## 文件系统层次结构标准 (FHS)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_12.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_14.png)

上一节我们介绍了Linux中“一切皆文件”的概念。本节中我们来看看这些文件是如何有组织地存放在系统中的。

Linux系统遵循一个名为**文件系统层次结构标准 (FHS)** 的规范。这意味着所有Linux发行版的主要目录结构都是相似的，你学会了红帽系统，也就掌握了其他发行版的基础。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_16.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_18.png)

以下是常见的目录分类及其用途：

*   **用户家目录**：普通用户的家目录位于 `/home/` 下，每个用户拥有一个同名子目录。例如，用户 `user1` 的家目录是 `/home/user1`。管理员`root`的家目录是 `/root`。
*   **配置文件目录**：`/etc/` 目录用于存放系统的配置文件，例如网络配置、主机名、DNS设置等。
*   **二进制可执行目录**：`/bin/`, `/sbin/`, `/usr/bin/`, `/usr/sbin/` 等目录存放着系统的命令和可执行程序。**切勿删除这些目录**，否则会导致许多命令无法使用。例如，`ls` 命令通常位于 `/usr/bin/ls`。
*   **设备文件目录**：`/dev/` 目录包含了系统所有的设备文件，例如硬盘 (`/dev/sda`)、网卡等。
*   **临时目录**：`/tmp/` 和 `/var/tmp/` 是临时文件目录。系统会定期清理其中的内容，**不要存放重要数据**。
*   **挂载点目录**：`/media/` 和 `/mnt/` 是系统提供的挂载点，用于挂载外部存储设备，如U盘、光盘等。
*   **信息目录**：`/proc/` 和 `/sys/` 是虚拟文件系统，它们提供内核和进程信息的接口。这些目录不占用硬盘空间，数据存放在内存中，修改立即生效，但重启后会丢失。
*   **可变数据目录**：`/var/` 用于存放经常变化的文件，如日志 (`/var/log`)、邮件、数据库文件等。
*   **启动目录**：`/boot/` 存放系统启动所需的文件，如内核、引导程序等。
*   **根目录**：`/` 是整个文件系统的起点，所有其他目录都挂载在其下。

## 路径引用：绝对路径与相对路径

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_22.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_24.png)

理解了目录结构后，我们需要知道如何定位和访问其中的文件。这涉及到两种路径引用方式。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_26.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_28.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_30.png)

### 绝对路径

绝对路径从根目录 `/` 开始，完整地描述到达目标文件或目录所经过的每一层目录。它的特点是**安全、明确**，与当前所在位置无关。

**公式**：`/目录1/目录2/.../目标文件`

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_32.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_34.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_36.png)

例如，访问文件 `/var/log/message` 的绝对路径就是 `/var/log/message`。

### 相对路径

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_38.png)

相对路径以**当前工作目录**为起点进行引用。使用前必须清楚自己当前的位置。

**命令**：`pwd` (Print Working Directory) 可以显示当前目录的绝对路径。

例如，如果当前目录是 `/var`，要进入 `log` 子目录，相对路径就是 `log`。如果写成 `/log`，则会被系统理解为从根目录开始的绝对路径，这通常是错误的。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_40.png)

相对路径中两个特殊的目录表示符：
*   `.` (一个点)：代表当前目录。
*   `..` (两个点)：代表上级目录（父目录）。

**技巧**：使用 `cd ..` 可以返回上级目录。使用 `cd -` 可以快速切换回上一次的工作目录。

## 目录切换与管理

`cd` (Change Directory) 是切换目录的核心命令。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_42.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_44.png)

以下是 `cd` 命令的常见用法：
*   `cd 目录名`：切换到指定目录。
*   `cd` 或 `cd ~`：切换到当前用户的家目录。
*   `cd ~用户名`：切换到指定用户的家目录（通常需要管理员权限）。
*   `cd -`：切换到上一次所在的目录。
*   `cd ..`：切换到上级目录。

**注意**：`source` 命令的缩写也是一个点 `.`，但用法是 `. 脚本文件名`（点后面有空格），用于使脚本修改立即生效，而不需要重启或重新登录。

## 文件与目录的基本操作

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_50.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_52.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_54.png)

本节中我们来看看如何创建、删除、移动和复制文件与目录。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_56.png)

### 文件操作

*   **创建空文件**：`touch 文件名`
    *   如果文件已存在，`touch` 会更新该文件的修改时间，**不会覆盖内容**。
*   **删除文件**：`rm 文件名`
    *   `-f` (force)：强制删除，不提示。
    *   `-i` (interactive)：交互式删除，删除前确认。
    *   系统通常为 `rm` 设置了别名 `rm -i`，以增加安全性。
*   **移动/重命名文件**：`mv 源文件 目标`
    *   如果“目标”是文件名，则为重命名。
    *   如果“目标”是目录路径，则为移动。
*   **复制文件**：`cp 源文件 目标`
    *   `-r` (recursive)：递归复制，用于复制目录。
    *   `-i` (interactive)：交互式覆盖提示。
    *   `-p` (preserve)：保留原文件的属性（如时间戳、权限）。
    *   可以在复制时直接重命名：`cp file1 /path/to/file2`

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_58.png)

### 目录操作

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_60.png)

*   **创建目录**：`mkdir 目录名`
    *   `-p` (parents)：自动创建路径中不存在的父目录。例如：`mkdir -p a/b/c/d`
*   **删除空目录**：`rmdir 目录名`
*   **删除目录（及内容）**：`rm -r 目录名`
    *   结合 `-f` 选项 (`rm -rf`) 可以强制递归删除，**非常危险，需谨慎使用**。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_62.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_64.png)

### 查看文件类型

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_66.png)

*   `file 文件名`：查看文件的类型（如文本、二进制、目录等）。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_68.png)

## 通配符

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_70.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_72.png)

通配符是一种用于匹配文件名的特殊字符，可以高效地批量操作文件。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_74.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_76.png)

以下是主要的通配符：
*   `*`：匹配零个或多个任意字符。例如 `ls *.txt` 列出所有 `.txt` 文件。
*   `?`：匹配任意一个字符。例如 `file?.log` 匹配 `file1.log`，但不匹配 `file10.log`。
*   `[abc]`：匹配方括号内列出的任意一个字符。例如 `[0-9]` 匹配任意一个数字。
*   `[!abc]` 或 `[^abc]`：匹配不在方括号内的任意一个字符。
*   `{string1,string2,...}`：扩展匹配。例如 `touch file{1,2,3}.txt` 会创建 `file1.txt`, `file2.txt`, `file3.txt`。

## Vim 文本编辑器

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_98.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_100.png)

Vim 是Linux系统中最强大、最常用的命令行文本编辑器之一。Vim 是 Vi 的增强版，两者操作基本兼容。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_102.png)

### Vim 的三种模式

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_104.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_106.png)

Vim 的核心在于其模式设计：
1.  **命令模式 (Command Mode)**：启动 Vim 后的默认模式。可以执行复制、粘贴、删除、搜索等命令，但不能直接输入文本。
2.  **插入模式 (Insert Mode)**：在此模式下可以像普通编辑器一样输入和编辑文本。
3.  **末行模式 (Ex Mode)**：在命令模式下按 `:` 进入。可以执行保存、退出、查找替换等操作。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_108.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_109.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_111.png)

**模式切换流程**：
*   启动 Vim (`vim 文件名`) -> **命令模式**
*   命令模式下按 `i`, `a`, `o` 等键 -> **插入模式**
*   插入模式下按 `Esc` 键 -> **命令模式**
*   命令模式下按 `:` 键 -> **末行模式**
*   末行模式下执行命令后自动回到 -> **命令模式**

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_113.png)

### 常用操作

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_114.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_116.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_118.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_120.png)

**插入模式进入方式**：
*   `i`：在光标前插入。
*   `a`：在光标后插入。
*   `o`：在当前行下方新建一行并插入。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_122.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_124.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_126.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_128.png)

**命令模式下的操作**：
*   `dd`：删除（剪切）当前行。
*   `yy`：复制当前行。
*   `p`：在光标下方粘贴。
*   `u`：撤销上一次操作。
*   `/关键字`：向下搜索关键字，按 `n` 查找下一个。
*   `数字+G`：跳转到指定行，例如 `5G` 跳转到第5行。`G` 跳转到最后一行，`gg` 跳转到第一行。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_130.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_132.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_134.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_136.png)

**末行模式下的操作**：
*   `:w`：保存文件。
*   `:q`：退出 Vim。
*   `:wq` 或 `:x`：保存并退出。
*   `:q!`：不保存强制退出。
*   `:set number` 或 `:set nu`：显示行号。
*   `:set nonumber` 或 `:set nonu`：取消显示行号。
*   `:%s/旧文本/新文本/g`：将全文中的“旧文本”替换为“新文本”。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_140.png)

## 输入输出重定向

Linux 命令执行时，会打开三个标准的“数据流”：
*   **标准输入 (stdin)**：文件描述符 `0`，默认为键盘。
*   **标准输出 (stdout)**：文件描述符 `1`，默认为屏幕（终端）。
*   **标准错误 (stderr)**：文件描述符 `2`，默认为屏幕（终端）。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_144.png)

重定向操作可以改变这些数据流的默认方向。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_146.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_148.png)

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_150.png)

**重定向符号**：
*   `命令 > 文件`：将命令的**标准输出**覆盖写入到文件。等同于 `1>`。
*   `命令 >> 文件`：将命令的**标准输出**追加到文件末尾。
*   `命令 2> 文件`：将命令的**标准错误**覆盖写入到文件。
*   `命令 2>> 文件`：将命令的**标准错误**追加到文件末尾。
*   `命令 &> 文件` 或 `命令 > 文件 2>&1`：将命令的**标准输出和标准错误**都覆盖写入到同一个文件。

![](img/d02eb1a3f8b779a012d51bd7e69bb19d_152.png)

**特殊设备文件**：
*   `/dev/null`：像一个“黑洞”，写入其中的任何数据都会被丢弃。常用于屏蔽不必要的输出。例如：`命令 2> /dev/null`。
*   `/dev/zero`：提供无限的空字符（`\0`）。常用于创建指定大小的空文件。

**警告**：`>` 操作会覆盖目标文件原有内容，使用时需确认。`>>` 追加操作更安全。
