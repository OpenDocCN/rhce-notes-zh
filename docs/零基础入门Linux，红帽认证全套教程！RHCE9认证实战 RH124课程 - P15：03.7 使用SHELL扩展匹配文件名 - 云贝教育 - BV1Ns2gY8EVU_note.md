# Linux入门与红帽认证：03.7：使用SHELL扩展匹配文件名 📂

在本节课中，我们将学习如何使用Bash Shell的各种扩展功能来匹配文件名和路径。这些功能包括模式匹配、波形符扩展、大括号扩展、变量扩展和命令替换等，它们能极大地提高我们在命令行中处理文件和目录的效率。

## 模式匹配（通配符） 🔍

上一节我们介绍了Shell的基本概念，本节中我们来看看如何使用模式匹配来筛选文件。模式匹配使用特定的元字符来代表一个或多个字符，从而实现灵活的文件名匹配。

![](img/6f12724b0af173810c84ed9a33bac808_1.png)

以下是Bash Shell中常用的模式匹配元字符：

*   `*`：匹配零个或多个任意字符组成的字符串。例如，`a*` 匹配所有以 `a` 开头的文件。
*   `?`：匹配任意单个字符。例如，`file?.txt` 匹配 `file1.txt`、`fileA.txt` 等。
*   `[abc]`：匹配方括号内列出的任意一个字符。例如，`[abc]*` 匹配以 `a`、`b` 或 `c` 开头的文件。
*   `[!abc]` 或 `[^abc]`：匹配不在方括号内列出的任意一个字符。例如，`[!a]*` 匹配不以 `a` 开头的文件。
*   `[[:alpha:]]`：匹配任意一个字母字符。
*   `[[:lower:]]`：匹配任意一个小写字母。
*   `[[:upper:]]`：匹配任意一个大写字母。
*   `[[:alnum:]]`：匹配任意一个字母或数字。
*   `[[:digit:]]`：匹配任意一个数字（0-9）。
*   `[[:space:]]`：匹配任意一个空白字符。

![](img/6f12724b0af173810c84ed9a33bac808_3.png)

**示例**：首先，我们创建一个包含多个文件的目录来演示。

![](img/6f12724b0af173810c84ed9a33bac808_5.png)

```bash
mkdir globdir && cd globdir
touch ALPHA BRAVO CHARLIE DELTA file1 file2 fileA fileB
```

![](img/6f12724b0af173810c84ed9a33bac808_7.png)

现在，我们可以使用模式匹配来查找文件：

![](img/6f12724b0af173810c84ed9a33bac808_9.png)

*   匹配所有以 `A` 开头的文件：`ls A*`
*   匹配所有包含字母 `A` 的文件：`ls *A*`
*   匹配文件名恰好为4个字符的文件：`ls ????`
*   匹配以 `A` 或 `B` 开头的文件：`ls [AB]*`
*   匹配不以 `A` 或 `B` 开头的文件：`ls [!AB]*`

![](img/6f12724b0af173810c84ed9a33bac808_11.png)

## 波形符扩展 🏠

![](img/6f12724b0af173810c84ed9a33bac808_13.png)

波形符 `~` 用于快速引用用户的家目录，这是一种非常便捷的路径扩展方式。

以下是波形符扩展的规则：

*   `~`：代表当前用户的家目录。
*   `~username`：代表指定用户 `username` 的家目录。

**示例**：

*   切换到当前用户家目录下的 `Documents` 文件夹：`cd ~/Documents`
*   打印 `root` 用户的家目录路径：`echo ~root`
*   打印当前用户的家目录路径：`echo ~`

![](img/6f12724b0af173810c84ed9a33bac808_15.png)

## 大括号扩展 {}

![](img/6f12724b0af173810c84ed9a33bac808_17.png)

大括号扩展 `{}` 用于生成任意字符串序列，可以简化需要创建多个有规律文件或目录的操作。

大括号扩展的规则如下：

*   逗号 `,` 用于分隔列表中的项目。
*   两个句点 `..` 用于生成一个连续的序列。

![](img/6f12724b0af173810c84ed9a33bac808_19.png)

**示例**：

*   创建多个日志文件：`touch {sunday,monday,tuesday}.log`
*   创建一系列有编号的目录：`mkdir archive{01..05}`
*   大括号可以嵌套使用，创建更复杂的组合：`touch song{A{1,2},B,C}.mp3`
*   快速复制文件并添加备份后缀：`cp sunday.log{,.bak}` 这等价于 `cp sunday.log sunday.log.bak`

![](img/6f12724b0af173810c84ed9a33bac808_21.png)

## 变量扩展 $

![](img/6f12724b0af173810c84ed9a33bac808_23.png)

![](img/6f12724b0af173810c84ed9a33bac808_24.png)

变量扩展允许我们在Shell中定义临时变量来存储值，并通过变量名来引用这些值，使命令更灵活。

![](img/6f12724b0af173810c84ed9a33bac808_26.png)

定义和引用变量的基本语法是：
```bash
变量名=值
echo $变量名
```

![](img/6f12724b0af173810c84ed9a33bac808_28.png)

**示例**：

*   定义一个变量并引用它：
    ```bash
    DATADIR=/tmp/data
    echo $DATADIR
    ```
*   为了防止变量名与后续字符混淆，建议用花括号 `{}` 将变量名括起来：
    ```bash
    echo ${DATADIR}_backup
    ```
*   **注意**：Shell变量名区分大小写，可以由字母、数字和下划线组成，但不能以数字开头。通常使用大写字母以便与命令区分。定义的变量仅在当前Shell会话中有效。

![](img/6f12724b0af173810c84ed9a33bac808_30.png)

## 命令替换 $()

命令替换允许我们将一个命令的输出作为另一个命令的参数或变量值，这是编写脚本和高效命令行操作的关键。

命令替换的语法是：
```bash
$(命令)
```
早期也使用反引号 `` `命令` ``，但推荐使用 `$()`，因为它更清晰且不易与单引号混淆。

![](img/6f12724b0af173810c84ed9a33bac808_32.png)

**示例**：

*   创建一个目录并立即进入该目录：
    ```bash
    cd $(mkdir -p backup)
    ```
*   创建一个以当前日期命名的备份文件：
    ```bash
    touch backup_file_$(date +%Y%m%d).log
    ```

![](img/6f12724b0af173810c84ed9a33bac808_34.png)

## 防止扩展与引号使用 🛡️

![](img/6f12724b0af173810c84ed9a33bac808_36.png)

有时我们不想让Shell解释特殊字符（如 `$`、`*`），而是希望将它们作为普通文本输出。这时就需要防止参数扩展。

![](img/6f12724b0af173810c84ed9a33bac808_38.png)

以下是防止扩展的方法：

![](img/6f12724b0af173810c84ed9a33bac808_40.png)

*   **反斜杠 `\`**：在特殊字符前加反斜杠，使其失去特殊含义。例如，`echo \$DATADIR` 会输出 `$DATADIR` 而不是变量的值。
*   **单引号 `'`**：单引号内的所有字符都会被原样输出，不会进行任何扩展。
*   **双引号 `"`**：双引号内的内容会进行变量扩展和命令替换，但会保留字符串的整体性（不拆分空格）。

![](img/6f12724b0af173810c84ed9a33bac808_42.png)

![](img/6f12724b0af173810c84ed9a33bac808_44.png)

**示例**：

*   使用单引号防止变量扩展：`echo '$DATADIR'` 输出 `$DATADIR`
*   使用双引号允许变量扩展：`echo "$DATADIR"` 输出变量的值，如 `/tmp/data`
*   当字符串中包含引号本身时，使用反斜杠进行转义：`echo "It's a \"test\"."`
