## 第六章：磁盘分区管理 💽

![](img/94e31a9f56af51c50ffa429653d4c5c4_8.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_10.png)

上一节我们介绍了课程的整体框架，本节中我们来看看存储管理的基础——磁盘分区。分区是磁盘管理的基本单元，它将磁盘空间根据需要进行分块或分段管理，以便更自由、方便地使用。

![](img/94e31a9f56af51c50ffa429653d4c5c4_12.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_13.png)

我们将重点解释两种主流的磁盘分区结构：

*   **MBR**：即主引导记录格式。这是一种历史悠久的磁盘分区模式，起源于20世纪70年代的DOS系统规范。其特点是支持最大2TB的磁盘和最多4个主分区。代码表示为 `fdisk` 工具常用于操作此格式。
*   **GPT**：即GUID分区表格式。这是一种现代的分区表格式，被Windows 10等新系统默认采用。它支持几乎无限数量的分区和远超2TB的磁盘容量。代码表示为 `gdisk` 或 `parted` 工具常用于操作此格式。

目前我们正处在新旧格式交替的阶段，因此两种都需要掌握。

## 第七章：逻辑卷管理器 📈

![](img/94e31a9f56af51c50ffa429653d4c5c4_15.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_17.png)

分区提供了基础的存储划分，但有时我们需要将多个分区组合成更大、更灵活的存储单元。这就是逻辑卷管理器的作用。

逻辑卷是一个用户无需关心底层物理分区大小的存储单元。它的核心思想是将多个物理分区（称为物理卷PV）聚合成一个存储池（称为卷组VG），再从这个池中划分出逻辑卷（LV）供文件系统使用。

![](img/94e31a9f56af51c50ffa429653d4c5c4_19.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_21.png)

逻辑卷的最大优势在于其灵活性，支持**在线扩展**。这意味着你可以在不卸载文件系统、不中断服务的情况下，动态增加逻辑卷的容量。其基本管理命令流程如下：
```bash
pvcreate /dev/sdb1          # 创建物理卷
vgcreate myvg /dev/sdb1     # 创建卷组
lvcreate -L 10G -n mylv myvg # 创建逻辑卷
mkfs.xfs /dev/myvg/mylv     # 创建文件系统
```

## 第八章：高级存储功能 🚀

![](img/94e31a9f56af51c50ffa429653d4c5c4_25.png)

既然逻辑卷可以手动扩展，那么能否实现自动扩展呢？答案是肯定的，这就引出了更高级的存储抽象。

![](img/94e31a9f56af51c50ffa429653d4c5c4_27.png)

以下是本章介绍的两个RHEL 8特有的高级存储功能：

*   **Stratis**：这是一个逻辑卷管理器的更高级抽象。它为用户提供了一个可以“自动精简配置”和“自动扩展”的文件系统。用户只需关注使用，当存储空间不足时，Stratis会自动从后台的存储池中分配空间，无需手动干预。公式上，可以理解为 `Stratis = 自动化的逻辑卷管理 + 文件系统`。
*   **VDO**：即虚拟数据优化器。这是一种具有**去重**和**压缩**功能的存储层。将数据存入VDO卷时，它会自动删除重复的数据块并压缩数据，从而显著节省存储空间。例如，存储10份相同的文件，在磁盘上实际只占用1份的空间。需要注意的是，去重功能在提高效率的同时，也带来了单点故障的风险，一份数据损坏可能导致所有依赖它的文件受损。因此，常建议将VDO构建在具有容错能力的RAID之上。

## 第九章：网络文件共享 🌐

![](img/94e31a9f56af51c50ffa429653d4c5c4_29.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_31.png)

如何将本机文件共享给网络上的其他用户？本章将讲解Linux内核原生的网络文件系统。

**NFS** 是一种由Linux内核直接支持的、高效的文件共享协议，特别适合在Linux系统之间共享文件。只要内核启动，NFS服务就已就绪，无需额外安装大型服务软件。

## 第十一章：管理网络安全 🛡️

![](img/94e31a9f56af51c50ffa429653d4c5c4_52.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_54.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_56.png)

网络安全是系统管理的重要一环。本章将综合讲解两个核心的安全工具。

![](img/94e31a9f56af51c50ffa429653d4c5c4_58.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_60.png)

我们将深入探讨：
*   **SELinux**：对第五章内容的扩展，重点讲解如何配置SELinux策略以控制网络服务（如HTTPD、FTP）的访问权限。
*   **Firewalld**：学习使用动态防火墙管理器来过滤和控制网络流量，例如开放或关闭特定服务的端口。常用命令如 `firewall-cmd --add-service=http --permanent`。

![](img/94e31a9f56af51c50ffa429653d4c5c4_64.png)

## 第十三章：容器化技术补充 🐳

![](img/94e31a9f56af51c50ffa429653d4c5c4_79.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_80.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_82.png)

自2021年起，红帽在RH134中新增了关于容器化技术的内容，这代表了未来的发展趋势。

![](img/94e31a9f56af51c50ffa429653d4c5c4_84.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_85.png)

![](img/94e31a9f56af51c50ffa429653d4c5c4_87.png)

**容器化技术**（如Podman/Docker）是一种轻量级的虚拟化方案。它在操作系统内核之上进行资源隔离，使得每个容器像是一个独立的服务器，但消耗的资源远少于传统虚拟机。这使得单台主机可以运行成百上千个容器。容器技术是云原生和PaaS平台的核心，深刻影响着软件的开发、交付和部署流程。我们将补充讲解如何管理和配置容器。
