在本节课中，我们将要学习Shell中的扩展字符，也称为通配符。这些特殊字符能帮助我们快速、高效地匹配和管理文件，是命令行操作中不可或缺的工具。我们还将学习如何通过变量和命令替换来动态处理信息，以及如何屏蔽这些特殊字符的字面含义。

上一节我们介绍了Shell扩展的基本概念，本节中我们来看看具体的通配符及其用法。通配符是Shell用来进行文件名模式匹配的特殊字符。

![](img/1505a1be42d031a66a0519655f11f748_32.png)

![](img/1505a1be42d031a66a0519655f11f748_34.png)

![](img/1505a1be42d031a66a0519655f11f748_36.png)

![](img/1505a1be42d031a66a0519655f11f748_38.png)

以下是常见的Shell通配符列表：

![](img/1505a1be42d031a66a0519655f11f748_40.png)

*   `*`：匹配零个或多个任意字符。
    *   **示例**：`ls *.txt` 会列出所有以 `.txt` 结尾的文件。
*   `?`：匹配任意单个字符。
    *   **示例**：`ls file?.txt` 会匹配 `file1.txt`、`fileA.txt`，但不会匹配 `file10.txt`。
*   `[abc...]`：匹配方括号中列出的任意一个字符。
    *   **示例**：`ls file[123].txt` 会匹配 `file1.txt`、`file2.txt`、`file3.txt`。
*   `[!abc...]` 或 `[^abc...]`：匹配不在方括号中列出的任意一个字符。
    *   **示例**：`ls file[!12].txt` 会匹配 `file3.txt`、`fileA.txt`，但不会匹配 `file1.txt` 或 `file2.txt`。
*   `[a-z]`：匹配指定范围内的任意一个字符。
    *   **示例**：`ls file[a-c].txt` 会匹配 `filea.txt`、`fileb.txt`、`filec.txt`。
*   `{string1,string2,...}`：大括号扩展，生成以逗号分隔的字符串列表。
    *   **示例**：`touch file{1,2,3}.txt` 会创建 `file1.txt`、`file2.txt`、`file3.txt` 三个文件。
*   `{start..end}`：大括号范围扩展，生成从 `start` 到 `end` 的序列。
    *   **示例**：`touch file{1..3}.txt` 效果同上。

![](img/1505a1be42d031a66a0519655f11f748_42.png)

![](img/1505a1be42d031a66a0519655f11f748_43.png)

![](img/1505a1be42d031a66a0519655f11f748_45.png)

**注意**：大括号扩展 `{}` 和方括号通配符 `[]` 有本质区别。`{}` 是展开操作，无论文件是否存在都会展开；而 `[]` 是匹配操作，只匹配当前目录下存在的文件。

理解了文件名匹配后，我们来看看如何在Shell中动态地使用数据。变量替换允许我们存储和引用值。

![](img/1505a1be42d031a66a0519655f11f748_65.png)

![](img/1505a1be42d031a66a0519655f11f748_67.png)

![](img/1505a1be42d031a66a0519655f11f748_69.png)

![](img/1505a1be42d031a66a0519655f11f748_70.png)

在Shell中，使用 `变量名=值` 的格式定义变量，使用 `$变量名` 或 `${变量名}` 来引用变量的值。花括号 `${}` 常用于明确变量名的边界。

![](img/1505a1be42d031a66a0519655f11f748_72.png)

**示例**：
```bash
username=laoma
echo $username        # 输出：laoma
echo ${username}_file # 输出：laoma_file
```

## 命令替换

![](img/1505a1be42d031a66a0519655f11f748_84.png)

![](img/1505a1be42d031a66a0519655f11f748_85.png)

有时我们需要获取一个命令的执行结果，并将其作为另一个命令的输入或参数，这时就需要用到命令替换。

![](img/1505a1be42d031a66a0519655f11f748_87.png)

命令替换的语法是 `$(command)` 或反引号 `` `command` ``。推荐使用 `$(command)`，因为它更清晰且支持嵌套。

![](img/1505a1be42d031a66a0519655f11f748_89.png)

**示例**：
```bash
# 将 date 命令的输出作为字符串的一部分
echo “Today is $(date +%A).”
# 根据当前日期创建日志文件
touch app-$(date +%Y%m%d).log
```

## 引用机制：屏蔽特殊含义

![](img/1505a1be42d031a66a0519655f11f748_95.png)

![](img/1505a1be42d031a66a0519655f11f748_97.png)

当我们希望通配符或特殊符号（如 `$`）保持其字面意义，而不是被Shell解释时，就需要使用引用机制。

![](img/1505a1be42d031a66a0519655f11f748_99.png)

![](img/1505a1be42d031a66a0519655f11f748_101.png)

![](img/1505a1be42d031a66a0519655f11f748_103.png)

以下是三种主要的引用方式：

![](img/1505a1be42d031a66a0519655f11f748_105.png)

![](img/1505a1be42d031a66a0519655f11f748_107.png)

![](img/1505a1be42d031a66a0519655f11f748_109.png)

1.  **反斜杠 `\`**：转义紧随其后的单个字符。
    *   **示例**：`echo \$username` 会输出 `$username`，而不是变量的值。
2.  **双引号 `"`**：弱引用。会解释变量替换 `$` 和命令替换 `$()`，但会抑制通配符（`*`, `?`, `[]`等）和大括号扩展 `{}` 的解释。
    *   **示例**：`echo “$username”` 输出变量值；`echo “*.txt”` 输出字符串 `*.txt`。
3.  **单引号 `'`**：强引用。抑制所有Shell扩展，引号内的所有字符都保持字面意义。
    *   **示例**：`echo ‘$username $(date)’` 会原样输出字符串 `$username $(date)`。
