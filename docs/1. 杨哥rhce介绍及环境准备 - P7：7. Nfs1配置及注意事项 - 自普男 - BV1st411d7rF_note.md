## 环境准备与软件安装

首先，我们需要在服务器端（s0）准备环境并安装必要的软件包。

上一节我们介绍了课程目标，本节中我们来看看如何准备基础环境。

**第一步是安装 NFS 软件包。** 服务器和客户端都需要安装相应的软件包。在服务器上执行以下命令：

```bash
yum install -y nfs-utils
```

![](img/48624234c7593a7cc842a3953898c890_6.png)

安装完成后，需要按要求创建共享目录。以下是需要创建的目录结构：

*   `/nfs1`：第一个共享目录。
*   `/nfs2`：第二个共享目录。
*   `/nfs2/private`：`/nfs2` 下的子目录，其所有者需设置为 `ldapuser5`。

使用以下命令创建目录并设置权限：

```bash
mkdir /nfs1 /nfs2 /nfs2/private
chown ldapuser5:ldapuser5 /nfs2/private
```

![](img/48624234c7593a7cc842a3953898c890_8.png)

此时，`ldapuser5` 用户已拥有对 `/nfs2/private` 目录的写入权限。

## 配置第一个 NFS 共享（/nfs1）

![](img/48624234c7593a7cc842a3953898c890_10.png)

![](img/48624234c7593a7cc842a3953898c890_11.png)

环境准备好后，我们开始配置第一个 NFS 共享。

上一节我们准备好了目录，本节中我们来看看如何配置 `/nfs1` 的共享。

**第二步是编辑 NFS 的配置文件 `/etc/exports`。** 该文件定义了共享目录、允许访问的客户端及访问权限。

![](img/48624234c7593a7cc842a3953898c890_13.png)

使用编辑器打开该文件：

![](img/48624234c7593a7cc842a3953898c890_14.png)

```bash
vim /etc/exports
```

![](img/48624234c7593a7cc842a3953898c890_15.png)

在文件中添加以下内容，共享 `/nfs1` 目录：

![](img/48624234c7593a7cc842a3953898c890_16.png)

![](img/48624234c7593a7cc842a3953898c890_17.png)

```
/nfs1 172.25.0.0/24(ro,sync)
```

![](img/48624234c7593a7cc842a3953898c890_18.png)

**配置参数解析：**
*   **`/nfs1`**：要共享的目录路径。
*   **`172.25.0.0/24`**：允许访问的网段，即 `example.com` 域中的主机。
*   **`ro`**：以只读（Read-Only）方式共享。
*   **`sync`**：采用同步写入方式，保证数据一致性。
*   **重要**：客户端网段与后面的权限选项 `(ro,sync)` 之间**不能有空格**。

## 启动 NFS 服务并配置防火墙

![](img/48624234c7593a7cc842a3953898c890_20.png)

配置文件编辑完成后，需要启动服务并确保其能通过防火墙。

![](img/48624234c7593a7cc842a3953898c890_22.png)

上一节我们配置了共享规则，本节中我们来看看如何让服务生效。

**第三步是启动 NFS 服务并设置为开机自启。** 执行以下命令：

```bash
systemctl restart nfs-server
systemctl enable nfs-server
```

![](img/48624234c7593a7cc842a3953898c890_24.png)

**第四步是配置防火墙，允许 NFS 服务。** 可以使用命令行方式添加防火墙规则：

```bash
firewall-cmd --permanent --add-service=nfs
firewall-cmd --reload
```

至此，第一个 NFS 共享 `/nfs1` 在服务器端的配置就全部完成了。

## 在客户端挂载 NFS 共享

![](img/48624234c7593a7cc842a3953898c890_26.png)

服务器配置好后，我们切换到客户端（desktop）进行挂载操作。

上一节我们完成了服务器端的配置，本节中我们来看看客户端如何挂载共享。

**在客户端，首先也需要安装 NFS 工具包：**

![](img/48624234c7593a7cc842a3953898c890_28.png)

![](img/48624234c7593a7cc842a3953898c890_29.png)

```bash
yum install -y nfs-utils
```

**接着，创建挂载点目录。** 题目要求挂载到 `/mnt/nfs1`：

```bash
mkdir /mnt/nfs1
```

**然后，编辑 `/etc/fstab` 文件实现永久挂载。** 添加如下一行：

![](img/48624234c7593a7cc842a3953898c890_31.png)

```
server0:/nfs1 /mnt/nfs1 nfs defaults 0 0
```

**参数解析：**
*   **`server0:/nfs1`**：NFS 服务器地址及共享路径。
*   **`/mnt/nfs1`**：本地挂载点。
*   **`nfs`**：文件系统类型。
*   **`defaults`**：使用默认挂载选项。
*   **`0 0`**：表示不备份且不进行磁盘检查。

![](img/48624234c7593a7cc842a3953898c890_33.png)

**最后，执行挂载命令使配置生效：**

```bash
mount -a
```

可以使用 `df -hT` 命令查看挂载是否成功。

## 验证与测试

配置完成后，进行简单的验证以确保功能正常。

![](img/48624234c7593a7cc842a3953898c890_35.png)

我们可以进行一个简单的读写测试。由于服务器端配置的是 `ro`（只读）和 `root_squash`（压制 root 权限），在客户端无法在挂载目录中创建文件。

![](img/48624234c7593a7cc842a3953898c890_36.png)

可以在服务器的 `/nfs1` 目录下创建一个测试文件：

![](img/48624234c7593a7cc842a3953898c890_37.png)

![](img/48624234c7593a7cc842a3953898c890_38.png)

```bash
touch /nfs1/testfile
```

![](img/48624234c7593a7cc842a3953898c890_40.png)

然后在客户端查看 `/mnt/nfs1` 目录，应该能看到此文件，这证明共享和挂载成功。

**访问控制测试（选做）：** 为了深入理解，可以修改 `/etc/exports` 文件，将允许的网段改为一个不存在的网段（例如 `10.0.0.0/24`），然后重启 `nfs-server` 服务并在客户端重新挂载。此时挂载会失败，这验证了 NFS 的访问控制列表（ACL）是有效的。

**注意：** 在正式考试中，只需按要求完成配置即可，无需进行额外的复杂测试。

## 总结

![](img/48624234c7593a7cc842a3953898c890_42.png)

本节课中我们一起学习了 NFS 网络文件系统的基本配置。
1.  我们首先在服务器端安装了软件包，并创建了所需的共享目录结构。
2.  接着，我们通过编辑 `/etc/exports` 文件，配置了第一个只读共享 `/nfs1`，并指定了允许访问的客户端网段。
3.  然后，我们启动了 NFS 服务，配置了防火墙规则以允许 NFS 流量。
4.  最后，我们在客户端安装了工具包，通过编辑 `/etc/fstab` 文件实现了 NFS 共享的永久挂载，并完成了挂载验证。

![](img/48624234c7593a7cc842a3953898c890_44.png)

通过以上步骤，我们掌握了 NFS 服务器和客户端配置的核心流程。下一道关于 `/nfs2` 共享（要求 Kerberos 加密）的题目将在此基础上进行扩展。
