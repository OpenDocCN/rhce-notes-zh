## 概述：什么是 if 判断

`if` 判断与我们之前学习的条件测试（如 `[ ]` 或 `test` 命令）功能相似，都是根据条件是否成立来决定执行哪些操作。当条件满足时，执行一组命令；当条件不满足时，则执行另一组命令（或什么都不做）。`if` 语句比简单的条件测试功能更强大，结构更清晰，适合实现复杂的逻辑控制。

## if 单分支判断：只能判断“对”的情况

![](img/619adfe9f9812d47f76f8a998ddc8196_18.png)

![](img/619adfe9f9812d47f76f8a998ddc8196_20.png)

![](img/619adfe9f9812d47f76f8a998ddc8196_22.png)

上一节我们介绍了 `if` 判断的基本概念，本节中我们来看看最简单的形式——单分支判断。单分支判断的特点是：**只能判断条件成立的情况**。如果条件成立，则执行指定的命令；如果条件不成立，则脚本会直接跳过，不执行任何操作。

它的语法结构有两种，效果完全相同：

![](img/619adfe9f9812d47f76f8a998ddc8196_24.png)

**语法一（常用）：**
```bash
if [ 条件 ]; then
    要执行的命令
fi
```

![](img/619adfe9f9812d47f76f8a998ddc8196_26.png)

**语法二：**
```bash
if [ 条件 ]
then
    要执行的命令
fi
```

以下是单分支判断的一个简单示例，用于检查文件是否存在：

![](img/619adfe9f9812d47f76f8a998ddc8196_28.png)

```bash
#!/bin/bash
# 判断 /etc/passwd 文件是否存在
if [ -f /etc/passwd ]; then
    echo “文件存在”
fi
```

![](img/619adfe9f9812d47f76f8a998ddc8196_30.png)

![](img/619adfe9f9812d47f76f8a998ddc8196_32.png)

在这个脚本中，`-f /etc/passwd` 是判断条件。如果文件 `/etc/passwd` 存在，则条件成立，执行 `echo “文件存在”` 命令。如果文件不存在，则条件不成立，`echo` 命令不会被执行，脚本直接结束。

![](img/619adfe9f9812d47f76f8a998ddc8196_34.png)

**关键点：**
*   `if` 语句以 `fi` 结束。
*   条件必须放在方括号 `[ ]` 内，并且方括号内**前后必须有空格**。
*   `then` 可以写在条件之后（用分号 `;` 隔开），也可以另起一行。

## if 双分支判断：既能判断“对”也能判断“错”

单分支判断只能处理条件成立的情况，这在实际应用中往往不够。因此，我们需要双分支判断。双分支判断在条件成立时执行一组命令，在条件不成立时执行另一组命令。

![](img/619adfe9f9812d47f76f8a998ddc8196_48.png)

它的语法结构如下：

![](img/619adfe9f9812d47f76f8a998ddc8196_50.png)

```bash
if [ 条件 ]; then
    条件成立时执行的命令
else
    条件不成立时执行的命令
fi
```

让我们通过一个安装软件包的脚本来理解双分支判断：

```bash
#!/bin/bash
# 检查 vsftpd 软件包是否已安装
rpm -q vsftpd &> /dev/null

![](img/619adfe9f9812d47f76f8a998ddc8196_52.png)

if [ $? -eq 0 ]; then
    echo “软件包已存在，启动服务...”
    systemctl start vsftpd
    systemctl enable vsftpd &> /dev/null
else
    echo “软件包不存在，正在安装...”
    yum install -y vsftpd &> /dev/null
    systemctl start vsftpd
    systemctl enable vsftpd &> /dev/null
    echo “服务已启动”
fi
```

![](img/619adfe9f9812d47f76f8a998ddc8196_54.png)

**脚本解析：**
1.  `rpm -q vsftpd &> /dev/null`：查询 `vsftpd` 包是否安装，并将所有输出（包括错误信息）重定向到 `/dev/null`（即丢弃）。
2.  `$?` 是一个特殊变量，代表**上一条命令的退出状态码**。`0` 通常表示成功（命令正常执行），非 `0` 表示失败。
3.  `[ $? -eq 0 ]`：判断上一条命令 (`rpm -q`) 的退出状态码是否等于 `0`。如果等于 `0`，说明软件包已安装。
4.  如果已安装 (`then`)，则启动并设置服务开机自启。
5.  如果未安装 (`else`)，则先安装软件包，再启动并设置服务开机自启。

这个例子展示了如何利用 `if` 双分支判断实现智能化的运维操作。

## if 多分支判断：处理多个条件

![](img/619adfe9f9812d47f76f8a998ddc8196_64.png)

当我们需要根据多种不同情况执行不同操作时，就需要用到多分支判断。它允许我们连续检查多个条件，直到某个条件成立，并执行对应的命令块。

它的语法结构如下：

```bash
if [ 条件1 ]; then
    条件1成立时执行的命令
elif [ 条件2 ]; then
    条件2成立时执行的命令
elif [ 条件3 ]; then
    条件3成立时执行的命令
else
    以上所有条件都不成立时执行的命令
fi
```

![](img/619adfe9f9812d47f76f8a998ddc8196_66.png)

以下是一个根据成绩评级的多分支判断脚本示例：

![](img/619adfe9f9812d47f76f8a998ddc8196_68.png)

![](img/619adfe9f9812d47f76f8a998ddc8196_70.png)

```bash
#!/bin/bash
read -p “请输入您的成绩进行查询：” score

if [ $score -ge 90 ]; then
    echo “成绩非常优秀！奖励哇塞女孩一枚。”
elif [ $score -ge 80 ]; then
    echo “成绩比较优秀！离奖励仅差一步。”
elif [ $score -ge 70 ]; then
    echo “成绩一般，还需努力。”
elif [ $score -ge 60 ]; then
    echo “成绩勉强及格。”
else
    echo “成绩不及格，请继续努力。”
fi
```

![](img/619adfe9f9812d47f76f8a998ddc8196_72.png)

![](img/619adfe9f9812d47f76f8a998ddc8196_74.png)

**脚本解析：**
1.  `read -p` 提示用户输入成绩，并存入变量 `score`。
2.  脚本会**从上到下依次判断**每个条件。
3.  例如，输入 `85` 分：
    *   首先判断 `[ 85 -ge 90 ]`，不成立。
    *   然后判断 `[ 85 -ge 80 ]`，成立！于是执行 `echo “成绩比较优秀！离奖励仅差一步。”`。
    *   **一旦某个 `elif` 条件成立，执行完对应的命令后，整个 `if` 语句就会结束，后面的条件不会再被判断。**
4.  如果所有 `if` 和 `elif` 的条件都不成立，则执行 `else` 部分的命令。

多分支判断非常适合用于实现菜单选择、分级策略等场景。

## 实践案例：编写一个猜数字游戏 🎮

为了综合运用变量、运算和 `if` 判断，我们来编写一个简单的猜数字游戏脚本。

![](img/619adfe9f9812d47f76f8a998ddc8196_76.png)

```bash
#!/bin/bash
# 生成一个 0-9 的随机数
correct=$(( RANDOM % 10 ))

read -p “猜一猜 0-9 之间的数字：” guess

![](img/619adfe9f9812d47f76f8a998ddc8196_78.png)

![](img/619adfe9f9812d47f76f8a998ddc8196_80.png)

if [ $guess -eq $correct ]; then
    echo “恭喜你，猜对了！奖励哇塞女孩一天的使用权。”
else
    echo “很遗憾，猜错了。哇塞女孩正在等你，请继续努力！”
    echo “本次的正确数字是：$correct”
fi
```

**脚本解析：**
1.  `RANDOM` 是 Bash 的内置变量，每次引用都会产生一个 `0-32767` 的随机数。
2.  `$(( RANDOM % 10 ))`：利用算术运算 `%`（取余）将随机数范围限制在 `0-9` 之间，并将结果赋值给变量 `correct`。
3.  使用 `read` 获取用户猜测的数字。
4.  使用 `if` 双分支判断用户输入是否等于随机生成的数字，并给出相应提示。

这个例子生动地展示了如何将所学知识组合起来，完成一个有趣的小程序。

## 总结与核心要点

![](img/619adfe9f9812d47f76f8a998ddc8196_82.png)

![](img/619adfe9f9812d47f76f8a998ddc8196_84.png)

本节课中我们一起学习了 Shell 脚本中 `if` 条件判断的三种形式：
1.  **单分支 (`if-then-fi`)**: 仅处理条件成立的情况。
2.  **双分支 (`if-then-else-fi`)**: 处理条件成立和不成立两种情况。
3.  **多分支 (`if-then-elif-else-fi`)**: 处理多个连续的条件。

**核心概念与格式牢记：**
*   **条件必须放在 `[ ]` 中**，并且括号内各部分**必须留有空格**，例如 `[ $a -eq $b ]`。
*   使用 `$?` 判断**上一条命令的执行结果**（`0` 为成功）。
*   使用 `$(( 算术表达式 ))` 进行**数值计算**。
*   所有 `if` 语句都必须以 `fi` 结尾。
*   对于变量比较，**数字比较**使用 `-eq`（等于）、`-ne`（不等于）、`-gt`（大于）、`-ge`（大于等于）、`-lt`（小于）、`-le`（小于等于）。
*   对于**字符串比较**，使用 `=`（等于）和 `!=`（不等于）。

![](img/619adfe9f9812d47f76f8a998ddc8196_86.png)

在实际工作中，我们很少从零开始编写复杂的脚本，更多的是阅读、理解和修改现有的脚本以满足需求。掌握 `if` 判断的结构和逻辑，是理解和修改他人脚本的基石。希望大家通过练习，能够熟练运用 `if` 判断来控制脚本的执行流程。
