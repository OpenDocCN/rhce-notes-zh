## 特殊变量与位置参数

![](img/a2df6e709ec3dbe7c74e925661b753d0_2.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_3.png)

上一节我们介绍了任务控制，本节中我们来看看Shell中的特殊变量，特别是预定义的位置参数。

![](img/a2df6e709ec3dbe7c74e925661b753d0_5.png)

位置参数是Shell预定义的变量，用于接收传递给脚本或函数的参数。以下是几个核心的位置参数：

*   **`$0`**：代表脚本自身的名称。
*   **`$1`, `$2`, ..., `$9`**：分别代表传递给脚本的第一个、第二个到第九个参数。
*   **`${10}`, `${11}`, ...**：当参数位置超过9时，必须使用大括号 `{}` 将数字括起来，例如 `${10}`。这是为了消除歧义，否则 `$10` 会被解释为 `$1` 后面连接一个字符 `0`。

![](img/a2df6e709ec3dbe7c74e925661b753d0_7.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_9.png)

### 脚本名称 `$0`

![](img/a2df6e709ec3dbe7c74e925661b753d0_11.png)

`$0` 变量存储了脚本的完整路径。我们可以使用 `basename` 命令来获取不带路径的纯脚本文件名。

```bash
#!/bin/bash
# 输出脚本的完整路径
echo "The script name is: $0"

![](img/a2df6e709ec3dbe7c74e925661b753d0_13.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_15.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_16.png)

# 使用 basename 命令获取纯文件名
script_name=$(basename $0)
echo "The script name (without path) is: $script_name"
```

![](img/a2df6e709ec3dbe7c74e925661b753d0_18.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_19.png)

### 处理带空格的参数

如果传递给脚本的参数包含空格，Shell会默认将空格视为参数的分隔符。为了将带空格的字符串作为一个整体参数传递，需要使用引号将其括起来。

![](img/a2df6e709ec3dbe7c74e925661b753d0_22.png)

```bash
#!/bin/bash
# 脚本内容：echo "Hello $1, welcome to the class."

![](img/a2df6e709ec3dbe7c74e925661b753d0_24.png)

# 错误调用方式（两个参数）
./script.sh John Doe
# 输出：Hello John, welcome to the class. (丢失了"Doe")

![](img/a2df6e709ec3dbe7c74e925661b753d0_26.png)

# 正确调用方式（一个参数）
./script.sh "John Doe"
# 输出：Hello John Doe, welcome to the class.
```

![](img/a2df6e709ec3dbe7c74e925661b753d0_28.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_29.png)

### `$*` 与 `$@` 的区别

![](img/a2df6e709ec3dbe7c74e925661b753d0_31.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_32.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_33.png)

`$*` 和 `$@` 都表示所有的位置参数，但它们在循环中的行为有细微差别。

*   **`$*`**：将所有参数视为一个**单一的单词**。
*   **`$@`**：将每个参数视为**独立的单词**。

以下示例展示了它们的区别：

![](img/a2df6e709ec3dbe7c74e925661b753d0_35.png)

```bash
#!/bin/bash
# 脚本1: 使用 $*
for arg in "$*"
do
    echo "Arg: $arg"
done

![](img/a2df6e709ec3dbe7c74e925661b753d0_37.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_38.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_40.png)

# 脚本2: 使用 $@
for arg in "$@"
do
    echo "Arg: $arg"
done
```

![](img/a2df6e709ec3dbe7c74e925661b753d0_42.png)

如果执行脚本 `./test.sh 1 2 3 4`，输出结果如下：
*   使用 `$*` 的输出：`Arg: 1 2 3 4` （一行输出，所有参数合并为一个字符串）
*   使用 `$@` 的输出：
    ```
    Arg: 1
    Arg: 2
    Arg: 3
    Arg: 4
    ```
    （四行输出，每个参数独立）

![](img/a2df6e709ec3dbe7c74e925661b753d0_44.png)

## 花括号扩展的字符串操作

![](img/a2df6e709ec3dbe7c74e925661b753d0_56.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_58.png)

接下来，我们学习如何使用花括号 `{}` 对变量进行字符串的截取和替换操作。这是一种强大而简洁的文本处理技巧。

假设我们定义了一个变量：
```bash
file="/tmp/test/file.tar.gz"
```

![](img/a2df6e709ec3dbe7c74e925661b753d0_60.png)

以下是常用的操作符和示例：

*   **`#`**：从字符串**开头**删除匹配的字符。单个 `#` 表示最小匹配，双个 `##` 表示最大匹配。
    ```bash
    echo ${file#*/}    # 输出：tmp/test/file.tar.gz (删除第一个`/`及左边)
    echo ${file##*/}   # 输出：file.tar.gz (删除最后一个`/`及左边)
    echo ${file#*.}    # 输出：tar.gz (删除第一个`.`及左边)
    echo ${file##*.}   # 输出：gz (删除最后一个`.`及左边)
    ```

![](img/a2df6e709ec3dbe7c74e925661b753d0_62.png)

*   **`%`**：从字符串**末尾**删除匹配的字符。单个 `%` 表示最小匹配，双个 `%%` 表示最大匹配。
    ```bash
    echo ${file%/*}    # 输出：/tmp/test (删除最后一个`/`及右边)
    echo ${file%%/*}   # 输出：(空) (删除第一个`/`及右边)
    echo ${file%.*}    # 输出：/tmp/test/file.tar (删除最后一个`.`及右边)
    echo ${file%%.*}   # 输出：/tmp/test/file (删除第一个`.`及右边)
    ```

![](img/a2df6e709ec3dbe7c74e925661b753d0_64.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_66.png)

**记忆口诀**：`#` 在键盘上位于 `$` 的左边，所以是删除**左边**；`%` 在键盘上位于 `$` 的右边，所以是删除**右边**。单符号是最小匹配，双符号是最大匹配。

![](img/a2df6e709ec3dbe7c74e925661b753d0_68.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_70.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_71.png)

### 子串提取和变量替换

除了删除，花括号还可以用于提取子串和进行字符替换。

![](img/a2df6e709ec3dbe7c74e925661b753d0_73.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_74.png)

*   **提取子串**：`${变量:起始位置:长度}`
    ```bash
    echo ${file:0:5}   # 输出：/tmp/ (从第0个字符开始，取5个字符)
    echo ${file:5:5}   # 输出：test/ (从第5个字符开始，取5个字符)
    ```

*   **变量替换**：`${变量/模式/替换字符串}`
    ```bash
    echo ${file/s/S}   # 输出：/tmp/teSt/file.tar.gz (替换第一个小写s为大写S)
    echo ${file//s/S}  # 输出：/tmp/teSt/file.tar.gz (替换所有小写s为大写S)
    ```

## 命令退出状态码

在Linux中，每个命令执行后都会返回一个**退出状态码**，也称为返回码。这个状态码用于判断命令是否成功执行。

![](img/a2df6e709ec3dbe7c74e925661b753d0_78.png)

*   **`0`**：表示命令**成功**执行。
*   **非 `0`**：表示命令执行**失败**，不同的非零值代表不同的错误类型。

![](img/a2df6e709ec3dbe7c74e925661b753d0_80.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_82.png)

状态码存储在特殊变量 `$?` 中，它总是保存**最近一条**执行命令的退出状态。

![](img/a2df6e709ec3dbe7c74e925661b753d0_84.png)

### 常见的退出状态码

![](img/a2df6e709ec3dbe7c74e925661b753d0_86.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_88.png)

以下是部分标准的退出状态码及其含义：

![](img/a2df6e709ec3dbe7c74e925661b753d0_89.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_91.png)

*   **0**：成功
*   **1**：一般性未知错误（例如，无效参数）
*   **2**：不适合的Shell命令（例如，文件或目录不存在）
*   **126**：命令不可执行（例如，权限不足）
*   **127**：命令未找到
*   **130**：通过 `Ctrl+C` 中断退出

![](img/a2df6e709ec3dbe7c74e925661b753d0_93.png)

### 示例

![](img/a2df6e709ec3dbe7c74e925661b753d0_95.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_97.png)

```bash
ls /tmp          # 成功执行
echo $?          # 输出：0

![](img/a2df6e709ec3dbe7c74e925661b753d0_99.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_100.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_101.png)

ls /nonexistent  # 目录不存在
echo $?          # 输出：2

![](img/a2df6e709ec3dbe7c74e925661b753d0_102.png)

./non_executable_file  # 文件没有执行权限
echo $?                # 输出：126

![](img/a2df6e709ec3dbe7c74e925661b753d0_104.png)

sleep 10               # 执行过程中按 Ctrl+C 中断
echo $?                # 输出：130
```

![](img/a2df6e709ec3dbe7c74e925661b753d0_106.png)

### 使用 `exit` 命令

![](img/a2df6e709ec3dbe7c74e925661b753d0_108.png)

在脚本中，可以使用 `exit` 命令来主动结束脚本，并指定一个退出状态码。如果不指定，脚本将以最后一条命令的退出状态码作为整个脚本的退出码。

![](img/a2df6e709ec3dbe7c74e925661b753d0_110.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_111.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_112.png)

```bash
#!/bin/bash
# test_exit.sh
for i in {1..3}
do
  echo "Number: $i"
  if [ $i -eq 2 ]; then
    exit 1  # 当 i 等于 2 时，主动退出并返回状态码 1
  fi
done
echo "This line will not be executed."

![](img/a2df6e709ec3dbe7c74e925661b753d0_114.png)

![](img/a2df6e709ec3dbe7c74e925661b753d0_115.png)

# 执行脚本后检查状态码
./test_exit.sh
echo "Exit code of the script is: $?" # 输出：1
```
