## 概述：MBR分区与设备文件

上一节我们介绍了磁盘分区的基本理论。本节中，我们来看看如何使用`fdisk`命令对MBR格式的磁盘进行实际操作。

![](img/9bef6761a51bd460fcfc360917ca5eff_11.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_13.png)

`fdisk`命令用于查看磁盘使用情况和进行磁盘分区，主要功能是分区，且适用于MBR分区格式。命令格式非常简单：`fdisk`后跟上设备路径即可。

*   **硬盘设备**：通常以`sd`开头，如`sda`、`sdb`、`sdc`。
*   **软盘驱动设备**：以`fd`开头。
*   **软RAID设备**：以`md`开头。
*   **回环设备**：`loop*`，代表本机回环网卡（`lo`）。
*   **光驱设备**：`sr0`，是`cdrom`的快捷方式。
*   **内存设备**：`mem`。
*   **空设备（黑洞设备）**：`null`，任何写入的数据都将被丢弃。
*   **满设备**：`full`，任何写入的数据都将失败。
*   **虚拟终端设备**：`tty*`，每打开一个终端就会占用一个，最多63个。
*   **随机数设备**：`random`和`urandom`。
*   **IO端口**：`port`。

在这些设备中，硬盘、光驱等是我们后续操作中比较常用的。

## 使用fdisk进行MBR分区

现在，我们开始对一块新硬盘（例如`/dev/sdb`）进行分区。注意，系统盘（如`sda`）通常已满，不建议操作。

执行命令 `fdisk /dev/sdb` 进入交互式分区界面。初始提示表明所有更改暂存于内存，只有在保存后才会写入磁盘。

输入 `m` 可以获取帮助信息。以下是几个常用命令：

*   `p`：打印当前磁盘的分区表。
*   `n`：添加一个新分区。
*   `d`：删除一个分区。
*   `w`：保存更改并退出。
*   `q`：不保存更改并退出。

使用 `p` 命令可以查看磁盘信息，其中 **Disk label type: dos** 表示该磁盘使用MBR分区格式。一块硬盘只能选择一种分区格式（MBR或GPT），两者不兼容。

接下来，我们开始创建分区。输入 `n` 创建新分区：

1.  选择分区类型：`p` 为主分区，`e` 为扩展分区。默认是主分区（`p`）。
2.  设置分区编号：对于主分区，编号为1-4。系统会自动建议下一个可用编号。
3.  设置起始扇区：通常使用默认值（直接回车）。
4.  设置分区大小：可以使用 `+size{K,M,G}` 的格式指定，例如 `+10G` 表示创建一个10GB的分区。

重复 `n` 命令可以创建更多分区。**MBR格式最多允许4个主分区**。当已有3个主分区后，创建第4个分区时，系统会默认建议创建扩展分区（`e`）。

**扩展分区**本身不直接存储数据，它是一个容器，用于在其中创建**逻辑分区**。逻辑分区的编号从5开始。

**重要原则**：如果计划创建超过4个分区，必须使用扩展分区。并且，**应将所有剩余空间分配给扩展分区**，否则剩余空间将无法使用。

![](img/9bef6761a51bd460fcfc360917ca5eff_42.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_44.png)

在扩展分区内，使用 `n` 命令创建的分区即为逻辑分区，其创建过程与主分区类似。

完成所有分区操作后，输入 `w` 保存并退出。可以使用 `fdisk -l /dev/sdb` 命令查看详细的分区信息，其中会标明分区类型（主分区、扩展分区）。

**如何区分主分区、扩展分区和逻辑分区？**
对于MBR磁盘，编号1-4的是主分区或扩展分区（通过`fdisk -l`查看类型）。编号5及以上的分区一定是逻辑分区。

![](img/9bef6761a51bd460fcfc360917ca5eff_46.png)

## 文件系统介绍

分区创建后，并不能直接存储文件。我们需要为分区建立一个“文件管理系统”，即**格式化**。

![](img/9bef6761a51bd460fcfc360917ca5eff_48.png)

文件系统是管理分区中文件存储、属性、读取等的系统。常见的Linux文件系统有：

*   **ext4**：CentOS 6默认文件系统。
    *   支持日志功能，可避免意外宕机导致的数据丢失。
    *   单个文件系统最大支持 **1EB** 的分区容量。
    *   单个文件最大支持 **16TB**。
*   **XFS**：CentOS 7及RHEL 8等现代系统的默认文件系统。
    *   日志功能数据恢复速度更快。
    *   单个文件系统最大支持 **8EB** 的分区容量。
    *   单个文件最大支持 **500TB**。
    *   数据读取速度快，被认为是当前主流的高性能文件系统。

![](img/9bef6761a51bd460fcfc360917ca5eff_50.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_52.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_54.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_55.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_57.png)

格式化命令为 `mkfs`。例如，将分区 `/dev/sdb1` 格式化为XFS文件系统：
```bash
mkfs.xfs /dev/sdb1
```
格式化后，可以使用 `lsblk -f /dev/sdb1` 查看分区的文件系统类型。

![](img/9bef6761a51bd460fcfc360917ca5eff_59.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_61.png)

## 挂载与卸载

在Linux中，用户不能直接访问硬件设备文件（如`/dev/sdb1`）。必须通过**挂载**，为分区提供一个访问入口，这个入口就是一个**目录**。

**挂载**的本质是建立分区与目录的关联。此后，向该目录存入文件，实际上就是存入对应的分区中。

![](img/9bef6761a51bd460fcfc360917ca5eff_63.png)

挂载命令是 `mount`，基本格式为：
```bash
mount <设备路径> <挂载点目录>
```
例如，将格式化的 `/dev/sdb1` 挂载到 `/mydata` 目录：
```bash
mount /dev/sdb1 /mydata
```
使用 `df -h` 命令可以查看已挂载的文件系统及其使用情况。

![](img/9bef6761a51bd460fcfc360917ca5eff_64.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_66.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_68.png)

**卸载**命令是 `umount`，可以指定设备路径或挂载点：
```bash
umount /dev/sdb1
# 或
umount /mydata
```

![](img/9bef6761a51bd460fcfc360917ca5eff_70.png)

![](img/9bef6761a51bd460fcfc360917ca5eff_72.png)

**挂载注意事项**：
1.  **挂载点通常应是一个空目录**。如果挂载到一个非空目录，原目录下的文件会被暂时隐藏（卸载后恢复）。
2.  **一个分区不能同时挂载到多个目录**。
3.  **一个目录不能同时挂载多个分区**。
4.  使用 `mount` 进行的挂载在系统重启后会失效，需要配置自动挂载（将在后续课程讲解）。

![](img/9bef6761a51bd460fcfc360917ca5eff_74.png)

**物理分区的局限性**：每个分区大小固定。当一个分区空间用尽后，无法直接扩容。需要卸载后，挂载一个新的、空间更大的分区到同一目录，或者将新分区挂载到另一个目录来继续存储数据。旧分区中的数据依然存在，可以重新挂载访问。

![](img/9bef6761a51bd460fcfc360917ca5eff_76.png)
