### 1.1 命令行与Shell简介

![](img/a7142e364802711818ad3c1a9f42cc80_53.png)

![](img/a7142e364802711818ad3c1a9f42cc80_55.png)

在Linux系统中，用户与内核的交互主要通过命令行界面进行。这与Windows系统以图形化界面（GUI）为核心的操作方式不同。Linux的图形化界面是可选的，而命令行是核心。

![](img/a7142e364802711818ad3c1a9f42cc80_57.png)

命令行（Command Line）是基于文本的界面，用户通过输入文本指令来操作计算机。在Linux中，命令行功能由**Shell**程序提供。Shell是一个命令解释器，负责接收用户输入的字符串，并将其传递给系统内核进行处理。

![](img/a7142e364802711818ad3c1a9f42cc80_59.png)

![](img/a7142e364802711818ad3c1a9f42cc80_61.png)

![](img/a7142e364802711818ad3c1a9f42cc80_63.png)

红帽企业Linux系统默认使用的Shell是**Bash**（Bourne-Again SHell）。

![](img/a7142e364802711818ad3c1a9f42cc80_65.png)

**打开终端的方法**：
1.  在图形化界面中，点击左上角“应用程序”菜单，选择“工具”中的“终端”。
2.  使用快捷键 `Ctrl + Alt + F2` 到 `F6` 可以切换到纯文本虚拟控制台。

### 1.3 命令的基本结构

![](img/a7142e364802711818ad3c1a9f42cc80_120.png)

![](img/a7142e364802711818ad3c1a9f42cc80_122.png)

![](img/a7142e364802711818ad3c1a9f42cc80_124.png)

一条完整的Shell命令通常由三部分组成，格式如下：
```
命令 [选项] [参数]
```
*   **命令**：要执行的操作或程序名称，是必须的部分。
*   **选项**：用于调整命令行为的修饰符，通常以 `-`（短选项）或 `--`（长选项）开头。例如 `-l`， `--help`。
*   **参数**：命令作用的对象，例如文件名、目录名等。

![](img/a7142e364802711818ad3c1a9f42cc80_126.png)

![](img/a7142e364802711818ad3c1a9f42cc80_128.png)

![](img/a7142e364802711818ad3c1a9f42cc80_129.png)

![](img/a7142e364802711818ad3c1a9f42cc80_131.png)

![](img/a7142e364802711818ad3c1a9f42cc80_133.png)

**示例分析**：
以 `ls -lh /home` 命令为例：
*   `ls` 是命令本身，意为“列出目录内容”。
*   `-lh` 是选项的组合。`-l` 表示以长格式（详细信息）列出，`-h` 表示以人类可读的格式（如K， M）显示文件大小。
*   `/home` 是参数，指定了 `ls` 命令要查看的目录。

![](img/a7142e364802711818ad3c1a9f42cc80_135.png)

绝大多数命令都支持 `--help` 选项，用于显示该命令的帮助信息。

**常用基础命令**：
*   `date`：显示或设置系统日期和时间。
    *   示例：`date +%Y-%m-%d` 可以仅输出年月日。
*   `passwd`：修改用户密码。root用户可以修改任何用户的密码。
*   `file`：查看文件类型。
*   `head` / `tail`：查看文件的开头或结尾部分，默认显示10行。
    *   示例：`head -n 5 file.txt` 查看文件前5行。
*   `wc`：统计文件的行数、单词数和字符数。
    *   示例：`wc -l file.txt` 统计文件行数。

![](img/a7142e364802711818ad3c1a9f42cc80_164.png)

![](img/a7142e364802711818ad3c1a9f42cc80_166.png)

**命令行效率技巧**：
*   **Tab键补全**：输入命令、选项或参数的前几个字符后按Tab键，Shell会尝试自动补全。按两次Tab键会列出所有可能的补全项。
*   **命令历史**：
    *   `history`：查看之前执行过的命令历史列表。
    *   `!!`：执行上一条命令。
    *   `!n`：执行历史列表中编号为 `n` 的命令。
    *   上下方向键：在历史命令中浏览。
    *   `Ctrl + R`：反向搜索历史命令。
*   **光标快捷操作**：
    *   `Ctrl + A` / `Ctrl + E`：将光标快速移动到**行首**/**行尾**。
    *   `Ctrl + K` / `Ctrl + U`：删除从光标处到**行尾**/**行首**的所有内容。
    *   `Alt + F` / `Alt + B`：将光标向前/向后移动一个**单词**。
*   **引用上一条命令的参数**：`Esc + .`（先按Esc键，再按点号）可以快速输入上一条命令的最后一个参数。

## 第二章：从命令行管理文件

### 2.1 Linux文件系统层次结构

![](img/a7142e364802711818ad3c1a9f42cc80_182.png)

Linux的文件系统采用单一的树形层次结构，一切皆从**根目录（`/`）**开始。这与Windows的多个盘符（C:， D:）结构不同。

![](img/a7142e364802711818ad3c1a9f42cc80_184.png)

![](img/a7142e364802711818ad3c1a9f42cc80_186.png)

以下是几个重要目录及其作用：
*   `/`：根目录，所有文件和目录的起点。
*   `/etc`：存放系统的**配置文件**。
*   `/home`：普通用户的**家目录**所在地。每个用户在此拥有一个以自己用户名命名的子目录。
*   `/root`：超级用户root的**家目录**。
*   `/var`：存放经常变化的（Variable）数据，如日志文件、数据库文件等。
*   `/dev`：存放**设备文件**。在Linux中，硬件设备（如磁盘、USB）也被抽象为文件。
*   `/usr`：存放系统软件和第三方应用程序。
*   `/tmp`：临时文件目录，系统会定期清理。
*   `/boot`：存放系统启动所需的文件，切勿随意删除。

![](img/a7142e364802711818ad3c1a9f42cc80_188.png)

![](img/a7142e364802711818ad3c1a9f42cc80_190.png)

### 2.2 绝对路径与相对路径

要定位一个文件或目录，需要使用**路径**。路径分为两种：

*   **绝对路径**：从根目录 `/` 开始写起的完整路径。
    *   示例：`/home/student/file.txt`
    *   特点：唯一且明确，无论当前位于何处，都能准确找到目标。
*   **相对路径**：相对于**当前工作目录**的路径。
    *   `.` 代表当前目录。
    *   `..` 代表上一级目录。
    *   `~` 代表当前用户的家目录。
    *   示例：假设当前在 `/home`，那么 `student/file.txt` 就是一个相对路径。
    *   特点：书写简短，但依赖当前所在位置。

![](img/a7142e364802711818ad3c1a9f42cc80_192.png)

![](img/a7142e364802711818ad3c1a9f42cc80_194.png)

使用 `pwd` 命令可以查看当前所在的**绝对路径**。

现在我们已经了解了路径的概念，本节中我们将学习如何使用命令来创建、查看、移动和删除文件与目录。

**导航与查看**：
*   `pwd`：显示当前工作目录。
*   `ls`：列出目录内容。
    *   `ls -l`：以长格式列出详细信息。
    *   `ls -a`：显示所有文件，包括隐藏文件（以 `.` 开头的文件）。
    *   `ls -R`：递归列出子目录内容。
*   `cd`：切换目录。
    *   `cd ~` 或 `cd`：切换到当前用户的家目录。
    *   `cd -`：切换到上一个工作目录。
    *   `cd ..`：切换到上一级目录。

![](img/a7142e364802711818ad3c1a9f42cc80_209.png)

**创建与删除**：
*   `touch`：创建空文件，或更新已有文件的时间戳。
    *   示例：`touch newfile.txt`
*   `mkdir`：创建新目录。
    *   `mkdir -p dir1/dir2/dir3`：递归创建多级目录。
*   `rm`：删除文件或目录。
    *   `rm file.txt`：删除文件。
    *   `rm -r directory`：递归删除目录及其内容。
    *   `rm -f file`：强制删除，不提示确认（**慎用**）。
*   `rmdir`：删除空目录。

**复制、移动与重命名**：
*   `cp`：复制文件或目录。
    *   `cp file1 file2`：复制文件。
    *   `cp -r dir1 dir2`：递归复制目录。
    *   `cp -a source dest`：归档模式复制，保留所有文件属性。
*   `mv`：移动或重命名文件/目录。
    *   `mv oldname newname`：重命名。
    *   `mv file /target/directory`：移动文件到目标目录。

### 2.4 路径名扩展（通配符）

为了高效地批量操作文件，Shell提供了路径名扩展功能，主要通过**通配符**实现：

*   `*`：匹配任意长度的任意字符（包括零个字符）。
    *   示例：`rm *.log` 删除所有 `.log` 结尾的文件。
*   `?`：匹配任意单个字符。
    *   示例：`ls file?.txt` 匹配 `file1.txt`， `fileA.txt` 等。
*   `[]`：匹配括号内的任意一个字符。
    *   示例：`ls file[123].txt` 匹配 `file1.txt`， `file2.txt`， `file3.txt`。
*   `{}`：生成序列或组合。
    *   示例：`mkdir dir{1,2,3}` 会创建 `dir1`， `dir2`， `dir3` 三个目录。
    *   示例：`touch file{1..10}.txt` 创建 `file1.txt` 到 `file10.txt`。

![](img/a7142e364802711818ad3c1a9f42cc80_237.png)

![](img/a7142e364802711818ad3c1a9f42cc80_239.png)

### 2.5 命令替换与转义

![](img/a7142e364802711818ad3c1a9f42cc80_241.png)

*   **命令替换**：`$(command)` 或 `` `command` ``。Shell会先执行括号或反引号内的命令，然后用其输出结果替换整个表达式。
    *   示例：`echo "Today is $(date)"` 会先执行 `date` 命令，然后将结果嵌入到字符串中输出。
*   **转义字符**：`\`。用于消除其后一个字符的特殊含义，使其作为普通字符处理。
    *   示例：如果文件名包含空格 `my file.txt`，在命令行中需要写为 `my\ file.txt` 或 `"my file.txt"`。

4.  **文件操作**：掌握了使用 `touch`， `mkdir`， `cp`， `mv`， `rm` 等命令进行创建、复制、移动/重命名和删除文件与目录的操作。
5.  **高级技巧**：了解了绝对路径与相对路径的区别，并学会了使用通配符（`*`， `?` 等）进行批量操作，以及命令替换和转义字符的基本用法。
