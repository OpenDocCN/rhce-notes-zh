在本节课中，我们将要学习Linux系统中关于文件与目录的两个核心概念：归属与权限。理解并掌握如何查看、修改文件的归属关系（属主与属组）以及基本权限（RWX），是管理Linux系统安全的基础。我们还将学习一种更精细的权限控制机制——访问控制列表（ACL），它允许我们对特定用户或组设置独立的权限。

## 文档的归属与权限概述 📄

![](img/6cef1a3de2e9540ecdc235a2951ad07a_8.png)

上一节我们介绍了用户账号和组账号的管理，本节中我们来看看文档的权限和归属。在RHCSA考试中，有多道题目与此相关，例如配置文件权限和创建共用目录。这些考点都围绕着文档的归属权和文档的权限展开。

通常，在系统中创建了账户后，我们需要考虑用户登录后对某个目录或文件能进行何种操作，以及该目录或文件属于谁。因此，系统中有两个关键概念：**归属**和**权限**。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_10.png)

## 理解文档归属 👤

归属指的是一个文件归哪个用户所有。Linux系统为每个文件标记了两个归属信息：
*   **属主**：文件属于哪个用户。
*   **属组**：文件属于哪个组。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_12.png)

这类似于公司里，一台笔记本电脑既属于某个具体员工（属主），也属于该员工所在的部门（属组）。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_14.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_16.png)

### 如何查看归属关系

![](img/6cef1a3de2e9540ecdc235a2951ad07a_18.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_20.png)

我们可以使用 `ls -l` 命令查看文件的详细属性，其中就包含了归属信息。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_22.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_24.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_26.png)

```bash
ls -l /etc/passwd
```

![](img/6cef1a3de2e9540ecdc235a2951ad07a_28.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_30.png)

输出结果中，`root root` 部分表示 `/etc/passwd` 这个文件属于 `root` 用户（第一个`root`）和 `root` 组（第二个`root`）。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_32.png)

### 三类访问身份

系统根据用户与文件的归属关系，将访问者分为三类：
1.  **属主**：文件的所有者。
2.  **属组**：与文件属主同组的用户。
3.  **其他人**：既不是属主，也不在属组内的任何其他用户。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_34.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_36.png)

设置权限时，就是针对这三类身份分别进行的。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_37.png)

## 理解基本权限 🔑

![](img/6cef1a3de2e9540ecdc235a2951ad07a_39.png)

了解了归属关系后，我们来看看权限。权限决定了不同身份的访问者能对文件或目录做什么操作。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_40.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_42.png)

### 权限标记与查看

使用 `ls -l` 命令时，输出信息的第一部分（如 `-rw-r--r--`）就是权限标记。
*   第一个字符表示文件类型（`-` 代表普通文件，`d` 代表目录）。
*   后面9个字符分为三组，每组三个字符，分别对应**属主**、**属组**和**其他人**的权限。
*   每组中的三个字符顺序固定为：`r` (读)、`w` (写)、`x` (执行)。拥有相应权限则显示对应字母，否则显示 `-`。

例如，`-rw-r--r--` 表示：
*   属主有读(`r`)、写(`w`)权限，无执行(`x`)权限。
*   属组有读(`r`)权限，无写和执行权限。
*   其他人有读(`r`)权限，无写和执行权限。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_44.png)

### 权限对文件和目录的含义

虽然标记相同，但 `r`、`w`、`x` 权限对文件和目录的含义不同：

![](img/6cef1a3de2e9540ecdc235a2951ad07a_46.png)

**对文件的含义：**
*   `r`：可以读取文件内容（如使用 `cat`, `vim`）。
*   `w`：可以修改文件内容。
*   `x`：可以将文件作为程序来执行。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_48.png)

**对目录的含义：**
*   `r`：可以列出目录内的内容（如使用 `ls`）。
*   `w`：可以在目录内创建、删除、重命名子文件或子目录。
*   `x`：可以进入该目录（如使用 `cd`）。

**重要提示**：对于目录，如果用户没有 `x`（执行）权限，则即使拥有 `r`（读）权限，也无法进入目录或列出其内容。

## 修改归属关系 🛠️

有时我们需要更改文件或目录的归属关系，这时需要使用 `chown` 命令。

以下是 `chown` 命令的基本用法：

*   **更改属主**：`chown 用户名 文件或目录`
*   **更改属组**：`chown :组名 文件或目录`
*   **同时更改属主和属组**：`chown 用户名:组名 文件或目录`
*   **递归更改**（同时更改目录及其内部所有内容的归属）：`chown -R 用户名:组名 目录`

例如，将 `/home/zhangsan` 目录的属主改为 `lisi`：
```bash
chown lisi /home/zhangsan
```

![](img/6cef1a3de2e9540ecdc235a2951ad07a_50.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_52.png)

## 修改基本权限 🛠️

要直接修改文件或目录的权限标记，需要使用 `chmod` 命令。

`chmod` 命令通过操作符和身份字母来修改权限：
*   **身份字母**：`u` (属主), `g` (属组), `o` (其他人), `a` (所有人，即`u+g+o`)。
*   **操作符**：`+` (添加权限), `-` (移除权限), `=` (直接设定权限)。

以下是 `chmod` 命令的用法示例：

*   为其他人添加读和执行权限：`chmod o+rx 目录`
*   移除属组的写权限：`chmod g-w 文件`
*   将属主的权限直接设置为读和执行：`chmod u=rx 文件`
*   同时为属组和其他人添加读权限：`chmod go+r 文件`
*   递归修改目录及其内部所有内容的权限：`chmod -R o+rx 目录`

## 访问控制列表（ACL）进阶 🎯

![](img/6cef1a3de2e9540ecdc235a2951ad07a_61.png)

基本权限模型（属主、属组、其他人）有时不够灵活。例如，一个属于 `root` 用户的文件，我们需要让用户 `zhangsan` 能读写，而用户 `lisi` 完全不能访问。此时，`zhangsan` 和 `lisi` 都属于“其他人”，无法通过基本权限区分对待。

![](img/6cef1a3de2e9540ecdc235a2951ad07a_63.png)

这时就需要使用**访问控制列表**。ACL允许我们为个别用户或组设置独立的、超出基本权限模型的权限。

### ACL管理命令

![](img/6cef1a3de2e9540ecdc235a2951ad07a_65.png)

![](img/6cef1a3de2e9540ecdc235a2951ad07a_67.png)

*   **查看ACL**：`getfacl 文件或目录`
*   **设置ACL**：`setfacl -m 条目 文件或目录`
*   **删除所有ACL条目**：`setfacl -b 文件或目录`

![](img/6cef1a3de2e9540ecdc235a2951ad07a_69.png)

### ACL条目格式

*   **为用户设置**：`user:用户名:权限` (例如 `user:zhangsan:rw`)
*   **为组设置**：`group:组名:权限`

![](img/6cef1a3de2e9540ecdc235a2951ad07a_71.png)

例如，让用户 `zhangsan` 对 `file1` 拥有读写权限，而用户 `lisi` 没有任何权限：
```bash
setfacl -m user:zhangsan:rw file1
setfacl -m user:lisi:--- file1
# 或者简写为 setfacl -m user:lisi:- file1
```
设置后，使用 `getfacl file1` 可以查看到添加的ACL条目。
