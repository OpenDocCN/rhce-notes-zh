# Linux运维培训教程：1.20：常用特殊符号与grep文件内容过滤 🔍

![](img/97677a5244ce354b231d907ac329a952_1.png)

在本节课中，我们将学习Linux系统中常用的特殊符号以及`grep`命令的使用。特殊符号能帮助我们高效地匹配和操作文件，而`grep`命令则是从文件中查找特定内容的强大工具。掌握这些知识，能极大地提升你在命令行下的工作效率。

## 常用特殊符号

![](img/97677a5244ce354b231d907ac329a952_3.png)

![](img/97677a5244ce354b231d907ac329a952_5.png)

![](img/97677a5244ce354b231d907ac329a952_6.png)

在Linux系统中，键盘上的许多符号（如 `!`、`@`、`#`、`$`、`%`、`^`、`&`、`*`、`()`、`{}`、`[]`、`<>`、`?`、`/`、`\`、`|` 等）都具有特殊含义。它们被称为特殊符号，能够完成特定的功能。

初级阶段，我们主要学习四个符号：`*`、`{}`、`?` 和 `[]`。其中，`*`和`{}`最为常用，`?`和`[]`了解即可。

### 星号 `*`

星号 `*` 是一个通配符，应用在文件名上，用于代表**任意多个任意字符**。
*   **任意字符**：可以是数字、字母（大小写）或其他符号。
*   **任意多个**：不限制字符的长度。

![](img/97677a5244ce354b231d907ac329a952_8.png)

它的功能非常强大，常用于文件搜索和批量操作。

![](img/97677a5244ce354b231d907ac329a952_10.png)

以下是`*`的一些常见用法示例：

![](img/97677a5244ce354b231d907ac329a952_12.png)

*   **搜索文件**：当你记不清完整文件名时，可以用`*`进行模糊匹配。
    ```bash
    # 搜索/etc目录下所有以`pass`开头的文件
    ls /etc/pass*
    # 搜索/var/log目录下所有以`.log`结尾的文件
    ls /var/log/*.log
    ```
*   **批量操作**：结合命令对匹配到的文件进行批量处理。
    ```bash
    # 将/var/log下所有.log文件备份到/opt目录
    cp -p /var/log/*.log /opt/
    ```
*   **危险操作**：`*`代表所有，需谨慎使用。
    ```bash
    # 此命令会删除根目录下的所有文件（极度危险！）
    rm -rf /*
    ```

### 问号 `?`

![](img/97677a5244ce354b231d907ac329a952_14.png)

![](img/97677a5244ce354b231d907ac329a952_15.png)

问号 `?` 同样应用在文件名上，用于代表**任意单个字符**。
*   它只匹配一个字符的位置，但该字符可以是任意内容。

![](img/97677a5244ce354b231d907ac329a952_17.png)

`?`的使用频率相对较低。

![](img/97677a5244ce354b231d907ac329a952_19.png)

![](img/97677a5244ce354b231d907ac329a952_21.png)

以下是`?`的用法示例：

```bash
# 搜索/dev目录下，以`tty`开头且结尾只有一个字符的文件
ls /dev/tty?
# 搜索/dev目录下，以`tty`开头且结尾只有两个字符的文件
ls /dev/tty??
```

### 中括号 `[]`

![](img/97677a5244ce354b231d907ac329a952_23.png)

![](img/97677a5244ce354b231d907ac329a952_24.png)

中括号 `[]` 用于匹配文件名中**指定范围或集合内的一个字符**。
*   它通常用于匹配**数字**或**字母**。
*   使用`-`可以表示一个连续的范围，例如 `[0-9]` 或 `[a-z]`。

以下是`[]`的用法示例：

```bash
# 搜索/var/log目录下，文件名中包含数字的文件
ls /var/log/*[0-9]*
# 搜索/var/log目录下，文件名中包含字母a、b、c、d的文件
ls /var/log/*[abcd]*
```

### 大括号 `{}`

大括号 `{}` 功能强大，常用于生成**序列**或代表**多组不同的字符串**，尤其擅长处理数字序列。

以下是`{}`的核心用法示例：

![](img/97677a5244ce354b231d907ac329a952_26.png)

![](img/97677a5244ce354b231d907ac329a952_27.png)

*   **生成数字序列**：这是最常用且实用的功能。
    ```bash
    # 在当前目录创建test1到test100，共100个文件
    touch test{1..100}
    # 删除test51到test60的文件
    rm -rf test{51..60}
    ```
*   **生成字符串列表**：也可以用于非数字的枚举。
    ```bash
    # 搜索/etc目录下，文件名包含a或b或c或d的文件
    ls /etc/*{a,b,c,d}*
    ```

![](img/97677a5244ce354b231d907ac329a952_29.png)

![](img/97677a5244ce354b231d907ac329a952_31.png)

**小结**：在以上四个符号中，`*`（任意多个字符）和`{}`（生成序列）是必须掌握的核心。`?`（单个字符）和`[]`（指定范围）了解其概念即可。

## grep文件内容过滤命令

![](img/97677a5244ce354b231d907ac329a952_35.png)

![](img/97677a5244ce354b231d907ac329a952_37.png)

上一节我们介绍了用于文件名匹配的特殊符号，本节中我们来看看如何过滤文件**内部**的内容。`grep`命令用于在文件中查找并显示包含指定字符串的行。

它的基本语法是：
```bash
grep [选项] “查找的字符串” 文件名
```

### 基本用法

直接从文件中过滤出包含特定关键词的行。

```bash
# 在/etc/passwd文件中查找包含“tom”的行
grep “tom” /etc/passwd
# 显示行号，方便定位
grep -n “tom” /etc/passwd
```

### 常用选项

`grep`命令有一些实用的选项来增强其功能。

以下是几个关键选项的介绍：

*   **`-n`**：显示匹配行在文件中的行号。
    ```bash
    grep -n “root” /etc/passwd
    ```
*   **`-i`**：忽略大小写进行匹配。
    ```bash
    # 匹配“user”、“USER”、“User”等
    grep -i “user” /etc/passwd
    ```
*   **`-v`**：反向选择，即显示**不包含**匹配字符串的所有行。
    ```bash
    # 显示/etc/passwd中不包含“root”的行
    grep -v “root” /etc/passwd
    ```

### 结合正则表达式符号

`grep`可以结合更精确的正则表达式符号进行过滤。

以下是两个常用的锚点符号：

*   **`^`**：匹配以指定字符串**开头**的行。
    ```bash
    # 匹配以“root”开头的行
    grep “^root” /etc/passwd
    ```
*   **`$`**：匹配以指定字符串**结尾**的行。
    ```bash
    # 匹配以“/bin/bash”结尾的行（可登录的普通用户）
    grep “/bin/bash$” /etc/passwd
    # 匹配以“/sbin/nologin”结尾的行（不可登录的系统用户）
    grep “/sbin/nologin$” /etc/passwd
    ```
*   **`^$`**：组合使用，匹配空行。
    ```bash
    # 统计/etc/login.defs文件中有多少空行
    grep “^$” /etc/login.defs | wc -l
    ```

### 高级过滤技巧

![](img/97677a5244ce354b231d907ac329a952_39.png)

![](img/97677a5244ce354b231d907ac329a952_41.png)

通过组合使用`grep`、选项和管道符`|`，可以实现复杂的过滤需求。

例如，要查看一个配置文件（如`/etc/login.defs`）中所有**非注释且非空行**的有效配置：

```bash
# 1. 先排除所有以#开头的注释行
# 2. 再排除所有空行
# 3. 显示行号
grep -v “^#” /etc/login.defs | grep -v “^$” | grep -n “.*”
```

![](img/97677a5244ce354b231d907ac329a952_45.png)

### 结合其他命令使用

![](img/97677a5244ce354b231d907ac329a952_47.png)

`grep`经常通过管道符`|`接收其他命令的输出结果，进行二次过滤。

![](img/97677a5244ce354b231d907ac329a952_49.png)

```bash
# 查看内存信息，只显示包含“Mem”的行
free -h | grep “Mem”
# 查看磁盘分区，只显示根分区`/`的信息
df -h | grep “/$”
```

![](img/97677a5244ce354b231d907ac329a952_51.png)

![](img/97677a5244ce354b231d907ac329a952_53.png)

本节课中我们一起学习了Linux中常用的特殊符号（`*`， `?`， `[]`， `{}`）和强大的文件内容过滤命令`grep`。`*`和`{}`用于灵活匹配和生成文件名，而`grep`则能精准定位文件内的文本。它们是命令行操作中不可或缺的工具，多加练习会让你在Linux系统管理上更加得心应手。
