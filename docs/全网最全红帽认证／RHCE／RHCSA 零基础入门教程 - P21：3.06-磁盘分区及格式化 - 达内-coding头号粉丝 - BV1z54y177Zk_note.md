*   **`/dev/vda`**：这是系统盘，包含启动分区、根分区和交换分区，我们无需改动。
*   **`/dev/vdb`**：这是我们的主要操作盘。它已有一个分区（`/dev/vdb1`）用于创建了一个卷组，其中包含一个约200MB的逻辑卷（如`vo`）。此外，我们还需要在这块磁盘上创建新分区，用于完成“添加交换分区”和“创建新逻辑卷”的题目。
*   **`/dev/vdc`**：这是一块完全空闲的磁盘，**必须保留**，专门用于完成“创建VDO卷”的题目。

## 磁盘使用流程

![](img/e593c03dd7ed1749cd6890914bdf542d_41.png)

在使用一块新磁盘存储数据前，需要遵循以下标准流程：
1.  **识别磁盘**：系统启动时内核自动识别。我们可以使用命令查看。
2.  **分区**：使用工具（如`fdisk`）在磁盘上划分逻辑区域。
3.  **格式化**：在分区上创建文件系统（如`ext4`），以便操作系统识别和存取数据。
4.  **挂载**：将格式化好的分区关联到Linux目录树的某个路径（挂载点），之后才能通过该目录访问存储空间。

![](img/e593c03dd7ed1749cd6890914bdf542d_43.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_45.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_47.png)

下面，我们将详细讲解每个步骤的操作。

## 识别与查看磁盘

首先，我们需要查看系统识别到的磁盘设备。以下是两个常用命令：

*   **`lsblk`**：以树状结构简洁地列出所有块设备（磁盘、分区）。
    ```bash
    lsblk
    ```
*   **`fdisk -l`**：列出所有磁盘的详细信息，包括分区表、容量等。
    ```bash
    fdisk -l
    ```
    或查看指定磁盘：
    ```bash
    fdisk -l /dev/vdb
    ```

## 磁盘分区

分区是划分磁盘空间的过程。Linux常用的分区工具有`fdisk`和`parted`。

![](img/e593c03dd7ed1749cd6890914bdf542d_49.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_51.png)

### 使用 `fdisk` 分区

![](img/e593c03dd7ed1749cd6890914bdf542d_53.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_54.png)

`fdisk` 是一个交互式分区工具，适合新手，因为它的更改在确认前不会生效。

![](img/e593c03dd7ed1749cd6890914bdf542d_56.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_58.png)

**基本操作流程如下：**
1.  执行 `fdisk /dev/vdb` 进入交互界面。
2.  输入 `m` 可以查看所有可用命令的帮助信息。
3.  常用命令：
    *   `p`：打印（显示）当前分区表。
    *   `n`：创建新分区。
    *   `d`：删除分区（**慎用**，建议分错了直接按`q`退出重来）。
    *   `w`：将分区表更改写入磁盘并退出。
    *   `q`：不保存更改，直接退出。

**示例：创建512MB分区**
假设要为`/dev/vdb`添加一个512MB的分区。
```bash
fdisk /dev/vdb
```
在交互界面中：
1.  输入 `n` 创建新分区。
2.  选择分区编号（通常直接回车接受默认值）。
3.  选择起始扇区（直接回车从默认位置开始）。
4.  设置结束位置时，**使用 `+512M`** 指定分区大小为512MB。
5.  输入 `p` 确认分区信息。
6.  确认无误后，输入 `w` 保存并退出。

![](img/e593c03dd7ed1749cd6890914bdf542d_60.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_61.png)

**分区生效**：分区完成后，需要让内核重新读取分区表。
*   最稳妥的方式是重启：`reboot`
*   若不重启，可尝试命令：`partprobe /dev/vdb` 或 `partx -a /dev/vdb`

![](img/e593c03dd7ed1749cd6890914bdf542d_63.png)

### 使用 `parted` 分区

`parted` 功能更强大，支持GPT等新分区表，但它的更改会**立即生效**，适合有经验的管理员。
```bash
parted /dev/vdb
```
在`parted`交互界面中，输入`help`查看命令。常用`print`查看分区表，`mkpart`创建分区。

## 格式化分区

分区完成后，需要格式化才能创建文件系统。使用 `mkfs` 命令族。

![](img/e593c03dd7ed1749cd6890914bdf542d_65.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_66.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_68.png)

**命令格式**：
```bash
mkfs.文件系统类型 分区设备
```
例如，将 `/dev/vdb2` 格式化为 `ext4` 文件系统：
```bash
mkfs.ext4 /dev/vdb2
```
输入 `mkfs` 后按两次Tab键，可以查看系统支持的所有文件系统类型。

## 实战：添加交换分区

现在，我们应用以上知识来完成“添加交换分区”的题目。题目要求：添加一个512MB的交换分区，并使其开机自动挂载。

![](img/e593c03dd7ed1749cd6890914bdf542d_70.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_72.png)

以下是操作步骤：

![](img/e593c03dd7ed1749cd6890914bdf542d_74.png)

![](img/e593c03dd7ed1749cd6890914bdf542d_76.png)

1.  **分区**：使用`fdisk`在`/dev/vdb`上创建一个512MB的新分区（例如`/dev/vdb2`），方法如前所述。
2.  **格式化**：交换分区需要特殊的格式化处理，使用 `mkswap` 命令。
    ```bash
    mkswap /dev/vdb2
    ```
3.  **启用交换分区**：使用 `swapon` 命令激活它。
    ```bash
    swapon /dev/vdb2
    ```
    使用 `swapon -s` 或 `free -h` 命令检查是否启用成功。
4.  **配置开机自动挂载**：编辑 `/etc/fstab` 文件，添加一行配置，使系统启动时自动启用此交换分区。
    ```bash
    vim /etc/fstab
    ```
    在文件末尾添加如下行（可以复制已有的`swap`行进行修改）：
    ```
    /dev/vdb2 swap swap defaults 0 0
    ```
5.  **验证配置**：可以不重启，使用以下命令测试配置是否正确。
    ```bash
    swapoff /dev/vdb2       # 先停用
    swapon -a               # 根据/etc/fstab启用所有交换分区
    swapon -s               # 再次检查，应能看到/dev/vdb2
    ```
