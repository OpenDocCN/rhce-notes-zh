`fdisk` 命令用于查看磁盘信息和管理MBR格式的分区。其基本语法为：
```bash
fdisk [设备路径]
```
设备路径通常位于 `/dev/` 目录下，例如 `/dev/sdb`。

![](img/101108a2d07d1989399d2a89142364e9_33.png)

![](img/101108a2d07d1989399d2a89142364e9_35.png)

![](img/101108a2d07d1989399d2a89142364e9_37.png)

### `/dev/` 目录简介
在开始分区前，我们先了解一下 `/dev/` 目录。该目录存放了系统中所有硬件设备的文件。
以下是 `/dev/` 目录下常见的设备文件类型：
*   **硬盘设备**：通常以 `sd` 开头，如 `sda`, `sdb`。
*   **虚拟终端设备**：以 `tty` 开头，代表用户可以登录的终端。
*   **回环设备**：`loop*`，常用于挂载镜像文件。
*   **光驱设备**：`sr0` 或 `cdrom`。
*   **空设备与满设备**：`null`（黑洞设备，丢弃所有写入数据）和 `full`（满设备，拒绝所有写入数据）。
*   **随机数设备**：`random` 和 `urandom`，提供随机数。

![](img/101108a2d07d1989399d2a89142364e9_39.png)

![](img/101108a2d07d1989399d2a89142364e9_40.png)

### 使用 `fdisk` 进行分区
现在，我们以 `/dev/sdb` 硬盘为例进行分区。

1.  **进入分区界面**：
    ```bash
    fdisk /dev/sdb
    ```
    输入命令后，会进入交互式分区界面。此时所有操作仅保存在内存中，直到执行保存命令才会写入磁盘。

2.  **常用交互命令**：
    在 `fdisk` 交互界面中，可以输入以下命令：
    *   `m`：显示帮助菜单。
    *   `p`：打印当前磁盘的分区表，可查看分区格式（如 `DOS` 代表MBR）和现有分区。
    *   `n`：创建新分区。
    *   `d`：删除分区。
    *   `w`：保存分区表并退出。
    *   `q`：不保存更改并退出。

3.  **创建分区流程**：
    输入 `n` 创建新分区。
    *   **选择分区类型**：`p` 为主分区，`e` 为扩展分区。MBR最多允许4个主分区，或3个主分区+1个扩展分区。
    *   **指定分区编号**：通常使用默认值。
    *   **指定起始扇区**：通常使用默认值。
    *   **指定分区大小**：使用 `+大小` 格式，如 `+10G` 表示创建10GB的分区。

4.  **创建扩展分区与逻辑分区**：
    当需要超过4个分区时，必须使用扩展分区。扩展分区本身不存储数据，它像一个容器，可以在其中创建多个逻辑分区。
    *   创建扩展分区时，选择类型 `e`，并建议将剩余所有空间分配给它。
    *   在扩展分区内，使用 `n` 命令创建的分区即为逻辑分区，编号从5开始。

## 文件系统：数据的管家

分区创建后，还不能直接存储数据，必须先进行“格式化”，即创建文件系统。文件系统是管理分区内文件存储、属性和访问的机制。

![](img/101108a2d07d1989399d2a89142364e9_42.png)

![](img/101108a2d07d1989399d2a89142364e9_44.png)

![](img/101108a2d07d1989399d2a89142364e9_46.png)

### 常见文件系统类型
Linux中主要有两种常见的文件系统：

![](img/101108a2d07d1989399d2a89142364e9_48.png)

1.  **ext4**：
    *   是 `ext` 文件系统家族的第四代。
    *   支持日志功能，可避免系统意外宕机导致的数据丢失。
    *   单个文件系统最大支持 **1EB** 的分区容量。
    *   单个文件最大支持 **16TB**。

2.  **XFS**：
    *   目前CentOS 7/RHEL 7及以上版本的默认文件系统。
    *   同样具备日志功能，且数据恢复速度通常比ext4更快。
    *   单个文件系统最大支持 **8EB** 的分区容量。
    *   单个文件最大支持 **500TB**。
    *   被认为是当前主流的、高性能的文件系统。

![](img/101108a2d07d1989399d2a89142364e9_50.png)

### 格式化分区
使用 `mkfs` 命令为分区创建文件系统。
```bash
# 格式化为 ext4 文件系统
mkfs.ext4 /dev/sdb1

![](img/101108a2d07d1989399d2a89142364e9_52.png)

# 格式化为 XFS 文件系统
mkfs.xfs /dev/sdb1
```
格式化后，可以使用 `lsblk -f` 命令查看分区的文件系统类型。

![](img/101108a2d07d1989399d2a89142364e9_54.png)

## 挂载：使用分区的入口

在Linux中，用户不能直接访问分区设备文件（如 `/dev/sdb1`）。必须通过“挂载”操作，将分区关联到一个目录（挂载点），才能通过该目录访问分区内的存储空间。

![](img/101108a2d07d1989399d2a89142364e9_63.png)

![](img/101108a2d07d1989399d2a89142364e9_65.png)

### 挂载与卸载命令
*   **挂载命令 `mount`**：
    ```bash
    mount [设备路径] [挂载点目录]
    ```
    例如，将 `/dev/sdb1` 挂载到 `/mnt/data`：
    ```bash
    mount /dev/sdb1 /mnt/data
    ```
*   **卸载命令 `umount`**：
    ```bash
    umount [挂载点目录]
    ```
    或
    ```bash
    umount [设备路径]
    ```

### 挂载注意事项
1.  **挂载点通常为空目录**：如果挂载点目录非空，挂载后原有文件会被暂时隐藏，卸载后重新出现。
2.  **一对一关系**：一个分区不能同时挂载到多个目录，一个目录也不能同时挂载多个分区。
3.  **查看挂载信息**：使用 `df -h` 命令可以查看所有已挂载的文件系统及其使用情况。

### 物理分区的局限性
使用 `fdisk` 创建的MBR分区是物理分区。一旦分区空间被存满，无法直接扩容。解决方法通常是：
1.  卸载已满的分区。
2.  创建新的分区并格式化。
3.  将新分区挂载到需要的目录，继续存储数据。
4.  旧的数据仍保留在原分区中，如需访问，可将其挂载到另一个目录。
