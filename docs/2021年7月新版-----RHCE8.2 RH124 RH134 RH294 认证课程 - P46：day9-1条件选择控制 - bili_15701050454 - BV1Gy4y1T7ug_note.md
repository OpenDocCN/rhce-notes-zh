# Shell脚本编程：条件选择控制

## 概述
在本节课中，我们将学习Shell脚本编程中的条件选择控制。主要内容包括`if`条件判断语句和`case`条件选择语句的使用。通过学习，你将能够编写可以根据不同条件执行不同代码块的脚本。

## 回顾上周内容
上一节我们介绍了Shell脚本的基础知识，本节中我们来看看条件控制结构。

以下是上周学习内容的总结：
*   **脚本基础**：脚本文件后缀通常使用`.sh`。文件第一行需要指定解释器，例如 `#!/bin/bash`。脚本编写完成后，需要为文件设置执行权限，例如 `chmod +x 脚本文件`。
*   **执行方式**：可以通过相对路径（如 `./脚本文件`）或绝对路径（如 `/root/bin/脚本文件`）执行脚本。也可以使用解释器直接运行，例如 `bash 脚本文件`，这种方式无需文件具有执行权限。
*   **脚本测试**：可以使用 `bash -n 脚本文件` 测试脚本语法，使用 `bash -x 脚本文件` 逐步显示并执行脚本以检查运行情况。语法测试只能检查语法错误，不能检查命令逻辑是否正确。
*   **变量声明**：变量名可以使用英文大写或驼峰法，不能使用保留字。变量赋值可以是字符串或数字，例如 `K=hello`。Shell本身不支持浮点数运算。
*   **变量运算**：运算可以使用 `let` 命令、`$(( ))` 或 `$[ ]` 结构，例如 `let Z=X+Y` 或 `Z=$((X+Y))`。
*   **环境变量**：可以使用 `env` 或 `export` 命令查看环境变量。使用 `export K=V` 可以定义环境变量。
*   **条件测试**：测试可以使用 `test` 命令或 `[ ]`、`[[ ]]` 结构。`[[ ]]` 支持通配符和正则表达式。常用测试运算符包括：
    *   `-eq`：等于
    *   `-ne`：不等于
    *   `-gt`：大于
    *   `-ge`：大于等于
    *   `-lt`：小于
    *   `-le`：小于等于
    使用测试时，括号内必须留有空格，例如 `[ $X -eq $Y ]`。
*   **逻辑运算**：主要是短路与(`&&`)和短路或(`||`)。如果 `CMD1` 为真，则执行 `CMD2`；如果 `CMD1` 为假，则执行 `CMD2`。
*   **接收输入**：使用 `read` 命令接收用户输入并赋值给变量，例如 `read -p "提示信息" ANSWER`。

## 条件选择控制
### 1️⃣ if 条件判断语句
`if` 语句根据条件测试的结果来决定执行哪部分代码。其基本逻辑是：程序入口后，判断一个条件（例如 `A > B`），根据结果为真或假，选择不同的执行路径。

#### 单分支 if 语句
单分支结构只有两种情况：条件为真时执行代码块，条件为假时直接结束。

**语法格式**：
```bash
if [ 条件测试 ]; then
    # 条件为真时执行的命令
fi
```

![](img/fb63846813c6d95074d04b24767569d7_1.png)

![](img/fb63846813c6d95074d04b24767569d7_2.png)

![](img/fb63846813c6d95074d04b24767569d7_3.png)

**示例**：
```bash
#!/bin/bash
X=10
Y=8
if [ $X -gt $Y ]; then
    echo "X大于Y"
fi
```
如果 `X` 大于 `Y`，则输出“X大于Y”，否则脚本直接结束。

![](img/fb63846813c6d95074d04b24767569d7_5.png)

#### 双分支 if 语句
双分支结构在条件为真和为假时，分别执行不同的代码块。

![](img/fb63846813c6d95074d04b24767569d7_7.png)

**语法格式**：
```bash
if [ 条件测试 ]; then
    # 条件为真时执行的命令
else
    # 条件为假时执行的命令
fi
```

![](img/fb63846813c6d95074d04b24767569d7_9.png)

![](img/fb63846813c6d95074d04b24767569d7_11.png)

**示例**：
```bash
#!/bin/bash
X=10
Y=8
if [ $X -eq $Y ]; then
    echo "X等于Y"
else
    echo "X不等于Y"
fi
```

#### 多分支 if 语句
当存在多个条件需要判断时，可以使用 `elif` 来添加更多的分支。

![](img/fb63846813c6d95074d04b24767569d7_13.png)

**语法格式**：
```bash
if [ 条件测试1 ]; then
    # 条件1为真时执行的命令
elif [ 条件测试2 ]; then
    # 条件2为真时执行的命令
else
    # 所有条件都为假时执行的命令
fi
```

![](img/fb63846813c6d95074d04b24767569d7_15.png)

**示例**：
```bash
#!/bin/bash
X=10
Y=8
if [ $X -eq $Y ]; then
    echo "X等于Y"
elif [ $X -lt $Y ]; then
    echo "X小于Y"
else
    echo "X大于Y"
fi
```
建议为最后一个分支写上 `else`，这是一个良好的编程习惯。

#### if 语句的嵌套
`if` 语句可以嵌套使用，即在一个 `if` 或 `else` 的代码块中再包含另一个完整的 `if` 语句。

![](img/fb63846813c6d95074d04b24767569d7_17.png)

![](img/fb63846813c6d95074d04b24767569d7_19.png)

**示例**：
```bash
#!/bin/bash
X=18
Y=17
if [ $X -ge $Y ]; then
    if [ $X -eq $Y ]; then
        echo "X等于Y"
    else
        echo "X大于Y"
    fi
fi
```
这个脚本首先判断 `X` 是否大于等于 `Y`，如果是，再进一步判断是否等于 `Y`。注意，每个 `if` 都必须有一个对应的 `fi` 来结束。嵌套时应确保逻辑清晰，避免不必要的复杂嵌套。

### 2️⃣ case 条件选择语句
`case` 语句用于基于某个变量的值进行多重选择，特别适合编写菜单类脚本。其语法比多个 `if-elif` 更简洁。

![](img/fb63846813c6d95074d04b24767569d7_27.png)

**语法格式**：
```bash
case $变量 in
模式1)
    # 匹配模式1时执行的命令
    ;;
模式2)
    # 匹配模式2时执行的命令
    ;;
*)
    # 默认情况（匹配任何其他模式）时执行的命令
    ;;
esac
```
*   `$变量`：需要被匹配的变量。
*   `模式)`：可能的匹配值，支持通配符（如 `*` 匹配任意字符串，`?` 匹配单个字符，`[abc]` 匹配a、b、c中的单个字符）。
*   `;;`：每个分支结束的标记。
*   `*)`：默认分支，匹配所有未列出的情况。
*   `esac`：`case` 语句的结束标记（反写）。

**示例**：
```bash
#!/bin/bash
echo "请输入选择 (1, 2, 3):"
read -p "Your choice: " CHOICE

![](img/fb63846813c6d95074d04b24767569d7_29.png)

![](img/fb63846813c6d95074d04b24767569d7_31.png)

![](img/fb63846813c6d95074d04b24767569d7_33.png)

case $CHOICE in
1)
    echo "你选择了 Hello"
    ;;
2)
    echo "你选择了 Hi"
    ;;
3)
    echo "你选择了 Hey"
    ;;
*)
    echo "输入无效"
    ;;
esac
```
当用户输入1、2或3时，脚本会输出对应的信息；输入其他内容则提示无效。`case` 语句通常用于实现简单的命令行菜单。

## 总结
本节课我们一起学习了Shell脚本中两种重要的条件控制结构：
1.  **`if` 条件判断语句**：用于基于逻辑条件进行分支选择，包括单分支、双分支、多分支以及嵌套使用。
2.  **`case` 条件选择语句**：用于基于一个变量的具体值进行多重匹配选择，语法简洁，常用于菜单实现。
