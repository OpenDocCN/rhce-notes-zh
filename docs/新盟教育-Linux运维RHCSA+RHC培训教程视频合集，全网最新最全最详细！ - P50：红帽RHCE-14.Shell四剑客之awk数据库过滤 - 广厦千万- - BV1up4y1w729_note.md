## 完整语法结构与内置变量

![](img/e2510927add1b11c641aad70c88f7c05_9.png)

上一节我们介绍了 `awk` 的基本用法，本节中我们来看看 `awk` 完整的命令结构。一个完整的 `awk` 命令可以包含 `BEGIN`、中间处理块和 `END` 三个部分。

**公式**：
```
awk 'BEGIN{前置命令} 模式{动作} END{后置命令}' 文件名
```

*   `BEGIN{}`：在处理文件**之前**执行一次，常用于初始化或打印表头。
*   `模式{动作}`：对文件中**每一行**进行匹配和处理。
*   `END{}`：在处理完文件**所有行之后**执行一次，常用于打印汇总信息。

![](img/e2510927add1b11c641aad70c88f7c05_11.png)

以下是 `awk` 命令的一个完整示例：
```bash
awk 'BEGIN{print "用户名\t\t解释器"} /^root/ {print $1, $7} END{print "总行数:", NR}' /etc/passwd
```
*   `BEGIN` 块：先打印出“用户名”和“解释器”作为表头，`\t` 代表制表符用于对齐。
*   中间处理块：匹配以 `root` 开头的行，打印其第1列（用户名）和第7列（解释器）。
*   `END` 块：最后打印内置变量 `NR` 的值，即文件的总行数。

**注意**：`awk` 对命令中的空格要求不严格，但为了可读性，建议在 `print`、变量和动作之间添加空格。

### 监控系统剩余内存

![](img/e2510927add1b11c641aad70c88f7c05_23.png)

如果我们只想查看系统剩余的内存（free），使用 `grep` 会包含无关行，而 `awk` 可以精确到列。

![](img/e2510927add1b11c641aad70c88f7c05_25.png)

![](img/e2510927add1b11c641aad70c88f7c05_26.png)

![](img/e2510927add1b11c641aad70c88f7c05_28.png)

![](img/e2510927add1b11c641aad70c88f7c05_30.png)

**命令**：
```bash
free -h | awk '/Mem:/ {print "剩余内存:", $4}'
```
*   管道 `|` 将 `free -h` 的结果传递给 `awk`。
*   `/Mem:/` 是一个正则表达式，用于匹配包含“Mem:”的行。
*   `{print "剩余内存:", $4}` 打印匹配行的第4列，即剩余内存量。

### 监控网卡流量信息

![](img/e2510927add1b11c641aad70c88f7c05_42.png)

![](img/e2510927add1b11c641aad70c88f7c05_44.png)

![](img/e2510927add1b11c641aad70c88f7c05_46.png)

查看特定网卡（如 `ens32`）的入口流量时，`awk` 可以提取特定列。

![](img/e2510927add1b11c641aad70c88f7c05_48.png)

![](img/e2510927add1b11c641aad70c88f7c05_49.png)

![](img/e2510927add1b11c641aad70c88f7c05_51.png)

**命令**：
```bash
ifconfig ens32 | awk '/RX packets/ {print "入口流量:", $6, $7}'
```
*   匹配包含“RX packets”的行。
*   打印该行的第6和第7列，通常包含接收的数据包数量和字节数。

**进阶思考**：如果输出结果中包含不需要的括号 `()`，可以尝试通过指定分隔符来去除。例如，使用 `-F` 参数指定以 `[()]`（即左括号或右括号）作为分隔符，然后打印相应的列。这需要根据实际输出结构进行调整。

## 在脚本中集成awk进行系统监控

![](img/e2510927add1b11c641aad70c88f7c05_59.png)

![](img/e2510927add1b11c641aad70c88f7c05_61.png)

`awk` 常被用于Shell脚本中，实现自动化的系统监控。以下是一个监控根分区使用情况的脚本思路。

![](img/e2510927add1b11c641aad70c88f7c05_63.png)

**脚本示例**：
```bash
#!/bin/bash
while true; do
    # 使用awk提取根分区剩余空间（假设为第4列），并去除单位‘G’
    remaining=$(df -h | awk '/\/$/ {print $4}' | sed 's/G//')
    # 进行数值判断
    if [ $remaining -lt 5 ]; then
        echo "警告：根分区剩余空间不足5G！"
        # 此处可以添加报警命令，如发送邮件
    fi
    sleep 60 # 每分钟检查一次
done
```
**代码解释**：
1.  `df -h` 查看磁盘使用情况。
2.  `awk '/\/$/ {print $4}'` 匹配以根目录“/”结尾的行，并打印其第4列（可用空间）。
3.  `sed 's/G//'` 使用 `sed` 命令移除字符串中的“G”字符，以便进行数值比较。
4.  `if [ $remaining -lt 5 ]` 判断剩余空间是否小于5GB。
5.  `sleep 60` 让循环每秒暂停60秒，避免过度消耗CPU资源。

![](img/e2510927add1b11c641aad70c88f7c05_65.png)

**注意**：若 `df -h` 输出的信息包含单位（如G、M），直接进行数值比较会出错，需要先处理字符串（如用 `sed` 移除单位）。

## 开机自启与计划任务

![](img/e2510927add1b11c641aad70c88f7c05_69.png)

对于需要长期运行的监控脚本，可以将其设置为开机自动启动。

![](img/e2510927add1b11c641aad70c88f7c05_71.png)

**方法**：将执行脚本的命令（需使用绝对路径）添加到 `/etc/rc.d/rc.local` 文件中。
1.  首先，确保 `/etc/rc.d/rc.local` 文件本身有执行权限：`chmod +x /etc/rc.d/rc.local`
2.  然后，编辑该文件，在末尾添加你的脚本执行命令，例如：`/bin/bash /root/monitor.sh &`
3.  同时，也要确保你的监控脚本 `monitor.sh` 有执行权限。

![](img/e2510927add1b11c641aad70c88f7c05_73.png)

**提示**：更规范的管理方式是使用 **计划任务（cron）** 来定期执行脚本，这将在后续课程中讲解。

## 总结

本节课中我们一起学习了 `awk` 命令的进阶知识：
1.  **完整结构**：掌握了 `awk` 的 `BEGIN`、模式-动作、`END` 三部分结构及其执行顺序。
2.  **精确过滤**：学习了如何结合正则表达式，对命令输出结果进行列级的精确提取，这在监控内存、网络流量时非常有用。
3.  **脚本集成**：了解了如何将 `awk` 嵌入到Shell脚本中，结合循环和条件判断，实现自动化监控任务。
4.  **自启管理**：知道了如何让脚本在开机时自动运行。

`awk` 作为一门功能强大的文本处理语言，其语法远不止于此，还支持数组、循环等复杂编程特性。但对于日常运维工作，掌握以上内容已能解决绝大多数文本过滤和数据提取问题。重点是理解其“按行处理、按列提取”的核心思想，并能够灵活运用。
