### 查看文件与目录权限
以下是查看权限的常用命令：

![](img/ed7638415e4f2b26a7b04345af7767cf_21.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_23.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_25.png)

*   `ls -l`：以长格式列出当前目录下的文件和目录信息，包含权限详情。
*   `ls -ld <目录名>`：专门查看某个目录本身的权限信息，而不是其内容。
*   `ll`：这是 `ls -l` 命令的一个常用别名，效果相同。

![](img/ed7638415e4f2b26a7b04345af7767cf_27.png)

使用 `ls -l` 命令后，输出行的首部信息解析如下：
```
-rw-r--r--. 1 root root 0 Dec 10 14:30 example.txt
```
*   **第一个字符**：代表文件类型。`-` 表示普通文件，`d` 表示目录。
*   **后续九个字符**：代表权限，每三个一组。第一组是**文件拥有人（user）**的权限，第二组是**文件拥有组（group）**的权限，第三组是**其他人（other）**的权限。
*   **权限字符**：`r` 代表读，`w` 代表写/修改，`x` 代表执行。
*   **链接数**：本例中的数字 `1`。
*   **拥有人与拥有组**：本例中均为 `root`。
*   **文件大小与修改时间**：本例中文件大小为0字节，最后修改时间为12月10日14:30。

![](img/ed7638415e4f2b26a7b04345af7767cf_29.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_31.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_33.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_35.png)

关于文件时间，系统记录三种时间戳：
*   **访问时间（Access）**：最后一次读取文件的时间。
*   **修改时间（Modify）**：最后一次修改文件**内容**的时间。
*   **状态改变时间（Change）**：最后一次修改文件**元数据**（如权限、拥有人）的时间。

![](img/ed7638415e4f2b26a7b04345af7767cf_37.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_39.png)

`ls -l` 默认显示的是**修改时间**。要查看全部三个时间，可以使用 `stat <文件名>` 命令。

### 权限（RWX）的含义
权限 `r`、`w`、`x` 对于文件和目录的意义不同，这是理解权限的关键。

![](img/ed7638415e4f2b26a7b04345af7767cf_41.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_43.png)

**对于文件：**
*   **读（r）**：允许读取文件内容。对应命令如 `cat`， `more`。
*   **写（w）**：允许修改文件**内容**。**注意**：此权限**不包含**删除文件本身的权力。删除文件取决于其所在目录的权限。
*   **执行（x）**：允许将文件作为程序或脚本运行。在Linux中，文件是否可执行不依赖扩展名，而取决于是否有 `x` 权限。

![](img/ed7638415e4f2b26a7b04345af7767cf_45.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_47.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_49.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_51.png)

**对于目录：**
*   **读（r）**：允许读取目录内容列表，即使用 `ls` 命令查看目录下有什么。
*   **写（w）**：允许在目录内**创建、删除、重命名**文件或子目录。**注意**：即使你对目录内的某个文件没有 `w` 权限，只要拥有该目录的 `w` 权限，你仍然可以删除此文件。
*   **执行（x）**：允许进入（`cd`）该目录，并访问目录内的元数据。这是访问目录内任何文件的前提。

![](img/ed7638415e4f2b26a7b04345af7767cf_53.png)

### 权限判定流程
当系统判断一个用户对某文件/目录的权限时，遵循以下顺序：
1.  判断用户是否为文件的**拥有人（User）**。如果是，则应用第一组权限（`u`）。
2.  如果不是拥有人，则判断用户是否属于文件的**拥有组（Group）**。如果是，则应用第二组权限（`g`）。
3.  如果以上都不是，则用户被视为**其他人（Other）**，应用第三组权限（`o`）。

### 修改权限：`chmod` 命令
使用 `chmod` 命令可以修改文件或目录的权限。

**语法格式：**
```bash
chmod [ugoa][+-=][rwx] <文件或目录名>
```
*   **身份指定符**：`u`（拥有人），`g`（拥有组），`o`（其他人），`a`（所有人，即 `ugo` 的简写）。
*   **操作符**：`+`（添加权限），`-`（移除权限），`=`（直接设定权限，覆盖原有）。
*   **权限指定符**：`r`， `w`， `x`。

**示例：**
*   `chmod o+w file.txt`：为其他人添加对 `file.txt` 的写权限。
*   `chmod g-r file.txt`：从拥有组移除对 `file.txt` 的读权限。
*   `chmod a=rx file.txt`：为所有人（拥有人、组、其他）设定读和执行权限，取消写权限。

**递归修改权限（-R）：**
使用 `-R` 选项可以递归地修改目录及其内部所有子项（文件和子目录）的权限。
```bash
chmod -R o+w mydir/
```

**关于大写 X 权限：**
在递归设置权限时，使用小写 `x` 会使目录和文件都获得执行权限。对于文件来说，这可能带来安全风险（例如，意外使脚本可执行）。使用大写 `X` 可以避免此问题：它只会为目录和**已经拥有至少一个执行权限位**的文件设置执行权限。
```bash
chmod -R a+X mydir/ # 安全地递归添加“进入目录”和“可执行”权限
```

**数字表示法：**
权限也可以用三位八进制数字表示，每位数字是 `r`（4）、`w`（2）、`x`（1）的加和。
*   拥有人权限（第一位）
*   拥有组权限（第二位）
*   其他人权限（第三位）

**示例：**
*   `chmod 755 file`：等同于 `u=rwx，g=rx，o=rx`。
*   `chmod 644 file`：等同于 `u=rw，g=r，o=r`。

### 修改文件拥有人和组：`chown` 命令
`chown` 命令用于改变文件的拥有人和/或拥有组。通常需要管理员权限执行。

![](img/ed7638415e4f2b26a7b04345af7767cf_64.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_66.png)

**语法格式：**
```bash
chown [新拥有人][:新拥有组] <文件或目录名>
```
*   使用 `:` 或 `.` 来分隔拥有人和组名。
*   如果只改变组，可以省略拥有人部分，直接以 `:组名` 开头。

**示例：**
*   `chown student file.txt`：将 `file.txt` 的拥有人改为 `student`。
*   `chown :developers file.txt`：将 `file.txt` 的拥有组改为 `developers`。
*   `chown student:developers file.txt`：同时修改拥有人和拥有组。
*   `chown -R student:developers mydir/`：递归修改目录及其所有内容。

**仅修改组：`chgrp` 命令**
`chgrp` 命令专门用于修改文件的拥有组。
```bash
chgrp developers file.txt
```

### 进程的概念
进程是**正在运行的程序**的一个实例。一个静态的程序文件存储在硬盘上，当它被加载到内存中并由CPU执行时，就产生了一个进程。每个进程都有一个唯一的进程ID（PID）。

![](img/ed7638415e4f2b26a7b04345af7767cf_74.png)

进程可以通过多种方式产生：
1.  **执行一条命令**：如 `ls`， `cp`。这类进程通常瞬间完成。
2.  **运行一个脚本**：如执行Shell脚本。
3.  **启动一个服务**：如启动Apache Web服务器。这类进程通常长时间运行在后台，称为**守护进程**。

### 进程状态
在Linux的多任务环境中，进程有不同的状态。可以使用 `ps` 或 `top` 命令查看进程状态（`S` 列或 `STAT` 列）。

![](img/ed7638415e4f2b26a7b04345af7767cf_76.png)

常见的进程状态包括：
*   **R（Running）**：正在运行或在运行队列中等待运行。
*   **S（Interruptible Sleep）**：可中断的睡眠状态。进程在等待某个事件完成（如I/O操作），期间可以被信号唤醒。
*   **D（Uninterruptible Sleep）**：不可中断的睡眠状态。进程正在等待某些硬件I/O操作（如磁盘读写），在此期间**不响应任何信号**。这是一种保护机制，防止数据不一致。
*   **T（Stopped）**：进程被停止（挂起），通常是由作业控制信号（如 `Ctrl+Z`）或调试器导致。
*   **Z（Zombie）**：僵尸进程。进程已终止，但其父进程尚未回收它，导致其进程描述符仍残留在系统中。少量僵尸进程无害，但过多会占用系统资源。

### 查看进程
以下是查看进程的主要命令：

![](img/ed7638415e4f2b26a7b04345af7767cf_86.png)

**1. `top` 命令**
`top` 提供一个动态、实时的系统状态视图，包括进程信息。默认每3秒刷新一次。
*   **系统概览区**：显示系统运行时间、登录用户数、平均负载（1， 5， 15分钟）、任务总数、CPU使用率、内存使用情况等。
    *   **平均负载**：指单位时间内，系统处于**可运行状态**和**不可中断状态**的平均进程数。与CPU核心数相关，理想情况下应接近或低于CPU核心数。
*   **进程列表区**：显示各个进程的详细信息，如PID、用户、优先级（PR）、Nice值（NI）、虚拟内存（VIRT）、常驻内存（RES）、共享内存（SHR）、状态（S）、CPU占用率（%CPU）、内存占用率（%MEM）、运行时间（TIME+）、命令（COMMAND）。
*   **常用交互命令**：
    *   `P`：按CPU使用率排序。
    *   `M`：按内存使用率排序。
    *   `k`：终止指定PID的进程。
    *   `d`：更改刷新间隔。
    *   `1`：切换显示所有CPU核心的单独使用情况。
    *   `q`：退出 `top`。

![](img/ed7638415e4f2b26a7b04345af7767cf_88.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_90.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_92.png)

![](img/ed7638415e4f2b26a7b04345af7767cf_94.png)

**2. `ps` 命令**
`ps` 用于查看当前时刻的进程快照。
*   `ps aux`：查看系统所有进程的详细信息，包括CPU和内存使用。
*   `ps -ef`：查看系统所有进程，并显示父进程ID（PPID）。
*   `ps -e -o pid，ppid，cmd`：自定义输出列，只显示PID、PPID和命令。
*   `ps --sort=-%cpu`：按CPU使用率降序排序输出。

### 系统硬件信息查看（补充）
作为系统管理员，了解服务器硬件配置至关重要。以下是一些常用命令：

*   **查看硬盘信息**：`lsblk`
*   **查看内存信息**：`free -h`
*   **查看CPU信息**：`lscpu`
*   **查看网卡信息**：`ethtool <网卡名>` （如 `ethtool eth0`）
*   **查看内核启动日志（含硬件信息）**：`dmesg`
*   **从BIOS读取最底层的硬件信息**：`dmidecode` （通常需要root权限）
    *   `dmidecode -t 4`：查看处理器信息。
    *   `dmidecode -t 17`：查看内存设备信息。
