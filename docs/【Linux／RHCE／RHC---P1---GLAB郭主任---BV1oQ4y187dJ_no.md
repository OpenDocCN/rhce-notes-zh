## Linux文件系统结构

![](img/6d531603ef2d0c77b5de741b72037419_51.png)

![](img/6d531603ef2d0c77b5de741b72037419_53.png)

Linux的文件系统结构与Windows截然不同。它没有“C盘”、“D盘”这样的磁盘分区概念，而是采用一个以根目录（`/`）为起点的单一树状结构。所有文件和目录都从根目录开始分支。

![](img/6d531603ef2d0c77b5de741b72037419_65.png)

我们可以通过命令查看根目录下的内容：
```bash
ls -l /
```
输出结果中，每个条目的第一个字符表示其类型：
*   **`d`**：表示目录（Directory）。
*   **`-`**：表示普通文件（File）。
*   **`l`**：表示链接文件（Link），类似于Windows的快捷方式。

以下是根目录下一些关键二级目录及其作用的说明：

![](img/6d531603ef2d0c77b5de741b72037419_79.png)

![](img/6d531603ef2d0c77b5de741b72037419_81.png)

![](img/6d531603ef2d0c77b5de741b72037419_83.png)

![](img/6d531603ef2d0c77b5de741b72037419_85.png)

*   **`/boot`**：存放系统启动相关的文件，如内核（`vmlinuz-*`）和引导加载程序（`grub2`）的配置文件。
*   **`/dev`**：存放设备文件。Linux遵循“一切皆文件”的理念，硬件设备如磁盘（`/dev/vda`）、终端（`/dev/tty`）等都以文件形式在此目录下。
*   **`/etc`**：**最重要的目录之一**，存放系统和应用程序的**配置文件**。例如，网络配置、Yum软件源配置、用户信息等都位于此目录。
*   **`/home`**：普通用户的**家目录**。每个用户在此目录下有一个以用户名命名的子目录（如`/home/student`），用于存放个人文件和配置。
*   **`/root`**：系统管理员（root用户）的**家目录**。
*   **`/bin` 和 `/sbin`**：存放可执行命令（二进制文件）。`/bin` 供所有用户使用，`/sbin` 通常供系统管理员使用。
*   **`/lib` 和 `/lib64`**：存放系统库文件，这些是程序运行所依赖的共享函数库。
*   **`/usr`**：存放用户安装的应用程序、库文件、文档等，可以看作是“UNIX System Resources”的缩写。
*   **`/var`**：存放经常变化的（Variable）数据，如日志文件（`/var/log`）、邮件队列等。
*   **`/tmp`**：存放**临时文件**。系统会定期清理此目录下的内容。
*   **`/proc` 和 `/sys`**：**伪文件系统**，它们映射内核和进程信息，主要用于系统监控和调优，其内容存在于内存中而非磁盘。
*   **`/media` 和 `/mnt`**：**临时挂载点**，用于手动挂载移动存储设备（如U盘、光盘）或网络文件系统。

## 创建与删除目录

![](img/6d531603ef2d0c77b5de741b72037419_101.png)

创建目录使用 `mkdir` 命令。

![](img/6d531603ef2d0c77b5de741b72037419_103.png)

![](img/6d531603ef2d0c77b5de741b72037419_105.png)

![](img/6d531603ef2d0c77b5de741b72037419_107.png)

![](img/6d531603ef2d0c77b5de741b72037419_109.png)

![](img/6d531603ef2d0c77b5de741b72037419_111.png)

以下是创建目录时的注意事项和常用方法：
*   **命名规范**：目录名中不能包含 `/`（斜杠），因为它是路径分隔符。长度通常不超过255个字符，且严格区分大小写。
*   **创建单个目录**：`mkdir dir_name`
*   **创建多级目录**：使用 `-p` 参数，可以一次性创建路径中所有不存在的父目录。
    ```bash
    mkdir -p /path/to/new/dir
    ```
*   **批量创建目录**：可以使用花括号 `{}` 扩展功能。
    ```bash
    mkdir dir_{a,b,c}        # 创建 dir_a, dir_b, dir_c
    mkdir -p project/{src,doc,log} # 在project下创建三个子目录
    ```

![](img/6d531603ef2d0c77b5de741b72037419_113.png)

![](img/6d531603ef2d0c77b5de741b72037419_115.png)

![](img/6d531603ef2d0c77b5de741b72037419_117.png)

![](img/6d531603ef2d0c77b5de741b72037419_119.png)

删除目录使用 `rmdir` 或 `rm` 命令。

以下是删除目录的常用方法：
*   `rmdir dir_name`：仅能删除**空目录**。
*   `rm -r dir_name`：递归删除目录及其内部所有内容。`-r` 代表递归（Recursive）。
*   `rm -rf dir_name`：强制递归删除，不进行任何确认提示。**此命令非常危险，请谨慎使用**。

## 复制、移动与重命名文件

复制文件使用 `cp` 命令。

![](img/6d531603ef2d0c77b5de741b72037419_123.png)

![](img/6d531603ef2d0c77b5de741b72037419_125.png)

![](img/6d531603ef2d0c77b5de741b72037419_129.png)

![](img/6d531603ef2d0c77b5de741b72037419_131.png)

以下是 `cp` 命令的常见用法和重要参数：
*   **基本语法**：`cp [选项] 源文件 目标文件或目录`
*   **复制文件到文件**：`cp file1 file2` （创建file1的副本file2）
*   **复制文件到目录**：`cp file1 /some/dir/`
*   **复制多个文件到目录**：`cp file1 file2 file3 /some/dir/`
*   **复制目录**：必须使用 `-r`（递归）参数。
    ```bash
    cp -r /source/dir /destination/
    ```
*   **保留文件属性**：使用 `-p` 参数，可以在复制时保留文件的原始属性（如所有者、时间戳）。
*   **强制覆盖**：使用 `-f` 参数。

移动或重命名文件使用 `mv` 命令。在Linux中，重命名被视为移动的一种特殊形式。

![](img/6d531603ef2d0c77b5de741b72037419_133.png)

![](img/6d531603ef2d0c77b5de741b72037419_135.png)

![](img/6d531603ef2d0c77b5de741b72037419_137.png)

![](img/6d531603ef2d0c77b5de741b72037419_139.png)

以下是 `mv` 命令的常见用法：
*   **移动文件**：`mv file1 /another/dir/`
*   **重命名文件**：`mv old_name new_name`
*   **移动并重命名**：`mv file1 /another/dir/new_name`
*   **移动目录**：`mv dir1 /another/location/` （无需 `-r` 参数）

## 创建文件链接

![](img/6d531603ef2d0c77b5de741b72037419_151.png)

Linux支持两种类型的链接：**硬链接**和**软链接（符号链接）**。创建链接使用 `ln` 命令。

![](img/6d531603ef2d0c77b5de741b72037419_153.png)

![](img/6d531603ef2d0c77b5de741b72037419_155.png)

![](img/6d531603ef2d0c77b5de741b72037419_157.png)

以下是两种链接的区别和创建方法：
*   **软链接（Symbolic Link）**
    *   类似于Windows的快捷方式，是一个独立的文件，内容指向目标文件的路径。
    *   可以跨文件系统，也可以链接到目录。
    *   如果原始文件被删除，软链接将失效（成为“断链”）。
    *   创建命令：`ln -s 目标文件 链接文件`
        ```bash
        ln -s /etc/passwd my_link
        ```
*   **硬链接（Hard Link）**
    *   是目标文件在文件系统中的另一个“名字”。它与原始文件共享相同的inode和数据块。
    *   不能跨文件系统，也不能链接到目录。
    *   删除原始文件，只要还有硬链接存在，文件数据就不会被真正删除。
    *   创建命令：`ln 目标文件 链接文件`
        ```bash
        ln /etc/passwd my_hardlink
        ```
