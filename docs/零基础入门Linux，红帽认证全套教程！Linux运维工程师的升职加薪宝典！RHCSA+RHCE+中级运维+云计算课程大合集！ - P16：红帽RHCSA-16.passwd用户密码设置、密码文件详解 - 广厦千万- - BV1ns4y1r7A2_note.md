## 用户密码设置

![](img/1ccd59b25576c34d9062b187b24642b5_9.png)

![](img/1ccd59b25576c34d9062b187b24642b5_11.png)

为用户设置密码使用 `passwd` 命令。其基本格式非常简单。

### 命令格式与基本用法

![](img/1ccd59b25576c34d9062b187b24642b5_13.png)

`passwd` 命令的基本格式如下：
```bash
passwd [选项] [用户名]
```

![](img/1ccd59b25576c34d9062b187b24642b5_15.png)

![](img/1ccd59b25576c34d9062b187b24642b5_17.png)

![](img/1ccd59b25576c34d9062b187b24642b5_19.png)

![](img/1ccd59b25576c34d9062b187b24642b5_20.png)

如果不指定用户名，则修改当前登录用户自身的密码。

*   **root用户修改自身密码**：root用户拥有最高权限，修改自身密码时无需验证旧密码。
    ```bash
    # 直接输入命令，然后根据提示输入新密码即可
    passwd
    ```

*   **root用户为其他用户设置密码**：root用户可以为任何用户设置或修改密码，同样无需知道该用户的旧密码。
    ```bash
    # 为 user1 用户设置密码
    passwd user1
    ```

*   **普通用户修改自身密码**：普通用户只能修改自己的密码，并且必须首先输入正确的当前密码。
    ```bash
    # 以普通用户身份登录后，执行此命令
    passwd
    ```

### 密码复杂度规范

系统对密码有严格的复杂度要求，普通用户必须遵守，而root用户设置密码时系统仅作警告，但仍可强制设置。

以下是主要的密码规范：
1.  **密码长度**：不能少于8个字符。
2.  **密码复杂度**：应包含以下四类字符中的至少三类：
    *   数字（例如：0-9）
    *   小写字母（例如：a-z）
    *   大写字母（例如：A-Z）
    *   特殊字符（例如：!@#$%^&*等）
3.  **密码历史**：新密码不能与旧密码过于相似。

## 密码文件 `/etc/shadow` 详解

用户设置密码后，加密的密码信息并非存储在 `/etc/passwd` 中，而是存放在 `/etc/shadow` 文件里。这是一个高度敏感的文件，普通用户没有任何读取权限。

![](img/1ccd59b25576c34d9062b187b24642b5_37.png)

![](img/1ccd59b25576c34d9062b187b24642b5_39.png)

![](img/1ccd59b25576c34d9062b187b24642b5_41.png)

### 文件结构与字段含义

`/etc/shadow` 文件每一行对应一个用户的密码信息，由冒号 `:` 分隔为9个字段。

我们以 `user1` 用户的一行为例进行解读：
```
user1:$6$xyz...abc:19077:0:60:7:7:::
```

以下是每个字段的详细解释：

1.  **用户名**：用户的登录名。
2.  **加密密码**：使用加密算法（如SHA512）处理后的密码字符串。如果此字段为 `*` 或 `!!`，则表示该账户密码被锁定或未设置密码。
3.  **最后一次密码修改时间**：表示从1970年1月1日（Unix纪元）到最后一次修改密码所经过的天数。例如 `19077`。
4.  **密码修改最小间隔**：两次修改密码之间必须间隔的最小天数。`0` 表示随时可以修改。
5.  **密码有效期**：密码保持有效的最大天数。例如 `60` 表示密码60天后过期。默认值 `99999` 表示几乎永不过期。
6.  **密码到期前警告天数**：在密码到期前多少天开始向用户发出警告。例如 `7` 表示到期前7天会收到警告。
7.  **密码过期后宽限天数**：密码过期后，账户仍可登录的宽限天数。在此期间登录，系统会强制用户修改密码。如果为空，则表示过期后立即失效。
8.  **账户失效日期**：指定账户失效的绝对日期（同样以1970年以来的天数表示）。到达此日期后，账户将无法登录。
9.  **保留字段**：目前未使用，留作未来扩展。

### 管理密码策略

我们可以通过修改 `/etc/shadow` 文件中的特定字段或使用 `chage` 命令来实施密码策略。

*   **强制用户首次登录时必须修改密码**：将密码的最后修改时间（第3字段）设置为 `0`。这可以通过 `chage` 命令方便地实现。
    ```bash
    # 强制 user1 在下次登录时更改密码
    chage -d 0 user1
    ```
    用户首次使用初始密码登录后，系统会强制要求其设置新密码。

![](img/1ccd59b25576c34d9062b187b24642b5_43.png)

![](img/1ccd59b25576c34d9062b187b24642b5_45.png)

*   **设置密码有效期**：例如，要求用户每60天必须更改一次密码。
    ```bash
    # 设置 user1 的密码有效期为60天
    chage -M 60 user1
    ```

![](img/1ccd59b25576c34d9062b187b24642b5_47.png)

*   **查看用户的密码策略信息**：
    ```bash
    # 查看 user1 的密码过期等信息
    chage -l user1
    ```

## `passwd` 命令的常用选项

![](img/1ccd59b25576c34d9062b187b24642b5_51.png)

![](img/1ccd59b25576c34d9062b187b24642b5_53.png)

除了设置密码，`passwd` 命令还有一些有用的管理选项。

以下是 `passwd` 命令的几个常用选项：
*   `-S, --status`：查看用户的密码状态。
    ```bash
    passwd -S user1
    ```
    输出示例：`user1 PS 2022-03-26 0 60 7 7 (密码已设置，使用 SHA512 算法。)`。其中信息依次对应：用户名、密码设置日期、最小修改间隔、有效期、警告期、宽限期。
*   `-l, --lock`：锁定用户密码，禁止该用户登录。
    ```bash
    passwd -l user1
    ```
*   `-u, --unlock`：解锁用户密码。
    ```bash
    passwd -u user1
    ```
*   `-d, --delete`：删除用户密码，使其可以无密码登录（极不安全，慎用）。
    ```bash
    passwd -d user1
    ```
*   `--stdin`：从标准输入读取密码，用于非交互式脚本中设置密码。
    ```bash
    echo "MyNewPass123!" | passwd --stdin user1
    ```

## 切换用户身份：`su` 命令

有时我们需要临时切换到另一个用户身份执行操作，这时可以使用 `su` 命令。

`su` 命令用于切换当前用户身份。

*   **基本切换**：切换到指定用户，但环境变量和工作目录可能保持不变。
    ```bash
    su user1
    ```
*   **完全切换**：使用 `-` 或 `-l` 选项，会模拟该用户的完整登录环境，包括加载其profile文件和切换到其家目录。
    ```bash
    su - user1
    ```
*   **退出切换**：执行 `exit` 命令或按 `Ctrl+D` 可以退回到原来的用户身份。
    ```bash
    exit
    ```
**注意**：root用户切换到任何其他用户都无需输入密码。普通用户使用 `su` 切换时，则需要输入目标用户的密码。
