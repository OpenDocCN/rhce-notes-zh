# Ansible 课程：第12章：管理变量

在本节课中，我们将要学习 Ansible 中变量的概念和使用方法。变量是 Ansible 自动化中非常重要的组成部分，它允许我们存储和复用值，从而使 Playbook 更加灵活和易于维护。

## 回顾与概述

上一节我们介绍了 Ansible 临时命令和 Playbook 的基本编写。本节中我们来看看如何定义和使用变量。

![](img/28b93c6324ab226c0b6e9999b3b90e72_1.png)

Ansible 变量可用于存储各种值，例如用户名、要安装的软件包、要重启的服务或要删除的文件。变量名应具有描述性，建议以字母开头，可以包含字母、数字和下划线。

![](img/28b93c6324ab226c0b6e9999b3b90e72_3.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_4.png)

## 在命令行中定义变量

![](img/28b93c6324ab226c0b6e9999b3b90e72_6.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_8.png)

我们可以在执行 `ansible` 或 `ansible-playbook` 命令时通过 `-e` 选项来定义变量。这种方式定义的变量优先级最高。

以下是在临时命令中使用变量的示例，用于安装多个软件包：
```bash
ansible servera -m yum -a "name={{ PKG }} state=present" -e 'PKG="httpd,mariadb"'
```
需要注意的是，在临时命令中传递列表形式的变量值（如多个软件包）时，需要正确的引用格式。

## 在 Playbook 中定义变量

![](img/28b93c6324ab226c0b6e9999b3b90e72_16.png)

在 Playbook 中定义变量是更常见的做法。变量可以在 Play 级别、Task 级别或独立的变量文件中定义。

![](img/28b93c6324ab226c0b6e9999b3b90e72_18.png)

### Play 级别变量

在 `hosts` 字段下方使用 `vars` 关键字定义的变量，对整个 Play 生效。

例如，我们定义一个服务列表变量，并通过循环来管理这些服务：
```yaml
---
- name: Manage services
  hosts: servera
  vars:
    web_services:
      - httpd
      - firewalld
      - mariadb
  tasks:
    - name: Ensure services are running
      service:
        name: "{{ item }}"
        state: started
        enabled: yes
      loop: "{{ web_services }}"
```
在这个例子中，`loop` 关键字会遍历 `web_services` 列表，将每个值依次赋给 `{{ item }}` 变量，从而执行三次 Task。这极大地提高了代码的复用性和简洁性。

### 使用变量文件

当变量较多或需要在多个 Playbook 间共享时，可以将变量定义在独立的 YAML 文件中，并通过 `vars_files` 关键字引入。

首先，创建一个变量文件，例如 `vars.yml`：
```yaml
---
packages_to_install:
  - httpd
  - php
  - mariadb-server

![](img/28b93c6324ab226c0b6e9999b3b90e72_20.png)

services_to_manage:
  - httpd
  - mariadb
  - firewalld
```
然后在 Playbook 中引入并使用它：
```yaml
---
- name: Install packages and manage services
  hosts: servera
  vars_files:
    - vars.yml
  tasks:
    - name: Install multiple packages
      yum:
        name: "{{ item }}"
        state: present
      loop: "{{ packages_to_install }}"

    - name: Ensure services are running
      service:
        name: "{{ item }}"
        state: started
      loop: "{{ services_to_manage }}"
```
这种方式使 Playbook 结构更清晰，变量管理更集中。

## 主机与主机组变量

我们可以为特定的主机或主机组定义变量，这些定义通常放在清单文件或特定的目录结构中。

### 在清单文件中定义

![](img/28b93c6324ab226c0b6e9999b3b90e72_31.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_33.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_34.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_36.png)

变量可以直接写在 Ansible 清单文件中。

![](img/28b93c6324ab226c0b6e9999b3b90e72_38.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_40.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_42.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_44.png)

*   **主机变量**：直接跟在主机地址后面。
    ```
    servera ansible_port=2222
    ```
*   **主机组变量**：使用 `:vars` 为整个组定义。
    ```
    [webservers]
    servera
    serverb

    [webservers:vars]
    ntp_server=ntp.example.com
    ```

![](img/28b93c6324ab226c0b6e9999b3b90e72_46.png)

### 使用目录结构定义（推荐）

更优雅的方式是使用特定的目录结构。在清单文件所在目录（或由 `ANSIBLE_CONFIG` 指定的目录）下创建 `host_vars` 和 `group_vars` 目录。

![](img/28b93c6324ab226c0b6e9999b3b90e72_48.png)

*   `host_vars/servera.yml` 文件用于定义 `servera` 主机的变量。
*   `group_vars/webservers.yml` 文件用于定义 `webservers` 组的变量。

![](img/28b93c6324ab226c0b6e9999b3b90e72_50.png)

![](img/28b93c6324ab226c0b6e9999b3b90e72_52.png)

Ansible 会自动加载这些目录下与主机或组名同名的 YAML 文件中的变量。这种方式将变量与清单文件分离，更易于管理。

## 注册变量

![](img/28b93c6324ab226c0b6e9999b3b90e72_58.png)

注册变量是一个特殊功能，它允许我们将一个任务的执行结果保存到变量中，供后续任务使用。这是通过 `register` 关键字实现的。

![](img/28b93c6324ab226c0b6e9999b3b90e72_60.png)

以下是一个示例，它执行一个命令并注册结果，然后通过 debug 模块打印出来：
```yaml
---
- name: Test register variable
  hosts: servera
  tasks:
    - name: Run a command and register the output
      command: echo "Hello World"
      register: command_result

    - name: Display the registered variable
      debug:
        var: command_result
```
`debug` 模块的 `var` 参数用于打印变量的值。注册的变量通常是一个字典，包含了命令执行的详细信息，如返回值、标准输出、开始时间等。

![](img/28b93c6324ab226c0b6e9999b3b90e72_62.png)

## 变量优先级与引用

Ansible 变量有优先级顺序，命令行变量 `-e` 的优先级最高。在 Playbook 中引用变量时，需要使用双花括号 `{{ variable_name }}`。

![](img/28b93c6324ab226c0b6e9999b3b90e72_64.png)

当变量需要与字符串拼接时，必须将整个表达式用引号括起来：
```yaml
- name: Create a user
  user:
    name: "{{ username }}-admin"
```
在这个例子中，`username` 变量的值会与 “-admin” 字符串拼接。
