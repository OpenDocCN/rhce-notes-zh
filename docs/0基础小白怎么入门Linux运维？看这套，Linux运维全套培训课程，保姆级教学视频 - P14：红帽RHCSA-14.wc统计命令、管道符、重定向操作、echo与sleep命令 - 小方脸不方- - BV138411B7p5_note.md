在本节课中，我们将学习几个非常核心且实用的Linux命令和操作符。我们将从统计文件信息的`wc`命令开始，然后深入理解功能强大的管道符`|`，接着掌握将命令结果保存到文件的重定向操作，最后了解用于输出文本和延迟执行的`echo`与`sleep`命令。这些工具是日常运维和脚本编写的基础。

## wc统计命令 📈

![](img/a300140479d001446a70b8164b3e22bb_7.png)

`wc`命令用于统计文件中的行数、单词数和字节数。它非常常用，尤其是在需要快速了解文件规模时。

![](img/a300140479d001446a70b8164b3e22bb_9.png)

![](img/a300140479d001446a70b8164b3e22bb_11.png)

![](img/a300140479d001446a70b8164b3e22bb_13.png)

命令的基本格式是：
```bash
wc [选项] 文件名
```

以下是`wc`命令的主要选项：
*   `-l`：仅统计行数。
*   `-w`：仅统计单词数（以空格分隔的字符串）。
*   `-c`：仅统计字节数。

例如，要统计`/etc/passwd`文件的总行数，可以执行：
```bash
wc -l /etc/passwd
```
这条命令会输出该文件的行数，这是`wc`命令最常用的功能。

## 管道符 `|` 🔄

上一节我们介绍了如何用`wc`统计单个文件的信息。本节中我们来看看管道符，它能将多个命令串联起来，让前一个命令的输出作为后一个命令的输入，从而组合出强大的功能。

![](img/a300140479d001446a70b8164b3e22bb_15.png)

管道符是一个竖线`|`。它的作用是将命令A的输出结果，交给命令B作为参数去处理。

![](img/a300140479d001446a70b8164b3e22bb_17.png)

以下是管道符的一些应用示例：
*   查看`/etc/passwd`文件的前10行中的后5行：
    ```bash
    head -10 /etc/passwd | tail -5
    ```
    这里，`head -10`先取出文件前10行，然后通过管道`|`将结果传递给`tail -5`，后者再取出输入流的最后5行。
*   统计上述操作最终得到的行数：
    ```bash
    head -10 /etc/passwd | tail -5 | wc -l
    ```
    我们可以在一条命令中连接多个管道。
*   查看网卡`ens32`的配置信息，并只显示前两行（通常包含网卡名和IP地址）：
    ```bash
    ifconfig ens32 | head -2
    ```

管道符功能强大，允许无限连接，是Linux命令行中不可或缺的工具。

## 重定向操作 📥

管道符实现了命令之间的数据传递，而重定向操作则实现了命令与文件之间的数据传递。它可以将命令的输出结果保存到文件中，或者从文件中读取数据作为命令的输入。

![](img/a300140479d001446a70b8164b3e22bb_19.png)

![](img/a300140479d001446a70b8164b3e22bb_21.png)

重定向主要通过大于号`>`和小于号`<`来实现。

### 输出重定向：`>` 与 `>>`

![](img/a300140479d001446a70b8164b3e22bb_23.png)

![](img/a300140479d001446a70b8164b3e22bb_25.png)

一个大于号`>`表示覆盖重定向。它会将命令的输出结果写入指定文件，如果文件已存在，则**覆盖**原有内容；如果文件不存在，则创建该文件。

![](img/a300140479d001446a70b8164b3e22bb_27.png)

例如，将`ifconfig ens32`命令的前两行结果保存到`net_ens32.txt`文件中：
```bash
ifconfig ens32 | head -2 > net_ens32.txt
```
执行后，屏幕没有输出，因为结果被重定向到了文件里。使用`cat net_ens32.txt`可以查看文件内容。

![](img/a300140479d001446a70b8164b3e22bb_29.png)

![](img/a300140479d001446a70b8164b3e22bb_31.png)

两个大于号`>>`表示追加重定向。它也会将命令的输出结果写入指定文件，但如果文件已存在，新内容会**追加**到文件末尾，而不是覆盖。

![](img/a300140479d001446a70b8164b3e22bb_33.png)

例如，向`net_ens32.txt`文件追加当前内存使用情况：
```bash
free -h >> net_ens32.txt
```
此时，`net_ens32.txt`文件中既包含了之前的网卡信息，也包含了新追加的内存信息。

![](img/a300140479d001446a70b8164b3e22bb_35.png)

![](img/a300140479d001446a70b8164b3e22bb_37.png)

### 错误输出重定向：`2>` 与 `2>>`

![](img/a300140479d001446a70b8164b3e22bb_39.png)

默认情况下，`>`和`>>`只重定向命令的**标准正确输出**。如果命令执行出错，错误信息（标准错误输出）仍会显示在屏幕上。

![](img/a300140479d001446a70b8164b3e22bb_41.png)

例如，查看一个不存在的文件：
```bash
cat /etc/xxxooo.tt
```
这条命令会报错，提示文件不存在。如果使用`cat /etc/xxxooo.tt > error.txt`，错误信息并不会被写入`error.txt`，因为`>`不收集错误输出。

![](img/a300140479d001446a70b8164b3e22bb_43.png)

为了收集错误输出，需要使用`2>`（覆盖）或`2>>`（追加）。

例如，将上述错误信息保存到文件：
```bash
cat /etc/xxxooo.tt 2> error.txt
```
此时，错误信息被保存到了`error.txt`文件中。这在脚本中用于记录故障主机或错误日志非常有用。

![](img/a300140479d001446a70b8164b3e22bb_45.png)

![](img/a300140479d001446a70b8164b3e22bb_47.png)

### 合并重定向：`&>` 与 `&>>`

![](img/a300140479d001446a70b8164b3e22bb_49.png)

如果想将命令的**正确输出和错误输出同时重定向**到同一个文件，可以使用`&>`（覆盖）或`&>>`（追加）。

![](img/a300140479d001446a70b8164b3e22bb_51.png)

例如，执行一条可能对也可能错的命令，并将所有输出保存：
```bash
cat /etc/hostname /etc/abc.tt &> all_output.txt
```
这条命令尝试查看两个文件，第一个存在，第二个不存在。执行后，正确结果和错误信息都会被保存到`all_output.txt`中。

![](img/a300140479d001446a70b8164b3e22bb_53.png)

## echo与sleep命令 ⏳

![](img/a300140479d001446a70b8164b3e22bb_55.png)

最后，我们来学习两个看似简单但后期非常有用的命令：`echo`和`sleep`。

### echo命令

`echo`命令用于在终端输出指定的字符串或变量值。

![](img/a300140479d001446a70b8164b3e22bb_57.png)

命令格式很简单：
```bash
echo [字符串]
```
例如：
```bash
echo Hello World
```
这会在屏幕上输出`Hello World`。

![](img/a300140479d001446a70b8164b3e22bb_59.png)

`echo`结合重定向可以快速创建或修改文件内容。例如，快速设置主机名：
```bash
echo “new_hostname” > /etc/hostname
```
这条命令会将“new_hostname”写入`/etc/hostname`文件，覆盖原有内容。如果只执行`echo > filename`，则相当于清空该文件。

![](img/a300140479d001446a70b8164b3e22bb_61.png)

![](img/a300140479d001446a70b8164b3e22bb_63.png)

### sleep命令

![](img/a300140479d001446a70b8164b3e22bb_65.png)

`sleep`命令用于使当前进程（或脚本）延迟（休眠）一段时间再继续执行。

![](img/a300140479d001446a70b8164b3e22bb_67.png)

![](img/a300140479d001446a70b8164b3e22bb_69.png)

命令格式为：
```bash
sleep [时间]
```
时间单位可以是：
*   `s`：秒（默认）
*   `m`：分钟
*   `h`：小时
*   `d`：天

![](img/a300140479d001446a70b8164b3e22bb_71.png)

![](img/a300140479d001446a70b8164b3e22bb_73.png)

例如，让终端暂停5秒：
```bash
sleep 5
```
执行后，终端会等待5秒钟才返回提示符。

![](img/a300140479d001446a70b8164b3e22bb_75.png)

`sleep`在编写Shell脚本时尤其有用，可以控制任务执行节奏，避免CPU持续高负载运行。例如，在循环中每次处理都暂停0.1秒，可以显著降低CPU使用率：
```bash
while true; do echo “Processing...”; sleep 0.1; done
```
