# Linux运维：RHCE：红帽认证：云计算-2：Jumpserver堡垒机使用教程 🚀

![](img/1d077f6b0c22e558dc259229f30f6d99_1.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_3.png)

在本教程中，我们将系统性地学习Jumpserver堡垒机的核心功能与使用方法。Jumpserver是一款开源的堡垒机系统，能够集中管理大量服务器资产，并提供严格的权限控制与操作审计。我们将从用户管理、资产管理、权限配置到最终通过Web和SSH客户端（如Xshell）进行连接与审计，一步步构建完整的堡垒机管理体系。

---

## 概述 📋

![](img/1d077f6b0c22e558dc259229f30f6d99_5.png)

Jumpserver作为堡垒机，主要提供以下核心功能：
*   **集中资产管理**：统一管理成千上万台服务器。
*   **精细化权限控制**：定义用户、用户组、资产组以及它们之间的访问关系。
*   **操作审计与录像**：记录并回放所有用户的操作命令和会话，实现安全追溯。
*   **多协议支持**：支持通过Web浏览器或SSH客户端（如Xshell）进行安全连接。

接下来，我们将按照操作流程，逐一详解每个功能模块的配置。

## 用户与用户组管理 👥

![](img/1d077f6b0c22e558dc259229f30f6d99_7.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_9.png)

首先，我们需要创建用于登录Jumpserver的管理用户。Jumpserver的初始管理员账号为 `admin`。

### 创建用户组

1.  登录Jumpserver Web控制台（例如：`http://192.168.1.63:8000`）。
2.  导航至 **用户管理 -> 用户组**。
3.  点击 **添加用户组**。
4.  输入组名，例如：`王者荣耀华北区运维部`。
5.  点击 **提交**。

![](img/1d077f6b0c22e558dc259229f30f6d99_11.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_13.png)

### 创建Jumpserver用户

用户是登录堡垒机本身的账户。

1.  导航至 **用户管理 -> 查看用户**。
2.  点击 **添加用户**。
3.  填写用户信息：
    *   **用户名**：例如 `mk123`
    *   **姓名**：例如 `申建明`
    *   **用户组**：选择上一步创建的 `王者荣耀华北区运维部`
    *   **角色**：选择 `超级管理员`（根据实际权限分配）
    *   **邮箱**：填写有效的邮箱地址（**重要**：系统生成的初始密码将发送至此邮箱）
4.  点击 **提交**。

**关键点**：创建成功后，系统会自动生成随机密码并发送到用户邮箱。管理员不直接设置用户密码，增强了安全性。用户需使用邮件中的用户名和密码登录Jumpserver Web界面。

## 资产管理 🖥️

资产管理是指将需要被管理的服务器（资产）添加到Jumpserver中。

### 创建资产组（主机组）

将服务器按逻辑分组，便于批量授权和管理。

1.  导航至 **资产管理 -> 资产列表**。
2.  点击 **添加主机组**。
3.  输入组名，例如：`王者荣耀华北区服务器组`。
4.  点击 **提交**。

### 添加服务器资产

将具体的服务器信息录入系统。

![](img/1d077f6b0c22e558dc259229f30f6d99_15.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_17.png)

1.  在资产列表页面，点击 **添加资产**。
2.  填写服务器信息：
    *   **主机名**：例如 `game1`
    *   **IP**：填写服务器的真实IP地址，例如 `192.168.1.63`（公网服务器则填写公网IP）。
    *   **管理用户**：填写能通过SSH登录该服务器的账号，通常是 `root`。
    *   **密码**：填写上述管理用户的密码。
    *   **端口**：SSH端口，默认为 `22`。
    *   **资产组**：选择上一步创建的 `王者荣耀华北区服务器组`。
3.  点击 **激活**。Jumpserver会尝试用填写的账号密码连接该服务器，验证资产有效性。

![](img/1d077f6b0c22e558dc259229f30f6d99_19.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_21.png)

### 关联资产与机房（可选）

可以将资产关联到具体的物理机房信息中。

1.  导航至 **资产管理 -> 机房**。
2.  点击 **添加**，填写机房信息（如名称、带宽、联系人等）。
3.  添加资产后，在资产列表点击对应资产的 **编辑**，在 **机房** 字段中选择已创建的机房。

## 权限与系统用户管理 🔐

上一节我们添加了服务器资产，但Jumpserver用户（如 `mk123`）还不能直接管理它们。我们需要创建“系统用户”并授权。

### 理解系统用户

系统用户是**存储在Jumpserver上**，用于**连接被管理服务器**的凭证。它不是Jumpserver的登录用户，也不是直接在目标服务器上创建的用户（除非执行推送）。

**访问流程**：`Jumpserver用户 (mk123)` -> 登录Jumpserver -> 使用 `系统用户 (manager1)` -> 连接并管理 `目标服务器 (game1)`。

### 创建系统用户

1.  导航至 **授权管理 -> 系统用户**。
2.  点击 **添加系统用户**。
3.  填写信息：
    *   **用户名**：例如 `manager1`（这是在连接目标服务器时使用的用户名）。
    *   **密码**：例如 `123456`。
    *   **协议**：选择 `SSH`。
    *   **自动生成密钥**：可选。
    *   **Sudo**：这里关联的是**命令过滤器**，定义该用户在被管理服务器上能执行哪些命令。可以稍后配置。
4.  点击 **提交**。

![](img/1d077f6b0c22e558dc259229f30f6d99_23.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_25.png)

### 推送系统用户到资产

![](img/1d077f6b0c22e558dc259229f30f6d99_27.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_28.png)

创建的系统用户需要“推送”到目标服务器上，Jumpserver会在目标服务器上创建同名用户并配置好SSH密钥或密码。

![](img/1d077f6b0c22e558dc259229f30f6d99_30.png)

1.  在系统用户列表，找到刚创建的 `manager1`，点击其名称。
2.  在详情页，点击 **推送** 标签页。
3.  选择要推送到的资产或资产组（例如 `game1`）。
4.  点击 **推送**。
    *   **推送成功**：意味着目标服务器上已成功创建用户 `manager1`，并且可以通过该账号连接。
    *   **推送失败**：需检查资产信息中的管理账号密码是否正确，以及网络连通性。

![](img/1d077f6b0c22e558dc259229f30f6d99_32.png)

## 授权规则配置 ⚖️

![](img/1d077f6b0c22e558dc259229f30f6d99_40.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_42.png)

现在，我们有了Jumpserver用户 (`mk123`)、资产 (`game1`) 和连接资产的系统用户 (`manager1`)。授权规则就是将这三者关联起来，定义“谁”能用“什么身份”管理“哪些服务器”。

![](img/1d077f6b0c22e558dc259229f30f6d99_44.png)

1.  导航至 **授权管理 -> 授权规则**。
2.  点击 **添加规则**。
3.  填写授权信息：
    *   **名称**：描述性名称，例如 `运维人员mk123管理game1`。
    *   **用户**/**用户组**：选择Jumpserver用户 `mk123` 或他所在的用户组。
    *   **资产**/**资产组**：选择资产 `game1` 或它所在的资产组。
    *   **系统用户**：选择 `manager1`。
    *   **动作**：选择 `连接`。
4.  点击 **提交**。

![](img/1d077f6b0c22e558dc259229f30f6d99_46.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_48.png)

至此，用户 `mk123` 就获得了通过系统用户 `manager1` 管理服务器 `game1` 的权限。

## 连接与操作审计 📹

配置完成后，用户可以通过两种方式连接被管资产：Web终端和SSH客户端。

![](img/1d077f6b0c22e558dc259229f30f6d99_58.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_60.png)

### 通过Web终端连接与审计

![](img/1d077f6b0c22e558dc259229f30f6d99_62.png)

1.  用户 `mk123` 使用邮箱收到的密码登录Jumpserver Web界面。
2.  导航至 **工作台 -> 我的资产**，可以看到被授权的资产 `game1`。
3.  点击 **连接**，会打开一个Web SSH终端。
4.  在此终端执行的所有命令（如 `ls`, `vim`, 文件操作等）都会被Jumpserver完整记录。

**审计功能**：
*   **实时监控**：管理员可以在 **会话管理 -> 在线会话** 中实时查看用户的操作。
*   **命令记录**：所有执行的命令历史保存在 **日志审计 -> 命令记录** 中。
*   **会话录像**：用户断开连接后，完整的操作过程会生成录像。管理员可以在 **日志审计 -> 登录历史** 中找到会话并进行 **回放**，精确复现用户的操作屏幕。

![](img/1d077f6b0c22e558dc259229f30f6d99_64.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_66.png)

### 通过Xshell（SSH客户端）连接

![](img/1d077f6b0c22e558dc259229f30f6d99_68.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_70.png)

![](img/1d077f6b0c22e558dc259229f30f6d99_72.png)

对于习惯使用专业SSH客户端的用户，Jumpserver也提供了接入方式。

![](img/1d077f6b0c22e558dc259229f30f6d99_74.png)

1.  **获取SSH密钥**：
    *   用户登录Web界面后，在个人信息页面下载私钥文件。
    *   或者，在创建用户时，系统邮件中会包含SSH密钥的下载地址。
2.  **配置Xshell连接**：
    *   打开Xshell，**新建** 会话。
    *   **名称**：自定义，如 `Jumpserver`。
    *   **协议**：`SSH`。
    *   **主机**：Jumpserver服务器的IP地址，例如 `192.168.1.63`。
    *   **端口号**：`2222`（**注意**：Jumpserver的SSH服务端口默认为2222，不是Web的8000端口）。
3.  **身份验证**：
    *   在 **用户身份验证** 部分，**方法** 选择 `Public Key`。
    *   **用户名**：填写你的Jumpserver用户名，例如 `mk123`。
    *   **用户密钥**：点击 **浏览** -> **导入**，选择刚才下载的私钥文件。
4.  **连接与使用**：
    *   连接后，会进入Jumpserver的字符界面。
    *   输入 `p` 可以查看你有权管理的所有资产列表。
    *   输入资产IP的部分字符（如 `63`）或资产ID，可以直接登录到对应的服务器。
    *   通过此方式进行的操作，同样会被Jumpserver完整审计和录像。

## 总结 🎯

![](img/1d077f6b0c22e558dc259229f30f6d99_82.png)

本节课我们一起学习了Jumpserver堡垒机的核心使用方法：

![](img/1d077f6b0c22e558dc259229f30f6d99_84.png)

1.  **用户体系搭建**：创建了Jumpserver登录用户和用户组，密码通过邮件安全分发。
2.  **资产统一纳管**：将服务器作为资产添加到系统中，并可以关联机房信息。
3.  **权限核心配置**：
    *   理解了 **系统用户** 作为连接代理的关键概念。
    *   创建系统用户并将其推送到目标服务器。
    *   通过 **授权规则** 将Jumpserver用户、系统用户和资产进行绑定，实现精细化权限控制。
4.  **安全连接与审计**：
    *   掌握了通过 **Web终端** 和 **SSH客户端（如Xshell）** 两种方式安全连接被管资产。
    *   深入了解了Jumpserver强大的 **操作审计** 能力，包括命令记录、实时监控和会话录像回放，这是堡垒机保障安全、定责溯源的核心功能。

![](img/1d077f6b0c22e558dc259229f30f6d99_86.png)

通过以上步骤，你可以利用Jumpserver有效地管理服务器集群，实现权限集中管控和安全合规审计。对于拥有数十台以上服务器的环境，部署堡垒机是运维安全的最佳实践。
