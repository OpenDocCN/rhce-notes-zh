## 计算机硬件组成 🧩

上一节我们回顾了基础命令，本节中我们来看看支撑这些命令运行的计算机硬件。计算机硬件是物理设备的集合，主要分为输入设备、主机设备、输出设备和外部存储设备。

以下是主要的硬件组成部分：

![](img/5494bb5e12e81c46243eb74ae26e9425_5.png)

![](img/5494bb5e12e81c46243eb74ae26e9425_7.png)

![](img/5494bb5e12e81c46243eb74ae26e9425_9.png)

*   **输入设备**：用于向计算机输入数据和指令。例如：**鼠标**、**键盘**、触摸屏。
*   **主机设备**：计算机的核心计算单元，位于机箱内部。主要包括：
    *   **主板**：连接所有硬件的核心电路板。
    *   **CPU（中央处理器）**：负责执行程序指令和处理数据。
    *   **内存（RAM）**：临时存储CPU正在处理或即将处理的数据，断电后数据丢失。
    *   **网卡**、**声卡**、**显卡**：负责网络通信、音频处理和图形显示。
*   **输出设备**：用于将计算机处理的结果呈现给用户。例如：**显示器**、**耳机**、**打印机**、**投影仪**。
*   **外部存储设备**：用于长期保存数据。例如：**硬盘**、**光盘**、**U盘**。

### 核心硬件详解：CPU、内存与硬盘

在主机设备中，CPU、内存和硬盘的速度和处理数据的层次不同，可以这样理解：

*   **CPU**：处理速度最快，像**火箭**。它负责执行计算任务。
*   **内存**：速度次之，像**飞机**。它为正在运行的程序和数据提供临时的高速存储空间。
*   **硬盘**：速度相对最慢，像**绿皮火车**。它用于永久性存储操作系统、程序文件和个人数据。

三者关系可以概括为：程序从**硬盘**加载到**内存**中运行，**CPU**从**内存**中读取指令和数据进行计算。

#### 关于CPU的更多信息

![](img/5494bb5e12e81c46243eb74ae26e9425_11.png)

*   **厂商**：全球主要CPU厂商有两家：**Intel（英特尔）** 和 **AMD**。
*   **架构与位数**：常见的架构是 **x86** 和 **x86_64**。位数决定了CPU一次能处理的数据量，现代计算机多为 **64位**。
    *   32位CPU：一次可处理约3.2GB数据。
    *   64位CPU：一次可处理约128GB数据。
*   **核心**：CPU内部的计算单元。核心数越多，并行处理任务的能力越强。公式可以理解为：`CPU总处理能力 ≈ 单核性能 × 核心数量`。

## Linux系统目录结构介绍 🌳

Linux系统采用一种倒置的树形目录结构，所有文件和目录都从根目录（`/`）开始。理解每个目录的用途是进行有效系统管理的关键。

### 必须掌握的核心目录

![](img/5494bb5e12e81c46243eb74ae26e9425_19.png)

![](img/5494bb5e12e81c46243eb74ae26e9425_21.png)

![](img/5494bb5e12e81c46243eb74ae26e9425_22.png)

![](img/5494bb5e12e81c46243eb74ae26e9425_24.png)

以下是Linux系统中一些最重要和最常用的目录：

*   **`/` （根目录）**：整个文件系统的起点，所有其他目录和文件都挂载在其下。
*   **`/bin` 与 `/usr/bin`**：存放普通用户和系统管理员可执行的**二进制程序文件**（即大部分命令）。例如 `ls`、`cp`、`mv` 命令对应的文件就存放在这里。`/bin` 通常是 `/usr/bin` 的软链接。
    ```bash
    ls -l /bin
    ```
*   **`/boot`**：存放**系统启动**所需的文件，包括Linux内核（`vmlinuz-*`文件）、引导加载程序（如GRUB）的配置文件。
*   **`/dev`**：存放**设备文件**。在Linux中，硬件设备（如硬盘、键盘）也被抽象为文件。例如，第一块硬盘可能表示为 `/dev/sda`。
*   **`/etc`**：存放系统和应用程序的**配置文件**。这是管理员最常修改的目录之一，例如用户账户文件 `/etc/passwd`、网络配置等。
*   **`/home`**：普通用户的**家目录**。每个用户在此目录下有一个以用户名命名的子目录（如 `/home/student`），用于存放个人文件和配置。
*   **`/root`**：系统超级管理员 **root** 的家目录。
*   **`/var`**：存放经常变化的**可变数据**，如日志文件、邮件、数据库文件等。其中 `/var/log` 专门存放系统日志。
    ```bash
    # 查看系统日志
    tail -f /var/log/messages
    ```
*   **`/tmp`**：存放**临时文件**。所有用户都可读写，系统重启后可能会清空，**不要存放重要数据**。

### 需要了解的常用目录

![](img/5494bb5e12e81c46243eb74ae26e9425_26.png)

![](img/5494bb5e12e81c46243eb74ae26e9425_28.png)

以下目录在特定操作中会用到，也需要熟悉：

*   **`/lib` 与 `/lib64`**：存放系统**共享库文件**和**内核模块**，是许多程序运行所依赖的基础文件。
*   **`/opt`**：用于安装**第三方可选应用程序**的目录。例如，你可以将自行下载的软件安装在此处。
*   **`/usr`**：存放**用户级的程序和数据**，可以看作第二个主要的文件系统层次。其子目录结构（如 `/usr/bin`, `/usr/lib`, `/usr/local`）与根目录类似。
    *   **`/usr/local`**：系统管理员**本地安装软件**的首选位置，通常用于编译安装的软件。
*   **`/mnt` 与 `/media`**：系统提供的**挂载点**，用于临时挂载文件系统，如光盘、USB驱动器等。
*   **`/proc` 与 `/sys`**：**虚拟文件系统**，其内容不在硬盘上，而是映射自**内存**中的内核和进程信息。用于查看和（有限地）配置系统内核参数。
