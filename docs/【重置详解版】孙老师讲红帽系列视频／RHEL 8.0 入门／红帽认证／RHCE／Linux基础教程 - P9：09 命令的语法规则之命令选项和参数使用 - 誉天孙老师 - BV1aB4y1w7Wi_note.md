## 概述：命令的构成

Linux命令行的基本语法结构可以总结为：**命令 [选项] [参数]**。其中，命令是必须的，它指明了要执行的操作；选项用于修饰命令的行为；参数则是命令操作的具体对象。

![](img/1d051518d9dc620b7113690e6b00ca92_8.png)

上一节我们介绍了如何在提示符下执行简单的命令，本节中我们来看看命令更详细的语法构成。

![](img/1d051518d9dc620b7113690e6b00ca92_10.png)

## 命令 (Command)

![](img/1d051518d9dc620b7113690e6b00ca92_12.png)

命令是您想要执行的操作，它永远是命令行中的第一个单词。例如，`ls` 是查看文件的命令，`passwd` 是修改密码的命令。如果系统无法识别您输入的第一个单词，就会报“命令未找到”的错误。

![](img/1d051518d9dc620b7113690e6b00ca92_14.png)

![](img/1d051518d9dc620b7113690e6b00ca92_15.png)

**核心概念**：命令行会将输入的第一个单词识别为命令。
```bash
# 示例：`somecommand` 被识别为命令，但因为它不存在，所以报错。
$ somecommand arg1 arg2
-bash: somecommand: command not found
```

![](img/1d051518d9dc620b7113690e6b00ca92_16.png)

![](img/1d051518d9dc620b7113690e6b00ca92_17.png)

## 选项 (Option)

![](img/1d051518d9dc620b7113690e6b00ca92_19.png)

选项用于修饰或改变命令的默认行为。它通常以一个短横线 `-`（短选项）或两个短横线 `--`（长选项）开头。

**核心概念**：选项是命令的“副词”，用于指定命令的执行方式。
```bash
# 示例：`ls -l` 中的 `-l` 就是一个选项，它让 `ls` 命令以详细列表格式显示。
$ ls -l
```

![](img/1d051518d9dc620b7113690e6b00ca92_21.png)

### 短选项与长选项

*   **短选项**：通常由单个字母表示，前面加一个短横线 `-`。例如 `-l`, `-a`, `-h`。
*   **长选项**：通常是一个完整的单词，前面加两个短横线 `--`。例如 `--all`, `--help`。长选项的可读性更好。

![](img/1d051518d9dc620b7113690e6b00ca92_23.png)

![](img/1d051518d9dc620b7113690e6b00ca92_24.png)

以下是短选项和长选项的使用示例：
```bash
# 短选项示例：`-a` 显示所有文件（包括隐藏文件）
$ ls -a

# 长选项示例：`--all` 与 `-a` 功能相同
$ ls --all
```

### 多个选项的联合使用

![](img/1d051518d9dc620b7113690e6b00ca92_26.png)

您可以在一个命令中使用多个选项。短选项可以合并书写，也可以分开书写；长选项则必须分开书写。

![](img/1d051518d9dc620b7113690e6b00ca92_28.png)

以下是多个选项联合使用的规则：
*   短选项可以合并：`ls -l -a -h` 可以写成 `ls -lah`。
*   短选项分开时，必须用空格隔开。
*   长选项必须分开书写，不能合并：`ls --all --help`。

**核心概念**：短选项可合并，长选项需分开。
```bash
# 短选项合并示例
$ ls -lah

# 长选项分开示例
$ ls --all --help
```

![](img/1d051518d9dc620b7113690e6b00ca92_30.png)

![](img/1d051518d9dc620b7113690e6b00ca92_32.png)

## 参数 (Argument)

参数是命令操作的具体对象，例如文件名、目录名或用户名。它直接跟在命令或选项后面，前面不加 `-` 或 `--`。

![](img/1d051518d9dc620b7113690e6b00ca92_34.png)

![](img/1d051518d9dc620b7113690e6b00ca92_36.png)

**核心概念**：参数是命令作用的“宾语”。
```bash
# 示例：`ls /home` 中的 `/home` 就是一个参数，指定了 `ls` 命令要查看的目录。
$ ls /home
```

您也可以为命令指定多个参数，参数之间同样需要用空格隔开。
```bash
# 示例：同时查看 `/home` 和 `/etc` 两个目录
$ ls /home /etc
```

## 语法规则总结

![](img/1d051518d9dc620b7113690e6b00ca92_38.png)

![](img/1d051518d9dc620b7113690e6b00ca92_39.png)

命令、选项、参数之间的分隔至关重要。请记住以下规则：

![](img/1d051518d9dc620b7113690e6b00ca92_41.png)

![](img/1d051518d9dc620b7113690e6b00ca92_43.png)

**命令与选项、选项与选项、选项与参数、参数与参数之间，都必须用空格隔开。**

![](img/1d051518d9dc620b7113690e6b00ca92_45.png)

本节课中我们一起学习了Linux命令的基本语法结构。我们明确了命令、选项和参数三者的定义与区别：命令指明操作，选项修饰行为，参数指定对象。同时，我们掌握了短选项与长选项的使用、多个选项的联合方法以及使用空格分隔各组成部分的关键规则。理解并熟练运用这些语法规则，是您高效使用Linux命令行的重要一步。

## 扩展知识：使用分号连接命令

![](img/1d051518d9dc620b7113690e6b00ca92_51.png)

有时您可能希望在一行中依次执行多个独立的命令。这时，可以使用分号 `;` 来连接它们。

**核心概念**：分号连接的命令会按顺序执行，且彼此之间没有逻辑关联。即使前一个命令执行失败，后一个命令依然会尝试执行。
```bash
# 示例：先执行 `date` 显示日期，再执行 `cal` 显示日历
$ date; cal

![](img/1d051518d9dc620b7113690e6b00ca92_53.png)

![](img/1d051518d9dc620b7113690e6b00ca92_55.png)

# 即使第一个命令错误，第二个命令仍会执行
$ wrongcommand; date
```
