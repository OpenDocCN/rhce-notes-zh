## 回顾与过渡 🔄

上一节我们介绍了DNS主从服务器之间的正向及反向解析同步。本节中，我们将在此基础上，探讨如何通过TSIG加密来保障主从服务器间数据传输的安全性。

### TSIG加密配置

![](img/1a0972b44a0bb14b123f0843f3061c20_58.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_60.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_62.png)

在DNS查询链中，任何一次查询被篡改都可能导致后续所有结果错误。TSIG（Transaction Signature）通过在通信双方共享一个密钥字符串，为每次事务添加数字签名，从而验证数据来源的合法性和完整性。

![](img/1a0972b44a0bb14b123f0843f3061c20_64.png)

**配置步骤如下：**

1.  **在主服务器上生成密钥**：
    ```bash
    dnssec-keygen -a HMAC-MD5 -b 128 -n HOST master-slave
    ```
    此命令会生成两个文件（`.key` 和 `.private`），但TSIG不使用它们进行非对称加解密，仅需使用其中的密钥字符串。

![](img/1a0972b44a0bb14b123f0843f3061c20_66.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_68.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_70.png)

2.  **在主服务器配置文件中加载密钥**：
    编辑 `/etc/named.conf`，添加密钥定义并允许从服务器使用：
    ```bash
    key "master-slave" {
        algorithm hmac-md5;
        secret "生成的密钥字符串";
    };
    server 192.168.10.20 {
        keys { master-slave; };
    };
    ```
    然后，将密钥文件链接到配置目录并重启服务：
    ```bash
    chmod 640 Kmaster-slave.*
    ln -s /root/Kmaster-slave.* /etc/named/
    systemctl restart named
    ```

![](img/1a0972b44a0bb14b123f0843f3061c20_72.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_74.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_76.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_78.png)

3.  **在从服务器上配置相同密钥**：
    在从服务器的 `/etc/named.conf` 中，添加完全相同的密钥定义，并指定主服务器地址和密钥名称。
    ```bash
    key "master-slave" {
        algorithm hmac-md5;
        secret "生成的密钥字符串"; # 与主服务器相同
    };
    server 192.168.10.10 {
        keys { master-slave; };
    };
    ```
    重启从服务器的DNS服务后，加密通道即建立。双方通过匹配密钥字符串来验证身份，确保同步安全。

![](img/1a0972b44a0bb14b123f0843f3061c20_82.png)

## DNS缓存（转发）服务器配置 ⚡

![](img/1a0972b44a0bb14b123f0843f3061c20_96.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_98.png)

上一节我们确保了主从同步的安全，本节我们来看看如何通过配置缓存（转发）服务器来优化本地网络的DNS查询效率。

![](img/1a0972b44a0bb14b123f0843f3061c20_100.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_102.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_104.png)

缓存服务器通常部署在内网网关位置。当内网主机首次查询外部域名时，缓存服务器会向上游DNS（如 `114.114.114.114`）请求并将结果缓存。后续相同查询可直接由缓存响应，减少外部查询延迟和带宽消耗。

![](img/1a0972b44a0bb14b123f0843f3061c20_106.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_108.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_110.png)

**配置方法非常简单：**

![](img/1a0972b44a0bb14b123f0843f3061c20_112.png)

1.  准备一台能同时连接内网和外网的服务器（服务器A），以及一台仅连接内网的客户端（服务器B）。
2.  在服务器A上安装 `bind-chroot`，编辑 `/etc/named.conf`。
3.  在配置文件中，使用 `forwarders` 参数指定上游DNS服务器地址即可。
    ```bash
    options {
        forwarders { 8.8.8.8; }; # 指定谷歌DNS作为上游
    };
    ```
4.  将客户端（服务器B）的DNS地址指向服务器A的内网IP。此后，客户端的所有DNS查询都将由服务器A转发，并且结果会被缓存以加速后续相同查询。

![](img/1a0972b44a0bb14b123f0843f3061c20_116.png)

## DNS分离解析技术 🌍

![](img/1a0972b44a0bb14b123f0843f3061c20_132.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_134.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_136.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_138.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_140.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_142.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_144.png)

缓存服务器优化了查询速度，而分离解析技术则能根据用户来源提供不同的解析结果，实现负载均衡和就近访问，这是CDN（内容分发网络）的核心技术之一。

![](img/1a0972b44a0bb14b123f0843f3061c20_146.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_148.png)

其原理是：DNS服务器根据查询请求的源IP地址所属网段，为其调用不同的区域数据配置文件，从而返回不同的解析结果。

![](img/1a0972b44a0bb14b123f0843f3061c20_150.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_152.png)

**配置示例：**

假设公司网站在北京和美国都有服务器，希望中国用户访问北京IP，美国用户访问美国IP。

![](img/1a0972b44a0bb14b123f0843f3061c20_154.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_156.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_158.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_160.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_162.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_164.png)

1.  **在主配置文件中定义ACL和视图**：
    编辑 `/etc/named.conf`，定义两个地址列表（ACL）并创建对应的视图（view）。
    ```bash
    acl "china" { 122.71.115.0/24; }; # 中国IP段
    acl "america" { 106.185.25.0/24; }; # 美国IP段

    view "china" {
        match-clients { "china"; };
        zone "linuxprobe.com" {
            type master;
            file "linuxprobe.com.china"; # 中国用户使用的数据文件
        };
    };

    view "america" {
        match-clients { "america"; };
        zone "linuxprobe.com" {
            type master;
            file "linuxprobe.com.america"; # 美国用户使用的数据文件
        };
    };
    ```

![](img/1a0972b44a0bb14b123f0843f3061c20_166.png)

2.  **创建不同的区域数据文件**：
    分别创建 `linuxprobe.com.china` 和 `linuxprobe.com.america` 文件。
    *   在 `.china` 文件中，将 `www.linuxprobe.com` 解析为北京服务器的IP（如 `122.71.115.1`）。
    *   在 `.america` 文件中，将同一域名解析为美国服务器的IP（如 `106.185.25.1`）。

![](img/1a0972b44a0bb14b123f0843f3061c20_168.png)

3.  **验证效果**：
    *   当源IP为 `122.71.115.88`（中国）的用户查询 `www.linuxprobe.com` 时，将获得IP `122.71.115.1`。
    *   当源IP为 `106.185.25.88`（美国）的用户查询同一域名时，将获得IP `106.185.25.1`。

![](img/1a0972b44a0bb14b123f0843f3061c20_170.png)

![](img/1a0972b44a0bb14b123f0843f3061c20_172.png)

这就实现了基于用户位置的智能解析，提升访问体验。

## 课程总结 📚

本节课中我们一起学习了DNS服务的三项高级功能：
1.  **TSIG加密**：通过共享密钥字符串，保障DNS主从服务器之间区域数据传输的安全性。
2.  **缓存（转发）服务器**：通过指定上游DNS并缓存结果，显著提升内网域名解析速度，减少外部查询。
3.  **分离解析**：根据查询者的源IP地址，返回不同的解析结果，是实现CDN和网络负载均衡的关键技术。
