## 解决视频访问问题 🎥

![](img/b23ad675e028b5115c2aa780706ac668_5.png)

上一节我们介绍了项目部署的整体架构，本节中我们来看看如何解决学习中心界面的数据加载问题。首先需要解决的是视频资源的访问问题。

在实际的学成在线项目中，视频由独立的媒体服务器提供访问。但在当前测试环境中，我们将通过门户服务来访问视频资源。

![](img/b23ad675e028b5115c2aa780706ac668_7.png)

![](img/b23ad675e028b5115c2aa780706ac668_8.png)

我们创建了一个独立的域名 `X-video-cloud` 来访问视频。在门户服务的Nginx配置中，已经设置了 `/video/` 路径指向视频目录。

![](img/b23ad675e028b5115c2aa780706ac668_9.png)

```nginx
location /video/ {
    # ... 配置详情
}
```

因此，只要将视频文件放置在门户服务容器内对应的 `/video/` 目录下，即可通过该域名路径访问。

以下是操作步骤：

![](img/b23ad675e028b5115c2aa780706ac668_11.png)

1.  将云服务器 `root` 目录下解压好的 `video` 文件夹，复制到门户服务容器映射的主机目录中（例如 `X` 目录）。
2.  同时，将 `static` 文件夹下的 `course` 和 `category` 子目录也复制到门户服务的主目录下，用于存放课程相关的静态资源（如图片）。
3.  视频将通过 `X-video-cloud` 域名访问，静态资源将通过门户服务自身的域名访问（例如 `/static/course/` 和 `/static/category/`）。

![](img/b23ad675e028b5115c2aa780706ac668_13.png)

完成以上步骤后，视频和图片等静态资源的访问路径就配置好了。

## 配置前端API请求地址 🔧

![](img/b23ad675e028b5115c2aa780706ac668_18.png)

解决了静态资源访问后，接下来需要让前端能够从微服务获取动态数据。当前学习中心页面会向用户中心和学习服务发起API请求，但这些请求地址目前不正确。

![](img/b23ad675e028b5115c2aa780706ac668_20.png)

我们需要修改门户服务的Nginx配置文件，将特定的API请求代理到微服务网关。

![](img/b23ad675e028b5115c2aa780706ac668_22.png)

以下是需要修改的API路径及其对应的网关路由：

![](img/b23ad675e028b5115c2aa780706ac668_24.png)

![](img/b23ad675e028b5115c2aa780706ac668_26.png)

1.  **搜索相关请求 (`/openapi/search/...`)**
    此请求应被转发至网关中配置的搜索服务路由。

2.  **学习进度请求 (`/openapi/learn/...`)**
    此请求应被转发至网关中配置的学习服务路由。在网关配置中，`/openapi/` 前缀的请求默认会路由到对应的微服务。

3.  **门户视图请求 (`/openapi/portalview/...`)**
    此请求同样可以通过网关的默认路由进行转发。

具体的修改方法是，在门户的Nginx配置文件中，找到对应的 `location` 配置块，将 `proxy_pass` 指令的值改为微服务网关的地址，并拼接上相应的路径前缀。

例如，将原本指向用户中心的地址：
```nginx
proxy_pass http://user-center;
```
修改为指向网关的地址：
```nginx
proxy_pass http://gateway-server:port/openapi/;
```

## 应用配置并验证 ✅

配置文件修改完成后，需要将其更新到服务器并重启门户服务容器以使配置生效。

![](img/b23ad675e028b5115c2aa780706ac668_28.png)

![](img/b23ad675e028b5115c2aa780706ac668_30.png)

操作流程如下：

![](img/b23ad675e028b5115c2aa780706ac668_32.png)

1.  将修改好的Nginx配置文件上传到云服务器，覆盖原有配置。
2.  进入华为云容器实例控制台，找到门户服务的实例。
3.  停止并删除当前实例，系统会自动根据最新镜像和配置创建一个新的实例。
4.  等待新实例启动成功。

![](img/b23ad675e028b5115c2aa780706ac668_34.png)

服务重启后，返回到学习中心页面进行刷新。此时，页面右侧的课程目录数据应能成功从微服务加载。点击任意课程章节，视频播放器也应能正常加载并播放视频。

至此，学习中心的前端界面与后端微服务已成功集成，动态数据和静态资源均可正常访问。

![](img/b23ad675e028b5115c2aa780706ac668_36.png)
