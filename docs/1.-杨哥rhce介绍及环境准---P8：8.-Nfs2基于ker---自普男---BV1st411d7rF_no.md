# RHCE课程：1.8：基于Kerberos安全认证的NFS共享配置 🔐

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_0.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_1.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_3.png)

在本节课中，我们将学习如何配置一个使用Kerberos进行安全认证的NFS共享。与传统的基于IP或主机的访问控制不同，Kerberos提供了基于用户的身份验证，确保只有经过验证的用户才能访问共享资源。我们将从服务器端配置开始，逐步完成客户端的挂载与验证。

## 概述

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_5.png)

本节教程将指导你完成一个安全的NFS共享配置。核心任务包括：在服务器端配置Kerberos密钥并导出共享目录；在客户端获取密钥并挂载共享；最后，验证特定用户能否通过Kerberos认证成功访问共享资源。整个过程涉及服务配置、防火墙规则（沿用之前设置）和用户票据管理。

---

## 服务器端配置

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_7.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_9.png)

上一节我们介绍了常规NFS共享，本节中我们来看看如何为其增加Kerberos安全层。首先需要在NFS服务器上进行一系列配置。

### 1. 下载Kerberos密钥表（Keytab）

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_11.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_13.png)

服务器需要密钥文件来进行安全通信。使用 `wget` 命令从指定URL下载，并保存到 `/etc/krb5.keytab`。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_15.png)

**关键点**：必须使用大写的 `-O` 选项来指定输出文件。若使用小写 `-o`，则会将下载日志而非密钥文件保存到目标路径，这是一个常见错误。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_17.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_18.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_19.png)

以下是正确的下载命令：
```bash
wget -O /etc/krb5.keytab http://server.example.com/path/to/keytab
```
如果系统没有 `wget`，请先安装：
```bash
yum install wget -y
```

### 2. 配置NFS服务使用NFSv4.2协议

编辑NFS服务的配置文件，启用NFSv4.2协议以支持更高级的特性（包括安全认证）。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_21.png)

打开 `/etc/sysconfig/nfs` 文件，找到 `RPCNFSDARGS` 参数所在行，确保其包含 `-V 4.2`。
```bash
RPCNFSDARGS="-V 4.2"
```

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_23.png)

### 3. 配置NFS共享导出

配置 `/etc/exports` 文件，以读写方式共享 `/nfs2` 目录给指定网段，并强制使用Kerberos v5隐私安全模式（`krb5p`）。

在 `/etc/exports` 文件中添加如下行：
```bash
/nfs2 172.25.0.0/24(rw,sync,sec=krb5p)
```
**公式说明**：`sec=krb5p` 表示使用Kerberos v5进行身份验证，并对数据传输进行加密（隐私模式）。

### 4. 启动相关服务

配置完成后，需要启动NFS服务器及其安全服务，并设置为开机自启。

以下是需要执行的服务管理命令：
```bash
systemctl restart nfs-server
systemctl enable nfs-server
systemctl restart nfs-secure-server
systemctl enable nfs-secure-server
```

## 客户端配置

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_28.png)

服务器配置完成后，我们转向客户端。客户端需要获取自己的密钥，并以安全方式挂载NFS共享。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_30.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_32.png)

### 1. 下载客户端Kerberos密钥表

与服务器类似，客户端也需要自己的密钥文件。使用 `wget` 命令下载并保存到相同路径。

执行以下命令：
```bash
wget -O /etc/krb5.keytab http://server.example.com/path/to/client/keytab
```

### 2. 配置挂载

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_34.png)

创建挂载点 `/mnt/nfssecure`，并在 `/etc/fstab` 中配置自动挂载。挂载选项需指定NFSv4.2协议和与服务器匹配的安全模式（`sec=krb5p`）。

在 `/etc/fstab` 文件中添加如下行：
```bash
server.example.com:/nfs2 /mnt/nfssecure nfs defaults,v4.2,sec=krb5p 0 0
```
**注意**：`v4.2` 中的 `v` 为小写，这与服务器端配置文件中的大写 `V` 不同。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_36.png)

### 3. 启动客户端安全服务并挂载

客户端需要启动特定的安全服务才能支持Kerberos认证的挂载。

以下是客户端的服务启动与挂载命令：
```bash
systemctl start nfs-secure
systemctl enable nfs-secure
mount -a
```
执行 `mount` 命令检查挂载是否成功，应能看到类似如下输出，包含 `v4.2` 和 `sec=krb5p`：
```bash
server.example.com:/nfs2 on /mnt/nfssecure type nfs4 (rw,relatime,vers=4.2,sec=krb5p,...)
```

## 验证与测试

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_40.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_41.png)

配置和挂载都完成后，最关键的一步是验证。普通用户必须通过Kerberos认证获取票据（Ticket），才能访问安全共享。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_42.png)

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_44.png)

### 1. 为用户获取Kerberos票据

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_46.png)

用户 `ldapuser5` 需要验证身份以获取访问票据。不能简单地使用 `su` 切换用户，因为那不会触发Kerberos密码验证。

推荐使用以下两种方法之一来获取票据：
*   **方法一：使用 `ssh` 登录本机**。这会提示输入用户的Kerberos密码（例如 `kerberos`）。
    ```bash
    ssh ldapuser5@localhost
    ```
*   **方法二：先切换用户，再初始化票据**。切换后，使用 `kinit` 命令并输入密码来获取票据。
    ```bash
    su - ldapuser5
    kinit
    ```

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_48.png)

### 2. 验证票据与访问权限

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_50.png)

获取票据后，可以查看票据列表，并测试在共享目录中创建文件。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_52.png)

执行以下命令进行验证：
```bash
# 查看当前用户的Kerberos票据
klist

# 切换到共享目录下的private子目录并创建测试文件
cd /mnt/nfssecure/private
touch testfile_ldapuser5.txt
```
如果文件创建成功，并且可以在NFS服务器的 `/nfs2/private/` 目录下看到该文件，则证明整个基于Kerberos的NFS安全共享配置成功。

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_54.png)

## 总结

![](img/4b3a1009aa0acdb45a6fcf3040ec7cab_56.png)

本节课中我们一起学习了如何配置基于Kerberos认证的安全NFS共享。我们回顾了核心步骤：在服务器端下载密钥、配置NFSv4.2和`krb5p`安全导出；在客户端下载密钥、配置挂载并启动安全服务；最后，指导用户通过`ssh`或`kinit`获取Kerberos票据以验证访问权限。整个过程的关键在于注意细节，如`wget`命令的`-O`选项大小写、服务器与客户端所需启动的服务不同（`nfs-secure-server` vs `nfs-secure`），以及用户必须通过认证获取票据才能访问资源。
