## 重定向：控制数据的流向

上一节我们介绍了命令的基本执行。本节中我们来看看如何改变命令输入和输出的默认目的地，这就是重定向。

在Linux中，计算机的输入设备（如键盘）和输出设备（如显示器）在命令行中有对应的抽象概念：
*   **标准输入 (STDIN)**： 通常指键盘输入，用文件描述符 `0` 表示。
*   **标准输出 (STDOUT)**： 命令执行后的正常输出，显示在屏幕上，用文件描述符 `1` 表示。
*   **错误输出 (STDERR)**： 命令执行出错时的提示信息，也显示在屏幕上，用文件描述符 `2` 表示。

默认情况下，标准输出 (`1`) 可以省略不写，但错误输出 (`2`) 必须明确指定。

![](img/98883775de87108044112c1087a69b2c_15.png)

重定向就是使用 `>`、`<`、`>>` 等符号，改变这些数据流的默认方向。

![](img/98883775de87108044112c1087a69b2c_17.png)

![](img/98883775de87108044112c1087a69b2c_19.png)

### 输出重定向

![](img/98883775de87108044112c1087a69b2c_21.png)

![](img/98883775de87108044112c1087a69b2c_22.png)

输出重定向是最常用的操作，它可以将原本输出到屏幕的内容，导入到一个文件中。

![](img/98883775de87108044112c1087a69b2c_24.png)

以下是输出重定向的几种常见用法：

![](img/98883775de87108044112c1087a69b2c_26.png)

*   **覆盖写入文件**： `命令 > 文件`
    *   将命令的**标准输出**覆盖写入到指定文件。如果文件不存在则创建，存在则清空原有内容。
    *   示例：`echo “Hello World” > test.txt` 会将 “Hello World” 写入 `test.txt`。
*   **错误信息写入文件**： `命令 2> 文件`
    *   将命令的**错误输出**覆盖写入到指定文件。
    *   示例：`ls /nonexistent 2> error.log` 会将 “ls: cannot access…” 这样的错误信息写入 `error.log`。
*   **追加写入文件**： `命令 >> 文件`
    *   将命令的**标准输出**追加到指定文件的末尾。
    *   示例：`echo “Another line” >> test.txt` 会在 `test.txt` 文件末尾新增一行。
*   **错误信息追加写入**： `命令 2>> 文件`
    *   将命令的**错误输出**追加到指定文件的末尾。
*   **混合输出到文件**： `命令 &> 文件` 或 `命令 > 文件 2>&1`
    *   将命令的**标准输出和错误输出同时**覆盖写入同一个文件。这是一种常用格式。
    *   示例：`ls /etc /nonexistent &> all_output.log` 会将正确列表和错误信息都保存到日志文件。

![](img/98883775de87108044112c1087a69b2c_28.png)

![](img/98883775de87108044112c1087a69b2c_30.png)

**注意**：`&>` 是一个整体，不能写成 `2>&1>`，这是错误的语法。

![](img/98883775de87108044112c1087a69b2c_32.png)

### 输入重定向

![](img/98883775de87108044112c1087a69b2c_34.png)

![](img/98883775de87108044112c1087a69b2c_36.png)

输入重定向相对较少使用，它可以将文件内容作为命令的输入。

![](img/98883775de87108044112c1087a69b2c_37.png)

![](img/98883775de87108044112c1087a69b2c_39.png)

![](img/98883775de87108044112c1087a69b2c_41.png)

*   **从文件读取输入**： `命令 < 文件`
    *   将文件内容作为命令的标准输入。
    *   示例：`wc -l < /etc/passwd` 会统计 `/etc/passwd` 文件的行数，但不会在结果中显示文件名。
*   **多行输入（Here Document）**： `命令 << 结束标记`
    *   允许在命令行中直接输入多行文本，直到遇到指定的“结束标记”为止。
    *   示例：
        ```bash
        mail -s “Subject” user@example.com << EOF
        This is the email body.
        It can have multiple lines.
        EOF
        ```
    *   这里的 `EOF` 是自定义的结束标记，可以用任何单词代替。

![](img/98883775de87108044112c1087a69b2c_43.png)

![](img/98883775de87108044112c1087a69b2c_45.png)

### 特殊设备文件

![](img/98883775de87108044112c1087a69b2c_47.png)

在重定向中，有两个特殊的设备文件非常有用：

![](img/98883775de87108044112c1087a69b2c_49.png)

*   **`/dev/null`**： 黑洞设备。任何写入它的数据都会被丢弃，通常用于屏蔽不必要的输出。
    *   示例：`command > /dev/null 2>&1` 会静默执行命令，不产生任何屏幕输出。
*   **`/dev/zero`**： 零设备。提供无限的空字符（`\0`），常用于生成特定大小的空文件或进行数据覆盖。

## 管道符：连接命令的桥梁

![](img/98883775de87108044112c1087a69b2c_61.png)

![](img/98883775de87108044112c1087a69b2c_63.png)

上一节我们学会了如何将命令输出保存到文件。本节中我们来看看如何将一个命令的输出，直接作为另一个命令的输入，这就是管道符 `|` 的魔力。

![](img/98883775de87108044112c1087a69b2c_67.png)

管道符的作用是：**将前一个命令的标准输出，作为后一个命令的标准输入**。

### 管道符的基本使用

![](img/98883775de87108044112c1087a69b2c_75.png)

管道符可以连接多个命令，形成强大的处理链条。

![](img/98883775de87108044112c1087a69b2c_77.png)

![](img/98883775de87108044112c1087a69b2c_78.png)

![](img/98883775de87108044112c1087a69b2c_80.png)

以下是管道符的几种典型应用：

*   **筛选内容**： 从一个命令的大量输出中，筛选出包含特定关键词的行。
    *   示例：`rpm -qa | grep httpd` 会列出所有已安装的软件包，然后从中筛选出名字包含 “httpd” 的包。
*   **接力处理**： 对数据进行多次加工。
    *   示例：`cat /etc/passwd | grep -v “^#” | grep -v “^$” | head -5`
        1.  读取 `/etc/passwd` 文件。
        2.  用 `grep -v` 反选，去掉所有以 `#` 开头的注释行。
        3.  再次用 `grep -v` 去掉所有空行。
        4.  用 `head` 只显示前5行结果。
*   **统计结果**： 对另一个命令的输出结果进行统计。
    *   示例：`who | wc -l` 会统计当前登录系统的用户数量。

![](img/98883775de87108044112c1087a69b2c_82.png)

![](img/98883775de87108044112c1087a69b2c_84.png)

## 环境变量：系统的记忆与配置

上一节我们通过管道符实现了命令间的协作。本节中我们来看看环境变量，它是Shell和系统用来存储配置信息、路径和临时数据的“记忆体”。

### 什么是变量？

![](img/98883775de87108044112c1087a69b2c_106.png)

变量是用于存储数据的命名实体，由 **变量名** 和 **变量值** 组成，通常称为键值对 (Key-Value)。

![](img/98883775de87108044112c1087a69b2c_108.png)

*   **定义变量**： `变量名=变量值`
    *   **注意**：等号两边不能有空格。
    *   示例：`PRICE=5`
*   **引用变量**： `$变量名` 或 `${变量名}`
    *   在变量名前加 `$` 符号来获取它的值。
    *   示例：`echo “The price is $PRICE”` 会输出 “The price is 5”。
*   **使用大括号**： 使用 `${变量名}` 是为了明确变量名的边界，避免歧义。
    *   示例：`echo “Value: ${PRICE}USD”` 是正确的。如果写成 `$PRICEUSD`，系统会尝试寻找名为 `PRICEUSD` 的变量。

![](img/98883775de87108044112c1087a69b2c_110.png)

![](img/98883775de87108044112c1087a69b2c_112.png)

![](img/98883775de87108044112c1087a69b2c_114.png)

### 变量的作用

![](img/98883775de87108044112c1087a69b2c_116.png)

引入变量主要是为了便于管理和复用。例如，在脚本中多次使用同一个路径或数值时，只需修改一处变量定义即可。

![](img/98883775de87108044112c1087a69b2c_118.png)

![](img/98883775de87108044112c1087a69b2c_120.png)

### 环境变量与局部变量

![](img/98883775de87108044112c1087a69b2c_122.png)

*   **局部变量**： 仅在当前Shell会话中有效，关闭终端或开启新会话后即消失。
    *   直接使用 `变量名=值` 定义的就是局部变量。
*   **环境变量（全局变量）**： 可以传递给子Shell（如新开的终端标签、执行的脚本）的变量。
    *   使用 `export 变量名` 命令将局部变量提升为环境变量。
    *   示例：
        ```bash
        NETWORK_PATH=”/etc/sysconfig/network-scripts/”
        export NETWORK_PATH
        # 或者合并为一行：export NETWORK_PATH=”/etc/sysconfig/network-scripts/”
        ```
        之后，在子Shell中也可以使用 `cd $NETWORK_PATH`。

![](img/98883775de87108044112c1087a69b2c_124.png)

![](img/98883775de87108044112c1087a69b2c_126.png)

**注意**：`export` 设置的全局变量在重启后也会失效。要设置永久生效的环境变量，需要将 `export` 语句写入Shell的配置文件（如 `~/.bashrc` 或 `/etc/profile`），这将在后续课程中讲解。

![](img/98883775de87108044112c1087a69b2c_128.png)

![](img/98883775de87108044112c1087a69b2c_130.png)

### 重要的系统环境变量

系统预定义了许多环境变量，使用 `env` 或 `printenv` 命令可以查看所有环境变量。

![](img/98883775de87108044112c1087a69b2c_132.png)

![](img/98883775de87108044112c1087a69b2c_134.png)

以下是一些关键的系统环境变量：

![](img/98883775de87108044112c1087a69b2c_136.png)

*   **`PATH`**： 命令搜索路径。当输入一个命令时，系统会按照 `PATH` 中定义的目录顺序去查找可执行文件。
    *   格式：`PATH=/usr/local/bin:/usr/bin:/bin:…`
    *   **安全提示**：通常不建议将当前目录 `.` 加入 `PATH`，以免误执行当前目录下的恶意脚本。
*   **`HOME`**： 当前用户的家目录路径。
*   **`USER`/`LOGNAME`**： 当前登录的用户名。
*   **`SHELL`**： 当前用户使用的Shell解释器路径。
*   **`PWD`**： 当前工作目录。
*   **`LANG`**： 当前系统的语言和字符集设置。
*   **`RANDOM`**： 每次引用都会生成一个随机整数。

![](img/98883775de87108044112c1087a69b2c_138.png)

### 命令执行过程与环境变量

![](img/98883775de87108044112c1087a69b2c_140.png)

了解命令执行过程有助于理解环境变量的作用：

1.  **判断路径**： 用户输入命令后，系统首先判断是否为绝对路径（如 `/bin/ls`）或相对路径（如 `./script.sh`）。如果是，则直接尝试执行。
2.  **检查别名**： 如果不是路径，则检查该命令是否为别名（`alias`）。
3.  **判断命令类型**： 使用 `type 命令名` 判断是内部命令（Shell内置）还是外部命令（独立可执行文件）。
4.  **查找外部命令**： 如果是外部命令，则系统会依据 `PATH` 环境变量中定义的目录列表，依次查找是否有对应的可执行文件。找到则执行，找不到则报错 “command not found”。

## 通配符：文件名的模式匹配

![](img/98883775de87108044112c1087a69b2c_142.png)

在处理文件时，我们经常需要匹配一组具有相似名称的文件，这时就需要用到通配符。

![](img/98883775de87108044112c1087a69b2c_144.png)

![](img/98883775de87108044112c1087a69b2c_146.png)

以下是常用的通配符：

*   **`*`**： 匹配任意长度的任意字符（包括零个字符）。
    *   示例：`ls *.txt` 列出所有 `.txt` 结尾的文件。`file*` 匹配以 `file` 开头的所有文件。
*   **`?`**： 匹配单个任意字符。
    *   示例：`file?.txt` 匹配 `file1.txt`，`fileA.txt`，但不匹配 `file10.txt`。
*   **`[]`**： 匹配方括号内的任意一个字符。
    *   示例：`file[123].txt` 匹配 `file1.txt`，`file2.txt`，`file3.txt`。
    *   可以使用连字符 `-` 表示范围：`file[a-c].txt` 匹配 `filea.txt`，`fileb.txt`，`filec.txt`。
    *   可以使用 `^` 表示排除：`file[^0-9].txt` 匹配不以数字结尾的 `file?.txt`。

![](img/98883775de87108044112c1087a69b2c_148.png)

**注意**：通配符是由Shell进行解释的，用于文件名扩展。它与后面会讲到的**正则表达式**中的元字符（如 `*`, `+`, `.`）含义不同，正则表达式用于文本内容匹配，功能更强大但语法也更复杂。

## 转义字符：让特殊字符变普通

![](img/98883775de87108044112c1087a69b2c_160.png)

在Shell中，某些字符具有特殊含义（如 `$`, `*`, `>`, `空格`）。如果我们想原样使用这些字符，就需要进行转义。

![](img/98883775de87108044112c1087a69b2c_162.png)

![](img/98883775de87108044112c1087a69b2c_164.png)

以下是主要的转义方式：

![](img/98883775de87108044112c1087a69b2c_166.png)

*   **反斜杠 `\`**： 转义紧随其后的单个字符。
    *   示例：`echo “The price is \$5”` 会输出 `The price is $5`，而不是尝试读取变量 `$5`。
*   **单引号 `’ ‘`**： 强引用。引号内所有字符都失去特殊含义，被当作普通字符串。
    *   示例：`echo ‘The price is $5’` 会原样输出 `The price is $5`。
*   **双引号 `” “`**： 弱引用。引号内大部分字符失去特殊含义，但 `$`（变量替换）、`` ` ``（命令替换）和 `\`（转义）仍然有效。
    *   示例：`echo “User is $USER”` 会输出 `User is root`。
*   **反引号 `` ` ``**： 命令替换。执行反引号内的命令，并将其输出结果替换到当前位置。
    *   示例：`echo “Today is `date`”` 会输出 `Today is Thu Nov 7 14:30:00 CST 2024`。
    *   **现代写法**：更推荐使用 `$(命令)` 的形式，功能相同且更清晰，嵌套时不易出错。
        *   示例：`echo “Today is $(date)”` 和 `for i in $(seq 1 10); do echo $i; done`。

## 总结

![](img/98883775de87108044112c1087a69b2c_170.png)

本节课中我们一起学习了Linux命令行的三大核心机制：

1.  **重定向**： 我们掌握了如何使用 `>`, `>>`, `2>`, `<` 等符号，灵活地控制命令的输入源和输出目标，将数据导入文件或从文件读取，这是日志记录和脚本自动化的重要基础。
2.  **管道符**： 我们理解了 `|` 符号的强大之处，它能够将多个简单的命令像流水线一样连接起来，实现复杂的数据处理功能，极大地提升了命令行的工作效率。
3.  **环境变量**： 我们认识了系统存储配置信息的“记忆体”，学会了如何定义、引用局部变量和全局变量，并深入了解了 `PATH` 等关键环境变量在命令执行过程中的核心作用。
