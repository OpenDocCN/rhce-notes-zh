# Shell脚本编程：06：使用条件结构与调试 🧠

## 条件结构概述

条件结构允许在编写Shell脚本时包含决策逻辑，以便仅在满足特定条件时才执行脚本的相应部分。这类可以根据条件跳过某些命令的结构，我们称之为**结构化命令**。

### if-then 语句

![](img/1f2ffadd27ca22a5b994158128eed933_14.png)

![](img/1f2ffadd27ca22a5b994158128eed933_15.png)

![](img/1f2ffadd27ca22a5b994158128eed933_16.png)

`if-then`语句的基本格式如下：
```bash
if command
then
    commands
fi
```
如果`if`后面的命令成功执行（退出状态码为0），则执行`then`和`fi`之间的命令。如果不成功（退出状态码非0），则跳过这些命令，继续执行脚本中的下一条命令。

**需要强调的是**：在Bash Shell的`if`语句中，`if`后面跟的是一个命令。Shell会运行这个命令，并根据其**退出状态码**（而非其他编程语言中的布尔值`true`或`false`）来决定是否执行`then`部分的命令。

### elif 语句（嵌套条件）

当需要检查多个条件时，可以使用`elif`（else if）来创建多重条件分支。
```bash
if command1
then
    commands
elif command2
then
    other-commands
else
    default-commands
fi
```
以下脚本检查用户是否存在，如果不存在则检查其家目录是否存在，并根据情况创建用户：
```bash
#!/bin/bash
username=“testuser”
if grep -q “^$username:” /etc/passwd
then
    echo “User $username exists.”
elif [ -d /home/$username ]
then
    echo “Home directory exists but user does not. Creating user...”
    useradd $username
else
    echo “User does not exist and home directory is missing.”
fi
```

## case 语句

![](img/1f2ffadd27ca22a5b994158128eed933_86.png)

![](img/1f2ffadd27ca22a5b994158128eed933_88.png)

当需要对同一个变量进行多次值匹配时，使用多个`if-elif`语句会显得冗长。`case`命令提供了一种更清晰的方式来进行多分支选择。

`case`语句的格式如下：
```bash
case variable in
pattern1 | pattern2)
    commands1
    ;;
pattern3)
    commands2
    ;;
*)
    default-commands
    ;;
esac
```
`case`命令会将指定的变量与不同的模式进行比较。如果匹配，则执行对应的命令块。每个命令块以两个分号`;;`结束。星号`*`用于匹配所有其他情况。

![](img/1f2ffadd27ca22a5b994158128eed933_90.png)

![](img/1f2ffadd27ca22a5b994158128eed933_91.png)

![](img/1f2ffadd27ca22a5b994158128eed933_92.png)

以下是一个判断输入字符类型的`case`语句示例：
```bash
#!/bin/bash
read -p “Enter a character: “ char
case $char in
    [a-zA-Z])
        echo “You entered a letter.”
        ;;
    [0-9])
        echo “You entered a digit.”
        ;;
    *)
        echo “You entered a special character or something else.”
        ;;
esac
```

### break 命令

![](img/1f2ffadd27ca22a5b994158128eed933_104.png)

![](img/1f2ffadd27ca22a5b994158128eed933_106.png)

`break`命令用于立即终止整个循环的执行，并继续执行循环体之后的代码。

![](img/1f2ffadd27ca22a5b994158128eed933_108.png)

![](img/1f2ffadd27ca22a5b994158128eed933_110.png)

**跳出单层循环示例：**
```bash
#!/bin/bash
a=1
while [ $a -le 5 ]
do
    if [ $a -eq 3 ]
    then
        break
    fi
    echo “Iteration $a”
    a=$((a + 1))
done
echo “Loop ended.”
```
当`a`等于3时，循环终止。

![](img/1f2ffadd27ca22a5b994158128eed933_112.png)

**跳出多层循环示例：**
使用`break n`可以跳出多层嵌套循环，其中`n`指定要跳出的循环层数。
```bash
#!/bin/bash
for i in 1 2
do
    echo “Outer loop: $i”
    for j in 1 2 3
    do
        if [ $j -eq 2 ]
        then
            break 2 # 跳出两层循环
        fi
        echo “ Inner loop: $j”
    done
done
```

### continue 命令

`continue`命令用于跳过当前循环迭代中剩余的语句，直接开始下一次迭代。

![](img/1f2ffadd27ca22a5b994158128eed933_120.png)

![](img/1f2ffadd27ca22a5b994158128eed933_122.png)

![](img/1f2ffadd27ca22a5b994158128eed933_124.png)

**continue 示例：**
```bash
#!/bin/bash
for num in {1..5}
do
    if [ $((num % 2)) -eq 0 ]
    then
        continue # 跳过偶数
    fi
    echo “Odd number: $num”
done
```
这个脚本只输出奇数（1, 3, 5），遇到偶数时使用`continue`跳过了`echo`命令。

### 良好实践建议
1.  **使用注释**：在脚本顶部和关键部分添加注释，说明脚本用途和逻辑。
2.  **代码结构化**：将长命令分成小块，使用缩进和空行来明确代码结构和层次。
3.  **格式一致**：在整个脚本中保持一致的编码风格。

### 调试模式

![](img/1f2ffadd27ca22a5b994158128eed933_138.png)

Bash Shell提供了几种调试脚本的模式：

![](img/1f2ffadd27ca22a5b994158128eed933_140.png)

![](img/1f2ffadd27ca22a5b994158128eed933_141.png)

![](img/1f2ffadd27ca22a5b994158128eed933_142.png)

![](img/1f2ffadd27ca22a5b994158128eed933_143.png)

*   **详细模式 (`-v`)**：在执行前将每条命令打印到标准输出。
    ```bash
    bash -v script.sh
    ```
*   **调试/跟踪模式 (`-x`)**：在执行时显示命令及其参数，有助于跟踪执行流程。
    ```bash
    bash -x script.sh
    ```
*   **语法检查 (`-n`)**：只检查脚本语法，而不执行它。
    ```bash
    bash -n script.sh
    ```
