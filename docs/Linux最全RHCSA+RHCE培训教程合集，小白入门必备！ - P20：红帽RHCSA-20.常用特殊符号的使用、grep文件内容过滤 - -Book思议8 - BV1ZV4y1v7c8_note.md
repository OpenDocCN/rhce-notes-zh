## 常用特殊符号

上一节我们介绍了Linux的基础命令，本节中我们来看看在文件名匹配和文本处理中扮演重要角色的特殊符号。在Linux系统下，特殊符号可以完成一些特殊的功能。

键盘上的许多符号都属于特殊符号，例如：`!`、`@`、`#`、`$`、`%`、`^`、`&`、`*`、`()`、`{}`、`[]`、`<>`、`?`、`/`、`\`、`` ` ``、`.`以及管道符`|`等。每种符号在系统中都有特定的含义。

在本阶段，我们主要学习以下四个符号：`*`、`{}`、`[]`和`?`。其中，`*`和`{}`是最常用且必须掌握的。

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_9.png)

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_11.png)

### 星号 `*`
星号`*`在文件名上用来代表任意多个任意字符。这意味着它可以匹配任何字符（数字、字母、符号等），并且不限制字符的数量。

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_13.png)

**公式/代码示例：**
*   `*` 匹配所有文件。
*   `pass*` 匹配所有以“pass”开头的文件。
*   `*.log` 匹配所有以“.log”结尾的文件。

例如，命令`ls /etc/pass*`会列出`/etc`目录下所有以“pass”开头的文件。命令`cp /var/log/*.log /opt`则会将`/var/log`目录下所有日志文件复制到`/opt`目录。

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_15.png)

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_17.png)

### 问号 `?`
问号`?`也是在文件名上使用的特殊符号，它用来代表任意单个字符。

**公式/代码示例：**
*   `tty?` 匹配像`tty1`、`ttyA`这样的文件名。
*   `tty??` 匹配像`tty12`、`ttyab`这样后跟两个字符的文件名。

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_19.png)

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_21.png)

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_23.png)

例如，`ls /dev/tty?`会列出`/dev`目录下以`tty`开头且后面只跟一个字符的所有设备文件。

### 中括号 `[]`
中括号`[]`用于匹配指定范围内的一个字符。它通常用来匹配数字或字母的连续范围。

**公式/代码示例：**
*   `[0-9]` 匹配任意一个数字。
*   `[a-z]` 匹配任意一个小写字母。
*   `*[0-9]*` 匹配文件名中包含任意数字的文件。

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_25.png)

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_27.png)

例如，`ls /var/log/*[0-9]*`会列出`/var/log`目录下文件名中包含数字的所有文件。但需要注意的是，`[]`在匹配不连续的数字范围（如`[15-25]`）时可能无法按预期工作。

### 大括号 `{}`
大括号`{}`功能强大，常用于生成序列或代表多组不同的字符串，尤其在处理数字序列时非常方便。

**公式/代码示例：**
*   `{1..100}` 生成从1到100的数字序列。
*   `{a,b,c}` 代表a、b、c三个独立的字符串。

例如，命令`touch test{1..100}.txt`可以快速创建100个名为`test1.txt`到`test100.txt`的文件。命令`rm -rf test{1..50}.txt`则会删除前50个文件。

## grep文件内容过滤

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_31.png)

了解了用于文件名匹配的特殊符号后，我们来看看如何深入文件内部查找内容。`grep`命令用于在文件中搜索符合条件的字符串，并将匹配到的行打印出来。

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_33.png)

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_35.png)

### grep基本语法
`grep`命令的基本语法格式如下：

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_37.png)

**代码示例：**
```bash
grep [选项] “搜索字符串” 文件名
```

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_39.png)

![](img/4b06783f2bcf8c68d5b653d2cef2acf1_41.png)

例如，我们想查看`/etc/passwd`文件中关于用户`tom`的信息，而不想看到文件的其他内容：

**代码示例：**
```bash
grep “tom” /etc/passwd
```
如果想同时显示行号，可以加上`-n`选项：
```bash
grep -n “tom” /etc/passwd
```

### 常用选项
以下是`grep`命令的一些常用选项：

*   `-n`：显示匹配行及其行号。
*   `-i`：忽略大小写进行匹配。
*   `-v`：反向选择，即显示不包含匹配字符串的所有行。

**代码示例：**
```bash
# 忽略大小写搜索“user”
grep -i “user” /etc/passwd

# 排除包含“root”的行
grep -v “root” /etc/passwd
```

### 结合正则表达式符号
`grep`可以结合更精确的正则表达式符号进行搜索：

*   `^`：匹配以指定字符串开头的行。
*   `$`：匹配以指定字符串结尾的行。
*   `^$`：匹配空行。

**代码示例：**
```bash
# 搜索以“root”开头的行
grep “^root” /etc/passwd

# 搜索以“/bin/bash”结尾的行（正常登录用户）
grep “/bin/bash$” /etc/passwd

# 统计文件中的空行数
grep “^$” /etc/default/useradd | wc -l
```

### 组合使用示例
一个实用的场景是查看配置文件中所有非注释且非空行的有效配置。例如，对于`/etc/default/useradd`文件：

**代码示例：**
```bash
# 1. 先排除以#开头的注释行
grep -v “^#” /etc/default/useradd
# 2. 再将上一步结果通过管道传递给下一个grep，排除空行
grep -v “^#” /etc/default/useradd | grep -v “^$”
# 3. 加上行号显示
grep -v “^#” /etc/default/useradd | grep -v “^$” | grep -n “.*”
```

### 结合管道使用
`grep`经常与管道`|`结合，对其他命令的输出结果进行过滤。

**代码示例：**
```bash
# 只查看内存信息
free -h | grep -i “mem”

# 只查看ens33网卡的信息
ifconfig | grep “ens33”
```
