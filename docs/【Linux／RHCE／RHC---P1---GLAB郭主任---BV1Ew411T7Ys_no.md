### 核心目录解析

![](img/6843ca73226eb91d491b05c7a60f65ff_39.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_41.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_43.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_45.png)

以下是Linux根目录下一些最重要的二级目录及其作用：

![](img/6843ca73226eb91d491b05c7a60f65ff_47.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_49.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_51.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_53.png)

*   **`/boot`**：存放系统启动相关的文件，如内核（`vmlinuz-*`）和引导加载程序（`grub2`）。
*   **`/dev`**：存放设备文件。Linux遵循“一切皆文件”的理念，所有硬件设备（如磁盘、CPU、内存）都在此目录下以文件形式呈现。例如，第一块虚拟磁盘通常为 `/dev/vda`。
*   **`/etc`**：**最重要的目录之一**，存放系统和应用程序的**配置文件**。例如，网络配置、软件源配置、用户信息等都位于此目录下。
*   **`/home`**：普通用户的**家目录**。每个用户在此目录下都有一个以自己用户名命名的子目录（如 `/home/student`），用于存放个人文件。
*   **`/root`**：系统管理员（root用户）的专属家目录。
*   **`/bin` 和 `/sbin`**：存放可执行的**命令文件**。`/bin` 包含普通用户和系统管理员都可用的基础命令（如 `ls`, `cp`），`/sbin` 则主要存放系统管理命令。
*   **`/lib` 和 `/lib64`**：存放系统运行所需的**库文件**。这些是应用程序调用内核功能的标准化接口。
*   **`/usr`**：存放用户安装的应用程序、共享库、文档等，可以看作是 `/` 目录的扩展。
*   **`/var`**：存放经常变化的（Variable）数据，如日志文件（`/var/log`）、邮件队列等。
*   **`/tmp`**：存放**临时文件**。系统会定期清理此目录下的内容。
*   **`/proc` 和 `/sys`**：**伪文件系统**，它们映射内核和进程信息，主要用于系统监控和性能调优，其中的文件并非真实存储在磁盘上。
*   **`/media` 和 `/mnt`**：**临时挂载点**。用于临时挂载移动设备（如U盘）或网络文件系统。

### 路径概念：绝对路径与相对路径

![](img/6843ca73226eb91d491b05c7a60f65ff_55.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_57.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_59.png)

在文件系统中定位文件或目录需要使用路径。
*   **绝对路径**：总是从根目录 `/` 开始，完整描述文件位置的路径。例如：`/home/student/file.txt`
*   **相对路径**：相对于当前工作目录的路径。常用符号有：
    *   `.` 代表当前目录。
    *   `..` 代表上一级目录。
    *   `~` 代表当前用户的家目录。

![](img/6843ca73226eb91d491b05c7a60f65ff_61.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_63.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_65.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_67.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_69.png)

**命令示例：**
```bash
pwd                 # 显示当前工作目录的绝对路径
cd /etc             # 使用绝对路径切换到 /etc 目录
cd ..               # 使用相对路径切换到上级目录
cd ~                # 切换到当前用户的家目录
```

### 创建目录 (`mkdir`)

![](img/6843ca73226eb91d491b05c7a60f65ff_87.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_89.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_91.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_93.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_95.png)

使用 `mkdir` 命令创建新目录。

![](img/6843ca73226eb91d491b05c7a60f65ff_97.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_99.png)

**命令语法与常用选项：**
```bash
mkdir [选项] 目录名...
```
*   `-p`：递归创建目录。如果父目录不存在，则一并创建。

**创建目录的注意事项：**
1.  目录名中不能包含 `/`。
2.  目录名长度通常不超过255个字符。
3.  Linux严格区分大小写和空格。

![](img/6843ca73226eb91d491b05c7a60f65ff_101.png)

**操作示例：**
```bash
mkdir dir1                  # 创建单个目录 dir1
mkdir -p parent/child       # 创建嵌套目录结构
mkdir dir{A,B,C}            # 使用通配符批量创建 dirA, dirB, dirC
```

### 删除目录与文件 (`rmdir`, `rm`)

![](img/6843ca73226eb91d491b05c7a60f65ff_103.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_105.png)

*   `rmdir`：仅用于删除**空目录**。
*   `rm`：功能强大，可删除文件或非空目录。

![](img/6843ca73226eb91d491b05c7a60f65ff_107.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_109.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_111.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_113.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_115.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_117.png)

**命令语法与常用选项：**
```bash
rmdir [选项] 目录名...      # 删除空目录
rm [选项] 文件或目录名...   # 删除文件或目录
```
*   `rm -r`：递归删除，用于删除目录及其包含的所有内容。
*   `rm -f`：强制删除，不进行任何确认提示。
*   `rm -rf`：**危险命令！** 强制递归删除，请谨慎使用，尤其是在根目录 `/` 下。

![](img/6843ca73226eb91d491b05c7a60f65ff_119.png)

**操作示例：**
```bash
rmdir empty_dir             # 删除空目录 empty_dir
rm file.txt                 # 删除文件 file.txt
rm -r my_folder             # 递归删除目录 my_folder 及其内容
# 请勿在根目录执行：rm -rf /
```

### 复制文件与目录 (`cp`)

使用 `cp` 命令复制文件或目录。

![](img/6843ca73226eb91d491b05c7a60f65ff_121.png)

**复制关系总结：**
*   ✅ 文件 -> 文件
*   ✅ 文件 -> 目录
*   ✅ 多个文件 -> 目录
*   ✅ 目录 -> 目录 (需加 `-r` 选项)
*   ❌ 多个文件 -> 文件
*   ❌ 目录 -> 文件

**命令语法与常用选项：**
```bash
cp [选项] 源... 目标
```
*   `-r` 或 `-R`：递归复制，用于复制目录。
*   `-f`：强制复制，覆盖目标文件时不提示。
*   `-p`：保留源文件的属性（如所有者、时间戳）。

![](img/6843ca73226eb91d491b05c7a60f65ff_123.png)

**操作示例：**
```bash
cp file1.txt file2.txt          # 复制文件并重命名
cp *.jpg /home/pictures/        # 复制所有.jpg文件到指定目录
cp -r source_dir/ dest_dir/     # 递归复制整个目录
```

### 移动/重命名文件与目录 (`mv`)

使用 `mv` 命令移动或重命名文件/目录。

**命令语法：**
```bash
mv [选项] 源... 目标
```
*   `-f`：强制移动，覆盖时不提示。

![](img/6843ca73226eb91d491b05c7a60f65ff_133.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_135.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_137.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_139.png)

**操作示例：**
```bash
mv old_name.txt new_name.txt    # 重命名文件
mv file.txt /tmp/               # 移动文件到 /tmp 目录
mv dir1/ dir2/                  # 将 dir1 移动到 dir2 内（若dir2存在），或重命名 dir1 为 dir2（若dir2不存在）
```

## 链接文件 🔗

![](img/6843ca73226eb91d491b05c7a60f65ff_145.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_147.png)

Linux中有两种链接文件：软链接（符号链接）和硬链接。

![](img/6843ca73226eb91d491b05c7a60f65ff_149.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_151.png)

*   **软链接（Symbolic Link）**：类似于Windows的快捷方式。它是一个独立的文件，内容指向目标文件的路径。删除原文件，软链接会失效。
*   **硬链接（Hard Link）**：是原文件的一个“别名”，与原文件共享相同的inode（数据块指针）。删除任何一个硬链接，只要还有一个链接存在，文件数据就不会被删除。

**创建链接的命令：**
```bash
ln -s 目标文件 链接名      # 创建软链接
ln 目标文件 链接名         # 创建硬链接
```

![](img/6843ca73226eb91d491b05c7a60f65ff_153.png)

![](img/6843ca73226eb91d491b05c7a60f65ff_155.png)

**操作示例：**
```bash
ln -s /var/log/messages log_link  # 创建指向系统日志的软链接
ls -l log_link                     # 查看链接详情，首字符为 ‘l’
```
