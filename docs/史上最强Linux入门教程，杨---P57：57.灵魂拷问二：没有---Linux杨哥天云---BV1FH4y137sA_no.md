## 概述

![](img/2c51b8825d1eb5408323b5774d32c95f_5.png)

在本节中，我们将探讨一个关于Linux文件权限的核心问题：当一个用户对某个文件没有任何访问权限时，是否就一定无法访问或操作该文件？我们将通过一个具体的场景来揭示“权限”与“权利”之间的重要区别。

![](img/2c51b8825d1eb5408323b5774d32c95f_7.png)

---

![](img/2c51b8825d1eb5408323b5774d32c95f_9.png)

上一节我们介绍了Linux的基本文件权限模型。本节中，我们来看看一个更深层次的问题：没有权限是否等同于绝对无法访问。

![](img/2c51b8825d1eb5408323b5774d32c95f_11.png)

我们准备了一个场景。在`/home`目录下有一个名为`password.txt`的文件。我们使用`root`用户将该文件的权限设置为`000`，即所有用户（所有者、所属组、其他用户）都**没有任何**读、写、执行权限。

![](img/2c51b8825d1eb5408323b5774d32c95f_13.png)

```bash
chmod 000 /home/password.txt
```

![](img/2c51b8825d1eb5408323b5774d32c95f_15.png)

此时，任何用户理论上都无法访问此文件。

![](img/2c51b8825d1eb5408323b5774d32c95f_17.png)

现在，我们以普通用户`tianyun`的身份尝试访问这个文件。

![](img/2c51b8825d1eb5408323b5774d32c95f_19.png)

![](img/2c51b8825d1eb5408323b5774d32c95f_21.png)

首先，尝试读取文件内容：
```bash
cat /home/password.txt
```
命令执行失败，提示权限不足，无法查看。

![](img/2c51b8825d1eb5408323b5774d32c95f_23.png)

接着，尝试向文件中写入内容：
```bash
echo "test" >> /home/password.txt
```
命令同样失败，提示权限不足，无法写入。

![](img/2c51b8825d1eb5408323b5774d32c95f_25.png)

执行操作更不可能成功。因此，从“权限”的角度看，用户`tianyun`确实被完全阻挡在外了。

![](img/2c51b8825d1eb5408323b5774d32c95f_27.png)

那么，用户`tianyun`是否就真的束手无策了呢？这里需要思考一个问题：设置文件权限的操作，是否只能是`root`用户的特权？

![](img/2c51b8825d1eb5408323b5774d32c95f_29.png)

我们让用户`tianyun`自己尝试修改这个文件的权限：
```bash
chmod 777 /home/password.txt
```
这个操作能否成功？普通用户能否为自己拥有的文件修改权限？

![](img/2c51b8825d1eb5408323b5774d32c95f_31.png)

![](img/2c51b8825d1eb5408323b5774d32c95f_33.png)

执行命令后，我们发现操作**成功**了。文件权限被修改为`777`，`tianyun`用户随即获得了读、写、执行的完整权限。

![](img/2c51b8825d1eb5408323b5774d32c95f_35.png)

这引出了一个关键概念：在Linux中，存在两种不同的东西。

![](img/2c51b8825d1eb5408323b5774d32c95f_37.png)

1.  **Permission（权限）**：指文件上设置的读、写、执行标志（如 `rwxr-xr--`）。它规定了“你能做什么”。
2.  **Power / Right（权利）**：指用户对文件的所有权或控制权。它决定了“你能改变什么”。

![](img/2c51b8825d1eb5408323b5774d32c95f_39.png)

用户`tianyun`最初对文件没有**权限**。但是，请不要忘记，他是该文件的**所有者**。作为所有者，他对这个文件拥有根本的**权利**。这种权利使得他可以自由地更改文件的**权限**设置。

![](img/2c51b8825d1eb5408323b5774d32c95f_41.png)

简单来说：**我有权利（Power），我就能为自己赋予权限（Permission）**。

![](img/2c51b8825d1eb5408323b5774d32c95f_43.png)

反之，如果一个用户既不是文件的所有者，也不在文件的所属组中，更没有其他特殊权限（如`sudo`），那么他对该文件既无**权利**，也无**权限**，自然无法修改其权限设置。

![](img/2c51b8825d1eb5408323b5774d32c95f_45.png)

所以，对于最初的灵魂拷问：“没有权限一定就访问不了吗？”，答案是不一定。

![](img/2c51b8825d1eb5408323b5774d32c95f_47.png)

![](img/2c51b8825d1eb5408323b5774d32c95f_49.png)

以下是核心结论：
*   文件的所有者（Owner）始终拥有修改该文件权限的**权利**，即使当前权限设置禁止他访问。
*   因此，**没有权限 ≠ 绝对无法访问**。如果用户是文件的所有者，他可以通过行使权利来重新获得权限。
*   修改文件权限并非`root`用户的专利，文件所有者同样可以。

![](img/2c51b8825d1eb5408323b5774d32c95f_51.png)

## 总结

![](img/2c51b8825d1eb5408323b5774d32c95f_55.png)

本节课中我们一起学习了Linux中“权限”与“权利”的关键区别。我们通过实验看到，文件的所有者凭借其所有权（权利），可以越过当前的文件权限（Permission）设置，直接修改权限，从而重新获得访问能力。这深刻说明了在Linux安全模型中，所有权是比一时权限设置更为根本的控制要素。理解这一点，对于深入掌握Linux文件系统和用户管理至关重要。
