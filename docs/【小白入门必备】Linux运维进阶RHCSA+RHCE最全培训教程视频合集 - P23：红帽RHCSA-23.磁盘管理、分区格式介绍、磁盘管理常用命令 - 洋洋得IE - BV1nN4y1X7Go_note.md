## 概述
在本节课中，我们将学习Linux系统中的磁盘管理基础知识。内容包括磁盘的物理结构、不同类型的分区格式（MBR与GPT）以及管理磁盘和文件系统的常用命令。通过本课的学习，你将能够理解磁盘的基本工作原理，并掌握查看和管理磁盘信息的核心技能。

---

## 磁盘物理结构简介
上一节我们介绍了课程的整体内容，本节中我们来看看磁盘的物理结构。在传统的物理服务器中，硬盘是可见的实体组件。

服务器硬盘的外观类似一块板砖，它被插入服务器机箱特定的硬盘插槽中。现代服务器硬盘大多支持热插拔功能，这意味着可以在服务器开机运行的状态下直接插入或拔出硬盘，系统能够立即识别，类似于使用U盘。

硬盘内部主要由以下几个核心部件组成：
*   **主轴马达**：用于带动盘片旋转。
*   **盘片**：数据实际存储的介质，类似于光盘。
*   **机械手臂与读写磁头**：机械手臂带动读写磁头在高速旋转的盘片上方移动，进行数据的读取和写入操作。

需要注意的是，硬盘内部是真空环境，以防止灰尘进入导致盘片损坏和数据丢失。因此，切勿随意拆解硬盘。

---

## 磁盘的存储单元：扇区
了解了物理结构后，我们需要知道数据是如何在磁盘上组织的。磁盘在划分分区和存储数据时，是以“扇区”为最小单位的。

**核心概念**：
*   扇区是磁盘存储数据的最小逻辑单元。
*   默认情况下，一个扇区的大小是 **512字节**。
*   当我们需要创建一个分区（例如10GB）时，系统会自动计算需要多少个扇区来组成这个空间，我们无需手动进行繁琐的换算。

---

## 磁盘接口类型
在为企业服务器或个人电脑选购硬盘时，会遇到不同的接口类型。接口决定了硬盘与主板之间的连接方式和数据传输速度。

以下是几种常见的硬盘接口类型：
*   **IDE**：一种较古老的接口，数据传输速度慢，价格便宜，现已基本淘汰。
*   **SCSI**：早期在服务器领域流行，速度快并支持热插拔，但如今也已不常见。
*   **SATA**：目前个人计算机领域主流的接口，也常见于服务器，性价比高。
*   **SAS**：当前服务器领域的主流接口，在速度和可靠性方面表现优异。
*   **SSD (固态硬盘)**：采用闪存颗粒存储数据，读写速度远快于传统机械硬盘，但价格较高。
*   **NVMe**：一种用于固态硬盘的高速通信协议，能充分发挥SSD的性能。

选择硬盘时，需确保其接口类型与服务器或主板的接口相匹配。

## 磁盘分区格式：MBR vs GPT
为硬盘创建分区前，必须选择一种分区格式。两种主流的格式是MBR和GPT，它们有不同的特点和限制。

### MBR (主引导记录)
MBR是一种较老的分区格式。

**特点与限制**：
1.  **分区数量限制**：一块硬盘最多只能创建**4个主分区**。
2.  **扩展分区与逻辑分区**：为了突破4个主分区的限制，可以将第4个分区创建为“扩展分区”。扩展分区本身不能直接存储数据，但它是一个容器，可以在其内部创建多个“逻辑分区”来存储数据。
3.  **磁盘容量限制**：MBR分区格式最大只支持 **2TB** 容量的硬盘。对于大于2TB的硬盘，无法使用MBR进行全部分区。

### GPT (GUID 分区表)
GPT是一种现代的分区格式，旨在克服MBR的限制。

**特点与优势**：
1.  **分区数量**：理论上，一块GPT硬盘可以创建**多达128个主分区**，无需使用扩展分区和逻辑分区这种复杂结构。
2.  **磁盘容量**：GPT支持巨大的磁盘容量，最大可达 **18EB**。这远远超过当前任何单个硬盘的容量，未来可扩展性极强。

**选择建议**：
*   对于容量小于2TB的硬盘，MBR和GPT均可使用。
*   对于容量大于2TB的硬盘，**必须使用GPT**分区格式。
*   鉴于GPT的优越性和未来兼容性，在新系统和新硬盘上通常推荐使用GPT。

## 磁盘管理常用命令
现在我们已经了解了磁盘的基础知识和分区格式，接下来学习在Linux系统中管理磁盘的常用命令。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_22.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_24.png)

### 1. `lsblk` - 列出块设备信息
这条命令用于查看系统中所有硬盘及其分区的信息。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_26.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_28.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_30.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_32.png)

**常用用法**：
```bash
lsblk
```
执行该命令会列出所有磁盘（如`sda`, `sdb`）和它们下面的分区（如`sda1`, `sda2`），并显示其挂载点。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_34.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_36.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_38.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_40.png)

**查看特定磁盘的详细信息**：
```bash
lsblk /dev/sdb
```

![](img/86a8f93f8e0a426d6aa10a219ce958b7_42.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_44.png)

### 2. `df` - 报告文件系统磁盘空间使用情况
这条命令用于查看已挂载文件系统的磁盘空间使用情况，如剩余空间、使用比例等。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_46.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_48.png)

**常用用法**：
```bash
df -h
```
选项 `-h` 表示以人类易读的格式（如K, M, G）显示大小。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_50.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_52.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_54.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_56.png)

### 3. `du` - 估算文件或目录的磁盘使用量
这条命令用于统计具体文件或目录占用了多少磁盘空间。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_58.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_60.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_62.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_64.png)

**常用用法**：
**统计当前目录下所有文件/目录的大小**：
```bash
du -sh *
```
选项 `-s` 表示仅显示总计，`-h` 表示人类易读格式。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_66.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_68.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_70.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_72.png)

**统计指定目录的总大小**：
```bash
du -sh /etc
```

![](img/86a8f93f8e0a426d6aa10a219ce958b7_74.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_76.png)

### 4. `blkid` - 查看块设备属性
这条命令可以查看磁盘或分区的属性，如文件系统类型、UUID等。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_78.png)

**常用用法**：
```bash
blkid /dev/sda1
```

## 在系统中添加新硬盘
我们可以在虚拟机中动态添加硬盘来模拟热插拔操作。

![](img/86a8f93f8e0a426d6aa10a219ce958b7_94.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_96.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_98.png)

操作后，使用 `lsblk` 命令可以立即看到新添加的硬盘（例如 `sdb`）。Linux中磁盘设备的命名规则如下：
*   `sd`：代表SCSI或SATA类型的磁盘。
*   第三个字母（`a`, `b`, `c`...）：代表第几块磁盘。`sda`是第一块，`sdb`是第二块，依此类推。
*   数字后缀（`1`, `2`, `3`...）：代表该磁盘上的第几个分区。

## 命令总结与对比
以下是本节课核心命令的快速参考指南：

![](img/86a8f93f8e0a426d6aa10a219ce958b7_110.png)

![](img/86a8f93f8e0a426d6aa10a219ce958b7_112.png)

| 命令 | 主要用途 | 常用示例 |
| :--- | :--- | :--- |
| `lsblk` | **查看所有硬盘和分区**的树状结构 | `lsblk` |
| `df` | **查看已挂载文件系统的使用情况**（空间、利用率） | `df -h` |
| `du` | **统计具体文件或目录的磁盘占用量** | `du -sh /home` |
| `blkid` | **查看块设备（硬盘/分区）的属性**（UUID，文件系统类型） | `blkid /dev/sda1` |

![](img/86a8f93f8e0a426d6aa10a219ce958b7_114.png)

**简单区分**：
*   想看看机器里插了几块硬盘，每个硬盘分了几个区，用 **`lsblk`**。
*   想看看根目录`/`或者`/home`目录还剩多少空间，会不会满了，用 **`df -h`**。
*   想弄清楚某个文件夹（比如放了很多日志的`/var/log`）到底占了多大地方，用 **`du -sh`**。
