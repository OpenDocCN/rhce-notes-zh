# 红帽认证入门教程：Day04：网络管理（下）🔧

在本节课中，我们将要学习如何在红帽系统（RHEL 8）中配置和管理网络。我们将重点掌握使用 `nmcli` 命令行工具修改IP地址、网关和DNS，理解设备与连接的关系，并学习直接修改配置文件的方法。

## 网络配置管理概述

上一节我们介绍了网络查看的基本命令。本节中我们来看看如何进行网络信息的配置管理。

在红帽8之前，我们通常直接修改网卡配置文件后重启 `network` 服务。但在红帽8中，`network` 服务不再提供，所有网络管理都通过 **NetworkManager** 来完成。虽然直接修改配置文件的方法依然有效，但我们将重点学习命令行的配置方式，因为这在生产环境中尤为重要。

![](img/524620945bc4ad61f0f722f2e79ac670_11.png)

![](img/524620945bc4ad61f0f722f2e79ac670_13.png)

NetworkManager 提供了图形化工具和命令行工具。我们着重讲解命令行工具 `nmcli`。其基本用法是 `nmcli connection`（可简写为 `con`），后面跟具体参数。NetworkManager 会将配置保存在 `/etc/sysconfig/network-scripts/` 目录下。

![](img/524620945bc4ad61f0f722f2e79ac670_15.png)

![](img/524620945bc4ad61f0f722f2e79ac670_16.png)

![](img/524620945bc4ad61f0f722f2e79ac670_18.png)

这里有两个核心概念：
*   **设备名**：指网络接口本身，如 `eth0`, `ens224`。
*   **连接名**：是为设备配置的一个设置集合，可以与设备名一致或不一致。

对于一个设备，同一时间只能有一个连接处于活动状态。但可以为同一个设备创建多个连接，以便在不同场景下切换使用。

## 添加与激活网络连接

以下是添加新网卡并为其创建连接的基本步骤。

首先，我们在虚拟机中添加一块新网卡。添加后，系统会识别到一个新设备（例如 `ens224`），但此时它没有活动的连接。

我们可以使用 `nmcli device status` 和 `ip addr show` 命令来确认新设备的存在及其状态（通常为 `disconnected`）。

要为此设备激活一个连接，使用以下命令：
```bash
nmcli device connect ens224
```
此命令会为设备 `ens224` 创建一个默认的连接，并在 `/etc/sysconfig/network-scripts/` 目录下生成对应的配置文件。

## 修改网络配置（IP、网关、DNS）

![](img/524620945bc4ad61f0f722f2e79ac670_20.png)

现在我们已经有了连接，接下来学习如何修改其配置，例如IP地址、子网掩码、网关和DNS。

![](img/524620945bc4ad61f0f722f2e79ac670_22.png)

![](img/524620945bc4ad61f0f722f2e79ac670_23.png)

![](img/524620945bc4ad61f0f722f2e79ac670_25.png)

使用 `nmcli connection modify` 命令可以修改连接的属性。以下是配置静态IP地址的示例：
```bash
nmcli connection modify ens224 ipv4.addresses 192.168.146.128/24
nmcli connection modify ens224 ipv4.method manual
```
第一条命令设置了IP地址和子网掩码。第二条命令将IP获取方式设置为手动（静态）。

修改配置后，需要重新加载连接并使更改生效：
```bash
nmcli connection reload
nmcli connection down ens224 && nmcli connection up ens224
```
最后，使用 `ip addr show ens224` 或 `nmcli connection show ens224` 来验证配置。

## 设备与连接的关系详解

我们再次强调设备与连接的关系。连接是配置的集合，其名称可以独立于设备名。

例如，我们可以修改连接的名字：
```bash
nmcli connection modify ens224 connection.id “my-connection”
```
修改后，在操作此连接时就需要使用新的连接名 `my-connection`。为了便于管理，通常建议保持连接名与设备名一致。

![](img/524620945bc4ad61f0f722f2e79ac670_27.png)

![](img/524620945bc4ad61f0f722f2e79ac670_29.png)

![](img/524620945bc4ad61f0f722f2e79ac670_30.png)

创建连接有两种主要方式：
1.  **自动关联**：使用 `nmcli device connect <设备名>`，系统会自动创建并激活一个默认连接。
2.  **手动创建**：使用 `nmcli connection add` 命令手动创建连接。
    ```bash
    nmcli connection add type ethernet ifname ens256 con-name ens256
    ```

## 配置网关与路由

![](img/524620945bc4ad61f0f722f2e79ac670_37.png)

网关配置通常只需在一个连接上设置，因为它是系统级别的。为连接 `ens256` 配置网关和DNS的命令如下：
```bash
nmcli connection modify ens256 ipv4.gateway 10.10.1.2 ipv4.dns 10.10.1.2
```
配置多个网关时，系统会根据路由的**跃点**值决定优先级，跃点值越低，路由优先级越高。可以使用 `ip route` 或 `route -n` 命令查看路由表。

## 直接修改配置文件

除了使用 `nmcli`，也可以直接编辑网卡配置文件，路径通常为 `/etc/sysconfig/network-scripts/ifcfg-<连接名>`。

![](img/524620945bc4ad61f0f722f2e79ac670_43.png)

![](img/524620945bc4ad61f0f722f2e79ac670_45.png)

![](img/524620945bc4ad61f0f722f2e79ac670_46.png)

以下是一个配置文件的关键参数示例：
```bash
TYPE=Ethernet
BOOTPROTO=none # 或 static， 表示静态IP
NAME=ens256 # 连接名
DEVICE=ens256 # 设备名
ONBOOT=yes # 开机启动
IPADDR=10.10.1.1 # IP地址
PREFIX=24 # 子网掩码， 也可用 NETMASK=255.255.255.0
GATEWAY=10.10.1.2 # 网关
DNS1=10.10.1.2 # DNS服务器， 注意从1开始编号
```
修改配置文件后，必须重新加载配置才能生效：
```bash
nmcli connection reload
nmcli connection down ens256 && nmcli connection up ens256
```
