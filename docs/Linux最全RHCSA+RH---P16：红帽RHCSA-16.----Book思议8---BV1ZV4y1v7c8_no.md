## 用户密码设置命令：`passwd`

`passwd` 命令用于设置或修改用户的登录密码。

![](img/f853d87279f9933e036ccb160d5e4361_14.png)

![](img/f853d87279f9933e036ccb160d5e4361_16.png)

![](img/f853d87279f9933e036ccb160d5e4361_18.png)

![](img/f853d87279f9933e036ccb160d5e4361_20.png)

### 基本用法

![](img/f853d87279f9933e036ccb160d5e4361_22.png)

*   **修改当前登录用户的密码**：直接执行 `passwd` 命令，系统会提示输入新密码。
    ```bash
    passwd
    ```
*   **修改指定用户的密码**：需要管理员（root）权限，在命令后指定用户名。
    ```bash
    passwd username
    ```

### 密码规范

系统对密码有安全规范要求，普通用户修改密码时必须遵守，而root用户则不受此限制。主要规范包括：
*   密码长度不少于8个字符。
*   密码应包含数字、字母（区分大小写）和特殊字符。
*   新密码不能与旧密码过于相似。

### `passwd` 命令常用选项

![](img/f853d87279f9933e036ccb160d5e4361_24.png)

![](img/f853d87279f9933e036ccb160d5e4361_26.png)

以下是 `passwd` 命令的一些常用选项：

![](img/f853d87279f9933e036ccb160d5e4361_28.png)

![](img/f853d87279f9933e036ccb160d5e4361_30.png)

![](img/f853d87279f9933e036ccb160d5e4361_32.png)

*   `-S, --status`：查看用户的密码状态信息。
    ```bash
    passwd -S username
    ```
*   `-l, --lock`：锁定用户密码，使其无法登录。
    ```bash
    passwd -l username
    ```
*   `-u, --unlock`：解锁已被锁定的用户密码。
    ```bash
    passwd -u username
    ```
*   `-d, --delete`：删除用户密码（清空密码字段）。
    ```bash
    passwd -d username
    ```
*   `--stdin`：从标准输入（如管道）读取密码，用于非交互式脚本中设置密码。
    ```bash
    echo "NewPassword123!" | passwd --stdin username
    ```

## 用户密码文件：`/etc/shadow`

![](img/f853d87279f9933e036ccb160d5e4361_40.png)

![](img/f853d87279f9933e036ccb160d5e4361_42.png)

![](img/f853d87279f9933e036ccb160d5e4361_44.png)

![](img/f853d87279f9933e036ccb160d5e4361_46.png)

用户的实际密码（加密后）及相关策略信息存储在 `/etc/shadow` 文件中。该文件权限严格，普通用户无法读取。

### 文件结构详解

`/etc/shadow` 文件每一行对应一个用户，由冒号 `:` 分隔为9个字段。

以用户 `user1` 的一行为例：`user1:$6$...$...:19077:0:60:7:7::`

1.  **用户名**：用户的登录名。
2.  **加密密码**：经过哈希算法（如SHA512）加密后的密码字符串。如果为 `!` 或 `*`，则表示该账户密码未设置或已锁定。
3.  **上次修改密码的日期**：从1970年1月1日（Unix纪元）到上次修改密码那天的天数。
4.  **密码最短有效期**：密码修改后，必须经过多少天才能再次修改。`0` 表示随时可以修改。
5.  **密码最长有效期**：密码多少天后过期，需要强制修改。例如 `60` 表示60天后过期。
6.  **密码过期前警告天数**：在密码过期前多少天开始向用户发出警告。
7.  **密码过期后宽限天数**：密码过期后，账户还可以宽限登录的天数。在此期间登录会强制要求修改密码。
8.  **账户失效日期**：从1970年1月1日起，账户将在多少天后完全失效（无法登录）。
9.  **保留字段**：目前未使用。

### 管理密码策略

![](img/f853d87279f9933e036ccb160d5e4361_48.png)

![](img/f853d87279f9933e036ccb160d5e4361_50.png)

我们可以直接编辑 `/etc/shadow` 文件或使用 `chage` 命令来修改上述字段，从而实施密码策略。

![](img/f853d87279f9933e036ccb160d5e4361_52.png)

一个常见的需求是**强制用户首次登录时必须修改密码**。这可以通过将“上次修改密码的日期”（第3字段）设置为 `0` 来实现。

![](img/f853d87279f9933e036ccb160d5e4361_54.png)

![](img/f853d87279f9933e036ccb160d5e4361_56.png)

使用 `chage` 命令可以更方便地完成这个操作：
```bash
chage -d 0 username
```
执行此命令后，用户 `username` 下次登录时，系统会强制要求其更改密码。

![](img/f853d87279f9933e036ccb160d5e4361_58.png)

## 切换用户身份命令：`su`

![](img/f853d87279f9933e036ccb160d5e4361_60.png)

`su` 命令用于切换当前用户的身份。

![](img/f853d87279f9933e036ccb160d5e4361_62.png)

![](img/f853d87279f9933e036ccb160d5e4361_64.png)

![](img/f853d87279f9933e036ccb160d5e4361_66.png)

![](img/f853d87279f9933e036ccb160d5e4361_68.png)

![](img/f853d87279f9933e036ccb160d5e4361_69.png)

*   **切换到其他用户**：`su username`。这会切换用户身份，但环境变量可能仍是原用户的。
*   **完全切换到其他用户（包括环境）**：`su - username` 或 `su -l username`。这会模拟一次完整的登录，加载目标用户的环境配置。
*   **退出切换的身份**：执行 `exit` 命令。

**注意**：root 用户切换到任何其他用户都无需输入密码。
