## 默认文件权限（umask）

![](img/8cc3d75b19f391b4800a717a5d120e34_7.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_9.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_11.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_13.png)

上一节我们介绍了文件权限的基本概念，本节中我们来看看当用户创建文件或目录时，系统是如何自动分配初始权限的，这涉及到`umask`（用户文件创建掩码）的概念。

![](img/8cc3d75b19f391b4800a717a5d120e34_15.png)

**什么是umask？**
当用户创建文件或目录时，即使没有显式指定权限，系统也会为其分配一个初始权限。这个分配过程由`umask`值控制。`umask`值定义了需要从默认权限中“屏蔽”掉的权限位。

![](img/8cc3d75b19f391b4800a717a5d120e34_17.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_19.png)

**如何查看和计算umask？**
使用`umask`命令可以查看当前用户的掩码值。该值通常是一个四位八进制数，但我们通常只关注后三位（普通权限位）。`umask`值是“反”的，计算实际权限时，需要用最高权限值减去`umask`值。

![](img/8cc3d75b19f391b4800a717a5d120e34_21.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_23.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_25.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_27.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_29.png)

*   对于目录，最高权限是`777`。例如，`umask`为`022`，则目录权限为 `777 - 022 = 755`。
*   对于文件，最高权限是`666`（因为新建文件通常没有执行权限）。例如，`umask`为`022`，则文件权限为 `666 - 022 = 644`。

**umask值的来源与设置**
`umask`值通常在系统启动时从配置文件中读取。在RHEL 9中，所有用户的默认`umask`值在`/etc/login.defs`文件中被定义为`022`。

![](img/8cc3d75b19f391b4800a717a5d120e34_31.png)

以下是修改`umask`值的方法：
*   **全局修改**：编辑`/etc/bashrc`或`/etc/profile`文件。
*   **用户自定义**：编辑用户家目录下的`.bashrc`文件。

![](img/8cc3d75b19f391b4800a717a5d120e34_33.png)

例如，将`student`用户的`umask`改为`002`：
```bash
echo "umask 002" >> /home/student/.bashrc
```
重新登录后，新创建的目录权限将变为`775`，文件权限变为`664`。

## 进程监控与管理

上一节我们介绍了文件的默认权限，本节中我们来看看Linux系统中另一个核心概念——进程。我们将学习什么是进程、如何查看进程状态、控制作业以及管理进程。

### 什么是进程？

进程可以理解为存储在磁盘中的应用程序被加载到内存中运行的一个实例。当系统启动一个程序时，内核会为其分配资源（如内存空间、CPU时间片），并创建一个包含该程序运行信息（如权限、环境变量、资源分配等）的进程。

![](img/8cc3d75b19f391b4800a717a5d120e34_35.png)

### 进程的创建与状态

一个新的进程通常由现有进程（父进程）通过`fork()`系统调用创建。`fork()`会创建一个与父进程几乎完全相同的子进程副本。随后，子进程通常会通过`exec()`系列函数来“替换”自身，加载并执行新的程序代码。

![](img/8cc3d75b19f391b4800a717a5d120e34_37.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_39.png)

进程在其生命周期中会处于不同的状态。以下是常见的进程状态代码及其含义：

*   **D**：不可中断的休眠状态（通常发生在等待I/O时，如磁盘写入）。
*   **R**：进程正在运行或处于可运行状态（在运行队列中）。
*   **S**：可中断的休眠状态（等待某个事件完成）。
*   **T**：进程被作业控制信号停止。
*   **Z**：僵尸进程。子进程已终止，但其父进程尚未回收其资源。

![](img/8cc3d75b19f391b4800a717a5d120e34_41.png)

可以使用`ps`或`top`命令查看进程状态。例如，`ps aux`命令输出中的`STAT`列显示了进程状态。

![](img/8cc3d75b19f391b4800a717a5d120e34_43.png)

### 控制作业（Jobs）

![](img/8cc3d75b19f391b4800a717a5d120e34_45.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_47.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_49.png)

在单个Shell会话中，默认一次只能在前台运行一个命令或作业。为了在同一个会话中运行多个任务，可以将作业放入后台。

![](img/8cc3d75b19f391b4800a717a5d120e34_51.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_53.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_55.png)

以下是管理后台作业的方法：
*   **将命令放入后台运行**：在命令末尾添加`&`符号，例如 `sleep 120 &`。
*   **查看后台作业**：使用`jobs`命令查看当前会话的后台作业。
*   **将后台作业调至前台**：使用`fg %作业编号`。
*   **暂停与恢复作业**：使用`Ctrl+Z`暂停前台作业，使用`bg %作业编号`在后台恢复运行被暂停的作业。

### 管理进程

![](img/8cc3d75b19f391b4800a717a5d120e34_57.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_59.png)

用户可以通过向进程发送信号来控制进程的行为。最常用的命令是`kill`。

以下是常用的信号：
*   **SIGINT (2)**：中断信号，相当于`Ctrl+C`。
*   **SIGKILL (9)**：强制终止信号。进程无法捕获或忽略此信号。
*   **SIGTERM (15)**：终止信号。这是`kill`命令默认发送的信号，允许进程进行清理工作。
*   **SIGSTOP (19)**：停止（暂停）信号。
*   **SIGCONT (18)**：继续运行信号。

![](img/8cc3d75b19f391b4800a717a5d120e34_61.png)

管理进程的命令示例：
*   **终止单个进程**：`kill -9 PID`
*   **终止进程组（同名进程）**：`killall -9 进程名`
*   **终止特定用户的进程**：`pkill -u 用户名`

![](img/8cc3d75b19f391b4800a717a5d120e34_63.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_65.png)

> **警告**：`SIGKILL (9)`信号非常强大，可能导致数据丢失或状态不一致，应谨慎使用。

![](img/8cc3d75b19f391b4800a717a5d120e34_67.png)

![](img/8cc3d75b19f391b4800a717a5d120e34_69.png)

### 监控系统负载

系统负载平均值（Load Average）是衡量系统繁忙程度的重要指标。它表示在特定时间间隔内，系统处于可运行状态和不可中断状态的平均进程数。

使用`uptime`命令可以查看负载平均值，它会显示最近1分钟、5分钟和15分钟的平均负载。
```bash
uptime
```
解读负载平均值时，需要参考系统的CPU核心数。理想情况下，负载平均值应接近但不超过CPU核心数。如果持续高于核心数，说明系统可能过载。

`top`命令是一个动态的、交互式的进程监控工具，可以实时查看系统资源使用情况（CPU、内存）和进程列表。其界面主要包含以下信息：
1.  系统时间、运行时间、登录用户数。
2.  负载平均值。
3.  任务总数及状态（运行、休眠、停止、僵尸）。
4.  CPU使用率概览（用户态、系统态、空闲、等待I/O等）。
5.  物理内存和交换空间使用情况。
6.  详细的进程列表。
