# 华为云PaaS微服务治理技术：P134：12-微服务治理-容错策略 🛡️

在本节课中，我们将要学习微服务治理中的容错策略。容错策略是保障服务调用稳定性的重要手段，它定义了当服务消费者调用一个异常的服务实例时，系统应如何处理。

## 容错策略概述

![](img/2f255e2289365e5a2a2d94dc7506623c_1.png)

上一节我们介绍了负载均衡等策略，本节中我们来看看容错策略。容错策略是指服务调用者（即服务消费方）在访问服务实例时，若该服务实例出现异常，系统所采取的处理策略。

![](img/2f255e2289365e5a2a2d94dc7506623c_3.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_5.png)

例如，服务A需要调用服务B。当服务A发起请求时，发现目标服务B的某个实例已经宕机。此时，服务A的处理方式就是容错策略。它可以选择直接返回失败，也可以尝试去调用服务B的其他健康实例。

![](img/2f255e2289365e5a2a2d94dc7506623c_7.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_9.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_10.png)

## 核心容错策略

容错策略主要围绕“当调用失败时，是否尝试重连”以及“如何重连”来设计。以下是四种主要的容错策略：

![](img/2f255e2289365e5a2a2d94dc7506623c_12.png)

1.  **Failover（故障切换）**：在不同的服务器上尝试连接其他实例。
2.  **Failfast（快速失败）**：不再尝试连接，直接返回失败结果。
3.  **Failsafe（安全失败）**：在同一服务器上尝试连接其他实例。
4.  **Failback（失败自动恢复）**：允许设置在同一服务器上尝试重连的次数。这是一个可配置的整数，例如 `retries = 3`。

## 策略配置与测试

理解了策略类型后，我们通过一个实际场景来测试容错策略的效果。测试的关键在于模拟服务实例故障的场景。

### 测试场景搭建

![](img/2f255e2289365e5a2a2d94dc7506623c_23.png)

首先，我们确保有两个相同的服务实例（例如 `portal-view`）在运行，端口分别为40100和40200。服务消费者（例如网关）通过负载均衡策略调用它们。

![](img/2f255e2289365e5a2a2d94dc7506623c_25.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_27.png)

为了更清晰地演示容错效果，我们先将网关的负载均衡策略设置为**会话粘滞（session sticky）**。这意味着来自同一用户的请求会持续发送到同一个服务实例。

![](img/2f255e2289365e5a2a2d94dc7506623c_29.png)

**代码示例：负载均衡策略配置**
```yaml
# 假设的配置，表示启用基于Cookie的会话粘滞
loadbalancer:
  strategy: session-sticky
  sticky-session: true
```

![](img/2f255e2289365e5a2a2d94dc7506623c_31.png)

### 测试容错机制

![](img/2f255e2289365e5a2a2d94dc7506623c_33.png)

1.  初始状态下，网关请求正常。
2.  我们手动停止网关当前正在访问的那个 `portal-view` 实例（例如端口40100的实例）。
3.  由于会话粘滞，后续请求仍会尝试访问已停止的40100实例，因此所有请求都会失败。

![](img/2f255e2289365e5a2a2d94dc7506623c_35.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_37.png)

此时，我们为网关配置容错策略。选择 **Failback** 策略，它允许在请求失败时，在同一服务器组内尝试其他实例。

![](img/2f255e2289365e5a2a2d94dc7506623c_39.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_41.png)

**操作步骤简述：**
1.  在治理控制台，选择网关服务。
2.  进入“容错”配置项，针对 `portal-view` 服务新增策略。
3.  选择 `Failback` 模式并保存。

![](img/2f255e2289365e5a2a2d94dc7506623c_43.png)

配置生效后，再次发起请求。虽然负载均衡策略仍试图将请求定向到已宕机的实例，但容错策略会介入，使这次调用自动切换到另一个健康的 `portal-view` 实例（端口40200），从而恢复服务。

![](img/2f255e2289365e5a2a2d94dc7506623c_45.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_46.png)

![](img/2f255e2289365e5a2a2d94dc7506623c_47.png)

这个测试表明，容错策略能够有效提升系统的可用性。在实际开发中，通常建议启用类似 `Failover` 或 `Failback` 的容错策略，以确保单个实例故障不会导致整个调用链失败。

![](img/2f255e2289365e5a2a2d94dc7506623c_49.png)

此外，还可以配置重试次数（如 `retries = 2`），以避免无限重试对系统资源造成不必要的消耗。

![](img/2f255e2289365e5a2a2d94dc7506623c_51.png)

**公式/概念强调：**
核心行为可以概括为：**调用失败 → 根据策略决策 → [放弃 / 重试其他实例]**。

![](img/2f255e2289365e5a2a2d94dc7506623c_53.png)

本节课中我们一起学习了微服务治理中的容错策略。我们首先明确了容错策略的定义：它是服务消费者对异常服务实例的调用处理方案。然后，我们介绍了四种常见的策略：故障切换、快速失败、安全失败和失败自动恢复。最后，通过一个结合了会话粘滞负载均衡的测试场景，我们实践了如何配置和验证 `Failback` 容错策略，观察到它如何自动将失败请求切换到健康实例，从而保障了服务的连续性。正确配置容错策略是构建高可用微服务系统的重要一环。
