# Linux运维：P27：逻辑卷扩容与RAID磁盘阵列

![](img/8e90719cb0e3faa3045378cefe7fb335_1.png)

在本节课中，我们将学习逻辑卷的扩容操作，并初步了解RAID磁盘阵列的基本概念。逻辑卷管理（LVM）的核心优势在于其动态调整存储空间的能力，而RAID则用于提升数据的可靠性和性能。

## 逻辑卷扩容操作

![](img/8e90719cb0e3faa3045378cefe7fb335_3.png)

上一节我们介绍了逻辑卷的创建。本节中，我们来看看如何对已创建的逻辑卷进行扩容。

![](img/8e90719cb0e3faa3045378cefe7fb335_7.png)

### 扩容的必要性

![](img/8e90719cb0e3faa3045378cefe7fb335_13.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_15.png)

在企业环境中，很难精确预估数据未来的增长量。例如，服务器的根分区空间需求会随着运行时间增长。使用逻辑卷存储数据，可以在空间不足时进行动态扩容，而无需在初始分配时预留过多空间。逻辑卷的核心思想就是支持在线、动态的扩容。

![](img/8e90719cb0e3faa3045378cefe7fb335_17.png)

### 检查当前空间状态

在对逻辑卷扩容前，需要先确认卷组是否有足够的剩余空间。

**命令示例：**
```bash
vgs  # 查看卷组信息及剩余空间
lvs  # 查看逻辑卷当前大小
```

### 扩展逻辑卷空间

确认卷组有空间后，即可使用 `lvextend` 命令扩展逻辑卷。

**命令格式：**
```bash
lvextend -L +[要增加的大小] [逻辑卷设备路径]
```
*   `-L +19G`：表示在原有基础上增加19GB空间。
*   `/dev/csystemvg/mylv`：是目标逻辑卷的路径。

![](img/8e90719cb0e3faa3045378cefe7fb335_19.png)

**重要说明：** 扩容逻辑卷时，无需指定卷组名。逻辑卷会自动从其所属的卷组中申请空间。

![](img/8e90719cb0e3faa3045378cefe7fb335_21.png)

### 扩展文件系统

![](img/8e90719cb0e3faa3045378cefe7fb335_23.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_25.png)

完成逻辑卷的空间扩展后，其内部的**文件系统**并不会自动增大。必须手动扩展文件系统，操作系统才能识别并使用新增的空间。

**命令示例（针对XFS文件系统）：**
```bash
xfs_growfs [挂载点目录]
```
例如，如果逻辑卷挂载在 `/webback` 目录，则执行 `xfs_growfs /webback`。

![](img/8e90719cb0e3faa3045378cefe7fb335_27.png)

**操作顺序总结：**
1.  `lvextend`：扩展逻辑卷的物理空间。
2.  `xfs_growfs`（或 `resize2fs` 用于ext系列）：扩展逻辑卷内的文件系统。

![](img/8e90719cb0e3faa3045378cefe7fb335_29.png)

完成这两步后，使用 `df -h` 命令即可看到容量已真正增加。

### 扩展卷组空间

![](img/8e90719cb0e3faa3045378cefe7fb335_31.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_32.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_34.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_36.png)

如果卷组空间也已耗尽，则需要先向卷组中添加新的物理卷（PV），然后再扩容逻辑卷。

![](img/8e90719cb0e3faa3045378cefe7fb335_38.png)

**命令格式：**
```bash
vgextend [卷组名] [新设备路径]
```
新设备可以是未使用的磁盘分区，甚至是整块新硬盘。

## 逻辑卷的缩减与删除

### 关于缩减

![](img/8e90719cb0e3faa3045378cefe7fb335_40.png)

逻辑卷支持缩减，但操作风险极高，且**必须离线（卸载状态）**进行。缩减顺序与扩容相反：
1.  卸载文件系统。
2.  缩减文件系统大小。
3.  缩减逻辑卷大小。
由于可能造成数据丢失，企业生产环境中极少进行缩容操作。

![](img/8e90719cb0e3faa3045378cefe7fb335_42.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_44.png)

### 删除逻辑卷

![](img/8e90719cb0e3faa3045378cefe7fb335_46.png)

若需删除整个逻辑卷配置，需按以下顺序操作：
1.  卸载文件系统，并移除 `/etc/fstab` 中的自动挂载配置。
2.  删除逻辑卷：`lvremove /dev/csystemvg/mylv`
3.  删除卷组：`vgremove csystemvg`
4.  （可选）删除物理卷属性：`pvremove [设备路径]`，或使用 `mkfs -F` 强制格式化以复用磁盘。

**警告：** 删除操作会清除该逻辑卷上的所有数据，且不可恢复。

## RAID磁盘阵列简介

接下来，我们初步了解RAID技术。RAID（独立磁盘冗余阵列）通过将多块物理磁盘组合起来，实现提升性能、增加容量或提供数据冗余的目的。

![](img/8e90719cb0e3faa3045378cefe7fb335_48.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_49.png)

### RAID的作用

![](img/8e90719cb0e3faa3045378cefe7fb335_51.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_53.png)

RAID的主要目标是：
*   **提高数据可靠性（冗余）**：部分RAID级别允许一块或多块磁盘损坏时不丢失数据。
*   **提升读写性能**：通过并行读写多个磁盘实现。
*   **整合存储容量**：将多块小容量磁盘组合为一个大逻辑盘。

![](img/8e90719cb0e3faa3045378cefe7fb335_55.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_56.png)

### RAID与LVM的结合

![](img/8e90719cb0e3faa3045378cefe7fb335_58.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_59.png)

![](img/8e90719cb0e3faa3045378cefe7fb335_61.png)

在实际的高可用存储方案中，RAID和LVM常结合使用：
1.  底层使用多块硬盘组建RAID，提供数据冗余保护。
2.  将RAID阵列形成的“大硬盘”作为物理卷（PV）加入卷组（VG）。
3.  在卷组上创建可灵活扩容的逻辑卷（LV）供系统使用。

![](img/8e90719cb0e3faa3045378cefe7fb335_63.png)

这种架构既利用了RAID的数据安全性，又获得了LVM的空间管理灵活性。

### RAID级别概述

RAID有不同级别（如RAID0, RAID1, RAID5, RAID10等），各有优缺点：
*   **RAID0（条带化）**：提升读写性能，但没有冗余，一块磁盘损坏即导致所有数据丢失。
*   **RAID1（镜像）**：提供数据冗余（完整备份），但磁盘利用率只有50%。
*   更高级别的RAID（如5，10）在性能、冗余和成本间取得不同平衡。

关于RAID的软件配置（如使用 `mdadm` 工具）将在后续课程中详细展开。

## 总结

本节课中我们一起学习了：
1.  **逻辑卷扩容**的完整流程：检查空间 -> `lvextend` 扩展LV -> `xfs_growfs` 扩展文件系统。
2.  当卷组空间不足时，使用 `vgextend` 命令添加新的物理磁盘或分区。
3.  逻辑卷的**缩减和删除**操作风险高，需谨慎进行，且通常需要离线操作。
4.  初步认识了**RAID磁盘阵列**的概念、作用及其与LVM结合使用的优势。

![](img/8e90719cb0e3faa3045378cefe7fb335_65.png)

掌握逻辑卷的动态管理能力，是Linux系统运维中高效、灵活管理存储空间的关键技能。
