## RPM软件包管理

RPM（Red Hat Package Manager）是红帽系列Linux发行版中用于软件包管理的核心工具。它允许用户安装、查询、验证、升级和卸载软件包。

上一节我们介绍了RPM的基本概念，本节中我们来看看如何使用RPM命令进行软件包管理。

以下是RPM常用的命令格式：

![](img/dff09b6f4cbf76b1007ae0a021556843_9.png)

*   **安装软件包**：`rpm -ivh 软件包全名.rpm`
    *   `-i`：安装
    *   `-v`：显示详细信息
    *   `-h`：显示安装进度条
*   **查询已安装的软件包**：`rpm -qa | grep 软件包名`
    *   `-qa`：查询所有已安装的包
*   **卸载软件包**：`rpm -e 软件包名`
    *   `-e`：卸载

![](img/dff09b6f4cbf76b1007ae0a021556843_11.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_13.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_15.png)

RPM的一个主要缺点是它不会自动处理软件包之间的依赖关系。例如，安装一个软件可能需要先安装其他多个软件包（依赖包），而RPM需要你手动逐个解决这些依赖，过程较为繁琐。

## YUM本地软件仓库搭建

![](img/dff09b6f4cbf76b1007ae0a021556843_19.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_21.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_23.png)

为了解决RPM的依赖问题，YUM（Yellowdog Updater, Modified）工具应运而生。YUM能够自动从软件仓库中下载软件包及其所有依赖项。首先，我们需要配置一个软件仓库。

![](img/dff09b6f4cbf76b1007ae0a021556843_25.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_27.png)

现在软件包有了之后，我们现在应该学习怎么去搭建本地仓库了。本地仓库的软件包就在本地的镜像文件里面，那我们现在要干的事情，就是我怎么去搭建这个仓库。

本地仓库也叫本地的YUM源。“源”指的是来源，本地YUM源就是这个软件包的来源是在我的本机，所以就叫本地软件仓库。

那怎么配呢？第一，路径严格要求在 `/etc/yum.repos.d/` 下边。因为在使用YUM命令安装软件包的时候，YUM会自动来到系统的这个路径，去找寻它的仓库文件。这个仓库文件的名字，要以 `.repo` 作为结尾，它才认识。如果你创建的文件结尾不是 `.repo`，它不会去读这个文件里的配置，也就不会按照文件里的内容去下载软件包。所以文件的名字一定要以 `.repo` 结尾。文件名前面叫什么没有严格要求。

![](img/dff09b6f4cbf76b1007ae0a021556843_29.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_31.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_33.png)

所以现在我们想创建一个本地的YUM源，路径一定要在 `/etc/yum.repos.d/`，文件名一定是 `.repo` 作为结尾。这两个条件满足了，就往文件里面开始写配置。

接下来我们直接使用 `vim` 命令在 `/etc/yum.repos.d/` 下面编写一个文件，例如 `local.repo`。

以下是配置本地仓库的步骤和关键配置项：

1.  **仓库名称**：写在第一行，用中括号 `[]` 括起来，名称自定义（不要用中文）。例如：`[local]`
2.  **仓库描述**：用 `name=` 来指定，是对仓库的备注说明。例如：`name=Local CentOS-ISO`
3.  **软件包地址**：用 `baseurl=` 来定义，这是最重要的配置项。使用 `file://` 固定格式来指定本地路径。例如，如果软件包在 `/mnt/centos` 目录下，则配置为：`baseurl=file:///mnt/centos`
    *   **注意**：`file://` 后跟三个斜杠，第三个斜杠是根目录 `/` 的开始。
4.  **启用仓库**：`enabled=1` 代表启用这个仓库（`0` 代表关闭）。如果不写，默认也是启用状态。
5.  **签名检查**：`gpgcheck=0` 代表不检测软件包的GPG签名。对于自己搭建的本地仓库，通常设置为 `0` 以避免签名验证失败导致安装失败。如果是官方网络仓库，则通常设置为 `1` 并配合 `gpgkey=` 指定密钥文件。

![](img/dff09b6f4cbf76b1007ae0a021556843_35.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_37.png)

一个完整的 `local.repo` 文件内容示例：
```
[local]
name=Local CentOS-ISO
baseurl=file:///mnt/centos
enabled=1
gpgcheck=0
```

配置中最重要的是 `baseurl=` 这一行，它指明了软件包的具体存放位置。如果路径指错，YUM将无法找到并下载软件包。

## YUM常用命令学习

![](img/dff09b6f4cbf76b1007ae0a021556843_39.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_41.png)

仓库搭建好后，我们就可以使用YUM命令来方便地管理软件了。YUM最大的优势就是能自动解决依赖关系。

![](img/dff09b6f4cbf76b1007ae0a021556843_43.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_45.png)

以下是YUM的常用命令：

![](img/dff09b6f4cbf76b1007ae0a021556843_47.png)

*   **列出所有可用仓库**：`yum repolist`
    *   这条命令会显示所有已配置仓库的名称、描述和软件包数量。
*   **搜索软件包**：`yum list | grep 关键字`
    *   `yum list` 会列出仓库中所有软件包（包括已安装和未安装的）。结合 `grep` 可以过滤出包含特定关键字的包，非常适合在记不住完整包名时使用。
*   **安装软件包**：`yum install -y 软件包名`
    *   `-y` 参数表示自动回答“yes”，在安装确认提示时直接继续，无需手动输入。
    *   YUM会自动计算并安装所有依赖包。
*   **卸载软件包**：`yum remove 软件包名`
    *   **注意**：默认不会卸载依赖包。卸载时需谨慎。
*   **升级软件包**：`yum update 软件包名`
    *   如果不指定包名，则升级所有可升级的软件包。操作前建议做好备份。

![](img/dff09b6f4cbf76b1007ae0a021556843_49.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_51.png)

例如，安装 `vim` 编辑器只需一条命令：`yum install -y vim`。YUM会自动处理其所需的几十个依赖包，而如果用RPM手动安装则需要逐个解决，非常耗时。

## 配置网络YUM仓库

![](img/dff09b6f4cbf76b1007ae0a021556843_53.png)

除了本地仓库，我们更多时候会使用网络仓库来获取更新、更全的软件包。配置网络仓库的方法与本地类似，只是 `baseurl=` 指向的是网络地址（HTTP/HTTPS）。

例如，配置Nginx官方YUM仓库：
1.  在 `/etc/yum.repos.d/` 下创建 `nginx.repo` 文件。
2.  将官方提供的仓库配置（通常包含 `baseurl=` 和 `gpgkey=`）复制到文件中。
3.  保存退出即可。

![](img/dff09b6f4cbf76b1007ae0a021556843_55.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_57.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_59.png)

![](img/dff09b6f4cbf76b1007ae0a021556843_61.png)

网络仓库通常会启用GPG检查（`gpgcheck=1`）以确保软件包来源可信，因此配置中会包含 `gpgkey=` 来指定签名密钥的地址。
