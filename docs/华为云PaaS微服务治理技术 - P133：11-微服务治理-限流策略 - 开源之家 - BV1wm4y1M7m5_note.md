# 华为云PaaS微服务治理技术：P133：11-微服务治理-限流策略 🚦

在本节课中，我们将要学习微服务治理中的限流策略。限流策略是保障微服务稳定运行、合理分配系统资源的关键技术之一。

## 概述

上一节我们介绍了负载均衡策略，本节中我们来看看限流策略。限流策略主要解决微服务之间的流量分配问题，其核心目标是保证每个微服务在自己的资源池内运行，互不影响。

## 什么是限流策略？🤔

![](img/90125c773cb86a63ff2bf1aaa9023899_1.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_2.png)

服务器上通常会运行多个容器，每个容器承载一个微服务应用。这些应用共享服务器有限的资源（如CPU、内存、线程池）。如果某个微服务（例如应用A）处理了过多的请求，占用了大量资源，就会导致其他微服务（例如应用B）因资源不足而无法正常运行。

限流策略就是为每个微服务设定一个处理能力的上限，例如每秒处理的事务数（QPS）。通过限制QPS，可以防止单个服务耗尽所有资源，从而保障整个系统的稳定性和可用性。

![](img/90125c773cb86a63ff2bf1aaa9023899_4.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_5.png)

## 核心概念：QPS

![](img/90125c773cb86a63ff2bf1aaa9023899_7.png)

QPS（Queries Per Second）是限流策略中的核心指标，代表**每秒处理的事务数**。其公式可以表示为：
`QPS = 请求数 / 时间（秒）`
在配置中，我们通过设置一个具体的数值来限制微服务的最大处理能力。

![](img/90125c773cb86a63ff2bf1aaa9023899_9.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_11.png)

## 如何配置限流策略？⚙️

![](img/90125c773cb86a63ff2bf1aaa9023899_13.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_15.png)

限流策略通常在**服务提供方**进行配置。以下是配置限流策略的步骤。

![](img/90125c773cb86a63ff2bf1aaa9023899_17.png)

1.  在微服务治理平台，找到目标服务提供方（例如 `port-view` 服务）。
2.  进入“限流”配置栏，点击“新增”。
3.  选择需要对其限流的消费方服务（例如 `gateway` 网关）。
4.  设置允许的QPS值（例如设置为 `1`，代表每秒最多处理1个来自该消费方的请求）。
5.  保存配置，策略会立即生效。

![](img/90125c773cb86a63ff2bf1aaa9023899_19.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_20.png)

## 策略测试与验证 🧪

![](img/90125c773cb86a63ff2bf1aaa9023899_21.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_23.png)

配置完成后，我们可以通过模拟请求来验证限流效果。

![](img/90125c773cb86a63ff2bf1aaa9023899_25.png)

*   **测试场景一：针对网关的限流**
    当我们为 `port-view` 服务设置针对 `gateway` 的QPS为1后，快速连续通过网关访问 `port-view` 服务。第一次请求会成功，但紧随其后的快速请求将会被拒绝，返回限流相关的错误提示。

![](img/90125c773cb86a63ff2bf1aaa9023899_26.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_27.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_29.png)

*   **测试场景二：针对其他消费方的限流**
    我们可以为 `port-view` 服务新增一条针对 `learning` 服务的限流规则（例如QPS=2）。此时，通过 `learning` 服务间接调用 `port-view` 的请求也会受到限制。而 `gateway` 直接调用 `learning` 的流量则不受此规则影响。

![](img/90125c773cb86a63ff2bf1aaa9023899_31.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_33.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_35.png)

*   **策略的灵活调整**
    云平台治理的优势在于策略可以动态调整。如果不再需要对某个消费方限流，只需在控制台删除或修改对应的限流规则即可，更改会实时生效，无需重启服务。

![](img/90125c773cb86a63ff2bf1aaa9023899_37.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_39.png)

## 总结

![](img/90125c773cb86a63ff2bf1aaa9023899_41.png)

![](img/90125c773cb86a63ff2bf1aaa9023899_42.png)

本节课中我们一起学习了微服务治理的限流策略。我们了解到，限流通过控制每个服务的QPS，解决了多服务共享资源时的流量分配问题，防止资源被单一服务过度占用，从而提升了整个微服务架构的健壮性和可靠性。在实际应用中，需要根据业务场景、资源情况和性能需求，为每个微服务合理设置限流阈值。
