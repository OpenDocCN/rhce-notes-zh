Linux系统基于GNU GPL协议发布，是一个自由开源的操作系统。红帽企业版（RHEL）是其中重要的商业发行版，以其稳定性著称。社区版如CentOS则提供免费的替代选择。

![](img/05621b55fbfa4b652434552e8f859246_11.png)

![](img/05621b55fbfa4b652434552e8f859246_13.png)

红帽认证体系分为多个层级和方向：
*   **RHCSA（红帽认证系统管理员）**：基础系统管理技能认证。
*   **RHCE（红帽认证工程师）**：在RHCSA基础上，侧重自动化运维技能（如Ansible）。
*   **RHCA（红帽认证架构师）**：高级认证，包含多个专业方向，如云计算、安全、存储等。

RHCE 8.0的考试相比7.0有了显著变化，从传统的服务配置考核转向更注重自动化运维和Shell脚本能力。考试形式为100%上机实操，满分300分，210分通过。考试环境为全英文。

![](img/05621b55fbfa4b652434552e8f859246_15.png)

## 学习建议与课程安排

本课程将从零基础开始，涵盖系统安装、命令行操作、服务配置等，并额外补充了链路聚合和Web服务（Apache）等实用内容。课程为期13天，节奏紧凑。

**给初学者的核心建议如下：**
*   **动手实践**：所有操作必须亲手完成，仅靠观看无法掌握。
*   **做好笔记**：跟随课程内容记录关键命令和步骤。
*   **反复练习**：考试通过的关键在于对实验环境的熟练操作。
*   **理解原理**：不仅要会操作，还要理解命令和配置背后的逻辑。

## 实验环境准备

为了进行后续学习，我们需要搭建自己的实验环境。以下是软硬件的最低要求：

**硬件要求：**
*   **内存**：最低8GB（用于RHCSA学习），建议16GB或以上（用于Ansible等后续内容）。
*   **CPU**：支持虚拟化技术（需在BIOS中开启）。
*   **硬盘**：固态硬盘（SSD）可显著提升虚拟机运行速度。
*   **存储空间**：至少50GB可用空间。

**软件要求：**
*   **虚拟化软件**：VMware Workstation 15.x 或更高版本。
*   **操作系统镜像**：RHEL 8.0 或 8.1/8.2 安装镜像。

## 安装RHEL 8.0 🖥️

上一节我们介绍了课程概况和环境要求，本节中我们来看看如何一步步安装RHEL 8.0系统。

### 创建虚拟机

![](img/05621b55fbfa4b652434552e8f859246_19.png)

![](img/05621b55fbfa4b652434552e8f859246_21.png)

以下是创建虚拟机的详细步骤：
1.  打开VMware Workstation，点击“文件”->“新建虚拟机”。
2.  选择“自定义（高级）”配置，点击下一步。
3.  虚拟机硬件兼容性选择“Workstation 15.x”，点击下一步。
4.  选择“稍后安装操作系统”，点击下一步。
5.  客户机操作系统选择“Linux”，版本选择“Red Hat Enterprise Linux 8 64位”，点击下一步。
6.  为虚拟机命名（例如 `serverA`），并选择保存位置，点击下一步。
7.  处理器配置：数量1，每个处理器的核心数2（可根据主机性能调整），点击下一步。
8.  内存分配：设置为2048 MB（2GB）或更高，点击下一步。
9.  网络类型：选择“使用网络地址转换（NAT）”，点击下一步。
10. I/O控制器和磁盘类型：保持默认推荐选项（LSI Logic, NVMe），点击下一步。
11. 选择磁盘：选择“创建新虚拟磁盘”，点击下一步。
12. 指定磁盘容量：大小设为20GB，选择“将虚拟磁盘拆分成多个文件”，点击下一步。
13. 指定磁盘文件：使用默认名称，点击下一步。
14. 在完成前，点击“自定义硬件”。
    *   在“新CD/DVD”选项中，选择“使用ISO映像文件”，浏览并选中你下载的RHEL 8 ISO镜像文件。
    *   在“处理器”选项中，勾选“虚拟化Intel VT-x/EPT或AMD-V/RVI”。
15. 点击“关闭”，然后点击“完成”。

### 安装操作系统

虚拟机创建完成后，启动它并开始安装系统。
1.  启动虚拟机，在引导界面选择第一项“Install Red Hat Enterprise Linux 8.x”，按回车。
2.  安装程序加载后，选择语言为“English”，点击“Continue”。
3.  在“INSTALLATION SUMMARY”界面，进行以下关键配置：
    *   **安装源**：确认是“Local media”即可。
    *   **软件选择**：选择“Server with GUI”（带图形界面的服务器），便于初学者操作。
    *   **安装目的地**：点击进入后，在“Storage Configuration”下选择“Custom”。选中你的20GB虚拟磁盘，点击“Done”。在分区方案界面，选择“Standard Partition”或“LVM”均可（后续课程会详解），这里为了简单，可以点击“Click here to create them automatically”让安装程序自动分区，然后点击“Done”。
4.  点击“Begin Installation”开始安装。
5.  安装过程中，需要设置**root密码**和**创建用户**。
    *   点击“Root Password”，设置密码（例如 `redhat`），由于是弱密码，需要点击两次“Done”确认。
    *   点击“User Creation”，创建普通用户（例如用户名 `student`，密码 `student`），同样需要点击两次“Done”确认。
6.  等待安装完成，点击“Reboot System”重启。

### 初始配置与克隆

系统重启后，进行初始设置并克隆出第二台实验机。
1.  重启后，接受许可协议，点击“Finish Configuration”。
2.  在登录界面，点击“Not listed?”，输入用户名 `root` 和密码 `redhat` 登录。
3.  完成初始向导（可选择跳过定位服务等）。
4.  **重要**：完成登录后，立即关机。在VMware中右键点击虚拟机，选择“管理”->“克隆”。选择“虚拟机中的当前状态”，创建“完整克隆”。将克隆机命名为 `serverB` 并保存到不同目录。这样我们就得到了两台初始状态完全相同的虚拟机。

### 配置静态IP地址

我们需要为虚拟机配置一个静态IP地址，以便远程连接和主机间通信。

1.  首先，确定你的VMware NAT网络的网段和网关。打开VMware的“编辑”->“虚拟网络编辑器”，查看“VMnet8 (NAT模式)”的子网IP（例如 `192.168.13.0`）和网关IP（例如 `192.168.13.2`）。
2.  编辑网络配置文件。RHEL 8的网络配置文件位于 `/etc/sysconfig/network-scripts/` 目录下，网卡名通常为 `ens160` 或类似。
    ```bash
    vim /etc/sysconfig/network-scripts/ifcfg-ens160
    ```
3.  修改或确保文件内容如下（请根据你的实际网络修改IP地址）：
    ```bash
    TYPE=Ethernet
    PROXY_METHOD=none
    BROWSER_ONLY=no
    BOOTPROTO=static        # 将dhcp改为static
    DEFROUTE=yes
    IPV4_FAILURE_FATAL=no
    IPV6INIT=yes
    IPV6_AUTOCONF=yes
    IPV6_DEFROUTE=yes
    IPV6_FAILURE_FATAL=no
    IPV6_ADDR_GEN_MODE=stable-privacy
    NAME=ens160
    UUID=你的网卡UUID（保持不变）
    DEVICE=ens160
    ONBOOT=yes              # 确保是yes，开机自启

    # 以下是新增的静态IP配置
    IPADDR=192.168.13.101   # 为serverA分配的IP，serverB可用102
    PREFIX=24               # 子网掩码，相当于255.255.255.0
    GATEWAY=192.168.13.2    # 你的VMnet8网关地址
    DNS1=114.114.114.114    # DNS服务器
    ```
4.  保存并退出编辑器（按 `Esc`，然后输入 `:wq` 回车）。
5.  重新加载网络连接配置并重启网卡：
    ```bash
    # 重新加载配置
    nmcli connection reload

    # 关闭再开启网卡（ens160替换为你的网卡名）
    nmcli connection down ens160
    nmcli connection up ens160
    ```
6.  验证网络配置：
    ```bash
    # 查看IP地址
    ip addr show ens160

    # 测试连通性，先ping网关
    ping 192.168.13.2

    # 再ping另一台虚拟机（配置好serverB后）
    ping 192.168.13.102
    ```
7.  **为serverB重复上述步骤**，注意将主机名改为 `serverb.lab.example.com`，IP地址改为 `192.168.13.102`。
