## 进程监控：`top`命令详解 🔍

上一节我们介绍了基础的进程概念，本节中我们来看看如何使用`top`命令动态地、全面地监控系统进程状态。

`top`命令提供了一个实时更新的系统概览。执行`top`命令后，屏幕会显示多行信息。

### 系统任务摘要区

第一行显示系统整体信息：
*   **当前系统时间**：例如 `14:46:08`。
*   **系统运行时间**：例如 `up 1 day, 2:30`。
*   **当前登录用户数**：例如 `2 users`。
*   **系统平均负载**：显示过去1分钟、5分钟、15分钟的平均负载，例如 `load average: 0.00, 0.01, 0.05`。

![](img/da48cd11fc3b22e50c69bc9609e83e22_7.png)

第二行显示进程统计信息：
*   **总进程数**：`total`
*   **运行中的进程数**：`running`
*   **休眠的进程数**：`sleeping`
*   **停止的进程数**：`stopped`
*   **僵尸进程数**：`zombie`

![](img/da48cd11fc3b22e50c69bc9609e83e22_9.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_11.png)

**僵尸进程**是指已经终止但其父进程尚未对其进行善后处理的进程。如果发现僵尸进程数量不为0，可以使用`ps aux | grep defunct`找到其PID，然后用`kill`命令终止其父进程。

![](img/da48cd11fc3b22e50c69bc9609e83e22_15.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_17.png)

### CPU状态区

![](img/da48cd11fc3b22e50c69bc9609e83e22_19.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_21.png)

第三行显示CPU使用情况百分比。我们需要重点关注的是`id`（空闲百分比）。如果`id`值持续很低（例如低于10%），则表明CPU非常繁忙，需要排查是哪个进程消耗了过多资源。

以下是各列含义：
*   `us`：用户空间进程占用CPU百分比。
*   `sy`：内核空间进程占用CPU百分比。
*   `ni`：改变过优先级的用户进程占用CPU百分比。
*   `id`：**空闲CPU百分比**。
*   `wa`：等待I/O的CPU时间百分比。
*   `hi`：处理硬件中断的CPU时间百分比。
*   `si`：处理软件中断的CPU时间百分比。
*   `st`：被虚拟化偷走的时间百分比。

![](img/da48cd11fc3b22e50c69bc9609e83e22_23.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_25.png)

### 内存与交换分区区

![](img/da48cd11fc3b22e50c69bc9609e83e22_27.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_29.png)

第四、五行分别显示物理内存（`Mem`）和交换分区（`Swap`）的使用情况。但通常查看内存更推荐使用`free -h`命令，因为它以更直观的单位（如G、M）显示。

### 进程列表区

下方区域以列表形式显示各个进程的详细信息，类似于`ps aux`命令的输出。

### `top`命令的交互操作

在`top`运行界面，可以按下特定键进行排序或操作：
*   **`P`**：按CPU使用率降序排序。
*   **`M`**：按内存使用率降序排序。
*   **`k`**：输入后提示输入进程PID以终止该进程。
*   **`q`**：退出`top`。

通过`top`命令，我们可以快速分析系统的健康状态，定位高负载的进程。

## 进程检索：`pgrep`命令 🔎

![](img/da48cd11fc3b22e50c69bc9609e83e22_35.png)

除了使用`ps aux | grep`组合来查找进程，还可以使用`pgrep`命令进行更简洁的进程检索。

![](img/da48cd11fc3b22e50c69bc9609e83e22_37.png)

以下是`pgrep`的常用选项：
*   `-l`：同时输出进程名和PID。
*   `-u <用户名>`：检索指定用户的进程。
*   `-t <终端号>`：检索指定终端的进程。
*   `-x`：进行进程名的精确匹配。

例如：
*   `pgrep -l sshd` 查找所有包含“sshd”的进程并显示名称和PID。
*   `pgrep -u test` 查找用户`test`的所有进程PID。
*   `pgrep -x sleep` 精确查找进程名为`sleep`的进程。

## 进程前后台调度 ⏸️▶️

![](img/da48cd11fc3b22e50c69bc9609e83e22_41.png)

进程可以在前台运行（占用当前终端），也可以在后台运行（不占用终端）。这对于运行耗时任务非常有用。

![](img/da48cd11fc3b22e50c69bc9609e83e22_43.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_45.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_47.png)

### 将进程放入后台

![](img/da48cd11fc3b22e50c69bc9609e83e22_48.png)

在命令末尾加上`&`符号，即可让命令在后台启动。
```bash
sleep 300 &
```
这条命令会让终端休眠300秒，但因为它被放到了后台，所以当前终端可以继续使用。

### 查看后台进程

![](img/da48cd11fc3b22e50c69bc9609e83e22_50.png)

使用`jobs -l`命令可以查看当前会话放入后台的进程列表及其作业编号（`[n]`）。

![](img/da48cd11fc3b22e50c69bc9609e83e22_52.png)

### 前后台调度

*   **`fg %n`**：将后台作业编号为`n`的进程调回**前台**运行。
*   **`bg %n`**：将已暂停（如按`Ctrl+Z`）的后台作业`n`，在**后台恢复**运行。
*   **`Ctrl+Z`**：暂停当前正在前台运行的进程，并将其放入后台（状态为`Stopped`）。

![](img/da48cd11fc3b22e50c69bc9609e83e22_54.png)

例如，一个长时间运行的`ping`命令可以放入后台，避免阻塞终端。

![](img/da48cd11fc3b22e50c69bc9609e83e22_56.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_58.png)

## 终止进程：`kill`, `killall`, `pkill` ⛔

当需要手动结束进程时，有以下几种命令。

### `kill` - 通过PID终止进程

`kill`通过进程ID（PID）来发送信号。最常用的信号是：
*   `15` (`SIGTERM`)：**正常终止**。这是默认信号，允许进程进行清理工作。
*   `9` (`SIGKILL`)：**强制终止**。立即结束进程，不给其清理机会，用于无法正常结束的进程（如僵尸进程）。

命令格式：
```bash
kill -15 1234    # 优雅地终止PID为1234的进程
kill -9 1234     # 强制杀死PID为1234的进程
```

### `killall` - 通过进程名终止进程

![](img/da48cd11fc3b22e50c69bc9609e83e22_60.png)

`killall`通过进程名来终止**所有**同名进程，更为彻底。
```bash
killall sleep          # 终止所有名为sleep的进程
killall -9 mysql       # 强制终止所有名为mysql的进程
```

![](img/da48cd11fc3b22e50c69bc9609e83e22_62.png)

### `pkill` - 高级进程终止

![](img/da48cd11fc3b22e50c69bc9609e83e22_64.png)

`pkill`功能更强大，可以根据进程名、用户名、终端等多种属性来终止进程。
*   `pkill -u <用户名>`：终止指定用户的所有进程。常用于将用户踢出系统。
*   `pkill -t <终端号>`：终止指定终端上的所有进程。

![](img/da48cd11fc3b22e50c69bc9609e83e22_66.png)

例如：
```bash
pkill -9 -u test      # 强制踢出用户test
pkill -9 -t pts/1     # 强制关闭终端pts/1上的所有进程
```

## 用户登录分析 👥

![](img/da48cd11fc3b22e50c69bc9609e83e22_68.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_72.png)

了解谁正在或曾经登录系统，是安全管理的重要部分。

![](img/da48cd11fc3b22e50c69bc9609e83e22_74.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_76.png)

### 查看当前已登录用户

![](img/da48cd11fc3b22e50c69bc9609e83e22_78.png)

![](img/da48cd11fc3b22e50c69bc9609e83e22_80.png)

*   `users`：最简洁，仅列出已登录的用户名。
*   `who`：显示详细信息，包括用户名、终端、登录时间和来源IP地址。
*   `w`：**最详细**。不仅显示`who`的信息，还显示系统当前时间、运行时长、平均负载，以及每个用户正在执行的任务和CPU占用情况。

### 查看历史登录记录

*   `last`：查看**历史登录成功**的记录。显示用户名、终端、登录IP、登录起止时间。
*   `lastb`：查看**历史登录失败**的记录。对于排查恶意登录尝试非常有用。

这些命令的信息通常来源于系统日志文件，如`/var/log/wtmp`（`last`）和`/var/log/btmp`（`lastb`）。

## 总结 📝

![](img/da48cd11fc3b22e50c69bc9609e83e22_82.png)

本节课中我们一起学习了Linux进程与用户管理的核心操作：
1.  使用 **`top`** 命令动态监控系统状态和进程资源占用，并学会按CPU或内存排序。
2.  使用 **`pgrep`** 命令快速检索进程。
3.  掌握进程的**前后台调度**：使用 **`&`** 将进程放入后台，使用 **`jobs`**、**`fg`**、**`bg`** 进行管理。
4.  学习三种终止进程的方法：通过PID的 **`kill`**、通过进程名的 **`killall`** 以及功能更强的 **`pkill`**，并理解了`SIGTERM(15)`和`SIGKILL(9)`信号的区别。
5.  使用 **`who`**、**`w`** 分析当前登录用户，使用 **`last`**、**`lastb`** 审计历史登录成功与失败记录。
