## 使用fdisk进行MBR分区 🛠️

上一节我们介绍了MBR分区格式的理论知识，本节中我们来看看如何实际操作。`fdisk`命令是用于查看磁盘使用情况和进行MBR格式分区的主要工具。

![](img/fed6a7e385d924a517de0e14858177d8_11.png)

命令的基本格式是：
```bash
fdisk [设备路径]
```
设备路径通常是`/dev/`目录下的硬盘设备文件，例如`/dev/sdb`。

![](img/fed6a7e385d924a517de0e14858177d8_13.png)

以下是`/dev`目录下一些常见的设备文件类型：

![](img/fed6a7e385d924a517de0e14858177d8_33.png)

*   **硬盘设备**：通常以`sd`开头，如`sda`、`sdb`、`sdc`。
*   **虚拟终端设备**：以`tty`开头，如`tty1`、`tty2`，代表用户可以登录的终端。
*   **光驱设备**：通常表示为`sr0`，其快捷方式是`cdrom`。
*   **空设备（黑洞设备）**：`/dev/null`。任何写入此设备的数据都会被丢弃。常用于清空文件内容，例如：
    ```bash
    find /var/log -name "*.log" -size +10M -exec cp /dev/null {} \;
    ```
*   **随机数设备**：`/dev/random`和`/dev/urandom`，用于生成随机数。
*   **回环网卡**：`lo`，代表本机网络回环地址。

### 分区实战演示

现在，我们以一块新硬盘（例如`/dev/sdb`）为例，演示如何使用`fdisk`进行MBR分区。

1.  **进入分区界面**：
    ```bash
    fdisk /dev/sdb
    ```
    输入命令后，所有操作都暂存于内存，直到执行保存命令才会写入磁盘。

2.  **获取帮助**：在`fdisk`交互界面中输入`m`可以查看所有可用命令。

3.  **常用命令**：
    *   `p`：打印当前磁盘的分区表。可以查看磁盘标签类型（如`DOS`代表MBR格式）、分区数量、大小等信息。
    *   `n`：创建一个新分区。
    *   `d`：删除一个分区。
    *   `w`：保存分区表并退出。
    *   `q`：不保存更改并退出。

4.  **创建分区流程**：
    *   输入`n`创建新分区。
    *   选择分区类型：`p`为主分区，`e`为扩展分区。MBR最多允许**4个主分区**，或**3个主分区+1个扩展分区**（扩展分区内可再划分多个逻辑分区）。
    *   指定分区编号（1-4）。
    *   指定起始扇区（通常直接回车使用默认值）。
    *   指定分区大小，支持`K`， `M`， `G`单位，例如`+10G`表示分配10GB空间。
    *   重复以上步骤创建所需分区。

5.  **处理四个分区的限制**：
    当需要划分超过四个分区时，必须使用扩展分区。例如，先创建3个主分区（sdb1， sdb2， sdb3），然后将第四个分区创建为扩展分区（sdb4）。接着，在扩展分区内继续创建逻辑分区（编号从5开始，如sdb5， sdb6...）。

6.  **保存与退出**：分区完成后，输入`w`保存所有更改。

**如何区分主分区、扩展分区和逻辑分区？**
对于MBR格式的磁盘，可以通过`fdisk -l /dev/sdb`查看。分区编号1-4的可能是主分区或扩展分区，其中会明确标出`Extended`的为扩展分区。编号5及以上的分区一定是逻辑分区。

## 文件系统介绍 💾

分区创建后，还不能直接存储数据，必须先进行**格式化**，为其创建一个**文件系统**。文件系统是操作系统用于管理分区中文件的**一套规则和方法**，负责文件的存储、检索、更新和安全控制。

![](img/fed6a7e385d924a517de0e14858177d8_47.png)

![](img/fed6a7e385d924a517de0e14858177d8_49.png)

### 常见的Linux文件系统

1.  **ext4**：
    *   是CentOS 6等旧系统的默认文件系统。
    *   支持日志功能，可在系统意外崩溃后帮助恢复数据。
    *   单个文件系统最大支持**1EB**的分区容量。
    *   单个文件最大支持**16TB**。

2.  **XFS**：
    *   是CentOS 7/8 及RHEL 7/8等现代系统的默认文件系统。
    *   同样具备日志功能，且数据恢复速度通常比ext4更快。
    *   单个文件系统最大支持**8EB**的分区容量。
    *   单个文件最大支持**500TB**。
    *   被认为是目前主流的、高性能的文件系统。

### 格式化分区

![](img/fed6a7e385d924a517de0e14858177d8_51.png)

使用`mkfs`命令为分区创建文件系统。

将分区格式化为XFS文件系统：
```bash
mkfs.xfs /dev/sdb1
```
将分区格式化为ext4文件系统：
```bash
mkfs.ext4 /dev/sdb1
```
格式化后，可以使用`lsblk -f`命令查看分区的文件系统类型。

![](img/fed6a7e385d924a517de0e14858177d8_53.png)

## 挂载介绍 🔗

![](img/fed6a7e385d924a517de0e14858177d8_55.png)

在Linux中，用户不能直接访问硬盘分区（如`/dev/sdb1`）。必须通过一个称为**挂载**的操作，将分区关联到目录树上的一个空目录（称为**挂载点**），才能通过该目录访问分区中的存储空间。

![](img/fed6a7e385d924a517de0e14858177d8_57.png)

![](img/fed6a7e385d924a517de0e14858177d8_59.png)

![](img/fed6a7e385d924a517de0e14858177d8_61.png)

![](img/fed6a7e385d924a517de0e14858177d8_63.png)

### 挂载与卸载命令

![](img/fed6a7e385d924a517de0e14858177d8_65.png)

![](img/fed6a7e385d924a517de0e14858177d8_67.png)

*   **挂载**：使用`mount`命令。
    ```bash
    mount [设备路径] [挂载点目录]
    ```
    例如，将格式化的`/dev/sdb1`挂载到`/mnt/data`：
    ```bash
    mount /dev/sdb1 /mnt/data
    ```

*   **卸载**：使用`umount`命令。
    ```bash
    umount [挂载点目录] 或 umount [设备路径]
    ```
    例如：
    ```bash
    umount /mnt/data
    ```

### 挂载的核心概念与注意事项

1.  **目录是入口**：文件存储在目录中，实质是通过目录这个“通道”写入到背后的分区里。挂载点目录最好是**空目录**，若非空，原内容在挂载期间会被隐藏，卸载后重新可见。

![](img/fed6a7e385d924a517de0e14858177d8_69.png)

![](img/fed6a7e385d924a517de0e14858177d8_70.png)

2.  **一对一关系**：
    *   一个分区不能同时挂载到多个目录。
    *   一个目录也不能同时挂载多个分区。

![](img/fed6a7e385d924a517de0e14858177d8_72.png)

![](img/fed6a7e385d924a517de0e14858177d8_74.png)

3.  **查看挂载信息**：使用`df -h`命令可以查看所有已挂载的文件系统及其使用情况。

![](img/fed6a7e385d924a517de0e14858177d8_76.png)

![](img/fed6a7e385d924a517de0e14858177d8_78.png)

### 工作流程示例

![](img/fed6a7e385d924a517de0e14858177d8_80.png)

假设我们为数据库备份专门划分了空间：
1.  创建挂载点目录：`mkdir /db_backup`
2.  格式化分区：`mkfs.xfs /dev/sdb1`
3.  挂载分区：`mount /dev/sdb1 /db_backup`
4.  现在，所有存入`/db_backup`目录的文件，实际上都存储在`/dev/sdb1`分区中。
5.  当`/dev/sdb1`空间用满后，可以卸载它：`umount /db_backup`，然后挂载一个更大的新分区（如`/dev/sdb2`）到同一个目录，继续使用。
