## 重定向回顾与进阶

![](img/c32f61355ce4986c39b89ab76e9e38fc_17.png)

![](img/c32f61355ce4986c39b89ab76e9e38fc_19.png)

上一节我们介绍了基本的重定向操作，本节中我们来看看重定向的更多细节。

![](img/c32f61355ce4986c39b89ab76e9e38fc_21.png)

![](img/c32f61355ce4986c39b89ab76e9e38fc_23.png)

重定向可以将命令的输出结果保存到文件中。例如，将`ls /`命令的输出保存到当前目录的`test.txt`文件中：
```bash
ls / > test.txt
```
使用`cat test.txt`可以查看文件内容，即根目录的列表。

![](img/c32f61355ce4986c39b89ab76e9e38fc_25.png)

然而，重定向默认只处理命令的**标准输出**。如果命令执行出错，错误信息（标准错误输出）不会被重定向到文件。例如：
```bash
ls a_non_existent_file > aa.txt
```
命令执行错误，但`aa.txt`文件是空的。

![](img/c32f61355ce4986c39b89ab76e9e38fc_27.png)

为了处理错误输出，需要使用特殊的重定向语法：
```bash
ls a_non_existent_file 2> error.txt
```
此时，错误信息会被保存到`error.txt`文件中。

![](img/c32f61355ce4986c39b89ab76e9e38fc_29.png)

![](img/c32f61355ce4986c39b89ab76e9e38fc_31.png)

以下是关于重定向类型的总结：
*   **`>` 或 `>>`**：重定向**标准输出**。`>`表示覆盖，`>>`表示追加。
*   **`2>` 或 `2>>`**：重定向**标准错误输出**。`2>`表示覆盖，`2>>`表示追加。

![](img/c32f61355ce4986c39b89ab76e9e38fc_33.png)

有时，我们希望将命令的标准输出和标准错误输出都重定向到同一个文件。可以使用以下语法：
```bash
command &> file        # 覆盖文件
command &>> file       # 追加到文件
# 或者等价的写法
command > file 2>&1    # 覆盖文件
command >> file 2>&1   # 追加到文件
```
这种写法在脚本或程序后台运行时记录所有输出（包括正常和错误信息）非常有用。

## 管道符与tee命令

![](img/c32f61355ce4986c39b89ab76e9e38fc_39.png)

![](img/c32f61355ce4986c39b89ab76e9e38fc_41.png)

理解了重定向后，我们来看看如何将多个命令串联起来，以及如何同时输出到屏幕和文件。

![](img/c32f61355ce4986c39b89ab76e9e38fc_43.png)

管道符 `|` 的作用是将前一个命令的**标准输出**作为后一个命令的**标准输入**。例如：
```bash
ls -l / | less
```
这个命令将`ls -l /`的输出结果通过管道传递给`less`命令进行分页查看。

![](img/c32f61355ce4986c39b89ab76e9e38fc_45.png)

但是，管道符与单纯的重定向结合使用时需要注意。观察以下命令：
```bash
ls -l / > /tmp/save.out | less /tmp/save.out
```
你会发现`less`可能无法正常显示内容。这是因为`ls -l / > /tmp/save.out`命令的输出被**直接重定向到了文件**，没有产生可供管道传递的标准输出，因此`less`命令没有接收到任何输入。

![](img/c32f61355ce4986c39b89ab76e9e38fc_47.png)

为了解决这个问题，我们需要使用 `tee` 命令。`tee`命令会从标准输入读取数据，同时将其**写入文件**并**输出到标准输出**。这样既能保存数据，又能让数据继续在管道中传递。
```bash
ls -l / | tee /tmp/save.out | less
```
在这个命令中，`ls -l /`的输出先通过`tee`命令写入`/tmp/save.out`文件，同时`tee`命令的输出（即同样的内容）继续传递给`less`命令进行显示。

![](img/c32f61355ce4986c39b89ab76e9e38fc_49.png)

![](img/c32f61355ce4986c39b89ab76e9e38fc_51.png)

简单来说：
*   **管道符 `|`**：用于连接命令，传递标准输出。
*   **`tee` 命令**：用于“分流”，在写入文件的同时，保留输出以供后续管道处理。

![](img/c32f61355ce4986c39b89ab76e9e38fc_53.png)

## 📝 Vim文本编辑器入门

现在，让我们转向Linux中最强大、最常用的文本编辑器之一——Vim。

![](img/c32f61355ce4986c39b89ab76e9e38fc_55.png)

![](img/c32f61355ce4986c39b89ab76e9e38fc_57.png)

Vim是Vi编辑器的增强版，是一个**纯文本编辑器**。它与Windows上的Word等文字处理软件不同，不处理字体、颜色等样式信息（尽管Vim支持语法高亮），专注于文本内容的编辑。Vi编辑器几乎存在于所有Linux系统中，而Vim通常需要额外安装。

![](img/c32f61355ce4986c39b89ab76e9e38fc_59.png)

### Vim的基本模式与工作流程

Vim有多种模式，最核心的是以下三种：
1.  **普通模式**：启动Vim后的默认模式。在此模式下可以移动光标、删除、复制、粘贴文本，但不能直接输入字符。
2.  **插入模式**：在此模式下可以像普通编辑器一样输入和修改文本。按 `i` 键进入插入模式。
3.  **命令行模式**：在普通模式下按 `:` 进入。在此模式下可以执行保存、退出、查找替换等命令。

![](img/c32f61355ce4986c39b89ab76e9e38fc_61.png)

![](img/c32f61355ce4986c39b89ab76e9e38fc_63.png)

基本工作流程为：**启动Vim -> 普通模式 -> 按 `i` 进入插入模式编辑 -> 按 `Esc` 返回普通模式 -> 按 `:` 进入命令行模式 -> 输入 `wq` 保存并退出**。

![](img/c32f61355ce4986c39b89ab76e9e38fc_65.png)

### Vim的快捷操作技巧

在普通模式下，Vim提供了大量高效的快捷键。

**1. 打开文件时的技巧**
*   `vim +n filename`：打开文件并定位到第n行。
*   `vim + filename`：打开文件并定位到最后一行。
*   `vim +/pattern filename`：打开文件并定位到第一个匹配“pattern”字符串的位置。

**2. 光标移动与编辑**
*   `x`：删除光标所在处的字符。
*   `dw`：删除一个单词（从光标处到单词结束）。
*   `dd`：删除当前行。
*   `u`：撤销上一次操作。
*   `yy`：复制当前行。
*   `p`：在光标后粘贴已复制或删除的内容。
*   `3yy`：复制从当前行开始的三行。
*   `r`：进入替换模式，随后输入的字符将直接替换光标处的字符。
*   `R`：进入连续替换模式，随后输入将覆盖后续文本，直到按`Esc`。

**3. 可视模式**
可视模式允许你选择文本块进行操作。
*   `v`：进入字符可视模式，按字符选择文本。
*   `V`：进入行可视模式，按整行选择文本。
*   `Ctrl + v`：进入块可视模式，按矩形列选择文本。
在选择文本后，可以按`d`删除、`y`复制等。

**4. 保存与退出**
在命令行模式下：
*   `:w`：保存文件。
*   `:q`：退出Vim（如果文件有修改未保存，会提示）。
*   `:wq` 或 `:x`：保存并退出。
*   `:q!`：强制退出，不保存任何修改。
