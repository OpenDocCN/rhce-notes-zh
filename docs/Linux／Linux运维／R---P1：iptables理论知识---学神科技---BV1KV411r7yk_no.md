在本节课中，我们将要学习Linux防火墙的核心组件之一：iptables。我们将从基本概念入手，了解其工作原理、核心组件（三表五链）以及数据包的匹配流程。通过本节课的学习，你将能够理解iptables的基本框架，为后续的实际配置打下坚实的理论基础。

## 概述：什么是iptables？

iptables信息包过滤系统实际上由两个组件组成：netfilter和iptables。

*   **netfilter组件**：称为内核空间，是内核的一部分，由一些信息包过滤表组成。这些表包含了内核用来控制信息包过滤处理的规则集。
*   **iptables组件**：称为用户空间，是一种工具，它使得插入、修改和删除信息包过滤表中的规则变得容易。

后期，netfilter和iptables被简称为iptables。iptables是基于内核的防火墙，功能非常强大。iptables内置了filter、NAT和mangle三张表。所有规则配置后立即生效，不需要重启服务。

## 核心概念：三表五链

![](img/db04beb62a68588e9f0095f4bb386098_1.png)

![](img/db04beb62a68588e9f0095f4bb386098_3.png)

上一节我们介绍了iptables的组成，本节中我们来看看其核心的“三表五链”结构。

### 三张表及其作用

以下是iptables中主要的三张表及其功能：

*   **filter表**：负责过滤数据包，是默认的表。
*   **NAT表**：负责网络地址转换。
*   **mangle表**：主要用来修改数据包内容，用于流量整形等。

### 五条链及其作用

![](img/db04beb62a68588e9f0095f4bb386098_5.png)

![](img/db04beb62a68588e9f0095f4bb386098_7.png)

每张表都包含若干条“链”，链是规则生效的位置。以下是主要的五条链：

![](img/db04beb62a68588e9f0095f4bb386098_9.png)

*   **INPUT链**：匹配目标IP是本机的数据包（即发送给本机的数据）。
*   **OUTPUT链**：匹配由本机发出的数据包。
*   **FORWARD链**：匹配流经本机但目标不是本机的数据包（即需要转发的数据）。
*   **PREROUTING链**：在数据包进入路由决策之前，修改目的地址（DNAT）。
*   **POSTROUTING链**：在数据包即将离开本机之前，修改源地址（SNAT）。

> **注意**：实际上还存在一个`raw`表，主要用于处理异常，其优先级最高。但由于不常用，所以我们通常说“三表五链”。

![](img/db04beb62a68588e9f0095f4bb386098_11.png)

### 查看表与链

![](img/db04beb62a68588e9f0095f4bb386098_13.png)

![](img/db04beb62a68588e9f0095f4bb386098_15.png)

你可以使用以下命令查看指定表中的规则：

![](img/db04beb62a68588e9f0095f4bb386098_17.png)

![](img/db04beb62a68588e9f0095f4bb386098_19.png)

```bash
iptables -t [表名] -L
```

*   `-t`：指定要操作的表（table），如 `filter`、`nat`、`mangle`。
*   `-L`：列出（List）规则。

例如，查看filter表的规则：
```bash
iptables -t filter -L
```
如果不使用 `-t` 参数，则默认操作 `filter` 表。

## 数据包处理流程

![](img/db04beb62a68588e9f0095f4bb386098_23.png)

![](img/db04beb62a68588e9f0095f4bb386098_25.png)

理解了表和链之后，我们来看看数据包是如何被处理的。数据包的过滤流程可以概括为：表 -> 链 -> 规则。

一个数据包到达防火墙后，会根据其类型（发给本机、由本机发出、需要转发）流经不同的链。在每个链上，系统会按照从上到下的顺序依次检查链中的规则，一旦找到匹配的规则，就执行该规则定义的动作（如允许或拒绝），并停止继续检查。如果所有规则都不匹配，则执行该链的默认策略。

![](img/db04beb62a68588e9f0095f4bb386098_27.png)

![](img/db04beb62a68588e9f0095f4bb386098_28.png)

### 数据包流向示意图

![](img/db04beb62a68588e9f0095f4bb386098_30.png)

![](img/db04beb62a68588e9f0095f4bb386098_32.png)

1.  **入站数据流向**：`PREROUTING` -> `INPUT`
2.  **出站数据流向**：`OUTPUT` -> `POSTROUTING`
3.  **转发数据流向**：`PREROUTING` -> `FORWARD` -> `POSTROUTING`

### 规则匹配顺序要点

![](img/db04beb62a68588e9f0095f4bb386098_38.png)

![](img/db04beb62a68588e9f0095f4bb386098_40.png)

![](img/db04beb62a68588e9f0095f4bb386098_42.png)

*   **链内规则**：自上而下按顺序检查，匹配即停止。
*   **默认策略**：当数据包不匹配链内任何规则时，将按该链的默认策略处理。
*   **规则顺序关键性**：越严格的规则应放在越靠前的位置。

上一节我们了解了数据包的旅程，本节中我们来看看如何通过命令来设置交通规则——即iptables的语法。

iptables命令的基本格式如下：

```bash
iptables [-t 表名] 管理选项 [链名] [匹配条件] [-j 控制类型/动作]
```

![](img/db04beb62a68588e9f0095f4bb386098_55.png)

![](img/db04beb62a68588e9f0095f4bb386098_57.png)

### 常用管理选项

![](img/db04beb62a68588e9f0095f4bb386098_59.png)

以下是管理规则时常用的选项：

![](img/db04beb62a68588e9f0095f4bb386098_61.png)

![](img/db04beb62a68588e9f0095f4bb386098_63.png)

*   `-A`：在指定链的末尾追加一条新规则。
*   `-I`：在指定链的指定位置插入一条新规则（默认为第一条）。
*   `-D`：从指定链中删除一条规则（可按序号或具体规则内容删除）。
*   `-R`：替换指定链中的某一条规则。
*   `-L`：列出指定链或所有链中的规则。
*   `-F`：清空指定链或所有链中的规则（不会改变默认策略）。
*   `-P`：设置指定链的默认策略。
*   `-n`：以数字形式显示IP和端口，常与 `-L` 联用。
*   `-v`：显示详细信息。
*   `--line-numbers`：显示规则序号，便于管理。

### 命令示例解析

1.  **追加规则**：在filter表的INPUT链末尾添加一条拒绝所有入站数据的规则。
    ```bash
    iptables -t filter -A INPUT -j DROP
    ```
    执行此命令后，所有发送到本机的数据包将被丢弃，可能导致SSH连接中断。

2.  **插入规则**：在filter表的INPUT链的第一条位置插入一条允许TCP协议访问22端口的规则。
    ```bash
    iptables -I INPUT -p tcp --dport 22 -j ACCEPT
    ```
    这可以确保SSH服务在严格防火墙规则下依然可用。

3.  **删除规则**：删除filter表INPUT链中的第一条规则。
    ```bash
    iptables -D INPUT 1
    ```

4.  **清空规则**：清空filter表所有链的规则。
    ```bash
    iptables -F
    ```

![](img/db04beb62a68588e9f0095f4bb386098_65.png)

![](img/db04beb62a68588e9f0095f4bb386098_67.png)

5.  **设置默认策略**：将filter表INPUT链的默认策略设置为DROP。
    ```bash
    iptables -P INPUT DROP
    ```
    > **警告**：在远程服务器上设置默认策略为DROP前，务必先设置允许SSH等必要服务的规则，或配置定时任务恢复策略，以防被锁在服务器外。

### 常用匹配条件

![](img/db04beb62a68588e9f0095f4bb386098_75.png)

规则中的匹配条件用于筛选数据包：

![](img/db04beb62a68588e9f0095f4bb386098_77.png)

*   `-p`：指定协议，如 `tcp`、`udp`、`icmp`。
*   `-s`：指定源IP地址或网段。
*   `-d`：指定目标IP地址或网段。
*   `-i`：指定数据包流入的网络接口（如eth0）。
*   `-o`：指定数据包流出的网络接口。
*   `--sport`：指定源端口。
*   `--dport`：指定目标端口。

**示例**：拒绝来自IP `192.168.1.100` 的所有流量。
```bash
iptables -A INPUT -s 192.168.1.100 -j DROP
```
