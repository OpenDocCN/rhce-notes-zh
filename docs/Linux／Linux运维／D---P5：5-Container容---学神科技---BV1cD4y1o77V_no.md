# Docker容器虚拟化平台：P5：容器端口映射 🌐

在本节课中，我们将学习Docker容器端口映射的核心概念与操作方法。端口映射是连接容器内部服务与外部网络的关键技术，理解它对于部署和访问容器化应用至关重要。

## 端口映射的必要性

![](img/07b0cde8a38cf4172b4eb43e147b73e4_1.png)

![](img/07b0cde8a38cf4172b4eb43e147b73e4_2.png)

上一节我们介绍了如何运行容器，本节中我们来看看如何从外部网络访问容器内部的服务。容器默认运行在隔离的网络环境中，外部无法直接访问。因此，需要将容器内部的端口映射到宿主机的端口上。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_4.png)

## 端口映射操作

![](img/07b0cde8a38cf4172b4eb43e147b73e4_6.png)

以下是使用 `docker run` 命令进行端口映射的步骤。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_8.png)

![](img/07b0cde8a38cf4172b4eb43e147b73e4_10.png)

1.  运行一个后台容器，并将容器的80端口映射到宿主机的80端口。
    ```bash
    docker run -d -p 80:80 httpd
    ```
    命令解析：
    *   `-d`：后台运行容器。
    *   `-p 80:80`：进行端口映射。格式为 `-p <宿主机端口>:<容器端口>`。此命令将容器的80端口映射到宿主机的80端口。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_12.png)

![](img/07b0cde8a38cf4172b4eb43e147b73e4_14.png)

2.  执行命令后，可以使用 `docker ps` 命令查看容器的运行状态及端口映射信息。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_16.png)

3.  在宿主机上，使用 `netstat` 命令可以查看到一个由 `docker-proxy` 进程监听的80端口，它负责代理外部到容器的网络请求。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_18.png)

4.  此时，通过访问宿主机的IP地址（例如 `http://宿主机IP`），即可成功访问到容器内运行的Web服务。

## 端口映射原理与网络模式

![](img/07b0cde8a38cf4172b4eb43e147b73e4_20.png)

![](img/07b0cde8a38cf4172b4eb43e147b73e4_22.png)

当前Docker容器的默认网络模式类似于虚拟机的 **NAT（网络地址转换）** 模式。

*   每个容器会获得一个内部IP（如 `172.17.0.2`）。
*   宿主机上会创建一个名为 `docker0` 的虚拟网桥（IP如 `172.17.0.1`）。
*   容器通过 `docker0` 网桥与宿主机通信。当外部请求访问宿主机映射端口时，`docker-proxy` 或宿主机的防火墙规则会将请求通过 `docker0` 网桥转发到对应的容器。

这种模式使得多个容器可以共享宿主机的IP出口，但宿主机的一个端口只能映射给一个容器。若需在宿主机同一端口运行多个容器实例，需借助负载均衡器（如Kubernetes的Service）或采用桥接模式为容器配置独立IP。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_24.png)

![](img/07b0cde8a38cf4172b4eb43e147b73e4_26.png)

## 进入容器内部

![](img/07b0cde8a38cf4172b4eb43e147b73e4_28.png)

![](img/07b0cde8a38cf4172b4eb43e147b73e4_30.png)

为了验证容器的网络配置，我们可以进入正在运行的容器内部。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_32.png)

1.  使用 `docker exec` 命令进入容器。
    ```bash
    docker exec -it <容器ID或名称> bash
    ```

![](img/07b0cde8a38cf4172b4eb43e147b73e4_34.png)

![](img/07b0cde8a38cf4172b4eb43e147b73e4_36.png)

2.  进入容器后，可以执行命令查看其网络配置，例如使用 `ip addr` 命令查看IP地址，确认其处于 `docker0` 网桥所在的网段（如 `172.17.0.0/16`）。

![](img/07b0cde8a38cf4172b4eb43e147b73e4_38.png)

## 总结

![](img/07b0cde8a38cf4172b4eb43e147b73e4_40.png)

本节课中我们一起学习了Docker端口映射。我们掌握了使用 `-p` 参数映射端口的基本命令，理解了其背后的NAT网络模式原理，并学会了通过 `docker exec` 进入容器进行验证。端口映射是连接容器内外网络的基础，是部署可访问服务的关键步骤。
