文件权限是对文件访问的控制机制。文件归创建它的用户所有，该用户对此文件拥有所有权。

![](img/63424467ac4424427a63147685e8a074_27.png)

例如，使用 `touch` 命令创建一个文件：
```bash
touch aa
ls -l aa
```
输出结果类似于：
```
-rw-r--r--. 1 root root 0 Jan 10 01:31 aa
```
此文件由 `root` 用户创建，因此 `root` 用户对其拥有所有权。

## 文件属性字段详解

以下是 `ls -l` 命令输出中各个字段的含义：

![](img/63424467ac4424427a63147685e8a074_41.png)

![](img/63424467ac4424427a63147685e8a074_43.png)

![](img/63424467ac4424427a63147685e8a074_45.png)

1.  **第一个字段（共11个字符）**：文件类型与权限。
    *   第一个字符表示**文件类型**：
        *   `-`： 普通文件
        *   `d`： 目录
        *   `l`： 符号链接文件
        *   `b`： 块设备文件
        *   `s`： socket文件
    *   后9个字符分为三组，每组三个，表示**权限**：
        *   第1-3位：**文件所有者（user）**的权限。
        *   第4-6位：**文件所属组（group）**的权限。
        *   第7-9位：**其他用户（other）**的权限。
    *   每组权限中的字符含义：
        *   `r`： 读权限
        *   `w`： 写权限
        *   `x`： 执行权限
        *   `-`： 无对应权限
    *   最后一个点 `.` 与SELinux上下文相关，初级学习可暂时忽略。

![](img/63424467ac4424427a63147685e8a074_47.png)

2.  **第二个字段**：硬链接计数。表示有多少个文件名指向同一个inode（数据块）。

3.  **第三字段**：文件所有者（属主）。

![](img/63424467ac4424427a63147685e8a074_49.png)

4.  **第四字段**：文件所属组（属组）。

![](img/63424467ac4424427a63147685e8a074_51.png)

5.  **第五字段**：文件大小（单位：字节）。

![](img/63424467ac4424427a63147685e8a074_53.png)

6.  **第六字段**：文件的最后修改时间。

![](img/63424467ac4424427a63147685e8a074_55.png)

![](img/63424467ac4424427a63147685e8a074_57.png)

7.  **第七字段**：文件名。

**权限判定流程**：当用户尝试访问一个文件时，系统按以下顺序检查权限：
1.  判断当前用户是否为文件**属主**。如果是，则应用“属主权限位”。
2.  如果不是属主，则判断当前用户是否属于文件**属组**。如果是，则应用“属组权限位”。
3.  如果以上都不是，则应用“其他用户权限位”。

## 目录权限与文件权限的区别

理解了文件的基本属性后，我们需要区分目录权限和文件权限的不同作用。

![](img/63424467ac4424427a63147685e8a074_63.png)

### 目录权限的作用
目录的权限控制用户能否**列出**、**进入**或**在目录中创建/删除**文件。

![](img/63424467ac4424427a63147685e8a074_65.png)

![](img/63424467ac4424427a63147685e8a074_67.png)

*   **读权限**：允许**列出**目录内的文件和子目录**名称**。
*   **写权限**：结合执行权限，允许在目录内**创建、删除、重命名**文件和子目录。
*   **执行权限**：允许**进入**该目录，并访问目录内文件的**元数据**（即`ls -l`看到的详细信息）。

![](img/63424467ac4424427a63147685e8a074_69.png)

以下是几种目录权限组合的示例：

*   `dr--r--r--`：可列出文件名，但所有文件属性显示为`?`，且无法进入目录。
*   `dr-xr-xr-x`：可进入目录，并能正常列出所有文件的详细属性，但无法在目录内创建或删除文件。
*   `d-wx-wx-wx`：无法列出目录内容，但可通过绝对路径直接访问目录内的已知文件，并可在目录内创建或删除文件。
*   `drwxrwxrwx`：拥有所有权限。

**重要结论**：删除一个文件，与文件本身的权限**无关**，只取决于其**父目录**是否对当前用户具有**写（w）和执行（x）**权限。

### 文件权限的作用
文件的权限直接控制用户对文件**内容**的操作。

*   **读权限**：允许读取文件内容。
*   **写权限**：允许修改文件内容。
*   **执行权限**：允许将文件作为程序或脚本运行。

**注意**：对于普通用户，要执行一个文件，通常需要同时具备该文件的**读（r）**和**执行（x）**权限。

## 修改权限的命令

![](img/63424467ac4424427a63147685e8a074_77.png)

![](img/63424467ac4424427a63147685e8a074_79.png)

![](img/63424467ac4424427a63147685e8a074_81.png)

了解了权限的含义后，本节中我们来看看如何修改它们。

![](img/63424467ac4424427a63147685e8a074_83.png)

### 1. `chmod` - 修改文件或目录权限
`chmod` 命令有两种主要的权限设置方法：数字法和字母法。

#### 数字法
将每组权限（rwx）视为一个二进制数，有权限为1，无权限为0，再转换为十进制。
*   `r` = 4
*   `w` = 2
*   `x` = 1
*   `-` = 0

![](img/63424467ac4424427a63147685e8a074_85.png)

权限组合计算：`rwx` = 4+2+1 = 7；`r-x` = 4+0+1 = 5；`rw-` = 4+2+0 = 6。

![](img/63424467ac4424427a63147685e8a074_87.png)

命令格式：`chmod [选项] 数字模式 文件/目录`
```bash
chmod 755 aa    # 将aa的权限设置为 rwxr-xr-x
chmod 644 bb    # 将bb的权限设置为 rw-r--r--
```
选项 `-R`：递归修改目录及其内部所有内容的权限。
```bash
chmod -R 755 mydir/
```

![](img/63424467ac4424427a63147685e8a074_89.png)

![](img/63424467ac4424427a63147685e8a074_91.png)

#### 字母法
使用字母来指定要修改的角色和操作。
*   **角色**：
    *   `u`： 文件所有者
    *   `g`： 文件所属组
    *   `o`： 其他用户
    *   `a`： 所有用户（默认）
*   **操作**：
    *   `+`： 添加权限
    *   `-`： 移除权限
    *   `=`： 直接设定权限

命令格式：`chmod [选项] 角色操作权限 文件/目录`
```bash
chmod u+x script.sh    # 给所有者添加执行权限
chmod g-w file.txt     # 从所属组移除写权限
chmod o=r-- public.txt # 设置其他用户权限为只读
chmod a+r document     # 给所有用户添加读权限
```

### 2. `chown` - 修改文件所有者和所属组
命令格式：`chown [选项] 新所有者:新所属组 文件/目录`
```bash
chown alice:developers project.tar.gz # 同时修改所有者和所属组
chown alice file.txt                  # 只修改所有者
chown :developers dir/                # 只修改所属组
```

### 3. `chgrp` - 修改文件所属组
`chown` 命令可以完成所有操作，因此 `chgrp` 命令使用较少。
```bash
chgrp developers file.txt
```

## 特殊权限位

![](img/63424467ac4424427a63147685e8a074_97.png)

除了基本的rwx权限，Linux还有三个特殊的权限位，用于实现更精细的控制。

![](img/63424467ac4424427a63147685e8a074_99.png)

![](img/63424467ac4424427a63147685e8a074_101.png)

### 1. SUID (Set User ID)
*   **位置**：设置在文件**所有者**的**执行位（x）**上。如果原有x权限，则显示为`s`；否则显示为`S`。
*   **作用**：当普通用户执行具有SUID位的程序时，在程序运行期间会暂时拥有该程序**所有者**的权限（通常是root）。
*   **常见示例**：`/usr/bin/passwd` 命令，允许普通用户修改自己的密码。
*   **设置方法**：
    ```bash
    chmod u+s /path/to/command
    # 或使用数字法，在三位数字前加4，如 4755
    chmod 4755 /path/to/command
    ```
*   **风险**：不当设置SUID是严重的安全隐患，需谨慎使用。

![](img/63424467ac4424427a63147685e8a074_103.png)

### 2. SGID (Set Group ID)
*   **作用于文件时**：
    *   **位置**：设置在文件**所属组**的**执行位（x）**上。显示为`s`或`S`。
    *   **作用**：类似SUID，执行时获得文件**所属组**的权限。
*   **作用于目录时**（更常用）：
    *   **作用**：在该目录下创建的任何新文件或子目录，其**所属组**会自动继承该目录的所属组，而不是创建者的默认组。便于团队协作共享文件。
*   **设置方法**：
    ```bash
    chmod g+s /path/to/directory
    # 或使用数字法，在三位数字前加2，如 2755
    chmod 2755 /path/to/directory
    ```

### 3. Sticky Bit (粘滞位)
*   **位置**：设置在**其他用户**的**执行位（x）**上。显示为`t`或`T`。
*   **作用**：通常用于全局可写目录（如`/tmp`）。即使目录权限是`777`，设置了粘滞位后，用户只能删除或重命名**自己创建**的文件，而不能删除其他用户的文件。
*   **设置方法**：
    ```bash
    chmod o+t /path/to/directory
    # 或使用数字法，在三位数字前加1，如 1777
    chmod 1777 /tmp
    ```

## 默认权限与umask

![](img/63424467ac4424427a63147685e8a074_111.png)

![](img/63424467ac4424427a63147685e8a074_113.png)

每次创建新文件或目录时，系统会赋予一个默认权限，这个默认值由 `umask`（权限掩码）控制。

![](img/63424467ac4424427a63147685e8a074_115.png)

![](img/63424467ac4424427a63147685e8a074_117.png)

![](img/63424467ac4424427a63147685e8a074_119.png)

*   **查看当前umask**：
    ```bash
    umask
    # 输出通常为 0022 或 0002
    ```
*   **umask的作用**：它指定了需要从默认权限中“屏蔽”掉的权限位。
    *   目录的默认最大权限是 `777`。
    *   文件的默认最大权限是 `666`（文件默认没有执行权限）。
*   **计算方法**（逻辑非与操作）：
    *   新建目录权限 = `777` - `umask`
    *   新建文件权限 = `666` - `umask`
*   **示例**：如果 `umask` 是 `022`
    *   新建目录权限 = `777 - 022 = 755` (`rwxr-xr-x`)
    *   新建文件权限 = `666 - 022 = 644` (`rw-r--r--`)
*   **修改umask**：
    ```bash
    umask 027  # 临时修改，仅对当前shell会话有效
    ```
    要永久生效，需要将 `umask` 命令写入shell的配置文件中，如 `~/.bashrc` 或 `/etc/profile`。

![](img/63424467ac4424427a63147685e8a074_121.png)

![](img/63424467ac4424427a63147685e8a074_123.png)

**安全实践**：对于Web服务器等场景，文件和目录的安全权限临界点通常就是默认的 `755`（目录）和 `644`（文件），不建议随意放宽。

## 总结
本节课中我们一起学习了Linux文件权限管理的核心知识：
1.  **理解权限**：掌握了文件属性中11个字符的含义，以及如何根据用户身份（属主、属组、其他）判定访问权限。
2.  **区分权限**：明确了目录权限（控制列出、进入、增删）与文件权限（控制读、写、执行内容）的根本区别。
3.  **修改权限**：学会了使用 `chmod`（数字法/字母法）、`chown`、`chgrp` 命令来修改权限和归属。
4.  **特殊权限**：了解了SUID（临时获取属主权限）、SGID（继承目录属组）、粘滞位（保护共享目录文件）的作用与设置。
5.  **默认权限**：明白了 `umask` 如何影响新创建文件和目录的初始权限。

![](img/63424467ac4424427a63147685e8a074_131.png)

掌握这些知识，你就能有效地管理Linux系统中的文件访问控制，为系统安全打下坚实基础。
