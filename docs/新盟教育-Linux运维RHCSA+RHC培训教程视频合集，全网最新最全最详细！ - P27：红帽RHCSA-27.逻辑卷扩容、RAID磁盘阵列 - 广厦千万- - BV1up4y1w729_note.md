## 逻辑卷扩容操作

![](img/187296b0f84d290949a0785982f59cc8_15.png)

![](img/187296b0f84d290949a0785982f59cc8_17.png)

上一节我们介绍了逻辑卷的创建与基本管理。本节中，我们来看看如何对已创建并使用的逻辑卷进行在线扩容，这是LVM在实际生产环境中的关键应用。

### 扩容前的检查

在对逻辑卷进行扩容之前，必须首先确认其所属的卷组（VG）是否有足够的剩余空间。

**检查卷组剩余空间：**
```bash
vgs
```
**检查逻辑卷当前大小：**
```bash
lvs
```
执行以上命令后，可以清楚地看到逻辑卷的当前容量以及卷组中可供分配的空间。

### 扩展逻辑卷容量

![](img/187296b0f84d290949a0785982f59cc8_19.png)

确认卷组有足够空间后，即可使用 `lvextend` 命令对逻辑卷进行扩容。

以下是扩展逻辑卷的命令格式：
```bash
lvextend -L +[要增加的大小] [逻辑卷的完整路径]
```
例如，为逻辑卷 `/dev/csystemvg/mylv` 增加19GB空间：
```bash
lvextend -L +19G /dev/csystemvg/mylv
```
命令执行成功后，系统会提示逻辑卷的大小已从原始值（如20G）调整为新值（如39G）。再次使用 `lvs` 命令可以验证逻辑卷的容量确实已经增大。

![](img/187296b0f84d290949a0785982f59cc8_21.png)

### 扩展文件系统

![](img/187296b0f84d290949a0785982f59cc8_23.png)

![](img/187296b0f84d290949a0785982f59cc8_25.png)

完成逻辑卷的空间扩展后，你会发现使用 `df -h` 命令查看挂载点，其显示的空间容量并未立即增大。这是因为逻辑卷上层的文件系统（如XFS）尚未感知到底层空间的变更。

**文件系统未扩展的原因：**
最初格式化时，文件系统只管理了原始大小的空间（如20G）。新增的空间（19G）并没有被文件系统管理，因此操作系统无法使用。

![](img/187296b0f84d290949a0785982f59cc8_27.png)

![](img/187296b0f84d290949a0785982f59cc8_29.png)

**扩展XFS文件系统：**
对于XFS类型的文件系统，需要使用 `xfs_growfs` 命令来扩展其管理范围。
```bash
xfs_growfs [挂载点目录]
```
例如，如果逻辑卷挂载在 `/webbak` 目录，则执行：
```bash
xfs_growfs /webbak
```
执行此命令后，再次使用 `df -h` 查看，挂载点的总容量就会显示为扩容后的正确大小（如39G）。原有数据在此过程中不会受到任何影响。

### 扩容卷组

![](img/187296b0f84d290949a0785982f59cc8_31.png)

![](img/187296b0f84d290949a0785982f59cc8_32.png)

![](img/187296b0f84d290949a0785982f59cc8_34.png)

如果逻辑卷需要继续扩容，但卷组的剩余空间已不足，我们就需要先对卷组本身进行扩容。

![](img/187296b0f84d290949a0785982f59cc8_36.png)

![](img/187296b0f84d290949a0785982f59cc8_38.png)

**扩展卷组容量：**
扩展卷组的命令是 `vgextend`。可以将新的物理硬盘或分区加入现有卷组。
```bash
vgextend [卷组名] [新设备路径]
```
例如，将新硬盘 `/dev/sdd` 加入到 `csystemvg` 卷组：
```bash
vgextend csystemvg /dev/sdd
```
卷组扩容成功后，即可再次使用 `lvextend` 和 `xfs_growfs` 命令对逻辑卷进行扩容。这个过程可以循环进行，从而实现存储空间的弹性扩展。

## 逻辑卷的缩减与删除

![](img/187296b0f84d290949a0785982f59cc8_40.png)

### 关于缩减操作

![](img/187296b0f84d290949a0785982f59cc8_42.png)

![](img/187296b0f84d290949a0785982f59cc8_44.png)

逻辑卷支持缩减容量，但此操作风险极高，且**不支持在线操作**。缩减前必须卸载文件系统，操作顺序为：先缩减文件系统，再缩减逻辑卷空间。由于可能直接导致数据丢失，**生产环境中强烈不建议执行缩容操作**。

![](img/187296b0f84d290949a0785982f59cc8_46.png)

### 删除逻辑卷

当不再需要某个逻辑卷时，可以将其删除。删除顺序与创建顺序相反，且同样需要先卸载。

以下是删除逻辑卷及相关资源的步骤：
1.  **卸载文件系统：** `umount [挂载点]`
2.  **删除逻辑卷：** `lvremove [逻辑卷完整路径]`
3.  **删除卷组：** `vgremove [卷组名]`
4.  **（可选）清除物理卷标签：** 删除后，原物理设备上的LVM标签可能依然存在。如需将其恢复为普通分区，可在重新格式化时使用 `-f` 选项强制覆盖，或使用 `pvremove` 命令清除标签。

## RAID磁盘阵列简介

![](img/187296b0f84d290949a0785982f59cc8_51.png)

![](img/187296b0f84d290949a0785982f59cc8_53.png)

在了解了逻辑卷的弹性扩展后，我们还需要关注数据的可靠性与安全性。RAID（独立磁盘冗余阵列）技术就是将多块物理磁盘通过特定方式组合起来，以实现提升性能、增加容量或提供数据冗余的目的。

![](img/187296b0f84d290949a0785982f59cc8_55.png)

![](img/187296b0f84d290949a0785982f59cc8_56.png)

### RAID的核心作用

![](img/187296b0f84d290949a0785982f59cc8_58.png)

![](img/187296b0f84d290949a0785982f59cc8_59.png)

![](img/187296b0f84d290949a0785982f59cc8_61.png)

![](img/187296b0f84d290949a0785982f59cc8_63.png)

RAID的主要功能可以概括为以下两点：
*   **提高数据可靠性（冗余）：** 通过数据镜像或校验机制，允许阵列中的一块或多块磁盘损坏时不丢失数据。
*   **提升读写性能：** 通过将数据分散存储在多个磁盘上并行读写，提高I/O速度。

### RAID常见级别

不同的RAID级别（如RAID 0, RAID 1, RAID 5, RAID 10等）在性能、容量和冗余之间提供了不同的权衡方案。例如：
*   **RAID 0（条带化）：** 将数据分块并行写入多个磁盘，**大幅提升读写速度**，但**无任何冗余**，一块磁盘损坏即导致全部数据丢失。
*   **RAID 1（镜像）：** 将数据完全复制到另一块磁盘，**提供数据冗余**，读取速度有所提升，但写入速度不变，且磁盘利用率只有50%。

在企业实际部署中，常将RAID与LVM结合使用：先使用RAID卡或软件RAID创建具有冗余能力的“虚拟大硬盘”，再在其上创建LVM卷组和逻辑卷。这样既能保障底层物理磁盘故障时的数据安全，又能通过LVM实现上层存储空间的灵活管理。
