# Linux基础入门教学：2：红帽系统安装与基础认知

## 概述
在本节课中，我们将要学习Linux操作系统的核心概念、红帽企业版Linux的安装过程，以及Linux系统与Windows系统在文件管理、系统结构等方面的根本区别。通过实践操作，我们将亲手安装一个红帽Linux系统，并初步认识其独特的文件系统结构。

## 课程内容

### 一、Linux与Windows的核心区别
上一节我们介绍了Linux的历史和内核概念，本节中我们来看看Linux与Windows在系统设计上的核心差异。

Linux与Windows在多个层面存在根本性区别，理解这些区别是学习Linux的基础。

以下是Linux与Windows的主要对比：
*   **安全性/稳定性**：Linux系统因其开源特性和严谨的权限管理，在安全性和稳定性上通常优于Windows。
*   **硬件支持**：Windows对消费级硬件（如游戏显卡、外设）的支持更广泛、更友好；Linux则更侧重于服务器和企业级硬件的驱动支持。
*   **源代码**：Windows源代码是闭源、保密的；Linux内核及大部分组件是开源的。
*   **可调节性**：Linux提供了极高的可定制性和可调节性，主要通过命令行进行深度控制；Windows则更侧重于图形界面的易用性。
*   **方便性**：Windows集成了丰富的多媒体和日常应用，对普通用户更友好；Linux在服务器管理和开发环境搭建上更为便捷。
*   **技术支持和文件系统**：Windows依赖厂商技术支持；Linux拥有强大的社区支持。两者的默认文件系统也不同，例如Windows常用NTFS，而Linux常用**XFS**或**EXT4**。

### 二、云计算与红帽产品体系
在深入技术细节前，我们需要了解Linux，特别是红帽Linux，在现代IT架构——云计算中的定位。

云计算是一种按需提供计算资源（如服务器、存储、网络）的服务模式，用户无需自建数据中心，即可通过互联网获取这些资源。红帽公司正是企业级Linux和混合云解决方案领域的领导者。

红帽提供的主要Linux发行版包括：
1.  **Red Hat Enterprise Linux (RHEL)**：企业版，以稳定性和长期支持著称，需订阅付费。
2.  **Fedora**：社区版，免费，是新技术的前沿试验场，功能更新快。
3.  **CentOS**：曾是RHEL的免费克隆版，现已被红帽收购并转向为滚动更新版本。

红帽的核心产品线围绕混合云构建，除了RHEL，还包括：
*   **Ansible**：自动化运维工具。
*   **Red Hat Virtualization**：企业级虚拟化平台。
*   **OpenShift**：基于Kubernetes的容器平台。

#### 1. 准备工作
首先，需要在你的Windows或Mac电脑上安装虚拟机软件**VMware Workstation Pro**，用于创建虚拟的计算机环境来安装Linux。安装完成后，还需要下载RHEL 8.4的ISO镜像文件。

![](img/a0fa6738366178094298e447dafdc041_28.png)

#### 2. 创建虚拟机
打开VMware，开始创建一台新的虚拟机，这相当于组装一台虚拟的电脑。

以下是创建虚拟机的关键步骤：
1.  选择“自定义”安装类型。
2.  选择稍后安装操作系统。
3.  客户机操作系统选择“Linux”，版本选择“Red Hat Enterprise Linux 8 64位”。
4.  为虚拟机命名并选择存放位置（建议放在非系统盘，如D盘）。
5.  处理器数量保持默认即可。
6.  内存大小建议分配**2GB**（2048MB），具体可根据宿主机内存调整。
7.  网络类型选择“桥接模式”（与宿主机在同一局域网）或“仅主机模式”（仅与宿主机通信）。
8.  创建新虚拟磁盘，大小建议**20GB**以上，选择“将虚拟磁盘拆分成多个文件”。
9.  完成虚拟机框架创建。

#### 3. 配置虚拟机并安装系统
虚拟机框架创建好后，需要进行配置并开始安装系统。

以下是安装RHEL 8.4的主要步骤：
1.  编辑虚拟机设置，在“CD/DVD”选项中加载下载好的RHEL 8.4 ISO镜像文件。
2.  开启此虚拟机，在启动界面选择“Install Red Hat Enterprise Linux 8.4”。
3.  安装语言选择“简体中文”。
4.  在“安装信息摘要”界面进行关键配置：
    *   **软件选择**：初学者建议选择“带GUI的服务器”，以便使用图形界面。
    *   **安装目的地**：选择“自定义”分区方案。
        *   点击“+”号，添加`/boot`分区，大小**500MB**。
        *   添加`swap`分区，大小**2048MB**。
        *   添加`/`（根）分区，分配剩余所有空间（例如**60GB**），文件系统选择**XFS**。
    *   **网络和主机名**：打开以太网连接，并设置主机名，格式建议为`姓名.runtimeedu.com`。
    *   **根密码**：为超级管理员用户`root`设置一个强密码。
    *   **创建用户**：创建一个普通用户（如`tangkai`）并设置密码。
5.  点击“开始安装”，等待系统安装完成。
6.  安装完成后重启，使用`root`用户和密码登录系统。

#### 1. 无后缀名与文件类型识别
与Windows通过后缀名（如`.exe`, `.txt`）识别文件类型不同，Linux**没有严格的后缀名概念**。在Linux中，`qq.exe`就是一个完整的文件名，系统不依赖`.exe`来判断它是否可执行。

Linux主要通过**文件颜色**和**权限属性**来区分文件类型。我们可以使用`ls`命令查看文件列表。

以下是常见的文件颜色与类型对应关系（在默认终端配色下）：
*   **白色**：普通文件（如文本文件、配置文件）。
*   **深蓝色**：目录（文件夹）。
*   **天蓝色**：链接文件（类似于Windows的快捷方式）。
*   **绿色**：可执行文件（类似于Windows的`.exe`文件）。
*   **黄色**：设备文件（如`/dev/sda`代表硬盘）。
*   **红色**：压缩包或软件包文件。

#### 2. 无盘符的树形目录结构
Windows使用`C:`、`D:`等盘符来划分存储空间。Linux则采用一种**树形的目录结构**，**没有盘符的概念**。整个文件系统从一个唯一的根目录开始，所有文件、目录、设备都挂载在这棵“树”的某个分支下。

![](img/a0fa6738366178094298e447dafdc041_109.png)

根目录用`/`表示，它是整个文件系统的起点。在根目录下，存在一系列标准的一级子目录，每个目录都有其特定用途：
*   `/bin`：存放基础命令。
*   `/etc`：存放系统配置文件。
*   `/home`：存放普通用户的家目录。
*   `/root`：超级管理员`root`的家目录。
*   `/var`：存放经常变化的文件，如日志。
*   `/boot`：存放系统启动相关的文件（这正是我们安装时创建的分区）。

![](img/a0fa6738366178094298e447dafdc041_111.png)

![](img/a0fa6738366178094298e447dafdc041_113.png)

当你为Linux分配一块硬盘空间（如安装时的60GB），这块空间就被格式化成文件系统（如XFS），并直接挂载到根目录`/`下。所有数据都存放在以`/`为起点的目录树中，通过路径来访问，而不是通过盘符。
