## 集合查询：IN 与 NOT IN

上一节我们介绍了使用 `BETWEEN...AND...` 进行范围查询。本节中我们来看看集合查询。集合查询与范围查询类似，但更侧重于指定一组具体的值，而非连续的范围。

集合查询使用 `IN` 关键字，其含义是“包含于”。它允许我们指定一个值列表，查询字段值匹配列表中任意一个值的记录。

以下是集合查询的基本语法：
```sql
SELECT * FROM 表名 WHERE 字段名 IN (值1, 值2, 值3, ...);
```
与范围查询不同，`IN` 可以用于字符型字段。对于字符值，必须使用引号（单引号或双引号均可）括起来。

例如，查询特定几位学生的信息：
```sql
SELECT * FROM students WHERE name IN (‘张三‘, ‘李四‘, ‘王五‘);
```
`IN` 的取反操作是 `NOT IN`，表示查询字段值不在指定集合中的记录。
```sql
SELECT * FROM students WHERE gender NOT IN (‘F‘);
```

## 模糊查询：LIKE 与 NOT LIKE

![](img/425edc8b56256906016499b09cfc7444_18.png)

![](img/425edc8b56256906016499b09cfc7444_20.png)

当我们需要查找包含特定字符模式的数据，但又无法精确匹配时，就需要用到模糊查询。模糊查询使用 `LIKE` 关键字，并配合通配符使用。

![](img/425edc8b56256906016499b09cfc7444_22.png)

![](img/425edc8b56256906016499b09cfc7444_24.png)

SQL中常用的通配符有两个：
*   **`%`**：代表零个、一个或多个任意字符。
*   **`_`**：代表一个任意字符。

以下是模糊查询的常见用法：
```sql
-- 查找名字以‘张‘开头的学生
SELECT * FROM students WHERE name LIKE ‘张%‘;

-- 查找名字第二个字是‘三‘的学生
SELECT * FROM students WHERE name LIKE ‘_三‘;

-- 查找名字中包含‘德‘字的学生
SELECT * FROM students WHERE name LIKE ‘%德%‘;
```
模糊查询同样支持取反操作 `NOT LIKE`。需要注意的是，即使查询数值型字段，使用 `LIKE` 时，匹配模式也需要用引号括起来。

## 空值判断：IS NULL 与 IS NOT NULL

在数据库中，空值（NULL）表示该字段没有值，它与数值0或空字符串‘ ‘ 是不同的概念。查询空值需要使用专门的 `IS NULL` 或 `IS NOT NULL` 操作符，而不能使用等号（=）。

![](img/425edc8b56256906016499b09cfc7444_32.png)

以下是空值判断的语法：
```sql
-- 查询‘成绩‘字段为空的记录
SELECT * FROM students WHERE score IS NULL;

![](img/425edc8b56256906016499b09cfc7444_34.png)

![](img/425edc8b56256906016499b09cfc7444_36.png)

-- 查询‘成绩‘字段不为空的记录
SELECT * FROM students WHERE score IS NOT NULL;
```
这个功能常用于检查数据完整性，发现漏填或待补充的数据。

## 多重条件查询：AND 与 OR

前面介绍的查询大多只针对单个字段设置条件。在实际应用中，我们经常需要根据多个字段的组合条件来筛选数据，这时就需要用到逻辑运算符 `AND`（与）和 `OR`（或）。

*   **`AND`**：连接多个条件，要求**所有**条件同时满足，结果取各条件的**交集**。
*   **`OR`**：连接多个条件，要求**至少一个**条件满足，结果取各条件的**并集**。

![](img/425edc8b56256906016499b09cfc7444_43.png)

以下是多重条件查询的示例：
```sql
-- 查询班级为1班‘并且‘成绩大于等于80分的学生（交集）
SELECT * FROM students WHERE class_id = 1 AND score >= 80;

-- 查询班级为1班‘或者‘成绩大于等于80分的学生（并集）
SELECT * FROM students WHERE class_id = 1 OR score >= 80;
```
通常，使用 `AND` 会使得结果集更精确、更小；而使用 `OR` 则会扩大结果集的范围。我们可以混合使用 `AND` 和 `OR`，并通过括号 `()` 来明确运算的优先级。

![](img/425edc8b56256906016499b09cfc7444_45.png)
