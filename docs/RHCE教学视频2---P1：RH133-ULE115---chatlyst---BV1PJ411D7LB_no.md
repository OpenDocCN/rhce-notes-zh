# Linux磁盘管理：2.1：fdisk, mkfs, fsck 详解 🖥️💾

![](img/0d06b3afdeb036834646c2193b4c0e3e_1.png)

在本节课中，我们将学习如何为Linux系统添加和使用新的存储设备。核心内容包括使用 `fdisk` 进行磁盘分区，使用 `mkfs` 格式化分区，以及使用 `fsck` 检查和修复文件系统。我们将遵循一个典型的操作流程：分区 -> 格式化 -> 挂载 -> 配置自动挂载。

## 概述

当需要为系统添加新的存储设备时，需要将其整合到虚拟文件系统（VFS）体系中。一个典型的操作流程是：首先对设备进行分区，然后格式化分区以创建文件系统，接着将分区挂载到目录树中，最后配置系统使其在启动时自动挂载。

## 磁盘分区工具 fdisk

上一节我们介绍了添加新存储的基本流程，本节中我们来看看第一步：磁盘分区。`fdisk` 是一个经典且常用的磁盘分区工具。需要明确的是，一块硬盘可以不分区直接使用，例如将整个1TB硬盘格式化为一个卷。但通常，我们会使用分区工具来管理磁盘空间。

以下是 `fdisk` 的常见用法和参数：

![](img/0d06b3afdeb036834646c2193b4c0e3e_3.png)

*   **列出分区信息**：`fdisk -l` 命令可以列出所有已识别硬盘的分区表。要查看指定设备（如 `/dev/sda`），可以使用 `fdisk -l /dev/sda`。
*   **进入交互式分区模式**：要对设备（如 `/dev/sda`）进行分区，需运行 `fdisk /dev/sda`。
*   **交互模式下的常用命令**：
    *   `n`：创建新分区。
    *   `d`：删除分区。
    *   `p`：打印当前分区表。
    *   `t`：更改分区类型ID。
    *   `l`：列出所有已知的分区类型ID。
    *   `w`：将分区表写入磁盘并退出。
    *   `q`：不保存更改并退出。

**重要提示**：在 `fdisk` 交互模式中，无论进行创建或删除操作，**必须输入 `w` 命令才会将更改实际写入磁盘**，否则所有操作都不会生效。

![](img/0d06b3afdeb036834646c2193b4c0e3e_5.png)

![](img/0d06b3afdeb036834646c2193b4c0e3e_6.png)

分区类型ID是一个重要的概念。例如，ID `83` 代表 Linux 原生文件系统（如 ext2/ext3/ext4），ID `82` 代表 Linux 交换分区（swap）。在早期，使用 LVM 时需要将分区类型改为对应的 LVM 类型，但现在许多工具（如 LVM）已不再严格依赖此类型标识。

![](img/0d06b3afdeb036834646c2193b4c0e3e_8.png)

![](img/0d06b3afdeb036834646c2193b4c0e3e_9.png)

![](img/0d06b3afdeb036834646c2193b4c0e3e_11.png)

![](img/0d06b3afdeb036834646c2193b4c0e3e_12.png)

完成分区并写入（`w`）后，**建议运行 `partprobe` 命令**。此命令会通知内核重新读取分区表，确保新创建的分区能够立即被系统识别和使用。否则，可能会出现虽然 `fdisk -l` 能看到新分区，但格式化或挂载时却提示设备不存在的错误。

上一节我们介绍了如何使用 `fdisk` 创建分区，本节中我们来看看如何为分区创建文件系统，即格式化。格式化操作会建立文件系统的元数据结构，如 inode 表和超级块。

![](img/0d06b3afdeb036834646c2193b4c0e3e_29.png)

![](img/0d06b3afdeb036834646c2193b4c0e3e_30.png)

![](img/0d06b3afdeb036834646c2193b4c0e3e_33.png)

格式化命令的核心是 `mkfs`（make filesystem）。对于 ext3 文件系统，有以下几种等效的格式化方式：

![](img/0d06b3afdeb036834646c2193b4c0e3e_35.png)

![](img/0d06b3afdeb036834646c2193b4c0e3e_36.png)

*   `mkfs.ext3 /dev/sda5`
*   `mkfs -t ext3 /dev/sda5`
*   `mke2fs -j /dev/sda5` （`-j` 选项表示为 ext2 添加日志功能，从而创建 ext3）

**重要警告**：**所有文件系统级别的操作（如格式化、调整属性、检查错误），都必须在分区未挂载（`umount`）的状态下进行**。对于根分区等无法卸载的分区，可以尝试以只读方式重新挂载（`mount -o remount,ro /`）。在已挂载的分区上执行这些操作极有可能导致数据损坏或丢失。

关于文件系统类型，ext3 本质上是 ext2 加上日志（journal）功能。可以使用 `tune2fs -j /dev/sda2` 命令为 ext2 分区添加日志，将其升级为 ext3。反之，使用 `tune2fs -O ^has_journal /dev/sda2` 可以移除日志，降级为 ext2。**请勿在生产系统上随意尝试这些操作。**

ext4 是 ext3 的演进，支持更大的单文件和分区容量（突破 ext3 的 2TB 限制），并拥有更好的性能。对于现代大容量存储，ext4 是更推荐的选择。

## 文件系统检查与修复 fsck

上一节我们介绍了如何创建文件系统，本节中我们来看看如何检查和修复文件系统。`fsck`（file system check）是用于检查并修复文件系统错误的工具。

执行文件系统检查的命令格式为 `fsck.ext3 /dev/sda5`。**再次强调，必须在分区未挂载时进行操作。**

`fsck` 检查过程可能非常危险。它会遍历文件系统结构，比对 inode 和 block 的对应关系。当发现不一致时，会询问用户是否修复。**处理此类问题的标准流程至关重要：**

1.  **评估损坏程度**：运行 `fsck` 时，先对所有询问选择 `n`（不修复），仅查看错误数量。如果错误数量巨大（成百上千），**应立即按 `Ctrl+C` 中断检查**，这通常意味着文件系统存在大面积损坏。
2.  **抢救数据**：在尝试修复前，应优先抢救数据。可以通过光盘启动或其他介质，将损坏分区挂载后，尽可能多地拷贝出尚可读取的数据。
3.  **创建磁盘镜像**：在修复前，使用 `dd` 等工具为损坏的磁盘或分区创建完整的镜像备份。这样即使修复失败，仍保有原始状态，可寻求专业数据恢复服务。
4.  **谨慎修复**：在完成数据抢救和备份后，方可尝试修复。可以使用 `fsck.ext3 -y /dev/sda5` 命令（`-y` 选项表示对所有询问自动回答“是”）。
5.  **文档与沟通**：记录问题发生时间、现象、采取的所有步骤。在操作关键系统前，应与上级或相关人员明确沟通并获得授权。

**日志（Journal）的作用**：ext3/ext4 等日志文件系统的 `fsck` 速度通常很快，因为它主要检查**日志**。日志记录了文件系统元数据的变更操作。系统意外崩溃后，`fsck` 通过重放（redo）或回滚日志，能快速将文件系统恢复到一致状态，而无需扫描整个磁盘。这是与 ext2 等非日志文件系统的根本区别。

如果需要强制进行完整的、非日志方式的深度检查（类似 ext2 的方式），可以使用 `-f` 参数，例如 `fsck.ext3 -f /dev/sda5`。此操作耗时较长，仅在怀疑日志本身或底层结构有严重问题时使用。

## 挂载与自动挂载

格式化完成后，需要将分区挂载到目录树才能使用。挂载命令为 `mount`，例如 `mount /dev/sda5 /mnt/data`。

为了使分区在系统启动时自动挂载，需要编辑 `/etc/fstab` 文件。添加一行的格式如下：
```
/dev/sda5  /mnt/data  ext3  defaults  1  2
```
各字段含义为：设备、挂载点、文件系统类型、挂载选项、dump备份标记、fsck检查顺序。

系统启动时，`init` 进程会执行 `mount -a` 命令，该命令会读取 `/etc/fstab` 文件并挂载其中列出的所有设备。

## 其他工具与文件系统简介

除了 `fdisk`，还有 `parted` 等图形或命令行分区工具。`parted` 功能强大，曾支持动态调整分区大小，但后续开发可能停滞，对某些文件系统（如 ext3）的高级功能支持有限。对于常规用途，`fdisk` 足够使用。

Linux 支持多种文件系统，各有特点：
*   **ext2/ext3/ext4**：Linux 原生系列，应用最广，生态支持最好。
*   **ReiserFS**：对小文件处理性能优异，采用了不同的数据结构算法。
*   **XFS/JFS**：64位文件系统，支持超大文件和分区，性能均衡，常用于企业级存储。
*   **Btrfs/ZFS**：下一代文件系统，提供高级功能如写时复制、快照、池化存储等。

选择文件系统时，需考虑发行版支持、功能需求、性能特点及社区活跃度。对于大多数 Red Hat Enterprise Linux 及其衍生版用户，ext4 或 XFS 是稳妥的选择。
