### 软链接

软链接（Symbolic Link）类似于Windows的快捷方式。以下是软链接的主要特点：

![](img/6234a15b2f4e38d19009c454ad6f2781_41.png)

*   **可以跨分区**：源文件和链接文件可以位于不同的磁盘分区。
*   **可以对目录创建链接**：可以为目录创建软链接。
*   **依赖源文件**：如果删除源文件，软链接将失效（变为“坏链接”）。

![](img/6234a15b2f4e38d19009c454ad6f2781_43.png)

创建软链接的命令格式是 `ln -s`。**注意：创建时必须使用绝对路径。**

以下是创建软链接的示例代码：
```bash
# 在/opt目录下创建源文件
touch /opt/hello_soft

# 为/opt/hello_soft创建软链接到/media目录（使用绝对路径）
ln -s /opt/hello_soft /media/hello_soft_link
```
软链接文件的权限显示为所有用户都有读写执行权限（`lrwxrwxrwx`），但其实际有效的权限取决于源文件的权限。

![](img/6234a15b2f4e38d19009c454ad6f2781_45.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_47.png)

### 硬链接

硬链接（Hard Link）与软链接有显著区别。以下是硬链接的主要特点：

![](img/6234a15b2f4e38d19009c454ad6f2781_49.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_51.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_52.png)

*   **不能跨分区**：源文件和硬链接必须在同一个文件系统（分区）内。
*   **不能对目录创建**：只能为文件创建硬链接。
*   **独立于源文件**：删除源文件后，硬链接文件仍然可用，因为硬链接与源文件共享相同的存储数据（inode）。

创建硬链接的命令是 `ln`（不加 `-s` 选项）。

以下是创建硬链接的示例代码：
```bash
# 在/opt目录下创建源文件
touch /opt/hello_hard

# 为/opt/hello_hard创建硬链接到/media目录
ln /opt/hello_hard /media/hello_hard_link
```
硬链接文件在外观和属性上与普通文件没有区别。对硬链接或源文件的修改会同步到另一方，因为它们指向同一数据块。

![](img/6234a15b2f4e38d19009c454ad6f2781_54.png)

**简单总结**：软链接是文件的“快捷方式”，硬链接是文件的“副本”（但共享存储空间）。软链接更灵活（可跨分区、对目录），但依赖源文件；硬链接限制更多，但删除源文件不影响链接。

## 获取命令帮助

![](img/6234a15b2f4e38d19009c454ad6f2781_56.png)

在Linux中，当我们不熟悉某个命令的用法时，可以查看其内置的帮助信息。

以下是几种获取命令帮助的方法：

![](img/6234a15b2f4e38d19009c454ad6f2781_58.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_60.png)

*   **`命令 --help`**：这是最常用、最快捷的方式，用于查看命令的基本用法和选项说明。例如：`ls --help`。
*   **`man 命令`**：查看命令的详细手册页（manual page），包含更全面的说明、选项和有时会有示例。例如：`man ls`。查看时可以使用与 `less` 命令相同的快捷键（如空格翻页，`q`退出）。
*   **`help 命令`**：主要用于查看Shell内部命令的帮助（如 `cd`, `echo`）。对于大多数外部命令（如 `ls`, `cp`）效果有限。
*   **`info 命令`**：提供另一种格式的帮助文档，但如今已较少使用。

![](img/6234a15b2f4e38d19009c454ad6f2781_62.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_64.png)

对于初学者，如果英文手册阅读困难，更推荐使用 `命令 --help` 结合互联网搜索（如百度）来学习命令的具体用法。

## 系统运行级别

![](img/6234a15b2f4e38d19009c454ad6f2781_72.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_74.png)

Linux系统有不同的运行级别（Runlevel），它定义了系统启动后运行哪些服务。

![](img/6234a15b2f4e38d19009c454ad6f2781_76.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_78.png)

系统共有7个运行级别（0-6），每个级别有特定用途：

![](img/6234a15b2f4e38d19009c454ad6f2781_80.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_82.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_84.png)

*   **0**：关机。
*   **1**：单用户模式。仅允许root用户登录，用于系统维护（如忘记密码）。
*   **2**：多用户模式（无NFS网络文件共享）。
*   **3**：完全的多用户模式（命令行界面），这是服务器最常用的标准级别。
*   **4**：未使用。
*   **5**：图形化界面模式，这是带桌面环境的系统的标准级别。
*   **6**：重启。

![](img/6234a15b2f4e38d19009c454ad6f2781_86.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_88.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_90.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_92.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_94.png)

可以使用 `runlevel` 命令查看当前和上一个运行级别。使用 `init N` 命令可以切换到指定运行级别（如 `init 3` 切换到命令行模式）。

![](img/6234a15b2f4e38d19009c454ad6f2781_96.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_98.png)

系统的默认运行级别由配置文件设定。可以使用 `systemctl get-default` 查看当前默认级别，使用 `systemctl set-default TARGET` 进行设置（例如 `systemctl set-default multi-user.target` 对应级别3，`graphical.target` 对应级别5）。

![](img/6234a15b2f4e38d19009c454ad6f2781_100.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_102.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_104.png)

## 关机与重启

![](img/6234a15b2f4e38d19009c454ad6f2781_106.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_108.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_110.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_112.png)

![](img/6234a15b2f4e38d19009c454ad6f2781_114.png)

在命令行下，我们可以使用以下命令来关机或重启系统：

*   **`poweroff`**：立即关闭系统。
*   **`reboot`**：立即重启系统。
*   **`shutdown`**：功能更丰富的关机命令。
    *   `shutdown -h now`：立即关机。
    *   `shutdown -r +10`：10分钟后重启。
*   **`init 0`**：切换到运行级别0（关机）。
*   **`init 6`**：切换到运行级别6（重启）。

![](img/6234a15b2f4e38d19009c454ad6f2781_116.png)

对于日常使用，记住 `poweroff` 和 `reboot` 这两个简单直接的命令即可。
