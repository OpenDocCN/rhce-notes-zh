### 用户与组管理作业

![](img/edd0768982b539599f88bbf49e265801_13.png)

第一道题是关于用户和组管理的。核心是使用 `useradd` 和 `usermod` 命令。

![](img/edd0768982b539599f88bbf49e265801_15.png)

*   **创建用户并修改组**：使用 `useradd` 创建用户，然后使用 `usermod -G` 将其添加到指定组。如果组不存在，需要先用 `groupadd` 创建。
*   **批量创建用户**：有两种方案。
    *   **方案一：直接使用命令**（注意：`useradd` 命令本身不支持 `{1..100}` 这样的花括号扩展来批量创建用户，这是一个常见的误解。此方法通常用于创建文件）。
    *   **方案二：编写Shell脚本**。这是RHCSA考试中会考察的脚本题。

以下是使用 `for` 循环脚本批量创建用户的示例：
```bash
#!/bin/bash
for i in `seq 1 100`
do
    useradd user$i
done
```
在这个脚本中，`seq 1 100` 生成从1到100的序列，`user$i` 中的 `$i` 是变量，在每次循环中依次取值为1,2,3...100。

### 用户密码的特殊设置

这是一个难点。题目要求使用 `useradd -p` 来设置密码，但直接使用明文密码是错误用法。

**正确步骤**：
1.  先创建一个临时用户（如 `tempuser`）并为其设置密码（例如 `123456`）。
2.  密码会加密后存储在 `/etc/shadow` 文件中。
3.  复制 `tempuser` 在 `/etc/shadow` 文件中对应的加密密码字符串（两个冒号之间的部分）。
4.  使用 `useradd -p` 创建新用户（如 `newuser`），并将复制的加密密码用**单引号**括起来作为参数。
    *   **命令示例**：`useradd newuser -p '$6$xxxx...xxxx'`
    *   **必须使用单引号**：因为加密字符串中包含 `$` 符号，在Shell中会被解释为变量。单引号能防止这种解释，双引号则不行。

### 软件包管理：RPM与YUM

*   **RPM与YUM的区别**：YUM是RPM的前端工具，是RPM包的集合。它们管理的都是二进制包。关键区别在于，YUM能**自动解决依赖关系**，而RPM需要手动安装所有依赖。
*   **企业中的选择**：
    *   **RPM**：常用于国企、银行等对软件来源有严格管控的环境。
    *   **YUM**：常用于大型互联网公司，便于批量部署，且常有自建的YUM仓库进行二次开发。
    *   **源码编译**：常用于中小企业，可控性高，能节省资源，但耗时较长。

## RAID（磁盘阵列）技术详解 💾

上一节我们学习了如何管理固定大小的分区，但它们有一个致命问题：**无法动态调整（扩容）**。同时，机械硬盘在大量IO操作下容易损坏或导致数据丢失。RAID技术就是为了解决**扩容、备份（冗余）和提速**这三个问题而诞生的。

RAID有多种级别（RAID 0, 1, 5, 6, 10等），数字不代表优劣，而是代表不同的技术方案。

### 常见RAID级别

*   **RAID 0（条带化 Stripping）**：
    *   **作用**：单纯**提速**，**无备份**。
    *   **原理**：数据分散存储在N块硬盘上，同时读写。
    *   **速率**：N块盘，速率提升近N倍。
    *   **利用率**：100%，但任意一块盘损坏，所有数据丢失。
    *   **最少盘数**：2块。

*   **RAID 1（镜像 Mirroring）**：
    *   **作用**：提供**备份**，**不提速**。
    *   **原理**：数据同时写入两块盘，完全镜像。
    *   **利用率**：50%（2块盘时）。原则上使用2N块盘，总容量为N块盘的容量。
    *   **最少盘数**：2块。

*   **RAID 5（分布式奇偶校验）**：
    *   **作用**：**兼顾提速和备份**，是单盘校验。
    *   **原理**：至少需要3块盘。数据条带化存储，同时会计算一个奇偶校验信息，轮流存储在每块盘上。允许损坏**至多1块**盘。
    *   **利用率**：(N-1)/N （N为盘数，例如3块盘利用率为2/3）。
    *   **最少盘数**：3块。

*   **RAID 6（双重分布式奇偶校验）**：
    *   **作用**：比RAID 5更高的可靠性，是双盘校验。
    *   **原理**：至少需要4块盘。使用两种不同的校验算法，允许损坏**至多2块**盘。
    *   **利用率**：(N-2)/N。
    *   **最少盘数**：4块。

*   **RAID 10（先镜像再条带） / RAID 01（先条带再镜像）**：
    *   **作用**：RAID 1和RAID 0的结合，**既提速又备份**。
    *   **原理**：至少需要4块盘。以RAID 10为例，先将盘两两做成RAID 1（镜像），再将这两个RAID 1组合成RAID 0（条带）。
    *   **利用率**：50%。
    *   **最少盘数**：4块。

> **生产环境选择**：成本足够选RAID 10，成本有限选RAID 5。

### 软RAID实战管理

![](img/edd0768982b539599f88bbf49e265801_37.png)

![](img/edd0768982b539599f88bbf49e265801_39.png)

在Linux中，我们使用 `mdadm` 命令来管理软RAID阵列。

![](img/edd0768982b539599f88bbf49e265801_41.png)

![](img/edd0768982b539599f88bbf49e265801_43.png)

![](img/edd0768982b539599f88bbf49e265801_45.png)

以下是 `mdadm` 命令的常见参数：
*   `-C` 或 `--create`：创建阵列。
*   `-v` 或 `--verbose`：显示详细过程。
*   `-l` 或 `--level`：指定RAID级别。
*   `-n` 或 `--raid-devices`：指定活动磁盘的数量。
*   `-x` 或 `--spare-devices`：指定备用磁盘的数量。
*   `-a` 或 `--auto`：同意创建设备，通常用于检测设备名。
*   `-S` 或 `--stop`：停止阵列。
*   `-D` 或 `--detail`：查看阵列详细信息。
*   `-f` 或 `--fail`：模拟标记一块磁盘为故障。
*   `-r` 或 `--remove`：移除一块磁盘。
*   `-a` 或 `--add`：添加一块磁盘。

![](img/edd0768982b539599f88bbf49e265801_47.png)

![](img/edd0768982b539599f88bbf49e265801_49.png)

**创建RAID 10阵列示例**：
假设我们有4块硬盘 `/dev/sdb`, `/dev/sdc`, `/dev/sdd`, `/dev/sde`。
```bash
mdadm -Cv /dev/md0 -l 10 -n 4 /dev/sdb /dev/sdc /dev/sdd /dev/sde
```
*   `/dev/md0` 是创建的RAID设备名。
*   `-l 10` 指定RAID级别为10。
*   `-n 4` 指定使用4块活动盘。

![](img/edd0768982b539599f88bbf49e265801_51.png)

**创建带备用盘的RAID 5阵列示例**：
假设我们有4块硬盘，其中3块做RAID 5，1块做备用。
```bash
mdadm -Cv /dev/md0 -l 5 -n 3 -x 1 /dev/sdb /dev/sdc /dev/sdd /dev/sde
```
*   `-x 1` 指定1块备用盘。**注意**：备用盘参数要放在所有活动盘之后。

![](img/edd0768982b539599f88bbf49e265801_53.png)

**管理RAID阵列**：
1.  **查看阵列状态**：`mdadm -D /dev/md0`
2.  **模拟磁盘损坏**：`mdadm /dev/md0 -f /dev/sdb`
3.  **移除损坏磁盘**：`mdadm /dev/md0 -r /dev/sdb`
4.  **添加新磁盘**：`mdadm /dev/md0 -a /dev/sdf`
5.  **停止阵列**：`mdadm -S /dev/md0`

![](img/edd0768982b539599f88bbf49e265801_55.png)

**格式化与挂载**：
创建RAID设备后，可以像普通磁盘一样进行格式化和挂载。
```bash
mkfs.ext4 /dev/md0          # 格式化为ext4文件系统
mkdir /mnt/raid10           # 创建挂载点
mount /dev/md0 /mnt/raid10  # 挂载
```
将挂载信息写入 `/etc/fstab` 以实现开机自动挂载。

> **注意**：软RAID设备在重启后，名称可能会从 `/dev/md0` 变为 `/dev/md127`。为了解决这个问题，需要创建配置文件 `/etc/mdadm.conf`。可以使用 `mdadm --detail --scan >> /etc/mdadm.conf` 命令生成初始配置。

## LVM（逻辑卷管理）技术详解 🗂️

RAID解决了部分问题，但扩容仍然不够灵活。LVM（Logical Volume Manager）就是为了实现存储空间的**动态、灵活管理**而设计的。

![](img/edd0768982b539599f88bbf49e265801_63.png)

![](img/edd0768982b539599f88bbf49e265801_65.png)

### LVM核心概念

![](img/edd0768982b539599f88bbf49e265801_67.png)

LVM将存储管理分为三层：
1.  **PV（Physical Volume，物理卷）**：可以是整块硬盘、硬盘分区，甚至是RAID设备（如 `/dev/md0`）。这是LVM的底层物理存储单元。
2.  **VG（Volume Group，卷组）**：由一个或多个PV组成。VG是LVM管理的基本单位，PV不能直接使用，必须加入VG。VG将多个不同大小的PV整合成一个大的、统一的存储池。
3.  **LV（Logical Volume，逻辑卷）**：从VG中划分出来的逻辑区块。LV可以被格式化并挂载使用，就像普通分区一样。我们最终操作的是LV。

![](img/edd0768982b539599f88bbf49e265801_69.png)

![](img/edd0768982b539599f88bbf49e265801_71.png)

**比喻**：
*   **PV（物理卷）** = 面粉袋
*   **VG（卷组）** = 和面盆里的面团（由多个面粉袋的面粉混合而成）
*   **LV（逻辑卷）** = 从面团上切下来准备蒸的小馒头
*   **PE（Physical Extent，物理扩展块）** = 面粉（默认4MB，是VG中最小的分配单位）

![](img/edd0768982b539599f88bbf49e265801_73.png)

![](img/edd0768982b539599f88bbf49e265801_75.png)

### LVM基本操作流程

![](img/edd0768982b539599f88bbf49e265801_77.png)

![](img/edd0768982b539599f88bbf49e265801_79.png)

**第一步：创建物理卷（PV）**
```bash
pvcreate /dev/sdb1 /dev/sdc1  # 将分区创建为PV
pvdisplay                     # 查看PV信息
```

![](img/edd0768982b539599f88bbf49e265801_81.png)

**第二步：创建卷组（VG）**
```bash
vgcreate vg_data /dev/sdb1 /dev/sdc1  # 创建名为vg_data的VG，并加入两个PV
vgdisplay                             # 查看VG信息
```

**第三步：创建逻辑卷（LV）**
创建LV时有两种指定大小的方法：
*   `-L`：指定具体容量（如 `-L 10G`）。如果容量不是PE大小的整数倍，系统会**向上取整**。
*   `-l`：指定PE的个数（如 `-l 2560`）。每个PE默认4MB，2560个PE就是10GB。
```bash
lvcreate -n lv_www -L 10G vg_data  # 在vg_data中创建名为lv_www，大小为10G的LV
lvdisplay                           # 查看LV信息
```
创建成功后，LV的设备路径通常为 `/dev/卷组名/逻辑卷名`，例如 `/dev/vg_data/lv_www`。

**第四步：格式化并挂载LV**
```bash
mkfs.ext4 /dev/vg_data/lv_www  # 格式化
mkdir /www                     # 创建挂载点
mount /dev/vg_data/lv_www /www # 挂载
```

### LVM扩容操作

![](img/edd0768982b539599f88bbf49e265801_83.png)

![](img/edd0768982b539599f88bbf49e265801_85.png)

这是LVM最大的优势。扩容前，请先使用 `vgdisplay` 确认VG有足够的剩余空间。

![](img/edd0768982b539599f88bbf49e265801_87.png)

![](img/edd0768982b539599f88bbf49e265801_89.png)

**情况一：VG空间充足，直接扩展LV**
```bash
lvextend -L +5G /dev/vg_data/lv_www  # 为lv_www增加5G空间
# 或扩展到指定大小
# lvextend -L 15G /dev/vg_data/lv_www
```
扩展LV后，文件系统并不会自动变大，需要同步调整文件系统大小。
```bash
# 对于ext2/3/4文件系统
resize2fs /dev/vg_data/lv_www
# 对于xfs文件系统
# xfs_growfs /www
```

![](img/edd0768982b539599f88bbf49e265801_91.png)

![](img/edd0768982b539599f88bbf49e265801_93.png)

**情况二：VG空间不足，需要先扩展VG**
1.  将新的硬盘或分区创建为PV。
    ```bash
    pvcreate /dev/sdd1
    ```
2.  将新的PV扩展到现有的VG中。
    ```bash
    vgextend vg_data /dev/sdd1
    ```
3.  此时VG有了新空间，再按照“情况一”扩展LV和文件系统。

### LVM快照卷

![](img/edd0768982b539599f88bbf49e265801_101.png)

![](img/edd0768982b539599f88bbf49e265801_103.png)

![](img/edd0768982b539599f88bbf49e265801_104.png)

![](img/edd0768982b539599f88bbf49e265801_106.png)

LVM快照卷提供了一种瞬间“冻结”LV状态的方法，常用于数据备份或恢复测试。
*   **特点**：快照卷创建后，对原LV的修改会被记录，快照卷大小应能容纳这些更改。用完后即删除。
*   **创建快照**：
    ```bash
    lvcreate -L 1G -s -n lv_www_snap /dev/vg_data/lv_www
    ```
    *   `-s`：创建快照。
    *   `-n lv_www_snap`：快照卷名称。
    *   `-L 1G`：为快照卷分配1G空间（用于存储变更数据）。
*   **恢复快照**：
    ```bash
    umount /www                              # 卸载原LV
    lvconvert --merge /dev/vg_data/lv_www_snap # 合并快照，恢复数据
    mount /dev/vg_data/lv_www /www           # 重新挂载
    ```
    恢复后，快照卷会自动删除。

### LVM缩容与删除

![](img/edd0768982b539599f88bbf49e265801_108.png)

![](img/edd0768982b539599f88bbf49e265801_109.png)

![](img/edd0768982b539599f88bbf49e265801_111.png)

*   **缩容**：**生产环境慎用！** 缩容需要先卸载文件系统，并强制进行文件系统检查 (`fsck`)，数据丢失风险高。操作顺序与扩容相反：先缩小文件系统，再缩小LV。
*   **删除LVM**：操作顺序与创建相反。
    1.  卸载LV：`umount /www`
    2.  删除LV：`lvremove /dev/vg_data/lv_www`
    3.  删除VG：`vgremove vg_data`
    4.  删除PV：`pvremove /dev/sdb1 /dev/sdc1`

## RAID与LVM结合实践 🔗

在实际生产环境中，为了同时获得RAID的性能/可靠性优势和LVM的灵活性，通常采用**底层用RAID，上层用LVM**的架构。

![](img/edd0768982b539599f88bbf49e265801_123.png)

**操作思路**：
1.  使用多块物理磁盘创建RAID阵列（如RAID 5, RAID 10），得到RAID设备（如 `/dev/md0`）。
2.  将这个RAID设备 `/dev/md0` 直接创建为物理卷（PV）：`pvcreate /dev/md0`。
3.  后续的VG创建、LV划分、扩容等操作，都基于这个RAID设备构成的PV进行。

![](img/edd0768982b539599f88bbf49e265801_129.png)

这样，你的逻辑卷就建立在了一个具有冗余或提速能力的RAID基础之上，并且仍然享受LVM带来的灵活管理特性。
