# RHCE8.0 红帽认证入门教程：Day03：进程管理 🖥️

在本节课中，我们将要学习 Linux 系统中的进程管理。进程是系统运行的核心，理解如何查看、分析和控制进程是系统管理的基础技能。我们将从进程的基本概念开始，逐步学习如何查看进程信息、分析资源占用以及理解进程之间的关系。

## 什么是进程？

进程可以理解为“正在运行的程序”。一个静态的程序文件（如 `/usr/bin/passwd`）被加载到内存并执行时，就产生了一个动态的进程。这与 Windows 系统中的“任务管理器”里看到的进程概念是相似的。

进程会占用系统的资源，主要包括 CPU、内存和硬盘 I/O。系统管理员需要监控这些资源的使用情况，以判断系统运行是否正常，是否存在异常进程（如病毒、挖矿程序）。

## 进程的来源与拥有者

![](img/5113a0f60670af36d3134f672fc27215_1.png)

上一节我们介绍了进程的基本概念，本节中我们来看看进程是如何产生的以及它的“拥有者”是谁。

![](img/5113a0f60670af36d3134f672fc27215_3.png)

进程通常由以下方式触发：
*   执行一个二进制程序文件，例如在终端输入 `passwd` 命令。
*   启动一个系统服务，例如使用 `systemctl start httpd` 启动 Web 服务。

![](img/5113a0f60670af36d3134f672fc27215_5.png)

![](img/5113a0f60670af36d3134f672fc27215_7.png)

关于进程的拥有者，需要理解以下几点：
*   默认情况下，谁执行了程序，产生的进程就属于谁。
*   许多系统服务（如 `httpd`, `nginx`, `mysqld`）会创建专门的系统用户来运行其进程，以提升安全性。这通常在服务的配置文件中定义。
*   进程在读取或写入文件时，使用的是其拥有者的身份和权限。因此，进程的拥有者必须具备访问所需资源的相应权限。

## 如何监控进程与系统资源

了解进程的来源后，我们需要掌握监控它们的方法。在 Linux 中，我们主要使用命令行工具来实时查看和分析进程状态及系统资源使用情况。

![](img/5113a0f60670af36d3134f672fc27215_27.png)

以下是三个核心命令：

![](img/5113a0f60670af36d3134f672fc27215_29.png)

**1. `ps` 命令**
`ps` 命令用于查看当前时刻的进程快照。不带任何参数时，它只显示当前终端会话的进程。

常用组合选项 `ps aux` 可以列出系统所有用户的详细进程信息。其输出包含的关键字段有：
*   `USER`: 进程拥有者。
*   `PID`: 进程的唯一 ID。
*   `%CPU` / `%MEM`: 占用 CPU 和内存的百分比。
*   `STAT`: 进程状态代码（例如，`R` 运行，`S` 睡眠，`Z` 僵尸进程）。
*   `COMMAND`: 启动进程的命令。

**2. `pstree` 命令**
`pstree` 命令以树状图形式显示进程之间的关系，可以直观地看到父进程与子进程的层级结构。使用 `pstree -p` 可以同时显示 PID。

![](img/5113a0f60670af36d3134f672fc27215_31.png)

![](img/5113a0f60670af36d3134f672fc27215_33.png)

![](img/5113a0f60670af36d3134f672fc27215_35.png)

![](img/5113a0f60670af36d3134f672fc27215_37.png)

**3. `top` 命令**
`top` 命令用于动态、实时地监控系统资源使用情况和进程列表。它提供了一个全屏交互式界面，信息持续更新。

`top` 界面主要分为两部分：
*   **系统概要区**：显示当前时间、系统运行时长、负载平均值、任务总数、CPU 总体使用率、物理内存和交换空间使用情况。
    *   负载平均值（load average）格式为：`1分钟, 5分钟, 15分钟`。
*   **进程列表区**：类似 `ps aux` 的实时列表，默认按 CPU 使用率排序。你可以按 `M` 键改为按内存使用率排序。

## 进程的状态

在使用 `ps` 或 `top` 命令时，`STAT` 字段显示了进程的状态。理解这些状态字母对排查问题很有帮助：

![](img/5113a0f60670af36d3134f672fc27215_39.png)

![](img/5113a0f60670af36d3134f672fc27215_41.png)

![](img/5113a0f60670af36d3134f672fc27215_43.png)

![](img/5113a0f60670af36d3134f672fc27215_45.png)

*   **R (Running)**: 正在运行或可运行（在运行队列中等待）。
*   **S (Sleeping)**: 睡眠状态，可中断，通常在等待某个事件（如 I/O 完成）。
*   **D (Uninterruptible Sleep)**: 不可中断的睡眠，通常是在等待硬件 I/O，不响应信号。
*   **T (Stopped)**: 进程被停止（如由调试器暂停）。
*   **Z (Zombie)**: 僵尸进程。进程已终止，但其父进程尚未回收它，残留部分信息在进程表中。
