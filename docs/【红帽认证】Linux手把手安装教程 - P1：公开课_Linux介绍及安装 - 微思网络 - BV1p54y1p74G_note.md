## Linux的历史与发展

想要了解Linux，首先需要知道它的历史。时间回到1958年，美国电话电报公司（AT&T）在当时垄断了长途电话业务。根据一份协议，AT&T不能涉足计算机行业或销售任何IT产品。

![](img/8460c11264de66eca5a9492af2fbe087_7.png)

![](img/8460c11264de66eca5a9492af2fbe087_9.png)

![](img/8460c11264de66eca5a9492af2fbe087_11.png)

到了1969年，AT&T旗下的贝尔实验室发明了Unix操作系统。由于版权属于AT&T且受协议限制不能销售，AT&T决定将Unix系统开源。

![](img/8460c11264de66eca5a9492af2fbe087_13.png)

![](img/8460c11264de66eca5a9492af2fbe087_15.png)

![](img/8460c11264de66eca5a9492af2fbe087_17.png)

![](img/8460c11264de66eca5a9492af2fbe087_19.png)

### Unix系统的特点
Unix系统有四个主要优点：
1.  **高可靠性**：采用模块化设计，每个组件独立工作。一个组件出问题不会影响整个系统。
2.  **兼容性好**：使用当时流行的C语言编写，开发人员使用同一种语言，便于沟通和开发。
3.  **高效**：是首款支持多任务的操作系统。当时的CPU按顺序执行任务，如果一个任务卡住，后续任务必须等待。Unix的多任务机制如同景区开设多个售票窗口，提升了资源利用效率。
4.  **生态系统好**：由于开源且免费，大家拿到源码后可以修改并发布自己的版本，促进了生态繁荣。

时间跳跃到1983年，AT&T发布了首款商业版Unix操作系统并开始收费闭源。这一举动在1984年引发了理查德·斯托曼发起的GNU计划。

### GNU计划与开源精神
GNU计划的目的是让系统重新免费分发。具体做法是用自己的语言重写Unix系统上的所有软件并免费发布。该计划认为从开源转向闭源违背了开放共享的精神。

![](img/8460c11264de66eca5a9492af2fbe087_21.png)

![](img/8460c11264de66eca5a9492af2fbe087_23.png)

GNU计划做了两件重要的事：
*   发布了许多免费软件（如后续会学到的Shell）。
*   提出了**GPL（GNU通用公共许可证）协议**。该协议规定，使用GPL发布的软件，他人可以免费修改，但二次分发时必须开源修改后的代码，以此传递开源精神。

![](img/8460c11264de66eca5a9492af2fbe087_25.png)

![](img/8460c11264de66eca5a9492af2fbe087_27.png)

![](img/8460c11264de66eca5a9492af2fbe087_29.png)

![](img/8460c11264de66eca5a9492af2fbe087_31.png)

除了严格的GPL协议，还有宽松的许可证，如**BSD协议**。BSD协议允许二次分发时不开源，苹果的macOS系统就基于BSD。

![](img/8460c11264de66eca5a9492af2fbe087_33.png)

![](img/8460c11264de66eca5a9492af2fbe087_35.png)

然而，GNU计划编写了大量软件，却始终未能成功开发出系统的核心——内核。没有内核，就像汽车有了轮胎和方向盘，但没有发动机，无法运行。

![](img/8460c11264de66eca5a9492af2fbe087_37.png)

## Linux内核的诞生与GNU/Linux

![](img/8460c11264de66eca5a9492af2fbe087_39.png)

![](img/8460c11264de66eca5a9492af2fbe087_41.png)

时间来到1991年，芬兰大学生林纳斯·托瓦兹编写出了**Linux内核**，并使用GPL协议发布。

![](img/8460c11264de66eca5a9492af2fbe087_43.png)

![](img/8460c11264de66eca5a9492af2fbe087_45.png)

![](img/8460c11264de66eca5a9492af2fbe087_47.png)

![](img/8460c11264de66eca5a9492af2fbe087_49.png)

此时，GNU组织拥有大量软件但缺少内核，而Linux内核恰好需要配套的软件才能构成完整可用的操作系统。两者结合，便形成了完整的**GNU/Linux操作系统**。

![](img/8460c11264de66eca5a9492af2fbe087_51.png)

![](img/8460c11264de66eca5a9492af2fbe087_53.png)

因此，严格来说，我们常说的“Linux操作系统”指的是GNU/Linux。单独说“Linux”时，通常指代其内核。

由于Linux系统开源且允许免费二次分发，开发者们创建了众多不同的发行版本。

![](img/8460c11264de66eca5a9492af2fbe087_55.png)

## 主要的Linux发行版

![](img/8460c11264de66eca5a9492af2fbe087_57.png)

![](img/8460c11264de66eca5a9492af2fbe087_59.png)

以下是部分知名的Linux发行版：
*   **Red Hat Enterprise Linux (RHEL)**：红帽企业版，商业发行版。
*   **Fedora**：红帽的社区版。
*   **CentOS**：社区企业操作系统，现已被红帽收编，基于RHEL源码重新编译。
*   **Ubuntu**：非常流行的桌面发行版。
*   **Deepin**：中国深度公司开发的发行版，针对国人使用习惯做了定制。
*   **Tails**：注重隐私安全的发行版，运行在U盘上，使用洋葱路由，数据不留痕。
*   **Kali Linux**：著名的渗透测试和安全审计发行版，内置大量安全工具。
*   **Oracle Linux**：甲骨文公司基于RHEL定制的发行版。
*   **SteamOS**：Valve公司为Steam游戏平台定制的发行版，简化了显卡驱动安装。

![](img/8460c11264de66eca5a9492af2fbe087_61.png)

若想了解更多发行版，可以访问 [DistroWatch](https://distrowatch.com) 网站查看排名和详细信息。

![](img/8460c11264de66eca5a9492af2fbe087_63.png)

### 发行版的两大体系
众多发行版主要基于两大体系：
1.  **Red Hat系**：如RHEL、CentOS、Fedora。在商业环境中应用广泛。
2.  **Debian系**：如Ubuntu、树莓派系统。社区维护的发行版数量较多。

![](img/8460c11264de66eca5a9492af2fbe087_65.png)

## 红帽公司与其认证体系

![](img/8460c11264de66eca5a9492af2fbe087_67.png)

红帽公司是全球领先的开源解决方案供应商，也是开源社区的主要贡献者之一。它针对自家系统推出了权威的认证体系：
*   **RHCSA (红帽认证系统管理员)**：基础认证。
*   **RHCE (红帽认证工程师)**：进阶认证，很多运维人员持有。考试为上机实操。
*   **RHCA (红帽认证架构师)**：高级认证，需通过多门考试。

根据红帽统计，财富500强公司中90%的服务器使用Linux系统。

## 安装CentOS Linux系统

![](img/8460c11264de66eca5a9492af2fbe087_71.png)

![](img/8460c11264de66eca5a9492af2fbe087_73.png)

![](img/8460c11264de66eca5a9492af2fbe087_74.png)

![](img/8460c11264de66eca5a9492af2fbe087_76.png)

我们将使用虚拟机软件（如VMware）来安装CentOS 8.3的最小化版本。

![](img/8460c11264de66eca5a9492af2fbe087_78.png)

![](img/8460c11264de66eca5a9492af2fbe087_80.png)

### 第一步：创建虚拟机
1.  打开虚拟机软件，选择“新建虚拟机”。
2.  选择“典型”配置，点击下一步。
3.  选择“稍后安装操作系统”，点击下一步。
4.  客户机操作系统选择“Linux”，版本选择“CentOS 8 64位”，点击下一步。
5.  命名虚拟机并选择存储位置，点击下一步。
6.  指定磁盘容量（例如30GB），点击下一步直至完成。

![](img/8460c11264de66eca5a9492af2fbe087_82.png)

![](img/8460c11264de66eca5a9492af2fbe087_84.png)

### 第二步：调整虚拟机设置与下载镜像
1.  编辑虚拟机设置，适当调高内存（如2GB）和处理器核心数（如2核）。
2.  从镜像站下载CentOS 8.3的`Minimal`版本ISO镜像文件。企业环境通常选择最小化安装以提升安全性。
3.  在虚拟机设置中，将下载的ISO镜像文件挂载到虚拟光驱。

### 第三步：启动安装程序
1.  启动虚拟机，在引导界面选择“Install CentOS Linux 8.0”。
2.  进入安装界面后，首先选择语言。**建议选择英文**，以避免后续可能出现的字符编码问题。
3.  需要配置以下几个关键部分（带红色感叹号的为必选项）：
    *   **安装目标 (INSTALLATION DESTINATION)**：选择之前创建的30GB硬盘，使用自动分区方案即可。
    *   **软件选择 (SOFTWARE SELECTION)**：选择 **`Minimal Install`** （最小化安装）。
    *   **时间和日期 (TIME & DATE)**：选择亚洲/上海时区，并设置正确时间。
    *   **根密码 (ROOT PASSWORD)**：为超级管理员用户`root`设置密码。
4.  所有必选项配置完成后，“开始安装 (Begin Installation)”按钮将变为可用，点击开始安装。
5.  安装完成后，点击“重启系统 (Reboot System)”。

### 第四步：首次登录与基础配置
1.  系统重启后，使用`root`用户名和设置的密码登录。
2.  登录后第一件事是**创建系统快照**，相当于游戏存档，方便后续实验出错时恢复。
3.  为虚拟机配置网络，使其能获取IP地址。使用命令 `nmtui` 激活网卡连接。
4.  使用 `ip a` 命令查看获取到的IP地址。
5.  在物理机上使用终端软件（如Xshell、SecureCRT）通过该IP地址连接到虚拟机，便于复制粘贴和操作。
6.  进行两项重要的基础配置（后续实验所需）：
    *   **关闭防火墙**：
        ```bash
        systemctl stop firewalld
        systemctl disable firewalld
        ```
    *   **禁用SELinux**：
        编辑配置文件 `/etc/selinux/config`，将 `SELINUX=enforcing` 改为 `SELINUX=disabled`。
7.  再次关机，并创建一个名为“预配置”的新快照，记录下IP地址和已完成的配置。
