## 概述：文件归属关系

![](img/9cc8ff33ade93b2f6f54530b35b04b87_11.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_13.png)

上一节我们介绍了用户和组的管理，本节中我们来看看如何通过权限来控制他们对文件的访问。首先，我们需要理解文件的归属关系。

![](img/9cc8ff33ade93b2f6f54530b35b04b87_15.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_17.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_18.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_19.png)

任何文件或目录都有三类归属关系：
*   **所有者 (User, U)**：文件的创建者，通常是文件的拥有者。
*   **所属组 (Group, G)**：文件所属的用户组，组内的成员可以继承该组的权限。
*   **其他人 (Other, O)**：既不是文件所有者，也不在文件所属组内的其他所有用户。

我们可以通过 `ls -l` 命令查看文件的详细信息，其中就包含了归属关系和权限。

```bash
[root@localhost ~]# ls -l /opt/hello.txt
-rw-r--r--. 1 root root 0  Nov 1 10:00 /opt/hello.txt
```
*   `-rw-r--r--`：第一部分代表文件类型和权限。
*   `root`：第一个 `root` 代表文件的所有者是 `root` 用户。
*   `root`：第二个 `root` 代表文件的所属组是 `root` 组。

## 基本权限详解

文件的权限由9个字符表示，每3个字符为一组，分别对应所有者(U)、所属组(G)和其他人(O)的权限。

权限的顺序永远是：**读 (r)** -> **写 (w)** -> **执行 (x)**。如果某个权限位是 `-`，则表示没有该权限。

以下是三类权限的具体含义：

![](img/9cc8ff33ade93b2f6f54530b35b04b87_21.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_23.png)

### 1. 针对文件的权限
*   **读 (r)**：允许查看文件的内容。例如，使用 `cat`、`less` 命令。
*   **写 (w)**：允许修改文件的内容。包括向文件中添加、删除、修改数据。
*   **执行 (x)**：允许将文件作为程序或脚本运行。

![](img/9cc8ff33ade93b2f6f54530b35b04b87_25.png)

### 2. 针对目录的权限
目录的权限含义与文件不同，这一点至关重要。
*   **读 (r)**：允许列出目录下的文件和子目录名称。例如，使用 `ls` 命令。
*   **写 (w)**：允许在目录内**创建、删除、重命名**文件和子目录。**注意**：即使你对目录下的某个文件没有写权限，只要你对目录有写权限，你仍然可以删除该文件。
*   **执行 (x)**：允许进入 (`cd`) 该目录，并访问目录内的元数据。

**核心区别总结**：文件的 `w` 权限控制文件**内容**的修改；目录的 `w` 权限控制目录**条目**（即文件本身）的增删。

![](img/9cc8ff33ade93b2f6f54530b35b04b87_27.png)

## 修改文件归属：chown命令

在设置权限前，有时我们需要先修改文件的拥有者或所属组。这需要使用 `chown` 命令。

`chown` 命令的基本格式如下：
```bash
chown [选项] 所有者:所属组 文件或目录
```

以下是几种常用用法示例：

```bash
# 1. 同时修改所有者和所属组
chown tom:tom hello.txt

# 2. 只修改所有者
chown root hello.txt

# 3. 只修改所属组 (两种写法等效)
chown :ops hello.txt
chgrp ops hello.txt # 使用专门的chgrp命令

# 4. 递归修改目录及其内部所有内容的归属
chown -R tom:tom /data/project/
```

## 修改文件权限：chmod命令

`chmod` 命令用于修改文件或目录的权限。它有两种主要的权限表示法：字母法和数字法。

### 字母表示法
语法：`chmod [ugoa][+-=][rwx] 文件或目录`
*   `[ugoa]`：指定权限作用对象。
    *   `u`：所有者 (User)
    *   `g`：所属组 (Group)
    *   `o`：其他人 (Other)
    *   `a`：所有人 (All)，是 `ugo` 的简写
*   `[+-=]`：指定操作。
    *   `+`：添加权限
    *   `-`：移除权限
    *   `=`：精确设置权限（覆盖原有）
*   `[rwx]`：权限类型。

以下是操作示例：

![](img/9cc8ff33ade93b2f6f54530b35b04b87_29.png)

```bash
# 给文件所有者添加执行权限
chmod u+x script.sh

![](img/9cc8ff33ade93b2f6f54530b35b04b87_31.png)

# 移除所属组和其他人的写权限
chmod go-w secret.txt

![](img/9cc8ff33ade93b2f6f54530b35b04b87_33.png)

# 为所有者设置读写执行，为所属组设置读执行，为其他人设置无权限
chmod u=rwx,g=rx,o= file.txt

![](img/9cc8ff33ade93b2f6f54530b35b04b87_35.png)

# 给所有人添加读权限
chmod a+r public.txt
```

### 数字表示法（八进制）
每个权限位可以用一个数字代表：
*   `r` (读) = 4
*   `w` (写) = 2
*   `x` (执行) = 1
*   `-` (无) = 0

![](img/9cc8ff33ade93b2f6f54530b35b04b87_37.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_39.png)

三类用户的权限数字由各自权限值相加得到，然后按顺序（U, G, O）组合成一个三位数。

**权限计算示例**：
*   `rwx` = 4+2+1 = **7**
*   `rw-` = 4+2+0 = **6**
*   `r-x` = 4+0+1 = **5**
*   `r--` = 4+0+0 = **4**

```bash
# 设置权限为 rwxr-xr-- (所有者读写执行，所属组读执行，其他人只读)
chmod 754 file.txt
# 7 -> u=rwx, 5 -> g=r-x, 4 -> o=r--
```

## 默认权限与umask

![](img/9cc8ff33ade93b2f6f54530b35b04b87_43.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_45.png)

当我们创建新的文件或目录时，系统会赋予一个默认的权限。这个默认权限由 `umask`（权限掩码）值决定。

![](img/9cc8ff33ade93b2f6f54530b35b04b87_47.png)

`umask` 值表示要从默认权限中“减去”的权限。查看当前 `umask`：

![](img/9cc8ff33ade93b2f6f54530b35b04b87_49.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_51.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_53.png)

```bash
# 以数字形式查看（常用）
umask
# 输出示例：0022

![](img/9cc8ff33ade93b2f6f54530b35b04b87_55.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_57.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_58.png)

# 以符号形式查看
umask -S
# 输出示例：u=rwx,g=rx,o=rx
```

**umask 计算规则**：
*   **目录的默认最大权限**是 `777` (rwxrwxrwx)。
*   **文件的默认最大权限**是 `666` (rw-rw-rw-)，因为新建文件通常不需要执行权限。
*   实际权限 = 最大权限 - umask值（进行按位“与”的非运算更准确，但减法便于理解）。

例如，当 `umask=022` 时：
*   新建目录权限：`777 - 022 = 755` -> `rwxr-xr-x`
*   新建文件权限：`666 - 022 = 644` -> `rw-r--r--`

你可以临时修改当前会话的 `umask`，或将其写入shell配置文件（如 `~/.bashrc`）使其永久生效。

![](img/9cc8ff33ade93b2f6f54530b35b04b87_60.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_62.png)

```bash
# 设置umask，使新建文件对同组用户可写
umask 002
```

## 特殊权限

除了基本的rwx权限，Linux还有三个特殊权限位：`SUID`, `SGID`, `Sticky Bit`。它们在权限字符串中占用执行位（x）的位置。

### 1. SUID (Set User ID)
*   **作用于文件**：当用户执行带有SUID位的程序时，程序会以**文件所有者**的身份运行，而不是执行者。
*   **设置方法**：
    *   字母法：`chmod u+s file`
    *   数字法：在三位权限数字前加4，如 `4755`
*   **典型示例**：`/usr/bin/passwd` 命令，普通用户执行时可以修改自己的密码（实质是修改了 `/etc/shadow` 文件，该文件只有root可写）。

![](img/9cc8ff33ade93b2f6f54530b35b04b87_64.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_66.png)

### 2. SGID (Set Group ID)
*   **作用于文件**：与SUID类似，程序运行时以**文件所属组**的身份运行。
*   **作用于目录**：**更为常用**。在该目录下创建的任何新文件或子目录，其所属组都会自动继承该目录的所属组，而不是创建者的默认组。这对于团队协作共享目录非常有用。
*   **设置方法**：
    *   字母法：`chmod g+s dir`
    *   数字法：在三位权限数字前加2，如 `2755`

![](img/9cc8ff33ade93b2f6f54530b35b04b87_68.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_70.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_72.png)

### 3. Sticky Bit (粘滞位)
*   **作用于目录**：在具有写权限的目录中（如 `/tmp`），用户可以创建和删除自己的文件，但**不能删除或重命名其他用户的文件**。
*   **设置方法**：
    *   字母法：`chmod o+t dir`
    *   数字法：在三位权限数字前加1，如 `1755`
*   **典型示例**：系统的 `/tmp` 和 `/var/tmp` 目录。

![](img/9cc8ff33ade93b2f6f54530b35b04b87_74.png)

![](img/9cc8ff33ade93b2f6f54530b35b04b87_76.png)

**特殊权限显示**：
*   如果原有执行位（x）存在，特殊权限显示为小写字母（`s`, `s`, `t`）。
*   如果原有执行位不存在，特殊权限显示为大写字母（`S`, `S`, `T`）。
```bash
# 查看/tmp目录权限
ls -ld /tmp
drwxrwxrwt. 15 root root 4096 Nov  1 11:00 /tmp
# 最后的 `t` 表示设置了粘滞位
```
