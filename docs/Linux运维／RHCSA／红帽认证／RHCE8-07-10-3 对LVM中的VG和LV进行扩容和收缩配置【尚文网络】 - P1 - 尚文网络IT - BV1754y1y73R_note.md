## 为卷组（VG）扩容 ➕

![](img/6df66d51f7775867ce63203c1307817d_78.png)

![](img/6df66d51f7775867ce63203c1307817d_80.png)

![](img/6df66d51f7775867ce63203c1307817d_82.png)

如果根分区 `/`（对应 `rhel-root` 逻辑卷）空间即将耗尽，我们需要为它所在的卷组扩容。核心思路是向卷组中添加新的物理卷（PV）。

![](img/6df66d51f7775867ce63203c1307817d_88.png)

![](img/6df66d51f7775867ce63203c1307817d_90.png)

![](img/6df66d51f7775867ce63203c1307817d_92.png)

以下是操作步骤：

![](img/6df66d51f7775867ce63203c1307817d_94.png)

![](img/6df66d51f7775867ce63203c1307817d_96.png)

![](img/6df66d51f7775867ce63203c1307817d_98.png)

![](img/6df66d51f7775867ce63203c1307817d_100.png)

1.  **添加新硬盘**：首先，在虚拟机中添加一块新硬盘（例如10G）。
2.  **扫描新磁盘**：在不重启系统的情况下，使用命令扫描新添加的磁盘。
    ```bash
    echo ‘- - -‘ > /sys/class/scsi_host/host2/scan
    ```
    使用 `lsblk` 命令确认新磁盘（如 `/dev/sdd`）已被识别。
3.  **对新磁盘分区**：使用 `parted` 工具对新磁盘进行分区。
    ```bash
    parted /dev/sdd
    ```
    在 `parted` 交互界面中执行：
    ```
    mklabel gpt
    mkpart p1 0% 100%
    print
    quit
    ```
4.  **设置分区类型**：将新分区的标志设置为 `lvm`，以便LVM识别。
    ```bash
    parted /dev/sdd set 1 lvm on
    ```
    使用 `parted /dev/sdd print` 确认分区标志已改为 `lvm`。
5.  **创建物理卷（PV）**：将新分区初始化为物理卷。
    ```bash
    pvcreate /dev/sdd1
    ```
6.  **扩展卷组（VG）**：将新的物理卷添加到需要扩容的卷组（例如 `rhel`）中。
    ```bash
    vgextend rhel /dev/sdd1
    ```
    使用 `vgs` 或 `vgdisplay rhel` 命令确认卷组容量已增加。

卷组扩容后，我们可以在其中创建新的逻辑卷。

以下是操作步骤：

![](img/6df66d51f7775867ce63203c1307817d_123.png)

![](img/6df66d51f7775867ce63203c1307817d_125.png)

![](img/6df66d51f7775867ce63203c1307817d_127.png)

![](img/6df66d51f7775867ce63203c1307817d_129.png)

1.  **创建逻辑卷**：在 `rhel` 卷组中创建一个大小为800M（200个PE）的逻辑卷，命名为 `test_lv1`。
    ```bash
    lvcreate -l 200 -n test_lv1 rhel
    ```
2.  **格式化逻辑卷**：使用 `ext4` 文件系统格式化新创建的逻辑卷。
    ```bash
    mkfs.ext4 /dev/rhel/test_lv1
    ```
3.  **挂载逻辑卷**：创建挂载点并将逻辑卷挂载上去。
    ```bash
    mkdir -p /app/test_lv1
    mount /dev/rhel/test_lv1 /app/test_lv1
    ```
    使用 `df -Th` 命令确认挂载成功。
4.  **测试写入**：向新挂载的分区写入测试数据。
    ```bash
    dd if=/dev/zero of=/app/test_lv1/testfile bs=10M count=10
    ```
    再次使用 `df -Th` 观察空间使用变化。
5.  **配置开机自动挂载**：编辑 `/etc/fstab` 文件，添加挂载配置以实现开机自动挂载。
    ```bash
    echo ‘/dev/rhel/test_lv1 /app/test_lv1 ext4 defaults 0 0‘ >> /etc/fstab
    ```

## 重要概念与最佳实践 💡

![](img/6df66d51f7775867ce63203c1307817d_141.png)

![](img/6df66d51f7775867ce63203c1307817d_143.png)

![](img/6df66d51f7775867ce63203c1307817d_145.png)

上一节我们完成了扩容操作，本节中我们来总结一些核心概念和最佳实践。

![](img/6df66d51f7775867ce63203c1307817d_147.png)

**关于分区类型标识**：
*   使用 `fdisk` 创建分区时，应将分区类型ID改为 `8e`（Linux LVM）。
*   使用 `parted` 创建分区时，应使用 `set <分区号> lvm on` 命令将分区标志设为 `lvm`。
*   虽然不修改标识也能用 `pvcreate` 创建物理卷，但修改标识是**最佳实践**。这能让你（或其他管理员）通过 `fdisk -l` 或 `parted print` 命令一目了然地识别出哪些分区是专供LVM使用的，便于后续管理和维护。

![](img/6df66d51f7775867ce63203c1307817d_149.png)

![](img/6df66d51f7775867ce63203c1307817d_151.png)

![](img/6df66d51f7775867ce63203c1307817d_153.png)

![](img/6df66d51f7775867ce63203c1307817d_155.png)

**核心命令公式**：
*   创建物理卷：`pvcreate <分区设备名，如 /dev/sdd1>`
*   扩展卷组：`vgextend <卷组名> <物理卷设备名>`
*   创建逻辑卷：`lvcreate -L <大小>G 或 -l <PE数量> -n <逻辑卷名> <卷组名>`

## 总结 📝

![](img/6df66d51f7775867ce63203c1307817d_164.png)

本节课中我们一起学习了LVM的核心管理操作：
1.  使用 `vgs`、`lvs`、`vgdisplay` 等命令检查LVM状态。
2.  通过添加新硬盘、创建物理卷（`pvcreate`）并将其加入现有卷组（`vgextend`）来完成**卷组（VG）的扩容**。
3.  在扩容后的卷组中创建新逻辑卷（`lvcreate`），格式化并挂载使用。
4.  理解了正确设置分区类型标识（`8e` 或 `lvm`）对于LVM管理的重要性。
