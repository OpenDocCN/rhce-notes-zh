# Shell脚本自动化编程实战：P31：4.14 FD和命名管道实现shell并发控制

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_0.png)

在本节课中，我们将学习如何利用文件描述符和命名管道来控制Shell脚本的并发执行数量，避免无限制并发可能带来的问题。

上一节我们介绍了传统的并发执行方法，本节中我们来看看如何对其进行控制。

## 传统并发方法回顾

传统的并发方法是将循环体放入后台执行，并使用`wait`命令等待所有后台进程结束。这种方法简单，但无法控制同时运行的进程数量。

以下是传统并发脚本的示例：

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_2.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_4.png)

```bash
#!/bin/bash
for i in {1..254}
do
    {
        ping -c 1 -W 1 192.168.1.$i &> /dev/null
        if [ $? -eq 0 ]; then
            echo "192.168.1.$i is up"
        else
            echo "192.168.1.$i is down"
        fi
    } &
done
wait
```

这种方法在处理少量任务时没有问题，但当并发数量极大时（例如创建上千个用户），可能会超出系统限制，导致错误。

## 实现并发控制的核心概念

为了实现并发控制，我们需要理解两个核心概念：**文件描述符**和**命名管道**。

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_6.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_8.png)

### 文件描述符

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_10.png)

文件描述符是操作系统为每个进程打开的文件分配的一个数字标识。每个进程默认打开三个文件描述符：
*   **0**：标准输入
*   **1**：标准输出
*   **2**：标准错误

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_12.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_14.png)

以下是关于文件描述符需要掌握的三点：
1.  **如何打开一个文件并分配描述符**：使用`exec`命令。
    ```bash
    exec 6<> /tmp/file1  # 打开/tmp/file1，并分配文件描述符6用于读写
    ```
2.  **如何关闭文件描述符**：同样使用`exec`命令。
    ```bash
    exec 6>&-  # 关闭文件描述符6
    ```
3.  **文件描述符的生命周期**：只要文件描述符未被关闭，即使对应的原始文件被删除，其内容依然可以通过该描述符访问和恢复。

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_16.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_18.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_20.png)

### 命名管道

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_21.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_23.png)

管道用于连接一个进程的输出和另一个进程的输入。匿名管道（`|`）只能在同一个Shell会话中使用。命名管道则是一个有名称的特殊文件，允许不同进程（甚至不同终端）进行通信。

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_25.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_27.png)

命名管道有两个关键特性：
*   **先进先出**：数据按照写入的顺序被读取。
*   **数据非持久化**：数据一旦被读取，就会从管道中移除，无法再次读取。

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_29.png)

以下是使用命名管道的步骤：
1.  **创建命名管道**：
    ```bash
    mkfifo /tmp/myfifo
    ```
2.  **进程A写入数据**：
    ```bash
    ls -l /dev > /tmp/myfifo
    ```
3.  **进程B读取数据**：
    ```bash
    cat < /tmp/myfifo
    ```

## 利用FD和管道控制并发数的原理

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_37.png)

结合文件描述符和命名管道，我们可以实现一个“令牌桶”机制来控制并发数量：
1.  在脚本开始时，创建一个命名管道，并打开一个文件描述符与之关联。
2.  向管道中预先写入一定数量的“令牌”（例如，换行符），这个数量就代表了允许的最大并发数。
3.  每当要启动一个后台任务前，从管道中读取一个令牌。由于管道读取是阻塞的，如果管道为空，则脚本会在此等待，直到有令牌被放回。
4.  任务执行结束后，向管道中写入一个令牌，表示一个“并发位置”被释放。
5.  这样，任何时候在管道中流动的令牌数量都是固定的，从而实现了并发数的控制。

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_39.png)

## 编写并发控制脚本

以下是利用该原理编写的脚本示例，它控制每次最多有5个`ping`进程同时运行：

```bash
#!/bin/bash

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_41.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_42.png)

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_44.png)

# 创建命名管道和文件描述符
trap “rm -f /tmp/fifo.$$” EXIT  # 脚本退出时清理管道
mkfifo /tmp/fifo.$$
exec 6<> /tmp/fifo.$$

# 向管道中放入5个令牌（换行符）
for i in {1..5}
do
    echo
done >&6

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_46.png)

# 主循环
for i in {1..254}
do
    # 读取一个令牌，控制并发数
    read -u6

    {
        # 要执行的任务
        ip=192.168.1.$i
        ping -c 1 -W 1 $ip &> /dev/null
        if [ $? -eq 0 ]; then
            echo “$ip is up”
        else
            echo “$ip is down”
        fi

        # 任务执行完毕，放回一个令牌
        echo >&6
    } &  # 将整个任务块放入后台执行
done

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_48.png)

wait  # 等待所有后台任务结束

# 关闭文件描述符
exec 6>&-
```

![](img/a71299eb2fc69eb77d10bff6d0c9d2d6_50.png)

**脚本解析**：
1.  `mkfifo`创建命名管道，`exec 6<>`打开该管道并关联到文件描述符6。
2.  初始循环向描述符6写入5个换行符，作为5个并发令牌。
3.  主`for`循环中，`read -u6`会从描述符6读取一行。如果管道为空，则在此阻塞，直到有任务完成并放回令牌。
4.  花括号`{}`内是具体的任务逻辑。
5.  任务结束时，`echo >&6`向管道放回一个令牌。
6.  花括号后的`&`将整个任务块置于后台运行。
7.  最后的`wait`等待所有后台任务结束，然后关闭文件描述符。
