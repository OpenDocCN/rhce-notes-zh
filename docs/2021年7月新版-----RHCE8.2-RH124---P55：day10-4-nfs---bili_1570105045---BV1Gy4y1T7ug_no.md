# Linux系统管理：Day10-4：NFS自动挂载器Autofs 🚀

在本节课中，我们将要学习如何使用Autofs（自动挂载器）来动态管理NFS共享的挂载与卸载。Autofs服务可以根据用户访问需求自动挂载网络文件系统，并在闲置一段时间后自动卸载，这比传统的静态挂载（如写入`/etc/fstab`）更加灵活和高效。

## 1. Autofs简介与安装

Autofs是一个内核级别的自动挂载服务。它的核心功能是监听指定的“触发目录”。当用户尝试访问该目录下的子目录时，Autofs会立即自动挂载预先配置好的远程共享；当该目录一段时间未被使用后，又会自动卸载，以节省系统资源。

![](img/cbfa4d76ff222819e4f3f11d4932f690_11.png)

要使用Autofs，首先需要安装对应的软件包。

![](img/cbfa4d76ff222819e4f3f11d4932f690_13.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_14.png)

```bash
yum install -y autofs
```

![](img/cbfa4d76ff222819e4f3f11d4932f690_16.png)

安装完成后，可以启动并设置服务开机自启。

```bash
systemctl start autofs
systemctl enable autofs
```

## 2. 理解Autofs配置文件

![](img/cbfa4d76ff222819e4f3f11d4932f690_18.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_20.png)

Autofs的配置主要涉及两个文件：主配置文件(`/etc/auto.master`)和映射文件。

### 主配置文件 `/etc/auto.master`
这个文件定义了“挂载点父目录”和与之对应的“映射文件”之间的关系。其基本格式为：
```
[挂载点父目录] [映射文件路径]
```
例如，一行配置 `/home /etc/auto.home` 表示：监听`/home`目录，其下的自动挂载规则由`/etc/auto.home`文件定义。

### 映射文件
映射文件（如`/etc/auto.home`）定义了具体的挂载规则。其基本格式为：
```
[子目录名] [挂载选项] [服务器地址:共享路径]
```
例如，一行配置 `user1 -fstype=nfs,rw 192.168.1.100:/shared/user1` 表示：当访问`/home/user1`时，自动将服务器`192.168.1.100`上的`/shared/user1`目录以NFS协议、读写方式挂载上来。

## 3. 配置实例：为NFS用户配置自动挂载家目录

下面我们通过一个完整的实例，演示如何为NFS用户`tom`配置自动挂载的家目录。

**实验目标**：用户`tom`在客户端登录时，能自动挂载NFS服务器上属于他的家目录。

![](img/cbfa4d76ff222819e4f3f11d4932f690_34.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_35.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_37.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_39.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_41.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_43.png)

**服务器端 (IP: 192.168.18.128) 操作：**

![](img/cbfa4d76ff222819e4f3f11d4932f690_45.png)

1.  创建共享目录和相应用户：
    ```bash
    mkdir /shared_tom
    useradd -u 1012 tom
    chown tom:tom /shared_tom
    ```

![](img/cbfa4d76ff222819e4f3f11d4932f690_47.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_49.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_51.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_53.png)

2.  配置NFS共享，编辑`/etc/exports`文件：
    ```
    /shared_tom 192.168.18.129(rw,sync,no_root_squash)
    ```
    导出共享并重启NFS服务：
    ```bash
    exportfs -arv
    systemctl restart nfs-server
    ```
3.  确保防火墙放行了NFS相关服务（nfs， rpc-bind， mountd）。

![](img/cbfa4d76ff222819e4f3f11d4932f690_55.png)

![](img/cbfa4d76ff222819e4f3f11d4932f690_56.png)

**客户端 (IP: 192.168.18.129) 操作：**

1.  安装并启动Autofs服务（如前所述）。
2.  创建与服务器UID相同的本地用户`tom`，并为其指定一个不存在的本地家目录（因为将由Autofs提供）：
    ```bash
    useradd -u 1012 -d /data/tom tom
    ```
3.  编辑主配置文件`/etc/auto.master`，添加以下行：
    ```
    /data /etc/auto.data
    ```
    这表示将监听`/data`目录。
4.  创建并编辑映射文件`/etc/auto.data`：
    ```
    tom -fstype=nfs,rw 192.168.18.128:/shared_tom
    ```
    这表示当访问`/data/tom`时，自动挂载服务器的`/shared_tom`目录。
5.  重启Autofs服务使配置生效：
    ```bash
    systemctl restart autofs
    ```

**测试：**
在客户端，使用`su - tom`切换用户。此时，系统会自动挂载NFS共享到`/data/tom`。使用`df -Th`命令可以看到挂载信息。在`/data/tom`目录下创建文件，实际上会保存在NFS服务器上。用户退出或一段时间不访问后，挂载会自动卸载。

## 4. 核心概念与命令总结

![](img/cbfa4d76ff222819e4f3f11d4932f690_64.png)

以下是本课涉及的核心操作命令列表：

*   **服务管理**：
    *   `systemctl start|stop|restart|enable autofs`：管理Autofs服务。
*   **配置文件**：
    *   `/etc/auto.master`：主配置文件，定义监听目录与映射文件的对应关系。
    *   `/etc/auto.*`：映射文件，定义具体的自动挂载规则。
*   **NFS相关**：
    *   `/etc/exports`：NFS服务器共享配置文件。
    *   `exportfs -arv`：重新导出NFS共享配置。
    *   `showmount -e [服务器地址]`：查看NFS服务器上的共享列表。

本节课中我们一起学习了Autofs自动挂载器的原理与配置方法。我们了解到，通过配置`/etc/auto.master`和对应的映射文件，可以实现对NFS等网络文件系统的按需自动挂载和卸载。这种方法特别适用于管理大量用户家目录或偶尔访问的网络资源，能够提升系统管理的灵活性和资源利用效率。关键点在于理解“触发目录”与“映射规则”的配合，以及确保NFS服务器和客户端之间的用户标识（如UID）一致，以避免权限问题。
