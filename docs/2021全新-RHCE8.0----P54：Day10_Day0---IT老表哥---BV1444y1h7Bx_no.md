上一节我们介绍了 Ansible 的核心概念，本节我们来回顾其基础架构。

![](img/35b02d1de0e97a44d002f3af946b331a_10.png)

Ansible 的基础架构可以理解为一个指挥中心（控制节点）向多个受管主机（被控节点）发号施令的过程。为了实现自动化，需要建立信任关系（通常通过 SSH 密钥认证）和明确的目标清单（资产清单）。

以下是 Ansible 基础架构的核心组件：
*   **控制节点**：运行 Ansible 命令的主机。
*   **受管主机**：被 Ansible 管理的目标主机。
*   **资产清单**：定义受管主机的列表和分组。
*   **模块**：Ansible 执行任务的具体工具。
*   **剧本**：定义一系列任务和流程的 YAML 文件。

## 第二章：配置文件优先级详解

理解了基础架构后，配置文件的正确使用至关重要。Ansible 支持多个配置文件，其优先级和作用范围不同。

![](img/35b02d1de0e97a44d002f3af946b331a_16.png)

配置文件生效的优先级从高到低，作用范围从大到小依次为：
1.  **工作目录下的 `ansible.cfg`**：优先级最高，仅在该工作目录下生效。这是**推荐**的配置方式，影响范围最小。
    *   例如：`/home/student/ansible/ansible.cfg`
2.  **用户家目录下的 `.ansible.cfg`**：优先级次之，对该用户生效。
    *   例如：`/home/student/.ansible.cfg`
3.  **系统默认的 `/etc/ansible/ansible.cfg`**：优先级最低，全局生效。

![](img/35b02d1de0e97a44d002f3af946b331a_18.png)

**核心原则**：在具体项目或考试中，应使用优先级最高且影响范围最小的配置文件，即在工作目录中创建 `ansible.cfg`。

## 第三章：临时命令与核心模块回顾

![](img/35b02d1de0e97a44d002f3af946b331a_28.png)

![](img/35b02d1de0e97a44d002f3af946b331a_30.png)

![](img/35b02d1de0e97a44d002f3af946b331a_32.png)

配置好环境后，就可以使用 Ansible 执行任务了。我们首先回顾使用临时命令的方式。

Ansible 临时命令的基本格式如下：
```bash
ansible <主机或主机组> -m <模块名> -a "<模块参数>"
```

### 文件模块

以下是常用的文件操作模块及其示例：

![](img/35b02d1de0e97a44d002f3af946b331a_34.png)

*   **`copy` 模块**：复制文件或生成内容。
    *   从控制节点复制文件：`ansible all -m copy -a "src=/local/file dest=/remote/path"`
    *   在受管主机上直接生成内容：`ansible all -m copy -a "content='Hello World' dest=/tmp/test.txt"`

*   **`file` 模块**：管理文件属性、创建目录或链接。
    *   创建目录：`ansible all -m file -a "path=/tmp/mydir state=directory"`
    *   创建软链接（注意源与目标顺序）：`ansible all -m file -a "src=/etc/fstab dest=/var/www/html/fstab state=link"`

*   **`fetch` 模块**：从受管主机拉取文件到控制节点。
    *   `ansible all -m fetch -a "src=/var/log/messages dest=/tmp/"`

*   **`lineinfile` 模块**：确保某行文本存在于文件中。
    *   `ansible all -m lineinfile -a "path=/etc/ssh/sshd_config regexp='^Port' line='Port 2222'"`

### 软件包模块

以下是管理软件包和仓库的模块：

*   **`yum_repository` 模块**：配置 YUM 仓库。
    ```bash
    ansible all -m yum_repository -a "name=myrepo description='My Repo' baseurl=http://mirror.example.com/repo gpgcheck=no"
    ```

![](img/35b02d1de0e97a44d002f3af946b331a_36.png)

![](img/35b02d1de0e97a44d002f3af946b331a_37.png)

![](img/35b02d1de0e97a44d002f3af946b331a_39.png)

*   **`yum` 模块**：安装、更新或删除软件包。
    *   安装软件包：`ansible all -m yum -a "name=httpd state=present"`
    *   删除软件包：`ansible all -m yum -a "name=httpd state=absent"`

## 第四章：常见问题与解答

![](img/35b02d1de0e97a44d002f3af946b331a_43.png)

![](img/35b02d1de0e97a44d002f3af946b331a_45.png)

在实际操作中，可能会遇到以下问题：

**问题1：为什么需要切换到工作目录下的 `ansible.cfg`？**
为了确保配置的精准性和项目的隔离性。使用工作目录下的配置文件，可以避免不同项目间的配置冲突，也符合考试和生产环境的最佳实践。

**问题2：如何将两个 YAML 任务写在一个临时命令中？**
可以使用换行符连接两个任务模块的参数。例如，连续执行两个 `yum_repository` 任务：
```bash
ansible all -m yum_repository -a "name=repo1 baseurl=..." -e "name=repo2 baseurl=..."
```
但在剧本或复杂任务中，建议分开写成独立的任务条目，清晰且不易出错。

**问题3：创建软链接时源路径和目标路径写反了怎么办？**
`file` 模块的 `src` 参数是已存在的源文件或目录，`dest`（或 `path`）参数是要创建的链接路径。如果创建后链接指向错误，检查并调换这两个参数的值即可。

**问题4：`ansible-doc` 是什么？**
`ansible-doc` 是查看 Ansible 模块详细使用说明的命令行工具。当不确定模块用法时，使用 `ansible-doc <模块名>` 可以查看参数、示例和说明，是学习和排查问题的好帮手。
