## 磁盘分区工具与类型

![](img/1daed523a3d934fd871297ff9ef69923_5.png)

上一节我们介绍了课程概述，本节中我们来看看磁盘分区的基本工具和类型。

![](img/1daed523a3d934fd871297ff9ef69923_7.png)

![](img/1daed523a3d934fd871297ff9ef69923_9.png)

在 Linux 系统中，常用的分区工具主要有三个：`fdisk`、`gdisk` 和 `parted`。

![](img/1daed523a3d934fd871297ff9ef69923_11.png)

![](img/1daed523a3d934fd871297ff9ef69923_13.png)

### MBR 分区类型
`fdisk` 工具主要针对 **MBR** 类型的分区表。MBR 全称为 **Master Boot Record**，即主引导记录。这是一种传统的分区表类型。

MBR 分区类型的特点如下：
*   **单个分区大小不能超过 2TB**。这是由于其寻址位决定的。
*   **存在主分区、扩展分区和逻辑分区的概念**。
    *   主分区最多只能有 **4** 个。
    *   如果需要多于4个分区，则需要创建一个**扩展分区**，扩展分区会占用一个主分区的位置。
    *   在扩展分区内部可以建立**逻辑分区**。
*   在 Linux 的命名规则中，逻辑分区的序号永远从 **5** 开始（因为主分区占用了1-4）。

在 Linux 系统中，MBR 分区类型通常也被称为 **msdos** 类型。

![](img/1daed523a3d934fd871297ff9ef69923_15.png)

### GPT 分区类型
`gdisk` 工具采用 **GUID** 分区表，即 **GPT**。

GPT 分区类型的特点如下：
*   它是一种更先进的分区方案，支持更大的磁盘容量。
*   **不再有主分区、扩展分区和逻辑分区的概念**，所有分区都是平等的。
*   最多支持 **128** 个分区（实际上是 2^7 个）。
*   可以轻松支持 **2TB 以上** 的单个分区，理论支持上限远高于当前硬件水平。

![](img/1daed523a3d934fd871297ff9ef69923_17.png)

![](img/1daed523a3d934fd871297ff9ef69923_19.png)

### MBR 与 GPT 的选择
以下是两者的一些核心区别：
*   **兼容性**：MBR 在传统 BIOS 系统上兼容性更好。GPT 则需要在支持 UEFI 的现代设备上使用。
*   **磁盘容量**：MBR 限制单个磁盘容量和分区容量（2TB）。GPT 无此限制。
*   **分区数量**：MBR 限制为4个主分区（或3主+1扩展）。GPT 支持更多分区。
*   **稳定性**：在传统 BIOS 环境下，MBR 的启动稳定性可能更优。

![](img/1daed523a3d934fd871297ff9ef69923_21.png)

![](img/1daed523a3d934fd871297ff9ef69923_22.png)

对于现代操作系统（如 Windows 8/10, 较新的 Linux 发行版）和新硬件，通常推荐使用 GPT。

## 基本分区操作方法

![](img/1daed523a3d934fd871297ff9ef69923_53.png)

![](img/1daed523a3d934fd871297ff9ef69923_54.png)

![](img/1daed523a3d934fd871297ff9ef69923_56.png)

上一节我们准备好了实验磁盘，本节中我们将通过 `fdisk` 工具，完整演示从创建分区到挂载使用的全过程。

![](img/1daed523a3d934fd871297ff9ef69923_58.png)

整个过程包括：**建立分区 -> 让内核识别分区 -> 格式化（创建文件系统）-> 挂载 -> 配置永久挂载**。

### 1. 查看当前磁盘状态
首先，使用 `lsblk` 命令确认新磁盘已被系统识别，且尚未分区。
```bash
lsblk
```

![](img/1daed523a3d934fd871297ff9ef69923_60.png)

### 2. 使用 fdisk 创建分区
我们以在新磁盘 `/dev/nvme0n2` 上创建一个 **2GB** 的主分区为例。

进入 `fdisk` 交互界面：
```bash
fdisk /dev/nvme0n2
```

在 `fdisk` 界面中，按以下步骤操作：
*   输入 `n` 创建新分区。
*   选择分区类型 `p` (主分区)。
*   分区号使用默认值 `1`。
*   起始扇区使用默认值。
*   在设置结束扇区或大小时，输入 `+2G` 来指定分区大小为 2GB。
*   输入 `p` 预览创建的分区信息。
*   确认无误后，输入 `w` 将分区表写入磁盘并退出。

![](img/1daed523a3d934fd871297ff9ef69923_62.png)

![](img/1daed523a3d934fd871297ff9ef69923_64.png)

**重要提示**：如果是在已有分区的单块磁盘上操作，需注意主分区数量不能超过4个。超过后需要创建扩展分区。

![](img/1daed523a3d934fd871297ff9ef69923_66.png)

![](img/1daed523a3d934fd871297ff9ef69923_67.png)

### 3. 让内核重新读取分区表
创建分区后，需要让系统内核重新读取分区表信息。
```bash
partprobe
```
之后，可以再次使用 `lsblk` 或 `lsblk -f` 查看新的分区（如 `/dev/nvme0n2p1`）是否出现。

![](img/1daed523a3d934fd871297ff9ef69923_69.png)

![](img/1daed523a3d934fd871297ff9ef69923_71.png)

### 4. 格式化分区（创建文件系统）
现在，我们需要为分区创建文件系统（即格式化）。在 RHEL 8 中，默认的文件系统是 **XFS**。

![](img/1daed523a3d934fd871297ff9ef69923_73.png)

使用 `mkfs` 命令进行格式化：
```bash
# 方法一：使用 mkfs.xfs 命令
mkfs.xfs /dev/nvme0n2p1

![](img/1daed523a3d934fd871297ff9ef69923_75.png)

# 方法二：使用 mkfs -t 选项
mkfs -t xfs /dev/nvme0n2p1
```

![](img/1daed523a3d934fd871297ff9ef69923_77.png)

### 5. 临时挂载分区
格式化后，需要将分区挂载到目录树中的某个位置（挂载点）才能使用。

![](img/1daed523a3d934fd871297ff9ef69923_79.png)

首先，创建一个挂载点目录：
```bash
mkdir /mnt/data1
```
然后，进行临时挂载：
```bash
mount /dev/nvme0n2p1 /mnt/data1
```
使用 `df -Th` 命令可以查看挂载是否成功。**临时挂载在系统重启后会失效**。

### 6. 配置永久挂载（编辑 /etc/fstab）
为了实现开机自动挂载，需要编辑 `/etc/fstab` 文件。

![](img/1daed523a3d934fd871297ff9ef69923_81.png)

![](img/1daed523a3d934fd871297ff9ef69923_83.png)

**建议使用 UUID 进行挂载**，因为设备名（如 `/dev/nvme0n2p1`）可能会变动，而 UUID 是唯一的。

![](img/1daed523a3d934fd871297ff9ef69923_85.png)

![](img/1daed523a3d934fd871297ff9ef69923_87.png)

![](img/1daed523a3d934fd871297ff9ef69923_89.png)

获取分区的 UUID：
```bash
blkid /dev/nvme0n2p1
```

![](img/1daed523a3d934fd871297ff9ef69923_91.png)

编辑 `/etc/fstab` 文件：
```bash
vim /etc/fstab
```
在文件末尾添加一行，格式如下（请替换为你的实际 UUID 和挂载点）：
```
UUID=你的分区UUID    /mnt/data1    xfs    defaults    0 0
```
*   第一列：设备标识（这里用 UUID）。
*   第二列：挂载点目录。
*   第三列：文件系统类型。
*   第四列：挂载选项（`defaults` 是常用选项）。
*   第五列：是否被 `dump` 备份工具使用（`0` 表示否）。
*   第六列：开机时是否检查文件系统及检查顺序（`0` 表示不检查）。

![](img/1daed523a3d934fd871297ff9ef69923_93.png)

**警告**：`/etc/fstab` 文件编辑错误可能导致系统无法正常启动，请务必仔细核对。

保存退出后，测试配置是否正确：
```bash
# 重新挂载所有在 /etc/fstab 中定义的文件系统
mount -a
# 再次查看挂载情况
df -Th
```
如果命令执行没有报错，且 `df -Th` 能显示挂载，则配置成功。

### 补充：Linux 磁盘设备命名规则
Linux 中的磁盘设备通常位于 `/dev/` 目录下，命名规则如下：
*   `/dev/sdX`：SCSI、SATA、USB 接口的磁盘。`X` 为字母，如 `a`, `b`, `c`，代表磁盘序号。
*   `/dev/vdX`：虚拟化环境（如 KVM）中的虚拟磁盘。
*   `/dev/nvmeXnY`：NVMe 协议固态硬盘。`X` 是控制器编号，`Y` 是命名空间编号（通常为1）。分区则在后面加 `p1`, `p2`，如 `/dev/nvme0n2p1`。
*   `/dev/sr0` 或 `/dev/cdrom`：光驱设备。
*   `/dev/hdX`：古老的 IDE 接口硬盘，现已少见。

其中，`sdX`, `vdX`, `nvmeXnY` 中的 `X` 代表**磁盘号**，后面的 `Y` 或 `pN` 代表该磁盘上的**分区号**。
