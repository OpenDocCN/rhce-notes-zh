## 回顾与过渡：`ls -d`选项的准确含义

上一节我们介绍了`ls`命令及其常用选项。有同学问到`ls -d`选项，这里需要明确其准确含义。

`ls -d`选项的作用是**查看指定参数（目录或文件）本身的属性信息，而不查看其下的内容**。即使参数是一个目录，它也不会显示该目录下的子文件和子目录。

例如，在当前目录（包含目录和文件）下执行：
```bash
ls -d a/b/c/*
```
此命令会列出`a/b/c/`路径下所有条目（无论它们是目录还是文件）自身的属性，而不会展开显示目录内部的内容。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_10.png)

如果参数本身是一个文件，由于文件没有下级内容，加不加`-d`选项效果差异不大。`-d`选项本身**不具备过滤功能**，它不会区分参数是目录还是文件。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_12.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_14.png)

若需要只列出目录，可以结合`-l`选项和过滤命令。`ls -l`列出的详细信息中，目录行首字符是`d`，文件行首字符是`-`。因此，可以通过管道进行过滤：
```bash
ls -l | grep ‘^d’
```
此命令会筛选出所有以`d`开头的行，即只显示目录。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_16.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_18.png)

另一个相关选项是`-R`（递归），用于列出目录及其所有子目录下的内容。例如：
```bash
ls -R /boot
```
此命令会递归显示`/boot`目录下的所有层次结构。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_20.png)

## 查看隐藏文件

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_22.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_24.png)

在Linux系统中，隐藏文件指的是名称以点号`.`开头的文档（包括目录和文件）。

默认情况下，使用`ls`命令不会显示隐藏文件。若要查看它们，需要使用`-A`选项。

例如，在root用户的主目录下：
```bash
ls /root
# 默认只显示非隐藏文件
ls -A /root
# 显示所有文件，包括隐藏文件
```
以点号开头的文件默认被隐藏，使用`ls -A`可以将其列出。

## 核心工具：VI/VIM文本编辑器 🛠️

下面，我们进入本节课的核心内容——学习使用VI/VIM文本编辑器。这是Unix/Linux系统中默认且功能强大的命令行文本编辑器。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_26.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_28.png)

VI是“Visual Interface”的缩写，是Unix/Linux系统的默认文本编辑器。VIM（Vi IMproved）是VI的增强版，提供了更多便利功能，如语法高亮，更适合编写脚本。在最小化安装的系统中，通常只预装了VI。VIM需要单独安装软件包`vim-enhanced`。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_30.png)

使用编辑器的核心目标是：**创建新文件**和**修改已有文件**。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_31.png)

### 编辑器的三种工作模式

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_33.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_34.png)

VI/VIM编辑器有三种主要的工作模式，理解它们是流畅使用编辑器的关键：

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_36.png)

1.  **命令模式**：启动VI/VIM后默认进入的模式。在此模式下，可以执行复制、粘贴、删除、查找等操作，但不能直接输入文本。
2.  **输入模式**：在此模式下，可以向文件中输入或编辑文本内容。
3.  **末行模式**：在此模式下，可以执行保存文件、退出编辑器等操作。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_38.png)

你可以将三种模式想象为编辑器的不同“形态”，就像汽车人变形一样，每种形态擅长不同的任务。

### 基本使用流程

以下是创建一个新文件或修改一个已有文件的标准流程：

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_40.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_42.png)

1.  **打开文件**：使用`vi`或`vim`命令后接文件名。
    ```bash
    vi myfile.txt
    ```
    如果文件不存在，编辑器会新建一个临时文件；如果文件存在，则打开它进行编辑。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_43.png)

2.  **进入输入模式**：打开文件后处于**命令模式**。按下字母键 `i`，编辑器底部出现`-- INSERT --`提示，表示已进入**输入模式**。此时可以输入文本内容。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_45.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_47.png)

3.  **返回命令模式**：文本输入或编辑完成后，按下 `Esc` 键，可以从**任何模式**返回到**命令模式**。

4.  **进入末行模式并保存退出**：在**命令模式**下，输入冒号 `:`，进入**末行模式**。此时光标会移动到屏幕底部命令行。
    以下是末行模式常用的退出命令：
    *   **保存并退出**：输入 `wq`，然后按回车。
    *   **不保存并强制退出**：输入 `q!`，然后按回车。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_49.png)

5.  **验证内容**：退出编辑器后，可以使用`cat`命令查看文件内容，确认修改已生效。
    ```bash
    cat myfile.txt
    ```

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_51.png)

**流程总结公式**：
`vi 文件名` -> **命令模式** -> 按 `i` -> **输入模式** (编辑文本) -> 按 `Esc` -> **命令模式** -> 按 `:` -> **末行模式** -> 输入 `wq` 或 `q!` -> 按 `Enter` 退出。

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_53.png)

![](img/5ff6f04eb9ef7d5e25cb66eb745492f7_55.png)
