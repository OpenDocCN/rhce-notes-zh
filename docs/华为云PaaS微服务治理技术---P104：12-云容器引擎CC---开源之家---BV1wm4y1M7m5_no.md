# 华为云PaaS微服务治理技术 - P104：12-云容器引擎CCE-容器镜像服务 🐳

在本节课中，我们将要学习华为云容器镜像服务（SWR）的核心概念与使用方法。容器镜像服务是管理Docker镜像的仓库，它允许您存储、管理和部署容器镜像。我们将重点学习如何将本地构建的镜像上传到云端仓库，以便在创建容器工作负载时使用。

## 概述

容器镜像服务用于管理Docker镜像。在之前CCE的快速体验中，我们直接从Docker官方仓库拉取了Nginx镜像。但在实际开发中，我们需要部署自己开发的微服务，这就需要构建私有镜像并上传到云平台进行管理。本节将介绍镜像管理的完整流程。

## 镜像使用流程

镜像管理的核心流程包括：创建组织分类、获取镜像、上传镜像，以及在部署工作负载时引用这些镜像。

以下是镜像管理的完整步骤：
1.  **创建组织**：对镜像进行分类管理。
2.  **获取镜像**：可以通过三种方式获取镜像。
3.  **上传镜像**：将镜像推送到云平台仓库。
4.  **部署与更新**：在CCE中创建工作负载时选择镜像，并可后续更新镜像版本。

上一节我们介绍了镜像管理的整体流程，本节中我们来看看每个步骤的具体操作。

## 创建组织 🗂️

组织用于对镜像进行分类管理。当镜像数量众多时，通过组织可以更清晰地进行管理。

操作步骤如下：
1.  登录华为云控制台。
2.  进入“云容器引擎CCE”服务。
3.  在左侧菜单栏点击“镜像仓库”。
4.  点击左侧菜单的“组织管理”。
5.  点击“创建组织”，输入组织名称（例如：`xcedu-cloud-01`），然后点击“确定”。

组织创建成功后，即可在该组织下上传和管理镜像。

以下是三种主要的镜像获取方式：
*   **上传自有镜像**：将本地服务器上已构建好的Docker镜像上传到镜像仓库。
*   **通过源代码构建**：编写Dockerfile文件，利用云平台的自动构建功能直接从源代码生成镜像。
*   **从Docker官方仓库拉取**：直接从Docker Hub等公共仓库拉取现有镜像。

在实际应用中，上传自有镜像和从官方仓库拉取镜像使用较多。接下来，我们将重点演示如何上传自有镜像到云平台。

## 上传自有镜像到云平台 🚀

![](img/9e38d128e963174ffe0267ac4fcb6585_9.png)

我们将通过Docker客户端，把本地服务器上的镜像推送到华为云镜像仓库。此方法相比网页上传更稳定、高效。

![](img/9e38d128e963174ffe0267ac4fcb6585_11.png)

整个流程需要您的本地开发环境或虚拟机中安装并运行Docker。假设本地服务器的IP为`192.168.101.64`。

### 步骤一：登录本地服务器并检查Docker

![](img/9e38d128e963174ffe0267ac4fcb6585_13.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_14.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_15.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_16.png)

首先，以root用户登录已安装Docker的本地服务器（虚拟机），并确保Docker版本高于1.11.2。

```bash
# 登录服务器
ssh root@192.168.101.64

![](img/9e38d128e963174ffe0267ac4fcb6585_18.png)

# 检查Docker版本
docker version
```

### 步骤二：获取访问凭证并登录云平台镜像仓库

![](img/9e38d128e963174ffe0267ac4fcb6585_20.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_21.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_22.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_23.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_24.png)

为了让本地Docker客户端有权限推送镜像到华为云仓库，需要先执行登录认证。

在华为云SWR控制台的“我的镜像”页面，点击“客户端上传”，页面会生成一条`docker login`命令。复制该命令到本地服务器执行。

![](img/9e38d128e963174ffe0267ac4fcb6585_26.png)

```bash
# 示例命令（具体命令以控制台生成为准）
docker login -u cn-north-1@ABCDEFG123456 -p xxxxxxxxxxxxx swr.cn-north-1.myhuaweicloud.com
```
执行成功后，会显示 `Login Succeeded`。

### 步骤三：为本地镜像打标（Tag）

![](img/9e38d128e963174ffe0267ac4fcb6585_28.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_29.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_30.png)

推送前，需要按照`[云平台仓库地址]/[组织名称]/[镜像名称]:[版本]`的格式，为本地镜像重新打标。

![](img/9e38d128e963174ffe0267ac4fcb6585_32.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_34.png)

假设我们要推送的本地镜像名为 `swr.demo/nginx:latest`，目标组织为 `xcedu-cloud-01`。

![](img/9e38d128e963174ffe0267ac4fcb6585_36.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_37.png)

```bash
# 打标命令格式
docker tag [本地镜像名]:[版本] [仓库地址]/[组织名]/[镜像名]:[版本]

![](img/9e38d128e963174ffe0267ac4fcb6585_38.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_39.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_40.png)

# 具体示例
docker tag swr.demo/nginx:latest swr.cn-north-1.myhuaweicloud.com/xcedu-cloud-01/nginx:latest
```
执行后，使用 `docker images` 命令可以看到一个符合新命名规则的新镜像标签。

![](img/9e38d128e963174ffe0267ac4fcb6585_42.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_44.png)

### 步骤四：推送镜像到云仓库

![](img/9e38d128e963174ffe0267ac4fcb6585_46.png)

使用 `docker push` 命令，将打好标的镜像推送到云端。

```bash
# 推送命令
docker push swr.cn-north-1.myhuaweicloud.com/xcedu-cloud-01/nginx:latest
```
命令行会显示上传进度。上传完成后，刷新华为云SWR控制台“我的镜像”页面，即可在对应的组织下看到刚刚推送的镜像。

![](img/9e38d128e963174ffe0267ac4fcb6585_48.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_49.png)

![](img/9e38d128e963174ffe0267ac4fcb6585_50.png)
