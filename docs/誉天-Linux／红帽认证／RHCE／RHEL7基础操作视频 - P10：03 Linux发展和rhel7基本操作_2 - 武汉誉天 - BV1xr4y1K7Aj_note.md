# Linux基础入门：01：Linux发展历史与核心原则 🐧

在本节课中，我们将要学习Linux操作系统的诞生背景、发展历程以及其核心设计哲学。了解这些基础知识，有助于我们理解Linux为何如此流行以及它的独特之处。

## Linux的诞生与发展

上一节我们提到了Unix系统及其版权问题。本节中我们来看看Linux是如何出现的。

1991年，一位名叫林纳斯·托瓦兹的学生接触到了Unix操作系统。由于Unix的版权限制，他无法自由地使用和修改。于是，他参考了Unix的设计思想，自己编写了一个操作系统内核。

林纳斯将这个内核的源代码公开在互联网上。世界各地的编程爱好者看到后非常激动，纷纷参与到这个内核的修改和完善工作中。

与此同时，GNU项目也看中了这个内核。GNU项目旨在创建一个完全自由的操作系统，但当时缺少一个可用的内核。于是，GNU组织决定将林纳斯的内核纳入其体系，并在其上开发操作系统的其他组件。

1991年，经过众人协作完善的Linux内核1.0版本正式发布。GNU组织将之前为Unix开发的一些组件移植到了Linux内核上，从而形成了一个完整的、开源的类Unix操作系统。因此，我们现在常说的“Linux操作系统”，其完整表述应为“GNU/Linux”。

Linux内核版本从最初的1.0，已经发展到今天的4.x甚至更高版本。其内核代码可以在官方网站 `kernel.org` 获取。

## Linux的标志：企鹅 🐧

Linux的官方标志是一只企鹅。关于这个标志的由来，主要有两种说法：
*   一种说法是林纳斯在动物园被企鹅咬过，因此选用企鹅作为标志。
*   另一种更被广泛接受的说法是：企鹅生活在南极，而南极不属于任何国家，这象征着Linux系统是自由、属于全世界的。

Linux内核是开源的，任何公司或组织都可以基于它进行二次开发，加入自己的软件和管理工具，形成自己的“发行版”。以下是几个重要的发行版：

**红帽企业版 Linux**
*   **公式/代码**: `Red Hat Enterprise Linux (RHEL)`
*   这是红帽公司面向企业服务器市场推出的商业发行版，特点是**极其稳定**，并提供官方的技术支持服务。企业需要付费订阅才能获得更新和支持。

![](img/e3b902ce5962fe408c74fe973b22575a_11.png)

**Fedora**
*   **公式/代码**: `Fedora Project`
*   这是由社区维护、红帽赞助的桌面版发行版。它集成了大量**最新**的软件和技术，相当于RHEL的“试验田”。新技术会先在Fedora上测试，稳定后再引入到RHEL中。

**CentOS**
*   **公式/代码**: `Community ENTerprise Operating System`
*   这是一个由社区维护的、**免费**的企业级发行版。它完全重建自RHEL的源代码，与RHEL在功能上**完全兼容**。许多公司使用CentOS来搭建服务器，并由自己的运维团队维护。红帽公司后来收购了CentOS项目。

![](img/e3b902ce5962fe408c74fe973b22575a_13.png)

**其他流行发行版**
*   **SUSE / openSUSE**: 另一个成功的商业发行版，在云计算和服务器领域应用广泛。
*   **Ubuntu**: 基于Debian，以易用性和强大的桌面环境著称，非常适合个人用户和初学者。
*   **Kali Linux**: 专注于网络安全测试，集成了大量安全审计工具。

**一切皆文件**
*   **含义**: 在Linux系统中，几乎所有的资源都被抽象为“文件”，包括硬件设备、系统配置、进程信息等。对硬件的操作，在用户看来就是读写某个特定的文件。
*   **示例**: 磁盘是文件（如 `/dev/sda`），配置信息保存在文本文件中。

**小型、单一目的的应用程序**
*   **含义**: Linux鼓励开发功能单一、小巧精悍的程序。每个程序只做好一件事，但要做到极致。
*   **示例**: `ls` 命令只负责列出目录内容，`grep` 命令只负责文本搜索。要完成复杂任务，可以通过管道等方式将多个小程序组合起来。
*   **代码示例**: `ls -la | grep “.txt”` 这个命令组合了 `ls` 和 `grep` 两个小程序，实现了“列出所有文件并过滤出包含.txt的文件”这个复杂任务。

**配置数据存储在文本文件中**
*   **含义**: 系统的所有配置都保存在纯文本文件中。这意味着用户可以直接通过编辑文本文件来修改系统设置，这种方式非常灵活且易于备份和版本控制。
*   **对比**: 这与Windows主要依靠图形界面和注册表进行配置的方式有很大不同。
