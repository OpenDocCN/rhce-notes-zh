# Linux进程管理：14：进程管理基础 🖥️

在本节课中，我们将学习Linux系统中进程管理的基础知识。进程是系统运行的核心，理解如何查看和管理进程是系统运维的关键技能。我们将从进程的基本概念开始，逐步介绍查看、控制和管理进程的常用命令。

## 进程的基本概念

上一节我们介绍了课程概述，本节中我们来看看什么是进程。进程是操作系统为了提供服务而分配的内存地址空间、安全属性和程序代码等资源的集合。进程执行需要环境变量的支持，并涉及文件、端口调度等协同工作。

在Linux操作系统中，所有的进程都通过一个“祖宗进程”派生而来。这个初始进程是 **`systemd`**，其进程ID（PID）为1。其他所有进程都是通过 **`fork`** 操作从 `systemd` 或其子进程复制（分支）出来的，形成树状结构。

![](img/a6e83539dc131db0b76d166904680b0d_1.png)

系统通过唯一的 **PID**（Process ID）来识别不同的进程。每个进程都有自己的PID。

![](img/a6e83539dc131db0b76d166904680b0d_3.png)

## 进程的状态

![](img/a6e83539dc131db0b76d166904680b0d_5.png)

进程在其生命周期中会处于不同的状态。以下是主要的进程状态及其表示符号：

![](img/a6e83539dc131db0b76d166904680b0d_7.png)

![](img/a6e83539dc131db0b76d166904680b0d_9.png)

*   **R (Running或Runnable)**: 进程正在运行或准备就绪，等待CPU调度。
*   **S (Interruptible Sleep)**: 进程处于可中断的等待状态，通常是在等待某个事件（如I/O操作）完成。
*   **D (Uninterruptible Sleep)**: 进程处于不可中断的等待状态，通常是在等待硬件I/O。
*   **T (Stopped)**: 进程已被停止（例如，通过发送 `SIGSTOP` 信号）。
*   **Z (Zombie)**: 僵尸进程。进程已终止，但其部分信息仍留在进程表中等待父进程读取。

![](img/a6e83539dc131db0b76d166904680b0d_11.png)

## 查看进程的命令

了解概念后，我们来看看如何查看系统中的进程。以下是两个最常用的命令。

### 1. top 命令

![](img/a6e83539dc131db0b76d166904680b0d_13.png)

`top` 命令提供了一个动态、实时的系统进程状态视图。它可以显示系统的总体负载、CPU和内存使用情况，以及详细的进程列表。

![](img/a6e83539dc131db0b76d166904680b0d_15.png)

![](img/a6e83539dc131db0b76d166904680b0d_17.png)

在 `top` 界面中，你可以使用以下快捷键进行交互操作：

![](img/a6e83539dc131db0b76d166904680b0d_19.png)

![](img/a6e83539dc131db0b76d166904680b0d_21.png)

*   **`k`**：终止一个进程。输入后，会提示你输入要终止的进程PID。
*   **`r`**：调整一个进程的优先级（Nice值）。
*   **`d`** 或 **`s`**：改变信息刷新延迟的时间（默认3秒）。
*   **`P`**：按CPU使用率排序。
*   **`M`**：按内存使用率排序。
*   **`l`**、**`t`**、**`m`**：切换显示系统负载、CPU状态和内存信息的摘要行。
*   **`q`**：退出 `top` 命令。

### 2. ps 命令

与 `top` 不同，`ps` 命令用于查看当前时刻的进程快照（静态信息）。一个常用的组合是 `ps aux`，它可以列出系统上所有用户的详细进程信息。

![](img/a6e83539dc131db0b76d166904680b0d_23.png)

![](img/a6e83539dc131db0b76d166904680b0d_25.png)

![](img/a6e83539dc131db0b76d166904680b0d_27.png)

`ps aux` 输出的各列含义如下：
*   **USER**: 启动进程的用户。
*   **PID**: 进程ID。
*   **%CPU**: 进程的CPU使用率。
*   **%MEM**: 进程的内存使用率。
*   **VSZ**: 进程占用的虚拟内存大小。
*   **RSS**: 进程占用的物理内存大小。
*   **TTY**: 进程关联的终端。
*   **STAT**: 进程状态（即上文提到的R、S、D等）。
*   **START**: 进程启动时间。
*   **COMMAND**: 启动进程的命令。

![](img/a6e83539dc131db0b76d166904680b0d_29.png)

![](img/a6e83539dc131db0b76d166904680b0d_31.png)

![](img/a6e83539dc131db0b76d166904680b0d_33.png)

如果你不记得 `ps` 命令的参数，可以使用 `man ps` 查看手册，然后输入 `/EXAMPLE` 搜索，可以找到常用的命令示例。

## 进程的前后台控制

在Shell中运行命令时，我们可以控制进程是在前台运行还是后台运行。这对于管理长时间运行的任务非常有用。

![](img/a6e83539dc131db0b76d166904680b0d_45.png)

![](img/a6e83539dc131db0b76d166904680b0d_47.png)

以下是相关的命令和操作：

![](img/a6e83539dc131db0b76d166904680b0d_49.png)

*   **`&`**：在命令末尾加上 `&` 符号，可以让该命令在**后台**立即运行。
    ```bash
    sleep 100 &
    ```
*   **`jobs`**：列出当前Shell会话中所有后台任务（作业），并显示它们的作业编号（如 `[1]`）。
*   **`fg %n`**：将作业编号为 `n` 的后台任务调到**前台**运行。
*   **`bg %n`**：将已停止的作业编号为 `n` 的任务在**后台**继续运行。
*   **`Ctrl+Z`**：快捷键，**暂停**当前在前台运行的进程，并将其放入后台，状态变为 `Stopped (T)`。
*   **`Ctrl+C`**：快捷键，**终止**当前在前台运行的进程。

![](img/a6e83539dc131db0b76d166904680b0d_51.png)

![](img/a6e83539dc131db0b76d166904680b0d_53.png)

## 管理进程：kill 命令

![](img/a6e83539dc131db0b76d166904680b0d_55.png)

![](img/a6e83539dc131db0b76d166904680b0d_57.png)

![](img/a6e83539dc131db0b76d166904680b0d_59.png)

当需要主动结束或控制进程时，我们使用 `kill` 命令。`kill` 命令通过向进程发送特定的**信号**来工作。

![](img/a6e83539dc131db0b76d166904680b0d_61.png)

![](img/a6e83539dc131db0b76d166904680b0d_63.png)

![](img/a6e83539dc131db0b76d166904680b0d_64.png)

![](img/a6e83539dc131db0b76d166904680b0d_66.png)

![](img/a6e83539dc131db0b76d166904680b0d_68.png)

常用的信号有：
*   **`SIGHUP (1)`**：挂起信号，通常用于让进程重新读取配置文件。
*   **`SIGINT (2)`**：中断信号，等同于按 `Ctrl+C`。
*   **`SIGKILL (9)`**：强制终止信号。这个信号无法被进程捕获或忽略，会立即结束进程。
*   **`SIGTERM (15)`**：终止信号。这是 `kill` 命令默认发送的信号，请求进程正常退出。
*   **`SIGSTOP (19)`**：停止信号，等同于按 `Ctrl+Z`，暂停进程。

![](img/a6e83539dc131db0b76d166904680b0d_70.png)

![](img/a6e83539dc131db0b76d166904680b0d_72.png)

`kill` 命令的使用格式如下：
```bash
kill [信号] <PID>
```
例如：
*   `kill 1234`：向PID为1234的进程发送默认的 `SIGTERM(15)` 信号，请求其终止。
*   `kill -9 1234`：向PID为1234的进程发送 `SIGKILL(9)` 信号，强制其立即终止。
*   `kill -SIGKILL 1234`：与上一条命令效果相同。

![](img/a6e83539dc131db0b76d166904680b0d_74.png)

此外，还有一些有用的变体命令：
*   **`pkill`**：通过进程名来结束进程。例如 `pkill sleep` 会结束所有名为 `sleep` 的进程。
*   **`killall`**：类似 `pkill`，结束指定名称的所有进程。
*   **`pkill -U <username>`**：结束指定用户运行的所有进程。这在需要清理某个用户的全部会话时非常有用。
