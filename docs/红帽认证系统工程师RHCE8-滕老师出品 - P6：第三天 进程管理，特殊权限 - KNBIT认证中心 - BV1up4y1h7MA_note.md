# Linux系统管理：P6：第三天 进程管理与特殊权限

在本节课中，我们将要学习Linux系统中的变量和特殊权限。变量是脚本编程的基础，理解其分类和用法至关重要。特殊权限则扩展了标准文件权限，用于实现特定的系统功能。我们将逐一解析这些概念，并通过实例帮助大家理解。

## 变量分类与使用

上一节我们介绍了进程管理的基础知识，本节中我们来看看Linux系统中的变量。变量主要分为环境变量、自定义变量、只读变量和位置参数。

![](img/d51cee42f12a808d1845a820093c9d0d_3.png)

### 环境变量

环境变量是系统预定义的、与操作环境相关的变量。它们通常为大写，用于存储系统路径、用户信息、语言设置等。

以下是查看所有环境变量的命令：
```bash
env
```
或
```bash
set
```

**PATH变量**是一个核心的环境变量，它定义了系统查找可执行命令的目录列表。
```bash
echo $PATH
```
其值通常包含如 `/usr/bin`、`/usr/sbin` 等目录。当用户输入一个命令（如 `ls`）时，系统会依次在 `PATH` 变量列出的目录中查找该命令的可执行文件。

若希望自己编写的脚本也能像系统命令一样在任何目录直接运行，有两种方法：
1.  将脚本复制到 `PATH` 变量中的某个目录（如 `/usr/local/bin`）。
2.  将脚本所在目录的路径添加到 `PATH` 变量中（通常不推荐将个人目录加入 `PATH`，存在安全风险）。

### 自定义变量

自定义变量由用户自行定义，其生命周期默认仅限于定义它的当前Shell进程。

定义与查看自定义变量的方法如下：
```bash
MY_VAR="Hello"  # 定义变量
echo $MY_VAR     # 查看变量值
```

自定义变量需注意以下几点：
*   变量名区分大小写。
*   变量名不能以数字开头，但可以包含数字。
*   变量名中不能有空格，如需可用引号包裹。

![](img/d51cee42f12a808d1845a820093c9d0d_5.png)

若希望自定义变量在子Shell中也可用，需将其提升为环境变量：
```bash
export MY_VAR
```
或定义时直接提升：
```bash
export MY_VAR="Hello"
```

![](img/d51cee42f12a808d1845a820093c9d0d_7.png)

使用 `unset` 命令可以取消一个变量：
```bash
unset MY_VAR
```

### 只读变量

只读变量一旦被定义和赋值，其值便不可更改。
```bash
readonly MY_READONLY_VAR="Initial Value"
# 尝试修改 MY_READONLY_VAR 会报错
```

### 位置参数

![](img/d51cee42f12a808d1845a820093c9d0d_9.png)

位置参数是一种特殊的变量，以数字命名（如 `$1`, `$2`），用于获取传递给脚本或函数的参数。

![](img/d51cee42f12a808d1845a820093c9d0d_11.png)

`$0` 代表脚本自身的名称。
`$1`, `$2`, `$3`... 分别代表第一个、第二个、第三个参数。
`$@` 或 `$*` 代表所有参数。
`$#` 代表参数的个数。

以下是一个示例脚本 `test_args.sh`：
```bash
#!/bin/bash
echo "脚本名称: $0"
echo "第一个参数: $1"
echo "第二个参数: $2"
echo "所有参数: $@"
echo "参数个数: $#"
```
运行该脚本：
```bash
bash test_args.sh arg1 arg2
```

此外，还有其他常用的以`$`开头的特殊变量：
*   `$$`：当前Shell进程的PID。
*   `$?`：上一个命令的退出状态码（0表示成功，非0表示失败）。

## 特殊权限

![](img/d51cee42f12a808d1845a820093c9d0d_17.png)

除了基本的读（r）、写（w）、执行（x）权限，Linux还有三种特殊权限：SUID、SGID和Sticky Bit。

![](img/d51cee42f12a808d1845a820093c9d0d_19.png)

![](img/d51cee42f12a808d1845a820093c9d0d_21.png)

![](img/d51cee42f12a808d1845a820093c9d0d_23.png)

### SUID (Set User ID)

![](img/d51cee42f12a808d1845a820093c9d0d_25.png)

SUID权限的数字表示为 **4**。
当为一个**可执行文件**设置SUID后，任何用户在执行该文件时，都将以**文件所有者**的权限身份运行，而不是执行者自身的权限。

![](img/d51cee42f12a808d1845a820093c9d0d_27.png)

一个典型例子是 `/usr/bin/passwd` 命令：
```bash
ls -l /usr/bin/passwd
```
输出中所有者权限位的 `x` 会变为 `s`（如果显示大写的`S`，则表示所有者没有执行权限`x`）。
这使得普通用户在执行 `passwd` 命令时，能临时获得root权限去修改受保护的 `/etc/shadow` 文件，从而更改自己的密码。

设置SUID：
```bash
chmod u+s filename
# 或使用数字
chmod 4755 filename
```

### SGID (Set Group ID)

SGID权限的数字表示为 **2**。
当为一个**目录**设置SGID后，在该目录下创建的任何新文件或子目录，其所属组将自动**继承**该目录的所属组，而不是创建者的默认组。

这常用于需要团队协作的共享目录：
```bash
mkdir shared_dir
chgrp team_group shared_dir  # 更改目录所属组
chmod g+s shared_dir         # 设置SGID权限
```
现在，任何用户在 `shared_dir` 中创建的文件，其所属组都是 `team_group`。

设置SGID：
```bash
chmod g+s directory_name
# 或使用数字
chmod 2775 directory_name
```

### Sticky Bit (粘滞位)

Sticky Bit权限的数字表示为 **1**。
当为一个**目录**设置Sticky Bit后，即使该目录对所有用户都有写权限（如权限为777），用户也**只能删除或重命名自己拥有的文件**，而不能删除其他用户的文件。

![](img/d51cee42f12a808d1845a820093c9d0d_29.png)

![](img/d51cee42f12a808d1845a820093c9d0d_30.png)

系统临时目录 `/tmp` 默认设置了此权限：
```bash
ls -ld /tmp
```
输出中其他人权限位的 `x` 会变为 `t`（如果显示大写的`T`，则表示其他人没有执行权限`x`）。

![](img/d51cee42f12a808d1845a820093c9d0d_32.png)

![](img/d51cee42f12a808d1845a820093c9d0d_34.png)

设置Sticky Bit：
```bash
chmod o+t directory_name
# 或使用数字
chmod 1777 directory_name
```

## 系统监控命令：vmstat

![](img/d51cee42f12a808d1845a820093c9d0d_40.png)

![](img/d51cee42f12a808d1845a820093c9d0d_42.png)

`vmstat` 命令用于报告关于系统进程、内存、交换分区、I/O和CPU使用情况的整体统计信息，是一个重要的系统性能监控工具。

![](img/d51cee42f12a808d1845a820093c9d0d_44.png)

基本用法：
```bash
vmstat [间隔时间] [次数]
```
例如，每秒刷新一次，总共显示5次：
```bash
vmstat 1 5
```

![](img/d51cee42f12a808d1845a820093c9d0d_46.png)

输出信息主要分为几个部分：
*   **Procs (进程)**:
    *   `r`: 等待运行或正在运行的进程数。
    *   `b`: 处于不可中断睡眠状态（通常与I/O相关）的进程数。
*   **Memory (内存)**:
    *   `swpd`: 已使用的虚拟内存（交换分区）大小。
    *   `free`: 空闲的物理内存大小。
    *   `buff`: 用作缓冲区的内存大小。
    *   `cache`: 用作高速缓存的记忆大小。
*   **Swap (交换分区)**:
    *   `si`: 每秒从磁盘交换到内存的数据量（swap in）。
    *   `so`: 每秒从内存交换到磁盘的数据量（swap out）。如果 `si`/`so` 持续过高，可能表示物理内存不足。
*   **IO (输入/输出)**:
    *   `bi`: 每秒从块设备接收的块数（读磁盘）。
    *   `bo`: 每秒发送到块设备的块数（写磁盘）。
*   **System (系统)**:
    *   `in`: 每秒的中断次数。
    *   `cs`: 每秒的上下文切换次数。过高可能意味着进程过多或CPU资源紧张。
*   **CPU (处理器)**:
    *   `us`: 用户进程占用CPU时间百分比。
    *   `sy`: 系统内核占用CPU时间百分比。
    *   `id`: CPU空闲时间百分比。
    *   `wa`: CPU等待I/O操作的时间百分比。如果过高，表示磁盘I/O可能是瓶颈。

通过观察 `vmstat` 的输出，管理员可以快速了解系统的负载状况和潜在瓶颈。
