# Linux运维：RHCE：无人值守安装配置与测试 🚀

在本节课中，我们将学习如何配置和测试一个无人值守（Kickstart）安装服务器。通过这种方式，可以实现Linux系统的大规模自动化部署，无需人工干预。我们将从理解应答文件开始，逐步完成整个环境的搭建与测试。

## 应答文件详解 📄

上一节我们介绍了无人值守安装的基本概念，本节中我们来看看其核心——Kickstart应答文件。这个文件定义了安装过程中的所有配置选项，在生产环境中，由于没有图形界面，我们必须手动编写或修改此文件。

![](img/2c983697469b6336a8b44f6825c4e166_1.png)

![](img/2c983697469b6336a8b44f6825c4e166_3.png)

以下是一个应答文件模板的关键部分及其含义：

![](img/2c983697469b6336a8b44f6825c4e166_5.png)

*   **安装模式**：`install` 表示全新安装。
*   **键盘布局**：`keyboard us` 使用美式键盘。
*   **Root密码**：`rootpw --iscrypted $6$...` 使用加密算法（如SHA-512）生成的密码。
*   **安装源**：`url --url="ftp://192.168.1.88/pub/"` 指定通过FTP获取系统镜像的地址。
*   **系统语言**：`lang zh_CN` 将系统语言设置为中文。
*   **防火墙**：`firewall --disabled` 表示禁用防火墙。
*   **认证加密**：`auth --useshadow --passalgo=sha512` 使用shadow密码和SHA-512算法。
*   **图形界面**：`xconfig --startxonboot` 安装并启动图形界面（此为可选项）。
*   **网络配置**：`network --bootproto=dhcp --device=ens33` 将ens33网卡配置为DHCP获取IP。
*   **时区设置**：`timezone Asia/Shanghai` 设置时区为亚洲/上海。
*   **引导程序**：`bootloader --location=mbr` 将引导程序安装到MBR。
*   **分区方案**：以下代码段定义了磁盘分区。
    ```bash
    clearpart --all --initlabel
    part /boot --fstype="xfs" --size=200
    part / --fstype="xfs" --size=10240
    part swap --fstype="swap" --size=1000
    ```
*   **安装后脚本**：`%post` 部分可以编写Shell脚本，用于系统安装完成后自动执行配置，例如配置Yum源。
*   **软件包选择**：`%packages` 部分指定要安装的软件包组，例如 `@development`。

![](img/2c983697469b6336a8b44f6825c4e166_9.png)

![](img/2c983697469b6336a8b44f6825c4e166_11.png)

## 配置文件部署与测试注意事项 ⚙️

![](img/2c983697469b6336a8b44f6825c4e166_15.png)

我们已经理解了应答文件的结构，现在需要将其部署到服务器并开始测试。首先，将编辑好的应答文件（例如 `ks.cfg`）上传到FTP服务器的指定目录，例如 `/var/ftp/pub/`。

![](img/2c983697469b6336a8b44f6825c4e166_17.png)

![](img/2c983697469b6336a8b44f6825c4e166_19.png)

![](img/2c983697469b6336a8b44f6825c4e166_21.png)

在开始测试前，有几点至关重要：

![](img/2c983697469b6336a8b44f6825c4e166_23.png)

![](img/2c983697469b6336a8b44f6825c4e166_25.png)

1.  **客户端内存**：进行自动化安装的客户机内存建议不少于2GB，否则安装过程可能出错。
2.  **测试机器选择**：可以使用全新裸机或已安装系统的机器测试。
    *   **全新裸机**：默认从硬盘启动，但硬盘无系统，会自动跳转至网络启动，测试效果最好。
    *   **已装系统机器**：需在启动时按特定快捷键（如F12）临时选择从网络启动，否则会进入原有系统。
3.  **“安装后重启”选项**：在应答文件中，`reboot` 指令需谨慎使用。如果客户机第一启动项仍是网卡，重启后会再次触发网络安装，导致循环安装。对于新机器，可以启用此项；对于测试机，建议关闭，或在安装后手动调整BIOS启动顺序。

![](img/2c983697469b6336a8b44f6825c4e166_27.png)

理论准备就绪，本节我们进行实战测试。我们分别用一台已安装系统的机器和一台全新虚拟机进行演示。

*   **已装系统机器测试**：重启机器，在启动瞬间按下 `F12` 键（具体按键因厂商而异），选择从网络启动。随后，机器会自动从DHCP服务器获取IP，下载引导文件和应答文件，并开始全自动安装。整个过程无需人工干预。
*   **全新虚拟机测试**：创建一台没有操作系统的新虚拟机，直接启动。它会自动尝试从网络启动，并开始自动化安装流程。

![](img/2c983697469b6336a8b44f6825c4e166_46.png)

![](img/2c983697469b6336a8b44f6825c4e166_48.png)

*   **获取应答文件失败**：通常是由于网络不通、FTP路径错误或匿名访问权限问题导致。
*   **安装缓慢**：如果在应答文件中选择了大量软件包（如开发工具组），会显著增加安装时间。生产环境建议最小化安装。
*   **内存不足**：客户端内存分配过低可能导致安装过程异常退出。

4.  可以挂载安装光盘，并使用 `yum makecache` 测试Yum源是否可用。
