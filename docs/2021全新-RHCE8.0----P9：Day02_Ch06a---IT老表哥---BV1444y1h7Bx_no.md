# RHCE8.0 红帽认证入门教程：Day02_Ch06a：管理用户和组 👥

![](img/735ad0f0cbe1761a525e59afe3dde071_0.png)

在本节课中，我们将要学习 Linux 系统中用户和组的管理。Linux 是一个多用户、多任务的操作系统，因此理解如何创建、管理和配置用户与组是系统管理的基础。我们将从核心概念讲起，逐步介绍相关的配置文件和管理命令。

## 用户分类 🧑‍💻

Linux 系统中的用户主要分为三类：

![](img/735ad0f0cbe1761a525e59afe3dde071_2.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_4.png)

1.  **超级管理员 (root)**：这是系统的最高权限用户，其用户标识符 **UID** 为 **0**。该用户拥有系统的完全控制权。
2.  **系统用户**：这类用户通常由系统或安装的软件服务创建和使用，用于运行后台进程。其特点是：
    *   **UID 范围**：**1 到 999**。
    *   **登录类型**：通常设置为不可登录（如 `/sbin/nologin`）。
3.  **普通用户**：由系统管理员创建，用于日常操作和系统管理。其 **UID** 通常从 **1000** 开始分配。在实际生产环境中，管理员通常使用普通用户进行管理，仅在需要时提升权限，这更符合安全审计的要求。

![](img/735ad0f0cbe1761a525e59afe3dde071_6.png)

## 用户标识符 (UID) 🔢

Linux 系统不是通过用户名，而是通过 **用户标识符 (UID)** 来唯一识别一个用户。每个用户都有一个特定的 UID。

例如，查看 `/etc/passwd` 文件的前两行：
```bash
head -n 2 /etc/passwd
```
输出类似于：
```
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/sbin/nologin
```
其中，第三字段就是 UID。`root` 的 UID 是 **0**，`bin` 的 UID 是 **1**。

![](img/735ad0f0cbe1761a525e59afe3dde071_8.png)

## 组的概念 👥

![](img/735ad0f0cbe1761a525e59afe3dde071_10.png)

用户可以被组织到组中，以方便进行权限管理。

![](img/735ad0f0cbe1761a525e59afe3dde071_11.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_13.png)

*   一个组内可以包含多个用户。
*   一个用户可以属于多个组。
*   每个用户有一个 **初始组（主要组）**，同时可以加入多个 **附加组**。
*   每个组也有一个唯一的 **组标识符 (GID)**。

当一个用户创建文件时，该文件的所属组默认就是该用户的初始组。

![](img/735ad0f0cbe1761a525e59afe3dde071_15.png)

Linux 使用四个主要文件来存储用户和组的信息：

![](img/735ad0f0cbe1761a525e59afe3dde071_26.png)

### 1. `/etc/passwd` - 用户账户信息

![](img/735ad0f0cbe1761a525e59afe3dde071_28.png)

此文件存储本地用户的基本信息，每行代表一个用户，由冒号 `:` 分隔为 7 个字段。

**格式**：`用户名:密码占位符:UID:GID:描述信息:家目录:Shell类型`

例如 `root:x:0:0:root:/root:/bin/bash`：
*   **字段1**：用户名 (`root`)
*   **字段2**：密码占位符 (`x`)，实际密码存储在 `/etc/shadow`
*   **字段3**：UID (`0`)
*   **字段4**：初始组 GID (`0`)
*   **字段5**：描述信息 (`root`)
*   **字段6**：家目录 (`/root`)
*   **字段7**：登录 Shell (`/bin/bash`，交互式；`/sbin/nologin`，不可登录)

![](img/735ad0f0cbe1761a525e59afe3dde071_30.png)

**注意**：通常不直接编辑此文件，而是使用管理命令。

![](img/735ad0f0cbe1761a525e59afe3dde071_32.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_34.png)

### 2. `/etc/shadow` - 用户密码信息

![](img/735ad0f0cbe1761a525e59afe3dde071_36.png)

此文件存储加密后的用户密码及相关策略，每行代表一个用户，由冒号 `:` 分隔为 9 个字段。

![](img/735ad0f0cbe1761a525e59afe3dde071_38.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_39.png)

**格式**：`用户名:加密密码:最近更改日期:最小修改间隔:密码有效期:警告期:宽限期:账号失效日:保留字段`

![](img/735ad0f0cbe1761a525e59afe3dde071_40.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_42.png)

例如 `bin:*:17784:0:99999:7:::`
*   **字段2**：`*` 或 `!` 表示未设置密码或密码被锁定。
*   **字段3**：从 1970年1月1日算起的天数。
*   **字段5**：`99999` 表示密码永不过期。

![](img/735ad0f0cbe1761a525e59afe3dde071_44.png)

使用 `id` 命令查看当前或指定用户的 UID、GID 及所属组。


![](img/735ad0f0cbe1761a525e59afe3dde071_88.png)

### 切换用户

使用 `su` (switch user) 命令切换用户身份。
*   `su username`：不完全切换，不加载该用户的环境变量。
*   `su - username`：完全切换，加载该用户的环境变量和家目录。切换到 root 时可简写为 `su -`。

### 创建用户

使用 `useradd` 命令创建新用户。默认会创建一个与用户名同名的组作为其初始组，UID/GID 自动递增。
```bash
useradd username
```
**常用选项**：
*   `-u UID`：指定 UID。
*   `-g GID或组名`：指定初始组。
*   `-G 附加组名`：指定附加组。
*   `-s Shell路径`：指定登录 Shell。
*   `-d 家目录路径`：指定家目录。
*   `-c "注释信息"`：添加描述。
*   `-r`：创建系统用户（UID 1-999）。

**完整示例**：
```bash
useradd -u 2000 -g student -G tgroup -s /bin/bash -c "Test User" -d /home/testuser testuser
```

### 修改用户属性

![](img/735ad0f0cbe1761a525e59afe3dde071_93.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_94.png)

使用 `usermod` 命令修改现有用户的属性，参数与 `useradd` 类似。
```bash
usermod -u 新UID username
usermod -G 新附加组 username
```

### 删除用户

使用 `userdel` 命令删除用户。
*   `userdel username`：仅删除用户记录，保留家目录和邮件。
*   `userdel -r username`：递归删除用户记录、家目录和邮件。

## 组管理命令 👥

### 创建和删除组

*   `groupadd groupname`：创建新组。
*   `groupdel groupname`：删除空组。

### 管理组成员

使用 `gpasswd` 命令管理组的成员。
*   `gpasswd -a username groupname`：将用户添加到附加组。
*   `gpasswd -d username groupname`：将用户从组中移除。

## 密码管理 🔐

### 设置或修改密码

![](img/735ad0f0cbe1761a525e59afe3dde071_99.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_100.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_101.png)

使用 `passwd` 命令。
*   `passwd`：修改当前用户自己的密码（需提供原密码）。
*   `passwd username`：root 用户修改其他用户的密码（无需原密码）。

![](img/735ad0f0cbe1761a525e59afe3dde071_103.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_104.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_106.png)

**使用管道快速设置密码（root权限）**：
```bash
echo "NewPassword" | passwd --stdin username
```

![](img/735ad0f0cbe1761a525e59afe3dde071_107.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_109.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_111.png)

### 查看和锁定密码

*   `passwd -S username`：查看用户密码状态。
*   `passwd -l username`：锁定用户密码，禁止登录。
*   `passwd -u username`：解锁用户密码。

![](img/735ad0f0cbe1761a525e59afe3dde071_113.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_114.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_116.png)

### 修改密码策略

![](img/735ad0f0cbe1761a525e59afe3dde071_118.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_120.png)

![](img/735ad0f0cbe1761a525e59afe3dde071_122.png)

使用 `chage` 命令修改密码的过期时间等信息。
```bash
chage -M 天数 username  # 设置密码有效期
chage -d 0 username     # 强制下次登录修改密码
```
