## 联合查询的注意事项 🔗

上一节我们介绍了连接查询，本节中我们来看看联合查询（UNION）的用法和需要注意的关键点。

联合查询连接两个SELECT命令的查询结果。要保证查询能正常执行且结果有意义，必须注意以下几点：

![](img/890cb5a844dc71beb01f5c0dc028b013_10.png)

以下是联合查询的核心要求：
1.  **列数必须相同**：两个SELECT语句查询出的字段数量必须一致，否则执行会报错。
2.  **字段含义应尽量一致**：列数相同只能保证语法不报错。查询结果的列名会以第一个SELECT语句的字段名为准。如果前后两个查询结果中字段的实际含义不同，那么合并后的数据将难以理解，失去实用价值。
3.  **数据应具有一致性**：理想情况下，前后查询的字段不仅在数量上，在数据类型和业务含义上也应保持一致，这样合并后的数据集才是清晰可用的。

例如，强行将`ID, name`字段与`class_id, class_name`字段联合，虽然列数相同，但数据含义混乱，结果难以使用。正确的联合查询应像连接查询中的全外连接示例那样，确保字段对应关系清晰。

因此，联合查询应确保两次查询的列数和意义相匹配，才能得到有意义的合并数据。

## 多表查询总结 📊

![](img/890cb5a844dc71beb01f5c0dc028b013_14.png)

我们已学习了单表查询、嵌套查询以及多表连接查询。下面来系统总结一下SELECT语句的各个部分及其常用用法。

![](img/890cb5a844dc71beb01f5c0dc028b013_16.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_18.png)

一个完整的SELECT查询语句结构如下：
```sql
SELECT 字段/函数
FROM 表名/子查询
[WHERE 条件]
[GROUP BY 分组字段]
[HAVING 分组后条件]
[ORDER BY 排序字段]
[LIMIT 分页];
```

![](img/890cb5a844dc71beb01f5c0dc028b013_20.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_22.png)

以下是各部分的详细说明：
*   **SELECT 部分**：可以是用星号`*`表示所有字段、具体字段名、或聚合函数（如`COUNT()`, `SUM()`）。
*   **FROM 部分**：
    *   单表查询：直接写表名。
    *   多表连接：使用`JOIN ... ON ...`语法或直接列出表名并用WHERE关联。
    *   子查询：可以将一个查询结果（必须是一个完整的虚拟表）放在FROM后，**并为其指定别名**。
*   **WHERE 部分**：这是条件过滤的核心，支持`=`、`>`、`LIKE`、`IN`、`BETWEEN`等多种操作符，以及逻辑连接`AND`、`OR`。
*   **GROUP BY 部分**：用于分组，常与聚合函数配合使用。
*   **HAVING 部分**：对分组后的结果进行二次过滤。WHERE是首次过滤（行级），HAVING是最终展示前的过滤（组级）。
*   **ORDER BY 部分**：用于对结果进行排序。
*   **LIMIT 部分**：用于分页，可以加在任何查询语句的末尾。

在MySQL中，连接查询主要有三种命令：
1.  **内连接 (INNER JOIN)**：返回两个表匹配的行。
2.  **左外连接 (LEFT OUTER JOIN)**：返回左表所有行，以及右表匹配的行。
3.  **右外连接 (RIGHT OUTER JOIN)**：返回右表所有行，以及左表匹配的行。

通过左外连接和右外连接的组合（使用`UNION`），可以模拟实现其他类型的连接，如全外连接（FULL OUTER JOIN）。这是因为MySQL本身不直接支持`FULL OUTER JOIN`命令。

嵌套查询主要用在两个位置：
1.  放在`FROM`后面作为一个虚拟表，**必须起别名**。
2.  放在`WHERE`后面作为一个过滤条件或数据来源。

查询命令繁多，需要多加练习才能熟练掌握。

## 复制表操作 📋

查询结束后，我们接下来学习一个实用的小命令：复制表。复制表主要有两种用途：复制表结构和复制表数据。

复制表在需要创建一个与现有表结构相似的新表时非常有用，可以节省重新设计表结构的时间。

### 复制表结构

使用`CREATE TABLE ... LIKE ...`命令可以复制源表的结构（包括字段、约束、索引等），但不会复制数据。
```sql
CREATE TABLE new_table_name LIKE existing_table_name;
```
这相当于复制了MySQL底层存储表结构的`.frm`文件。

### 复制表结构及数据

![](img/890cb5a844dc71beb01f5c0dc028b013_24.png)

这是一个分两步的过程：
1.  先用上述方法复制表结构。
2.  使用`INSERT INTO ... SELECT ...`命令将数据从旧表插入到新表。
```sql
INSERT INTO new_table_name SELECT * FROM existing_table_name;
```
这保证了新表拥有和旧表完全一致的结构和数据。

### 快速复制数据（不推荐用于完整复制）

还有一种更直接的方式，在创建表的同时导入数据：
```sql
CREATE TABLE new_table_name SELECT * FROM existing_table_name;
```
但这种方法**主要复制数据**，表结构中的一些关键信息（如主键约束、自增属性、索引等）可能会丢失，导致新表不是一个“完整”或“规范”的表。因此，建议使用前两种方法进行完整的表复制。

## 破解MySQL root密码 🔑

![](img/890cb5a844dc71beb01f5c0dc028b013_30.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_32.png)

当忘记MySQL的root密码时，可以通过特殊方式重置。此操作需在数据库服务器本地进行，远程无法完成。

![](img/890cb5a844dc71beb01f5c0dc028b013_34.png)

破解密码的原理是：通过修改MySQL配置文件，使其在启动时跳过权限验证，从而可以无密码登录，然后修改密码。

![](img/890cb5a844dc71beb01f5c0dc028b013_36.png)

以下是具体步骤：
1.  **编辑MySQL配置文件**：找到MySQL的配置文件`my.cnf`（位置因安装方式而异，可能是`/etc/my.cnf`或`/usr/local/mysql/etc/my.cnf`）。
2.  **添加跳过授权配置**：在`[mysqld]`配置段下添加一行：
    ```
    skip-grant-tables
    ```
    为了方便，平时可以将这行用`#`注释掉，需要时再取消注释。
3.  **重启MySQL服务**：保存配置文件后，重启MySQL服务使配置生效。
    ```bash
    systemctl restart mysqld
    ```
4.  **无密码登录并修改密码**：此时可以直接用`mysql -u root`无密码登录。登录后，通过SQL语句修改`mysql.user`表中的密码。
    ```sql
    UPDATE mysql.user SET authentication_string=PASSWORD('你的新密码') WHERE User='root';
    ```
    **注意**：在MySQL 5.7中，密码字段是`authentication_string`；在更早版本中可能是`password`。`PASSWORD()`是加密函数。
5.  **恢复配置文件并重启**：修改密码后，退出MySQL，将配置文件中的`skip-grant-tables`一行再次注释或删除。然后重启MySQL服务。
6.  **使用新密码登录**：现在就可以使用刚设置的新密码正常登录了。

**版本差异**：在MySQL 5.7中，启用`skip-grant-tables`后所有用户都可无密码连接。在MySQL 8.0及以后，此模式下只有root用户可以无密码连接。

## 授权远程访问 🌐

![](img/890cb5a844dc71beb01f5c0dc028b013_40.png)

默认情况下，MySQL只允许本地连接。要允许其他服务器或客户端远程连接数据库，需要进行授权。

![](img/890cb5a844dc71beb01f5c0dc028b013_42.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_44.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_46.png)

授权命令的基本格式如下：
```sql
GRANT 权限列表 ON 数据库名.表名 TO '用户名'@'客户端IP' IDENTIFIED BY '密码';
```

![](img/890cb5a844dc71beb01f5c0dc028b013_48.png)

以下是关键参数解释：
*   **权限列表**：如`ALL PRIVILEGES`（所有权限）、`SELECT`、`INSERT`、`UPDATE`等。
*   **数据库名.表名**：`*.*`代表所有数据库的所有表。
*   **'用户名'@'客户端IP'**：
    *   `'root'@'192.168.1.100'`：仅允许IP为192.168.1.100的客户端使用root用户连接。
    *   `'root'@'%'`：允许任何IP的客户端使用root用户连接（`%`是通配符）。
*   **IDENTIFIED BY '密码'**：指定该用户从该IP连接时使用的密码。**这个密码可以与本地登录密码不同**，它专用于此次授权。

执行授权命令后，通常需要刷新权限：
```sql
FLUSH PRIVILEGES;
```

![](img/890cb5a844dc71beb01f5c0dc028b013_50.png)

**重要提醒**：授权后，如果仍无法远程连接，请检查服务器防火墙是否放行了MySQL的默认端口（**3306**）。需要关闭防火墙或在防火墙规则中放行3306端口。

## 存储引擎简介 💾

![](img/890cb5a844dc71beb01f5c0dc028b013_52.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_54.png)

最后，我们简要了解一下MySQL的存储引擎。存储引擎决定了数据如何存储、索引如何创建以及事务等特性。

![](img/890cb5a844dc71beb01f5c0dc028b013_56.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_57.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_59.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_61.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_63.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_64.png)

在MySQL数据目录（如`/var/lib/mysql/`）下，每个数据库是一个文件夹，每张表对应两个主要文件：
*   **`.frm`文件**：存储表的结构信息（框架）。
*   **`.ibd`文件**：存储表的数据和索引（如果使用InnoDB引擎）。

这种文件分离的存储方式，也解释了为什么我们可以分别复制表结构（对应.frm）和表数据（对应.ibd）。

MySQL最常用的两种存储引擎是：
*   **MyISAM**：较早的引擎，不支持事务和外键，但读取速度快。
*   **InnoDB**：MySQL 5.5版本后的默认引擎，支持**事务**、**外键**和**行级锁**，提供了更好的数据完整性和并发性能。

![](img/890cb5a844dc71beb01f5c0dc028b013_66.png)

![](img/890cb5a844dc71beb01f5c0dc028b013_68.png)

在现代数据库应用中，InnoDB通常是推荐的选择。

![](img/890cb5a844dc71beb01f5c0dc028b013_70.png)
