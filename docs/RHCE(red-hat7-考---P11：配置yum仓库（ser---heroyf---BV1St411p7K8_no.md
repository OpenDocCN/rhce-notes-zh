# RHCE 考前讲解：P11：配置 YUM 仓库 📦

![](img/0f1e09c7dca10b77176fb58a18b22711_0.png)

在本节课中，我们将学习如何在 Red Hat 环境中配置 YUM 软件仓库。这是 RHCE 考试中的一项基础但重要的任务，涉及在服务器和客户端上正确设置仓库配置文件，并理解关键参数的含义。

## 环境准备与任务概述

现在我们已经打开了两台设备：右边是服务器（server），左边是客户端（desktop）。第一道题的主要任务是配置 YUM 仓库的配置文件。操作时请注意熟练运用 `Tab` 键进行命令补全，以提高效率。

![](img/0f1e09c7dca10b77176fb58a18b22711_2.png)

## 在服务器上配置仓库

![](img/0f1e09c7dca10b77176fb58a18b22711_4.png)

上一节我们介绍了任务目标，本节中我们来看看具体的配置步骤。首先，我们需要在服务器上编辑 YUM 仓库配置文件。

![](img/0f1e09c7dca10b77176fb58a18b22711_6.png)

以下是创建仓库配置文件的具体步骤：

![](img/0f1e09c7dca10b77176fb58a18b22711_8.png)

![](img/0f1e09c7dca10b77176fb58a18b22711_10.png)

1.  使用 `vi` 或 `vim` 编辑器创建或编辑仓库文件：
    ```bash
    vim /etc/yum.repos.d/base.repo
    ```
2.  在文件中输入以下配置内容。注意，`name` 字段可以自定义，`gpgcheck` 必须设置为 `0` 以关闭 GPG 密钥检查，因为考试环境中通常不提供官方密钥。
    ```ini
    [RHCE]
    name=RHCE
    baseurl=http://content.example.com/rhel7.0/x86_64/dvd
    enabled=1
    gpgcheck=0
    ```
    *   **`[RHCE]`**: 仓库的唯一 ID。
    *   **`name`**: 仓库的可读名称。
    *   **`baseurl`**: 软件包的实际位置（URL）。建议从浏览器中复制，避免手动输入错误。
    *   **`enabled=1`**: 启用此仓库。
    *   **`gpgcheck=0`**: **公式**：`gpgcheck = 0`，表示禁用软件包签名验证。

![](img/0f1e09c7dca10b77176fb58a18b22711_12.png)

**关键点解释**：`gpgcheck` 参数用于验证软件包是否来自可信来源。若设置为 `1` 但未配置正确的 GPG 密钥，YUM 将无法工作。因此在无密钥的考试环境中，必须将其设为 `0`。

![](img/0f1e09c7dca10b77176fb58a18b22711_14.png)

## 获取并填写正确的 BaseURL

配置中的 `baseurl` 需要指向正确的软件包路径。你可以通过打开桌面环境的火狐浏览器，访问题目指定的地址来获取。

![](img/0f1e09c7dca10b77176fb58a18b22711_16.png)

操作方法是：复制浏览器地址栏中 `content.example.com/rhel7.0/x86_64/dvd` 部分的完整路径，直接粘贴到配置文件中。这能最大程度避免因手动输入导致的拼写错误。

![](img/0f1e09c7dca10b77176fb58a18b22711_18.png)

## 将配置同步到客户端

![](img/0f1e09c7dca10b77176fb58a18b22711_20.png)

在服务器上配置完成后，需要将这份配置文件复制到客户端（desktop）上，使客户端也能使用这个仓库。

![](img/0f1e09c7dca10b77176fb58a18b22711_22.png)

理论上，可以使用 `scp` 命令进行复制：
```bash
scp /etc/yum.repos.d/base.repo root@172.25.0.10:/etc/yum.repos.d/
```
这条命令会将服务器上的 `base.repo` 文件复制到 IP 为 `172.25.0.10` 的客户端的相同目录下。

![](img/0f1e09c7dca10b77176fb58a18b22711_24.png)

## 处理防火墙问题的实践建议

![](img/0f1e09c7dca10b77176fb58a18b22711_26.png)

在实际操作中，如果之前没有预先配置好服务器防火墙以允许相关连接，上述 `scp` 命令可能会被防火墙阻止而失败。

![](img/0f1e09c7dca10b77176fb58a18b22711_27.png)

此时，更稳妥高效的做法是：**不要花费时间去临时配置防火墙**，以免打乱解题思路。直接在客户端上手动创建一份相同的配置文件。

![](img/0f1e09c7dca10b77176fb58a18b22711_28.png)

![](img/0f1e09c7dca10b77176fb58a18b22711_30.png)

以下是手动创建的步骤：

![](img/0f1e09c7dca10b77176fb58a18b22711_31.png)

1.  登录到客户端（desktop）。
2.  同样使用编辑器创建文件：
    ```bash
    vim /etc/yum.repos.d/base.repo
    ```
3.  输入与服务器上**完全相同的配置内容**。

![](img/0f1e09c7dca10b77176fb58a18b22711_33.png)

**经验分享**：在考试时，分别在两台机器上手敲一遍配置是可靠的做法。这样可以避免因网络或防火墙问题导致的意外错误，也能让你更清晰地掌控每一步操作。

![](img/0f1e09c7dca10b77176fb58a18b22711_34.png)

![](img/0f1e09c7dca10b77176fb58a18b22711_35.png)

## 验证配置

![](img/0f1e09c7dca10b77176fb58a18b22711_37.png)

配置完成后，可以在客户端上运行以下命令来清理缓存并测试仓库是否可用：
```bash
yum clean all
yum repolist
```
如果配置正确，`yum repolist` 命令会显示出名为 `RHCE` 的仓库及其包含的软件包数量。

![](img/0f1e09c7dca10b77176fb58a18b22711_38.png)

![](img/0f1e09c7dca10b77176fb58a18b22711_40.png)

## 课程总结

![](img/0f1e09c7dca10b77176fb58a18b22711_41.png)

本节课中我们一起学习了 RHCE 考试中配置 YUM 仓库的完整流程。我们掌握了在服务器上创建仓库配置文件的关键参数设置，特别是将 `gpgcheck` 设置为 `0`。我们还学习了通过复制浏览器地址来准确填写 `baseurl`，并理解了在防火墙环境未预先配置时，直接在客户端手动创建配置文件是最优实践。确保配置能在服务器和客户端上同时生效，是完成后续软件管理任务的基础。
