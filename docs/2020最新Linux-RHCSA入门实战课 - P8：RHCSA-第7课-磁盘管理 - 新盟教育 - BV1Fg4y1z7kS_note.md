## 概述

在本节课中，我们将要学习Linux系统中的磁盘管理。我们将从理解计算机的基本架构和硬盘的物理结构开始，逐步深入到文件系统、分区、格式化以及挂载等核心概念和操作。通过本课的学习，你将能够掌握在Linux系统中管理磁盘和文件系统的基本技能。

## 计算机架构与存储

上一节我们介绍了课程概述，本节中我们来看看计算机的基本架构和存储器的角色。

计算机的架构由五大部件组成，这是由冯·诺依曼总结的。这五大部件分别是：
*   **运算器**
*   **控制器**
*   **存储器**
*   **输入设备**
*   **输出设备**

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_7.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_9.png)

其中，运算器和控制器结合起来就是我们常说的**CPU**。存储器包括内存和硬盘等。硬盘的主要目的就是**存储**数据。

## 文件系统的作用与类型

既然硬盘用于存储，我们就需要有规则来管理数据的存放。这个规则就是**文件系统**。

文件系统的作用是合理地规划硬盘，保证数据能够被正常地写入、读取和管理。它定义了一系列规则，例如：
*   数据存放在硬盘的哪个位置。
*   数据以何种格式存储。
*   文件的命名规则和权限原则。

不同的操作系统或场景使用不同的文件系统类型。常见的文件系统类型有：
*   **FAT/FAT32**：常用于U盘。
*   **NTFS**：常用于Windows系统硬盘和光盘。
*   **EXT3/EXT4**：Linux系统中较早期的文件系统。
*   **XFS**：红帽Linux 7及以后版本的默认文件系统，支持超大容量。

文件系统类型的区别主要在于支持的**最大存储容量**、**性能**和**崩溃恢复速度**等方面。例如，XFS支持的最大容量可达18EB，足以满足绝大多数需求。

## 硬盘的物理结构

要理解数据如何存取，我们需要了解硬盘的物理结构。硬盘主要分为**机械硬盘（HDD）** 和**固态硬盘（SSD）**。

机械硬盘的速度通常远慢于固态硬盘，因为其数据读取依赖物理运动。一个机械硬盘主要由以下部分组成：
*   **磁盘片**：存储数据的磁性盘片。
*   **磁道**：磁盘片上的同心圆轨道。
*   **扇区**：磁道上划分的扇形区域，是硬盘最小的物理存储单元，通常为**512字节**。
*   **柱面**：所有磁盘片上相同半径的磁道组成的柱状区域。
*   **磁头**：在磁道上读取和写入数据的部件。
*   **磁臂**：带动磁头移动到指定磁道的机械臂。

操作系统读取数据时，并非逐个读取扇区，而是连续读取多个扇区，这个逻辑单位称为**块**。在Linux中，常见的块大小为**4KB**。

## Inode：文件的“身份证”

在Linux文件系统中，一个文件的数据分为两部分存储：
1.  实际的文件内容，存储在**数据块**中。
2.  文件的元数据，如权限、所有者、时间戳等，存储在**Inode**中。

每个Inode都有一个唯一的编号。当你使用 `ls -li` 命令时，第一列显示的就是文件的Inode编号。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_11.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_13.png)

**硬链接**和**软链接**的区别与Inode密切相关：
*   **硬链接**：直接指向文件的Inode。创建硬链接会增加Inode的链接计数，删除原文件不会影响硬链接，因为它们指向同一个数据块。公式表示为：`ln <源文件> <链接名>`
*   **软链接**：是一个独立的文件，其内容是指向目标文件的路径。它拥有自己的Inode。删除原文件会导致软链接失效。公式表示为：`ln -s <源文件> <链接名>`

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_17.png)

## 磁盘分区管理

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_19.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_21.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_23.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_24.png)

一块新硬盘需要经过**分区**、**格式化**、**挂载**三个步骤才能使用。

### 分区类型与编号

一块硬盘最多只能有**4个主分区**。为了创建更多分区，我们可以使用**扩展分区**。扩展分区本身不能直接存储数据，但它可以再划分为多个**逻辑分区**。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_26.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_28.png)

常见的分区方案是 **3个主分区 + 1个扩展分区**，然后在扩展分区内创建所需的逻辑分区。

在Linux中，分区编号规则如下：
*   主分区编号：1-4
*   扩展分区编号：4
*   逻辑分区编号：从5开始

例如，一个有6个可用分区的硬盘，其编号可能是：1, 2, 3, 5, 6, 7（其中4是扩展分区，不可直接使用）。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_30.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_31.png)

### 使用 `fdisk` 进行分区（MBR，适用于<2TB硬盘）

`fdisk` 是一个交互式的分区工具，适用于管理小于2TB的硬盘（使用MBR分区表）。

以下是使用 `fdisk` 对 `/dev/sdb` 硬盘进行分区的核心步骤：

```bash
fdisk /dev/sdb
```
进入交互界面后，常用命令有：
*   `n`：创建新分区
*   `d`：删除分区
*   `p`：打印当前分区表
*   `w`：保存并退出
*   `q`：不保存退出

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_33.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_35.png)

创建分区时，需要选择分区类型（主分区 `p` 或扩展分区 `e`），指定起始扇区和分区大小（如 `+10G` 表示分配10GB空间）。操作完成后，使用 `partprobe` 命令让系统重新读取分区表。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_37.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_38.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_39.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_41.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_43.png)

### 使用 `gdisk` 或 `parted` 进行分区（GPT，适用于≥2TB硬盘）

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_45.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_47.png)

对于大于等于2TB的硬盘，需要使用GPT分区表，工具可以是 `gdisk` 或 `parted`。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_49.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_51.png)

`gdisk` 的使用方式与 `fdisk` 非常相似，也是交互式命令。而 `parted` 命令则更偏向于一行命令完成操作。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_53.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_54.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_56.png)

例如，使用 `parted` 将 `/dev/sdb` 设置为GPT格式并创建一个主分区：
```bash
parted /dev/sdb mklabel gpt
parted /dev/sdb mkpart primary 0% 100%
```

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_58.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_60.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_61.png)

## 格式化与挂载

分区完成后，需要格式化为具体的文件系统才能存储数据。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_63.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_65.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_67.png)

### 格式化

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_69.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_71.png)

使用 `mkfs` 命令进行格式化。例如，将 `/dev/sdb1` 格式化为EXT4文件系统：
```bash
mkfs.ext4 /dev/sdb1
```
或者格式化为XFS文件系统：
```bash
mkfs.xfs /dev/sdb1
```

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_73.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_75.png)

### 挂载

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_77.png)

格式化后，需要将分区挂载到系统目录树中的一个空目录（挂载点）上才能访问。
```bash
mount /dev/sdb1 /data
```
这条命令将 `/dev/sdb1` 分区挂载到 `/data` 目录。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_79.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_81.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_83.png)

### 实现开机自动挂载

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_85.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_87.png)

手动挂载的分区在重启后会失效。为了实现开机自动挂载，需要编辑 `/etc/fstab` 文件。

首先，可以使用 `blkid` 命令获取分区的UUID，这比使用设备名更稳定。
```bash
blkid /dev/sdb1
```

然后，在 `/etc/fstab` 文件中添加一行配置：
```
UUID=你的分区UUID    /data    ext4    defaults    0 0
```
各字段含义为：设备标识、挂载点、文件系统类型、挂载选项、是否备份、是否开机检查。

编辑完成后，可以使用 `mount -a` 命令测试配置是否正确，它会挂载所有在 `/etc/fstab` 中定义但未挂载的设备。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_89.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_91.png)

## Swap交换分区管理

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_93.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_95.png)

Swap分区相当于虚拟内存，当物理内存不足时，系统会将部分数据暂时存放到Swap分区。

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_97.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_99.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_101.png)

### 查看Swap信息
```bash
free -h
```

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_103.png)

### 创建并激活Swap分区

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_105.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_107.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_109.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_111.png)

可以将一个分区（如 `/dev/sdb2`）创建为Swap：
```bash
mkswap /dev/sdb2  # 格式化分区为Swap
swapon /dev/sdb2  # 激活Swap分区
```
同样，需要将Swap分区加入 `/etc/fstab` 以实现开机自动激活：
```
/dev/sdb2    swap    swap    defaults    0 0
```

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_113.png)

### 使用文件创建Swap

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_115.png)

![](img/03f0e8229a9040eb8374b1a84b8c3e6d_117.png)

除了使用独立分区，也可以用一个文件来充当Swap空间：
```bash
dd if=/dev/zero of=/swapfile bs=1M count=1024  # 创建一个1GB的文件
mkswap /swapfile  # 格式化为Swap
chmod 600 /swapfile # 修改权限
swapon /swapfile # 激活
```
同样，将 `/swapfile swap swap defaults 0 0` 加入 `/etc/fstab`。
