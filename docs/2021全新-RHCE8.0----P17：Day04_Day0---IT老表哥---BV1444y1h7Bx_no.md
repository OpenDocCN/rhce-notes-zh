## 第七章回顾：权限管理 🔐

上一节我们介绍了文件系统的基本操作，本节中我们来看看Linux系统中至关重要的权限管理机制。

### 用户身份与基本权限

一个文件或目录的权限由9位字符表示，分为三个部分。每个部分对应一类用户身份：

*   **所有者 (User/u)**：文件或目录的创建者。
*   **所属组 (Group/g)**：文件或目录所属的用户组。
*   **其他用户 (Other/o)**：既不是所有者，也不在所属组中的其他所有用户。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_5.png)

每类用户身份对应三种基本权限：

![](img/817a6e1884fd9bbe044d6ddc26b7a108_7.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_9.png)

*   **读 (r)**：对于文件，表示可以读取内容；对于目录，表示可以列出目录内的文件列表。
*   **写 (w)**：对于文件，表示可以修改内容；对于目录，表示可以在其中创建、删除文件或子目录。
*   **执行 (x)**：对于文件，表示可以将其作为程序运行；对于目录，表示可以进入（`cd`）该目录。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_11.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_13.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_15.png)

权限也可以用数字表示：`r=4`, `w=2`, `x=1`。组合权限即为数字相加，例如 `rwx=7`, `r-x=5`。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_17.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_19.png)

### 文件与目录的常见权限组合

以下是实践中常见的权限设置：

对于目录，通常只有以下几种组合有意义：
*   `rwx` (7)：拥有完全控制权。
*   `r-x` (5)：可以进入并列出内容，但不能修改。
*   `---` (0)：没有任何权限。

对于文件，常见的组合有：
*   `rw-` (6)：可读可写。
*   `r--` (4)：只读。
*   `r-x` (5)：可读可执行。
*   `---` (0)：无权限。

### 默认权限与umask

系统创建文件或目录时，会赋予一个默认权限。这个默认权限由“最大可赋予权限”减去“umask（权限掩码）”得到。

*   目录的最大权限为 `0777` (`rwxrwxrwx`)。
*   文件的最大权限为 `0666` (`rw-rw-rw-`)。
*   查看当前umask：`umask`
*   root用户的umask通常为 `0022`，普通用户为 `0002`。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_21.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_23.png)

计算公式为：**默认权限 = 最大权限 - umask**。例如，root用户创建文件：`0666 - 0022 = 0644` (`rw-r--r--`)。

### 特殊权限

除了基本的rwx，还有三种特殊权限，它们占用权限位的第一位（千位）。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_25.png)

1.  **SUID (Set User ID)**：权限值为 `4xxx`。当设置在**可执行文件**上时，任何用户执行该文件期间，都将临时拥有文件**所有者**的权限。命令为 `chmod u+s 文件名`。
    *   **例子**：`/usr/bin/passwd` 命令就设置了SUID，允许普通用户临时获得root权限来修改自己的密码（写入 `/etc/shadow` 文件）。

2.  **SGID (Set Group ID)**：权限值为 `2xxx`。有两种情况：
    *   设置在**可执行文件**上时（`chmod g+s 文件`），执行期间将拥有文件**所属组**的权限。
    *   设置在**目录**上时（`chmod g+s 目录`），在该目录下新建的任何文件或子目录，其所属组都会自动继承该目录的所属组。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_27.png)

3.  **Sticky Bit (粘滞位)**：权限值为 `1xxx`。仅对**目录**有效（`chmod o+t 目录`）。设置了粘滞位的目录，其中的文件只有**文件所有者**、**目录所有者**或 **root** 用户才能删除，其他用户即使有写权限也无法删除。常用于共享目录（如 `/tmp`）。

### 隐藏权限与ACL

1.  **隐藏属性 (chattr)**：使用 `chattr` 命令设置，`lsattr` 命令查看。
    *   `i` (immutable)：文件被设置为**不可修改**（不能删除、重命名、添加链接，也不能写入数据）。相当于完全写保护。
    *   `a` (append only)：文件只能**追加数据**，不能修改或删除已有内容。相当于部分写保护。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_29.png)

2.  **文件访问控制列表 (ACL)**：提供比传统9位权限更精细的权限控制，可以为特定用户或组单独设置权限。
    *   查看ACL：`getfacl 文件名`
    *   设置ACL：`setfacl -m u:用户名:权限 文件名` 或 `setfacl -m g:组名:权限 文件名`
    *   递归设置（对目录及其内容）：`setfacl -R -m ...`

### 作业控制 (Job Control)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_78.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_80.png)

在单个终端会话中，可以管理前台和后台作业。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_84.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_86.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_88.png)

*   在命令后加 `&` 符号：让命令在**后台**立即运行（如 `ping baidu.com &`）。
*   `Ctrl + Z`：暂停当前**前台**作业，并将其放入后台。
*   `jobs`：列出当前终端的所有后台作业。
*   `fg %作业号`：将指定的后台作业调至**前台**运行。
*   `bg %作业号`：让暂停的后台作业**继续在后台**运行。
*   `nohup 命令 &`：使命令在用户退出登录后仍能继续运行，输出默认重定向到 `nohup.out` 文件。

### 信号与进程终止

![](img/817a6e1884fd9bbe044d6ddc26b7a108_96.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_98.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_100.png)

可以使用 `kill` 命令向进程发送信号来控制它。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_102.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_104.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_106.png)

*   `kill -9 PID`：发送 `SIGKILL` 信号，**强制立即终止**进程，不给进程清理的机会。
*   `kill -15 PID`：发送 `SIGTERM` 信号，请求进程**正常终止**，允许进程进行保存等收尾工作（默认信号）。
*   `killall -9 进程名`：根据进程名，强制终止所有同名进程。
*   其他常用信号：
    *   `-1` (`SIGHUP`)：让进程重新加载配置文件（挂起）。
    *   `-0`：用于检测进程是否存在。

## 第九章回顾：服务管理 🖥️

![](img/817a6e1884fd9bbe044d6ddc26b7a108_114.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_116.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_118.png)

系统服务是在后台持续运行、提供特定功能的进程。红帽RHEL 7/8使用 `systemd` 作为初始化系统和服务管理器。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_120.png)

### 使用 systemctl 管理服务

所有服务都通过 `systemctl` 命令进行管理，替代了旧版的 `service` 和 `chkconfig` 命令。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_122.png)

基本操作如下：
*   启动服务：`systemctl start 服务名`
*   停止服务：`systemctl stop 服务名`
*   重启服务：`systemctl restart 服务名`
*   重新加载配置（不重启）：`systemctl reload 服务名`
*   查看服务状态：`systemctl status 服务名`
*   设置开机自启：`systemctl enable 服务名`
*   禁用开机自启：`systemctl disable 服务名`

### 运行级别与目标 (Target)

`systemd` 使用“目标 (target)”来替代传统的运行级别。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_124.png)

*   切换到多用户文本模式：`systemctl isolate multi-user.target`
*   切换到图形界面模式：`systemctl isolate graphical.target`
*   设置默认启动目标：`systemctl set-default multi-user.target`

**重要原则**：修改任何服务的配置文件后，通常需要**重启该服务** (`systemctl restart`) 才能使新配置生效。

## 第十章回顾：远程连接管理 🌐

![](img/817a6e1884fd9bbe044d6ddc26b7a108_128.png)

管理Linux服务器通常通过远程连接进行，最常用的协议是SSH。

![](img/817a6e1884fd9bbe044d6ddc26b7a108_130.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_131.png)

### 基本SSH连接

![](img/817a6e1884fd9bbe044d6ddc26b7a108_133.png)

*   连接远程主机：`ssh 用户名@远程主机IP或域名`
*   连接并直接执行单条命令：`ssh 用户名@主机 命令`

![](img/817a6e1884fd9bbe044d6ddc26b7a108_135.png)

![](img/817a6e1884fd9bbe044d6ddc26b7a108_137.png)

首次连接某台主机时，客户端会保存该主机的公钥指纹到 `~/.ssh/known_hosts` 文件中，以建立信任。如果服务器密钥变更，需要删除该文件中对应的条目才能重新连接。

### SSH密钥对认证（免密登录）

为了实现更安全、便捷的免密码登录，可以使用公钥认证。

1.  在**客户端**生成密钥对：`ssh-keygen -t rsa`。默认在 `~/.ssh/` 下生成 `id_rsa` (私钥) 和 `id_rsa.pub` (公钥)。
2.  将**客户端公钥**上传到**服务器**对应用户的 `~/.ssh/authorized_keys` 文件中。可以使用便捷命令：`ssh-copy-id 用户名@服务器IP`。
3.  完成以上步骤后，再次SSH连接该服务器用户时，就不再需要输入密码。

### SSH服务器配置

配置文件位于 `/etc/ssh/sshd_config`。常见修改项：

*   **禁止root远程登录**：设置 `PermitRootLogin no`。
*   **修改默认端口**：修改 `Port 22` 为其他端口（如 `Port 2222`），以增强安全性。注意要选择1024-65535之间未被占用的端口。
*   **应用配置**：**修改配置文件后，必须重启SSH服务**：`systemctl restart sshd`。
