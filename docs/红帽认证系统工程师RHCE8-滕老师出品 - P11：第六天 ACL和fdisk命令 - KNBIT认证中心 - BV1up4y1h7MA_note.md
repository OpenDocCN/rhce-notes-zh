## 什么是ACL？ 🔍

ACL（Access Control List，访问控制列表）是一种对文件和目录进行访问控制的机制。与传统的用户/组/其他（UGO）权限模型相比，ACL允许我们为**特定的用户**或**特定的组**设置独立的权限，从而实现更细粒度的访问控制。

### 传统权限的局限性

传统的`chmod`命令只能为三类对象设置权限：文件所有者（user）、所属组（group）和其他用户（other）。例如，有一个文件`abc`，用户`user1`和`user2`都属于“其他用户”类别。如果我们希望`user1`能读，`user2`能写，传统方法无法单独为`user1`和`user2`设置不同的权限。给“其他用户”写权限会导致`user1`也获得写权限，这不符合要求。

![](img/02cfdf75f8678a9d3483722b555c9a8c_6.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_8.png)

### ACL的优势

![](img/02cfdf75f8678a9d3483722b555c9a8c_10.png)

ACL解决了这个问题，它允许我们为单个用户或组指定精确的权限。例如，可以为`user1`设置读权限，为`user2`设置写权限。

![](img/02cfdf75f8678a9d3483722b555c9a8c_12.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_14.png)

**核心公式**：
`setfacl -m u:username:permissions file`
或
`setfacl -m g:groupname:permissions file`

## 查看和设置ACL权限 🛠️

### 查看ACL权限

使用`ls -l`命令查看文件时，如果权限字符串末尾有一个**加号（+）**，则表示该文件或目录设置了ACL。

**示例代码**：
```bash
ls -l filename
```
输出中若出现类似 `-rw-rw-r--+` 的格式，末尾的 `+` 表示存在ACL。

要查看详细的ACL条目，需使用 `getfacl` 命令。

**示例代码**：
```bash
getfacl filename
```

### 设置ACL权限

![](img/02cfdf75f8678a9d3483722b555c9a8c_26.png)

使用 `setfacl` 命令设置ACL权限。`-m` 选项表示修改。

![](img/02cfdf75f8678a9d3483722b555c9a8c_28.png)

**示例场景**：为文件 `file1` 赋予用户 `user2` 读和写权限。
**示例代码**：
```bash
setfacl -m u:user2:rw file1
```

设置后，切换到 `user2` 用户即可验证其拥有写入权限。

![](img/02cfdf75f8678a9d3483722b555c9a8c_30.png)

## 理解Mask（掩码） 🎭

![](img/02cfdf75f8678a9d3483722b555c9a8c_36.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_38.png)

使用 `getfacl` 查看文件时，会看到一个名为 **mask** 的条目。Mask是一个重要的概念，它限制了**指定用户**、**拥有组**和**指定组**的有效权限。

### Mask的作用

*   **受影响的对象**：Mask的值会影响ACL中“指定用户”、“拥有组”和“指定组”这三个条目的**最大有效权限**。
*   **权限计算**：上述三个条目的实际生效权限，是其设置的权限与Mask值进行**“与”操作**的结果。即，它们的权限不能超过Mask定义的权限。
*   **不受影响的对象**：文件“所有者”和“其他用户”的权限不受Mask影响。

**关键点**：如果将一个用户的ACL权限设置为 `rwx`，但Mask是 `r--`，那么该用户的实际有效权限只有 `r--`。

**示例代码**（修改Mask）：
```bash
setfacl -m m::r file1
```
此命令将文件 `file1` 的Mask设置为只读。

## 为目录设置ACL 📂

为目录设置ACL有两种主要方式，它们对新创建的文件和子目录的继承行为不同。

![](img/02cfdf75f8678a9d3483722b555c9a8c_40.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_42.png)

### 1. 递归设置（-R）

使用 `-R` 选项可以递归地为目录及其**现有**的所有内容设置ACL。

**特点**：此方式只对执行命令时已存在的文件和子目录生效。之后在该目录下**新建**的文件或子目录**不会继承**这些ACL权限。

**示例代码**：
```bash
setfacl -R -m u:user2:rwx /path/to/directory
```

![](img/02cfdf75f8678a9d3483722b555c9a8c_44.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_46.png)

### 2. 默认ACL（Default ACL）

![](img/02cfdf75f8678a9d3483722b555c9a8c_48.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_50.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_52.png)

使用 `-d` 选项可以设置默认ACL。默认ACL会被该目录下**新建**的子目录和文件继承。

![](img/02cfdf75f8678a9d3483722b555c9a8c_54.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_56.png)

**特点**：默认ACL**本身不适用于父目录**，只适用于在父目录下**新建**的项目。这对于确保新创建的项目自动具有特定ACL非常有用。

**示例代码**：
```bash
setfacl -d -m u:user2:rx /path/to/directory
```
此命令为目录设置一个默认ACL，使得之后在该目录下新建的子目录和文件，用户 `user2` 都自动拥有读和执行权限。

## 删除ACL权限 🗑️

*   **删除特定条目**：使用 `-x` 选项。
    **示例代码**：
    ```bash
    setfacl -x u:user2 file1  # 删除文件上user2的ACL条目
    setfacl -d -x u:user2 dir1 # 删除目录上user2的默认ACL条目
    ```
*   **删除所有ACL条目**：使用 `-b` 选项。
    **示例代码**：
    ```bash
    setfacl -b file1  # 删除文件上的所有ACL
    setfacl -b dir1   # 删除目录上的所有ACL（包括递归设置的）
    ```
*   **删除所有默认ACL条目**：使用 `-k` 选项。
    **示例代码**：
    ```bash
    setfacl -k dir1   # 删除目录上的所有默认ACL
    ```

## 磁盘分区管理（fdisk） 💾

![](img/02cfdf75f8678a9d3483722b555c9a8c_64.png)

上一节我们介绍了如何通过ACL精细控制文件访问权限，本节中我们来看看如何管理系统的存储空间——即磁盘分区。`fdisk` 是一个传统的命令行磁盘分区工具。

### 分区步骤概述

管理一个新磁盘或空间通常包含以下四步：
1.  **分区**：使用 `fdisk` 在磁盘上创建分区。
2.  **格式化**：在分区上创建文件系统（如XFS、ext4）。
3.  **挂载**：将分区关联到文件系统树的一个目录上。
4.  **永久挂载**：编辑配置文件，使系统启动时自动挂载。

### 1. 使用fdisk创建分区

**示例代码**（对 `/dev/vdb` 磁盘进行操作）：
```bash
fdisk /dev/vdb
```
进入交互界面后，常用命令：
*   `n`：创建新分区。
*   `p`：打印当前分区表。
*   `d`：删除分区。
*   `w`：保存并退出。
*   `q`：不保存退出。

**注意**：在MBR格式下，如果需要超过4个分区，需要先创建**扩展分区（extended）**，然后在其中创建**逻辑分区（logical）**。考试时通常要求创建逻辑分区。

### 2. 格式化分区

![](img/02cfdf75f8678a9d3483722b555c9a8c_66.png)

创建文件系统，即“格式化”。红帽8默认使用XFS文件系统。

**示例代码**（将 `/dev/vdb1` 格式化为XFS）：
```bash
mkfs.xfs /dev/vdb1
```
如果使用ext4，命令为 `mkfs.ext4 /dev/vdb1`。

### 3. 临时挂载分区

将格式化好的分区挂载到一个目录（挂载点）。

**示例代码**：
```bash
mkdir /mnt/data
mount /dev/vdb1 /mnt/data
```
使用 `df -h` 或 `mount` 命令可以查看挂载情况。

### 4. 配置永久挂载

临时挂载在重启后会失效。需要编辑 `/etc/fstab` 文件实现开机自动挂载。

**文件格式**：
`<设备标识> <挂载点> <文件系统类型> <挂载选项> <备份标记> <检查顺序>`

**示例**，在 `/etc/fstab` 中添加一行：
```
/dev/vdb1 /mnt/data xfs defaults 0 0
```
更推荐使用设备的UUID（唯一标识符），可以通过 `blkid` 命令查看。
```
UUID=xxxx-xxxx-xxxx /mnt/data xfs defaults 0 0
```

![](img/02cfdf75f8678a9d3483722b555c9a8c_68.png)

![](img/02cfdf75f8678a9d3483722b555c9a8c_70.png)

编辑保存后，使用 `mount -a` 命令重新加载 `/etc/fstab` 中的所有配置，或重启系统生效。

![](img/02cfdf75f8678a9d3483722b555c9a8c_72.png)

**重要提示**：编辑 `/etc/fstab` 文件时必须非常小心，任何错误都可能导致系统无法正常启动。
