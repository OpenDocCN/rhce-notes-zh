# RHCE认证课程：P27：日志转储

在本节课中，我们将要学习Linux系统中的日志转储功能。日志转储，也称为日志切割或日志轮转，是管理系统日志文件大小和数量的重要机制。通过配置日志转储，可以防止日志文件无限增长，从而节省磁盘空间并方便查阅历史日志。

## 日志转储的必要性

服务器在持续运行过程中会不断产生日志。例如，系统信息级别（`info`）及以上的日志通常存储在 `/var/log/messages` 文件中。如果服务器运行时间很长或操作频繁，这个日志文件会变得非常庞大。

![](img/dffe59a98ad1df9923dda01d8d5569f6_1.png)

庞大的日志文件会带来两个主要问题：首先，它会占用大量磁盘空间；其次，文件内容过多会导致查阅历史日志变得非常不便。因此，我们需要日志转储功能来管理这些日志文件。

![](img/dffe59a98ad1df9923dda01d8d5569f6_3.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_5.png)

## 日志转储的工作原理

![](img/dffe59a98ad1df9923dda01d8d5569f6_7.png)

日志转储的核心功能是周期性地对日志文件进行归档和新建。例如，对于 `/var/log/messages` 文件，可以设置转储规则，使其在7天后自动执行以下操作：
1.  将当前的 `messages` 文件重命名，例如加上日期后缀 `messages-20230731`。
2.  创建一个新的、空的 `messages` 文件来记录后续的日志。

这样，历史日志就被归档到带日期的文件中，而当前日志则记录在新的小文件中。转储的触发条件不仅可以是时间（如每周），也可以是文件大小（如达到10MB时触发）。

![](img/dffe59a98ad1df9923dda01d8d5569f6_9.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_10.png)

## 日志转储的依赖与工具

![](img/dffe59a98ad1df9923dda01d8d5569f6_12.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_14.png)

Linux系统的日志转储功能依赖于 `logrotate` 工具。该工具通常已预装，如需安装可使用包管理器。
*   主程序文件位于 `/usr/sbin/logrotate`。
*   手动执行转储的命令是 `logrotate`。
*   其核心配置文件是 `/etc/logrotate.conf`。
*   具体的转储规则配置文件位于 `/etc/logrotate.d/` 目录下。

![](img/dffe59a98ad1df9923dda01d8d5569f6_16.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_18.png)

## 解读全局配置文件

![](img/dffe59a98ad1df9923dda01d8d5569f6_24.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_26.png)

首先查看全局配置文件 `/etc/logrotate.conf`。该文件包含一些默认的全局设置，并且会通过 `include` 指令包含 `/etc/logrotate.d/` 目录下的所有特定规则配置文件。

![](img/dffe59a98ad1df9923dda01d8d5569f6_28.png)

以下是配置文件中常见的关键参数及其含义：

![](img/dffe59a98ad1df9923dda01d8d5569f6_30.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_32.png)

*   **`weekly`**： 设置转储周期为每周执行一次。
*   **`rotate 4`**： 设置保留的旧日志文件份数。例如，设为4表示会保留最新的4份历史日志，更早的则会被删除。
*   **`create`**： 转储后，创建一个新的空日志文件继续记录。
*   **`compress`**： 使用gzip压缩历史日志文件以节省空间。
*   **`dateext`**： 使用日期作为历史日志文件的后缀名（如 `messages-20230731`），而不是简单的序列号。
*   **`missingok`**： 如果日志文件不存在，不报错，继续处理下一个。
*   **`notifempty`**： 如果日志文件为空，则不执行转储。

## 自定义日志转储规则

![](img/dffe59a98ad1df9923dda01d8d5569f6_40.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_42.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_44.png)

全局配置通常作为默认值，更常见的做法是在 `/etc/logrotate.d/` 目录下为特定服务或日志文件创建独立的规则文件。

![](img/dffe59a98ad1df9923dda01d8d5569f6_46.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_48.png)

一个规则文件的基本格式是：先指定日志文件的路径（支持通配符），然后用大括号 `{}` 包裹具体的规则参数。

![](img/dffe59a98ad1df9923dda01d8d5569f6_50.png)

例如，系统自带的 `/etc/logrotate.d/yum` 规则可能如下所示，它管理 `/var/log/yum.log` 文件：
```
/var/log/yum.log {
    missingok
    notifempty
    size 30k
    yearly
    create 0600 root root
}
```

![](img/dffe59a98ad1df9923dda01d8d5569f6_52.png)

## 实践：创建自定义转储规则

![](img/dffe59a98ad1df9923dda01d8d5569f6_58.png)

假设我们需要管理一个自定义日志文件 `/var/log/test.log`，规则是：当文件大小超过400KB时转储，保留4份历史日志，转储后压缩旧文件，并创建权限为0644、所有者为student的新日志文件。

以下是创建该规则的步骤：

![](img/dffe59a98ad1df9923dda01d8d5569f6_60.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_62.png)

1.  **创建规则文件**：在 `/etc/logrotate.d/` 目录下新建一个文件，例如 `test`。
    ```bash
    sudo vim /etc/logrotate.d/test
    ```
2.  **编写规则内容**：将以下配置写入文件。
    ```
    /var/log/test.log {
        create 0644 student student
        compress
        rotate 4
        size 400k
    }
    ```
3.  **创建测试日志文件**：
    ```bash
    sudo touch /var/log/test.log
    sudo chown student:student /var/log/test.log
    ```
4.  **手动触发转储以测试规则**：使用 `-f` 选项强制立即执行转储。
    ```bash
    sudo logrotate -f /etc/logrotate.d/test
    ```
5.  **验证结果**：检查 `/var/log/` 目录，应该能看到类似 `test.log.1.gz` 的压缩归档文件，并且新的 `test.log` 文件已创建。

![](img/dffe59a98ad1df9923dda01d8d5569f6_64.png)

如果需要为历史日志文件名添加日期后缀，可以在规则中添加 `dateext` 参数。但请注意，使用 `dateext` 后，通常每天只会在第一次达到条件时转储一次。

## 日志转储与计划任务

![](img/dffe59a98ad1df9923dda01d8d5569f6_70.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_72.png)

![](img/dffe59a98ad1df9923dda01d8d5569f6_74.png)

`logrotate` 工具本身通常由系统的计划任务（cron job）驱动执行。例如，在 `/etc/cron.daily/` 目录下可能存在一个 `logrotate` 脚本，它每天自动运行一次，检查所有配置规则并执行符合条件的转储操作。这意味着我们自定义的规则也会被这个每日任务自动调度执行。

在规则配置中，还可以使用 `prerotate` 和 `postrotate` 指令，指定在转储开始前或结束后需要执行的额外命令，这为日志管理提供了更大的灵活性。
