## 通配符的深入应用

![](img/3851de4cdcfca8431e074919bbc7345f_8.png)

上一节我们介绍了星号（`*`）和问号（`?`）的基本用法，本节中我们来看看如何在实际场景中组合使用它们。

![](img/3851de4cdcfca8431e074919bbc7345f_10.png)

![](img/3851de4cdcfca8431e074919bbc7345f_12.png)

### 清空目录的注意事项

使用 `rm -rf *` 命令可以清空当前目录下的所有非隐藏文件。但隐藏文件（以点 `.` 开头的文件）默认不会被匹配到。

![](img/3851de4cdcfca8431e074919bbc7345f_14.png)

![](img/3851de4cdcfca8431e074919bbc7345f_15.png)

如果想同时清空隐藏文件（注意：`.` 和 `..` 代表当前目录和上级目录，通常不会被删除），可以使用 `rm -rf .*` 命令。但在当前目录下执行时，`.` 和 `..` 由于是系统目录链接，可能不会被删除，在其他目录下则有可能被匹配并删除，使用时需格外小心。

### 实战练习：保留特定文件

![](img/3851de4cdcfca8431e074919bbc7345f_17.png)

![](img/3851de4cdcfca8431e074919bbc7345f_18.png)

假设当前目录有以下文件，我们想只保留 `abc.txt` 文件，删除其他所有文件。
```
abc.txt
ab1.txt
ac2.txt
bc3.txt
...
```

![](img/3851de4cdcfca8431e074919bbc7345f_20.png)

![](img/3851de4cdcfca8431e074919bbc7345f_22.png)

这个问题的关键在于找出目标文件 `abc.txt` 与其他文件的**不同点**。观察发现，只有 `abc.txt` 文件的**第二个字母是 `b`**。

因此，我们的思路是：匹配所有**第二个字母不是 `b`** 的文件，然后删除它们。匹配模式可以这样构建：
*   第一个字母：任意字符，用 `?` 表示。
*   第二个字母：不能是 `b`，用 `[^b]` 表示。
*   第三个及之后的字母：任意字符，用 `*` 表示。

![](img/3851de4cdcfca8431e074919bbc7345f_24.png)

组合起来的命令就是：
```bash
rm -rf ?[^b]*
```
这个命令会匹配并删除所有第二个字母不是 `b` 的文件，从而只保留了第二个字母是 `b` 的 `abc.txt`。

![](img/3851de4cdcfca8431e074919bbc7345f_26.png)

**核心要点**：通配符 `[^...]` 用于匹配**不在**指定范围内的**一个**字符。

![](img/3851de4cdcfca8431e074919bbc7345f_28.png)

![](img/3851de4cdcfca8431e074919bbc7345f_30.png)

## 扩展的通配符类

![](img/3851de4cdcfca8431e074919bbc7345f_32.png)

除了基础通配符，Shell还预定义了一些字符类，可以更方便地匹配特定类型的字符。

![](img/3851de4cdcfca8431e074919bbc7345f_34.png)

![](img/3851de4cdcfca8431e074919bbc7345f_36.png)

以下是常用的通配符类及其含义：
*   `[[:lower:]]`：匹配任意一个小写字母（a-z）。
*   `[[:upper:]]`：匹配任意一个大写字母（A-Z）。
*   `[[:digit:]]`：匹配任意一个数字（0-9）。
*   `[[:alpha:]]`：匹配任意一个字母（包括大小写）。
*   `[[:alnum:]]`：匹配任意一个字母或数字。
*   `[[:space:]]`：匹配任意一个空白字符（如空格、制表符）。

![](img/3851de4cdcfca8431e074919bbc7345f_37.png)

![](img/3851de4cdcfca8431e074919bbc7345f_39.png)

**使用方法**：这些字符类需要放在 `[[]]` 双中括号内使用。例如，`[[:lower:]]` 整体代表“任意一个小写字母”。如果想匹配一个以小写字母开头的文件名，可以这样写：
```bash
ls [[:lower:]]*
```

![](img/3851de4cdcfca8431e074919bbc7345f_41.png)

## 命令扩展

![](img/3851de4cdcfca8431e074919bbc7345f_43.png)

命令扩展是一种强大的功能，它允许我们将一个命令的输出结果，直接嵌入到另一个命令行中。

![](img/3851de4cdcfca8431e074919bbc7345f_45.png)

![](img/3851de4cdcfca8431e074919bbc7345f_46.png)

### 语法与示例

![](img/3851de4cdcfca8431e074919bbc7345f_48.png)

![](img/3851de4cdcfca8431e074919bbc7345f_50.png)

命令扩展有两种等价的语法格式：
1.  **反引号**：`` `command` ``
2.  **美元符号加小括号**：`$(command)`

![](img/3851de4cdcfca8431e074919bbc7345f_52.png)

![](img/3851de4cdcfca8431e074919bbc7345f_54.png)

![](img/3851de4cdcfca8431e074919bbc7345f_56.png)

**推荐使用 `$(command)` 格式**，因为它更清晰，且不易与单引号混淆。

**作用**：Shell遇到这两种语法时，会先执行 `command` 命令，然后用该命令的**标准输出结果**替换掉整个 `$(command)` 或 `` `command` `` 结构。

**示例1：在字符串中嵌入命令结果**
```bash
echo “The system hostname is: $(hostname)”
```
这条命令会先执行 `hostname` 得到主机名（例如 `yutianedu`），然后输出：`The system hostname is: yutianedu`。

![](img/3851de4cdcfca8431e074919bbc7345f_58.png)

![](img/3851de4cdcfca8431e074919bbc7345f_60.png)

**示例2：直接使用命令结果作为参数**
```bash
echo $(date)
```
这会直接输出 `date` 命令的结果。如果不用命令扩展，`echo date` 只会输出字符串 “date”。

**错误示例**：
```bash
$(hostname)
```
这条命令会先执行 `hostname` 得到 `yutianedu`，然后Shell会试图将 `yutianedu` 当作一个命令来执行，显然会报“命令未找到”的错误。

### 实战应用：动态生成文件名

命令扩展在需要动态参数时非常有用。例如，每天备份一个带当前日期的文件：
```bash
cp -r /etc /tmp/etc_backup_$(date +%Y%m%d)
```
`$(date +%Y%m%d)` 会先被执行，输出格式化的当前日期（如 `20230419`），因此最终复制的目录名会是 `/tmp/etc_backup_20230419`。这样，每次运行命令都会自动生成一个带有执行日期的新备份，无需手动修改名称。

![](img/3851de4cdcfca8431e074919bbc7345f_62.png)

## 总结

![](img/3851de4cdcfca8431e074919bbc7345f_64.png)

本节课中我们一起学习了Shell中两个高级技巧：
1.  **通配符的灵活运用**：通过分析文件名的特征，组合使用 `?`、`*`、`[]`、`[^]` 以及 `[[:class:]]` 等通配符，可以精确匹配目标文件集合。
2.  **命令扩展**：使用 `$(command)` 语法，可以在命令行中直接引用另一个命令的执行结果，从而实现动态、自动化的操作，这是编写Shell脚本和进行高效系统管理的基础技能。

![](img/3851de4cdcfca8431e074919bbc7345f_66.png)

通过理解和练习这些概念，你将能更加得心应手地驾驭Linux命令行。
