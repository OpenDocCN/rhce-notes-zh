## 1. 软件包管理 📦

![](img/71d671f38cfd8db43051eea76793fb55_14.png)

![](img/71d671f38cfd8db43051eea76793fb55_16.png)

上一节我们介绍了Ansible的基础架构，本节中我们来看看如何使用Ansible管理软件包。

![](img/71d671f38cfd8db43051eea76793fb55_18.png)

Ansible提供了多种模块来管理RHEL系统中的软件包，其功能类似于命令行工具`yum`或`dnf`。

![](img/71d671f38cfd8db43051eea76793fb55_20.png)

![](img/71d671f38cfd8db43051eea76793fb55_21.png)

### 1.1 yum与dnf模块
`yum`和`dnf`模块都用于安装、升级或删除软件包。它们的用法几乎完全相同。
```yaml
- name: 使用yum安装软件包
  yum:
    name: httpd
    state: present

![](img/71d671f38cfd8db43051eea76793fb55_22.png)

![](img/71d671f38cfd8db43051eea76793fb55_24.png)

![](img/71d671f38cfd8db43051eea76793fb55_25.png)

![](img/71d671f38cfd8db43051eea76793fb55_27.png)

- name: 使用dnf安装软件包
  dnf:
    name: mariadb-server
    state: present
```

![](img/71d671f38cfd8db43051eea76793fb55_29.png)

![](img/71d671f38cfd8db43051eea76793fb55_31.png)

### 1.2 package_fact模块
`package_fact`模块用于收集受管主机上已安装软件包的信息。
以下是使用`package_fact`的示例：
```yaml
- name: 收集软件包信息
  package_facts:
    manager: auto

![](img/71d671f38cfd8db43051eea76793fb55_33.png)

![](img/71d671f38cfd8db43051eea76793fb55_35.png)

- name: 显示特定软件包版本
  debug:
    msg: “{{ ansible_facts.packages[‘httpd’][0].version }}”
```
此任务首先收集所有软件包信息，然后通过过滤器仅显示`httpd`软件的版本。

![](img/71d671f38cfd8db43051eea76793fb55_37.png)

![](img/71d671f38cfd8db43051eea76793fb55_39.png)

### 1.3 package模块
`package`模块是一个通用的包管理模块，可以视为`rpm`命令的Ansible封装，同样可以用于安装软件包。
```yaml
- name: 使用package模块安装软件
  package:
    name: vim
    state: present
```
目前，安装软件包主要有`yum`、`dnf`和`package`三种模块可供选择。

### 1.4 rpm_key模块
`rpm_key`模块用于导入RPM包的GPG密钥，通常在部署软件仓库时使用。
```yaml
- name: 导入GPG密钥
  rpm_key:
    key: “http://example.com/RPM-GPG-KEY-example”
    state: present
```

## 2. 用户与组管理 👥

![](img/71d671f38cfd8db43051eea76793fb55_43.png)

在掌握了软件包管理后，我们进入用户和组的管理。这些模块能帮助我们批量创建和管理系统账户。

![](img/71d671f38cfd8db43051eea76793fb55_50.png)

![](img/71d671f38cfd8db43051eea76793fb55_52.png)

![](img/71d671f38cfd8db43051eea76793fb55_54.png)

### 2.1 user模块
`user`模块用于管理用户账户，功能非常全面。
以下是创建一个用户的综合示例：
```yaml
- name: 创建用户示例
  hosts: all
  tasks:
    - name: 创建用户
      user:
        name: user20
        uid: 2000
        shell: /sbin/nologin
        password: “{{ ‘redhat’ | password_hash(‘sha512’) }}”
```
此任务创建了一个名为`user20`、UID为`2000`、使用`/sbin/nologin` shell且密码经过SHA512哈希加密的用户。

![](img/71d671f38cfd8db43051eea76793fb55_55.png)

`user`模块还可以用于生成SSH密钥对。
```yaml
- name: 为用户生成SSH密钥
  user:
    name: alice
    generate_ssh_key: yes
    ssh_key_bits: 2048
    ssh_key_file: .ssh/id_rsa
```

![](img/71d671f38cfd8db43051eea76793fb55_57.png)

![](img/71d671f38cfd8db43051eea76793fb55_59.png)

### 2.2 group模块
`group`模块用于管理用户组，例如创建或删除组。
```yaml
- name: 创建用户组
  group:
    name: developers
    state: present
```

![](img/71d671f38cfd8db43051eea76793fb55_61.png)

![](img/71d671f38cfd8db43051eea76793fb55_63.png)

### 2.3 known_hosts模块
`known_hosts`模块用于在受管主机的`/etc/ssh/ssh_known_hosts`文件中添加或删除主机密钥。
```yaml
- name: 添加主机密钥到已知列表
  known_hosts:
    name: server.example.com
    key: “{{ lookup(‘file’, ‘/path/to/server_key.pub’) }}”
```

![](img/71d671f38cfd8db43051eea76793fb55_65.png)

### 2.4 authorized_key模块
`authorized_key`模块用于为用户添加或删除SSH授权密钥，实现免密登录。
```yaml
- name: 添加SSH授权密钥
  authorized_key:
    user: alice
    key: “{{ lookup(‘url’, ‘https://github.com/alice.keys’) }}”
    state: present
```

## 3. 进程与服务管理 ⚙️

![](img/71d671f38cfd8db43051eea76793fb55_73.png)

![](img/71d671f38cfd8db43051eea76793fb55_75.png)

管理好用户后，系统的日常运行离不开进程和服务。Ansible提供了强大的计划任务和服务管理模块。

![](img/71d671f38cfd8db43051eea76793fb55_77.png)

![](img/71d671f38cfd8db43051eea76793fb55_78.png)

### 3.1 at模块
`at`模块用于安排一次性任务，在未来的特定时间点执行。
```yaml
- name: 安排一次性任务
  at:
    command: “/usr/sbin/useradd user201”
    count: 2
    units: minutes
    unique: yes
```
此任务将在2分钟后执行`useradd user201`命令，且`unique: yes`参数确保不会重复安排相同任务。可以使用`at -l`命令查看已安排的任务。

![](img/71d671f38cfd8db43051eea76793fb55_80.png)

![](img/71d671f38cfd8db43051eea76793fb55_82.png)

![](img/71d671f38cfd8db43051eea76793fb55_83.png)

![](img/71d671f38cfd8db43051eea76793fb55_85.png)

### 3.2 cron模块
`cron`模块用于管理周期性计划任务（cron job），将任务写入指定用户的crontab中。
以下是`cron`模块的参数和示例：
*   `special_time`: 预定义的时间宏（如`hourly`, `daily`）。
*   `state`: `present`创建，`absent`删除。
*   `cron_file`: 如果维护大量服务器，可以使用预编写的cron文件。
```yaml
- name: 创建周期性备份任务
  cron:
    name: “Daily backup”
    user: root
    hour: 1
    minute: 0
    job: “/usr/bin/tar -czf /backup/backup.tar.gz /data”
    state: present
```
此任务会创建一个每天凌晨1点运行的备份任务，并写入`/etc/crontab`文件。

![](img/71d671f38cfd8db43051eea76793fb55_87.png)

![](img/71d671f38cfd8db43051eea76793fb55_88.png)

![](img/71d671f38cfd8db43051eea76793fb55_89.png)

### 3.3 service与systemd模块
`service`和`systemd`模块用于管理系统服务。`systemd`模块功能更全，例如允许重新加载守护进程。
```yaml
- name: 启动并启用httpd服务
  systemd:
    name: httpd
    state: started
    enabled: yes
```

![](img/71d671f38cfd8db43051eea76793fb55_91.png)

### 3.4 reboot模块
`reboot`模块用于安全地重启受管主机，比直接使用shell命令更安全。它可以在任务运行期间重启主机，然后继续执行后续的playbook任务。
```yaml
- name: 重启主机
  reboot:
    reboot_timeout: 600
```

`shell`和`command`模块用于执行任意命令。`shell`模块支持管道符和变量操作，而`command`模块则更安全，但某些操作符无法使用。

### 3.6 ansible_facts环境变量
通过`ansible_facts.env`可以访问受管主机上的所有环境变量。
```yaml
- name: 打印所有环境变量
  debug:
    msg: “{{ ansible_facts.env }}”
```
也可以使用`lookup`插件来返回变量。
```yaml
- name: 使用lookup插件获取环境变量
  debug:
    msg: “{{ lookup(‘env’, ‘HOME’) }}”
```

## 4. 系统信息收集与过滤 🔍

在管理多台主机时，快速获取并筛选所需信息至关重要。Ansible的`setup`模块和过滤器能帮助我们高效完成这项工作。

默认情况下，运行playbook时会自动收集`ansible_facts`。我们可以使用`filter`参数来筛选所需信息。
```bash
# 收集所有事实，但信息过多

![](img/71d671f38cfd8db43051eea76793fb55_120.png)

![](img/71d671f38cfd8db43051eea76793fb55_123.png)

# 使用exclude_filter排除不需要的信息
ansible all -m setup -a “exclude_filter=ansible_mounts”
```

## 5. 存储管理 💾

![](img/71d671f38cfd8db43051eea76793fb55_131.png)

![](img/71d671f38cfd8db43051eea76793fb55_133.png)

存储管理是系统运维的核心。Ansible提供了一系列模块来简化磁盘分区、逻辑卷和文件系统的管理。

![](img/71d671f38cfd8db43051eea76793fb55_134.png)

### 5.1 parted模块
`parted`模块用于磁盘分区，替代了`fdisk`命令。
以下是`parted`模块的参数和示例：
*   `device`: 要分区的块设备。
*   `number`: 分区编号。
*   `state`: 分区状态（如`present`）。
*   `part_end`: 分区结束位置（用于指定大小）。
```yaml
- name: 创建10GB分区
  parted:
    device: /dev/vdb
    number: 1
    state: present
    part_end: 10GiB
```
此任务相当于在`/dev/vdb`磁盘上创建了一个10GB的分区。

![](img/71d671f38cfd8db43051eea76793fb55_136.png)

![](img/71d671f38cfd8db43051eea76793fb55_138.png)

![](img/71d671f38cfd8db43051eea76793fb55_140.png)

### 5.2 lvg与lvol模块
`lvg`模块用于管理卷组（VG），`lvol`模块用于管理逻辑卷（LV）。
以下是`lvg`和`lvol`模块的示例：
```yaml
# 示例1：创建物理卷并加入卷组
- name: 创建卷组
  lvg:
    vg: myvg
    pvs: /dev/vdc1
    state: present

![](img/71d671f38cfd8db43051eea76793fb55_142.png)

# 示例2：调整卷组的物理卷
- name: 调整卷组物理卷
  lvg:
    vg: vg1
    pvs: /dev/vdb
    state: present

![](img/71d671f38cfd8db43051eea76793fb55_144.png)

![](img/71d671f38cfd8db43051eea76793fb55_146.png)

# 示例3：创建逻辑卷
- name: 创建逻辑卷
  lvol:
    vg: myvg
    lv: mylv
    size: 2g
    state: present
```

### 5.3 filesystem模块
`filesystem`模块用于创建和调整文件系统，相当于`mkfs`命令。
```yaml
- name: 创建XFS文件系统
  filesystem:
    fstype: xfs
    dev: /dev/vdb1
```

![](img/71d671f38cfd8db43051eea76793fb55_148.png)

![](img/71d671f38cfd8db43051eea76793fb55_149.png)

![](img/71d671f38cfd8db43051eea76793fb55_151.png)

![](img/71d671f38cfd8db43051eea76793fb55_152.png)

### 5.4 mount模块
`mount`模块用于在`/etc/fstab`中配置挂载点，实现永久挂载。
```yaml
- name: 永久挂载文件系统
  mount:
    path: /data
    src: UUID=12345678-1234-1234-1234-123456789abc
    fstype: xfs
    state: present
```
此任务将指定UUID的设备，以XFS格式永久挂载到`/data`目录。

![](img/71d671f38cfd8db43051eea76793fb55_154.png)

![](img/71d671f38cfd8db43051eea76793fb55_156.png)

### 5.5 交换分区管理
目前Ansible可能没有专门的交换分区管理模块。一种可行的实现思路是：先使用`lvol`模块创建逻辑卷，然后使用`command`或`shell`模块执行`mkswap`和`swapon`命令。
```yaml
- name: 创建交换分区
  command: /sbin/mkswap /dev/myvg/swaplv
```

## 6. 网络管理 🌐

![](img/71d671f38cfd8db43051eea76793fb55_164.png)

![](img/71d671f38cfd8db43051eea76793fb55_166.png)

最后，我们探讨网络配置的自动化。虽然Ansible在此领域的模块仍在发展中，但已有一些可用的工具。

### 6.1 使用系统角色
一种方法是使用RHEL系统角色，例如`rhel-system-roles.network`。它通过定义变量来实现网络配置，但目前功能可能尚不完善。
```yaml
- hosts: all
  vars:
    network_connections:
      - name: eth0
        type: ethernet
        ip:
          address:
            - 192.168.1.10/24
  roles:
    - rhel-system-roles.network
```

### 6.2 nmcli模块
`nmcli`模块是对`nmcli`命令行工具的封装，可以配置连接、IP地址、DNS、网关等。
```yaml
- name: 配置网络连接
  nmcli:
    conn_name: my-eth0
    ifname: eth0
    type: ethernet
    ip4: 192.168.1.10/24
    gw4: 192.168.1.1
    state: present
```

### 6.3 hostname模块
`hostname`模块用于修改受管主机的主机名。
```yaml
- name: 修改主机名
  hostname:
    name: server01.example.com
```

![](img/71d671f38cfd8db43051eea76793fb55_182.png)

### 6.4 firewalld模块
`firewalld`模块用于配置防火墙规则，我们在之前的章节中已经接触过。
以下是两个配置示例：
```yaml
# 示例1：在默认区域（public）中放行HTTP服务
- name: 在public区域放行HTTP
  firewalld:
    service: http
    permanent: yes
    state: enabled
    immediate: yes

# 示例2：配置富规则（rich rule）
- name: 添加富规则允许特定IP
  firewalld:
    rich_rule: “rule family=“ipv4” source address=“192.168.1.100” port port=“22” protocol=“tcp” accept”
    zone: public
    permanent: yes
    state: enabled
```
