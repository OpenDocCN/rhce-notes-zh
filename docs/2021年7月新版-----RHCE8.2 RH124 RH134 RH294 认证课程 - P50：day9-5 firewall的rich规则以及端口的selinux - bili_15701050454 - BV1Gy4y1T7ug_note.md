# Linux防火墙与SELinux：Day9-5：Firewall富规则与端口SELinux 🔥

在本节课中，我们将学习Firewalld防火墙的富规则（Rich Rules）以及如何为网络端口配置SELinux安全上下文。富规则提供了比基本规则更精细的控制能力，而端口的SELinux标签则能进一步控制服务对端口的访问权限。

![](img/554bb2af56ee44846305dcd415309b3d_1.png)

## 富规则（Rich Rules）概述

![](img/554bb2af56ee44846305dcd415309b3d_3.png)

![](img/554bb2af56ee44846305dcd415309b3d_5.png)

上一节我们介绍了防火墙的基本规则。本节中，我们来看看功能更强大的富规则。富规则允许我们设置更复杂的匹配条件，例如源IP地址、协议、端口、连接限制，甚至进行端口转发。

富规则的基本语法结构如下：
```bash
rule [family="ipv4|ipv6"]
     [source|destination address="address[/mask]"]
     [service name="service name"]
     [port port="number" protocol="tcp|udp"]
     [protocol value="protocol"]
     [icmp-block name="icmp type"]
     [masquerade]
     [forward-port port="number" protocol="tcp|udp" to-port="number" to-addr="address"]
     [log [prefix="prefix text"] [level="log level"]]
     [audit]
     [accept|reject|drop]
```

![](img/554bb2af56ee44846305dcd415309b3d_7.png)

![](img/554bb2af56ee44846305dcd415309b3d_9.png)

以下是管理富规则的主要命令选项：
*   `--add-rich-rule`：添加一条富规则。
*   `--remove-rich-rule`：删除一条富规则。
*   `--query-rich-rule`：查询某条富规则是否存在。
*   `--list-rich-rules`：列出所有富规则。

## 富规则实践：基于源IP的访问控制

![](img/554bb2af56ee44846305dcd415309b3d_19.png)

![](img/554bb2af56ee44846305dcd415309b3d_21.png)

让我们通过一个实验来理解富规则。假设我们要拒绝来自特定IP地址（例如192.168.18.129）的所有访问。

![](img/554bb2af56ee44846305dcd415309b3d_23.png)

![](img/554bb2af56ee44846305dcd415309b3d_25.png)

![](img/554bb2af56ee44846305dcd415309b3d_27.png)

![](img/554bb2af56ee44846305dcd415309b3d_29.png)

首先，我们添加一条拒绝规则：
```bash
firewall-cmd --permanent --zone=public --add-rich-rule='rule family="ipv4" source address="192.168.18.129" reject'
firewall-cmd --reload
```
执行后，来自192.168.18.129的ping或SSH连接都将被拒绝。

![](img/554bb2af56ee44846305dcd415309b3d_31.png)

![](img/554bb2af56ee44846305dcd415309b3d_33.png)

![](img/554bb2af56ee44846305dcd415309b3d_35.png)

![](img/554bb2af56ee44846305dcd415309b3d_37.png)

然后，我们可以删除这条规则：
```bash
firewall-cmd --permanent --zone=public --remove-rich-rule='rule family="ipv4" source address="192.168.18.129" reject'
firewall-cmd --reload
```
删除后，连接恢复。

![](img/554bb2af56ee44846305dcd415309b3d_39.png)

![](img/554bb2af56ee44846305dcd415309b3d_40.png)

![](img/554bb2af56ee44846305dcd415309b3d_42.png)

![](img/554bb2af56ee44846305dcd415309b3d_44.png)

**一个重要结论是：防火墙规则的匹配顺序是先匹配富规则，再匹配基本规则。**

## 富规则实践：连接速率限制

![](img/554bb2af56ee44846305dcd415309b3d_53.png)

富规则还可以用来限制连接速率。例如，限制每分钟到SSH服务的新连接数不超过2个。

![](img/554bb2af56ee44846305dcd415309b3d_55.png)

以下是添加速率限制规则的命令：
```bash
firewall-cmd --permanent --zone=public --add-rich-rule='rule service name="ssh" limit value="2/m" accept'
firewall-cmd --reload
```
这条规则允许SSH服务，但限制每分钟只接受2个新连接。超过此限制的连接将被丢弃。请注意，此限制通常基于客户端IP地址来区分不同连接。

![](img/554bb2af56ee44846305dcd415309b3d_59.png)

## 富规则实践：端口转发

端口转发是富规则的一个重要应用。它可以将到达本机某个端口的流量转发到本机的另一个端口，常用于隐藏真实服务端口。

![](img/554bb2af56ee44846305dcd415309b3d_63.png)

![](img/554bb2af56ee44846305dcd415309b3d_65.png)

![](img/554bb2af56ee44846305dcd415309b3d_67.png)

例如，将到达本机8080/TCP端口的流量转发到80/TCP端口：
```bash
firewall-cmd --permanent --zone=public --add-rich-rule='rule family="ipv4" forward-port port="8080" protocol="tcp" to-port="80"'
firewall-cmd --reload
```
配置后，外部客户端访问本机的8080端口，实际会被转发到80端口进行处理。这提供了一层简单的端口隐藏。

**需要注意的是，如果本机8080端口已有服务监听，则会发生冲突。**

![](img/554bb2af56ee44846305dcd415309b3d_69.png)

## 端口的SELinux安全上下文

![](img/554bb2af56ee44846305dcd415309b3d_71.png)

现在，我们切换话题，看看SELinux如何管理网络端口。之前我们学习过，SELinux为主体（进程）和客体（如文件）打上标签。对于网络通信，端口也是客体，拥有自己的SELinux标签。

使用以下命令可以查看端口的SELinux标签：
```bash
semanage port -l
```
输出中会列出各种服务类型（如`http_port_t`）所允许绑定的TCP/UDP端口范围。

## 端口SELinux标签实践

![](img/554bb2af56ee44846305dcd415309b3d_73.png)

默认情况下，HTTP服务（类型为`http_port_t`）只能绑定到特定端口（如80、443、8080等）。如果尝试将HTTP服务改为监听一个非允许的端口（例如82），服务将无法启动。

首先，查看当前HTTP服务允许的端口：
```bash
semanage port -l | grep http_port_t
```

![](img/554bb2af56ee44846305dcd415309b3d_75.png)

![](img/554bb2af56ee44846305dcd415309b3d_77.png)

假设我们修改Web服务配置，将其监听端口改为82。重启服务时会失败，并提示权限被拒绝，因为SELinux策略禁止`httpd`进程绑定到`http_port_t`类型未包含的端口。

为了解决这个问题，我们需要将82端口添加到`http_port_t`类型的允许端口列表中：
```bash
semanage port -a -t http_port_t -p tcp 82
```
再次查看端口列表，确认82端口已添加。现在，Web服务就可以成功绑定到82端口并启动了。

![](img/554bb2af56ee44846305dcd415309b3d_79.png)

这个机制确保了服务只能运行在策略明确允许的端口上，增强了系统安全性。当然，在生产环境中，如果确定要禁用这种限制，可以直接关闭SELinux，但更推荐的方法是正确修改策略。
