# Linux用户与权限管理：5.1：系统用户UID与passwd文件详解 🔍

在本节课中，我们将要学习Linux系统中用户的核心概念，特别是用户的唯一标识符UID，以及存储用户关键信息的`/etc/passwd`文件。理解这些内容是后续管理用户、组和文件权限的基础。

## 用户与UID概念

上一节我们介绍了本章将学习用户、组和权限。本节中我们来看看用户的基本概念。

![](img/656aa97765fcccf46af5e4c740e0f042_1.png)

Linux是一个多用户操作系统。用户是访问系统资源的身份标识。每个用户有两个关键属性：
*   **用户名**：一个字符串，用于标识用户身份，例如 `root` 或 `user1`。用户名是供使用者识别和登录使用的。
*   **用户ID**：一个唯一的数字标识符，称为 **UID**。系统内部通过UID来识别和区分用户，而不是用户名。

![](img/656aa97765fcccf46af5e4c740e0f042_3.png)

系统管理员`root`的UID固定为 **0**。普通用户的UID默认从 **1000** 开始分配。

![](img/656aa97765fcccf46af5e4c740e0f042_5.png)

![](img/656aa97765fcccf46af5e4c740e0f042_7.png)

![](img/656aa97765fcccf46af5e4c740e0f042_8.png)

用户名和UID通常是一一对应的，但技术上允许两个不同的用户名拥有相同的UID。在这种情况下，系统会认为他们是同一个用户，因为系统只认UID。

## 详解 /etc/passwd 文件

用户账户信息存储在 `/etc/passwd` 文件中。要查看此文件的详细格式说明，可以使用命令 `man 5 passwd`。

![](img/656aa97765fcccf46af5e4c740e0f042_16.png)

![](img/656aa97765fcccf46af5e4c740e0f042_18.png)

![](img/656aa97765fcccf46af5e4c740e0f042_20.png)

![](img/656aa97765fcccf46af5e4c740e0f042_22.png)

`/etc/passwd` 文件中的每一行代表一个用户账户，各字段由冒号 `:` 分隔。以下是每个字段的含义：

![](img/656aa97765fcccf46af5e4c740e0f042_24.png)

![](img/656aa97765fcccf46af5e4c740e0f042_26.png)

以下是`/etc/passwd`文件各字段的详细解释：

![](img/656aa97765fcccf46af5e4c740e0f042_28.png)

![](img/656aa97765fcccf46af5e4c740e0f042_30.png)

1.  **用户名**：用户登录系统时使用的名称。
2.  **密码占位符**：历史上这里存放加密后的密码，现在密码已移至 `/etc/shadow` 文件以提高安全性。此处的 `x` 表示该用户需要密码验证（在某些旧版本系统中，删除`x`可使该用户无需密码登录，但新版本中此行为可能已改变）。
3.  **UID**：用户的数字ID。**0** 代表root用户，**1000** 及以上通常代表普通用户。
4.  **GID**：用户**主组**的数字ID。每个用户必须属于至少一个组，我们将在下一节详细讲解组。
5.  **描述信息**：用户的注释或全名字段。例如，在图形登录界面显示的可能就是这个信息，而非用户名。
6.  **家目录**：用户登录后进入的默认工作目录。通常是 `/home/用户名`。
7.  **登录Shell**：用户登录后启动的命令行解释器。最常见的是 `/bin/bash`，它为用户提供命令行交互界面。

您可以通过直接编辑 `/etc/passwd` 文件（需root权限）来手动添加用户。例如，复制一行现有用户信息，修改用户名、UID、GID和家目录等字段，即可创建一个新的用户条目。

然而，手动修改需要注意同步创建对应的家目录和用户组，否则用户登录或切换时可能会报错。更安全、规范的做法是使用 `useradd` 等用户管理命令，我们将在后续课程中学习。

![](img/656aa97765fcccf46af5e4c740e0f042_48.png)
