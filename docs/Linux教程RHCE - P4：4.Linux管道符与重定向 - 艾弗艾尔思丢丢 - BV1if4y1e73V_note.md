## 重定向：命令与文件的桥梁
上一节我们介绍了基本的Linux命令，本节中我们来看看如何控制命令的输入和输出流向。

![](img/f252fb8f87c2a92229ff66f626d0800c_11.png)

重定向操作符可以将命令的输出信息导入到一个文件中，或者将文件的内容导回给命令作为输入。这个过程实现了命令与文件之间的信息流交换。

![](img/f252fb8f87c2a92229ff66f626d0800c_13.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_15.png)

### 输入与输出重定向
重定向主要分为两类：
*   **输入重定向**：将文件的内容作为命令的输入。
*   **输出重定向**：将命令的输出结果写入到文件。

对于输出重定向，它又细分为两种模式：
*   **标准输出**：命令执行后产生的正常、预期的信息。
*   **错误输出**：命令执行过程中产生的错误或警告信息。

![](img/f252fb8f87c2a92229ff66f626d0800c_17.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_18.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_20.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_22.png)

此外，写入文件时有两种模式：
*   **覆盖写入**：清空文件原有内容，再写入新内容。
*   **追加写入**：在文件原有内容的末尾，添加新内容。

### 操作符详解与实验
以下是核心操作符及其含义：

![](img/f252fb8f87c2a92229ff66f626d0800c_24.png)

*   **`>`**：将**标准输出**以**覆盖**模式写入文件。
    *   命令格式：`命令 > 文件`
*   **`>>`**：将**标准输出**以**追加**模式写入文件。
    *   命令格式：`命令 >> 文件`
*   **`2>`**：将**错误输出**以**覆盖**模式写入文件。
    *   命令格式：`命令 2> 文件`
*   **`2>>`**：将**错误输出**以**追加**模式写入文件。
    *   命令格式：`命令 2>> 文件`
*   **`&>`**：将**标准输出和错误输出**全部以**覆盖**模式写入文件。
    *   命令格式：`命令 &> 文件`
*   **`<`**：将文件内容作为命令的**输入**。
    *   命令格式：`命令 < 文件`

## 管道符：命令之间的任意门
上一节我们学会了如何连接命令和文件，本节中我们来看看如何连接命令与命令。

![](img/f252fb8f87c2a92229ff66f626d0800c_66.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_67.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_69.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_70.png)

管道符 `|` 像一个“任意门”，它能够将前一个命令（命令A）的**标准输出**，直接传递给后一个命令（命令B）作为**输入**进行处理。

![](img/f252fb8f87c2a92229ff66f626d0800c_72.png)

### 管道符的作用
其核心作用是：**命令A | 命令B**
将命令A的输出结果，交给命令B进行二次处理。

![](img/f252fb8f87c2a92229ff66f626d0800c_74.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_76.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_78.png)

**实验5：使用管道进行过滤和统计**
```bash
# 1. 查看系统所有用户
cat /etc/passwd

# 2. 使用管道和 grep 过滤出可以登录系统的用户（shell为/bin/bash）
cat /etc/passwd | grep /bin/bash

![](img/f252fb8f87c2a92229ff66f626d0800c_80.png)

# 3. 在上一步基础上，再用管道和 wc -l 统计可以登录的用户数量
cat /etc/passwd | grep /bin/bash | wc -l

# 4. 统计 /etc 目录下有多少个文件
ls /etc | wc -l
```

![](img/f252fb8f87c2a92229ff66f626d0800c_82.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_83.png)

**实验6：管道符在命令参数中的应用**
```bash
# echo 命令输出一个字符串，通过管道传递给 passwd 命令的 --stdin 参数，为用户 root 设置密码
echo "NewPassword123" | passwd --stdin root
# 注意：--stdin 参数表示从标准输入读取密码，这里标准输入的内容来自管道。
```

## 通配符：模糊匹配的利器
当我们需要操作多个具有相似名称的文件，但又记不住完整名字时，通配符就派上用场了。

![](img/f252fb8f87c2a92229ff66f626d0800c_85.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_87.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_88.png)

![](img/f252fb8f87c2a92229ff66f626d0800c_89.png)

以下是常用的通配符：
*   **`*`**：匹配任意多个字符（包括0个字符）。
    *   示例：`ls /dev/sda*` 匹配所有以 `sda` 开头的文件。
*   **`?`**：匹配任意一个字符。
    *   示例：`ls /dev/sda?` 匹配 `sda1`, `sda2`，但不匹配 `sda` 本身。
*   **`[ ]`**：匹配括号内的任意一个字符。
    *   示例：`ls /dev/sda[0-9]` 匹配 `sda0` 到 `sda9`。
    *   示例：`ls /dev/sda[135]` 只匹配 `sda1`, `sda3`, `sda5`。
*   **`{ }`**：生成一个序列，进行精确匹配。
    *   示例：`ls /dev/sda{1,3,5}` 只匹配 `sda1`, `sda3`, `sda5`。如果文件不存在，会报错。

## 转义符：让特殊字符变普通
命令行中有许多具有特殊功能的字符（如 `$`, `*`, `>` 等）。如果我们想将它们作为普通字符输出，就需要使用转义符。

*   **`\`**（反斜杠）：转义紧随其后的一个字符。
    *   示例：`echo "价格是 \$5"` 会输出 `价格是 $5`，否则 `$` 会被当作变量引用符。
*   **`‘ ‘`**（单引号）：强引用，引号内所有字符都失去特殊含义，变为普通字符。
    *   示例：`echo ‘价格是 $5’` 会原样输出 `价格是 $5`。
*   **`“ ”`**（双引号）：弱引用，引号内大部分字符失去特殊含义，但 `$`（变量引用）、`` ` ``（命令替换）和 `\`（转义）依然有效。
    *   示例：`echo “当前用户是 $USER”` 会输出 `当前用户是 root`。
*   **`` ` ` ``（反引号）：命令替换，执行反引号内的命令，并将其输出结果替换到当前位置。
    *   示例：`current_time=\`date\`` 将 `date` 命令的输出结果赋值给变量 `current_time`。

## 环境变量：系统的全局设置
环境变量是操作系统或Shell中预先设置好的变量，用于存储系统信息、配置程序行为等。它们通常全局有效。

### 常用环境变量
*   **`PATH`**：定义系统查找命令的路径。当输入一个命令时，系统会按照 `PATH` 中的路径顺序去寻找对应的可执行文件。
    *   查看：`echo $PATH`
*   **`HOME`**：当前用户的家目录路径。
    *   查看：`echo $HOME`
*   **`SHELL`**：当前使用的Shell解释器路径。
    *   查看：`echo $SHELL`
*   **`LANG`**：当前系统的语言和字符集设置。
    *   查看：`echo $LANG`
*   **`USER`**：当前登录的用户名。
    *   查看：`echo $USER`
*   **`RANDOM`**：每次引用都会生成一个随机整数。
    *   查看：`echo $RANDOM`

### 自定义与提升变量作用域
```bash
# 1. 定义一个普通变量（局部变量）
workdir=/etc/nginx
echo $workdir

# 2. 使用变量
cd $workdir

# 3. 切换到其他用户（如 testuser），该变量不可用
su - testuser
echo $workdir # 输出为空

![](img/f252fb8f87c2a92229ff66f626d0800c_98.png)

# 4. 返回原用户，将局部变量提升为环境变量（全局变量）
exit
export workdir

![](img/f252fb8f87c2a92229ff66f626d0800c_100.png)

# 5. 再次切换到 testuser，变量可用了
su - testuser
echo $workdir
```
**注意**：通过 `export` 提升的变量只在当前Shell会话及其子进程中有效。若要永久生效，需要将 `export 变量名=值` 写入Shell的配置文件（如 `~/.bashrc` 或 `/etc/profile`）。

## 总结
本节课中我们一起学习了Linux中提高效率的核心工具：
1.  **重定向**：掌握了如何用 `>`, `>>`, `2>`, `<` 等操作符控制命令的输入输出流，实现命令与文件的交互。
2.  **管道符**：理解了 `|` 如何像“任意门”一样，将多个命令串联起来，让前一个命令的输出成为后一个命令的输入。
3.  **通配符**：学会了使用 `*`, `?`, `[ ]`, `{ }` 来快速匹配和操作多个文件。
4.  **转义符**：知道了如何用 `\`, `‘ ‘`, `“ ”` 来处理命令行中的特殊字符。
5.  **环境变量**：认识了 `PATH`, `HOME` 等关键环境变量的作用，并学会了如何定义和提升自定义变量的作用域。
