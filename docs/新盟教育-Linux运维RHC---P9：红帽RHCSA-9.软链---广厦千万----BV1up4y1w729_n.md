## 软链接与硬链接 🔗

上一节我们介绍了文件和目录的基本操作，本节中我们来看看Linux系统中的链接文件，它类似于Windows系统中的快捷方式。

![](img/83268310834077f3a00a01d2e7eb11e3_29.png)

![](img/83268310834077f3a00a01d2e7eb11e3_31.png)

### 什么是链接文件？
链接文件是Linux中一种特殊的文件类型，它指向另一个文件或目录。主要分为软链接（符号链接）和硬链接两种。

![](img/83268310834077f3a00a01d2e7eb11e3_33.png)

![](img/83268310834077f3a00a01d2e7eb11e3_35.png)

![](img/83268310834077f3a00a01d2e7eb11e3_36.png)

### 软链接 (Symbolic Link)
软链接类似于Windows的快捷方式。它是一个独立的文件，其中存储的是目标文件的路径信息。

**软链接的特点如下：**
*   **可以跨分区**：源文件和链接文件可以位于不同的磁盘分区。
*   **可以对目录创建链接**：可以为目录创建软链接。
*   **依赖源文件**：如果删除源文件，软链接将失效（变成“坏链接”）。
*   **权限显示不同**：软链接自身的权限通常是`rwxrwxrwx`，但其有效权限取决于源文件的权限。

**创建软链接的命令格式为：**
```bash
ln -s <源文件路径> <链接文件路径>
```

![](img/83268310834077f3a00a01d2e7eb11e3_38.png)

![](img/83268310834077f3a00a01d2e7eb11e3_40.png)

**以下是创建软链接时的注意事项：**
*   创建链接时，**必须使用绝对路径**指定源文件，否则链接可能失效。
*   删除软链接不会影响源文件。
*   修改软链接或源文件的内容，另一方会同步更新。

### 硬链接 (Hard Link)
硬链接可以理解为给同一个文件数据块起了另一个名字。它与源文件共享相同的`inode`编号和数据块。

**硬链接的特点如下：**
*   **不能跨分区**：硬链接和源文件必须在同一个文件系统（分区）内。
*   **不能对目录创建**：只能对文件创建硬链接。
*   **不依赖源文件**：删除源文件后，硬链接文件仍然可以正常访问，因为数据块并未被释放。
*   **与源文件地位平等**：硬链接和源文件在本质上没有区别，可以看作文件的另一个“入口”。

**创建硬链接的命令格式为：**
```bash
ln <源文件路径> <链接文件路径>
```

![](img/83268310834077f3a00a01d2e7eb11e3_42.png)

**硬链接的实质**类似于执行了一次保持属性的拷贝命令`cp -p`，但不同的是，硬链接与源文件会保持数据同步更新。

![](img/83268310834077f3a00a01d2e7eb11e3_44.png)

## 获取命令帮助 📖

![](img/83268310834077f3a00a01d2e7eb11e3_46.png)

![](img/83268310834077f3a00a01d2e7eb11e3_47.png)

上一节我们学习了链接文件，本节中我们来看看当不熟悉某个命令时，如何获取它的使用帮助。

在Linux中，命令主要分为内部命令（Shell内置）和外部命令（独立程序）。对于用户而言，更重要的是掌握如何查询命令的用法。

**以下是几种获取命令帮助的方法：**

1.  **`--help` 选项**
    大多数外部命令都支持`--help`选项，它能输出该命令的简要用法和选项说明。
    ```bash
    ls --help
    ```

2.  **`man` 手册**
    `man`命令提供更详细、更正式的命令手册，包括语法、选项、描述、示例等。
    ```bash
    man ls
    ```
    在`man`手册页面中，可以使用快捷键导航，例如空格键翻页，`/`键搜索，`q`键退出。

![](img/83268310834077f3a00a01d2e7eb11e3_50.png)

3.  **`info` 手册**
    `info`是另一种格式的帮助文档，通常比`man`手册更详细，结构更复杂。
    ```bash
    info ls
    ```

4.  **`help` 命令**
    主要用于查看Shell内部命令的帮助。
    ```bash
    help cd
    ```

**建议**：对于初学者，如果英文手册阅读困难，可以优先使用`--help`获取核心选项，或借助互联网搜索引擎（如百度）查询中文教程和示例，这通常更易于理解。

## 系统运行级别 🏃

了解了如何获取帮助后，我们来看看Linux系统的运行级别。运行级别定义了系统启动后处于何种工作状态。

![](img/83268310834077f3a00a01d2e7eb11e3_54.png)

Linux系统通常有7个运行级别（0-6），每个级别对应不同的服务和功能组合。

![](img/83268310834077f3a00a01d2e7eb11e3_56.png)

**以下是各运行级别的含义：**

![](img/83268310834077f3a00a01d2e7eb11e3_58.png)

![](img/83268310834077f3a00a01d2e7eb11e3_60.png)

![](img/83268310834077f3a00a01d2e7eb11e3_62.png)

*   **0**: 关机模式。
*   **1**: 单用户模式。仅`root`用户可登录，用于系统维护（如忘记密码）。大多数服务不启动。
*   **2**: 多用户模式（无NFS）。允许多用户登录，但不支持网络文件系统（NFS）。
*   **3**: 完全多用户模式（文本模式）。**标准的服务器运行级别**，提供完整的命令行界面。
*   **4**: 未使用，保留。
*   **5**: 图形界面模式。在级别3的基础上启动了图形化登录界面。
*   **6**: 重启模式。

![](img/83268310834077f3a00a01d2e7eb11e3_64.png)

![](img/83268310834077f3a00a01d2e7eb11e3_66.png)

**查看当前运行级别：**
```bash
runlevel
```
输出类似 `N 3`，表示之前级别为N（N表示开机后直接进入），当前级别为3。

![](img/83268310834077f3a00a01d2e7eb11e3_68.png)

![](img/83268310834077f3a00a01d2e7eb11e3_70.png)

**临时切换运行级别：**
```bash
init <级别号>
```
例如，`init 6`会重启系统。

![](img/83268310834077f3a00a01d2e7eb11e3_72.png)

**查看和设置默认运行级别：**
*   查看默认级别：
    ```bash
    systemctl get-default
    ```
*   设置默认级别（例如设为图形界面）：
    ```bash
    systemctl set-default graphical.target
    ```
*   设置默认级别（例如设为文本模式）：
    ```bash
    systemctl set-default multi-user.target
    ```
    系统的默认级别配置文件位于`/etc/inittab`（在较新系统中，该文件可能只起兼容作用，实际由`systemd`管理）。

![](img/83268310834077f3a00a01d2e7eb11e3_74.png)

![](img/83268310834077f3a00a01d2e7eb11e3_76.png)

**注意**：如果系统安装时未选择图形界面，则即使切换到级别5，也不会出现图形界面。

## 关机与重启 ⏻

![](img/83268310834077f3a00a01d2e7eb11e3_92.png)

最后，我们来学习如何在命令行下安全地关闭或重启Linux系统。

![](img/83268310834077f3a00a01d2e7eb11e3_94.png)

![](img/83268310834077f3a00a01d2e7eb11e3_96.png)

![](img/83268310834077f3a00a01d2e7eb11e3_98.png)

**关机命令主要有以下几种：**

![](img/83268310834077f3a00a01d2e7eb11e3_100.png)

*   `poweroff`：**最常用**，立即关闭系统。
*   `init 0`：切换到运行级别0（关机）。
*   `shutdown -h now`：立即关机。
*   `shutdown -h +10`：10分钟后关机，并向所有登录用户发送警告信息。

![](img/83268310834077f3a00a01d2e7eb11e3_102.png)

![](img/83268310834077f3a00a01d2e7eb11e3_104.png)

![](img/83268310834077f3a00a01d2e7eb11e3_106.png)

![](img/83268310834077f3a00a01d2e7eb11e3_108.png)

![](img/83268310834077f3a00a01d2e7eb11e3_110.png)

**重启命令主要有以下几种：**

*   `reboot`：**最常用**，立即重启系统。
*   `init 6`：切换到运行级别6（重启）。
*   `shutdown -r now`：立即重启。

![](img/83268310834077f3a00a01d2e7eb11e3_112.png)

**建议**：在生产环境中，使用`shutdown`命令可以更友好地安排关机或重启时间，并通知用户。在个人学习或测试中，使用`poweroff`和`reboot`更为直接便捷。

![](img/83268310834077f3a00a01d2e7eb11e3_114.png)

![](img/83268310834077f3a00a01d2e7eb11e3_116.png)
