## 程序、进程与线程的概念

![](img/fed9e32a958eb04bd6321fc925bca91f_13.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_15.png)

上一节我们结束了实操内容，本节我们来学习一些重要的理论概念。理解这些概念是进行系统管理和故障排查的基础。

![](img/fed9e32a958eb04bd6321fc925bca91f_17.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_18.png)

### 什么是程序？
用计算机语言编写的命令序列的集合，用来实现特定的目标或解决特定的问题。简单来说，程序就是安装在计算机硬盘上的软件。例如，从应用商店下载的聊天工具，在安装到电脑后，就成为了一个应用程序。

![](img/fed9e32a958eb04bd6321fc925bca91f_20.png)

*   **程序占用磁盘空间**。
*   **程序是静态且永久的**。在未运行时，它不会占用内存空间。

![](img/fed9e32a958eb04bd6321fc925bca91f_22.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_23.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_25.png)

### 什么是进程？
进程是正在运行中的程序。当程序被加载到内存并开始执行时，它就变成了一个进程。

![](img/fed9e32a958eb04bd6321fc925bca91f_27.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_29.png)

*   **进程占用内存空间**。
*   **进程是动态的**，其状态会不断变化。
*   **进程具有生命周期**。有些进程执行完毕会自动结束（如 `ls` 命令），有些则需要用户手动结束（如一个持续运行的应用）。
*   每个进程启动时，系统会为其分配独立的内存空间和一个唯一的进程标识符 **PID**。
*   每个进程都有一个父进程（PPID），父进程可以创建多个子进程，形成父子关系。
*   进程可以以前台或后台方式运行，大多数系统进程在后台运行。

![](img/fed9e32a958eb04bd6321fc925bca91f_31.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_33.png)

### 什么是线程？
线程，也称为轻量级进程，是进程中的一个执行单元，是进程中实际运作的单位。一个进程可以并发多个线程。

![](img/fed9e32a958eb04bd6321fc925bca91f_35.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_37.png)

我们可以用一个比喻来理解：
*   将 **CPU** 比作一个**大工厂**。
*   将 **CPU核心** 比作工厂里的**生产车间**（例如，四核CPU有四个车间）。
*   将 **线程** 比作每个车间里**真正干活的员工**（例如，每个核心有两个线程，意味着每个车间有两名员工）。

对于运维工作而言，只需理解线程是进程内部真正负责执行任务的单元即可。

## 查看进程树：`pstree` 命令

![](img/fed9e32a958eb04bd6321fc925bca91f_39.png)

了解了基本概念后，我们来看看如何查看系统中的进程。首先介绍以树状结构显示进程关系的 `pstree` 命令。

`pstree` 命令以树状结构显示进程信息，清晰展示进程间的父子关系。

以下是 `pstree` 命令的一些常用选项示例：

**1. 基本使用**
直接运行 `pstree` 会显示系统的进程树。树的根通常是 `systemd`（所有进程的父进程，PID为1）。

```bash
pstree
```

**2. 显示进程PID**
使用 `-p` 选项可以显示每个进程的PID。

```bash
pstree -p
```

**3. 显示完整命令行与指定用户进程**
使用 `-a` 选项显示完整命令行。结合用户名，可以查看特定用户运行的进程。

![](img/fed9e32a958eb04bd6321fc925bca91f_41.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_43.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_45.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_46.png)

```bash
# 查看用户 ‘test’ 运行了哪些进程及其完整命令
pstree -a test
```

**4. 显示进程所属用户**
使用 `-u` 选项可以列出每个进程所属的用户名。

![](img/fed9e32a958eb04bd6321fc925bca91f_48.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_50.png)

```bash
pstree -u
```

![](img/fed9e32a958eb04bd6321fc925bca91f_52.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_54.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_56.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_57.png)

**关键点说明**：
*   在树状结构中，可以直观看到父子进程关系。结束父进程通常会同时结束其所有子进程。
*   `systemd` 是系统的初始化进程（PID 1），负责启动和管理其他所有进程。
*   输出中类似 `6*` 的标识，表示该进程下有6个线程。

![](img/fed9e32a958eb04bd6321fc925bca91f_60.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_62.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_64.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_66.png)

除了树状视图，我们更常用 `ps` 命令来获取系统进程的快照信息。`ps` 命令功能强大，选项组合多样。

![](img/fed9e32a958eb04bd6321fc925bca91f_67.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_69.png)

### 1. `ps aux` 命令详解
`ps aux` 是常用的组合，用于查看系统所有进程的详细信息。

```bash
ps aux
```

![](img/fed9e32a958eb04bd6321fc925bca91f_71.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_73.png)

以下是 `ps aux` 输出中各列的含义：

![](img/fed9e32a958eb04bd6321fc925bca91f_75.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_77.png)

*   **USER**: 进程属于哪个用户。
*   **PID**: 进程的ID号。
*   **%CPU**: 进程占用CPU的百分比。
*   **%MEM**: 进程占用物理内存的百分比。
*   **VSZ**: 进程使用的虚拟内存总量（单位KB）。
*   **RSS**: 进程占用的固定内存总量（单位KB）。
*   **TTY**: 进程在哪个终端运行。
    *   `?` 表示由内核直接调用，无关联终端。
    *   `ttyN` 表示在本机终端登录的用户进程。
    *   `pts/N` 表示在远程终端登录的用户进程。
*   **STAT**: 进程状态。常见状态有：
    *   **R**: 正在运行或可运行。
    *   **S**: 休眠状态。
    *   **T**: 停止状态。
    *   **Z**: 僵尸进程。
    *   **+**: 位于后台的进程组。
*   **START**: 进程启动时间。
*   **TIME**: 进程占用CPU的总时间。
*   **COMMAND**: 启动进程的命令行。

**关于僵尸进程**：僵尸进程是已终止但其父进程未能成功回收其资源的进程。通常需要结束其父进程来清除僵尸进程。

### 2. `ps -ef` 命令
这是另一种风格的命令，同样用于查看所有进程，其输出格式略有不同。

```bash
ps -ef
# 或使用长格式
ps -elf
```

![](img/fed9e32a958eb04bd6321fc925bca91f_79.png)

`ps -elf` 输出中多了一个 **PPID** 列，即父进程的PID。这在排查僵尸进程时非常有用，因为我们需要根据PPID找到并处理其父进程。

![](img/fed9e32a958eb04bd6321fc925bca91f_81.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_83.png)

### 3. 常用过滤技巧
`ps` 命令常与 `grep` 结合，用于查找特定进程。

![](img/fed9e32a958eb04bd6321fc925bca91f_85.png)

```bash
# 查找与 ‘sshd’ 相关的所有进程
ps aux | grep sshd
```

![](img/fed9e32a958eb04bd6321fc925bca91f_87.png)

查找僵尸进程及其父进程的命令（如果系统存在僵尸进程）：

![](img/fed9e32a958eb04bd6321fc925bca91f_89.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_91.png)

```bash
ps -eo pid,ppid,stat,cmd | grep -E ‘Z|defunct’
```

![](img/fed9e32a958eb04bd6321fc925bca91f_93.png)

![](img/fed9e32a958eb04bd6321fc925bca91f_95.png)

## 总结
本节课我们一起学习了程序、进程和线程的核心概念及其区别。我们掌握了使用 `pstree` 命令以树状结构查看进程关系，以及使用 `ps aux` 和 `ps -ef` 命令查看详细的进程快照信息。理解这些命令的输出列含义，特别是进程状态（STAT）和父子进程关系（PID/PPID），是进行系统监控和问题诊断的重要基础。下节课我们将学习功能更强大的动态进程监控工具——`top` 命令。
