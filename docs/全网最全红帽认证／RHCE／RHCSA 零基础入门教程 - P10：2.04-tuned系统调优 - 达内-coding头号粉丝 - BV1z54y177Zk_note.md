在本节课中，我们将学习红帽认证考试中的一个重要考点：使用 `tuned` 服务进行系统调优。这个考点看似复杂，但实际上非常简单，核心是学会使用红帽预设的优化方案。

## 概述：什么是系统调优？

![](img/e60d59e07682e0244a70e62145b4ec0e_7.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_9.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_11.png)

系统调优，英文为 `tune`，其目的是调整Linux系统的内核参数，以优化其在特定应用场景下的性能。例如，作为数据库服务器、网络服务器或虚拟机时，所需的优化侧重点各不相同。

![](img/e60d59e07682e0244a70e62145b4ec0e_13.png)

传统的手动调优方法需要修改 `/proc` 目录下的内核参数（临时生效）或编辑 `/etc/sysctl.conf` 文件（永久生效），过程繁琐且容易出错。

在红帽7和红帽8系统中，红帽公司提供了一个名为 `tuned` 的服务，它预置了大量针对不同场景的优化方案（称为“配置集”或“profile”）。我们只需选择合适的方案即可，极大地简化了调优工作。

## tuned服务的核心操作

`tuned` 服务在红帽系统上通常默认已安装并启用。我们无需关心复杂的参数细节，只需掌握核心管理工具 `tuned-adm` 的几个命令即可。

![](img/e60d59e07682e0244a70e62145b4ec0e_15.png)

以下是 `tuned-adm` 最常用的几个指令：

![](img/e60d59e07682e0244a70e62145b4ec0e_17.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_19.png)

*   **`list`**：列出所有可用的优化方案。
*   **`active`**：查看当前正在使用的优化方案。
*   **`recommend`**：获取系统推荐的优化方案。
*   **`profile`**：切换到指定的优化方案。

![](img/e60d59e07682e0244a70e62145b4ec0e_21.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_23.png)

### 1. 列出可用方案

![](img/e60d59e07682e0244a70e62145b4ec0e_25.png)

要查看 `tuned` 提供了哪些预设方案，可以使用 `list` 命令。

![](img/e60d59e07682e0244a70e62145b4ec0e_27.png)

```bash
tuned-adm list
```

![](img/e60d59e07682e0244a70e62145b4ec0e_29.png)

执行后会显示一个列表，例如 `balanced`（平衡模式）、`throughput-performance`（吞吐性能模式）、`virtual-guest`（虚拟机客户机模式）等，每种方案都有简要说明。

![](img/e60d59e07682e0244a70e62145b4ec0e_31.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_33.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_35.png)

### 2. 查看系统推荐方案

![](img/e60d59e07682e0244a70e62145b4ec0e_37.png)

考试题目通常要求“为系统选择建议的调优配置集”。我们可以使用 `recommend` 命令来获取红帽根据当前系统环境（如检测到是虚拟机）给出的推荐方案。

![](img/e60d59e07682e0244a70e62145b4ec0e_39.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_41.png)

```bash
tuned-adm recommend
```
命令会输出一个方案名称，例如 `virtual-guest`。

![](img/e60d59e07682e0244a70e62145b4ec0e_43.png)

### 3. 查看当前活动方案

![](img/e60d59e07682e0244a70e62145b4ec0e_45.png)

要确认系统当前正在使用哪个优化方案，使用 `active` 命令。

![](img/e60d59e07682e0244a70e62145b4ec0e_47.png)

```bash
tuned-adm active
```

![](img/e60d59e07682e0244a70e62145b4ec0e_49.png)

### 4. 切换优化方案

![](img/e60d59e07682e0244a70e62145b4ec0e_51.png)

如果推荐方案与当前活动方案不一致，我们需要使用 `profile` 命令进行切换。

![](img/e60d59e07682e0244a70e62145b4ec0e_53.png)

```bash
tuned-adm profile [方案名称]
```
例如，要切换到 `virtual-guest` 方案：
```bash
tuned-adm profile virtual-guest
```

## 考试实战步骤与总结

![](img/e60d59e07682e0244a70e62145b4ec0e_55.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_56.png)

上一节我们介绍了 `tuned-adm` 的核心命令，本节中我们来看看在RHCSA考试中如何应用这些知识。

![](img/e60d59e07682e0244a70e62145b4ec0e_57.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_58.png)

考试时，针对 `tuned` 的题目通常按以下步骤操作：

1.  **验证服务状态**：通常 `tuned` 服务默认已安装并运行。如需确认，可使用 `systemctl status tuned` 查看。
2.  **获取推荐方案**：运行 `tuned-adm recommend`，记下输出的方案名称。
3.  **检查当前方案**：运行 `tuned-adm active`，查看当前使用的方案。
4.  **应用推荐方案**：如果两个命令的结果不一致，则使用 `tuned-adm profile [推荐方案名]` 进行切换。如果一致，则无需任何操作。

![](img/e60d59e07682e0244a70e62145b4ec0e_60.png)

**请注意**：在练习环境中，系统很可能已经自动应用了正确的方案（如 `virtual-guest`）。因此，这道题有很高概率是“零操作”的送分题，但你必须通过上述命令验证并知晓整个过程。

![](img/e60d59e07682e0244a70e62145b4ec0e_62.png)

![](img/e60d59e07682e0244a70e62145b4ec0e_64.png)

本节课中我们一起学习了如何使用红帽的 `tuned` 服务进行系统调优。我们了解到，调优的本质是选择合适的预设方案，而无需手动修改复杂的内核参数。你只需要掌握 `tuned-adm list`、`recommend`、`active` 和 `profile` 这四个关键命令，就能轻松应对相关考题。记住，验证是关键，确保系统当前使用的方案与推荐方案一致即可。
