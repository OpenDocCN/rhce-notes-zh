## 登录 Shell 与非登录 Shell 的定义

![](img/958b1e6f67aa2c70398f48389d79769e_11.png)

上一节我们介绍了环境变量的概念，本节中我们来看看 Shell 的两种不同启动模式。

简单来说，**登录 Shell** 是指用户需要输入用户名和密码进行身份验证后才能进入的 Shell 会话。而**非登录 Shell** 则是在已经登录的会话中直接启动的 Shell，无需再次验证身份。

![](img/958b1e6f67aa2c70398f48389d79769e_13.png)

![](img/958b1e6f67aa2c70398f48389d79769e_15.png)

以下是登录 Shell 的几种典型情况：

![](img/958b1e6f67aa2c70398f48389d79769e_17.png)

![](img/958b1e6f67aa2c70398f48389d79769e_19.png)

*   通过图形界面登录系统后，首次进入的桌面环境。
*   在字符界面（如 tty1）直接输入用户名和密码登录。
*   使用 `su - <用户名>` 命令切换用户（注意命令中的短横线 `-`）。

以下是非登录 Shell 的几种典型情况：

![](img/958b1e6f67aa2c70398f48389d79769e_21.png)

![](img/958b1e6f67aa2c70398f48389d79769e_23.png)

*   在图形界面中打开的终端窗口（如 GNOME Terminal）。
*   使用 `su <用户名>` 命令切换用户（**不**带短横线 `-`）。
*   在已存在的 Shell 中执行脚本时启动的子 Shell。

这两种 Shell 的核心区别在于：登录 Shell 会“完全变身”为目标用户，加载该用户完整的环境配置；而非登录 Shell 则可能“半变身”，会继承一部分父 Shell 的环境属性。

![](img/958b1e6f67aa2c70398f48389d79769e_25.png)

## 配置文件读取机制

理解了 Shell 的类型，接下来我们看看它们如何读取配置文件。Shell 通过读取特定的配置文件来设置环境变量、别名等。

![](img/958b1e6f67aa2c70398f48389d79769e_27.png)

![](img/958b1e6f67aa2c70398f48389d79769e_29.png)

关键的配置文件有四个：
1.  `/etc/profile` - 系统全局配置文件。
2.  `~/.bash_profile` - 用户个人配置文件。
3.  `/etc/bashrc` - 系统全局的 Bash 配置文件。
4.  `~/.bashrc` - 用户个人的 Bash 配置文件。

![](img/958b1e6f67aa2c70398f48389d79769e_31.png)

**登录 Shell** 会按顺序读取以下所有四个文件：
`/etc/profile` -> `~/.bash_profile` -> `~/.bashrc` -> `/etc/bashrc`

![](img/958b1e6f67aa2c70398f48389d79769e_33.png)

![](img/958b1e6f67aa2c70398f48389d79769e_35.png)

![](img/958b1e6f67aa2c70398f48389d79769e_37.png)

**非登录 Shell** 通常只读取以下两个文件：
`~/.bashrc` -> `/etc/bashrc`

![](img/958b1e6f67aa2c70398f48389d79769e_39.png)

![](img/958b1e6f67aa2c70398f48389d79769e_41.png)

> **注意**：`~/.bashrc` 文件通常会显式地调用 `/etc/bashrc`，以确保系统全局配置生效。

![](img/958b1e6f67aa2c70398f48389d79769e_43.png)

为了更直观地理解，我们通过一个实验来观察不同 Shell 如何读取变量。

![](img/958b1e6f67aa2c70398f48389d79769e_51.png)

![](img/958b1e6f67aa2c70398f48389d79769e_53.png)

假设我们在四个配置文件中分别定义变量：
*   `/etc/profile` 中定义：`A=100`
*   `~/.bash_profile` 中定义：`B=200`
*   `/etc/bashrc` 中定义：`C=300`
*   `~/.bashrc` 中定义：`D=400`

![](img/958b1e6f67aa2c70398f48389d79769e_55.png)

以下是不同场景下的读取结果：

![](img/958b1e6f67aa2c70398f48389d79769e_57.png)

1.  **使用 `su - admin`（登录 Shell）切换到 admin 用户**：
    由于是登录 Shell，它会读取全部四个文件。因此，变量 `A`, `B`, `C`, `D` 都将存在。

2.  **使用 `su admin`（非登录 Shell）切换到 admin 用户**：
    由于是非登录 Shell，它只读取 `~/.bashrc` 和 `/etc/bashrc`。因此，只有变量 `C` 和 `D` 存在（`A` 和 `B` 在未读取的文件中）。

![](img/958b1e6f67aa2c70398f48389d79769e_59.png)

3.  **在图形界面新打开一个终端（非登录 Shell）**：
    同样是非登录 Shell，只读取 `~/.bashrc` 和 `/etc/bashrc`。因此，只有变量 `C` 和 `D` 存在。

![](img/958b1e6f67aa2c70398f48389d79769e_61.png)

![](img/958b1e6f67aa2c70398f48389d79769e_62.png)

![](img/958b1e6f67aa2c70398f48389d79769e_64.png)

这个实验清晰地展示了登录与非登录 Shell 在读取配置文件范围上的根本差异。

## 环境变量的继承

你可能会问：为什么在图形界面新打开的终端里，虽然没有读取 `/etc/profile`，但仍然能使用像 `PATH` 这样的系统环境变量呢？

![](img/958b1e6f67aa2c70398f48389d79769e_66.png)

这是因为环境变量具有**继承性**。当用户通过图形界面登录（这是一个登录 Shell）时，`/etc/profile` 等文件中的环境变量已经被加载并设置好了。随后在这个会话中打开的任何终端（非登录 Shell），作为子进程，都会自动继承其父进程（即登录会话）的环境变量。所以，`PATH` 等变量得以传递下来。

![](img/958b1e6f67aa2c70398f48389d79769e_68.png)

## 配置建议与应用

了解了原理，我们可以给出清晰的配置建议：

![](img/958b1e6f67aa2c70398f48389d79769e_70.png)

*   **设置全局环境变量**（对所有用户生效）：应编辑 `/etc/profile` 文件。
*   **设置用户个人环境变量**：应编辑 `~/.bash_profile` 文件。
*   **设置命令别名**：
    *   若要为所有用户设置别名，应编辑 `/etc/bashrc` 文件。
    *   若只为当前用户设置别名，应编辑 `~/.bashrc` 文件。

遵循这些规则，可以确保你的配置在正确的 Shell 类型下生效，避免出现“配置了却不生效”的困惑。
