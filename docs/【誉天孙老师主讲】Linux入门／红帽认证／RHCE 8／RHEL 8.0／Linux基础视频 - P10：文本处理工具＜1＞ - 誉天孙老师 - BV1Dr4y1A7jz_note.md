## 文本内容查看工具

![](img/0abb4089125f351630d8b78b399b8608_11.png)

![](img/0abb4089125f351630d8b78b399b8608_13.png)

上一节我们介绍了文本处理的重要性，本节中我们来看看如何查看文件内容。以下是三个基础的查看工具。

![](img/0abb4089125f351630d8b78b399b8608_15.png)

![](img/0abb4089125f351630d8b78b399b8608_17.png)

*   **`cat`**：用于连接文件并打印到标准输出设备。常用选项 `-n` 可以为输出的行添加行号。
    *   **查看单个文件**：`cat /etc/passwd`
    *   **查看多个文件**：`cat /etc/passwd /etc/profile`
    *   **添加行号**：`cat -n /etc/passwd`
*   **`more`**：一个基础的分页查看工具，适用于一次性从头到尾阅读。使用空格键翻页，回车键向下滚动一行，浏览完毕后自动退出。
    *   **基本使用**：`more /etc/passwd`
*   **`less`**：一个功能更强大的分页查看工具，可以前后自由翻看。它也是 `man` 命令的默认分页器。
    *   **基本使用**：`less /etc/passwd`
    *   **搜索关键词**：在 `less` 界面中，输入 `/` 后跟关键词（如 `/user`）进行搜索，按 `n` 跳转到下一个匹配项，按 `N` 跳转到上一个匹配项。
    *   **编辑文件**：在 `less` 界面中，按 `v` 键会调用 `vi` 编辑器打开当前文件，进行编辑后保存退出，会返回 `less` 视图。
    *   **退出**：按 `q` 键退出 `less`。

![](img/0abb4089125f351630d8b78b399b8608_19.png)

**过渡**：学会了如何查看整个文件后，我们常常只需要关注文件的特定部分，比如开头或结尾的若干行。

## 文件头部与尾部内容提取

![](img/0abb4089125f351630d8b78b399b8608_21.png)

![](img/0abb4089125f351630d8b78b399b8608_23.png)

当文件很大时，我们可能只想查看其开头或结尾的几行内容。这时就需要用到 `head` 和 `tail` 命令。

![](img/0abb4089125f351630d8b78b399b8608_27.png)

![](img/0abb4089125f351630d8b78b399b8608_29.png)

以下是 `head` 和 `tail` 命令的常见用法。

![](img/0abb4089125f351630d8b78b399b8608_31.png)

![](img/0abb4089125f351630d8b78b399b8608_33.png)

*   **`head`**：默认显示文件的前10行。
    *   **显示前N行**：`head -n 5 /etc/passwd` （显示前5行）
    *   **显示除最后N行外的所有行**：`head -n -6 /etc/passwd` （显示除了最后6行以外的所有内容）
*   **`tail`**：默认显示文件的末尾10行。
    *   **显示末尾N行**：`tail -n 4 /etc/passwd` （显示末尾4行）
    *   **从第N行开始显示到文件末尾**：`tail -n +4 /etc/passwd` （从第4行开始显示）
    *   **实时监控文件内容（常用于查看日志）**：`tail -f /var/log/messages`。此命令会持续显示文件新增的内容，对于监控实时变化的日志文件非常有用。

**过渡**：提取了文件的首尾行之后，更常见的需求是在文件中搜索包含特定关键词的行。这就是 `grep` 命令的用武之地。

`grep` 是一个强大的文本搜索工具，它能使用正则表达式搜索文本，并将匹配的行打印出来。它是文本处理中最常用的命令之一。

![](img/0abb4089125f351630d8b78b399b8608_51.png)

![](img/0abb4089125f351630d8b78b399b8608_53.png)

![](img/0abb4089125f351630d8b78b399b8608_55.png)

![](img/0abb4089125f351630d8b78b399b8608_57.png)

以下是 `grep` 命令的一些核心选项。

*   **`-i`**：忽略搜索关键词的大小写。`grep -i “root” /etc/passwd`
*   **`-w`**：精确匹配整个单词，而不是作为字符串的一部分。`grep -w “root” /etc/passwd`
*   **`-v`**：反向选择，即输出不包含匹配模式的所有行。`grep -v “root” /etc/passwd`
*   **`-n`**：显示匹配行在原文件中的行号。`grep -n “root” /etc/passwd`
*   **`-o`**：只输出匹配到的字符串本身，而不是整行。`grep -o “root” /etc/passwd`
*   **`-c`**：统计匹配到的行数。`grep -c “root” /etc/passwd`
*   **`-A num`**：显示匹配行及其后面 `num` 行。`grep -A 2 “root” /etc/passwd`
*   **`-B num`**：显示匹配行及其前面 `num` 行。`grep -B 2 “root” /etc/passwd`
*   **`-C num`**：显示匹配行及其前后各 `num` 行。`grep -C 2 “root” /etc/passwd`
*   **`-q`**：静默模式，不输出任何结果。通常通过命令的返回值 `$?` 来判断是否匹配成功（0为成功，非0为失败）。
*   **`-r`**：递归搜索，在指定目录及其子目录下的所有文件中进行搜索。`grep -r “HISTSIZE” /etc`
*   **`-l`**：与 `-r` 结合使用，只输出包含匹配模式的文件名，而不显示具体内容。`grep -rl “HISTSIZE” /etc`

## 正则表达式基础

![](img/0abb4089125f351630d8b78b399b8608_72.png)

![](img/0abb4089125f351630d8b78b399b8608_74.png)

正则表达式是一种用于描述字符串模式的强大语言。`grep` 默认支持**基本正则表达式（BRE）**，使用 `-E` 选项或 `egrep` 命令则支持功能更丰富的**扩展正则表达式（ERE）**。

![](img/0abb4089125f351630d8b78b399b8608_76.png)

以下是基本正则表达式的常用元字符。

![](img/0abb4089125f351630d8b78b399b8608_78.png)

![](img/0abb4089125f351630d8b78b399b8608_79.png)

![](img/0abb4089125f351630d8b78b399b8608_81.png)

*   **`^`**：匹配行的开头。`grep “^root” /etc/passwd` （匹配以 `root` 开头的行）
*   **`$`**：匹配行的结尾。`grep “bash$” /etc/passwd` （匹配以 `bash` 结尾的行）
*   **`^$`**：匹配空行。
*   **`.`**：匹配任意一个字符。`grep “r..t” /etc/passwd` （匹配像 `root`， `rapt` 这样的字符串）
*   **`[]`**：匹配方括号内的任意一个字符。`grep “^[abc]” /etc/passwd` （匹配以 a 或 b 或 c 开头的行）
*   **`[^]`**：匹配不在方括号内的任意一个字符。`grep “^[^abc]” /etc/passwd` （匹配不以 a 或 b 或 c 开头的行）
*   **`[a-z]`**, **`[A-Z]`**, **`[0-9]`**：匹配一个范围。`grep “[0-9]” /etc/passwd` （匹配包含数字的行）
*   **`*`**：匹配前一个字符0次或多次。`grep “ro*t” /etc/passwd` （匹配 `rt`， `rot`， `root`， `rooot` 等）
*   **`\`**：转义字符，使其后的特殊字符失去特殊含义。例如，要匹配真正的点号 `.`，需要使用 `\.`。

![](img/0abb4089125f351630d8b78b399b8608_83.png)

![](img/0abb4089125f351630d8b78b399b8608_85.png)

![](img/0abb4089125f351630d8b78b399b8608_87.png)

以下是扩展正则表达式新增的常用元字符（需使用 `grep -E` 或 `egrep`）。

![](img/0abb4089125f351630d8b78b399b8608_89.png)

![](img/0abb4089125f351630d8b78b399b8608_91.png)

![](img/0abb4089125f351630d8b78b399b8608_93.png)

![](img/0abb4089125f351630d8b78b399b8608_95.png)

![](img/0abb4089125f351630d8b78b399b8608_97.png)

![](img/0abb4089125f351630d8b78b399b8608_98.png)

![](img/0abb4089125f351630d8b78b399b8608_100.png)

*   **`|`**：或操作，匹配两者之一。`egrep “root|user” /etc/passwd`
*   **`+`**：匹配前一个字符1次或多次。`egrep “ro+t” /etc/passwd` （匹配 `rot`， `root` 等，但不匹配 `rt`）
*   **`?`**：匹配前一个字符0次或1次。`egrep “ro?t” /etc/passwd` （匹配 `rt` 或 `rot`）
*   **`()`**：将括号内的模式作为一个整体。`egrep “(oo)+” /etc/passwd`
*   **`{m}`**, **`{m,n}`**：匹配前一个字符 m 次，或 m 到 n 次。`egrep “o{2,3}” /etc/passwd` （匹配连续2个或3个 `o`）

## 按列提取工具：cut

![](img/0abb4089125f351630d8b78b399b8608_132.png)

`cut` 命令用于从文件的每一行中截取部分内容（字段或字符）。

![](img/0abb4089125f351630d8b78b399b8608_134.png)

![](img/0abb4089125f351630d8b78b399b8608_136.png)

以下是 `cut` 命令的常见用法。

*   **按分隔符提取字段**：这是最常用的方式，使用 `-d` 指定字段分隔符，`-f` 指定要提取的字段编号（从1开始）。
    *   **提取`/etc/passwd`文件的第一列（用户名）**：`cut -d “:” -f 1 /etc/passwd`
    *   **提取第一和第三列**：`cut -d “:” -f 1,3 /etc/passwd`
*   **按字符位置提取**：使用 `-c` 选项按字符位置提取。
    *   **提取每行的第1到第5个字符**：`cut -c 1-5 /etc/passwd`

以下是几个核心的文本分析工具。

![](img/0abb4089125f351630d8b78b399b8608_187.png)

![](img/0abb4089125f351630d8b78b399b8608_189.png)

![](img/0abb4089125f351630d8b78b399b8608_191.png)

![](img/0abb4089125f351630d8b78b399b8608_195.png)

*   **`wc`**：统计文件的行数、单词数和字节数。
    *   **`-l`**：统计行数。`wc -l /etc/passwd`
    *   **`-w`**：统计单词数。
    *   **`-c`**：统计字节数。
*   **`sort`**：对文本行进行排序。
    *   **`-r`**：降序排序。
    *   **`-n`**：按数值大小排序，而不是字符串。
    *   **`-t`**：指定字段分隔符。
    *   **`-k`**：指定按第几个字段排序。`sort -t “:” -k 3 -n /etc/passwd` （按第三列UID的数值升序排序）
    *   **`-u`**：排序并去除重复行。
*   **`uniq`**：报告或忽略文件中的重复行。**注意**：`uniq` 通常需要与 `sort` 结合使用，因为它只能检测相邻的重复行。
    *   **`-c`**：在每行旁边显示该行重复出现的次数。`sort /etc/passwd | uniq -c`
*   **`diff`**：比较两个文件的差异。`diff file1 file2`
*   **`tr`**：用于转换或删除字符。
    *   **字符替换**：`cat /etc/passwd | tr “:” “\n”` （将冒号替换为换行符）
    *   **压缩重复字符**：`echo “aaabbbccc” | tr -s “ab”` （将连续的 `a` 和 `b` 压缩成一个，输出 `abccc`）

## 流编辑器：sed (简介)

![](img/0abb4089125f351630d8b78b399b8608_213.png)

![](img/0abb4089125f351630d8b78b399b8608_215.png)

`sed` 是一个非交互式的流编辑器。它一次处理一行内容，将当前行存储在临时缓冲区（称为“模式空间”）中进行处理，处理完成后输出到屏幕，然后清空缓冲区，读取下一行。除非使用 `-i` 选项，否则它不会修改原始文件。

![](img/0abb4089125f351630d8b78b399b8608_217.png)

![](img/0abb4089125f351630d8b78b399b8608_219.png)

![](img/0abb4089125f351630d8b78b399b8608_221.png)

![](img/0abb4089125f351630d8b78b399b8608_223.png)

以下是 `sed` 命令的一个简单示例。

![](img/0abb4089125f351630d8b78b399b8608_225.png)

![](img/0abb4089125f351630d8b78b399b8608_227.png)

![](img/0abb4089125f351630d8b78b399b8608_229.png)

![](img/0abb4089125f351630d8b78b399b8608_231.png)

![](img/0abb4089125f351630d8b78b399b8608_233.png)

*   **打印特定行**：`sed -n ‘5,10p’ /etc/passwd` （打印第5到第10行）。`-n` 选项用于禁止自动打印模式空间，`p` 命令表示打印。
*   **工作原理**：默认情况下，`sed` 会先自动打印读取的每一行（原始行），再执行命令。如果命令中包含 `p`（打印），则会再打印一次处理后的行。使用 `-n` 可以关闭自动打印，只输出由命令显式指定的内容。
