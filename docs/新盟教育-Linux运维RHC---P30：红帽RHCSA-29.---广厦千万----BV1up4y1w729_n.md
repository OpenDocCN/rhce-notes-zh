### 什么是程序？
程序是用计算机语言编写的命令序列的集合，用于实现特定目标或解决特定问题。简单来说，程序就是安装在计算机上的软件。例如，从应用商店下载的聊天工具就是一个程序。

![](img/f081544b2304fd7016bb60e35f8ff0d9_23.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_25.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_27.png)

*   **程序占用磁盘空间**。
*   **程序是静态且永久的**。在未运行时，它只是存储在硬盘上的文件。

![](img/f081544b2304fd7016bb60e35f8ff0d9_29.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_31.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_33.png)

### 什么是进程？
进程是正在运行中的程序。当程序被加载到内存并开始执行时，它就变成了一个进程。

![](img/f081544b2304fd7016bb60e35f8ff0d9_35.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_37.png)

*   **进程占用内存空间**。
*   **进程是动态的**，它有生命周期，会经历创建、运行和终止。
*   每个进程都有独立的**内存空间**和系统分配的**唯一PID（进程ID）**，用于识别。
*   每个进程通常都有一个**父进程（PPID）**，父进程可以创建多个**子进程**，形成父子关系。
*   进程可以**前台运行**（占用当前终端）或**后台运行**（不占用当前终端，默默执行）。

### 什么是线程？
线程，常被称为**轻量级进程**，是进程中的一个执行单元，是进程中实际运作的单位。

*   线程被包含在进程内部，是进程的子集。
*   一个进程可以并发多个线程，共享进程的资源。
*   可以将CPU核心比作工厂的车间，线程则是车间里真正干活的工人。线程是**真正执行计算和处理任务**的实体。

![](img/f081544b2304fd7016bb60e35f8ff0d9_39.png)

对于运维工作，理解线程是“进程中实际运作的单位”即可，无需像开发人员那样深入。

## 查看进程树：`pstree`命令

了解了基本概念后，我们来看看如何在Linux系统中直观地查看进程及其关系。

`pstree`命令以树状结构显示进程信息，清晰地展示了进程之间的父子关系。

以下是`pstree`命令的常用选项：

![](img/f081544b2304fd7016bb60e35f8ff0d9_41.png)

*   **基本命令**：直接运行`pstree`会显示简明的进程树。
    ```bash
    pstree
    ```
*   **显示PID**：使用`-p`选项显示每个进程的PID。
    ```bash
    pstree -p
    ```
*   **显示完整命令行**：使用`-a`选项显示进程的完整启动命令。
*   **显示用户**：使用`-u`选项列出每个进程所属的用户名。
    ```bash
    pstree -u
    ```
*   **查看指定用户的进程**：可以直接在命令后跟上用户名，查看该用户启动的进程。
    ```bash
    pstree username
    ```

![](img/f081544b2304fd7016bb60e35f8ff0d9_43.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_45.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_46.png)

**关键点**：
*   树状结构的根通常是 **`systemd`**（PID为1），它是所有进程的**父进程**，也称为**初始化进程**，负责启动和管理系统中的其他进程。
*   进程名后的`*`号表示该进程包含线程（轻量级进程），例如 `(6*)` 表示该进程下有6个线程。

## 查看进程信息：`ps`命令

除了树状视图，我们还需要一种方式来查看进程的详细状态信息。`ps`命令用于静态查看系统进程的快照。

![](img/f081544b2304fd7016bb60e35f8ff0d9_59.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_60.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_62.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_64.png)

### `ps aux` 命令
这是最常用的命令组合之一，用于查看系统所有进程的详细信息。

![](img/f081544b2304fd7016bb60e35f8ff0d9_66.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_68.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_69.png)

```bash
ps aux
```

![](img/f081544b2304fd7016bb60e35f8ff0d9_71.png)

以下是`ps aux`输出中各列的含义：

![](img/f081544b2304fd7016bb60e35f8ff0d9_73.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_75.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_77.png)

1.  **USER**: 进程所属的用户。
2.  **PID**: 进程的ID号。
3.  **%CPU**: 进程占用CPU的百分比。
4.  **%MEM**: 进程占用物理内存的百分比。
5.  **VSZ**: 进程使用的虚拟内存总量（以KB为单位）。
6.  **RSS**: 进程占用的固定内存总量（以KB为单位）。
7.  **TTY**: 进程在哪个终端运行。
    *   `?`：由内核直接调用，无关联终端。
    *   `ttyN`：表示在本机终端登录的用户进程。
    *   `pts/N`：表示在远程终端（如SSH连接）登录的用户进程。
8.  **STAT**: 进程状态。这是需要重点关注的列。
    *   **R (Running)**: 正在运行或可运行（在运行队列中）。
    *   **S (Sleeping)**: 休眠状态，等待某个事件完成。
    *   **D**：不可中断的休眠状态（通常与IO相关）。
    *   **T (Stopped)**: 进程被停止（如由调试器暂停）。
    *   **Z (Zombie)**: **僵尸进程**。进程已终止，但其父进程未能成功回收其资源。需要关注并处理。
    *   **<**：高优先级进程。
    *   **N**：低优先级进程。
    *   **s**：该进程是会话首进程（包含子进程）。
    *   **+**：进程位于**前台进程组**。
9.  **START**: 进程的启动时间。
10. **TIME**: 进程占用CPU的总时间。
11. **COMMAND**: 启动该进程的命令。

![](img/f081544b2304fd7016bb60e35f8ff0d9_79.png)

**僵尸进程处理**：如果发现僵尸进程（STAT为Z），通常需要终止其父进程（PPID），然后由`systemd`回收资源。可以使用以下命令查找僵尸进程及其父进程：

![](img/f081544b2304fd7016bb60e35f8ff0d9_81.png)

```bash
ps -ef | grep defunct
# 或更精确的过滤
ps -eo pid,ppid,stat,cmd | grep -E ‘[Zz]’
```

![](img/f081544b2304fd7016bb60e35f8ff0d9_83.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_85.png)

### `ps -ef` 命令
这是另一种显示风格（BSD vs UNIX），同样常用，尤其会显示**PPID（父进程ID）**。

![](img/f081544b2304fd7016bb60e35f8ff0d9_87.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_89.png)

```bash
ps -ef
# 或
ps -elf
```

![](img/f081544b2304fd7016bb60e35f8ff0d9_91.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_93.png)

**最常用场景**：结合`grep`过滤特定进程。
```bash
ps aux | grep sshd
```

## 总结
本节课中，我们一起学习了程序、进程和线程的核心概念：
*   **程序**是静态的软件实体，存储在磁盘。
*   **进程**是运行中的程序，动态占用内存，有独立PID和生命周期。
*   **线程**是进程内的执行单元，是实际工作的实体。

![](img/f081544b2304fd7016bb60e35f8ff0d9_101.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_103.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_105.png)

我们还掌握了两个关键的进程查看工具：
*   **`pstree`**：以树状图形式直观展示进程间的父子关系。
*   **`ps`**：查看进程的详细状态信息，特别是`ps aux`和`ps -ef`的组合，并能通过`STAT`字段识别进程状态（如运行、休眠、僵尸）。

![](img/f081544b2304fd7016bb60e35f8ff0d9_107.png)

![](img/f081544b2304fd7016bb60e35f8ff0d9_109.png)
