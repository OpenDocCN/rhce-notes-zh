# Ansible入门教程：P49：Day09_Ch01b_安装Ansible 🚀

在本节课中，我们将要学习如何在RHEL 8.0系统上安装和初始化Ansible自动化工具。这是构建Ansible控制节点的基础步骤。

## 概述 📋

Ansible是一个强大的自动化工具，用于配置管理、应用部署和任务编排。本节将指导您完成Ansible的安装过程，并验证其是否成功运行。我们只需要在一台机器（控制节点）上安装Ansible，其他机器（被控节点）则无需安装。

## 环境准备 🖥️

在开始安装之前，需要确保您的环境满足以下要求。

### 硬件与系统要求

以下是搭建Ansible练习环境的基本要求：
*   **内存**：练习环境建议8G至16G内存。若使用三台裸机，8G内存足够。
*   **硬盘**：至少100G，建议使用固态硬盘（SSD）以获得更好的性能。
*   **虚拟机数量**：至少需要三台虚拟机。
*   **操作系统**：统一使用RHEL 8.0。RHEL 7与8的主要区别在于Python版本（7使用2.7.5，8使用3.6.8）。

## 安装Ansible ⚙️

上一节我们介绍了环境要求，本节中我们来看看具体的安装步骤。Ansible 2.9.10-1是当前的最新版本。需要注意的是，Ansible及其依赖的Python包并不在标准的RHEL 8仓库中，因此我们需要通过挂载本地安装光盘来辅助安装。

### 步骤一：挂载本地镜像源

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_10.png)

首先，我们需要在控制节点（controller）上挂载RHEL 8的安装镜像，以获取本地yum源。

1.  创建一个用于挂载的目录：
    ```bash
    mkdir /mnt/iso
    ```
2.  挂载ISO镜像文件（请根据您的镜像实际路径调整）：
    ```bash
    mount /dev/sr0 /mnt/iso
    ```

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_12.png)

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_13.png)

### 步骤二：配置本地Yum仓库

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_15.png)

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_17.png)

挂载完成后，需要配置yum以使用这个本地源。

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_19.png)

1.  编辑yum仓库配置文件：
    ```bash
    vim /etc/yum.repos.d/local.repo
    ```
2.  在文件中添加以下内容（注意方括号`[]`内不能有空格）：
    ```ini
    [BaseOS]
    name=BaseOS
    baseurl=file:///mnt/iso/BaseOS
    enabled=1
    gpgcheck=0

    [AppStream]
    name=AppStream
    baseurl=file:///mnt/iso/AppStream
    enabled=1
    gpgcheck=0
    ```
3.  保存并退出编辑器。

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_20.png)

### 步骤三：安装EPEL仓库

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_22.png)

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_23.png)

Ansible包由Fedora项目维护，存放在EPEL（Extra Packages for Enterprise Linux）仓库中。我们需要先安装EPEL仓库。

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_24.png)

*   在联网状态下，执行以下命令安装EPEL仓库包：
    ```bash
    dnf install -y https://dl.fedoraproject.org/pub/epel/epel-release-latest-8.noarch.rpm
    ```
*   如果网络下载困难，也可以将提前下载好的rpm包上传至服务器进行安装。
*   安装完成后，更新yum元数据缓存：
    ```bash
    dnf makecache
    ```

### 步骤四：安装Ansible

配置好EPEL仓库后，就可以安装Ansible了。请确保在已挂载本地yum源的情况下执行此命令，因为安装过程会依赖本地光盘中的Python组件。

*   执行安装命令：
    ```bash
    dnf install -y ansible
    ```
*   安装完成后，可以查看Ansible的版本以确认：
    ```bash
    ansible --version
    ```
    您将看到类似 `ansible 2.9.10-1` 的输出信息。

## 验证安装 ✅

安装完成后，我们需要验证Ansible是否能够正常工作。

### 方法一：查看Ansible配置信息

运行以下命令可以查看Ansible的默认配置文件路径、Python模块位置等详细信息：
```bash
ansible --version
```
输出信息会包含配置文件路径、模块搜索路径、Python版本以及可执行文件位置（通常在 `/usr/bin/` 下）。

### 方法二：运行临时命令

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_26.png)

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_27.png)

通过运行一个针对本机的临时命令，可以测试Ansible的核心功能是否正常。

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_28.png)

![](img/e7a82b0d4f83aab98b362a2f93d8fe4a_29.png)

*   使用 `setup` 模块收集本机（localhost）的事实信息：
    ```bash
    ansible localhost -m setup
    ```
*   如果安装成功，命令将返回一长串JSON格式的数据，其中包含了本机的网络配置（IPv4、IPv6地址）、系统信息等“事实”（facts）。这证明Ansible已成功安装并可运行。
