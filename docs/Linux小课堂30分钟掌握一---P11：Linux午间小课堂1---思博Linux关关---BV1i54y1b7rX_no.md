# Linux小课堂：P11：SSH远程登录配置与管理 🖥️

在本节课中，我们将学习如何配置和使用SSH协议来远程登录和管理Linux服务器。这是部署线上服务（如Apache）的第一步，也是IT从业者必须掌握的核心技能。

## 概述：为什么需要SSH？

在日常工作中，服务器通常位于机房或云平台，我们无法每次都到物理设备前操作。SSH（Secure Shell，安全外壳协议）允许我们通过网络，安全地远程登录到服务器并进行管理，就像在本地操作一样。

### 安装SSH软件包

![](img/b6ae09f5c76578b35acf81251230147a_11.png)

大部分Linux发行版默认已安装SSH。如果未安装，可以使用包管理器进行安装。在基于RHEL/CentOS的系统上，使用 `yum` 命令。

以下是安装命令：
```bash
yum install openssh*
```
此命令会安装所有以 `openssh` 开头的软件包及其依赖。

### 启动SSH服务

安装完成后，需要启动SSH服务并设置为开机自启。

以下是管理服务的命令：
```bash
# 启动SSH服务
systemctl start sshd.service

# 停止SSH服务
systemctl stop sshd.service

# 设置开机自启
systemctl enable sshd.service

# 禁止开机自启
systemctl disable sshd.service

# 查看服务状态
systemctl status sshd.service
```

## SSH服务端核心配置

![](img/b6ae09f5c76578b35acf81251230147a_29.png)

上一节我们启动了SSH服务，本节中我们来看看其核心配置文件，以进行安全性和功能定制。

SSH服务端的主要配置文件是 `/etc/ssh/sshd_config`。以下是几个关键参数及其含义：

以下是需要关注的核心配置项：

![](img/b6ae09f5c76578b35acf81251230147a_31.png)

*   **`PasswordAuthentication yes`**： 是否允许使用密码登录。设置为 `no` 则禁用密码登录，必须使用密钥等方式。
*   **`Port 22`**： SSH服务监听的端口号。出于安全考虑，可以修改为其他高端口号（如 `5000`）。
*   **`ListenAddress 0.0.0.0`**： 监听所有网络接口的IP地址。如果服务器有多个IP，可以指定为某个具体IP。
*   **`PermitRootLogin yes`**： 是否允许root用户直接登录。出于安全考虑，生产环境通常设置为 `no`。
*   **`GSSAPIAuthentication no`**： 建议设置为 `no`，可以加快连接速度，避免因DNS解析问题导致登录缓慢。
*   **`UseDNS no`**： 建议设置为 `no`，同样用于避免反向DNS查询，提升登录效率。

修改配置文件后，需要重启SSH服务使配置生效：
```bash
systemctl restart sshd.service
```

> **注意**： 线上服务器操作需谨慎，错误的配置可能导致无法远程登录。

## SSH客户端连接与使用

上一节我们配置好了服务端，本节中我们来看看客户端如何连接，并了解密码登录的原理。

### 密码登录流程

SSH密码登录是一个加密的验证过程：
1.  客户端向服务器（IP:端口）发起连接请求。
2.  服务器将自己的公钥发送给客户端。
3.  客户端使用该公钥加密账号密码，并发送回服务器。
4.  服务器用自己的私钥解密，验证账号密码是否正确。
5.  验证通过，建立加密连接。

![](img/b6ae09f5c76578b35acf81251230147a_39.png)

### 使用客户端工具连接

在Windows系统上，可以使用 `SecureCRT`、`Xshell`、`PuTTY` 等工具。连接时需要提供：
*   **主机**： 服务器的IP地址
*   **端口**： SSH端口（默认为22）
*   **用户名**： 要登录的账户名（如 `root`）
*   **密码**： 对应用户的密码

![](img/b6ae09f5c76578b35acf81251230147a_41.png)

连接成功后，命令行提示符中：
*   `$` 符号表示当前是普通用户。
*   `#` 符号表示当前是root超级管理员。

以下是配置日志记录的推荐方案：
1.  打开 **全局选项** 或 **会话选项**。
2.  找到 **日志文件** 设置。
3.  配置日志文件名格式，例如：
    ```
    %S-%Y%m%d-%H%M%S.log
    ```
    这会将日志按“会话名-年月日-时分秒”的格式保存。
4.  同时启用“连接时”、“断开时”、“每行”的时间戳记录，使日志更清晰。

### 配置颜色方案

为不同类型的文件（目录、可执行文件、压缩包等）设置不同颜色，可以直观区分，避免误操作。
1.  在 **全局选项** 中找到 **终端** -> **外观** -> **ANSI颜色**。
2.  根据文件类型配置对应的颜色。
3.  也可以在网上搜索并导入优秀的配色方案。
