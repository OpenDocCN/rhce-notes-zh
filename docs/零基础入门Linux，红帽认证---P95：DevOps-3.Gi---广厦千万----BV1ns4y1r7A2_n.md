## 设置界面语言

GitLab的默认界面语言是英文。为了操作方便，我们需要将其调整为简体中文。

以下是操作步骤：

1.  登录GitLab后，在用户界面中找到语言设置选项。
2.  将语言从“English”更改为“简体中文”。此时界面翻译完成度约为67%，并非完全汉化。
3.  保存更改并刷新页面，界面将显示为中文。

完成设置后，所有菜单项如“用户资料”、“账号”、“应用”等都将以中文显示。

## 修改管理员密码

首次登录GitLab使用的是系统生成的初始密码。出于安全考虑，这个密码文件会在24小时后或系统首次重新配置时自动删除，因此必须立即修改。

以下是修改密码的步骤：

1.  点击右上角用户头像，进入“偏好设置”。
2.  在左侧菜单中找到并点击“密码”。
3.  在“当前密码”栏中输入初始密码（可在安装时生成的初始密码文件中找到）。
4.  在“新密码”和“确认密码”栏中，输入满足要求的新密码（GitLab要求密码长度至少为8个字符）。
5.  点击“保存”按钮完成修改。

修改成功后，下次登录请使用新设置的密码。

## 创建群组与项目

![](img/5f9f5fa80600bff23e62dc2a3c11b534_15.png)

在GitLab中，代码存储在“项目”中。为了更好地管理项目和用户权限，我们通常先创建“群组”。

![](img/5f9f5fa80600bff23e62dc2a3c11b534_17.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_18.png)

### 理解核心概念

![](img/5f9f5fa80600bff23e62dc2a3c11b534_20.png)

*   **项目**：即代码仓库，用于存储和管理代码。
*   **用户**：可以登录GitLab平台的个人账号。
*   **群组**：用于集中管理项目和用户。可以将项目归属到某个群组，并将用户加入该群组，从而实现组内成员共享项目资源。

### 创建群组

![](img/5f9f5fa80600bff23e62dc2a3c11b534_22.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_24.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_26.png)

以下是创建群组的步骤：

![](img/5f9f5fa80600bff23e62dc2a3c11b534_28.png)

1.  点击左侧导航栏的“菜单”，选择“管理”。
2.  在管理区域，点击“新建群组”。
3.  填写群组信息：
    *   **群组名称**：例如 `test`。
    *   **群组URL**：系统会自动生成，格式为 `访问地址/组名`。
    *   **群组描述**：（可选）例如“这是一个测试项目的组”。
    *   **可见性级别**：分为私有、内部和公开。
        *   **私有**：只有被明确添加到该群组的成员才能查看组内项目。
        *   **内部**：任何登录到GitLab的用户都可以查看。
        *   **公开**：任何人都无需登录即可通过URL访问。
    *   其他设置如“允许创建项目”等可保持默认。
4.  点击“创建群组”。

![](img/5f9f5fa80600bff23e62dc2a3c11b534_30.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_31.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_33.png)

### 创建项目

![](img/5f9f5fa80600bff23e62dc2a3c11b534_35.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_37.png)

群组创建完成后，即可在该群组下创建项目。

以下是创建项目的步骤：

1.  在管理区域，点击“新建项目”。
2.  选择“创建空白项目”。
3.  填写项目信息：
    *   **项目名称**：例如 `my_lottery`。
    *   **项目URL**：选择将项目放在之前创建的 `test` 群组下，URL格式为 `访问地址/组名/项目名`。
    *   **项目描述**：（可选）例如“这是一个幸运转盘抽奖项目，目前处于测试阶段”。
    *   **可见性级别**：选择“私有”，确保只有授权人员可访问。
4.  点击“新建项目”。

项目创建成功后，页面会显示一个空的代码仓库，并附有命令行操作指引。

### 添加远程仓库地址

![](img/5f9f5fa80600bff23e62dc2a3c11b534_48.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_50.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_51.png)

首先，需要将GitLab上的远程仓库地址关联到本地仓库。

1.  进入本地项目所在的目录（例如 `/data`）。
2.  执行以下命令，添加远程仓库。其中 `origin` 是远程仓库在本地的别名，可以自定义。
    ```bash
    git remote add origin http://web.gitlab.com/test/my_lottery.git
    ```
3.  使用以下命令验证远程仓库是否添加成功。
    ```bash
    git remote -v
    ```
    此命令用于查看已配置的远程仓库列表。

![](img/5f9f5fa80600bff23e62dc2a3c11b534_53.png)

### 推送代码

![](img/5f9f5fa80600bff23e62dc2a3c11b534_55.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_57.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_59.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_60.png)

关联远程仓库后，即可将本地代码推送上去。

![](img/5f9f5fa80600bff23e62dc2a3c11b534_62.png)

![](img/5f9f5fa80600bff23e62dc2a3c11b534_64.png)

1.  执行推送命令。`-u` 参数设置上游分支，`--all` 参数推送所有分支。
    ```bash
    git push -u origin --all
    ```
    如果 `--all` 参数无效，可以指定分支名，如 `master` 或 `main`。
    ```bash
    git push -u origin master
    ```
2.  首次推送时，GitLab会要求进行身份验证。输入有权限访问该仓库的用户名（如 `root`）和密码。
3.  推送成功后，刷新GitLab项目页面，即可看到本地代码文件已全部出现在远程仓库中。

这个过程模拟了开发人员将代码提交到中央仓库的流程。
