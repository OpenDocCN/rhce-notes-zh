在考试环境中，系统通常会提供三块虚拟磁盘。第一块磁盘（`vda`）已安装操作系统，包含根分区和交换分区，我们无需操作。我们需要重点关注的是另外两块磁盘：`vdb`和`vdc`。

![](img/88e36055e8557fbef288e04c9ef8db6b_32.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_33.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_35.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_37.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_39.png)

*   `vdb` 磁盘：这块磁盘上已经存在一个分区（`vdb1`），用于创建了一个卷组和一个逻辑卷（例如名为 `vo` 的逻辑卷）。考试中有一道题目要求你**调整这个逻辑卷的大小**，并且**必须保证其内部文件系统的数据不丢失**，因此**绝对不能重新格式化**。
*   `vdc` 磁盘：这是一块完全空闲的磁盘，专门留给另一道题目**创建VDO卷**使用。因此，在做其他题目（如添加交换分区）时，**切勿使用 `vdc` 磁盘**。

基于以上需求，我们需要在 `vdb` 磁盘上完成以下操作：
1.  调整现有逻辑卷的大小。
2.  创建一个新的分区，用于**添加交换分区**。
3.  创建另一个新的分区，为**创建新逻辑卷**准备空间。

## Linux磁盘使用流程概述

![](img/88e36055e8557fbef288e04c9ef8db6b_45.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_47.png)

在使用一块新磁盘前，需要遵循一个标准的流程，以下是这个流程的步骤：

1.  **识别磁盘**：系统启动时，内核会自动识别硬件磁盘。
2.  **分区**：使用分区工具（如 `fdisk`, `parted`）在磁盘上划分逻辑区域。
3.  **格式化**：使用格式化工具（如 `mkfs`, `mkswap`）在分区上创建文件系统。
4.  **挂载**：使用 `mount` 命令将格式化好的分区关联到系统目录树。
5.  **使用**：通过挂载点目录访问和存储数据。

## 核心操作详解

### 1. 识别与查看磁盘

我们可以使用以下命令查看系统识别的磁盘和分区信息。

*   **`lsblk`**：以树状图形式简洁地列出所有块设备。
    ```bash
    lsblk
    ```
*   **`fdisk -l`**：列出所有磁盘的详细信息，包括分区表、容量等。
    ```bash
    fdisk -l
    ```

![](img/88e36055e8557fbef288e04c9ef8db6b_49.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_51.png)

### 2. 磁盘分区

![](img/88e36055e8557fbef288e04c9ef8db6b_53.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_54.png)

分区工具主要有两种：`fdisk` 和 `parted`。

![](img/88e36055e8557fbef288e04c9ef8db6b_56.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_58.png)

*   **`fdisk`**：传统分区工具，主要支持 MBR（MS-DOS）分区表。它采用**交互式界面**，修改后需要执行 `w` 命令才会保存，适合新手练习。
    ```bash
    fdisk /dev/vdb
    ```
    进入交互界面后，常用命令有：
    *   `p`：打印（查看）当前分区表。
    *   `n`：创建新分区。
    *   `d`：删除分区。
    *   `w`：保存更改并退出。
    *   `q`：不保存更改并退出。

*   **`parted`**：更现代的分区工具，同时支持 MBR 和 GPT 分区表。它的特点是**操作立即生效**，适合有经验的管理员，但风险较高。
    ```bash
    parted /dev/vdb
    ```
    在 `parted` 交互界面中，常用命令有：
    *   `print` 或 `p`：查看分区表。
    *   `mkpart`：创建分区。
    *   `rm`：删除分区。
    *   `quit`：退出。

![](img/88e36055e8557fbef288e04c9ef8db6b_60.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_61.png)

**分区后刷新内核**：分区完成后，需要让内核重新读取分区表。
*   最彻底的方式是重启系统：`reboot`
*   如果不重启，可以尝试以下命令刷新指定磁盘：
    ```bash
    partprobe /dev/vdb
    # 或
    partx -a /dev/vdb
    ```

![](img/88e36055e8557fbef288e04c9ef8db6b_63.png)

### 3. 格式化分区

格式化是为分区创建文件系统的过程。不同的用途使用不同的格式化命令。

*   **创建常规文件系统（如 ext4）**：
    ```bash
    mkfs.ext4 /dev/vdb3
    ```
*   **创建交换空间**：
    ```bash
    mkswap /dev/vdb2
    ```

### 4. 挂载与自动挂载

![](img/88e36055e8557fbef288e04c9ef8db6b_65.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_66.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_68.png)

*   **临时挂载**：使用 `mount` 命令。
    ```bash
    mount /dev/vdb3 /mnt/data
    ```
*   **开机自动挂载**：需要编辑 `/etc/fstab` 配置文件。
    ```bash
    vim /etc/fstab
    ```
    在文件中添加一行，格式如下：
    ```
    /dev/vdb3  /mnt/data  ext4  defaults  0 0
    ```

## 实战：添加交换分区

现在，我们以一道具体的考题为例，演示如何“添加一个512MB的交换分区，并使其开机自动挂载”。

![](img/88e36055e8557fbef288e04c9ef8db6b_70.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_72.png)

假设我们已经在 `/dev/vdb` 上创建了第二个分区 `/dev/vdb2`（512MB）。以下是操作步骤：

![](img/88e36055e8557fbef288e04c9ef8db6b_74.png)

![](img/88e36055e8557fbef288e04c9ef8db6b_76.png)

1.  **格式化分区为交换空间**：
    ```bash
    mkswap /dev/vdb2
    ```
2.  **立即启用该交换分区**：
    ```bash
    swapon /dev/vdb2
    ```
3.  **验证交换分区已启用**：
    ```bash
    swapon -s
    # 或
    free -h
    ```
4.  **配置开机自动挂载**：
    *   编辑 `/etc/fstab` 文件。
    *   在文件末尾添加一行：
        ```
        /dev/vdb2 swap swap defaults 0 0
        ```
5.  **测试自动挂载配置**：
    *   先关闭我们刚启用的交换分区：`swapoff /dev/vdb2`
    *   然后让系统根据 `/etc/fstab` 自动启用所有交换分区：`swapon -a`
    *   再次检查：`swapon -s`，如果能看到 `/dev/vdb2` 则配置成功。
