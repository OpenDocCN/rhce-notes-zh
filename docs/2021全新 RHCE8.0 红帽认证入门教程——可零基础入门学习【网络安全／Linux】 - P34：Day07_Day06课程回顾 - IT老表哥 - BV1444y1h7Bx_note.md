## 一、Bash Shell基础回顾

上一节我们介绍了Bash Shell的基本概念，本节中我们来看看变量与赋值以及简单的数学运算。

### 1. 变量与赋值
变量是存储数据的容器。以下是关于变量的核心规则和操作：
*   **命名规则**：变量名不能以数字开头，可以以字母或下划线开头。
*   **变量取值**：使用 `$` 符号后跟变量名来获取变量的值，例如 `$variable_name`。
*   **数学运算**：可以使用 `let`、`$(( ))` 或 `expr` 命令进行基本的数学运算。

### 2. 文本处理工具
文本处理是系统管理中的常见任务。以下是几个关键工具的介绍：

**awk**
`awk` 是一个强大的文本分析工具，可以按字段处理文本。
*   **基本语法**：`awk -F ‘分隔符’ ‘模式 {动作}’ 文件名`
*   **模式匹配**：用于查找包含特定关键字的行。

![](img/8d1c3bdab1d9f407c63a6ea1abf40a9b_16.png)

![](img/8d1c3bdab1d9f407c63a6ea1abf40a9b_17.png)

![](img/8d1c3bdab1d9f407c63a6ea1abf40a9b_19.png)

**sed**
`sed` 是流编辑器，用于对文本进行流式处理（按行处理）。
*   **基本语法**：`sed ‘选项 动作’ 文件名`
*   **常用动作**：
    *   `p`：打印输出。
    *   `a`：在指定行后追加文本。
    *   `i`：在指定行前插入文本。
    *   `d`：删除指定行。
    *   `s`：替换文本。
*   **重要提示**：除了 `-i` 选项会直接修改原文件外，其他操作默认只模拟输出。使用 `-i` 前务必做好文件备份。

**grep**
`grep` 用于在文件中搜索包含特定模式的行。
*   **常用选项**：
    *   `-i`：忽略大小写。
    *   `-A n`：显示匹配行及其后n行。
    *   `-B n`：显示匹配行及其前n行。
    *   `-E`：使用扩展正则表达式，支持多条件匹配（条件间是“或”关系）。

**cut**
`cut` 命令用于从文件的每一行中截取部分内容。
*   **特点**：`cut` 命令的分隔符只能识别单个字符，对于非标准格式的文本，`awk` 的处理通常更精准。

### 3. 正则表达式与特殊符号
正则表达式是文本匹配的强大工具，以下是一些基础概念：
*   `^`：匹配行首。
*   `$`：匹配行尾。
*   `*`：匹配前一个字符零次或多次。
*   `?`：匹配前一个字符零次或一次。
*   `${variable}`：引用变量，花括号用于消除变量名歧义。
*   `$?`：获取上一条命令的退出状态码。`0` 表示成功，非 `0` 表示失败。在脚本中常用于条件判断。

## 二、Shell脚本编程核心

掌握了基础工具后，我们进入Shell脚本编程的核心部分。

### 1. 变量进阶操作
变量在脚本中有更多用法：
*   **命令替换**：将命令的输出赋值给变量，推荐使用 `$(command)` 格式，便于嵌套。
*   **引号区别**：
    *   双引号：允许变量和命令替换。
    *   单引号：将所有内容视为普通字符。
*   **变量类型**：主要有整型（用于数学运算）和字符串型。
*   **特殊变量**：
    *   `$0`：脚本名。
    *   `$1`, `$2`...：脚本参数。
    *   `$@`：所有参数列表。
    *   `$*`：所有参数合并为一个字符串。
    *   `$#`：参数个数。

### 2. 条件判断语句
条件判断用于控制程序流程。

**判断条件类型**
*   **文件判断**：
    *   `-e`：文件/目录是否存在。
    *   `-f`：是否为常规文件。
    *   `-d`：是否为目录。
*   **字符串判断**：
    *   `-n`：字符串长度是否非零。
    *   `-z`：字符串长度是否为零。
    *   `=` 或 `==`：两个字符串是否相等。
*   **数值判断**：
    *   `-eq`：等于。
    *   `-ne`：不等于。
    *   `-lt`：小于。
    *   `-gt`：大于。
    *   `-le`：小于等于。
    *   `-ge`：大于等于。
*   **逻辑非**：使用 `!` 符号，例如 `! -e file` 判断文件不存在。

**if 语句**
`if` 语句用于基于条件执行不同代码块。
```bash
if [ 条件表达式 ]; then
    # 条件为真时执行的命令
elif [ 其他条件 ]; then
    # 其他条件为真时执行的命令
else
    # 所有条件为假时执行的命令
fi
```
**注意**：`[ ]` 内条件表达式的前后必须留有空格。条件判断以命令的退出状态码 `$?` 为准。

![](img/8d1c3bdab1d9f407c63a6ea1abf40a9b_37.png)

![](img/8d1c3bdab1d9f407c63a6ea1abf40a9b_39.png)

**case 语句**
`case` 语句用于多重分支选择，比多个 `if-elif` 更清晰。
```bash
case $变量 in
    模式1)
        命令集1
        ;;
    模式2)
        命令集2
        ;;
    *)
        默认命令集
        ;;
esac
```
**注意**：每个模式以 `)` 结尾，命令集以 `;;` 结束，`*` 代表默认情况。

### 3. 循环语句
循环用于重复执行一段代码。

**for 循环**
遍历一个列表中的值。
```bash
for 变量 in 列表
do
    # 循环体
done
```

**while 循环**
当条件为真时执行循环体。
```bash
while [ 条件 ]
do
    # 循环体
done
```

**until 循环**
与 `while` 相反，当条件为假时执行循环体。
```bash
until [ 条件 ]
do
    # 循环体
done
```

## 三、系统任务管理

最后，我们回顾系统任务自动化的关键工具：计划任务。

### 1. 一次性计划任务 (at)
`at` 命令用于安排一次性任务在将来某个特定时间点执行。
*   **基本用法**：`at 时间`
*   **时间格式**：可以是具体时间点（如 `14:30`），也可以是相对时间（如 `now + 2 hours`）。

### 2. 周期性计划任务 (cron)
`crontab` 命令用于创建、编辑和管理周期性任务。
*   **编辑任务**：`crontab -e`
*   **查看任务**：`crontab -l`
*   **格式说明**：用户级任务格式为 `分 时 日 月 周 命令`。
    *   `*`：代表任何值。
    *   `,`：指定多个值（如 `2,6,10`）。
    *   `-`：指定一个范围（如 `1-5`）。
    *   `/`：指定间隔频率（如 `*/2` 表示每两单位一次）。
*   **系统级任务**：root用户可以管理 `/etc/cron.hourly/`, `/etc/cron.daily/`, `/etc/cron.weekly/`, `/etc/cron.monthly/` 目录下的任务，这些任务由系统按预设延迟时间自动执行。
