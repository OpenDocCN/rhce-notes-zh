### 什么是程序？

![](img/a33effd18a0cb2476a571b9d46075f08_20.png)

![](img/a33effd18a0cb2476a571b9d46075f08_22.png)

程序是用计算机语言编写的命令序列的集合，用来实现特定的目标或解决特定的问题。

![](img/a33effd18a0cb2476a571b9d46075f08_24.png)

![](img/a33effd18a0cb2476a571b9d46075f08_26.png)

![](img/a33effd18a0cb2476a571b9d46075f08_28.png)

简单来说，程序就是我们通常所说的“软件”。例如，从应用商店下载一个聊天工具（如微信），这个软件安装到电脑后，就成为了电脑里的一个应用程序。程序在未运行时，是静态的，永久存储在磁盘上，占用的是硬盘空间。

![](img/a33effd18a0cb2476a571b9d46075f08_30.png)

![](img/a33effd18a0cb2476a571b9d46075f08_32.png)

**核心概念**：程序 = 存储在磁盘上的静态软件。

![](img/a33effd18a0cb2476a571b9d46075f08_34.png)

![](img/a33effd18a0cb2476a571b9d46075f08_36.png)

### 什么是进程？

![](img/a33effd18a0cb2476a571b9d46075f08_38.png)

![](img/a33effd18a0cb2476a571b9d46075f08_40.png)

进程是正在运行中的程序。

当程序被加载到内存中并开始执行时，它就变成了一个进程。进程是动态的，有生命周期的，并且占用的是内存空间。进程的生命周期可能很短（如一条命令瞬间执行完毕），也可能很长（如一个持续运行的后台服务）。

以下是进程的几个关键特性：
*   **独立内存空间**：每个进程都有系统为其分配的独立内存空间。
*   **进程ID (PID)**：系统为每个进程分配一个唯一的ID号，用于识别和管理该进程，类似于人的身份证号。
*   **父子关系**：每个进程（子进程）通常由一个父进程创建。父进程可以创建多个子进程。结束父进程通常会同时结束其所有子进程。
*   **运行方式**：进程可以在前台运行（占用当前终端），也可以在后台运行（不占用终端，默默提供服务）。

![](img/a33effd18a0cb2476a571b9d46075f08_42.png)

**核心概念**：进程 = 运行中的程序 = 动态的、有生命周期的实体。

### 什么是线程？

线程，也称为轻量级进程，是进程中实际运作的单元，一个进程可以并发多个线程。

我们可以用一个比喻来理解：将CPU比作一个工厂，CPU的每个核心（Core）是工厂里的一个生产车间，而每个线程就是车间里真正干活的工人。例如，一颗“四核八线程”的CPU，就相当于一个有4个车间，每个车间有2个工人的工厂。

对于运维工作而言，我们只需要理解线程是进程内部真正执行任务的最小单位即可，无需像开发人员那样深入探究。

**核心概念**：线程 = 进程内的执行单元 = “真正干活的”。

### 使用 `pstree` 命令查看进程树 🌳

`pstree` 命令以树状结构显示进程信息，清晰地展示了进程之间的父子关系。

![](img/a33effd18a0cb2476a571b9d46075f08_52.png)

![](img/a33effd18a0cb2476a571b9d46075f08_54.png)

![](img/a33effd18a0cb2476a571b9d46075f08_56.png)

首先，确保系统已安装该工具：
```bash
yum install -y psmisc
```

![](img/a33effd18a0cb2476a571b9d46075f08_58.png)

![](img/a33effd18a0cb2476a571b9d46075f08_60.png)

![](img/a33effd18a0cb2476a571b9d46075f08_61.png)

安装后，直接运行 `pstree`：
```bash
pstree
```
输出结果以树状图形式显示，根进程通常是 `systemd` (PID为1)，它是所有进程的“祖先”（父进程）。

以下是 `pstree` 的常用选项：
*   `-p`：显示每个进程的PID。
    ```bash
    pstree -p
    ```
*   `-u`：显示进程所属的用户名。
*   `-a`：显示完整的命令行。可以结合用户名查看特定用户运行的进程。
    ```bash
    pstree -a testuser # 查看 testuser 用户运行的进程树
    ```

### 使用 `ps` 命令查看进程快照 📸

![](img/a33effd18a0cb2476a571b9d46075f08_71.png)

![](img/a33effd18a0cb2476a571b9d46075f08_73.png)

![](img/a33effd18a0cb2476a571b9d46075f08_75.png)

`ps` 命令用于查看当前时刻的进程快照，是静态的。最常用的组合是 `ps aux`。

`ps aux` 命令会列出系统所有进程的详细信息。其输出包含多列，每列含义如下：

![](img/a33effd18a0cb2476a571b9d46075f08_77.png)

![](img/a33effd18a0cb2476a571b9d46075f08_79.png)

![](img/a33effd18a0cb2476a571b9d46075f08_81.png)

| 列名 | 含义 |
| :--- | :--- |
| USER | 进程所属的用户 |
| PID | 进程ID |
| %CPU | 进程占用CPU百分比 |
| %MEM | 进程占用物理内存百分比 |
| VSZ | 进程使用的虚拟内存大小（KB） |
| RSS | 进程占用的固定内存大小（KB） |
| TTY | 进程在哪个终端运行。`?` 表示内核调用，`ttyN` 表示本地终端，`pts/N` 表示远程终端。 |
| STAT | 进程状态。常见状态：`R` (运行), `S` (休眠), `T` (停止), `Z` (僵尸), `+` (前台进程组)。 |
| START | 进程启动时间 |
| TIME | 进程占用CPU的总时间 |
| COMMAND | 启动进程的命令行 |

![](img/a33effd18a0cb2476a571b9d46075f08_83.png)

**进程状态 `STAT` 详解**：
*   **R (Running)**：进程正在运行或在运行队列中等待。
*   **S (Sleeping)**：进程处于休眠状态，等待某个事件完成。
*   **T (Stopped)**：进程被停止（如通过 `Ctrl+Z`）。
*   **Z (Zombie)**：僵尸进程。指子进程已终止，但其父进程尚未回收其资源。需要关注并处理。
*   **+**：位于前台进程组。

`ps aux` 常与 `grep` 结合，用于过滤查找特定进程：
```bash
ps aux | grep sshd
```

### 使用 `ps -ef` 命令查看进程

`ps -ef` 是另一种风格的命令，同样用于查看所有进程。其中 `-e` 显示所有进程，`-f` 显示完整格式信息。

![](img/a33effd18a0cb2476a571b9d46075f08_85.png)

![](img/a33effd18a0cb2476a571b9d46075f08_87.png)

![](img/a33effd18a0cb2476a571b9d46075f08_89.png)

一个特别有用的格式是 `ps -elf`，它会多显示一列 `PPID`，即父进程的PID。这在排查僵尸进程时非常有用，因为需要结束其父进程来回收资源。

![](img/a33effd18a0cb2476a571b9d46075f08_91.png)

查找僵尸进程及其父进程的命令如下：
```bash
ps -ef | grep -E ‘Z|PPID’ | grep -v grep
```
或者使用更精确的过滤：
```bash
ps -eo pid,ppid,stat,cmd | grep -E ‘Z|PPID’
```

![](img/a33effd18a0cb2476a571b9d46075f08_93.png)

### 关于僵尸进程的处理 🧟

![](img/a33effd18a0cb2476a571b9d46075f08_95.png)

![](img/a33effd18a0cb2476a571b9d46075f08_97.png)

僵尸进程是已终止但资源未被父进程回收的进程。处理方法是终止其父进程，之后系统父进程（`systemd`, PID 1）会接管并清理它。如果父进程是 `systemd` 或终止父进程无效，则可能需要重启系统。
