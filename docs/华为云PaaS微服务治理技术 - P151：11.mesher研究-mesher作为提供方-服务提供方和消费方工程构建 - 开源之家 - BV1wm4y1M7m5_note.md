# 华为云PaaS微服务治理技术：P151：11.mesher研究-mesher作为提供方-服务提供方和消费方工程构建 🛠️

在本节课中，我们将学习如何构建一个完整的测试场景，其中Mesher作为服务提供方的代理。我们将分别构建一个不具备微服务能力的普通服务提供方，以及一个基于CSE框架、具备微服务能力的服务消费方，并编写它们之间的调用代码。

## 概述

![](img/1ce6193894850398c29f8b69c7c55c0c_3.png)

我们将按照Mesher作为提供方的架构图，依次完成服务提供方和消费方程序的编写与配置。核心目标是让具备微服务能力的消费方，能够通过Mesher调用到不具备微服务能力的普通服务。

![](img/1ce6193894850398c29f8b69c7c55c0c_5.png)

## 构建服务提供方

上一节我们介绍了Mesher作为提供方的架构图，本节中我们来看看如何构建图中的服务提供方。

![](img/1ce6193894850398c29f8b69c7c55c0c_7.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_9.png)

服务提供方本身不具备微服务能力，因此我们只需开发一个普通的Spring Boot工程即可。

![](img/1ce6193894850398c29f8b69c7c55c0c_11.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_12.png)

以下是具体步骤：

![](img/1ce6193894850398c29f8b69c7c55c0c_14.png)

1.  在已有的普通Spring Boot工程中，创建一个新的Controller类，例如 `ProviderController`。
2.  在该Controller中定义一个简单的REST接口，例如路径为 `/hello`。
3.  该接口接收一个参数（如 `id`），并返回一个简单的JSON数据。

**提供方示例代码：**
```java
@RestController
public class ProviderController {
    @RequestMapping("/hello")
    public String sayHello(@RequestParam String id) {
        return "{\"message\": \"Hello from Provider, id: \" + id}";
    }
}
```

![](img/1ce6193894850398c29f8b69c7c55c0c_16.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_17.png)

构建完成后，我们可以直接启动该Spring Boot应用（假设端口为40000）并进行测试，通过浏览器或工具访问 `http://localhost:40000/hello?id=test`，确认接口可以正常返回数据。这证明我们的服务提供方已经就绪。

![](img/1ce6193894850398c29f8b69c7c55c0c_19.png)

## 构建服务消费方

![](img/1ce6193894850398c29f8b69c7c55c0c_21.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_23.png)

服务提供方构建完成后，接下来我们需要构建服务消费方。与提供方不同，消费方需要具备微服务能力，以便能够接入CSE服务注册与发现中心。

我们将在一个已经接入CSE框架的微服务工程（例如 `portalview`）中，添加一个作为消费方的Controller。

![](img/1ce6193894850398c29f8b69c7c55c0c_25.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_26.png)

以下是具体步骤：

![](img/1ce6193894850398c29f8b69c7c55c0c_28.png)

1.  在目标微服务工程中，创建一个新的Controller类。
2.  使用 `@RequestMapping` 和 `@RestSchema` 注解将其定义为CSE框架的REST接口。
3.  在接口方法中，编写通过CSE框架发起服务调用的代码。

![](img/1ce6193894850398c29f8b69c7c55c0c_30.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_31.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_33.png)

**消费方示例代码：**
```java
@RestSchema(schemaId = "consumerController")
@RequestMapping(path = "/")
public class ConsumerController {

    @Autowired
    private RestTemplate restTemplate;

    @RequestMapping("/invokeHello")
    public String invokeHello(@RequestParam String id) {
        // 构建要调用的服务URL
        String url = "cse://provider-service/hello?id=" + id;
        // 通过RestTemplate发起调用
        String result = restTemplate.getForObject(url, String.class);
        return result;
    }
}
```

在这段代码中，最关键的部分是构建调用URL：`cse://provider-service/hello`。这表示消费方将通过CSE框架，去调用名为 `provider-service` 的服务的 `/hello` 接口。这里的 `provider-service` 正是后续将由Mesher代理并注册到CSE中心的服务名。

## 当前进展与后续工作

![](img/1ce6193894850398c29f8b69c7c55c0c_35.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_37.png)

至此，我们已完成了服务提供方和消费方的基础代码构建。

*   **服务提供方**：是一个运行在端口40000的普通Spring Boot应用，提供了 `/hello` 接口。
*   **服务消费方**：是一个已接入CSE的微服务，其 `ConsumerController` 中包含了调用 `cse://provider-service/hello` 的逻辑。

![](img/1ce6193894850398c29f8b69c7c55c0c_40.png)

![](img/1ce6193894850398c29f8b69c7c55c0c_41.png)

然而，目前还无法进行测试。因为名为 `provider-service` 的服务及其 `/hello` 接口契约并未在CSE服务中心注册。消费方无法找到这个服务目标。这正是我们下一阶段需要完成的工作：配置并启动Mesher，让它代理我们的普通服务提供方，并将其以 `provider-service` 的名义注册到CSE服务中心，从而打通整个调用链路。

![](img/1ce6193894850398c29f8b69c7c55c0c_43.png)

## 总结

![](img/1ce6193894850398c29f8b69c7c55c0c_47.png)

本节课中我们一起学习了在Mesher作为提供方的场景下，如何构建服务提供方和消费方工程。
*   服务提供方是一个**普通Spring Boot应用**，无需微服务能力。
*   服务消费方是一个**基于CSE框架的微服务**，具备服务发现和调用能力。
*   消费方通过 **`cse://{serviceName}/{interface}`** 格式的URL来发起调用，其中的 `serviceName` 将由Mesher后续提供。
