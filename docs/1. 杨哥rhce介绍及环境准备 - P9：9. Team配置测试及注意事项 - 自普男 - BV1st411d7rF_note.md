## 环境准备 🛠️

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_11.png)

上一节我们介绍了链路聚合的基本概念，本节中我们来看看如何为实验准备环境。在练习环境中，虚拟机默认可能没有足够的网卡，因此需要手动添加。考试时，所有环境均已预先配置好，无需此步骤。

以下是添加虚拟网卡的步骤：
1.  在物理主机上打开虚拟化管理工具（如 `virt-manager`）。
2.  选择目标虚拟机，点击“添加硬件”。
3.  选择“网络设备”，并选择“RTL8139”或“e1000”这类全虚拟化网卡驱动，以确保网卡名称为 `ethX` 格式。
4.  为每台虚拟机添加两块网卡（`eth1` 和 `eth2`）。

完成添加后，在服务器上使用 `ip addr` 命令应能看到 `eth1` 和 `eth2` 网卡。

## 配置Team连接 ⚙️

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_18.png)

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_20.png)

环境准备好后，我们开始配置Team。我们将使用图形化工具 `nm-connection-editor` 进行配置。如果远程图形连接有问题，可以在服务器本地执行 `startx` 命令启动图形界面。

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_22.png)

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_23.png)

以下是配置Team连接的具体步骤：
1.  打开 `nm-connection-editor`。
2.  点击“添加”按钮，选择连接类型为“Team”。
3.  在“常规”选项卡中，勾选“自动连接”。
4.  在“Team”配置界面，点击“添加”按钮，为Team添加两个以太网接口成员。
    *   第一个接口：选择 `eth1`，并确保勾选“自动连接”。
    *   第二个接口：选择 `eth2`，并确保勾选“自动连接”。
5.  在“IPv4设置”中，将方法改为“手动”，并添加指定的IP地址（例如 `192.168.0.1/24`）。
6.  **关键步骤**：在“Team配置”的JSON字段中，定义Team的工作模式。为实现主备（active-backup）功能，需输入以下配置：
    ```json
    {"runner": {"name": "activebackup"}}
    ```
7.  点击“保存”完成配置。

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_25.png)

按照相同步骤，在另一台服务器上配置Team，使用IP地址 `192.168.0.2/24`。

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_27.png)

## 验证与测试 ✅

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_29.png)

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_30.png)

配置完成后，我们需要验证Team是否正常工作。上一节我们完成了配置，本节中我们来看看如何进行验证。

以下是验证Team状态的命令：
*   查看IP地址：`ip addr show team0`
*   查看Team详细状态：`teamdctl team0 state`
    *   此命令应显示运行模式为 `activebackup`，且两个端口状态均为 `up`。
*   测试连通性：使用 `ping` 命令测试两台服务器之间的网络连通性。
    *   例如：`ping 192.168.0.2`

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_32.png)

## 故障模拟与注意事项 ⚠️

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_34.png)

为了确保配置符合题目要求（一个接口失效时另一个仍能工作），可以进行故障模拟测试。

以下是测试主备切换的方法：
1.  断开一个端口（例如 `eth1`）的连接（在虚拟化管理器中禁用该网卡或执行 `ip link set eth1 down`）。
2.  立即使用 `teamdctl team0 state` 命令观察状态变化，应显示 `eth1` 为 `down`，而 `eth2` 接管流量。
3.  此时 `ping` 测试应仍然成功，证明高可用性生效。
4.  恢复 `eth1` 连接，观察状态是否恢复。

![](img/6c7592b5ee47c3a2c5e24bb7940f75e2_36.png)

**重要注意事项**：
*   考试时无需手动添加网卡。
*   配置中务必为Team接口和每个成员接口勾选“自动连接”。
*   JSON配置的格式必须正确，引号和括号需完整。
*   配置文件位于 `/etc/sysconfig/network-scripts/` 目录下（如 `ifcfg-team0`, `ifcfg-team0-eth1`）。
