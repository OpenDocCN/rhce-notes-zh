# 华为认证ICT学院HCIA/HCIP-Datacom教程：P41：第3册-第2章-3-链路聚合 🔗

在本节课中，我们将要学习一种在局域网中极为常见的高可靠性技术——链路聚合。我们将了解它的作用、两种工作模式以及基础的配置方法。

## 概述 📖

链路聚合是一种端口冗余技术，它通过将多条物理链路捆绑成一条逻辑链路，以提高设备间的总带宽和可靠性。本节将从三个方面展开：链路聚合的介绍、工作模式以及简单配置。

## 链路聚合介绍

上一节我们介绍了其他高可靠性技术，本节中我们来看看链路聚合。在局域网中，为了增加交换机之间的链路带宽，例如在接入层与汇聚层之间，我们可能会连接多条线路。

![](img/856561f5f2ce310522136b7dddd3c884_1.png)

然而，在二层网络环境中，由于生成树协议（STP）会阻塞冗余端口以防止环路，即使连接了多条链路，实际上也只有一条链路处于转发状态，无法达到增加带宽的目的。

![](img/856561f5f2ce310522136b7dddd3c884_3.png)

在三层网络环境中，虽然不存在生成树阻塞的问题，但通过配置多条独立链路会增加IP地址管理的复杂性，每条链路都需要一对独立的IP地址，造成资源浪费且扩展性差。

为了解决上述问题，链路聚合技术应运而生。它的核心思想是将多条物理链路捆绑成一个逻辑的“以太通道”（Eth-Trunk）。对于上层协议和网络管理而言，这个通道就是一条链路。

![](img/856561f5f2ce310522136b7dddd3c884_5.png)

**公式/概念**：
*   **物理链路**：`接口1 + 接口2 + 接口3 + ...`
*   **逻辑链路**：`Eth-Trunk = 捆绑(接口1, 接口2, 接口3, ...)`

这样做的好处是：
1.  **增加带宽**：逻辑链路的带宽是所有成员链路带宽的总和。
2.  **提高可靠性**：当某条成员链路故障时，流量可以自动切换到其他正常链路上。
3.  **负载分担**：流量可以在所有成员链路上进行分担，提升效率。
4.  **简化管理**：在三层应用中，只需在逻辑链路上配置一个IP地址，简化了配置。

![](img/856561f5f2ce310522136b7dddd3c884_7.png)

对于二层网络，生成树协议会将整个Eth-Trunk视为一条逻辑链路，因此其内部的所有物理端口都不会被阻塞，从而实现了带宽的叠加且无环路风险。

![](img/856561f5f2ce310522136b7dddd3c884_9.png)

## 链路聚合模式

了解了链路聚合的基本概念后，本节我们来看看它的两种主要工作模式。链路聚合有两种模式：手动负载分担模式和基于LACP（链路聚合控制协议）的模式。

以下是两种模式的详细介绍：

*   **手动负载分担模式**：
    *   管理员需要手工将物理接口加入Eth-Trunk。配置简单，但无法检测到链路层错误、链路错连等问题。
    *   一旦配置完成，所有成员链路都会参与数据转发，实现负载分担。没有主备链路的概念。

![](img/856561f5f2ce310522136b7dddd3c884_11.png)

*   **LACP模式**：
    *   设备之间通过交互LACPDU（链路聚合控制协议数据单元）报文来协商聚合。这种模式能检测链路状态和对端信息，可靠性更高。
    *   LACP模式可以配置活动接口的上限，实现**M:N**备份，即M条链路处于活动（Active）转发状态，N条链路处于备份（Inactive）状态。当活动链路故障时，备份链路可以接替工作。

**代码/概念**：
*   **手动模式配置关键命令**：`mode manual load-balance`
*   **LACP模式配置关键命令**：`mode lacp-static`

## 链路聚合简单配置

![](img/856561f5f2ce310522136b7dddd3c884_15.png)

![](img/856561f5f2ce310522136b7dddd3c884_17.png)

![](img/856561f5f2ce310522136b7dddd3c884_19.png)

上一节我们介绍了链路聚合的两种模式，本节中我们通过一个简单的例子来看看如何进行配置。配置主要分为创建Eth-Trunk、配置工作模式、将物理接口加入Eth-Trunk等步骤。

![](img/856561f5f2ce310522136b7dddd3c884_21.png)

![](img/856561f5f2ce310522136b7dddd3c884_23.png)

以下是手动负载分担模式的配置示例（以华为设备为例）：

![](img/856561f5f2ce310522136b7dddd3c884_25.png)

1.  **创建Eth-Trunk接口**：
    ```bash
    system-view
    interface eth-trunk 1
    ```
2.  **配置Eth-Trunk为Trunk类型并允许VLAN通过**（针对二层聚合）：
    ```bash
    port link-type trunk
    port trunk allow-pass vlan all
    ```
3.  **将物理接口加入Eth-Trunk**：
    ```bash
    quit
    interface gigabitethernet 0/0/1
    eth-trunk 1
    interface gigabitethernet 0/0/2
    eth-trunk 1
    interface gigabitethernet 0/0/3
    eth-trunk 1
    ```

![](img/856561f5f2ce310522136b7dddd3c884_27.png)

![](img/856561f5f2ce310522136b7dddd3c884_29.png)

配置完成后，可以使用 `display eth-trunk 1` 命令查看聚合组的状态，确认所有成员接口都已正常加入并处于Up状态。

![](img/856561f5f2ce310522136b7dddd3c884_31.png)

对于LACP模式，配置略有不同，需要在Eth-Trunk接口视图下指定模式，并可以设置活动接口上限和抢占等高级特性。

![](img/856561f5f2ce310522136b7dddd3c884_33.png)

![](img/856561f5f2ce310522136b7dddd3c884_35.png)

以下是LACP模式的部分关键配置：

1.  **创建Eth-Trunk并指定为LACP模式**：
    ```bash
    interface eth-trunk 1
    mode lacp-static
    ```
2.  **（可选）设置最大活动链路数**（实现M:N备份）：
    ```bash
    max active-linknumber 2  # 设置最多2条链路处于活动状态
    ```
3.  **（可选）配置系统优先级和端口优先级**，用于在聚合组中选举主动端和确定活动链路。
4.  **（可选）开启抢占功能**：当故障链路恢复且优先级更高时，可以重新成为活动链路。
    ```bash
    lacp preempt enable
    lacp preempt delay 10  # 设置抢占延迟为10秒
    ```
