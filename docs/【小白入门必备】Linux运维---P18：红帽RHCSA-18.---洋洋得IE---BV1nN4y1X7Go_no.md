## 归属关系与基本权限

![](img/60df99bdc72cab28d70d36628177a599_11.png)

上一节我们介绍了用户和组的管理，本节中我们来看看如何通过权限来控制这些用户对文件的操作。Linux系统的核心特点之一是“一切皆文件”，因此权限管理本质上就是控制用户对文件的访问。

![](img/60df99bdc72cab28d70d36628177a599_13.png)

![](img/60df99bdc72cab28d70d36628177a599_15.png)

![](img/60df99bdc72cab28d70d36628177a599_17.png)

![](img/60df99bdc72cab28d70d36628177a599_18.png)

任何文件或目录的归属关系都分为三类，用首字母表示：
*   **U (User)**：代表文件的所有者，通常是文件的创建者。
*   **G (Group)**：代表文件的所属组。
*   **O (Other)**：代表既不是所有者，也不在所属组内的其他所有用户。

我们可以使用 `ls -l` 命令查看文件的详细信息，其中包含了归属关系和权限。

```bash
[root@localhost ~]# ls -l hello.txt
-rw-r--r--. 1 root root 0 Aug 10 15:30 hello.txt
```

输出结果解读如下：
*   **`-rw-r--r--`**：第一部分是权限标识。第一个字符 `-` 表示这是一个普通文件（`d` 表示目录，`l` 表示链接文件）。后面的9个字符每3个为一组，分别代表 **所有者(U)**、**所属组(G)**、**其他人(O)** 的权限。
*   **`root`**：第一个 `root` 表示文件的所有者是 `root` 用户。
*   **`root`**：第二个 `root` 表示文件的所属组是 `root` 组。

权限位 `rwx` 的含义如下：
*   **r (Read，读取)**：对于文件，表示可以查看文件内容；对于目录，表示可以列出目录内的文件列表。
*   **w (Write，写入)**：对于文件，表示可以修改文件内容（增、删、改）；对于目录，表示可以在目录内创建、删除、重命名文件或子目录。
*   **x (Execute，执行)**：对于文件，表示可以运行该文件（如脚本或程序）；对于目录，表示可以进入 (`cd`) 该目录。

如果某个权限位是 `-`，则表示没有对应的权限。例如 `rw-` 表示有读和写权限，但没有执行权限。

## 修改文件归属关系 (chown)

在设置权限前，有时我们需要先修改文件的归属关系。`chown` 命令用于更改文件的所有者和所属组。

![](img/60df99bdc72cab28d70d36628177a599_24.png)

以下是 `chown` 命令的几种常用格式：

![](img/60df99bdc72cab28d70d36628177a599_26.png)

![](img/60df99bdc72cab28d70d36628177a599_28.png)

```bash
# 同时修改所有者和所属组
chown tom:devops hello.txt

# 仅修改所有者
chown root hello.txt

# 仅修改所属组 (注意冒号前没有所有者)
chown :devops hello.txt

# 递归修改目录及其内部所有内容的归属关系
chown -R tom:devops /data/project/
```

**示例场景**：公司有运维部（`devops`组）和开发部（`develop`组）。我们希望将 `/data/scripts/` 目录及其下所有文件的所有权交给运维组。可以这样做：
1.  创建运维组：`groupadd devops`
2.  将运维员工（如用户 `tom`）加入该组：`gpasswd -a tom devops`
3.  修改目录归属：`chown -R :devops /data/scripts/`

这样，`tom` 用户就通过组继承了对 `/data/scripts/` 目录的权限（具体权限还需通过 `chmod` 设置）。

## 修改文件权限 (chmod)

`chmod` 命令是修改文件或目录权限的核心工具。它有两种主要的权限表示法：**字母表示法**和**数字表示法**。

### 字母表示法

字母表示法直观易懂，格式为：`chmod [ugoa][+-=][rwx] 文件/目录名`。

*   **身份指定符**：
    *   `u`：所有者 (User)
    *   `g`：所属组 (Group)
    *   `o`：其他人 (Other)
    *   `a`：所有人 (All)，即 `ugo` 的总和，是默认选项。
*   **操作符**：
    *   `+`：添加权限
    *   `-`：移除权限
    *   `=`：直接设定权限（覆盖原有）
*   **权限指定符**：`r`， `w`， `x`

以下是具体操作示例：

```bash
# 为所有者添加执行权限
chmod u+x backup.sh

# 为所属组移除写权限
chmod g-w document.txt

# 为其他人设置只读权限（等于移除写和执行）
chmod o=r document.txt

# 同时为所有者和所属组添加写权限
chmod ug+w report.txt

# 使用逗号分隔，同时进行多项权限操作
chmod u+x,g=rw,o-rwx script.sh
```

![](img/60df99bdc72cab28d70d36628177a599_30.png)

### 数字表示法

![](img/60df99bdc72cab28d70d36628177a599_32.png)

数字表示法更为简洁，使用三个八进制数字分别代表 **所有者**、**所属组**、**其他人** 的权限。每个数字是 `r`、`w`、`x` 对应值的和：
*   `r` (读) = **4**
*   `w` (写) = **2**
*   `x` (执行) = **1**
*   `-` (无) = **0**

计算方式：所需权限值相加。例如：
*   `rw-` = 4 (r) + 2 (w) + 0 (x) = **6**
*   `r-x` = 4 (r) + 0 (w) + 1 (x) = **5**
*   `rwx` = 4+2+1 = **7**

![](img/60df99bdc72cab28d70d36628177a599_34.png)

![](img/60df99bdc72cab28d70d36628177a599_36.png)

以下是常用权限的数字表示：

```bash
# 设置文件为常见权限：所有者可读可写(rw-)，所属组和其他人只读(r--)
chmod 644 index.html

# 设置脚本文件：所有者可读可写可执行(rwx)，所属组和其他人可读可执行(r-x)
chmod 755 startup.sh

![](img/60df99bdc72cab28d70d36628177a599_38.png)

# 设置私有文件：只有所有者可读可写(rw-)，组和其他人无权限(---)
chmod 600 secret.key

![](img/60df99bdc72cab28d70d36628177a599_40.png)

# 设置共享目录：所有者有完全权限(rwx)，组内成员可读可执行(r-x)，其他人无权限(---)
chmod 750 shared_folder/
```

**重要提示**：使用数字表示法时，必须提供完整的三个数字。例如 `chmod 7 file.txt` 会被系统理解为 `chmod 007 file.txt`，导致所有者和所属组权限被清空。

## 文件与目录权限的差异

![](img/60df99bdc72cab28d70d36628177a599_46.png)

![](img/60df99bdc72cab28d70d36628177a599_48.png)

理解权限时，最关键的一点是区分 **文件权限** 和 **目录权限**。相同的 `r`、`w`、`x` 符号在两者身上意义完全不同。

![](img/60df99bdc72cab28d70d36628177a599_50.png)

| 权限 | 对文件的意义 | 对目录的意义 |
| :--- | :--- | :--- |
| **r (读)** | 可以读取文件内容 | 可以**列出**目录下的文件和子目录名（如 `ls` 命令） |
| **w (写)** | 可以修改文件内容 | 可以在目录内**创建、删除、重命名**文件和子目录 |
| **x (执行)** | 可以执行该文件（如脚本） | 可以**进入**该目录（如 `cd` 命令），并访问其内部元数据 |

![](img/60df99bdc72cab28d70d36628177a599_52.png)

**核心结论**：用户能否删除一个文件，**不取决于**他对这个文件本身的权限，而取决于他对该文件**所在目录**是否拥有 **`w`** 权限。

![](img/60df99bdc72cab28d70d36628177a599_54.png)

![](img/60df99bdc72cab28d70d36628177a599_56.png)

![](img/60df99bdc72cab28d70d36628177a599_58.png)

**验证实验**：
1.  用户 `tom` 对一个文件拥有 `rwx` 完全权限。
2.  但 `tom` 对该文件所在的父目录只有 `r-x` 权限（缺少 `w`）。
3.  此时，`tom` 可以打开并修改文件内容，但**无法删除**这个文件，因为他对目录没有删除（`w`）权限。

## 默认权限与umask

当我们创建新的文件或目录时，系统会赋予一个默认的权限。这个默认权限由 **umask（权限掩码）** 值决定。umask 指定了需要从默认权限中“扣除”的权限。

查看当前 umask 值：
```bash
# 以数字形式显示（八进制）
umask
# 输出示例：0022

# 以符号形式显示（更直观）
umask -S
# 输出示例：u=rwx,g=rx,o=rx
```

![](img/60df99bdc72cab28d70d36628177a599_66.png)

![](img/60df99bdc72cab28d70d36628177a599_68.png)

对于目录，默认权限是 `777` (即 `rwxrwxrwx`)；对于文件，默认权限是 `666` (即 `rw-rw-rw-`，文件默认没有执行权限是出于安全考虑)。创建时，系统会用默认权限 **减去** umask 值，得到最终权限。

**计算示例**（umask=022）：
*   **新建目录**：`777 - 022 = 755` -> `rwxr-xr-x`
*   **新建文件**：`666 - 022 = 644` -> `rw-r--r--`

修改 umask 值（通常只在当前Shell会话中临时生效）：
```bash
# 设置为更严格的权限，新建文件为600，目录为700
umask 077
```

## 特殊权限简介

![](img/60df99bdc72cab28d70d36628177a599_70.png)

除了基本的 `rwx`，Linux 还有三个特殊权限位，它们在权限字符串中占用执行位 (`x`) 的位置，并用特殊字符表示：

![](img/60df99bdc72cab28d70d36628177a599_72.png)

![](img/60df99bdc72cab28d70d36628177a599_74.png)

![](img/60df99bdc72cab28d70d36628177a599_76.png)

1.  **SUID (Set User ID)**：出现在**所有者**的执行位。
    *   **符号表示**：`s` (如果所有者原本有 `x` 权限) 或 `S` (如果所有者原本无 `x` 权限)。
    *   **作用**：当用户执行带有 SUID 位的文件时，在执行期间会暂时获得文件**所有者**的权限。典型例子是 `/usr/bin/passwd` 命令，普通用户执行它可以修改自己的密码（实际修改的是 `/etc/shadow` 文件，该文件只有 root 可写）。
    *   **设置**：`chmod u+s file` 或 `chmod 4xxx file` (数字法，`4`代表SUID)。

![](img/60df99bdc72cab28d70d36628177a599_78.png)

![](img/60df99bdc72cab28d70d36628177a599_80.png)

2.  **SGID (Set Group ID)**：出现在**所属组**的执行位。
    *   **符号表示**：`s` 或 `S`。
    *   **作用**：
        *   对**文件**：执行时暂时获得文件**所属组**的权限。
        *   对**目录**：在该目录下创建的任何新文件或子目录，其所属组会自动继承该目录的所属组，而不是创建者的默认组。常用于团队协作的共享目录。
    *   **设置**：`chmod g+s dir` 或 `chmod 2xxx dir` (数字法，`2`代表SGID)。

![](img/60df99bdc72cab28d70d36628177a599_82.png)

3.  **Sticky Bit (粘滞位)**：出现在**其他人**的执行位。
    *   **符号表示**：`t` 或 `T`。
    *   **作用**：仅对目录有效。设置了粘滞位的目录，用户只能删除或重命名**自己创建**的文件，而不能删除其他用户的文件。系统临时目录 `/tmp` 就是典型例子。
    *   **设置**：`chmod o+t /tmp` 或 `chmod 1xxx /tmp` (数字法，`1`代表粘滞位)。
