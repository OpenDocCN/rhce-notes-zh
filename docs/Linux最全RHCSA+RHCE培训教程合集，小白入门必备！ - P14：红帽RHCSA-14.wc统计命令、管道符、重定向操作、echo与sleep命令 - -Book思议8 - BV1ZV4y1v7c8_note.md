### wc统计命令 📊

`wc`命令用于统计文件中的行数、单词数和字节数。其基本语法为：
```bash
wc [选项] 文件名
```

以下是`wc`命令的常用选项：
*   `-l`：仅统计行数。
*   `-w`：仅统计单词数（以空格分隔的字符串）。
*   `-c`：仅统计字节数。

例如，统计`/etc/passwd`文件的行数：
```bash
wc -l /etc/passwd
```
这条命令会输出文件的总行数，这是`wc`命令最常用的功能。

### 管道符 `|` 🔄

上一节我们介绍了`wc`命令，本节中我们来看看管道符。管道符`|`的功能非常强大，它可以将前一个命令的输出结果，作为参数传递给后一个命令进行处理。

管道符的基本用法是：
```bash
命令A | 命令B
```

![](img/9421524ff35a8f7225121be8c064a0f2_19.png)

![](img/9421524ff35a8f7225121be8c064a0f2_21.png)

以下是管道符的应用示例：
*   **查看文件前十行中的后五行**：
    ```bash
    head -10 /etc/passwd | tail -5
    ```
    这个组合先使用`head`获取文件前十行，然后通过管道将结果传给`tail`，最终显示这十行中的后五行。
*   **统计网卡配置信息的行数**：
    ```bash
    ifconfig ens32 | head -2 | wc -l
    ```
    这个例子展示了管道的链式使用：先获取`ens32`网卡信息，再取前两行，最后统计行数。

![](img/9421524ff35a8f7225121be8c064a0f2_23.png)

管道符允许无限连接命令，只要逻辑需要，可以一直“管道”下去。

### 重定向操作 📥📤

![](img/9421524ff35a8f7225121be8c064a0f2_33.png)

管道符用于连接命令，而重定向操作则用于控制命令的输入和输出流向，特别是与文件进行交互。

![](img/9421524ff35a8f7225121be8c064a0f2_35.png)

![](img/9421524ff35a8f7225121be8c064a0f2_37.png)

![](img/9421524ff35a8f7225121be8c064a0f2_39.png)

以下是主要的重定向操作符及其功能：

![](img/9421524ff35a8f7225121be8c064a0f2_41.png)

*   **`>` 覆盖重定向**：将命令的**正确输出结果**覆盖写入到指定文件。如果文件不存在，则创建它。
    ```bash
    ifconfig ens32 | head -2 > net_ens32.txt
    ```

*   **`>>` 追加重定向**：将命令的**正确输出结果**追加到指定文件的末尾，不覆盖原有内容。
    ```bash
    free -h >> system_info.txt
    ```

![](img/9421524ff35a8f7225121be8c064a0f2_43.png)

*   **`2>` 错误覆盖重定向**：将命令的**错误输出信息**覆盖写入到指定文件。
    ```bash
    cat /nonexistent_file 2> error.log
    ```

![](img/9421524ff35a8f7225121be8c064a0f2_45.png)

![](img/9421524ff35a8f7225121be8c064a0f2_47.png)

*   **`2>>` 错误追加重定向**：将命令的**错误输出信息**追加到指定文件。
    ```bash
    ls /invalid_path 2>> error.log
    ```

![](img/9421524ff35a8f7225121be8c064a0f2_49.png)

![](img/9421524ff35a8f7225121be8c064a0f2_51.png)

*   **`&>` 全部覆盖重定向**：将命令的**正确和错误输出**全部覆盖写入到指定文件。
    ```bash
    cat /etc/hostname /fakefile &> all_output.log
    ```

![](img/9421524ff35a8f7225121be8c064a0f2_53.png)

*   **`&>>` 全部追加重定向**：将命令的**正确和错误输出**全部追加到指定文件。
    ```bash
    ls /etc/passwd /fakefile &>> all_output.log
    ```

![](img/9421524ff35a8f7225121be8c064a0f2_55.png)

> **应用场景提示**：在企业环境中，重定向（尤其是错误重定向）非常有用。例如，在编写脚本批量检测服务器是否在线时，可以将无法`ping`通的服务器IP（错误输出）收集到日志文件中，便于快速定位故障机器。

### echo与sleep命令 ⏳

![](img/9421524ff35a8f7225121be8c064a0f2_59.png)

最后，我们来了解两个看似简单但后期非常有用的命令：`echo`和`sleep`。

![](img/9421524ff35a8f7225121be8c064a0f2_61.png)

![](img/9421524ff35a8f7225121be8c064a0f2_63.png)

**`echo`命令**用于在终端输出指定的字符串或变量。
```bash
echo “Hello World”
```
结合重定向，`echo`可以快速创建或修改文件内容。例如，快速设置主机名：
```bash
echo “new_hostname” > /etc/hostname
```
一个空`echo`命令加覆盖重定向，可以清空文件：
```bash
echo “” > file.txt
# 或更常见的
> file.txt
```

![](img/9421524ff35a8f7225121be8c064a0f2_65.png)

![](img/9421524ff35a8f7225121be8c064a0f2_67.png)

**`sleep`命令**用于使当前进程（或脚本）暂停（睡眠）一段时间。
```bash
sleep 时间
```
时间单位可以是秒（默认）、分钟（`m`）、小时（`h`）、天（`d`）。
```bash
sleep 5    # 休眠5秒
sleep 0.1  # 休眠0.1秒
```
`sleep`命令在编写Shell脚本时尤为重要。它可以控制命令的执行节奏，避免CPU因循环任务而过载。例如，在快速循环中插入`sleep 0.1`，可以显著降低CPU使用率，让系统有喘息之机。

### 总结 🎯

![](img/9421524ff35a8f7225121be8c064a0f2_79.png)

![](img/9421524ff35a8f7225121be8c064a0f2_81.png)

本节课中我们一起学习了Linux中的几个核心工具：
1.  **`wc`命令**：用于统计文件的行数、单词数和字节数。
2.  **管道符 `|`**：用于连接多个命令，将前一个命令的输出作为后一个命令的输入，实现功能组合。
3.  **重定向操作**：包括覆盖（`>`）、追加（`>>`）、错误重定向（`2>`）等，用于灵活控制命令的输入输出与文件的交互。
4.  **`echo`命令**：输出文本或变量，常与重定向配合用于文件操作。
5.  **`sleep`命令**：暂停进程执行，常用于脚本中以控制执行间隔，减轻系统负载。
