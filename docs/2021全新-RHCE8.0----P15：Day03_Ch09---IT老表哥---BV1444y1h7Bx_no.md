# 红帽认证入门教程：03：服务管理 🔧

![](img/53e4a818daf8695e993b3b4c52b7b97a_0.png)

在本节课中，我们将要学习Linux系统中的服务管理。我们将重点了解`systemctl`命令，它是红帽7及以后版本中管理服务的核心工具。通过本课，你将理解服务与进程的区别，掌握使用`systemctl`管理服务的基本操作。

## 服务与进程的概念

上一节我们介绍了进程管理，本节中我们来看看服务管理。在Linux系统中，我们通常通过进程来处理任务。例如，`httpd`服务是通过执行`/usr/sbin/httpd`这个二进制文件来创建进程的。

进程负责处理具体任务，服务则是通过进程来启动的。那么，为什么需要“服务”这个概念呢？当我们直接通过进程管理时，例如修改配置文件后使用`kill -HUP`重启进程，或者使用`kill -2`、`kill -15`停止进程，这些操作都是可行的。但是，进程管理无法实现“开机自动启动”这样的功能。

为了解决进程管理在自动化、依赖关系等方面的不足，系统引入了“服务”的管理方式。

![](img/53e4a818daf8695e993b3b4c52b7b97a_2.png)

## Systemd 与 Systemctl

![](img/53e4a818daf8695e993b3b4c52b7b97a_4.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_6.png)

在红帽7以前的版本中，使用`service`模块来管理服务。从红帽7开始，服务管理交给了`systemd`。`systemd`是系统启动的第一个进程，它负责系统启动、一般服务的启动与管理，相当于一个功能升级的“管家”。

![](img/53e4a818daf8695e993b3b4c52b7b97a_8.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_10.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_12.png)

管理`systemd`的工具是`systemctl`命令。在红帽7之后，我们统一使用`systemctl`来管理服务。

![](img/53e4a818daf8695e993b3b4c52b7b97a_14.png)

## 单元配置文件

`systemctl`管理服务时，需要读取“单元配置文件”。这些文件定义了如何启动和管理对应的进程。

![](img/53e4a818daf8695e993b3b4c52b7b97a_16.png)

单元配置文件通常存放在以下目录中：
*   `/usr/lib/systemd/system/`
*   `/etc/systemd/system/`

![](img/53e4a818daf8695e993b3b4c52b7b97a_18.png)

这些文件以后缀区分类型，例如：
*   `.service`：一般服务
*   `.target`：启动目标（与开机启动相关）
*   `.socket`：套接字

![](img/53e4a818daf8695e993b3b4c52b7b97a_20.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_22.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_23.png)

例如，我们安装`httpd`后，可以在`/usr/lib/systemd/system/`目录下找到`httpd.service`文件。这个文件定义了服务的描述、依赖关系、启动命令、重启方式等信息。在初学阶段，我们不需要掌握如何编写这个文件，但需要知道它的存在和作用。

![](img/53e4a818daf8695e993b3b4c52b7b97a_27.png)

## 服务管理基本操作

![](img/53e4a818daf8695e993b3b4c52b7b97a_32.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_34.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_35.png)

`systemctl`命令功能强大，但对于初学者，我们首先掌握最常用的几个操作即可。

![](img/53e4a818daf8695e993b3b4c52b7b97a_37.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_39.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_41.png)

以下是管理服务状态的基本命令：

![](img/53e4a818daf8695e993b3b4c52b7b97a_42.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_44.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_46.png)

*   **启动服务**：`systemctl start 服务名`
*   **停止服务**：`systemctl stop 服务名`
*   **重启服务**：`systemctl restart 服务名`
*   **重新加载配置文件**：`systemctl reload 服务名`
*   **查看服务状态**：`systemctl status 服务名`

![](img/53e4a818daf8695e993b3b4c52b7b97a_48.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_50.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_52.png)

**注意**：在使用`systemctl`操作服务（如`httpd`）时，可以省略`.service`后缀。例如，`systemctl restart httpd` 等同于 `systemctl restart httpd.service`。

![](img/53e4a818daf8695e993b3b4c52b7b97a_54.png)

以下是管理服务开机自启的命令：

![](img/53e4a818daf8695e993b3b4c52b7b97a_56.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_58.png)

*   **设置服务开机自启**：`systemctl enable 服务名`
*   **禁止服务开机自启**：`systemctl disable 服务名`
*   **查看服务是否开机自启**：`systemctl is-enabled 服务名`

![](img/53e4a818daf8695e993b3b4c52b7b97a_60.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_62.png)

## 其他实用命令

除了管理单个服务，`systemctl`还有一些查看系统信息的命令。

![](img/53e4a818daf8695e993b3b4c52b7b97a_64.png)

*   列出所有已加载的单元：`systemctl list-units`
*   列出所有已安装的单元文件：`systemctl list-unit-files`
*   查看单元的依赖关系：`systemctl list-dependencies 单元名`
*   查看`systemctl`支持的所有单元类型：`systemctl -t help`
*   获取`systemctl`命令的详细帮助：`systemctl --help`

![](img/53e4a818daf8695e993b3b4c52b7b97a_66.png)

![](img/53e4a818daf8695e993b3b4c52b7b97a_68.png)

## 总结

![](img/53e4a818daf8695e993b3b4c52b7b97a_70.png)

本节课中我们一起学习了Linux的服务管理。我们理解了服务与进程的区别，认识了红帽7/8系统中负责管理服务的`systemd`及其工具`systemctl`。我们知道了服务由“单元配置文件”定义，并重点掌握了使用`systemctl`进行服务启停、状态查看以及设置开机自启等核心操作。这些是后续学习和日常系统管理中最常用到的技能。
