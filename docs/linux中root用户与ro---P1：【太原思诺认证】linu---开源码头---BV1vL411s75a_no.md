Linux入门：P1：理解root用户与root权限

在本节课中，我们将要学习Linux系统中至关重要的两个概念：root用户与root权限。理解它们是安全、高效管理系统的基础。

---

### **root用户与超级管理员权限**

root用户是系统中的超级管理员，其用户ID（UID）等于零。除了系统特殊文件等少数情况，root用户能够操作任何文件和服务进程。

但是，在root组里的普通用户并不会自动拥有root权限，他们和其他普通用户一样，受限于基本权限和用户组权限控制。

---

### **用户切换命令：`su` 与 `su -`**

在Linux中切换用户的常用命令是 `su` 或 `su -`。这两者之间存在重要区别。

`su` 命令可以将当前用户切换为root（或其他用户），但用户的Shell环境仍然是切换前的普通用户环境。

`su -` 命令则会连带用户身份和Shell环境全部切换为目标用户（如root）的身份。这可以避免因环境变量（如PATH）设置错误导致的问题。

---

### **临时获取权限：`sudo` 命令**

普通用户如果需要临时执行需要root权限的操作，可以使用 `sudo` 工具。

`sudo` 的使用可以减少频繁切换用户的时间，并且提高安全性。`sudo` 命令可以精细地限制特定用户在某台主机上只能运行某些命令，并且所有操作都会被记录在日志中。

例如，在安装程序时，普通用户可以使用命令 `sudo apt install 程序名`，然后在系统提示下输入自己的密码（而非root密码）来完成安装。

---

### **总结**

本节课中我们一起学习了Linux权限管理的核心。我们明确了**root用户**是UID为0的超级管理员。我们比较了切换用户的两种方式：`su` 与 `su -`。最后，我们介绍了 `sudo` 命令，它允许普通用户临时、受控地执行管理员任务，这是一种更安全、更高效的工作方式。
