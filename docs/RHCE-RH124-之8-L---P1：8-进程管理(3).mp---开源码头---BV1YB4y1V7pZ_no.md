在本节课中，我们将学习如何使用 `jobs` 命令管理后台作业，以及如何使用 `kill` 和 `pkill` 命令向进程发送信号以控制其状态。我们将通过具体的命令示例，演示如何暂停、终止进程，以及如何根据进程名批量终止进程。

## 查看作业列表

![](img/f921db673dbe963b3ee4992771433d04_4.png)

上一节我们介绍了进程的基本概念，本节中我们来看看如何管理由当前终端启动的作业。每个用户从不同终端登录时，其作业是相互独立的。

![](img/f921db673dbe963b3ee4992771433d04_6.png)

使用 `jobs` 命令可以查看当前终端会话中运行的所有后台作业。

```
jobs
```

执行该命令后，会列出当前终端下的所有作业及其状态。

![](img/f921db673dbe963b3ee4992771433d04_8.png)

## 控制作业状态

![](img/f921db673dbe963b3ee4992771433d04_10.png)

以下是控制作业状态的常用方法，主要通过向作业发送信号来实现。

### 暂停作业

我们可以使用 `kill` 命令向指定作业发送 `SIGSTOP` 信号，使其暂停运行。

![](img/f921db673dbe963b3ee4992771433d04_12.png)

```
kill -SIGSTOP %1
```
或使用作业号：
```
kill -19 %1
```
其中 `%1` 表示作业号为1的作业。执行后，该作业将暂停。

![](img/f921db673dbe963b3ee4992771433d04_14.png)

### 终止作业

![](img/f921db673dbe963b3ee4992771433d04_16.png)

要彻底终止一个作业，需要向其发送 `SIGTERM` 信号（默认信号，编号为15）。

```
kill -15 %2
```
或直接使用：
```
kill %2
```
作业被终止后，将从 `jobs` 列表中消失。

![](img/f921db673dbe963b3ee4992771433d04_18.png)

![](img/f921db673dbe963b3ee4992771433d04_20.png)

## 查看系统所有进程

![](img/f921db673dbe963b3ee4992771433d04_22.png)

作业管理仅限于当前终端。要查看系统内运行的所有进程，需要使用 `ps` 命令。

`ps -ef` 命令可以以完整格式列出所有进程的详细信息。

```
ps -ef
```
另一种常用风格是 BSD 格式的 `ps aux`，两者功能类似。

## 根据进程名终止进程

`pkill` 命令可以根据进程名称来发送信号，默认发送 `SIGTERM` 信号以终止进程。

![](img/f921db673dbe963b3ee4992771433d04_24.png)

例如，要终止所有名为 `tail` 的进程：

![](img/f921db673dbe963b3ee4992771433d04_26.png)

```
pkill tail
```
此命令会终止所有由 `tail` 命令启动的进程，无论其进程号是多少。

## 总结

![](img/f921db673dbe963b3ee4992771433d04_28.png)

本节课中我们一起学习了 Linux 进程管理的高级操作。我们掌握了使用 `jobs` 查看和管理后台作业，使用 `kill` 命令通过作业号或进程号来暂停或终止进程，以及使用 `pkill` 命令根据进程名批量终止进程。这些技能对于有效管理系统运行的任务至关重要。
