## 概述
周期性计划任务用于在特定时间点自动重复执行命令或脚本，例如每周一14点执行一次系统备份。在Linux系统中，这主要通过 `cron` 服务来实现。

## Cron服务组件
Cron服务由几个关键组件构成，以下是其主要部分：

*   **crond**：提供cron守护进程的主程序包。
*   **cronie**：提供cron守护进程、相关配置文件及辅助工具。
*   **cronie-anacron**：包含anacron，用于补充系统的维护任务。
*   **crontabs**：包含`crond`所需的系统及用户crontab文件。

![](img/f324443c665bdd3d293772d398067dd6_7.png)

通常，安装`cronie`时会自动安装其依赖。在RHEL 7及以后版本，可以使用 `systemctl status crond` 命令查看服务状态。

![](img/f324443c665bdd3d293772d398067dd6_9.png)

## 系统级Cron任务
系统级Cron任务由root用户管理，其配置文件位于 `/etc/crontab`。我们先来了解一下系统级计划任务。

![](img/f324443c665bdd3d293772d398067dd6_11.png)

![](img/f324443c665bdd3d293772d398067dd6_13.png)

查看 `/etc/crontab` 文件，可以看到其基本结构：

![](img/f324443c665bdd3d293772d398067dd6_15.png)

![](img/f324443c665bdd3d293772d398067dd6_17.png)

```bash
SHELL=/bin/bash
PATH=/sbin:/bin:/usr/sbin:/usr/bin
MAILTO=root
...
```

![](img/f324443c665bdd3d293772d398067dd6_19.png)

文件开头定义了环境变量，如使用的SHELL、PATH以及邮件接收者（MAILTO）。任务条目的格式如下：

![](img/f324443c665bdd3d293772d398067dd6_21.png)

```
分 时 日 月 星期 用户 命令
```

以下是时间字段的说明：

*   `*`：代表“每”，例如分钟位置的 `*` 表示每分钟。
*   数字：指定具体时间点，例如 `1` 在“时”字段表示凌晨1点。
*   列表：用逗号分隔多个时间点，例如 `1,15,20,40` 在“分”字段表示每小时的第1、15、20、40分钟执行。
*   范围：用短横线指定一个连续范围，例如 `20-23` 在“时”字段表示晚上8点到11点。
*   步长：与`*`或范围结合使用，格式为 `*/数值`，例如 `*/2` 在“分”字段表示每两分钟执行一次。

**注意**：步长值必须能被对应时间字段的最大值整除（例如分钟步长须被60整除，小时步长须被24整除）。

一个示例任务：每分钟以root身份执行一次 `echo hello` 命令。
```
* * * * * root /bin/echo hello
```

## 用户级Cron任务
普通用户通常只能管理自己的计划任务。接下来我们看看用户级计划任务。

每个用户都可以使用 `crontab` 命令管理自己的任务。用户cron文件存放在 `/var/spool/cron/` 目录下，以用户名命名。root用户可以修改任何用户的cron任务。

以下是 `crontab` 命令的常用选项：

*   `crontab -e`：编辑当前用户的cron任务。
*   `crontab -l`：列出当前用户的cron任务。
*   `crontab -r`：删除当前用户的所有cron任务。
*   `crontab -u username -e`：root用户编辑指定用户的cron任务。

![](img/f324443c665bdd3d293772d398067dd6_23.png)

用户cron任务的格式与系统级类似，但**不需要指定“用户”字段**，默认为任务所属用户。格式为：
```
分 时 日 月 星期 命令
```

![](img/f324443c665bdd3d293772d398067dd6_25.png)

例如，用户student添加一个每分钟执行 `echo hello` 的任务：
```bash
[student@server ~]$ crontab -e
* * * * * /bin/echo hello
```

![](img/f324443c665bdd3d293772d398067dd6_27.png)

![](img/f324443c665bdd3d293772d398067dd6_29.png)

![](img/f324443c665bdd3d293772d398067dd6_31.png)

## 环境变量与输出处理
用户cron任务默认的PATH环境变量通常只包含 `/usr/bin:/bin`，比交互式Shell的PATH简单。因此，在cron任务中执行命令时，**强烈建议使用绝对路径**。

![](img/f324443c665bdd3d293772d398067dd6_33.png)

如果不想接收任务执行结果（标准输出或错误）的邮件通知，可以将输出重定向。例如，将输出丢弃：
```
* * * * * /bin/echo hello > /dev/null 2>&1
```

![](img/f324443c665bdd3d293772d398067dd6_35.png)

![](img/f324443c665bdd3d293772d398067dd6_37.png)

![](img/f324443c665bdd3d293772d398067dd6_39.png)

也可以在crontab文件顶部定义环境变量，如自定义PATH：
```
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin
* * * * * /bin/echo hello
```

![](img/f324443c665bdd3d293772d398067dd6_41.png)

![](img/f324443c665bdd3d293772d398067dd6_43.png)

## 时间格式的深入理解
时间字段的组合需要仔细理解，特别是“日”和“星期”字段的关系。

一个复杂的例子：`30 4 1,15 * 5 /some/command`
*   **含义**：在每月1号和15号，如果当天**同时**也是星期五，则在凌晨4:30执行命令。
*   **关键点**：“日”（1,15）和“星期”（5，代表星期五）是**逻辑与(AND)** 关系，两个条件必须同时满足任务才会执行。这就是为什么“星期”字段放在最后，需要单独判断。
