# Ansible循环任务控制：Day11：RH294_Ch05_循环任务控制 🔄

在本节课中，我们将要学习Ansible中的循环任务控制。之前我们介绍了变量的使用，本节中我们来看看如何使用循环来高效地执行重复性任务，例如同时创建多个用户或安装多个软件包。

## 概述

![](img/138795d9f4cb593553319109342570a8_1.png)

在编写Ansible剧本时，有时需要一次性执行多个相似的任务。例如，同时创建20个用户或安装3个软件包。在Shell脚本中，我们通常使用`for`循环来实现。同样，在Ansible中，我们可以使用`loop`关键字来实现循环控制，这能让我们的剧本更加简洁和高效。

## 基础循环示例

以下是使用`loop`进行基础循环的示例。我们将创建一个剧本，用于在目标主机上创建一系列用户。

![](img/138795d9f4cb593553319109342570a8_9.png)

![](img/138795d9f4cb593553319109342570a8_11.png)

```yaml
---
- name: 基础循环示例
  hosts: all
  tasks:
    - name: 创建多个用户
      user:
        name: "{{ item }}"
        state: present
      loop:
        - user130
        - user131
        - user132
```

![](img/138795d9f4cb593553319109342570a8_13.png)

在这个剧本中：
*   `user`模块用于创建用户。
*   `{{ item }}`是一个特殊的变量，它代表`loop`列表中的当前元素。
*   `loop`关键字后面跟着一个列表，Ansible会遍历列表中的每个元素并执行任务。

![](img/138795d9f4cb593553319109342570a8_15.png)

![](img/138795d9f4cb593553319109342570a8_17.png)

![](img/138795d9f4cb593553319109342570a8_18.png)

## 在单个任务中执行多个循环

![](img/138795d9f4cb593553319109342570a8_20.png)

![](img/138795d9f4cb593553319109342570a8_21.png)

![](img/138795d9f4cb593553319109342570a8_22.png)

![](img/138795d9f4cb593553319109342570a8_23.png)

![](img/138795d9f4cb593553319109342570a8_24.png)

![](img/138795d9f4cb593553319109342570a8_25.png)

一个任务可以包含多个循环操作。例如，我们可以在一个剧本中既创建用户又安装软件包。

![](img/138795d9f4cb593553319109342570a8_27.png)

![](img/138795d9f4cb593553319109342570a8_29.png)

![](img/138795d9f4cb593553319109342570a8_31.png)

```yaml
---
- name: 多任务循环示例
  hosts: all
  tasks:
    - name: 创建多个用户
      user:
        name: "{{ item }}"
        state: present
      loop:
        - user130
        - user131

    - name: 安装多个软件包
      yum:
        name: "{{ item }}"
        state: latest
      loop:
        - httpd
        - vsftpd
        - nfs-utils
```

![](img/138795d9f4cb593553319109342570a8_33.png)

![](img/138795d9f4cb593553319109342570a8_35.png)

请注意，每个任务中的`{{ item }}`变量是独立的，它只在其所属任务的`loop`范围内生效。

![](img/138795d9f4cb593553319109342570a8_53.png)

![](img/138795d9f4cb593553319109342570a8_54.png)

![](img/138795d9f4cb593553319109342570a8_55.png)

![](img/138795d9f4cb593553319109342570a8_56.png)

以下是使用变量定义循环列表的示例：

![](img/138795d9f4cb593553319109342570a8_59.png)

![](img/138795d9f4cb593553319109342570a8_60.png)

![](img/138795d9f4cb593553319109342570a8_61.png)

![](img/138795d9f4cb593553319109342570a8_62.png)

![](img/138795d9f4cb593553319109342570a8_63.png)

```yaml
---
- name: 使用变量循环示例
  hosts: all
  vars:
    user_list:
      - user140
      - user141
      - user142
    package_list:
      - httpd
      - vsftpd
      - nfs-utils

  tasks:
    - name: 使用变量列表创建用户
      user:
        name: "{{ item }}"
        state: present
      loop: "{{ user_list }}"

    - name: 使用变量列表安装软件包
      yum:
        name: "{{ item }}"
        state: present
      loop: "{{ package_list }}"
```

![](img/138795d9f4cb593553319109342570a8_65.png)

![](img/138795d9f4cb593553319109342570a8_66.png)

![](img/138795d9f4cb593553319109342570a8_68.png)

在这个剧本中：
*   我们在`vars`部分定义了两个列表变量：`user_list`和`package_list`。
*   在任务中，`loop`关键字通过`"{{ variable_name }}"`的形式引用这些变量列表。
*   这种方法使剧本结构更清晰，易于修改。

## 处理字典列表的循环（复杂数据结构）

当循环列表中的元素是字典（即包含键值对）时，我们需要在任务中引用具体的键。

![](img/138795d9f4cb593553319109342570a8_80.png)

![](img/138795d9f4cb593553319109342570a8_82.png)

![](img/138795d9f4cb593553319109342570a8_84.png)

![](img/138795d9f4cb593553319109342570a8_85.png)

![](img/138795d9f4cb593553319109342570a8_86.png)

以下是循环处理字典列表的示例：

![](img/138795d9f4cb593553319109342570a8_88.png)

![](img/138795d9f4cb593553319109342570a8_90.png)

```yaml
---
- name: 字典列表循环示例
  hosts: all
  tasks:
    - name: 创建用户并指定Shell
      user:
        name: "{{ item.name }}"
        shell: "{{ item.shell }}"
        state: present
      loop:
        - { name: 'user150', shell: '/bin/bash' }
        - { name: 'user151', shell: '/sbin/nologin' }
```

![](img/138795d9f4cb593553319109342570a8_92.png)

![](img/138795d9f4cb593553319109342570a8_94.png)

![](img/138795d9f4cb593553319109342570a8_96.png)

![](img/138795d9f4cb593553319109342570a8_97.png)

![](img/138795d9f4cb593553319109342570a8_99.png)

在这个示例中，`loop`列表中的每个元素都是一个字典。我们使用`{{ item.key }}`（例如`{{ item.name }}`）来访问字典中特定键的值。

## 使用 `dict2items` 过滤器转换变量

![](img/138795d9f4cb593553319109342570a8_105.png)

有时，我们定义的变量可能是一个复杂的字典，而我们需要循环其子项。这时可以使用`dict2items`过滤器将字典转换为适合循环的列表。

![](img/138795d9f4cb593553319109342570a8_107.png)

![](img/138795d9f4cb593553319109342570a8_108.png)

![](img/138795d9f4cb593553319109342570a8_109.png)

![](img/138795d9f4cb593553319109342570a8_110.png)

以下是使用 `dict2items` 的示例：

![](img/138795d9f4cb593553319109342570a8_112.png)

![](img/138795d9f4cb593553319109342570a8_113.png)

![](img/138795d9f4cb593553319109342570a8_115.png)

![](img/138795d9f4cb593553319109342570a8_117.png)

```yaml
---
- name: 使用 dict2items 过滤器
  hosts: all
  vars:
    users_info:
      user160:
        shell: /bin/bash
      user161:
        shell: /sbin/nologin

  tasks:
    - name: 使用 dict2items 创建用户
      user:
        name: "{{ item.key }}"
        shell: "{{ item.value.shell }}"
        state: present
      loop: "{{ users_info | dict2items }}"
```

![](img/138795d9f4cb593553319109342570a8_118.png)

![](img/138795d9f4cb593553319109342570a8_119.png)

![](img/138795d9f4cb593553319109342570a8_120.png)

![](img/138795d9f4cb593553319109342570a8_122.png)

在这个剧本中：
*   `users_info`是一个嵌套字典。
*   `| dict2items`是一个Jinja2过滤器，它将字典转换为一个包含`key`和`value`键的字典列表。
*   在循环中，我们可以使用`{{ item.key }}`获取用户名，使用`{{ item.value.shell }}`获取对应的shell配置。

![](img/138795d9f4cb593553319109342570a8_124.png)

![](img/138795d9f4cb593553319109342570a8_126.png)

![](img/138795d9f4cb593553319109342570a8_128.png)

## 总结

![](img/138795d9f4cb593553319109342570a8_130.png)

![](img/138795d9f4cb593553319109342570a8_131.png)

本节课中我们一起学习了Ansible中循环任务控制的核心方法。

![](img/138795d9f4cb593553319109342570a8_133.png)

![](img/138795d9f4cb593553319109342570a8_135.png)

![](img/138795d9f4cb593553319109342570a8_137.png)

我们主要掌握了：
1.  **基础循环**：使用`loop`和`{{ item }}`执行简单列表循环。
2.  **变量化列表**：将循环列表定义为变量，提升剧本可维护性。
3.  **复杂数据循环**：处理字典列表，通过`{{ item.key }}`访问值。
4.  **数据结构转换**：使用`dict2items`过滤器将复杂字典转换为可循环的列表。

![](img/138795d9f4cb593553319109342570a8_139.png)

![](img/138795d9f4cb593553319109342570a8_141.png)

通过使用循环，我们可以避免在剧本中编写大量重复的任务代码，使自动化脚本更加简洁和强大。
