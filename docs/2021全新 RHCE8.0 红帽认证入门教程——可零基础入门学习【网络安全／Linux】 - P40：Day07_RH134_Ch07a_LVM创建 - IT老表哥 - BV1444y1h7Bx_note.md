在本节课中，我们将要学习逻辑卷管理（LVM）的创建过程。LVM是一种灵活的磁盘管理方案，允许我们动态调整存储空间，相比传统的分区管理方式具有显著优势。

![](img/cd60b3244398b937491fbe8ee5ec73e7_9.png)

## 概述 📋

上一节我们介绍了磁盘分区和挂载的基本操作。本节中我们来看看如何创建和管理逻辑卷。逻辑卷管理（LVM）通过将物理存储设备抽象化，提供了更灵活、可扩展的存储管理方式。

![](img/cd60b3244398b937491fbe8ee5ec73e7_11.png)

## LVM核心概念与优势 🧠

![](img/cd60b3244398b937491fbe8ee5ec73e7_13.png)

LVM，即逻辑卷管理器，其核心优势在于可以**在线调整分区大小**，实现存储空间的灵活分配。它还可以实现软件级别的冗余功能。

![](img/cd60b3244398b937491fbe8ee5ec73e7_15.png)

LVM涉及几个重要的专有名词：
*   **分区**：可用于读写的块设备。
*   **物理卷**：物理卷是卷组的基本组成单位。一个分区或整块磁盘在打上特定标记后，可以成为物理卷。
*   **卷组**：一个或多个物理卷可以合并形成一个存储池，称为卷组。
*   **逻辑卷**：在卷组中划分出的逻辑存储单元，经过格式化和挂载后即可用于存储数据。

![](img/cd60b3244398b937491fbe8ee5ec73e7_17.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_19.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_21.png)

LVM的创建流程可以概括为：**物理卷 -> 卷组 -> 逻辑卷**。这相当于将分散的存储资源整合后重新划分，便于统一管理。

![](img/cd60b3244398b937491fbe8ee5ec73e7_23.png)

接下来，将物理卷加入到一个卷组中。卷组是存储池。以下是相关命令：

![](img/cd60b3244398b937491fbe8ee5ec73e7_80.png)

*   **创建卷组**：
    ```bash
    vgcreate vg0 /dev/nvme0n2p5 /dev/nvme0n2p6
    ```

*   **查看卷组**：使用 `vgs`、`vgdisplay` 或 `vgscan` 命令。
*   **删除卷组**：使用 `vgremove` 命令。

### 第四步：创建逻辑卷

![](img/cd60b3244398b937491fbe8ee5ec73e7_90.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_92.png)

在卷组的基础上，我们可以划分出逻辑卷。以下是创建逻辑卷的两种方式：

![](img/cd60b3244398b937491fbe8ee5ec73e7_95.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_97.png)

*   **方式一：直接指定大小**
    ```bash
    lvcreate -L 100M -n lv0 vg0
    ```
*   **方式二：指定物理扩展单元数量**
    ```bash
    lvcreate -l 30 -n lv1 vg0
    ```
    总容量 = PE数量 × PE大小。

### 第五步：格式化与挂载

![](img/cd60b3244398b937491fbe8ee5ec73e7_115.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_117.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_119.png)

逻辑卷创建后，需要格式化和挂载才能使用。以下是操作步骤：

![](img/cd60b3244398b937491fbe8ee5ec73e7_121.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_122.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_123.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_124.png)

1.  **格式化逻辑卷**：
    ```bash
    mkfs.xfs /dev/vg0/lv0
    mkfs.ext4 /dev/vg0/lv1
    ```
2.  **创建挂载点**：
    ```bash
    mkdir /mnt/lv0 /mnt/lv1
    ```
3.  **挂载逻辑卷**：建议使用设备的UUID进行挂载，这比使用设备路径更稳定可靠。可以通过 `blkid` 命令查看UUID。
    *   编辑 `/etc/fstab` 文件，添加类似如下行：
        ```
        UUID=<lv0的UUID> /mnt/lv0 xfs defaults 0 0
        UUID=<lv1的UUID> /mnt/lv1 ext4 defaults 0 0
        ```
    *   执行 `mount -a` 命令挂载所有在 `/etc/fstab` 中定义的文件系统。

![](img/cd60b3244398b937491fbe8ee5ec73e7_126.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_128.png)

![](img/cd60b3244398b937491fbe8ee5ec73e7_130.png)

**注意**：格式化后的可用空间可能略小于创建时指定的大小，这是因为部分空间被系统用于存储元数据，这在考试允许的误差范围内。
