## 硬盘物理结构简介

上一节我们介绍了Linux系统的一些基础操作，本节中我们来看看计算机中用于存储数据的核心硬件——硬盘。

硬盘在传统物理服务器上，其外观类似于一块板砖。服务器机箱上有专门的硬盘插槽，硬盘可以直接插入。现代服务器硬盘大多支持热插拔功能，这意味着可以在服务器开机运行的状态下直接插入或拔出硬盘，系统能够立即识别，类似于我们日常使用U盘。

硬盘的内部结构主要由以下几部分组成：
*   **主轴马达**：用于带动盘片旋转。
*   **盘片**：数据实际存储的介质，类似于光盘。
*   **机械手臂与读写磁头**：机械手臂带动读写磁头在高速旋转的盘片上移动，进行数据的读取和写入操作。

硬盘内部是真空环境，以防止灰尘进入导致盘片损坏和数据丢失。因此，普通用户切勿自行拆解硬盘。

## 磁盘的接口类型

了解了硬盘的物理结构后，我们来看看硬盘与服务器连接的“桥梁”——接口类型。不同的接口类型在速度和适用场景上有所区别。

以下是几种常见的硬盘接口类型：
*   **IDE**：一种古老的接口，数据传输速度慢，价格便宜。大约在2009年后，在服务器领域已被淘汰，曾主要用于个人电脑。
*   **SCSI**：在2013年至2017年间在服务器领域流行，数据传输速度快，并较早支持热插拔功能。
*   **SATA**：目前个人电脑领域主流的接口，服务器中也有使用，特点是速度较快。
*   **SAS**：目前在服务器领域比较流行的接口类型。
*   **SSD (固态硬盘)**：利用闪存颗粒存储数据，没有机械部件，速度远快于传统机械硬盘。通常使用SATA或更快的接口。
*   **NVMe**：一种通信协议，专为固态硬盘设计，通过PCIe通道与CPU直连，能提供极高的数据传输速度。

对于运维工作而言，无需过度纠结接口类型，公司会根据性能需求和预算选择相应的硬件。

## 磁盘分区格式：MBR vs GPT

有了硬盘之后，我们需要对其进行分区才能使用。分区格式决定了如何组织和管理硬盘上的空间。主要有两种分区格式：MBR和GPT。

### MBR分区格式

MBR是一种较为古老的分区格式。它有一个显著的限制：**一块硬盘最多只能划分4个主分区**。

对于许多应用场景来说，4个分区可能不够用。为了解决这个问题，MBR引入了“扩展分区”的概念。你可以将第4个分区创建为“扩展分区”，它本身不直接存储数据，而是作为一个容器，在这个容器内部可以再划分出多个“逻辑分区”来存储数据。

以下是MBR分区结构的特点：
*   最多4个主分区，或3个主分区 + 1个扩展分区。
*   在扩展分区内可以创建多个逻辑分区（例如，IDE硬盘最多59个，SCSI硬盘最多11个）。
*   **最大仅支持2.2TB以内的磁盘容量**。对于超过2.2TB的大容量硬盘，MBR无法有效管理。

### GPT分区格式

GPT是一种现代的分区格式，旨在克服MBR的限制。

以下是GPT分区格式的主要优势：
*   **支持几乎无限数量的分区**（在Linux系统中通常最多可创建128个主分区）。
*   **支持超大容量磁盘**，最大可支持18EB（Exabyte）的磁盘容量。容量换算公式为：`1024 KB = 1 MB; 1024 MB = 1 GB; 1024 GB = 1 TB; 1024 TB = 1 PB; 1024 PB = 1 EB`。

在企业环境中，对于小容量磁盘（<2.2TB）可以选择MBR，而对于大容量磁盘则必须使用GPT分区格式。

## Linux磁盘管理常用命令

前面我们介绍了分区格式的理论知识，本节中我们来看看在Linux系统中如何查看和管理磁盘。首先，我们需要为虚拟机添加一块新硬盘用于实验。

### 添加新硬盘并查看

![](img/1940f65a0dc2cd9312319524fd1a9413_13.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_15.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_16.png)

最常用的命令是 `lsblk`，它可以列出系统当前所有的块设备信息（如硬盘和分区）。

![](img/1940f65a0dc2cd9312319524fd1a9413_22.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_24.png)

```bash
lsblk
```
执行命令后，你会看到类似以下的输出：
```
NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda      8:0    0   20G  0 disk
├─sda1   8:1    0    1G  0 part /boot
└─sda2   8:2    0   19G  0 part /
sdb      8:16   0  100G  0 disk
sdc      8:32   0  100G  0 disk
```

![](img/1940f65a0dc2cd9312319524fd1a9413_26.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_28.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_30.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_32.png)

输出结果解析：
*   **NAME**：设备名称。
    *   以 `sd` 开头表示SCSI或SATA类型的磁盘。
    *   后面的字母 `a`, `b`, `c`... 表示第1、2、3...块磁盘。
    *   名称后面的数字 `1`, `2`... 表示该磁盘上的第1、2...个分区。
*   **SIZE**：设备容量。
*   **TYPE**：设备类型，`disk` 表示磁盘，`part` 表示分区。
*   **MOUNTPOINT**：挂载点，表示该分区当前被哪个目录使用。

![](img/1940f65a0dc2cd9312319524fd1a9413_36.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_38.png)

### 查看文件系统使用情况

![](img/1940f65a0dc2cd9312319524fd1a9413_50.png)

`df` 命令用于显示文件系统的磁盘空间使用情况，它主要关注**正在使用中**的分区。

![](img/1940f65a0dc2cd9312319524fd1a9413_56.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_58.png)

通常我们使用 `-h` 选项，以人类易读的格式（K, M, G）显示信息。

![](img/1940f65a0dc2cd9312319524fd1a9413_62.png)

```bash
df -h
```
输出示例：
```

/dev/sda2        19G  1.2G   17G   7% /
/dev/sda1       976M  139M  770M  16% /boot
```
这个命令可以清晰地看到每个挂载分区的总容量、已用空间、可用空间和使用百分比。

### 统计文件或目录大小

![](img/1940f65a0dc2cd9312319524fd1a9413_74.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_76.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_78.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_80.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_82.png)

`du` 命令用于估算文件或目录的磁盘使用空间。

一个常用的组合是 `du -sh`：
*   `-s`：仅显示总计，不列出每个子目录的详情。
*   `-h`：以人类易读的格式显示。

![](img/1940f65a0dc2cd9312319524fd1a9413_84.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_86.png)

```bash
# 统计 /etc 目录的总大小
du -sh /etc

![](img/1940f65a0dc2cd9312319524fd1a9413_88.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_90.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_92.png)

# 与 ls -lh 命令对比，ls 显示的是目录元数据大小，而非目录内所有内容的总大小
ls -lh -d /etc
```
**注意**：`ls -lh` 在查看目录时，显示的是目录条目本身的大小，而不是目录下所有文件内容的总和。而 `du` 命令统计的是实际的数据占用空间。

### 命令总结与选用指南

![](img/1940f65a0dc2cd9312319524fd1a9413_116.png)

![](img/1940f65a0dc2cd9312319524fd1a9413_118.png)

为了帮助大家记忆，以下是这些命令的常用场景总结：

![](img/1940f65a0dc2cd9312319524fd1a9413_120.png)

1.  **`lsblk`**：**查看系统中有多少块硬盘以及每个硬盘的分区情况**。这是获取磁盘整体布局的首选命令。
2.  **`df -h`**：**查看正在使用的各个分区的空间使用率**。用于监控磁盘空间是否充足。
3.  **`du -sh`**：**准确统计文件或目录的实际磁盘占用大小**。当需要知道某个文件夹到底占了多大空间时使用。
4.  **`blkid`**：查看分区或磁盘的详细属性，如文件系统类型、UUID。可作为 `lsblk -f` 的补充。
