# Ansible入门教程：1.5：管理Ansible配置文件 🛠️

在本节课中，我们将要学习如何管理Ansible的配置文件。配置文件是控制Ansible行为的关键，理解其优先级和结构对于高效使用Ansible至关重要。

## 配置文件优先级 📊

![](img/c2ba2feed5ed1e890b56580d9fa82337_1.png)

Ansible控制节点会从多个可能的位置之一读取其配置文件。这些位置有不同的优先级，数字越大代表优先级越低。

![](img/c2ba2feed5ed1e890b56580d9fa82337_3.png)

以下是配置文件的优先级顺序：

![](img/c2ba2feed5ed1e890b56580d9fa82337_5.png)

1.  **当前工作目录下的 `ansible.cfg` 文件**：优先级最高。如果执行`ansible`命令的目录中存在此文件，则使用它。
2.  **用户家目录下的 `.ansible.cfg` 文件**：优先级次之。这是一个隐藏文件，仅对当前用户生效。
3.  **`/etc/ansible/ansible.cfg` 文件**：优先级最低。这是全局默认配置文件，当其他位置没有配置文件时使用。

**核心概念**：优先级遵循 **“就近原则”**，范围越小的配置文件优先级越高。这允许我们为不同的项目或环境使用不同的配置。

![](img/c2ba2feed5ed1e890b56580d9fa82337_7.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_9.png)

## 配置文件结构解析 📝

上一节我们介绍了配置文件的优先级，本节中我们来看看配置文件内部的具体结构。配置文件通常包含多个节（section），其中 `[defaults]` 节定义了Ansible的基本行为选项。

以下是 `[defaults]` 节中一些关键配置项的说明：

*   **`inventory`**：定义资产清单文件（即受管主机列表）的路径。这是必需项。
    *   **示例**：`inventory = /home/student/ansible/inventory`
*   **`remote_user`**：指定连接受管主机时使用的远程登录用户名。
    *   **示例**：`remote_user = student`
*   **`remote_port`**：指定连接受管主机时使用的SSH端口，默认为22。
    *   **示例**：`remote_port = 22`
*   **`ask_pass`**：设置为 `True` 时，每次执行任务都会提示输入SSH密码；设置为 `False` 时则不会。为了实现自动化，通常设置为 `False` 并配合SSH密钥认证。
    *   **示例**：`ask_pass = False`

## 实战：创建自定义配置文件 🚀

![](img/c2ba2feed5ed1e890b56580d9fa82337_14.png)

了解了配置文件的结构后，让我们通过一个实战练习来创建并使用一个自定义的配置文件。

![](img/c2ba2feed5ed1e890b56580d9fa82337_16.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_18.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_19.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_21.png)

**实验目标**：以普通用户 `student` 身份，在其工作目录 `/home/student/ansible` 下创建配置文件，并配置基本的连接和权限提升选项。

![](img/c2ba2feed5ed1e890b56580d9fa82337_23.png)

以下是具体操作步骤：

![](img/c2ba2feed5ed1e890b56580d9fa82337_25.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_27.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_29.png)

1.  **切换到工作目录并创建资产清单**：
    ```bash
    # 切换到student用户
    su - student
    # 创建ansible工作目录并进入
    mkdir ~/ansible && cd ~/ansible
    # 创建资产清单文件，定义两台受管主机
    echo -e "[servers]\nnode1.lab.example.com\nnode2.lab.example.com" > inventory
    ```

![](img/c2ba2feed5ed1e890b56580d9fa82337_31.png)

2.  **创建并编辑Ansible配置文件**：
    ```bash
    # 创建ansible.cfg文件
    vim ansible.cfg
    ```
    在文件中输入以下内容：
    ```ini
    [defaults]
    # 指定资产清单文件路径
    inventory = /home/student/ansible/inventory
    # 指定远程连接端口
    remote_port = 22
    # 指定远程连接用户
    remote_user = student
    # 禁用连接时的密码询问（需已配置SSH密钥）
    ask_pass = False

    [privilege_escalation]
    # 启用权限提升
    become = True
    # 指定权限提升方法为sudo
    become_method = sudo
    # 提升到root用户
    become_user = root
    # 权限提升时不需要输入密码（需在受管主机上配置sudo免密）
    become_ask_pass = False
    ```
    **核心概念**：`[privilege_escalation]` 节用于配置当普通用户需要执行管理员命令时的权限提升方式。

![](img/c2ba2feed5ed1e890b56580d9fa82337_33.png)

3.  **验证配置**：
    ```bash
    # 列出所有受管主机，验证配置和连接
    ansible all --list-hosts
    # 对所有主机执行一个命令，验证权限提升是否生效
    ansible all -m command -a “id”
    ansible all -m command -a “fdisk -l” # 此命令需要root权限
    ```

![](img/c2ba2feed5ed1e890b56580d9fa82337_35.png)

## 权限提升的两种方法 🔑

当使用普通用户远程连接并需要执行特权命令时，必须在受管主机上进行相应配置。主要有两种方法：

![](img/c2ba2feed5ed1e890b56580d9fa82337_37.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_39.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_41.png)

**方法一：配置sudo（推荐）**
此方法通过修改受管主机上的 `/etc/sudoers` 文件，允许特定用户无需密码即可使用 `sudo` 执行所有命令。
1.  在受管主机上，使用 `visudo` 命令编辑 `/etc/sudoers` 文件。
2.  添加一行：`student ALL=(ALL) NOPASSWD: ALL`。
3.  此配置与Ansible配置文件中的 `become_method = sudo` 和 `become_ask_pass = False` 配合使用。

![](img/c2ba2feed5ed1e890b56580d9fa82337_43.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_45.png)

**方法二：配置su（了解即可）**
此方法通过修改受管主机上的PAM（可插拔认证模块）配置，允许特定用户组的成员使用 `su` 命令切换到root而无需密码。
1.  在受管主机上，编辑 `/etc/pam.d/su` 文件。
2.  取消注释 `auth required pam_wheel.so use_uid` 这一行。
3.  将执行用户（如 `student`）添加到 `wheel` 组：`usermod -aG wheel student`。
4.  此配置与Ansible配置文件中的 `become_method = su` 配合使用。

![](img/c2ba2feed5ed1e890b56580d9fa82337_47.png)

![](img/c2ba2feed5ed1e890b56580d9fa82337_49.png)

**注意**：出于安全和管理一致性考虑，**强烈推荐使用方法一（sudo）**。

![](img/c2ba2feed5ed1e890b56580d9fa82337_51.png)
