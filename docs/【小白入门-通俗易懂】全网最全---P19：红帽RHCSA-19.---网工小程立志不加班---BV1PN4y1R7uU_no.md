在本节课中，我们将要学习Linux系统中的四种特殊权限：SUID、SGID、SBIT和ACL。这些权限扩展了基本权限（rwx）的功能，用于满足更精细和特殊的安全管理需求。我们将逐一了解它们的作用、应用场景和设置方法。

## 概述 📋

![](img/870350dbed590393c127ae365ebc72e5_11.png)

上一节我们介绍了Linux文件的基本权限和归属关系。本节中，我们来看看几种特殊的权限机制。这些权限并非日常必需，但在特定场景下非常有用。前两种（SUID、SGID）主要针对可执行文件，用于临时改变执行者的身份；后两种（SBIT、ACL）则更为常用，分别用于目录保护和为特定用户/组定制权限。

## SUID（Set User ID）权限 🛡️

![](img/870350dbed590393c127ae365ebc72e5_13.png)

![](img/870350dbed590393c127ae365ebc72e5_15.png)

![](img/870350dbed590393c127ae365ebc72e5_17.png)

![](img/870350dbed590393c127ae365ebc72e5_19.png)

SUID权限主要针对可执行程序文件。当一个可执行文件被设置了SUID权限后，任何普通用户在执行该文件时，都会临时拥有该文件所有者（通常是root）的身份和权限。该权限仅在程序执行过程中有效，程序执行完毕后用户恢复原有身份。

![](img/870350dbed590393c127ae365ebc72e5_21.png)

![](img/870350dbed590393c127ae365ebc72e5_23.png)

### 核心概念与示例

其功能可以用一个“尚方宝剑”或“圣旨”的比喻来理解：普通用户手持此令，在执行特定命令时便如同皇帝（root）亲临，拥有至高权力，但用完即止。

一个典型的例子是`/usr/bin/passwd`命令。普通用户可以使用它修改自己的密码，但密码最终存储在`/etc/shadow`文件中，而该文件对普通用户是不可写的。

**查看`passwd`命令的权限：**
```bash
ls -l /usr/bin/passwd
```
输出中，所有者的执行位是 `s` 而非 `x`，这表示设置了SUID权限。因此，当用户tom执行`passwd`时，实际是以root身份在修改`/etc/shadow`文件。

### 设置与取消SUID权限

设置SUID权限的命令是 `chmod u+s <文件名>`。
取消SUID权限的命令是 `chmod u-s <文件名>`。

**示例：为`cat`命令设置SUID（演示用，实际慎用）**
```bash
which cat          # 找到cat命令的路径，例如 /usr/bin/cat
chmod u+s /usr/bin/cat
```
设置后，普通用户使用`cat`查看任何文件都将拥有root权限，这非常危险。务必谨慎设置SUID。

**注意：** 如果文件本身没有执行权限（x），设置SUID后会显示为大写 `S`，表示SUID有效但文件不可执行，这通常没有意义。

## SGID（Set Group ID）权限 👥

SGID权限同样主要针对可执行文件。设置了SGID权限后，普通用户在执行该文件时，会临时拥有该文件所属组的身份和权限。

### 核心概念与示例

![](img/870350dbed590393c127ae365ebc72e5_25.png)

![](img/870350dbed590393c127ae365ebc72e5_27.png)

其作用是让执行者临时成为文件所属组的成员，从而继承该组对某些资源的访问权限。

例如，`locate`命令（对应文件`/usr/bin/locate`）用于搜索文件，它需要读取数据库文件`/var/lib/mlocate/mlocate.db`。普通用户对此数据库文件没有读权限。

**查看`locate`命令的权限：**
```bash
ls -l /usr/bin/locate
```
输出中，所属组的执行位是 `s`，表示设置了SGID权限，且文件属于`slocate`组。当用户执行`locate`时，便临时成为`slocate`组的一员，从而可以读取组内有权访问的数据库文件。

![](img/870350dbed590393c127ae365ebc72e5_29.png)

### 设置与取消SGID权限

设置SGID权限的命令是 `chmod g+s <文件名>`。
取消SGID权限的命令是 `chmod g-s <文件名>`。

![](img/870350dbed590393c127ae365ebc72e5_31.png)

**注意：** SGID权限同样需谨慎使用，日常管理中我们很少主动设置它。

## SBIT（Sticky Bit）权限 📌

![](img/870350dbed590393c127ae365ebc72e5_33.png)

![](img/870350dbed590393c127ae365ebc72e5_35.png)

![](img/870350dbed590393c127ae365ebc72e5_37.png)

SBIT权限**只针对目录有效**。它对目录的写权限（w）进行了重要限制。

![](img/870350dbed590393c127ae365ebc72e5_39.png)

![](img/870350dbed590393c127ae365ebc72e5_41.png)

![](img/870350dbed590393c127ae365ebc72e5_43.png)

![](img/870350dbed590393c127ae365ebc72e5_45.png)

### 核心概念与示例

当一个目录被设置为777权限（即所有用户可读、写、执行）时，任何用户都可以删除该目录下的任何文件，这可能导致混乱。SBIT权限就是为了解决这个问题。

![](img/870350dbed590393c127ae365ebc72e5_47.png)

设置了SBIT权限的目录（权限位中“其他用户”的执行位变为 `t`），除了目录所有者和root用户外，**普通用户只能删除或重命名自己创建的文件和目录**，而不能删除其他用户创建的文件。

![](img/870350dbed590393c127ae365ebc72e5_49.png)

系统临时目录`/tmp`就是一个典型的例子：
```bash
ls -ld /tmp
```
输出中，权限末尾有一个 `t`，表示设置了SBIT权限。因此，用户A无法删除用户B在`/tmp`中创建的文件。

![](img/870350dbed590393c127ae365ebc72e5_51.png)

### 设置与取消SBIT权限

![](img/870350dbed590393c127ae365ebc72e5_53.png)

设置SBIT权限的命令是 `chmod o+t <目录名>`。
取消SBIT权限的命令是 `chmod o-t <目录名>`。

**示例：为`/test`目录设置SBIT权限**
```bash
chmod 777 /test     # 首先赋予完全权限
chmod o+t /test     # 设置SBIT权限
ls -ld /test        # 查看权限，末尾应显示 ‘drwxrwxrwt’
```
设置后，即使目录权限是777，用户也只能操作自己的文件，无法删除他人的文件。

![](img/870350dbed590393c127ae365ebc72e5_55.png)

![](img/870350dbed590393c127ae365ebc72e5_57.png)

## ACL（Access Control List）访问控制列表 🗂️

ACL提供了比传统“所有者-所属组-其他人”更精细的权限控制。它可以为**单个用户**或**单个用户组**单独设置对某个文件或目录的访问权限。

![](img/870350dbed590393c127ae365ebc72e5_59.png)

![](img/870350dbed590393c127ae365ebc72e5_61.png)

### 核心概念与示例

假设有一个目录`/ops`属于`ops`组，你希望组员（如tom、老王）有完整权限，而其他部门的人（如小张）完全无权限。这时，一个实习生小薇来了，你希望她只能查看和进入目录，但不能修改或删除。传统的权限模型无法直接满足这种需求，而ACL可以。

![](img/870350dbed590393c127ae365ebc72e5_63.png)

![](img/870350dbed590393c127ae365ebc72e5_65.png)

![](img/870350dbed590393c127ae365ebc72e5_67.png)

### ACL权限管理命令

1.  **设置ACL权限**：`setfacl -m <规则> <文件/目录>`
    *   `-m` 表示修改ACL规则。
    *   规则格式：
        *   为用户设置：`u:用户名:权限` (如 `u:xiaowei:rx`)
        *   为用户组设置：`g:组名:权限` (如 `g:interns:rx`)
    *   权限用 `r`、`w`、`x` 表示。

![](img/870350dbed590393c127ae365ebc72e5_69.png)

![](img/870350dbed590393c127ae365ebc72e5_71.png)

![](img/870350dbed590393c127ae365ebc72e5_73.png)

2.  **查看ACL权限**：`getfacl <文件/目录>`

![](img/870350dbed590393c127ae365ebc72e5_75.png)

3.  **删除指定ACL条目**：`setfacl -x <条目> <文件/目录>`
    *   条目格式：`u:用户名` 或 `g:组名`

4.  **清空所有ACL条目**：`setfacl -b <文件/目录>`

### 操作示例

**场景：** 为目录`/ops`设置ACL，允许用户`xiaowei`具有读和执行权限。
```bash
# 1. 设置ACL
setfacl -m u:xiaowei:rx /ops

# 2. 查看ACL
getfacl /ops
# 输出中会多出 `user:xiaowei:r-x` 这一行

![](img/870350dbed590393c127ae365ebc72e5_77.png)

![](img/870350dbed590393c127ae365ebc72e5_79.png)

![](img/870350dbed590393c127ae365ebc72e5_81.png)

# 3. 此时用 ls -ld /ops 查看，权限末尾会多一个 ‘+’ 号，表示有ACL权限
ls -ld /ops
```

**删除用户`xiaowei`的ACL权限：**
```bash
setfacl -x u:xiaowei /ops
```

**清空`/ops`目录的所有ACL权限：**
```bash
setfacl -b /ops
```

ACL功能强大，是实现复杂权限管理的利器。

## 总结 🎯

本节课中我们一起学习了Linux的四种特殊权限：
*   **SUID**：让普通用户临时拥有文件所有者的身份执行命令，**针对可执行文件**。
*   **SGID**：让普通用户临时拥有文件所属组的身份执行命令，**针对可执行文件**。
*   **SBIT**：限制用户在公共目录中只能删除自己的文件，**针对目录**。
*   **ACL**：为单个用户或用户组设置独立的文件/目录访问权限，实现**精细化的权限控制**。

![](img/870350dbed590393c127ae365ebc72e5_83.png)

![](img/870350dbed590393c127ae365ebc72e5_85.png)

理解这些特殊权限，能帮助你在实际运维中更灵活、更安全地管理系统资源和用户访问。请务必在理解其原理的基础上谨慎使用，特别是SUID/SGID，不当设置可能带来严重的安全风险。
