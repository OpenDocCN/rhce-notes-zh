# 进程管理：08a：进程管理基础

![](img/8da272e13ab9bc62b5577fffb2a7186c_0.png)

在本节课中，我们将要学习Linux系统中的进程管理。进程是系统运行程序时的动态实例，理解和管理进程对于维护系统稳定性和性能至关重要。我们将从进程的基本概念开始，逐步学习如何查看、分析和管理系统中的进程。

## 什么是进程？

进程可以理解为静态程序被触发后产生的动态执行实例。例如，在Windows系统中，通过任务管理器看到的“进程”选项卡，其中列出了正在运行的程序和后台服务。每个进程都占用一定的系统资源，如CPU、内存、硬盘I/O和网络带宽。

在Linux系统中，进程的概念是类似的。系统通过执行一个二进制文件或启动一项服务来触发进程。进程是动态的，而程序文件是静态存储在磁盘上的。

## 进程的来源与拥有者

上一节我们介绍了进程的基本概念，本节中我们来看看进程是如何产生的以及它的拥有者是谁。

进程通常通过两种方式触发：
1.  执行一个二进制文件，例如执行 `passwd` 命令。
2.  启动一项系统服务，例如启动 `httpd` 服务。

关于进程的拥有者，需要理解以下几点：
*   大部分情况下，谁执行了命令，产生的进程拥有者就是谁。
*   对于系统服务，进程的拥有者通常在服务的配置文件中定义，而不是启动服务的用户。例如，`httpd` 服务的进程通常以 `apache` 用户身份运行，这是在 `/etc/httpd/conf/httpd.conf` 配置文件中设置的。
*   进程在读取或写入文件时，使用的是其拥有者的身份和权限。因此，进程拥有者必须具备访问所需文件的相应权限。

### 1. `ps` 命令
`ps` 命令用于查看当前系统的进程状态。如果不加任何参数，它只显示与当前终端相关的进程。

![](img/8da272e13ab9bc62b5577fffb2a7186c_19.png)

![](img/8da272e13ab9bc62b5577fffb2a7186c_21.png)

以下是 `ps` 命令输出中一些关键字段的含义：
*   **PID**: 进程的唯一标识符。
*   **PPID**: 父进程的PID，表示是哪个进程创建了它。
*   **TTY**: 进程所在的终端。如果显示为 `?`，则表示该进程与终端无关（通常是后台服务）。
*   **TIME**: 进程占用CPU的累计时间。
*   **CMD**: 启动该进程的命令。

常用的 `ps` 命令选项组合：
*   `ps aux`: 以BSD格式列出所有用户的详细进程信息。
*   `ps -ef`: 以标准格式列出所有进程的完整信息。
*   `ps -l`: 以长格式显示当前终端的进程信息。

![](img/8da272e13ab9bc62b5577fffb2a7186c_23.png)

![](img/8da272e13ab9bc62b5577fffb2a7186c_25.png)

### 2. `pstree` 命令
`pstree` 命令以树状图的形式显示进程之间的父子关系，这有助于我们理解进程的层次结构。

![](img/8da272e13ab9bc62b5577fffb2a7186c_27.png)

```bash
pstree -aup
```

### 3. `top` 命令
`top` 命令提供了一个动态、实时的系统资源使用情况视图，包括进程列表及其资源占用（CPU、内存等）。它默认会持续刷新。

```bash
top
```
在 `top` 界面中，可以按 `P` 键按CPU使用率排序，按 `M` 键按内存使用率排序。按 `q` 键退出。

## 进程的状态

![](img/8da272e13ab9bc62b5577fffb2a7186c_29.png)

在 `ps` 或 `top` 命令的输出中，进程的状态（STAT）用一个字母表示。理解这些状态对于诊断问题非常重要：

*   **R (Running)**: 正在运行或可运行（在运行队列中等待）。
*   **S (Sleeping)**: 可中断的睡眠状态，正在等待某个事件完成（如I/O操作）。
*   **D (Uninterruptible Sleep)**: 不可中断的睡眠状态，通常是在等待硬件I/O，不响应信号。
*   **T (Stopped)**: 进程被停止（例如，由调试器暂停）。
*   **Z (Zombie)**: 僵尸进程。进程已终止，但其父进程尚未回收它，残留部分信息在进程表中。

## 进程的优先级（Nice值）

![](img/8da272e13ab9bc62b5577fffb2a7186c_31.png)

每个进程都有一个优先级，这会影响内核调度器分配CPU时间的顺序。优先级由两部分组成：
*   内核分配的基准优先级（通常为80）。
*   用户可调整的 **Nice值**，范围从 -20 到 19。

![](img/8da272e13ab9bc62b5577fffb2a7186c_33.png)

![](img/8da272e13ab9bc62b5577fffb2a7186c_35.png)

**公式：进程优先级 = 基准优先级 + Nice值**

![](img/8da272e13ab9bc62b5577fffb2a7186c_37.png)

![](img/8da272e13ab9bc62b5577fffb2a7186c_39.png)

Nice值越低（负数），进程的优先级越高，获得CPU时间的机会就越多。默认情况下，进程的Nice值为0。普通用户只能调低自己进程的优先级（增加Nice值），而 `root` 用户可以提高或降低任何进程的优先级。

![](img/8da272e13ab9bc62b5577fffb2a7186c_40.png)

## 总结

![](img/8da272e13ab9bc62b5577fffb2a7186c_44.png)

本节课中我们一起学习了Linux进程管理的基础知识。我们首先明确了进程是程序执行的动态实例，并了解了进程的来源和拥有者概念。接着，我们重点学习了三个核心命令：`ps` 用于查看进程快照，`pstree` 用于查看进程树关系，`top` 用于实时监控系统资源和进程状态。我们还解释了进程的各种状态（R、S、D、T、Z）以及如何通过Nice值来影响进程的优先级。掌握这些内容是进行有效的系统监控和性能调优的第一步。
