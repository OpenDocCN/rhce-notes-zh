在本节课中，我们将要学习网络通信的基础框架：TCP/IP协议栈与OSI七层模型。理解这些模型是学习后续网络配置、服务部署和故障排查的基石。

## TCP/IP协议栈与OSI七层模型

![](img/9ef472bf85ccca3d58006a684992c58f_3.png)

![](img/9ef472bf85ccca3d58006a684992c58f_4.png)

![](img/9ef472bf85ccca3d58006a684992c58f_5.png)

TCP/IP协议栈和OSI七层模型是描述网络通信过程的两种理论框架。它们会与我们后面学习的许多服务和应用程序密切相关。

![](img/9ef472bf85ccca3d58006a684992c58f_6.png)

下图展示了OSI七层模型（右侧）与TCP/IP四层模型（左侧）的对应关系：

![](img/9ef472bf85ccca3d58006a684992c58f_8.png)

![](img/9ef472bf85ccca3d58006a684992c58f_9.png)

![](img/9ef472bf85ccca3d58006a684992c58f_1.png)

OSI模型从下至上分为七层：
1.  物理层
2.  数据链路层
3.  网络层
4.  传输层
5.  会话层
6.  表示层
7.  应用层

![](img/9ef472bf85ccca3d58006a684992c58f_11.png)

TCP/IP模型则通常被归纳为四层，它与OSI模型的对应关系如下：
*   **应用层**：对应OSI的应用层、表示层和会话层（上三层）。
*   **传输层**：与OSI的传输层对应。
*   **网络层**：与OSI的网络层对应。
*   **网络接口层**：对应OSI的数据链路层和物理层（下两层）。

![](img/9ef472bf85ccca3d58006a684992c58f_12.png)

![](img/9ef472bf85ccca3d58006a684992c58f_13.png)

了解这些协议模型至关重要。OSI七层模型主要定义了网络通信的完整过程。从事网络设备相关工作的人员会更关注OSI模型，例如数据包如何在网络中传输、如何寻址以及传输过程中的变化。

而对于运维人员来说，我们可能更关注具体的协议，例如HTTP协议，需要进行抓包和分析。因此，我们关注更多的是TCP/IP模型。

## 各层功能与协议

![](img/9ef472bf85ccca3d58006a684992c58f_15.png)

![](img/9ef472bf85ccca3d58006a684992c58f_16.png)

每一层都有其特定的功能和对应的协议与服务。

![](img/9ef472bf85ccca3d58006a684992c58f_17.png)

![](img/9ef472bf85ccca3d58006a684992c58f_18.png)

以下是各层关注的要点：

![](img/9ef472bf85ccca3d58006a684992c58f_19.png)

![](img/9ef472bf85ccca3d58006a684992c58f_20.png)

*   **链路层/网络接口层**：关注物理连接标准。例如**MAC地址**、以太网标准（如 `IEEE 802.3`）以及Wi-Fi标准（如 `IEEE 802.11`）。这一层更多地涉及物理协议和标准的定义。

*   **网络层**：关注数据传输和寻址。从OSI模型来看，这就是第三层。我们常说的三层、四层、七层都是指OSI模型。这一层定义了如何传输和寻址数据。例如，**IPv4**、**IPv6**、**ICMP**（Ping命令使用的协议）和**IGMP**等协议都在此层定义。IP报文的格式、长度和封装形式也由这一层决定。

*   **传输层**：关注端到端的可靠或不可靠传输。这一层有两个非常重要的协议：**TCP**（传输控制协议）和**UDP**（用户数据报协议）。我们会在后面看到，许多服务使用TCP协议进行可靠的传输，有些则使用UDP方式。两者的区别在于报文结构和控制机制不同，因此传输速度和可靠性也不同。

![](img/9ef472bf85ccca3d58006a684992c58f_22.png)

TCP协议为了保证可靠性（如确认、重传、流量控制），会在原始数据上添加更多的头部信息，因此带来的数据量更大，传输速度可能相对较慢。UDP协议则更为简单高效，但不保证可靠性。

![](img/9ef472bf85ccca3d58006a684992c58f_23.png)

![](img/9ef472bf85ccca3d58006a684992c58f_24.png)

![](img/9ef472bf85ccca3d58006a684992c58f_25.png)

*   **应用层**：关注具体的应用程序协议。作为运维人员，我们关注的重点较多在上三层，即应用层。这里的协议非常丰富。

![](img/9ef472bf85ccca3d58006a684992c58f_26.png)

![](img/9ef472bf85ccca3d58006a684992c58f_27.png)

以下是应用层常见协议示例：

*   **HTTP/HTTPS**：`HTTP` 是客户端浏览器与服务器之间传输网页的超文本传输协议。`HTTPS` 是增加了SSL/TLS加密的HTTP协议。
*   **FTP**：用于文件传输的协议。
*   **Telnet/SSH**：用于远程连接和管理网络设备的协议。早期设备多使用明文的Telnet，现在普遍支持加密的SSH。
*   **NTP**：用于时间同步的协议。
*   **DNS**：域名解析协议。
*   **DHCP**：动态主机配置协议，用于自动分配IP地址。
*   **SNMP**：简单网络管理协议。
*   **SMTP**：简单邮件传输协议。

因此，作为运维人员，我们关注的点较多在应用层。当然，在进行网络故障排查和抓包分析时，我们也必须关注下面的四层（网络接口层、网络层、传输层），因为底层直接影响数据的传输。而上三层更多地影响不同应用程序之间如何使用特定协议进行“交流”，就像对不同国家的人要说不同的语言，数据的表示和会话管理也在上三层实现。
