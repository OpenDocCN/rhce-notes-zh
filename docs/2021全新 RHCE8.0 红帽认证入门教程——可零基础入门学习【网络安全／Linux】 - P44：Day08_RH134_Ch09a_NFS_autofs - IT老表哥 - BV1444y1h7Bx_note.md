## NFS网络文件系统概念

上一节我们介绍了课程的整体安排，本节中我们来看看NFS网络文件系统的基本概念。

![](img/332cdfe33baea640624e3340f92cb91c_1.png)

网络文件系统（NFS）主要用于Unix和Linux系统之间的资源共享。它允许客户端像访问本地目录一样访问服务器上共享的目录。这与Windows系统中使用的SMB/CIFS协议类似，但NFS在Unix/Linux环境中更为常见，且通常被认为安全性更高。

![](img/332cdfe33baea640624e3340f92cb91c_3.png)

当服务器发布一个共享目录后，客户端可以通过挂载（mount）的方式访问该目录。挂载时需要注意以下几点：
*   挂载点目录应为空目录，否则原内容将被隐藏。
*   若需写入文件，共享目录必须具备相应的写权限。
*   可以在挂载时设置本地访问权限。

关于安全性，NFS默认会将客户端的root用户权限“降级”为一个低权限用户（在RHEL 8中为`nobody`，UID 65534），以增强安全性。更高级的配置可以使用Kerberos（如KRB5P）进行完全加密的验证。

## 配置NFS服务端

![](img/332cdfe33baea640624e3340f92cb91c_7.png)

![](img/332cdfe33baea640624e3340f92cb91c_9.png)

理论介绍完毕，接下来我们通过实践来配置NFS服务端。

![](img/332cdfe33baea640624e3340f92cb91c_11.png)

![](img/332cdfe33baea640624e3340f92cb91c_13.png)

首先，我们需要一台服务器（例如`servera`）来提供NFS共享服务。确保已安装必要的软件包并启动服务。

![](img/332cdfe33baea640624e3340f92cb91c_15.png)

**1. 安装与启动NFS服务**
如果系统是最小化安装，可能需要安装`nfs-utils`软件包。
```bash
dnf install -y nfs-utils
```
启动NFS服务并设置为开机自启。
```bash
systemctl enable --now nfs-server
systemctl status nfs-server
```

![](img/332cdfe33baea640624e3340f92cb91c_17.png)

![](img/332cdfe33baea640624e3340f92cb91c_19.png)

**2. 创建并配置共享目录**
创建一个用于共享的目录，并在其中生成示例文件，以便后续验证。
```bash
mkdir -p /mnt/share
cd /mnt/share
touch file1
mkdir dir1
```
编辑NFS的导出配置文件`/etc/exports`，定义共享目录、允许访问的客户端网络及权限。
```bash
vim /etc/exports
```
以下是配置文件的格式说明：
*   **共享目录**：服务器上要共享的目录路径。
*   **网络范围**：允许访问此共享的客户端IP地址或网段。
*   **权限**：设置在括号`()`内，例如：
    *   `rw`：读写权限。
    *   `ro`：只读权限。
    *   `sync`：同步写入（数据先写入服务器内存，再同步到磁盘）。

一个配置示例如下：
```
/mnt/share 192.168.14.0/24(rw,sync)
```
此配置表示将`/mnt/share`目录共享给`192.168.14.0/24`网段的所有主机，并授予读写权限。

![](img/332cdfe33baea640624e3340f92cb91c_21.png)

![](img/332cdfe33baea640624e3340f92cb91c_23.png)

**3. 应用配置并设置防火墙**
使新的导出配置生效。
```bash
exportfs -rv
```
为了让客户端能够访问，需要在服务端防火墙放行NFS相关服务。
```bash
firewall-cmd --permanent --add-service=nfs
firewall-cmd --permanent --add-service=mountd
firewall-cmd --permanent --add-service=rpc-bind
firewall-cmd --reload
```

## 配置NFS客户端挂载

![](img/332cdfe33baea640624e3340f92cb91c_27.png)

![](img/332cdfe33baea640624e3340f92cb91c_29.png)

服务端配置完成后，我们来看看客户端如何挂载并使用共享目录。

现在，我们在另一台客户端机器（例如`serverb`）上进行操作。首先，验证能否发现服务端发布的共享。
```bash
showmount -e 192.168.14.128
```
如果能看到共享目录列表，说明网络和防火墙配置正确。

以下是客户端挂载的步骤。

![](img/332cdfe33baea640624e3340f92cb91c_31.png)

**1. 创建本地挂载点**
```bash
mkdir -p /mnt/nfs_share
```

![](img/332cdfe33baea640624e3340f92cb91c_33.png)

![](img/332cdfe33baea640624e3340f92cb91c_34.png)

![](img/332cdfe33baea640624e3340f92cb91c_36.png)

**2. 临时挂载NFS共享**
使用`mount`命令进行临时挂载，重启后失效。
```bash
mount -t nfs 192.168.14.128:/mnt/share /mnt/nfs_share
```
*   `-t nfs`：指定文件系统类型为NFS。
*   `192.168.14.128:/mnt/share`：NFS服务端IP及共享目录路径。
*   `/mnt/nfs_share`：本地挂载点目录。

![](img/332cdfe33baea640624e3340f92cb91c_38.png)

![](img/332cdfe33baea640624e3340f92cb91c_39.png)

![](img/332cdfe33baea640624e3340f92cb91c_40.png)

![](img/332cdfe33baea640624e3340f92cb91c_41.png)

![](img/332cdfe33baea640624e3340f92cb91c_43.png)

![](img/332cdfe33baea640624e3340f92cb91c_44.png)

使用`df -Th`命令可以查看挂载结果。

**3. 永久挂载NFS共享**
编辑`/etc/fstab`文件实现开机自动挂载。
```bash
vim /etc/fstab
```
添加如下一行：
```
192.168.14.128:/mnt/share  /mnt/nfs_share  nfs  defaults  0 0
```
保存后，使用`mount -a`命令测试配置是否正确。

**4. 解决客户端写入权限问题**
挂载后，客户端可能无法在共享目录中创建文件。这是因为NFS默认的root_squash安全机制将客户端的root用户映射为服务端的`nobody`用户，而`nobody`通常没有写入权限。

有两种解决方法：
*   **方法一（推荐）**：在NFS服务端，为共享目录赋予其他用户（others）写权限。
    ```bash
    chmod -R o+w /mnt/share
    ```
*   **方法二（不推荐，安全性低）**：在服务端的`/etc/exports`配置中，为共享选项添加`no_root_squash`。这会允许客户端的root用户以root身份写入，存在安全风险。
    ```
    /mnt/share 192.168.14.0/24(rw,sync,no_root_squash)
    ```
    修改后需重启NFS服务：`systemctl restart nfs-server`。

![](img/332cdfe33baea640624e3340f92cb91c_47.png)

![](img/332cdfe33baea640624e3340f92cb91c_48.png)

## 配置Autofs自动挂载

![](img/332cdfe33baea640624e3340f92cb91c_50.png)

手动挂载需要预先执行命令，而Autofs可以实现按需自动挂载，当访问特定目录时自动挂载远程共享，闲置一段时间后自动卸载。

上一节我们完成了手动挂载，本节中我们来看看如何配置更智能的自动挂载。Autofs服务会监控指定的“触发目录”，当用户访问该目录下的子目录时，根据配置规则自动挂载对应的远程资源。

![](img/332cdfe33baea640624e3340f92cb91c_52.png)

![](img/332cdfe33baea640624e3340f92cb91c_54.png)

以下是在客户端（`serverb`）上配置Autofs的步骤。

![](img/332cdfe33baea640624e3340f92cb91c_56.png)

![](img/332cdfe33baea640624e3340f92cb91c_58.png)

![](img/332cdfe33baea640624e3340f92cb91c_60.png)

![](img/332cdfe33baea640624e3340f92cb91c_61.png)

![](img/332cdfe33baea640624e3340f92cb91c_63.png)

**1. 安装并启动Autofs服务**
```bash
dnf install -y autofs
systemctl enable --now autofs
```

**2. 配置主映射文件**
编辑主配置文件`/etc/auto.master`，定义触发目录和对应的映射文件。
```bash
vim /etc/auto.master
```
添加一行，例如：
```
/mnt    /etc/auto.ld
```
*   `/mnt`：触发目录。当访问`/mnt`下的特定子目录时，会触发自动挂载。
*   `/etc/auto.ld`：规则映射文件，名称可以自定义。

**3. 创建并编辑映射文件**
创建上一步定义的映射文件。
```bash
vim /etc/auto.ld
```
在文件中添加挂载规则，格式为：
```
子目录名   [-挂载选项]   服务器:共享目录
```
例如，配置访问`/mnt/nfs_auto`时自动挂载NFS共享：
```
nfs_auto   -fstype=nfs   192.168.14.128:/mnt/share
```

![](img/332cdfe33baea640624e3340f92cb91c_65.png)

**4. 重启服务并测试**
重启Autofs服务使配置生效。
```bash
systemctl restart autofs
```
现在进行测试。最初，`/mnt/nfs_auto`目录并不存在。
```bash
ls /mnt/
```
当你尝试访问或列出这个目录时，Autofs会自动创建它并挂载远程共享。
```bash
ls /mnt/nfs_auto
```
使用`df -Th`命令，可以看到NFS共享已被自动挂载到`/mnt/nfs_auto`。如果超过默认时间（通常5分钟）没有访问，Autofs会自动卸载该共享。

![](img/332cdfe33baea640624e3340f92cb91c_67.png)

## 总结
本节课中我们一起学习了NFS网络文件系统的核心知识。我们实践了如何配置NFS服务端发布共享目录，如何在客户端进行手动挂载和永久挂载，并解决了常见的写入权限问题。最后，我们探索了Autofs自动挂载服务，它能够实现按需挂载和卸载，提升了资源访问的灵活性和便捷性。这些技能是管理Linux系统网络存储的重要组成部分。
