在本节课中，我们将学习Linux中文件重定向的几个简单应用案例。通过具体的例子，你将掌握如何利用重定向快速创建文件、合并文件内容以及处理命令输出，这些技巧在日常系统管理中非常实用。

### 创建新文件

![](img/5d8890fc537442707fc4979881246330_5.png)

我们可以使用重定向符号 `>` 来快速创建一个新文件。例如，使用 `touch` 命令：

![](img/5d8890fc537442707fc4979881246330_7.png)

```bash
touch newfile.txt
```

![](img/5d8890fc537442707fc4979881246330_9.png)

这个命令会创建一个名为 `newfile.txt` 的空文件。需要注意的是，如果目标文件已存在，`>` 操作会覆盖原有内容。

![](img/5d8890fc537442707fc4979881246330_11.png)

---

### 使用 `echo` 创建带内容的文件

`echo` 命令通常将内容输出到屏幕（标准输出）。我们可以通过重定向将其输出保存到文件中。

![](img/5d8890fc537442707fc4979881246330_13.png)

例如，将字符串 “hello 杨哥” 写入文件：

![](img/5d8890fc537442707fc4979881246330_15.png)

```bash
echo "hello 杨哥" > file1.txt
```

![](img/5d8890fc537442707fc4979881246330_17.png)

执行后，`file1.txt` 文件被创建，其内容为 `hello 杨哥`（不包含引号）。建议为包含空格的字符串加上引号。

![](img/5d8890fc537442707fc4979881246330_19.png)

### 创建多行文件

要创建一个包含多行内容的文件，可以使用 `cat` 命令配合重定向。

一种方法是交互式输入。直接运行 `cat > file2.txt`，然后在终端中输入内容，每行结束后按回车。输入完成后，按 `Ctrl + D` 结束输入。

![](img/5d8890fc537442707fc4979881246330_21.png)

```bash
cat > file2.txt
111
222
333
# 按 Ctrl + D 结束
```

![](img/5d8890fc537442707fc4979881246330_23.png)

此时，`file2.txt` 文件包含三行内容：111、222 和 333。在同一个 `cat` 进程中，多次输入不会覆盖，只有再次运行该命令并重定向时才会覆盖原文件。

![](img/5d8890fc537442707fc4979881246330_25.png)

### 在脚本中创建多行文件

![](img/5d8890fc537442707fc4979881246330_27.png)

![](img/5d8890fc537442707fc4979881246330_29.png)

在自动化脚本中，无法手动按 `Ctrl + D`。这时可以使用“Here Document”语法。

其基本格式是 `cat > 文件名 << ‘结束标记’`。输入内容，直到遇到单独一行的“结束标记”为止。

![](img/5d8890fc537442707fc4979881246330_31.png)

![](img/5d8890fc537442707fc4979881246330_33.png)

```bash
cat > file3.txt << YANG
第一行内容
第二行内容
YANG
```

执行后，`file3.txt` 文件被创建，内容为两行文本。标记 `YANG` 本身不会写入文件。常见的标记是 `EOF`，但你可以使用任何独特的字符串作为标记。

![](img/5d8890fc537442707fc4979881246330_35.png)

### 合并多个文件

![](img/5d8890fc537442707fc4979881246330_39.png)

利用 `cat` 命令读取多个文件并重定向输出，可以轻松合并文件内容。

![](img/5d8890fc537442707fc4979881246330_41.png)

假设已有 `yang1.txt`、`yang2.txt`、`yang3.txt` 三个文件，要将它们合并到新文件 `yang_all.txt` 中：

![](img/5d8890fc537442707fc4979881246330_43.png)

```bash
cat yang1.txt yang2.txt yang3.txt > yang_all.txt
```

`cat` 命令会按照参数给定的顺序（`yang1.txt`、`yang2.txt`、`yang3.txt`）读取文件内容，然后将所有输出重定向到 `yang_all.txt` 文件中。

![](img/5d8890fc537442707fc4979881246330_45.png)

### 截取命令输出并保存

![](img/5d8890fc537442707fc4979881246330_51.png)

我们也可以将任何命令的标准输出重定向到文件。例如，使用 `head` 命令查看文件前几行并保存：

查看 `/etc/passwd` 文件的前10行并保存：
```bash
head /etc/passwd > passwd_head.txt
```

![](img/5d8890fc537442707fc4979881246330_53.png)

查看 `/etc/passwd` 文件的前5行并保存：
```bash
head -n 5 /etc/passwd > passwd_head5.txt
```

![](img/5d8890fc537442707fc4979881246330_55.png)

Linux系统不依赖扩展名识别文件类型，因此文件名可以自由定义。

### 核心概念回顾

![](img/5d8890fc537442707fc4979881246330_61.png)

在文件管理操作中，重定向会频繁使用。请记住以下核心概念：

1.  **三个标准文件描述符**：
    *   `0`：标准输入 (stdin)
    *   `1`：标准输出 (stdout)
    *   `2`：标准错误输出 (stderr)

![](img/5d8890fc537442707fc4979881246330_63.png)

2.  **重定向符号**：
    *   `>` 或 `1>`：重定向标准输出（覆盖）。
    *   `>>`：重定向标准输出（追加）。
    *   `2>`：重定向标准错误输出。
    *   `&>`：重定向标准输出和标准错误输出。

![](img/5d8890fc537442707fc4979881246330_65.png)

3.  **特殊设备文件**：
    *   `/dev/null`：空设备。任何输出重定向到此文件都将被丢弃，常用于屏蔽不必要的输出信息。

![](img/5d8890fc537442707fc4979881246330_67.png)
