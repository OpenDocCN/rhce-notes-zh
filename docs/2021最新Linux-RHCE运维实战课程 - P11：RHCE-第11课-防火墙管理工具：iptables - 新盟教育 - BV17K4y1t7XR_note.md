## 防火墙管理工具概述
从红帽7系统开始，防火墙的默认管理工具是firewalld。但需要明确，firewalld只是一个管理工具，类似于一个软件程序。Linux系统同样支持iptables。无论是红帽6还是7系统，其底层的防火墙功能实际上都是由iptables提供的。因此，本课程将以iptables作为主要讲解内容，后续会简要介绍firewalld。

iptables功能强大，参数众多，但无需死记硬背，关键在于理解原理并学会查询使用。

## iptables的组成与核心概念
iptables实际上由两部分组成：
1.  **Netfilter**：位于内核空间，是Linux内核的一部分，负责提供数据包过滤、网络地址转换等核心功能。
2.  **iptables**：位于用户空间，是一个命令行工具，用于插入、修改和删除Netfilter规则表中的规则。

![](img/c6a05523406c80a53870b368287f7464_44.png)

![](img/c6a05523406c80a53870b368287f7464_52.png)

![](img/c6a05523406c80a53870b368287f7464_54.png)

我们通常将这两者统称为iptables。它内置了三张主要的表（table）：
*   **filter表**：负责过滤数据包，是默认的表。
*   **nat表**：负责网络地址转换。
*   **mangle表**：用于修改数据包内容，如做流量整形。

## 三表五链详解
上一节我们介绍了iptables的组成，本节中我们来看看其核心架构“三表五链”。

### 表与链的关系
可以将iptables看作一个选美大赛，不同的“表”（table）是不同的评委组（如filter组、nat组、mangle组），每个评委组内部又有几条“链”（chain），链上挂着具体的“规则”（rule）。

### 各表与链的功能
*   **filter表**：负责过滤。包含三条链：
    *   `INPUT`：处理**进入本机**的数据包。
    *   `FORWARD`：处理**经过本机转发**的数据包。
    *   `OUTPUT`：处理**从本机发出**的数据包。
*   **nat表**：负责地址转换。包含三条链：
    *   `PREROUTING`：在数据包进入路由决策之前修改目的地址（DNAT）。
    *   `POSTROUTING`：在数据包离开路由决策之后修改源地址（SNAT）。
    *   `OUTPUT`：处理本机产生的数据包的地址转换。
*   **mangle表**：用于修改数据包内容。五条链（PREROUTING, INPUT, FORWARD, OUTPUT, POSTROUTING）都有。

### 数据包流经顺序
理解数据包流经防火墙的顺序至关重要。下图描绘了其核心流程：

```
数据包进入
    |
    v
[PREROUTING] (mangle, nat) // 路由前处理
    |
    v
路由决策
    |
    |---> 目的地是本机？ ---> [INPUT] (mangle, filter) ---> 本地进程
    |                           |
    |                           v
    |                        [OUTPUT] (mangle, nat, filter)
    |                           |
    |                           v
    |                       [POSTROUTING] (mangle, nat)
    |
    |---> 需要转发？ -------> [FORWARD] (mangle, filter) ---> [POSTROUTING] (mangle, nat)
```

![](img/c6a05523406c80a53870b368287f7464_95.png)

![](img/c6a05523406c80a53870b368287f7464_97.png)

![](img/c6a05523406c80a53870b368287f7464_99.png)

![](img/c6a05523406c80a53870b368287f7464_101.png)

**匹配原则**：
1.  规则匹配顺序为**自上而下，依次进行**。
2.  一旦某条规则匹配成功，就执行对应动作（如ACCEPT, DROP），并**停止继续匹配**。
3.  如果链中所有规则都不匹配，则执行该链的**默认策略**（policy）。

![](img/c6a05523406c80a53870b368287f7464_103.png)

![](img/c6a05523406c80a53870b368287f7464_105.png)

因此，**越严格、越具体的规则应该放在越前面**。

## iptables基本操作与规则管理
上一节我们理清了数据包的流转路径，本节中我们来看看如何使用iptables命令进行实际操作。

### 安装与启动
在红帽7系统中，iptables通常已安装。如需安装，可使用：
```bash
yum install -y iptables iptables-services
```
启动并设置开机自启：
```bash
systemctl start iptables
systemctl enable iptables
```
**注意**：为避免冲突，通常需要停止并禁用firewalld：
```bash
systemctl stop firewalld
systemctl disable firewalld
```

### 规则配置文件
iptables的规则默认保存在 `/etc/sysconfig/iptables` 文件中。使用 `service iptables save` 或 `iptables-save > /etc/sysconfig/iptables` 命令可以将当前内存中的规则保存到此文件，以便重启后生效。

![](img/c6a05523406c80a53870b368287f7464_119.png)

### 命令语法结构
iptables命令的基本语法如下：
```
iptables [-t 表名] 管理选项 [链名] [匹配条件] [-j 目标动作]
```
*   `-t 表名`：指定操作的表，如filter、nat、mangle。**省略时默认为filter表**。
*   `管理选项`：指定操作方式，如添加、删除、查看规则。
*   `链名`：指定操作的链。
*   `匹配条件`：指定要匹配的数据包特征，如源IP、目标IP、协议、端口等。
*   `-j 目标动作`：指定匹配后的处理动作，如ACCEPT、DROP。

### 常用管理选项
以下是管理规则的核心命令：

![](img/c6a05523406c80a53870b368287f7464_126.png)

![](img/c6a05523406c80a53870b368287f7464_128.png)

**规则添加与删除**
*   `-A`：在指定链的**末尾追加**一条规则。
    ```bash
    iptables -A INPUT -s 192.168.1.100 -j DROP # 在INPUT链末尾添加拒绝192.168.1.100的规则
    ```
*   `-I`：在指定链的**指定位置插入**一条规则（默认插入到第一行）。
    ```bash
    iptables -I INPUT 3 -p tcp --dport 80 -j ACCEPT # 在INPUT链第3行插入允许80端口的规则
    ```
*   `-D`：从指定链中**删除**一条规则（可按规则内容或行号删除）。
    ```bash
    iptables -D INPUT -s 192.168.1.100 -j DROP # 按内容删除
    iptables -D INPUT 3 # 按行号删除
    ```

**规则查看与清空**
*   `-L`：列出指定链或所有链的规则。
    ```bash
    iptables -L # 查看filter表所有规则
    iptables -t nat -L # 查看nat表所有规则
    iptables -L --line-numbers # 查看规则并显示行号
    ```
*   `-F`：清空指定链或所有链的规则。
    ```bash
    iptables -F INPUT # 清空INPUT链规则
    iptables -F # 清空所有链规则
    ```
*   `-P`：设置链的**默认策略**。只有DROP和ACCEPT两种。
    ```bash
    iptables -P INPUT DROP # 将INPUT链默认策略设置为拒绝（高危操作！）
    ```
    **警告**：在生产环境中，如果远程设置默认策略为DROP，必须先放行SSH等管理端口，否则会导致无法连接。

### 常用匹配条件
匹配条件用于筛选出需要处理的数据包。

![](img/c6a05523406c80a53870b368287f7464_177.png)

![](img/c6a05523406c80a53870b368287f7464_179.png)

![](img/c6a05523406c80a53870b368287f7464_181.png)

![](img/c6a05523406c80a53870b368287f7464_183.png)

![](img/c6a05523406c80a53870b368287f7464_185.png)

![](img/c6a05523406c80a53870b368287f7464_187.png)

![](img/c6a05523406c80a53870b368287f7464_189.png)

**基于网络接口**
*   `-i`：匹配数据包的**进入**网卡（如 `-i eth0`）。
*   `-o`：匹配数据包的**流出**网卡（如 `-o eth1`）。

![](img/c6a05523406c80a53870b368287f7464_191.png)

![](img/c6a05523406c80a53870b368287f7464_192.png)

![](img/c6a05523406c80a53870b368287f7464_193.png)

**基于地址**
*   `-s`：匹配**源IP地址**或网段（如 `-s 192.168.1.0/24`）。
*   `-d`：匹配**目标IP地址**或网段。

**基于协议和端口**
*   `-p`：指定协议，如 `tcp`、`udp`、`icmp`。
*   `--sport`：匹配**源端口**（需配合 `-p` 使用）。
*   `--dport`：匹配**目标端口**（需配合 `-p` 使用）。
    ```bash
    iptables -A INPUT -p tcp --dport 22 -j ACCEPT # 允许访问本机SSH端口
    iptables -A INPUT -p udp --dport 53 -j DROP # 拒绝访问本机DNS端口
    ```
    **端口范围**使用冒号连接，如 `1000:3000`。

**其他匹配**
*   `-m state --state`：基于连接状态匹配，如 `NEW`（新连接）、`ESTABLISHED`（已建立的连接）、`RELATED`（相关的连接）。**这是实现“状态防火墙”的关键，非常有用**。
    ```bash
    iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT # 允许已建立和相关连接
    ```
*   `-m mac --mac-source`：基于源MAC地址匹配。
*   `-m limit --limit`：限制数据包速率。

### 处理动作（-j 参数）
*   `ACCEPT`：允许数据包通过。
*   `DROP`：丢弃数据包，**不返回任何信息**。
*   `REJECT`：拒绝数据包，**通常会返回拒绝信息**（如`port unreachable`）。
*   `SNAT`：源地址转换，用于共享上网。
    ```bash
    iptables -t nat -A POSTROUTING -s 192.168.0.0/24 -o eth0 -j SNAT --to-source 公网IP
    ```
*   `DNAT`：目的地址转换，用于端口映射或负载均衡。
    ```bash
    iptables -t nat -A PREROUTING -i eth0 -p tcp --dport 80 -j DNAT --to-destination 内网服务器IP:80
    ```
*   `MASQUERADE`：动态SNAT，常用于拨号上网等IP不固定的情况。
    ```bash
    iptables -t nat -A POSTROUTING -o ppp0 -j MASQUERADE
    ```
*   `LOG`：记录日志，然后继续匹配下一条规则。
