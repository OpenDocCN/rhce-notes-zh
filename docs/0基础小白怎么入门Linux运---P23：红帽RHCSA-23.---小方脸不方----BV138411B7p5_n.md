## 磁盘物理结构简介

上一节我们介绍了课程概述，本节中我们来看看磁盘的物理结构。在企业环境中，如果使用云主机，通常看不到物理硬盘。但在传统物理服务器上，硬盘是可见的硬件组件。

服务器硬盘的外观类似一块板砖，安装在服务器的硬盘插槽中。一个服务器可能有多个硬盘插槽，例如12个或16个。硬盘可以插入对应的插口，通常有一个开关按钮，按下后可以弹出托架以便插入硬盘。

现代硬盘大多支持热插拔功能。热插拔意味着可以在系统开机运行时插入或移除硬盘，系统能立即识别，无需重启。这类似于使用U盘。

硬盘内部主要由以下部件组成：
*   **主轴马达**：驱动盘片旋转。
*   **盘片**：数据存储的介质，类似光盘。
*   **机械手臂与读写磁头**：机械手臂末端的读写磁头在盘片旋转时，通过摩擦盘片表面进行数据的读取和写入。

硬盘内部是真空环境，以防止灰尘进入导致盘片损坏和数据丢失。因此，硬盘一旦被拆开，通常就无法再使用了。

## 磁盘存储的基本概念：扇区

在深入分区之前，我们需要理解一个核心概念：**扇区**。扇区是磁盘存储数据的最小单元。

一块硬盘需要被划分成多个分区来使用。在划分分区时，计算机以扇区为单位进行计算。默认情况下，一个扇区可以存储 **512个字节** 的数据。

当划分一个分区（例如10GB）时，计算机会计算需要多少个扇区来组成这个空间。公式可以简化为：
`所需扇区数 = 分区大小 (字节) / 512字节`

不过，在实际操作中，我们通常直接指定分区大小（如10G、20G），而无需手动计算扇区数量。

## 磁盘接口类型

服务器硬盘有不同的接口型号，必须与服务器插槽匹配。常见的接口类型有：

以下是几种常见的磁盘接口类型：
*   **IDE**：早期接口，数据传输速度慢，价格便宜。约在2009年后在服务器领域被淘汰，曾用于个人电脑。
*   **SCSI**：早期服务器领域常用接口，数据传输速度快。在2013年至2017年间流行，并开始支持热插拔。
*   **SATA**：目前个人电脑领域主流接口，服务器也有使用，数据传输速度较快。
*   **SAS**：当前服务器领域流行接口，性能较好。
*   **SSD (固态硬盘)**：利用内存机制存储数据，速度远快于传统机械硬盘，但价格较高。
*   **NVMe**：一种用于固态硬盘的高速传输协议，能进一步提升数据传输速度。

选择硬盘时，接口性能与价格相关。高性能硬件通常意味着更高的成本。

## 磁盘分区格式

对硬盘进行分区时，需要选择一种分区格式。主要有两种格式：MBR和GPT。

### MBR分区格式

MBR是一种较古老的分区格式。它有一个主要限制：**一块硬盘最多只能创建4个主分区**。

对于许多用户来说，4个分区可能足够（例如：系统盘、软件盘、资料盘、娱乐盘）。但对于有更多分类需求的用户或企业服务器，4个分区可能不够。

![](img/9d2cc618fab23493e5dc767f070739b8_9.png)

![](img/9d2cc618fab23493e5dc767f070739b8_11.png)

![](img/9d2cc618fab23493e5dc767f070739b8_13.png)

![](img/9d2cc618fab23493e5dc767f070739b8_15.png)

为了解决这个限制，MBR引入了**扩展分区**的概念。扩展分区本身不直接存储数据，而是作为一个容器，在其内部可以划分多个**逻辑分区**。

![](img/9d2cc618fab23493e5dc767f070739b8_16.png)

![](img/9d2cc618fab23493e5dc767f070739b8_18.png)

![](img/9d2cc618fab23493e5dc767f070739b8_20.png)

因此，在MBR格式下，一块硬盘的分区结构通常是：
*   1-3个**主分区** (用于存储数据)
*   1个**扩展分区** (作为容器)
*   在扩展分区内创建N个**逻辑分区** (用于存储数据)

![](img/9d2cc618fab23493e5dc767f070739b8_22.png)

![](img/9d2cc618fab23493e5dc767f070739b8_24.png)

![](img/9d2cc618fab23493e5dc767f070739b8_26.png)

![](img/9d2cc618fab23493e5dc767f070739b8_28.png)

逻辑分区的数量限制取决于磁盘类型，例如IDE硬盘最多59个，SCSI硬盘最多11个。

![](img/9d2cc618fab23493e5dc767f070739b8_32.png)

![](img/9d2cc618fab23493e5dc767f070739b8_34.png)

然而，MBR还有一个致命的缺点：**最大仅支持2.2TB以内的磁盘容量**。对于更大的硬盘，MBR无法有效管理。

### GPT分区格式

![](img/9d2cc618fab23493e5dc767f070739b8_44.png)

![](img/9d2cc618fab23493e5dc767f070739b8_46.png)

GPT是一种现代的分区格式，专为管理大容量硬盘设计。

![](img/9d2cc618fab23493e5dc767f070739b8_48.png)

![](img/9d2cc618fab23493e5dc767f070739b8_50.png)

![](img/9d2cc618fab23493e5dc767f070739b8_52.png)

GPT格式的优点非常明显：
*   **分区数量几乎无限制**：理论上可创建**128个主分区**，无需使用扩展分区和逻辑分区的复杂概念。
*   **支持超大容量**：最大可支持约 **18EB** 的磁盘容量。容量换算关系为：
    `1024字节 = 1KB` -> `1024KB = 1MB` -> `1024MB = 1GB` -> `1024GB = 1TB` -> `1024TB = 1PB` -> `1024PB = 1EB`

![](img/9d2cc618fab23493e5dc767f070739b8_54.png)

![](img/9d2cc618fab23493e5dc767f070739b8_56.png)

![](img/9d2cc618fab23493e5dc767f070739b8_58.png)

**总结如何选择**：
*   如果磁盘容量小于2.2TB，可以选择MBR或GPT格式。
*   如果磁盘容量大于2.2TB，或者需要创建大量分区，则必须使用GPT格式。

**1. `lsblk` - 列出块设备信息**
这条命令用于列出系统中所有的磁盘和分区信息。
```bash
lsblk
```
执行后，你会看到类似 `sda`、`sdb` 的磁盘名称，以及 `sda1`、`sda2` 等分区名称。
*   名称解读：`sd` 代表接口类型（如SATA, SCSI），`a`、`b`、`c`... 代表第1、2、3...块磁盘，后面的数字 `1`、`2`... 代表该磁盘上的分区编号。
*   常用选项：
    *   `lsblk -d`：仅显示磁盘本身信息，不显示分区。
    *   `lsblk -f`：显示文件系统类型、UUID等详细信息。

![](img/9d2cc618fab23493e5dc767f070739b8_94.png)

![](img/9d2cc618fab23493e5dc767f070739b8_96.png)

![](img/9d2cc618fab23493e5dc767f070739b8_98.png)

**2. `df` - 报告文件系统磁盘空间使用情况**
这条命令用于查看**已挂载**文件系统的磁盘空间使用情况。
```bash
df -h
```
*   `-h` 选项表示以人类易读的格式（如K, M, G）显示大小。
*   该命令显示每个挂载点的总空间、已用空间、可用空间和使用百分比。

![](img/9d2cc618fab23493e5dc767f070739b8_100.png)

![](img/9d2cc618fab23493e5dc767f070739b8_102.png)

**3. `du` - 估算文件或目录的磁盘使用量**
这条命令用于统计具体文件或目录占用的磁盘空间大小。
```bash
du -sh /etc
```
*   `-s` 选项表示只显示总用量。
*   `-h` 选项表示以人类易读的格式显示。
*   与 `ls -lh` 命令的区别：`ls -lh` 查看目录时，只显示目录元数据的大小，而不统计目录内所有文件内容的总大小。`du` 命令则会递归统计目录下所有文件的实际数据总大小。

![](img/9d2cc618fab23493e5dc767f070739b8_104.png)

![](img/9d2cc618fab23493e5dc767f070739b8_106.png)

![](img/9d2cc618fab23493e5dc767f070739b8_108.png)

**4. `blkid` - 显示块设备的属性**
这条命令用于查看块设备（如分区）的文件系统类型、UUID等属性。
```bash
blkid /dev/sda1
```
其功能与 `lsblk -f /dev/sda1` 类似。

![](img/9d2cc618fab23493e5dc767f070739b8_110.png)

![](img/9d2cc618fab23493e5dc767f070739b8_112.png)

### 命令使用总结

![](img/9d2cc618fab23493e5dc767f070739b8_114.png)

为了更清晰地理解，我们对这些命令的常见用途进行总结：
*   想查看系统中有多少块硬盘以及每个硬盘的分区情况 → 使用 **`lsblk`**。
*   想查看各个已挂载分区的空间使用率（用了多少，还剩多少） → 使用 **`df -h`**。
*   想统计某个具体目录或文件总共占用了多大磁盘空间 → 使用 **`du -sh`**。
*   想查看某个分区的文件系统类型等属性 → 使用 **`blkid`** 或 **`lsblk -f`**。
