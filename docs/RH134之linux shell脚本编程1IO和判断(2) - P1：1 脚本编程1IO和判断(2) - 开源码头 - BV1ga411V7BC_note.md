## 概述
在本节课中，我们将要学习Linux Shell脚本编程的基础知识，重点包括脚本的创建与执行、变量的使用，以及字符串输出中的引用机制。我们将通过简单的例子，理解如何编写一个可执行的脚本，并掌握`echo`命令在输出字符串时，单引号、双引号和转义符的不同作用。

## 脚本的创建与执行 🚀

Linux Shell脚本的运行环境是命令行，因此不需要像C语言那样包含头文件或调用库。

首先，我们创建一个目录并进入该目录：
```bash
mkdir ~/SCRIPT
cd ~/SCRIPT
```

接着，我们使用`vim`编辑器创建一个脚本文件。建议使用`.sh`作为扩展名，这样编辑器能提供语法高亮和验证。
```bash
vim hello.sh
```

在脚本的第一行，我们需要写入“魔术行”（Shebang），它告诉系统使用哪个解释器来执行此脚本。对于Bash，我们这样写：
```bash
#!/bin/bash
```

![](img/77895045e1a93098eeaaed3c7a913ce8_2.png)

魔术行之后，我们可以编写命令。让我们写一个最简单的`echo`命令来输出“Hello World”：
```bash
echo Hello World
```

![](img/77895045e1a93098eeaaed3c7a913ce8_4.png)

保存并退出编辑器（按`Esc`键，然后输入`:wq`）。默认情况下，新建的脚本文件没有执行权限。我们可以使用`chmod`命令为其添加执行权限：
```bash
chmod +x hello.sh
```

现在，文件对所有用户都有了执行权限。要执行脚本，如果其所在目录不在系统的`PATH`环境变量中，我们需要指定路径：
```bash
./hello.sh
```

![](img/77895045e1a93098eeaaed3c7a913ce8_6.png)

如果希望在任何目录都能直接通过文件名调用脚本，可以将其移动到`PATH`包含的目录中，例如`~/bin`：
```bash
mkdir -p ~/bin
cp hello.sh ~/bin/
```
之后，就可以直接输入`hello.sh`来运行脚本了。

![](img/77895045e1a93098eeaaed3c7a913ce8_8.png)

## 字符串输出与引用机制 📝

![](img/77895045e1a93098eeaaed3c7a913ce8_10.png)

上一节我们介绍了如何创建和执行一个简单的脚本。本节中，我们来看看`echo`命令输出字符串时，如何处理特殊字符。

![](img/77895045e1a93098eeaaed3c7a913ce8_12.png)

### 引用的必要性
在脚本中，井号(`#`)通常表示注释。如果我们想原样输出包含井号的字符串，直接写`echo #Hello World!`是无效的，因为`#`之后的内容会被视为注释。

为了解决这个问题，我们需要使用引号。

![](img/77895045e1a93098eeaaed3c7a913ce8_14.png)

![](img/77895045e1a93098eeaaed3c7a913ce8_16.png)

### 双引号 - 弱引用
使用双引号(`"`)将字符串括起来，可以保留大部分字符的原意，但允许进行**变量替换**和**命令替换**。

以下是变量替换的例子。我们先定义一个变量：
```bash
A=张三
```
然后在`echo`命令中引用它：
```bash
echo "$A Hello World!"
```
执行后，变量`$A`的内容“张三”会被替换到字符串中输出。

![](img/77895045e1a93098eeaaed3c7a913ce8_18.png)

![](img/77895045e1a93098eeaaed3c7a913ce8_20.png)

以下是命令替换的例子。我们使用反引号或`$()`来执行一个命令，并将其结果嵌入字符串：
```bash
echo "当前主机名: $(hostname)"
```
双引号内的`$(hostname)`会被执行，其输出结果成为字符串的一部分。

**总结双引号的特点**：允许变量和命令替换，但会取消大部分特殊字符（如`$`, `` ` ``, `\`）的特殊含义，使其成为普通字符。

![](img/77895045e1a93098eeaaed3c7a913ce8_22.png)

### 单引号 - 强引用
使用单引号(`'`)将字符串括起来，会强制将其中所有字符都作为普通文本处理，**禁止任何形式的替换或解释**。

例如：
```bash
echo '$A Hello World! $(hostname)'
```
执行后，会原样输出字符串`$A Hello World! $(hostname)`，而不会进行变量替换或命令替换。

![](img/77895045e1a93098eeaaed3c7a913ce8_24.png)

**总结单引号的特点**：所有内容都按字面意义输出，不做任何解释。

### 转义符
反斜杠(`\`)是转义符，用于取消单个紧随其后的字符的特殊含义。

例如，我们想输出一个美元符号：
```bash
echo The price is \$100.
```
这里的`\$`表示输出字符`$`本身，而不是引入一个变量。

如果字符串中只有少量特殊字符，使用转义符很方便。但如果特殊字符很多，在每个前面都加转义符会很繁琐，这时使用单引号是更好的选择。

![](img/77895045e1a93098eeaaed3c7a913ce8_26.png)

![](img/77895045e1a93098eeaaed3c7a913ce8_28.png)

## 总结
本节课中我们一起学习了Linux Shell脚本编程的入门知识。

![](img/77895045e1a93098eeaaed3c7a913ce8_30.png)

我们首先了解了如何创建、赋予权限并执行一个Shell脚本，包括“魔术行”的作用和`PATH`环境变量的重要性。

接着，我们深入探讨了`echo`命令输出字符串时的三种引用机制：
1.  **双引号（弱引用）**：允许变量和命令替换，适合需要动态生成内容的字符串。
2.  **单引号（强引用）**：禁止一切替换，所有字符原样输出，适合包含大量特殊字符的固定字符串。
3.  **转义符**：用于临时取消单个字符的特殊含义，适合处理字符串中个别的特殊符号。

![](img/77895045e1a93098eeaaed3c7a913ce8_32.png)

理解并熟练运用这些引用机制，是编写健壮、清晰Shell脚本的基础。在后续课程中，我们将利用这些知识，学习更复杂的逻辑判断和流程控制。
