## 压缩的作用与好处

![](img/c48328c9ce17d46a4be02982353077ad_5.png)

![](img/c48328c9ce17d46a4be02982353077ad_7.png)

压缩的主要目的是减少文件的大小。这可以带来两个直接的好处：一是节约磁盘存储空间，二是提高文件在网络上的传输速度。例如，在传输大型项目文件或备份系统日志时，先进行压缩可以显著提升效率。

![](img/c48328c9ce17d46a4be02982353077ad_9.png)

![](img/c48328c9ce17d46a4be02982353077ad_11.png)

![](img/c48328c9ce17d46a4be02982353077ad_13.png)

在互联网中，常见的音视频（如MP3、MP4）和图片（如JPG、PNG）格式本身就是压缩格式，这使得它们能够被快速加载和传输。

## Linux下的常见压缩格式

![](img/c48328c9ce17d46a4be02982353077ad_17.png)

在Linux系统中，有三种主流的压缩命令，它们对应不同的压缩格式和算法：

![](img/c48328c9ce17d46a4be02982353077ad_19.png)

![](img/c48328c9ce17d46a4be02982353077ad_21.png)

![](img/c48328c9ce17d46a4be02982353077ad_23.png)

1.  **gzip**: 压缩速度较快，压缩比例一般。
2.  **bzip2**: 压缩速度中等，压缩比例较高。
3.  **xz**: 压缩速度较慢，压缩比例最高。

![](img/c48328c9ce17d46a4be02982353077ad_25.png)

选择哪种压缩格式，取决于你对速度和压缩率的需求。如果需要快速压缩，可以选择`gzip`；如果需要尽可能减小文件体积，可以选择`xz`。

![](img/c48328c9ce17d46a4be02982353077ad_27.png)

## 压缩格式对比演示

![](img/c48328c9ce17d46a4be02982353077ad_33.png)

![](img/c48328c9ce17d46a4be02982353077ad_35.png)

为了直观地比较这三种格式，我们以系统文件 `/etc/services` 为例进行压缩。

![](img/c48328c9ce17d46a4be02982353077ad_37.png)

首先，复制文件并查看其原始大小：
```bash
cp /etc/services /opt/
du -h /opt/services
```
输出显示文件大小约为 **656K**。

![](img/c48328c9ce17d46a4be02982353077ad_39.png)

![](img/c48328c9ce17d46a4be02982353077ad_41.png)

![](img/c48328c9ce17d46a4be02982353077ad_43.png)

接下来，我们分别用三种格式进行压缩。

![](img/c48328c9ce17d46a4be02982353077ad_45.png)

![](img/c48328c9ce17d46a4be02982353077ad_47.png)

![](img/c48328c9ce17d46a4be02982353077ad_49.png)

### 1. 使用 gzip 压缩
命令格式为 `gzip [文件名]`。
```bash
cd /opt
gzip services
```
压缩后，原文件 `services` 会变成 `services.gz`。查看压缩后大小：
```bash
du -h services.gz
```
输出约为 **136K**。

![](img/c48328c9ce17d46a4be02982353077ad_51.png)

**查看压缩文件内容**：不能使用 `cat`，需使用 `zcat`。
```bash
zcat services.gz
```

![](img/c48328c9ce17d46a4be02982353077ad_53.png)

![](img/c48328c9ce17d46a4be02982353077ad_55.png)

![](img/c48328c9ce17d46a4be02982353077ad_57.png)

**解压缩**：使用 `-d` 选项。
```bash
gzip -d services.gz
```
解压后，文件恢复为原始大小。

![](img/c48328c9ce17d46a4be02982353077ad_59.png)

### 2. 使用 bzip2 压缩
如果系统未安装 `bzip2`，需先安装：
```bash
yum install -y bzip2
```
压缩命令：
```bash
bzip2 services
```
压缩后文件为 `services.bz2`，大小约为 **124K**。

![](img/c48328c9ce17d46a4be02982353077ad_61.png)

**查看内容**：使用 `bzcat`。
```bash
bzcat services.bz2
```

![](img/c48328c9ce17d46a4be02982353077ad_63.png)

![](img/c48328c9ce17d46a4be02982353077ad_64.png)

**解压缩**：同样使用 `-d` 选项。
```bash
bzip2 -d services.bz2
```

### 3. 使用 xz 压缩
压缩命令：
```bash
xz services
```
压缩后文件为 `services.xz`，大小约为 **100K**。可以看到，`xz` 的压缩率最高，但压缩过程会稍有停顿，速度最慢。

![](img/c48328c9ce17d46a4be02982353077ad_66.png)

**解压缩**：
```bash
xz -d services.xz
```

![](img/c48328c9ce17d46a4be02982353077ad_68.png)

**总结对比**：
*   **gzip**: 速度最快，压缩后 136K。
*   **bzip2**: 速度中等，压缩后 124K。
*   **xz**: 速度最慢，压缩后 100K（压缩率最高）。

## tar 打包工具

![](img/c48328c9ce17d46a4be02982353077ad_86.png)

![](img/c48328c9ce17d46a4be02982353077ad_88.png)

![](img/c48328c9ce17d46a4be02982353077ad_90.png)

![](img/c48328c9ce17d46a4be02982353077ad_92.png)

![](img/c48328c9ce17d46a4be02982353077ad_94.png)

上一节我们介绍了三种针对单个文件的压缩工具。但它们在面对目录时有一个共同的缺陷：**无法直接压缩目录**。

![](img/c48328c9ce17d46a4be02982353077ad_96.png)

这时，就需要用到 `tar` 命令。`tar` 本身是一个“打包”工具，它可以将多个文件或整个目录合并成一个单独的包文件（通常以 `.tar` 结尾）。打包本身并不改变文件大小，但将目录转换为单个文件后，我们就可以方便地使用 `gzip`、`bzip2` 或 `xz` 对这个包文件进行压缩了。

### 打包与压缩的联合使用

![](img/c48328c9ce17d46a4be02982353077ad_98.png)

`tar` 命令的选项较多，以下是创建打包文件时的核心选项组合：
*   `-c`: 创建新的打包文件。
*   `-f`: 指定打包后的文件名。**此选项必须放在所有选项的最后**。
*   `-z`: 调用 `gzip` 进行压缩/解压。
*   `-j`: 调用 `bzip2` 进行压缩/解压。
*   `-J`: 调用 `xz` 进行压缩/解压。
*   `-v`: 显示详细的处理过程（可选）。
*   `-x`: 解包/解压。
*   `-C`: 指定解压到的目录。

![](img/c48328c9ce17d46a4be02982353077ad_100.png)

以下是常见操作示例：

**1. 仅打包（不压缩）**
将 `/var/log` 目录打包成 `log.tar`。
```bash
tar -cf log.tar /var/log
```

**2. 打包并直接用 gzip 压缩**
打包 `/var/log` 目录，并直接用 gzip 压缩，生成 `log.tar.gz`。
```bash
tar -czf log.tar.gz /var/log
```

**3. 打包并直接用 xz 压缩**
打包 `/var/log` 目录，并直接用 xz 压缩，生成 `log.tar.xz`。
```bash
tar -cJf log.tar.xz /var/log
```

**4. 查看打包/压缩包内的文件列表**
使用 `-t` 选项。
```bash
tar -tzf log.tar.gz
```

**5. 解压到当前目录**
```bash
tar -xzf log.tar.gz
```

**6. 解压到指定目录**
使用 `-C` 选项，解压到 `/opt` 目录。
```bash
tar -xzf log.tar.gz -C /opt
```

**关于“删除开头的`/`”提示**：在执行打包命令时，你可能会看到“从成员名中删除开头的`/`”的提示。这是 `tar` 的安全特性，它移除了文件路径前的根目录符号 `/`。这样在解压时，你就可以自由地指定解压路径（如 `-C /opt`），而不会强制覆盖系统原始路径。

本节课中我们一起学习了Linux下的文件压缩与打包。
*   我们了解了 **gzip、bzip2、xz** 三种压缩工具，知道了它们分别在速度和压缩率上各有侧重。
*   我们掌握了 **tar** 这个强大的打包工具，学会了如何将目录打包成单个文件，并联合压缩工具进行一步到位的打包压缩操作。
*   我们熟悉了 `tar` 命令的 `-c`（创建）、`-x`（解压）、`-z/-j/-J`（调用压缩）、`-f`（指定文件名）、`-C`（指定解压路径）等关键选项的使用。
