## 软链接与硬链接 🔗

上一节我们介绍了文件的基本操作，本节中我们来看看如何创建文件的链接。链接文件类似于Windows系统中的快捷方式，但在Linux中分为**软链接**和**硬链接**两种，它们有本质的区别。

### 核心概念与区别

![](img/80d048d7e04503f33678ec98d9d6d31e_3.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_5.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_7.png)

简单来说：
*   **软链接**：完全等同于Windows的快捷方式。它只是一个指向目标文件的**路径指针**。如果目标文件被删除，软链接将失效。创建命令为 `ln -s`。
*   **硬链接**：相当于为同一个文件数据创建了**多个访问入口**（文件名）。所有硬链接都指向磁盘上相同的**数据块**。只要还有一个硬链接存在，文件数据就不会丢失。创建命令为 `ln`。

![](img/80d048d7e04503f33678ec98d9d6d31e_9.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_11.png)

**公式/代码描述**：
*   创建软链接：`ln -s <源文件> <链接文件名>`
*   创建硬链接：`ln <源文件> <链接文件名>`

### 访问原理浅析

为了更好地理解，我们可以看看文件的访问原理：
1.  Linux中每个文件都有一个唯一的**inode编号**，存储文件的元数据（属性、权限等）和指向数据块的指针。
2.  目录项将**文件名**映射到对应的**inode**。
3.  **硬链接**就是创建一个新的目录项（新文件名），指向**同一个inode**。因此，inode的“链接数”会增加。
4.  **软链接**本身是一个独立的文件，拥有自己的inode和数据块，但其数据块中存储的是**目标文件的路径**。访问软链接时，系统会读取这个路径，然后去查找目标文件。

### 特性对比

以下是软链接与硬链接的主要特性总结：

*   **软链接**
    *   可以指向目录。
    *   可以跨文件系统（分区）。
    *   源文件被删除后，链接失效（成为“悬空链接”）。
    *   文件大小是其指向的路径字符串的长度。
*   **硬链接**
    *   不能指向目录（仅限超级用户在某些系统上可创建目录的硬链接，但不推荐）。
    *   不能跨文件系统（因为inode编号仅在同一个文件系统内唯一）。
    *   删除一个硬链接，不影响其他硬链接对文件的访问。
    *   与源文件完全平等，无法区分谁是“原始文件”。

**何时使用**：
*   当你需要为文件或目录创建一个方便的快捷方式，或者想隐藏真实文件位置时，使用**软链接**。
*   当你需要为重要文件创建多个等效的访问路径，防止误删除导致数据丢失时，可以使用**硬链接**（但更常见的备份方式是复制）。

## 通配符 🌟

在文件管理中，我们经常需要匹配多个文件。Linux的通配符可以帮助我们通过模糊匹配的方式，快速查找和操作成组文件。

通配符主要用于命令行中，结合特殊符号进行模式匹配。

以下是常用的通配符及其用法：

*   `*`（星号）：匹配任意长度的任意字符（包括零个字符）。
    *   示例：`ls *.txt` 列出所有以`.txt`结尾的文件。
*   `?`（问号）：匹配任意**单个**字符。
    *   示例：`ls file?.txt` 匹配 `file1.txt`, `fileA.txt`，但不匹配 `file10.txt`。
*   `[ ]`（中括号）：匹配括号内列出的**任意一个**字符。
    *   示例：`ls file[123].txt` 匹配 `file1.txt`, `file2.txt`, `file3.txt`。
    *   可以使用短横线表示范围：`[0-9]` 匹配数字，`[a-z]` 匹配小写字母。
*   `[! ]` 或 `[^ ]`：匹配**不在**括号内列出的任意一个字符（取反）。
    *   示例：`ls file[!0-9].txt` 匹配不以数字结尾的`fileX.txt`。
*   `{ }`（大括号）：生成一个序列或列表，匹配其中用逗号分隔的任意一个**完整字符串**。这不是模式匹配，而是扩展。
    *   示例：`touch file{1,3,5}.txt` 会创建 `file1.txt`, `file3.txt`, `file5.txt` 三个文件。
    *   示例：`cp /path/to/file.{txt,bak}` 等同于 `cp /path/to/file.txt /path/to/file.bak`。

## 获取帮助 📚

Linux系统拥有强大的内置帮助系统。当你不熟悉某个命令的用法时，无需立即上网搜索，可以优先使用以下方法获取帮助。

![](img/80d048d7e04503f33678ec98d9d6d31e_17.png)

### 命令内置帮助 (`--help`)

![](img/80d048d7e04503f33678ec98d9d6d31e_18.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_20.png)

大多数命令都支持 `--help` 或 `-h` 选项，它会输出该命令的简要用法说明、参数和选项列表。

![](img/80d048d7e04503f33678ec98d9d6d31e_21.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_22.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_24.png)

**代码示例**：
```bash
ls --help
```

### 手册页 (`man`)

![](img/80d048d7e04503f33678ec98d9d6d31e_31.png)

`man`（manual的缩写）命令提供非常详细的手册页，包含命令的描述、语法、选项、示例等。通常分为多个章节（如1-用户命令，5-文件格式）。

![](img/80d048d7e04503f33678ec98d9d6d31e_33.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_35.png)

**口诀**：“是男人，就 man 一下”。

![](img/80d048d7e04503f33678ec98d9d6d31e_37.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_38.png)

![](img/80d048d7e04503f33678ec98d9d6d31e_39.png)

**代码示例**：
```bash
man ls          # 查看ls命令的手册
man 5 passwd    # 查看passwd配置文件的格式手册（第5章）
man -k passwd   # 在所有手册页中搜索包含“passwd”关键词的条目
```

### 官方文档目录

![](img/80d048d7e04503f33678ec98d9d6d31e_56.png)

系统的完整文档通常存放在以下目录中，你可以直接去查阅：
*   `/usr/share/doc`：存放各类软件包的官方文档。
*   `/usr/share/man`：存放所有 `man` 手册页的源文件。
