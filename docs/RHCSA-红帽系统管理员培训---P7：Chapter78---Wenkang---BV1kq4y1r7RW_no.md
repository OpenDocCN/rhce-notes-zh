## 什么是进程？🤔

上一节我们介绍了文件权限，本节中我们来看看进程。这一章节主要分为四个部分。首先，我们需要了解什么是进程。

进程是指一个已经在系统中运行起来的程序。这个程序的状态必须是在运行中。进程包含几个核心含义：

1.  **内存分配**：进程在内存中肯定分配有空间。
2.  **执行单元**：代码的执行方式，可能是单进程或多线程。
3.  **状态**：进程一旦运行，就会有运行、退出、异常等状态。

这三种条件必须同时具备，我们才称之为一个进程。

### 进程的环境

进程的运行环境包括以下几个方面：

*   **本地变量与全局变量**：例如，系统语言设置是全局变量，而某个程序内部的语言设置是本地变量。
*   **CPU调度关系**：CPU每次只能执行一个任务，它依靠操作系统分配的极短时间片（如5毫秒）在不同进程间快速切换，造成“同时运行”的假象。公式表示为：`CPU时间片轮转`。
*   **系统资源**：进程运行可能需要各种资源，如磁盘、网络端口、麦克风、摄像头等。

### 进程的生命周期

进程通常有父子关系。父进程（PPID）可以创建子进程（PID）。例如，QQ主程序是父进程，它可能派生出处理消息接收、发送、广告等不同任务的子进程。

所有进程的始祖是 `systemd`（PID 为 1）。关机时，系统会向 PID 1 发送信号，由它通知所有子进程（如QQ、浏览器）关闭，最后自己退出。

子进程会继承父进程申请的资源权限。当子进程运行时，父进程通常会进入睡眠状态，等待子进程完成任务并返回通知。

### 进程的状态

现代操作系统都是多任务系统。一个CPU在一个时间点上只能处理一个进程。进程在运行中会因CPU时间片和资源分配的变化而进入不同状态，用不同标识符表示。

以下是进程的主要状态流转：

1.  **新建 (New)**：代码已加载到内存，但还未被调度。
2.  **可运行 (Runnable, R)**：进程已被放入调度队列，等待CPU时间片。
3.  **运行 (Running, R)**：CPU正在执行该进程。
4.  **退出 (Exit)**：进程正常结束，释放所有资源。
5.  **僵尸 (Zombie, Z)**：进程已结束，但部分资源或PID号未被完全释放，占用系统标识符。需要手动清理。
6.  **停止 (Stopped, T)**：进程被暂停（如按 `Ctrl+Z`），可以恢复运行。
7.  **睡眠 (Sleep)**：进程因等待某些条件（如硬件资源）而暂停执行。分为几种：
    *   **可中断睡眠 (Interruptible Sleep, S)**：等待条件满足（如插入麦克风）后可自动唤醒。
    *   **不可中断睡眠 (Uninterruptible Sleep, D)**：深度睡眠，不响应任何信号，必须等待自身操作完成。
    *   **终止睡眠 (Killable Sleep, K)**：处于睡眠但可以被终止信号干掉。

在这些状态中，最重要的是 **僵尸状态 (Z)**，发现后需要及时清理，否则可能耗尽系统资源。其他睡眠状态一般无需干预，条件满足后进程会自动恢复。

## 如何查看进程？🔍

![](img/93270747878fcd732f42f961d6d5af6a_7.png)

了解了进程的概念后，我们来看看如何查看系统中的进程信息。

我们使用 `ps` 命令来查看进程。常用命令是 `ps aux` 或 `ps -ef`，它们以不同风格显示所有进程信息。

```
ps aux
```

执行命令后，会看到类似下面的列信息：

*   **USER**：运行该进程的用户。
*   **PID**：进程的唯一ID号。
*   **%CPU, %MEM**：进程占用CPU和内存的百分比。
*   **VSZ, RSS**：虚拟内存大小和实际使用的物理内存大小。Linux中进程申请的内存（VSZ）和实际使用的内存（RSS）是分开统计的。
*   **TTY**：进程所在的终端编号。`?` 表示后台进程。
*   **STAT**：进程状态（就是上面提到的 R, S, Z, T, D 等）。
*   **START**：进程启动时间。
*   **TIME**：进程累计占用CPU的时间。
*   **COMMAND**：启动该进程的命令。

![](img/93270747878fcd732f42f961d6d5af6a_9.png)

![](img/93270747878fcd732f42f961d6d5af6a_11.png)

另一个有用的命令是 `pstree`，它可以以树状图形式显示进程的父子关系，更直观。

```
pstree -p
```

## 作业控制：前台与后台🔄

![](img/93270747878fcd732f42f961d6d5af6a_18.png)

现在我们知道如何查看进程了，接下来学习如何控制作业（Job），即进程的前后台运行与切换。

作业控制是Shell的一个功能，允许用户管理多个命令。例如，在编辑文件时突然需要处理其他事情，可以暂停编辑器，稍后再恢复，而无需关闭它。

进程分为两种运行方式：

*   **前台进程 (Foreground Process)**：运行在用户当前可见的终端，会占用输入焦点。
*   **后台进程 (Background Process)**：在后台运行，不占用当前终端输入，输出可能被重定向。

![](img/93270747878fcd732f42f961d6d5af6a_20.png)

![](img/93270747878fcd732f42f961d6d5af6a_22.png)

![](img/93270747878fcd732f42f961d6d5af6a_24.png)

以下是作业控制的常用操作：

![](img/93270747878fcd732f42f961d6d5af6a_26.png)

![](img/93270747878fcd732f42f961d6d5af6a_28.png)

1.  **暂停进程**：`Ctrl + Z`。将当前前台进程暂停，并放入作业列表。
2.  **查看作业**：`jobs`。列出当前会话中的所有作业及其编号。
3.  **恢复到前台**：`fg [%作业编号]`。将指定作业或最近暂停的作业恢复到前台运行。
4.  **恢复到后台**：`bg [%作业编号]`。将暂停的作业在后台继续运行。

![](img/93270747878fcd732f42f961d6d5af6a_30.png)

**示例**：
```
vim file1.txt  # 开始编辑
# 按下 Ctrl+Z 暂停
jobs           # 查看，假设输出 [1] Stopped vim file1.txt
fg %1          # 将1号作业恢复到前台继续编辑
```

你也可以用 `ps -j` 命令查看作业的详细信息，包括PID。

![](img/93270747878fcd732f42f961d6d5af6a_32.png)

## 进程信号与控制⚡

我们已经可以查看和暂停进程了，本节学习如何向进程发送信号，特别是终止进程。

![](img/93270747878fcd732f42f961d6d5af6a_38.png)

信号是进程间通信的一种方式，用于通知进程某个事件的发生。最常用的信号是终止信号。

以下是常用的进程控制信号及触发方式：

| 信号编号 | 信号名 | 触发方式 | 说明 |
| :--- | :--- | :--- | :--- |
| 2 | SIGINT | `Ctrl + C` | **中断**。温和地终止前台进程，进程可以捕获并处理。 |
| 3 | SIGQUIT | `Ctrl + \` | **退出**。终止进程并生成核心转储文件。 |
| 9 | SIGKILL | `kill -9 PID` | **强制杀死**。立即终止进程，**无法被捕获或忽略**，最强制。 |
| 15 | SIGTERM | `kill PID` (默认) | **终止**。请求进程正常退出，允许进程清理后结束。 |
| 18 | SIGCONT | `bg`, `fg` | **继续**。让停止的进程继续运行。 |
| 19 | SIGSTOP | `Ctrl + Z` | **停止**。暂停进程执行。 |

**核心区别**：
*   `SIGTERM (15)`：是“请礼貌地离开”，进程可以执行清理代码。
*   `SIGKILL (9)`：是“立即消失”，进程没有机会处理，可能导致数据丢失或残留临时文件。

**示例**：强制杀死进程
```
kill -9 1234
```

### 更安全的进程终止方法

![](img/93270747878fcd732f42f961d6d5af6a_40.png)

直接使用PID可能输错。以下是更安全的方法：

1.  **使用 `pidof` 获取PID**：`pidof 进程名` 可以列出该进程名的所有PID。
    ```
    pidof vim
    ```

![](img/93270747878fcd732f42f961d6d5af6a_42.png)

2.  **使用命令替换**：将 `pidof` 的结果传递给 `kill`。
    ```
    kill -15 $(pidof vim)
    ```

![](img/93270747878fcd732f42f961d6d5af6a_44.png)

![](img/93270747878fcd732f42f961d6d5af6a_46.png)

3.  **使用 `pkill`**：直接通过进程名杀死。
    ```
    pkill -9 vim
    ```

![](img/93270747878fcd732f42f961d6d5af6a_47.png)

![](img/93270747878fcd732f42f961d6d5af6a_49.png)

4.  **踢出登录用户**：`w` 命令查看当前登录用户和终端。使用 `pkill -t` 可以终止特定终端上的所有进程。
    ```
    w # 查看用户和终端（如 pts/0）
    pkill -9 -t pts/0 # 强制踢出该终端用户
    ```

5.  **终止用户所有进程**：`pkill -u` 可以终止指定用户运行的所有进程。
    ```
    pkill -9 -u username
    ```

## 系统负载监控📈

![](img/93270747878fcd732f42f961d6d5af6a_55.png)

进程过多或某些进程异常可能导致系统变慢，这就需要监控系统负载。

系统负载（Load Average）是指系统中处于可运行状态和不可中断睡眠状态的平均进程数。可以理解为“CPU的待办事项队列长度”。

有多个命令可以查看系统负载：

![](img/93270747878fcd732f42f961d6d5af6a_57.png)

1.  **`w` 命令**：第一行显示负载和登录用户。
2.  **`uptime` 命令**：精简地显示负载和运行时间。
3.  **`top` 命令**：动态显示进程和系统状态，顶部也显示负载。

![](img/93270747878fcd732f42f961d6d5af6a_59.png)

![](img/93270747878fcd732f42f961d6d5af6a_61.png)

```
w
# 或
uptime
# 或
top (按 q 退出)
```

![](img/93270747878fcd732f42f961d6d5af6a_63.png)

输出中的 `load average: 0.01, 0.05, 0.15` 三个数字分别代表：
*   **过去1分钟**的平均负载
*   **过去5分钟**的平均负载
*   **过去15分钟**的平均负载

### 如何判断负载高低？

负载高低是相对于CPU核心数而言的。**单核CPU**的负载为1.0表示满负荷。对于多核CPU，需要将负载值除以核心数来判断。

**查看CPU核心数**：
```
cat /proc/cpuinfo | grep "processor" | wc -l
```

**判断标准**：
*   对于 **4核CPU**，负载持续在 **4.0** 左右，表示CPU已满负荷。
*   如果 **1分钟负载**远高于 **15分钟负载**，说明系统刚刚经历了一个短时高压。
*   如果负载持续高于 `(CPU核心数 * 0.7)`，就需要关注并排查原因（可能是CPU、内存、磁盘I/O或网络瓶颈）。

## 系统服务管理：systemd 🛠️

![](img/93270747878fcd732f42f961d6d5af6a_87.png)

最后，我们学习如何管理系统服务。在Linux中，许多功能（如Web服务器、数据库）是以服务（Service）形式运行的，它们通常由多个进程组成，需要统一管理。

![](img/93270747878fcd732f42f961d6d5af6a_89.png)

这个管理器就是 `systemd`。`systemctl` 是管理 `systemd` 单元（如服务）的主要命令。

### 服务单元类型

![](img/93270747878fcd732f42f961d6d5af6a_91.png)

![](img/93270747878fcd732f42f961d6d5af6a_93.png)

![](img/93270747878fcd732f42f961d6d5af6a_95.png)

`systemd` 管理的主要单元类型有：
*   **`.service`**：**服务单元**。最常见的类型，用于管理后台守护进程，如 `httpd` (Apache)。
*   **`.socket`**：**套接字单元**。监听网络或本地套接字，有连接时才启动对应服务，节省资源。
*   **`.path`**：**路径单元**。监控文件系统路径变化，触发服务启动。

### 管理服务状态

![](img/93270747878fcd732f42f961d6d5af6a_97.png)

以下是管理服务状态的常用命令，以 `httpd` 服务为例：

*   **查看状态**：`systemctl status httpd`
*   **启动服务**：`systemctl start httpd`
*   **停止服务**：`systemctl stop httpd`
*   **重启服务**：`systemctl restart httpd` （先停止再启动，会中断服务）
*   **重载配置**：`systemctl reload httpd` （重新加载配置，不中断服务）
*   **启用开机自启**：`systemctl enable httpd`
*   **禁用开机自启**：`systemctl disable httpd`
*   **查看是否开机自启**：`systemctl is-enabled httpd`
*   **查看是否正在运行**：`systemctl is-active httpd`

![](img/93270747878fcd732f42f961d6d5af6a_99.png)

### 服务依赖关系

服务之间可能存在依赖。例如，网络服务可能依赖于基础系统目标。

![](img/93270747878fcd732f42f961d6d5af6a_101.png)

![](img/93270747878fcd732f42f961d6d5af6a_103.png)

*   **查看依赖**：查看指定服务依赖哪些其他单元。
    ```
    systemctl list-dependencies httpd --reverse
    ```
*   **屏蔽服务**：防止服务被手动或依赖自动启动。
    ```
    systemctl mask httpd
    ```
*   **取消屏蔽**：
    ```
    systemctl unmask httpd
    ```

![](img/93270747878fcd732f42f961d6d5af6a_105.png)

![](img/93270747878fcd732f42f961d6d5af6a_107.png)

### 实验：管理Web服务

1.  安装Apache：`yum install httpd -y`
2.  启动服务：`systemctl start httpd`
3.  关闭防火墙（临时）：`systemctl stop firewalld`
4.  浏览器访问服务器IP，应能看到测试页。
5.  设置开机自启：`systemctl enable httpd`
6.  重启服务器后，Apache服务会自动启动。
