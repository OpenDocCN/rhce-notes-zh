## 软链接 🔗

上一节我们介绍了链接的基本概念，本节中我们来看看如何创建和使用软链接。

![](img/9f459cb42415e6fab3925e97c1f4c63a_13.png)

软链接，也称为符号链接，类似于Windows系统中的快捷方式。它是一个独立的文件，其内容是指向另一个文件或目录的路径。

创建软链接的命令是 `ln -s`。

![](img/9f459cb42415e6fab3925e97c1f4c63a_15.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_17.png)

以下是创建软链接的步骤：
1.  使用 `ln -s` 命令。
2.  命令格式为：`ln -s <源文件路径> <链接文件路径>`。

![](img/9f459cb42415e6fab3925e97c1f4c63a_19.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_21.png)

例如，将 `/home/student/c` 文件链接到当前目录下的 `test`：
```bash
ln -s /home/student/c test
```
使用 `ls -l` 查看时，软链接文件会显示为 `lrwxrwxrwx`，并有一个箭头指向源文件。

当你通过软链接访问文件时，实际上访问的是源文件。如果删除源文件，软链接将失效（通常会显示闪烁或报错），因为它指向的路径不存在了。

## 硬链接 🔗

理解了软链接后，本节我们来探讨硬链接。硬链接与软链接有本质区别。

硬链接是直接指向文件系统中同一份数据的另一个名称。多个硬链接指向的是磁盘上的同一块数据区域。

![](img/9f459cb42415e6fab3925e97c1f4c63a_23.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_25.png)

创建硬链接的命令是 `ln`（不加 `-s` 参数）。

![](img/9f459cb42415e6fab3925e97c1f4c63a_27.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_29.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_30.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_32.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_34.png)

以下是创建硬链接的步骤：
1.  使用 `ln` 命令。
2.  命令格式为：`ln <源文件> <链接文件>`。

例如，为文件 `b` 创建一个名为 `2b` 的硬链接：
```bash
ln b 2b
```
使用 `ls -l` 查看，硬链接文件看起来和普通文件一样，没有特殊的箭头标识。

**硬链接的核心特性**：
*   所有硬链接（包括原始文件）地位平等，都指向同一份数据。
*   修改任何一个硬链接的内容，所有链接看到的内容都会同步改变。
*   删除其中一个硬链接，只要还有其他硬链接存在，文件数据就不会丢失。只有当所有硬链接都被删除（即链接数降为0），文件数据才会被系统回收。

![](img/9f459cb42415e6fab3925e97c1f4c63a_36.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_38.png)

**硬链接的限制**：
1.  只能用于常规文件（`-`），不能用于目录或特殊文件（如设备文件）。
2.  必须在同一个文件系统（分区）内创建，不能跨文件系统。

![](img/9f459cb42415e6fab3925e97c1f4c63a_40.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_42.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_44.png)

硬链接常用于对重要文件进行数据保护，防止误删除。

## 文件存储原理：i-node与block 💾

![](img/9f459cb42415e6fab3925e97c1f4c63a_46.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_48.png)

了解了链接的操作后，我们需要深入底层，看看文件是如何在磁盘上被存储和读取的。这涉及到i-node和block两个核心概念。

在磁盘使用前，需要经过**分区**和**格式化**。格式化会确定文件系统类型（如ext4），并创建固定数量的 **i-node** 和 **block**。

![](img/9f459cb42415e6fab3925e97c1f4c63a_50.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_52.png)

*   **block（块）**：是存储实际文件内容（如文本、图片数据）的基本单位。
*   **i-node（索引节点）**：是存储文件**元数据**（属性）的地方，**不包括文件名**。

![](img/9f459cb42415e6fab3925e97c1f4c63a_54.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_56.png)

i-node中存储的信息包括：
*   文件权限、所有者、所属组
*   文件大小、时间戳（创建、修改、访问时间）
*   文件类型（普通文件、目录等）
*   指向文件数据所在block的指针

**文件读取过程**：
当使用 `cat test` 命令读取文件时：
1.  系统首先在目录中找到文件名 `test` 对应的 **i-node 编号**。
2.  根据 i-node 编号，读取 i-node 中的元数据，检查访问权限。
3.  权限通过后，根据 i-node 中的指针，找到对应的 block。
4.  从 block 中读取实际的文件内容。

![](img/9f459cb42415e6fab3925e97c1f4c63a_58.png)

可以使用 `ls -i` 命令查看文件的 i-node 编号：
```bash
ls -i filename
```

![](img/9f459cb42415e6fab3925e97c1f4c63a_60.png)

**硬链接与i-node的关系**：
创建硬链接时，并没有创建新的数据块（block），只是创建了一个新的文件名，并指向**同一个 i-node**。因此，硬链接文件和原文件的 i-node 编号是相同的。`ls -l` 结果中第二列的数字，就表示指向该 i-node 的硬链接数量。

**软链接与i-node的关系**：
创建软链接时，会创建一个新的、独立的文件（拥有自己的 i-node 和 block）。这个软链接文件的内容是其指向的目标文件的路径字符串。

## 文件删除原理 🗑️

![](img/9f459cb42415e6fab3925e97c1f4c63a_62.png)

理解了文件的存储结构，我们就能明白文件删除的本质。

![](img/9f459cb42415e6fab3925e97c1f4c63a_64.png)

完全删除一个文件，需要满足两个条件：
1.  **i-link 计数为 0**：即指向该文件数据的所有硬链接（文件名）都被删除。
2.  **i-count 计数为 0**：即没有进程正在使用该文件。

**常见问题**：
有时删除一个大文件后，磁盘空间并未释放。这通常是因为仍有进程在打开并使用这个文件（i-count > 0）。此时文件数据在磁盘上标记为“已删除”，但空间要等到进程关闭文件后才会真正释放。

可以使用 `lsof | grep deleted` 命令查找已被删除但仍被进程占用的文件。

![](img/9f459cb42415e6fab3925e97c1f4c63a_66.png)

## 格式化与性能考量 ⚙️

![](img/9f459cb42415e6fab3925e97c1f4c63a_68.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_70.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_72.png)

![](img/9f459cb42415e6fab3925e97c1f4c63a_74.png)

在格式化磁盘时，可以指定 i-node 大小和 block 大小，这会影响文件系统的性能和应用场景。

![](img/9f459cb42415e6fab3925e97c1f4c63a_76.png)

*   **block 大小**：如果存放大量小文件（如日志文件），使用较大的 block 会浪费空间，因为一个 block 只能属于一个文件，即使它没被填满。如果存放大型文件（如视频），较大的 block 可以减少寻址次数，提高IO性能。通常 ext4 文件系统默认 block 大小为 4KB。
*   **i-node 数量**：如果文件系统用于存放海量小文件，需要确保有足够的 i-node 数量。可以使用 `df -i` 命令查看 i-node 的使用情况。如果 i-node 用尽，即使磁盘还有空间，也无法创建新文件。

![](img/9f459cb42415e6fab3925e97c1f4c63a_78.png)

格式化时指定参数的示例：
```bash
mkfs.ext4 -b 4096 -I 256 /dev/sdb1
# -b 指定block大小
# -I 指定i-node大小（字节）
```

![](img/9f459cb42415e6fab3925e97c1f4c63a_80.png)

## 总结 📝

本节课中我们一起学习了：
1.  **软链接**：创建快捷方式，命令为 `ln -s`。删除源文件后链接失效。
2.  **硬链接**：创建文件的别名，命令为 `ln`。为同一份数据提供多个访问入口，用于数据保护。有不能用于目录和不能跨文件系统的限制。
3.  **文件存储原理**：文件通过 **i-node** 存储属性，通过 **block** 存储内容。读取文件时先查 i-node，再找 block。
4.  **文件删除原理**：需要 i-link（硬链接数）和 i-count（进程占用数）均为 0，文件才会被彻底删除。
5.  **性能考量**：根据存储文件的类型（大文件或小文件），在格式化时选择合适的 block 和 i-node 大小，以优化磁盘空间利用率和IO性能。
