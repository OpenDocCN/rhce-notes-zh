## 用户与权限回顾
上一节我们介绍了用户、组以及文件权限的修改。本节中，我们来看看如何修改文件对不同身份用户的权限。

修改权限可以使用 `chmod` 命令。其基本语法是：
```bash
chmod [选项] 模式 文件
```
其中，`模式` 部分遵循 `who what which` 的格式：
*   **who**：指定修改哪类用户的权限。
    *   `u`：文件拥有者（user）
    *   `g`：文件所属组（group）
    *   `o`：其他用户（others）
    *   `a`：所有用户（all）
*   **what**：指定要进行的操作。
    *   `+`：增加权限
    *   `-`：移除权限
    *   `=`：设置权限为指定值
*   **which**：指定具体的权限。
    *   `r`：读权限
    *   `w`：写权限
    *   `x`：执行权限

![](img/ea074f4501eb5f471c9d58f59f130853_21.png)

以下是具体操作示例：
*   为文件 `passwd` 的所属组增加写权限：`chmod g+w passwd`
*   为文件 `passwd` 的所属组和其他用户移除写权限：`chmod go-w passwd`
*   设置文件 `passwd` 的其他用户权限为只读和执行：`chmod o=rx passwd`
*   为文件 `passwd` 的拥有者增加执行权限，同时为所属组增加写权限：`chmod u+x,g+w passwd`
*   移除文件 `passwd` 所有用户的执行权限：`chmod a-x passwd`

除了字符表示法，权限也可以用数字表示：
*   `r`（读）对应数字 **4**
*   `w`（写）对应数字 **2**
*   `x`（执行）对应数字 **1**

每种身份的权限是其对应数字之和。例如，`rwx` 权限是 `4+2+1=7`，`r-x` 权限是 `4+0+1=5`。

![](img/ea074f4501eb5f471c9d58f59f130853_46.png)

![](img/ea074f4501eb5f471c9d58f59f130853_48.png)

使用数字方式修改权限时，需要指定三个数字，分别代表拥有者、所属组和其他用户的权限。例如：
*   `chmod 777 file`：所有用户拥有全部权限。
*   `chmod 644 file`：拥有者可读写，所属组和其他用户只可读。
*   `chmod 755 file`：拥有者可读、写、执行，所属组和其他用户可读、执行。

![](img/ea074f4501eb5f471c9d58f59f130853_51.png)

**注意**：如果使用数字方式时少写了一位，系统会在**前面**自动补零。例如，`chmod 66 file` 实际效果是 `066`，即拥有者和所属组无权限，其他用户拥有读写权限。

## IO重定向与管道

### 标准输入、输出与错误
在Linux中，每个命令执行时都会打开三个标准数据流：
*   **标准输入（stdin）**：文件描述符为 **0**，默认来自键盘。
*   **标准输出（stdout）**：文件描述符为 **1**，默认输出到终端屏幕。
*   **标准错误（stderr）**：文件描述符为 **2**，默认也输出到终端屏幕。

### 输出重定向
输出重定向可以将命令的输出从默认的屏幕转移到其他地方，通常是文件。

以下是输出重定向的操作符：
*   `>`：将**标准输出**重定向到文件。如果文件存在，会**覆盖**原有内容。
    *   示例：`ls > ls.out` 将 `ls` 命令的结果保存到 `ls.out` 文件。
*   `>>`：将**标准输出**追加到文件末尾。
    *   示例：`ls >> ls.out` 将 `ls` 命令的结果追加到 `ls.out` 文件末尾。
*   `2>`：将**标准错误**重定向到文件。
    *   示例：`find /etc -name passwd 2> find.err` 将查找过程中的错误信息保存到 `find.err`。
*   `2>>`：将**标准错误**追加到文件末尾。
*   `&>`：将**标准输出**和**标准错误**都重定向到同一个文件。
    *   示例：`find /etc -name passwd &> find.all` 将所有输出（正确和错误）都保存到 `find.all`。
*   `&>>`：将**标准输出**和**标准错误**都追加到同一个文件末尾。

### 特殊设备文件 `/dev/null`
`/dev/null` 是一个特殊的字符设备文件，被称为“黑洞”。任何写入它的数据都会被丢弃，且不占用存储空间。它常用于丢弃不需要的命令输出。
*   示例：`ping -c 2 www.baidu.com &> /dev/null` 执行 ping 命令，但丢弃所有输出，只关心命令是否成功（通过退出码 `$?` 判断）。

### 管道符 `|`
管道符 `|` 用于连接多个命令，将前一个命令的**标准输出**作为后一个命令的**标准输入**。

![](img/ea074f4501eb5f471c9d58f59f130853_105.png)

![](img/ea074f4501eb5f471c9d58f59f130853_106.png)

基本语法：
```bash
command1 | command2 | command3 ...
```

![](img/ea074f4501eb5f471c9d58f59f130853_108.png)

![](img/ea074f4501eb5f471c9d58f59f130853_110.png)

![](img/ea074f4501eb5f471c9d58f59f130853_112.png)

![](img/ea074f4501eb5f471c9d58f59f130853_114.png)

示例：
*   `find /etc -name passwd | wc -l`：查找 `/etc` 下名为 `passwd` 的文件，并统计找到的行数。
*   `df -h | grep ‘/$’ | tr -s ‘ ‘ | cut -d‘ ‘ -f5`：这是一个组合命令，用于提取根分区 `/` 的使用百分比。

![](img/ea074f4501eb5f471c9d58f59f130853_116.png)

![](img/ea074f4501eb5f471c9d58f59f130853_118.png)

**关键点**：
1.  默认情况下，**标准错误（stderr）无法通过管道传递**。
2.  可以使用 `2>&1` 将标准错误合并到标准输出，从而使其能够通过管道。
    *   示例：`find /etc -name passwd 2>&1 | wc -l` 现在可以统计包括错误信息在内的所有输出行数。
3.  输出重定向 (`>`) 和管道是互斥的。如果一个命令的输出被重定向到文件，它就无法再通过管道传递给下一个命令。
4.  可以使用 `tee` 命令实现“三通”，既能将输出保存到文件，又能继续通过管道传递。
    *   示例：`df -h | tee df.out | grep ‘/$’ | tee grep.out` 将 `df -h` 的结果同时保存到 `df.out` 并传递给 `grep`。

### 输入重定向
输入重定向可以将命令的输入源从默认的键盘改为文件。

![](img/ea074f4501eb5f471c9d58f59f130853_127.png)

![](img/ea074f4501eb5f471c9d58f59f130853_129.png)

操作符：
*   `<`：从文件读取内容作为命令的标准输入。
    *   示例：`tr ‘a-z’ ‘A-Z’ < .bashrc` 将 `.bashrc` 文件内容从小写转换为大写。
*   `<<`：称为“Here Document”，用于向命令输入多行文本，直到遇到指定的结束标记。
    *   示例：
        ```bash
        cat > newfile.txt << EOF
        这是第一行。
        这是第二行。
        EOF
        ```
        这段代码会将 `EOF` 标记之间的两行文本写入 `newfile.txt` 文件。这在脚本中批量编辑文件时非常有用。

## Vim编辑器高级使用

### Vim的三种工作模式
Vim 高效的核心在于其三种工作模式，理解并熟练切换是掌握Vim的关键。
1.  **命令模式（Normal Mode）**：打开文件后默认进入的模式。在此模式下可以移动光标、复制、粘贴、删除文本，但不能直接输入字符。
2.  **插入模式（Insert Mode）**：在此模式下可以像普通编辑器一样输入和编辑文本。从命令模式按 `i`, `a`, `o` 等键进入。
3.  **退出模式（Ex Mode / 命令行模式）**：在命令模式下按 `:` 进入。在此模式下可以保存文件、退出编辑器、进行搜索替换等操作。

模式切换：
*   命令模式 -> 插入模式：按 `i` (在光标前插入), `a` (在光标后插入), `o` (在下一行插入), `O` (在上一行插入)。
*   插入模式 -> 命令模式：按 `Esc` 键。
*   命令模式 -> 退出模式：按 `:` 键。
*   退出模式 -> 命令模式：按 `Esc` 键。
