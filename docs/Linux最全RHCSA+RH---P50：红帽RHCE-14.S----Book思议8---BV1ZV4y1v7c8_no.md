## 使用 BEGIN 和 END 模式

![](img/91173e3956168390a21b18943a9ca18a_7.png)

上一节我们介绍了 `awk` 的基本过滤和打印功能。本节中，我们来看看如何利用 `BEGIN` 和 `END` 模式在数据处理前后执行特定操作。

![](img/91173e3956168390a21b18943a9ca18a_9.png)

`BEGIN` 模式下的命令会在读取任何输入行之前执行一次，常用于打印表头或初始化变量。`END` 模式下的命令则会在所有输入行处理完毕后执行一次，常用于打印汇总信息。

以下是一个示例，我们在处理 `/etc/passwd` 文件前，先打印自定义的表头。

![](img/91173e3956168390a21b18943a9ca18a_11.png)

```bash
awk 'BEGIN{print "用户名\t\t解释器"} /^mai/{print $1, $7}' /etc/passwd
```

![](img/91173e3956168390a21b18943a9ca18a_13.png)

**注意**：在 `BEGIN` 或 `END` 块中，如果要打印的内容不是 `awk` 的内置变量或语法（例如制表符 `\t` 或中文字符），需要将其用引号括起来，否则 `awk` 可能无法识别导致语法错误。

一个更完整的例子，结合了 `BEGIN`、主处理逻辑和 `END`：

```bash
awk 'BEGIN{print "用户名\t\t解释器"} /^mai/{print $1 "\t" $7} END{print "总行数：" NR}' /etc/passwd
```
*   `BEGIN{print “用户名\t\t解释器”}`：在处理文件前，先输出一行表头。
*   `/^mai/{print $1 “\t” $7}`：对每一行，如果以 “mai” 开头，则打印第1列和第7列，中间用制表符分隔。
*   `END{print “总行数：” NR}`：在所有行处理完后，打印总行数。

`awk` 对命令中的空格要求不严格，但为了代码清晰，建议在适当位置添加空格。

## 结合管道进行数据过滤

`awk` 的强大之处在于它能轻松处理 `grep` 难以应对的列数据提取场景。本节我们来看看如何将其他命令的输出通过管道传递给 `awk` 进行精准过滤。

### 示例1：过滤系统剩余内存

![](img/91173e3956168390a21b18943a9ca18a_21.png)

![](img/91173e3956168390a21b18943a9ca18a_23.png)

如果我们想查看系统的剩余内存，使用 `free -h` 命令后，用 `grep` 很难单独提取出“剩余内存”这个数值。

![](img/91173e3956168390a21b18943a9ca18a_25.png)

![](img/91173e3956168390a21b18943a9ca18a_27.png)

以下是使用 `awk` 的解决方案：

![](img/91173e3956168390a21b18943a9ca18a_29.png)

![](img/91173e3956168390a21b18943a9ca18a_31.png)

![](img/91173e3956168390a21b18943a9ca18a_33.png)

![](img/91173e3956168390a21b18943a9ca18a_35.png)

```bash
free -h | awk '/Mem:/ {print “剩余内存：” $4}'
```
*   `free -h`：输出内存使用情况。
*   `|`：管道符，将前一个命令的输出作为后一个命令的输入。
*   `awk ‘/Mem:/ {print “剩余内存：” $4}’`：匹配包含 “Mem:” 的行，然后打印该行的第4列，即剩余内存。

### 示例2：过滤网卡流量信息

![](img/91173e3956168390a21b18943a9ca18a_45.png)

![](img/91173e3956168390a21b18943a9ca18a_47.png)

查看特定网卡（如 `ens33`）的入口流量信息，`grep` 会输出整行，而 `awk` 可以精确提取数值。

![](img/91173e3956168390a21b18943a9ca18a_49.png)

![](img/91173e3956168390a21b18943a9ca18a_51.png)

![](img/91173e3956168390a21b18943a9ca18a_53.png)

```bash
ifconfig ens33 | awk '/RX packets/ {print “入口流量：” $5}'
```
这条命令会匹配包含 “RX packets” 的行，并打印其第5列，即接收的数据包数量（入口流量）。如果想同时查看接收和发送的字节数，可以打印多列：
```bash
ifconfig ens33 | awk '/RX packets/ {print “接收包:” $5 “， 接收字节:” $6}'
```

**进阶思考**：如果输出结果中包含不需要的括号 `()`，如何去除？可以尝试在 `awk` 中指定更复杂的分隔符，例如使用 `-F` 参数结合字符集合 `[]`。这需要根据实际输出格式进行调试。

## 编写监控脚本

![](img/91173e3956168390a21b18943a9ca18a_57.png)

理解了基础用法后，我们可以将 `awk` 命令嵌入到 Shell 脚本中，实现自动化监控。本节我们将学习如何编写一个简单的脚本，用于周期性地监控系统状态。

![](img/91173e3956168390a21b18943a9ca18a_59.png)

![](img/91173e3956168390a21b18943a9ca18a_61.png)

以下是一个示例脚本，它循环监控网卡的 IP 地址和流量：

![](img/91173e3956168390a21b18943a9ca18a_63.png)

![](img/91173e3956168390a21b18943a9ca18a_65.png)

![](img/91173e3956168390a21b18943a9ca18a_67.png)

```bash
#!/bin/bash
while true
do
    # 获取并打印IP地址
    IP=`ifconfig ens33 | awk '/inet / {print $2}'`
    echo “IP地址：$IP”
    
    # 获取并打印入口流量（接收字节数）
    RX=`ifconfig ens33 | awk '/RX packets/ {print $6}'`
    echo “入口流量：$RX”
    
    # 获取并打印出口流量（发送字节数）
    TX=`ifconfig ens33 | awk '/TX packets/ {print $6}'`
    echo “出口流量：$TX”
    
    # 清屏并等待2秒
    clear
    sleep 2
done
```
**脚本解析**：
1.  `#!/bin/bash`：指定脚本解释器为 Bash。
2.  `while true`：开启一个无限循环。
3.  **` 反引号**：用于执行命令并将结果赋值给变量。
4.  `clear`：清空终端屏幕，使每次输出更清晰。
5.  `sleep 2`：让循环每次执行后暂停2秒，避免消耗过多CPU资源。

![](img/91173e3956168390a21b18943a9ca18a_69.png)

对于初学者，现阶段的核心目标是能**看懂**并理解此类脚本的逻辑，知道每部分命令的作用。在需要时，能够基于现有脚本进行修改，而不必从头编写。

### 示例3：监控磁盘空间并告警

我们还可以编写脚本监控根分区的剩余空间，并在空间不足时发出警告。

![](img/91173e3956168390a21b18943a9ca18a_71.png)

```bash
#!/bin/bash
while true
do
    # 获取根分区剩余空间（例如”5G“），但注意这里获取的是带单位的字符串
    space_left=`df -h | awk ‘/\/$/ {print $4}’`
    echo “根分区剩余空间：$space_left”
    
    # 注意：字符串”5G“无法直接与数字5进行数值比较
    # 一种方法是进行字符串比较
    if [ “$space_left” = “5G” ]; then
        echo “警告：根分区空间仅剩5G！”
    fi
    
    sleep 5
done
```
**重要提示**：`df -h` 输出的空间信息带有单位（如 G、M），这是一个字符串。如果需要进行数值比较（例如判断是否小于5G），需要先处理掉单位。可以使用 `awk` 的 `-F` 参数指定分隔符来只提取数字部分：
```bash
space_left_num=`df -h | awk -F”[ G]” ‘/\/$/ {print $4}’`
# 此时 $space_left_num 就是一个纯数字
if [ $space_left_num -lt 5 ]; then
    echo “警告：根分区空间不足5G！”
fi
```

## 设置开机自启动脚本

如果我们希望某个监控脚本在服务器启动时自动运行，可以将它添加到系统的启动脚本中。

![](img/91173e3956168390a21b18943a9ca18a_75.png)

![](img/91173e3956168390a21b18943a9ca18a_77.png)

在 RHEL/CentOS 7 及以上系统中，可以编辑 `/etc/rc.d/rc.local` 文件（请注意，首先需要给予该文件执行权限）。

1.  给 `rc.local` 文件添加执行权限：
    ```bash
    chmod +x /etc/rc.d/rc.local
    ```
2.  编辑 `/etc/rc.d/rc.local` 文件，在文件末尾添加要执行的命令，**建议使用命令的绝对路径**：
    ```bash
    /bin/bash /path/to/your/monitor_script.sh &
    ```
    *   `/bin/bash`：Bash解释器的绝对路径。
    *   `/path/to/your/monitor_script.sh`：你的监控脚本的绝对路径。
    *   `&`：让脚本在后台运行，避免阻塞启动过程。
3.  同样，确保你的 `monitor_script.sh` 脚本本身也有执行权限 (`chmod +x`)。

这样，当系统启动时，就会自动执行你指定的脚本。

## 总结

本节课中我们一起学习了 `awk` 命令的进阶应用：
1.  **`BEGIN` 与 `END` 模式**：用于在数据处理前后执行一次性操作，如打印表头或汇总信息。
2.  **结合管道过滤**：将 `grep`、`free`、`ifconfig` 等命令的输出通过管道交给 `awk`，实现精准的列数据提取，这是 `awk` 在实际工作中最常用的场景之一。
3.  **编写监控脚本**：将 `awk` 命令嵌入到 Shell 脚本的循环和条件判断中，实现系统资源（内存、流量、磁盘空间）的自动监控。我们强调，初学者首要目标是理解脚本逻辑，具备改编能力。
4.  **开机自启动**：了解了如何通过 `/etc/rc.d/rc.local` 文件让脚本在系统启动时自动运行。

![](img/91173e3956168390a21b18943a9ca18a_85.png)

`awk` 功能非常强大，本身甚至是一门编程语言，支持数组、循环和复杂判断。但对于日常运维和 RHCE 认证，掌握其文本过滤和字段提取的核心功能即可应对绝大多数场景。
