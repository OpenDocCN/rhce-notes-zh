## 概述
在本节课中，我们将要学习Linux系统中关于进程管理的几个核心操作。我们将了解如何查看进程的实时状态，如何将进程在前后台之间进行调度，如何安全或强制地结束进程，以及如何分析用户的登录情况。这些技能对于日常系统维护和故障排查至关重要。

## 进程状态实时监控：`top`命令详解
上一节我们介绍了使用`ps`命令查看进程快照。本节中我们来看看如何动态地监控进程状态，这就需要用到`top`命令。

`top`命令提供了一个实时更新的系统状态视图，包括进程列表和资源使用情况。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_7.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_9.png)

### `top`命令输出解读
运行`top`命令后，屏幕会分为几个部分。我们来逐一解读：

![](img/9c337f75f06c4ba4d00f190bf7c192a1_11.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_13.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_15.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_17.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_19.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_21.png)

**第一行：系统概况**
这一行显示了系统的整体状态。
*   **当前系统时间**：例如 `16:46:13`。
*   **系统运行时间**：例如 `up 1 day, 2:30`，表示系统已运行1天2小时30分钟。
*   **当前登录用户数**：例如 `2 users`。
*   **系统平均负载**：例如 `load average: 0.00, 0.01, 0.05`。这三个数字分别代表过去1分钟、5分钟、15分钟的系统平均负载。对于单核CPU，1.00表示满负荷。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_23.png)

**第二行：进程概况**
这一行显示了进程的统计信息。
*   **总进程数**：`total`
*   **运行中的进程数**：`running`
*   **休眠中的进程数**：`sleeping`
*   **停止的进程数**：`stopped`
*   **僵尸进程数**：`zombie`。僵尸进程是已结束但未被父进程回收的进程。如果这个数字不为0且持续增长，可能需要关注。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_25.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_27.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_29.png)

**第三行：CPU使用情况**
这一行显示了CPU时间在各种状态下的分配百分比。
*   **`us`**：用户空间进程占用CPU百分比。
*   **`sy`**：内核空间进程占用CPU百分比。
*   **`ni`**：改变过优先级的用户进程占用CPU百分比。
*   **`id`**：**空闲CPU百分比**。这是需要重点关注的指标，如果这个值持续很低（例如低于10%），说明CPU非常繁忙，需要排查原因。
*   **`wa`**：等待I/O的CPU时间百分比。
*   **`hi`**：处理硬件中断的CPU时间百分比。
*   **`si`**：处理软件中断的CPU时间百分比。
*   **`st`**：被虚拟化环境“偷走”的CPU时间百分比。

**第四、五行：内存使用情况**
*   **第四行 (`Mem`)**：物理内存使用情况，包括总量、已用、空闲、缓冲/缓存。
*   **第五行 (`Swap`)**：交换分区使用情况。
> **提示**：查看内存更直观的方式是使用 `free -h` 命令。

**进程列表区域**
这部分列出了各个进程的详细信息，默认按CPU使用率排序。各列含义如下：
*   **`PID`**：进程ID。
*   **`USER`**：进程所有者。
*   **`PR` / `NI`**：进程优先级。
*   **`VIRT` / `RES` / `SHR`**：虚拟内存、常驻内存、共享内存使用量。
*   **`%CPU` / `%MEM`**：进程占用CPU和物理内存的百分比。
*   **`TIME+`**：进程使用的总CPU时间。
*   **`COMMAND`**：启动进程的命令行。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_31.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_33.png)

### `top`命令的交互操作
在`top`运行界面，可以输入单字母命令进行交互：
*   **`P`**：按CPU使用率排序（默认）。
*   **`M`**：按内存使用率排序。
*   **`k`**：终止一个进程。输入后需要指定进程PID。
*   **`q`**：退出`top`。
*   **`h`**：显示帮助。

例如，当发现CPU使用率过高时，可以按 **`P`** 键，将消耗CPU最多的进程排在最前面，便于定位问题进程。

## 进程检索：`pgrep`命令
有时我们需要快速查找特定进程的PID。`ps`命令结合`grep`可以做到，而`pgrep`命令更为简洁。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_39.png)

以下是`pgrep`命令的常用选项示例：
*   查找名为`sshd`的进程PID：
    ```bash
    pgrep sshd
    ```
*   显示进程名和PID：
    ```bash
    pgrep -l sshd
    ```
*   查找指定用户（如`test`）的进程：
    ```bash
    pgrep -l -u test
    ```
*   精确匹配进程名（必须完全一致）：
    ```bash
    pgrep -x sshd
    ```

![](img/9c337f75f06c4ba4d00f190bf7c192a1_41.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_43.png)

> **注意**：`pgrep`默认进行模糊匹配。`-x`选项用于精确匹配。

## 进程的前后台调度
在Linux中，进程可以在前台运行（占用当前终端），也可以在后台运行（不占用终端）。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_50.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_52.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_53.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_55.png)

### 将进程放入后台
在命令的末尾加上 `&` 符号，可以让该命令在后台启动。
```bash
sleep 300 &
```
这条命令会让终端“睡眠”300秒，但因为有`&`，它会在后台运行，你依然可以在当前终端输入其他命令。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_57.png)

### 查看后台进程
使用 `jobs` 命令可以查看当前终端放入后台的进程列表。
```bash
jobs -l
```
输出会显示后台任务的编号（如`[1]`）、状态、PID和命令。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_59.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_61.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_63.png)

### 前后台调度
*   **`fg`**：将后台进程调至前台运行。
    ```bash
    fg %1  # 将编号为1的后台任务调到前台
    ```
*   **`Ctrl + Z`**：挂起当前在前台运行的进程（使其进入“已停止”状态）。
*   **`bg`**：将挂起的进程在后台继续运行。
    ```bash
    bg %1  # 让编号为1的已停止任务在后台继续运行
    ```

**应用场景**：当你运行一个耗时很长的命令（如大数据拷贝、编译）时，可以将其放入后台，从而释放终端进行其他操作。

## 杀死进程
当需要终止一个进程时，可以使用以下命令。

### `kill` 命令：通过PID终止进程
`kill` 通过向进程发送信号来工作。最常用的信号有：
*   **`15` (`SIGTERM`)**：**正常终止**。这是`kill`命令的默认信号。进程会收到通知，并有机会清理资源后退出。
*   **`9` (`SIGKILL`)**：**强制终止**。进程会立即被系统杀死，没有机会进行清理。用于无法正常终止的进程（如僵尸进程或程序无响应）。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_65.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_67.png)

命令格式：
```bash
kill [信号] <PID>
kill -9 <PID>  # 强制杀死PID为1234的进程
```

![](img/9c337f75f06c4ba4d00f190bf7c192a1_69.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_71.png)

### `pkill` 命令：通过进程名终止进程
`pkill` 可以根据进程名来发送信号，避免手动查找PID。
```bash
pkill sleep          # 终止所有名为sleep的进程
pkill -9 firefox     # 强制终止所有firefox进程
pkill -u test        # 终止test用户的所有进程
```
`pkill` 使用方便，但需谨慎，因为它可能终止多个同名进程。

![](img/9c337f75f06c4ba4d00f190bf7c192a1_73.png)

### `killall` 命令：通过进程名终止进程（更彻底）
`killall` 与 `pkill` 类似，但要求进程名必须精确匹配。
```bash
killall sleep
```

![](img/9c337f75f06c4ba4d00f190bf7c192a1_75.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_77.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_79.png)

![](img/9c337f75f06c4ba4d00f190bf7c192a1_81.png)

### 踢出登录用户
结合进程管理，我们可以终止用户的登录会话进程（通常是`bash`），从而将其踢出系统。
```bash
pkill -9 -u test  # 强制终止test用户的所有进程，包括其登录会话
```
或者，更精确地通过终端号踢出：
```bash
pkill -9 -t pts/1  # 强制终止在pts/1终端上的进程
```

## 用户登录分析
了解谁登录了系统以及登录历史，是系统安全管理的一部分。

### 查看当前已登录用户
*   **`who`**：显示当前已登录用户的用户名、终端、登录时间和来源IP。
    ```bash
    who
    ```
*   **`w`**：比`who`更详细，显示用户正在执行的命令以及系统负载信息。
    ```bash
    w
    ```
*   **`users`**：仅显示当前已登录的用户名列表，非常简洁。

### 查看登录历史记录
*   **`last`**：查看**成功登录**的历史记录。显示用户名、终端、登录IP、登录时长和登出时间。
    ```bash
    last
    ```
*   **`lastb`**：查看**失败登录**的尝试记录。对于排查暴力破解等安全问题非常有用。
    ```bash
    lastb
    ```
> **注意**：查看失败记录通常需要`root`权限。

## 总结
本节课中我们一起学习了Linux进程管理的核心操作：
1.  使用 **`top`** 命令动态监控系统进程和资源状态，并学会通过`P`、`M`键排序来定位资源消耗大的进程。
2.  使用 **`pgrep`** 命令快速检索进程。
3.  掌握了进程的**前后台调度**：使用 `&` 将进程放入后台，使用 `jobs`、`fg`、`bg`、`Ctrl+Z` 进行管理。
4.  学习了多种**终止进程**的方法：
    *   `kill [信号] PID`：通过PID终止。
    *   `pkill/pkillall [选项] 进程名`：通过进程名终止。
    *   理解了信号 `15`（正常终止）和 `9`（强制终止）的区别。
    *   学会了如何踢出指定用户（`pkill -u`）。
5.  学会了使用 **`who`**、**`w`**、**`last`**、**`lastb`** 等命令分析用户登录情况，这对于系统安全审计至关重要。
