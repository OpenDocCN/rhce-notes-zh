## FTP基础概念

上一节我们概述了课程内容，本节中我们来看看FTP的基础概念。

FTP的全称是File Transfer Protocol，即文件传输协议。它是一种在互联网中，基于TCP协议进行端到端数据传输的协议。其功能类似于个人使用的百度网盘，但FTP服务器搭建在企业内部，数据存储在自有服务器上，更适合传输需要保密的内部文件。

FTP采用C/S架构：
*   **C** 代表客户端，用于连接服务器并下载或上传数据。
*   **S** 代表服务端，用于存储和共享数据。

FTP默认使用两个端口工作：
*   **21端口**：命令端口，用于接收客户端的指令（如登录、请求文件列表）。
*   **20端口**：数据端口，在主动模式下用于向客户端传输文件数据。

## FTP的工作模式

![](img/71690302f0288fccb8336e6dd04f3de6_7.png)

![](img/71690302f0288fccb8336e6dd04f3de6_9.png)

了解了FTP的基本架构后，本节中我们来看看它的两种工作模式，这关系到防火墙的配置。

![](img/71690302f0288fccb8336e6dd04f3de6_13.png)

![](img/71690302f0288fccb8336e6dd04f3de6_14.png)

![](img/71690302f0288fccb8336e6dd04f3de6_16.png)

![](img/71690302f0288fccb8336e6dd04f3de6_18.png)

FTP协议有两种工作模式：主动模式和被动模式。其核心区别在于**建立数据传输连接时由谁主动发起**。

![](img/71690302f0288fccb8336e6dd04f3de6_20.png)

![](img/71690302f0288fccb8336e6dd04f3de6_22.png)

**主动模式**
1.  客户端使用一个随机端口（如1024）连接服务端的21号命令端口，发送指令（如请求下载文件），并告知服务端自己的数据端口号（如1024）。
2.  服务端收到指令后，用自己的20号数据端口**主动连接**客户端指定的数据端口（1024）。
3.  连接建立后，开始传输数据。
**潜在问题**：如果客户端开启了防火墙，可能会阻止服务端发起的这次主动连接。

![](img/71690302f0288fccb8336e6dd04f3de6_24.png)

![](img/71690302f0288fccb8336e6dd04f3de6_26.png)

**被动模式**
1.  客户端连接服务端的21号命令端口，并发送`PASV`指令进入被动模式。
2.  服务端随机开启一个高端口（如5000）作为数据端口，并将端口号告知客户端。
3.  客户端使用另一个随机端口**主动连接**服务端指定的数据端口（5000）。
4.  连接建立后，开始传输数据。
**优点**：数据传输连接由客户端发起，通常能更好地兼容客户端的防火墙设置。

![](img/71690302f0288fccb8336e6dd04f3de6_28.png)

![](img/71690302f0288fccb8336e6dd04f3de6_30.png)

![](img/71690302f0288fccb8336e6dd04f3de6_32.png)

![](img/71690302f0288fccb8336e6dd04f3de6_34.png)

**注意**：现代FTP软件和服务器（如vsftpd）默认通常使用被动模式。

![](img/71690302f0288fccb8336e6dd04f3de6_36.png)

## 安装与启动vsftpd

了解了可用的工具后，我们现在开始动手搭建FTP服务器。

![](img/71690302f0288fccb8336e6dd04f3de6_62.png)

![](img/71690302f0288fccb8336e6dd04f3de6_64.png)

我们将在IP为`192.168.0.40`的服务器上安装`vsftpd`，并确保网络和基础环境正常。

![](img/71690302f0288fccb8336e6dd04f3de6_66.png)

![](img/71690302f0288fccb8336e6dd04f3de6_68.png)

**操作步骤如下：**

1.  **修改主机名（可选，便于识别）**
    ```bash
    hostnamectl set-hostname ftp-server
    ```
2.  **安装vsftpd软件包**
    ```bash
    yum install -y vsftpd
    ```
3.  **启动vsftpd服务并设置开机自启**
    ```bash
    systemctl start vsftpd
    systemctl enable vsftpd
    ```
4.  **检查服务状态**
    ```bash
    systemctl status vsftpd
    ```
5.  **配置防火墙（如果防火墙开启）**
    *   方法一：直接关闭防火墙（学习环境推荐）。
        ```bash
        systemctl stop firewalld
        systemctl disable firewalld
        ```
    *   方法二：在防火墙中永久放行FTP服务。
        ```bash
        firewall-cmd --permanent --add-service=ftp
        firewall-cmd --reload
        ```

**注意**：如果使用`iptables`，默认规则通常允许FTP连接，无需特殊配置。

## vsftpd的访问模式与匿名访问

服务启动后，本节中我们来看看客户端如何连接，并理解vsftpd的用户认证机制。

![](img/71690302f0288fccb8336e6dd04f3de6_84.png)

![](img/71690302f0288fccb8336e6dd04f3de6_86.png)

![](img/71690302f0288fccb8336e6dd04f3de6_87.png)

`vsftpd`支持三种用户访问模式：
1.  **本地用户模式**：使用服务器系统账户（如`root`, `ftpuser`）登录。
2.  **匿名用户模式**：无需密码，使用`anonymous`或`ftp`用户登录。
3.  **虚拟用户模式**：使用独立的数据库文件存储账号，安全性更高。

![](img/71690302f0288fccb8336e6dd04f3de6_89.png)

**默认情况下，vsftpd启用匿名访问模式。** 匿名用户登录后，会进入一个特定的共享目录。

![](img/71690302f0288fccb8336e6dd04f3de6_91.png)

**客户端连接测试：**
在客户端机器（IP如`192.168.0.11`）上安装`lftp`工具并连接服务器。
```bash
yum install -y lftp
lftp 192.168.0.40
```
连接成功后，执行`ls`命令，可以看到服务器共享目录下的内容（默认是一个`pub`文件夹）。

**匿名用户的共享目录**位于服务器的`/var/ftp/`路径下。匿名用户登录后看到的“根目录”即为此目录。
```bash
# 在服务端查看匿名共享目录
ls -ld /var/ftp/
```
任何放入`/var/ftp/pub/`目录的文件，匿名用户都可以访问和下载。

## 客户端操作与权限配置

![](img/71690302f0288fccb8336e6dd04f3de6_93.png)

![](img/71690302f0288fccb8336e6dd04f3de6_95.png)

成功连接后，本节中我们学习如何在客户端进行文件操作，并解决常见的权限问题。

使用`lftp`连接后，可以执行很多命令。输入`help`可以查看支持的命令列表。

**常用命令示例：**
*   `ls`：列出服务器文件。
*   `cd pub`：进入服务器的`pub`目录。
*   `get 文件名`：下载文件到客户端当前目录。
*   `put 文件名`：上传客户端文件到服务器当前目录。
*   `mkdir 目录名`：在服务器创建目录（需有权限）。
*   `exit`：退出连接。

**上传文件时遇到的权限问题**
默认情况下，匿名用户只有下载权限。尝试上传文件会收到“Permission denied”错误。

**解决方法：**
需要同时修改服务器上的**目录权限**和**vsftpd配置文件**。

1.  **修改共享目录的所有者**（以`pub`目录为例）：
    ```bash
    chown ftp:ftp /var/ftp/pub/
    ```
2.  **编辑vsftpd配置文件，启用匿名写入权限**：
    ```bash
    vim /etc/vsftpd/vsftpd.conf
    ```
    找到并修改或添加以下行（移除行首的`#`以取消注释）：
    ```bash
    # 允许匿名用户上传文件
    anon_upload_enable=YES
    # 允许匿名用户创建目录
    anon_mkdir_write_enable=YES
    # 允许匿名用户写入（覆盖、修改）
    anon_other_write_enable=YES
    ```
3.  **重启vsftpd服务使配置生效**：
    ```bash
    systemctl restart vsftpd
    ```
4.  **客户端重新连接并测试**：
    现在可以在`/pub`目录内进行上传、创建文件夹等操作。
