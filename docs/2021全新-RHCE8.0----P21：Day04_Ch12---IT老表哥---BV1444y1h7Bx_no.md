上一节我们介绍了IPv4的网络配置，本节中我们来看看IPv6的简单配置方法。IPv6是下一代互联网协议，地址长度为128位。

![](img/b3107af461914c14f8fe1f2e81d96ada_15.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_17.png)

IPv6的配置主要在网卡配置文件中进行。以下是关键步骤：

1.  首先，需要关闭IPv6的自动配置功能。
2.  然后，手动指定一个IPv6地址。

一个典型的IPv6地址配置示例如下：
```bash
IPV6_AUTOCONF=no
IPV6ADDR=2020::1/64
```
在这个例子中，`2020::1`是简写形式，代表`2020:0000:0000:0000:0000:0000:0000:0001`。IPv6地址共128位，通常由冒号分隔的8组十六进制数表示，每组16位。`/64`表示网络前缀的长度。

配置完成后，可以使用`ping6`命令测试IPv6网络的连通性：
```bash
ping6 2020::2
```
请注意，测试的前提是通信双方都配置了IPv6地址。

## 链路聚合（Team）概念与模式 ⛓️

链路聚合是一种网络技术，它将多个物理网络端口汇聚在一起，形成一个逻辑端口。这样做主要有两个目的：**增加总吞吐量**和**提供链路冗余**。当一条物理链路出现故障时，流量可以自动切换到其他正常链路，保证网络不中断。

在RHEL8中，我们通常使用`teamd`服务来实现链路聚合，它比传统的`bond`方式更灵活。常见的聚合模式有以下几种：

*   **负载均衡（loadbalance）**：所有网卡同时工作，总带宽是各成员带宽之和。
*   **轮询（roundrobin）**：在所有活动端口间轮流发送数据包。
*   **主备（activebackup）**：只有一个端口处于活动状态，其他端口作为备份。当活动端口故障时，备份端口会接替其工作。

下面，我们将以**主备（activebackup）** 模式为例，演示如何配置链路聚合。

## 实战：配置Active Backup模式链路聚合 🛠️

我们将使用两台物理网卡（例如`ens161`和`ens193`）来创建一个逻辑聚合端口`team0`。

![](img/b3107af461914c14f8fe1f2e81d96ada_32.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_34.png)

以下是配置链路聚合的核心步骤：

![](img/b3107af461914c14f8fe1f2e81d96ada_35.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_37.png)

### 步骤一：创建Team逻辑接口

![](img/b3107af461914c14f8fe1f2e81d96ada_39.png)

首先，我们需要创建一个类型为`team`的逻辑接口，并指定其运行模式。
```bash
nmcli connection add type team con-name team0 ifname team0 config '{"runner": {"name": "activebackup"}}'
```
这条命令创建了一个名为`team0`的逻辑接口，并设置其运行模式为`activebackup`（主备模式）。

![](img/b3107af461914c14f8fe1f2e81d96ada_41.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_43.png)

### 步骤二：将物理网卡添加为Team成员

接着，需要将两块物理网卡作为“从属”成员加入到这个`team0`接口中。
```bash
nmcli connection add type team-slave con-name team0-port1 ifname ens161 master team0
nmcli connection add type team-slave con-name team0-port2 ifname ens193 master team0
```
**重要提示**：物理网卡（`ens161`, `ens193`）本身不需要配置IP地址，IP地址应配置在逻辑接口`team0`上。

![](img/b3107af461914c14f8fe1f2e81d96ada_45.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_47.png)

### 步骤三：为Team接口配置IP地址

![](img/b3107af461914c14f8fe1f2e81d96ada_49.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_51.png)

现在，为逻辑接口`team0`配置IP地址，使其能够进行网络通信。
```bash
nmcli connection modify team0 ipv4.method manual ipv4.addresses 192.168.145.200/24
```

![](img/b3107af461914c14f8fe1f2e81d96ada_53.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_55.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_57.png)

### 步骤四：激活所有连接

配置完成后，激活`team0`接口及其成员。
```bash
nmcli connection up team0
nmcli connection up team0-port1
nmcli connection up team0-port2
```

![](img/b3107af461914c14f8fe1f2e81d96ada_59.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_61.png)

![](img/b3107af461914c14f8fe1f2e81d96ada_62.png)

使用以下命令检查`team0`的状态，可以看到当前活动的端口。
```bash

```
通过`ping`命令持续测试`team0`接口的IP地址。然后，尝试断开当前活动的物理网卡（例如`nmcli device disconnect ens161`），观察`ping`测试是否中断，并使用`teamdctl`命令查看活动端口是否已自动切换到备份网卡（如`ens193`）。恢复断开的网卡后，流量可能会切回原主端口或保持现状，这体现了主备模式的冗余能力。

## 简化配置方法：使用配置文件示例 📄

如果你觉得命令行复杂，RHEL系统提供了配置示例文件，可以直接修改使用。这是一个更快捷的方法。

示例文件通常位于`/usr/share/doc/`目录下。你可以找到名为`teamd.conf`的示例，复制并修改其中的接口名、IP地址和成员网卡名，然后将其放到`/etc/sysconfig/network-scripts/`目录下（如`ifcfg-team0`），最后使用`nmcli connection reload`和`nmcli connection up team0`激活即可。
