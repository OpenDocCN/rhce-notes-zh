## 开源软件与Linux起源

![](img/624a7666131e4ded3b5c98f6d0ea0818_11.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_13.png)

上一节我们完成了RHEL7的安装，本节中我们来看看Linux背后的理念和它的起源。

![](img/624a7666131e4ded3b5c98f6d0ea0818_15.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_16.png)

开源（Open Source）指的是软件的源代码（Source Code）对所有人开放，可以被自由获取、学习、修改和分发。但请注意，**开源不等于免费**。例如，红帽企业版Linux（RHEL）是开源的，但企业用户通常需要付费订阅以获得技术支持服务。

![](img/624a7666131e4ded3b5c98f6d0ea0818_18.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_20.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_22.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_23.png)

Linux的理念源于自由软件运动。自由软件基金会（FSF）定义了四大自由：
*   以任何目的运行软件的自由。
*   学习和修改软件源代码的自由。
*   重新分发软件副本的自由。
*   改进软件并向社区发布改进版本的自由。

![](img/624a7666131e4ded3b5c98f6d0ea0818_25.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_26.png)

Linux本身是一个年轻的操作系统，其内核（Kernel）由林纳斯·托瓦兹于1991年首次发布。它的诞生与GNU项目密不可分。GNU项目旨在创建一个完全自由的操作系统，但在1990年代初期，它缺少一个可用的内核。Linux内核的出现恰好填补了这一空白，两者结合形成了我们今天所说的“GNU/Linux”操作系统。

![](img/624a7666131e4ded3b5c98f6d0ea0818_28.png)

重启后，首先会看到“许可协议”界面。此处需要勾选“我接受许可协议”的复选框，然后点击右下角的“完成配置”。

![](img/624a7666131e4ded3b5c98f6d0ea0818_55.png)

接下来，系统会介绍 **Kdump** 机制。Kdump是内核崩溃转储机制，用于在内核发生严重错误（类似Windows蓝屏）时，将内存中的运行状态信息捕获并保存到硬盘，生成一个转储文件（`vmcore`）。这个文件可以发送给技术支持人员，用于分析崩溃原因。对于学习环境，如果内存资源紧张，可以选择禁用此功能。我们点击右下角的“前进”，然后选择“启用kdump”或“禁用kdump”后继续。

![](img/624a7666131e4ded3b5c98f6d0ea0818_57.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_59.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_61.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_63.png)

![](img/624a7666131e4ded3b5c98f6d0ea0818_65.png)

随后是“订阅注册”界面。RHEL是商业发行版，需要订阅红帽服务才能获得官方更新和技术支持。对于实验和学习环境，我们不需要注册。请选择“不，我以后注册”或类似选项，然后点击“完成”。

完成上述配置后，系统会再次重启，进入登录界面。

## 系统登录与用户切换

配置完成后，我们终于可以进入系统了。本节将介绍如何使用不同用户账户登录。

系统重启后，将显示图形化登录界面。你可能会看到创建的非root用户（例如“张三”）的图标。点击该图标，输入密码即可登录。

**root** 用户是系统的超级管理员，默认出于安全考虑不会直接显示在登录列表。要使用root用户登录，请执行以下操作：
1.  在登录界面，点击“未列出”或“Not listed”。
2.  在用户名输入框中键入 **`root`**。
3.  点击“下一步”。
4.  输入你在安装过程中为root用户设置的密码。

登录成功后，你将看到RHEL7的桌面环境，这标志着操作系统安装与初始配置全部完成。

## 连接课堂环境（可选）

为了在实体机课堂上同步观看教师的演示屏幕，本地学员可能需要配置网络并安装远程桌面软件。

首先，确保你的电脑通过网线连接到教室网络。网络配置信息通常由教师提供。

其次，你需要安装VNC Viewer软件来连接教师的演示机。以下是在RHEL7终端中安装一个VNC客户端（例如`tigervnc`）的命令示例：
```bash
yum install tigervnc
```
请注意，新安装的纯净系统可能无法直接使用`yum`命令，因为它需要配置软件仓库。此步骤仅作演示，课堂环境会有特定安排。

安装后，你可以使用VNC Viewer连接教师提供的IP地址（例如 `172.25.254.77`）来观看演示。

**注意**：如果你的系统设置了自动登录，需要先注销当前用户才能看到登录界面并切换至root用户。点击屏幕右上角的用户名，选择“注销”即可。

本节课中我们一起学习了开源软件的核心概念，了解了Linux的起源与发展，并完成了RHEL7操作系统安装后的基本配置和登录操作。

我们明确了开源（开放源代码）与免费的区别，认识了Linux作为GNU项目关键组成部分的历史地位。在实践部分，我们逐步操作了接受许可、了解Kdump、跳过订阅注册，最终以root用户和非root用户成功登录系统。
