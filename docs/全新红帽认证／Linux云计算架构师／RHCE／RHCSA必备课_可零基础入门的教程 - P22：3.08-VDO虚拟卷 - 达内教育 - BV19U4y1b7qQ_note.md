# Linux存储管理：3.08：VDO虚拟卷

![](img/772654e052213d03b56db71b60129c43_1.png)

![](img/772654e052213d03b56db71b60129c43_3.png)

在本节课中，我们将要学习一种名为VDO（Virtual Data Optimizer，虚拟数据优化器）的存储技术。这是一种通过删除重复数据来优化存储空间使用的方法，能够让我们用较小的物理磁盘空间，为用户提供更大的逻辑存储空间。

![](img/772654e052213d03b56db71b60129c43_5.png)

## 认识VDO 🧐

上一节我们介绍了传统的磁盘分区与逻辑卷管理，本节中我们来看看一种更“聪明”的存储优化技术——VDO。

VDO是红帽8系统中引入的一个内核模块。它的核心作用是**减少重复数据在磁盘上的存储**。例如，如果同一个文件被存储了多份，VDO可以确保在物理磁盘上只保留一份数据，从而节省空间。但对于访问这些文件的用户或程序来说，他们各自看到的仍然是完整的、独立的一份文件。

我们可以用一个简单的公式来理解其空间节省原理：
**实际物理占用空间 ≈ 唯一数据量 + 少量元数据**
**用户感知的逻辑空间 = 配置的虚拟大小**

## VDO的工作原理 🛠️

![](img/772654e052213d03b56db71b60129c43_12.png)

![](img/772654e052213d03b56db71b60129c43_14.png)

VDO主要通过两种技术实现空间优化：
1.  **零区块排除**：忽略全由零组成的数据块，不实际存储。
2.  **重复数据删除**：识别并合并完全相同的数据块。

![](img/772654e052213d03b56db71b60129c43_16.png)

![](img/772654e052213d03b56db71b60129c43_18.png)

这类似于网盘服务的工作原理：当多个用户存储了相同的电影文件时，服务商在服务器上可能只保存一份，但每个用户都认为自己独享了该文件的存储空间。VDO在本地服务器上实现了类似的功能。

## 实践：创建与配置VDO卷 💻

![](img/772654e052213d03b56db71b60129c43_24.png)

接下来，我们通过一个具体任务来学习如何配置VDO卷。任务目标是：使用一块10GB的物理磁盘（`/dev/vdc`），创建一个名为`myvdo`、逻辑大小为50GB的VDO卷，并将其格式化和挂载。

以下是完成此任务所需的步骤：

![](img/772654e052213d03b56db71b60129c43_26.png)

![](img/772654e052213d03b56db71b60129c43_27.png)

### 1. 安装必要软件包
首先，我们需要安装VDO管理工具。
```bash
yum install vdo -y
```
安装的软件包、管理命令和系统服务名称都叫`vdo`。

### 2. 创建VDO卷
创建VDO卷是核心步骤。如果不记得命令格式，可以查看帮助手册：
```bash
man vdo
```
在手册中，输入`/EXAMPLE`查找示例，可以找到创建命令的模板。根据我们的任务要求，创建命令如下：
```bash
vdo create --name=myvdo --device=/dev/vdc --vdoLogicalSize=50G
```
*   `--name`：指定VDO卷的名称。
*   `--device`：指定使用的物理磁盘设备。
*   `--vdoLogicalSize`：指定逻辑（虚拟）大小，这里设置为50GB。

![](img/772654e052213d03b56db71b60129c43_29.png)

![](img/772654e052213d03b56db71b60129c43_31.png)

创建成功后，虚拟设备路径通常为`/dev/mapper/myvdo`。

![](img/772654e052213d03b56db71b60129c43_33.png)

![](img/772654e052213d03b56db71b60129c43_35.png)

### 3. 查看VDO卷信息
创建后，可以使用以下命令查看状态：
```bash
vdo status --name=myvdo            # 查看简要状态
vdo stats --name=myvdo             # 查看详细统计信息（如物理/逻辑大小）
```

![](img/772654e052213d03b56db71b60129c43_37.png)

![](img/772654e052213d03b56db71b60129c43_39.png)

### 4. 格式化VDO卷
接下来格式化VDO卷。**为了加快格式化速度，必须使用特定参数**：
*   如果格式化为XFS文件系统：
    ```bash
    mkfs.xfs -K /dev/mapper/myvdo
    ```
    `-K` 参数用于跳过丢弃数据块，能极大加速大容量虚拟设备的格式化过程。
*   如果格式化为EXT4文件系统：
    ```bash
    mkfs.ext4 -E nodiscard /dev/mapper/myvdo
    ```

![](img/772654e052213d03b56db71b60129c43_41.png)

![](img/772654e052213d03b56db71b60129c43_43.png)

### 5. 配置挂载
首先，创建挂载点并临时挂载，测试是否正常：
```bash
mkdir /vblock
mount /dev/mapper/myvdo /vblock
```
使用`df -h`命令查看，会发现`/vblock`显示的大小约为50GB。

![](img/772654e052213d03b56db71b60129c43_45.png)

![](img/772654e052213d03b56db71b60129c43_47.png)

![](img/772654e052213d03b56db71b60129c43_49.png)

![](img/772654e052213d03b56db71b60129c43_51.png)

### 6. 配置开机自动挂载
为了让VDO卷在系统启动时自动挂载，需要编辑`/etc/fstab`文件。**这里有一个关键点**：必须确保挂载时VDO服务已经启动，否则会挂载失败导致系统启动异常。

![](img/772654e052213d03b56db71b60129c43_53.png)

![](img/772654e052213d03b56db71b60129c43_55.png)

![](img/772654e052213d03b56db71b60129c43_57.png)

![](img/772654e052213d03b56db71b60129c43_59.png)

在`/etc/fstab`中添加如下一行：
```bash
/dev/mapper/myvdo /vblock xfs defaults,_netdev 0 0
```
重点是 `_netdev` 这个挂载选项。它表示这是一个网络型设备（尽管VDO是本地设备），系统会等待基础网络就绪后再挂载，这间接保证了VDO服务已启动。这是确保开机挂载成功的可靠方法。

最后，启用VDO服务使其开机自启：
```bash
systemctl enable vdo
```

## 验证去重效果 🔍

配置完成后，我们可以验证VDO的去重功能。向VDO卷中多次复制同一个大文件（例如一个ISO镜像），并观察物理空间的使用情况。

1.  复制一个大文件到挂载点多次：
    ```bash
    cp /path/to/large_file.iso /vblock/file1.iso
    cp /path/to/large_file.iso /vblock/file2.iso
    cp /path/to/large_file.iso /vblock/file3.iso
    ```
2.  检查逻辑空间使用（用户视角）：
    ```bash
    df -h /vblock
    ```
    显示的空间使用量会累加。
3.  检查物理空间使用（管理员视角）：
    ```bash
    vdo stats --name=myvdo | grep -i “physical”
    ```
    你会发现物理空间的增长量远小于多个文件大小的总和，因为重复的数据只存储了一次。

本节课中我们一起学习了VDO虚拟卷技术。我们了解到VDO是一种通过零块排除和重复数据删除来优化存储空间的解决方案。我们实践了从安装软件包、创建逻辑卷大于物理盘的VDO设备、到使用特殊参数进行格式化、以及正确配置开机挂载的完整流程。

![](img/772654e052213d03b56db71b60129c43_77.png)

![](img/772654e052213d03b56db71b60129c43_79.png)

![](img/772654e052213d03b56db71b60129c43_81.png)

关键要点总结：
*   **核心价值**：用更少的物理空间提供更大的逻辑存储空间，节省成本。
*   **关键命令**：使用 `vdo create` 创建卷，注意 `--vdoLogicalSize` 参数。
*   **格式化技巧**：格式化XFS用 `-K`，EXT4用 `-E nodiscard` 以提升速度。
*   **挂载关键**：在`/etc/fstab`中必须为VDO设备添加 `_netdev` 挂载选项，以确保启动顺序正确。
