## 用户密码设置

为用户设置密码是使其能够登录系统的关键步骤。我们使用 `passwd` 命令来完成此操作。

![](img/49cd58bf7244e86a67f72a691c66d517_10.png)

![](img/49cd58bf7244e86a67f72a691c66d517_12.png)

### passwd命令基础用法

`passwd` 命令的格式非常简单。

```
passwd [用户名]
```

![](img/49cd58bf7244e86a67f72a691c66d517_14.png)

![](img/49cd58bf7244e86a67f72a691c66d517_16.png)

*   如果不指定用户名，则修改当前登录用户自身的密码。
*   如果指定用户名（仅root用户有此权限），则为指定用户修改密码。

![](img/49cd58bf7244e86a67f72a691c66d517_18.png)

![](img/49cd58bf7244e86a67f72a691c66d517_20.png)

![](img/49cd58bf7244e86a67f72a691c66d517_22.png)

以下是具体操作示例：

*   **root用户修改自身密码**：直接输入 `passwd` 命令，无需输入旧密码即可设置新密码。
*   **root用户为其他用户（如user1）设置密码**：使用 `passwd user1` 命令。
*   **普通用户修改自身密码**：使用 `passwd` 命令，但必须首先输入正确的当前密码，并且新密码必须符合系统的密码复杂度规则。

### 密码复杂度规则

系统对密码有严格的复杂度要求，普通用户必须遵守，而root用户则不受此限制。主要规则包括：
*   密码长度不少于8个字符。
*   密码应包含数字、字母（区分大小写）以及特殊字符。
*   新密码不能与旧密码过于相似。

如果密码不符合规范，系统会给出相应提示。普通用户连续尝试失败次数过多会被暂时禁止修改。

## 用户密码文件详解

用户设置密码后，加密的密码信息并非存储在 `/etc/passwd` 文件中，而是存放在 `/etc/shadow` 文件里。这个文件对系统安全至关重要。

![](img/49cd58bf7244e86a67f72a691c66d517_24.png)

![](img/49cd58bf7244e86a67f72a691c66d517_26.png)

### 文件权限与关系

![](img/49cd58bf7244e86a67f72a691c66d517_28.png)

![](img/49cd58bf7244e86a67f72a691c66d517_30.png)

`/etc/shadow` 文件权限严格，普通用户无法查看，只有root用户可以读写。

![](img/49cd58bf7244e86a67f72a691c66d517_32.png)

![](img/49cd58bf7244e86a67f72a691c66d517_34.png)

![](img/49cd58bf7244e86a67f72a691c66d517_36.png)

```
ls -l /etc/shadow
---------- 1 root root 日期 时间 /etc/shadow
```

![](img/49cd58bf7244e86a67f72a691c66d517_38.png)

`/etc/passwd` 文件存储用户的基本信息（如用户名、UID、家目录等），而 `/etc/shadow` 文件则专门存储用户的密码及相关策略信息。两个文件的行数一致，每一行对应一个用户。

![](img/49cd58bf7244e86a67f72a691c66d517_40.png)

![](img/49cd58bf7244e86a67f72a691c66d517_42.png)

![](img/49cd58bf7244e86a67f72a691c66d517_44.png)

### 字段含义解读

![](img/49cd58bf7244e86a67f72a691c66d517_46.png)

`/etc/shadow` 文件中每一行代表一个用户的密码信息，以冒号 `:` 分隔为9个字段。我们以 `user1` 用户行为例进行解读：

1.  **用户名**：用户的登录名，例如 `user1`。
2.  **加密密码**：经过加密算法（如SHA512）处理后的密码字符串。如果该字段是 `*` 或 `!!`，则表示该用户密码未设置或已锁定，不能用于登录。
3.  **最后修改日期**：从1970年1月1日（Unix纪元）到上次修改密码之日所经过的天数。
4.  **最小修改间隔**：密码修改后，必须经过多少天才可以再次修改。如果为 `0`，表示随时可以修改。
5.  **密码有效期**：密码保持有效的最大天数。例如 `99999` 表示密码几乎永不过期。可以设置为如 `60`，表示密码有效期为60天。
6.  **到期前警告天数**：在密码到期前多少天开始向用户发出警告。例如 `7` 表示在密码到期前7天发出警告。
7.  **过期后宽限天数**：密码过期后，用户仍然可以登录并修改密码的天数。在此期间，用户登录后会强制要求修改密码。
8.  **账号失效日期**：从1970年1月1日起，到账号自动失效的天数。到达此日期后，账号将无法使用。
9.  **保留字段**：目前未使用，留作未来扩展。

## 密码策略管理

了解文件结构后，我们可以通过命令或直接修改文件来管理密码策略。

### 使用chage命令

`chage` 命令是专门用来修改用户密码过期信息的工具，它实际上就是修改 `/etc/shadow` 文件中的相关字段。

一个非常重要的应用是**强制用户首次登录时必须修改密码**。这通过将“最后修改日期”字段设置为 `0` 来实现。

```
chage -d 0 user6
```

执行此命令后，`user6` 用户首次使用初始密码登录时，系统会强制要求其立即修改密码，否则无法登录。这确保了初始密码的安全性。

![](img/49cd58bf7244e86a67f72a691c66d517_48.png)

![](img/49cd58bf7244e86a67f72a691c66d517_50.png)

### 使用passwd命令选项

`passwd` 命令还有一些有用的选项来管理密码状态：

![](img/49cd58bf7244e86a67f72a691c66d517_52.png)

![](img/49cd58bf7244e86a67f72a691c66d517_54.png)

*   **查看密码信息**：`passwd -S user1` 可以查看用户的密码状态、设置时间、有效期等摘要信息。
*   **锁定与解锁用户密码**：
    *   锁定：`passwd -l user1` （用户将无法使用密码登录）
    *   解锁：`passwd -u user1`
*   **清空用户密码**：`passwd -d user1` （将密码字段清空，用户可无需密码登录，但通常仅用于特定场景）

![](img/49cd58bf7244e86a67f72a691c66d517_56.png)

### 非交互式设置密码

![](img/49cd58bf7244e86a67f72a691c66d517_58.png)

在脚本中，我们可能需要非交互式地设置密码，这时可以使用 `--stdin` 选项。

```
echo "MyNewPass123!" | passwd --stdin user8
```

![](img/49cd58bf7244e86a67f72a691c66d517_60.png)

这种方式会将 `echo` 输出的字符串直接作为新密码设置给 `user8`。**注意**：在命令行历史中可能会留下明文密码记录，因此需谨慎使用。

## 用户身份切换

最后，我们简单了解 `su` 命令，它用于切换当前用户身份。

*   **切换用户**：`su username` 切换到指定用户，但环境变量可能仍是原用户的。
*   **完全切换（推荐）**：`su - username` 或 `su -l username`，这会完全切换到目标用户的环境，包括家目录和Shell配置。
*   **退出切换**：输入 `exit` 命令或按 `Ctrl+D` 即可退回到之前的用户身份。

![](img/49cd58bf7244e86a67f72a691c66d517_71.png)

**注意**：root用户切换到任何其他用户都无需输入密码。普通用户切换身份则需要知道目标用户的密码。
