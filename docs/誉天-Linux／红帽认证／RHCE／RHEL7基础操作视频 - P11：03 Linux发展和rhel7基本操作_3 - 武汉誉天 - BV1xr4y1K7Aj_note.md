## 系统登录方式

![](img/42b51c2765d7b2da2fd44999a75d7de3_4.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_6.png)

Linux系统提供了两种主要的登录方式：基于文本的字符界面登录和基于图形的图形界面登录。这两种方式都需要使用用户名和密码进行身份验证。

### 字符界面登录

![](img/42b51c2765d7b2da2fd44999a75d7de3_14.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_16.png)

如果系统以最小化方式安装，或者从控制台启动，您将看到一个黑色的字符界面。界面上会显示 `login:` 提示符，您需要在此输入用户名，然后在 `Password:` 提示符后输入密码。密码输入时不会显示任何字符（如星号或圆点），这是正常的安全措施。

登录成功后，您会看到命令提示符，例如 `[root@localhost ~]#`，这表示您已成功进入系统。

![](img/42b51c2765d7b2da2fd44999a75d7de3_25.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_27.png)

### 图形界面登录

如果系统安装了图形界面（如GNOME），开机后通常会进入图形登录界面。您需要在此选择或输入用户名，然后在弹出的窗口中输入密码。登录后，您将进入图形化的桌面环境。

## 虚拟控制台与界面切换

![](img/42b51c2765d7b2da2fd44999a75d7de3_30.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_32.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_33.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_35.png)

Linux系统默认提供了多个独立的虚拟控制台，方便用户在不同工作环境间切换。

![](img/42b51c2765d7b2da2fd44999a75d7de3_37.png)

### 控制台分布与切换

系统通常有6个控制台：1个图形控制台和5个虚拟字符控制台。它们之间是相互独立的。

![](img/42b51c2765d7b2da2fd44999a75d7de3_39.png)

*   **从图形界面切换到字符界面**：按下 `Ctrl + Alt + F2` 到 `Ctrl + Alt + F6` 可以切换到对应的虚拟字符控制台（tty2 到 tty6）。
*   **从字符界面切换回图形界面**：按下 `Ctrl + Alt + F1` 可以切换回图形控制台（tty1）。

![](img/42b51c2765d7b2da2fd44999a75d7de3_41.png)

### 在字符界面启动图形界面

如果您当前处于字符界面，但系统已安装图形环境，可以通过执行 `startx` 命令来启动图形界面。

```bash
startx
```

**注意**：此命令成功的前提是系统已安装图形界面所需的软件包（如X Window系统和桌面环境）。

## 图形终端（GNOME Terminal）的使用技巧

![](img/42b51c2765d7b2da2fd44999a75d7de3_43.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_44.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_46.png)

![](img/42b51c2765d7b2da2fd44999a75d7de3_48.png)

在图形界面中，通常通过右键菜单选择“在终端中打开”来启动命令行终端。这个终端工具提供了一些便捷操作：

*   **新建标签页**：按 `Ctrl + Shift + T` 可以在同一个终端窗口中新建一个标签页。
*   **切换标签页**：按 `Alt + 数字键`（如1, 2, 3）可以快速切换到对应的标签页。也可以使用 `Ctrl + PageUp` 或 `Ctrl + PageDown`。
*   **复制与粘贴**：用鼠标选中文本后，右键菜单选择“复制”和“粘贴”。快捷键是 `Ctrl + Shift + C`（复制）和 `Ctrl + Shift + V`（粘贴）。
*   **调整字体大小**：按 `Ctrl + Shift + +` 可以放大终端字体，按 `Ctrl + -` 可以缩小字体。也可以在终端菜单的 Edit -> Profile Preferences 中进行设置。
*   **清屏**：按 `Ctrl + L` 可以快速清空当前终端屏幕。
*   **退出终端**：建议使用 `exit` 命令安全退出终端，而不是直接点击窗口的关闭按钮。

## 用户与密码管理

每个用户都有一个专属的家目录（Home Directory），用于存放个人文件。例如，root用户的家目录是 `/root`，普通用户的家目录通常在 `/home/用户名` 下。

### 修改用户密码

使用 `passwd` 命令可以修改密码。

*   **root用户修改自身或其他用户密码**：root用户拥有最高权限，修改密码时无需验证旧密码。
    ```bash
    # 修改root自身密码
    passwd
    # 修改其他用户（如user1）的密码
    passwd user1
    ```
    执行命令后，根据提示输入新密码并确认即可。

*   **普通用户修改自身密码**：普通用户执行 `passwd` 时，首先会被要求输入当前密码以验证身份，然后才能设置新密码。
    ```bash
    passwd
    ```

**重要提示**：出于安全考虑，系统通常要求密码长度不少于8个字符。如果密码过于简单，命令会提示“BAD PASSWORD”。

## 理解root用户与权限切换

### root用户的特殊性

root用户是Linux系统中的超级管理员账户，拥有对系统的完全控制权。权力越大，责任和风险也越大，不当操作可能导致系统损坏。因此，在实际生产环境中，应尽量避免直接使用root账户进行日常操作。

### 切换用户身份

使用 `su` 命令可以切换当前用户身份。

*   **切换到其他用户（推荐方式）**：使用 `su - 用户名`。`-` 选项会创建一个完整的登录环境，完全模拟目标用户的身份。
    ```bash
    # 切换到user1用户
    su - user1
    ```
*   **切换回root用户**：
    ```bash
    su - root
    # 或使用简写
    su -
    ```
    从普通用户切换回root需要输入root的密码。

*   **退出当前用户身份**：执行 `exit` 命令可以退回之前的用户身份。

**注意**：应避免连续嵌套切换多个用户（如 `su - user1` 后再 `su - user2`），这可能导致环境变量混乱。正确的做法是退出一层身份后再切换。

### 临时提升权限（sudo）

普通用户在执行某些需要root权限的命令时，可以使用 `sudo` 命令临时获取权限。

```bash
# 以root权限执行passwd命令
sudo passwd user2
```
首次使用 `sudo` 时，系统会显示一段提醒文字，强调权力与责任，并要求输入当前用户自己的密码（不是root密码）进行确认。能否使用 `sudo` 以及能执行哪些命令，由系统管理员通过配置决定。
