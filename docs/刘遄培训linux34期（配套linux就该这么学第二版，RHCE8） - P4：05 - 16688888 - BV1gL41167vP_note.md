## 管道符 | 🚪

![](img/58a3b0e928051b5e51038fbd664d8b19_29.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_31.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_33.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_35.png)

管道符（`|`）的作用是将前一个命令的输出结果，作为后一个命令的输入。它实现了命令与命令之间的数据交互。

![](img/58a3b0e928051b5e51038fbd664d8b19_37.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_39.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_41.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_43.png)

**核心公式：**
```
命令A | 命令B
```
此格式表示将命令A的输出结果，交给命令B进行二次处理。管道符可以连续使用多次。

![](img/58a3b0e928051b5e51038fbd664d8b19_47.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_53.png)

以下是管道符的几个典型应用场景：

![](img/58a3b0e928051b5e51038fbd664d8b19_57.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_67.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_69.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_71.png)

*   **统计登录用户数量**：我们想统计系统中可以登录的用户数量。首先，`/etc/passwd`文件保存了用户信息，其中最后一列是用户的登录终端。`/bin/bash`表示可以登录，`/sbin/nologin`表示不能登录。我们可以先用`grep`命令过滤出可以登录的用户行，再通过管道符将结果交给`wc -l`命令统计行数。
    ```
    grep "/bin/bash" /etc/passwd | wc -l
    ```
*   **统计当前目录文件数**：`ls`命令默认输出格式会随终端宽度变化，但每行一个文件是固定的。我们可以用`wc -l`统计`ls`命令输出的行数。
    ```
    ls | wc -l
    ```
*   **查找特定进程**：`ps aux`命令会列出所有进程，信息很多。我们可以通过管道符用`grep`命令快速过滤出我们关心的进程。
    ```
    ps aux | grep sshd
    ```
*   **非交互式设置密码**：`passwd`命令通常需要交互式输入密码。我们可以用`echo`命令输出密码，然后通过管道符传递给`passwd`命令。但需要注意，`passwd`命令默认不接收标准输入，需要加上`--stdin`参数。
    ```
    echo "redhat" | passwd --stdin linuxprobe
    ```

## 重定向符 > 与 < 📤📥

![](img/58a3b0e928051b5e51038fbd664d8b19_87.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_89.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_91.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_93.png)

重定向符的作用是改变命令的输入源或输出目标，主要实现命令与文件之间的数据交互。

![](img/58a3b0e928051b5e51038fbd664d8b19_95.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_97.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_99.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_101.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_103.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_105.png)

**核心概念：**
*   **输出重定向**：将命令原本要输出到屏幕（标准输出）的内容，写入到文件中。
    *   **覆盖写入**：`命令 > 文件`
    *   **追加写入**：`命令 >> 文件`
*   **输入重定向**：将文件中的内容，作为命令的输入。
    *   **基本格式**：`命令 < 文件`

![](img/58a3b0e928051b5e51038fbd664d8b19_107.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_109.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_111.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_113.png)

以下是重定向的几种常见用法：

![](img/58a3b0e928051b5e51038fbd664d8b19_115.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_117.png)

*   **将命令结果保存到文件**：
    ```
    ls -l > file_list.txt
    ```
*   **将多行命令结果追加到文件**：
    ```
    echo "Line 1" >> log.txt
    echo "Line 2" >> log.txt
    ```
*   **将文件内容作为命令输入**：
    ```
    wc -l < /etc/passwd
    ```

## 通配符 * 与 ? 🔍

![](img/58a3b0e928051b5e51038fbd664d8b19_129.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_131.png)

通配符用于匹配文件名或字符串，当我们只知道部分信息时，可以用它来批量操作。

![](img/58a3b0e928051b5e51038fbd664d8b19_133.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_135.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_137.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_139.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_141.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_143.png)

**核心代码：**
*   `*`：匹配零个、一个或多个任意字符。
    *   `ls /dev/sda*` 匹配所有以`sda`开头的文件。
*   `?`：匹配单个任意字符。
    *   `ls /dev/sda?` 匹配如`sda1`, `sda2`等，但不匹配`sda`本身。
*   `[0-9]`：匹配单个数字。
*   `[a-z]`：匹配单个小写字母。
*   `[A-Z]`：匹配单个大写字母。
*   `[a,z]`或`[135]`：匹配括号内指定的单个字符。

![](img/58a3b0e928051b5e51038fbd664d8b19_147.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_157.png)

## 转义符 \ 与引号 🛡️

![](img/58a3b0e928051b5e51038fbd664d8b19_165.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_167.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_169.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_171.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_173.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_175.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_177.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_179.png)

转义符和引号用于让Shell正确理解我们的意图，特别是当命令或字符串中包含特殊字符（如空格、`$`、`*`等）时。

![](img/58a3b0e928051b5e51038fbd664d8b19_181.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_183.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_185.png)

**核心代码：**
*   **反斜杠 `\`**：转义紧随其后的单个字符，使其失去特殊含义，变成普通字符。
    ```
    echo Price is \$5
    # 输出：Price is $5 （否则$5会被当作变量）
    ```
*   **单引号 `‘’`**：强引用。引号内的所有字符都视为普通字符，变量和命令都不会被解析。
    ```
    echo ‘Price is $5’
    # 输出：Price is $5
    ```
*   **双引号 `“”`**：弱引用。引号内的内容作为一个整体，但其中的变量（`$VAR`）和命令替换（`` `command` `` 或 `$(command)`）会被解析。
    ```
    name=“Linux”
    echo “Hello, $name”
    # 输出：Hello, Linux
    ```
*   **反引号 `` ` ``**：命令替换。执行反引号内的命令，并将其输出结果替换到当前位置。等效于`$(命令)`。
    ```
    current_dir=`pwd`
    # 或 current_dir=$(pwd)
    echo $current_dir
    ```

## 环境变量 🌍

![](img/58a3b0e928051b5e51038fbd664d8b19_189.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_191.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_193.png)

环境变量是操作系统或Shell中预先设置好的参数，用于定义系统的工作环境。它们通常用大写字母表示。

![](img/58a3b0e928051b5e51038fbd664d8b19_195.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_197.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_199.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_201.png)

**查看与使用变量：**
*   `echo $变量名`：查看变量的值。
*   `变量名=值`：定义变量（等号两边不能有空格）。
*   `export 变量名`：将局部变量提升为全局环境变量，使其他用户或Shell也能使用。

![](img/58a3b0e928051b5e51038fbd664d8b19_203.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_205.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_207.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_209.png)

**重要环境变量示例：**
*   `PATH`：系统查找命令的路径列表，用冒号分隔。
*   `HOME`：当前用户的家目录。
*   `SHELL`：当前用户使用的Shell解释器。
*   `HISTSIZE`：历史命令记录条数。
*   `LANG`：系统语言和字符集设置。

![](img/58a3b0e928051b5e51038fbd664d8b19_211.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_213.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_215.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_217.png)

**修改环境变量：**
要永久修改环境变量，通常需要编辑配置文件（如`/etc/profile`或用户家目录的`.bashrc`），然后使用`source`命令使更改立即生效。
```
vim /etc/profile
# 在文件中修改变量，例如：HISTSIZE=5000
source /etc/profile
```

## Vim文本编辑器 ✏️

![](img/58a3b0e928051b5e51038fbd664d8b19_223.png)

Vim是Linux中最强大、最通用的文本编辑器之一。它有三种主要模式：

![](img/58a3b0e928051b5e51038fbd664d8b19_225.png)

![](img/58a3b0e928051b5e51038fbd664d8b19_227.png)

1.  **命令模式**：启动Vim后的默认模式。可以执行复制(`yy`)、粘贴(`p`)、删除(`dd`)、查找(`/关键词`)等操作，也是进入其他模式的桥梁。
2.  **输入模式**：在命令模式下按 `i`（光标前插入）、`a`（光标后插入）、`o`（下一行插入）等键进入。可以自由编辑文本。按 `Esc` 键返回命令模式。
3.  **末行模式**：在命令模式下按 `:` 进入。可以执行保存(`:w`)、退出(`:q`)、保存并退出(`:wq`)、强制操作(`:wq!`)等。

**基本使用流程：**
```
vim filename          # 打开/创建文件（命令模式）
i                     # 进入输入模式，开始编辑
[编辑内容]
Esc                   # 返回命令模式
:wq!                  # 进入末行模式，保存并强制退出
```
