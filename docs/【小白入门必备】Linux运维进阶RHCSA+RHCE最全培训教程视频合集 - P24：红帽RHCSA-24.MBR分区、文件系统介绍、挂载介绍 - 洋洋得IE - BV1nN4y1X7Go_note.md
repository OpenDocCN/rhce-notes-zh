## 认识 `/dev` 目录

![](img/9b4604ee275337b875648a275d2b3efc_11.png)

![](img/9b4604ee275337b875648a275d2b3efc_13.png)

在Linux系统中，所有的硬件设备都以文件的形式存在于`/dev`目录下。了解这些设备文件是进行磁盘管理的基础。

![](img/9b4604ee275337b875648a275d2b3efc_15.png)

![](img/9b4604ee275337b875648a275d2b3efc_17.png)

![](img/9b4604ee275337b875648a275d2b3efc_19.png)

以下是`/dev`目录下一些常见的设备文件类型：

![](img/9b4604ee275337b875648a275d2b3efc_21.png)

![](img/9b4604ee275337b875648a275d2b3efc_23.png)

![](img/9b4604ee275337b875648a275d2b3efc_25.png)

*   **硬盘设备**：通常以`sd`开头，例如`sda`、`sdb`、`sdc`。较早的IDE硬盘可能以`hd`开头。
*   **光驱设备**：通常为`sr0`，其快捷方式为`cdrom`。
*   **虚拟终端设备**：以`tty`开头，例如`tty1`、`tty2`。每打开一个终端窗口，就会占用一个此类设备。
*   **空设备与满设备**：
    *   `/dev/null`：称为“黑洞设备”，任何写入此设备的数据都会被丢弃。常用于丢弃命令的输出。
    *   `/dev/zero`：称为“满设备”，提供无限的零字节流。常用于创建空文件或测试。
*   **随机数设备**：`/dev/random`和`/dev/urandom`，用于生成随机数。
*   **回环网卡**：`lo`设备，代表本机网络回环地址。

## 使用 `fdisk` 进行MBR分区

上一节我们认识了设备文件，本节中我们来看看如何使用`fdisk`命令对硬盘进行MBR格式的分区。

**命令格式**：`fdisk [设备路径]`
例如，对第二块硬盘进行操作：`fdisk /dev/sdb`

进入`fdisk`交互界面后，所有操作首先保存在内存中，只有在执行保存命令后才会写入磁盘。输入`m`可以获取帮助菜单。

以下是分区过程中常用的几个交互命令：

*   `p`：打印当前磁盘的分区表。可以查看磁盘标签类型（如`DOS`代表MBR格式）、分区数量及大小。
*   `n`：创建一个新分区。系统会依次询问：
    1.  分区类型：主分区(`p`) 或 扩展分区(`e`)。
    2.  分区编号：MBR格式下，主分区和扩展分区编号为1-4。
    3.  起始扇区：通常直接回车使用默认值。
    4.  分区大小：可以使用`+size{K,M,G}`的格式指定，如`+10G`表示10GB。
*   `d`：删除一个分区。
*   `w`：保存所有更改并退出。
*   `q`：不保存更改并退出。

**MBR分区要点**：
*   一块MBR格式的硬盘最多只能有**4个主分区**，或**3个主分区 + 1个扩展分区**。
*   扩展分区本身不能直接存储数据，它像一个容器，可以在其内部创建**逻辑分区**（编号从5开始）。
*   逻辑分区的数量可以很多（例如十多个），它们用于实际存储数据。

## 文件系统介绍

![](img/9b4604ee275337b875648a275d2b3efc_43.png)

![](img/9b4604ee275337b875648a275d2b3efc_45.png)

分区创建完成后，并不能直接用来存储文件。我们需要为分区建立一个“文件管理系统”，即**格式化**，为其赋予一个文件系统。

文件系统负责管理分区内的文件，包括文件的存储、读取、属性以及日志记录等。不同的文件系统有不同的特性和限制。

![](img/9b4604ee275337b875648a275d2b3efc_47.png)

![](img/9b4604ee275337b875648a275d2b3efc_49.png)

目前Linux系统中两种主流的文件系统是：

1.  **ext4**：
    *   曾是CentOS 6等系统的默认文件系统。
    *   支持日志功能，可避免意外宕机导致的数据丢失。
    *   单个文件系统最大支持 **1EB** 的分区容量。
    *   单个文件最大支持 **16TB**。

2.  **XFS**：
    *   是CentOS 7/8 及 RHEL 7/8 等现代系统的默认文件系统。
    *   同样支持日志功能，且数据恢复速度通常比ext4更快。
    *   单个文件系统最大支持 **8EB** 的分区容量。
    *   单个文件最大支持 **500TB**。
    *   被认为是目前主流的、高性能的文件系统。

**格式化命令**：
*   格式化为XFS文件系统：`mkfs.xfs /dev/sdb1`
*   格式化为ext4文件系统：`mkfs.ext4 /dev/sdb1`

![](img/9b4604ee275337b875648a275d2b3efc_51.png)

格式化后，可以使用`lsblk -f`命令查看分区的文件系统类型。

## 挂载介绍

![](img/9b4604ee275337b875648a275d2b3efc_55.png)

![](img/9b4604ee275337b875648a275d2b3efc_57.png)

![](img/9b4604ee275337b875648a275d2b3efc_59.png)

![](img/9b4604ee275337b875648a275d2b3efc_61.png)

上一节我们为分区创建了文件系统，本节中我们来看看如何通过“挂载”来使用这个分区。

![](img/9b4604ee275337b875648a275d2b3efc_63.png)

在Linux中，用户不能直接访问硬件设备（如硬盘分区）。必须通过一个**目录**作为入口来访问，这个目录称为**挂载点**。将分区关联到某个目录的操作，就叫做**挂载**。

![](img/9b4604ee275337b875648a275d2b3efc_65.png)

![](img/9b4604ee275337b875648a275d2b3efc_67.png)

**挂载原理**：
当我们将分区`/dev/sdb1`挂载到目录`/mnt/data`后，所有存入`/mnt/data`目录的文件，实际上都被存储到了`/dev/sdb1`这个分区中。目录`/mnt/data`就像一个“通道”或“窗口”。

**挂载命令**：`mount [设备路径] [挂载点目录]`
例如：`mount /dev/sdb1 /mnt/data`

**卸载命令**：`umount [挂载点目录]`
例如：`umount /mnt/data`

![](img/9b4604ee275337b875648a275d2b3efc_69.png)

![](img/9b4604ee275337b875648a275d2b3efc_70.png)

**重要注意事项**：
1.  **挂载点通常应为空目录**。如果挂载点目录非空，挂载后原有文件会被暂时隐藏（卸载后恢复）。
2.  **一个分区不能同时挂载到多个目录**。
3.  **一个目录不能同时挂载多个分区**。
4.  使用`df -h`命令可以查看当前已挂载的文件系统及其使用情况。

![](img/9b4604ee275337b875648a275d2b3efc_72.png)

![](img/9b4604ee275337b875648a275d2b3efc_74.png)

**操作示例**：
1.  创建挂载点：`mkdir /backup`
2.  格式化分区：`mkfs.xfs /dev/sdb1`
3.  挂载分区：`mount /dev/sdb1 /backup`
4.  使用分区：此时所有存入`/backup`的文件都实际保存在`/dev/sdb1`中。
5.  卸载分区：`umount /backup`

![](img/9b4604ee275337b875648a275d2b3efc_78.png)
