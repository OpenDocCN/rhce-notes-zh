在本节课中，我们将要学习Ansible Vault。Ansible Vault是Ansible中用于加密敏感数据（如密码、密钥等）的工具，它能确保这些信息在存储和传输过程中的安全。我们将学习其基本概念、使用方法以及如何在Playbook中调用加密文件。

## 概述

![](img/7314385d90542d30a025304d1d1bf94f_10.png)

Ansible Vault的作用是加密敏感数据，例如密码等信息。这些数据通常定义在变量文件中。其主要应用场景是加密变量文件或证书。本节课程将重点讲解加密变量文件的方法。

![](img/7314385d90542d30a025304d1d1bf94f_12.png)

## Ansible Vault说明

我们可以通过查看帮助文档来了解Ansible Vault的详细用法。在Linux系统中，遇到不熟悉的命令时，可以使用 `man` 命令或 `--help` 选项来获取帮助信息。

![](img/7314385d90542d30a025304d1d1bf94f_14.png)

![](img/7314385d90542d30a025304d1d1bf94f_15.png)

以下是Ansible Vault的简化版用法说明，其中包含了一些常用参数：

![](img/7314385d90542d30a025304d1d1bf94f_19.png)

*   **create**: 创建新的加密文件。
*   **decrypt**: 解密已加密的文件。
*   **edit**: 编辑已加密的文件。
*   **view**: 查看已加密文件的内容。
*   **encrypt**: 对已存在的文件进行加密。
*   **rekey**: 更改加密文件的密码。
*   **encrypt_string**: 加密单个字符串。

## 创建加密文件

![](img/7314385d90542d30a025304d1d1bf94f_29.png)

首先，我们学习如何创建一个新的加密文件。使用以下命令：

![](img/7314385d90542d30a025304d1d1bf94f_31.png)

![](img/7314385d90542d30a025304d1d1bf94f_33.png)

```bash
ansible-vault create secret.yml
```

执行命令后，系统会提示你输入并确认加密密码。之后，它会调用Vim编辑器，你可以在其中定义需要加密的变量内容。例如：

```yaml
user: testansibleuser
uid: 2020
```

保存并退出后，文件 `secret.yml` 就被创建并加密了。如果使用 `cat` 命令查看，你只能看到加密后的乱码内容。

![](img/7314385d90542d30a025304d1d1bf94f_35.png)

## 查看加密文件内容

要查看加密文件的内容，需要使用 `ansible-vault view` 命令并输入密码：

```bash
ansible-vault view secret.yml
```

输入正确的密码后，即可看到文件中的明文内容。

![](img/7314385d90542d30a025304d1d1bf94f_37.png)

## 在Playbook中调用加密文件

上一节我们介绍了如何创建和查看加密文件，本节中我们来看看如何在Ansible Playbook中调用这些加密的变量。

首先，我们创建一个Playbook文件，例如 `tvault.yml`：

```yaml
---
- name: Create Users via Vault
  hosts: servers
  vars_files:
    - ./secret.yml
  tasks:
    - name: Create user
      user:
        name: "{{ user }}"
        state: present
        uid: "{{ uid }}"
```

![](img/7314385d90542d30a025304d1d1bf94f_39.png)

![](img/7314385d90542d30a025304d1d1bf94f_40.png)

![](img/7314385d90542d30a025304d1d1bf94f_41.png)

这个Playbook引用了我们之前加密的 `secret.yml` 文件。直接运行这个Playbook会失败，因为Ansible无法自动获取Vault的密码。

![](img/7314385d90542d30a025304d1d1bf94f_43.png)

![](img/7314385d90542d30a025304d1d1bf94f_44.png)

以下是三种为Playbook提供Vault密码的方法：

![](img/7314385d90542d30a025304d1d1bf94f_45.png)

![](img/7314385d90542d30a025304d1d1bf94f_46.png)

![](img/7314385d90542d30a025304d1d1bf94f_47.png)

**方法一：使用 `--ask-vault-pass` 参数**
运行Playbook时添加此参数，系统会提示你输入Vault密码。
```bash
ansible-playbook tvault.yml --ask-vault-pass
```

![](img/7314385d90542d30a025304d1d1bf94f_48.png)

![](img/7314385d90542d30a025304d1d1bf94f_49.png)

![](img/7314385d90542d30a025304d1d1bf94f_50.png)

**方法二：使用 `--vault-id` 参数（推荐）**
这是当前版本推荐的方法。通过此参数指定密码提示符。
```bash
ansible-playbook tvault.yml --vault-id @prompt
```

![](img/7314385d90542d30a025304d1d1bf94f_51.png)

![](img/7314385d90542d30a025304d1d1bf94f_52.png)

**方法三：使用密码文件**
创建一个纯文本文件（例如 `pass.yml`）来存储Vault密码。
```bash
echo ‘redhat‘ > pass.yml
```
然后运行Playbook时指定该密码文件。
```bash
ansible-playbook tvault.yml --vault-password-file=pass.yml
```
**注意**：此文件包含明文密码，务必采取安全措施，例如将其设置为隐藏文件（文件名以 `.` 开头）。

![](img/7314385d90542d30a025304d1d1bf94f_54.png)

![](img/7314385d90542d30a025304d1d1bf94f_56.png)

## 其他Vault操作

![](img/7314385d90542d30a025304d1d1bf94f_58.png)

![](img/7314385d90542d30a025304d1d1bf94f_60.png)

掌握了创建和调用后，我们再来看看Ansible Vault的其他管理功能。

![](img/7314385d90542d30a025304d1d1bf94f_62.png)

![](img/7314385d90542d30a025304d1d1bf94f_63.png)

**解密文件**
使用 `decrypt` 命令可以将加密文件永久解密为明文。
```bash
ansible-vault decrypt secret.yml
```

![](img/7314385d90542d30a025304d1d1bf94f_65.png)

![](img/7314385d90542d30a025304d1d1bf94f_66.png)

**重新加密文件**
使用 `encrypt` 命令可以对已解密的文件或普通文件进行加密。
```bash
ansible-vault encrypt secret.yml
```

![](img/7314385d90542d30a025304d1d1bf94f_68.png)

![](img/7314385d90542d30a025304d1d1bf94f_70.png)

**更改加密密码**
使用 `rekey` 命令可以修改已加密文件的密码。执行时会要求先输入当前密码，再输入并确认新密码。
```bash
ansible-vault rekey secret.yml
```

![](img/7314385d90542d30a025304d1d1bf94f_72.png)

**编辑加密文件**
不能直接用文本编辑器打开加密文件。需要使用 `edit` 命令，它会要求输入密码，然后在解密状态下打开编辑器供你修改，保存后会自动重新加密。
```bash
ansible-vault edit secret.yml
```

![](img/7314385d90542d30a025304d1d1bf94f_73.png)

![](img/7314385d90542d30a025304d1d1bf94f_75.png)

![](img/7314385d90542d30a025304d1d1bf94f_76.png)

## 安全建议

![](img/7314385d90542d30a025304d1d1bf94f_78.png)

![](img/7314385d90542d30a025304d1d1bf94f_80.png)

通常，使用加密文件来保存敏感变量是最佳实践。为了进一步增强安全性，建议将加密文件设置为隐藏文件（在文件名前加一个点，例如 `.secret.yml`），并在Playbook中相应更新引用路径。虽然这并非绝对安全，但能增加一层基础防护。

## 总结

![](img/7314385d90542d30a025304d1d1bf94f_87.png)

![](img/7314385d90542d30a025304d1d1bf94f_88.png)

本节课中我们一起学习了Ansible Vault的核心功能。我们了解了它的用途是加密敏感数据，学习了如何**创建**、**查看**、**编辑**、**解密**和**更改密码**（`rekey`）加密文件。重点是掌握了在Playbook中调用加密文件的三种方法：交互式输入（`--ask-vault-pass`）、通过提示符（`--vault-id @prompt`）以及使用密码文件（`--vault-password-file`）。最后，我们还讨论了将加密文件设置为隐藏文件的安全最佳实践。掌握这些技能，你就能安全地在Ansible中管理密码和密钥等机密信息了。
