## 用户账号概述

![](img/0d8dc47b97799e89ece894f64757702c_7.png)

![](img/0d8dc47b97799e89ece894f64757702c_9.png)

![](img/0d8dc47b97799e89ece894f64757702c_11.png)

用户账号是登录系统的基础。系统中有两类用户：超级管理员`root`和普通用户。`root`用户权限最大，但企业环境中通常只授权给核心运维人员，普通员工则使用权限受限的普通用户账号，以保障系统安全。

## 用户模板目录

在深入用户管理之前，我们先了解一个概念：用户模板目录`/etc/skel`。这个目录存放着创建新用户时，会自动复制到其家目录下的默认配置文件（如`.bashrc`、`.bash_profile`等）。这些文件为用户提供了基础的Shell环境配置。

```
ls -la /etc/skel/
```

## 创建用户：`useradd`命令

![](img/0d8dc47b97799e89ece894f64757702c_17.png)

![](img/0d8dc47b97799e89ece894f64757702c_19.png)

![](img/0d8dc47b97799e89ece894f64757702c_21.png)

创建用户的命令是`useradd`，只有`root`用户有权执行。其基本语法是`useradd [选项] 用户名`。

![](img/0d8dc47b97799e89ece894f64757702c_23.png)

```
useradd user1
```

上述命令创建了一个名为`user1`的用户。创建后，用户信息被记录在系统文件中，但此时用户尚未设置密码，无法登录。

## 用户信息文件：`/etc/passwd`详解

用户创建后，其核心信息存储在`/etc/passwd`文件中。这个文件至关重要，删除它会导致所有用户（包括`root`）无法登录。

![](img/0d8dc47b97799e89ece894f64757702c_25.png)

![](img/0d8dc47b97799e89ece894f64757702c_27.png)

```
cat /etc/passwd
```

文件中的每一行代表一个用户，以冒号`:`分隔为7个字段。以下是每个字段的含义：

1.  **用户名**：用户登录时使用的名称。
2.  **密码占位符**：历史上用于存放加密密码，现在密码已移至`/etc/shadow`文件，此处统一用`x`表示。
3.  **用户ID (UID)**：用户的唯一数字标识。
    *   `UID=0`：超级管理员 (`root`)。
    *   `UID=1-999`：系统伪用户，供系统进程使用，不能登录。
    *   `UID>=1000`：普通用户。
4.  **组ID (GID)**：用户所属**基本组（初始组）**的ID。创建用户时会自动生成一个与用户名同名的组作为其基本组。
5.  **描述信息 (GECOS)**：用户的备注信息，如全名、电话等，可为空。
6.  **家目录**：用户登录后的默认工作目录。普通用户通常在`/home/`下。
7.  **登录Shell**：用户登录后使用的命令解释器。默认为`/bin/bash`。如果设置为`/sbin/nologin`，则该用户无法登录系统。

## `useradd`命令常用选项

![](img/0d8dc47b97799e89ece894f64757702c_33.png)

`useradd`命令提供了丰富的选项来定制用户属性。以下是几个常用选项：

![](img/0d8dc47b97799e89ece894f64757702c_35.png)

![](img/0d8dc47b97799e89ece894f64757702c_37.png)

*   `-u UID`：指定用户的UID。
    ```
    useradd -u 6666 user2
    ```
*   `-d 目录`：指定用户的家目录。
    ```
    useradd -d /opt/user3 user3
    ```
*   `-c “备注”`：添加用户的描述信息。
    ```
    useradd -c “运维部，电话123” user4
    ```
*   `-G 组名1,组名2,...`：将用户加入到指定的**附加组**。
    ```
    groupadd devops
    useradd -G devops user5
    ```
*   `-s Shell路径`：指定用户的登录Shell。
    ```
    useradd -s /sbin/nologin user6 # 创建不能登录的用户
    ```

这些选项可以组合使用，一次性完成复杂的用户创建。

## 查看用户和组信息：`id`命令

`id`命令用于查看用户及其所属组的信息。

```
id user1
```
输出会显示用户的`UID`、`GID`以及所属的所有组（基本组和附加组）。
