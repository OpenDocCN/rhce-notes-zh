## 磁盘分区工具概述

![](img/04da126c1733b82647b80a39b174f132_8.png)

![](img/04da126c1733b82647b80a39b174f132_10.png)

上一节我们介绍了课程安排，本节中我们来看看Linux中常用的磁盘分区工具。主要有两种分区方案及其对应的工具。

![](img/04da126c1733b82647b80a39b174f132_11.png)

![](img/04da126c1733b82647b80a39b174f132_13.png)

![](img/04da126c1733b82647b80a39b174f132_15.png)

![](img/04da126c1733b82647b80a39b174f132_16.png)

*   **MBR (Master Boot Record) 分区方案**：使用 `fdisk` 工具。
    *   单个分区最大支持 **2TB**。
    *   分区数量有限制，最多 **4个主分区**，或 **3个主分区 + 1个扩展分区**（扩展分区内可创建多个逻辑分区）。
    *   在Linux中，其分区类型常被称为 `msdos`。
*   **GPT (GUID Partition Table) 分区方案**：使用 `gdisk` 或 `parted` 工具。
    *   支持 **2TB以上** 的单个分区。
    *   没有主分区、扩展分区的概念，最多支持 **128个分区**。
    *   使用全局唯一标识符 (GUID) 来标识分区。

对于初学者，我们首先掌握 `fdisk` 工具的基本操作。

现在，我们以新添加的磁盘（假设为 `/dev/sdb`）为例，演示从创建分区到挂载使用的完整过程。

![](img/04da126c1733b82647b80a39b174f132_45.png)

### 1. 使用 `fdisk` 创建分区

![](img/04da126c1733b82647b80a39b174f132_47.png)

![](img/04da126c1733b82647b80a39b174f132_48.png)

首先，我们使用 `fdisk` 工具对磁盘进行分区。

![](img/04da126c1733b82647b80a39b174f132_50.png)

```bash
fdisk /dev/sdb
```
进入交互界面后，可以输入 `m` 查看帮助。常用命令如下：
*   `n`：创建新分区。
*   `p`：打印当前分区表。
*   `t`：更改分区类型。
*   `w`：保存并退出。
*   `q`：不保存退出。

![](img/04da126c1733b82647b80a39b174f132_52.png)

![](img/04da126c1733b82647b80a39b174f132_54.png)

![](img/04da126c1733b82647b80a39b174f132_56.png)

![](img/04da126c1733b82647b80a39b174f132_58.png)

创建主分区的步骤如下：
1.  输入 `n` 开始创建新分区。
2.  选择分区类型 `p` (主分区)。
3.  设置分区号（通常默认即可）。
4.  设置起始扇区（通常默认即可）。
5.  设置分区大小，例如 `+2G` 表示创建2GB的分区。
6.  输入 `p` 确认分区信息。
7.  输入 `w` 将更改写入磁盘。

### 2. 让内核识别新分区

![](img/04da126c1733b82647b80a39b174f132_60.png)

创建分区后，需要让系统内核重新读取分区表信息。

```bash
partprobe /dev/sdb
```
然后使用 `lsblk` 命令确认新分区（如 `/dev/sdb1`）已出现。

![](img/04da126c1733b82647b80a39b174f132_62.png)

![](img/04da126c1733b82647b80a39b174f132_64.png)

![](img/04da126c1733b82647b80a39b174f132_66.png)

### 3. 格式化分区（创建文件系统）

![](img/04da126c1733b82647b80a39b174f132_68.png)

分区创建后，需要格式化为特定的文件系统才能存储数据。RHEL 8 默认使用 `xfs` 文件系统。

![](img/04da126c1733b82647b80a39b174f132_70.png)

```bash
mkfs.xfs /dev/sdb1
```
或者使用 `-t` 选项指定文件系统类型：
```bash
mkfs -t xfs /dev/sdb1
```

![](img/04da126c1733b82647b80a39b174f132_72.png)

![](img/04da126c1733b82647b80a39b174f132_73.png)

![](img/04da126c1733b82647b80a39b174f132_75.png)

### 4. 挂载分区

![](img/04da126c1733b82647b80a39b174f132_77.png)

格式化后，需要将分区挂载到目录树中的某个位置（挂载点）才能访问。

![](img/04da126c1733b82647b80a39b174f132_79.png)

**临时挂载**（重启后失效）：
```bash
mkdir /mnt/data1
mount /dev/sdb1 /mnt/data1
```
使用 `df -Th` 命令可以查看挂载情况。

![](img/04da126c1733b82647b80a39b174f132_81.png)

![](img/04da126c1733b82647b80a39b174f132_83.png)

![](img/04da126c1733b82647b80a39b174f132_85.png)

**永久挂载**（需编辑配置文件）：
1.  获取分区的UUID（推荐使用UUID，比设备名更稳定）：
    ```bash
    blkid /dev/sdb1
    ```
2.  编辑 `/etc/fstab` 文件，在末尾添加一行：
    ```
    UUID=你的分区UUID /mnt/data1 xfs defaults 0 0
    ```
    *   第一列：设备标识（`UUID=` 或 `/dev/sdb1`）。
    *   第二列：挂载点目录。
    *   第三列：文件系统类型。
    *   第四列：挂载选项（`defaults` 是常用选项）。
    *   第五列：是否被 `dump` 备份（`0` 表示否）。
    *   第六列：开机时是否用 `fsck` 检查磁盘顺序（`0` 表示否）。
3.  保存文件后，测试挂载配置是否正确：
    ```bash
    mount -a
    ```
    如果无报错，则配置成功。再次使用 `df -Th` 确认。

## Linux磁盘设备命名规则

了解设备命名有助于识别磁盘类型。设备文件通常位于 `/dev/` 目录下。

![](img/04da126c1733b82647b80a39b174f132_91.png)

以下是常见命名规则：
*   `/dev/sd[a-z]`：SCSI、SATA、USB接口的磁盘或U盘。`sd` 后的字母代表磁盘顺序，数字代表分区号（如 `sda1`）。
*   `/dev/vd[a-z]`：虚拟化环境（如KVM）中的虚拟磁盘。
*   `/dev/nvme[0-9]n[1-9]`：NVMe协议的高速固态硬盘。`nvme0n1` 代表第一个NVMe磁盘的第一个命名空间。
*   `/dev/sr0` 或 `/dev/cdrom`：光驱设备。

## 总结

本节课中我们一起学习了Linux磁盘管理的基础操作。我们了解了MBR和GPT两种分区方案的区别，并重点演练了使用 `fdisk` 工具进行分区创建、使用 `partprobe` 通知内核、使用 `mkfs` 格式化分区以及通过 `mount` 命令和 `/etc/fstab` 文件进行挂载的完整流程。掌握这些步骤是进行更高级存储管理（如LVM）的前提。
