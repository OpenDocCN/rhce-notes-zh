Linux入门与红帽认证RHCE通关教程：P82：修改SSH端口及排错指南

在本节课中，我们将学习如何修改Linux系统中SSH服务的默认端口，并解决在此过程中可能遇到的两个主要障碍：SELinux安全模块和防火墙规则。

### **修改SSH配置文件**

![](img/3d8c5904c3fb7f0dae24975b36c51781_2.png)

首先，我们需要编辑SSH服务的主配置文件来更改默认端口。SSH的默认端口是22，出于安全考虑，我们可以将其修改为1到65535之间的任意端口，例如2222。

![](img/3d8c5904c3fb7f0dae24975b36c51781_3.png)

![](img/3d8c5904c3fb7f0dae24975b36c51781_4.png)

1.  使用文本编辑器打开SSH主配置文件：
    ```bash
    vi /etc/ssh/sshd_config
    ```

![](img/3d8c5904c3fb7f0dae24975b36c51781_5.png)

![](img/3d8c5904c3fb7f0dae24975b36c51781_6.png)

2.  在文件中找到关于端口（Port）的配置行。请注意，配置项区分大小写。
3.  移除该行开头的注释符号 `#`，并将端口号修改为目标值，例如：
    ```
    Port 2222
    ```
4.  保存并退出编辑器。

![](img/3d8c5904c3fb7f0dae24975b36c51781_7.png)

至此，配置文件修改完成。

![](img/3d8c5904c3fb7f0dae24975b36c51781_8.png)

### **重启服务与排错：SELinux**

上一节我们修改了配置文件，本节中我们来看看如何重启服务并解决第一个常见问题。

![](img/3d8c5904c3fb7f0dae24975b36c51781_12.png)

修改配置后，需要重启SSH服务以使更改生效。尝试执行重启命令时，可能会遇到第一个障碍，系统提示启动失败。

![](img/3d8c5904c3fb7f0dae24975b36c51781_13.png)

**问题根源**：这通常是由于SELinux（安全增强Linux）的安全策略限制所导致。SELinux会阻止服务使用非标准端口。

![](img/3d8c5904c3fb7f0dae24975b36c51781_14.png)

**解决方案**：根据实际需求，可以选择临时或永久调整SELinux策略。在生产或考试环境中，可能需要更精细地配置SELinux策略。在本教程的演示环境中，我们将选择关闭SELinux。

以下是操作步骤：

1.  **临时关闭SELinux（重启后失效）**：
    执行以下命令将SELinux设置为宽容模式，仅记录违规而不阻止：
    ```bash
    setenforce 0
    ```
    执行此命令后，通常可以成功重启SSH服务。

2.  **永久关闭SELinux**：
    若要永久禁用SELinux，需要编辑其配置文件。
    *   打开SELinux配置文件：
        ```bash
        vi /etc/selinux/config
        ```
    *   找到 `SELINUX=` 这一行。它通常有三个选项：
        *   `enforcing`：强制模式（默认），阻止并记录违规。
        *   `permissive`：宽容模式，仅记录违规。
        *   `disabled`：禁用模式，完全不加载SELinux策略。
    *   将值修改为 `disabled`：
        ```
        SELINUX=disabled
        ```
    *   保存并退出。此更改将在系统下次重启后生效。

![](img/3d8c5904c3fb7f0dae24975b36c51781_16.png)

解决SELinux问题后，SSH服务应能成功重启并监听新的端口（如2222）。

![](img/3d8c5904c3fb7f0dae24975b36c51781_17.png)

### **连接测试与排错：防火墙**

![](img/3d8c5904c3fb7f0dae24975b36c51781_20.png)

服务重启成功后，我们尝试从客户端使用新端口进行连接。假设服务器IP为 `192.168.1.20`。

![](img/3d8c5904c3fb7f0dae24975b36c51781_21.png)

![](img/3d8c5904c3fb7f0dae24975b36c51781_22.png)

使用 `ssh` 命令并指定端口参数 `-p` 进行连接：
```bash
ssh username@192.168.1.20 -p 2222
```
此时，可能会遇到第二个障碍：连接超时或被拒绝。

![](img/3d8c5904c3fb7f0dae24975b36c51781_23.png)

**问题根源**：系统的防火墙（如firewalld）没有开放我们新配置的SSH端口（2222）。防火墙默认只允许少数已知服务端口（如22）的流量通过。

**解决方案**：我们需要在防火墙规则中永久添加一条规则，放行TCP协议在2222端口的流量。

![](img/3d8c5904c3fb7f0dae24975b36c51781_25.png)

![](img/3d8c5904c3fb7f0dae24975b36c51781_26.png)

以下是使用 `firewall-cmd` 管理工具的操作步骤：

1.  **添加永久规则**：
    执行以下命令，添加一条允许TCP协议访问2222端口的永久规则。
    ```bash
    firewall-cmd --permanent --add-port=2222/tcp
    ```
    参数说明：
    *   `--permanent`：表示规则永久生效。
    *   `--add-port=2222/tcp`：指定添加的端口和协议。

2.  **重载防火墙配置**：
    添加永久规则后，需要重载防火墙才能使新规则立即生效，而无需重启防火墙服务。
    ```bash
    firewall-cmd --reload
    ```

![](img/3d8c5904c3fb7f0dae24975b36c51781_28.png)

完成上述步骤后，再次尝试使用 `ssh -p 2222` 命令进行连接，即可成功登录服务器。

![](img/3d8c5904c3fb7f0dae24975b36c51781_29.png)

![](img/3d8c5904c3fb7f0dae24975b36c51781_30.png)

### **总结**

![](img/3d8c5904c3fb7f0dae24975b36c51781_32.png)

本节课中我们一起学习了修改SSH服务端口的完整流程及排错方法。

![](img/3d8c5904c3fb7f0dae24975b36c51781_34.png)

1.  **核心步骤**是编辑 `/etc/ssh/sshd_config` 文件，修改 `Port` 配置项并重启 `sshd` 服务。
2.  **主要障碍**通常有两个：
    *   **SELinux限制**：可通过将其设置为宽容模式（`setenforce 0`）或禁用（修改 `/etc/selinux/config`）来解决。
    *   **防火墙阻拦**：需要使用防火墙工具（如 `firewall-cmd --permanent --add-port=端口号/tcp` 并 `--reload`）开放对应的端口。

![](img/3d8c5904c3fb7f0dae24975b36c51781_35.png)

请注意，在正式生产环境中，直接关闭SELinux和防火墙并非最佳安全实践。更推荐的做法是：为SELinux配置正确的端口上下文，并在防火墙中精确开放所需的最小端口范围。本教程的方法主要用于快速理解和通过实验环境测试。
