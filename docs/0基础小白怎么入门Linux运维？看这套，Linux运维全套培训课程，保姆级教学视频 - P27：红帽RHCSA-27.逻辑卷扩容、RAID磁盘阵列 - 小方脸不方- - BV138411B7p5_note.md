![](img/9b59b31b71b755714753fd60db6a7241_1.png)

在本节课中，我们将深入学习逻辑卷管理（LVM）的核心操作——扩容，并初步了解RAID磁盘阵列的基本概念。逻辑卷的灵活扩容是其最重要的特性，而RAID则为数据安全提供了另一层保障。

## 逻辑卷扩容详解 💾

![](img/9b59b31b71b755714753fd60db6a7241_3.png)

### 为何需要逻辑卷扩容？

![](img/9b59b31b71b755714753fd60db6a7241_15.png)

![](img/9b59b31b71b755714753fd60db6a7241_17.png)

对于服务器上的数据，你很难在最初划分分区时就准确预估其未来的增长量。例如根分区，随着服务器不断运行，其所需空间难以预测。因此，使用逻辑卷存储数据的核心优势在于其**可扩容性**。你无需在初始阶段分配过大的空间，当空间不足时，直接对其进行扩容即可。

### 扩容前检查

在扩容逻辑卷之前，必须首先确认其所属的卷组（VG）是否有足够的剩余空间。逻辑卷的扩容空间完全依赖于卷组的可用空间。

以下是检查命令：
```bash
# 查看卷组信息及剩余空间
vgs
# 查看逻辑卷当前大小
lvs
```

### 扩容逻辑卷空间

![](img/9b59b31b71b755714753fd60db6a7241_19.png)

确认卷组有足够空间后，即可对逻辑卷进行扩容。扩容操作支持**在线进行**，无需卸载文件系统或重启服务器。

扩展逻辑卷的命令格式如下：
```bash
lvresize -L +[要增加的大小] [逻辑卷设备路径]
```
例如，为逻辑卷 `/dev/system_vg/my_lv` 增加 19GB 空间：
```bash
lvresize -L +19G /dev/system_vg/my_lv
```
命令执行后，逻辑卷的**物理边界**会扩大，但此时文件系统还无法使用新增的空间。

![](img/9b59b31b71b755714753fd60db6a7241_21.png)

### 扩展文件系统

![](img/9b59b31b71b755714753fd60db6a7241_23.png)

![](img/9b59b31b71b755714753fd60db6a7241_25.png)

这是扩容过程中**最关键的一步**。逻辑卷空间扩大后，其上的文件系统并不会自动扩展。新增的空间没有文件系统，操作系统无法识别和管理。

因此，需要根据文件系统类型，执行相应的扩展命令：
*   对于 **XFS** 文件系统，使用 `xfs_growfs`。
*   对于 **ext2/3/4** 文件系统，使用 `resize2fs`。

![](img/9b59b31b71b755714753fd60db6a7241_27.png)

![](img/9b59b31b71b755714753fd60db6a7241_29.png)

由于我们之前格式化为XFS，因此执行：
```bash
# 指定挂载点扩展文件系统（更简便）
xfs_growfs /web_backup
# 或直接指定逻辑卷设备
xfs_growfs /dev/system_vg/my_lv
```
执行此命令后，使用 `df -h` 查看，逻辑卷的**可用空间**才会真正变大。

### 扩容卷组

![](img/9b59b31b71b755714753fd60db6a7241_31.png)

![](img/9b59b31b71b755714753fd60db6a7241_33.png)

![](img/9b59b31b71b755714753fd60db6a7241_35.png)

如果卷组空间已耗尽，需要先对卷组进行扩容，然后才能继续扩容逻辑卷。

![](img/9b59b31b71b755714753fd60db6a7241_37.png)

![](img/9b59b31b71b755714753fd60db6a7241_39.png)

卷组扩容的命令是 `vgextend`，可以将新的物理卷（PV，可以是整块硬盘或分区）加入现有卷组。
```bash
# 将新的硬盘/dev/sdd加入卷组system_vg
vgextend system_vg /dev/sdd
```
添加成功后，使用 `vgs` 查看卷组空间变大，之后便可以继续按上述步骤扩容逻辑卷。

### 逻辑卷缩容与删除 ⚠️

**缩容（减小空间）** 在理论上是可行的，但**风险极高**，企业环境中极少使用。因为缩容需要先缩减文件系统，此过程极易导致数据丢失。若必须操作，需先卸载文件系统，流程复杂且危险。

![](img/9b59b31b71b755714753fd60db6a7241_41.png)

**删除逻辑卷** 的操作同样需要谨慎，并遵循固定顺序：
1.  **卸载** 逻辑卷的挂载点：`umount /web_backup`
2.  **删除** 逻辑卷：`lvremove /dev/system_vg/my_lv`
3.  **删除** 卷组（如果不再需要）：`vgremove system_vg`
4.  **删除** 物理卷属性（如需将磁盘恢复为普通分区）：`pvremove /dev/sdb1`

![](img/9b59b31b71b755714753fd60db6a7241_43.png)

![](img/9b59b31b71b755714753fd60db6a7241_45.png)

删除后，原有的分区或硬盘即可重新格式化作为普通存储设备使用。

![](img/9b59b31b71b755714753fd60db6a7241_47.png)

## RAID磁盘阵列初探 🛡️

在了解了逻辑卷提供的灵活存储管理后，我们来看看如何从硬件层面提升数据的可靠性和性能，这就是RAID（独立磁盘冗余阵列）技术。

### RAID是什么？

![](img/9b59b31b71b755714753fd60db6a7241_49.png)

![](img/9b59b31b71b755714753fd60db6a7241_51.png)

RAID技术通过将多块独立的物理硬盘以特定方式组合起来，形成一个逻辑上的大容量硬盘。其主要目的有两个：
1.  **提升数据可靠性（冗余）**：即使部分硬盘损坏，数据也不会丢失。
2.  **提升读写性能**：通过并行读写操作加快速度。

![](img/9b59b31b71b755714753fd60db6a7241_53.png)

![](img/9b59b31b71b755714753fd60db6a7241_55.png)

### 常见RAID级别简介

![](img/9b59b31b71b755714753fd60db6a7241_57.png)

![](img/9b59b31b71b755714753fd60db6a7241_58.png)

RAID根据组合方式的不同分为多个级别，以下是两种基础且重要的级别：

![](img/9b59b31b71b755714753fd60db6a7241_60.png)

![](img/9b59b31b71b755714753fd60db6a7241_62.png)

![](img/9b59b31b71b755714753fd60db6a7241_64.png)

![](img/9b59b31b71b755714753fd60db6a7241_66.png)

*   **RAID 0（条带化）**：
    *   **原理**：将数据拆分后，同时写入多块硬盘。
    *   **优点**：读写速度最快，容量利用率100%（总容量为所有硬盘之和）。
    *   **缺点**：**无冗余**，任何一块硬盘损坏，所有数据都会丢失。
    *   **适用场景**：对性能要求极高，但对数据安全要求不高的场景，如视频缓存。

*   **RAID 1（镜像）**：
    *   **原理**：将相同的数据同时写入两块硬盘，实现完全备份。
    *   **优点**：数据安全性高，一块硬盘损坏，另一块保有完整数据。
    *   **缺点**：容量利用率只有50%（总容量为单块硬盘容量），写入速度无提升。
    *   **适用场景**：对数据安全要求极高的场景，如操作系统盘、数据库日志。

### RAID与LVM的结合

在实际的高可用存储方案中，RAID和LVM常常结合使用，形成多层存储架构：
1.  底层使用多块硬盘组建RAID（如RAID 1或RAID 5），提供硬件级的**数据冗余保护**。
2.  将RAID阵列形成的逻辑盘作为物理卷（PV），加入卷组（VG）。
3.  在卷组上创建逻辑卷（LV），利用LVM实现存储空间的**灵活分配与在线扩容**。

这种组合既能保障数据安全，又能提供极佳的存储管理灵活性。
