## sed工作原理与基本选项

![](img/284624387c77a1d86226e42aec60b69b_5.png)

![](img/284624387c77a1d86226e42aec60b69b_7.png)

上一节我们介绍了多种文本处理命令，本节中我们来看看 `sed` 的核心机制。`sed` 是一种流编辑器，其工作原理是：逐行读取文件内容至一个称为“模式空间”的缓存区，所有编辑操作都在此缓存中进行。默认情况下，`sed` 只将修改后的结果输出到终端，而**不改变原文件**。若需直接修改原文件，必须使用 `-i` 选项。

![](img/284624387c77a1d86226e42aec60b69b_9.png)

以下是 `sed` 的几个关键选项：

![](img/284624387c77a1d86226e42aec60b69b_10.png)

*   **`-n`**：静默模式。禁止自动打印模式空间的内容，常与打印命令 `p` 配合使用，仅输出处理过的行。
*   **`-i`**：直接编辑文件。使用此选项后，`sed` 会将修改写回原文件。**操作前建议备份**。
*   **`-e`**：连接多个编辑命令。可以在一个 `sed` 命令中执行多项操作。
*   **`-r`**：启用扩展正则表达式。使 `sed` 支持更简洁强大的正则语法（如 `+`, `?`, `|`）。

![](img/284624387c77a1d86226e42aec60b69b_12.png)

![](img/284624387c77a1d86226e42aec60b69b_14.png)

例如，仅打印文件 `file.txt` 的第5行，需要使用 `-n` 选项来抑制默认输出：
```bash
sed -n '5p' file.txt
```

![](img/284624387c77a1d86226e42aec60b69b_16.png)

## 地址定位：指定要处理的行

![](img/284624387c77a1d86226e42aec60b69b_18.png)

`sed` 的强大之处在于能精确指定要操作的行。如果不指定地址，则默认对全文所有行进行操作。

![](img/284624387c77a1d86226e42aec60b69b_20.png)

以下是几种主要的地址指定方法：

![](img/284624387c77a1d86226e42aec60b69b_22.png)

![](img/284624387c77a1d86226e42aec60b69b_23.png)

*   **数字地址**：直接指定行号。例如，`5` 表示第5行。
*   **最后一行**：使用美元符号 `$` 表示文件的最后一行。
*   **正则表达式地址**：使用 `/正则表达式/` 的格式，匹配包含该模式的行。这继承了之前 `grep` 课程中学到的所有正则知识。
*   **地址范围**：使用逗号 `,` 分隔两个地址，表示从第一个地址匹配的行到第二个地址匹配的行之间的范围。

![](img/284624387c77a1d86226e42aec60b69b_25.png)

![](img/284624387c77a1d86226e42aec60b69b_27.png)

让我们通过打印命令 `p` 来举例说明。打印以 `root` 开头的行：
```bash
sed -n '/^root/p' /etc/passwd
```
打印第10行到第20行：
```bash
sed -n '10,20p' /etc/passwd
```

![](img/284624387c77a1d86226e42aec60b69b_29.png)

## 正则表达式的应用与高级定界符

在 `sed` 中，正则表达式的用法与 `grep` 高度一致。你可以使用之前学过的所有元字符，如 `^`（行首）、`$`（行尾）、`[]`（字符组）等。

![](img/284624387c77a1d86226e42aec60b69b_31.png)

一个常见的需求是匹配包含特定路径（含有斜杠 `/`）的行。如果直接使用 `/bin/bash/` 作为地址，斜杠会与地址定界符冲突。此时有两种解决方案：

![](img/284624387c77a1d86226e42aec60b69b_33.png)

1.  **使用反斜杠转义**：在每个字面意义的斜杠前加反斜杠 `\`。
    ```bash
    sed -n '/\/bin\/bash/p' /etc/passwd
    ```

![](img/284624387c77a1d86226e42aec60b69b_35.png)

2.  **使用其他定界符**：`sed` 允许使用除反斜杠外的任意字符作为地址定界符，常用 `#` 或 `|`。这能极大地提高命令的可读性。
    ```bash
    sed -n '\#/bin/bash#p' /etc/passwd
    ```

![](img/284624387c77a1d86226e42aec60b69b_37.png)

此外，在正则表达式后添加 `I` 可以**忽略大小写**进行匹配。例如，匹配包含 “root” 或 “ROOT” 的行：
```bash
sed -n '/root/Ip' /etc/passwd
```
