# Linux运维RHCSA：P20：常用特殊符号与grep文件内容过滤 📚

![](img/7e6705974e860a37974be51768921c31_1.png)

在本节课中，我们将要学习Linux系统中几个常用的特殊符号，以及如何使用`grep`命令来过滤文件内容。这些工具在日常系统管理和文件操作中非常实用，能够极大地提高工作效率。

## 常用特殊符号 🔣

![](img/7e6705974e860a37974be51768921c31_3.png)

![](img/7e6705974e860a37974be51768921c31_5.png)

![](img/7e6705974e860a37974be51768921c31_6.png)

在Linux系统下，特殊符号可以完成一些特殊的功能。键盘上的许多符号，如叹号、井号、星号、大括号等都属于特殊符号。每种符号在系统中都有特定的含义。在初级阶段，我们主要学习以下四个符号：星号、大括号、问号和中括号。其中，星号和大括号是最常用、必须掌握的。

### 星号 (*)

星号是一个通配符，应用在文件名上，用来代表**任意多个任意字符**。

*   **任意字符**：可以代表任何字符，如数字、字母（大小写）、点等。
*   **任意多个**：代表字符的长度不限。

**主要用途**：常用于文件搜索和批量操作。

![](img/7e6705974e860a37974be51768921c31_8.png)

![](img/7e6705974e860a37974be51768921c31_10.png)

以下是星号的一些应用示例：

*   搜索`/etc`目录下所有以`pass`开头的文件：
    ```bash
    ls /etc/pass*
    ```
*   搜索`/var/log`目录下所有以`.log`结尾的日志文件：
    ```bash
    ls /var/log/*.log
    ```
*   将`/var/log`目录下所有`.log`文件备份到`/opt`目录：
    ```bash
    cp -p /var/log/*.log /opt
    ```
*   星号可以灵活地放在文件名的开头、中间或结尾进行匹配。

![](img/7e6705974e860a37974be51768921c31_12.png)

### 问号 (?)

问号同样应用在文件名上，用来代表**任意单个字符**。

![](img/7e6705974e860a37974be51768921c31_14.png)

![](img/7e6705974e860a37974be51768921c31_15.png)

*   **任意字符**：可以代表任何字符。
*   **单个**：只能精确匹配一个字符的位置。

**主要用途**：用于需要精确匹配单个未知字符的场景，但使用频率相对较低。

![](img/7e6705974e860a37974be51768921c31_17.png)

![](img/7e6705974e860a37974be51768921c31_19.png)

![](img/7e6705974e860a37974be51768921c31_21.png)

以下是问号的一些应用示例：

*   搜索`/dev`目录下以`tty`开头且结尾只有一个字符的文件：
    ```bash
    ls /dev/tty?
    ```
*   搜索`/dev`目录下以`tty`开头且结尾有两个字符的文件：
    ```bash
    ls /dev/tty??
    ```

### 中括号 ([])

![](img/7e6705974e860a37974be51768921c31_23.png)

![](img/7e6705974e860a37974be51768921c31_24.png)

中括号应用在文件名上，用来匹配**指定范围内的一个字符**。

*   **指定范围**：通常用于匹配连续的数字或字母范围。
*   **单个字符**：匹配括号内定义的集合中的任意一个字符。

**主要用途**：用于匹配具有特定字符特征的文件名。

以下是中括号的一些应用示例：

*   搜索`/var/log`目录下文件名中包含数字（0-9）的文件：
    ```bash
    ls /var/log/*[0-9]*
    ```
*   搜索`/var/log`目录下文件名中包含字母（a-z）的文件：
    ```bash
    ls /var/log/*[a-z]*
    ```
*   搜索`/dev`目录下以`tty`开头，且结尾是0到9之间数字的文件：
    ```bash
    ls /dev/tty[0-9]
    ```
    *注意：中括号对于不连续的数字范围（如`[15-25]`）匹配效果不佳。*

### 大括号 ({})

![](img/7e6705974e860a37974be51768921c31_26.png)

大括号功能强大，应用在文件名上，主要用于生成序列或代表**多组不同的字符串**，尤其擅长处理数字序列。

![](img/7e6705974e860a37974be51768921c31_27.png)

**主要用途**：批量创建、删除或匹配具有序列特征的文件。

![](img/7e6705974e860a37974be51768921c31_29.png)

![](img/7e6705974e860a37974be51768921c31_31.png)

以下是大括号的一些应用示例：

![](img/7e6705974e860a37974be51768921c31_33.png)

*   在`/opt`目录下快速创建100个名为`test1`到`test100`的文件：
    ```bash
    touch /opt/test{1..100}
    ```
*   删除`/opt`目录下`test51`到`test60`的文件：
    ```bash
    rm -rf /opt/test{51..60}
    ```
*   匹配`/dev`目录下以`tty`开头，结尾为15到25之间数字的文件（效果优于中括号）：
    ```bash
    ls /dev/tty{15..25}
    ```
*   匹配文件名中包含`a`、`b`、`c`或`d`的文件：
    ```bash
    ls /etc/*[a,b,c,d]*
    ```

![](img/7e6705974e860a37974be51768921c31_35.png)

![](img/7e6705974e860a37974be51768921c31_37.png)

上一节我们介绍了几个用于文件名匹配的特殊符号，本节中我们来看看如何深入文件内部，查找特定的文本内容。

## grep文件内容过滤 🔍

`grep`命令用于在文件中查找并显示包含指定字符串的行。当你需要从一个大型文件中快速定位特定信息，而不想逐行阅读时，`grep`非常有用。

**基本语法**：
```bash
grep [选项] “查找字符串” 文件名
```

### 基本用法

*   在`/etc/passwd`文件中查找包含“tom”的行：
    ```bash
    grep “tom” /etc/passwd
    ```
*   显示匹配行所在的行号（使用`-n`选项）：
    ```bash
    grep -n “tom” /etc/passwd
    ```
*   查找包含“laowang”的行：
    ```bash
    grep “laowang” /etc/passwd
    ```

### 结合管道使用

`grep`常与其他命令通过管道(`|`)结合，对前一个命令的输出结果进行过滤。

*   查看网络配置，并只显示包含“ens33”的行：
    ```bash
    ifconfig | grep ens33
    ```
*   查看内存信息，并只显示包含“Mem”的行：
    ```bash
    free -h | grep Mem
    ```

### 常用选项

`grep`命令有一些实用的选项来增强其功能。

以下是`grep`命令的几个常用选项：

*   **`-i`**：忽略大小写进行匹配。
    ```bash
    grep -i “user” /etc/passwd # 会匹配“user”、“User”、“USER”等
    ```
*   **`-v`**：反向选择，即显示**不包含**匹配字符串的所有行。
    ```bash
    grep -v “root” /etc/passwd # 显示所有不包含“root”的行
    ```
*   **`-n`**：显示匹配行及其在文件中的行号。

### 结合正则表达式符号

`grep`可以结合简单的正则表达式符号进行更精确的匹配。

![](img/7e6705974e860a37974be51768921c31_39.png)

![](img/7e6705974e860a37974be51768921c31_41.png)

以下是几个常用的结合模式：

*   **`^字符串`**：匹配以特定字符串**开头**的行。
    ```bash
    grep “^root” /etc/passwd # 只匹配以“root”开头的行
    ```
*   **`字符串$`**：匹配以特定字符串**结尾**的行。
    ```bash
    grep “/bin/bash$” /etc/passwd # 匹配使用bash作为登录shell的用户
    grep “/sbin/nologin$” /etc/passwd # 匹配不可登录系统的用户
    ```
*   **`^$`**：匹配空行。
    ```bash
    grep “^$” /etc/login.defs | wc -l # 统计文件中的空行数
    ```

### 综合应用示例

![](img/7e6705974e860a37974be51768921c31_43.png)

一个常见的需求是查看配置文件中所有**非注释且非空**的有效配置行。

![](img/7e6705974e860a37974be51768921c31_45.png)

例如，查看`/etc/login.defs`文件的有效配置：
1.  首先排除所有以注释符`#`开头的行。
2.  再排除所有的空行。
3.  最后显示行号以便查看。

![](img/7e6705974e860a37974be51768921c31_47.png)

```bash
grep -v “^#” /etc/login.defs | grep -v “^$” | nl
```
或者使用`-n`选项并放在最后：
```bash
grep -v “^#” /etc/login.defs | grep -v “^$” | grep -n “.*”
```
这个组合命令能清晰地向我们展示文件的核心配置内容。
