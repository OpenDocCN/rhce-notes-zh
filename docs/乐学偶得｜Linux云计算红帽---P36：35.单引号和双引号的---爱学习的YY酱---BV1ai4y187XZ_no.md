在本节课中，我们将学习Linux命令行中单引号（`'`）和双引号（`"`）的核心区别。理解它们如何对待字符串中的变量，是编写Shell脚本和高效使用命令行的关键。

![](img/53378691d7d620c185f3a796a2817ed3_0.png)

## 引号的基本作用

在Shell中，`echo`命令用于在终端显示文本。当我们需要显示的文本内容很长，并且其中既包含普通字符串又包含变量时，就需要使用引号来界定这个文本块。

例如，我们想显示一句话：“当前正在使用这台电脑的用户是XXX”。这里的“XXX”我们希望是一个动态的、根据实际登录用户而变化的变量，而不是固定的文字。

## 双引号的作用

双引号允许我们在字符串中嵌入变量，Shell会解释这些变量并将其值替换到字符串中。

请看以下命令：
```bash
echo “Your user that is currently using this PC is $LOGNAME”
```
在这个例子中，`$LOGNAME`是一个环境变量，它代表当前登录系统的用户名。当命令执行时，Shell会找到`$LOGNAME`变量对应的值（例如“william”），并将其替换到字符串中。因此，输出结果会是：
```
Your user that is currently using this PC is william
```
这验证了当前终端会话的用户名确实是“william”（即`william@localhost`中的“william”）。

## 单引号的作用

与双引号不同，单引号会将其内部的所有内容都视为纯粹的文本字符，Shell不会对其中的任何部分（如变量或特殊符号）进行解释。

现在，我们将上面的双引号改为单引号：
```bash
echo ‘Your user that is currently using this PC is $LOGNAME’
```
执行这个命令后，输出结果会变成：
```
Your user that is currently using this PC is $LOGNAME
```
可以看到，`$LOGNAME`没有被替换成用户名，而是被原封不动地作为普通文本打印了出来。单引号“字面地”保护了其中的所有内容。

## 核心区别总结

上一节我们通过具体例子感受了两种引号的效果，本节我们来明确它们的核心机制区别。

以下是单引号与双引号的核心区别对比：

*   **双引号 (`"`)**: 会对其中的**变量**（如`$VAR`）、**命令替换**（如`` `command` ``或`$(command)`）和**某些转义字符**（如`\n`换行）进行解释和替换。
*   **单引号 (`'`)**: 将其中的所有内容都视为**纯文本**，不进行任何解释。变量、命令替换和转义字符都会失去其特殊含义，被直接输出。

简单来说，双引号是“智能的”或“可扩展的”，而单引号是“纯粹的”或“字面的”。

## 如何选择？

在实际使用时：
*   当字符串中**需要包含变量或命令执行结果**时，使用**双引号**。
*   当需要**原样输出所有字符**，确保其中的美元符号`$`、反引号`` ` ``等不被Shell解释时，使用**单引号**。

本节课中，我们一起学习了Linux Shell中单引号和双引号的根本区别。记住“双引号解释变量，单引号原样输出”这个核心原则，将帮助你在命令行操作和脚本编写中更精准地控制文本输出。
