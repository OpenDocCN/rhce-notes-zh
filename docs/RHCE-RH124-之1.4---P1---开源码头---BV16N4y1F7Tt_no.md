## 虚拟机管理命令

上一节我们介绍了教学环境的构成，本节中我们来看看如何通过命令行高效地管理这些虚拟机。

我们使用一套定制化的命令来管理虚拟机，这些命令以 `rht` 开头，代表 Red Hat Training。

### 启动与查看虚拟机

以下是启动 `classroom` 虚拟机的命令：
```bash
rht-vmctl start classroom
```
命令执行后，系统会提示虚拟机已启动。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_9.png)

![](img/8e06f005cf0a6a00fb4928f1b9f762db_11.png)

我们可以通过以下命令查看虚拟机的控制台界面：
```bash
rht-vmview classroom
```
`classroom` 虚拟机提供了教学资源，通常我们只需要保证它运行即可，无需登录。

要一次性启动所有其他4台虚拟机（`servera`, `serverb`, `workstation`, `bastion`），可以使用：
```bash
rht-vmctl start all
```
**注意**：首次运行此命令时，系统会从 `classroom` 服务器下载虚拟机磁盘映像，这个过程需要时间，并且要求宿主机有足够的内存（建议6GB以上）。执行此命令前，务必先启动 `classroom` 虚拟机。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_13.png)

### 查看虚拟机状态

![](img/8e06f005cf0a6a00fb4928f1b9f762db_15.png)

使用以下命令可以查看所有虚拟机的运行状态：
```bash
rht-vmctl status
```
由于虚拟机启动需要时间，执行后可能不会立即显示为运行状态。

### 虚拟机的其他操作

![](img/8e06f005cf0a6a00fb4928f1b9f762db_17.png)

![](img/8e06f005cf0a6a00fb4928f1b9f762db_19.png)

每个命令都支持 `-h` 或 `--help` 选项来获取帮助信息。命令的基本格式为：`rht-vmctl [操作] [虚拟机名]`。

以下是可用的操作选项：
*   `start`：启动一台虚拟机。
*   `stop`：向一台运行的虚拟机发送关机信号，使其正常关闭。
*   `reset`：重置虚拟机。如果虚拟机已关机，则会瞬间还原到初始状态。这是一个非常有用的功能，当实验环境被意外修改时，可以快速恢复。
*   `poweroff`：强制关闭一台虚拟机。当 `stop` 命令无效时使用。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_21.png)

例如，关闭 `serverb` 虚拟机：
```bash
rht-vmctl stop serverb
```
再次查看状态，会发现 `serverb` 的状态变为 `shut off`。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_23.png)

在做实验时，我们通常按需启动相关的虚拟机。其中，`bastion` 虚拟机作为路由器角色，连接实验机器和资源服务器，通常也需要开启。

### 理解命令行提示符与目录结构

![](img/8e06f005cf0a6a00fb4928f1b9f762db_30.png)

![](img/8e06f005cf0a6a00fb4928f1b9f762db_31.png)

![](img/8e06f005cf0a6a00fb4928f1b9f762db_32.png)

Linux系统的文件组织采用树状结构。包含文件的容器称为**目录**（在Windows中常称为文件夹）。整个文件系统从根目录 `/` 开始，逐级分支。

命令行提示符包含了重要信息，格式通常为 `[用户名@主机名 当前目录]$`。
*   `用户名`：当前登录的用户。
*   `主机名`：当前机器的主机名（短名称）。
*   `当前目录`：用户当前所在的目录。`~` 符号代表当前用户的家目录（主目录）。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_34.png)

![](img/8e06f005cf0a6a00fb4928f1b9f762db_36.png)

使用 `pwd` 命令可以打印出当前所在目录的完整路径。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_38.png)

![](img/8e06f005cf0a6a00fb4928f1b9f762db_40.png)

### 文件与目录操作

以下是几个核心的文件与目录操作命令，它们展示了命令行的高效性。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_44.png)

**创建文件**
`touch` 命令用于创建新的空文件或更新已有文件的时间戳。
```bash
touch file{1..100}
```
这条命令利用大括号扩展，一次性创建了100个名为 file1, file2, ..., file100 的文件。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_46.png)

**删除文件**
`rm` 命令用于删除文件或目录。`-f` 选项表示强制删除，不进行确认提示。
```bash
rm -f file{1..100}
```
这条命令会强制删除刚才创建的100个文件。

**创建目录**
`mkdir` 命令用于创建新目录。
```bash
mkdir dir{1..100}
```
这条命令一次性创建100个目录。

**删除目录**
`rmdir` 命令用于删除空目录。要删除非空目录，通常使用 `rm -r`。
```bash
rmdir dir{1..100}
```

**向文件写入内容**
`echo` 命令用于在终端输出文本。结合重定向符号 `>`（覆盖）或 `>>`（追加），可以将输出内容写入文件。
```bash
echo “Hello” >> file111
```
这条命令将 “Hello” 字符串追加到 `file111` 文件的末尾。如果文件不存在，则会创建它。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_48.png)

### 图形界面与命令行的关联

在Linux中，图形桌面的“桌面”区域实际上对应着用户家目录下的一个特定目录（例如 `/home/student/Desktop`）。当我们在该目录下执行创建或删除文件的操作时，图形桌面上会实时反映出这些变化。这体现了Linux系统后台逻辑与前台表现的映射关系。

同样，在图形界面中删除桌面图标，实际上也是在对那个目录下的文件进行操作。

## 命令总结

本节课中我们一起学习了以下内容：
1.  **虚拟机管理**：使用 `rht-vmctl` 命令启动 (`start`)、停止 (`stop`)、重置 (`reset`) 和查看状态 (`status`)。
2.  **目录导航**：使用 `pwd` 查看当前目录，使用 `cd` 切换目录。
3.  **文件操作**：使用 `touch` 创建文件，使用 `rm` 删除文件。
4.  **目录操作**：使用 `mkdir` 创建目录，使用 `rmdir` 删除空目录。
5.  **内容写入**：使用 `echo` 结合 `>>` 向文件追加内容。

![](img/8e06f005cf0a6a00fb4928f1b9f762db_50.png)

这些命令是Linux操作的基础，通过命令行可以高效、批量地完成许多任务，这是掌握Linux系统管理的重要一步。
