## 回顾上周内容

上一节我们介绍了脚本的基础知识，本节中我们来看看上周的核心内容总结。

脚本文件后缀一般使用`.sh`结尾。

文件第一行需要指定解释器，格式是`#!/bin/bash`。

脚本编写完成后，需要保存。

文件的权限设置需要有执行权限，命令是`chmod +x 脚本文件名`。

执行脚本的方式有以下几种。

*   绝对路径执行：例如 `/root/bin/下的文件`
*   使用解释器直接运行：`bash 脚本文件`

使用解释器运行脚本时，可以没有执行权限。

脚本的测试有以下方法。

以下是脚本测试的方法：
*   `bash -n 脚本文件`：测试脚本语法是否正确。
*   `bash -x 脚本文件`：测试脚本运行情况，显示每一步执行结果。

变量赋值可以是字符串、数字或命令。

但是Shell默认不支持小数运算。

运算有以下几种方式。

以下是变量运算的方式：
*   `let`命令：`let sum=$x+$y`
*   `$(( ))`：`sum=$((x+y))`

然后是测试命令。

测试命令有`test`和`[ ]`，`[[ ]]`支持使用通配符或正则表达式。

使用中括号测试时，要注意括号内必须有空格，例如 `[ $x -eq $y ]`。

然后是双分支条件。

双分支也是一样，通过`if`进行条件判断。

如果为真，执行一条路；如果为假，执行另一条路，执行完后结束。

## 单分支if语句

我们先看一下单分支`if`语句。

我们来看一个简单的例子。

![](img/5ca10ee4348a6b839bb380fd12aa5897_9.png)

首先定义两个变量：`x=10`，`y=8`。

然后使用`if`语句判断。

`if`语句写完条件后，需要一个分号，代表条件判断结束，然后是`then`。

![](img/5ca10ee4348a6b839bb380fd12aa5897_11.png)

如果条件为真，就输出一个`hello`。

最后用`fi`结束。

我们执行一下这个单分支脚本。

```bash
x=10
y=8
if [ $x -eq $y ]; then
    echo "hello"
fi
```

脚本直接结束了，我们查看它的返回值。

`$?`的值为1，表示条件为假，直接退出。

我们修改一下条件，让`x`大于`y`。

```bash
if [ $x -gt $y ]; then
    echo "hello"
fi
```

现在条件为真，执行了`echo "hello"`。

这就是单分支`if`语句，如果条件正确，就执行语句块。

## 双分支与多分支if语句

接下来我们看看双分支`if`语句。

双分支使用`if-else`结构。

![](img/5ca10ee4348a6b839bb380fd12aa5897_15.png)

如果条件为真，执行`then`后面的语句；否则，执行`else`后面的语句。

```bash
if [ $x -gt $y ]; then
    echo "x is greater than y"
else
    echo "x is not greater than y"
fi
```

如果条件为真，输出`x is greater than y`；否则输出`x is not greater than y`。

对于多分支条件，我们不能一直用`if`或`else`。

多分支使用`if-elif-else`结构。

最后一个条件建议写上`else`，这是一个好习惯。

我们来看一个多分支的例子。

```bash
if [ $x -lt $y ]; then
    echo "x is less than y"
elif [ $x -eq $y ]; then
    echo "x is equal to y"
else
    echo "x is greater than y"
fi
```

首先判断`x`是否小于`y`，如果是，输出对应信息。

然后判断`x`是否等于`y`，如果是，输出对应信息。

如果以上都不满足，那么`x`肯定大于`y`，执行`else`分支。

![](img/5ca10ee4348a6b839bb380fd12aa5897_17.png)

这是一个简单的`if`语法。

记住，`if`语句结束时必须加上`fi`。

## if语句的嵌套

`if`语句是可以嵌套的。

![](img/5ca10ee4348a6b839bb380fd12aa5897_21.png)

我们来看一个嵌套的例子。

```bash
if [ $x -ge $y ]; then
    if [ $x -eq $y ]; then
        echo "x equals y"
    else
        echo "x is greater than y"
    fi
fi
```

这个脚本首先判断`x`是否大于等于`y`。

![](img/5ca10ee4348a6b839bb380fd12aa5897_23.png)

如果不是，直接退出。

![](img/5ca10ee4348a6b839bb380fd12aa5897_25.png)

如果是，则进入第二个条件判断`x`是否等于`y`。

如果等于，输出`x equals y`；如果不等于，则输出`x is greater than y`。

![](img/5ca10ee4348a6b839bb380fd12aa5897_27.png)

注意，只要嵌套了一个`if`，结尾就必须有一个对应的`fi`。

嵌套`if`时，条件之间一定要有关联，并且尽量少写无关的嵌套，保持代码简洁。

## case条件选择语句

第二个内容是条件判断选择语句：`case`。

`case`的语法如下。

首先，`case`后面跟一个变量。

然后使用`in`关键字开始匹配。

如果变量的值等于某个模式，就执行对应的语句。

如果都不匹配，可以设定一个默认分支或直接退出。

结束语句是`esac`（`case`反过来写）。

`case`支持通配符风格：`*`匹配任意字符串，`?`匹配任意单个字符，`[ ]`匹配指定范围内的单个字符。

![](img/5ca10ee4348a6b839bb380fd12aa5897_29.png)

我们来看一个范例。

![](img/5ca10ee4348a6b839bb380fd12aa5897_31.png)

这个脚本首先定义了一个随机颜色，然后打印一个菜单。

![](img/5ca10ee4348a6b839bb380fd12aa5897_33.png)

菜单选项有：1备份数据库，2清理日志，3软件升级等。

![](img/5ca10ee4348a6b839bb380fd12aa5897_35.png)

然后通过`read -p`接收用户输入的数字。

![](img/5ca10ee4348a6b839bb380fd12aa5897_37.png)

用户输入的值会传递给`case`语句进行匹配。

![](img/5ca10ee4348a6b839bb380fd12aa5897_39.png)

如果输入`1`，就匹配到`1)`，然后执行系统备份的命令。

`case`语句通常用于制作菜单，而`if`语句多用于运维逻辑判断。

当然，在`case`中嵌套`for`或`if`也很常见。
