## 用户密码设置

![](img/c2a1a806f6f647590d772e6429f79129_8.png)

![](img/c2a1a806f6f647590d772e6429f79129_10.png)

为用户设置密码的命令是 `passwd`。其基本用法非常简单。

### 命令格式

![](img/c2a1a806f6f647590d772e6429f79129_12.png)

`passwd` 命令的基本格式如下：
```bash
passwd [选项] [用户名]
```

### 使用方法

以下是 `passwd` 命令的几种常见使用场景：

*   **修改当前用户自身密码**：直接执行 `passwd` 命令，系统会提示输入新密码。
*   **管理员为其他用户设置密码**：root用户可以使用 `passwd 用户名` 的格式为指定用户设置或修改密码，且无需知道原密码。
*   **普通用户修改自身密码**：普通用户也可以使用 `passwd` 命令修改自己的密码，但必须首先输入自己当前的密码。

**注意**：普通用户在设置新密码时，必须遵循系统的密码策略，例如密码长度、复杂度等，否则修改可能失败。

## 密码策略与规范

Linux系统对密码有默认的复杂性要求，主要包含以下几点：
*   **长度**：通常不少于8个字符。
*   **复杂度**：应包含数字、字母（区分大小写）以及特殊字符。
*   **历史**：新密码不能与旧密码过于相似。

root用户在设置密码时不受这些策略的严格限制，但普通用户必须遵守。

## 密码信息文件 `/etc/shadow`

![](img/c2a1a806f6f647590d772e6429f79129_28.png)

![](img/c2a1a806f6f647590d772e6429f79129_29.png)

![](img/c2a1a806f6f647590d772e6429f79129_31.png)

![](img/c2a1a806f6f647590d772e6429f79129_32.png)

用户设置密码后，加密的密码信息并非存储在 `/etc/passwd` 中，而是存放在 `/etc/shadow` 文件里。这是一个安全性更高的文件。

### 文件权限

![](img/c2a1a806f6f647590d772e6429f79129_34.png)

![](img/c2a1a806f6f647590d772e6429f79129_36.png)

![](img/c2a1a806f6f647590d772e6429f79129_38.png)

![](img/c2a1a806f6f647590d772e6429f79129_40.png)

首先，我们查看一下该文件的权限：
```bash
ls -l /etc/shadow
```
你会发现该文件权限严格，普通用户甚至无法读取，只有root用户有权限管理。

### 文件结构详解

`/etc/shadow` 文件每一行对应一个用户的密码信息，由冒号 `:` 分隔为9个字段。

我们以用户 `user1` 的一行为例进行解读：
```
user1:$6$...加密字符串...:19077:0:99999:7:::
```

以下是每个字段的含义：
1.  **用户名**：用户的登录名。
2.  **加密密码**：用户密码经过加密算法（如SHA512）处理后的字符串。如果此字段为 `!` 或 `*`，则表示该用户密码未设置或已锁定。
3.  **最后修改时间**：从1970年1月1日到密码最后一次被修改的天数。
4.  **最小修改间隔**：密码修改后，多少天内不允许再次修改。`0` 表示随时可修改。
5.  **密码有效期**：密码多少天后过期。例如 `99999` 表示几乎永不过期，`60` 表示60天后过期。
6.  **警告期**：密码过期前多少天开始向用户发出警告。
7.  **宽限期**：密码过期后，允许用户在此宽限天数内修改密码并登录。
8.  **账号失效时间**：从1970年1月1日起，指定账号在多少天后失效。
9.  **保留字段**：目前未使用。

## 密码管理高级操作

`passwd` 命令配合一些选项，可以实现更丰富的密码管理功能。

以下是 `passwd` 命令的几个常用选项：

![](img/c2a1a806f6f647590d772e6429f79129_42.png)

![](img/c2a1a806f6f647590d772e6429f79129_44.png)

*   **`-S, --status`**：查看用户的密码状态信息。
    ```bash
    passwd -S user1
    ```
*   **`-l, --lock`**：锁定用户密码，使其无法登录。
    ```bash
    passwd -l user1
    ```
*   **`-u, --unlock`**：解锁用户密码。
    ```bash
    passwd -u user1
    ```
*   **`-d, --delete`**：清空用户密码。
    ```bash
    passwd -d user1
    ```
*   **`--stdin`**：从标准输入（如管道）非交互式地设置密码，常用于脚本中。
    ```bash
    echo "MyNewPass123!" | passwd --stdin user1
    ```

## 强制用户首次登录修改密码

![](img/c2a1a806f6f647590d772e6429f79129_48.png)

在企业环境中，管理员分发初始账号时，常希望用户首次登录就必须修改密码。这可以通过修改 `/etc/shadow` 文件中密码的“最后修改时间”字段来实现。

![](img/c2a1a806f6f647590d772e6429f79129_50.png)

![](img/c2a1a806f6f647590d772e6429f79129_52.png)

使用 `chage` 命令可以方便地达成此目的：
```bash
chage -d 0 user1
```
此命令将 `user1` 的密码最后修改时间设置为0（1970年1月1日），系统会认为该密码从未修改过。用户 `user1` 下次登录时，系统将强制要求其更改密码。

## 用户身份切换

![](img/c2a1a806f6f647590d772e6429f79129_54.png)

有时我们需要临时切换到另一个用户身份执行操作，这时可以使用 `su` 命令。

![](img/c2a1a806f6f647590d772e6429f79129_56.png)

![](img/c2a1a806f6f647590d772e6429f79129_58.png)

![](img/c2a1a806f6f647590d772e6429f79129_60.png)

![](img/c2a1a806f6f647590d772e6429f79129_62.png)

![](img/c2a1a806f6f647590d772e6429f79129_63.png)

`su` 命令用于切换当前用户身份到其他用户身份。

### 使用方法

*   **切换用户（不切换环境变量）**：
    ```bash
    su username
    ```
*   **完全切换用户（切换环境变量和家目录）**：推荐使用 `-` 或 `-l` 选项。
    ```bash
    su - username
    ```
*   **退出切换的身份**：执行 `exit` 命令即可返回原用户身份。

![](img/c2a1a806f6f647590d772e6429f79129_65.png)

![](img/c2a1a806f6f647590d772e6429f79129_67.png)

**注意**：root用户切换到任何用户都无需输入密码。普通用户使用 `su` 切换时，需要知道目标用户的密码。
