# Linux进程管理与计划任务：P26：day5-2 - 进程优先级与计划任务 🚀

在本节课中，我们将要学习Linux系统中的进程优先级管理和计划任务。我们将首先了解进程优先级的分类与调整方法，然后学习如何使用`at`和`cron`工具来安排一次性或周期性的任务。

## 进程优先级介绍

上一节我们介绍了进程的基本概念，本节中我们来看看如何管理进程的优先级。Linux系统中的进程优先级分为两个主要范围。

*   **实时进程优先级**：范围为 **0到99**。数字越小，优先级越高。这类进程用于需要实时响应的任务。
*   **非实时进程优先级**：范围为 **100到139**。数字越小，优先级越高。这是普通用户进程的默认范围。

此外，还有一个与用户相关的`nice`值，它影响非实时进程的优先级。

*   **nice值**：范围为 **-20到19**。`nice`值越低（如-20），进程的“友好度”越低，意味着它更“自私”，会获得更高的系统优先级。默认启动的进程`nice`值为0，对应的系统优先级为120。

![](img/e078c2bc2c16c5441da5a858d834c208_1.png)

![](img/e078c2bc2c16c5441da5a858d834c208_2.png)

系统优先级（0-139）数字越小，优先级越高。系统维护着运行队列和过期队列来管理这些进程。

![](img/e078c2bc2c16c5441da5a858d834c208_3.png)

## 调整进程优先级

了解了优先级的概念后，我们来看看如何调整它。进程优先级的调整主要分为两种：启动时指定和运行时调整。

![](img/e078c2bc2c16c5441da5a858d834c208_5.png)

以下是调整优先级的主要命令：

![](img/e078c2bc2c16c5441da5a858d834c208_7.png)

![](img/e078c2bc2c16c5441da5a858d834c208_9.png)

1.  **`nice`命令**：以指定的优先级启动一个新进程。
    *   命令格式：`nice -n NICE_VALUE COMMAND`
    *   例如：`nice -n -5 /usr/bin/some_task` 会以较高的优先级（`nice`值为-5）启动任务。

2.  **`renice`命令**：调整一个已经正在运行的进程的优先级。
    *   命令格式：`renice NICE_VALUE -p PID`
    *   例如：`renice 10 -p 1234` 会将PID为1234的进程的`nice`值调整为10（较低优先级）。

需要注意的是，只有`root`用户可以将进程的`nice`值调低（即提高优先级），普通用户只能调高自己的进程的`nice`值（即降低优先级）。

## 一次性任务计划：at

![](img/e078c2bc2c16c5441da5a858d834c208_17.png)

![](img/e078c2bc2c16c5441da5a858d834c208_19.png)

除了调整正在运行的进程，我们还可以安排任务在未来某个时间点自动执行。首先，我们来看一次性的任务计划工具`at`。

![](img/e078c2bc2c16c5441da5a858d834c208_21.png)

`at`工具用于在指定的时间执行一次性任务。要使用`at`，需要确保`atd`服务正在运行。

以下是`at`命令的基本用法：

*   **安排任务**：使用`at TIME`进入交互式界面输入要执行的命令，或使用`at -f FILE TIME`从文件读取命令。
*   **查看队列**：使用`atq`或`at -l`查看等待执行的`at`任务队列。
*   **删除任务**：使用`atrm JOB_ID`或`at -d JOB_ID`删除指定的任务。

`at`的时间格式非常灵活，例如：
*   `at now + 5 minutes` （5分钟后执行）
*   `at 14:30 tomorrow` （明天下午2点30分执行）
*   `at 3:00 PM next week` （下星期下午3点执行）

任务执行时，会使用执行`at`命令的用户身份和环境变量。命令的输出结果默认会通过邮件发送给该用户。

## 周期性任务计划：cron

对于需要周期性重复执行的任务，我们使用`cron`服务。这是Linux系统中最常用的计划任务工具。

`cron`依赖于`crond`服务，它通常默认安装并运行。`cron`任务分为两类：

1.  **系统任务**：由`root`用户配置，通常用于系统维护，配置文件位于`/etc/crontab`及`/etc/cron.d/`目录下，以及`/etc/cron.hourly/`, `/etc/cron.daily/`等目录。
2.  **用户任务**：每个用户都可以使用`crontab`命令创建、编辑、列出或删除自己的`cron`任务。

### crontab时间格式

![](img/e078c2bc2c16c5441da5a858d834c208_23.png)

用户使用`crontab -e`编辑任务时，每行代表一个任务，其基本格式如下：

![](img/e078c2bc2c16c5441da5a858d834c208_25.png)

```
* * * * * COMMAND_TO_EXECUTE
- - - - -
| | | | |
| | | | +----- 星期几 (0 - 6) (星期天 = 0)
| | | +------- 月份 (1 - 12)
| | +--------- 日期 (1 - 31)
| +----------- 小时 (0 - 23)
+------------- 分钟 (0 - 59)
```

![](img/e078c2bc2c16c5441da5a858d834c208_27.png)

![](img/e078c2bc2c16c5441da5a858d834c208_29.png)

![](img/e078c2bc2c16c5441da5a858d834c208_31.png)

每个星号(`*`)代表该时间字段的每个有效值。我们也可以用数字、范围(`-`)、列表(`,`)和步长(`/`)来指定时间。

![](img/e078c2bc2c16c5441da5a858d834c208_33.png)

![](img/e078c2bc2c16c5441da5a858d834c208_35.png)

以下是几个例子：

![](img/e078c2bc2c16c5441da5a858d834c208_36.png)

*   `* * * * *`：每分钟执行一次。
*   `0 * * * *`：每小时的0分时执行（即每小时执行一次）。
*   `*/5 * * * *`：每5分钟执行一次。
*   `0 2 * * 0`：每周日凌晨2点执行。
*   `30 14 1 * *`：每月1日下午2点30分执行。

![](img/e078c2bc2c16c5441da5a858d834c208_38.png)

### 管理cron任务

![](img/e078c2bc2c16c5441da5a858d834c208_40.png)

以下是管理用户`cron`任务的常用命令：

*   `crontab -e`：编辑当前用户的`cron`任务表。
*   `crontab -l`：列出当前用户的所有`cron`任务。
*   `crontab -r`：删除当前用户的所有`cron`任务（谨慎使用）。

![](img/e078c2bc2c16c5441da5a858d834c208_42.png)

用户的任务文件存储在`/var/spool/cron/`目录下，以用户名命名。任务执行产生的标准输出和错误默认会通过邮件发送给该用户。

![](img/e078c2bc2c16c5441da5a858d834c208_43.png)
