在本节课中，我们将要学习逻辑卷管理（LVM）的进阶操作，特别是如何对已创建的逻辑卷进行在线扩容，以及了解RAID磁盘阵列的基本概念和作用。逻辑卷的核心优势在于其动态调整大小的能力，这对于管理不断增长的数据至关重要。

## 逻辑卷扩容操作 🔄

![](img/6401beb9522d66d71a10068fbe7f9526_3.png)

![](img/6401beb9522d66d71a10068fbe7f9526_5.png)

![](img/6401beb9522d66d71a10068fbe7f9526_7.png)

上一节我们介绍了逻辑卷的创建，本节中我们来看看如何对已存在的逻辑卷进行扩容。扩容的核心思想是：当逻辑卷的存储空间不足时，我们可以动态地为其增加容量，而无需停机或迁移数据。

![](img/6401beb9522d66d71a10068fbe7f9526_11.png)

![](img/6401beb9522d66d71a10068fbe7f9526_13.png)

### 检查当前空间状态

![](img/6401beb9522d66d71a10068fbe7f9526_17.png)

在对逻辑卷扩容前，首先需要检查卷组（VG）中是否有足够的剩余空间可供分配。

以下是检查命令：
```bash
# 查看卷组信息，关注剩余空间
vgs
# 查看逻辑卷信息，关注当前大小
lvs
```
执行上述命令后，你可以看到逻辑卷的当前大小和卷组的剩余空间。扩容逻辑卷的空间不能超过卷组的剩余空间。

### 扩展逻辑卷容量

确认卷组有足够空间后，即可使用 `lvextend` 命令扩展逻辑卷。

![](img/6401beb9522d66d71a10068fbe7f9526_19.png)

以下是扩展逻辑卷的命令格式：
```bash
lvextend -L +[要增加的大小] /dev/[卷组名]/[逻辑卷名]
```
例如，为逻辑卷 `mylv` 增加19GB空间：
```bash
lvextend -L +19G /dev/system_vg/mylv
```
命令执行后，系统会提示逻辑卷的大小已从原始值（如20G）调整为目标值（如39G）。此时，使用 `lvs` 命令可以确认逻辑卷的容量已经变大。

### 扩展文件系统

![](img/6401beb9522d66d71a10068fbe7f9526_21.png)

然而，仅仅扩展逻辑卷的物理空间是不够的。新增加的空间还没有被文件系统识别和管理，因此操作系统无法使用这部分空间。

![](img/6401beb9522d66d71a10068fbe7f9526_23.png)

以下是解决此问题的关键步骤：
我们需要扩展逻辑卷上的文件系统，使其涵盖新增的空间。对于XFS文件系统，使用 `xfs_growfs` 命令；对于ext2/3/4文件系统，则使用 `resize2fs` 命令。

![](img/6401beb9522d66d71a10068fbe7f9526_25.png)

以下是扩展XFS文件系统的命令：
```bash
# 指定挂载点进行扩展
xfs_growfs /挂载点目录
# 或直接指定逻辑卷设备
xfs_growfs /dev/system_vg/mylv
```
执行此命令后，再使用 `df -h` 查看，你会发现挂载点的总容量已经成功增加，新增的空间现在可以被正常使用了。

![](img/6401beb9522d66d71a10068fbe7f9526_27.png)

## 卷组扩容操作 🧩

![](img/6401beb9522d66d71a10068fbe7f9526_37.png)

![](img/6401beb9522d66d71a10068fbe7f9526_39.png)

当卷组本身的剩余空间不足以为逻辑卷扩容时，我们就需要对卷组进行扩容，即向卷组中添加新的物理存储设备（如硬盘或分区）。

### 向卷组添加新设备

我们可以将新的硬盘或分区加入到现有的卷组中，以增加其总容量。

![](img/6401beb9522d66d71a10068fbe7f9526_41.png)

以下是扩展卷组的命令格式：
```bash
vgextend [卷组名] /dev/新设备名
```
例如，将新硬盘 `/dev/sdd` 加入到 `system_vg` 卷组：
```bash
vgextend system_vg /dev/sdd
```
命令执行成功后，使用 `vgs` 命令可以看到卷组的总容量和剩余空间都增加了。之后，你就可以使用新增的空间来进一步扩展逻辑卷。

![](img/6401beb9522d66d71a10068fbe7f9526_43.png)

![](img/6401beb9522d66d71a10068fbe7f9526_45.png)

## 逻辑卷与卷组的删除 🗑️

![](img/6401beb9522d66d71a10068fbe7f9526_47.png)

在某些情况下，可能需要删除不再使用的逻辑卷或卷组。删除操作需要按照特定顺序进行，且通常要求先卸载文件系统。

以下是删除操作的顺序和注意事项：
1.  卸载逻辑卷的挂载点：`umount /挂载点目录`
2.  删除逻辑卷：`lvremove /dev/[卷组名]/[逻辑卷名]`
3.  删除卷组：`vgremove [卷组名]`
4.  删除物理卷（可选）：`pvremove /dev/设备名` 或直接格式化该设备。

**警告**：删除操作会清除该逻辑卷上的所有数据，且不可恢复，请务必谨慎操作。

在了解了逻辑卷的动态管理后，我们还需要关注数据存储的另一个重要方面：可靠性与性能。RAID（独立磁盘冗余阵列）技术就是将多个物理磁盘组合起来，以实现数据冗余、提升性能或两者兼得。

![](img/6401beb9522d66d71a10068fbe7f9526_57.png)

![](img/6401beb9522d66d71a10068fbe7f9526_58.png)

### RAID的作用与优势

![](img/6401beb9522d66d71a10068fbe7f9526_60.png)

![](img/6401beb9522d66d71a10068fbe7f9526_62.png)

![](img/6401beb9522d66d71a10068fbe7f9526_64.png)

RAID的主要目的是通过磁盘组合来提高数据的安全性和/或读写速度。在企业环境中，它常与LVM结合使用，构建既灵活又可靠的存储架构。

![](img/6401beb9522d66d71a10068fbe7f9526_66.png)

### 常见的RAID级别

不同的RAID级别提供了不同的特性组合，例如在性能、冗余和成本之间进行权衡。

以下是几种常见RAID级别的简要介绍：
*   **RAID 0（条带化）**：将数据分散存储在多个磁盘上，提高了读写速度，但没有冗余功能。一块磁盘损坏会导致所有数据丢失。
*   **RAID 1（镜像）**：将相同的数据同时写入两块磁盘，提供了完全的冗余备份。读取速度可能提升，但写入速度不变，磁盘利用率只有50%。
*   **RAID 5（带奇偶校验的条带化）**：至少需要三块磁盘。将数据和奇偶校验信息分布在所有磁盘上。允许损坏一块磁盘而不丢失数据，兼顾了性能、冗余和存储效率。
*   **RAID 10（先镜像再条带化）**：结合了RAID 1和RAID 0，至少需要四块磁盘。它提供了优秀的读写性能和数据冗余，允许每个镜像组中坏掉一块磁盘。

**注意**：RAID不能替代定期备份。它主要防止因硬件故障导致的服务中断和数据丢失，但无法防范误删除、病毒或整个系统故障。
