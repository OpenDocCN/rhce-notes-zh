## 通配符详解 🔍

![](img/0879bce6e59c49e7334f621c7946393e_7.png)

上一节我们介绍了文件的基本操作，本节中我们来看看如何利用通配符来批量处理文件。在Linux系统中，通配符是一种特殊的符号，用于匹配文件名或路径名。它与更复杂的正则表达式不同，是日常文件操作中最常用的工具。

以下是几种最常用的通配符及其含义：

*   **星号 `*`**：代表**任意数量**的任意字符。
    *   例如：`ls *.txt` 会列出当前目录下所有以 `.txt` 结尾的文件。
*   **问号 `?`**：代表**任意一个**字符。
    *   例如：`ls file?.txt` 会匹配 `file1.txt`、`fileA.txt`，但不会匹配 `file10.txt`。
*   **中括号 `[]`**：代表一个字符范围，匹配括号内**任意一个**字符。
    *   例如：`ls [a-c].txt` 会匹配 `a.txt`、`b.txt`、`c.txt`。
    *   范围可以组合：`ls [a-zA-Z0-9].txt` 匹配所有单个字母或数字命名的 `.txt` 文件。
*   **中括号内加叹号 `[!]`**：代表“非”，匹配**不在**括号内的任意一个字符。
    *   例如：`ls [!0-9].txt` 会匹配第一个字符不是数字的所有 `.txt` 文件。

除了用于文件匹配的通配符，系统里还有一些具有特殊功能的符号：

![](img/0879bce6e59c49e7334f621c7946393e_9.png)

![](img/0879bce6e59c49e7334f621c7946393e_11.png)

*   **管道符 `|`**：将前一个命令的输出，作为后一个命令的输入。
    *   公式：`command1 | command2`
*   **井号 `#`**：代表注释，其后的内容在脚本或配置文件中不会被当作命令执行。
*   **波浪线 `~`**：代表当前用户的**家目录**。
*   **连字符 `-`**：代表**上一次访问的目录**，常用于 `cd -` 命令。
*   **美元符号 `$`**：用于**调用变量**。
    *   例如：`echo $PATH` 会输出 `PATH` 环境变量的值。
*   **斜杠 `/`**：代表**根目录**或作为**目录之间的分隔符**。
*   **重定向符号 `>` 和 `>>`**：
    *   `>`：将命令输出**覆盖**到指定文件。`command > file`
    *   `>>`：将命令输出**追加**到指定文件末尾。`command >> file`
*   **花括号 `{}`**：用于生成序列，常用于快速创建文件。
    *   例如：`touch file{1..3}.txt` 会创建 `file1.txt`、`file2.txt`、`file3.txt` 三个文件。

### 引号的特殊区别

![](img/0879bce6e59c49e7334f621c7946393e_13.png)

在Shell中，单引号、双引号和反引号的行为截然不同。

![](img/0879bce6e59c49e7334f621c7946393e_15.png)

*   **反引号 `` ` ``**：**优先执行**反引号内的命令，并将其输出结果替换到原位置。
    *   代码示例：`echo `date`` 会先执行 `date` 命令得到当前时间，然后 `echo` 输出这个时间。
*   **双引号 `"`**：引用的内容会被当作一个整体，但**其中的变量（`$var`）和反引号会被解析**。
    *   代码示例：
        ```bash
        name="GLAB"
        echo "Hello, $name" # 输出：Hello, GLAB
        ```
*   **单引号 `'`**：引用的内容会被**原样输出**，不解析任何变量或命令。
    *   代码示例：
        ```bash
        name="GLAB"
        echo 'Hello, $name' # 输出：Hello, $name
        ```

> **关键总结**：当需要引用字符串并保留变量替换功能时，使用**双引号**；当需要完全原样输出字符串时，使用**单引号**；当需要执行命令并获取其输出时，使用**反引号**（现代写法更推荐使用 `$(command)`）。

## 帮助命令手册 📖

了解了通配符的强大功能后，当我们遇到不熟悉的命令时该怎么办呢？本节中我们来看看Linux内置的“说明书”——帮助命令。最核心、最常用的帮助命令是 `man`。

`man` 命令是 **manual（手册）** 的缩写，它提供了系统中绝大多数命令、函数和配置文件的详细说明。

![](img/0879bce6e59c49e7334f621c7946393e_17.png)

基本用法非常简单：
```bash
man [命令名]
```
例如，输入 `man ls` 即可查看 `ls` 命令的完整手册。

### 使用 `man` 手册的技巧

![](img/0879bce6e59c49e7334f621c7946393e_19.png)

![](img/0879bce6e59c49e7334f621c7946393e_21.png)

浏览 `man` 手册时，有几个实用技巧：

![](img/0879bce6e59c49e7334f621c7946393e_23.png)

![](img/0879bce6e59c49e7334f621c7946393e_25.png)

1.  **查找关键字**：在手册页面内，按下 `/` 键，然后输入你想查找的关键词（如 `-l`），按回车即可搜索。按 `n` 键查找下一个匹配项，按 `N` 键查找上一个。
2.  **翻页**：使用**空格键**向下翻页，使用 **`b`** 键向上翻页。
3.  **退出**：按下 **`q`** 键即可退出 `man` 手册。

![](img/0879bce6e59c49e7334f621c7946393e_29.png)

### 如何查找命令

![](img/0879bce6e59c49e7334f621c7946393e_33.png)

如果你连命令的全名都不知道，只知道一个关键词，可以使用 `man -k` 命令进行搜索。
```bash
man -k [关键词]
```
例如，`man -k password` 会列出所有手册页描述中包含 “password” 的命令和配置文件。

![](img/0879bce6e59c49e7334f621c7946393e_35.png)

![](img/0879bce6e59c49e7334f621c7946393e_37.png)

![](img/0879bce6e59c49e7334f621c7946393e_39.png)

有时系统帮助数据库未更新，可能导致搜索无结果。此时可以运行 `sudo mandb` 命令更新数据库，然后再进行搜索。

![](img/0879bce6e59c49e7334f621c7946393e_41.png)

![](img/0879bce6e59c49e7334f621c7946393e_43.png)

![](img/0879bce6e59c49e7334f621c7946393e_45.png)

找到你感兴趣的命令后，`man -k` 会显示其名称和所在手册的章节号（如 `passwd (1)`）。你可以通过指定章节号来查看：
```bash
man [章节号] [命令名]
# 例如
man 1 passwd
```
