## 远程连接Elasticsearch

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_12.png)

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_13.png)

要远程连接Elasticsearch，首先需要获取其外网访问地址。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_15.png)

在华为云容器服务的访问方式中，可以找到Elasticsearch的外网地址和端口（默认为9200）。通过这个地址和端口，即可从外部网络连接到ES服务。

连接ES服务，我们可以使用一个名为`elasticsearch-head`的浏览器插件。该插件提供了一个图形化界面来管理和操作ES。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_17.png)

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_18.png)

以下是启动和连接`elasticsearch-head`插件的步骤：
1.  确保本地已安装Node.js运行环境。
2.  解压提供的插件包，在命令行中进入其目录。
3.  运行命令 `npm run start` 来启动插件服务。
4.  在浏览器中访问插件提供的本地地址（通常是 `http://localhost:9100`）。
5.  在插件的连接界面，将ES的外网地址和端口填入，然后点击连接。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_20.png)

连接成功后，即可在插件界面中看到ES集群的状态信息。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_22.png)

## 创建索引与映射

成功连接ES后，下一步是创建索引并定义其映射（Mapping），这类似于在数据库中创建表和定义字段结构。

首先，我们需要创建名为 `xc_course` 的索引。这个名称是固定的，因为后端程序已配置好读取此索引的数据。

在`elasticsearch-head`的“索引”选项卡中，点击“新建索引”。输入索引名 `xc_course`。由于是测试环境，分片数设置为1，副本数设置为0。点击确定即可创建索引。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_24.png)

创建索引后，需要为其定义映射。映射规定了索引中每个字段的数据类型和属性。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_26.png)

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_28.png)

我们使用Postman工具来发送HTTP请求以创建映射。以下是操作步骤：
1.  打开Postman，创建一个新的POST请求。
2.  在请求地址栏中，填入ES的外网访问地址，并加上索引和`_mapping`路径，格式为：`http://{外网IP}:9200/xc_course/_mapping`。
3.  在请求体（Body）中，选择`raw`格式和`JSON`类型。
4.  将提供的映射JSON数据复制到请求体中。
5.  点击“Send”发送请求。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_30.png)

请求成功后，映射即创建完成。此时可以在`elasticsearch-head`插件的“数据浏览”中看到已创建的字段结构。

## 导入初始化数据

映射创建好后，就可以向索引中导入数据了。我们使用Postman通过HTTP POST请求来批量添加文档。

数据导入分为两部分，分别对应两个不同的类型（类似于数据库中的表）：`xc_course`（课程信息）和`xc_course_media`（课程媒体信息，如视频地址）。

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_32.png)

![](img/1465beaeddb7ad9dc6e8873aadd3ec44_33.png)

以下是导入数据的步骤：
1.  在Postman中，创建一个新的POST请求。
2.  请求地址格式为：`http://{外网IP}:9200/xc_course/_doc/1`。其中 `_doc` 表示文档类型，`1` 是文档的ID。
3.  在请求体（Body）中，选择`raw`格式和`JSON`类型。
4.  将第一条课程数据的JSON内容复制到请求体中。
5.  点击“Send”发送请求，即可添加第一条数据。
6.  重复以上步骤，修改请求地址中的文档ID和请求体中的JSON内容，将其余测试数据依次导入。
7.  对于 `xc_course_media` 类型的数据，使用地址 `http://{外网IP}:9200/xc_course_media/_doc/{id}` 进行同样的操作。

所有数据导入完成后，可以在`elasticsearch-head`插件的“数据浏览”中，选择对应的索引和类型，查看已成功导入的所有文档记录。
