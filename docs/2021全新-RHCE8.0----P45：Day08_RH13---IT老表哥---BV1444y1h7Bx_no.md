# RHCE8.0 红帽认证入门教程：Day08：RH134_Ch09b_autofs实例 🚀

在本节课中，我们将学习如何配置 **autofs** 服务来实现一个特定的应用场景：为远程用户自动挂载其家目录。这类似于 Windows AD 域环境中的漫游配置文件功能，用户在任何一台域成员机器上登录，都能访问到位于中央服务器上的个人文件。

## 概述

上一节我们介绍了 autofs 的基本概念和配置方法。本节中，我们来看一个具体的实例：模拟一个域环境，当指定用户登录客户端时，自动将其在服务器上的家目录挂载到本地的家目录位置，实现文件的集中存储和共享。

## 问题情境与解决思路

![](img/2dfae09850a14dd97e9855796bb9c43e_4.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_6.png)

我们思考一个问题：如何为指定用户创建账户，并自动挂载其家目录？这种需求常见于 AD 域环境。在域中，用户文件通常保存在远程服务器上。用户从机器 A 登录保存文件后，在机器 B 登录依然能看到这些文件，因为这些文件并非存储在本地，而是存储在远程目录。

Linux 系统也需要实现相同的原理。我们的解决思路是：在本地创建一个指向远程服务器的用户账户（前提是该用户已在服务器端存在）。当该用户登录时，其本地的家目录会自动挂载服务器端对应的家目录文件，从而实现文件共享。

![](img/2dfae09850a14dd97e9855796bb9c43e_8.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_10.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_12.png)

下面我们通过一个假定情境来实现。

![](img/2dfae09850a14dd97e9855796bb9c43e_13.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_14.png)

1.  在客户端，我们创建一个本地用户 `remoteuser`。
    ```bash
    useradd -u 2000 remoteuser
    ```
    检查其家目录：
    ```bash
    ll -d /home/remoteuser
    ```
    此时可以正常切换到该用户并访问其家目录。

![](img/2dfae09850a14dd97e9855796bb9c43e_16.png)

2.  我们删除这个本地家目录，模拟家目录不存在的情况。
    ```bash
    rm -rf /home/remoteuser
    ```
    再次切换用户时，会因为家目录不存在而失败。我们的目标就是：当用户 `remoteuser` 登录时，能自动将服务器上的家目录挂载过来。

![](img/2dfae09850a14dd97e9855796bb9c43e_18.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_19.png)

## 实验目标与架构

![](img/2dfae09850a14dd97e9855796bb9c43e_21.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_23.png)

我们需要完成以下事项：
*   **服务端 (Server A)**：发布 `/home/remoteuser` 目录。

*   **实现效果**：当用户 `remoteuser` 登录客户端时，自动将服务端的 `/home/remoteuser` 目录挂载成为该用户的家目录。

### 步骤一：配置 NFS 服务端 (Server A)

![](img/2dfae09850a14dd97e9855796bb9c43e_39.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_41.png)

首先，在服务端配置 NFS，共享远程用户的家目录。

1.  编辑 NFS 导出配置文件 `/etc/exports`，添加共享规则。
    ```bash
    vim /etc/exports
    ```
    添加以下内容（假设客户端 IP 网段为 192.168.145.0/24）：
    ```
    /home/remoteuser 192.168.145.0/24(rw,sync,no_root_squash)
    ```
    关键参数说明：
    *   `rw`：允许读写。
    *   `sync`：同步写入，保证数据一致性。
    *   `no_root_squash`：不映射 root 用户，对于家目录挂载场景可能需要。

![](img/2dfae09850a14dd97e9855796bb9c43e_43.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_44.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_45.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_46.png)

2.  重新发布 NFS 共享列表。
    ```bash
    exportfs -rv
    ```

![](img/2dfae09850a14dd97e9855796bb9c43e_48.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_49.png)

### 步骤二：配置 Autofs 客户端 (Server B)

![](img/2dfae09850a14dd97e9855796bb9c43e_50.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_51.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_53.png)

接下来，在客户端配置 autofs，实现按需自动挂载。

![](img/2dfae09850a14dd97e9855796bb9c43e_54.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_56.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_57.png)

1.  首先，测试能否看到服务端发布的共享。
    ```bash
    showmount -e 192.168.145.128
    ```
    应该能看到 `/home/remoteuser` 这一行。

![](img/2dfae09850a14dd97e9855796bb9c43e_59.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_61.png)

2.  编辑主配置文件 `/etc/auto.master`，指定监控目录和对应的映射文件。
    ```bash
    vim /etc/auto.master
    ```
    添加以下一行：
    ```
    /home /etc/auto.remote
    ```
    这表示当访问 `/home` 目录下的子目录时，参考 `/etc/auto.remote` 文件中的规则。

![](img/2dfae09850a14dd97e9855796bb9c43e_63.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_65.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_67.png)

3.  创建并编辑映射文件 `/etc/auto.remote`。
    ```bash
    cp /etc/auto.misc /etc/auto.remote
    vim /etc/auto.remote
    ```
    在文件中添加挂载规则：
    ```
    remoteuser -fstype=nfs,rw 192.168.145.128:/home/remoteuser
    ```
    这条规则的意思是：当访问 `/home/remoteuser` 时，以读写方式挂载服务端 `192.168.145.128` 的 `/home/remoteuser` 目录。

![](img/2dfae09850a14dd97e9855796bb9c43e_69.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_71.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_72.png)

4.  重启 autofs 服务使配置生效。
    ```bash
    systemctl restart autofs
    ```

![](img/2dfae09850a14dd97e9855796bb9c43e_75.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_80.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_81.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_85.png)

1.  在客户端，切换到 `remoteuser` 用户。
    ```bash

    ```
    如果配置正确，你会成功登录，并且当前目录就是自动挂载的远程家目录。

2.  进行读写测试，验证功能完整性。
    ```bash
    touch testfile
    ls -l
    ```
    如果能看到新创建的 `testfile` 文件，说明挂载成功且具有读写权限。

## 应用场景总结

![](img/2dfae09850a14dd97e9855796bb9c43e_95.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_97.png)

本节课我们一起学习了如何配置 autofs 实现远程用户家目录的自动挂载。这个实例模拟了类似 Windows AD 域的环境，其核心应用场景在于**集中化的用户文件管理**。通过这种方式，用户可以在网络中的任何一台客户端机器上登录，都能访问到统一、最新的个人文件，非常适合企业或学校等需要管理多台主机的环境。

![](img/2dfae09850a14dd97e9855796bb9c43e_99.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_100.png)

![](img/2dfae09850a14dd97e9855796bb9c43e_102.png)

**关键点回顾**：
*   **服务端**：通过 NFS 共享指定用户的家目录。
*   **客户端**：通过 autofs 配置，实现用户登录时自动挂载远程目录。
*   **效果**：实现了用户配置文件的“漫游”，数据集中存储在服务器，提高了管理效率和数据安全性。
