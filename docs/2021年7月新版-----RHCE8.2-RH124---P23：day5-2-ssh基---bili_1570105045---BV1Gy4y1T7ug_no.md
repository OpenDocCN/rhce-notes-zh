## 基于密码的SSH认证回顾

上一节我们介绍了SSH的基本概念，本节中我们来看看基于密码的认证流程。

![](img/5e9e0cb2154ef323a13761a1a59d514b_4.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_6.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_8.png)

基于密码的SSH认证流程如下：
1.  客户端发起连接请求。
2.  服务器响应并发送自己的公钥。
3.  客户端验证服务器公钥（首次连接时会提示用户确认）。
4.  用户输入密码进行验证。
5.  验证通过后，建立加密通道进行通信。

![](img/5e9e0cb2154ef323a13761a1a59d514b_10.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_12.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_14.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_15.png)

这种方式的缺点是每次连接都需要手动输入密码，在自动化脚本或频繁连接时很不方便。

![](img/5e9e0cb2154ef323a13761a1a59d514b_17.png)

## 基于公钥的SSH认证原理

![](img/5e9e0cb2154ef323a13761a1a59d514b_19.png)

为了解决密码登录的麻烦，SSH支持基于公钥的认证。其核心优势在于实现免密码登录。

以下是公钥认证的步骤：
1.  **客户端生成密钥对**：在客户端使用 `ssh-keygen` 命令生成一对非对称密钥（公钥和私钥）。
2.  **发送公钥至服务器**：将客户端的公钥手动传输并添加到服务器对应用户的 `~/.ssh/authorized_keys` 文件中。此步骤通常需要基于密码认证来完成首次连接。
3.  **认证过程**：
    *   客户端发起连接，服务器随机生成一个字符串。
    *   服务器用客户端预存的公钥加密该字符串，并发回给客户端。
    *   客户端使用自己的私钥解密，得到原始字符串。
    *   客户端将解密后的字符串发送给服务器进行比对。
    *   如果匹配，则认证成功，建立连接。

![](img/5e9e0cb2154ef323a13761a1a59d514b_21.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_23.png)

整个过程客户端无需输入密码，其安全性依赖于私钥的保密性。

![](img/5e9e0cb2154ef323a13761a1a59d514b_25.png)

## 配置SSH公钥认证实践

接下来，我们通过实际操作来配置公钥认证。

![](img/5e9e0cb2154ef323a13761a1a59d514b_27.png)

### 1. 生成SSH密钥对

![](img/5e9e0cb2154ef323a13761a1a59d514b_29.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_31.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_33.png)

在客户端机器上，使用以下命令生成密钥对。默认使用RSA算法。

![](img/5e9e0cb2154ef323a13761a1a59d514b_35.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_37.png)

```bash
ssh-keygen -t rsa
```
执行命令后，连续按回车接受默认设置（密钥保存路径为 `~/.ssh/id_rsa`，不设置密钥密码）。命令会生成两个文件：
*   `~/.ssh/id_rsa`：私钥文件，必须严格保密。
*   `~/.ssh/id_rsa.pub`：公钥文件，可以发送给任何人。

![](img/5e9e0cb2154ef323a13761a1a59d514b_39.png)

### 2. 将公钥发送到服务器

![](img/5e9e0cb2154ef323a13761a1a59d514b_41.png)

生成密钥对后，需要将公钥发送到目标服务器。推荐使用 `ssh-copy-id` 命令，它可以自动完成公钥的追加工作。

![](img/5e9e0cb2154ef323a13761a1a59d514b_43.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_45.png)

```bash
ssh-copy-id user@server_ip
```
例如，发送到IP为192.168.1.100的root用户：
```bash
ssh-copy-id root@192.168.1.100
```
执行此命令时，会提示输入一次服务器用户的密码。成功后，客户端的公钥就会被添加到服务器上 `~/.ssh/authorized_keys` 文件的末尾。

### 3. 使用公钥登录服务器

![](img/5e9e0cb2154ef323a13761a1a59d514b_47.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_49.png)

公钥配置完成后，再次使用SSH连接时，就会自动尝试公钥认证，无需输入密码。

![](img/5e9e0cb2154ef323a13761a1a59d514b_51.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_52.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_54.png)

```bash
ssh root@192.168.1.100
```

![](img/5e9e0cb2154ef323a13761a1a59d514b_56.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_58.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_60.png)

### 4. 使用非默认密钥文件

![](img/5e9e0cb2154ef323a13761a1a59d514b_62.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_64.png)

如果密钥对不是默认名称或不在默认路径，需要在连接时指定私钥文件。

![](img/5e9e0cb2154ef323a13761a1a59d514b_66.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_68.png)

```bash
ssh -i /path/to/private_key user@server_ip
```
例如，使用自定义密钥 `my_key` 进行连接：
```bash
ssh -i ~/.ssh/my_key root@192.168.1.100
```
相应地，在复制公钥时也需要指定公钥文件：
```bash
ssh-copy-id -i ~/.ssh/my_key.pub root@192.168.1.100
```

![](img/5e9e0cb2154ef323a13761a1a59d514b_72.png)

## SSH客户端常用命令选项

![](img/5e9e0cb2154ef323a13761a1a59d514b_74.png)

![](img/5e9e0cb2154ef323a13761a1a59d514b_76.png)

除了基本连接，SSH客户端还提供了一些有用的选项。

![](img/5e9e0cb2154ef323a13761a1a59d514b_78.png)

以下是常用SSH客户端选项：
*   `-p port`：指定远程服务器的SSH端口（默认为22）。
*   `-v`：启用详细模式，用于调试连接问题。
*   `-i identity_file`：指定用于身份验证的私钥文件。
*   远程执行命令：SSH可以在连接后执行单条命令并退出，适用于自动化任务。
    ```bash
    ssh user@server_ip “ls -l /tmp”
    ```

## SSH服务端配置

![](img/5e9e0cb2154ef323a13761a1a59d514b_82.png)

### 常用服务端配置项

以下是一些重要的配置参数及其说明：
*   `Port 22`：设置SSH服务监听的端口号。改为非默认端口（如8822）可以增强安全性。
*   `PermitRootLogin yes`：是否允许root用户直接登录。建议改为 `no` 以提升安全。
*   `PubkeyAuthentication yes`：是否启用公钥认证。
*   `PasswordAuthentication yes`：是否启用密码认证。在启用公钥认证后，可以考虑禁用此项。
*   `AuthorizedKeysFile .ssh/authorized_keys`：指定存储授权公钥的文件路径。
*   `ClientAliveInterval 60`：服务器每60秒向客户端发送一次保活消息。
*   `ClientAliveCountMax 3`：如果连续3次未收到客户端响应，则断开连接。
*   `UseDNS no`：禁用DNS反向解析，可以加快SSH登录速度。

![](img/5e9e0cb2154ef323a13761a1a59d514b_94.png)

### 配置示例与生效

![](img/5e9e0cb2154ef323a13761a1a59d514b_96.png)

例如，我们希望更改SSH端口并禁止root密码登录：
1.  编辑配置文件：
    ```bash
    vi /etc/ssh/sshd_config
    ```
2.  找到并修改以下行：
    ```
    Port 8822
    PermitRootLogin prohibit-password # 允许root使用公钥登录，但禁止密码登录
    ```
3.  如果系统启用了SELinux，需要为新端口添加规则：
    ```bash
    semanage port -a -t ssh_port_t -p tcp 8822
    ```
4.  如果系统启用了防火墙，需要放行新端口：
    ```bash
    firewall-cmd --permanent --add-port=8822/tcp
    firewall-cmd --reload
    ```
5.  重启SSH服务使配置生效：
    ```bash
    systemctl restart sshd
    ```
**重要**：在重启 `sshd` 服务前，请确保已通过当前连接测试了新配置（例如，在另一个终端用新端口成功连接），或者保留了当前活动会话，以免配置错误导致无法远程登录。
