# 备考红帽认证必修课：P13：2.08-附加权限 📚

![](img/3353efa715b2c1e117d3276d1bf33ce7_1.png)

在本节课中，我们将要学习Linux系统中的附加权限。上一节我们介绍了文件的基本权限和访问控制列表（ACL），它们解决了权限分类不够细致的问题。本节中我们来看看，除了读写执行（RWX）之外，Linux还有哪些特殊的权限标记，它们能实现哪些独特的功能。

![](img/3353efa715b2c1e117d3276d1bf33ce7_4.png)

## 概述 📖

![](img/3353efa715b2c1e117d3276d1bf33ce7_6.png)

Linux文件的基本权限位（RWX）有时无法满足特定的安全或功能需求。因此，系统引入了三种附加权限：Set UID、Set GID和粘滞位（Sticky Bit）。这些权限通过修改用户、组或其他人的执行位（X）来实现特殊功能。本节课我们将重点理解Set GID权限，并完成一道相关的实践题目。

![](img/3353efa715b2c1e117d3276d1bf33ce7_8.png)

## 附加权限简介 🔍

![](img/3353efa715b2c1e117d3276d1bf33ce7_10.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_12.png)

Linux的权限标记共有9位，分别代表所有者（User）、所属组（Group）和其他人（Other）的读（R）、写（W）、执行（X）权限。附加权限没有额外的位置表示，因此它们会覆盖原有的执行位（X），通过将`x`变为`s`或`t`来体现。

以下是三种附加权限：
*   **Set UID (SUID)**：作用于**文件**的所有者执行位。当普通用户执行具有SUID权限的程序时，该程序会暂时获得其**所有者**的权限。例如，`/usr/bin/passwd`命令就具有SUID权限，使得普通用户可以修改自己的密码（密码存储在`/etc/shadow`文件中，而普通用户对此文件并无写权限）。
    *   **设置命令**：`chmod u+s 文件名`
    *   **权限标记**：所有者执行位显示为 `s`（如果原有x权限）或 `S`（如果原有x权限）。
*   **Set GID (SGID)**：作用于**目录**的所属组执行位时，功能特殊，也是我们本节课的重点。它使得在该目录下**新建**的任何文件或子目录，其所属组会自动继承该目录的所属组，而不是创建者的主要组。
    *   **设置命令**：`chmod g+s 目录名`
    *   **权限标记**：所属组执行位显示为 `s`（如果原有x权限）或 `S`（如果原有x权限）。
*   **粘滞位 (Sticky Bit)**：作用于**目录**的其他人执行位。它确保在该目录中，用户只能删除或重命名**自己创建**的文件，而不能删除其他用户的文件。系统临时目录`/tmp`通常就设置有粘滞位。
    *   **设置命令**：`chmod o+t 目录名`
    *   **权限标记**：其他人执行位显示为 `t`（如果原有x权限）或 `T`（如果原有x权限）。

## 实践：创建共用目录 🛠️

![](img/3353efa715b2c1e117d3276d1bf33ce7_14.png)

现在，我们通过一道实践题目来深入理解Set GID权限的应用。题目要求如下：
1.  在`/home`目录下创建名为`tools`的目录。
2.  将此目录的所属组设置为`admins`。
3.  确保`admins`组的成员对该目录拥有读、写和执行访问权限。
4.  确保其他用户（除了root）没有任何权限。
5.  设置该目录，使得在其下创建的任何文件，其所属组自动为`admins`组。

![](img/3353efa715b2c1e117d3276d1bf33ce7_16.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_18.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_20.png)

以下是完成此任务的步骤和命令：

**第一步：创建目录**
```bash
mkdir /home/tools
```

![](img/3353efa715b2c1e117d3276d1bf33ce7_22.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_24.png)

**第二步：更改目录的所属组**
使用`chown`命令将目录的所属组改为`admins`。
```bash
chown :admins /home/tools
```

![](img/3353efa715b2c1e117d3276d1bf33ce7_26.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_28.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_30.png)

**第三步：设置目录的基本权限**
我们需要为`admins`组设置`rwx`权限，同时移除其他人的所有权限。使用`chmod`命令的符号模式可以清晰地做到这一点。
```bash
chmod g=rwx,o= /home/tools
```
*   `g=rwx`：将所属组（`admins`组）的权限设置为读、写、执行。
*   `o=`：将其他人的权限设置为空（即无任何权限）。

![](img/3353efa715b2c1e117d3276d1bf33ce7_32.png)

**第四步：为目录添加Set GID权限**
这是实现“自动继承组所有权”功能的关键。我们为所属组的执行位添加`s`标记。
```bash
chmod g+s /home/tools
```

完成以上步骤后，你可以使用`ls -ld /home/tools`命令查看最终结果。权限位应显示为 `drwxrws---`，并且所属组为`admins`。

现在，任何用户在`/home/tools`目录下创建新文件时，该文件的所属组都会自动设置为`admins`，而不是创建者的主要组。你可以通过以下命令验证：
```bash
touch /home/tools/testfile.txt
ls -l /home/tools/testfile.txt
```
观察输出，`testfile.txt`的所属组应为`admins`。

![](img/3353efa715b2c1e117d3276d1bf33ce7_34.png)

## 权限的数字表示法 🔢

![](img/3353efa715b2c1e117d3276d1bf33ce7_36.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_38.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_40.png)

除了符号表示法（如`u+s`， `g=rwx`），`chmod`命令也支持八进制数字表示法。这对于快速设置组合权限非常方便。

![](img/3353efa715b2c1e117d3276d1bf33ce7_42.png)

![](img/3353efa715b2c1e117d3276d1bf33ce7_44.png)

基本规则如下：
*   **读 (r)** = 4
*   **写 (w)** = 2
*   **执行 (x)** = 1
*   无权限 = 0

权限数字是相应位值的**求和**。例如：
*   `rwx` = 4+2+1 = **7**
*   `rw-` = 4+2+0 = **6**
*   `r-x` = 4+0+1 = **5**

![](img/3353efa715b2c1e117d3276d1bf33ce7_46.png)

一个完整的权限用三个数字表示，分别对应**所有者**、**所属组**和**其他人**。例如，`755` 表示 `rwxr-xr-x`。

**附加权限的数字表示**：
附加权限在三个基本数字**之前**增加一个数字位：
*   **Set UID** = 4
*   **Set GID** = 2
*   **粘滞位** = 1

![](img/3353efa715b2c1e117d3276d1bf33ce7_48.png)

因此，要使用数字法设置我们刚才创建的`/home/tools`目录的权限（`drwxrws---`），可以这样计算：
*   所有者：`rwx` = 7
*   所属组：`rws` (基本`rwx`=7， 加上Set GID) -> 该位整体仍为7，但Set GID由前置数字表示。
*   其他人：`---` = 0
*   附加位：需要Set GID，所以是 2。

最终命令为：
```bash
chmod 2770 /home/tools
```
这个命令一次性设置了Set GID权限，并将基本权限设置为`770`（即所有者`rwx`， 所属组`rwx`， 其他人无权限）。

![](img/3353efa715b2c1e117d3276d1bf33ce7_50.png)

本节课中我们一起学习了Linux的附加权限。我们了解到，为了解决基本RWX权限无法满足的特定场景，Linux引入了Set UID、Set GID和粘滞位这三种特殊权限。我们重点探讨了Set GID权限在目录上的应用——它能使目录下新建的文件自动继承目录的所属组，这对于团队协作和共享目录管理非常有用。通过“创建共用目录”的实践，我们掌握了如何综合运用`mkdir`、`chown`和`chmod`命令来实现复杂的权限控制需求。最后，我们还简要介绍了权限的八进制数字表示法，作为设置权限的另一种高效方式。理解这些概念和操作，对于Linux系统管理和红帽认证考试都至关重要。
