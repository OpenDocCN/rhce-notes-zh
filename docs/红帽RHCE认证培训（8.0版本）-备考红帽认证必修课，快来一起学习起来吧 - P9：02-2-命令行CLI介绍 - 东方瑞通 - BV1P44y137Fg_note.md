## 什么是命令行？ 🔍

![](img/55a87a90873bc9c8098a5209bb037f1b_8.png)

命令行是一个访问Linux系统的接口。Linux系统的构建核心就是围绕着命令行展开的。我们在命令行中执行各种命令来维护操作系统。

![](img/55a87a90873bc9c8098a5209bb037f1b_10.png)

例如，我们打开一个终端窗口，就是下图所示的样子。它用于向计算机系统输入指令。我们之前在这个窗口里修改密码，就是通过命令行完成的。

![](img/55a87a90873bc9c8098a5209bb037f1b_1.png)

我们输入的这些字符串称为**指令**。指令可能是正确的，也可能是错误的。计算机要识别指令，必须遵循一定的规则，不能随意输入。随意输入的字符串，系统是无法识别的。

## Shell程序：命令行的提供者 🐚

![](img/55a87a90873bc9c8098a5209bb037f1b_12.png)

Linux的命令行是由一个叫做 **Shell** 的程序提供的。虽然我们看到的是一个终端窗口，但这个窗口后端的应用程序是哪个程序呢？

## Shell的种类与Bash 🐢

![](img/55a87a90873bc9c8098a5209bb037f1b_17.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_19.png)

Linux发展多年，有很多发行版本，每个版本可能使用不同的Shell程序。在红帽系统中，默认提供的Shell程序是 **GNU Bourne-Again Shell**，简称为 **Bash**。

![](img/55a87a90873bc9c8098a5209bb037f1b_21.png)

Bash是类Unix系统中最成功、使用最广泛的Shell之一，它是早期Shell（sh）的改进和升级版本。

那么，Shell到底扮演什么角色呢？更准确地说，Shell是用户和系统内核之间的**翻译官**。

我们可以这样理解：
*   **内核**：是操作系统的核心，负责管理硬件和基础资源。
*   **Shell**：是包裹在内核外的一层，负责接收和解释用户输入的命令。
*   **命令**：如 `ls`（查看文件）、`passwd`（修改密码）、`cd`（切换目录）等，是用户输入的字符串。

其工作流程如下图所示：

![](img/55a87a90873bc9c8098a5209bb037f1b_12.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_23.png)

1.  用户在Shell程序（终端）中输入一个命令字符串（如 `ls`）。
2.  Shell程序解析这个字符串。
3.  Shell将解析后的指令交给系统内核去执行。
4.  内核执行完毕后，将结果返回给Shell。
5.  Shell将结果显示在终端上，这称为**标准输出**。

了解这个流程很重要。当命令执行出错时，我们需要分析问题是出在命令本身、Shell解析，还是内核执行阶段。

## 终端窗口的基本操作 🖱️

上一节我们介绍了Shell的概念，本节我们来看看如何操作提供Shell环境的终端窗口。

打开终端后，我们可能会对界面感到陌生。下面介绍一些基本操作：

*   **放大/缩小**：点击窗口的放大/缩小按钮，或使用快捷键 `Ctrl + Shift + +`（放大）和 `Ctrl + -`（缩小）。
*   **全屏**：点击全屏按钮或按 `F11` 键。
*   **新建标签页**：点击“文件”菜单选择“新建标签页”，或使用快捷键 `Ctrl + Shift + T`。多个标签页可以通过 `Alt + 数字键`（如Alt+1， Alt+2）快速切换。
*   **新建独立窗口**：点击“文件”菜单选择“新建窗口”，或使用快捷键 `Ctrl + Shift + N`。
*   **分离标签页**：在标签页上右键，可以选择“分离终端”，将其变成一个独立窗口。
*   **设置**：通过“编辑”菜单下的“首选项”，可以调整字体、颜色、背景等终端外观。

## 命令提示符的含义 📍

![](img/55a87a90873bc9c8098a5209bb037f1b_35.png)

终端窗口中闪烁的光标前的那一串字符就是**命令提示符**。它提供了关键信息，格式通常类似 `[user@host directory]$`。

![](img/55a87a90873bc9c8098a5209bb037f1b_36.png)

我们来分解一下 `[kiosk@foundation0 Desktop]$` 这个例子：
*   `kiosk`：代表当前登录的用户账户。
*   `@`：分隔符。
*   `foundation0`：代表当前主机的主机名。
*   `Desktop`：代表当前所在的目录（工作路径）。
*   `$` 或 `#`：提示符结尾符号。`$` 表示当前是**普通用户**，`#` 表示当前是**管理员用户**。

![](img/55a87a90873bc9c8098a5209bb037f1b_38.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_39.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_41.png)

通过命令提示符，我们可以一目了然地知道：在哪个机器上、以什么用户身份、在哪个目录下进行操作。

![](img/55a87a90873bc9c8098a5209bb037f1b_43.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_45.png)

命令提示符的样式可以通过 `PS1` 这个Shell变量来定制。例如，执行 `echo $PS1` 可以查看当前值。修改它可以改变显示内容，比如显示完整路径等。这部分内容我们会在后续深入学习Shell时详细讲解。

## 虚拟终端（TTY） 🖥️➡️🖥️

之前我们操作的是图形界面下的终端。在纯字符界面下，Linux也提供了命令行访问方式，这就是**虚拟终端**。

![](img/55a87a90873bc9c8098a5209bb037f1b_53.png)

在红帽工作站上，我们可以通过快捷键在图形界面和虚拟终端之间切换：
*   `Ctrl + Alt + F2` 到 `Ctrl + Alt + F6`：切换到字符界面的虚拟终端（tty2 - tty6）。
*   `Ctrl + Alt + F1` 或 `Ctrl + Alt + F7`（取决于系统）：切换回图形界面。

在虚拟终端登录后，同样会出现命令提示符。执行 `tty` 命令可以查看当前所在的终端设备名，例如 `/dev/tty3`。

![](img/55a87a90873bc9c8098a5209bb037f1b_55.png)

而在图形界面打开的终端里执行 `tty`，显示的名称通常是 `/dev/pts/0`、`/dev/pts/1` 等（`pts` 指伪终端）。

![](img/55a87a90873bc9c8098a5209bb037f1b_56.png)

简单来说，`tty` 是物理或虚拟的字符终端，`pts` 是图形界面下模拟的终端。对于初学者，知道两者都是能输入命令的终端窗口即可。

## 命令行的结构 🧱

![](img/55a87a90873bc9c8098a5209bb037f1b_64.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_66.png)

有了终端，我们就要执行命令了。那么，Shell是如何解析我们输入的一长串字符的呢？

![](img/55a87a90873bc9c8098a5209bb037f1b_68.png)

一个完整的命令行通常由三部分组成：**命令**、**选项** 和 **参数**。

![](img/55a87a90873bc9c8098a5209bb037f1b_70.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_72.png)

其基本语法可以概括为：
```
命令 [选项...] [参数...]
```

以下是每个部分的解释：

**1. 命令**
命令行中第一个空格前的单词就是需要执行的命令。例如，在 `ls -l /home` 中，`ls` 就是命令。

![](img/55a87a90873bc9c8098a5209bb037f1b_74.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_76.png)

**2. 选项**
选项用于调整命令的行为或输出格式，通常以一个 (`-`) 或两个 (`--`) 横杠开头。
*   例如，`ls -a` 中的 `-a` 选项表示显示所有文件（包括隐藏文件）。
*   `ls -l` 中的 `-l` 选项表示以长列表格式显示文件详细信息。
*   有时，单横杠和双横杠选项意义相同，如 `ls -a` 和 `ls --all`。这是历史遗留的两种风格。

**3. 参数**
参数通常是命令操作的目标对象。
*   例如，`ls /home` 中的 `/home` 就是参数，表示要查看 `/home` 目录下的内容。
*   一个命令可以跟多个参数，如 `ls /home /etc` 表示同时查看两个目录。

![](img/55a87a90873bc9c8098a5209bb037f1b_78.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_80.png)

选项和参数可以组合使用，例如 `ls -la /home` 表示以详细列表格式查看 `/home` 目录下的所有文件。

![](img/55a87a90873bc9c8098a5209bb037f1b_82.png)

## 命令语法与帮助 📖

![](img/55a87a90873bc9c8098a5209bb037f1b_82.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_84.png)

当我们不清楚一个命令怎么用时，可以查看它的帮助。很多命令支持 `--help` 选项。

![](img/55a87a90873bc9c8098a5209bb037f1b_86.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_87.png)

例如，输入 `cat --help` 会显示 `cat` 命令的使用方法。帮助信息中常用一些符号约定：
*   **中括号 `[ ]`**：表示括号内的内容是可选的（即可加可不加的选项或参数）。
*   **竖线 `|`**：表示“或”，在多个选项中只能选一个。
*   **省略号 `...`**：表示相同类型的项目可以重复多个。

例如，在 `cat [OPTION]... [FILE]...` 中：
*   `[OPTION]...` 表示可以加多个选项（如 `-A -n`）。
*   `[FILE]...` 表示可以查看多个文件的内容。

![](img/55a87a90873bc9c8098a5209bb037f1b_89.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_93.png)

## 远程访问命令行 🌐

到目前为止，我们操作的都是本地机器。在实际工作中，我们经常需要远程管理服务器。Linux下最常用的远程登录工具是 **SSH**。

![](img/55a87a90873bc9c8098a5209bb037f1b_93.png)

基本用法是：
```bash
ssh 用户名@主机名或IP地址
```

![](img/55a87a90873bc9c8098a5209bb037f1b_95.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_96.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_98.png)

例如：
*   `ssh student@servera`：以 `student` 用户身份登录到 `servera` 这台机器。
*   首次连接时会询问是否信任主机密钥，输入 `yes` 即可。
*   接着会提示输入对应用户的密码。

认证成功后，你就会进入远程服务器的命令行环境，命令提示符会变为远程服务器的信息。操作完成后，输入 `exit` 命令或按 `Ctrl + D` 即可退出远程连接，回到本地终端。

SSH还支持更安全的密钥对认证方式（无需每次输入密码），这将在后续章节详细介绍。

## 总结 🎯

![](img/55a87a90873bc9c8098a5209bb037f1b_102.png)

![](img/55a87a90873bc9c8098a5209bb037f1b_104.png)

本节课中我们一起学习了Linux命令行的基础知识：
1.  **命令行**是用户与Linux系统交互的核心接口。
2.  **Shell** 是提供命令行的程序，它解析用户输入，与内核通信。红帽系统默认使用 **Bash**。
3.  我们可以在**图形终端**或**虚拟终端**中使用命令行。
4.  **命令提示符** 显示了当前用户、主机名和工作目录等重要信息。
5.  一个命令行通常由 **命令**、**选项** 和 **参数** 三部分组成。
6.  使用 `--help` 可以查看命令的基本用法和语法说明。
7.  通过 **SSH** 工具，我们可以安全地远程登录和管理其他Linux服务器。
