在RHCSA的第二个虚拟机中，系统通常会准备三块磁盘：`/dev/vda`、`/dev/vdb` 和 `/dev/vdc`。

![](img/4d59b8a21fd5834a341a13043b3345ae_32.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_33.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_35.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_37.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_39.png)

*   **`/dev/vda`**：这是系统盘，包含了启动分区、根分区和交换分区。我们无需对其进行操作。
*   **`/dev/vdb`**：这是我们的“工作盘”。考试中多道题目会共用此磁盘：
    *   它已有一个分区（如`/dev/vdb1`），上面建立了卷组和逻辑卷（如`vo`），用于“**调整逻辑卷大小**”题目。**注意：** 此逻辑卷已格式化并存有数据，切勿重新格式化。
    *   我们需要在此磁盘上**新增分区**，用于“**添加交换分区**”和“**创建新逻辑卷**”题目。
*   **`/dev/vdc`**：这是一块**完全空闲**的磁盘，专门预留给“**创建VDO卷**”题目使用。请勿在其他题目中使用此磁盘，务必为其保留。

## 磁盘使用流程概述

![](img/4d59b8a21fd5834a341a13043b3345ae_41.png)

在使用一块新磁盘前，需要遵循以下标准流程：
1.  **识别磁盘**：系统启动时内核自动识别。我们可以使用命令查看。
2.  **创建分区**：使用分区工具在磁盘上划分逻辑区域。
3.  **格式化**：在分区上创建特定的文件系统结构。
4.  **挂载**：将格式化好的分区关联到系统目录树，方可存取数据。

![](img/4d59b8a21fd5834a341a13043b3345ae_45.png)

接下来，我们详细看看每个步骤的工具与操作。

## 识别磁盘与分区

要查看系统已识别的磁盘和分区信息，可以使用以下命令：

**命令示例：**
```bash
lsblk
```
或
```bash
fdisk -l
```
`lsblk` 命令以树状图形式清晰展示磁盘和分区关系，而 `fdisk -l` 则列出更详细的分区表信息。

## 分区工具

创建或修改分区主要使用 `fdisk` 或 `parted` 工具。

![](img/4d59b8a21fd5834a341a13043b3345ae_49.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_51.png)

**`fdisk` 命令**：传统交互式分区工具，适合新手。其特点是**修改后需输入 `w` 命令才会保存**，输入 `q` 则不保存退出，安全性较高。
```bash
fdisk /dev/vdb
```

![](img/4d59b8a21fd5834a341a13043b3345ae_53.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_54.png)

**`parted` 命令**：更强大的分区工具，支持更多分区表类型（如GPT）。其特点是**修改立即生效**，适合有经验的管理员，但风险较高。
```bash
parted /dev/vdb
```

![](img/4d59b8a21fd5834a341a13043b3345ae_56.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_58.png)

**重要提示：** 分区完成后，内核可能无法立即识别新分区表。最稳妥的方法是重启系统 (`reboot`)。若不想重启，可尝试让内核重新读取分区表：
```bash
partprobe /dev/vdb
```

## 格式化工具

![](img/4d59b8a21fd5834a341a13043b3345ae_60.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_61.png)

分区完成后，需要格式化以创建文件系统。使用 `mkfs` (make filesystem) 命令族。

![](img/4d59b8a21fd5834a341a13043b3345ae_63.png)

**命令格式：**
```bash
mkfs.文件系统类型 /dev/分区名
```
例如，格式化为 xfs 文件系统：
```bash
mkfs.xfs /dev/vdb3
```
要查看系统支持哪些 `mkfs` 工具，可以输入 `mkfs` 后按两次Tab键。

## 实战：添加交换分区

现在，我们运用所学知识来完成“添加交换分区”的题目。题目要求是：在系统中添加一个512MB的交换分区，并确保其在系统启动时自动挂载。

以下是操作步骤：

![](img/4d59b8a21fd5834a341a13043b3345ae_65.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_66.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_68.png)

**第一步：在 `/dev/vdb` 上创建新分区**
使用 `fdisk` 为交换分区划出512MB空间。
```bash
fdisk /dev/vdb
```
在交互界面中：
1.  输入 `n` 创建新分区。
2.  选择分区编号（例如2），直接回车使用默认值。
3.  设置起始扇区，直接回车使用默认值。
4.  设置结束位置，输入 `+512M` 指定大小为512MB。
5.  输入 `p` 查看分区表确认。
6.  输入 `w` 保存并退出。

**第二步：格式化新分区为交换空间**
交换分区需要特殊的格式化处理，使用 `mkswap` 命令。
```bash
mkswap /dev/vdb2
```

**第三步：启用交换分区**
使用 `swapon` 命令激活新创建的交换分区。
```bash
swapon /dev/vdb2
```
使用 `swapon -s` 命令可以验证新的交换分区是否已启用。

![](img/4d59b8a21fd5834a341a13043b3345ae_70.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_72.png)

**第四步：配置开机自动挂载**
编辑 `/etc/fstab` 文件，添加一行配置，使系统启动时能自动启用此交换分区。
```bash
vim /etc/fstab
```
在文件末尾添加如下行（可复制已有的交换分区行进行修改）：
```
/dev/vdb2 swap swap defaults 0 0
```
保存并退出编辑器。

![](img/4d59b8a21fd5834a341a13043b3345ae_74.png)

![](img/4d59b8a21fd5834a341a13043b3345ae_76.png)

**第五步：验证配置**
为了不重启系统而验证配置是否正确，可以先禁用该交换分区，然后使用 `-a` 参数让 `swapon` 读取 `/etc/fstab` 文件并启用所有配置的交换分区。
```bash
swapoff /dev/vdb2
swapon -a
swapon -s # 再次检查，应能看到 /dev/vdb2
```
