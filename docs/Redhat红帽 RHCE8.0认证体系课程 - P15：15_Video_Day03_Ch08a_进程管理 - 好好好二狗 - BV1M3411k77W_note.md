# Redhat红帽 RHCE8.0认证体系课程：P15：进程管理基础 🖥️

在本节课中，我们将要学习Linux系统中的进程管理。进程是系统运行的核心，理解如何查看、分析和控制进程是系统管理员必备的技能。我们将从进程的基本概念开始，逐步学习如何查看进程信息、分析资源占用以及理解进程状态。

## 什么是进程？

我们从Windows系统来看，打开任务管理器，在“进程”选项卡中可以看到一个个运行的程序，包括应用和后台服务。进程可以理解为：通过执行一个静态的程序文件而触发的一个动态运行实例。静态程序是存储在磁盘上的文件，而进程是这个文件被加载到内存中运行时的动态实体。

例如，运行腾讯会议程序，是通过执行 `WeMeet.app` 这个文件来触发其进程的。一个进程会占用CPU、内存、磁盘I/O和网络带宽等系统资源。在系统中，进程数量可能很多，例如当前演示的电脑就有241个进程和3000多个线程。进程内部可以产生多个线程，即子任务。

我们需要关注哪些进程占用了较多资源，通常从CPU、内存和硬盘I/O这三个方面来考虑。例如，某个程序占用了10%的CPU，或者大量内存。同时，也要判断进程的运行是否正常，是否存在异常进程（如木马、挖矿程序），并评估系统当前和未来的资源是否充足。

## 进程的拥有者

![](img/0562c49032108c344d7011d9a8436d1b_1.png)

在Linux系统中，进程的拥有者不一定就是执行该文件的用户。这与我们之前讨论的SUID权限有关。进程在读取或写入文件时，是以其拥有者的身份进行的，因此该拥有者必须对目标文件拥有相应的权限。

![](img/0562c49032108c344d7011d9a8436d1b_3.png)

![](img/0562c49032108c344d7011d9a8436d1b_5.png)

例如，以root用户身份启动Nginx服务，但Nginx进程的拥有者通常是专门的 `nginx` 用户，而不是root。同样，Apache服务通常由 `apache` 用户运行。服务的拥有者通常在配置文件中定义，例如在 `httpd.conf` 中定义 `User` 和 `Group` 字段。

### 使用 `ps` 命令

![](img/0562c49032108c344d7011d9a8436d1b_67.png)

![](img/0562c49032108c344d7011d9a8436d1b_69.png)

`ps` 命令用于显示当前进程的状态快照。

![](img/0562c49032108c344d7011d9a8436d1b_77.png)

![](img/0562c49032108c344d7011d9a8436d1b_79.png)

![](img/0562c49032108c344d7011d9a8436d1b_81.png)

![](img/0562c49032108c344d7011d9a8436d1b_83.png)

![](img/0562c49032108c344d7011d9a8436d1b_85.png)

*   **基本用法**：直接输入 `ps` 会显示当前终端会话下的进程。
    ```bash
    ps
    ```
    输出包含以下关键字段：
    *   `PID`: 进程的唯一ID。
    *   `TTY`: 进程所在的终端。如果显示 `?`，则表示该进程是后台守护进程，不依赖于任何终端。
    *   `TIME`: 进程累计占用CPU的时间。
    *   `CMD`: 启动该进程的命令名称。

*   **查看详细信息**：使用 `ps -lf` 或 `ps aux` 可以查看更全面的信息。
    ```bash
    ps aux
    ```
    `ps aux` 的输出包含用户（`USER`）、CPU占用率（`%CPU`）、内存占用率（`%MEM`）、虚拟内存大小（`VSZ`）、驻留内存大小（`RSS`）、终端（`TTY`）、状态（`STAT`）、启动时间（`START`）和完整命令（`COMMAND`）等。

### 进程状态（STAT）

在 `ps` 命令的输出中，`STAT` 列表示进程状态，常见状态有：

![](img/0562c49032108c344d7011d9a8436d1b_87.png)

![](img/0562c49032108c344d7011d9a8436d1b_89.png)

![](img/0562c49032108c344d7011d9a8436d1b_91.png)

*   **R (Running)**: 正在运行或可运行（在运行队列中等待）。
*   **S (Sleeping)**: 睡眠状态，可中断，正在等待某个事件完成（如I/O操作）。
*   **D (Uninterruptible Sleep)**: 不可中断的睡眠，通常是在等待硬件I/O，不响应信号。
*   **T (Stopped)**: 进程被停止（如通过 `Ctrl+Z` 发送了SIGTSTP信号）。
*   **Z (Zombie)**: 僵尸进程，进程已终止，但其父进程尚未回收它。

### 使用 `pstree` 命令

![](img/0562c49032108c344d7011d9a8436d1b_106.png)

![](img/0562c49032108c344d7011d9a8436d1b_108.png)

`pstree` 命令以树状图形式显示进程间的父子关系，让进程层次结构一目了然。
```bash
pstree -aup
```
参数 `-a` 显示完整命令，`-u` 显示用户，`-p` 显示PID。这有助于理解系统服务（如 `systemd`）如何派生和管理子进程。

### 使用 `top` 命令

`top` 命令提供系统资源的动态实时视图，类似于Windows的任务管理器。

![](img/0562c49032108c344d7011d9a8436d1b_110.png)

直接运行 `top` 会进入一个交互式界面，显示：
*   系统运行时间、登录用户数、平均负载（1分钟、5分钟、15分钟）。
*   CPU使用情况（用户空间、内核空间、空闲等）。
*   内存和交换空间使用情况。
*   下方是一个不断刷新的进程列表，默认按CPU使用率排序。

![](img/0562c49032108c344d7011d9a8436d1b_112.png)

![](img/0562c49032108c344d7011d9a8436d1b_114.png)

常用交互命令（在 `top` 运行时按）：
*   `P`: 按CPU使用率排序。
*   `M`: 按内存使用率排序。
*   `k`: 终止一个进程（需要输入PID）。
*   `q`: 退出 `top`。

![](img/0562c49032108c344d7011d9a8436d1b_116.png)

![](img/0562c49032108c344d7011d9a8436d1b_118.png)

![](img/0562c49032108c344d7011d9a8436d1b_120.png)

![](img/0562c49032108c344d7011d9a8436d1b_122.png)

`top` 命令也支持非交互模式，例如 `top -d 1 -n 3` 表示每1秒刷新一次，总共刷新3次后退出。

## 总结

![](img/0562c49032108c344d7011d9a8436d1b_130.png)

本节课我们一起学习了Linux进程管理的基础知识。我们首先理解了进程是程序运行的动态实例，会消耗系统资源。然后，我们探讨了进程的拥有者概念，这对于文件访问权限至关重要。接着，我们学习了触发进程的两种主要方式。最后，我们重点掌握了使用 `ps`、`pstree` 和 `top` 命令来查看和分析进程信息、状态及系统资源使用情况。这些是监控系统健康、排查性能问题和进行日常运维的基础技能。
