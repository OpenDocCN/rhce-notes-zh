在本节课中，我们将要学习Linux系统中至关重要的权限管理。权限是控制用户对文件或目录能执行哪些操作的核心机制。理解权限管理，能帮助我们更好地结合之前学习的用户和组管理知识，实现对系统资源的精细化控制。

## 归属关系 👥

![](img/839e61fdc0537fff9b27d73532e9fd38_11.png)

![](img/839e61fdc0537fff9b27d73532e9fd38_13.png)

![](img/839e61fdc0537fff9b27d73532e9fd38_15.png)

上一节我们介绍了用户和组，本节中我们来看看权限的归属关系。任何文件或目录都有三类归属关系：所有者、所有组和其他人。

*   **所有者**：文件的创建者，通常拥有对该文件的最高控制权。用字母 **U** 表示。
*   **所有组**：文件所属的用户组，组内的成员可以继承该组的权限。用字母 **G** 表示。
*   **其他人**：既不是文件所有者，也不在文件所属组内的所有其他用户。用字母 **O** 表示。

我们可以使用 `ls -l` 命令查看文件的归属关系。例如，`-rw-r--r-- 1 root root` 表示文件所有者为 `root`，所有组也为 `root`。

## 基本权限解读 📖

文件的权限由9个字符表示，每3个字符为一组，分别对应所有者、所有组和其他人的权限。

权限顺序固定为：**r**（读）、**w**（写）、**x**（执行）。如果某个位置是 `-`，则表示没有对应权限。

![](img/839e61fdc0537fff9b27d73532e9fd38_17.png)

![](img/839e61fdc0537fff9b27d73532e9fd38_19.png)

![](img/839e61fdc0537fff9b27d73532e9fd38_21.png)

以下是三类权限的具体含义：

*   **r (Read)**：读取权限。对于文件，表示可以查看文件内容；对于目录，表示可以列出目录内的文件列表。
*   **w (Write)**：写入权限。对于文件，表示可以修改文件内容；对于目录，表示可以在目录内创建、删除、重命名文件。
*   **x (Execute)**：执行权限。对于文件，表示可以运行该文件（如脚本或程序）；对于目录，表示可以进入（`cd`）该目录。

例如，权限 `rw-r--r--` 解读为：
*   所有者（`root`）: `rw-` 可读、可写，但不可执行。
*   所有组（`root组`）: `r--` 仅可读。
*   其他人: `r--` 仅可读。

## 修改归属关系 (chown) 🔄

在设置权限前，有时需要先修改文件的归属关系。这可以通过 `chown` 命令实现。

命令格式如下：
```bash
chown [选项] 所有者[:所属组] 文件或目录
```

以下是几种常见用法：

*   **同时修改所有者和所属组**：`chown tom:devops hello.txt`
*   **仅修改所有者**：`chown root hello.txt`
*   **仅修改所属组**：`chown :devops hello.txt`

选项 `-R` 用于递归修改目录及其内部所有内容的归属关系。

## 修改权限 (chmod) ⚙️

核心的权限修改命令是 `chmod`。它有两种主要的权限表示法：字母法和数字法。

### 字母表示法

使用 `u`（所有者）、`g`（所有组）、`o`（其他人）、`a`（所有人）来指定对象，用 `+`（添加）、`-`（移除）、`=`（精确设置）来指定操作，用 `r`、`w`、`x` 来指定权限。

![](img/839e61fdc0537fff9b27d73532e9fd38_23.png)

以下是常见操作示例：

![](img/839e61fdc0537fff9b27d73532e9fd38_25.png)

*   **给所有者添加执行权限**：`chmod u+x hello.sh`
*   **移除所有组的写权限**：`chmod g-w hello.txt`
*   **设置其他人只有读权限**：`chmod o=r hello.txt`
*   **一次性为多类用户设置权限**：`chmod u=rwx,g=rx,o=r hello.sh`

### 数字表示法

![](img/839e61fdc0537fff9b27d73532e9fd38_27.png)

![](img/839e61fdc0537fff9b27d73532e9fd38_29.png)

每个权限对应一个数字：**r=4**，**w=2**，**x=1**，**无权限=0**。将三类用户的权限数字相加，得到一个三位数。

例如，权限 `rwxr-xr--` 对应的数字计算如下：
*   所有者: r(4)+w(2)+x(1) = **7**
*   所有组: r(4)+x(1) = **5**
*   其他人: r(4) = **4**
因此，该权限的数字表示为 **754**。

设置权限时，直接使用该数字即可：`chmod 754 hello.sh`

![](img/839e61fdc0537fff9b27d73532e9fd38_31.png)

**注意**：使用数字法时，必须指定完整的三位数字，即使某一部分不想改变，也需要写出其当前值（或计算出的值）。

## 文件与目录权限的差异 📁 vs 📄

![](img/839e61fdc0537fff9b27d73532e9fd38_39.png)

![](img/839e61fdc0537fff9b27d73532e9fd38_41.png)

文件与目录的 `r`、`w`、`x` 权限含义有根本区别，这是权限管理的难点和重点。

以下是核心差异总结：

| 权限 | 对文件的影响 | 对目录的影响 |
| :--- | :--- | :--- |
| **r (读)** | 可读取文件内容 | 可列出目录内容（如 `ls` 命令） |
| **w (写)** | 可修改文件内容 | 可在目录内**创建、删除、重命名**文件 |
| **x (执行)** | 可执行文件（如脚本） | 可进入该目录（如 `cd` 命令） |

**关键理解**：用户能否删除一个文件，**不取决于**他对该文件本身的权限，而取决于他对该文件**所在目录**是否拥有 **w** 权限。

## 默认权限与umask 🎭

新创建的文件和目录会有一个默认权限。这个默认权限由系统的 `umask`（权限掩码）值决定。`umask` 指定了要从默认权限中“减去”的权限。

![](img/839e61fdc0537fff9b27d73532e9fd38_43.png)

*   查看当前umask：`umask` 或 `umask -S`（更易读的字母形式）。
*   通常目录的默认权限是 `777`，文件的默认权限是 `666`。
*   `umask` 值（如 `022`）表示：创建目录时，权限为 `777 - 022 = 755`（`rwxr-xr-x`）；创建文件时，权限为 `666 - 022 = 644`（`rw-r--r--`）。

![](img/839e61fdc0537fff9b27d73532e9fd38_45.png)

![](img/839e61fdc0537fff9b27d73532e9fd38_47.png)

一般无需修改系统默认的 `umask` 值。

![](img/839e61fdc0537fff9b27d73532e9fd38_49.png)
