## 系统重要目录详解

![](img/1327ce911fda6912259486f774e86c81_10.png)

![](img/1327ce911fda6912259486f774e86c81_12.png)

上一节我们介绍了分区与挂载的基本概念，本节中我们来看看Linux系统根目录下一些关键的子目录及其作用。

![](img/1327ce911fda6912259486f774e86c81_14.png)

![](img/1327ce911fda6912259486f774e86c81_16.png)

以下是Linux系统中一些核心目录的说明：

![](img/1327ce911fda6912259486f774e86c81_18.png)

*   **`/root`**：这是超级用户（root）的家目录。家目录是用户登录系统后默认所在的个人工作空间。
*   **`/home`**：这是所有普通用户的家目录所在地。每个用户都会在此目录下拥有一个以自己用户名命名的子目录作为其家目录。例如，用户 `admin` 的家目录路径是 `/home/admin`。
*   **`/usr`**：此目录用于存放系统软件和应用程序。它包含几个重要的子目录：
    *   `/usr/bin`：存放普通用户可执行的命令文件。例如，`ls` 命令的文件通常位于此处。
    *   `/usr/sbin`：存放系统管理员（root）执行的系统管理命令。例如，磁盘分区命令 `fdisk`。
    *   `/usr/local`：通常用于存放用户自行编译安装的第三方软件。
*   **`/etc`**：此目录存放系统的配置文件。修改这些文件可以改变系统或软件的行为。例如，主机名配置文件是 `/etc/hostname`，网络配置文件通常在 `/etc/sysconfig/network-scripts/` 目录下。
*   **`/var`**：此目录存放经常变化的（Variable）数据，例如系统日志文件、网站内容、邮件队列等。
*   **`/tmp`**：此目录是系统的临时文件目录，用于存放程序运行时产生的临时文件。系统会定期清理此目录。注意，它并非类似Windows的“回收站”。
*   **`/boot`**：此目录存放系统启动时所需的核心文件，如内核镜像、引导加载程序（GRUB）配置文件等。
*   **`/dev`**：此目录包含了所有的设备文件。在Linux中，硬件设备（如硬盘、键盘）都被抽象为文件，存放在这里。
*   **`/proc` 和 `/sys`**：这两个目录比较特殊，它们不是真实的磁盘目录，而是内核提供的接口，映射了当前系统的运行状态和内存中的数据。`/proc` 主要反映进程信息，`/sys` 主要反映内核和硬件信息。

![](img/1327ce911fda6912259486f774e86c81_20.png)

![](img/1327ce911fda6912259486f774e86c81_22.png)

对于初学者，无需死记硬背所有目录。随着日常使用，你会自然而然地熟悉它们。另外，根目录下的 `/bin`、`/sbin`、`/lib`、`/lib64` 通常是 `/usr` 目录下对应目录的软链接（类似于Windows的快捷方式）。例如，`/bin` 通常链接到 `/usr/bin`。

## 文件名与路径规则

在创建文件或目录时，需要遵循一些基本规则。

![](img/1327ce911fda6912259486f774e86c81_26.png)

以下是关于文件名和路径的注意事项：

![](img/1327ce911fda6912259486f774e86c81_28.png)

![](img/1327ce911fda6912259486f774e86c81_30.png)

*   **长度限制**：文件名（包括路径）的长度不能超过255个字符。
*   **字符有效性**：几乎所有的字符都可以用于命名，包括特殊字符（如 `*`、`?`、`$`）。但斜杠 `/` 除外，因为它被用作路径分隔符和根目录的标识。
*   **命名建议**：为了清晰和易于管理，建议使用有意义的英文或拼音命名，避免使用难以辨识的特殊字符。
*   **大小写敏感**：Linux系统严格区分文件名的大小写。例如，`File.txt`、`file.txt` 和 `FILE.TXT` 是三个不同的文件。

## 相对路径与绝对路径

![](img/1327ce911fda6912259486f774e86c81_38.png)

![](img/1327ce911fda6912259486f774e86c81_40.png)

理解了目录结构后，我们需要学习如何定位文件，这就涉及到路径。路径分为绝对路径和相对路径两种。

![](img/1327ce911fda6912259486f774e86c81_42.png)

![](img/1327ce911fda6912259486f774e86c81_44.png)

![](img/1327ce911fda6912259486f774e86c81_46.png)

![](img/1327ce911fda6912259486f774e86c81_48.png)

**绝对路径**是从根目录 `/` 开始的完整路径。它的特点是**在任何当前工作目录下都能唯一、准确地指向同一个文件或目录**。

![](img/1327ce911fda6912259486f774e86c81_50.png)

![](img/1327ce911fda6912259486f774e86c81_52.png)

**公式**：`绝对路径 = / + 目录1/目录2/.../文件名`

**示例代码**：`ls /etc/sysconfig/network-scripts/`

**相对路径**是相对于当前工作目录的路径。它不以 `/` 开头。它的有效性**依赖于你当前所在的位置**。

![](img/1327ce911fda6912259486f774e86c81_54.png)

**公式**：`相对路径 = [./]目录A/目录B/.../文件名` （开头的 `./` 表示当前目录，通常可以省略）

![](img/1327ce911fda6912259486f774e86c81_56.png)

**示例代码**：假设当前目录是 `/etc`，那么 `sysconfig/network-scripts/` 就是一个相对路径。

为了更直观地理解，我们来看一个例子。假设你想查看 `/etc/sysconfig/network-scripts/` 目录下的内容：

*   使用绝对路径：无论你在系统的哪个位置（例如 `/home` 或 `/tmp`），命令 `ls /etc/sysconfig/network-scripts/` 都能正确执行。
*   使用相对路径：如果你当前就在 `/etc` 目录下，那么使用 `ls sysconfig/network-scripts/` 会更简洁。但如果你在 `/home` 目录下执行同样的相对路径命令，系统就会报错，因为在 `/home` 目录下找不到 `sysconfig` 这个子目录。

![](img/1327ce911fda6912259486f774e86c81_58.png)

![](img/1327ce911fda6912259486f774e86c81_60.png)

简单来说，绝对路径像是一个完整的邮寄地址（如“中国北京市海淀区xx路xx号”），而相对路径像是基于地标的指引（如“从公司门口右转的便利店”）。后者更简洁，但只有知道“公司门口”这个当前位置时才有效。

## `cd` 命令与路径实践

`cd` (Change Directory) 命令用于切换当前工作目录，是使用路径最频繁的命令之一。

以下是 `cd` 命令的常见用法示例：

*   **`cd [目录路径]`**：切换到指定路径。可以是绝对路径或相对路径。
    *   `cd /tmp` （使用绝对路径切换到 `/tmp`）
    *   `cd etc` （使用相对路径，切换到当前目录下的 `etc` 子目录）
*   **`cd` 或 `cd ~`**：直接返回当前用户的家目录。这是“一键回家”的快捷方式。
    *   `cd` （返回当前用户家目录）
    *   `cd ~` （同上）
*   **`cd ~用户名`**：切换到指定用户的家目录。
    *   `cd ~admin` （切换到用户 `admin` 的家目录，即 `/home/admin`）
*   **`cd ..`**：返回上一级目录（父目录）。
    *   在 `/etc/sysconfig` 目录下执行 `cd ..`，会切换到 `/etc`。
*   **`cd -`**：在两个最近工作过的目录之间快速切换。
    *   从 `/home` 切换到 `/etc` 后，执行 `cd -` 会立刻跳回 `/home`；再执行一次 `cd -` 又会跳回 `/etc`。
*   **`cd .`**：切换到当前目录（这个操作本身不会改变位置，但在某些脚本或命令组合中有用）。`.` 代表当前目录本身。

**代码示例**：
```bash
pwd                     # 显示当前目录，例如 /home/user
cd /etc                 # 使用绝对路径切换到 /etc
pwd                     # 输出 /etc
cd sysconfig            # 使用相对路径切换到 /etc/sysconfig
pwd                     # 输出 /etc/sysconfig
cd ..                   # 返回上一级目录
pwd                     # 输出 /etc
cd ~                    # 返回家目录 /home/user
pwd                     # 输出 /home/user
cd -                    # 跳回上一个目录 /etc
pwd                     # 输出 /etc
```
