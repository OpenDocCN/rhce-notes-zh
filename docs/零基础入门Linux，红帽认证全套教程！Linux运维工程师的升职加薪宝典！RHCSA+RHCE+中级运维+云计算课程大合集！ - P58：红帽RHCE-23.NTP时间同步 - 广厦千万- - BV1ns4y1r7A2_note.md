## 为什么需要时间同步？
在企业的集群环境中，通常会有多台服务器协同工作。例如，一组服务器提供网站服务，另一组服务器提供数据库服务。为了让这些服务器能够高效协同工作，它们的时间必须保持一致。如果每台服务器的时间都不同，例如一台是2018年，另一台是2022年，它们将无法有效协同工作。因此，确保所有服务器的时间同步是集群管理的基础。

---

## 什么是NTP？
NTP（Network Time Protocol，网络时间协议）是一种用于同步计算机系统时间的协议。它通过互联网或局域网与时间服务器通信，确保系统时间的准确性。NTP服务器的时间非常精确，据说几百年都不会误差一秒，远优于手动配置的时间。

## 在CentOS中配置NTP客户端
上一节我们介绍了NTP的基本概念，本节中我们来看看如何在CentOS系统中配置NTP客户端，使其自动与公共NTP服务器同步时间。

![](img/0d72895c0c58aa5270004491211cf388_10.png)

### 安装NTP软件包
在CentOS 7中，NTP服务通常默认安装。如果未安装，可以使用以下命令安装：
```bash
yum -y install chrony
```

![](img/0d72895c0c58aa5270004491211cf388_12.png)

![](img/0d72895c0c58aa5270004491211cf388_14.png)

![](img/0d72895c0c58aa5270004491211cf388_16.png)

### 启动NTP服务
安装完成后，启动NTP服务并设置为开机自启：
```bash
systemctl start chronyd
systemctl enable chronyd
```

### 验证时间同步
NTP服务安装并启动后，会自动与配置文件中的NTP服务器同步时间。可以通过以下命令查看当前时间：
```bash
date
```
如果时间不正确，可以手动修改时间，然后观察NTP服务是否自动将其同步为标准时间。

## 配置NTP服务器
上一节我们学习了如何配置NTP客户端，本节中我们来看看如何配置NTP服务器，使其作为内网的时间源。

![](img/0d72895c0c58aa5270004491211cf388_18.png)

### 修改NTP配置文件
NTP的主配置文件位于`/etc/chrony.conf`。默认情况下，该文件配置了与公共NTP服务器同步。以下是配置步骤：

![](img/0d72895c0c58aa5270004491211cf388_20.png)

1. 打开配置文件：
   ```bash
   vi /etc/chrony.conf
   ```

![](img/0d72895c0c58aa5270004491211cf388_22.png)

![](img/0d72895c0c58aa5270004491211cf388_23.png)

2. 注释掉默认的公共NTP服务器，并添加阿里云的NTP服务器：
   ```bash
   # server 0.centos.pool.ntp.org iburst
   # server 1.centos.pool.ntp.org iburst
   # server 2.centos.pool.ntp.org iburst
   # server 3.centos.pool.ntp.org iburst
   server ntp.aliyun.com iburst
   server ntp1.aliyun.com iburst
   ```

![](img/0d72895c0c58aa5270004491211cf388_25.png)

![](img/0d72895c0c58aa5270004491211cf388_27.png)

![](img/0d72895c0c58aa5270004491211cf388_29.png)

3. 允许内网客户端同步时间：
   在配置文件中添加以下行，允许所有IP地址的客户端同步时间：
   ```bash
   allow 0.0.0.0/0
   ```

![](img/0d72895c0c58aa5270004491211cf388_30.png)

4. 保存并退出配置文件。

### 重启NTP服务
修改配置文件后，重启NTP服务使配置生效：
```bash
systemctl restart chronyd
```

### 验证NTP服务器状态
使用以下命令查看NTP服务器的同步状态：
```bash
chronyc sources -v
```
如果输出中显示`*`标记，表示当前正在与指定的NTP服务器同步时间。

![](img/0d72895c0c58aa5270004491211cf388_32.png)

## 内网NTP中继配置
上一节我们配置了NTP服务器，本节中我们来看看如何在内网中配置NTP中继，使内网服务器通过中继服务器同步时间。

### 内网NTP中继架构
在企业内网中，如果服务器无法直接访问外网，可以通过一台能够访问外网的服务器作为NTP中继。具体架构如下：
1. 中继服务器与外网NTP服务器（如阿里云NTP）同步时间。
2. 内网服务器与中继服务器同步时间。

![](img/0d72895c0c58aa5270004491211cf388_34.png)

![](img/0d72895c0c58aa5270004491211cf388_36.png)

### 配置内网客户端
在内网客户端上，修改NTP配置文件，指定中继服务器的IP地址：
1. 打开配置文件：
   ```bash
   vi /etc/chrony.conf
   ```

![](img/0d72895c0c58aa5270004491211cf388_38.png)

![](img/0d72895c0c58aa5270004491211cf388_40.png)

2. 注释掉默认的NTP服务器，添加中继服务器的IP地址：
   ```bash
   # server 0.centos.pool.ntp.org iburst
   # server 1.centos.pool.ntp.org iburst
   # server 2.centos.pool.ntp.org iburst
   # server 3.centos.pool.ntp.org iburst
   server 192.168.0.40 iburst
   ```

![](img/0d72895c0c58aa5270004491211cf388_42.png)

3. 保存并退出配置文件。

![](img/0d72895c0c58aa5270004491211cf388_44.png)

![](img/0d72895c0c58aa5270004491211cf388_46.png)

4. 重启NTP服务：
   ```bash
   systemctl restart chronyd
   ```

![](img/0d72895c0c58aa5270004491211cf388_48.png)

![](img/0d72895c0c58aa5270004491211cf388_50.png)

### 验证内网客户端同步状态
使用以下命令查看内网客户端是否与中继服务器同步时间：
```bash
chronyc sources -v
```
如果输出中显示`*`标记，并且IP地址为中继服务器的IP，表示同步成功。

![](img/0d72895c0c58aa5270004491211cf388_52.png)
