## SCP：简单文件复制 🔄

上一节我们介绍了SSH协议，本节中我们来看看基于SSH的`scp`命令。`scp`用于在本地主机和远程主机之间复制文件，其语法与`ssh`类似。

**基本命令格式如下：**
```bash
scp [选项] 源文件 目标地址
```
或
```bash
scp [选项] 目标地址 源文件
```

![](img/2bb31c823bb2669b75a5bb57f8b8f902_7.png)

以下是`scp`的常用选项：
*   `-P 端口号`：指定SSH端口（默认22）。
*   `-q`：静默模式，不显示任何输出信息。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_9.png)

### 使用示例

![](img/2bb31c823bb2669b75a5bb57f8b8f902_11.png)

**1. 从远程主机拉取文件到本地**
假设将远程主机`192.168.18.128`上`/data/community.tar.gz`文件拉取到本地家目录。
```bash
scp -q 192.168.18.128:/data/community.tar.gz ~/
```
使用`-q`选项时，传输过程无信息输出。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_13.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_15.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_17.png)

**2. 将本地文件发送到远程主机**
将本地`/tmp/test`文件发送到远程主机`192.168.18.128`的`/tmp`目录下。
```bash
scp /tmp/test 192.168.18.128:/tmp/
```
不使用`-q`选项时，会显示传输速度、进度和时间等信息。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_19.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_20.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_21.png)

`scp`每次传输都会完整复制文件，无论目标位置是否已存在相同文件。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_23.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_24.png)

## Rsync：高效同步工具 ⚡

![](img/2bb31c823bb2669b75a5bb57f8b8f902_28.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_30.png)

接下来我们看看`rsync`命令。它的用法与`scp`类似，但功能更强大。`rsync`使用专门的协议，支持增量同步，传输速度通常比`scp`更快。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_32.png)

**基本命令格式如下：**
```bash
rsync [选项] 源文件 目标地址
```

![](img/2bb31c823bb2669b75a5bb57f8b8f902_34.png)

### 核心特性与示例

![](img/2bb31c823bb2669b75a5bb57f8b8f902_36.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_37.png)

**1. 增量传输**
`rsync`在传输前会比较源和目标目录的差异，只传输发生变化的部分。
```bash
# 首次同步整个data目录到远程主机
rsync -av /data/ 192.168.18.128:/data/

![](img/2bb31c823bb2669b75a5bb57f8b8f902_39.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_40.png)

# 在本地/data/目录新增一个message文件后，再次执行
rsync -av /data/ 192.168.18.128:/data/
```
第二次命令只会传输新增的`message`文件，实现了增量同步。而`scp`则会重新传输整个目录。

**2. 保持文件属性**
使用`-a`（archive）选项可以保留文件的权限、时间戳等元数据。
```bash
rsync -av /data/ 192.168.18.128:/data/
```

![](img/2bb31c823bb2669b75a5bb57f8b8f902_42.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_44.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_46.png)

**3. 目录完全同步（删除多余文件）**
默认情况下，`rsync`只同步源目录有的内容到目标目录。如果希望两个目录完全一致（即目标目录中源目录没有的文件也被删除），需要使用`--delete`选项。
```bash
# 假设远程/data/目录有一个secret文件，而本地没有
rsync -av --delete /data/ 192.168.18.128:/data/
```
执行后，远程主机的`secret`文件会被删除，确保两个目录内容完全相同。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_48.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_50.png)

**重要提示：** 使用`rsync`时，路径末尾的`/`含义不同：
*   `rsync /data/ remote:/backup/`：同步`/data/`目录下的**内容**到远程`/backup/`目录下。
*   `rsync /data remote:/backup/`：同步整个`/data`**目录本身**到远程`/backup/`目录下。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_52.png)

`rsync`常用于多服务器间的目录同步和备份脚本中。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_54.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_56.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_58.png)

## SFTP：安全的交互式文件传输 🗂️

最后，我们了解`sftp`。这是一个基于SSH的交互式文件传输程序，类似于传统的FTP，但更加安全。

**基本连接命令如下：**
```bash
sftp user@hostname
```

连接成功后，会进入`sftp>`提示符。以下是在`sftp`会话中的常用命令：

**1. 导航与查看**
*   `lpwd`：查看**本地**当前工作目录。
*   `lls`：列出**本地**当前目录的文件。
*   `pwd`：查看**远程**当前工作目录。
*   `ls`：列出**远程**当前目录的文件。
*   `cd`：更改**远程**工作目录。
*   `lcd`：更改**本地**工作目录。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_62.png)

![](img/2bb31c823bb2669b75a5bb57f8b8f902_64.png)

**2. 文件传输**
*   `get 远程文件`：将远程文件下载到本地当前目录。
*   `put 本地文件`：将本地文件上传到远程当前目录。

### 使用示例

连接远程主机并传输文件：
```bash
sftp root@192.168.18.129
sftp> lpwd
/local/path
sftp> lls
sftp> cd /data
sftp> get remote_file.txt  # 下载文件
sftp> put local_file.txt   # 上传文件
sftp> exit
```
`sftp`提供了交互式的文件管理界面，但在实际自动化脚本中不如`scp`或`rsync`常用。

## 总结 📝

![](img/2bb31c823bb2669b75a5bb57f8b8f902_66.png)

本节课中我们一起学习了三种基于SSH的文件传输工具：
1.  **`scp`**：简单直接，适用于一次性的文件复制或拉取，但每次都是全量传输。
2.  **`rsync`**：功能强大，支持增量同步和目录镜像，是进行定期备份或多服务器同步的首选工具。
3.  **`sftp`**：提供交互式会话，可以像操作本地文件一样浏览和管理远程文件，适合手动交互场景。

![](img/2bb31c823bb2669b75a5bb57f8b8f902_68.png)

你可以根据实际需求（如是否需要增量、是否需要交互等）来选择合适的工具。
