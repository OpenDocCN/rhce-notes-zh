## 练习一：打印等差数列

**题目要求**：使用脚本打印出从2到50，步长为4的等差数列（即 2, 6, 10, ..., 50）。

上一节我们介绍了`seq`命令，本节中我们来看看如何用它来生成等差数列。

以下是几种实现方法：

**方法一：使用 `seq` 命令**
这是最简便直接的方法。`seq`命令可以生成一个数字序列。
```bash
#!/bin/bash
seq 2 4 50
```
**代码解释**：`seq 起始值 步长 结束值`。这里从2开始，每次增加4，直到50。

![](img/935b4e1c5bc374092b9e65e5f6aceb20_8.png)

![](img/935b4e1c5bc374092b9e65e5f6aceb20_10.png)

![](img/935b4e1c5bc374092b9e65e5f6aceb20_11.png)

**方法二：使用 `for` 循环**
```bash
#!/bin/bash
for ((i=2; i<=50; i+=4)); do
    echo $i
done
```
**方法三：使用 `while` 循环**
```bash
#!/bin/bash
i=2
while [ $i -le 50 ]; do
    echo $i
    i=$((i+4))
done
```

![](img/935b4e1c5bc374092b9e65e5f6aceb20_12.png)

**核心要点**：同一个脚本任务可以有多种写法，只要能达到预期结果即可。`seq`命令在处理这类等差数列问题时通常最为高效。

## 练习二：查询软件包安装日期

![](img/935b4e1c5bc374092b9e65e5f6aceb20_24.png)

![](img/935b4e1c5bc374092b9e65e5f6aceb20_26.png)

**题目要求**：编写一个Shell脚本，列出所有名称中包含“kernel”的软件包的安装日期，输出格式类似于“软件包名 was installed on 安装日期”。

![](img/935b4e1c5bc374092b9e65e5f6aceb20_28.png)

上一节我们学习了如何用`rpm`命令查询软件包信息，本节中我们来看看如何提取并格式化其中的安装日期。

![](img/935b4e1c5bc374092b9e65e5f6aceb20_30.png)

![](img/935b4e1c5bc374092b9e65e5f6aceb20_32.png)

以下是解题的关键步骤和一种实现方法：

![](img/935b4e1c5bc374092b9e65e5f6aceb20_34.png)

![](img/935b4e1c5bc374092b9e65e5f6aceb20_36.png)

1.  **使用 `rpm -qa | grep kernel` 获取所有包含“kernel”的软件包列表。**
2.  遍历这个列表，对每个软件包使用 `rpm -qi` 查询其详细信息。
3.  从详细信息中提取“Install Date”这一行。
4.  使用`awk`或`cut`等工具对提取出的字符串进行分割，只保留日期部分。
5.  按照指定格式进行输出。

![](img/935b4e1c5bc374092b9e65e5f6aceb20_38.png)

**参考脚本**：
```bash
#!/bin/bash
for package in $(rpm -qa | grep -i kernel); do
    # 使用awk提取日期：以“: ”作为分隔符，取第二个字段
    install_date=$(rpm -qi $package | grep "Install Date" | awk -F': ' '{print $2}')
    echo "$package was installed on $install_date"
done
```
**代码解释**：
*   `rpm -qa | grep -i kernel`：查询所有已安装包，并用`grep -i`（忽略大小写）过滤出含“kernel”的包名。
*   `rpm -qi $package`：查询指定软件包的详细信息。
*   `grep “Install Date”`：从详细信息中筛选出包含“Install Date”的行。
*   `awk -F‘: ’ ‘{print $2}’`：设置分隔符为“冒号+空格”（`: `），然后打印第二个字段，即纯日期字符串。

![](img/935b4e1c5bc374092b9e65e5f6aceb20_40.png)

**另一种思路**：也可以使用`rpm -q --qf`（查询格式）命令直接输出定制化的信息，但需要对输出格式字符串有更深入的了解。

**核心要点**：这道题练习了命令的管道（`|`）连接、循环遍历命令执行结果、以及使用文本处理工具（`grep`, `awk`）从结构化文本中提取特定信息。

## 练习三：自动化备份与清理

**题目要求**：编写一个备份脚本，将`/etc/sysconfig`目录备份到`/usr/local/backup`目录下，备份文件命名为`sysconfig-backup-日期.tar.bz2`（日期格式为YYYYMMDD），并自动删除该目录下7天前的同名备份文件。

![](img/935b4e1c5bc374092b9e65e5f6aceb20_46.png)

上一节我们介绍了归档压缩命令和`find`命令，本节中我们来看看如何将它们组合起来实现一个实用的自动化任务。

以下是实现此功能的脚本框架和步骤解析：

**参考脚本**：
```bash
#!/bin/bash
# 1. 定义变量
backup_date=$(date +%Y%m%d)                # 获取当前日期，格式为20211201
source_dir="/etc/sysconfig"                # 要备份的源目录
backup_dir="/usr/local/backup"             # 备份文件存放目录
backup_name="sysconfig-backup-$backup_date.tar.bz2" # 备份文件名

# 2. 执行备份 (使用tar命令创建bz2压缩包)
tar -cjf "$backup_dir/$backup_name" "$source_dir"

# 3. 清理7天前的旧备份文件
find "$backup_dir" -name "sysconfig-backup-*.tar.bz2" -mtime +7 -exec rm -f {} \;
```
**代码解释**：
1.  **变量定义**：使用`$(command)`进行命令替换，将`date`命令的输出赋值给变量。清晰定义源目录、目标目录和文件名。
2.  **执行备份**：使用`tar -cjf`命令。
    *   `-c`：创建归档。
    *   `-j`：使用bzip2压缩。
    *   `-f`：指定归档文件名。这里使用了变量拼接来构成完整的路径和文件名。
3.  **清理旧文件**：使用`find`命令。
    *   `$backup_dir`：在备份目录中查找。
    *   `-name “sysconfig-backup-*.tar.bz2”`：匹配符合命名模式的文件。
    *   `-mtime +7`：查找修改时间在7天以前（大于7天）的文件。
    *   `-exec rm -f {} \;`：对找到的每个文件执行`rm -f`（强制删除）命令。

![](img/935b4e1c5bc374092b9e65e5f6aceb20_48.png)

**核心要点**：这个脚本综合运用了变量、命令替换、归档压缩和文件查找操作。它体现了Shell脚本在生产环境中的典型应用：将重复、繁琐的运维操作（手动备份、清理）自动化，提高效率并减少人为错误。
