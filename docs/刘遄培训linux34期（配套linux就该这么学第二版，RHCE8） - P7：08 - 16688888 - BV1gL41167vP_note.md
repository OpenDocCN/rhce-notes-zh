## 5.2 文件的一般权限

上一节我们介绍了用户身份与账号管理，本节中我们来看看文件上的权限，即我们常说的可读、可写、可执行权限。

对于一个文件，其权限主要由三部分组成：
*   **读取**：允许查看文件内容。
*   **写入**：允许修改文件内容。
*   **执行**：允许运行文件（如脚本或程序）。

在Linux系统中，使用 `ls -l` 命令查看文件时，会看到类似 `-rwxr-xr--` 的权限字符串。这个字符串共有10位：
*   第1位代表**文件类型**（例如 `-` 表示普通文件，`d` 表示目录）。
*   第2-4位代表**文件所有者**的权限。
*   第5-7位代表**文件所属用户组**的权限。
*   第8-10位代表**其他用户**的权限。

权限字符 `r`、`w`、`x` 分别对应读取、写入和执行。如果对应位置是 `-`，则表示没有该权限。

### 权限的数字表示法

为了更简洁、高效地设置和读取权限，Linux引入了数字表示法。每个权限对应一个数字：
*   **r (读取)** = **4**
*   **w (写入)** = **2**
*   **x (执行)** = **1**
*   **- (无权限)** = **0**

![](img/1888b43c09db70b7fb5637ebf5b86f48_11.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_13.png)

计算权限时，将同一用户类别（所有者、组、其他）的权限值相加即可。

![](img/1888b43c09db70b7fb5637ebf5b86f48_15.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_17.png)

**示例：将权限 `rwxr-xr--` 转换为数字法**
1.  所有者：`rwx` = 4 + 2 + 1 = **7**
2.  所属组：`r-x` = 4 + 0 + 1 = **5**
3.  其他用户：`r--` = 4 + 0 + 0 = **4**
因此，该权限的数字表示为 **754**。

**示例：将数字 `765` 转换为权限法**
1.  所有者：7 = 4+2+1 -> `rwx`
2.  所属组：6 = 4+2+0 -> `rw-`
3.  其他用户：5 = 4+0+1 -> `r-x`
因此，权限字符串为 `rwxrw-r-x`。

### 权限对于文件和目录的不同含义

![](img/1888b43c09db70b7fb5637ebf5b86f48_27.png)

理解权限对文件和目录的不同影响至关重要。

![](img/1888b43c09db70b7fb5637ebf5b86f48_29.png)

以下是不同权限在文件和目录上的具体含义：

![](img/1888b43c09db70b7fb5637ebf5b86f48_31.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_33.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_35.png)

| 权限 | 对文件的影响 | 对目录的影响 |
| :--- | :--- | :--- |
| **读取 (r)** | 可读取文件内容（如使用 `cat`） | 可列出目录内的文件列表（如使用 `ls`） |
| **写入 (w)** | 可修改文件内容（如使用 `vim`） | 可在目录内创建、删除、重命名文件 |
| **执行 (x)** | 可执行该文件（如运行脚本） | 可进入该目录（如使用 `cd`） |

### 修改权限和属性的命令

掌握了权限的表示方法后，我们来看看如何修改它们。

*   **`chmod`**：修改文件或目录的权限。
    *   格式：`chmod [权限] 文件或目录名`
    *   示例：`chmod 754 filename` 或 `chmod u=rwx,g=rx,o=r filename`
*   **`chown`**：修改文件或目录的所有者和所属组。
    *   格式：`chown [所有者]:[所属组] 文件或目录名`
    *   示例：`chown root:root filename` 或 `chown alice filename`

> **注意**：`chown` 命令中的分隔符可以是冒号 `:` 或点号 `.`，两者等效。

## 5.3 文件的特殊权限

除了基本读写执行权限，Linux还有三种特殊权限，它们是对一般权限的重要补充。

![](img/1888b43c09db70b7fb5637ebf5b86f48_37.png)

### SUID (Set User ID)

*   **作用**：当二进制可执行文件设置SUID后，任何用户在执行该文件时，都将**临时获得文件所有者的权限**。
*   **典型示例**：`/usr/bin/passwd` 命令。普通用户无权直接修改 `/etc/shadow` 密码文件，但通过执行带有SUID权限的 `passwd` 命令，可以临时获得root权限从而修改自己的密码。
*   **设置方法**：
    *   符号法：`chmod u+s filename`
    *   数字法：在三位数字权限前增加一位，SUID值为 **4**。例如 `4755`。
*   **显示**：在所有者执行位，如果原本有 `x` 则显示为 `s`，否则显示为 `S`。

### SGID (Set Group ID)

*   **作用**：
    1.  对**文件**：执行时临时获得文件**所属组的权限**。
    2.  对**目录**：在该目录下创建的任何新文件，其所属组将**自动继承该目录的所属组**。
*   **设置方法**：
    *   符号法：`chmod g+s directory_name`
    *   数字法：SGID值为 **2**。例如 `2755` 或对目录设置 `2770`。
*   **显示**：在所属组执行位，如果原本有 `x` 则显示为 `s`，否则显示为 `S`。

### SBIT (Sticky Bit)

![](img/1888b43c09db70b7fb5637ebf5b86f48_39.png)

*   **作用**：仅对目录有效。设置后，目录内的文件**仅允许其所有者或root用户删除或重命名**。其他用户即使有写权限也无法删除他人的文件。
*   **典型示例**：系统的 `/tmp` 临时目录。
*   **设置方法**：
    *   符号法：`chmod o+t directory_name`
    *   数字法：SBIT值为 **1**。例如 `1777`。
*   **显示**：在其他用户执行位，如果原本有 `x` 则显示为 `t`，否则显示为 `T`。

### 特殊权限的数字计算

特殊权限的数字与基本权限数字组合使用，形成一个四位数：`[特殊权限][所有者权限][所属组权限][其他用户权限]`。

**示例：计算权限 `rwsr-Sr-T` 的数字表示**
1.  基本权限：`rwx r-x r-x` -> 7 5 5
2.  特殊权限：所有者位是 `s` (SUID)，所属组位是 `S` (无执行位的SGID)，其他用户位是 `T` (无执行位的SBIT)。
    *   SUID = 4， SGID = 2， SBIT = 1。
    *   此处有SUID(4)和SGID(2)，所以特殊位 = 4 + 2 = 6。
3.  最终数字权限为：**6755**。

## 5.4 文件的隐藏权限

隐藏权限是另一层保护，用于限制一些特定的文件操作，默认情况下不可见。

管理隐藏权限的主要命令是 **`chattr`** (设置) 和 **`lsattr`** (查看)。

![](img/1888b43c09db70b7fb5637ebf5b86f48_49.png)

*   **设置隐藏权限**：`chattr [+-=] [属性] 文件或目录名`
*   **查看隐藏权限**：`lsattr 文件或目录名`

![](img/1888b43c09db70b7fb5637ebf5b86f48_57.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_59.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_61.png)

以下是两个常用的隐藏权限属性：

![](img/1888b43c09db70b7fb5637ebf5b86f48_63.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_65.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_67.png)

*   **`i` (immutable)**：设置后，文件将无法被修改、删除、重命名，甚至无法创建链接。**root用户也会受此限制**。
    *   设置：`chattr +i filename`
    *   取消：`chattr -i filename`
*   **`a` (append only)**：设置后，文件只能以追加方式写入数据，无法修改或删除已有内容。
    *   设置：`chattr +a filename`
    *   取消：`chattr -a filename`

![](img/1888b43c09db70b7fb5637ebf5b86f48_71.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_73.png)

> **提示**：在RHCE考试或实际排错中，如果遇到无法删除的文件，且常规权限检查无误，记得使用 `lsattr` 命令查看是否设置了隐藏权限（如 `i`）。

## 5.5 文件的访问控制列表 (ACL)

标准权限系统（所有者、组、其他人）有时不够灵活。ACL允许我们为**单个用户或用户组**设置精细的文件访问权限。

### ACL管理命令

![](img/1888b43c09db70b7fb5637ebf5b86f48_79.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_81.png)

*   **`setfacl`**：设置ACL规则。
    *   格式：`setfacl -m u:用户名:权限 文件` 或 `setfacl -m g:组名:权限 文件`
    *   示例：`setfacl -m u:st:rx /project` (授予用户st对/project目录的读和执行权)
*   **`getfacl`**：查看ACL规则。
    *   格式：`getfacl 文件`
*   **删除特定ACL规则**：`setfacl -x u:用户名 文件`
*   **删除所有ACL规则**：`setfacl -b 文件`

![](img/1888b43c09db70b7fb5637ebf5b86f48_83.png)

### `sudo` 服务

为了让普通用户能够安全、受控地执行部分管理员命令，而不需要知道root密码，我们使用 `sudo` 服务。

![](img/1888b43c09db70b7fb5637ebf5b86f48_131.png)

![](img/1888b43c09db70b7fb5637ebf5b86f48_133.png)

配置 `sudo` 是通过编辑 **`/etc/sudoers`** 文件实现的，**强烈建议使用 `visudo` 命令进行编辑**，因为该命令会进行语法检查。

**配置示例**：允许用户 `linuxprobe` 执行 `reboot` 命令。
```bash
# 在 /etc/sudoers 文件中添加
linuxprobe ALL=(ALL) /usr/sbin/reboot
```
*   **第一个字段**：用户名。
*   **第二个字段**：允许使用sudo的主机（`ALL` 表示所有）。
*   **第三个字段**：以谁的身份执行（`(ALL)` 表示可以切换成任何用户，通常指root）。
*   **第四个字段**：允许执行的命令（**必须使用绝对路径**）。

**用户使用方法**：
```bash
sudo /usr/sbin/reboot
```
执行时，系统会要求输入**当前用户自己的密码**进行验证，而不是root密码。

> **重要原则**：
> 1.  权限最小化：只授予完成工作所必需的最少命令权限。
> 2.  禁止授予编辑文件的命令（如 `vim`, `echo >>`），否则用户可能通过修改系统文件（包括 `/etc/sudoers` 本身）来提升权限。
