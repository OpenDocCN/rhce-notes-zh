## 权限问题与解决方案

上一节我们介绍了用户和组的管理。本节中我们来看看当普通用户需要执行某些管理员命令时遇到的问题及其解决方案。

以普通用户 `test2` 执行 `fdisk -l` 命令查看硬盘分区为例，系统会提示权限不足（Permission denied）。这是因为 `fdisk` 这类磁盘管理命令需要 `root` 权限才能执行。

## 认识 `/etc/sudoers` 文件

![](img/bda912ee54906b8f240727f8616b6d98_26.png)

`sudo` 的权限配置存储在 `/etc/sudoers` 文件中。直接编辑此文件存在风险，因此系统提供了 `visudo` 命令。

使用 `ls -l /etc/sudoers` 查看文件属性，可以看到该文件对所有者（`root`）有读写权限，对其他用户只有读权限。这意味着普通用户无法直接修改它。

![](img/bda912ee54906b8f240727f8616b6d98_28.png)

![](img/bda912ee54906b8f240727f8616b6d98_8.png)

`visudo` 命令会调用一个临时副本文件进行编辑，并在保存前进行语法检查，确保配置正确无误，然后再写回原文件。这是一种安全的编辑方式。

## 基础配置：授予单个用户权限

![](img/bda912ee54906b8f240727f8616b6d98_36.png)

现在，我们来看看如何编辑 `/etc/sudoers` 文件来为普通用户授权。

以下是配置的基本格式：
```
用户名    主机名=(可切换到的用户)    命令列表
```

例如，我们要允许用户 `test2` 在任何主机上以任何用户身份执行 `/usr/sbin/fdisk` 命令。

1.  首先，以 `root` 身份执行 `visudo` 命令。
2.  在配置文件中找到类似 `root ALL=(ALL) ALL` 的行，在其下方添加新规则。
3.  添加以下内容：
    ```
    test2    ALL=(ALL)       /usr/sbin/fdisk
    ```
    *   `test2`: 要授权的用户名。
    *   `ALL`: 表示所有主机。
    *   `(ALL)`: 表示 `test2` 可以切换到任何其他用户身份来执行命令（通常指 `root`）。
    *   `/usr/sbin/fdisk`: 允许执行的命令的**绝对路径**。可以使用 `which fdisk` 命令查找路径。

![](img/bda912ee54906b8f240727f8616b6d98_28.png)
![](img/bda912ee54906b8f240727f8616b6d98_30.png)

4.  保存并退出 `visudo`。如果没有语法错误，配置即生效。

配置完成后，切换到 `test2` 用户，使用 `sudo` 执行命令：
```bash
sudo fdisk -l
```
系统会提示输入 `test2` 用户自己的密码（不是 `root` 密码）。输入正确后，命令将以 `root` 权限执行。默认情况下，认证成功后会有 **5分钟** 的有效期，在此期间执行其他 `sudo` 命令可能无需再次输入密码。

## 进阶配置：免密码与用户组授权

上一节我们为单个用户配置了需要密码的 `sudo` 权限。本节中我们来看看如何实现免密码操作以及为用户组授权。

### 配置免密码执行

如果希望用户执行特定命令时无需输入密码，可以在命令前加上 `NOPASSWD:` 标签。

在 `visudo` 中，将之前的规则修改为：
```
test2    ALL=(ALL)       NOPASSWD: /usr/sbin/fdisk
```
这样，用户 `test2` 执行 `sudo fdisk` 时就不再需要输入密码。

### 为用户组授权

![](img/bda912ee54906b8f240727f8616b6d98_38.png)

![](img/bda912ee54906b8f240727f8616b6d98_39.png)

![](img/bda912ee54906b8f240727f8616b6d98_40.png)

可以授权给整个用户组，组内所有成员都拥有相同的 `sudo` 权限。在组名前加上百分号 `%` 表示这是一个组。

![](img/bda912ee54906b8f240727f8616b6d98_42.png)

例如，授权给 `test2` 组的所有成员可以免密码执行 `fdisk`：
```
%test2   ALL=(ALL)       NOPASSWD: /usr/sbin/fdisk
```

![](img/bda912ee54906b8f240727f8616b6d98_44.png)

## 使用别名简化管理

![](img/bda912ee54906b8f240727f8616b6d98_45.png)

当需要管理大量用户或命令时，可以使用别名来简化配置。别名分为用户别名、命令别名等，**别名必须大写**。

### 用户别名 (User_Alias)

![](img/bda912ee54906b8f240727f8616b6d98_47.png)

用户别名可以将多个用户归为一组。

在 `visudo` 文件中找到 `User_Alias` 示例部分，或在文件末尾添加：
```
User_Alias TESTADMINS = test1, test2
```
这定义了一个名为 `TESTADMINS` 的用户别名，包含了 `test1` 和 `test2` 两个用户。

然后，可以像使用普通用户名一样使用这个别名：
```
TESTADMINS ALL=(ALL) NOPASSWD: /usr/sbin/fdisk
```

![](img/bda912ee54906b8f240727f8616b6d98_49.png)

![](img/bda912ee54906b8f240727f8616b6d98_50.png)

### 命令别名 (Cmnd_Alias)

![](img/bda912ee54906b8f240727f8616b6d98_52.png)

![](img/bda912ee54906b8f240727f8616b6d98_54.png)

![](img/bda912ee54906b8f240727f8616b6d98_56.png)

命令别名可以将多个命令归为一组。

![](img/bda912ee54906b8f240727f8616b6d98_58.png)

在 `visudo` 文件末尾添加：
```
Cmnd_Alias STORAGECMDS = /usr/sbin/fdisk, /usr/bin/passwd
```
这定义了一个名为 `STORAGECMDS` 的命令别名，包含了 `fdisk` 和 `passwd` 命令。

然后，在授权规则中引用这个命令别名：
```
TESTADMINS ALL=(ALL) NOPASSWD: STORAGECMDS
```
这样，`TESTADMINS` 别名中的用户就可以免密码执行 `STORAGECMDS` 中的所有命令。

## 总结与注意事项

本节课中我们一起学习了如何使用 `visudo` 安全地为普通用户配置 `sudo` 权限。主要内容包括：

1.  **理解需求**：普通用户执行特权命令需要提权，但应遵循最小权限原则。
2.  **使用 `visudo`**：通过 `visudo` 命令安全编辑 `/etc/sudoers` 配置文件，它具有语法检查功能。
3.  **基础授权**：掌握 `用户名 主机=(身份) 命令` 的基本配置格式，并为用户 `test2` 授权执行 `fdisk`。
4.  **进阶配置**：学习了如何配置 `NOPASSWD:` 实现免密码操作，以及如何通过 `%组名` 为用户组授权。
5.  **使用别名**：通过 `User_Alias` 和 `Cmnd_Alias` 定义用户别名和命令别名，简化复杂环境下的权限管理。

**核心要点回顾**：
*   命令必须使用**绝对路径**。
*   配置更改后立即生效。
*   `sudo` 默认要求输入**用户自己的密码**，并有5分钟缓存。
*   使用别名（大写）可以使配置更清晰、易于维护。
*   始终遵循**最小权限原则**，只授予必要的命令权限，以保障系统安全。

![](img/bda912ee54906b8f240727f8616b6d98_60.png)

这种机制在日常系统运维中非常有用，它允许管理员将部分管理任务委派给普通用户，同时保持对系统的整体控制和安全。
