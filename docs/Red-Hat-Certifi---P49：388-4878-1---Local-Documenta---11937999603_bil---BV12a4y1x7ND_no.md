# Ansible 文档教程：第13章：本地文档 📚

在本节课中，我们将要学习如何使用 Ansible 的本地文档。掌握查阅文档的能力对于高效使用 Ansible 至关重要，尤其是在面对不熟悉的任务时。本节将详细介绍 `ansible-doc` 命令及其各种选项，帮助你快速查找模块信息和参数用法。

上一节我们介绍了 Ansible 文档的重要性，本节中我们来看看如何利用系统自带的本地文档工具。

![](img/287a2161df03cf39336992dccadb55b7_1.png)

## 访问命令帮助

由于我们的 Ansible 是从源代码安装的，因此无法使用 `man` 命令查看手册页。但我们可以使用 `-h` 或 `--help` 标志来获取任何 Ansible 命令的帮助信息。

![](img/287a2161df03cf39336992dccadb55b7_3.png)

![](img/287a2161df03cf39336992dccadb55b7_4.png)

例如，要查看 `ansible-doc` 命令的基本用法和选项，可以运行：
```bash
ansible-doc -h
```
该命令会显示命令的语法、描述以及所有可用选项。

## 理解插件与模块

在 Ansible 文档中，你可能会看到“插件”和“模块”这两个术语。根据 Ansible 官方文档：
*   **模块** 是可重用的独立脚本，由 Ansible API、`ansible` 命令和 `ansible-playbook` 命令调用，在目标主机（通常是远程系统）上执行。
*   **插件** 用于扩展 Ansible 的核心功能，并在控制节点上执行。

本课程主要关注**模块**。`ansible-doc` 命令默认用于查看随 Ansible 发行的模块信息。

## 使用 `ansible-doc` 命令

`ansible-doc` 是查询本地 Ansible 文档的主要工具。以下是其几个关键选项：

![](img/287a2161df03cf39336992dccadb55b7_12.png)

以下是 `ansible-doc` 命令的核心选项列表：
*   **`-h`**：显示帮助信息，列出所有可用选项。
*   **`-l`** 或 **`--list`**：列出所有可用的插件。**默认列出的是模块**。
*   **`-t <TYPE>`**：指定要列出的插件类型。若不指定，则默认为 `module`。
*   **`-s`** 或 **`--snippet`**：显示指定插件的 Playbook 代码片段。这会列出可用参数及其描述，但信息不如完整文档详细。
*   **`-v`**：启用详细输出模式。

### 列出所有模块

![](img/287a2161df03cf39336992dccadb55b7_19.png)

![](img/287a2161df03cf39336992dccadb55b7_20.png)

![](img/287a2161df03cf39336992dccadb55b7_22.png)

要查看系统上所有可用的 Ansible 模块，可以使用 `-l` 选项：
```bash
ansible-doc -l
```
执行此命令后，会进入一个文本视图，你可以滚动浏览所有模块及其简短描述。例如，你可能会看到 `lineinfile` 模块，其描述是管理文件中的行。

### 查看完整模块文档

![](img/287a2161df03cf39336992dccadb55b7_28.png)

![](img/287a2161df03cf39336992dccadb55b7_30.png)

要查看某个特定模块的完整文档，包括详细描述、所有参数、示例等，只需在 `ansible-doc` 后跟上模块名。

例如，查看 `service` 模块的文档：
```bash
ansible-doc service
```
文档结构通常包含：
1.  **描述**：模块功能的概述。
2.  **参数**：
    *   **必需参数**通常用等号（`=`）表示。
    *   **可选参数**通常用破折号（`-`）表示。
    *   每个参数都有详细说明。
3.  **说明**：关于模块使用的额外注意事项。
4.  **另请参阅**：列出相关或类似的插件。
5.  **作者**：模块作者信息。
6.  **示例**：**这是非常重要的部分**，它提供了使用不同参数的真实 Playbook 示例，涵盖了该模块的大多数常见用法。

### 查看模块代码片段

如果你只需要快速查看模块的可用参数及其简短描述，可以使用 `-s` 选项。输出格式与在 Playbook 中编写任务的方式一致。

例如，查看 `service` 模块的片段：
```bash
ansible-doc -s service
```
输出会显示一个任务框架，包含 `name`、`service` 模块调用以及所有参数列表。这适用于快速查阅，但若需要查看使用示例，建议使用不带 `-s` 标志的完整文档模式。
