## Linux系统与红帽认证体系
上一节我们介绍了讲师背景，本节中我们来看看Linux系统和红帽认证的基本情况。

![](img/b6fd6b6fc4754886c0dd15f860a3375c_12.png)

![](img/b6fd6b6fc4754886c0dd15f860a3375c_14.png)

Linux是一个开源操作系统内核，由林纳斯·托瓦兹于1991年首次发布。它基于GNU通用公共许可证（GPL）发布，这意味着任何人都可以自由使用、修改和分发其源代码。

![](img/b6fd6b6fc4754886c0dd15f860a3375c_16.png)

GNU/Linux系统包含多个发行版，主要分为以下几类：
*   **企业版**：如Red Hat Enterprise Linux (RHEL)，提供商业支持与订阅服务。
*   **社区版**：如CentOS，基于RHEL源代码重新编译，免费但不提供官方支持。
*   **个人/桌面版**：如Fedora、Ubuntu、Debian等，常用于个人电脑或特定领域。

红帽认证是业界公认的Linux技能标准，主要分为三个级别：
*   **RHCSA (红帽认证系统管理员)**：基础系统管理技能认证。
*   **RHCE (红帽认证工程师)**：高级系统管理及自动化技能认证。
*   **RHCA (红帽认证架构师)**：顶级架构师级别认证，包含多个方向。

RHCE 8.0的考试重点已从传统的服务管理转向**自动化运维**，强调Ansible和Shell脚本的应用能力。考试形式为100%上机实操，满分300分，210分通过。

## 安装RHEL 8.0操作系统
现在，我们开始动手安装RHEL 8.0操作系统。请跟随以下步骤在VMware中创建虚拟机并安装系统。

**步骤1：创建新的虚拟机**
1.  打开VMware，选择“文件” -> “新建虚拟机”。
2.  选择“自定义（高级）”配置，点击下一步。
3.  虚拟机硬件兼容性选择“Workstation 15.x”。
4.  安装来源选择“稍后安装操作系统”。
5.  客户机操作系统选择“Linux”，版本选择“Red Hat Enterprise Linux 8 64位”。
6.  命名虚拟机并选择存储位置（例如：`RHCE8-ServerA`）。
7.  处理器配置：1个处理器，2个核心。
8.  内存分配：2048 MB（2GB）。
9.  网络类型：选择“使用网络地址转换(NAT)”。
10. I/O控制器和磁盘类型保持默认推荐选项。
11. 创建新虚拟磁盘，大小设置为20GB，选择“将虚拟磁盘拆分成多个文件”。
12. 指定磁盘文件名称（例如：`RHCE8-ServerA.vmdk`）。
13. 在“自定义硬件”中，加载RHEL 8.0的ISO镜像文件，并确保已勾选“虚拟化Intel VT-x/EPT或AMD-V/RVI”选项。
14. 点击“完成”创建虚拟机。

**步骤2：安装操作系统**
1.  启动刚创建的虚拟机。
2.  在引导界面，选择第一项“Install Red Hat Enterprise Linux 8.0”并回车。
3.  选择安装过程中的语言（建议选择英文“English”）。
4.  进入安装信息摘要界面，需要配置以下几项：
    *   **SOFTWARE SELECTION**：选择“Server with GUI”（带图形界面的服务器）。
    *   **INSTALLATION DESTINATION**：选择20GB的虚拟磁盘，分区方案选择“Automatic”。
    *   **NETWORK & HOST NAME**：暂时不配置，安装完成后配置。
5.  点击“Begin Installation”开始安装。
6.  在安装过程中，设置**root**用户的密码（例如：`redhat`）。
7.  创建一个普通用户（例如：用户名`student`，密码`student`）。
8.  等待安装完成，点击“Reboot”重启系统。

**步骤3：初始配置与克隆**
1.  系统重启后，接受许可协议。
2.  完成初始配置，跳过订阅注册。
3.  使用`root`用户和设定的密码登录系统。
4.  为了后续实验需要两台主机，我们将此虚拟机完整克隆一份。
    *   关闭当前虚拟机。
    *   右键点击虚拟机，选择“管理” -> “克隆”。
    *   选择“虚拟机中的当前状态”。
    *   选择“创建完整克隆”。
    *   命名克隆虚拟机（例如：`RHCE8-ServerB`）并指定存储位置。
5.  分别启动ServerA和ServerB。

## 配置主机名与网络
系统安装完成后，我们需要为每台主机配置唯一的主机名和IP地址，以便它们能够相互通信。

**配置主机名**
在ServerA上执行以下命令：
```bash
hostnamectl set-hostname servera.lab.example.com
```
在ServerB上执行以下命令：
```bash
hostnamectl set-hostname serverb.lab.example.com
```
使用 `hostnamectl` 命令可以验证配置是否生效。

**配置网络（以ServerA为例）**
1.  首先，查看VMware虚拟网络编辑器（VMnet8）中NAT模式的子网地址和网关（例如：子网`192.168.13.0/24`，网关`192.168.13.2`）。
2.  编辑网卡配置文件（通常为`ens160`或类似名称）：
    ```bash
    vim /etc/sysconfig/network-scripts/ifcfg-ens160
    ```
3.  修改或确保文件包含以下关键内容：
    ```bash
    BOOTPROTO=static # 或 none，表示静态IP
    ONBOOT=yes # 开机自启
    IPADDR=192.168.13.101 # 为ServerA分配的IP地址
    PREFIX=24 # 子网掩码，等同于 NETMASK=255.255.255.0
    GATEWAY=192.168.13.2 # 网关地址
    DNS1=114.114.114.114 # DNS服务器地址
    ```
    *注意：ServerB应配置不同的IP地址，如`192.168.13.102`。*
4.  保存并退出编辑器。
5.  重新加载网络连接配置并重启网卡：
    ```bash
    nmcli connection reload
    nmcli connection down ens160
    nmcli connection up ens160
    ```
6.  使用 `ip addr show ens160` 或 `ping` 命令测试网络连通性。
