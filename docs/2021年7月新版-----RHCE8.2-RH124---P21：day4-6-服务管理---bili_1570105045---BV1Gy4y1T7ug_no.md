# RHCE8.2 认证课程：P21：服务管理 🛠️

在本节课中，我们将要学习 RHEL 8 系统中的服务管理。服务是系统后台运行的程序，负责提供特定功能。从 RHEL 7 开始，系统使用 `systemd` 作为默认的初始化系统和服务管理器，它取代了传统的 `init` 进程，带来了更快的启动速度和更强大的管理功能。

## 核心概念：systemd 与单元

上一节我们介绍了系统启动流程，本节中我们来看看 `systemd` 的核心概念——单元。

`systemd` 通过 **单元** 来管理系统资源。单元是 `systemd` 管理的基本对象，用配置文件进行标识和配置。单元文件定义了服务、套接字、设备等多种类型的对象。

![](img/31eefa8b659cb99079bfbd2b8564d75d_1.png)

最重要的单元类型有以下几种：
*   **Service 单元**：用于定义系统服务，后缀为 `.service`。
*   **Socket 单元**：用于定义进程间通信的套接字文件，后缀为 `.socket`。
*   **Target 单元**：用于将多个单元组合成一个组，模拟实现传统的运行级别，后缀为 `.target`。

单元配置文件通常存放在以下几个目录，优先级从高到低：
1.  `/etc/systemd/system/`：系统管理员创建和管理的自定义单元文件，优先级最高。
2.  `/run/systemd/system/`：运行时生成的单元文件，优先级次之。
3.  `/usr/lib/systemd/system/`：软件包安装的默认单元文件，优先级最低。

## 管理工具：systemctl

![](img/31eefa8b659cb99079bfbd2b8564d75d_7.png)

了解了单元的概念后，我们使用 `systemctl` 命令来管理它们。`systemctl` 是管理 `systemd` 单元的主要命令行工具。

其基本命令格式为：
```bash
systemctl [命令] [单元名称]
```

以下是常用的服务管理命令列表：

![](img/31eefa8b659cb99079bfbd2b8564d75d_9.png)

*   **查看服务状态**：`systemctl status <服务名>`
*   **启动服务**：`systemctl start <服务名>`
*   **停止服务**：`systemctl stop <服务名>`
*   **重启服务**：`systemctl restart <服务名>`
*   **设置开机自启**：`systemctl enable <服务名>`
*   **禁用开机自启**：`systemctl disable <服务名>`
*   **查看所有已启动的服务**：`systemctl list-units --type=service`
*   **重新加载单元配置文件**：`systemctl daemon-reload`

![](img/31eefa8b659cb99079bfbd2b8564d75d_11.png)

例如，管理 `httpd` 服务：
```bash
# 查看httpd服务状态
systemctl status httpd

![](img/31eefa8b659cb99079bfbd2b8564d75d_13.png)

# 启动httpd服务
systemctl start httpd

# 设置httpd开机自启
systemctl enable httpd

![](img/31eefa8b659cb99079bfbd2b8564d75d_15.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_17.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_19.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_21.png)

# 禁用httpd开机自启并停止服务
systemctl disable --now httpd
```

![](img/31eefa8b659cb99079bfbd2b8564d75d_23.png)

`systemctl enable` 命令实际上是在 `/etc/systemd/system/` 下的对应 `target` 目录中，创建了一个指向单元文件的软链接。`disable` 命令则是删除这个软链接。

## 服务单元文件详解

![](img/31eefa8b659cb99079bfbd2b8564d75d_33.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_35.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_37.png)

上一节我们学习了如何使用命令管理服务，本节中我们来看看定义服务的核心——单元文件。

![](img/31eefa8b659cb99079bfbd2b8564d75d_39.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_41.png)

服务单元文件（`.service`）通常由三个部分组成：
1.  `[Unit]`：定义单元的元数据、描述以及与其他单元的依赖关系。
2.  `[Service]`：定义服务的具体执行参数和环境。
3.  `[Install]`：定义如何安装此单元，例如在哪个 `target` 中启用。

以下是一个服务单元文件的关键选项示例：
```ini
[Unit]
Description=Apache Web Server # 服务描述
After=network.target # 本服务在 network.target 之后启动
Requires=network.target # 强依赖 network.target

[Service]
Type=simple
ExecStart=/usr/sbin/httpd -DFOREGROUND # 服务启动命令
ExecReload=/usr/sbin/httpd -k graceful
KillMode=mixed
Restart=on-failure # 服务意外终止后自动重启

[Install]
WantedBy=multi-user.target # 在 multi-user.target 中启用
```

*   **依赖关系**：`After` 和 `Requires` 定义了启动顺序和强依赖。`Wants` 表示弱依赖，即使依赖单元未启动，本单元也可启动。
*   **启动命令**：`ExecStart` 是最关键的选项，指定了启动服务要执行的命令或脚本的绝对路径。
*   **重启策略**：`Restart` 选项可以设置为 `on-failure` 等，使服务在意外退出时自动重启。

![](img/31eefa8b659cb99079bfbd2b8564d75d_43.png)

修改单元文件后，必须执行 `systemctl daemon-reload` 命令，让 `systemd` 重新加载配置，更改才会生效。

![](img/31eefa8b659cb99079bfbd2b8564d75d_45.png)

## Target 单元与运行级别

在传统的 `init` 系统中，使用运行级别来定义系统状态。在 `systemd` 中，这一概念由 **Target 单元** 实现。

![](img/31eefa8b659cb99079bfbd2b8564d75d_47.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_49.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_51.png)

Target 单元将多个服务和其他单元组合在一起，形成一个系统状态。例如：
*   `graphical.target`：对应运行级别 5，图形化界面模式。
*   `multi-user.target`：对应运行级别 3，多用户文本模式。
*   `rescue.target`：对应运行级别 1，单用户救援模式。

![](img/31eefa8b659cb99079bfbd2b8564d75d_53.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_55.png)

可以使用以下命令管理运行级别：
```bash
# 查看当前默认的 target
systemctl get-default

![](img/31eefa8b659cb99079bfbd2b8564d75d_57.png)

# 设置默认启动到图形界面
systemctl set-default graphical.target

![](img/31eefa8b659cb99079bfbd2b8564d75d_59.png)

# 临时切换到多用户文本模式（无需重启）
systemctl isolate multi-user.target
```

![](img/31eefa8b659cb99079bfbd2b8564d75d_61.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_63.png)

**注意**：不要将默认 `target` 设置为 `poweroff.target`（关机）、`reboot.target`（重启）或 `rescue.target`（救援），否则系统可能无法正常启动。

## Socket 单元：按需激活服务

![](img/31eefa8b659cb99079bfbd2b8564d75d_65.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_67.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_69.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_73.png)

除了直接管理服务，`systemd` 还可以通过 **Socket 单元** 实现按需激活。

![](img/31eefa8b659cb99079bfbd2b8564d75d_75.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_77.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_79.png)

Socket 单元（`.socket`）监听一个套接字（网络端口或 Unix 域套接字）。当有连接到达该套接字时，`systemd` 才启动对应的服务。这对于不常使用但需要快速响应的服务非常高效。

![](img/31eefa8b659cb99079bfbd2b8564d75d_81.png)

例如，可以配置一个服务，使其仅在有人访问特定端口时才被启动，访问结束后一段时间再自动停止，以节省系统资源。

![](img/31eefa8b659cb99079bfbd2b8564d75d_83.png)

![](img/31eefa8b659cb99079bfbd2b8564d75d_85.png)
