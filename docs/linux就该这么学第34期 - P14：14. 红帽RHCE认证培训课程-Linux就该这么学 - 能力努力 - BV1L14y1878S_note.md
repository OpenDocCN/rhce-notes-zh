## 9.2：SSH远程连接服务

上一节我们介绍了Linux系统服务配置的基本原则。本节中，我们来看看一个非常核心的服务：SSH（Secure Shell）。

![](img/7dde111767c722ed1ef71e138505803c_3.png)

![](img/7dde111767c722ed1ef71e138505803c_5.png)

SSH是一种安全的远程连接协议。例如，如果一台服务器位于上海，而管理员在北京，就可以通过SSH协议远程控制上海的服务器。与Windows的图形化远程桌面相比，SSH基于命令行界面，传输的是文本指令，因此具有延迟更低、效率更高的特点。

![](img/7dde111767c722ed1ef71e138505803c_7.png)

### 配置服务的基本原则回顾
在深入配置SSH之前，我们再次强调Linux服务配置的四个基本原则：
1.  Linux系统中一切皆文件。
2.  配置一个服务，本质上是修改该服务对应的配置文件。
3.  修改配置后，通常需要重启服务才能使新参数生效（少数服务如网卡服务例外）。
4.  若希望服务在系统重启后自动运行，需要将其加入启动项。

### 定位SSH配置文件
根据FHS（文件系统层次结构标准），系统的配置文件大多存放在`/etc`目录下。SSH服务的配置文件也不例外。

![](img/7dde111767c722ed1ef71e138505803c_9.png)

![](img/7dde111767c722ed1ef71e138505803c_11.png)

服务的配置文件通常分为两种：
*   **主配置文件**：保存服务最重要的全局参数。
*   **一般配置文件**：保存次重要的参数，通常由主配置文件调用。

![](img/7dde111767c722ed1ef71e138505803c_13.png)

![](img/7dde111767c722ed1ef71e138505803c_15.png)

![](img/7dde111767c722ed1ef71e138505803c_17.png)

![](img/7dde111767c722ed1ef71e138505803c_19.png)

![](img/7dde111767c722ed1ef71e138505803c_21.png)

![](img/7dde111767c722ed1ef71e138505803c_23.png)

寻找主配置文件有一个通用口诀：在`/etc`目录下，存在一个以服务名称命名的文件夹，里面有一个以服务名称命名并以`.conf`结尾的文件。虽然存在例外，但绝大多数服务符合此规律。

![](img/7dde111767c722ed1ef71e138505803c_25.png)

![](img/7dde111767c722ed1ef71e138505803c_27.png)

![](img/7dde111767c722ed1ef71e138505803c_29.png)

![](img/7dde111767c722ed1ef71e138505803c_31.png)

![](img/7dde111767c722ed1ef71e138505803c_33.png)

对于SSH服务，其主配置文件是：
```
/etc/ssh/sshd_config
```

![](img/7dde111767c722ed1ef71e138505803c_35.png)

![](img/7dde111767c722ed1ef71e138505803c_37.png)

![](img/7dde111767c722ed1ef71e138505803c_39.png)

![](img/7dde111767c722ed1ef71e138505803c_41.png)

![](img/7dde111767c722ed1ef71e138505803c_43.png)

### 查看与编辑SSH配置
使用`vim`编辑器可以查看和修改SSH的主配置文件。
```
vim /etc/ssh/sshd_config
```
配置文件中，以`#`或`;`开头的行是注释信息，仅用于说明，不会生效。

![](img/7dde111767c722ed1ef71e138505803c_45.png)

![](img/7dde111767c722ed1ef71e138505803c_47.png)

![](img/7dde111767c722ed1ef71e138505803c_49.png)

以下是一个配置SSH服务的小实例，我们将禁止root用户直接远程登录以增强安全性。

![](img/7dde111767c722ed1ef71e138505803c_51.png)

![](img/7dde111767c722ed1ef71e138505803c_53.png)

![](img/7dde111767c722ed1ef71e138505803c_55.png)

![](img/7dde111767c722ed1ef71e138505803c_57.png)

1.  编辑主配置文件，找到`PermitRootLogin`参数。
    ```
    vim /etc/ssh/sshd_config
    ```
2.  将该参数的值从`yes`改为`no`。
    ```
    PermitRootLogin no
    ```
3.  保存并退出编辑器（在vim命令模式下输入`:wq`）。
4.  **重启SSH服务**使配置生效。
    ```
    systemctl restart sshd
    ```
5.  （可选）将SSH服务加入开机自启。
    ```
    systemctl enable sshd
    ```
完成以上步骤后，尝试用root账户进行远程连接将会被拒绝。实验完成后，请记得将参数改回`yes`，并重启服务，以免影响后续操作。

![](img/7dde111767c722ed1ef71e138505803c_59.png)

![](img/7dde111767c722ed1ef71e138505803c_61.png)

## SSH的两种身份验证机制

![](img/7dde111767c722ed1ef71e138505803c_63.png)

![](img/7dde111767c722ed1ef71e138505803c_65.png)

![](img/7dde111767c722ed1ef71e138505803c_67.png)

![](img/7dde111767c722ed1ef71e138505803c_69.png)

![](img/7dde111767c722ed1ef71e138505803c_71.png)

上一节我们通过修改配置体验了SSH的基本管理。本节中我们来看看SSH协议提供的两种身份验证机制，这是其“安全”特性的核心。

![](img/7dde111767c722ed1ef71e138505803c_73.png)

![](img/7dde111767c722ed1ef71e138505803c_75.png)

![](img/7dde111767c722ed1ef71e138505803c_77.png)

![](img/7dde111767c722ed1ef71e138505803c_79.png)

SSH协议支持两种验证方式：
1.  **口令验证**：使用账号和密码进行验证。
2.  **密钥对验证**：使用一对加密密钥（公钥和私钥）进行验证。

![](img/7dde111767c722ed1ef71e138505803c_81.png)

![](img/7dde111767c722ed1ef71e138505803c_83.png)

密钥对验证在安全性和便利性上通常优于口令验证。私钥用于本地加密/签名，公钥需上传至服务器用于验证/解密。密钥长度（如2048位）远超普通密码，且算法更复杂。

![](img/7dde111767c722ed1ef71e138505803c_85.png)

![](img/7dde111767c722ed1ef71e138505803c_87.png)

### 配置密钥对验证
我们将演示如何在客户端生成密钥对，并将公钥部署到服务器，最终配置为仅允许密钥登录。

![](img/7dde111767c722ed1ef71e138505803c_89.png)

**实验环境设定**：
*   服务器A：IP地址 `192.168.10.10`
*   客户端B：IP地址 `192.168.10.20`

![](img/7dde111767c722ed1ef71e138505803c_91.png)

![](img/7dde111767c722ed1ef71e138505803c_93.png)

![](img/7dde111767c722ed1ef71e138505803c_95.png)

**步骤**：
1.  **在客户端生成密钥对**。
    在客户端B上执行以下命令，按提示回车即可（如需为私钥设置密码可在此输入）。
    ```
    ssh-keygen
    ```
    生成的密钥对默认保存在用户家目录的`.ssh/`隐藏文件夹中（`id_rsa`为私钥，`id_rsa.pub`为公钥）。

![](img/7dde111767c722ed1ef71e138505803c_97.png)

![](img/7dde111767c722ed1ef71e138505803c_99.png)

![](img/7dde111767c722ed1ef71e138505803c_101.png)

![](img/7dde111767c722ed1ef71e138505803c_103.png)

![](img/7dde111767c722ed1ef71e138505803c_105.png)

2.  **将公钥上传至服务器**。
    使用`ssh-copy-id`命令将客户端的公钥传输到服务器A的指定用户（如root）下。
    ```
    ssh-copy-id root@192.168.10.10
    ```
    此命令会要求输入服务器root用户的密码。传输成功后，公钥内容会被追加到服务器对应用户家目录的`~/.ssh/authorized_keys`文件中。

![](img/7dde111767c722ed1ef71e138505803c_107.png)

![](img/7dde111767c722ed1ef71e138505803c_109.png)

![](img/7dde111767c722ed1ef71e138505803c_111.png)

![](img/7dde111767c722ed1ef71e138505803c_113.png)

3.  **在服务器上禁用口令验证**。
    登录服务器A，编辑SSH主配置文件。
    ```
    vim /etc/ssh/sshd_config
    ```
    找到`PasswordAuthentication`参数，将其值改为`no`。
    ```
    PasswordAuthentication no
    ```
    保存退出，并重启SSH服务。
    ```
    systemctl restart sshd
    ```

![](img/7dde111767c722ed1ef71e138505803c_115.png)

![](img/7dde111767c722ed1ef71e138505803c_117.png)

![](img/7dde111767c722ed1ef71e138505803c_119.png)

![](img/7dde111767c722ed1ef71e138505803c_121.png)

4.  **测试验证**。
    此时，从客户端B再次连接服务器A，将不再需要输入密码，而是直接通过密钥对验证登录。这证明了密钥对验证的优先级高于口令验证。

![](img/7dde111767c722ed1ef71e138505803c_123.png)

## 基于SSH协议的文件传输

![](img/7dde111767c722ed1ef71e138505803c_127.png)

上一节我们实现了安全的密钥登录。本节中我们来看看如何利用SSH协议进行安全的远程文件传输。

既然SSH可以传输命令文本，自然也可以传输文件。系统提供了`scp`命令，它基于SSH协议，无需额外配置即可在主机间安全地拷贝文件。

### 将本地文件传输到远程主机
命令格式如下：
```
scp [本地文件路径] [用户名]@[远程主机地址]:[远程目录路径]
```
**实例**：将客户端B上的文件`local_file.txt`传输到服务器A的`/root`目录下。
```
scp /path/to/local_file.txt root@192.168.10.10:/root
```

![](img/7dde111767c722ed1ef71e138505803c_129.png)

![](img/7dde111767c722ed1ef71e138505803c_131.png)

### 从远程主机下载文件到本地
命令格式如下：
```
scp [用户名]@[远程主机地址]:[远程文件路径] [本地目录路径]
```
**实例**：将服务器A上`/root/remote_file.txt`文件下载到客户端B的当前目录。
```
scp root@192.168.10.10:/root/remote_file.txt ./
```

![](img/7dde111767c722ed1ef71e138505803c_133.png)

![](img/7dde111767c722ed1ef71e138505803c_135.png)

## 使用Tmux管理远程会话

上一节我们学会了传输文件。本节中我们来看看一个非常实用的终端复用器：Tmux。它解决了远程会话因网络中断而终止的问题，并能高效管理多个终端窗口。

![](img/7dde111767c722ed1ef71e138505803c_137.png)

在进行长时间任务（如编译软件）时，如果网络连接意外断开，所有进度都会丢失。Tmux可以将会话（Session）保存在服务器端，实现**不间断会话**。此外，它还能在一个终端内进行**窗口分割**和**会话共享**。

![](img/7dde111767c722ed1ef71e138505803c_139.png)

![](img/7dde111767c722ed1ef71e138505803c_141.png)

![](img/7dde111767c722ed1ef71e138505803c_143.png)

### 安装Tmux
在RHEL 8/CentOS 8中，Tmux服务取代了旧版的Screen。
```
dnf install tmux
```

![](img/7dde111767c722ed1ef71e138505803c_145.png)

![](img/7dde111767c722ed1ef71e138505803c_147.png)

### 不间断会话演示
1.  **创建新会话**。
    ```
    tmux new -s session_name
    ```
    例如：`tmux new -s mywork`。屏幕底部会出现一个状态栏，表示已进入Tmux会话。

2.  **模拟会话中断**。
    在会话中执行一些命令（如`ping`），然后直接关闭终端窗口或断开SSH连接。

![](img/7dde111767c722ed1ef71e138505803c_149.png)

3.  **恢复会话**。
    重新登录服务器后，列出所有Tmux会话。
    ```
    tmux ls
    ```
    可以看到`mywork`会话处于`detached`状态。使用以下命令恢复它：
    ```
    tmux attach -t mywork
    ```
    你会发现之前运行的任务和命令历史都完好无损。

![](img/7dde111767c722ed1ef71e138505803c_151.png)

![](img/7dde111767c722ed1ef71e138505803c_153.png)

### 窗口分割与管理
Tmux允许将一个终端窗口分割成多个窗格（Pane），提高工作效率。

![](img/7dde111767c722ed1ef71e138505803c_155.png)

![](img/7dde111767c722ed1ef71e138505803c_157.png)

*   **上下分割窗口**：`tmux split-window`
*   **左右分割窗口**：`tmux split-window -h`
*   **切换窗格**：先按`Ctrl+b`，然后按方向键（上`↑`，下`↓`，左`←`，右`→`）
*   **交换窗格位置**：`tmux swap-pane -U`（与上方窗格交换），`tmux swap-pane -D`（与下方窗格交换）

![](img/7dde111767c722ed1ef71e138505803c_159.png)

![](img/7dde111767c722ed1ef71e138505803c_161.png)

### 会话共享
Tmux支持会话共享，允许多个用户同时连接到同一个Tmux会话，实时查看和操作，非常适合远程协作或教学。

![](img/7dde111767c722ed1ef71e138505803c_163.png)

![](img/7dde111767c722ed1ef71e138505803c_165.png)

![](img/7dde111767c722ed1ef71e138505803c_167.png)

![](img/7dde111767c722ed1ef71e138505803c_169.png)

1.  用户A在服务器上创建会话。
    ```
    tmux new -s shared_session
    ```
2.  用户B连接到同一台服务器后，可以接入这个会话。
    ```
    tmux attach -t shared_session
    ```
    此时，双方可以看到完全相同的终端界面，并且操作会实时同步。

![](img/7dde111767c722ed1ef71e138505803c_171.png)

## 使用Journalctl检索日志信息

![](img/7dde111767c722ed1ef71e138505803c_179.png)

![](img/7dde111767c722ed1ef71e138505803c_181.png)

![](img/7dde111767c722ed1ef71e138505803c_183.png)

![](img/7dde111767c722ed1ef71e138505803c_185.png)

上一节我们使用Tmux增强了终端操作的韧性。本节中，我们将学习如何使用`journalctl`命令高效地检索系统日志。

![](img/7dde111767c722ed1ef71e138505803c_187.png)

![](img/7dde111767c722ed1ef71e138505803c_189.png)

![](img/7dde111767c722ed1ef71e138505803c_191.png)

系统日志记录了所有重要事件，默认存储在`/var/log/`目录下，其中`/var/log/messages`是一个综合日志文件。直接查看文件内容冗杂，`journalctl`是systemd配套的日志管理工具，可以按多种条件进行过滤和检索。

![](img/7dde111767c722ed1ef71e138505803c_193.png)

![](img/7dde111767c722ed1ef71e138505803c_195.png)

### 按优先级过滤
系统日志有不同的优先级（从高到低：emerg, alert, crit, err, warning, notice, info, debug）。例如，只查看“严重”及以上级别的日志：
```
journalctl -p crit
```

### 按时间过滤
*   查看今日日志：`journalctl --since today`
*   查看指定时间段的日志（例如从昨天12点至今）：
    ```
    journalctl --since "yesterday 12:00:00"
    ```
*   查看某个时间区间的日志：
    ```
    journalctl --since "2022-04-01 00:00:00" --until "2022-05-01 23:59:59"
    ```

### 按服务单元过滤
这是最常用的方式之一，可以只看特定服务（如sshd）的日志。
```
journalctl -u sshd
```
