## AWK命令：标准化文本处理

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_13.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_15.png)

上一节我们介绍了 `cut` 命令的局限性，本节中我们来看看 `AWK` 如何解决这些问题。`AWK` 能够以指定的分隔符截取文本，并进行标准化输出。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_17.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_18.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_20.png)

其基本命令格式如下：
```bash
awk -F '分隔符' '动作指令' 文件名
```
*   `-F` 用于指定分隔符。
*   `动作指令` 需要用单引号包裹，通常使用 `print` 来输出指定的列（字段）。

### AWK基础应用

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_32.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_34.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_38.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_40.png)

以下是 `AWK` 的一个基础应用示例。假设我们有一个内容不规则的文本文件 `t2.txt`：
```
1   2 3    4  5     6   7 8
```
使用 `AWK` 命令以空格为分隔符，输出第1、5、7列：
```bash
awk -F ' ' '{print $1,$5,$7}' t2.txt
```
这个命令会将连续的多个空格视为一个分隔符，并输出标准化的结果 `1 5 7`。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_41.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_43.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_44.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_45.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_47.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_49.png)

相比之下，使用 `cut` 命令处理不规则空格会得到不完整的结果：
```bash
cut -d ' ' -f 1,5,7 t2.txt
```
`cut` 无法将多个连续空格合并处理，导致输出不准确。

### 使用关键字筛选行

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_59.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_60.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_62.png)

`AWK` 还可以根据关键字筛选特定的行进行处理。例如，从 `/etc/passwd` 文件中筛选出包含 `root` 的行，并输出其第1、2、3列，列之间用制表符分隔：
```bash
awk -F ':' '/root/ {print $1 "\t" $2 "\t" $3}' /etc/passwd
```
*   `/root/` 用于指定匹配的关键字。
*   `"\t"` 代表制表符，用于格式化输出。

### 特殊内置变量：NF与NR

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_70.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_72.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_74.png)

`AWK` 提供了两个有用的内置变量：
*   **`NF`**：代表当前行的字段总数（列数）。`$NF` 表示最后一列。
    ```bash
    awk '{print $NF}' test.txt # 输出每一行的最后一个字段
    ```
*   **`NR`**：代表当前处理的行号。`$NR` 表示第NR行的第NR个字段（即矩阵的对角线元素）。
    ```bash
    awk '{print $NR}' test.txt # 输出每一行行号对应的那个字段
    ```
    若要输出指定行，应使用条件判断，例如输出第2行：`awk 'NR==2 {print $0}' file.txt`。

### 实践练习：编写网络信息查询脚本

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_81.png)

接下来，我们通过一个练习来巩固 `AWK` 的用法。目标是编写一个脚本，根据用户输入的网卡名称，输出该网卡的IP地址和MAC地址。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_89.png)

参考脚本如下：
```bash
#!/bin/bash
# 提示用户输入网卡设备名
read -p "Input the network device name: " DEV_NAME

# 使用awk获取MAC地址（通常位于ip addr show输出的第二行）
DEV_MAC=$(ip addr show $DEV_NAME | awk -F ' ' 'NR==2 {print $2}')

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_91.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_93.png)

# 使用awk获取IP地址（匹配以inet开头的行）
DEV_IP=$(ip addr show $DEV_NAME | awk -F ' ' '/inet / {print $2}')

# 格式化输出结果
echo "The network device information is listed below:"
echo "Device Name: $DEV_NAME"
echo "MAC Address: $DEV_MAC"
echo "IP Address: $DEV_IP"
```
这个脚本演示了如何将命令输出通过管道传递给 `AWK` 进行精准提取，并将结果赋值给变量，最终整合输出。此类脚本常用于系统巡检等自动化任务。

## SED命令：流编辑器

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_105.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_107.png)

上一节我们掌握了 `AWK` 的数据提取能力，本节中我们来看看 `SED`，它主要用于流式文本编辑和转换。`SED` 依照脚本指令处理文本，擅长自动化、重复性的编辑操作。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_109.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_110.png)

其基本语法为：
```bash
sed [选项] ‘动作指令‘ 文件名
```
常用选项包括：
*   `-e`：直接在指令列模式上进行 `sed` 的动作编辑（可接多个动作）。
*   `-f`：执行一个包含 `sed` 动作的文件。
*   `-n`：仅显示处理后的结果。
*   `-i`：**直接修改文件内容**（使用需谨慎）。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_112.png)

### SED基础编辑操作

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_114.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_116.png)

以下是 `SED` 的几种常见操作，我们以一个示例文件 `textfile.txt` 进行演示。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_118.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_119.png)

**1. 增加行**
在第四行后插入一行 “new line”：
```bash
sed -e ‘4a\new line‘ textfile.txt
```
*   `4a` 表示在第四行后追加（append）。
*   使用 `\` 可以换行输入多行内容。

**2. 删除行**
删除第2到第5行：
```bash
nl passwd | sed ‘2,5d‘
```
*   `2,5d` 表示删除第2至第5行。
*   `d` 代表删除（delete）。

**3. 替换行**
将第2到第5行内容替换为字符串 “number two to five”：
```bash
sed ‘2,5c\number two to five‘ textfile.txt
```
*   `2,5c` 表示将第2至第5行替换（change）为后续字符串。

**4. 搜索与显示**
仅显示包含 “root” 关键字的行：
```bash
sed -n ‘/root/p‘ /etc/passwd
```
*   `/root/` 用于模式匹配。
*   `-n` 与 `p` 结合，只打印匹配的行。

### SED的搜索与替换

`SED` 最强大的功能之一是搜索并替换，语法与 `Vim` 类似：
```bash
sed ‘s/要被替换的字符串/新的字符串/g‘ 文件名
```
*   `s` 代表替换（substitute）。
*   `g` 代表全局替换（global），不加 `g` 则只替换每行第一个匹配项。

**示例：提取IP地址**
我们可以利用 `SED` 的分步处理来从 `ip addr show` 命令的输出中精确提取IP地址：
```bash
ip addr show eth0 | sed -n ‘/inet /p‘ | sed ‘s/^.*inet //‘ | sed ‘s/\/.*$//‘
```
这个管道操作分三步：
1.  `sed -n ‘/inet /p‘`：筛选出包含 “inet ” 的行。
2.  `sed ‘s/^.*inet //‘`：删除行首至 “inet ” 的所有字符。
3.  `sed ‘s/\/.*$//‘`：删除第一个 “/” 之后直到行尾的所有字符。

### 直接修改文件内容

使用 `-i` 选项，`SED` 可以直接修改源文件，这在进行批量文本替换时非常高效，但风险也较高。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_121.png)

**示例：批量修改文件后缀**
假设文件 `sed.txt` 内容如下：
```
google.
taobao.
tencent.
```
将所有行末尾的句点 `.` 替换为感叹号 `!`：
```bash
sed -i ‘s/\.$/!/g‘ sed.txt
```
*   `\.` 对点进行转义，匹配字面意义的 “.”。
*   `$` 匹配行尾。
*   `-i` 使得修改直接生效于 `sed.txt` 文件。

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_123.png)

![](img/4ab0770c5518c4b86a269cf5bdbb3f54_125.png)

**示例：在文件末尾添加一行**
```bash
sed -i ‘$a\This is a test line.‘ sed.txt
```
*   `$` 代表最后一行。
*   `a` 代表追加。

本节课中我们一起学习了 `AWK` 和 `SED` 这两个核心的文本处理工具。`AWK` 擅长基于列的结构化数据提取和报告生成，而 `SED` 则擅长流式文本编辑、搜索和替换。掌握它们能够极大地提升在Linux命令行下处理文本数据的效率和能力。记住，对于 `SED` 的 `-i`（直接修改）选项，操作前务必确认或备份，以免造成数据丢失。
