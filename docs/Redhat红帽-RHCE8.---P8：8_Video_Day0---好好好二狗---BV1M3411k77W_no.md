## 软链接与硬链接 🔗

![](img/973e5c14bb4152f9f9b804a9c3293c34_7.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_9.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_11.png)

上一节我们介绍了文件的基本操作，本节中我们来看看如何创建文件的链接。链接文件类似于Windows系统中的快捷方式，但在Linux中分为软链接和硬链接两种，它们有本质区别。

**软链接**完全等同于Windows的快捷方式。它本身不存储文件内容，仅包含一个指向目标文件的路径。如果目标文件被删除，软链接就会失效。其创建命令为：
```bash
ln -s 源文件 链接名
```

**硬链接**则是为同一个文件数据块创建另一个访问入口。你可以将硬链接理解为给同一个文件起了另一个名字。只要还有一个硬链接存在，文件数据就不会丢失。其创建命令为：
```bash
ln 源文件 链接名
```

以下是两者的核心区别：
*   **本质不同**：软链接是路径的指针；硬链接是文件数据块的另一个入口。
*   **跨分区**：软链接可以跨文件系统或分区；硬链接必须在同一分区内。
*   **指向目录**：软链接可以指向目录；硬链接通常不能指向目录。
*   **原文件删除**：删除原文件，软链接失效（称为“断链”）；删除原文件，硬链接仍可正常访问文件内容。

### 文件访问原理简析

为了理解软硬链接，可以简单了解文件访问原理。Linux文件系统通过**inode**（索引节点）来标识文件，inode存储了文件的元数据（如权限、所有者）和指向数据块的指针。

*   **硬链接**：直接指向同一个inode，因此inode的链接计数会增加。使用 `ls -i` 命令可以看到，硬链接和原文件具有相同的inode编号。
*   **软链接**：拥有自己独立的inode，该inode中存储的内容是目标文件的路径字符串。当通过软链接访问时，系统会读取这个路径，然后重新定位到目标文件。

## 通配符 🌟

在文件管理中，我们经常需要匹配多个文件。通配符可以帮助我们使用模糊匹配的方式快速查找和操作文件。

![](img/973e5c14bb4152f9f9b804a9c3293c34_13.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_15.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_17.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_18.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_19.png)

以下是常用的通配符及其含义：
*   `*`：匹配任意长度（包括零个）的任意字符。
*   `?`：匹配任意单个字符。
*   `[abc]`：匹配方括号内列出的任意一个字符（如a、b或c）。
*   `[a-z]`：匹配指定范围内的任意一个字符（如a到z）。
*   `[!abc]` 或 `[^abc]`：匹配不在方括号内列出的任意一个字符。
*   `[[:alpha:]]`：匹配任意字母。
*   `[[:digit:]]`：匹配任意数字。
*   `{word1,word2}`：匹配花括号内以逗号分隔的任意一个字符串（如word1或word2）。

![](img/973e5c14bb4152f9f9b804a9c3293c34_21.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_22.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_24.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_26.png)

例如：
*   `ls *.txt` 列出所有.txt文件。
*   `ls file?.log` 列出像file1.log、fileA.log这样的文件。
*   `ls [abc]*` 列出所有以a、b或c开头的文件。

在Linux系统中，如果你忘记了命令的用法，有多种方式可以获取帮助。这在RHCE考试中也是允许使用的。

![](img/973e5c14bb4152f9f9b804a9c3293c34_40.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_42.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_43.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_44.png)

![](img/973e5c14bb4152f9f9b804a9c3293c34_45.png)

以下是几种主要的获取帮助的方法：
1.  **`--help` 选项**：大多数命令都支持此选项，能提供简洁的使用说明。
    ```bash
    ls --help
    ```
2.  **`man` 手册**：提供最详细、最权威的命令说明文档。口诀：“是男人，就man一下”。
    ```bash
    man ls
    ```
    *   `man -k 关键词` 可以搜索包含该关键词的手册页。
    *   手册通常分章节，常见的有：
        *   **第1章**：用户命令。
        *   **第5章**：文件格式和配置文件。
3.  **`info` 页面**：某些软件提供了比man更结构化的info文档，通常包含超链接导航。
    ```bash
    info ls
    ```
4.  **文档目录**：系统中存放官方文档的位置。
    *   `/usr/share/doc`：存放各类软件包的详细文档。
    *   `/usr/share/man`：存放man手册页的源文件。
