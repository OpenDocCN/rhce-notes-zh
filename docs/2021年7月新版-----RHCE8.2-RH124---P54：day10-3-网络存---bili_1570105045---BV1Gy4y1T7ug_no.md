## NFS简介与工作原理
上一节我们介绍了本地存储管理，本节中我们来看看网络存储。NFS是一种通过网络挂载存储的常用服务。

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_3.png)

NFS的实践过程如下：例如，在本地有一个 `/apps` 目录，需要挂载到NFS服务器上的 `/nfs/apps` 目录。数据实际存放在服务器的 `/nfs/apps` 中，客户端通过NFS挂载后，可以像操作本地目录一样进行读写和删除操作。

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_5.png)

NFS的底层原理涉及两个关键组件：NFS工具和RPC服务。服务器端创建NFS服务并暴露本地文件系统，通过RPC服务将数据封装在TCP/IP协议栈中，再通过网络传输到客户端。客户端同样通过RPC服务获取数据，并由NFS进程将其挂载到本地文件系统上。

## NFS的优势与软件包
NFS的主要优势在于节省本地存储空间并集中管理数据。例如，可以将用户的家目录统一挂载到NFS服务器上，从而减少各终端设备的存储需求。这个功能也常用于构建分布式存储系统。

NFS服务的主要进程包括：
*   `rpc.mountd`：管理挂载请求和权限。
*   `rpc.lockd`：管理文件锁（非必需）。
*   `rpc.statd`：检查文件一致性，可修复文件（非必需）。
*   `rpc.rquotad`：管理磁盘配额。
*   `nfsd`：NFS服务的主要进程。

NFS的主要配置文件有两个，其写法类似，主要是包含关系：
*   `/etc/exports`
*   `/etc/sysconfig/nfs`

## 实践：配置NFS服务器与客户端
上一节我们介绍了NFS的基本概念，本节我们来动手配置NFS服务。配置需要两台机器：一台作为服务器，一台作为客户端。

首先，在两台机器上安装必要的工具包：
```bash
dnf install nfs-utils -y
```
安装完成后，启动 `rpcbind` 和 `nfs-server` 服务（在服务器端）：
```bash
systemctl enable --now rpcbind
systemctl enable --now nfs-server
```
为了方便实验，可以先关闭防火墙：
```bash
systemctl stop firewalld
systemctl disable firewalld
```
在服务器上创建两个用于共享的目录：
```bash
mkdir /data/t1 /data/t2
```
编辑NFS的主配置文件 `/etc/exports`，定义共享规则：
```
/data/t1 *(rw)
/data/t2 *(rw)
```
这条配置表示将 `/data/t1` 和 `/data/t2` 目录共享给所有客户端（`*`），并赋予读写权限（`rw`）。

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_51.png)

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_53.png)

使配置生效：
```bash
exportfs -r
```
查看当前的共享状态：
```bash
exportfs -v
```
在客户端上，可以查看服务器提供的共享列表：
```bash
showmount -e <服务器IP地址>
```
在客户端创建本地挂载点并挂载NFS共享：
```bash
mkdir /data/t1 /data/t2
mount -t nfs <服务器IP地址>:/data/t1 /data/t1
mount -t nfs <服务器IP地址>:/data/t2 /data/t2
```
使用 `df -h` 命令可以查看挂载是否成功。

## NFS权限与用户映射
在挂载成功后尝试写入文件时，可能会遇到权限被拒绝的问题。即使配置了 `rw`（读写）权限，这是因为NFS涉及用户身份映射。

NFS服务器和客户端各自维护独立的用户体系。当客户端的root用户访问NFS服务器时，服务器并不会直接承认其root身份。默认情况下，客户端的root用户会被“压缩”映射为服务器上的一个普通用户（通常是 `nobody` 或 `nfsnobody`）。

NFS提供了 `root_squash`（默认）和 `no_root_squash` 选项来控制root用户的映射：
*   `root_squash`：将客户端的root用户映射为服务器上的匿名用户（安全推荐）。
*   `no_root_squash`：客户端的root用户在服务器上保持root身份（存在安全风险）。

在 `/etc/exports` 中，可以这样指定：
```
/data/t2 *(rw,no_root_squash)
```
对于普通用户，NFS默认使用 `all_squash` 选项将所有用户映射为匿名用户。可以使用 `anonuid` 和 `anongid` 选项指定映射到的特定UID和GID，这在多台服务器需要统一访问身份时非常有用，例如Web集群共享存储：
```
/data/www *(rw,all_squash,anonuid=48,anongid=48)
```
这条配置将所有访问用户都映射为服务器上UID=48、GID=48的用户（通常是Apache用户 `apache`）。

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_57.png)

修改配置后，需要重新加载：

## 配置文件详解与自动挂载
NFS的配置文件 `/etc/exports` 格式清晰。每一行定义一个共享。

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_63.png)

其基本格式为：
```
共享的目录路径 允许访问的客户端(选项1,选项2,...)
```
*   **共享的目录路径**：服务器上要共享出去的绝对路径。
*   **允许访问的客户端**：可以是单个IP（`192.168.1.100`）、网段（`192.168.1.0/24`）或代表所有主机的 `*`。
*   **选项**：控制访问权限和行为，用逗号分隔。常见选项有：
    *   `rw`：读写。
    *   `ro`：只读。
    *   `sync`：同步写入（数据安全，性能较低）。
    *   `async`：异步写入（性能高，有数据丢失风险）。
    *   `root_squash` / `no_root_squash`：是否压缩root用户。
    *   `all_squash`：压缩所有用户。
    *   `anonuid` / `anongid`：指定匿名映射的UID/GID。

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_65.png)

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_67.png)

为了使NFS共享在客户端开机时自动挂载，需要将其写入 `/etc/fstab` 文件。
```bash
<服务器IP地址>:/data/t2 /data/t2 nfs defaults,_netdev,vers=4 0 0
```
这里有两个重要选项：
*   `_netdev`：指明这是一个网络设备。系统启动时如果网络未就绪，会跳过此挂载，防止系统进入紧急模式。
*   `vers=4`：指定使用NFSv4协议，这是当前推荐版本。

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_69.png)

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_71.png)

![](img/01e89926d2f3cbbc73a3e2e297a24f8a_73.png)

写入 `/etc/fstab` 后，可以使用 `mount -a` 命令测试并立即挂载。
