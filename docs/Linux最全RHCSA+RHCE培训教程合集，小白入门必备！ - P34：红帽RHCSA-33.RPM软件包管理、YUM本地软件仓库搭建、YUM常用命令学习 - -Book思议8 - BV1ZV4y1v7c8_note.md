### 本地仓库配置详解

本地软件仓库也称为本地YUM源。“源”指的是软件包的来源。本地YUM源意味着软件包来源于本机，因此称为本地软件仓库。

以下是搭建本地YUM仓库的步骤：

**1. 创建仓库配置文件**

YUM命令在安装软件时会自动到系统的特定路径下寻找仓库配置文件。这个路径是 `/etc/yum.repos.d/`。在此路径下创建的文件，必须以 `.repo` 作为扩展名，YUM才会识别。如果文件扩展名不是 `.repo`，YUM将不会读取其中的配置，也就无法根据配置下载软件包。

因此，创建本地YUM源的第一步是：在 `/etc/yum.repos.d/` 路径下，创建一个以 `.repo` 结尾的文件。文件名前缀可以自定义，YUM只认 `.repo` 扩展名。

**2. 编写配置文件内容**

使用 `vim` 命令在指定路径创建并编辑文件，例如 `local.repo`。

```
vim /etc/yum.repos.d/local.repo
```

打开文件后，需要写入以下配置内容：

![](img/908ac43f088b2faa62c811217da5a196_12.png)

```
[local]
name=local - centos-iso
baseurl=file:///mnt/centos
enabled=1
gpgcheck=0
```

![](img/908ac43f088b2faa62c811217da5a196_14.png)

接下来，我们详细解释每一行配置的含义：

*   **`[local]`**： 定义仓库的名称。名称放在中括号内，可以自定义，但建议不要使用中文。这里命名为 `local`，表示本地。
*   **`name=local - centos-iso`**： 定义仓库的描述信息。`name` 是固定语法，后面跟的描述内容可以自定义，用于说明这个仓库的来源或用途。这里描述为来源于本地的CentOS镜像文件。
*   **`baseurl=file:///mnt/centos`**： **这是最重要的一行配置**。它定义了软件包的实际存放路径。
    *   `baseurl` 是固定语法。
    *   `file://` 是固定格式，用于指定本地路径。
    *   `/mnt/centos` 是软件包所在目录的路径。YUM安装软件时，会自动在此路径下寻找 `Packages` 目录并从中下载软件。**此路径绝对不能指错**，否则YUM将找不到软件包。
*   **`enabled=1`**： 指定是否启用此仓库。
    *   `1` 表示启用。
    *   `0` 表示禁用。如果禁用，YUM将不会从这个仓库下载软件。
    *   如果不写此行，默认状态也是启用 (`1`)。
*   **`gpgcheck=0`**： 指定是否检查软件包的GPG签名。
    *   `1` 表示检查。如果检查签名，但系统没有对应的签名密钥文件，软件包会安装失败。
    *   `0` 表示不检查。对于我们自己搭建的本地仓库，通常设置为 `0` 以简化操作。
    *   对于网络仓库（如官方源），通常必须设置为 `1` 并配合 `gpgkey` 指定密钥文件，以确保软件包的安全性。

![](img/908ac43f088b2faa62c811217da5a196_16.png)

![](img/908ac43f088b2faa62c811217da5a196_18.png)

配置完成后，保存并退出文件。至此，本地YUM仓库就搭建完成了。

### YUM的优势：自动解决依赖

![](img/908ac43f088b2faa62c811217da5a196_20.png)

![](img/908ac43f088b2faa62c811217da5a196_22.png)

YUM最强大的功能在于它能自动解决软件包的依赖关系。

例如，当我们使用 `yum -y install gcc` 命令安装GCC编译器时，YUM不仅会安装GCC主包，还会自动安装其所需的6个依赖包。这些依赖包同样来自于我们配置的本地仓库路径。

如果使用RPM命令手动安装，则需要自己逐个找到并安装所有依赖包，过程非常繁琐。YUM极大地简化了软件安装流程。

## YUM常用命令学习

仓库搭建好后，我们来学习YUM工具的核心命令。

### 1. 列出仓库信息：`yum repolist`

![](img/908ac43f088b2faa62c811217da5a196_24.png)

使用 `yum repolist` 命令可以列出所有已配置并启用的仓库信息。

![](img/908ac43f088b2faa62c811217da5a196_26.png)

```
yum repolist
```

执行该命令后，会显示类似以下信息：
*   **仓库ID (repo id)**： 对应配置文件中的 `[local]`，即仓库名称。
*   **仓库名称 (repo name)**： 对应配置文件中的 `name` 描述。
*   **状态 (status)**： 显示该仓库中可用的软件包数量。

这个命令可以帮助你确认仓库是否配置正确并已启用。

### 2. 安装软件包：`yum install`

安装软件包的基本命令是 `yum install`。

```
yum install 软件包名
```

**使用 `-y` 选项：**
在安装过程中，YUM会列出将要安装的主包和依赖包，并询问 “Is this ok [y/d/N]?”。其中：
*   `y`： 同意安装。
*   `d`： 仅下载，不安装。
*   `N`： 取消。

如果我们确定要安装，可以使用 `-y` 选项来自动回答 “yes”，避免手动确认。

![](img/908ac43f088b2faa62c811217da5a196_28.png)

```
yum -y install 软件包名
# 或者
yum install -y 软件包名
# 或者
yum install 软件包名 -y
```

![](img/908ac43f088b2faa62c811217da5a196_30.png)

`-y` 选项的位置没有严格要求。例如，安装VIM编辑器：`yum -y install vim`。VIM可能需要28个依赖包，YUM会自动处理，非常便捷。

### 3. 搜索软件包：`yum list` 与 `grep`

当我们想安装一个软件但记不清完整包名时，可以使用 `yum list` 命令进行搜索。

*   `yum list`： 列出仓库中所有可用的软件包（包括已安装和未安装的）。直接使用会列出所有包，数量庞大。
*   `yum list | grep 关键词`： 结合管道符 `|` 和 `grep` 命令进行过滤，只显示包含特定关键词的包。

例如，想安装与Java相关的包但记不清全名：

```
yum list | grep java
```

![](img/908ac43f088b2faa62c811217da5a196_32.png)

如果结果太多，可以进一步过滤：

![](img/908ac43f088b2faa62c811217da5a196_34.png)

```
yum list | grep java | grep jdk
```

![](img/908ac43f088b2faa62c811217da5a196_36.png)

这种方法非常灵活，是查找软件包的实用技巧。

![](img/908ac43f088b2faa62c811217da5a196_38.png)

![](img/908ac43f088b2faa62c811217da5a196_40.png)

### 4. 查看已安装软件包：`rpm -qa`

虽然YUM功能强大，但RPM命令在某些场景下依然不可替代。例如，查看系统中所有已安装的软件包：

```
rpm -qa
```

可以使用 `wc -l` 统计数量：

![](img/908ac43f088b2faa62c811217da5a196_42.png)

![](img/908ac43f088b2faa62c811217da5a196_44.png)

```
rpm -qa | wc -l
```

## 网络YUM仓库简介

除了本地仓库，在实际工作中更常用的是网络YUM仓库（如软件官方源、EPEL源等）。

![](img/908ac43f088b2faa62c811217da5a196_46.png)

网络仓库的配置文件同样位于 `/etc/yum.repos.d/` 目录下，但 `baseurl` 指向的是HTTP或HTTPS的网络地址。

一个典型的网络仓库配置示例如下（以Nginx官方稳定版仓库为例）：

```
[nginx-stable]
name=nginx stable repo
baseurl=http://nginx.org/packages/centos/$releasever/$basearch/
gpgcheck=1
enabled=1
gpgkey=https://nginx.org/keys/nginx_signing.key
```

与本地仓库配置的主要区别在于：
1.  **`baseurl`**： 使用 `http://` 或 `https://` 开头的网络地址。
2.  **`gpgcheck=1`**： 通常需要开启GPG签名检查以保证软件来源安全。
3.  **`gpgkey`**： 当 `gpgcheck=1` 时，必须通过 `gpgkey` 指定用于验证签名的公钥文件地址。

YUM在搜索软件包时，默认会**优先使用网络仓库**，然后才是本地仓库。
