在本节课中，我们将要学习Shell脚本中的`if`条件判断语句。`if`判断是编程中控制流程的核心，它允许我们根据条件是否成立来执行不同的命令。我们将从最简单的单分支判断开始，逐步深入到双分支和多分支判断，并通过实例演示其应用。

![](img/7ffb899997274e1964c7b6f4e1374eee_4.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_6.png)

## 概述

![](img/7ffb899997274e1964c7b6f4e1374eee_8.png)

`if`判断与我们之前学习的条件测试（如`[ ]`或`test`）目的一致，都是根据条件是否满足来决定执行何种操作。但`if`语句提供了更结构化和强大的逻辑控制能力，适合实现复杂的脚本逻辑。

## if单分支判断

![](img/7ffb899997274e1964c7b6f4e1374eee_10.png)

上一节我们介绍了条件测试的基础，本节中我们来看看最简单的`if`判断形式——单分支。单分支`if`语句的特点是：**只能判断条件成立的情况，无法处理条件失败的情况**。也就是说，只有当条件为真时，才会执行指定的命令；如果条件为假，则直接跳过。

![](img/7ffb899997274e1964c7b6f4e1374eee_12.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_13.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_15.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_16.png)

它的语法结构有两种形式。

![](img/7ffb899997274e1964c7b6f4e1374eee_18.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_20.png)

以下是`if`单分支的两种语法格式：

1.  **格式一**：
    ```bash
    if [ 条件 ]
    then
        要执行的命令
    fi
    ```

![](img/7ffb899997274e1964c7b6f4e1374eee_22.png)

2.  **格式二**：
    ```bash
    if [ 条件 ]; then
        要执行的命令
    fi
    ```

![](img/7ffb899997274e1964c7b6f4e1374eee_24.png)

**代码说明**：
*   `if` 是语句的开始。
*   `[ 条件 ]` 是判断条件，可以是文件测试、数值比较、字符串比较等。
*   `then` 表示条件成立后要执行后续命令。
*   `fi` 是`if`语句的结束标记。
*   第二种格式将`then`与`if`写在同一行，用分号`;`隔开，更为紧凑。

为了语法的美观，我们通常会对`then`后面的命令进行缩进。

![](img/7ffb899997274e1964c7b6f4e1374eee_26.png)

让我们通过一个实例来理解。创建一个脚本`if01.sh`，判断`/etc/passwd`文件是否存在。

![](img/7ffb899997274e1964c7b6f4e1374eee_28.png)

```bash
#!/bin/bash

![](img/7ffb899997274e1964c7b6f4e1374eee_30.png)

# 使用格式一
if [ -f /etc/passwd ]
then
    echo “文件存在”
fi

![](img/7ffb899997274e1964c7b6f4e1374eee_32.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_34.png)

# 使用格式二
if [ -f /etc/passwd ]; then
    echo “文件存在”
fi
```

![](img/7ffb899997274e1964c7b6f4e1374eee_36.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_38.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_40.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_42.png)

给脚本添加执行权限并运行：
```bash
chmod +x if01.sh
./if01.sh
```
脚本会输出“文件存在”，因为`/etc/passwd`文件确实存在。

![](img/7ffb899997274e1964c7b6f4e1374eee_44.png)

如果我们将条件改为判断一个不存在的文件，例如`/opt/abcd.txt`：
```bash
if [ -f /opt/abcd.txt ]; then
    echo “文件存在”
fi
```
再次运行脚本，将不会有任何输出。因为条件不成立，`then`后面的`echo`命令不会被执行。

这就是单分支`if`的局限性：它只能处理条件成立时的操作。

## if双分支判断

![](img/7ffb899997274e1964c7b6f4e1374eee_46.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_48.png)

单分支`if`无法处理条件失败的情况，这在实际应用中往往不够。因此，我们需要双分支`if`语句。**双分支`if`既能判断条件成立，也能判断条件不成立**，并分别执行不同的命令块。

它的语法结构如下：

```bash
if [ 条件 ]
then
    条件成立时执行的命令
else
    条件不成立时执行的命令
fi
```

**代码说明**：
*   `if [ 条件 ]` 和 `then` 部分与单分支相同。
*   `else` 是新的关键字，代表“否则”。
*   如果`if`后面的条件成立，则执行`then`和`else`之间的命令。
*   如果条件不成立，则执行`else`和`fi`之间的命令。

![](img/7ffb899997274e1964c7b6f4e1374eee_50.png)

让我们编写一个脚本`if02.sh`，实现一个更实用的功能：检查软件包`vsftpd`是否安装，如果未安装则自动安装并启动服务。

![](img/7ffb899997274e1964c7b6f4e1374eee_52.png)

```bash
#!/bin/bash

# 检查vsftpd是否安装，将命令结果（成功或失败的状态码）重定向到/dev/null，不显示输出
rpm -q vsftpd &> /dev/null

![](img/7ffb899997274e1964c7b6f4e1374eee_54.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_56.png)

# 判断上一条命令的执行状态码
if [ $? -eq 0 ]
then
    # 状态码为0，表示命令成功，即软件包已安装
    echo “软件包已存在，启动服务...”
    systemctl start vsftpd
    systemctl enable vsftpd &> /dev/null
    echo “vsftpd服务已启动并设置开机自启”
else
    # 状态码非0，表示命令失败，即软件包未安装
    echo “软件包不存在，正在安装...”
    yum install -y vsftpd &> /dev/null
    systemctl start vsftpd
    systemctl enable vsftpd &> /dev/null
    echo “vsftpd安装完成，服务已启动并设置开机自启”
fi
```

![](img/7ffb899997274e1964c7b6f4e1374eee_58.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_60.png)

**核心概念解释**：
*   `$?` 是一个特殊变量，用于获取**上一条命令的退出状态码**。通常，0表示成功，非0表示失败。
*   `-eq` 是数值比较运算符，表示“等于”。
*   `&> /dev/null` 将命令的标准输出和标准错误都重定向到空设备，即不显示任何输出。

![](img/7ffb899997274e1964c7b6f4e1374eee_62.png)

这个脚本清晰地展示了双分支的逻辑：**如果...就...否则...**。

为了增加趣味性，我们再来看一个猜数字的小游戏脚本`if03.sh`，它综合运用了变量、随机数和双分支判断。

```bash
#!/bin/bash

# 生成一个0-9的随机数作为正确答案
correct_answer=$((RANDOM % 10))

![](img/7ffb899997274e1964c7b6f4e1374eee_64.png)

# 提示用户输入
read -p “请输入一个0到9之间的数字：” user_guess

![](img/7ffb899997274e1964c7b6f4e1374eee_66.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_67.png)

# 判断用户猜测是否正确
if [ $user_guess -eq $correct_answer ]
then
    echo “恭喜你，猜对了！奖励哇塞女孩一枚！”
else
    echo “很遗憾，猜错了。哇塞女孩正在等你，请继续努力！”
    echo “本次的正确数字是：$correct_answer”
fi
```

**核心概念解释**：
*   `RANDOM` 是Shell的一个环境变量，每次引用都会产生一个0到32767之间的随机整数。
*   `$((RANDOM % 10))` 是算术运算，表示对随机数取除以10的余数，结果范围是0-9。

![](img/7ffb899997274e1964c7b6f4e1374eee_69.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_71.png)

运行这个脚本，体验一下双分支判断带来的交互逻辑。

## if多分支判断

在实际场景中，我们可能面临多个条件的选择，而不仅仅是“是”或“否”。这时就需要多分支`if`语句。**多分支`if`允许我们依次检查多个条件，执行第一个满足条件所对应的命令块**。

它的语法结构如下：

```bash
if [ 条件1 ]
then
    条件1成立时执行的命令
elif [ 条件2 ]
then
    条件2成立时执行的命令
elif [ 条件3 ]
then
    条件3成立时执行的命令
...
else
    以上所有条件都不成立时执行的命令
fi
```

![](img/7ffb899997274e1964c7b6f4e1374eee_73.png)

**代码说明**：
*   `elif` 是 `else if` 的缩写，用于添加新的判断条件。
*   脚本会从上到下依次判断每个`if`或`elif`后的条件。
*   一旦某个条件成立，就会执行其对应的`then`后面的命令块，然后**直接跳到`fi`结束整个判断**，后面的条件不再检查。
*   如果所有`if`和`elif`的条件都不成立，则执行`else`后面的命令块。

一个典型的应用是成绩等级评定脚本`if04.sh`：

```bash
#!/bin/bash

![](img/7ffb899997274e1964c7b6f4e1374eee_75.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_77.png)

read -p “请输入您的成绩（0-100）：” score

if [ $score -ge 90 ]
then
    echo “成绩优秀！奖励哇塞女孩一天！”
elif [ $score -ge 80 ]
then
    echo “成绩良好！离哇塞女孩仅一步之遥。”
elif [ $score -ge 70 ]
then
    echo “成绩中等，还需努力。”
elif [ $score -ge 60 ]
then
    echo “成绩及格，勉强过关。”
else
    echo “成绩不及格。加油吧！”
fi
```

**核心概念解释**：
*   `-ge` 是数值比较运算符，表示“大于等于”。
*   注意条件的顺序：必须先判断`>=90`，再判断`>=80`。如果反过来，一个95分的成绩会首先满足`>=80`的条件，被判定为“良好”，而不会继续判断是否为“优秀”。

运行脚本，输入不同的分数，观察其如何根据分数区间跳转到不同的分支。

## 总结与核心要点

本节课中我们一起学习了Shell脚本中强大的流程控制工具——`if`条件判断语句。

![](img/7ffb899997274e1964c7b6f4e1374eee_79.png)

![](img/7ffb899997274e1964c7b6f4e1374eee_80.png)

我们掌握了三种形式：
1.  **单分支`if`**：仅处理条件成立的情况，结构简单。
2.  **双分支`if`**：通过`if-else`结构处理“成立”与“不成立”两种对立情况。
3.  **多分支`if`**：通过`if-elif-else`结构处理多个连续或并列的条件。

**编写`if`语句的核心要点**：
*   **条件必须放在方括号`[ ]`内**，这是进行测试的标准格式。
*   **方括号内各部分必须有空格**，例如 `[ $a -eq $b ]`，`[`、`$a`、`-eq`、`$b`、`]`之间都要有空格。
*   **使用变量时一定要加上`$`符号**，如 `$score`。
*   判断命令是否成功，最常用的方法是检查特殊变量 **`$?`** 是否等于0。
*   `then`、`else`、`elif`、`fi` 都是关键字，需要正确拼写。

![](img/7ffb899997274e1964c7b6f4e1374eee_82.png)

`if`判断是Shell脚本编程的基石，结合之前学习的变量、运算符和测试命令，你已经可以编写出具有实用逻辑的脚本了。记住，学习脚本的初期主要是理解和修改现有脚本，多练习、多分析，自然会逐渐掌握其精髓。
