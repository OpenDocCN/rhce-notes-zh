# 华为云PaaS微服务治理技术 - P121：13.学成在线项目部署-前端门户-创建工作负载 🚀

![](img/10202787ab2dd180ced5cdcfe41d61dc_1.png)

在本节课中，我们将要学习如何为“学成在线”项目的前端门户部分创建工作负载。我们将使用华为云平台，基于Nginx镜像部署一个无状态工作负载，并配置数据卷和公网访问方式，为后续部署工程代码做好准备。

![](img/10202787ab2dd180ced5cdcfe41d61dc_3.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_5.png)

## 概述

![](img/10202787ab2dd180ced5cdcfe41d61dc_6.png)

前端部署包含门户、学习中心和搜索三部分。门户是整个工程的入口，学习中心和搜索的访问也需要通过门户中的Nginx进行代理，因此我们首先部署门户。

## 镜像获取与工作负载创建

门户的运行需要使用Nginx。我们可以直接从Docker Hub获取Nginx的官方镜像。

接下来，我们进入华为云平台，开始创建工作负载。

1.  在控制台中选择“工作负载” -> “无状态负载”。
2.  点击“创建无状态负载”。
3.  为工作负载命名，例如 `nginx-portal`。
4.  开启“时钟同步”功能，然后进入下一步。

![](img/10202787ab2dd180ced5cdcfe41d61dc_8.png)

## 容器配置

![](img/10202787ab2dd180ced5cdcfe41d61dc_10.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_12.png)

在容器配置步骤，我们需要设置镜像和资源。

![](img/10202787ab2dd180ced5cdcfe41d61dc_14.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_16.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_17.png)

1.  在镜像搜索框中输入 `nginx` 进行搜索。
2.  选择官方的 `nginx` 镜像，并选用一个稳定版本（例如 `latest` 或特定版本号）。
3.  配置容器资源。对于Nginx门户，建议CPU请求设为 `0.25` 核，限制设为 `1` 核；内存请求设为 `1024Mi`。

![](img/10202787ab2dd180ced5cdcfe41d61dc_19.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_21.png)

## 数据卷配置 📂

![](img/10202787ab2dd180ced5cdcfe41d61dc_23.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_25.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_26.png)

上一节我们配置了容器的基础信息，本节中我们来看看如何配置数据卷。配置数据卷的目的是为了方便管理日志、配置文件和工程代码，这与工作负载是“无状态”还是“有状态”没有直接关系。

![](img/10202787ab2dd180ced5cdcfe41d61dc_28.png)

“无状态工作负载”的特点是应用实例本身不持久化状态数据，可以快速进行横向扩展。而“有状态工作负载”（如多节点数据库）则需要解决数据共享和持久化问题。我们为Nginx配置数据卷，主要是为了在容器外部管理这些经常变动的文件。

![](img/10202787ab2dd180ced5cdcfe41d61dc_30.png)

以下是需要配置的三个数据卷及其作用：

![](img/10202787ab2dd180ced5cdcfe41d61dc_32.png)

*   **日志卷**：将容器内的Nginx日志目录挂载到宿主机，方便日志收集和查看。
*   **配置卷**：将Nginx的配置文件目录挂载到宿主机，便于修改和更新配置，无需重新构建镜像。
*   **代码卷**：用于存放门户前端工程代码的目录，方便代码更新。

![](img/10202787ab2dd180ced5cdcfe41d61dc_34.png)

具体配置如下：

1.  添加第一个数据卷，类型选择“主机路径”，挂载路径填写宿主机上用于存放日志的目录（如 `/data/nginx/logs`），容器挂载路径填写 `/var/log/nginx`。
2.  添加第二个数据卷，宿主机路径填写配置文件目录（如 `/data/nginx/conf`），容器挂载路径填写 `/etc/nginx`。
3.  添加第三个数据卷，宿主机路径填写代码目录（如 `/data/nginx/html`），容器挂载路径填写 `/usr/share/nginx/html`。

![](img/10202787ab2dd180ced5cdcfe41d61dc_36.png)

配置完成后，点击“下一步”。

![](img/10202787ab2dd180ced5cdcfe41d61dc_38.png)

## 访问方式配置 🌐

![](img/10202787ab2dd180ced5cdcfe41d61dc_40.png)

工作负载和存储配置完成后，我们需要为其配置访问方式，以便用户能够从公网访问门户网站。

![](img/10202787ab2dd180ced5cdcfe41d61dc_42.png)

如果选择“弹性IP”访问，云平台会随机分配一个30000-32767范围的高端口，这不符合网站通常使用80或443端口的惯例。因此，我们选择“负载均衡”方式。

![](img/10202787ab2dd180ced5cdcfe41d61dc_44.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_46.png)

负载均衡器可以接收公网80端口的请求，并将其转发到内部容器的80端口。在华为云中，我们需要创建或选择一个负载均衡器实例。

![](img/10202787ab2dd180ced5cdcfe41d61dc_48.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_49.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_51.png)

1.  在访问方式设置中，选择“公网访问”，访问类型选择“负载均衡”。
2.  点击“新建负载均衡器”。
3.  为负载均衡器命名（如 `elb-portal`），选择“经典型”或“增强型”（本例中选择经典型），并根据需求选择计费模式（按带宽或按流量）。
4.  点击“立即购买”并完成创建。
5.  回到工作负载创建页面，刷新并选择刚刚创建的负载均衡器。
6.  设置服务端口和容器端口均为 `80`。
7.  系统会自动为负载均衡器分配一个弹性公网IP。确认配置无误后，点击“创建”。

![](img/10202787ab2dd180ced5cdcfe41d61dc_53.png)

## 创建完成与后续步骤

![](img/10202787ab2dd180ced5cdcfe41d61dc_55.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_57.png)

工作负载创建成功后，在详情页可以看到其状态和访问地址。这个地址就是负载均衡器的弹性公网IP，通过它即可在标准80端口访问我们的门户网站。

至此，门户的工作负载已创建完成。我们使用了一个新的云服务——弹性负载均衡，来实现公网访问。在接下来的步骤中，我们需要将门户的前端工程代码和Nginx配置文件上传到之前配置的宿主机目录中，然后即可测试访问。

![](img/10202787ab2dd180ced5cdcfe41d61dc_59.png)

## 总结

![](img/10202787ab2dd180ced5cdcfe41d61dc_61.png)

![](img/10202787ab2dd180ced5cdcfe41d61dc_63.png)

本节课中我们一起学习了在华为云上为前端门户创建无状态工作负载的全过程。关键步骤包括：从Docker Hub获取Nginx镜像、配置容器资源、挂载数据卷以管理日志、配置和代码文件，以及通过创建弹性负载均衡器配置公网访问。这为部署实际的前端应用代码打下了基础。
