接下来，我们通过一个拓扑结构来了解路由的基本概念。图中包含几个网段：`5.0.0.0/24`网段、`6.0.0.0/24`网段，以及外部网络`172.17.0.0/16`网段。

我们以主机B为例。主机B有两个网络接口：`eth0`和`eth1`，分别属于`5.0.0.0/24`和`6.0.0.0/24`网段。

所谓路由表，就是描述了到达某个网段或某个IP地址时，数据包应该从哪个网络接口发出，以及是否需要将数据包交给路由器转发。可以想到，如果目标地址与我们直接相连（例如在同一个网段），那么主机自己就可以处理，这称为**直接路由**。如果目标地址不在直接相连的网段，那么就需要将数据包交给路由器，这称为**间接路由**。

![](img/0eb5b43171458f4dae1f5818d87ea309_23.png)

这个道理很简单。如果你要把一封信送给同班的女朋友A，你们在同一个楼里，你直接走过去就可以。但如果你要把花送给在上海的女朋友B，你就需要把花交给快递员，经过一系列的中转，最终到达你女朋友手上。

无论是网络中的路由器，还是进行通信的主机，它们都有自己的路由表。路由器有路由表，主机也有主机的路由表。它们所描述的内容是相似的，即：到达某个目标（可能是一个网段，也可能是一个具体的IP地址），应该从哪个接口出去，以及下一跳交给谁（是自己处理还是交给某个网关）。

路由表中的每一条记录，我们称之为**路由条目**。例如，第一条条目表示：到达`5.0.0.0/24`这个网段，数据包从`eth0`接口发出。有同学可能会问，为什么这里接口的名字和拓扑图上的名字不一样？这在之前简单解释过，**接口的逻辑名称和网卡配置的名称是可以不同的**。第二条条目表示：到达`6.0.0.0/24`网段，从`eth1`接口发出。因为这个接口本身就连接着`6.0.0.0/24`网段，所以数据包不需要经过路由器，直接从这个接口发出，然后通过ARP协议寻找目标主机的MAC地址即可。

![](img/0eb5b43171458f4dae1f5818d87ea309_32.png)

那么，如果现在主机B要发送数据包给DNS服务器（IP为`172.17.53.3`），该如何处理呢？首先，主机B会判断这个IP地址属于哪个网段。通过比对，发现它属于`172.17.0.0/16`网段。因此，主机B会查询路由表，并使用第三条路由条目来决定数据包的走向：数据包从`eth0`接口发出，并交给IP地址为`5.0.0.1`的路由器。之后如何到达目标主机，就是路由器的事情了。

![](img/0eb5b43171458f4dae1f5818d87ea309_34.png)

![](img/0eb5b43171458f4dae1f5818d87ea309_35.png)

如果我们要访问的是互联网上的其他主机呢？例如，通过DNS解析后，我们需要访问一个IP地址为`202.x.x.x`的主机。这时，主机B查询路由表：目标IP不属于`5.0.0.0/24`网段（第一条无效），不属于`6.0.0.0/24`网段（第二条无效），也不属于`172.17.0.0/16`网段（第三条无效）。那么，主机就会使用最后一条**默认路由**。默认路由也常被称为**默认网关**，它表示：除了前面几条明确指出的路由之外，所有其他目的地都走这条路径。在本例中，数据包同样从`eth0`接口发出，交给路由器`5.0.0.1`，之后的事情主机B就不再管了。

![](img/0eb5b43171458f4dae1f5818d87ea309_36.png)

![](img/0eb5b43171458f4dae1f5818d87ea309_37.png)

![](img/0eb5b43171458f4dae1f5818d87ea309_38.png)

![](img/0eb5b43171458f4dae1f5818d87ea309_39.png)

所以，所有主机在进行通信时，无论是访问局域网还是互联网，都需要查询路由表。**如果路由表中没有匹配任何条目（包括默认路由），那么系统就会返回“网络不可达”的错误**。这个概念非常重要。在后续课程中，我们会教大家如何查看和配置路由表，但首先必须理解这个核心概念。

本节课中，我们一起学习了IPv4路由的基础知识。我们了解了**路由表**的作用，它由多条**路由条目**组成，每条条目指明了到达特定目标的出口接口和下一跳网关。我们区分了**直接路由**（目标在同一网段）和**间接路由**（目标在不同网段，需要网关转发），并认识了至关重要的**默认路由**（默认网关）。记住，没有匹配路由条目会导致通信失败。理解这些概念是后续进行网络配置和故障排查的基础。
