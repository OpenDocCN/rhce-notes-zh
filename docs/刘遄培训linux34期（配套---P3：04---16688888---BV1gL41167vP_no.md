## 2.6 文本文件查看命令 👀

![](img/0e97be835b329f8182e1909717503965_9.png)

上一节我们介绍了如何查找文件和目录，本节中我们来看看如何查看文件的内容。Linux系统遵循“一切皆文件”的哲学，因此掌握查看文件的方法是基础中的基础。

### cat命令

![](img/0e97be835b329f8182e1909717503965_11.png)

![](img/0e97be835b329f8182e1909717503965_13.png)

![](img/0e97be835b329f8182e1909717503965_15.png)

`cat`命令用于查看内容较少的纯文本文件。其名称来源于英文单词“concatenate”（连接），但在这里主要用于显示文件内容。

![](img/0e97be835b329f8182e1909717503965_17.png)

**命令格式**：
```bash
cat [文件名]
```

![](img/0e97be835b329f8182e1909717503965_19.png)

执行该命令后，文件的所有内容会一次性显示在终端屏幕上。对于小文件来说，这非常方便直观。

![](img/0e97be835b329f8182e1909717503965_21.png)

### more命令

![](img/0e97be835b329f8182e1909717503965_23.png)

![](img/0e97be835b329f8182e1909717503965_25.png)

当查看内容较多的文本文件时，使用`cat`命令会导致信息快速滚过屏幕，难以阅读。此时，`more`命令是更好的选择。

![](img/0e97be835b329f8182e1909717503965_27.png)

![](img/0e97be835b329f8182e1909717503965_29.png)

**命令格式**：
```bash
more [文件名]
```

![](img/0e97be835b329f8182e1909717503965_31.png)

![](img/0e97be835b329f8182e1909717503965_33.png)

`more`命令会以分页的形式显示文件内容。您可以使用以下按键进行交互：
*   按**空格键**向下翻一页。
*   按**回车键**向下滚动一行。
*   按 **`q`** 键退出查看。
*   屏幕左下角会显示当前阅读的进度百分比。

![](img/0e97be835b329f8182e1909717503965_35.png)

![](img/0e97be835b329f8182e1909717503965_37.png)

**建议**：如果您不确定文件大小，可以统一使用`more`命令来查看，以避免信息刷屏。

![](img/0e97be835b329f8182e1909717503965_39.png)

### head命令

![](img/0e97be835b329f8182e1909717503965_41.png)

有时我们只关心文件开头部分的内容，这时可以使用`head`命令。

**命令格式**：
```bash
head -n [行数] [文件名]
```

![](img/0e97be835b329f8182e1909717503965_43.png)

**参数解释**：
*   `-n`：这是一个格式参数，本身没有具体含义，但它后面接的数字指定了要显示的行数。
*   `[行数]`：要查看文件前多少行。
*   `[文件名]`：要查看的目标文件。

![](img/0e97be835b329f8182e1909717503965_45.png)

例如，`head -n 15 initial-setup-ks.cfg` 会显示该文件的前15行。

### tail命令

![](img/0e97be835b329f8182e1909717503965_47.png)

![](img/0e97be835b329f8182e1909717503965_49.png)

与`head`命令相反，`tail`命令用于查看文件末尾部分的内容。

![](img/0e97be835b329f8182e1909717503965_51.png)

**命令格式**：
```bash
tail -n [行数] [文件名]
```

![](img/0e97be835b329f8182e1909717503965_53.png)

例如，`tail -n 15 initial-setup-ks.cfg` 会显示该文件的最后15行。

![](img/0e97be835b329f8182e1909717503965_55.png)

`tail`命令还有一个强大的功能：**实时刷新日志文件**。这在系统排错或监控服务状态时非常有用。

![](img/0e97be835b329f8182e1909717503965_57.png)

![](img/0e97be835b329f8182e1909717503965_59.png)

**命令格式（实时刷新）**：
```bash
tail -f [文件名]
```

![](img/0e97be835b329f8182e1909717503965_61.png)

执行此命令后，终端会持续监控并显示文件新增的内容，直到您按下 **`Ctrl + C`** 终止。例如，在一个终端窗口执行 `tail -f /var/log/messages` 监控系统日志，同时在另一个终端重启网络服务，您就能实时看到日志的更新。

![](img/0e97be835b329f8182e1909717503965_63.png)

### tr命令

`tr`命令用于替换文本文件中的字符。它本身不能直接读取文件，需要配合管道符或其他命令使用。

![](img/0e97be835b329f8182e1909717503965_65.png)

![](img/0e97be835b329f8182e1909717503965_67.png)

![](img/0e97be835b329f8182e1909717503965_69.png)

**命令格式（常用）**：
```bash
cat [文件名] | tr [原始字符] [目标字符]
```

**参数解释**：
*   `|`：管道符。它的作用是将前一个命令的输出结果，作为后一个命令的输入进行处理。
*   `[原始字符]`：需要被替换的字符或字符范围。
*   `[目标字符]`：替换后的字符。

![](img/0e97be835b329f8182e1909717503965_71.png)

**示例**：
1.  将所有小写字母转换为大写字母：
    ```bash
    cat test.txt | tr [a-z] [A-Z]
    ```
2.  将文件中所有的字母`a`替换为字母`b`：
    ```bash
    cat test.txt | tr a b
    ```

![](img/0e97be835b329f8182e1909717503965_73.png)

![](img/0e97be835b329f8182e1909717503965_75.png)

![](img/0e97be835b329f8182e1909717503965_77.png)

![](img/0e97be835b329f8182e1909717503965_79.png)

**注意**：以上命令仅将替换结果显示在屏幕上，并未修改原文件。如需修改原文件，需要结合后续讲到的重定向功能。

![](img/0e97be835b329f8182e1909717503965_81.png)

![](img/0e97be835b329f8182e1909717503965_83.png)

### wc命令

![](img/0e97be835b329f8182e1909717503965_85.png)

![](img/0e97be835b329f8182e1909717503965_87.png)

`wc`命令用于统计文本文件的行数、单词数和字节数。

![](img/0e97be835b329f8182e1909717503965_89.png)

**命令格式**：
```bash
wc [参数] [文件名]
```

![](img/0e97be835b329f8182e1909717503965_91.png)

**常用参数**：
*   `-l`：统计行数。
*   `-w`：统计单词数。
*   `-c`：统计字节数。

例如：
*   `wc -l test.txt` 统计文件行数。
*   `wc -w test.txt` 统计文件单词数。
*   `wc -c test.txt` 统计文件字节数。

![](img/0e97be835b329f8182e1909717503965_93.png)

### stat命令

![](img/0e97be835b329f8182e1909717503965_95.png)

`stat`命令用于查看文件的详细时间属性等信息，比简单的`ls -l`命令显示的时间更精确。

![](img/0e97be835b329f8182e1909717503965_97.png)

**命令格式**：
```bash
stat [文件名]
```

![](img/0e97be835b329f8182e1909717503965_99.png)

![](img/0e97be835b329f8182e1909717503965_101.png)

![](img/0e97be835b329f8182e1909717503965_103.png)

执行后会显示文件的三个重要时间戳：
*   **Access Time (atime)**：最后一次**访问**文件内容的时间。
*   **Modify Time (mtime)**：最后一次**修改**文件内容的时间。
*   **Change Time (ctime)**：最后一次**更改**文件属性（如权限、所有者）的时间。

**注意**：修改文件内容通常会导致文件大小变化，因此`mtime`和`ctime`常常会同时更新。

### grep命令

`grep`命令用于按行提取包含指定关键词的文本。它是Linux中非常强大和常用的命令。

![](img/0e97be835b329f8182e1909717503965_105.png)

![](img/0e97be835b329f8182e1909717503965_107.png)

**命令格式**：
```bash
grep [参数] “关键词” [文件名]
```

![](img/0e97be835b329f8182e1909717503965_109.png)

![](img/0e97be835b329f8182e1909717503965_111.png)

**常用参数**：
*   `-n`：显示匹配行及其在文件中的行号。
*   `-v`：反选，显示**不包含**关键词的所有行。

![](img/0e97be835b329f8182e1909717503965_113.png)

**示例**：
在文件 `test.txt` 中搜索包含 “oo” 的行：
```bash
grep “oo” test.txt
```
搜索并显示行号：
```bash
grep -n “oo” test.txt
```

![](img/0e97be835b329f8182e1909717503965_115.png)

### cut命令

![](img/0e97be835b329f8182e1909717503965_117.png)

![](img/0e97be835b329f8182e1909717503965_119.png)

![](img/0e97be835b329f8182e1909717503965_121.png)

`grep`命令是按行提取，而`cut`命令则是**按列提取**文本。它特别适用于处理由特定分隔符（如冒号、逗号、制表符）格式化的数据。

![](img/0e97be835b329f8182e1909717503965_123.png)

![](img/0e97be835b329f8182e1909717503965_125.png)

![](img/0e97be835b329f8182e1909717503965_127.png)

**命令格式**：
```bash
cut -d [分隔符] -f [列号] [文件名]
```

**参数解释**：
*   `-d`：指定列之间的分隔符。
*   `-f`：指定要提取的列编号。

**经典示例**：
Linux系统的用户信息保存在 `/etc/passwd` 文件中，每行一个用户，字段间用冒号`:`分隔。
1.  提取系统中所有用户的用户名（第一列）：
    ```bash
    cut -d : -f 1 /etc/passwd
    ```
2.  提取所有用户的用户ID（第三列）：
    ```bash
    cut -d : -f 3 /etc/passwd
    ```

![](img/0e97be835b329f8182e1909717503965_129.png)

![](img/0e97be835b329f8182e1909717503965_131.png)

### diff命令

![](img/0e97be835b329f8182e1909717503965_133.png)

![](img/0e97be835b329f8182e1909717503965_135.png)

`diff`命令用于比较**两个文本文件的差异**。

**命令格式**：
```bash
diff [参数] 文件A 文件B
```

![](img/0e97be835b329f8182e1909717503965_137.png)

**常用参数**：
*   `–brief` 或 `-q`：仅报告文件是否不同，不显示具体差异。
*   `-c`：以上下文格式详细显示差异。

![](img/0e97be835b329f8182e1909717503965_139.png)

**示例**：
比较文件 `a.txt` 和 `b.txt`：
```bash
diff --brief a.txt b.txt
```
如果文件不同，使用详细模式查看差异：
```bash
diff -c a.txt b.txt
```
输出结果中，`!` 表示该行有变化，`-` 表示文件A独有的行，`+` 表示文件B独有的行。

![](img/0e97be835b329f8182e1909717503965_141.png)

![](img/0e97be835b329f8182e1909717503965_143.png)

### uniq命令

![](img/0e97be835b329f8182e1909717503965_145.png)

![](img/0e97be835b329f8182e1909717503965_147.png)

![](img/0e97be835b329f8182e1909717503965_149.png)

`uniq`命令用于**去除文本文件中连续的重复行**。

**命令格式**：
```bash
uniq [文件名]
```

**重要限制**：`uniq`命令默认只能去除**相邻的**重复行。如果重复行不相邻，则需要先使用`sort`命令排序，再用管道符传递给`uniq`。

![](img/0e97be835b329f8182e1909717503965_151.png)

![](img/0e97be835b329f8182e1909717503965_153.png)

**示例**：
文件 `c.txt` 内容为：apple, banana, apple, orange。
1.  直接去重无效（两个apple不相邻）：
    ```bash
    uniq c.txt
    ```
2.  先排序，再去重：
    ```bash
    sort c.txt | uniq
    ```
    或者使用`sort`命令的 `-u` 参数一步完成：
    ```bash
    sort -u c.txt
    ```

![](img/0e97be835b329f8182e1909717503965_155.png)

### sort命令

![](img/0e97be835b329f8182e1909717503965_157.png)

`sort`命令用于**对文本文件的行进行排序**。

![](img/0e97be835b329f8182e1909717503965_159.png)

![](img/0e97be835b329f8182e1909717503965_161.png)

![](img/0e97be835b329f8182e1909717503965_163.png)

**命令格式**：
```bash
sort [参数] [文件名]
```

![](img/0e97be835b329f8182e1909717503965_165.png)

![](img/0e97be835b329f8182e1909717503965_167.png)

**常用参数**：
*   `-n`：按照数字大小进行排序（默认按字符ASCII码排序）。
*   `-u`：去除重复行（相当于 `sort | uniq`）。

![](img/0e97be835b329f8182e1909717503965_169.png)

![](img/0e97be835b329f8182e1909717503965_171.png)

**示例**：
1.  对单词进行字母排序：
    ```bash
    sort fruits.txt
    ```
2.  对数字进行排序（必须加 `-n` 参数）：
    ```bash
    sort -n numbers.txt
    ```

## 2.7 文件目录管理命令 📁

![](img/0e97be835b329f8182e1909717503965_175.png)

上一节我们学习了如何查看和处理文件内容，本节中我们来看看如何管理文件本身，包括创建、复制、移动和删除等操作。

![](img/0e97be835b329f8182e1909717503965_177.png)

### touch命令

![](img/0e97be835b329f8182e1909717503965_179.png)

![](img/0e97be835b329f8182e1909717503965_181.png)

![](img/0e97be835b329f8182e1909717503965_183.png)

`touch`命令有两个主要功能：
1.  **创建空白的新文件**。
2.  **修改文件的时间戳**（atime和mtime）。

![](img/0e97be835b329f8182e1909717503965_185.png)

**命令格式（创建文件）**：
```bash
touch [文件名]
```

![](img/0e97be835b329f8182e1909717503965_187.png)

**注意**：在Linux中，文件类型不依赖于文件名后缀（如`.txt`），而是由文件本身的元数据决定。后缀名主要是为了方便用户识别。

**命令格式（修改时间戳）**：
```bash
touch -d “YYYY-MM-DD HH:MM” [文件名]
```
例如，将文件 `test.txt` 的时间戳改为2023年10月1日10点30分：
```bash
touch -d “2023-10-01 10:30” test.txt
```

### mkdir命令

`mkdir`命令用于**创建新的目录（文件夹）**。

![](img/0e97be835b329f8182e1909717503965_189.png)

**命令格式**：


![](img/0e97be835b329f8182e1909717503965_191.png)

如果需要创建一串嵌套的目录（即父目录不存在），则需要使用 `-p` 参数进行递归创建。

**示例**：
创建 `/home/test/a/b/c` 这一系列目录：
```bash
mkdir -p /home/test/a/b/c
```

### rm命令

![](img/0e97be835b329f8182e1909717503965_221.png)

`rm`命令用于**删除文件或目录**。**请谨慎使用此命令，因为Linux下删除的文件通常难以恢复，且没有回收站机制。**

**命令格式（删除文件）**：
```bash
rm [文件名]
```
系统会询问是否确认删除。

**命令格式（强制删除文件，不询问）**：
```bash
rm -f [文件名]
```

**命令格式（删除目录）**：
删除目录必须使用 `-r`（递归）参数。
```bash
rm -r [目录名]
```
通常结合 `-f` 参数强制删除目录及其内部所有内容：
```bash
rm -rf [目录名]
```

![](img/0e97be835b329f8182e1909717503965_223.png)

**⚠️ 严重警告**：命令 `rm -rf /*` 会尝试删除根目录下的所有文件，可能导致系统崩溃，**绝对不要在生产环境或重要系统中尝试**。

### dd命令

![](img/0e97be835b329f8182e1909717503965_225.png)

`dd`命令是一个功能强大的数据复制工具，可以**按块复制文件或设备，并支持转换格式**。它常用于备份、磁盘克隆、创建测试文件等。

**命令格式**：
```bash
dd if=[输入文件] of=[输出文件] bs=[块大小] count=[块数量]
```

**参数解释**：
*   `if`：输入文件（input file）。
*   `of`：输出文件（output file）。
*   `bs`：每个块的大小（bytes）。可以使用单位，如 `K`（千字节）、`M`（兆字节）、`G`（吉字节）。
*   `count`：要复制的块数量。
*   **总复制大小 = bs * count**

**示例**：
1.  从 `/dev/zero`（一个能无限提供空字符的设备文件）创建大小为1GB的测试文件：
    ```bash
    dd if=/dev/zero of=testfile bs=1M count=1024
    ```
    这个命令也常用来简单测试磁盘的写入速度。
2.  备份磁盘的MBR（主引导记录，前512字节）：
    ```bash
    dd if=/dev/sda of=mbr_backup bs=512 count=1
    ```

### tar命令

`tar`命令用于**对文件或目录进行打包和压缩**，是Linux中最常用的归档工具。

![](img/0e97be835b329f8182e1909717503965_247.png)

**命令格式（打包并压缩）**：
```bash
tar czvf [压缩包名.tar.gz] [要打包的文件或目录...]
```

![](img/0e97be835b329f8182e1909717503965_249.png)

![](img/0e97be835b329f8182e1909717503965_251.png)

![](img/0e97be835b329f8182e1909717503965_253.png)

**命令格式（解压）**：
```bash
tar xzvf [压缩包名.tar.gz]
```

![](img/0e97be835b329f8182e1909717503965_255.png)

![](img/0e97be835b329f8182e1909717503965_257.png)

**参数解释**：
*   `c`：创建新的压缩包。
*   `x`：从压缩包解压文件。
*   `z`：使用gzip算法进行压缩或解压（对应`.tar.gz`或`.tgz`后缀）。
*   `v`：显示详细的处理过程。
*   `f`：指定压缩包的文件名。**这个参数必须放在所有参数的最后**。
*   `-C [目录]`：解压到指定目录。

![](img/0e97be835b329f8182e1909717503965_259.png)

![](img/0e97be835b329f8182e1909717503965_261.png)

**示例**：
1.  将 `/etc` 目录打包压缩为 `etc_backup.tar.gz`：
    ```bash
    tar czvf etc_backup.tar.gz /etc
    ```
2.  将压缩包解压到当前目录：
    ```bash
    tar xzvf etc_backup.tar.gz
    ```
3.  将压缩包解压到 `/tmp` 目录：
    ```bash
    tar xzvf etc_backup.tar.gz -C /tmp
    ```

![](img/0e97be835b329f8182e1909717503965_263.png)

**注意**：在RHEL 8/CentOS 8中，解压时通常可以省略 `z` 参数，`tar` 命令会自动识别压缩格式。

## 2.8 管道符、重定向与环境变量 🔄

本节我们将学习如何让命令之间、命令与文件之间协同工作，这是提升Linux操作效率的关键。

### 管道符 |

![](img/0e97be835b329f8182e1909717503965_269.png)

管道符 `|` 的作用是：**将前一个命令的输出结果，作为后一个命令的输入**。这就像工厂的流水线，让数据在不同命令间传递和处理。

**命令格式**：
```bash
命令A | 命令B
```

**示例**：
1.  在 `/etc/passwd` 文件中查找包含 “root” 的行：
    ```bash
    cat /etc/passwd | grep root
    ```
    （这等价于 `grep root /etc/passwd`，但演示了管道符的用法）
2.  统计当前目录下文件和目录的总数：
    ```bash
    ls -l | wc -l
    ```

![](img/0e97be835b329f8182e1909717503965_271.png)

### 输出重定向

![](img/0e97be835b329f8182e1909717503965_273.png)

输出重定向用于**将命令的输出结果写入到文件中**，而不是显示在屏幕上。它分为以下几种情况：

![](img/0e97be835b329f8182e1909717503965_275.png)

**1. 标准输出重定向**
*   `>`：**覆盖写入**。清空文件原有内容，再写入新内容。
    ```bash
    echo “Hello World” > hello.txt
    ```
*   `>>`：**追加写入**。在文件原有内容的末尾添加新内容。
    ```bash
    echo “Another Line” >> hello.txt
    ```

![](img/0e97be835b329f8182e1909717503965_277.png)

**2. 错误输出重定向**
命令的执行结果分为“标准输出”（正确信息）和“错误输出”（报错信息）。错误输出重定向专门用于捕获错误信息。
*   `2>`：覆盖写入错误信息。
*   `2>>`：追加写入错误信息。
    ```bash
    ls no_such_file 2> error.log
    ```

**3. 混合输出重定向**
将标准输出和错误输出都重定向到同一个文件。
*   `&>`：覆盖写入所有输出。
*   `&>>`：追加写入所有输出。
    ```bash
    ls some_file no_such_file &> output.log
    ```

### 输入重定向

输入重定向 `<` 用于**将文件的内容作为命令的输入**。它与管道符功能类似，但数据流向是从文件到命令。

**命令格式**：
```bash
命令 < [文件名]
```

**示例**：
使用 `wc -l` 统计文件行数：
1.  常规用法（文件作为参数）：
    ```bash
    wc -l /etc/passwd
    ```
    输出：`45 /etc/passwd`
2.  输入重定向用法：
    ```bash
    wc -l < /etc/passwd
    ```
    输出：`45`

![](img/0e97be835b329f8182e1909717503965_279.png)

两者的区别在于，第二种方式文件内容是通过重定向“流入”`wc`命令的，命令本身并未“看到”文件名，因此只输出行数结果。
