## 概述

在本节课中，我们将学习Linux系统中两个非常重要的工具：vim文本编辑器和计划任务。vim是编辑配置文件的核心工具，而计划任务则用于自动化执行周期性或一次性的系统任务。掌握这两项技能是进行Linux系统管理和服务配置的基础。

### Vim的三种模式

Vim编辑器包含三种主要模式，理解它们之间的切换是熟练使用vim的关键。

*   **命令模式**：这是打开文件后的默认模式。在此模式下，可以移动光标、使用快捷键（如复制、粘贴、删除），但不能直接输入文本。
*   **输入模式**：在此模式下，可以向文件中插入或修改文本。从命令模式可以进入输入模式。
*   **末行模式**：在此模式下，可以执行保存、退出、查找替换等文件级操作。末行模式的特征是光标会移动到屏幕底部，并以冒号`:`开始。

![](img/0d7497831b577d813431aec7a37ef748_21.png)

![](img/0d7497831b577d813431aec7a37ef748_23.png)

![](img/0d7497831b577d813431aec7a37ef748_25.png)

三种模式之间的切换关系如下图所示：

**从命令模式进入输入模式**，有以下几种常用方式：
*   **`a`**：在**光标后**一位开始插入。
*   **`i`**：在**光标当前位置**开始插入。
*   **`o`**：在**当前行下方**新建一行并开始插入。

在输入模式下，按 **`ESC`** 键可以退回到命令模式。

![](img/0d7497831b577d813431aec7a37ef748_47.png)

![](img/0d7497831b577d813431aec7a37ef748_49.png)

在命令模式下，按 **`:`** 键可以进入末行模式。在末行模式下，可以执行以下操作：
*   **`:w`**：保存文件。
*   **`:q`**：退出vim。如果文件有未保存的修改，此命令会失败。
*   **`:q!`**：强制退出，不保存任何修改。
*   **`:wq`**：保存并退出。

![](img/0d7497831b577d813431aec7a37ef748_51.png)

![](img/0d7497831b577d813431aec7a37ef748_53.png)

**注意**：命令必须严格按照顺序，例如 `:qw` 是无效的。

### 命令模式下的常用快捷键

![](img/0d7497831b577d813431aec7a37ef748_63.png)

在命令模式下，熟练使用快捷键可以极大提升编辑效率。以下是移动光标和文本编辑的常用快捷键。

**光标移动**：
*   **`h`**, **`j`**, **`k`**, **`l`**：分别控制光标向左、下、上、右移动。
*   **`gg`**：跳转到文件**首行**。
*   **`G`** (Shift + g)：跳转到文件**末行**。
*   **`nG`** 或 **`:n`**：跳转到第n行。例如，`:100` 跳转到第100行。

**文本搜索**：
*   **`/关键词`**：自顶向下搜索文件中的“关键词”。按 **`n`** 跳转到下一个匹配项，按 **`N`** (Shift + n) 跳转到上一个匹配项。
*   **`?关键词`**：自底向上搜索文件中的“关键词”。

**复制、粘贴与删除**：
*   **`yy`**：复制光标所在行。
*   **`nyy`**：复制从光标行开始的n行。例如，`5yy` 复制5行。
*   **`dd`**：删除（剪切）光标所在行。
*   **`ndd`**：删除从光标行开始的n行。
*   **`p`**：将复制或删除的内容粘贴到光标**之后**。
*   **`u`**：撤销上一步操作。

### 末行模式进阶功能

除了保存退出，末行模式还支持显示行号和文本替换。

![](img/0d7497831b577d813431aec7a37ef748_65.png)

**显示行号**：
在末行输入 `:set nu` 可以显示行号，输入 `:set nonu` 可以取消显示。

![](img/0d7497831b577d813431aec7a37ef748_67.png)

**文本替换**：
替换命令的基本格式为 `:[范围]s/旧文本/新文本/[选项]`。
*   **`:s/one/two`**：将光标所在行的**第一个** “one” 替换为 “two”。
*   **`:s/one/two/g`**：将光标所在行的**所有** “one” 替换为 “two”。
*   **`:%s/one/two/g`**：将**全文**的 “one” 替换为 “two”。
*   **`:1,4s/one/two/g`**：将第1行到第4行的所有 “one” 替换为 “two”。

## 计划任务 ⏰

![](img/0d7497831b577d813431aec7a37ef748_76.png)

![](img/0d7497831b577d813431aec7a37ef748_78.png)

系统管理员经常需要执行一些周期性任务，如每日备份。手动执行既繁琐又容易遗忘，Linux的计划任务功能可以完美解决这个问题。

### 计划任务概述

![](img/0d7497831b577d813431aec7a37ef748_80.png)

Linux中的计划任务主要分为两类：
*   **一次性任务**：在指定的未来某个时间点执行一次。使用 `at` 命令管理。
*   **周期性任务**：按照设定的周期（如每分钟、每天、每周）重复执行。使用 `cron` 服务管理。

![](img/0d7497831b577d813431aec7a37ef748_82.png)

这两类任务都依赖于后台的守护进程（`atd` 和 `crond`）来调度和执行。

![](img/0d7497831b577d813431aec7a37ef748_84.png)

### 一次性任务：at

![](img/0d7497831b577d813431aec7a37ef748_86.png)

![](img/0d7497831b577d813431aec7a37ef748_87.png)

![](img/0d7497831b577d813431aec7a37ef748_89.png)

![](img/0d7497831b577d813431aec7a37ef748_91.png)

![](img/0d7497831b577d813431aec7a37ef748_93.png)

![](img/0d7497831b577d813431aec7a37ef748_95.png)

`at` 命令用于安排一次性任务。任务执行后的输出会以邮件形式发送给用户。

![](img/0d7497831b577d813431aec7a37ef748_97.png)

![](img/0d7497831b577d813431aec7a37ef748_99.png)

**基本用法**：
1.  使用 `at [时间]` 进入at命令交互界面。
2.  输入要执行的命令。
3.  按 **`Ctrl + D`** 保存并退出。

![](img/0d7497831b577d813431aec7a37ef748_101.png)

**时间格式示例**：
*   `at now + 2 minutes`：2分钟后执行。
*   `at 15:00`：今天下午3点执行。
*   `at 11:00 PM`：晚上11点执行。
*   `at 11:00 2024-12-31`：2024年12月31日上午11点执行。

**管理at任务**：
*   `atq` 或 `at -l`：查看等待执行的at任务列表。
*   `atrm [任务编号]` 或 `at -d [任务编号]`：删除指定的at任务。

**使用脚本文件**：
可以将多条命令写入一个脚本文件（如 `at_job.sh`），然后通过输入重定向让at执行。
```bash
at now + 1 minute < /path/to/at_job.sh
```

### 周期性任务：cron

`cron` 是用于设置周期性任务的服务。用户通过 `crontab` 命令来管理自己的cron任务。

![](img/0d7497831b577d813431aec7a37ef748_103.png)

![](img/0d7497831b577d813431aec7a37ef748_105.png)

**crontab命令**：
*   `crontab -e`：编辑当前用户的cron任务表。
*   `crontab -l`：列出当前用户的cron任务。
*   `crontab -r`：删除当前用户的所有cron任务。

**cron时间格式**：
cron任务的一行由6个字段组成，格式如下：
```
分 时 日 月 周 要执行的命令
```
*   **分**：0-59
*   **时**：0-23
*   **日**：1-31
*   **月**：1-12
*   **周**：0-7 (0和7都代表周日)
*   **命令**：需要执行的命令或脚本的**绝对路径**。

**特殊符号**：
*   `*`：代表所有可能的值。例如，在“分”字段是 `*` 表示每分钟。
*   `,`：代表分隔多个值。例如，`1,3,5` 表示第1、3、5个时间单位。
*   `-`：代表一个范围。例如，`1-5` 表示第1到第5个时间单位。
*   `/`：代表步长。例如，在“分”字段是 `*/5` 表示每5分钟。

**格式示例**：
*   `0 2 * * * /backup.sh`：每天凌晨2点整执行 `/backup.sh` 脚本。
*   `*/5 * * * * /monitor.sh`：每5分钟执行一次 `/monitor.sh` 脚本。
*   `0 17 * * 1,3,5 /report.sh`：每周一、三、五的下午5点执行 `/report.sh` 脚本。
*   `30 3 1 * * /monthly_backup.sh`：每月1日凌晨3点30分执行 `/monthly_backup.sh` 脚本。

![](img/0d7497831b577d813431aec7a37ef748_107.png)

**注意事项**：
1.  “日”和“周”字段最好不要同时指定，因为它们是“或”的关系，可能产生非预期的执行时间。
2.  命令一定要使用绝对路径，因为cron执行任务时的环境变量可能与用户登录时不同。
3.  如果系统在计划任务时间点关机，错过的任务**不会**补执行。对于非常重要的每日任务，可以考虑将其脚本放入 `/etc/cron.daily/` 等系统目录，由 `anacron` 服务管理，它会在开机后执行错过的重要任务。
