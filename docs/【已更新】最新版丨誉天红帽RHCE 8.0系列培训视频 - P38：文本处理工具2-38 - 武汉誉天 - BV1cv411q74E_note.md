在本节课中，我们将深入学习流编辑器SED的进阶用法，包括如何使用地址范围精确指定操作目标，以及执行删除、追加、插入、替换等编辑操作。掌握这些技巧能让你高效地处理文本文件。

## 地址范围的指定方法

上一节我们介绍了SED的基本模式匹配，本节中我们来看看如何更精确地指定操作的行范围。

![](img/4c16db4232edfe9d2647ae396ee002f0_5.png)

SED允许你通过多种方式指定一个地址范围，以限定命令生效的区域。

![](img/4c16db4232edfe9d2647ae396ee002f0_7.png)

以下是几种常见的地址范围指定方式：

![](img/4c16db4232edfe9d2647ae396ee002f0_9.png)

*   **指定单行**：使用行号，例如 `2` 表示只对第二行进行操作。
*   **指定多行范围**：使用起始行号和结束行号，中间用逗号分隔。例如 `1,3` 表示对第一到第三行进行操作。
*   **模式匹配到行号**：使用模式匹配作为起点，行号作为终点。例如 `/root/,3` 表示从第一个包含“root”的行开始，到第三行结束。
*   **模式匹配到模式匹配**：使用两个模式匹配来界定范围。例如 `/user/,/root/` 表示从第一个包含“user”的行开始，到第一个包含“root”的行结束。
*   **指定步长**：使用波浪号 `~` 指定步长。例如 `1~2` 表示从第一行开始，每隔一行操作一次（即操作第1、3、5、7...行）。
*   **指定行及其后N行**：使用加号 `+`。例如 `1,+5` 表示从第一行开始，以及其后的五行（共六行）。

![](img/4c16db4232edfe9d2647ae396ee002f0_11.png)

## 基于地址范围执行操作

![](img/4c16db4232edfe9d2647ae396ee002f0_13.png)

在指定了地址范围后，我们可以执行多种编辑操作。首先，我们回顾一下打印操作。

![](img/4c16db4232edfe9d2647ae396ee002f0_15.png)

### 打印内容

![](img/4c16db4232edfe9d2647ae396ee002f0_17.png)

使用 `p` 命令可以打印匹配的行。通常需要与 `-n` 选项配合使用，以抑制默认输出，只显示我们要求打印的行。

![](img/4c16db4232edfe9d2647ae396ee002f0_19.png)

```bash
sed -n ‘2p’ /etc/passwd
```

### 删除内容

![](img/4c16db4232edfe9d2647ae396ee002f0_21.png)

使用 `d` 命令可以删除匹配的行。**执行删除操作时，通常不需要 `-n` 选项**，因为SED会默认输出未被删除的行。

![](img/4c16db4232edfe9d2647ae396ee002f0_23.png)

以下是删除操作的示例：

```bash
# 删除第二行
sed ‘2d’ /etc/passwd

![](img/4c16db4232edfe9d2647ae396ee002f0_25.png)

# 删除第一到第五行
sed ‘1,5d’ /etc/passwd
```

![](img/4c16db4232edfe9d2647ae396ee002f0_27.png)

**重要提示**：以上命令默认只修改输出到屏幕的内容，**原文件不会被改变**。若想直接修改原文件，需要添加 `-i` 选项。

```bash
# 直接修改原文件，删除前五行
sed -i ‘1,5d’ /etc/passwd
```

由于 `-i` 选项会直接覆盖原文件且无法撤销，为安全起见，可以结合 `-i.bak` 选项。这会在修改前为原文件创建一个备份副本。

```bash
# 修改原文件，并自动创建名为 /etc/passwd.bak 的备份文件
sed -i.bak ‘1,5d’ /etc/passwd
```

![](img/4c16db4232edfe9d2647ae396ee002f0_29.png)

### 追加与插入内容

![](img/4c16db4232edfe9d2647ae396ee002f0_31.png)

`a` 和 `i` 命令用于在指定行的后面或前面添加新内容。

*   **`a`**：在匹配行**之后**追加文本。
*   **`i`**：在匹配行**之前**插入文本。

```bash
# 在第二行之后追加一行 “ABC”
sed ‘2 a\ABC’ /etc/passwd

![](img/4c16db4232edfe9d2647ae396ee002f0_33.png)

# 在第二行之前插入一行 “ABC”
sed ‘2 i\ABC’ /etc/passwd
```

![](img/4c16db4232edfe9d2647ae396ee002f0_35.png)

### 替换整行内容

![](img/4c16db4232edfe9d2647ae396ee002f0_37.png)

`c` 命令用于替换（change）整行的内容。

![](img/4c16db4232edfe9d2647ae396ee002f0_39.png)

```bash
# 将第二行的内容完全替换为 “New Line Content”
sed ‘2 c\New Line Content’ /etc/passwd
```

### 查找并替换部分内容

`s` 命令用于查找并替换行内的文本，其语法与Vim编辑器中的替换命令非常相似。基本格式为 `s/旧模式/新模式/标志`。

*   **旧模式**：可以是一个具体的字符串，也可以是一个正则表达式。
*   **新模式**：要替换成的文本。
*   **标志**：
    *   `g`：全局替换（替换行内所有匹配项，默认只替换第一个）。
    *   `i`：忽略大小写。
    *   `p`：如果替换成功，则打印该行。

![](img/4c16db4232edfe9d2647ae396ee002f0_41.png)

```bash
# 将每行中第一个 “root” 替换为 “AAAA”
sed ‘s/root/AAAA/’ /etc/passwd

# 将每行中所有的 “root” 替换为 “AAAA”
sed ‘s/root/AAAA/g’ /etc/passwd

![](img/4c16db4232edfe9d2647ae396ee002f0_43.png)

![](img/4c16db4232edfe9d2647ae396ee002f0_44.png)

# 全局替换并打印发生替换的行
sed -n ‘s/root/AAAA/gp’ /etc/passwd
```

## 课程总结

![](img/4c16db4232edfe9d2647ae396ee002f0_46.png)

本节课中我们一起学习了SED命令的核心编辑功能。我们掌握了如何通过行号、模式匹配和步长来精确指定操作的地址范围。在此基础上，我们实践了打印(`p`)、删除(`d`)、追加(`a`)、插入(`i`)、整行替换(`c`)以及查找替换(`s`)等关键操作，并了解了安全修改原文件的 `-i.bak` 选项。这些技能是进行自动化文本处理和脚本编写的坚实基础。
