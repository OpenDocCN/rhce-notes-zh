## 2.6 文本文件编辑命令

上一节我们介绍了如何查找文件和目录，本节中我们来看看如何查看和初步处理文件内容。Linux系统遵循“一切皆文件”的原则，因此掌握文件操作是基础中的基础。

![](img/73e66a8e069da75c78306e793b25dc9d_15.png)

![](img/73e66a8e069da75c78306e793b25dc9d_17.png)

### cat命令：查看文件内容

![](img/73e66a8e069da75c78306e793b25dc9d_19.png)

![](img/73e66a8e069da75c78306e793b25dc9d_21.png)

`cat`命令用于查看文件内容，其名称源于“concatenate”（连接）的缩写。该命令会将文件内容全部输出到终端屏幕上。

![](img/73e66a8e069da75c78306e793b25dc9d_23.png)

![](img/73e66a8e069da75c78306e793b25dc9d_25.png)

![](img/73e66a8e069da75c78306e793b25dc9d_27.png)

**命令格式**：
```bash
cat [文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_29.png)

例如，查看一个名为 `anaconda-ks.cfg` 的文件：
```bash
cat anaconda-ks.cfg
```
执行后，文件内容会完整显示在屏幕上。

![](img/73e66a8e069da75c78306e793b25dc9d_31.png)

![](img/73e66a8e069da75c78306e793b25dc9d_33.png)

![](img/73e66a8e069da75c78306e793b25dc9d_35.png)

**注意事项**：`cat`命令适合查看内容较少的文件。如果文件内容很长，信息会在屏幕上快速滚动，导致用户难以阅读。

![](img/73e66a8e069da75c78306e793b25dc9d_37.png)

![](img/73e66a8e069da75c78306e793b25dc9d_39.png)

### more命令：分页查看文件

为了解决查看大文件的问题，可以使用 `more` 命令。该命令会以分页形式显示文件内容，方便用户逐行或逐页浏览。

![](img/73e66a8e069da75c78306e793b25dc9d_41.png)

**命令格式**：
```bash
more [文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_43.png)

使用 `more` 命令时：
*   按 **回车键** 向下滚动一行。
*   按 **空格键** 向下翻一页。
*   屏幕左下角会显示当前阅读的进度百分比。

### head与tail命令：查看文件首尾

![](img/73e66a8e069da75c78306e793b25dc9d_45.png)

![](img/73e66a8e069da75c78306e793b25dc9d_47.png)

有时我们只关心文件的开头或结尾部分，这时可以使用 `head` 和 `tail` 命令。

![](img/73e66a8e069da75c78306e793b25dc9d_49.png)

![](img/73e66a8e069da75c78306e793b25dc9d_51.png)

*   **head命令**：查看文件开头若干行。
    **命令格式**：
    ```bash
    head -n [行数] [文件名]
    ```
    例如，查看文件前15行：
    ```bash
    head -n 15 [文件名]
    ```

![](img/73e66a8e069da75c78306e793b25dc9d_53.png)

*   **tail命令**：查看文件末尾若干行。
    **命令格式**：
    ```bash
    tail -n [行数] [文件名]
    ```
    例如，查看文件末尾15行：
    ```bash
    tail -n 15 [文件名]
    ```

![](img/73e66a8e069da75c78306e793b25dc9d_55.png)

![](img/73e66a8e069da75c78306e793b25dc9d_57.png)

**tail命令的-f参数**：`tail` 命令有一个非常实用的 `-f` 参数，用于持续刷新并显示文件的最新内容，常用于实时监控日志文件。
```bash
tail -f [日志文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_59.png)

### tr命令：替换文件内容

![](img/73e66a8e069da75c78306e793b25dc9d_61.png)

`tr` 命令用于替换或删除文件中的字符。它通常需要与其他命令（如 `cat`）结合使用，通过“管道符”（`|`）将前一个命令的输出作为其输入。

![](img/73e66a8e069da75c78306e793b25dc9d_63.png)

**命令格式（通过管道）**：
```bash
cat [文件名] | tr [原字符] [目标字符]
```

![](img/73e66a8e069da75c78306e793b25dc9d_65.png)

![](img/73e66a8e069da75c78306e793b25dc9d_67.png)

以下是 `tr` 命令的一些应用示例：

*   **将小写字母转换为大写字母**：
    ```bash
    cat [文件名] | tr [a-z] [A-Z]
    ```
*   **将特定字符`a`替换为`b`**：
    ```bash
    cat [文件名] | tr a b
    ```

![](img/73e66a8e069da75c78306e793b25dc9d_69.png)

![](img/73e66a8e069da75c78306e793b25dc9d_71.png)

![](img/73e66a8e069da75c78306e793b25dc9d_73.png)

**注意**：以上命令仅将替换结果显示在屏幕上，并未实际修改原文件。若需保存修改，需结合后续将学到的“重定向”操作。

![](img/73e66a8e069da75c78306e793b25dc9d_75.png)

![](img/73e66a8e069da75c78306e793b25dc9d_77.png)

![](img/73e66a8e069da75c78306e793b25dc9d_79.png)

![](img/73e66a8e069da75c78306e793b25dc9d_81.png)

### wc命令：统计文件信息

![](img/73e66a8e069da75c78306e793b25dc9d_83.png)

![](img/73e66a8e069da75c78306e793b25dc9d_85.png)

`wc` 命令用于统计文件的行数、单词数和字节数。

![](img/73e66a8e069da75c78306e793b25dc9d_87.png)

**命令格式与参数**：
```bash
wc [参数] [文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_89.png)

常用参数：
*   `-l`：统计行数。
*   `-w`：统计单词数。
*   `-c`：统计字节数。

![](img/73e66a8e069da75c78306e793b25dc9d_91.png)

例如：
```bash
wc -l [文件名]  # 统计文件行数
```

![](img/73e66a8e069da75c78306e793b25dc9d_93.png)

### stat命令：查看文件时间属性

![](img/73e66a8e069da75c78306e793b25dc9d_95.png)

`stat` 命令可以查看文件的详细时间属性，每个文件在Linux中都有三个重要的时间戳：

![](img/73e66a8e069da75c78306e793b25dc9d_97.png)

![](img/73e66a8e069da75c78306e793b25dc9d_99.png)

![](img/73e66a8e069da75c78306e793b25dc9d_101.png)

*   **Access Time (atime)**：最后一次访问（读取）文件内容的时间。
*   **Change Time (ctime)**：最后一次更改文件属性（如权限、所有者）的时间。
*   **Modify Time (mtime)**：最后一次修改文件内容的时间。

**命令格式**：
```bash
stat [文件名]
```

**注意**：修改文件内容通常会导致其文件大小属性变化，因此 `ctime` 和 `mtime` 经常会同时更新。

![](img/73e66a8e069da75c78306e793b25dc9d_103.png)

### grep命令：按行提取信息

![](img/73e66a8e069da75c78306e793b25dc9d_105.png)

`grep` 命令用于在文件中搜索包含指定关键字的行，并将其显示出来。这是文本处理中极其常用的命令。

![](img/73e66a8e069da75c78306e793b25dc9d_107.png)

**命令格式**：
```bash
grep [关键词] [文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_109.png)

常用参数：
*   `-n`：显示匹配行及其在文件中的行号。
*   `-v`：反向选择，显示不包含关键词的所有行。

![](img/73e66a8e069da75c78306e793b25dc9d_111.png)

![](img/73e66a8e069da75c78306e793b25dc9d_113.png)

![](img/73e66a8e069da75c78306e793b25dc9d_115.png)

![](img/73e66a8e069da75c78306e793b25dc9d_117.png)

例如，在文件中搜索包含“root”的行：
```bash
grep root [文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_119.png)

### cut命令：按列提取信息

`cut` 命令用于按列提取文本信息，特别适用于处理由特定分隔符（如冒号、逗号）分隔的格式化文件，例如 `/etc/passwd`。

![](img/73e66a8e069da75c78306e793b25dc9d_121.png)

![](img/73e66a8e069da75c78306e793b25dc9d_123.png)

**命令格式**：
```bash
cut -d [分隔符] -f [列号] [文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_125.png)

![](img/73e66a8e069da75c78306e793b25dc9d_127.png)

应用示例：
*   **提取系统所有用户名**（`/etc/passwd`文件第一列）：
    ```bash
    cut -d : -f 1 /etc/passwd
    ```
*   **统计系统用户数量**（即统计`/etc/passwd`的行数）：
    ```bash
    wc -l /etc/passwd
    ```

![](img/73e66a8e069da75c78306e793b25dc9d_129.png)

### diff命令：比较文件差异

`diff` 命令用于比较两个文件的内容差异。

![](img/73e66a8e069da75c78306e793b25dc9d_131.png)

![](img/73e66a8e069da75c78306e793b25dc9d_133.png)

![](img/73e66a8e069da75c78306e793b25dc9d_135.png)

**命令格式**：
```bash
diff [选项] 文件A 文件B
```

![](img/73e66a8e069da75c78306e793b25dc9d_137.png)

![](img/73e66a8e069da75c78306e793b25dc9d_139.png)

![](img/73e66a8e069da75c78306e793b25dc9d_141.png)

常用选项：
*   `-brief` 或 `-q`：仅报告文件是否不同，不显示具体差异。
*   `-c`：以详细格式显示差异，并标识出不同的行。

例如，简要比较两个文件：
```bash
diff -brief fileA.txt fileB.txt
```

### sort命令：对文件内容排序

![](img/73e66a8e069da75c78306e793b25dc9d_153.png)

![](img/73e66a8e069da75c78306e793b25dc9d_155.png)

![](img/73e66a8e069da75c78306e793b25dc9d_157.png)

`sort` 命令用于对文本文件的行进行排序。

![](img/73e66a8e069da75c78306e793b25dc9d_159.png)

![](img/73e66a8e069da75c78306e793b25dc9d_161.png)

![](img/73e66a8e069da75c78306e793b25dc9d_163.png)

**命令格式**：
```bash
sort [选项] [文件名]
```

![](img/73e66a8e069da75c78306e793b25dc9d_165.png)

常用选项：
*   默认按字母顺序排序。
*   `-n`：按数值大小进行排序（用于处理数字）。
*   `-u`：排序并去除重复行（相当于 `sort | uniq`）。

![](img/73e66a8e069da75c78306e793b25dc9d_167.png)

应用示例：
*   **对数字文件进行正确排序**：
    ```bash
    sort -n number.txt
    ```
*   **排序并去重**：
    ```bash
    sort -u data.txt
    ```

## 2.7 文件目录管理命令

### touch命令：创建文件与修改时间

`touch` 命令主要有两个功能：
1.  **创建新的空文件**。
2.  **修改文件的时间戳**（atime和mtime）。

**命令格式**：
```bash
touch [文件名]
```
例如，创建一个名为 `testfile` 的文件：
```bash
touch testfile
```

**修改文件时间**：
```bash
touch -a -m -t 0801010101 [文件名] # 将访问和修改时间设为指定时间
```

`mv` 命令用于移动文件/目录，或对其进行重命名。其本质是“剪切”操作。

![](img/73e66a8e069da75c78306e793b25dc9d_207.png)

例如，将文件 `old.txt` 重命名为 `new.txt`：
```bash
mv old.txt new.txt
```

### rm命令：删除文件或目录

`rm` 命令用于删除文件或目录。**此命令需谨慎使用，因为Linux没有回收站，删除后难以恢复。**

![](img/73e66a8e069da75c78306e793b25dc9d_213.png)

**命令格式**：
*   **删除文件**：
    ```bash
    rm [文件名]
    ```
    系统会询问是否确认删除。
*   **强制删除文件（不询问）**：
    ```bash
    rm -f [文件名]
    ```
*   **删除目录**：必须使用 `-r` 参数进行递归删除。
    ```bash
    rm -r [目录名]
    ```
*   **强制删除目录**：
    ```bash
    rm -rf [目录名]
    ```

![](img/73e66a8e069da75c78306e793b25dc9d_215.png)

**⚠️ 严重警告**：命令 `rm -rf /*` 会尝试删除根目录下的所有文件，可能导致系统崩溃，**绝对不要在生产环境或重要系统中尝试**。

### dd命令：按块复制/转换文件

`dd` 命令是一个功能强大的低级别复制工具，可以按照数据块进行精确复制，常用于备份、磁盘克隆、创建特定大小文件等。

**命令格式**：
```bash
dd if=输入文件 of=输出文件 bs=块大小 count=块数量
```

![](img/73e66a8e069da75c78306e793b25dc9d_217.png)

![](img/73e66a8e069da75c78306e793b25dc9d_219.png)

![](img/73e66a8e069da75c78306e793b25dc9d_221.png)

参数说明：
*   `if`：输入文件（input file）。
*   `of`：输出文件（output file）。
*   `bs`：每个块的大小（如 512、1M、1G）。
*   `count`：要复制的块数量。
*   **总复制大小 = bs * count**

![](img/73e66a8e069da75c78306e793b25dc9d_223.png)

![](img/73e66a8e069da75c78306e793b25dc9d_225.png)

![](img/73e66a8e069da75c78306e793b25dc9d_227.png)

应用示例：
1.  **备份磁盘分区表**（备份硬盘第一个扇区）：
    ```bash
    dd if=/dev/sda of=partition_backup bs=512 count=1
    ```
2.  **创建指定大小的测试文件**（从“零设备” `/dev/zero` 读取）：
    ```bash
    dd if=/dev/zero of=testfile bs=1G count=1
    ```
3.  **测试磁盘写入速度**（通过创建大文件观察耗时）：
    ```bash
    dd if=/dev/zero of=speed_test bs=1G count=1 oflag=direct
    ```

![](img/73e66a8e069da75c78306e793b25dc9d_233.png)

### tar命令：打包与压缩

![](img/73e66a8e069da75c78306e793b25dc9d_243.png)

![](img/73e66a8e069da75c78306e793b25dc9d_245.png)

![](img/73e66a8e069da75c78306e793b25dc9d_247.png)

`tar` 命令用于将多个文件或目录打包成一个归档文件，并可选进行压缩。

![](img/73e66a8e069da75c78306e793b25dc9d_249.png)

**命令格式（常用组合）**：
*   **打包并压缩**：
    ```bash
    tar czvf 归档名.tar.gz 要打包的目录或文件
    ```
*   **解压**：
    ```bash
    tar xzvf 归档名.tar.gz
    ```

![](img/73e66a8e069da75c78306e793b25dc9d_251.png)

![](img/73e66a8e069da75c78306e793b25dc9d_253.png)

参数说明：
*   `c`：创建归档。
*   `x`：解压归档。
*   `z`：使用gzip压缩/解压（生成`.tar.gz`）。
*   `j`：使用bzip2压缩/解压（生成`.tar.bz2`）。
*   `v`：显示详细过程。
*   `f`：指定归档文件名。

## 3.2 重定向符

命令和文件如何互动？重定向符扮演了关键角色。它能够将命令的输出结果写入文件，或者将文件内容作为命令的输入。

### 输出重定向

输出重定向将命令的执行结果（本应显示在屏幕上）导入到文件中。

![](img/73e66a8e069da75c78306e793b25dc9d_257.png)

**类型**：
*   **标准输出重定向**：
    *   `命令 > 文件`：将**正确**输出结果**覆盖**写入文件。
    *   `命令 >> 文件`：将**正确**输出结果**追加**写入文件。
*   **错误输出重定向**：
    *   `命令 2> 文件`：将**错误**信息**覆盖**写入文件。
    *   `命令 2>> 文件`：将**错误**信息**追加**写入文件。
*   **混合输出重定向**（无论对错，全部写入）：
    *   `命令 &> 文件`：覆盖写入。
    *   `命令 &>> 文件`：追加写入。

![](img/73e66a8e069da75c78306e793b25dc9d_259.png)

![](img/73e66a8e069da75c78306e793b25dc9d_261.png)

**示例**：
```bash
ls > list.txt        # 将当前目录列表覆盖写入list.txt
ls not_exist 2> error.log # 将错误信息写入error.log
ls &> all_output.log # 将所有输出（正确和错误）写入all_output.log
```

![](img/73e66a8e069da75c78306e793b25dc9d_263.png)

### 输入重定向

输入重定向将文件的内容作为命令的输入。

**命令格式**：
```bash
命令 < 文件
```

**示例**：统计一个文件的行数。
```bash
# 传统方式，文件作为命令对象
wc -l filename.txt
# 使用输入重定向，文件内容作为输入流
wc -l < filename.txt
```
两者结果的主要区别在于，后者仅输出行数，不输出文件名。

**Here Document**：一种特殊的输入重定向，允许在命令行中直接嵌入多行输入，直到遇到指定的结束标记。
```bash
命令 << EOF
输入内容...
输入内容...
EOF
```
