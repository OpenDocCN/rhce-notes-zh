## 什么是Shell？它能做什么？

上一节我们结束了RHCSA阶段的内容，本节中我们来看看RHCE阶段的核心技能之一——Shell脚本。

![](img/64201f98c64e13fc5691c212dd65f17e_9.png)

Shell本身是一门编程语言。它就像计算机中的C语言、Java、Python等编程语言一样，是人与计算机交流的媒介。我们想管理或控制计算机，就必须使用计算机能理解的语言，即二进制（0和1）。然而，我们日常使用的命令（如 `ls`、`pwd`）并非二进制，这是因为Shell作为一门**解释型语言**，在背后充当了“翻译官”的角色。

![](img/64201f98c64e13fc5691c212dd65f17e_11.png)

编程语言主要分为两类：
*   **编译型语言**：如C、C++、Go。程序运行前需要专门的编译器将代码**提前编译**成计算机认识的二进制文件。
    *   **公式/代码示例**：`gcc source.c -o program` （将C源代码编译为可执行程序）
*   **解释型语言**：如Shell、Python、PHP。程序在**运行时**由解释器**一边执行一边翻译**成机器指令。

![](img/64201f98c64e13fc5691c212dd65f17e_13.png)

我们自学习Linux以来，输入的所有命令都属于Shell语言。它功能强大，支持变量、数组、函数、条件判断（`if`）、循环（`for`）等大部分编程语言具备的功能。

以下是Shell脚本可以完成的一些任务：
*   服务器的批量维护与初始化
*   软件的自动化部署与安装
*   系统监控与故障自动检测报警
*   定时任务（如凌晨自动备份数据）

![](img/64201f98c64e13fc5691c212dd65f17e_15.png)

![](img/64201f98c64e13fc5691c212dd65f17e_17.png)

![](img/64201f98c64e13fc5691c212dd65f17e_19.png)

Shell属于**高级语言**，离用户更近，使用起来相对方便。很多常用功能（如 `cat`、`ls`）已经由他人实现，我们直接组合使用即可，这降低了入门门槛，但其编程思维逻辑仍需认真培养。

## 为什么需要Shell脚本？

![](img/64201f98c64e13fc5691c212dd65f17e_21.png)

![](img/64201f98c64e13fc5691c212dd65f17e_22.png)

脚本的本质是**命令的堆积**。它将一系列需要手动输入的命令，按照顺序写入一个文件。

![](img/64201f98c64e13fc5691c212dd65f17e_24.png)

设想一个场景：公司每上线一台新服务器，都需要你部署一套完整的LNMP（Linux, Nginx, MySQL, PHP）环境。手动敲击几十甚至上百条命令不仅效率低下，而且容易出错。

此时，你可以将这些部署命令写成一个Shell脚本文件（例如 `deploy_lnmp.sh`）。下次再有新服务器上线，只需执行这个脚本文件，所有命令便会自动按序运行。你可以节省大量时间，让工作变得更加高效和“幸福”。

另一个典型场景是**数据备份**。备份操作通常安排在凌晨服务器空闲时进行，以避免影响用户体验。通过Shell脚本结合计划任务（如`cron`），你可以设定脚本在凌晨3点自动执行备份命令，而无需熬夜加班。

## 如何编写一个规范的Shell脚本？

![](img/64201f98c64e13fc5691c212dd65f17e_26.png)

下面我们来看看一个规范Shell脚本的组成部分。我们将创建一个简单的脚本，实现在屏幕上输出“Hello World”。

![](img/64201f98c64e13fc5691c212dd65f17e_27.png)

首先，创建一个用于存放练习脚本的目录，并进入该目录：
```bash
mkdir /script
cd /script
```

然后，创建一个脚本文件。脚本文件名通常以 `.sh` 结尾，这只是约定俗成，便于识别。
```bash
vim hello.sh
```

![](img/64201f98c64e13fc5691c212dd65f17e_29.png)

![](img/64201f98c64e13fc5691c212dd65f17e_31.png)

![](img/64201f98c64e13fc5691c212dd65f17e_33.png)

一个规范的Shell脚本主要包含以下几个部分：

![](img/64201f98c64e13fc5691c212dd65f17e_35.png)

### 1. 环境声明（Shebang）
脚本的第一行需要指定解释器，格式为 `#!` 后接解释器路径。这告诉系统使用哪个“翻译官”来执行脚本中的命令。
```bash
#!/bin/bash
```
*   `#!` 是一个特殊标记，称为“Shebang”。
*   `/bin/bash` 是Linux系统中最常用的Shell解释器路径。

### 2. 注释信息
注释以 `#` 开头，其后的内容不会被解释器执行。注释用于解释脚本的功能、作者、更新时间等，是良好的编程习惯，便于自己和他人阅读。
```bash
# Author: Your Name
# Description: This script prints “Hello World” to the screen.
```

### 3. 可执行命令
从第三行开始，写入你需要执行的命令。我们的目标是输出“Hello World”，因此使用 `echo` 命令。
```bash
echo “Hello World”
```

完整的 `hello.sh` 脚本内容如下：
```bash
#!/bin/bash
# Author: Your Name
# Description: This script prints “Hello World” to the screen.
echo “Hello World”
```

![](img/64201f98c64e13fc5691c212dd65f17e_37.png)

![](img/64201f98c64e13fc5691c212dd65f17e_39.png)

保存并退出编辑器（在Vim中按 `Esc` 后输入 `:wq`）。

## 如何执行Shell脚本？

![](img/64201f98c64e13fc5691c212dd65f17e_49.png)

![](img/64201f98c64e13fc5691c212dd65f17e_51.png)

![](img/64201f98c64e13fc5691c212dd65f17e_52.png)

创建好的脚本文件默认没有执行权限。我们需要先为其添加执行权限。

为文件所有者添加执行权限：
```bash
chmod u+x hello.sh
```
或者为所有用户添加执行权限：
```bash
chmod +x hello.sh
```
添加权限后，文件名在 `ls` 列表中通常会显示为绿色。

![](img/64201f98c64e13fc5691c212dd65f17e_54.png)

![](img/64201f98c64e13fc5691c212dd65f17e_56.png)

![](img/64201f98c64e13fc5691c212dd65f17e_57.png)

![](img/64201f98c64e13fc5691c212dd65f17e_59.png)

![](img/64201f98c64e13fc5691c212dd65f17e_61.png)

执行脚本有以下几种方式：
1.  **使用绝对路径或相对路径**（推荐）：
    ```bash
    /script/hello.sh  # 绝对路径
    ./hello.sh        # 相对路径（当前目录下）
    ```
    *   `./` 表示当前目录。直接输入 `hello.sh` 会被系统当作命令去查找，而非执行当前目录下的文件。

![](img/64201f98c64e13fc5691c212dd65f17e_63.png)

![](img/64201f98c64e13fc5691c212dd65f17e_65.png)

2.  **通过指定解释器直接执行**（无需给脚本执行权限）：
    ```bash
    bash hello.sh
    /bin/bash hello.sh
    ```

执行后，终端将输出：
```
Hello World
```

## 一个复杂的脚本示例

![](img/64201f98c64e13fc5691c212dd65f17e_73.png)

为了让大家更直观地感受脚本的威力，这里有一个系统初始化脚本的片段。它通过大量的 `echo` 命令输出提示信息，让执行过程一目了然，同时自动完成一系列繁琐的配置工作。
```bash
#!/bin/bash
echo “========================================”
echo “     系统初始化环境脚本”
echo “========================================”
echo “[1/10] 关闭防火墙…”
systemctl stop firewalld
systemctl disable firewalld
echo “[2/10] 配置Yum仓库…”
# ... 此处省略具体的配置命令
echo “[3/10] 设置时区并同步时间…”
# ... 此处省略具体的配置命令
# ... 后续更多步骤
echo “所有初始化操作已完成！”
```
这样的脚本可以一键对成百上千台服务器进行统一配置，将运维人员从重复劳动中解放出来。
