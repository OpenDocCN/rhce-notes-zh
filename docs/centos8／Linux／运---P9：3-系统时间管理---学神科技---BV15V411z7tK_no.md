在本节课中，我们将要学习Linux系统中的时间管理。我们将了解硬件时钟与系统时钟的区别，学习如何查看和设置系统时间，并掌握使用`date`命令格式化输出时间，以及使用`time`命令测量程序运行时间。

## 硬件时钟与系统时钟

上一节我们介绍了系统的基本操作，本节中我们来看看系统时间的管理。在Linux系统中，存在两种时钟：硬件时钟和系统时钟。

![](img/9ef8834e3568656a361030f8ad28e591_3.png)

*   **硬件时钟**：指的是主板BIOS中的时间，即使计算机关机，它也会依靠主板电池继续运行。
*   **系统时钟**：指的是Linux内核启动后维护的时间。系统启动时，会首先读取硬件时钟，然后以此为基础运行。

![](img/9ef8834e3568656a361030f8ad28e591_5.png)

我们可以使用`hwclock`命令查看硬件时钟。

![](img/9ef8834e3568656a361030f8ad28e591_7.png)

```bash
hwclock
```

使用`date`命令可以查看当前的系统时间。

```bash
date
```

![](img/9ef8834e3568656a361030f8ad28e591_9.png)

正常情况下，硬件时钟和系统时间应该是一致的。如果不一致，通常是由于时区设置或手动调整造成的。

![](img/9ef8834e3568656a361030f8ad28e591_10.png)

## 设置系统时间

了解了如何查看时间后，我们来看看如何修改系统时间。修改系统时间主要使用`date`命令，并通过`-s`参数指定一个时间字符串。

![](img/9ef8834e3568656a361030f8ad28e591_12.png)

例如，将系统时间设置为2020年8月6日10点07分：

```bash
date -s “2020-08-06 10:07:00”
```

![](img/9ef8834e3568656a361030f8ad28e591_14.png)

执行后，系统时间即被修改。

## 格式化输出时间

直接使用`date`命令会输出完整的日期和时间信息。但在很多场景下，我们只需要特定格式的时间字符串，例如用于备份文件的命名。

以下是使用`date`命令自定义输出格式的方法：

![](img/9ef8834e3568656a361030f8ad28e591_16.png)

`date`命令后使用双引号，内部以`+`开头，后面跟上格式控制符。格式控制符以`%`开头。

例如，只显示年月日：

```bash
date +“%Y-%m-%d”
```

![](img/9ef8834e3568656a361030f8ad28e591_18.png)

常见的日期格式表达式如下：

*   `%Y`：四位数的年份（如2020）
*   `%y`：两位数的年份（如20）
*   `%m`：两位数的月份（01-12）
*   `%d`：两位数的日期（01-31）
*   `%H`：24小时制的小时（00-23）
*   `%M`：分钟（00-59）
*   `%S`：秒（00-60）

你可以自由组合这些符号，并使用`-`、`/`或空格作为分隔符。

```bash
date +“%Y/%m/%d %H:%M:%S”
```

![](img/9ef8834e3568656a361030f8ad28e591_20.png)

此外，`date`命令还可以计算相对时间。例如，显示一个月后的日期：

![](img/9ef8834e3568656a361030f8ad28e591_22.png)

```bash
date -d “+1 month” +“%F”
```

## 测量命令运行时间

![](img/9ef8834e3568656a361030f8ad28e591_24.png)

除了管理时间，Linux还提供了测量程序运行时间的工具。`time`命令用于统计一个命令执行所消耗的时间，这在后期进行系统或程序性能调优时非常有用。

`time`命令的用法很简单，直接在需要测量的命令前加上`time`即可。

例如，测量`ls -l /`命令的执行时间：

![](img/9ef8834e3568656a361030f8ad28e591_26.png)

```bash
time ls -l /
```

命令执行后，会输出类似以下三行时间信息：

![](img/9ef8834e3568656a361030f8ad28e591_28.png)

*   **real**：实际流逝的时间，即从命令开始到结束的“墙钟时间”。
*   **user**：程序在用户态（非内核）消耗的CPU时间。
*   **sys**：程序在内核态消耗的CPU时间。

一个简单的比喻：中午去吃饭，**real**时间是从离开座位到回到座位的总时间（包括走路、排队、吃饭、返回）。**user**时间可能相当于你亲自吃饭所花的时间，而**sys**时间则可能相当于食堂工作人员为你服务（打饭）所花的时间。`real`时间通常大于`user`和`sys`时间之和，因为其中包含了等待（如I/O操作）的时间。

![](img/9ef8834e3568656a361030f8ad28e591_30.png)

![](img/9ef8834e3568656a361030f8ad28e591_31.png)

## 时间同步与历史问题

![](img/9ef8834e3568656a361030f8ad28e591_33.png)

在现代Linux系统中（如CentOS 8），通常会将硬件时钟设置为UTC（协调世界时），并根据系统设置的时区（如中国的CST，东八区）来显示系统时间。这样，硬件时钟与系统时间在数值上就是一致的。

但在早期的系统（如CentOS 6）中，默认将硬件时钟视为本地时间。对于东八区的用户，系统时间会在硬件时钟的基础上加8小时，这会导致在安装Windows和Linux双系统时，两个系统的时间显示不一致。如今，大多数主流Linux发行版都已改进，默认使用UTC时间，避免了这个问题。

## 总结

![](img/9ef8834e3568656a361030f8ad28e591_35.png)

本节课中我们一起学习了Linux系统的时间管理。我们首先区分了**硬件时钟**和**系统时钟**，并学会了使用`hwclock`和`date`命令分别查看它们。接着，我们掌握了使用`date -s`命令设置系统时间，以及使用`date +“格式”`来灵活地格式化输出时间字符串，这在脚本编程和日常管理中非常实用。最后，我们介绍了`time`命令，它可以用来测量一个命令的实际运行时间及CPU占用情况，是性能分析的入门工具。理解并掌握这些时间管理操作，是进行高效系统运维的基础。
