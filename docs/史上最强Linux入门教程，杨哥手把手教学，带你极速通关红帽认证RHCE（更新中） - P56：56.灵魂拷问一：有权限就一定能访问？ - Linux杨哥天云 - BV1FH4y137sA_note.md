Linux权限管理：P56：灵魂拷问一：有权限就一定能访问？ 🔍

![](img/7b6dd2e6e430ab5836e9719feced68ac_1.png)

在本节课中，我们将探讨Linux权限系统中的一个核心问题：拥有对某个文件的访问权限，是否就意味着一定能成功访问它？我们将通过一个具体的场景来揭示权限访问的完整路径依赖。

![](img/7b6dd2e6e430ab5836e9719feced68ac_3.png)

---

### 场景构建与问题提出

![](img/7b6dd2e6e430ab5836e9719feced68ac_5.png)

首先，我们创建一个场景。将 `/etc/hosts` 文件复制到当前目录。

![](img/7b6dd2e6e430ab5836e9719feced68ac_7.png)

```bash
cp /etc/hosts ./
```

![](img/7b6dd2e6e430ab5836e9719feced68ac_9.png)

现在，当前目录下存在一个名为 `hosts` 的文件。

![](img/7b6dd2e6e430ab5836e9719feced68ac_11.png)

接下来，我们切换到普通用户 `tianyun`，并检查该用户对当前目录下 `hosts` 文件的读权限。

```bash
ls -l hosts
```

![](img/7b6dd2e6e430ab5836e9719feced68ac_13.png)

输出显示，`tianyun` 用户作为“其他人”，拥有该文件的读权限（`r--`）。

![](img/7b6dd2e6e430ab5836e9719feced68ac_15.png)

然而，当我们尝试使用绝对路径读取位于 `/root/hosts` 的原始文件时，却遇到了权限拒绝的错误。

```bash
cat /root/hosts
```
**输出：** `Permission denied`

![](img/7b6dd2e6e430ab5836e9719feced68ac_17.png)

这就引出了核心问题：为什么对文件本身有读权限，访问却失败了？

![](img/7b6dd2e6e430ab5836e9719feced68ac_19.png)

---

### 权限访问的路径原理

![](img/7b6dd2e6e430ab5836e9719feced68ac_21.png)

访问一个文件，就像进入一个深宅大院取东西，需要经过多个关卡：
1.  进入院子（对应根目录 `/`）。
2.  打开房门（对应 `/root` 目录）。
3.  打开保险箱（对应 `/root/hosts` 文件）。

![](img/7b6dd2e6e430ab5836e9719feced68ac_23.png)

即使你拥有保险箱的钥匙（文件权限），但如果院门或房门被锁住（目录权限不足），你依然无法到达保险箱的位置。

因此，要成功访问一个文件，用户必须对**该文件路径上的每一级目录**都拥有相应的访问权限。

![](img/7b6dd2e6e430ab5836e9719feced68ac_25.png)

![](img/7b6dd2e6e430ab5836e9719feced68ac_27.png)

### 逐级目录权限分析

上一节我们介绍了权限访问的路径依赖原理，本节中我们来看看如何具体分析每一级目录的权限。

![](img/7b6dd2e6e430ab5836e9719feced68ac_29.png)

以下是使用 `ls -ld` 命令检查路径上各级目录权限的步骤：

![](img/7b6dd2e6e430ab5836e9719feced68ac_31.png)

1.  **检查根目录 `/` 的权限：**
    ```bash
    ls -ld /
    ```
    对于 `tianyun` 用户（其他人身份），根目录的权限通常包含 `r-x`（读和执行）。这表示用户可以查看根目录下的内容列表（读），并可以进入根目录（执行）。因此，根目录这一关通常不是障碍。

![](img/7b6dd2e6e430ab5836e9719feced68ac_33.png)

2.  **检查 `/root` 目录的权限：**
    ```bash
    ls -ld /root
    ```
    输出显示，`/root` 目录的所有者和所属组都是 `root`，而“其他人”的权限是 `---`（无任何权限）。
    *   `tianyun` 用户不是所有者。
    *   `tianyun` 用户也不是所属组 `root` 的成员。
    *   因此，`tianyun` 用户只能匹配“其他人”的权限，即**没有任何权限**。

**关键点：** 对于一个目录，“执行”权限（`x`）意味着可以“进入”该目录。没有执行权限，用户将无法访问该目录下的任何内容，无论其对下层文件或目录拥有何种权限。

![](img/7b6dd2e6e430ab5836e9719feced68ac_35.png)

由于 `tianyun` 用户对 `/root` 目录没有执行权限，导致其在第一步就被阻挡，根本无法进入 `/root` 目录，因此更谈不上访问其下的 `hosts` 文件了。

![](img/7b6dd2e6e430ab5836e9719feced68ac_37.png)

### 默认目录权限的启示

![](img/7b6dd2e6e430ab5836e9719feced68ac_41.png)

这个例子也解释了Linux系统中用户家目录的默认安全设置。

![](img/7b6dd2e6e430ab5836e9719feced68ac_43.png)

在 `/home` 目录下，每个普通用户的家目录（如 `/home/tianyun`）默认权限通常是 `drwx------`。这意味着：
*   所有者（用户自己）拥有读、写、执行全部权限。
*   所属组和其他人没有任何权限。

这种设置确保了用户文件的私密性，其他用户无法进入或浏览其家目录。

![](img/7b6dd2e6e430ab5836e9719feced68ac_45.png)

### 核心结论总结

本节课中我们一起学习了Linux权限访问的关键规则：

![](img/7b6dd2e6e430ab5836e9719feced68ac_49.png)

**拥有对某个文件的权限，不代表一定能访问它。访问的成功与否，取决于对文件路径上每一级目录的权限。**

![](img/7b6dd2e6e430ab5836e9719feced68ac_51.png)

核心要求是：用户必须对目标文件所在路径的**每一级父目录**至少拥有**执行权限**。这可以形象地理解为获得进入每一道“门”的许可。

用公式概括访问成功的条件就是：
**访问成功 = 对路径中所有父目录拥有 `x` 权限 + 对目标文件拥有所需权限**

![](img/7b6dd2e6e430ab5836e9719feced68ac_53.png)

因此，在管理和排查文件访问权限问题时，务必检查从根目录开始的完整路径，而不仅仅是文件本身的权限设置。
