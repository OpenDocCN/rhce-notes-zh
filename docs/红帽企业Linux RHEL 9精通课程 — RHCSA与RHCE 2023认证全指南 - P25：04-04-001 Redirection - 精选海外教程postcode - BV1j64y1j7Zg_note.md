# Linux 精通课程：P25：04-04-001 重定向 📥📤

在本节课中，我们将要学习 Linux 中一个非常核心且强大的概念——重定向。你将了解命令如何获取输入、产生输出，以及如何通过重定向操作符来控制这些数据流的去向，从而将简单的命令组合成强大的数据处理管道。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_1.png)

## 命令的输入与输出概述

正如您之前了解到的，每个命令基本上都是它自己的计算机程序。计算机程序是一种获取输入、处理输入并给出输出的自动化方式。

一般的 Linux 命令可以从两个地方获取输入，并将输出到另外两个地方。总共有四种可能的方式将数据输入和输出传统的 Linux 命令。

*   **输入来源**：命令参数和标准输入。
*   **输出目标**：标准输出和标准错误。

其中三个由“标准”一词开头，称为**标准数据流**：标准输入、标准输出和标准错误。而命令行参数则不是标准数据流。

标准数据流的特殊之处在于它们可以被**重定向**和**连接**在一起。本节中，我们将详细探讨这些数据流以及如何操作它们。

## 标准输出 (stdout) 💬

当命令成功运行时，从命令输出数据的主要通道称为**标准输出**。默认情况下，标准输出流指向您的终端屏幕。这就是为什么当我们键入命令时，命令的输出会出现在屏幕上。

例如，运行 `date` 命令：
```bash
date
```
输出会显示在终端上。关于输出数据流的一个重要特性是，您可以使用称为**重定向**的过程来改变它们的去向。

## 标准错误 (stderr) ⚠️

当命令产生错误消息、日志信息或任何非主要输出的信息时，这些内容会被发送到**标准错误**流。这有助于将有用的输出和错误/日志信息分开。

例如，给 `date` 命令一个无效参数：
```bash
date invalid_arg
```
您会看到一条错误消息。因为它是一条错误消息，所以它被发送到标准错误流。默认情况下，标准错误也连接到终端屏幕。同样，标准错误流也可以被重定向。

## 标准输入 (stdin) ⌨️

命令获取输入的另一种标准方式是**标准输入**。默认情况下，标准输入连接到键盘。

我们可以用 `cat` 命令来演示。`cat` 命令通常用于连接文件，但如果没有给定文件参数，它会从标准输入读取。

运行 `cat` 命令而不带任何参数：
```bash
cat
```
此时终端似乎在等待。如果您开始输入（例如输入“hello”并按回车），`cat` 会从键盘（标准输入）读取文本，进行处理，然后将结果输出到标准输出（终端屏幕）。按 `Ctrl+C` 可以停止 `cat` 命令。

因为标准输入是数据流，所以它也可以被重定向。这意味着我们可以告诉命令从文件或其他命令的输出（另一个数据流）中读取数据。

将数据流连接在一起的能力非常强大。您可以将一个命令的标准输出传递给另一个命令的标准输入，依此类推，建立起强大的处理管道。这种方式被称为**管道**，是 Linux 命令行强大和高效的核心概念之一。

## 命令行参数

![](img/01a3f19c5a4579a25c4b9bd6d487f212_5.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_7.png)

我们已经多次见过命令行参数。例如，在命令 `cal 12 2017` 中，`12` 和 `2017` 就是传递给 `cal` 命令的参数。选项也可以有自己的参数，例如 `cal -A 1 -B 1 12 2017`。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_9.png)

命令行参数和数据流的关键区别在于：数据流可以流动、重定向并通过管道连接；而命令行参数是静态的，仅与当前正在处理的命令或选项相关联。

另一个区别是，并非所有命令都接受标准输入（例如 `echo` 命令），但几乎每个命令都可以接受命令行参数。

## 重定向标准输出

![](img/01a3f19c5a4579a25c4b9bd6d487f212_11.png)

如果我们想改变标准输出的目的地，而不是让它显示在终端上，该怎么做呢？这就是**重定向**。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_13.png)

假设我们想将 `cat` 命令的标准输出重定向到一个名为 `output.txt` 的文件。每个数据流都有一个关联的编号：
*   标准输入 (stdin): **0**
*   标准输出 (stdout): **1**
*   标准错误 (stderr): **2**

![](img/01a3f19c5a4579a25c4b9bd6d487f212_15.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_16.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_18.png)

我们可以使用 `>` 符号（大于号）进行重定向。其完整形式是指定流编号 `1`：
```bash
cat 1> output.txt
```
然后输入一些内容（如“linux is amazing”），按 `Ctrl+C` 停止。此时终端没有输出，但当前目录下会生成一个 `output.txt` 文件，其中包含了我们输入的文本。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_20.png)

由于重定向标准输出非常常见，Linux 允许我们省略流编号 `1`，简写为：
```bash
cat > output.txt
```
效果完全相同。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_22.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_24.png)

**重要提示**：默认情况下，单个 `>` 符号会在写入文件前**清空（截断）** 该文件的所有原有内容。

如果我们希望将输出**追加**到文件末尾，而不是清空文件，需要使用两个连续的 `>` 符号：
```bash
cat >> output.txt
```
此时再输入内容，新内容会被添加到文件原有内容之后。

## 重定向标准错误

重定向标准错误的方式与标准输出类似，但需要使用其流编号 `2`。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_26.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_28.png)

例如，运行一个会产生错误的命令，并将其标准错误重定向到文件：
```bash
cat -k blah 2> error.txt
```
终端不会显示错误信息，错误内容被写入 `error.txt` 文件。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_30.png)

同样，使用 `2>>` 可以将错误信息追加到文件：
```bash
cat -k blah 2>> error.txt
```
这对于记录日志非常有用，可以保存历史错误信息。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_32.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_34.png)

## 同时重定向标准输出和标准错误

![](img/01a3f19c5a4579a25c4b9bd6d487f212_36.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_38.png)

您可以同时重定向两个输出流到不同的文件：
```bash
cat linux_is_amazing 1>> output.txt 2>> error.txt
```
因为 `cat` 命令成功运行，所以 `output.txt` 会收到内容，而 `error.txt` 为空。

对于标准输出，省略 `1` 的简写形式仍然有效：
```bash
cat linux_is_amazing >> output.txt 2>> error.txt
```

![](img/01a3f19c5a4579a25c4b9bd6d487f212_40.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_42.png)

## 重定向标准输入

![](img/01a3f19c5a4579a25c4b9bd6d487f212_44.png)

我们也可以改变标准输入的来源，使其不从键盘读取，而从文件读取。这使用 `<` 符号（小于号），标准输入的流编号是 `0`。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_46.png)

首先，创建一个输入文件：
```bash
cat > input.txt
```
输入“Hello World”，按 `Ctrl+C` 停止。现在，让 `cat` 命令从 `input.txt` 文件读取标准输入：
```bash
cat 0< input.txt
```
或者使用简写形式（因为标准输入是唯一通常用 `<` 重定向的流）：
```bash
cat < input.txt
```
`cat` 会读取文件内容并输出到终端（标准输出）。

**综合练习**：如何让 `cat` 从 `input.txt` 读取输入，并将其输出重定向到 `hello.txt` 文件？

![](img/01a3f19c5a4579a25c4b9bd6d487f212_48.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_50.png)

**答案**：
```bash
cat < input.txt > hello.txt
```
或使用完整流编号：
```bash
cat 0< input.txt 1> hello.txt
```
执行后，`hello.txt` 文件将包含 “Hello World”。

## 一个有趣的演示：终端也是文件

Linux 中“一切皆文件”，甚至终端设备也被视为文件。您可以通过重定向在不同终端间传递数据。

1.  打开两个终端窗口。
2.  在其中一个终端运行 `tty` 命令，它会返回该终端对应的设备文件路径（例如 `/dev/pts/1`）。
3.  在另一个终端中，将内容重定向到那个设备文件：
    ```bash
    echo “Message from another terminal” > /dev/pts/1
    ```
4.  您会看到消息出现在第一个终端中。这演示了重定向如何在整个系统甚至网络上传递数据。

## 总结 🎯

![](img/01a3f19c5a4579a25c4b9bd6d487f212_54.png)

![](img/01a3f19c5a4579a25c4b9bd6d487f212_56.png)

本节课中我们一起学习了 Linux 重定向的核心知识：

1.  **三种标准数据流**：标准输入 (stdin/0)、标准输出 (stdout/1)、标准错误 (stderr/2)。
2.  **重定向操作符**：
    *   `>` 或 `1>`：将标准输出重定向到文件（覆盖）。
    *   `>>` 或 `1>>`：将标准输出追加到文件。
    *   `2>`：将标准错误重定向到文件（覆盖）。
    *   `2>>`：将标准错误追加到文件。
    *   `<` 或 `0<`：将文件内容重定向为命令的标准输入。
3.  **核心概念**：重定向改变了数据流的“流向”，而管道（`|`）则是将一个命令的标准输出连接到另一个命令的标准输入，这是构建复杂命令行操作的基础。

![](img/01a3f19c5a4579a25c4b9bd6d487f212_58.png)

掌握重定向是有效使用 Linux 命令行的关键一步，它让您能够灵活地控制数据的来源和去向，实现自动化处理和日志记录。
