## 程序、进程与线程的概念

![](img/1409254f55efd7e52cb0f34e3cf1e77f_18.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_20.png)

上一节我们概述了本节课的内容，本节中我们来详细看看程序、进程和线程这三个核心概念。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_22.png)

### 什么是程序？
程序是用计算机语言编写的命令序列的集合，用于实现特定目标或解决特定问题。通俗地说，程序就是我们从软件商店下载并安装到电脑里的软件，例如聊天工具、浏览器等。在未运行时，程序以静态文件的形式存储在硬盘上，占用磁盘空间。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_24.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_26.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_28.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_30.png)

**核心概念**：程序是静态的、永久的，存储在磁盘上。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_32.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_34.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_36.png)

### 什么是进程？
进程是正在运行中的程序。当程序被加载到内存并开始执行时，它就变成了一个进程。进程占用系统的内存空间和CPU时间，是动态的、有生命周期的实体。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_38.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_40.png)

以下是进程的几个关键特性：
*   **生命周期**：进程从启动到结束的过程。有的进程（如一条简单的命令）生命周期极短，执行完毕即结束；有的进程（如后台服务）则长期运行，直到被手动终止或系统关机。
*   **独立内存空间**：每个进程都拥有系统为其分配的独立内存区域。
*   **进程ID (PID)**：系统为每个进程分配的唯一标识符，用于识别和管理进程，类似于人的身份证号。
*   **父子关系**：在Linux中，进程之间存在父子关系。一个进程（父进程）可以创建多个其他进程（子进程）。父进程终止时，其子进程通常也会被终止。

**核心概念**：`进程 = 运行中的程序`，它是动态的，拥有独立的PID和内存空间。

### 什么是线程？
线程，常被称为轻量级进程，是进程中的一个执行单元，是进程中实际运作的单位。一个进程可以包含多个线程，这些线程共享进程的内存空间和资源，但各自拥有独立的执行序列。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_42.png)

我们可以用一个比喻来理解：
*   **CPU工厂**：将整个CPU比喻为一个工厂。
*   **核心 (Cores)**：工厂里的生产车间。例如，四核CPU就有四个车间。
*   **线程 (Threads)**：每个车间里真正干活的工人。例如，一个核心有两个线程，就意味着每个车间有两个工人。

**核心概念**：线程是进程内部的执行流，是CPU调度的基本单位。对于运维工作，理解“线程是实际干活的实体”即可。

## 查看进程树：`pstree` 命令

了解了基本概念后，我们来看看如何查看系统中进程的组织结构。`pstree`命令能以树状图形式显示进程及其父子关系。

首先，确保系统已安装`pstree`工具：
```bash
yum install -y psmisc
```

![](img/1409254f55efd7e52cb0f34e3cf1e77f_44.png)

### `pstree` 基本使用
直接运行`pstree`命令，可以看到以树状结构展示的系统进程：
```bash
pstree
```
输出中，树根通常是 **`systemd`**（进程ID为1），它是所有用户进程的祖先进程（init进程）。树的分支显示了进程间的派生关系。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_46.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_48.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_50.png)

### 常用选项
以下是`pstree`命令的一些有用选项：

**显示进程PID**
使用`-p`选项可以在进程名后显示其PID。
```bash
pstree -p
```

![](img/1409254f55efd7e52cb0f34e3cf1e77f_52.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_54.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_56.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_58.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_60.png)

**显示指定用户的进程**
如果你想查看特定用户（例如`test`用户）运行的进程树，可以直接在命令后跟上用户名。
```bash
pstree test
```

**显示完整命令行**
`-a`选项可以显示进程的完整启动命令及参数，这对于了解进程的具体作用很有帮助。
```bash
pstree -a
```

![](img/1409254f55efd7e52cb0f34e3cf1e77f_62.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_64.png)

**结合`-u`选项查看进程所属用户**
`-u`选项可以列出进程所属的用户名。你可以单独使用，也可以与指定用户结合使用。
```bash
# 显示所有进程及其所属用户
pstree -u
# 显示test用户的进程及其完整命令行
pstree test -a
```
在树状图中，进程后的`*`号及数字（如`6*`）表示该进程下包含的线程数量。

## 查看进程快照：`ps` 命令

![](img/1409254f55efd7e52cb0f34e3cf1e77f_76.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_78.png)

`pstree`展示了进程关系，而`ps`命令则用于获取当前时刻进程状态的快照。它功能强大，选项组合多样。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_80.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_82.png)

### `ps aux` 命令详解
`ps aux`是一组非常常用的选项组合，用于查看系统所有进程的详细信息。
```bash
ps aux
```
这条命令的输出包含多列信息，每一列的含义如下：

*   **USER**: 进程所有者的用户名。
*   **PID**: 进程ID。
*   **%CPU**: 进程占用CPU的百分比。
*   **%MEM**: 进程占用物理内存的百分比。
*   **VSZ**: 进程使用的虚拟内存大小（单位：KB）。
*   **RSS**: 进程占用的固定内存大小（单位：KB）。
*   **TTY**: 进程在哪个终端运行。
    *   `?` 表示由系统内核直接启动，无控制终端。
    *   `tty1` 等表示本地终端登录。
    *   `pts/0`、`pts/1` 等表示远程终端登录（如通过SSH连接）。
*   **STAT**: 进程状态。这是需要重点关注的列。
    *   **R (Running)**: 正在运行或可运行（在运行队列中）。
    *   **S (Sleeping)**: 休眠中，可被唤醒。
    *   **D (Disk Sleep)**: 不可中断的休眠（通常与I/O操作相关）。
    *   **T (Stopped)**: 进程被停止（如通过`Ctrl+Z`）。
    *   **Z (Zombie)**: **僵尸进程**。进程已终止，但其父进程未能成功回收其资源。需要关注并处理。
    *   **<** : 高优先级进程。
    *   **N** : 低优先级进程。
    *   **s** : 该进程是会话首进程（即父进程）。
    *   **+** : 位于前台进程组。
*   **START**: 进程启动时间。
*   **TIME**: 进程占用CPU的总时间。
*   **COMMAND**: 启动进程的命令行。

### `ps -ef` 命令
这是另一种常见的风格（选项前带`-`），同样用于查看所有进程。
```bash
ps -ef
# 或使用长格式，显示更多信息
ps -elf
```
`ps -elf`输出中一个重要的列是 **PPID**，它表示父进程的ID。当需要追踪进程关系或处理僵尸进程时，PPID非常有用。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_84.png)

### 进程信息过滤
`ps`命令常与`grep`结合，用于查找特定进程。
```bash
# 查找与 sshd 相关的所有进程
ps aux | grep sshd
```
这是运维工作中**最常用**的命令之一，用于快速定位进程。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_86.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_88.png)

### 查找僵尸进程
僵尸进程（状态为`Z`）需要特别关注。可以使用以下命令组合进行查找：
```bash
ps -ef | grep defunct
# 或者更精确地过滤STAT状态为Z的进程
ps aux | awk '$8=="Z" {print $0}'
```
如果发现僵尸进程，通常需要终止其父进程（通过PPID找到），系统初始化进程（`systemd`）会负责回收其资源。

## 总结
本节课我们一起学习了操作系统中的核心概念和进程管理工具。

![](img/1409254f55efd7e52cb0f34e3cf1e77f_96.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_98.png)

![](img/1409254f55efd7e52cb0f34e3cf1e77f_100.png)

1.  **概念区分**：我们明确了**程序**是静态的磁盘文件，**进程**是运行中的程序实例，而**线程**是进程内部更小的执行单元。
2.  **进程查看**：我们掌握了`pstree`命令，它能直观展示进程间的**树状父子关系**。
3.  **进程快照**：我们深入学习了`ps`命令，特别是`ps aux`组合，它能列出系统所有进程的详细信息，包括PID、状态、资源占用等，并且我们学会了如何解读关键的**STAT状态位**（如运行`R`、休眠`S`、僵尸`Z`）。
4.  **实用技巧**：我们了解了如何使用`ps aux | grep [进程名]`来快速过滤查找进程，以及如何识别和初步处理**僵尸进程**。
