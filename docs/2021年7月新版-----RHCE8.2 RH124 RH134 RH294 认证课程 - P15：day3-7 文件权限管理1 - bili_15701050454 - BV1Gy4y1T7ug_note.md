在本节课中，我们将要学习Linux系统中文件权限管理的基础知识。理解文件权限是系统安全和管理的重要基石，它决定了谁可以读取、修改或执行系统中的文件。

## 文件权限概述

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_7.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_9.png)

上一节我们介绍了文件链接，本节中我们来看看文件权限。当我们使用 `ls -l` 命令查看文件时，会看到类似 `-rwxr-xr--` 的输出，这代表了文件的权限信息。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_11.png)

一个文件的完整权限信息通常包含11个字符。第一个字符代表**文件类型**，例如 `-` 表示普通文件，`d` 表示目录。接下来的9个字符，每3个为一组，分别代表**所有者**、**所属组**和**其他人**对该文件的权限。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_13.png)

## 权限字符的含义

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_15.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_17.png)

以下是权限字符 `r`、`w`、x` 的具体含义：
*   **r**： 读权限。对于文件，表示可以查看文件内容；对于目录，表示可以列出目录内的文件列表。
*   **w**： 写权限。对于文件，表示可以修改文件内容；对于目录，表示可以在目录内创建、删除或重命名文件。
*   **x**： 执行权限。对于文件，表示可以将其作为程序或脚本执行；对于目录，表示可以进入该目录。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_19.png)

## 修改文件所有者和所属组

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_21.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_23.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_25.png)

要修改文件的所有者和所属组，我们使用 `chown` 命令。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_27.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_29.png)

以下是 `chown` 命令的基本用法：
```bash
# 将文件 file1 的所有者改为 student
chown student file1

# 将文件 file1 的所有者改为 student，所属组也改为 student
chown student:student file1
# 或者使用点号
chown student.student file1

# 只修改文件的所属组为 root
chown :root file1
# 或者使用专门的命令 chgrp
chgrp root file1
```

`chown` 命令还支持一些有用的选项：
*   `--reference=参照文件`： 将目标文件的权限设置为与参照文件相同。
    ```bash
    chown --reference=file2 file1
    ```
*   `-R`： 递归地修改目录及其内部所有文件和子目录的权限。**此操作需谨慎使用**，因为它会改变目录树下所有项目的所有者。
    ```bash
    chown -R student:student /data/testdir
    ```

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_33.png)

## 修改文件权限

修改文件权限的核心命令是 `chmod`。主要有三种方法：符号法、数字法和加减法。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_35.png)

### 1. 符号法

符号法使用 `u`（所有者）、`g`（所属组）、`o`（其他人）、`a`（所有人）与 `+`（添加）、`-`（移除）、`=`（精确设置）进行操作。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_37.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_39.png)

以下是符号法示例：
```bash
# 为所属组添加读写权限
chmod g+rw file1

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_41.png)

# 为其他人设置只写权限（不推荐，因为无法读取）
chmod o=w file1

# 移除所有人的执行权限
chmod a-x file1
```

**重要提示**：对于文件，仅有写权限(`w`)通常没有意义，因为无法读取内容进行编辑。有意义的最小权限组合是读(`r`)权限。对于脚本文件，需要同时具备读(`r`)和执行(`x`)权限才能运行。

### 2. 数字法

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_43.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_45.png)

数字法使用八进制数字表示权限，每个数字对应 `rwx` 一组权限。其计算规则是：
*   `r` (读) = 4
*   `w` (写) = 2
*   `x` (执行) = 1

权限数字是这些值的和。例如：
*   `rwx` = 4+2+1 = 7
*   `rw-` = 4+2+0 = 6
*   `r-x` = 4+0+1 = 5

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_47.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_49.png)

以下是数字法示例：
```bash
# 设置文件权限为 rwxr-xr-- (所有者可读可写可执行，组可读可执行，其他人只读)
chmod 754 file1

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_51.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_53.png)

# 设置文件权限为 rw-r--r-- (所有者可读可写，组和其他人只读)
chmod 644 file1
```

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_55.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_57.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_59.png)

### 3. 目录权限的特殊性

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_61.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_62.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_64.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_66.png)

目录的权限与文件略有不同：
*   目录的**读**权限(`r`)：允许列出目录内容（如使用 `ls`）。
*   目录的**写**权限(`w`)：允许在目录内创建、删除或重命名文件。
*   目录的**执行**权限(`x`)：允许进入该目录（如使用 `cd`）。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_68.png)

**关键点**：用户能否删除或重命名目录内的某个文件，**取决于用户对该目录是否具有写权限**，而与文件本身的权限无关。目录有意义的最小权限是执行(`x`)权限，这样才能进入目录。

## 默认权限与umask

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_70.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_72.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_74.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_75.png)

新创建的文件和目录并非拥有满权限(777)，其默认权限由系统的 `umask`（权限掩码）值决定。

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_77.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_79.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_81.png)

查看当前 `umask` 值：
```bash
umask
```

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_83.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_85.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_87.png)

`umask` 定义了需要从默认权限中“屏蔽”掉的权限位。计算方式如下：
*   **目录的默认权限** = `777` - `umask`
*   **文件的默认权限** = `666` - `umask` （注意：系统默认不会给新文件赋予执行权限，因此如果计算结果中出现奇数权限，会自动加1）

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_89.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_91.png)

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_93.png)

例如，如果 `umask` 值为 `022`：
*   新目录权限 = 777 - 022 = 755 (`rwxr-xr-x`)
*   新文件权限 = 666 - 022 = 644 (`rw-r--r--`)。计算过程：6-0=6， 6-2=4， 6-2=4。

修改 `umask` 值（仅对当前会话有效）：
```bash
umask 027
```

![](img/33cd0e52e22f6b3684bc1d2df8be8bb4_95.png)

要使 `umask` 设置永久生效，可以将其写入用户的shell配置文件中（如 `~/.bashrc`）：
```bash
echo “umask 027” >> ~/.bashrc
# 然后重新加载配置文件或重新登录
source ~/.bashrc
```

## 总结

本节课中我们一起学习了Linux文件权限管理的基础知识。我们了解了：
1.  权限字符 `r`、`w`、`x` 的含义及其对文件和目录的不同影响。
2.  如何使用 `chown` 命令修改文件的所有者和所属组。
3.  使用 `chmod` 命令修改权限的三种方法：符号法、数字法和加减法。
4.  目录权限的特殊性，特别是删除文件权限取决于目录的写权限。
5.  系统默认权限是如何通过 `umask` 值来控制的，以及如何修改它。
