### 1. 管理临时文件

管理临时文件主要涉及自定义清理策略。其核心是通过创建 `/etc/tmpfiles.d/` 目录下的配置文件来实现。

**操作步骤**：
1.  在 `/etc/tmpfiles.d/` 目录下创建配置文件。
2.  使用 `systemd-tmpfiles --create` 命令应用配置。

### 2. 系统调优

系统调优工具 `tuned` 用于优化系统性能。我们需要掌握如何查看和设置调优方案。

**核心命令**：
*   查看当前激活的方案：`tuned-adm active`
*   查看推荐方案：`tuned-adm recommend`
*   切换方案：`tuned-adm profile <方案名称>`

**完整步骤示例**：
若当前方案不是 `virtual-guest`，而考试要求使用它，则执行：
```bash
tuned-adm profile virtual-guest
```
操作前请确保 `tuned` 服务已安装并运行。

### 3. 文件访问控制列表

文件ACL用于为特定用户或组设置精细的文件访问权限，而非传统的用户/组/其他大类权限。

**核心操作**：
*   **获取ACL**：`getfacl <文件名>`
*   **设置ACL**：`setfacl -m u:<用户名>:<权限> <文件名>`

### 4. SELinux 管理

SELinux 提供强制性的访问控制安全机制。

**SELinux 的三种模式**：
*   **enforcing**：强制模式，拒绝违规操作并记录。
*   **permissive**：宽容模式，仅记录违规操作。
*   **disabled**：禁用模式。

**模式切换方法**：
*   **临时切换**（仅在 `enforcing` 和 `permissive` 之间）：
    ```bash
    setenforce 1  # 切换到 enforcing
    setenforce 0  # 切换到 permissive
    ```
    使用 `getenforce` 命令查看当前模式。
*   **永久切换**：修改 `/etc/selinux/config` 文件中的 `SELINUX=` 值。
    > **重要提示**：涉及 `disabled` 模式的切换（无论启用或禁用）都必须重启系统才能生效。

![](img/96b54b060e31a8cedaa29dd168b0747b_11.png)

**安全性上下文管理**：
*   **更改默认上下文**：使用 `semanage fcontext` 命令。
    ```bash
    semanage fcontext -a -t <类型> “<目录或文件路径>”
    ```
    对于目录，路径可使用通配符（如 `/webcontent/*`）使其对所有子项生效。修改后需使用 `restorecon -Rv <路径>` 恢复上下文。
*   **临时修改上下文**：使用 `chcon` 命令。
    ```bash
    chcon -t <类型> <目标文件>
    或
    chcon --reference=<参考文件> <目标文件>
    ```

### 5. 磁盘与分区管理

我们学习了传统分区和LVM逻辑卷管理两种磁盘管理方式。

**传统基本分区操作步骤**：
1.  **分区**：使用 `fdisk`、`gdisk` 或 `parted` 工具。
2.  **格式化**：使用 `mkfs` 系列命令创建文件系统。
3.  **临时挂载**：使用 `mount` 命令。
4.  **永久挂载**：将挂载信息写入 `/etc/fstab` 文件。

**LVM 逻辑卷管理操作步骤**：
1.  **创建物理卷**：`pvcreate /dev/sdX`
2.  **创建卷组**：`vgcreate <卷组名> /dev/sdX`
3.  **创建逻辑卷**：`lvcreate -n <逻辑卷名> -L <大小> <卷组名>`
4.  **创建文件系统并挂载**。

![](img/96b54b060e31a8cedaa29dd168b0747b_13.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_15.png)

**LVM 动态调整**：
*   **在线扩容**：
    1.  确保卷组有足够空间：`vgs`
    2.  扩展逻辑卷：`lvextend -L +<增加大小> /dev/<卷组名>/<逻辑卷名>`
    3.  扩展文件系统：`xfs_growfs /dev/<卷组名>/<逻辑卷名>` (针对XFS) 或 `resize2fs` (针对EXT4)。
*   **离线缩容**（文件系统需支持，如EXT4）：
    1.  卸载文件系统。
    2.  检查文件系统：`e2fsck -f /dev/<卷组名>/<逻辑卷名>`
    3.  缩小文件系统：`resize2fs /dev/<卷组名>/<逻辑卷名> <新大小>`
    4.  缩小逻辑卷：`lvreduce -L <新大小> /dev/<卷组名>/<逻辑卷名>`

![](img/96b54b060e31a8cedaa29dd168b0747b_19.png)

### 练习1：LVM逻辑卷扩容

**题目要求**：在 `/dev/sda` 上创建本地LVM，逻辑卷大小为200MiB，使用XFS文件系统并挂载，然后将其扩容至500MiB。

**操作步骤**：
1.  **分区**：在 `/dev/sda` 上创建新分区（例如 `/dev/sda7`），类型设置为 `8e` (Linux LVM)。
2.  **创建物理卷**：
    ```bash
    pvcreate /dev/sda7
    ```
3.  **创建卷组**：
    ```bash
    vgcreate vg1 /dev/sda7
    ```
4.  **创建逻辑卷并格式化**：
    ```bash
    lvcreate -n lv1 -L 200M vg1
    mkfs.xfs /dev/vg1/lv1
    mkdir /mnt/lvm1
    mount /dev/vg1/lv1 /mnt/lvm1
    ```
5.  **扩容逻辑卷**：
    ```bash
    lvextend -L 500M /dev/vg1/lv1
    xfs_growfs /mnt/lvm1
    ```
6.  **验证**：使用 `df -h` 命令查看挂载点容量是否已变为500MiB左右。

### 练习2：创建Swap交换分区

![](img/96b54b060e31a8cedaa29dd168b0747b_32.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_34.png)

**题目要求**：创建并启用一个512MiB的Swap分区。

**操作步骤**：
1.  **分区**：在目标磁盘上创建新分区（例如 `/dev/sda8`），类型设置为 `82` (Linux swap)。
2.  **格式化并启用**：
    ```bash
    mkswap /dev/sda8
    swapon /dev/sda8
    ```
3.  **永久生效**：将以下行添加到 `/etc/fstab` 文件末尾。
    ```
    /dev/sda8 swap swap defaults 0 0
    ```
4.  执行 `mount -a` 测试配置是否正确。

### 练习3：Shell脚本 - 等差数列

**题目要求**：编写脚本，输出公差为4，从2开始到50结束的等差数列。

**脚本示例** (`seq_script.sh`)：
```bash
#!/bin/bash
for i in $(seq 2 4 50); do
    echo $i
done
```
**执行**：`bash seq_script.sh`

### 练习4：Shell脚本 - 防火墙规则管理

**题目要求**：编写脚本，可批量添加防火墙服务规则并永久生效。

![](img/96b54b060e31a8cedaa29dd168b0747b_36.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_38.png)

**脚本示例** (`firewall_script.sh`)：
```bash
#!/bin/bash
read -p "Please input the services: " services
for service in $services; do
    echo "Adding $service service to firewall..."
    firewall-cmd --permanent --add-service=$service
done
firewall-cmd --reload
```
**执行**：`bash firewall_script.sh`，然后输入服务名，如 `ssh http cockpit`。

### 练习5：Shell脚本 - 批量创建用户

![](img/96b54b060e31a8cedaa29dd168b0747b_42.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_44.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_45.png)

**题目要求**：根据提供的用户列表文件，批量创建无需密码的系统用户。

![](img/96b54b060e31a8cedaa29dd168b0747b_47.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_49.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_50.png)

**脚本示例** (`batch_add_users.sh`)：
```bash
#!/bin/bash
if [ $# -eq 0 ]; then
    echo "Usage: $0 <userlist_file>"
    exit 1
fi
if [ ! -f $1 ]; then
    echo "Error: File $1 not found."
    exit 2
fi
for username in $(cat $1); do
    useradd -s /bin/bash $username
    echo "User $username created."
done
```
**准备用户列表文件** (`user.list`)：
```
alice
bob
charlie
```
**执行**：`bash batch_add_users.sh user.list`

### 练习9：Shell脚本 - 系统识别

**题目要求**：编写脚本，根据输入参数输出对应的系统名称。

![](img/96b54b060e31a8cedaa29dd168b0747b_54.png)

![](img/96b54b060e31a8cedaa29dd168b0747b_56.png)

**脚本示例** (`os_identify.sh`)：
```bash
#!/bin/bash
case $1 in
    rhel)
        echo "Red Hat Enterprise Linux"
        ;;
    centos)
        echo "Community Enterprise Operating System"
        ;;
    *)
        echo "Usage: $0 {rhel|centos}"
        ;;
esac
```
**执行**：
```bash
bash os_identify.sh rhel   # 输出：Red Hat Enterprise Linux
bash os_identify.sh        # 输出：Usage: ...
```
