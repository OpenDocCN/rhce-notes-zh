在考试环境的第二个虚拟机中，系统通常会准备三块磁盘：
*   **`/dev/vda`**：这是系统盘，包含启动分区、根分区和交换分区，由卷组和逻辑卷构成。我们无需操作此盘。
*   **`/dev/vdb`**：这是关键的数据盘，我们需要在此盘上完成多道题目。
*   **`/dev/vdc`**：这是一块空磁盘，专门用于“创建VDO卷”这道题目，切勿提前使用。

## Linux磁盘使用流程 🔄

在开始具体操作前，我们先来了解在Linux中使用一块新磁盘的标准流程。这个过程是线性的，每一步都依赖于前一步的完成。

![](img/3c94ef84362921039c5a60383bfd6407_41.png)

![](img/3c94ef84362921039c5a60383bfd6407_43.png)

![](img/3c94ef84362921039c5a60383bfd6407_45.png)

Linux系统使用磁盘存储设备通常遵循以下步骤：
1.  **识别磁盘**：系统启动时，内核自动识别硬件磁盘。
2.  **创建分区**：使用分区工具（如`fdisk`, `parted`）在磁盘上划分存储区域。
3.  **格式化分区**：使用格式化工具（如`mkfs`, `mkswap`）在分区上创建文件系统。
4.  **挂载使用**：使用`mount`命令将格式化好的分区关联到目录树上的某个挂载点，之后才能通过该目录访问存储空间。

![](img/3c94ef84362921039c5a60383bfd6407_47.png)

## 磁盘与分区管理工具 🛠️

上一节我们介绍了磁盘使用的完整流程，本节中我们来看看流程中涉及的核心工具。

### 识别与查看磁盘

系统识别磁盘后，我们可以使用命令查看磁盘及分区信息。
*   **`lsblk`**：以树状图简洁地列出所有块设备（磁盘和分区）。
*   **`fdisk -l`**：详细列出所有磁盘的分区表、容量等信息。查看特定磁盘可用 `fdisk -l /dev/vdb`。

### 创建与管理分区

要对磁盘进行分区，我们需要选择合适的分区工具。主要的分区表格式有两种：
*   **MBR (MS-DOS)**：传统模式，最多支持4个主分区（或3主分区+1扩展分区），单个磁盘容量支持不超过2TB。
*   **GPT**：新模式，支持超过2TB的磁盘，主分区数量理论上无限制（Linux通常支持最多128个）。

以下是常用的分区工具：
*   **`fdisk`**：传统交互式工具，主要支持MBR格式。其特点是**操作后需输入`w`命令才会保存并生效**，输入`q`则放弃更改退出，适合新手练习。
*   **`parted`**：更强大的交互式工具，**同时支持MBR和GPT格式**。其特点是**操作指令一经执行立即生效**，适合有经验的管理员，使用时需谨慎。
*   **`gdisk`**：专门用于GPT格式分区的交互式工具（在RHEL8中可能需要单独安装`gdisk`软件包）。

![](img/3c94ef84362921039c5a60383bfd6407_49.png)

![](img/3c94ef84362921039c5a60383bfd6407_51.png)

**分区表更新**：分区操作完成后，需要让Linux内核重新读取分区表。
*   最彻底的方式是**重启系统** (`reboot`)。
*   如果不想重启，可以尝试使用 **`partprobe /dev/vdb`** 或 **`partx -a /dev/vdb`** 通知内核刷新指定磁盘的分区信息。

![](img/3c94ef84362921039c5a60383bfd6407_53.png)

![](img/3c94ef84362921039c5a60383bfd6407_54.png)

### 格式化分区

![](img/3c94ef84362921039c5a60383bfd6407_56.png)

分区创建后，必须格式化才能用于存储文件。格式化即创建文件系统。
*   **`mkfs`**：通用的格式化命令，其后可接文件系统类型，如 `mkfs.xfs`, `mkfs.ext4`。
*   **`mkswap`**：专门用于将分区格式化为交换空间（Swap）的命令。

![](img/3c94ef84362921039c5a60383bfd6407_58.png)

你可以通过 `mkfs` 后按两次Tab键，查看系统支持的所有格式化工具。

## 实战演练：分区操作 💪

![](img/3c94ef84362921039c5a60383bfd6407_60.png)

![](img/3c94ef84362921039c5a60383bfd6407_61.png)

理论介绍完毕，现在让我们通过实际操作来巩固知识。我们将模拟考试需求，在`/dev/vdb`磁盘上创建两个新分区。

![](img/3c94ef84362921039c5a60383bfd6407_63.png)

假设通过`lsblk`查看，`/dev/vdb`有10GB空间，已用约2GB，剩余约8GB。我们需要创建一个512MB的分区（用于交换分区）和一个1000MB的分区（用于逻辑卷，略大于800MB需求）。

以下是使用 `fdisk` 工具进行分区的步骤：

1.  运行 `fdisk /dev/vdb` 进入交互界面。
2.  输入 `p` 打印当前分区表，确认现有分区。
3.  输入 `n` 创建新分区。
4.  选择分区编号（例如，直接回车使用默认值2）。
5.  选择起始扇区（直接回车使用默认值）。
6.  设置结束位置时，使用 **`+512M`** 指定分区大小为512MB。
7.  再次输入 `p` 查看，确认新分区`/dev/vdb2`已创建。
8.  重复步骤3-6，创建第三个分区，结束位置设置为 **`+1000M`**。
9.  输入 `p` 最终确认分区表包含`/dev/vdb2`(512M)和`/dev/vdb3`(1000M)。
10. 确认无误后，输入 **`w`** 保存分区表并退出。如果操作有误，则输入 **`q`** 放弃更改退出。
11. 执行 `partprobe /dev/vdb` 或重启系统，使新分区生效。

## 实战演练：添加交换分区 🔄

![](img/3c94ef84362921039c5a60383bfd6407_65.png)

![](img/3c94ef84362921039c5a60383bfd6407_66.png)

上一节我们完成了分区创建，本节中我们利用创建好的分区来完成“添加交换分区”这道题目。

![](img/3c94ef84362921039c5a60383bfd6407_68.png)

题目要求：为系统添加一个512MB的交换分区，并确保其在系统启动时自动挂载。

操作步骤如下：

1.  **格式化分区为交换空间**：
    ```bash
    mkswap /dev/vdb2
    ```
    执行后，可以使用 `blkid /dev/vdb2` 查看，其 `TYPE` 会显示为 `swap`。

![](img/3c94ef84362921039c5a60383bfd6407_70.png)

![](img/3c94ef84362921039c5a60383bfd6407_72.png)

2.  **临时启用交换分区**：
    ```bash
    swapon /dev/vdb2
    ```
    使用 `swapon -s` 或 `free -h` 命令检查，应能看到新增的交换空间。

![](img/3c94ef84362921039c5a60383bfd6407_74.png)

![](img/3c94ef84362921039c5a60383bfd6407_76.png)

3.  **配置开机自动挂载**：
    编辑 `/etc/fstab` 文件，在末尾添加一行配置：
    ```
    /dev/vdb2 swap swap defaults 0 0
    ```
    *   第一列：设备名 (`/dev/vdb2`)
    *   第二列：挂载点 (交换分区固定为 `swap`)
    *   第三列：文件系统类型 (`swap`)
    *   第四列：挂载参数 (`defaults`)
    *   第五、六列：dump备份和fsck检查标记（均为`0`）

4.  **验证自动挂载配置**：
    *   可以先执行 `swapoff /dev/vdb2` 停用。
    *   然后执行 `swapon -a`，此命令会尝试启用`/etc/fstab`中所有配置为`swap`的设备。
    *   再次执行 `swapon -s`，如果能看到`/dev/vdb2`，则证明配置成功，系统重启后也会自动挂载。
