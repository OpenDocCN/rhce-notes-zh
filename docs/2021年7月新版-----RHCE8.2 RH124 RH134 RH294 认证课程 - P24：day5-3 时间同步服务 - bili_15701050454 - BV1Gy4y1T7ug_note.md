## 概述
在本节课中，我们将要学习Linux系统中的时间同步服务。时间同步在多主机协作环境中至关重要，它能确保集群内所有服务器的时间保持一致，避免因时间差异导致的应用错误、日志混乱和证书失效等问题。我们将重点介绍现代化的时间同步服务 `chrony`，包括其工作原理、配置方法以及相关的管理工具。

## 时间同步的重要性
在多主机协同工作的环境中，时间同步非常重要。如果主机之间的系统时间不一致，会导致诸多问题。

例如，假设有服务器A和服务器B。如果A的系统时间是13点，而B的系统时间是14点，那么它们记录的系统日志时间就会出现误差。这不仅仅是文本记录上的误差。

如果涉及到应用程序，时间差可能导致它们的工作时间不一致。例如，两台电脑都设定在13:05分执行计划任务，但B的系统时间快了1小时，它就会在12:05（实际时间）提前执行任务，如果任务涉及数据收集，就会造成数据不一致。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_3.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_5.png)

在实际工作中，服务很少是单机运行，通常是集群或分布式部署。这些服务需要互相沟通和依赖。如果时间对不上，会造成很大的问题。

## NTP 与 Chrony
时间同步主要有两种服务：NTP和Chrony。

NTP服务目前可能已经较少使用。当然，一些较早入行的工程师可能仍习惯使用NTP。

新兴的服务是 `chrony`。它的误差更小，配置也更简单。现在一般推荐使用 `chrony`。

时间同步利用NTP协议进行服务调度。下面看一下它们的区别：
*   **NTP服务**：在局域网内误差可达0.1毫秒以内；在互联网上精度为1到50毫秒。
*   **Chrony服务**：同样是基于NTP协议的自研软件。其精度在几毫秒之内；在局域网内可达几十微秒。这比NTP更加精细。

## Chrony 服务架构
让我们了解一下时间同步服务的架构。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_7.png)

假设我们有一台时间服务器，运行 `chrony` 服务。还有多台需要同步时间的服务器，例如服务器A、B、C。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_9.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_11.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_13.png)

这三台服务器需要时间同步。它们通过 `chrony` 服务，与时间服务器同步时间，从而确保彼此时间一致。

那么时间服务器自身的时间如何同步呢？它通过互联网连接到公共的时间同步服务器（例如 `time.windows.com`）。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_15.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_17.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_19.png)

整个架构可以这样理解：公司内部的集群服务器（客户端）通过内网与内部的 `chrony` 时间服务器（服务端）同步；内部的 `chrony` 服务器再通过外网与公共时间服务器同步。

从这个架构可以看出，时间同步服务也是一个C/S（客户端/服务器）结构。内部的 `chrony` 服务器对于公共服务器是客户端，对于内部集群又是服务端。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_21.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_23.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_25.png)

## Chrony 的优势
`chrony` 具有以下优势：
*   **同步更快**：初始同步只需数分钟，而非数小时，从而最大程度减少了时间和频率的误差。这对于并非24小时运行的虚拟机非常有用。
*   **更好的稳定性**：能更好地适应间歇性网络连接，发现时间不对时会自动同步。
*   **配置简单**：不需要频繁的手动调整。

## Chrony 的配置与使用
`chrony` 的服务端和客户端都通过同一个配置文件进行配置。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_37.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_39.png)

### 配置文件
主要的配置文件是 `/etc/chrony.conf`。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_41.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_43.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_45.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_47.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_49.png)

配置的核心在于指定时间同步服务器。以下是关键参数：
*   `pool`：指定一组时间服务器。
*   `server`：指定单一的时间同步服务器。
格式通常为：`server [地址] iburst`。`iburst` 选项可以加速初始同步。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_51.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_53.png)

例如，指定阿里云和腾讯云的时间服务器：
```bash
server ntp.aliyun.com iburst
server time1.tencentyun.com iburst
```
建议指定多个服务器以确保冗余性。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_57.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_59.png)

### 服务管理
以下是管理 `chrony` 服务的常用命令：
*   启动服务：`systemctl start chronyd`
*   停止服务：`systemctl stop chronyd`
*   重启服务：`systemctl restart chronyd`
*   查看状态：`systemctl status chronyd`
*   设置开机自启：`systemctl enable chronyd`

![](img/cb992f5bb674cca3cf8b11eb6068fe43_63.png)

### 客户端工具
常用的客户端命令是 `chronyc`。使用 `chronyc sources` 可以查看当前同步的源服务器状态。
输出结果前的符号含义如下：
*   `*`：当前正在使用的同步源。
*   `+`：可用的备用同步源。
*   `-`：不可用的同步源。
*   `?`：失去连接的同步源（能解析IP但无法连接）。

### 配置内部时间服务器
若要将一台主机配置为内部网络的时间服务器（既同步外部时间，又为内部客户端提供服务），需要修改其配置文件 `/etc/chrony.conf`。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_75.png)

在配置文件中添加 `allow` 指令来允许特定网段的客户端同步，例如：
```bash
allow 192.168.18.0/24
```
然后重启 `chronyd` 服务。内部的其他客户端只需将 `server` 指向这台内部服务器的IP地址即可。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_78.png)

## 系统时间与日期管理
除了 `chrony`，系统也提供了其他时间管理工具。

### `timedatectl` 命令
`timedatectl` 是一个强大的时间日期管理工具。

![](img/cb992f5bb674cca3cf8b11eb6068fe43_81.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_83.png)

以下是常用操作：
*   **查看当前时间设置**：`timedatectl status`
*   **列出所有时区**：`timedatectl list-timezones`
*   **设置系统时区**：`timedatectl set-timezone Asia/Shanghai`
*   **手动设置时间**（需先关闭NTP同步）：`timedatectl set-time “YYYY-MM-DD HH:MM:SS”`
*   **开启/关闭NTP时间同步**：`timedatectl set-ntp true/false`

![](img/cb992f5bb674cca3cf8b11eb6068fe43_85.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_86.png)

![](img/cb992f5bb674cca3cf8b11eb6068fe43_88.png)

**注意**：手动修改时间时，如果NTP服务开启，操作会被禁止。需要先使用 `timedatectl set-ntp false` 关闭NTP同步，修改后再开启。
