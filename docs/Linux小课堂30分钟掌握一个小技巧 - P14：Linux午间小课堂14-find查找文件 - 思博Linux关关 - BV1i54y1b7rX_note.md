# Linux小课堂：P14：使用find和locate命令查找文件 🔍

在本节课中，我们将学习Linux系统中两种强大的文件查找工具：`find`和`locate`。我们将重点探讨`find`命令如何根据时间、用户属性和文件大小等属性来精确查找文件，并了解`locate`命令基于数据库的快速查找方式。掌握这些技巧对于系统管理和安全维护至关重要。

## 通过时间属性查找文件 ⏰

上一节我们介绍了网络数据包分析工具，本节中我们来看看如何根据文件的时间属性进行查找。在Linux中，文件具有访问时间（atime）、修改时间（mtime）和状态变更时间（ctime）等属性。当文件被访问、修改或元数据（如权限）改变时，这些时间戳会更新。利用`find`命令，我们可以筛选在特定时间段内被修改过的文件，这在排查系统异常或安全事件时非常有用。

以下是基于分钟数查找的命令格式：

```bash
find [路径] -mmin [+/-]分钟数
```

*   **`-20`**： 查找在当前时间**20分钟以内**被修改过的文件。
*   **`+20`**： 查找在当前时间**20分钟以前**（更早）被修改过的文件。
*   **`20`**： 查找在**恰好20分钟前**那个时间点被修改过的文件。

对于更长时间跨度的查找，使用`-mtime`参数更为方便，它以天为单位。

![](img/12634f69778c6325af6ec6f471604191_1.png)

以下是基于天数查找的命令格式：

```bash
find [路径] -mtime [+/-]天数
```

![](img/12634f69778c6325af6ec6f471604191_3.png)

![](img/12634f69778c6325af6ec6f471604191_4.png)

*   **`-4`**： 查找在当前时间**4天以内**被修改过的文件。
*   **`+4`**： 查找在当前时间**4天以前**（更早）被修改过的文件。
*   **`4`**： 查找在**恰好4天前**那一天被修改过的文件。

![](img/12634f69778c6325af6ec6f471604191_6.png)

> **提示**：将参数中的 `-mmin` 或 `-mtime` 替换为 `-amin` 或 `-atime` 可以基于访问时间查找，替换为 `-cmin` 或 `-ctime` 可以基于状态变更时间查找。

![](img/12634f69778c6325af6ec6f471604191_8.png)

## 通过用户属性查找文件 👤

![](img/12634f69778c6325af6ec6f471604191_10.png)

![](img/12634f69778c6325af6ec6f471604191_11.png)

![](img/12634f69778c6325af6ec6f471604191_12.png)

如果怀疑有非法用户登录并创建了文件，我们可以根据文件的所有者（用户）或所属组来查找文件。这有助于快速定位可疑文件并进行清理。

以下是基于用户查找的命令：

![](img/12634f69778c6325af6ec6f471604191_14.png)

![](img/12634f69778c6325af6ec6f471604191_15.png)

![](img/12634f69778c6325af6ec6f471604191_16.png)

![](img/12634f69778c6325af6ec6f471604191_17.png)

![](img/12634f69778c6325af6ec6f471604191_18.png)

```bash
find [路径] -user 用户名
find [路径] -uid 用户ID
```

![](img/12634f69778c6325af6ec6f471604191_20.png)

第一条命令通过用户名查找，第二条命令通过用户ID查找。例如，`find /home -user chenxi` 会找出`/home`目录下所有属于用户`chenxi`的文件。

![](img/12634f69778c6325af6ec6f471604191_22.png)

同样，我们也可以根据组来查找：

![](img/12634f69778c6325af6ec6f471604191_24.png)

![](img/12634f69778c6325af6ec6f471604191_26.png)

```bash
find [路径] -group 组名
find [路径] -gid 组ID
```

![](img/12634f69778c6325af6ec6f471604191_27.png)

这些命令匹配的信息与系统文件`/etc/passwd`和`/etc/group`中的记录相对应。

## 通过文件大小查找文件 📏

![](img/12634f69778c6325af6ec6f471604191_33.png)

![](img/12634f69778c6325af6ec6f471604191_34.png)

![](img/12634f69778c6325af6ec6f471604191_35.png)

![](img/12634f69778c6325af6ec6f471604191_37.png)

当磁盘空间不足时，我们需要找出占用空间较大的文件。`find`命令的`-size`参数可以根据文件大小进行筛选。

![](img/12634f69778c6325af6ec6f471604191_39.png)

![](img/12634f69778c6325af6ec6f471604191_40.png)

以下是基于文件大小查找的命令格式：

![](img/12634f69778c6325af6ec6f471604191_42.png)

![](img/12634f69778c6325af6ec6f471604191_44.png)

```bash
find [路径] -size [+/-]大小
```

![](img/12634f69778c6325af6ec6f471604191_46.png)

![](img/12634f69778c6325af6ec6f471604191_48.png)

![](img/12634f69778c6325af6ec6f471604191_50.png)

*   **`-20k`**： 查找**小于20KB**的文件。
*   **`+20k`**： 查找**大于20KB**的文件。
*   **`20k`**： 查找**等于20KB**的文件。

![](img/12634f69778c6325af6ec6f471604191_52.png)

大小单位可以是`c`（字节）、`k`（千字节）、`M`（兆字节）、`G`（吉字节）。**需要注意**，`find`命令在匹配大小时会进行**四舍五入**。例如，一个0.1KB的文件会被归入“0k”的查找范围，而不是“1k”。

## find与locate命令的对比 ⚖️

`find`命令通过递归遍历目录树来查找文件，这种方式虽然精确，但在目录结构庞大（如从根目录`/`开始查找）时可能效率较低，并消耗大量系统资源。

`locate`命令则通过查询预建的文件名数据库来实现快速查找。因为它不实时扫描磁盘，所以速度极快。

以下是`locate`命令的基本用法：

```bash
locate 关键词
```

例如，`locate passwd`会找出所有文件名中包含“passwd”的文件。

![](img/12634f69778c6325af6ec6f471604191_54.png)

![](img/12634f69778c6325af6ec6f471604191_56.png)

**两者的主要区别与注意事项：**

![](img/12634f69778c6325af6ec6f471604191_58.png)

1.  **速度**：`locate`远快于`find`，因为它查询的是数据库。
2.  **实时性**：`locate`的数据库（通常由`updatedb`命令更新，并有定时任务自动运行）并非实时更新。新创建的文件可能需要手动运行`updatedb`后，才能被`locate`找到。
3.  **适用场景**：在脚本中或需要精确匹配各种属性（时间、权限等）时，使用`find`更可靠。临时快速查找已知文件名的文件时，使用`locate`更高效。
