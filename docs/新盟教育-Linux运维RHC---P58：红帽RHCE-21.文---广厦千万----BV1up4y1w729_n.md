## FTP服务简介
FTP是一种在企业内部用于文件传输和共享的服务，其功能类似于个人使用的百度网盘，但数据存储在企业自己的服务器上，保证了数据的私密性。

FTP的全称是File Transfer Protocol，即文件传输协议。它可以传输各种类型的文件，例如MP3、MP4、脚本、图片、文本文件等。在企业网络环境中，访问网站通常使用HTTP/HTTPS协议，而进行文件数据传输则使用FTP协议。

FTP是一种基于TCP协议的端到端数据传输协议。底层通过TCP协议（如三次握手）建立可靠连接，上层则通过FTP协议进行文件传输。

---

## FTP架构与端口
FTP采用客户端/服务器（C/S）架构。
*   **客户端（Client）**：用于连接服务器并下载或上传文件。
*   **服务器（Server）**：用于存储和共享文件。

FTP服务默认使用两个端口：
*   **端口21**：命令端口，用于接收客户端的指令（如请求下载某个文件）。
*   **端口20**：数据端口，用于实际的数据传输。

![](img/7890579e5f185ac8c485bae22f186d0b_5.png)

客户端在连接时也会使用随机的命令端口和数据端口，分别与服务端的对应端口进行通信。

## FTP的工作模式
FTP有两种主要的工作模式，区别在于数据传输连接由谁主动发起。

![](img/7890579e5f185ac8c485bae22f186d0b_11.png)

![](img/7890579e5f185ac8c485bae22f186d0b_13.png)

![](img/7890579e5f185ac8c485bae22f186d0b_14.png)

![](img/7890579e5f185ac8c485bae22f186d0b_16.png)

![](img/7890579e5f185ac8c485bae22f186d0b_18.png)

### 主动模式
在主动模式下：
1.  客户端使用随机端口向服务器的**21号命令端口**发起连接，并发送指令（例如请求下载文件）。在指令中，客户端会告知服务器自己的**数据端口号**（例如1024）。
2.  服务器收到指令后，使用自己的**20号数据端口**，主动去连接客户端告知的**数据端口**（1024）。
3.  连接建立后，开始数据传输。

![](img/7890579e5f185ac8c485bae22f186d0b_20.png)

![](img/7890579e5f185ac8c485bae22f186d0b_22.png)

**核心特点**：数据传输连接由**服务器端**主动向客户端发起。
**潜在问题**：如果客户端开启了防火墙，可能会拦截服务器发起的连接请求。解决方法是在客户端防火墙放行FTP服务或相关端口，或者直接关闭防火墙。

![](img/7890579e5f185ac8c485bae22f186d0b_24.png)

### 被动模式
在被动模式下：
1.  客户端向服务器的**21号命令端口**发起连接，并发送`PASV`指令，进入被动模式。
2.  服务器收到指令后，随机开启一个**数据端口**（例如5000），并将此端口号告知客户端。
3.  客户端使用自己的随机数据端口，主动去连接服务器的**随机数据端口**（5000）。
4.  连接建立后，开始数据传输。

![](img/7890579e5f185ac8c485bae22f186d0b_26.png)

![](img/7890579e5f185ac8c485bae22f186d0b_28.png)

![](img/7890579e5f185ac8c485bae22f186d0b_30.png)

**核心特点**：数据传输连接由**客户端**主动向服务器端发起。
**注意事项**：服务器端需要放行随机范围的数据端口，或直接关闭防火墙。`vsftpd`默认使用被动模式。

## 常见的FTP软件
要在Linux系统上搭建FTP服务器，可以选择以下软件：
*   `wu-ftpd`：非常古老的FTP软件。
*   `ProFTPD`：专业的FTP软件。
*   `PureFTPd`：纯粹的FTP软件。
*   **`vsftpd`**：非常安全的FTP守护进程，在企业中应用广泛。

在Windows系统上，则有`Serv-U`等软件。
本节课我们将重点学习 **`vsftpd`**。

客户端访问FTP服务器时，也需要使用工具，例如：
*   `ftp`：古老的命令行客户端。
*   `lftp`：`ftp`的增强版命令行客户端，功能更强大。
*   `FileZilla`：图形化的FTP客户端工具。

## 安装与启动vsftpd
上一节我们介绍了FTP的基本概念，本节我们来看看如何在服务器上安装和配置`vsftpd`。

![](img/7890579e5f185ac8c485bae22f186d0b_43.png)

### 环境准备
1.  确保服务器网络连通。
2.  关闭防火墙或放行FTP服务（学习环境建议直接关闭）。
    ```bash
    # 关闭firewalld防火墙
    systemctl stop firewalld
    systemctl disable firewalld

    # 若使用iptables，也将其关闭
    systemctl stop iptables
    systemctl disable iptables
    ```
3.  临时关闭SELinux（可选，学习环境建议关闭）。
    ```bash
    setenforce 0
    ```

![](img/7890579e5f185ac8c485bae22f186d0b_49.png)

### 安装vsftpd
使用yum包管理器安装`vsftpd`软件。

## vsftpd的用户访问模式
`vsftpd`支持三种用户访问模式，用于控制客户端的认证方式。

1.  **本地用户模式**：客户端必须使用服务器系统上存在的真实用户名和密码进行登录。
2.  **匿名用户模式**：客户端无需任何认证，可以直接访问服务器提供的共享目录。这是默认启用的模式。
3.  **虚拟用户模式**：用户信息存储在独立的数据库（如MySQL）中，而非系统用户。安全性更高，配置也更复杂。

默认情况下，`vsftpd`启用的是**匿名用户模式**。匿名用户登录时，实际是以系统内置的`ftp`用户身份访问服务器。

### 安装客户端工具
在客户端机器上安装访问工具，例如`ftp`和其增强版`lftp`。
```bash
yum install -y ftp lftp
```

### 使用lftp连接服务器
`lftp`功能更强大，默认以匿名用户身份连接，使用更方便。
```bash
# 连接FTP服务器，IP地址替换为你的服务器IP
lftp 192.168.0.40
```
连接成功后，命令行提示符会发生变化，表示已进入FTP会话。

![](img/7890579e5f185ac8c485bae22f186d0b_85.png)

![](img/7890579e5f185ac8c485bae22f186d0b_87.png)

### 常用FTP命令
在`lftp`会话中，可以执行很多命令。输入`help`可以查看所有支持的命令。
以下是文件操作相关的常用命令：

*   `ls`：列出服务器当前目录的文件和子目录。
*   `cd [目录名]`：切换服务器上的目录。
*   `pwd`：显示服务器上的当前目录。
*   `get [文件名]`：从服务器下载单个文件到客户端当前目录。
    ```bash
    get hello.txt
    ```
*   `put [文件名]`：将客户端当前目录的文件上传到服务器。
    ```bash
    put local_file.txt
    ```
*   `mget [文件模式]`：批量下载文件（支持通配符`*`）。
*   `mput [文件模式]`：批量上传文件。
*   `mkdir [目录名]`：在服务器上创建目录。
*   `rm [文件名]`：删除服务器上的文件。
*   `exit` 或 `bye`：退出`lftp`会话。

## 配置匿名用户上传权限
默认情况下，匿名用户可以浏览和下载文件，但无法上传、创建或删除文件。如果需要这些功能，需要进行配置。

### 1. 修改共享目录权限
匿名用户的默认共享目录是`/var/ftp/pub`。我们需要确保`ftp`用户对这个目录有写权限。
```bash
# 在FTP服务器上执行
# 将/var/ftp/pub目录的所有者和组改为ftp用户
chown ftp:ftp /var/ftp/pub

![](img/7890579e5f185ac8c485bae22f186d0b_89.png)

![](img/7890579e5f185ac8c485bae22f186d0b_91.png)

# 为该目录添加写权限
chmod o+w /var/ftp/pub
# 或者直接赋予777权限（测试环境可用，生产环境需谨慎）
# chmod 777 /var/ftp/pub
```

### 2. 修改vsftpd配置文件
编辑`vsftpd`的主配置文件`/etc/vsftpd/vsftpd.conf`。
```bash
vim /etc/vsftpd/vsftpd.conf
```
找到并修改以下配置项，取消注释（删除行首的`#`）或将值改为`YES`：
```bash
# 允许匿名用户上传文件
anon_upload_enable=YES

# 允许匿名用户创建新目录
anon_mkdir_write_enable=YES

# 允许匿名用户进行写操作（包括重命名、删除等）
anon_other_write_enable=YES
```

### 3. 重启vsftpd服务
每次修改配置文件后，都需要重启服务使配置生效。
```bash
systemctl restart vsftpd
```

### 4. 客户端测试
配置完成后，在客户端重新连接服务器，并切换到`pub`目录，即可进行上传、创建文件夹等操作。
```bash
lftp 192.168.0.40
cd pub
put myfile.txt
mkdir new_folder
```
