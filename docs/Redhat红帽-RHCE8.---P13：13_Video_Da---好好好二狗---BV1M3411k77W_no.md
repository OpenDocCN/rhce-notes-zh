在本节课中，我们将要学习Linux系统中至关重要的文件及目录权限管理。理解权限是系统安全和管理的基础，它决定了用户能对文件和目录执行哪些操作。

![](img/3647f721fb1ab2bb9117dcd068cdd086_5.png)

## 权限的基本概念

上一节我们介绍了Linux系统的基本操作，本节中我们来看看权限的核心概念。权限定义了用户对文件或目录的访问能力，主要分为三种基本操作：读、写、执行。

![](img/3647f721fb1ab2bb9117dcd068cdd086_7.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_8.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_10.png)

*   **读**：对于文件，意味着可以查看其内容；对于目录，意味着可以列出目录内的文件和子目录名称。
*   **写**：对于文件，意味着可以修改其内容；对于目录，意味着可以在其中创建、删除或重命名文件和子目录。
*   **执行**：对于文件，意味着可以将其作为程序或脚本运行；对于目录，意味着可以进入该目录，使其成为当前工作目录。

![](img/3647f721fb1ab2bb9117dcd068cdd086_12.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_14.png)

## 身份与权限的对应关系

每个文件或目录都关联着三种身份，权限正是针对这三种身份分别设置的。

![](img/3647f721fb1ab2bb9117dcd068cdd086_16.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_18.png)

*   **所有者**：文件的创建者或当前拥有者。
*   **所属组**：文件所有者所属的用户组。
*   **其他人**：既不是所有者，也不在所属组中的其他所有用户。

**注意**：`root`用户是超级管理员，但`root`组只是一个普通用户组，不具备特殊权限。具有特殊权限的系统组是`wheel`组。

## 文件与目录权限的差异

![](img/3647f721fb1ab2bb9117dcd068cdd086_20.png)

文件与目录的“执行”权限含义不同，这是初学者容易混淆的地方。以下通过具体例子说明目录权限的特殊性。

创建一个测试目录并观察其默认权限：
```bash
mkdir /tmp/test_privilege
ls -ld /tmp/test_privilege
```
默认权限可能是 `drwxr-xr-x`，表示所有者有全部权限，所属组和其他人只有读和执行权限。

**目录执行权限的作用**：
如果目录没有执行权限，用户将无法进入该目录。即使有读权限，也只能看到文件名，无法获取详细信息。

![](img/3647f721fb1ab2bb9117dcd068cdd086_22.png)

**目录读权限的作用**：
如果目录只有执行权限而没有读权限，用户可以进入目录，但无法列出其中的内容。

因此，对于一个希望被正常访问的目录，通常需要同时具备读和执行权限。

![](img/3647f721fb1ab2bb9117dcd068cdd086_24.png)

## 权限的表示与修改方法

![](img/3647f721fb1ab2bb9117dcd068cdd086_26.png)

了解了权限的含义后，我们来看看如何查看和修改权限。

![](img/3647f721fb1ab2bb9117dcd068cdd086_28.png)

### 符号表示法
使用 `ls -l` 命令查看时，权限以10个字符表示，例如 `-rwxr-xr--`。
*   第1位：文件类型（`-` 普通文件，`d` 目录，`l` 链接等）。
*   第2-4位：所有者权限。
*   第5-7位：所属组权限。
*   第8-10位：其他人权限。

### 修改权限命令：`chmod`
`chmod` 命令用于修改文件或目录的权限，主要有两种方法：

**1. 符号法**
使用 `u`（所有者）、`g`（所属组）、`o`（其他人）、`a`（所有人）结合 `+`（增加）、`-`（移除）、`=`（设置）来操作。
例如：
```bash
chmod u+x file.txt      # 给所有者增加执行权限
chmod g-w file.txt      # 移除所属组的写权限
chmod o=r file.txt      # 设置其他人权限为只读
chmod a+x file.txt      # 给所有人增加执行权限
```

![](img/3647f721fb1ab2bb9117dcd068cdd086_30.png)

**2. 数字法**
将读（r）、写（w）、执行（x）分别对应数字4、2、1，将同一身份的权限值相加，得到一组三位数字。
例如：
```bash
chmod 755 directory/    # 对应 rwxr-xr-x
chmod 644 file.txt      # 对应 rw-r--r--
```
数字法更为简洁常用。

## 修改文件所有者和所属组

除了权限，我们还可以修改文件的所有者和所属组身份。

![](img/3647f721fb1ab2bb9117dcd068cdd086_32.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_34.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_36.png)

*   **`chown`**：修改文件所有者和/或所属组。
    ```bash
    chown user1 file.txt          # 修改所有者
    chown :group1 file.txt        # 修改所属组
    chown user1:group1 file.txt   # 同时修改所有者和所属组
    chown -R user1:group1 dir/    # 递归修改目录下所有文件
    ```
*   **`chgrp`**：专门用于修改文件所属组。
    ```bash
    chgrp group1 file.txt
    ```

![](img/3647f721fb1ab2bb9117dcd068cdd086_40.png)

## 默认权限与umask

![](img/3647f721fb1ab2bb9117dcd068cdd086_42.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_44.png)

当我们创建新文件或目录时，系统会赋予一个默认权限，这由 `umask`（权限掩码）值决定。

![](img/3647f721fb1ab2bb9117dcd068cdd086_46.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_48.png)

![](img/3647f721fb1ab2bb9117dcd068cdd086_50.png)

*   目录的默认最大权限是 `777`。
*   文件的默认最大权限是 `666`（默认不赋予执行权限）。
*   实际权限 = 最大权限 - umask值。

查看当前umask值：
```bash
umask
```
通常root用户的umask是`0022`，普通用户是`0002`。这意味着：
*   root用户创建的文件权限是 `666 - 022 = 644`。
*   root用户创建的目录权限是 `777 - 022 = 755`。

临时修改umask：
```bash
umask 027
```
永久修改需要编辑 `/etc/profile` 或用户shell配置文件。

![](img/3647f721fb1ab2bb9117dcd068cdd086_52.png)

## 特殊权限

除了基本的rwx，Linux还有三种特殊权限位。

**1. SUID（Set User ID）**
*   **作用**：当用户执行一个设置了SUID的可执行文件时，该进程将暂时拥有文件**所有者**的权限，而不是执行者的权限。
*   **设置**：`chmod u+s file`
*   **位置与表示**：出现在所有者执行位，若原位置有`x`则显示为`s`，否则为`S`。
*   **数值表示**：在数字法前加4，如 `4755`。
*   **典型例子**：`/usr/bin/passwd` 命令，允许普通用户修改自己的密码（写入 `/etc/shadow` 文件）。

**2. SGID（Set Group ID）**
*   **对文件的作用**：类似SUID，进程将暂时拥有文件**所属组**的权限。
*   **对目录的作用（更常用）**：在该目录下创建的新文件或子目录，将自动继承该目录的所属组，而不是创建者的主组。
*   **设置**：`chmod g+s directory`
*   **位置与表示**：出现在所属组执行位，显示为`s`或`S`。
*   **数值表示**：在数字法前加2，如 `2755`。

![](img/3647f721fb1ab2bb9117dcd068cdd086_54.png)

**3. Sticky Bit（粘滞位）**
*   **作用**：仅对目录有效。设置了粘滞位的目录，即使所有用户都有写权限，也只有文件**所有者**、目录**所有者**或**root**用户才能删除或重命名其中的文件。
*   **典型应用**：系统临时目录 `/tmp`。
*   **设置**：`chmod o+t directory`
*   **位置与表示**：出现在其他人执行位，显示为`t`或`T`。
*   **数值表示**：在数字法前加1，如 `1755`。

![](img/3647f721fb1ab2bb9117dcd068cdd086_56.png)

## 文件扩展属性

最后，我们介绍两个用于写保护的扩展属性，通过 `chattr` 命令管理。

![](img/3647f721fb1ab2bb9117dcd068cdd086_58.png)

*   **`i`（不可变）属性**：文件不能被修改、删除、重命名或创建链接，超级用户也无法直接操作。
    ```bash
    chattr +i important_file.txt  # 添加不可变属性
    lsattr important_file.txt     # 查看属性
    chattr -i important_file.txt  # 移除不可变属性
    ```
*   **`a`（仅追加）属性**：文件内容只能追加，不能修改或删除已有内容。
    ```bash
    chattr +a log_file.log
    ```
