## SSH客户端连接

上一节我们介绍了SSH服务端的配置，本节中我们来看看如何使用客户端进行连接。

在配置好SSH服务后，我们通常使用SSH客户端工具进行远程连接。最基本的连接命令格式如下：

```bash
ssh [用户名@]IP地址 [-p 端口号]
```

例如，从一台IP为 `192.168.31.146` 的机器连接到 `192.168.31.145`，可以使用：

```bash
ssh 192.168.31.145
```
或指定用户（默认为root）和端口（默认为22）：
```bash
ssh root@192.168.31.145 -p 22
```

如果是首次连接某台服务器，系统会提示你确认主机的密钥指纹，输入 `yes` 后，再输入对应用户的密码即可建立连接。之后再次连接同一台主机时，将不再需要确认密钥。

## 无密码验证登录

在实际运维中，频繁手动输入密码非常低效，尤其是在脚本自动化场景下。因此，我们需要配置无密码验证登录。

无密码验证的核心是**非对称加密**。客户端生成一对密钥（公钥和私钥），将公钥上传到服务端。之后客户端连接时，服务端会用存储的公钥来验证客户端的私钥，验证通过则无需密码。

以下是配置无密码登录的步骤：

1.  **在客户端生成密钥对**
    在希望发起无密码登录的机器（客户端）上执行以下命令生成密钥对：
    ```bash
    ssh-keygen -t rsa
    ```
    执行命令后，连续按回车接受默认设置（密钥存放路径为 `~/.ssh/id_rsa`，不设置密钥密码）。完成后会在 `~/.ssh/` 目录下生成两个文件：`id_rsa`（私钥）和 `id_rsa.pub`（公钥）。

2.  **将公钥上传到服务端**
    使用 `ssh-copy-id` 命令将客户端的公钥传输到目标服务器（服务端）：
    ```bash
    ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.168.31.145
    ```
    此命令会将公钥内容自动追加到服务端 `~/.ssh/authorized_keys` 文件中。执行过程中需要输入一次服务端用户的密码。

![](img/742643619c1d77fb04eabdbf311fb84b_7.png)

3.  **验证无密码登录**
    完成上述步骤后，再次从客户端SSH连接服务端，将不再需要输入密码：
    ```bash
    ssh root@192.168.31.145
    ```

## 无密码验证的应用与原理

配置无密码验证后，可以极大提升自动化操作的效率。

![](img/742643619c1d77fb04eabdbf311fb84b_15.png)

*   **SCP文件传输**：基于SSH通道的 `scp` 命令在传输文件时也不再需要密码。
    ```bash
    # 将本地文件同步到远程服务器
    scp /tmp/111 root@192.168.31.145:/tmp/
    ```

![](img/742643619c1d77fb04eabdbf311fb84b_17.png)

*   **远程执行命令**：无需完全登录到远程服务器，可以直接在本地执行远程命令并返回结果。
    ```bash
    ssh root@192.168.31.145 "ifconfig"
    ```

![](img/742643619c1d77fb04eabdbf311fb84b_19.png)

其工作原理简述如下：
1.  客户端向服务端发起连接请求。
2.  服务端生成一个随机字符串，用客户端预存的公钥加密后，发送给客户端。
3.  客户端用自己的私钥解密该字符串，并将其与会话ID组合，再次加密后发回服务端。
4.  服务端用公钥解密并验证字符串与会话ID，匹配成功则建立加密连接。

![](img/742643619c1d77fb04eabdbf311fb84b_21.png)

这个过程确保了连接的安全性和身份验证，同时免去了手动输入密码的环节。

## 常见问题与故障排除

![](img/742643619c1d77fb04eabdbf311fb84b_34.png)

在配置和使用无密码登录时，可能会遇到一些问题，以下是几个关键点：

![](img/742643619c1d77fb04eabdbf311fb84b_36.png)

![](img/742643619c1d77fb04eabdbf311fb84b_38.png)

1.  **服务端密码更改是否影响无密码登录？**
    **不影响**。无密码登录依赖密钥对验证，与服务端用户密码无关。

![](img/742643619c1d77fb04eabdbf311fb84b_40.png)

2.  **服务端IP地址变更后，无密码登录是否有效？**
    **有效，但首次连接需要处理**。如果服务端IP改变，客户端首次连接时会因为主机密钥不匹配而报错或提示确认。需要删除客户端 `~/.ssh/known_hosts` 文件中旧IP对应的记录，然后重新连接。

![](img/742643619c1d77fb04eabdbf311fb84b_42.png)

![](img/742643619c1d77fb04eabdbf311fb84b_44.png)

![](img/742643619c1d77fb04eabdbf311fb84b_46.png)

![](img/742643619c1d77fb04eabdbf311fb84b_48.png)

3.  **客户端重新生成密钥对后，无密码登录是否有效？**
    **无效**。因为服务端 `authorized_keys` 文件中存储的是旧的公钥，与新生成的密钥对不匹配。需要重新执行 `ssh-copy-id` 上传新的公钥。

![](img/742643619c1d77fb04eabdbf311fb84b_50.png)

![](img/742643619c1d77fb04eabdbf311fb84b_52.png)

4.  **Windows客户端如何实现无密码登录？**
    可以使用如Xshell、SecureCRT、PuTTY等SSH客户端工具。这些工具通常也支持生成密钥对，并将公钥上传至Linux服务器，从而实现基于密钥的认证。
