地址可以是一个数字、一个正则表达式，或者一个范围。

![](img/fa6bdd1b77cb8407f4613df364d5b65f_200.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_202.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_204.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_208.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_210.png)

*   **空地址**：对全文所有行进行处理。
*   **单行地址**：`n` 指定第 `n` 行。
*   **正则地址**：`/pattern/` 匹配包含该模式的行。
*   **范围地址**：
    *   `n,m`：第 `n` 行到第 `m` 行。
    *   `n,+m`：从第 `n` 行开始的连续 `m` 行。
    *   `/pattern1/,/pattern2/`：从匹配`pattern1`的行到匹配`pattern2`的行。
    *   `n,/pattern/`：从第 `n` 行到匹配`pattern`的行。
*   **步进地址**：`1~2` 匹配所有奇数行，`2~2` 匹配所有偶数行。

以下是一些最常用的`sed`编辑命令：

![](img/fa6bdd1b77cb8407f4613df364d5b65f_241.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_243.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_245.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_247.png)

*   **`p`**：打印当前模式空间内容。常与 `-n` 选项一起使用，`-n` 选项会**禁止默认输出**，只有被 `p` 命令处理的行才会被打印。
    ```bash
    sed -n ‘/root/p’ /etc/passwd # 只打印包含root的行
    ```
*   **`d`**：删除模式空间内容，不再进行后续处理，也不打印。
    ```bash
    sed ‘/root/d’ /etc/passwd # 删除包含root的行
    ```
*   **`a \text`**：在匹配行**之后**追加文本 `text`。使用 `\n` 可以追加多行。
    ```bash
    sed ‘/^root/a \# This is the root user’ /etc/passwd
    ```
*   **`i \text`**：在匹配行**之前**插入文本 `text`。
*   **`c \text`**：将匹配行**替换**为文本 `text`。
*   **`w file`**：将匹配行写入指定文件 `file`。
    ```bash
    sed -n ‘/root/w output.txt’ /etc/passwd
    ```
*   **`r file`**：将指定文件 `file` 的内容读取并插入到匹配行之后。
*   **`=`**：打印匹配行的行号。

## sed进阶：搜索与替换 🔍➡️✏️

![](img/fa6bdd1b77cb8407f4613df364d5b65f_257.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_259.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_261.png)

`sed`最核心的功能之一是搜索替换，其语法与Vim编辑器中的替换非常相似。

![](img/fa6bdd1b77cb8407f4613df364d5b65f_267.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_269.png)

基本格式为：`s/查找模式/替换内容/标志`

*   **查找模式**：支持基本和扩展正则表达式。
*   **替换内容**：普通文本，也可以使用后向引用 `\1, \2, ...` 来引用前面分组 `()` 匹配的内容。
*   **标志**：
    *   `g`：全局替换，行内所有匹配项都被替换。
    *   `i`：忽略大小写。
    *   `p`：打印替换成功的行。
    *   `w file`：将替换成功的行写入文件。

![](img/fa6bdd1b77cb8407f4613df364d5b65f_271.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_273.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_275.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_277.png)

**示例1：简单替换**
```bash
sed ‘s/old/new/’ file.txt # 将每行第一个old替换为new
sed ‘s/old/new/g’ file.txt # 将行内所有old替换为new
sed ‘s/old/new/2’ file.txt # 将每行第二个old替换为new
```

![](img/fa6bdd1b77cb8407f4613df364d5b65f_281.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_283.png)

![](img/fa6bdd1b77cb8407f4613df364d5b65f_285.png)

**示例2：使用分组和后向引用**
分组是`sed`中一个极其有用的特性。圆括号 `()` 用于分组，`\1, \2` 用于引用对应分组的内容。
```bash
# 提取eth0网卡的IP地址
ifconfig eth0 | sed -n ‘2p’ | sed -n ‘s/.*inet //p’
# 更优雅的方式：使用分组
ifconfig eth0 | sed -rn ‘2s/.*inet (addr:)?(([0-9]{1,3}\.){3}[0-9]{1,3}).*/\2/p’
```
在这个复杂例子中：
1.  `sed -rn`：`-r`启用扩展正则，`-n`禁止默认输出。
2.  `2s`：只处理第二行。
3.  `.*inet (addr:)?`：匹配 “inet “ 或 “inet addr:”。
4.  `(([0-9]{1,3}\.){3}[0-9]{1,3})`：这是一个分组（\2），匹配IP地址格式。
5.  `.*`：匹配行尾剩余内容。
6.  `/\2/p`：用第二个分组（即IP地址）替换整行，并打印。

**案例1：提取文件的基名和目录名**
```bash
path=“/usr/local/src/package.tar.gz”
# 提取目录名（/usr/local/src）
echo $path | sed -r ‘s|(.*/).*|\1|’
# 提取基名（package.tar.gz）
echo $path | sed -r ‘s|.*/||’
# 同时提取（使用多个分组）
echo $path | sed -r ‘s|(.*/)([^/]*)$|\1 is DIR, \2 is FILE|’
```

**案例2：清理配置文件（删除注释和空行）**
这是一个经典用例，常用于处理像 `/etc/fstab` 这样的配置文件。
```bash
# 方法1：分步执行
sed ‘/^#/d’ /etc/fstab | sed ‘/^$/d’
# 方法2：使用 `-e` 指定多个脚本
sed -e ‘/^#/d’ -e ‘/^$/d’ /etc/fstab
# 方法3：使用分号分隔多个命令
sed ‘/^#/d； /^$/d’ /etc/fstab
```

**案例3：就地编辑与备份**
`-i` 选项允许`sed`直接修改源文件，这是一个危险但强大的操作。**务必谨慎使用，并养成备份的习惯。**
```bash
# 直接修改文件，不备份
sed -i ‘s/old/new/g’ file.txt
# 修改文件前先备份，备份文件后缀为.bak
sed -i.bak ‘s/old/new/g’ file.txt
# 先测试无误，再使用 -i
sed ‘s/old/new/g’ file.txt # 预览效果
sed -i ‘s/old/new/g’ file.txt # 确认后执行
```
