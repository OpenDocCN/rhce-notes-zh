## 一、Bash Shell基础回顾

上一节我们介绍了课程的整体结构，本节中我们来看看Bash Shell的基础概念。

![](img/abc8d8b49410af722eea0bee5c74d0d6_8.png)

### 1. 变量与赋值
变量是存储数据的容器。以下是关于变量的核心操作：
*   **命名规则**：变量名不能以数字开头，可以使用字母或下划线开头。
*   **变量取值**：使用 `$` 符号引用变量值，例如 `$variable_name`。
*   **数学运算**：可以使用 `let`、`$(( ))` 或 `declare -i` 进行数值计算。

![](img/abc8d8b49410af722eea0bee5c74d0d6_10.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_12.png)

### 2. 引号与转义
引号和转义字符用于处理字符串中的特殊含义。
*   **双引号**：允许变量和命令替换。
*   **单引号**：保留所有字符的字面值。
*   **反斜杠**：用于转义单个字符。

![](img/abc8d8b49410af722eea0bee5c74d0d6_14.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_16.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_18.png)

### 3. 变量类型
Shell中主要有两种变量类型：
*   **整型**：用于数学运算。
*   **字符串型**：用于文本处理和比较。

![](img/abc8d8b49410af722eea0bee5c74d0d6_20.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_22.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_23.png)

### 4. 特殊变量
以下是一些常用的特殊变量：
*   `$0`：脚本名称。
*   `$1`, `$2`...：脚本参数。
*   `$*`：所有参数作为一个整体。
*   `$@`：所有参数的列表。
*   `$#`：参数个数。
*   `$?`：上一条命令的退出状态码（0表示成功，非0表示失败）。

## 二、文本处理工具 🔧

![](img/abc8d8b49410af722eea0bee5c74d0d6_35.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_37.png)

掌握了变量基础后，我们来看看处理文本数据的强大工具。

![](img/abc8d8b49410af722eea0bee5c74d0d6_39.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_41.png)

### 1. `sed` 流编辑器
`sed` 用于对文本进行流式编辑，按行处理。
*   **基本格式**：`sed [选项] ‘模式/动作’ 文件名`
*   **常用动作**：
    *   `p`：打印行。
    *   `d`：删除行。
    *   `s/old/new/`：替换文本。
    *   `a\text`：在行后追加文本。
    *   `i\text`：在行前插入文本。
*   **重要提示**：默认只输出到屏幕，不修改原文件。使用 `-i` 选项会直接修改文件，操作前建议备份。

### 2. `grep` 文本搜索
`grep` 用于搜索包含特定模式的行。
以下是 `grep` 的常用选项：
*   `-i`：忽略大小写。
*   `-v`：反向选择，显示不匹配的行。
*   `-A n`：显示匹配行及其后n行。
*   `-B n`：显示匹配行及其前n行。
*   `-E`：使用扩展正则表达式，支持多条件匹配（条件间为“或”关系）。

### 3. `cut` 列截取工具
`cut` 用于从文件的每一行中截取部分内容。
*   **常用选项**：`-d‘分隔符’ -f字段编号`
*   **局限性**：分隔符只能识别单个字符，对于连续分隔符（如多个空格）处理不佳。

### 4. `awk` 文本分析工具
`awk` 更适合处理非标准格式的文本（如连续空格分隔）。
*   **基本格式**：`awk -F‘分隔符’ ‘模式 {动作}’ 文件名`
*   **优势**：能将连续的同类型分隔符视为一个，切割更精准。

## 三、条件判断与流程控制 ⚙️

文本处理工具能帮我们提取数据，而条件判断和流程控制则让脚本具备逻辑判断能力。

![](img/abc8d8b49410af722eea0bee5c74d0d6_43.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_45.png)

![](img/abc8d8b49410af722eea0bee5c74d0d6_47.png)

### 1. 条件判断表达式
判断通常在 `[ ]` 或 `[[ ]]` 中进行，**注意表达式前后必须有空格**。
以下是常见的判断条件：
*   **文件/目录判断**：
    *   `-e 文件`：文件/目录是否存在。
    *   `-f 文件`：是否为常规文件。
    *   `-d 文件`：是否为目录。
*   **字符串判断**：
    *   `-z 字符串`：字符串长度是否为0（空）。
    *   `-n 字符串`：字符串长度是否非0。
    *   `字符串1 == 字符串2`：字符串是否相等。
*   **数值比较**：
    *   `-eq`：等于。
    *   `-ne`：不等于。
    *   `-gt`：大于。
    *   `-lt`：小于。
    *   `-ge`：大于等于。
    *   `-le`：小于等于。

![](img/abc8d8b49410af722eea0bee5c74d0d6_49.png)

### 2. `if` 条件判断语句
`if` 语句用于基于条件执行不同代码块。
```bash
if [ 条件1 ]; then
    命令集1
elif [ 条件2 ]; then
    命令集2
else
    命令集3
fi
```
**关键点**：判断以命令的退出状态码 `$?` 为准（0为真，非0为假）。语句以 `fi` 结束。

### 3. `case` 多分支选择语句
`case` 语句用于匹配一个值与多个模式。
```bash
case $变量 in
模式1)
    命令集1
    ;;
模式2)
    命令集2
    ;;
*)
    默认命令集
    ;;
esac
```
**关键点**：每个模式以 `)` 结束，命令集以 `;;` 结束。`*)` 是默认情况。语句以 `esac` 结束。

### 4. 循环语句
循环用于重复执行一段代码。
*   **`for` 循环**：遍历列表。
    ```bash
    for 变量 in 列表; do
        命令集
    done
    ```
*   **`while` 循环**：当条件为真时执行。
    ```bash
    while [ 条件 ]; do
        命令集
    done
    ```
*   **`until` 循环**：当条件为假时执行（与 `while` 相反）。
    ```bash
    until [ 条件 ]; do
        命令集
    done
    ```

## 四、计划任务 ⏰

最后，我们回顾如何让系统在特定时间自动执行任务。

### 1. 一次性计划任务 `at`
`at` 用于安排在未来某一时刻执行一次任务。
*   **基本用法**：`at 时间`
*   **时间格式**：可以是具体时间点（如 `14:30`），也可以是相对时间（如 `now + 3 hours`）。

### 2. 周期性计划任务 `cron`
`cron` 用于安排周期性执行的任务。
*   **编辑任务**：使用 `crontab -e`。
*   **查看任务**：使用 `crontab -l`。
*   **格式说明**：`分 时 日 月 周 命令`
    *   `*`：代表任何值。
    *   `,`：指定多个值（如 `2,6,10` 表示2点、6点、10点）。
    *   `-`：指定范围（如 `1-5` 表示1到5）。
    *   `/`：指定间隔（如 `*/2` 表示每2单位）。
*   **系统级任务**：可以放置在 `/etc/cron.hourly/`, `daily/`, `weekly/`, `monthly/` 目录下，由系统按小时、天、周、月周期执行（可能有几分钟延迟）。
