在本节课中，我们将要学习Shell脚本中一个常见的概念：自增运算符 `i++` 和 `++i` 的区别。虽然在实际脚本编写中，它们对变量本身值的影响通常没有区别，但理解其底层原理有助于我们更深入地掌握变量运算。

上一节我们介绍了变量的多种运算方式，本节中我们来看看自增运算符的细微差别。

## 对变量值的影响

![](img/81c59632de6d4965f1d4210ef44d8076_6.png)

首先，我们来看 `i++` 和 `++i` 对变量**自身值**的影响。结论是：**两者对变量值的最终影响完全相同**。

![](img/81c59632de6d4965f1d4210ef44d8076_7.png)

![](img/81c59632de6d4965f1d4210ef44d8076_9.png)

以下是验证步骤：
1.  定义变量 `i` 并赋值为1。
2.  使用 `let` 命令分别执行 `i++` 和 `++i`。
3.  观察执行后变量 `i` 的值。

```bash
# 定义变量 i 和 j
i=1
j=1

![](img/81c59632de6d4965f1d4210ef44d8076_11.png)

![](img/81c59632de6d4965f1d4210ef44d8076_13.png)

# 使用 i++ 自增
let i++
echo $i  # 输出结果为 2

![](img/81c59632de6d4965f1d4210ef44d8076_15.png)

![](img/81c59632de6d4965f1d4210ef44d8076_17.png)

# 使用 ++j 自增
let ++j
echo $j  # 输出结果同样为 2
```

![](img/81c59632de6d4965f1d4210ef44d8076_19.png)

![](img/81c59632de6d4965f1d4210ef44d8076_21.png)

从以上操作可以看出，无论是 `i++` 还是 `++i`，执行后变量自身的值都从1变成了2。因此，在仅需要改变变量值而不关心表达式结果的场景下（例如循环中的计数器递增），两者可以互换使用。

![](img/81c59632de6d4965f1d4210ef44d8076_22.png)

![](img/81c59632de6d4965f1d4210ef44d8076_24.png)

## 对表达式结果的影响

![](img/81c59632de6d4965f1d4210ef44d8076_26.png)

然而，当我们将自增操作放在一个**表达式**中，并试图获取该表达式的值时，`i++` 和 `++i` 的行为就产生了关键区别。

![](img/81c59632de6d4965f1d4210ef44d8076_28.png)

以下是核心区别：
*   **`i++`**：**先赋值，后自增**。整个表达式的值是变量自增**之前**的值。
*   **`++i`**：**先自增，后赋值**。整个表达式的值是变量自增**之后**的值。

![](img/81c59632de6d4965f1d4210ef44d8076_30.png)

![](img/81c59632de6d4965f1d4210ef44d8076_32.png)

让我们通过一个实验来清晰地展示这个区别：

![](img/81c59632de6d4965f1d4210ef44d8076_34.png)

![](img/81c59632de6d4965f1d4210ef44d8076_36.png)

```bash
# 取消变量定义，重新开始
unset i j x y

![](img/81c59632de6d4965f1d4210ef44d8076_38.png)

# 重新赋值
i=1
j=1

![](img/81c59632de6d4965f1d4210ef44d8076_40.png)

# 将 i++ 的表达式结果赋值给 x
let x=i++
echo “x = $x”  # 输出：x = 1
echo “i = $i”  # 输出：i = 2

# 将 ++j 的表达式结果赋值给 y
let y=++j
echo “y = $y”  # 输出：y = 2
echo “j = $j”  # 输出：j = 2
```

![](img/81c59632de6d4965f1d4210ef44d8076_42.png)

分析以上结果：
1.  `let x=i++`：先将 `i` 的原始值（1）赋给表达式，即 `x=1`，然后 `i` 自身再执行自增变为2。
2.  `let y=++j`：先对 `j` 执行自增变为2，然后将自增后的值（2）赋给表达式，即 `y=2`。

![](img/81c59632de6d4965f1d4210ef44d8076_44.png)

![](img/81c59632de6d4965f1d4210ef44d8076_46.png)

所以，虽然最终 `i` 和 `j` 的值都变成了2，但表达式赋值的结果 `x` 和 `y` 却不同。

## 实际应用场景探讨

![](img/81c59632de6d4965f1d4210ef44d8076_48.png)

![](img/81c59632de6d4965f1d4210ef44d8076_50.png)

那么，这个区别在Shell脚本编程中有实际用途吗？答案是：**极少**。

![](img/81c59632de6d4965f1d4210ef44d8076_52.png)

一个可能涉及到的场景是在定义数组下标时：
*   使用 `arr[i++]` 可能会从下标0开始赋值（先取 `i` 的值0，再自增）。
*   使用 `arr[++i]` 可能会从下标1开始赋值（先自增 `i` 为1，再取值）。

然而，在Shell脚本中，我们通常直接通过索引来操作数组，并不特别关心下标是从0还是1开始，因此这个区别的影响微乎其微。对于大多数情况，包括在 `while` 或 `for` 循环中作为计数器，你使用 `i++` 或 `++i` 都是可以的，因为它们对变量 `i` 的最终影响一致。

![](img/81c59632de6d4965f1d4210ef44d8076_54.png)

## 关于未定义变量的情况

![](img/81c59632de6d4965f1d4210ef44d8076_56.png)

最后，补充一个知识点：如果对一个未定义的变量使用自增运算符，Shell会将其初始值视为0。

![](img/81c59632de6d4965f1d4210ef44d8076_57.png)

```bash
unset i
let i++
echo $i  # 输出：1

unset j
let ++j
echo $j  # 输出：1
```

本节课中我们一起学习了 `i++` 和 `++i` 的区别。总结如下：
1.  **对变量自身**：两者效果相同，都是将变量值加1。
2.  **对表达式结果**：`i++` 返回加1前的值，`++i` 返回加1后的值。这是最核心的区别。
3.  **实际应用**：在Shell脚本编程中，这个区别很少产生影响，你可以根据习惯选择使用哪一种。

![](img/81c59632de6d4965f1d4210ef44d8076_59.png)

理解这个概念有助于你阅读更复杂的代码，但在自己编写脚本时，无需过度纠结。
