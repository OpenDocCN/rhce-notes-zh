## 命令行概述

上一节我们介绍了课程的整体安排，本节中我们来看看什么是命令行以及它的核心概念。

命令行是管理员输入的一串字符，用于完成特定的系统任务。用户通过按回车键提交这串字符，系统便会执行相应的操作。例如，命令 `ip address list` 用于查看网络接口的地址信息。

Linux系统能够理解这些命令，是因为存在一个特殊的程序——**解释器**。解释器负责将用户提交的、人类可读的指令，翻译成操作系统内核能够理解和执行的机器指令。在Linux中，这个解释器通常被称为 **Shell**（外壳程序）。

**Shell** 是包裹在操作系统**内核**（Kernel）外的一层程序。内核是操作系统的核心，直接管理CPU、内存、磁盘等硬件。而Shell作为用户与内核之间的桥梁，使得用户无需直接操作复杂的机器指令。

## Linux目录结构

![](img/440952c40b93dd1372c7621388f09f46_22.png)

![](img/440952c40b93dd1372c7621388f09f46_24.png)

![](img/440952c40b93dd1372c7621388f09f46_26.png)

理解了命令行的基本概念后，我们需要熟悉Linux系统的“地图”——目录结构。Linux采用树形结构来组织文件，这与Windows类似，但具体路径表示和顶级目录有所不同。

在Linux命令行中，路径的层次使用**斜杠（/）** 来分隔。同时，单个斜杠 **`/`** 也表示整个文件系统的起点，即**根目录**。所有其他目录和文件都位于根目录之下。

![](img/440952c40b93dd1372c7621388f09f46_28.png)

![](img/440952c40b93dd1372c7621388f09f46_30.png)

以下是系统一些常见的一级目录及其用途简介：

*   **`/bin`** 与 **`/sbin`**：存放可执行程序文件。`/sbin`下的程序通常需要管理员（root）权限才能执行。
*   **`/dev`**：存放各种硬件设备的文件接口。
*   **`/home`**：普通用户的主目录所在地。例如，用户`zhangsan`的主目录通常是`/home/zhangsan`。
*   **`/root`**：系统管理员（root）用户的主目录。
*   **`/boot`**：存放系统启动所需的文件，包括内核。切勿随意删除此目录。
*   **`/etc`**：存放系统的各种配置文件。
*   **`/var`**：存放经常变化的（Variable）数据，如系统日志、邮件等。
*   **`/tmp`**：存放临时文件。
*   **`/mnt`** 与 **`/media`**：用于挂载外部存储设备（如U盘、光盘）。`/mnt`通常用于管理员手动挂载，`/media`用于系统自动识别并挂载。

## 命令格式与快捷键

熟悉了系统结构，我们来看看如何高效地书写和输入命令。

一条完整的Linux命令通常由三部分组成：**命令字**、**选项**和**参数**。其基本格式如下：
```
命令字 [选项] [参数]
```
*   **命令字**：指定要执行的操作，是必须的部分。
*   **选项**：用于调节命令的具体执行方式或效果，通常以短横线`-`开头。多个单字母选项可以合并，例如 `-l -h` 可以写成 `-lh`。
*   **参数**：指定命令操作的对象，如文件或目录路径。

以 `ls` 命令为例：
*   `ls`：仅列出当前目录下的文件名。
*   `ls -l`：以长格式列出文件的详细信息（如权限、大小、时间）。
*   `ls -lh /home`：以易读的格式（如KB，MB）列出`/home`目录下文件的详细信息。

为了提升操作效率，Linux命令行提供了一些实用的快捷键：

*   **Tab键**：命令或路径补全。输入部分字符后按Tab，系统会自动补全唯一可能的结果，或列出所有可能的选择。
*   **Ctrl + L**：清空当前终端屏幕，效果等同于执行 `clear` 命令。
*   **Ctrl + C**：终止当前正在运行的前台任务。
*   **Esc + .**：快速粘贴上一条命令的最后一个参数到当前光标位置。
