在本教程中，我们将学习如何在Linux系统上配置iSCSI服务器（Target）和客户端（Initiator）。iSCSI是一种基于IP网络的存储协议，允许客户端通过网络访问远程存储设备，如同访问本地硬盘一样。我们将从安装软件包开始，逐步完成服务端配置、客户端发现与连接，最终实现远程磁盘的挂载和使用。

---

## 服务器端配置 🖥️

上一节我们介绍了iSCSI的基本概念，本节中我们来看看如何在服务器端进行配置。

![](img/39660e4c92330b6323ab7fd2fe3fcb79_9.png)

### 安装软件包

![](img/39660e4c92330b6323ab7fd2fe3fcb79_11.png)

![](img/39660e4c92330b6323ab7fd2fe3fcb79_13.png)

![](img/39660e4c92330b6323ab7fd2fe3fcb79_14.png)

首先，需要在服务器上安装提供iSCSI Target功能的软件包。

![](img/39660e4c92330b6323ab7fd2fe3fcb79_15.png)

![](img/39660e4c92330b6323ab7fd2fe3fcb79_17.png)

![](img/39660e4c92330b6323ab7fd2fe3fcb79_19.png)

```bash
yum install -y targetcli
```

### 配置iSCSI Target

安装完成后，使用 `targetcli` 工具来创建后端存储、iSCSI Target和LUN。

![](img/39660e4c92330b6323ab7fd2fe3fcb79_21.png)

![](img/39660e4c92330b6323ab7fd2fe3fcb79_22.png)

1.  进入 `targetcli` 交互式配置界面。
    ```bash
    targetcli
    ```
2.  创建后端存储块。这相当于创建一个虚拟磁盘文件。
    ```
    /> backstores/block create disk1 /dev/sdb
    ```
3.  创建iSCSI Target。系统会自动生成一个唯一的iSCSI限定名称（IQN）。
    ```
    /> iscsi/ create iqn.2020-06.com.example:server
    ```
4.  为刚创建的Target添加一个LUN（逻辑单元），将之前创建的存储块关联上去。
    ```
    /> iscsi/iqn.2020-06.com.example:server/tpg1/luns create /backstores/block/disk1
    ```
5.  设置访问控制列表（ACL），允许指定的客户端IQN进行连接。客户端的IQN需要预先确定。
    ```
    /> iscsi/iqn.2020-06.com.example:server/tpg1/acls create iqn.2020-06.com.example:client
    ```
6.  设置监听地址和端口，使服务在服务器的IP上监听连接。
    ```
    /> iscsi/iqn.2020-06.com.example:server/tpg1/portals create 172.25.0.10 3260
    ```
7.  配置完成后，输入 `exit` 退出。配置会自动保存。

![](img/39660e4c92330b6323ab7fd2fe3fcb79_24.png)

### 配置防火墙与启动服务

![](img/39660e4c92330b6323ab7fd2fe3fcb79_26.png)

为了让客户端能够访问，需要开放防火墙的相应端口，并确保服务开机自启。

以下是需要执行的命令：
```bash
# 开放iSCSI默认端口3260
firewall-cmd --permanent --add-port=3260/tcp
firewall-cmd --reload

![](img/39660e4c92330b6323ab7fd2fe3fcb79_28.png)

# 启动并启用target服务
systemctl start target
systemctl enable target
```

## 客户端配置 💻

![](img/39660e4c92330b6323ab7fd2fe3fcb79_35.png)

![](img/39660e4c92330b6323ab7fd2fe3fcb79_37.png)

上一节我们完成了服务器的搭建，本节中我们来看看客户端如何发现并连接服务器提供的存储。

![](img/39660e4c92330b6323ab7fd2fe3fcb79_39.png)

### 安装客户端软件包

![](img/39660e4c92330b6323ab7fd2fe3fcb79_41.png)

在客户端机器上，需要安装iSCSI Initiator工具。

```bash
yum install -y iscsi-initiator-utils
```

### 配置Initiator名称

![](img/39660e4c92330b6323ab7fd2fe3fcb79_43.png)

客户端需要有一个唯一的IQN，并需要与服务器ACL中允许的IQN一致。

![](img/39660e4c92330b6323ab7fd2fe3fcb79_45.png)

1.  编辑客户端initiator名称配置文件。
    ```bash
    vim /etc/iscsi/initiatorname.iscsi
    ```
2.  将内容修改为服务器ACL中允许的IQN。
    ```
    InitiatorName=iqn.2020-06.com.example:client
    ```
3.  保存并退出后，重启iscsi服务使配置生效。
    ```bash
    systemctl restart iscsid
    systemctl enable iscsid
    ```

![](img/39660e4c92330b6323ab7fd2fe3fcb79_47.png)

### 发现与登录Target

现在，客户端可以尝试发现服务器上的Target并建立连接。

![](img/39660e4c92330b6323ab7fd2fe3fcb79_49.png)

1.  使用 `iscsiadm` 命令发现指定服务器IP上的Target。
    ```bash
    iscsiadm -m discovery -t st -p 172.25.0.10
    ```
    如果成功，将显示服务器的Target IQN。
2.  登录到发现的Target。
    ```bash
    iscsiadm -m node -T iqn.2020-06.com.example:server -p 172.25.0.10 -l
    ```

### 识别并使用新磁盘

登录成功后，客户端系统会识别到一个新的磁盘设备（例如 `/dev/sdc`）。

以下是使用新磁盘的步骤：
1.  **查看新磁盘**：使用 `lsblk` 或 `fdisk -l` 命令确认新磁盘已被识别。
2.  **分区**：使用 `fdisk` 或 `parted` 工具对新磁盘进行分区。
    ```bash
    fdisk /dev/sdc
    # 在交互界面中创建新分区
    ```
3.  **格式化**：使用 `mkfs` 命令对分区进行格式化。
    ```bash
    mkfs.ext4 /dev/sdc1
    ```
4.  **挂载使用**：创建挂载点，并将新分区挂载上去。
    ```bash
    mkdir /mnt/iscsi_data
    mount /dev/sdc1 /mnt/iscsi_data
    ```
5.  **实现开机自动挂载**：获取分区的UUID，并将其写入 `/etc/fstab` 文件。
    ```bash
    blkid /dev/sdc1
    # 复制输出的UUID
    echo "UUID=你的-UUID /mnt/iscsi_data ext4 _netdev 0 0" >> /etc/fstab
    ```
    **注意**：iSCSI磁盘需要网络，因此挂载选项必须包含 `_netdev`。

## 总结 📝

本节课中我们一起学习了iSCSI存储网络的完整配置流程。

我们首先在服务器端使用 `targetcli` 工具创建了存储资源、iSCSI Target和访问控制。随后，在客户端配置了Initiator名称，通过发现和登录操作连接到了远程存储。最后，我们对远程磁盘进行了分区、格式化、挂载和自动挂载配置，使其可以像本地磁盘一样使用。
