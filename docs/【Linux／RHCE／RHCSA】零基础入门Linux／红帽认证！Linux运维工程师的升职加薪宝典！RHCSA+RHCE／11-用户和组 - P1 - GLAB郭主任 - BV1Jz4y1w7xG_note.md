在本节课中，我们将要学习Linux系统中用户和组管理的基础知识。操作系统是一个多用户系统，允许多个用户同时使用，并通过用户和组来分配不同的权限，实现安全隔离。

## 用户基础概念

![](img/198c0890c81ff7a3e16c143981c4e39c_8.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_10.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_13.png)

上一节我们介绍了课程概述，本节中我们来看看用户的基本概念。

![](img/198c0890c81ff7a3e16c143981c4e39c_15.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_17.png)

用户的账户用于运行命令，并在不同的人员和程序之间提供安全界限。这意味着不同用户在系统上的操作在逻辑上是隔离的。在系统外部，我们使用用户名来区分用户；而在系统内部，则使用UID（用户ID）作为唯一标识。通常，每个用户都需要分配密码来验证其登录权限。

系统中的每个进程都以特定用户的身份运行，每个文件也都有一个特定的用户作为其所有者。这种所有权控制有助于实施系统安全访问策略。

![](img/198c0890c81ff7a3e16c143981c4e39c_19.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_21.png)

用户主要分为三种类型：
*   **超级用户**：即系统管理员 `root`，UID为0，拥有对系统的完全访问权限。
*   **系统用户**：通常不用于交互式登录，而是为系统服务或进程提供运行身份，以实现相互隔离。
*   **普通用户**：对系统拥有有限的访问权限。

![](img/198c0890c81ff7a3e16c143981c4e39c_23.png)

我们可以通过 `id` 命令查看当前登录用户的UID等信息。
```bash
id
```
通过查看 `/etc/passwd` 文件，可以列出系统上的所有用户。
```bash
cat /etc/passwd
```
在该文件中，`root` 用户排在首位，其UID为0。其他如 `bin`、`daemon` 等用户通常是系统用户。

## 理解 `/etc/passwd` 文件

![](img/198c0890c81ff7a3e16c143981c4e39c_37.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_39.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_41.png)

上一节我们了解了用户的类型，本节中我们来看看存储用户信息的核心文件。

`/etc/passwd` 文件中的每一行代表一个用户，由冒号分隔为7个字段。以下是每个字段的含义：

1.  **用户名**：用户登录时使用的名称。
2.  **密码**：历史上密码存储在此字段，现在通常显示为 `x`，表示密码已移至 `/etc/shadow` 文件。
3.  **UID**：用户ID，系统识别用户的唯一标识。
4.  **GID**：主组ID，用户所属主要组的唯一标识。
5.  **描述**：用户的真实姓名或备注信息。
6.  **家目录**：用户登录后进入的默认工作目录。
7.  **登录Shell**：用户登录后使用的默认Shell程序（如 `/bin/bash`）。系统用户的Shell常设置为 `/sbin/nologin`。

![](img/198c0890c81ff7a3e16c143981c4e39c_43.png)

## 创建与管理用户

上一节我们解析了用户配置文件，本节中我们来学习如何创建和管理用户。

![](img/198c0890c81ff7a3e16c143981c4e39c_45.png)

创建用户的基本命令是 `useradd`。如果不指定任何选项，系统会使用默认值创建用户。
```bash
useradd glab1
```
创建后，可以查看 `/etc/passwd` 文件确认用户已添加，并且系统会在 `/home` 目录下自动创建同名家目录。

![](img/198c0890c81ff7a3e16c143981c4e39c_47.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_49.png)

`useradd` 命令支持多个选项来自定义用户属性：
*   `-u`：指定用户的UID。
*   `-g`：指定用户的主组GID。
*   `-c`：指定用户的描述/注释。
*   `-d`：指定用户的家目录路径。
*   `-s`：指定用户登录时使用的Shell。
*   `-M`：不创建用户家目录（常用于创建系统用户）。

以下是使用选项创建用户的示例：
```bash
useradd glab2 -u 1234 -c "This is a test user" -d /tmp/glab2 -s /sbin/nologin
```

## 用户密码管理

上一节我们学会了创建用户，本节中我们来看看如何为用户设置和管理密码。

![](img/198c0890c81ff7a3e16c143981c4e39c_51.png)

新创建的用户默认没有密码，需要使用 `passwd` 命令来设置。
```bash
passwd glab1
```
然后根据提示输入两次密码即可。

`passwd` 命令还包含其他管理功能：
*   `-l`：锁定用户密码，禁止该用户登录。
*   `-u`：解锁用户密码。
*   `-d`：删除用户密码，允许无密码登录。
*   `-n`：设置密码的最短有效天数。
*   `-x`：设置密码的最长有效天数。

![](img/198c0890c81ff7a3e16c143981c4e39c_53.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_55.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_57.png)

例如，设置用户 `glab3` 的密码最短10天后才能修改，最长20天后必须修改：
```bash
passwd glab3 -n 10 -x 20
```
用户密码信息实际存储在 `/etc/shadow` 文件中。该文件同样由冒号分隔字段，其中包含加密的密码、上次修改日期、密码有效期、警告期等信息。

![](img/198c0890c81ff7a3e16c143981c4e39c_59.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_61.png)

## 修改与删除用户

![](img/198c0890c81ff7a3e16c143981c4e39c_63.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_65.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_67.png)

上一节我们介绍了密码管理，本节中我们来看看如何修改用户属性和删除用户。

![](img/198c0890c81ff7a3e16c143981c4e39c_69.png)

修改现有用户属性使用 `usermod` 命令，其许多选项与 `useradd` 类似。
```bash
usermod glab2 -u 2345 -c "Updated description"
```
`usermod` 还有一些特有选项：
*   `-L`：锁定用户账户（与 `passwd -l` 效果类似，但使用大写 `L`）。
*   `-U`：解锁用户账户。
*   `-d -m`：指定新的家目录，并将旧家目录内容移动到新位置。这两个选项需分开使用。
    ```bash
    usermod glab1 -d /home/glab11 -m
    ```
*   `-aG`：为用户追加新的附加组（组的概念将在后面详细说明）。

![](img/198c0890c81ff7a3e16c143981c4e39c_71.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_73.png)

删除用户使用 `userdel` 命令。
```bash
userdel glab1
```
默认情况下，`userdel` 不会删除用户的家目录。这可能导致问题：如果后续新创建的用户被分配了相同的UID，该用户将自动获得旧家目录的所有权。有两种解决方案：
1.  删除用户时使用 `-r` 选项，同时删除其家目录。
    ```bash
    userdel -r glab3
    ```
2.  创建用户时，习惯性地使用 `-u` 手动指定唯一的UID，避免UID冲突。

![](img/198c0890c81ff7a3e16c143981c4e39c_75.png)

## 用户组管理

![](img/198c0890c81ff7a3e16c143981c4e39c_77.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_79.png)

上一节我们完成了对用户操作的讲解，本节中我们来看看组的管理。

![](img/198c0890c81ff7a3e16c143981c4e39c_81.png)

组是用户的集合，系统通过GID来区分不同的组。将权限授予一个组，就等于授予该组内的所有用户，这比单独为每个用户授权更高效。组信息存储在 `/etc/group` 文件中。

创建用户时，系统会自动创建一个与用户名同名的组作为该用户的**主要组**（或称为基本组）。用户还可以属于零个、一个或多个**附加组**。

关于主要组和附加组，需要理解以下几点：
*   一个用户只能属于一个主要组。
*   一个用户可以属于零个、一个或多个附加组。
*   一个组（无论是主要组还是附加组）可以包含多个用户。

![](img/198c0890c81ff7a3e16c143981c4e39c_83.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_85.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_87.png)

![](img/198c0890c81ff7a3e16c143981c4e39c_88.png)

使用 `groupadd` 命令可以创建新组。
```bash
groupadd developers
```
使用 `usermod` 命令的 `-aG` 选项可以为用户添加附加组。
```bash
usermod -aG developers glab2
```
使用 `id` 命令可以查看用户所属的组。
```bash
id glab2
```
输出中，`groups` 后面的第一个是主要组，其后的是附加组。
