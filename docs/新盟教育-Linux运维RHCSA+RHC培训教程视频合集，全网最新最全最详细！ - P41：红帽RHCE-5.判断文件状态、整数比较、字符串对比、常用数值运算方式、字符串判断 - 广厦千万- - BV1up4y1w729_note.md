## 判断文件状态 📂

上一节我们介绍了Shell的基础命令，本节中我们来看看如何判断文件的状态。Shell提供了一系列以单个字母开头的选项，用于在条件判断中检查文件的各种属性。

![](img/5754e0381cbf6ba1da2144f491453ef6_6.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_8.png)

以下是常用的文件状态判断选项及其用法：

![](img/5754e0381cbf6ba1da2144f491453ef6_9.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_11.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_13.png)

*   **`-e`**：判断文件或目录是否存在。
    *   示例：`[ -e /etc/passwd ] && echo "文件存在"`
*   **`-d`**：判断路径是否为目录。
    *   示例：`[ -d /etc ] && echo "这是一个目录"`
*   **`-f`**：判断路径是否为普通文件。
    *   示例：`[ -f /etc/passwd ] && echo "这是一个文件"`
*   **`-r`**：判断当前用户对文件是否具有读权限。
    *   示例：`[ -r /etc/passwd ] && echo "可读"`
*   **`-w`**：判断当前用户对文件是否具有写权限。
    *   示例：`[ -w /etc/passwd ] && echo "可写"`
*   **`-x`**：判断当前用户对文件是否具有执行权限。
    *   示例：`[ -x /etc/passwd ] && echo "可执行"`

**注意**：使用这些选项时，必须放在方括号 `[ ]` 内，并且选项、路径和方括号之间都需要有空格。命令执行后默认不显示结果，可以通过 `$?` 获取上一条命令的退出状态码（0表示成功/真，非0表示失败/假）。

```bash
[ -e /etc/passwd ]
echo $? # 输出 0 表示文件存在
```

## 整数比较 🔢

了解了对文件的判断后，我们来看看如何对数字进行比较。Shell中进行**整数比较**需要使用特定的操作符，不能直接使用数学符号（如 `=`、`>`），否则会被当作字符串处理。

![](img/5754e0381cbf6ba1da2144f491453ef6_15.png)

以下是整数比较的操作符：

![](img/5754e0381cbf6ba1da2144f491453ef6_17.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_19.png)

*   **`-eq`**：等于（Equal）
    *   示例：`[ 1 -eq 1 ] && echo "相等"`
*   **`-ne`**：不等于（Not Equal）
    *   示例：`[ 2 -ne 1 ] && echo "不相等"`
*   **`-gt`**：大于（Greater Than）
    *   示例：`[ 2 -gt 1 ] && echo "大于"`
*   **`-ge`**：大于等于（Greater or Equal）
    *   示例：`[ 1 -ge 1 ] && echo "大于等于"`
*   **`-lt`**：小于（Less Than）
    *   示例：`[ 1 -lt 2 ] && echo "小于"`
*   **`-le`**：小于等于（Less or Equal）
    *   示例：`[ 1 -le 1 ] && echo "小于等于"`

**语法格式**：`[ 数值1 操作符 数值2 ]`

![](img/5754e0381cbf6ba1da2144f491453ef6_21.png)

```bash
[ 10 -gt 5 ]
echo $? # 输出 0，表示判断为真（10大于5）
```

## 字符串对比 🔤

![](img/5754e0381cbf6ba1da2144f491453ef6_25.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_27.png)

与整数比较不同，字符串对比使用的是我们更熟悉的符号。本节我们来学习字符串的对比方法。

![](img/5754e0381cbf6ba1da2144f491453ef6_29.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_31.png)

字符串对比主要使用以下操作符：

*   **`=` 或 `==`**：判断两个字符串是否相等。
    *   示例：`[ "$USER" = "root" ] && echo "当前用户是root"`
*   **`!=`**：判断两个字符串是否不相等。
    *   示例：`[ "hello" != "world" ] && echo "字符串不同"`

![](img/5754e0381cbf6ba1da2144f491453ef6_33.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_35.png)

**重要提示**：字符串对比时，变量建议用双引号引起来，防止变量值为空时导致语法错误。`=` 和 `==` 在 `[ ]` 测试中功能相同。

![](img/5754e0381cbf6ba1da2144f491453ef6_37.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_39.png)

```bash
name="Alice"
[ "$name" = "Alice" ] && echo "名字匹配"
```

## 常用数值运算方式 ➕➖✖️➗

![](img/5754e0381cbf6ba1da2144f491453ef6_45.png)

在脚本中，我们经常需要进行数学运算。Shell提供了多种数值运算的方法，各有特点。

![](img/5754e0381cbf6ba1da2144f491453ef6_47.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_49.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_50.png)

以下是几种常用的数值运算方法：

1.  **`$[ ]` 或 `$(( ))` 表达式**：这是最常用、最简洁的方式。
    *   示例：
        ```bash
        echo $[ 5 + 3 ]     # 输出 8
        echo $(( 10 / 2 ))  # 输出 5
        echo $(( 10 % 3 ))  # 输出 1 (求余)
        ```

![](img/5754e0381cbf6ba1da2144f491453ef6_52.png)

2.  **`expr` 命令**：一种较老的运算方式，操作符两侧需要空格，部分符号（如 `*`）需要转义。
    *   示例：
        ```bash
        expr 5 + 3
        expr 5 \* 3  # 乘法符号需要转义
        ```

![](img/5754e0381cbf6ba1da2144f491453ef6_54.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_56.png)

3.  **`let` 命令**：通常用于将运算结果赋值给变量，支持 `++`、`--`、`+=`、`-=` 等简洁的自增/自减操作。
    *   示例：
        ```bash
        let a=5+3
        echo $a  # 输出 8
        
        let a++
        echo $a  # 输出 9
        
        let a+=10
        echo $a  # 输出 19
        ```

![](img/5754e0381cbf6ba1da2144f491453ef6_58.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_60.png)

对于初学者，推荐优先掌握 `$(( ))` 进行算术运算。

![](img/5754e0381cbf6ba1da2144f491453ef6_62.png)

## 字符串判断（空值判断）⚖️

除了对比字符串内容，判断一个字符串变量是否为空也是常见操作。本节我们学习空值判断。

字符串空值判断主要使用以下两个选项：

![](img/5754e0381cbf6ba1da2144f491453ef6_64.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_66.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_68.png)

*   **`-z`**：判断字符串长度是否为零（Zero）。如果为空，则返回真（0）。
    *   示例：`[ -z "$var" ] && echo "变量为空"`
*   **`-n`**：判断字符串长度是否非零（Non-zero）。如果不为空，则返回真（0）。
    *   示例：`[ -n "$var" ] && echo "变量不为空"`

![](img/5754e0381cbf6ba1da2144f491453ef6_70.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_72.png)

```bash
empty_str=""
non_empty_str="Hello"

[ -z "$empty_str" ] && echo "这是空字符串"      # 会执行
[ -n "$non_empty_str" ] && echo "这不是空字符串" # 会执行
```

## 条件判断符号（命令连接符）⛓️

![](img/5754e0381cbf6ba1da2144f491453ef6_74.png)

掌握了基本的判断和运算后，我们来看看如何将多个命令根据逻辑关系组合起来执行。这通过条件判断符号（也称命令连接符）实现。

![](img/5754e0381cbf6ba1da2144f491453ef6_76.png)

以下是三个重要的命令连接符：

1.  **`&&`（逻辑与）**：只有**前面**的命令执行成功（退出状态码为0），**后面**的命令才会执行。
    *   **场景**：确保上一步成功后再执行下一步。
    *   示例：`[ -f /etc/passwd ] && cp /etc/passwd /opt/` （如果文件存在，则拷贝它）

![](img/5754e0381cbf6ba1da2144f491453ef6_78.png)

![](img/5754e0381cbf6ba1da2144f491453ef6_80.png)

2.  **`||`（逻辑或）**：只有**前面**的命令执行失败（退出状态码非0），**后面**的命令才会执行。
    *   **场景**：上一步失败时执行补救操作。
    *   示例：`rpm -q vsftpd || yum install -y vsftpd` （如果查询不到vsftpd包，则安装它）

3.  **`;`（分号）**：按顺序执行多个命令，**无论前面命令成功与否**，后面的命令都会执行。命令之间没有逻辑依赖关系。
    *   **场景**：希望一次性顺序执行多个独立命令。
    *   示例：`cd /opt; ls; pwd` （切换到/opt目录，列出内容，然后显示当前路径）

![](img/5754e0381cbf6ba1da2144f491453ef6_82.png)

**组合使用示例**：模拟一个安装服务的流程。
```bash
# 如果软件包未安装，则安装它；安装成功后，启动并设置开机自启
rpm -q vsftpd || yum install -y vsftpd && systemctl start vsftpd && systemctl enable vsftpd
```
**逻辑解释**：`||` 左侧查询失败则执行右侧安装；`&&` 确保安装成功后才会启动服务，启动成功后再设置自启。
