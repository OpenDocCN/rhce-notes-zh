在本节课中，我们将要学习Linux系统中关于进程管理的核心操作，包括如何查看进程状态、进行进程的前后台调度、杀死进程以及分析用户的登录情况。这些技能对于日常的系统维护和故障排查至关重要。

## 进程状态监控：top命令详解 📊

上一节我们介绍了使用`ps`命令查看进程快照。本节中我们来看看一个动态监控进程的工具——`top`命令。`top`命令可以实时显示系统中各个进程的资源占用状况，类似于Windows的任务管理器。

执行`top`命令后，界面主要分为几个部分：

![](img/1865fdbb20644e85be2e679e6f98192f_6.png)

![](img/1865fdbb20644e85be2e679e6f98192f_8.png)

![](img/1865fdbb20644e85be2e679e6f98192f_10.png)

![](img/1865fdbb20644e85be2e679e6f98192f_12.png)

**第一行：系统概况**
*   **当前系统时间**：例如 `10:13:16`。
*   **系统运行时间**：例如 `up 1 day, 2:30`，表示系统已运行1天2小时30分钟。
*   **当前登录用户数**：例如 `2 users`。
*   **系统平均负载**：显示过去1、5、15分钟的系统平均负载，例如 `load average: 0.00, 0.01, 0.05`。

![](img/1865fdbb20644e85be2e679e6f98192f_14.png)

![](img/1865fdbb20644e85be2e679e6f98192f_16.png)

![](img/1865fdbb20644e85be2e679e6f98192f_18.png)

![](img/1865fdbb20644e85be2e679e6f98192f_20.png)

**第二行：进程概况**
*   **总进程数**：`total`
*   **运行中的进程数**：`running`
*   **休眠中的进程数**：`sleeping`
*   **停止的进程数**：`stopped`
*   **僵尸进程数**：`zombie`

![](img/1865fdbb20644e85be2e679e6f98192f_22.png)

![](img/1865fdbb20644e85be2e679e6f98192f_24.png)

僵尸进程是指已经终止但其父进程尚未对其进行善后处理的进程。如果发现僵尸进程数量不为0，可以使用`ps aux | grep defunct`找到其父进程ID（PPID），然后结束父进程来清理。

![](img/1865fdbb20644e85be2e679e6f98192f_26.png)

![](img/1865fdbb20644e85be2e679e6f98192f_27.png)

**第三行：CPU状态**
*   **用户空间占用CPU百分比**：`us`
*   **内核空间占用CPU百分比**：`sy`
*   **用户进程优先级改变占用CPU百分比**：`ni`
*   **空闲CPU百分比**：`id` **（重点关注）**
*   等待I/O的CPU时间百分比：`wa`
*   硬中断占用百分比：`hi`
*   软中断占用百分比：`si`
*   虚拟机占用百分比：`st`

`id`值表示CPU的闲置率。如果这个值持续很低（例如低于10%），则表明CPU非常繁忙，需要排查是哪个进程消耗了大量资源。

**第四、五行：内存与交换分区**
这部分信息以KB为单位显示，不如使用`free -h`命令查看直观。`top`中主要关注`Mem`（物理内存）和`Swap`（交换分区）的使用情况。

![](img/1865fdbb20644e85be2e679e6f98192f_29.png)

**进程列表区域**
以下是系统中各个进程的详细信息列表，默认按CPU使用率排序。每一列的含义如下：
*   `PID`：进程ID。
*   `USER`：进程所有者。
*   `PR`：进程优先级。
*   `NI`：Nice值，负值表示高优先级。
*   `VIRT`：进程使用的虚拟内存总量。
*   `RES`：进程使用的、未被换出的物理内存大小。
*   `SHR`：共享内存大小。
*   `S`：进程状态（R=运行，S=睡眠，Z=僵尸等）。
*   `%CPU`：上次更新到现在的CPU时间占用百分比。
*   `%MEM`：进程使用的物理内存百分比。
*   `TIME+`：进程使用的CPU时间总计。
*   `COMMAND`：命令名/命令行。

![](img/1865fdbb20644e85be2e679e6f98192f_31.png)

在`top`交互界面中，可以使用以下常用命令：
*   **`P`**：按CPU使用率排序。
*   **`M`**：按内存使用率排序。
*   **`k`**：输入后提示输入PID以杀死进程。
*   **`q`**：退出`top`。

## 进程检索与过滤 🔎

除了`top`，我们还可以使用`pgrep`命令来快速检索进程。`pgrep`通过匹配进程名来查找进程ID（PID），比`ps`结合`grep`更简洁。

![](img/1865fdbb20644e85be2e679e6f98192f_37.png)

以下是`pgrep`的常用选项：
*   `-l`：同时列出进程名和PID。
    ```bash
    pgrep -l sshd
    ```
*   `-u`：检索指定用户的进程。
    ```bash
    pgrep -l -u root
    ```
*   `-t`：检索指定终端（tty）的进程。
    ```bash
    pgrep -l -t pts/0
    ```
*   `-x`：进行精确匹配，要求进程名完全一致。
    ```bash
    pgrep -x sshd
    ```

## 进程的前后台调度 ⏯️

在Linux中，进程可以在前台运行（占用当前终端），也可以在后台运行（不占用终端）。这对于运行耗时任务非常有用。

![](img/1865fdbb20644e85be2e679e6f98192f_49.png)

**1. 将进程放入后台运行**
在命令的末尾加上`&`符号，可以让命令在后台启动。
```bash
sleep 300 &
```
这条命令会让系统休眠300秒，但因为它是在后台运行，所以不会阻塞当前终端。

![](img/1865fdbb20644e85be2e679e6f98192f_51.png)

**2. 查看后台进程列表**
使用`jobs`命令可以查看当前终端放入后台的进程列表。
```bash
jobs -l
```

![](img/1865fdbb20644e85be2e679e6f98192f_53.png)

![](img/1865fdbb20644e85be2e679e6f98192f_55.png)

![](img/1865fdbb20644e85be2e679e6f98192f_57.png)

**3. 将后台进程调至前台**
使用`fg`命令，后面跟上`jobs`命令显示的编号（如`%1`），可以将后台进程调回前台运行。
```bash
fg %1
```

**4. 挂起与恢复前台进程**
*   按下 `Ctrl + Z` 可以挂起（暂停）当前在前台运行的进程。
*   使用 `bg %[编号]` 命令可以让被挂起的进程在后台继续运行。
*   使用 `fg %[编号]` 命令可以让被挂起的进程回到前台继续运行。

## 杀死进程 🛑

当需要终止一个进程时，可以使用`kill`系列命令。

**1. kill：通过PID结束进程**
`kill`命令通过向进程发送信号来工作。最常用的信号有：
*   `15` (`SIGTERM`)：**默认信号**。优雅地终止进程，允许进程清理后退出。
*   `9` (`SIGKILL`)：强制杀死进程，立即终止，不给进程清理的机会。
```bash
kill 1234         # 优雅结束PID为1234的进程
kill -9 1234      # 强制杀死PID为1234的进程
```

![](img/1865fdbb20644e85be2e679e6f98192f_59.png)

![](img/1865fdbb20644e85be2e679e6f98192f_61.png)

**2. pkill：通过进程名结束进程**
`pkill`命令可以根据进程名来结束进程，适合批量结束同名进程。
```bash
pkill sleep        # 结束所有名为sleep的进程
pkill -9 -u test   # 强制结束用户test的所有进程
```

![](img/1865fdbb20644e85be2e679e6f98192f_63.png)

![](img/1865fdbb20644e85be2e679e6f98192f_65.png)

![](img/1865fdbb20644e85be2e679e6f98192f_66.png)

**3. killall：通过进程名结束进程（更彻底）**
`killall`与`pkill`类似，但通常匹配更严格。
```bash
killall sleep
```

**4. 通过终端踢出用户**
结合`w`或`who`命令查看用户登录的终端，然后使用`pkill`可以踢出指定终端的用户。
```bash
who                # 查看登录用户和终端
pkill -9 -t pts/1  # 强制结束在pts/1终端上的所有进程（即踢出该用户）
```

## 用户登录分析 👥

![](img/1865fdbb20644e85be2e679e6f98192f_74.png)

![](img/1865fdbb20644e85be2e679e6f98192f_76.png)

![](img/1865fdbb20644e85be2e679e6f98192f_78.png)

了解谁登录了系统以及登录历史，是系统安全审计的重要部分。

**1. 查看当前已登录用户**
*   `users`：仅显示已登录的用户名列表。
*   `who`：显示更详细的信息，包括用户名、终端、登录时间和来源IP。
    ```bash
    who
    ```
*   `w`：显示最详细的信息，包括`who`的信息以及用户当前正在执行的命令和系统负载。
    ```bash
    w
    ```

**2. 查看历史登录记录**
*   `last`：查看系统历史登录成功的记录（从`/var/log/wtmp`文件读取）。
    ```bash
    last
    ```
*   `lastb`：查看系统历史登录失败的记录（从`/var/log/btmp`文件读取）。这常用于排查暴力破解攻击。
    ```bash
    lastb
    ```
