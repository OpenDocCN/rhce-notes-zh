## 权限匹配机制

上一节我们介绍了用户和组的基本概念，本节中我们来看看进程如何根据其身份来匹配文件的权限。

任何进程在运行时都带有其启动用户的身份信息，即一个UID（用户ID）和一个或多个GID（组ID）。当一个进程尝试访问一个文件时，系统会将该进程的身份与文件的属性进行比对，以确定访问权限。

访问权限的匹配遵循以下优先级顺序：

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_11.png)

1.  **文件拥有者（Owner）**：如果进程的UID与文件的UID（拥有者）相同，则进程以“文件拥有者”的身份访问，并拥有文件为拥有者设置的权限。
2.  **文件所属组（Group）**：如果进程的UID与文件UID不匹配，但进程的GID（主组或附加组）与文件的GID（所属组）匹配，则进程以“文件所属组”的身份访问，并拥有文件为所属组设置的权限。
3.  **其他用户（Others）**：如果以上两者都不匹配，则进程以“其他用户”的身份访问，并拥有文件为其他用户设置的权限。

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_13.png)

**核心概念**：权限匹配的优先级是 **拥有者 > 所属组 > 其他用户**。

## 如何查看权限

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_19.png)

我们可以使用 `ls -l` 命令来查看文件或目录的详细信息，其中就包含了权限信息。

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_21.png)

```
-rw-r--r--. 1 root root 0 Aug 10 10:00 example.txt
```

输出结果中，第一个字符 `-` 表示这是一个普通文件（`d` 表示目录）。随后的9个字符每3个为一组，分别代表：

*   **前3位**：文件拥有者（user）的权限。
*   **中间3位**：文件所属组（group）的权限。
*   **后3位**：其他用户（others）的权限。

每组中的三个字符依次代表 **读（r）**、**写（w）**、**执行（x）** 权限。如果对应位置是 `-`，则表示没有该权限。

例如，`rw-r--r--` 表示：
*   拥有者（root）有 **读** 和 **写** 权限。
*   所属组（root组）有 **读** 权限。
*   其他用户有 **读** 权限。

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_23.png)

## 文件权限详解

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_25.png)

对于普通文件，`r`、`w`、`x` 权限的含义非常直观。

以下是文件权限的具体含义：

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_27.png)

*   **读权限（r）**：允许查看文件的内容。例如，使用 `cat`、`less`、`more` 等命令读取文件。
*   **写权限（w）**：允许修改文件的内容。例如，使用文本编辑器保存对文件的更改，或者向文件追加内容。
*   **执行权限（x）**：允许将文件作为程序或脚本来运行。例如，直接执行一个二进制程序（如 `/usr/bin/ls`）或一个Shell脚本。

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_29.png)

## 目录权限详解

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_31.png)

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_33.png)

目录在Linux中也被视为一种特殊文件，其 `r`、`w`、`x` 权限的含义与普通文件截然不同。

以下是目录权限的具体含义：

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_35.png)

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_37.png)

*   **读权限（r）**：允许**列出**目录中包含的文件和子目录的名称。仅凭 `r` 权限，你无法进入目录，也无法查看文件的详细信息（如大小、权限等）。
*   **执行权限（x）**：允许**进入**该目录，并将其作为当前工作目录。这是访问目录内任何文件或子目录的前提。只有 `x` 权限，你可以进入目录，但无法列出内容。
*   **写权限（w）**：允许在目录内**创建、删除、重命名**文件和子目录。**注意**：删除一个文件所需的权限，取决于你对文件**所在目录**是否有 `w` 权限，而不是文件本身的权限。

**核心概念**：对目录的常见有效权限组合是 `rx`（可进入并查看）和 `rwx`（可进入、查看并修改内容）。

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_39.png)

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_41.png)

## 权限的八进制表示法

除了使用 `rwx` 这种字符表示法，权限也常用三位八进制数字表示，便于计算和设置。

每个权限位对应一个数字：
*   `r` (读) = 4
*   `w` (写) = 2
*   `x` (执行) = 1
*   `-` (无) = 0

将同一组（拥有者、所属组、其他用户）的权限数字相加，就得到该组的权限值。再将三组的值连起来，就是完整的八进制权限。

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_43.png)

**公式**：`权限数字 = (r?4:0) + (w?2:0) + (x?1:0)`

![](img/6d3e0d608f4b40e52d3e61df86dbc57e_45.png)

例如：
*   `rwxr-xr--` 转换为数字：
    *   拥有者：`r+w+x` = 4+2+1 = **7**
    *   所属组：`r+ -+x` = 4+0+1 = **5**
    *   其他用户：`r+ -+ -` = 4+0+0 = **4**
    *   所以，`rwxr-xr--` 等价于 **754**。
