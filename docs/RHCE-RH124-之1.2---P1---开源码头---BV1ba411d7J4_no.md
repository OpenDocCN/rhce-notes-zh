# RHCE RH124 课程：1.2：Linux虚拟机的使用 🖥️

在本节课中，我们将学习如何配置和使用一个预置的Linux虚拟机环境，为后续的RHCE实验做准备。我们将了解虚拟机的基本设置、登录过程以及如何调整桌面环境以适应个人习惯。

## 虚拟机配置检查

上一节我们导入了虚拟机文件，本节中我们来看看虚拟机的具体配置。这些配置在导入OVA文件时已自动设置，但了解其含义有助于理解实验环境。

编辑虚拟机设置，主要查看以下配置：
*   **内存**：分配了约6GB。如果您的物理机内存为8GB，此配置尚可；若为4GB，则可能不足，建议至少使用8GB物理内存。
*   **处理器**：数量为1个，但关键是需要启用处理器的虚拟化指令支持（如Intel VT-x或AMD-V）。此选项在导入的虚拟机中已默认勾选。
*   **磁盘**：大小为250GB，但实际占用空间远小于此值。
*   **显示器**：已禁用“3D图形加速”功能。这是因为当前系统版本（RHEL 8.0）的图形界面与新版的VMware 3D加速存在兼容性问题，可能导致界面卡死。后续版本（如8.2/8.3）可能会修复此问题。

## 实验环境架构

![](img/8eb4e23ccc2e052f0028cbf63d49ec91_7.png)

需要理解的是，我们的实验环境采用了嵌套虚拟化架构。

*   首先，我们启动的这台虚拟机（称为`F0`）本身运行在VMware Workstation（或同类软件）中。
*   在这台`F0`虚拟机内部，我们将运行另外五台用于实验的虚拟机（`classroom`， `servera`， `serverb`等）。

![](img/8eb4e23ccc2e052f0028cbf63d49ec91_9.png)

这意味着物理机需要较强的性能来支撑两层虚拟机运行。如果资源紧张，可以直接在`F0`虚拟机内操作。为此，建议在启动内部虚拟机前，为`F0`虚拟机创建一个**快照**，以便在环境出错时能快速恢复到初始状态。

以下是登录步骤：

3.  输入密码后按回车登录。

首次登录后，会看到RHEL 8的默认桌面环境（GNOME），其界面风格类似Windows 8或手机系统，对于桌面操作可能不太习惯。我们可以将其切换为更传统的经典模式。

## 切换桌面环境

为了使界面更符合传统PC的使用习惯，我们将桌面环境从默认的“Wayland”切换到经典的“X11”模式。

以下是具体步骤：
1.  点击屏幕右上角的用户菜单，选择 **`注销(Log Out)`**。
2.  返回到登录界面后，在密码输入框下方，找到并点击**齿轮图标**。
3.  在弹出的菜单中，选择 **`GNOME Classic (X11 display server)`**。
4.  再次输入用户名(`kiosk`)和密码(`redhat`)，然后登录。

**说明**：
*   `X11`是Linux系统上运行多年的经典图形显示服务，稳定。
*   `Wayland`是旨在取代`X11`的现代显示协议，是未来发展方向，但在某些版本中可能仍有小瑕疵。
*   两者在桌面外观上几乎无差异，主要区别在于底层图形渲染架构。选择`GNOME Classic`能提供更接近Windows 7/10的传统桌面布局。

## 熟悉经典桌面

![](img/8eb4e23ccc2e052f0028cbf63d49ec91_27.png)

![](img/8eb4e23ccc2e052f0028cbf63d49ec91_29.png)

登录到经典桌面后，界面会更易于使用。

*   **左上角菜单**：功能类似Windows的“开始”菜单，包含应用程序、位置、系统工具等分类。
*   **应用程序**：可找到系统安装的所有软件，如火狐浏览器(`Firefox`)、系统设置工具等。教学环境中还预装了`Education`分类下的教学软件。
*   **桌面与文档**：双击桌面上的“`kiosk`的主文件夹”图标，可以访问个人文档，逻辑与Windows一致。

如果希望使用中文界面，可以在系统设置中进行语言切换。
