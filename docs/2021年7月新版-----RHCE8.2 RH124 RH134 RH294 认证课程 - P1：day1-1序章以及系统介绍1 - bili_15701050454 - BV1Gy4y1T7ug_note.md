# RHCE认证课程：P1：序章与系统介绍

## 概述
在本节课中，我们将学习RHCE认证课程的整体结构、Linux系统的历史背景，并完成RHEL 8.2系统的安装与基础配置。课程内容旨在为后续学习打下坚实基础。

## RHCE 8.2 课程体系介绍
在正式开课前，我们先了解RHCE 8与RHCE 7的异同。

上一代RHCE 7包含三门课程：**RH124**、**RH134**和**RH254**。而RHCE 8的核心内容是**RH294**，这门课程主要教授Ansible自动化运维工具。RH294原属于红帽更高级的认证体系，下放至RHCE 8后，其能力要求较高，但通过综合练习掌握后，应用于生产环境并不困难。

Ansible是目前较为流行的自动化运维工具。但需要注意的是，RHCE 8课程体系减少了对传统服务（如RH254所教授内容）的深入讲解，这可能导致学员在基础服务技能上有所薄弱。

关于认证考试，目前通常要求在同一天内上午考**RHCSA**，下午考**RHCE**。如果仅通过其中一门，则只需补考未通过的部分。但必须**RHCSA考试及格**，才能最终获得RHCE证书。红帽考试均为实操题（Lab），考生需根据题目要求进行操作。自RHEL 8.2起，RHCSA考试时长增至3小时，RHCE（即RH294）考试时长为4小时。对于熟练者，实际所需时间通常更短。

## Linux系统历史
了解Linux，需要从其前身Unix系统说起。

Unix诞生于1969年，由Ken Thompson在贝尔实验室开发。最初，他为了能在当时的系统上运行一款游戏，使用B语言编写了一个操作系统内核，这被视为Unix的雏形。随后，Ken Thompson与Dennis Ritchie合作，于1973年使用C语言重写了Unix系统，此举也推动了C语言的普及。

后来，Unix系统走向商业化并闭源，这促使了自由软件运动的发展。1984年，Richard Stallman发起了**GNU项目**，旨在创建一个完全自由的操作系统。GNU项目制定了**GPL协议**，允许用户自由使用、修改和分发软件。

![](img/2473aef33deddf8755c05ef7d8504db1_3.png)

与此同时，林纳斯·托瓦兹在1991年编写了**Linux内核**。将Linux内核与GNU项目的工具组合起来，才构成了一个完整的、可用的操作系统，即我们今天所说的**GNU/Linux系统**。

## RHEL 8.2 安装与初始化配置
本节我们将进行RHEL 8.2系统的安装。

![](img/2473aef33deddf8755c05ef7d8504db1_64.png)

![](img/2473aef33deddf8755c05ef7d8504db1_66.png)

![](img/2473aef33deddf8755c05ef7d8504db1_68.png)

### 创建虚拟机
1.  打开VMware Workstation，选择“创建新的虚拟机”。
2.  选择“自定义”配置，并选择稍后安装操作系统。
3.  客户机操作系统选择“Linux”，版本选择“Red Hat Enterprise Linux 8 64位”。
4.  为虚拟机命名并选择存储位置。
5.  处理器数量建议为2，内存设置为2GB。
6.  网络类型选择“使用网络地址转换（NAT）”。
7.  创建新虚拟磁盘，大小设置为100GB，并选择“将虚拟磁盘存储为单个文件”。

![](img/2473aef33deddf8755c05ef7d8504db1_70.png)

### 安装系统
1.  启动虚拟机，在启动界面选择第一项“Install Red Hat Enterprise Linux 8.2”直接安装。
2.  安装语言选择“English”。
3.  进入安装信息摘要界面，需配置以下关键部分：
    *   **安装目的地**：选择手动分区。
        *   点击“+”添加挂载点。
        *   `/boot`：引导分区，分配 **1GB**。
        *   `swap`：交换分区，分配 **4GB**（约为内存的2倍）。
        *   `/`：根分区，分配剩余的大部分空间（例如45GB）。
        *   `/data`：数据分区，分配剩余所有空间，用于统一管理第三方应用。
    *   **网络和主机名**：开启网络连接，主机名可暂时保留默认值。
    *   **软件选择**：初学者可选择带GUI的服务器，生产环境通常选择最小化安装。
4.  设置root用户密码，并可选择创建一个普通用户（如`student`）。
5.  确认所有配置后，开始安装。安装完成后重启系统。

## Linux系统框架与终端
### 用户类型
Linux系统主要有两类用户：
1.  **root用户**：超级管理员，拥有系统最高权限，需谨慎使用。
2.  **普通用户**：权限受限，只能在自己的家目录及拥有权限的目录中进行操作。

### 终端类型
终端是用户与系统交互的界面，主要分为：
*   **物理终端**：直接连接在主机上的设备。
*   **伪终端**：通过远程连接或图形界面模拟出来的终端，例如通过SSH工具连接的会话。

### 多用户与多任务
Linux支持多用户同时登录，且每个用户可以运行不同的任务。使用`w`或`who`命令可以查看当前登录系统的用户及其活动。

### 运行级别与目标
系统启动后的模式称为“目标”。图形界面对应`graphical.target`，纯文本界面对应`multi-user.target`。可以使用`systemctl`命令进行切换：
```bash
# 切换到纯文本模式
systemctl set-default multi-user.target
# 切换到图形模式
systemctl set-default graphical.target
```
切换后需要重启生效。图形界面会消耗较多内存资源，因此服务器通常使用最小化安装的文本模式。

## Shell基础
### 什么是Shell
Shell是系统的用户界面，提供了用户与内核进行交互操作的一种接口。它接收用户输入的命令，并将其转换成系统指令执行。常见的Shell有Bash、Zsh等，RHEL 8默认使用**Bash**。

![](img/2473aef33deddf8755c05ef7d8504db1_146.png)

### 命令提示符
命令行前的提示符格式由环境变量`PS1`定义。例如：
```bash
# 查看当前PS1设置
echo $PS1
# 自定义PS1，加入时间和完整路径
PS1="[\u@\h \t \w]\\$ "
```
其中，`\u`代表用户名，`\h`代表主机名，`\w`代表当前工作目录的完整路径，`\$`对root用户显示`#`，对普通用户显示`$`。

![](img/2473aef33deddf8755c05ef7d8504db1_148.png)

![](img/2473aef33deddf8755c05ef7d8504db1_150.png)

![](img/2473aef33deddf8755c05ef7d8504db1_152.png)

### 命令执行过程
当用户在Shell中输入命令并回车后，系统会经历以下步骤：
1.  Shell解析用户输入的命令。
2.  在`$PATH`环境变量定义的路径中查找对应的可执行程序。
3.  找到程序后，Shell通过系统调用将程序提交给内核。
4.  内核为程序分配资源（CPU、内存等），使其运行。
5.  程序执行完毕，将结果输出到终端。
