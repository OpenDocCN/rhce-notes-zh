# RHCE 8.2 认证课程：Day 7-3：救援模式 🛠️

![](img/66cfffc820a0ac293c28a666b98ffa76_1.png)

在本节课中，我们将学习 Linux 系统的救援模式。这是一种在系统无法正常启动时，用于修复和恢复系统的重要工具。我们将探讨三种常见的问题场景及其解决方案。

![](img/66cfffc820a0ac293c28a666b98ffa76_3.png)

## 回顾上午内容

上一节我们介绍了系统优化和计划任务。本节中，我们来看看当系统出现严重故障时，如何进入和使用救援模式进行修复。

![](img/66cfffc820a0ac293c28a666b98ffa76_5.png)

上午的核心内容包括：
*   **系统优化**：使用 `tuned` 工具调整系统性能。
    *   查看当前优化方案：`tuned-adm active`
    *   查看系统推荐方案：`tuned-adm recommend`
    *   更改优化方案：`tuned-adm profile <profile_name>`
    *   关闭优化：`tuned-adm off`
*   **进程优先级**：使用 `nice` 和 `renice` 命令。
    *   启动时指定优先级：`nice -n <优先级> <命令>`，优先级范围是 **-20 到 19**。
    *   重置运行中进程优先级：`renice -n <优先级> -p <PID>`
*   **计划任务**：分为一次性任务 (`at`) 和周期性任务 (`cron`)。
    *   `at` 命令有三种执行方式：交互式、输入重定向、通过 `-f` 指定文件。
    *   管理 `at` 任务：
        *   列出任务：`at -l` 或 `atq`
        *   删除任务：`at -d <任务号>` 或 `atrm <任务号>`
        *   查看任务详情：`at -c <任务号>`
    *   `cron` 表达式格式：`分钟 小时 日 月 星期`
        *   特殊字符示例：
            *   `0 1 * * *`：每天凌晨1点0分执行。
            *   `0,15,20,30,45 1 * * *`：每天1点的0、15、20、30、45分执行。
            *   `0-15 1 * * *`：每天1点的0到15分，每分钟执行。
            *   `*/2 1 * * *`：每天1点开始，每2分钟执行一次。
            *   `@yearly`：每年1月1日0点执行。
            *   `@monthly`：每月1日0点执行。
            *   `@weekly`：每周日0点执行。
            *   `@daily` 或 `@hourly`：每天0点或每小时0分执行。
            *   `@reboot`：每次重启时执行。
    *   **黑白名单机制**：通过 `/etc/at.deny`, `/etc/at.allow`, `/etc/cron.deny`, `/etc/cron.allow` 文件控制用户权限。

## 救援模式详解

救援模式主要用于处理系统无法正常启动的紧急情况。主要有以下三种应用场景：

### 场景一：错误的启动目标导致系统循环重启

![](img/66cfffc820a0ac293c28a666b98ffa76_7.png)

当系统默认启动目标（target）被错误地设置为 `reboot` 或 `poweroff` 时，系统会陷入无限重启或关机循环。

![](img/66cfffc820a0ac293c28a666b98ffa76_9.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_11.png)

**解决方案：中断 GRUB 引导流程**
1.  系统启动时，在 GRUB 菜单界面快速按下 `e` 键进入编辑模式。
2.  找到以 `linux` 开头的那一行，在行末添加 `rd.break` 参数以中断启动流程。
3.  为了获得读写权限，找到同一行的 `ro`（只读）参数，将其改为 `rw`。
4.  按 `Ctrl+X` 执行修改后的启动项。
5.  系统会进入一个临时的 shell 环境。此时需要切换回真实的根文件系统：
    ```bash
    chroot /sysroot
    ```
6.  修复错误的启动目标设置：
    ```bash
    systemctl set-default multi-user.target
    ```
7.  退出并重启系统：
    ```bash
    exit
    reboot
    ```

![](img/66cfffc820a0ac293c28a666b98ffa76_13.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_15.png)

### 场景二：忘记 root 用户密码

![](img/66cfffc820a0ac293c28a666b98ffa76_17.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_18.png)

这是救援模式最常用的功能之一。可以通过系统内置的救援入口或使用安装光盘/镜像进入。

![](img/66cfffc820a0ac293c28a666b98ffa76_20.png)

**使用安装光盘进入救援模式（通用性强）**
1.  从光盘启动，在安装界面选择 **“Troubleshooting”** -> **“Rescue a Red Hat Enterprise Linux system”**。
2.  进入救援环境后，选择 **“1) Continue”** 继续。
3.  系统会自动尝试查找并挂载原有的根文件系统。成功后，会提示已挂载到 `/mnt/sysroot`。
4.  切换根环境：
    ```bash
    chroot /mnt/sysroot
    ```
5.  修改 root 密码：
    ```bash
    passwd root
    ```
6.  **关键步骤**：如果系统启用了 SELinux（默认开启），必须重建文件安全上下文，否则修改可能不生效：
    ```bash
    touch /.autorelabel
    ```
7.  退出并重启：
    ```bash
    exit
    reboot
    ```

![](img/66cfffc820a0ac293c28a666b98ffa76_24.png)

### 场景三：文件系统挂载错误导致系统进入紧急模式

![](img/66cfffc820a0ac293c28a666b98ffa76_26.png)

当 `/etc/fstab` 文件配置错误（如挂载点不存在、文件系统类型错误）时，系统在启动时会进入紧急模式（emergency shell），要求输入 root 密码。

![](img/66cfffc820a0ac293c28a666b98ffa76_28.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_30.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_32.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_34.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_36.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_38.png)

**解决方案**
1.  在紧急模式下输入 root 密码进入 shell。
2.  检查 `/etc/fstab` 文件，修正错误的行（例如，错误的设备名、挂载点或文件系统类型）。
    ```bash
    vim /etc/fstab
    ```
3.  重新挂载所有文件系统并退出：
    ```bash
    mount -a
    exit
    ```
4.  系统会继续正常的启动流程。

**注意**：对于严重的文件系统损坏（如硬盘物理坏道），救援模式可能无法修复，此时需要更换硬件或从备份恢复。

## 总结

![](img/66cfffc820a0ac293c28a666b98ffa76_44.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_46.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_48.png)

本节课中我们一起学习了 Linux 救援模式的核心用法。我们了解了三种典型的故障场景：
1.  **启动目标错误**：通过编辑 GRUB 启动参数，进入临时环境修复。
2.  **忘记密码**：通过光盘或系统救援入口，切换根环境后重置密码，并注意 SELinux 的 `autorelabel` 步骤。
3.  **挂载错误**：在紧急模式下修正 `/etc/fstab` 配置文件。

![](img/66cfffc820a0ac293c28a666b98ffa76_50.png)

![](img/66cfffc820a0ac293c28a666b98ffa76_52.png)

救援模式是系统管理员的“救命稻草”，掌握它对于维护系统稳定性和处理紧急故障至关重要。请务必在实验环境中多加练习，熟悉整个流程。
