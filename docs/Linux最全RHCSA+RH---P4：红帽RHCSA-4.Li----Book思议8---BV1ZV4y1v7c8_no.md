### 多用户系统
Linux是一个多用户操作系统。这意味着在同一台Linux服务器上，可以允许多个用户同时登录并使用系统资源。例如，在企业环境中，不同的运维人员可以使用各自的账号登录同一台服务器进行工作。

### 多任务系统
Linux是一个多任务操作系统。它允许系统同时运行多个程序或进程。这就像在Windows系统中，你可以一边听音乐，一边浏览网页，一边运行其他软件。

### 严格区分大小写
Linux系统对大小写是严格区分的。无论是文件名、目录名还是命令本身，`File.txt`和`file.txt`会被系统视为两个完全不同的文件。

### 一切皆文件
在Linux哲学中，“一切皆文件”是一个核心概念。硬件设备（如CPU、内存、硬盘）、目录、普通文件等，在系统中都以文件的形式进行管理和访问。例如，查看CPU信息就是读取`/proc/cpuinfo`这个特殊的文件。

![](img/be285920a3861062f1010b438a00e96c_43.png)

![](img/be285920a3861062f1010b438a00e96c_45.png)

### 树形目录结构
Linux文件系统采用倒置的树形结构进行组织，所有文件和目录都从**根目录（/**）开始。根目录是整个文件系统的起点，类似于大树的树根。如果根目录被破坏，整个系统的数据都可能丢失。

![](img/be285920a3861062f1010b438a00e96c_47.png)

### 无扩展名依赖
与Windows系统不同，Linux不依赖文件扩展名来识别文件类型。文件能否被正确执行或读取，取决于文件本身的属性和内容，而非`.exe`或`.txt`这样的后缀。不过，为了用户便于识别，我们仍然会使用如`.sh`（脚本）、`.conf`（配置文件）、`.log`（日志）等扩展名。

![](img/be285920a3861062f1010b438a00e96c_49.png)

![](img/be285920a3861062f1010b438a00e96c_51.png)

### 无回收站机制
Linux系统默认没有类似Windows的回收站功能。使用命令行删除文件通常是直接、永久性的操作。因此，执行删除命令时需要格外谨慎，因为数据恢复非常困难。

![](img/be285920a3861062f1010b438a00e96c_53.png)

![](img/be285920a3861062f1010b438a00e96c_55.png)

### 沉默即成功
在Linux命令行中，大多数命令执行成功后不会给出明确的“成功”提示。**没有错误信息往往就是最好的提示**，意味着命令已顺利执行。如果命令有误，系统通常会返回明确的错误信息。

## 命令终端字段含义介绍

![](img/be285920a3861062f1010b438a00e96c_65.png)

![](img/be285920a3861062f1010b438a00e96c_67.png)

理解了系统的基本概念后，我们来看看如何与系统交互。命令终端是我们输入指令的地方。下图展示了一个典型的命令提示符：

```
[root@localhost ~]#
```

![](img/be285920a3861062f1010b438a00e96c_77.png)

![](img/be285920a3861062f1010b438a00e96c_79.png)

![](img/be285920a3861062f1010b438a00e96c_81.png)

以下是每个字段的含义解析：

![](img/be285920a3861062f1010b438a00e96c_85.png)

![](img/be285920a3861062f1010b438a00e96c_87.png)

![](img/be285920a3861062f1010b438a00e96c_89.png)

*   **`root`**：**当前登录的用户名**。这里显示的是超级管理员账号`root`。
*   **`@`**：分隔符，用于分隔用户名和主机名。
*   **`localhost`**：**当前系统的主机名**。安装后默认为`localhost`，后期可以根据用途修改，例如改为`web-server`或`db-server`。
*   **`~`**：**波浪号，代表当前用户的家目录**。用户首次登录系统时，默认进入自己的家目录。
    *   超级管理员`root`的家目录是：`/root`
    *   普通用户（如`test`）的家目录是：`/home/用户名`，例如`/home/test`
*   **`#`** 或 **`$`**：**命令提示符结尾符**，用于指示当前用户的权限等级。
    *   **`#`**：表示当前用户是**超级管理员（root）**，拥有最高系统权限。
    *   **`$`**：表示当前用户是**普通用户**，权限受到限制。

## 命令行格式介绍

现在，我们开始学习如何输入命令。Linux命令有特定的格式，不能随意书写。一条完整的命令通常遵循以下结构：

```
命令字 [选项] [参数]
```

为了更好地理解，我们以最常用的`ls`命令为例进行拆解。`ls`命令的基本功能是**列出目录中的内容**。

### 命令字
命令字是命令本身，代表要执行的核心操作。例如，`ls`就是一个命令字。

![](img/be285920a3861062f1010b438a00e96c_99.png)

![](img/be285920a3861062f1010b438a00e96c_103.png)

**示例代码：**
```bash
ls
```
执行此命令会列出当前所在目录下的所有文件和子目录。

![](img/be285920a3861062f1010b438a00e96c_105.png)

![](img/be285920a3861062f1010b438a00e96c_107.png)

### 选项
选项用于调整或增强命令的功能。选项通常以`-`或`--`开头。

*   **短选项**：以单个`-`开头，后接单个字符（如`-l`）。多个短选项可以合并，例如`-lh`。
*   **长选项**：以`--`开头，后接一个完整的单词（如`--help`）。长选项不能合并。

![](img/be285920a3861062f1010b438a00e96c_116.png)

![](img/be285920a3861062f1010b438a00e96c_118.png)

![](img/be285920a3861062f1010b438a00e96c_120.png)

![](img/be285920a3861062f1010b438a00e96c_122.png)

**示例代码：**
```bash
ls -l
```
`-l`选项表示以**长格式**显示，可以查看文件/目录的详细信息（权限、所有者、大小、修改时间等）。

![](img/be285920a3861062f1010b438a00e96c_124.png)

![](img/be285920a3861062f1010b438a00e96c_126.png)

![](img/be285920a3861062f1010b438a00e96c_128.png)

**示例代码：**
```bash
ls -lh
```
这里合并使用了`-l`和`-h`两个选项。`-h`的作用是**以人性化的单位（K, M, G）显示文件大小**，使信息更易读。

![](img/be285920a3861062f1010b438a00e96c_130.png)

![](img/be285920a3861062f1010b438a00e96c_132.png)

### 参数
参数是命令的操作对象，可以是文件、目录或程序等。

**示例代码：**
```bash
ls -lh /etc
```
这里的`/etc`就是一个参数，表示让`ls`命令列出`/etc`目录下的内容，而不是当前目录。

### 格式符号说明
在命令格式 `命令字 [选项] [参数]` 中：
*   **`[]`（中括号）**：表示括号内的内容是可选的。你可以根据需求决定是否使用选项或参数。
*   **`...`（省略号）**：表示前面的元素（选项或参数）可以同时指定多个。

**综合示例代码：**
```bash
ls -la /etc /var/log
```
这条命令的含义是：使用`-l`（详情）和`-a`（显示所有文件，包括隐藏文件）选项，同时列出`/etc`和`/var/log`两个目录的内容。
