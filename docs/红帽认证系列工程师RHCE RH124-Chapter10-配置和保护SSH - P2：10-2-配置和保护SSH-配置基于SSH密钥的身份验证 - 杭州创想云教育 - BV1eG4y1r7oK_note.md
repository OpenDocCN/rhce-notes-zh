# 红帽认证系列工程师RHCE RH124-Chapter10：配置和保护SSH - P2：10-2-配置基于SSH密钥的身份验证 🔑

在本节课中，我们将要学习如何配置基于SSH密钥的身份验证。这是一种比密码认证更安全的方法，通过非对称加密技术来验证用户身份，可以有效防范中间人攻击等风险。

上一节我们介绍了密码认证的潜在风险，本节中我们来看看如何配置和使用更安全的SSH密钥认证。

## 概述：SSH密钥认证原理

SSH密钥认证使用一对密钥：公钥和私钥。公钥可以公开分享，放置在服务器上；私钥必须由用户秘密保管。当客户端尝试连接时，服务器会使用存储的公钥加密一个随机字符串并发送给客户端。客户端使用私钥解密后，将结果发回服务器进行验证。如果匹配，则认证成功。

整个过程的核心是**非对称加密**，其安全性远高于传统的密码认证。

## 生成SSH密钥对

首先，我们需要在客户端生成一个密钥对。这可以在任何一台受信任的机器上完成。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_1.png)

以下是生成密钥对的两种方法：

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_3.png)

**1. 交互式生成（默认）**
使用 `ssh-keygen` 命令，它会引导你完成整个过程，默认使用RSA算法。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_5.png)

```bash
ssh-keygen
```
执行后，默认会在用户家目录的 `.ssh/` 文件夹下生成两个文件：
*   `id_rsa`：私钥文件，必须妥善保管。
*   `id_rsa.pub`：公钥文件，需要上传到服务器。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_7.png)

**2. 非交互式生成**
如果你希望一次性完成所有设置，可以使用带参数的命令。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_9.png)

```bash
ssh-keygen -t rsa -b 2048 -f ~/.ssh/mykey -N ""
```
*   `-t rsa`：指定使用RSA算法。
*   `-b 2048`：指定密钥长度为2048位。
*   `-f ~/.ssh/mykey`：指定生成的密钥文件名。
*   `-N ""`：设置私钥的密码为空（即免密码私钥）。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_11.png)

## 将公钥部署到服务器

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_13.png)

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_15.png)

生成密钥对后，需要将公钥文件传输到目标服务器上对应用户的家目录中。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_17.png)

以下是部署公钥的方法：

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_19.png)

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_21.png)

**使用 `ssh-copy-id` 命令（推荐）**
这个命令可以自动将公钥追加到服务器对应用户的 `~/.ssh/authorized_keys` 文件中。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_23.png)

```bash
ssh-copy-id -i ~/.ssh/id_rsa.pub student@serverb
```
*   `-i`：指定要传送的公钥文件路径。
*   `student@serverb`：目标服务器的用户名和主机名。执行后需要输入一次服务器用户的密码。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_25.png)

**手动复制**
你也可以通过其他方式（如U盘、SCP命令）将公钥内容复制到服务器 `~/.ssh/authorized_keys` 文件的末尾。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_27.png)

## 使用SSH密钥登录

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_29.png)

公钥部署成功后，即可使用对应的私钥进行连接。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_31.png)

**连接命令示例：**
```bash
ssh -i ~/.ssh/id_rsa student@serverb
```
*   `-i`：指定使用的私钥文件路径。如果使用默认的 `id_rsa` 文件，此参数可省略。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_33.png)

如果私钥在生成时设置了密码，首次连接需要输入私钥密码进行解密。

## 管理私钥密码与代理

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_35.png)

为私钥设置密码能增加安全性，但每次连接都需要输入，可能带来不便。SSH代理可以帮助我们在一次会话中缓存解密后的私钥。

以下是使用SSH代理的步骤：

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_37.png)

1.  启动SSH代理并设置环境变量。
    ```bash
    eval $(ssh-agent)
    ```
2.  将私钥添加到代理中，并输入私钥密码。
    ```bash
    ssh-add ~/.ssh/id_rsa
    ```
3.  完成上述步骤后，在同一终端会话中再次使用SSH连接时，就无需重复输入私钥密码了。

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_39.png)

## 安全建议

![](img/a3e594ed70a0a06fdd0bb98418d1fadb_41.png)

*   **私钥保管**：私钥等同于密码，必须严格保密，切勿泄露给他人。
*   **加密私钥**：对于需要通过互联网访问的服务器，强烈建议为私钥设置强密码。
*   **免密码私钥的适用场景**：通常仅建议在高度信任的内部网络环境（如自动化脚本、CI/CD流水线）中使用免密码私钥。
*   **使用代理**：在安全的环境下，使用SSH代理可以平衡安全性与便利性。
