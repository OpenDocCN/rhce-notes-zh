## 5.2 文件的一般权限

上一节我们介绍了用户身份与文件所有者的概念，本节中我们来看看文件权限的具体构成。

Linux系统中的文件权限主要分为三种基本类型：
*   **读取（Read）**：用字母 `r` 表示，数字 `4` 表示。
*   **写入（Write）**：用字母 `w` 表示，数字 `2` 表示。
*   **执行（Execute）**：用字母 `x` 表示，数字 `1` 表示。

一个文件的完整权限由9个字符组成，分为三段，每段三位，分别对应三类用户：
*   **前三位**：文件**所有者（Owner）**的权限。
*   **中间三位**：文件**所属用户组（Group）**的权限。
*   **后三位**：**其他用户（Others）**的权限。

例如，使用 `ls -l` 命令查看文件时，可能会看到类似 `-rwxr-xr--` 的权限字符串。第一位 `-` 表示文件类型（后续会讲），后面的 `rwxr-xr--` 就是9位权限。

### 权限对于文件和目录的不同含义

权限对于普通文件和目录的意义有所不同，理解这一点至关重要。

以下是针对普通文件的权限解释：
*   **读取权限（r）**：允许查看文件内容。对应命令如 `cat`、`more`。
*   **写入权限（w）**：允许修改文件内容。对应命令如 `vim`、`echo`。
*   **执行权限（x）**：允许将文件作为程序或脚本运行。执行方式为 `./filename`。

以下是针对目录的权限解释：
*   **读取权限（r）**：允许**列出**目录内的文件列表。对应命令 `ls`。
*   **写入权限（w）**：允许在目录内**创建、删除、重命名**文件。对应命令 `touch`、`rm`、`mv`。
*   **执行权限（x）**：允许**进入**该目录。对应命令 `cd`。

> **注意**：删除一个文件的权限取决于其**所在目录**是否具有写入（w）权限，而非文件本身的权限。

### 数字表示法（八进制表示法）

![](img/7bd977cd29654fed1d6b4bc6733f847e_9.png)

为了方便设置和阅读，权限常用数字表示法。其换算规则基于二进制思想，将 `r`、`w`、`x` 分别视为一个二进制位。

![](img/7bd977cd29654fed1d6b4bc6733f847e_11.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_13.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_15.png)

**换算公式**：
`r` = 4， `w` = 2， `x` = 1， 无权限 = 0。

将每一类用户（所有者、组、其他人）的三种权限值相加，得到一个0-7的数字。例如：
*   `rwx` = 4+2+1 = **7**
*   `rw-` = 4+2+0 = **6**
*   `r-x` = 4+0+1 = **5**
*   `r--` = 4+0+0 = **4**
*   `---` = 0+0+0 = **0**

因此，一个权限 `rwxr-xr--` 用数字表示就是 `754`（所有者7， 所属组5， 其他人4）。

![](img/7bd977cd29654fed1d6b4bc6733f847e_17.png)

**权限换算练习**：
*   将 `rw-rw-r--` 转换为数字：`664`
*   将数字 `765` 转换为权限符号：`rwxrw-r-x`

![](img/7bd977cd29654fed1d6b4bc6733f847e_19.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_21.png)

### 修改文件权限和属性的命令

![](img/7bd977cd29654fed1d6b4bc6733f847e_23.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_25.png)

掌握了权限的表示方法后，我们来看看如何修改它们。

![](img/7bd977cd29654fed1d6b4bc6733f847e_27.png)

**1. 修改文件权限：`chmod` 命令**
`chmod` 命令用于修改文件或目录的权限。可以使用符号法（u/g/o/a +/-/= r/w/x）或更简洁的数字法。
```bash
# 数字法示例：将文件file.txt的权限设置为rwxr-xr-- (754)
chmod 754 file.txt

# 符号法示例：给所有用户增加执行权限
chmod a+x file.txt
```

![](img/7bd977cd29654fed1d6b4bc6733f847e_29.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_31.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_33.png)

**2. 修改文件所有者和所属组：`chown` 命令**
`chown` 命令用于修改文件的所有者和所属组。
```bash
# 语法：chown 所有者:所属组 文件名
# 将file.txt的所有者改为user1，所属组改为group1
chown user1:group1 file.txt

# 只修改所属组
chown :group1 file.txt
# 或使用 chgrp 命令
chgrp group1 file.txt
```
> **历史小知识**：在较早的系统中，`chown` 命令的间隔符是点号（.），如 `user1.group1`。现代系统中冒号（:）和点号均可使用。

## 5.3 文件的特殊权限

一般权限 `rwx` 的功能相对基础。Linux系统还提供了三种特殊权限，作为一般权限的补充，用于实现更灵活的权限控制。

### SUID（Set User ID）

**作用**：当二进制程序文件被设置了SUID权限后，任何用户在执行该程序时，都将**临时获得该程序文件所有者**的权限。

**典型示例**：`/usr/bin/passwd` 命令。
普通用户可以使用 `passwd` 命令修改自己的密码，而密码实际存储在 `/etc/shadow` 文件中。查看该文件权限：
```bash
ls -l /etc/shadow
```
你会发现 `/etc/shadow` 的权限是 `---------`（000），普通用户对其没有任何读写权限。那么 `passwd` 命令是如何修改它的呢？查看 `passwd` 命令本身的权限：
```bash
ls -l /usr/bin/passwd
```
你会看到权限中有 `rws`，其中的 `s` 就代表SUID权限。当普通用户执行 `passwd` 时，会临时获得文件所有者（root）的权限，从而能够修改 `/etc/shadow` 文件。

**设置与查看**：
*   SUID的数字表示为 **4**。
*   在权限符号中，它出现在**所有者**的执行位（x）上。如果原来有x权限，则显示为小写 `s`（如 `rws`）；如果原来没有x权限，则显示为大写 `S`（如 `rwS`）。

### SGID（Set Group ID）

SGID权限有两种应用场景：

**1. 针对可执行文件**：
**作用**：与SUID类似，但执行者将临时获得**文件所属组**的权限。
**数字表示**：**2**。

![](img/7bd977cd29654fed1d6b4bc6733f847e_35.png)

**2. 针对目录（更常用）**：
**作用**：在设置了SGID权限的目录中，**新建的文件会自动继承该目录的所属组**，而不是创建者的默认组。
**典型用途**：用于协作目录，确保所有在该目录下创建的文件都属于同一个项目组。

**设置与查看**：
*   SGID的数字表示为 **2**。
*   在权限符号中，它出现在**所属组**的执行位（x）上。显示规则同SUID（有小写`s`和大写`S`之分）。

### SBIT（Sticky Bit）

**作用**：又称为“粘滞位”。当目录被设置SBIT权限后，该目录中的文件**仅允许其所有者或root用户删除或重命名**。
**典型示例**：系统的临时目录 `/tmp`。所有用户都可以在 `/tmp` 中创建文件，但只能删除自己的文件，无法删除他人的文件。

**设置与查看**：
*   SBIT的数字表示为 **1**。
*   在权限符号中，它出现在**其他人**的执行位（x）上。如果原来有x权限，则显示为小写 `t`（如 `rwt`）；如果原来没有x权限，则显示为大写 `T`（如 `rwT`）。

### 特殊权限的数字表示与计算

特殊权限可以叠加在一般权限的数字表示法之前，形成一个四位数字。

![](img/7bd977cd29654fed1d6b4bc6733f847e_37.png)

**计算公式**：
`特殊权限数字` + `一般权限数字（所有者）` + `一般权限数字（所属组）` + `一般权限数字（其他人）`

其中：
*   SUID = 4
*   SGID = 2
*   SBIT = 1

**示例**：
*   权限 `rwsr-xr-t` 如何计算？
    1.  一般权限：`rwxr-xr-x` = 755
    2.  特殊权限：所有者位有`s`（SUID）=4，其他人位有`t`（SBIT）=1。特殊权限总和=4+1=5。
    3.  完整数字权限：**5755**
*   数字 `6750` 代表什么权限？
    1.  第一位6是特殊权限：6 = 4(SUID) + 2(SGID)。
    2.  后三位750是一般权限：7(`rwx`)，5(`r-x`)，0(`---`)。
    3.  转换后：`rwsr-s---`（注意：所属组5是`r-x`，加上SGID后变为`r-s`）。

## 5.4 文件的隐藏属性

除了我们能看到的一般权限和特殊权限，Linux文件系统还支持一些“隐藏属性”（Hidden Attributes）。这些属性不直接显示在 `ls -l` 的输出中，需要使用特定命令查看和设置，它们提供了更深层次的文件保护和控制功能。

**管理命令**：
*   **设置/删除隐藏属性**：`chattr [+-=] [属性] 文件或目录`
*   **查看隐藏属性**：`lsattr 文件或目录`

![](img/7bd977cd29654fed1d6b4bc6733f847e_39.png)

**常用隐藏属性参数**：
以下是几个重要且常用的隐藏属性：

![](img/7bd977cd29654fed1d6b4bc6733f847e_41.png)

*   **`i`（immutable）**：
    *   **作用**：设置后，文件将无法被修改、删除、重命名，也无法创建指向它的链接。即使是root用户也不例外。
    *   **用途**：保护极其重要的配置文件，防止被意外或恶意修改。
    ```bash
    # 为文件添加i属性
    chattr +i important.conf
    # 此时尝试删除或修改都会失败
    rm important.conf # 报错：Operation not permitted
    # 移除i属性
    chattr -i important.conf
    ```

*   **`a`（append only）**：
    *   **作用**：设置后，文件内容只能以**追加（append）** 方式写入，无法修改已有内容或截断文件。
    *   **用途**：非常适合保护日志文件，确保日志记录只能增加，不能被篡改或删除。
    ```bash
    # 为日志文件添加a属性
    chattr +a /var/log/secure.log
    # 可以追加内容
    echo "New log entry" >> /var/log/secure.log
    # 但不能清空或覆盖
    echo "Clear" > /var/log/secure.log # 报错：Operation not permitted
    ```

![](img/7bd977cd29654fed1d6b4bc6733f847e_43.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_45.png)

*   **`s`（secure deletion）**：
    *   **作用**：文件被删除时，其数据块会被**用零填充**，确保无法通过数据恢复工具找回。需要文件系统支持。
    *   **用途**：安全删除敏感文件。

![](img/7bd977cd29654fed1d6b4bc6733f847e_47.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_49.png)

*   **`A`（no atime update）**：
    *   **作用**：禁止更新文件的访问时间（atime）。每次读取文件都会更新atime，对于频繁访问的文件，禁用此功能可以提升I/O性能。

![](img/7bd977cd29654fed1d6b4bc6733f847e_51.png)

> **提示**：隐藏属性是文件系统级别的功能，并非所有文件系统都支持全部属性。常用的ext3/ext4/xfs文件系统都支持上述常见属性。

![](img/7bd977cd29654fed1d6b4bc6733f847e_53.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_55.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_57.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_59.png)

## 5.5 文件的访问控制列表（ACL）

![](img/7bd977cd29654fed1d6b4bc6733f847e_61.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_63.png)

之前学习的权限控制（所有者、组、其他人）虽然强大，但粒度仍不够细。例如，我们想精确地允许用户A读写某个文件，而禁止同组的用户B，用传统权限就很难实现。

![](img/7bd977cd29654fed1d6b4bc6733f847e_65.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_67.png)

**文件的访问控制列表（ACL， Access Control List）** 就是为了解决这个问题而生的。它可以为**单个用户或用户组**设置针对**单个文件或目录**的精细权限。

![](img/7bd977cd29654fed1d6b4bc6733f847e_69.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_71.png)

### ACL的管理命令

*   **为文件设置ACL规则**：`setfacl`
*   **查看文件的ACL规则**：`getfacl`

**`setfacl` 常用参数**：
*   `-m`：修改或添加ACL规则。
*   `-x`：删除指定的ACL规则。
*   `-b`：删除文件的所有ACL规则。
*   `-R`：递归地对目录及其内容进行操作。

**基本语法**：
```bash
# 针对用户设置
setfacl -m u:username:permissions file
# 针对组设置
setfacl -m g:groupname:permissions file
```

**示例场景**：
有一个目录 `/project`，希望：
1.  用户 `zhangsan` 拥有所有权限（rwx）。
2.  用户 `lisi` 只有读取和执行权限（r-x）。
3.  其他用户无任何权限（---）。

![](img/7bd977cd29654fed1d6b4bc6733f847e_73.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_75.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_77.png)

```bash
# 1. 先设置目录的基本权限，例如给所属组和其他人无权限
chmod 750 /project # 假设所有者是root，权限为rwxr-x---

![](img/7bd977cd29654fed1d6b4bc6733f847e_79.png)

# 2. 使用ACL为zhangsan添加rwx权限
setfacl -m u:zhangsan:rwx /project

![](img/7bd977cd29654fed1d6b4bc6733f847e_81.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_83.png)

# 3. 使用ACL为lisi添加r-x权限
setfacl -m u:lisi:r-x /project

![](img/7bd977cd29654fed1d6b4bc6733f847e_85.png)

# 4. 查看ACL规则
getfacl /project
# 输出会显示除了传统的owner/group/others权限外，还有user:zhangsan和user:lisi的特定规则。
```

![](img/7bd977cd29654fed1d6b4bc6733f847e_87.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_89.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_91.png)

**`getfacl` 命令输出解读**：
`getfacl` 命令的输出清晰展示了文件的ACL信息，包括：
*   `# file:` 文件名
*   `# owner:` 所有者
*   `# group:` 所属组
*   `user::rwx` 所有者的权限
*   `user:zhangsan:rwx` 用户zhangsan的ACL权限
*   `group::r-x` 所属组的权限
*   `group:devteam:r--` 组devteam的ACL权限（如果有）
*   `mask::rwx` 有效权限掩码（限制ACL条目的最大有效权限）
*   `other::---` 其他人的权限

![](img/7bd977cd29654fed1d6b4bc6733f847e_93.png)

> **权限优先级**：当文件同时拥有传统权限和ACL时，**ACL规则的优先级更高**。系统会先检查ACL，如果用户或组在ACL中有明确规则，则使用该规则；否则，才回退到传统的所有者、组、其他人的权限判断。

## 5.6 切换用户身份与sudo服务

![](img/7bd977cd29654fed1d6b4bc6733f847e_95.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_97.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_99.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_101.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_103.png)

在Linux系统中，我们经常需要切换用户身份来执行不同的任务。除了直接使用 `su` 命令，`sudo` 服务提供了更安全、更可控的权限委托方式。

![](img/7bd977cd29654fed1d6b4bc6733f847e_105.png)

### `su` 命令

![](img/7bd977cd29654fed1d6b4bc6733f847e_107.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_109.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_111.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_113.png)

`su`（switch user）命令用于切换用户身份。

```bash
# 切换到指定用户（不加载该用户的环境变量）
su username

![](img/7bd977cd29654fed1d6b4bc6733f847e_115.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_117.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_119.png)

# 完全切换（连环境变量一起切换，推荐）
su - username
```
*   **root用户切换到任何普通用户都不需要密码**。
*   **普通用户切换到其他用户（包括root）都需要输入目标用户的密码**。

![](img/7bd977cd29654fed1d6b4bc6733f847e_121.png)

`su -` 的 `-` 参数代表“登录式切换”，它会模拟一次完整的登录，初始化目标用户的环境变量（如PATH、HOME等），使切换后的环境与直接登录该用户完全一致。

![](img/7bd977cd29654fed1d6b4bc6733f847e_123.png)

![](img/7bd977cd29654fed1d6b4bc6733f847e_125.png)

### `sudo` 服务

`su` 命令需要知道目标用户的密码，这存在安全隐患（例如，多人需要知道root密码）。`sudo` 服务的理念是：**让受信任的用户以其他用户（通常是root）的身份运行特定的命令，而无需知道其密码**。

**工作原理**：
1.  系统管理员通过编辑 `/etc/sudoers` 配置文件，精确指定哪个用户（或用户组）可以在哪台主机上，以哪个用户的身份，运行哪些命令。
2.  普通用户在执行命令时，在命令前加上 `sudo`。
3.  系统会验证该用户是否有权限执行此命令（根据 `/etc/sudoers`），并提示输入**该用户自己的密码**进行确认（并非root密码）。
4.  验证通过后，命令将以指定的身份（如root）执行。

**配置 `sudoers` 文件**：
**永远不要直接使用普通文本编辑器编辑 `/etc/sudoers`！** 请使用 `visudo` 命令。该命令会在保存时检查语法，防止配置错误导致所有用户都无法使用sudo。
```bash
visudo
```

**配置语法示例**：
在 `/etc/sudoers` 文件中，找到类似以下格式的行进行添加或修改：
```
# 语法：谁    在哪些主机=（以谁的身份）    可以执行哪些命令
user    host=(runas_user)    command
```
**示例**：
1.  允许用户 `zhangsan` 在任何主机上以root身份执行 `systemctl restart nginx` 命令：
    ```
    zhangsan ALL=(root) /usr/bin/systemctl restart nginx
    ```
2.  允许 `wheel` 组的成员在任何主机上以任何用户身份执行任何命令（相当于赋予管理员权限）：
    ```
    %wheel ALL=(ALL) ALL
    ```
3.  允许用户 `lisi` 无需密码即可执行 `/sbin/reboot` 命令：
    ```
    lisi ALL=(root) NOPASSWD: /sbin/reboot
    ```

**用户使用方式**：
配置好后，用户 `zhangsan` 可以这样重启nginx：
```bash
sudo systemctl restart nginx
# 系统会提示输入zhangsan自己的密码
```

**`sudo` 的优势**：
1.  **审计性**：所有通过 `sudo` 执行的命令都会被记录到日志（通常是 `/var/log/secure`），便于追踪。
2.  **安全性**：不需要共享root密码，权限可以精确到单个命令。
3.  **灵活性**：可以配置免密码执行特定命令。

> **重要安全原则**：
> 1.  **最小权限原则**：只授予完成工作所必需的最小权限。不要轻易赋予 `ALL` 权限。
> 2.  **禁止编辑权限**：尽量避免授予用户能够编辑系统文件（如 `vim`、`echo >` 到系统文件）或 `sudoers` 文件本身的 `sudo` 权限，否则他们可能借此提升权限。
