# Shell脚本编程：第1章：Shell基础05

![](img/fa61eba61871094f5884990ca6472d06_0.png)

在本节课中，我们将要学习Shell脚本中的测试表达式。测试是脚本逻辑判断的基础，它允许我们检查文件状态、比较数值和字符串，从而控制脚本的执行流程。我们将详细讲解`test`命令的语法、各种比较运算符以及逻辑运算符的使用。

## 测试表达式与退出代码

上一节我们介绍了命令的退出代码。本节中我们来看看如何使用`test`命令（或其等效的方括号 `[ ]` 语法）进行条件测试。

`test`命令用于评估一个表达式。和所有命令一样，`test`命令执行后也会生成一个退出代码，存储在 `$?` 变量中。如果测试结果为真（条件成立），则退出代码为 `0`；如果测试结果为假（条件不成立），则退出代码为非零值（通常是 `1`）。

测试表达式通常使用二进制比较运算符，这些运算符需要两个操作数，并根据等式或不等式进行求值。

## 数值比较

`test`命令的数值比较仅限于整数。

以下是用于整数比较的二进制运算符：
*   `-eq`：等于（equals）
*   `-ne`：不等于（not equals）
*   `-gt`：大于（greater than）
*   `-ge`：大于等于（greater than or equals）
*   `-lt`：小于（less than）
*   `-le`：小于等于（less than or equals）

![](img/fa61eba61871094f5884990ca6472d06_2.png)

让我们通过一些例子来理解。请注意，表达式和方括号 `[`、`]` 之间必须有空格。

![](img/fa61eba61871094f5884990ca6472d06_4.png)

```bash
# 错误的示范：test不支持浮点数比较
[ 3.14 -eq 3.14 ]
echo $? # 输出非0，表示错误

![](img/fa61eba61871094f5884990ca6472d06_6.png)

![](img/fa61eba61871094f5884990ca6472d06_8.png)

![](img/fa61eba61871094f5884990ca6472d06_10.png)

# 正确的整数比较
[ 1 -eq 1 ]
echo $? # 输出0，表示真

[ 1 -ne 1 ]
echo $? # 输出1，表示假

![](img/fa61eba61871094f5884990ca6472d06_12.png)

![](img/fa61eba61871094f5884990ca6472d06_13.png)

[ 5 -ne 3 ]
echo $? # 输出0，表示真

[ 5 -gt 3 ]
echo $? # 输出0，表示真

[ 2 -ge 2 ]
echo $? # 输出0，表示真

[ 2 -lt 2 ]
echo $? # 输出1，表示假

[ 1 -le 2 ]
echo $? # 输出0，表示真
```

![](img/fa61eba61871094f5884990ca6472d06_15.png)

![](img/fa61eba61871094f5884990ca6472d06_16.png)

除了使用 `-gt` 这类选项，也可以使用双括号 `(( ))` 进行算术比较，其效果是相同的。

```bash
# 使用双括号进行算术比较
(( 5 > 3 ))
echo $? # 输出0，表示真
```

![](img/fa61eba61871094f5884990ca6472d06_18.png)

## 字符串比较

![](img/fa61eba61871094f5884990ca6472d06_20.png)

![](img/fa61eba61871094f5884990ca6472d06_21.png)

接下来，我们看看字符串比较。字符串比较主要有三种运算符。

以下是字符串比较的运算符：
*   `=` 或 `==`：等于。通常习惯使用两个等号 `==`。
*   `!=`：不等于。

![](img/fa61eba61871094f5884990ca6472d06_23.png)

![](img/fa61eba61871094f5884990ca6472d06_25.png)

![](img/fa61eba61871094f5884990ca6472d06_26.png)

```bash
# 字符串比较示例
[ "abc" = "abc" ]
echo $? # 输出0，表示真

![](img/fa61eba61871094f5884990ca6472d06_28.png)

[ "abc" == "abc" ]
echo $? # 输出0，表示真

![](img/fa61eba61871094f5884990ca6472d06_30.png)

[ "abc" = "ab" ]
echo $? # 输出1，表示假

[ "abc" != "ab" ]
echo $? # 输出0，表示真
```

![](img/fa61eba61871094f5884990ca6472d06_32.png)

![](img/fa61eba61871094f5884990ca6472d06_33.png)

**重要提示**：当在 `test` 表达式中使用 `>` 和 `<` 进行字符串大小比较时，必须使用反斜杠 `\` 进行转义，否则它们会被Shell解释为重定向符号。

![](img/fa61eba61871094f5884990ca6472d06_35.png)

字符串比较的规则需要特别注意，它比较的是字符串中**首字符的ASCII码值**，而不是字符串的长度或字母顺序。这与 `sort` 命令的排序规则可能不同。

![](img/fa61eba61871094f5884990ca6472d06_37.png)

![](img/fa61eba61871094f5884990ca6472d06_38.png)

![](img/fa61eba61871094f5884990ca6472d06_40.png)

```bash
#!/bin/bash
# 脚本：compare1.sh
var1="basketball"
var2="football"

![](img/fa61eba61871094f5884990ca6472d06_42.png)

![](img/fa61eba61871094f5884990ca6472d06_44.png)

![](img/fa61eba61871094f5884990ca6472d06_45.png)

# 情况一：未转义的 >，进行的是字符串长度比较（数值比较）
if [ $var1 > $var2 ]; then
    echo "$var1 is greater than $var2 (by length)"
elif [ $var1 \< $var2 ]; then
    echo "$var1 is less than $var2 (by length)"
fi

![](img/fa61eba61871094f5884990ca6472d06_46.png)

# 情况二：转义的 \>，进行的是字符串（首字符ASCII码）比较
if [ $var1 \> $var2 ]; then
    echo "$var1 is greater than $var2 (by ASCII)"
elif [ $var1 \< $var2 ]; then
    echo "$var1 is less than $var2 (by ASCII)"
fi
```
执行第一个 `if` 块时，比较的是 `"basketball"` 和 `"football"` 的长度（10和8），因此输出 `basketball is greater than football (by length)`。
执行第二个 `if` 块时，比较的是首字母 `'b'` (ASCII 98) 和 `'f'` (ASCII 102)，因此输出 `basketball is less than football (by ASCII)`。

![](img/fa61eba61871094f5884990ca6472d06_48.png)

关于大小写字母的比较，由于小写字母的ASCII码值大于对应的大写字母，所以在字符串比较中，小写字母“大于”大写字母。

![](img/fa61eba61871094f5884990ca6472d06_50.png)

![](img/fa61eba61871094f5884990ca6472d06_51.png)

![](img/fa61eba61871094f5884990ca6472d06_52.png)

```bash
[ "apple" \> "Apple" ]
echo $? # 输出0，表示真，因为 'a'(97) > 'A'(65)
```
这与 `ls` 或 `sort` 命令的默认排序（通常按语言环境排序，小写字母可能优先）可能相反。

## 一元运算符

![](img/fa61eba61871094f5884990ca6472d06_54.png)

![](img/fa61eba61871094f5884990ca6472d06_56.png)

![](img/fa61eba61871094f5884990ca6472d06_58.png)

一元运算符只对一个操作数进行求值，常用于检查字符串是否为空。

![](img/fa61eba61871094f5884990ca6472d06_59.png)

![](img/fa61eba61871094f5884990ca6472d06_61.png)

![](img/fa61eba61871094f5884990ca6472d06_62.png)

常用的字符串一元运算符：
*   `-z STRING`：字符串长度为0（空）则为真。
*   `-n STRING`：字符串长度非0（非空）则为真。

![](img/fa61eba61871094f5884990ca6472d06_64.png)

```bash
str=""
[ -z "$str" ]
echo $? # 输出0，表示真，字符串为空

[ -n "$str" ]
echo $? # 输出1，表示假，字符串为空，非空条件不成立

![](img/fa61eba61871094f5884990ca6472d06_66.png)

![](img/fa61eba61871094f5884990ca6472d06_67.png)

str="hello"
[ -n "$str" ]
echo $? # 输出0，表示真，字符串非空
```

## 文件和目录测试

在与文件系统交互时，测试文件和目录的状态是一种良好实践。`test` 命令提供了丰富的运算符用于此目的。

以下是一些常用的文件测试运算符：
*   `-e FILE`：文件存在即为真。
*   `-f FILE`：文件存在且为常规文件即为真。
*   `-d FILE`：文件存在且为目录即为真。
*   `-L FILE`：文件存在且为符号链接即为真。
*   `-r FILE`：文件存在且可读即为真。
*   `-w FILE`：文件存在且可写即为真。
*   `-x FILE`：文件存在且可执行即为真。
*   `-s FILE`：文件存在且大小大于0即为真。
*   `-b FILE`：文件存在且为块设备文件即为真。
*   `-c FILE`：文件存在且为字符设备文件即为真。

**再次强调**：测试表达式中的空格是语法必须的，不是可选的。

```bash
# 检查 /dev/nvme0n1p1 是否存在且为块设备
[ -b /dev/nvme0n1p1 ]
echo $?

![](img/fa61eba61871094f5884990ca6472d06_69.png)

# 检查 /dev/sr0 (光驱) 是否存在且为字符设备
[ -c /dev/sr0 ]
echo $?

![](img/fa61eba61871094f5884990ca6472d06_71.png)

![](img/fa61eba61871094f5884990ca6472d06_73.png)

# 检查 /tmp 是否存在且为目录
[ -d /tmp ]
echo $?

![](img/fa61eba61871094f5884990ca6472d06_74.png)

# 检查 /etc/passwd 是否存在且为常规文件
[ -f /etc/passwd ]
echo $?
```

## 逻辑运算符（复合测试）

为了构建更复杂的条件，我们需要逻辑运算符来组合多个测试。

逻辑运算符允许我们执行复合条件测试：
*   **逻辑与 (`&&` 或 `-a`)**：只有所有条件都为真，整个表达式才为真。使用 `&&` 连接命令时，遵循“短路”逻辑：只有前一个命令成功（返回0），才会执行下一个命令。
*   **逻辑或 (`||` 或 `-o`)**：只要有一个条件为真，整个表达式就为真。使用 `||` 连接命令时，遵循“短路”逻辑：只有前一个命令失败（返回非0），才会执行下一个命令。

```bash
# 逻辑与示例
[ 1 -eq 1 ] && [ 2 -eq 2 ]
echo $? # 输出0，两个都为真

[ 1 -eq 1 ] && [ 2 -eq 3 ]
echo $? # 输出1，第二个为假

![](img/fa61eba61871094f5884990ca6472d06_76.png)

![](img/fa61eba61871094f5884990ca6472d06_77.png)

# 使用 -a 的等效写法
[ 1 -eq 1 -a 2 -eq 2 ]
echo $? # 输出0

# 逻辑或示例
[ 1 -eq 2 ] || [ 2 -eq 2 ]
echo $? # 输出0，第二个为真

# 使用 -o 的等效写法
[ 1 -eq 2 -o 2 -eq 2 ]
echo $? # 输出0

# 短路逻辑演示
[ 1 -eq 1 ] || echo "This will NOT be printed" # 左边为真，右边不执行
[ 1 -eq 2 ] && echo "This will NOT be printed" # 左边为假，右边不执行
[ 1 -eq 2 ] || echo "This WILL be printed"     # 左边为假，右边执行
```

![](img/fa61eba61871094f5884990ca6472d06_79.png)

让我们看一个结合文件测试和逻辑运算符的脚本例子：

```bash
#!/bin/bash
# 脚本：compare3.sh
if [ -d "$HOME" -a -w "$HOME/testfile" ]; then
    echo "The file exists and you can write to it."
else
    echo "The file does NOT exist or you can NOT write to it."
fi
```
这个脚本检查家目录是否存在且可访问，并同时检查家目录下的 `testfile` 文件是否存在且可写。只有两个条件都满足，才会输出第一条信息。

![](img/fa61eba61871094f5884990ca6472d06_81.png)

本节课中我们一起学习了Shell脚本中测试表达式的核心知识。我们掌握了如何使用 `test` 命令进行数值、字符串的比较，如何检查文件和目录的状态，以及如何使用逻辑运算符 `&&`、`||`、`-a`、`-o` 来构建复杂的条件判断。理解测试表达式是编写具有决策能力脚本的关键第一步。在接下来的课程中，我们将学习如何利用这些测试结果，通过 `if`、`case` 等条件结构来控制脚本的分支执行流程。
