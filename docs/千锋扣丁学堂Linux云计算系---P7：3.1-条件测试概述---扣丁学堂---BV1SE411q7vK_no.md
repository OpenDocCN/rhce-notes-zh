# Shell脚本自动化编程实战：P7：3.1 条件测试概述 📚

![](img/e816ff5f00ac1c24d290637349220730_0.png)

在本节课中，我们将要学习Shell脚本中一个非常核心的基础点——条件测试。条件测试用于判断某个条件是否成立，是控制脚本流程的关键。我们将了解条件测试的三种主要类型，并通过具体示例学习如何使用它们。

![](img/e816ff5f00ac1c24d290637349220730_2.png)

## 条件测试简介

条件测试指的是判断某个对象或条件是否成立。在Shell脚本中，条件测试主要分为三类：**文件测试**、**字符串比较**和**数值比较**。

文件测试主要用于测试文件属性，例如判断文件是否存在、是否为常规文件、是否具有特定权限等。字符串比较用于比较两个字符串是否相等或判断字符串长度。数值比较则用于比较两个整数的大小关系。

## 条件测试的基本语法

条件测试可以使用 `test` 命令或方括号 `[ ]` 来实现。这两种方式功能相同，但方括号的写法更常见，可读性更好。

以下是使用 `test` 命令的基本语法：
```bash
test 表达式
```
如果表达式为真，则命令返回退出状态码 `0`；如果为假，则返回非零值。

以下是使用方括号 `[ ]` 的基本语法：
```bash
[ 表达式 ]
```
**注意**：方括号 `[` 和 `]` 与内部的表达式之间**必须**有空格，因为 `[` 本身是一个命令。

## 文件测试示例

![](img/e816ff5f00ac1c24d290637349220730_8.png)

![](img/e816ff5f00ac1c24d290637349220730_10.png)

上一节我们介绍了条件测试的基本概念，本节中我们来看看文件测试的具体应用。文件测试常用于在脚本中判断文件或目录的状态，以确保后续操作能够顺利进行。

![](img/e816ff5f00ac1c24d290637349220730_12.png)

以下是一些常见的文件测试操作符：

*   **`-e file`**：测试文件是否存在。
*   **`-f file`**：测试文件是否存在且为常规文件。
*   **`-d file`**：测试文件是否存在且为目录。
*   **`-r file`**：测试文件是否存在且可读。
*   **`-w file`**：测试文件是否存在且可写。
*   **`-x file`**：测试文件是否存在且可执行。

让我们通过一个简单的备份脚本示例来理解文件测试的用法。假设我们需要备份数据到一个目录，如果该目录不存在，则先创建它。

```bash
#!/bin/bash

![](img/e816ff5f00ac1c24d290637349220730_14.png)

# 定义备份目录变量
BACKUP_DIR="/var/mysql_backup"

![](img/e816ff5f00ac1c24d290637349220730_16.png)

echo "开始备份..."

![](img/e816ff5f00ac1c24d290637349220730_18.png)

# 测试备份目录是否存在
if [ ! -d "$BACKUP_DIR" ]; then
    echo "备份目录不存在，正在创建..."
    mkdir -p "$BACKUP_DIR"
fi

echo "备份操作进行中..."
# 此处可添加实际的备份命令
```

![](img/e816ff5f00ac1c24d290637349220730_20.png)

在这个脚本中，`[ ! -d "$BACKUP_DIR" ]` 就是一个条件测试。`-d` 测试 `$BACKUP_DIR` 变量代表的路径是否为一个目录。`!` 是逻辑“非”操作符，表示取反。因此，整个测试的含义是：**如果 `$BACKUP_DIR` 不是一个目录**，则执行 `if` 语句块内的命令（创建目录）。

![](img/e816ff5f00ac1c24d290637349220730_22.png)

## 字符串与数值比较示例

了解了文件测试后，我们来看看字符串和数值比较。这两种测试常用于判断用户输入、环境变量或脚本参数。

以下是字符串和数值比较的常见操作符：

**字符串比较：**
*   **`= 或 ==`**：测试两个字符串是否相等。
*   **`!=`**：测试两个字符串是否不相等。
*   **`-z string`**：测试字符串长度是否为零（空字符串）。
*   **`-n string`**：测试字符串长度是否非零。

**数值比较：**
*   **`-eq`**：等于（equal）
*   **`-ne`**：不等于（not equal）
*   **`-gt`**：大于（greater than）
*   **`-ge`**：大于等于（greater than or equal）
*   **`-lt`**：小于（less than）
*   **`-le`**：小于等于（less than or equal）

**重要提示**：字符串比较使用 `=` 和 `!=`，而数值比较必须使用 `-eq`， `-ne` 等操作符，不能使用数学符号 `>` 和 `<`（在双括号 `[[ ]]` 中可以直接使用，但这是高级特性）。

让我们看一个安装脚本的示例，它需要检查当前用户是否为root：

```bash
#!/bin/bash

# 方法1：通过数值比较 UID
if [ "$UID" -ne 0 ]; then
    echo "Error: You must have root privileges to run this script."
    exit 1
fi

![](img/e816ff5f00ac1c24d290637349220730_24.png)

# 方法2：通过字符串比较用户名
if [ "$USER" != "root" ]; then
    echo "Error: You must be root to run this script."
    exit 1
fi

![](img/e816ff5f00ac1c24d290637349220730_26.png)

echo "正在安装软件包..."
# 此处可添加实际的安装命令，例如 yum install httpd
```

![](img/e816ff5f00ac1c24d290637349220730_28.png)

在这个脚本中：
1.  `[ "$UID" -ne 0 ]` 是数值比较，判断当前用户的UID是否不等于0（root用户的UID为0）。
2.  `[ "$USER" != "root" ]` 是字符串比较，判断当前用户名是否不是“root”。
如果条件成立（用户不是root），则打印错误信息并退出脚本。

![](img/e816ff5f00ac1c24d290637349220730_30.png)

## 组合测试条件

![](img/e816ff5f00ac1c24d290637349220730_32.png)

![](img/e816ff5f00ac1c24d290637349220730_34.png)

![](img/e816ff5f00ac1c24d290637349220730_36.png)

有时我们需要测试多个条件。可以使用逻辑操作符将多个测试表达式组合起来。

![](img/e816ff5f00ac1c24d290637349220730_38.png)

![](img/e816ff5f00ac1c24d290637349220730_40.png)

*   **`-a` 或 `&&`**：逻辑与（AND）。两个条件都为真，整个表达式才为真。
    *   在单括号 `[ ]` 中使用 `-a`，例如 `[ 条件1 -a 条件2 ]`。
    *   在双括号 `[[ ]]` 中可以使用更直观的 `&&`。
*   **`-o` 或 `||`**：逻辑或（OR）。两个条件有一个为真，整个表达式即为真。
    *   在单括号 `[ ]` 中使用 `-o`。
    *   在双括号 `[[ ]]` 中可以使用 `||`。

![](img/e816ff5f00ac1c24d290637349220730_42.png)

示例：检查一个文件是否存在且可读。
```bash
if [ -f "/path/to/file" -a -r "/path/to/file" ]; then
    echo "文件存在且可读。"
fi
```

## 双括号 `[[ ]]` 扩展测试

![](img/e816ff5f00ac1c24d290637349220730_50.png)

![](img/e816ff5f00ac1c24d290637349220730_52.png)

除了单括号 `[ ]`，Bash还提供了双括号 `[[ ]]` 进行条件测试。`[[ ]]` 是关键字而非命令，功能更强大，语法也更直观。

![](img/e816ff5f00ac1c24d290637349220730_54.png)

![](img/e816ff5f00ac1c24d290637349220730_56.png)

`[[ ]]` 的优势包括：
1.  支持使用 `&&` 和 `||` 进行逻辑组合，无需转义。
2.  支持字符串模式匹配（使用 `=~` 操作符）。
3.  在字符串比较中，可以直接使用 `>` 和 `<` 进行比较（按字典顺序）。

![](img/e816ff5f00ac1c24d290637349220730_58.png)

![](img/e816ff5f00ac1c24d290637349220730_60.png)

![](img/e816ff5f00ac1c24d290637349220730_62.png)

示例：
```bash
filename="example.txt"
if [[ -f "$filename" && -r "$filename" ]]; then
    echo "文件存在且可读。"
fi

![](img/e816ff5f00ac1c24d290637349220730_64.png)

![](img/e816ff5f00ac1c24d290637349220730_66.png)

if [[ "$USER" == r* ]]; then # 模式匹配：用户名以r开头
    echo "你好，名字以r开头的用户。"
fi
```

![](img/e816ff5f00ac1c24d290637349220730_67.png)

对于初学者，在简单脚本中使用 `[ ]` 即可。当需要更复杂的模式匹配或逻辑组合时，`[[ ]]` 是更好的选择。

## 总结

![](img/e816ff5f00ac1c24d290637349220730_72.png)

![](img/e816ff5f00ac1c24d290637349220730_74.png)

本节课中我们一起学习了Shell脚本中的条件测试。我们了解到：
1.  条件测试是判断条件真假的基石，主要通过 `test` 命令或 `[ ]`、`[[ ]]` 结构实现。
2.  测试主要分为三类：**文件测试**（如 `-f`， `-d`）、**字符串比较**（如 `=`， `-z`）和**数值比较**（如 `-eq`， `-gt`）。
3.  通过逻辑操作符（`-a`， `-o` 或 `&&`， `||`）可以组合多个测试条件。
4.  双括号 `[[ ]]` 提供了比单括号 `[ ]` 更强大和直观的测试功能。

![](img/e816ff5f00ac1c24d290637349220730_76.png)

![](img/e816ff5f00ac1c24d290637349220730_77.png)

![](img/e816ff5f00ac1c24d290637349220730_79.png)

掌握条件测试是编写健壮、自动化Shell脚本的关键一步，它使得脚本能够根据不同的系统状态或用户输入做出智能决策。
