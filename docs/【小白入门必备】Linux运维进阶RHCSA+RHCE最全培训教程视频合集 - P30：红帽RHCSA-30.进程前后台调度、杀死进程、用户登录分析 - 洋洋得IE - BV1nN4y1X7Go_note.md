## 进程监控：`top`命令详解 📊

上一节我们介绍了基础的进程概念，本节中我们来看看如何使用`top`命令动态地、全面地监控系统进程状态。

`top`命令提供了一个实时更新的系统状态视图。执行`top`命令后，界面主要分为几个部分。

### 进程总体信息

![](img/f870b3d4d260fe941095b95864e0d0d7_7.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_9.png)

第一行显示了系统的概况。
*   **当前系统时间**：例如 `16:46:12`。
*   **系统运行时间**：例如 `up 1 day, 2:30`，表示系统已运行1天2小时30分钟。
*   **当前登录用户数**：例如 `2 users`。
*   **系统平均负载**：显示过去1分钟、5分钟、15分钟的平均负载，例如 `load average: 0.00, 0.01, 0.05`。

**僵尸进程**是指已经终止但其父进程尚未对其进行善后（获取终止信息、释放资源）的进程。如果发现僵尸进程数量不为0，可以通过`ps aux`过滤出僵尸进程（状态为`Z`），然后使用`kill`命令结束其父进程来清理。

*   **空闲CPU百分比**：`id` **（这是需要重点关注的值）**
*   等待I/O的CPU时间：`wa`
*   硬中断占用：`hi`
*   软中断占用：`si`

**重点关注`id`（空闲百分比）**。如果这个值持续很低（例如低于10%），说明CPU非常繁忙，需要进一步排查是哪个进程消耗了大量资源。可以使用`top`内部命令进行排序来定位问题进程。

### 内存与交换分区信息

第四行和第五行分别显示物理内存（`Mem`）和交换分区（`Swap`）的使用情况，以KiB为单位。不过，查看内存更直观的方式是使用`free -h`命令。

### `top`命令的交互操作

![](img/f870b3d4d260fe941095b95864e0d0d7_31.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_33.png)

在`top`运行界面中，可以按下特定键来改变显示或排序方式。
*   **按下 `P`**：按照CPU使用率降序排列进程。这有助于快速找到消耗CPU最多的进程。
*   **按下 `M`**：按照内存使用量降序排列进程。这有助于快速找到消耗内存最多的进程。
*   **按下 `q`**：退出`top`命令。

**总结**：`top`命令是分析系统实时状态的核心工具。我们需要关注**系统负载**、**僵尸进程数**、**CPU空闲率（`id`）**，并学会使用`P`和`M`键来排序，从而定位资源消耗大户。

## 进程检索：`pgrep`命令

除了`ps`命令结合`grep`过滤，还有一个专门用于检索进程ID的命令`pgrep`。

以下是`pgrep`命令的常用选项示例：
*   **基本检索**：`pgrep sshd` 会列出所有包含“sshd”的进程ID。
*   **显示进程名**：`pgrep -l sshd` 在列出PID的同时也显示进程名。
*   **检索指定用户**：`pgrep -u test` 检索属于`test`用户的进程。
*   **检索指定终端**：`pgrep -t pts/1` 检索在终端`pts/1`上运行的进程。
*   **精确匹配**：`pgrep -x mysqld` 只匹配进程名恰好是`mysqld`的进程。

![](img/f870b3d4d260fe941095b95864e0d0d7_39.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_41.png)

`pgrep`命令输出简洁，适合快速获取进程ID，但信息不如`ps aux`详细。

## 进程前后台调度 ⏸️▶️

![](img/f870b3d4d260fe941095b95864e0d0d7_48.png)

Linux中的进程可以运行在前台或后台。前台进程会占用当前终端，后台进程则在“背后”运行，不占用终端输入。

以下是进程调度的核心操作：
*   **将进程放入后台运行**：在命令的末尾加上 `&` 符号。例如：`sleep 300 &` 会让`sleep`命令在后台休眠300秒，终端可立刻使用。
*   **查看后台作业列表**：使用 `jobs -l` 命令。
*   **将后台作业调至前台**：使用 `fg %作业编号` 命令。例如 `fg %1`。
*   **暂停前台进程并放入后台**：按下 `Ctrl + Z` 组合键，这会挂起当前前台进程。然后使用 `bg %作业编号` 命令让它在后台继续运行。

![](img/f870b3d4d260fe941095b95864e0d0d7_50.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_52.png)

**应用场景**：当你需要运行一个耗时很长的命令（如网络测试`ping`、大数据复制），但又不想它阻塞当前终端时，就可以使用 `&` 将其放入后台执行。

## 杀死进程 🛑

![](img/f870b3d4d260fe941095b95864e0d0d7_54.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_56.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_58.png)

当需要终止一个进程时，可以使用`kill`系列命令。

### 通过进程ID（PID）杀死进程：`kill`

`kill`命令通过向进程发送特定信号来工作。常用信号有：
*   **`15` (`SIGTERM`)**：**正常终止**（默认信号）。通知进程结束，允许其完成收尾工作。
*   **`9` (`SIGKILL`)**：**强制杀死**。立即终止进程，不给进程任何清理机会。

**命令格式**：`kill [-信号] PID`
*   `kill 1234`：向PID为1234的进程发送默认的`SIGTERM(15)`信号，优雅地结束它。
*   `kill -9 1234`：强制杀死PID为1234的进程。通常用于进程无响应（“卡死”）时。

### 通过进程名杀死进程：`pkill`

`pkill`命令根据进程名来杀死进程，它会杀死所有匹配该名称的进程。
*   `pkill sleep`：杀死所有名为`sleep`的进程。
*   `pkill -9 sleep`：强制杀死所有名为`sleep`的进程。
*   `pkill -u test`：杀死`test`用户的所有进程。**这常用于将某个用户踢出系统**。
*   `pkill -t pts/1`：杀死在`pts/1`终端上运行的所有进程。

![](img/f870b3d4d260fe941095b95864e0d0d7_60.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_62.png)

### 通过终端杀死进程：`killall`

![](img/f870b3d4d260fe941095b95864e0d0d7_64.png)

`killall`与`pkill`类似，也是通过名称杀死进程，语法略有不同。
*   `killall sleep`：杀死所有名为`sleep`的进程。

![](img/f870b3d4d260fe941095b95864e0d0d7_66.png)

![](img/f870b3d4d260fe941095b95864e0d0d7_68.png)

**总结**：`kill PID`用于精确终止特定进程；`pkill 进程名`用于批量终止同类进程，特别实用的是`pkill -u 用户名`来踢出用户。

### 查看当前已登录用户

![](img/f870b3d4d260fe941095b95864e0d0d7_80.png)

以下命令按详细程度递增：
1.  **`users`**：仅显示当前登录的用户名列表，非常简洁。
2.  **`who`**：显示详细信息，包括**用户名**、**登录终端（`tty`本地/`pts`远程）**、**登录时间**和**登录来源IP地址**。
    ```bash
    root     pts/0        2022-04-16 14:46 (192.168.10.1)
    ```
3.  **`w`**：**最详细的命令**。它不仅显示`who`的信息，还额外显示系统负载（同`top`第一行）、用户空闲时间以及该用户所有进程消耗的CPU时间。
    ```bash
    16:46:12 up 1 day,  2:30,  2 users,  load average: 0.00, 0.01, 0.05
    USER     TTY      FROM             LOGIN@   IDLE   JCPU   PCPU WHAT
    root     pts/0    192.168.10.1     14:46    0.00s  0.10s  0.00s w
    ```

### 查看历史登录记录

*   **`last`**：查看**历史登录成功**的记录。显示用户名、终端、登录IP、登录起止时间。
*   **`lastb`**：查看**历史登录失败**的记录。这对于发现恶意登录尝试非常有用。记录来自`/var/log/btmp`文件。

**总结**：使用`w`或`who`查看当前谁在线上；使用`last`和`lastb`审计历史登录情况，是维护系统安全的基础。
