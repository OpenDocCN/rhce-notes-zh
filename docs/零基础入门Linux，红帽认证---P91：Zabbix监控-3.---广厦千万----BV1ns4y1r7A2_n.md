# Zabbix监控：P91：Zabbix故障报警配置教程

在本节课中，我们将学习如何在Zabbix监控系统中配置故障报警。主要内容包括配置邮件报警媒介、创建触发器来定义报警条件、设置动作来执行报警通知，以及最终验证报警流程是否正常工作。

## 概述

Zabbix具备灵活的故障告警功能，允许用户为几乎任何监控项配置多种报警方式，如电子邮件、短信、微信或钉钉。这样，管理员可以快速获知服务器问题。本教程将指导你完成邮件报警的完整配置流程。

---

## 配置邮件报警媒介

首先，我们需要配置Zabbix的邮件发送功能。这要求你的邮箱（如QQ邮箱或163邮箱）已开启SMTP服务。

以下是配置邮箱的通用步骤：
1.  登录你的邮箱（如QQ邮箱或163邮箱）。
2.  进入邮箱设置，找到“账号”或“POP3/SMTP/IMAP”相关选项。
3.  开启“POP3/SMTP服务”和“IMAP/SMTP服务”。这些服务允许第三方工具通过你的账号收发邮件。

完成邮箱设置后，我们可以在Zabbix中配置报警媒介。

1.  在Zabbix Web界面，导航至 **管理** -> **报警媒介类型**。
2.  在列表中找到类型为“电子邮件”的条目，点击进入编辑页面。
3.  配置以下关键参数：
    *   **名称**： 保持为“Email”即可。
    *   **类型**： 选择“电子邮件”。
    *   **SMTP服务器**： 输入你的邮箱SMTP服务器地址。例如：
        *   163邮箱：`smtp.163.com`
        *   QQ邮箱：`smtp.qq.com`
    *   **SMTP服务器端口**： 通常为`25`（非加密方式）。
    *   **SMTP helo**： 填写邮箱服务商的域名，如`163.com`。
    *   **SMTP电邮**： 填写你的完整邮箱地址，作为发件人。例如：`your_email@163.com`。
    *   **认证**： 勾选“用户名和密码”。
        *   **用户名称**： 填写你的邮箱账号（@符号前的部分）。
        *   **密码**： **此处非常重要**，需要填写你的**邮箱授权码**，而非邮箱登录密码。你需要在邮箱设置中生成此授权码。
4.  点击 **更新** 保存配置。
5.  你可以点击 **测试** 按钮，输入一个收件邮箱来验证配置是否成功。如果测试邮件能正常收到，说明邮件报警媒介已配置正确。

## 创建触发器

![](img/fd1a54568e858f6ca153e27ee959e7ce_5.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_7.png)

上一节我们配置了报警的发送渠道。本节中，我们来看看如何定义报警的触发条件，这需要通过“触发器”来实现。

触发器用于评估监控项收集到的数据。当数据满足特定异常条件时，触发器状态会变为“问题”，从而触发后续的报警动作，无需管理员手动在前端查看。

![](img/fd1a54568e858f6ca153e27ee959e7ce_9.png)

以下是创建触发器的步骤：
1.  导航至 **配置** -> **主机**，找到你想要配置报警的主机。
2.  点击该主机行的 **触发器**。
3.  点击右上角的 **创建触发器**。
4.  配置触发器参数：
    *   **名称**： 为触发器起一个易于识别的名字，例如“已登录用户过多”。
    *   **严重性**： 选择报警级别，例如“警告”。
    *   **表达式**： 这是核心，用于定义触发条件。点击 **添加** 来构建表达式。
        *   **监控项**： 选择你想要监控的指标，例如“已登录系统的用户数量”。
        *   **功能**： 选择评估数据的方式。常用选项有：
            *   `last()`: 最新值。适用于状态类（如服务是否运行）或瞬时阈值报警。
            *   `avg()`: 平均值。适用于需要在特定时间段内持续异常才报警的场景，如CPU负载。使用此功能时，需要指定时间范围，例如 `avg(5m)` 表示过去5分钟的平均值。
        *   **结果**： 选择判断条件，如“大于”、“小于”等。
        *   **数值**： 输入触发阈值。例如，对于“已登录系统的用户数量”，可以设置为 `> 2`。
5.  点击 **插入**，表达式会自动生成。你可以直接修改表达式中的数值来调整阈值。
6.  点击 **添加** 完成触发器创建。

![](img/fd1a54568e858f6ca153e27ee959e7ce_11.png)

**示例：创建CPU负载触发器**
假设我们想监控CPU的1分钟平均负载，当过去1分钟内平均负载持续超过0.10（即10%）时触发报警。
*   **表达式**可以配置为：`{主机:system.cpu.load[percpu,avg1].avg(1m)}>0.10`
*   这表示：对监控项“1分钟平均负载”取过去1分钟的平均值，如果该值大于0.10，则触发触发器。

创建完成后，你可以通过实际制造条件（如多开终端登录以触发用户数报警，或运行消耗CPU的脚本）来测试触发器。在 **监测** -> **问题** 或仪表板的问题区域，可以看到被触发的报警。

## 配置报警动作

![](img/fd1a54568e858f6ca153e27ee959e7ce_13.png)

触发器定义了“何时报警”，而动作则定义了“报警时做什么”。本节我们将创建一个动作，将触发器的状态变化通过邮件通知管理员。

![](img/fd1a54568e858f6ca153e27ee959e7ce_15.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_17.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_19.png)

1.  导航至 **配置** -> **动作**。
2.  点击右上角的 **创建动作**。
3.  配置动作参数：
    *   **名称**： 为动作命名，例如“用户登录报警动作”。
    *   **条件**： 添加触发此动作的条件。通常选择“触发器”，并等于你上一步创建的触发器（如“已登录用户过多”）。这意味着当该触发器被触发时，执行此动作。
4.  在 **操作** 标签页配置具体操作：
    *   **默认操作步骤持续时间**： 设置动作执行的间隔。例如设为 `10m`，表示如果问题持续存在，则每10分钟重复发送一次报警通知。
    *   **操作**： 点击 **添加**，定义单次操作详情。
        *   **发送到用户/用户组**： 必须至少选择一个。这里可以先选择一个预置组（如`Zabbix administrators`），实际收件人在用户个人设置中指定。
        *   **仅发送到**： 选择报警媒介，例如“Email”。
        *   **自定义消息**： 勾选此项以自定义邮件内容。
            *   **主题** 和 **消息**： 可以使用Zabbix预设的宏来动态生成信息。例如，故障告警的主题可以使用：
                ```
                {TRIGGER.STATUS}: {TRIGGER.NAME}
                ```
                消息内容可以使用：
                ```
                告警主机: {HOST.NAME}
                主机地址: {HOST.IP}
                告警时间: {EVENT.DATE} {EVENT.TIME}
                告警等级: {TRIGGER.SEVERITY}
                告警信息: {TRIGGER.NAME}
                问题详情: {ITEM.NAME}: {ITEM.VALUE}
                当前状态: {TRIGGER.STATUS}
                ```
5.  配置 **恢复操作**（可选但建议）：当问题解决、触发器状态恢复正常时，可以发送恢复通知。配置方式与“操作”类似，主题和消息可以调整为恢复相关的模板。
6.  点击 **添加** 完成动作创建。

**关键概念**：动作的“步骤”决定了在“默认操作步骤持续时间”内执行操作的次数。例如，持续时间为10m，步骤为1-1，则表示在这10分钟内只执行一次操作（发送一封邮件）。如果设置为1-0，则会持续重复执行。

## 设置用户与收件人

![](img/fd1a54568e858f6ca153e27ee959e7ce_33.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_35.png)

动作配置中我们指定了发送到“用户组”，但最终邮件发给哪个具体的邮箱地址，需要在相应用户的个人配置中设置。

![](img/fd1a54568e858f6ca153e27ee959e7ce_37.png)

1.  导航至 **管理** -> **用户**，找到接收报警的用户（通常是`Admin`或你创建的管理员用户）。
2.  点击用户名进入配置页面，切换到 **报警媒介** 标签页。
3.  点击 **添加**。
4.  选择类型为“Email”。
5.  在 **收件人** 栏填写接收报警的完整邮箱地址。
6.  **当启用时** 可以设置发送时间，默认`1-7,00:00-24:00`表示一周七天全天候发送。
7.  **如果类型** 选择报警严重性，例如“所有严重性”。
8.  点击 **添加**，然后 **更新** 用户信息。

![](img/fd1a54568e858f6ca153e27ee959e7ce_39.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_41.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_43.png)

至此，完整的报警链路已经配置完成：监控项收集数据 -> 触发器判断异常 -> 动作执行 -> 通过用户配置的邮箱地址发送邮件。

所有配置完成后，需要进行端到端的测试。

![](img/fd1a54568e858f6ca153e27ee959e7ce_67.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_69.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_71.png)

![](img/fd1a54568e858f6ca153e27ee959e7ce_73.png)

1.  **触发报警**： 手动制造一个满足触发器条件的情况。例如，打开三个SSH终端连接到被监控主机，使“已登录用户数量”超过2。
2.  **检查问题列表**： 在 **监测** -> **问题** 或仪表板中，确认对应的触发器状态变为“问题”。
3.  **检查动作日志**： 在 **报表** -> **动作日志** 中，查看对应的动作是否已执行，状态是否为“已送达”。
4.  **检查收件箱**： 登录你在用户报警媒介中设置的邮箱，检查是否收到了告警邮件。邮件应包含你自定义的主机、触发器、时间等信息。
5.  **测试恢复通知**： 消除报警条件（如退出多余的SSH终端）。等待片刻，检查是否收到了问题恢复的邮件。
6.  **调整触发器严重性显示（可选）**： 为了让报警邮件中的严重性级别显示为中文，可以进入 **管理** -> **一般** -> **触发器显示**，将各个严重性级别的名称从英文（如“Warning”）改为中文（如“警告”）。

## 其他功能与总结

![](img/fd1a54568e858f6ca153e27ee959e7ce_83.png)

*   **审计日志**： 在 **报表** -> **审计** 中，可以查看所有用户在Zabbix前端进行的配置更改记录，便于追溯和审计。
*   **微信/钉钉报警**： 原理类似，需要在“报警媒介类型”中配置对应的Webhook或接口信息。但请注意，某些方式（如企业微信）可能需要额外的企业资质或可信域名，配置可能更为复杂。邮件报警是通用且稳定的首选方式。

本节课中我们一起学习了Zabbix故障报警的核心配置流程。我们首先配置了邮件发送服务器，然后创建了触发器来定义监控项的异常阈值，接着设置了动作来关联触发器并定义报警行为（发送邮件），最后在用户配置中指定了具体的收件邮箱。通过完整的测试，我们验证了从问题发生到邮件通知的整个自动化报警流程。掌握这些步骤，你就能为你的Zabbix监控系统搭建起有效的故障通知机制。
