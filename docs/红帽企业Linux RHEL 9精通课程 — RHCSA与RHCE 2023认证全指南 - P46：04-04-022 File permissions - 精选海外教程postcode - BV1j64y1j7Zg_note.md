## 文件权限基础

![](img/20a34e660dd42ee6a223f127c2c4ce60_6.png)

![](img/20a34e660dd42ee6a223f127c2c4ce60_7.png)

每个文件都有一个所有者，该所有者拥有读取、写入或执行该文件的特定权限。每个文件还将具有允许一组用户（即所属组）拥有读、写或执行权限的设置。最后，每个文件都为非所有者且非所属组成员的“其他人”定义了权限。

![](img/20a34e660dd42ee6a223f127c2c4ce60_9.png)

查看特定文件权限的方法是使用 `ls` 命令与 `-l` 选项，该选项代表长列表格式。

![](img/20a34e660dd42ee6a223f127c2c4ce60_11.png)

## 常用 `ls` 命令选项

![](img/20a34e660dd42ee6a223f127c2c4ce60_15.png)

上一节我们介绍了文件权限的基本概念，本节中我们来看看如何查看这些信息。`ls` 命令是查看文件列表的基础，配合不同的选项可以获取更详细的信息。

![](img/20a34e660dd42ee6a223f127c2c4ce60_17.png)

以下是 `ls` 命令的一些常见选项：

*   `-a`：列出所有文件，包括隐藏文件（即以点 `.` 开头的文件）。
*   `-l`：以长格式显示，会输出包括权限、所有者、大小等在内的一大堆信息。
*   `-h`：与 `-l` 选项结合使用，使文件大小以人类可读的格式（如K、M、G）显示，而不是精确的字节数。
*   `-R`：递归列出目录内容，即从当前目录开始，遍历所有子文件夹。

![](img/20a34e660dd42ee6a223f127c2c4ce60_19.png)

例如，执行 `ls -lah` 可以同时查看所有文件（包括隐藏文件）的详细信息，并以易读的格式显示大小。

![](img/20a34e660dd42ee6a223f127c2c4ce60_21.png)

## 解读 `ls -l` 的输出

![](img/20a34e660dd42ee6a223f127c2c4ce60_27.png)

现在，让我们具体看看 `ls -l` 命令的输出内容。每一行都包含多组信息，理解它们对于管理权限至关重要。

以下是 `ls -l` 输出中各列的含义：

![](img/20a34e660dd42ee6a223f127c2c4ce60_29.png)

1.  **第一组**：文件类型和访问权限。
2.  **第二组**：文件的硬链接数量。
3.  **第三组**：文件所有者。
4.  **第四组**：文件所属组。
5.  **第五组**：文件大小（以字节为单位，配合 `-h` 选项可转为易读格式）。
6.  **第六组**：文件的最后修改日期和时间。
7.  **第七列**：文件的名称。

![](img/20a34e660dd42ee6a223f127c2c4ce60_31.png)

让我们详细看看第一组信息。第一个字符代表**文件类型**：

![](img/20a34e660dd42ee6a223f127c2c4ce60_33.png)

*   `-`：常规文件。
*   `d`：目录（文件夹）。
*   `l`：符号链接（软链接）。
*   `c`：字符设备文件。
*   `b`：块设备文件。
*   `s`：套接字文件。
*   `p`：命名管道文件。

![](img/20a34e660dd42ee6a223f127c2c4ce60_35.png)

第一组中剩余的9个字符称为**文件权限位**，它们被分为三组，每组三个字符：

*   **前三位**：文件**所有者**的读（`r`）、写（`w`）、执行（`x`）权限。
*   **中间三位**：文件**所属组**成员的读、写、执行权限。
*   **后三位**：**其他人**（既非所有者也非所属组成员）的读、写、执行权限。

![](img/20a34e660dd42ee6a223f127c2c4ce60_37.png)

例如，权限 `-rwxr-xr--` 表示：
*   所有者（`rwx`）：可读、可写、可执行。
*   所属组（`r-x`）：可读、可执行，但不可写。
*   其他人（`r--`）：仅可读。

## 更改文件所有者和所属组

了解了如何查看权限后，我们来看看如何修改它们。首先学习如何更改文件的所有者。

更改文件所有者需要使用 `chown` 命令。

![](img/20a34e660dd42ee6a223f127c2c4ce60_39.png)

![](img/20a34e660dd42ee6a223f127c2c4ce60_41.png)

```bash
sudo chown 新所有者 文件名
```

例如，`sudo chown student file.txt` 将 `file.txt` 的所有者改为用户 `student`。通常需要使用 `sudo` 提权，因为只有 root 用户或文件当前所有者才能更改此属性。

如果想同时更改所有者和所属组，可以使用冒号分隔：

```bash
sudo chown 新所有者:新所属组 文件名
```

![](img/20a34e660dd42ee6a223f127c2c4ce60_43.png)

例如，`sudo chown bob:developers file.txt`。

![](img/20a34e660dd42ee6a223f127c2c4ce60_45.png)

此外，还有一个专门的命令用于只更改文件的所属组，即 `chgrp` 命令。

![](img/20a34e660dd42ee6a223f127c2c4ce60_47.png)

```bash
sudo chgrp 新所属组 文件名
```

例如，`sudo chgrp root file.txt` 将文件的所属组改回 `root`。

![](img/20a34e660dd42ee6a223f127c2c4ce60_49.png)

## 更改文件权限

现在，让我们学习如何更改文件本身的权限位。为此，我们将使用 `chmod`（change mode）命令。

`chmod` 命令有两种设置权限的方法：**符号表示法**和**八进制（数字）表示法**。

### 方法一：符号表示法

这种方法使用字母来指定要修改哪类用户（`u`=所有者，`g`=组，`o`=其他人，`a`=所有用户）的权限，以及要执行的操作（`+`=添加，`-`=移除，`=`=精确设置），最后跟上权限字母（`r`， `w`， `x`）。

基本语法如下：

```bash
chmod [用户类别][操作符][权限] 文件名
```

例如：
*   `sudo chmod u+x script.sh`：为文件所有者添加执行（`x`）权限。
*   `sudo chmod o-r file.txt`：移除其他人对该文件的读取（`r`）权限。
*   `sudo chmod g=rw data.csv`：将所属组的权限精确设置为可读可写。

![](img/20a34e660dd42ee6a223f127c2c4ce60_51.png)

你可以在一条命令中用逗号分隔多个设置：

```bash
sudo chmod u-w,g+r,o=rx file.txt
```
这条命令的意思是：移除所有者的写权限，为所属组添加读权限，将其他人的权限精确设置为可读可执行。

### 方法二：八进制（数字）表示法

![](img/20a34e660dd42ee6a223f127c2c4ce60_53.png)

这种方法使用三个八进制数字（0-7）来分别代表所有者、所属组和其他人的权限。每个数字由读（4）、写（2）、执行（1）权限值相加得到。

权限值对应关系：
*   `r` (读) = 4
*   `w` (写) = 2
*   `x` (执行) = 1
*   无权限 = 0

计算方式是将所需权限的值相加：
*   `rwx` = 4+2+1 = **7**
*   `rw-` = 4+2+0 = **6**
*   `r-x` = 4+0+1 = **5**
*   `r--` = 4+0+0 = **4**
*   `---` = 0+0+0 = **0**

基本语法如下：

```bash
chmod 三位八进制数 文件名
```

例如：
*   `sudo chmod 755 script.sh`：权限为 `rwxr-xr-x`（所有者可读可写可执行，组和其他人可读可执行）。
*   `sudo chmod 644 file.txt`：权限为 `rw-r--r--`（所有者可读可写，组和其他人只读）。
*   `sudo chmod 760 secret.dat`：权限为 `rwxrw----`（所有者可读可写可执行，所属组可读可写，其他人无任何权限）。

## 总结

![](img/20a34e660dd42ee6a223f127c2c4ce60_57.png)

本节课中我们一起学习了Linux文件权限管理的核心知识。我们首先了解了文件权限的三个基本维度：所有者、所属组和其他人。接着，我们深入探讨了如何使用 `ls -l` 命令查看这些详细信息。

![](img/20a34e660dd42ee6a223f127c2c4ce60_59.png)

然后，我们掌握了两个关键的管理命令：使用 `chown` 和 `chgrp` 来更改文件的所有者和所属组。最后，我们重点学习了使用 `chmod` 命令修改文件权限的两种方法——直观的符号表示法和高效的数字（八进制）表示法。理解并熟练运用这些命令，是控制文件访问安全、备考RHCSA/RHCE认证以及进行日常系统管理的基础。
