## 📂 判断文件状态

![](img/44f2af324236795a606194824cf66cb4_9.png)

![](img/44f2af324236795a606194824cf66cb4_11.png)

上一节我们介绍了Shell的基本操作，本节中我们来看看如何判断文件的状态。Shell提供了一系列以单个字母表示的操作符，用于检查文件或目录的各种属性。其基本格式是在中括号 `[ ]` 内使用这些操作符，并且**每个部分之间必须有空格**。

![](img/44f2af324236795a606194824cf66cb4_13.png)

![](img/44f2af324236795a606194824cf66cb4_15.png)

以下是常用的文件状态判断操作符及其含义：

*   **`-e`**：判断文件或目录是否存在。
    ```bash
    [ -e /etc/passwd ] && echo "文件存在"
    ```
*   **`-d`**：判断指定路径是否为一个目录。
    ```bash
    [ -d /etc ] && echo "/etc 是一个目录"
    ```
*   **`-f`**：判断指定路径是否为一个普通文件。
    ```bash
    [ -f /etc/passwd ] && echo "/etc/passwd 是一个文件"
    ```
*   **`-r`**：判断当前用户对文件是否具有读权限。
    ```bash
    [ -r /etc/passwd ] && echo "可读"
    ```
*   **`-w`**：判断当前用户对文件是否具有写权限。
    ```bash
    [ -w /etc/passwd ] && echo "可写"
    ```
*   **`-x`**：判断当前用户对文件是否具有执行权限。
    ```bash
    [ -x /etc/passwd ] && echo "可执行"
    ```

> **提示**：这些命令执行后不会直接输出结果，需要通过 `$?` 获取上一条命令的返回值（0表示成功/真，非0表示失败/假）。现阶段，你只需要认识这些操作符的含义，在阅读脚本时能理解其作用即可。

## 🔢 整数比较

![](img/44f2af324236795a606194824cf66cb4_19.png)

![](img/44f2af324236795a606194824cf66cb4_21.png)

了解了文件判断，我们进入数值运算领域。首先学习如何进行整数比较。在Shell中，整数比较需要使用特定的操作符，不能直接使用数学符号（如 `=`， `>`），因为它们会被解释为字符串操作或重定向。

![](img/44f2af324236795a606194824cf66cb4_23.png)

以下是整数比较的操作符列表：

![](img/44f2af324236795a606194824cf66cb4_25.png)

*   **`-eq`**：等于（Equal）。
    ```bash
    [ 1 -eq 1 ] && echo “1等于1”
    ```
*   **`-ne`**：不等于（Not Equal）。
    ```bash
    [ 2 -ne 1 ] && echo “2不等于1”
    ```
*   **`-gt`**：大于（Greater Than）。
    ```bash
    [ 2 -gt 1 ] && echo “2大于1”
    ```
*   **`-ge`**：大于等于（Greater or Equal）。
    ```bash
    [ 1 -ge 1 ] && echo “1大于等于1”
    ```
*   **`-lt`**：小于（Less Than）。
    ```bash
    [ 1 -lt 2 ] && echo “1小于2”
    ```
*   **`-le`**：小于等于（Less or Equal）。
    ```bash
    [ 1 -le 1 ] && echo “1小于等于1”
    ```

## 🔤 字符串对比

![](img/44f2af324236795a606194824cf66cb4_33.png)

与整数比较不同，字符串对比使用我们更熟悉的符号。字符串对比也常用于判断变量内容或用户输入。

![](img/44f2af324236795a606194824cf66cb4_35.png)

![](img/44f2af324236795a606194824cf66cb4_37.png)

以下是字符串对比的常用方法：

![](img/44f2af324236795a606194824cf66cb4_39.png)

![](img/44f2af324236795a606194824cf66cb4_41.png)

![](img/44f2af324236795a606194824cf66cb4_43.png)

![](img/44f2af324236795a606194824cf66cb4_45.png)

*   **`==` 或 `=`**：判断两个字符串是否相等。
    ```bash
    [ “$USER” == “root” ] && echo “当前用户是root”
    ```
*   **`!=`**：判断两个字符串是否不相等。
    ```bash
    [ “$USER” != “root” ] && echo “当前用户不是root”
    ```

> **注意**：数字在字符串对比中也被当作字符处理。`[ “1” == “1” ]` 为真，但这是字符‘1’的对比，而非数学意义上的比较。

## ➕ 常用数值运算方式

在脚本中，我们经常需要进行算术运算。Shell提供了多种实现方式，各有特点。

![](img/44f2af324236795a606194824cf66cb4_55.png)

![](img/44f2af324236795a606194824cf66cb4_57.png)

![](img/44f2af324236795a606194824cf66cb4_59.png)

以下是几种常用的数值运算方法：

![](img/44f2af324236795a606194824cf66cb4_61.png)

![](img/44f2af324236795a606194824cf66cb4_63.png)

*   **`$[ ]` 表达式**：这是一种简洁的运算方式。
    ```bash
    echo $[1 + 1] # 输出 2
    echo $[5 * 2] # 输出 10
    ```
*   **`$(( ))` 表达式**：这是C语言风格的运算方式，功能与 `$[ ]` 类似。
    ```bash
    echo $((10 / 3)) # 输出 3 (整数除法)
    ```
*   **`expr` 命令**：一个古老的外部命令，使用时需要注意空格和特殊符号的转义。
    ```bash
    expr 1 + 1 # 输出 2
    expr 5 \* 2 # 乘法符号*需要转义
    ```
*   **`let` 命令**：常用于将算术运算结果赋值给变量，在脚本中很实用，支持一些简写形式。
    ```bash
    let a=1+1
    echo $a # 输出 2
    let a++ # 等同于 let a=a+1
    echo $a # 输出 3
    let a*=2 # 等同于 let a=a*2
    echo $a # 输出 6
    ```

## ❓ 字符串判断（空值判断）

除了对比字符串内容，判断一个字符串变量或文件内容是否为空也是常见操作。

![](img/44f2af324236795a606194824cf66cb4_67.png)

![](img/44f2af324236795a606194824cf66cb4_69.png)

![](img/44f2af324236795a606194824cf66cb4_71.png)

以下是字符串空值判断的操作符：

![](img/44f2af324236795a606194824cf66cb4_73.png)

![](img/44f2af324236795a606194824cf66cb4_75.png)

*   **`-z`**：判断字符串长度是否为零（Zero）。如果为空，则返回真（0）。
    ```bash
    [ -z “$myVar” ] && echo “变量myVar为空”
    ```
*   **`-n`**：判断字符串长度是否非零（Non-zero）。如果不为空，则返回真（0）。
    ```bash
    [ -n “$USER” ] && echo “USER变量非空，值为 $USER”
    ```

## ⚙️ 条件判断的简单应用：命令连接符

![](img/44f2af324236795a606194824cf66cb4_77.png)

![](img/44f2af324236795a606194824cf66cb4_79.png)

掌握了基础判断和运算后，我们来看看如何将它们与命令结合起来，实现简单的条件逻辑。Shell提供了几个命令连接符。

以下是命令连接符及其逻辑：

![](img/44f2af324236795a606194824cf66cb4_81.png)

![](img/44f2af324236795a606194824cf66cb4_83.png)

*   **`&&`（并且）**：只有**前一个命令执行成功**（返回值为0），才会执行后一个命令。
    ```bash
    # 如果vsftpd软件包已安装，则启动该服务
    rpm -q vsftpd && systemctl start vsftpd
    ```
*   **`||`（或者）**：只有**前一个命令执行失败**（返回值非0），才会执行后一个命令。
    ```bash
    # 如果vsftpd软件包未安装，则安装它
    rpm -q vsftpd || yum install -y vsftpd
    ```
*   **`;`（分号）**：顺序执行命令，**无论前一个命令成功与否**，都会执行下一个命令。命令之间没有逻辑依赖关系。
    ```bash
    # 依次执行三条命令：创建文件、查询包、列出目录
    touch /opt/test.txt; rpm -q vsftpd; ls /root
    ```

![](img/44f2af324236795a606194824cf66cb4_85.png)

**综合示例**：一个安装并启动服务的逻辑。
```bash
# 查询vsftpd是否安装，如果没安装（查询失败），则安装它。
rpm -q vsftpd || yum install -y vsftpd
# 安装成功后（或已安装），启动服务并设为开机自启。
systemctl start vsftpd && systemctl enable vsftpd
```
