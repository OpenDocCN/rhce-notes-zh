## 配置软件仓库以安装图形化工具

![](img/02a226413774ed27694003c105245250_30.png)

`firewall-config`工具在默认系统中并未安装，我们需要先配置本地软件仓库（YUM/DNF仓库）来获取并安装它。

以下是配置软件仓库的三个步骤：

### 第一步：挂载系统安装光盘

![](img/02a226413774ed27694003c105245250_34.png)

![](img/02a226413774ed27694003c105245250_36.png)

![](img/02a226413774ed27694003c105245250_38.png)

我们需要将Linux系统安装时使用的光盘镜像文件挂载到本地目录，作为软件包的来源。

![](img/02a226413774ed27694003c105245250_42.png)

![](img/02a226413774ed27694003c105245250_44.png)

![](img/02a226413774ed27694003c105245250_46.png)

1.  在虚拟机设置中，确保CD/DVD设备已连接，并指向系统安装镜像文件（即`.iso`文件）。
2.  在系统中创建一个挂载点目录：
    ```bash
    mkdir -p /media/cdrom
    ```
3.  将光盘设备挂载到该目录：
    ```bash
    mount /dev/cdrom /media/cdrom
    ```
4.  为了永久生效，需要将挂载信息写入`/etc/fstab`文件：
    ```
    /dev/cdrom /media/cdrom iso9660 defaults 0 0
    ```

### 第二步：编写软件仓库配置文件

![](img/02a226413774ed27694003c105245250_58.png)

![](img/02a226413774ed27694003c105245250_60.png)

接下来，我们需要创建DNF/YUM的仓库配置文件，告诉系统从哪里查找软件包。

![](img/02a226413774ed27694003c105245250_62.png)

![](img/02a226413774ed27694003c105245250_64.png)

1.  进入仓库配置目录：
    ```bash
    cd /etc/yum.repos.d/
    ```
2.  由于RHEL 8的光盘内容被分为`BaseOS`和`AppStream`两个目录，我们需要为两者分别创建仓库配置。新建一个以`.repo`结尾的文件，例如`local.repo`，并写入以下内容：
    ```ini
    [BaseOS]
    name=BaseOS Repository
    baseurl=file:///media/cdrom/BaseOS
    enabled=1
    gpgcheck=0

    [AppStream]
    name=AppStream Repository
    baseurl=file:///media/cdrom/AppStream
    enabled=1
    gpgcheck=0
    ```
    *   `[ ]`：仓库的唯一标识符。
    *   `name`：仓库的描述信息。
    *   `baseurl`：软件包的实际路径，`file://`表示本地文件。
    *   `enabled=1`：启用此仓库。
    *   `gpgcheck=0`：不进行GPG签名校验（适用于本地可信源）。

## 使用firewall-config配置防火墙

`firewall-config`工具提供了直观的界面来管理防火墙规则，其底层操作与`firewall-cmd`命令是完全同步的。

![](img/02a226413774ed27694003c105245250_108.png)

![](img/02a226413774ed27694003c105245250_110.png)

### 核心界面与操作

![](img/02a226413774ed27694003c105245250_112.png)

![](img/02a226413774ed27694003c105245250_114.png)

![](img/02a226413774ed27694003c105245250_116.png)

![](img/02a226413774ed27694003c105245250_118.png)

![](img/02a226413774ed27694003c105245250_120.png)

工具主界面主要包含以下几个区域和功能：
*   **重新加载**：按钮功能等同于`firewall-cmd --reload`，使永久配置立即生效。
*   **区域选择**：可以为不同网络接口（如内网、外网卡）分配不同的安全策略区域。
*   **永久生效配置**：勾选此选项后，所做的修改将保存并永久生效。
*   **服务列表**：以复选框形式列出所有可放行的预定义服务（如SSH、HTTP）。

![](img/02a226413774ed27694003c105245250_122.png)

![](img/02a226413774ed27694003c105245250_124.png)

**基本操作流程**：
1.  勾选“永久生效”选项。
2.  在服务列表中，勾选你需要放行的服务（例如，要允许SSH访问，就勾选`ssh`）。
3.  点击“重新加载”按钮，使设置生效。

### 处理未知端口或协议

![](img/02a226413774ed27694003c105245250_164.png)

考试或工作中可能会遇到需要放行未知端口的情况。

![](img/02a226413774ed27694003c105245250_166.png)

![](img/02a226413774ed27694003c105245250_168.png)

![](img/02a226413774ed27694003c105245250_170.png)

以下是几种解决方法：
1.  **通过服务名查找**：在`firewall-config`的“服务”选项卡中浏览或搜索，看该端口是否已有对应的预定义服务。
2.  **直接添加端口**：在“端口”选项卡中，点击“添加”，直接输入端口号（如`12345`）和协议（`tcp`或`udp`）。
3.  **“全部放行”策略**：如果题目只要求结果且没有冲突，一个取巧的方法是在图形界面中勾选所有服务（或大量相关服务）。红帽考试通常按点给分，只要结果符合要求，多开的服务一般不会扣分。

**重要提示**：如果考试题目只要求放行某个端口号（例如`8866`）但**未指明协议**，务必同时放行TCP和UDP协议，这是一个常见的考察点。

### 配置端口转发

![](img/02a226413774ed27694003c105245250_188.png)

![](img/02a226413774ed27694003c105245250_190.png)

我们可以用图形界面轻松实现端口转发，例如将访问本机`888`端口的流量转发到`22`端口（SSH）。

![](img/02a226413774ed27694003c105245250_192.png)

![](img/02a226413774ed27694003c105245250_194.png)

![](img/02a226413774ed27694003c105245250_196.png)

1.  在`firewall-config`的“端口转发”选项卡中，选择“IPv4”并点击“添加”。
2.  填写转发规则：
    *   **源端口**：`888`
    *   **目标端口**：`22`
    *   **目标IP**：留空或填写本机IP（表示转发到本机）。
3.  点击“确定”并“重新加载”规则。
4.  之后，就可以使用`ssh user@ip -p 888`来连接服务器了。

### 配置富规则（Rich Rules）

![](img/02a226413774ed27694003c105245250_198.png)

![](img/02a226413774ed27694003c105245250_200.png)

富规则允许进行更精细的流量控制，例如禁止特定网段访问某项服务。

以下示例是禁止`192.168.10.0/24`网段访问本机的SSH服务：
1.  在“富规则”选项卡中点击“添加”。
2.  配置规则元素：
    *   **规则**：选择`ssh`服务。
    *   **操作**：选择“拒绝”。
    *   **源地址**：填写`192.168.10.0/24`。
    *   （可选）**日志**：可以勾选记录日志，便于审计。
3.  点击“确定”并“重新加载”。配置完成后，来自该网段的SSH连接将被拒绝。

### 网卡会话（NetworkManager Connection）

![](img/02a226413774ed27694003c105245250_252.png)

![](img/02a226413774ed27694003c105245250_254.png)

网卡会话的本质是**网卡配置参数的模板**。它允许你为同一张物理网卡保存多套配置（如“公司静态IP”、“家庭DHCP”），并在不同场景下快速切换。

![](img/02a226413774ed27694003c105245250_256.png)

![](img/02a226413774ed27694003c105245250_258.png)

![](img/02a226413774ed27694003c105245250_260.png)

以下是创建和使用网卡会话的步骤：

![](img/02a226413774ed27694003c105245250_262.png)

![](img/02a226413774ed27694003c105245250_264.png)

![](img/02a226413774ed27694003c105245250_266.png)

![](img/02a226413774ed27694003c105245250_268.png)

![](img/02a226413774ed27694003c105245250_270.png)

![](img/02a226413774ed27694003c105245250_272.png)

![](img/02a226413774ed27694003c105245250_274.png)

1.  **创建“家庭”会话**（使用DHCP自动获取IP）：
    ```bash
    nmcli connection add con-name Home type ethernet ifname ens160 ipv4.method auto
    ```
2.  **创建“公司”会话**（使用固定IP）：
    ```bash
    nmcli connection add con-name Company type ethernet ifname ens160 ipv4.method manual ipv4.addresses 192.168.10.88/24 ipv4.gateway 192.168.10.1
    ```
3.  **查看所有会话**：
    ```bash
    nmcli connection show
    ```
4.  **切换会话**：
    *   切换到家庭配置：`nmcli connection up Home`
    *   切换到公司配置：`nmcli connection up Company`
    执行后，使用`ip addr show ens160`可以看到IP地址被瞬间修改。

![](img/02a226413774ed27694003c105245250_276.png)

![](img/02a226413774ed27694003c105245250_278.png)

![](img/02a226413774ed27694003c105245250_280.png)

![](img/02a226413774ed27694003c105245250_282.png)

这些会话实际上对应着`/etc/sysconfig/network-scripts/`目录下的配置文件，切换会话就是激活不同的配置文件。

### 网卡绑定（Bonding）

网卡绑定（Bonding）是将多块物理网卡**虚拟成一块逻辑网卡**的技术，主要目的是实现**负载均衡**和**故障冗余**（高可用）。

**工作原理**：以`bond0`为主逻辑网卡，`ens160`和`ens192`作为其从属网卡。所有网络配置（IP、网关等）只在`bond0`上设置。数据流由`bond0`调度到各从属网卡。当一块从属网卡故障时，流量会自动切换到另一块，保证网络不中断。

以下是配置网卡绑定的步骤（以`balance-rr`轮询模式为例）：

1.  **删除原有网卡配置文件**（因为配置将集中在bond上）：
    ```bash
    cd /etc/sysconfig/network-scripts/
    rm -f ifcfg-ens160 ifcfg-ens192
    ```
2.  **创建绑定主接口`bond0`**：
    ```bash
    nmcli connection add type bond con-name bond0 ifname bond0 bond.options "mode=balance-rr"
    ```
3.  **将两块物理网卡添加为`bond0`的从属接口**：
    ```bash
    nmcli connection add type ethernet slave-type bond con-name bond0-port1 ifname ens160 master bond0
    nmcli connection add type ethernet slave-type bond con-name bond0-port2 ifname ens192 master bond0
    ```
4.  **为`bond0`配置IP地址**：
    ```bash
    nmcli connection modify bond0 ipv4.addresses 192.168.10.10/24 ipv4.method manual
    nmcli connection up bond0
    ```
5.  **验证绑定效果**：
    *   从另一台机器持续`ping`绑定后的IP（`192.168.10.10`）。
    *   在虚拟机设置中，突然移除（或禁用）其中一块网卡（如`ens160`）。
    *   观察`ping`的结果。在理想的物理环境下，网络应几乎无感知，可能仅有个别延迟增大或丢失1个数据包。这表明另一块网卡`ens192`立即接管了流量，实现了高可用。

**模式说明**：`mode=balance-rr`（轮询模式）提供负载均衡和故障转移。还有其他模式，如`active-backup`（主备模式），平时只有一块网卡工作，另一块作为热备。

## 总结

本节课中我们一起学习了：
1.  **图形化防火墙管理**：通过配置本地软件仓库安装`firewall-config`工具，并利用其图形界面完成服务放行、端口转发、富规则设置等防火墙配置，其操作与命令行完全等效且更直观。
2.  **高级网络配置**：
    *   **网卡会话**：将网卡配置保存为可快速切换的模板，极大提升了在不同网络环境间切换的效率。
    *   **网卡绑定**：通过将多块物理网卡绑定为一块逻辑网卡，实现了网络带宽的叠加和故障时的自动切换，增强了网络的可靠性和性能。
