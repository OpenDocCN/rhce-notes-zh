## 日志管理回顾

上一节我们介绍了日志管理，本节中我们来看看具体的日志查看与配置方法。

![](img/7a64d0c98690fdc8278ee84d3d56104d_3.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_5.png)

系统日志主要由 `systemd-journald` 服务管理。日志默认保存在内存中，路径为 `/run/log/journal/`，以二进制格式存储。若需永久保存，需修改配置文件。

查看系统日志的主要命令是 `journalctl`。

以下是 `journalctl` 的常用参数：
*   `-f`：实时追踪日志末尾。
*   `-n`：显示指定行数的日志，例如 `-n 50` 显示最后50行。
*   `--since` 和 `--until`：查看特定时间段的日志。

查看单个服务的状态及日志可以使用：
```bash
systemctl status <服务名>
```
此命令会显示服务的状态和最近几行日志。

系统的重要日志文件还包括：
*   `/var/log/messages`：记录除系统启动外的大部分常规日志。
*   `/var/log/dmesg`：记录系统启动过程中的内核消息。
*   `/var/log/secure`：记录与安全认证相关的日志，如用户登录、sudo使用等。
*   `/var/log/boot.log`：记录系统启动时显示在屏幕上的信息。

![](img/7a64d0c98690fdc8278ee84d3d56104d_7.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_8.png)

要使 `journald` 的日志永久存储，需要编辑配置文件 `/etc/systemd/journald.conf`，将 `Storage=` 选项设置为 `persistent`，然后重启 `systemd-journald` 服务。永久日志将存储在 `/var/log/journal/` 目录下，但需使用 `journalctl` 命令查看。

`rsyslog` 服务负责将 `journald` 产生的日志分类并转储为文本文件。其配置文件为 `/etc/rsyslog.conf`，其中定义了日志的设施、优先级和存储路径。

日志级别从0到7，数字越小优先级越高：
*   **0**: `emerg` - 系统不可用
*   **1**: `alert` - 必须立即采取措施
*   **2**: `crit` - 严重情况
*   **3**: `err` - 错误条件
*   **4**: `warning` - 警告条件
*   **5**: `notice` - 正常但重要的情况
*   **6**: `info` - 信息性消息
*   **7**: `debug` - 调试级消息

![](img/7a64d0c98690fdc8278ee84d3d56104d_10.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_12.png)

典型的日志条目格式如下：
```
月 日 时:分:秒 主机名 服务/进程[PID]: 日志内容
```
例如：`Jul 11 09:50:01 servera systemd[1]: Started User Manager for UID 1000.`

日志轮替由 `logrotate` 管理，默认规则通常是每周轮替一次，保留4个旧日志文件。轮替后，旧文件会加上日期后缀（如 `.20230711`），然后服务会重新打开一个新日志文件。

## 时间管理回顾

上一节我们介绍了日志管理，本节中我们来看看系统时间的查看与同步。

![](img/7a64d0c98690fdc8278ee84d3d56104d_16.png)

使用 `timedatectl` 命令可以查看和设置系统时间与时区。

![](img/7a64d0c98690fdc8278ee84d3d56104d_18.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_20.png)

设置时区，例如设置为上海时间：
```bash
timedatectl set-timezone Asia/Shanghai
```
设置本地时间前，需要先关闭网络时间同步（NTP）：
```bash
timedatectl set-ntp false
timedatectl set-time “YYYY-MM-DD HH:MM:SS”
```

![](img/7a64d0c98690fdc8278ee84d3d56104d_22.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_24.png)

在RHEL 8中，时间同步服务为 `chronyd`。其配置文件为 `/etc/chrony.conf`。

![](img/7a64d0c98690fdc8278ee84d3d56104d_26.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_28.png)

配置时间服务器，例如添加一个内部NTP服务器：
```bash
server classroom.example.com iburst
```
`iburst` 参数可以在服务启动时快速进行时间同步。

修改配置后，需要重启 `chronyd` 服务以使配置生效：
```bash
systemctl restart chronyd
```
可以使用以下命令检查时间同步状态：
```bash
chronyc sources -v
```

## 网络管理回顾

上一节我们介绍了时间管理，本节中我们来看看网络接口的配置与管理。

![](img/7a64d0c98690fdc8278ee84d3d56104d_30.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_32.png)

首先，需要确认网卡是否被系统识别并存在连接配置文件。可以使用 `nmcli connection show` 查看。

如果网卡未被管理，可以手动添加连接。有两种方法：

![](img/7a64d0c98690fdc8278ee84d3d56104d_34.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_35.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_37.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_39.png)

**方法一：使用 `nmcli` 命令完整添加**
```bash
nmcli connection add type ethernet con-name <连接名> ifname <网卡设备名>
```
例如：`nmcli connection add type ethernet con-name eth256 ifname eth256`

![](img/7a64d0c98690fdc8278ee84d3d56104d_41.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_43.png)

**方法二：让 NetworkManager 自动生成连接**
```bash
nmcli device connect <网卡设备名>
```
此方法生成的连接名可能为 `Wired connection 1`，与设备名不一致。如需修改连接名以匹配设备名，可使用：
```bash
nmcli connection modify “Wired connection 1” connection.id eth256
```

在 RHEL 8 中，传统的 `network` 服务默认未安装且已被淘汰，网络管理由 `NetworkManager` 负责。重启单个网络连接的方法是：
```bash
nmcli connection reload <连接名>
nmcli connection down <连接名>
nmcli connection up <连接名>
```

![](img/7a64d0c98690fdc8278ee84d3d56104d_45.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_47.png)

配置网络信息同样有两种方法：

![](img/7a64d0c98690fdc8278ee84d3d56104d_49.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_51.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_53.png)

**方法一：使用 `nmcli` 命令配置**
```bash
nmcli connection modify <连接名> ipv4.addresses <IP地址/掩码> ipv4.gateway <网关> ipv4.dns <DNS> ipv4.method manual
nmcli connection up <连接名>
```
关键是将 `ipv4.method` 设置为 `manual`（手动）。

**方法二（推荐）：直接编辑配置文件**
网络接口的配置文件位于 `/etc/sysconfig/network-scripts/` 目录下，命名格式为 `ifcfg-<连接名>`。

![](img/7a64d0c98690fdc8278ee84d3d56104d_55.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_56.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_58.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_60.png)

一个典型的静态IP配置示例如下：
```bash
TYPE=Ethernet
BOOTPROTO=none
DEFROUTE=yes
NAME=eth256
DEVICE=eth256
ONBOOT=yes
IPADDR0=192.168.1.100
PREFIX0=24
GATEWAY0=192.168.1.1
DNS1=8.8.8.8
DNS2=8.8.4.4
```
**重要提示**：请仔细检查拼写，常见的错误如 `GATEWAY` 拼错、`PREFIX`（子网掩码）写错、`IPADDR` 后未跟数字 `0` 等，都会导致网络不通。

![](img/7a64d0c98690fdc8278ee84d3d56104d_62.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_64.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_66.png)

也可以配置全局DNS，编辑 `/etc/resolv.conf` 文件：
```bash
nameserver 8.8.8.8
nameserver 8.8.4.4
```

![](img/7a64d0c98690fdc8278ee84d3d56104d_68.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_70.png)

链路聚合可以将多个物理网卡绑定成一个逻辑接口，实现负载均衡或故障转移。虽然RHCE 8.0考试不考，但在企业环境中非常重要。配置时可以参考 `/usr/share/doc/teamd/` 目录下的示例配置文件。

上一节我们介绍了网络配置，本节中我们来看看文件的打包、压缩和传输。

![](img/7a64d0c98690fdc8278ee84d3d56104d_81.png)

Linux 中常见的三种压缩格式及其对应参数：
*   **gzip**：压缩率均衡，使用最广。对应 `tar` 命令的 `-z` 参数。
*   **bzip2**：压缩率较高。对应 `tar` 命令的 `-j` 参数。
*   **xz**：压缩率最高。对应 `tar` 命令的 `-J` 参数。

`tar` 命令是主要的归档工具，常用参数如下：
*   `-c`：创建归档文件。
*   `-x`：解压归档文件。
*   `-t`：列出归档文件内容，测试完整性。
*   `-v`：显示详细过程。
*   `-f`：指定归档文件名。
*   `-C`：解压到指定目录。

例如，创建一个用 gzip 压缩的归档包并解压到指定目录：
```bash
tar -czvf archive.tar.gz /path/to/dir
tar -xzvf archive.tar.gz -C /target/directory
```

![](img/7a64d0c98690fdc8278ee84d3d56104d_83.png)

文件传输命令：
*   **scp**：基于 SSH 的安全复制。
    ```bash
    scp local_file user@remote_host:/remote/directory
    scp user@remote_host:/remote/file local_directory
    ```
*   **rsync**：增量同步工具，效率高，常用于备份。
    ```bash
    rsync -av local_dir/ user@remote_host:/remote_dir/
    ```

## 软件包管理回顾（上）

![](img/7a64d0c98690fdc8278ee84d3d56104d_89.png)

上一节我们介绍了文件传输，本节中我们来看看软件包的基础管理。

![](img/7a64d0c98690fdc8278ee84d3d56104d_91.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_92.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_94.png)

RPM 是 Red Hat 系的软件包格式。一个完整的 RPM 包名示例：`bash-5.0.17-1.el8.x86_64.rpm`
*   `bash`：软件包名称
*   `5.0.17`：版本号
*   `1`：发行号
*   `el8`：适用于 RHEL 8
*   `x86_64`：CPU架构
*   `rpm`：文件扩展名

![](img/7a64d0c98690fdc8278ee84d3d56104d_95.png)

`rpm` 命令主要用于查询，安装时难以解决依赖关系。

![](img/7a64d0c98690fdc8278ee84d3d56104d_97.png)

以下是 `rpm` 的常用查询命令：
*   `rpm -qa`：列出所有已安装的包。
*   `rpm -qc <包名>`：查询包的配置文件列表。
*   `rpm -qd <包名>`：查询包的文档列表。
*   `rpm -qf <文件名>`：查询文件由哪个包安装。
*   `rpm -qi <包名>`：查询包的详细信息。
*   `rpm -q --scripts <包名>`：查询包安装时使用的脚本。

![](img/7a64d0c98690fdc8278ee84d3d56104d_99.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_101.png)

![](img/7a64d0c98690fdc8278ee84d3d56104d_103.png)

在 RHEL 8 中，高级软件包管理工具是 `dnf`（`yum` 是其软链接）。首先需要配置软件仓库（repository）。

本地仓库配置示例（编辑 `/etc/yum.repos.d/local.repo`）：
```bash
[BaseOS]
name=BaseOS
baseurl=file:///mnt/BaseOS
enabled=1
gpgcheck=0

![](img/7a64d0c98690fdc8278ee84d3d56104d_105.png)

[AppStream]
name=AppStream
baseurl=file:///mnt/AppStream
enabled=1
gpgcheck=0
```
**重要提示**：
1.  RHEL 8 的仓库分为 `BaseOS`（基础系统包）和 `AppStream`（应用流包），通常需要同时配置。
2.  `baseurl` 中，`file://` 表示文件协议，后面跟三个斜杠，第三个斜杠是挂载点的根路径。
3.  注意拼写，如 `enabled`、`gpgcheck` 不要写错。

`dnf`/`yum` 的基本用法：
*   `dnf clean all`：清理缓存。
*   `dnf makecache`：建立元数据缓存。
*   `dnf repolist`：列出已配置的仓库。
*   `dnf install <包名>`：安装软件包。
*   `dnf remove <包名>`：移除软件包。
*   `dnf history`：查看事务历史。
*   `dnf provides <文件名>`：查询哪个包提供指定文件。
