## 创建虚拟机框架

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_10.png)

在安装操作系统之前，我们需要先在虚拟机软件中创建一个“空壳”电脑，即定义虚拟机的硬件配置。

操作步骤如下：

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_14.png)

1.  打开虚拟机软件，选择“创建新的虚拟机”。
2.  选择“典型”配置，点击下一步。
3.  **关键步骤**：在“安装客户机操作系统”界面，**务必选择“稍后安装操作系统”**。如果此处直接选择了镜像文件，虚拟机将使用简易安装模式，导致安装结果与教程不符。
4.  选择客户机操作系统为“Linux”，版本为“Red Hat Enterprise Linux 7 64位”。
5.  为虚拟机命名（例如“我的第一台Linux系统”），并选择安装位置（建议放在空间充足的分区）。
6.  指定磁盘容量，例如20GB，然后点击“完成”。

## 配置虚拟机硬件

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_29.png)

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_30.png)

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_32.png)

创建好虚拟机框架后，我们需要对其硬件进行微调，以优化性能并移除不必要的设备。

以下是具体的配置建议：

*   **内存**：根据物理机内存大小分配。若物理内存≤2GB，分配1GB；若>2GB，分配2GB即可。
*   **处理器**：处理器核心数建议与物理机保持一致，可在“任务管理器”中查看。若不确定，设置为1个核心也可。
*   **镜像关联**：在“虚拟机设置”中，选择“CD/DVD”，关联之前下载好的RHEL 7.0系统镜像文件。
*   **网络适配器**：暂时选择“仅主机模式”，网络详细知识将在后续章节讲解。
*   **移除无用设备**：建议移除“USB控制器”、“声卡”和“打印机”。移除声卡可以避免输入错误命令时系统提示音干扰思绪。

配置完成后，即可开启此虚拟机进行系统安装。

## 安装红帽企业版Linux系统

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_34.png)

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_36.png)

现在，我们将开始正式的安装过程。请使用键盘上下方向键选择“Install Red Hat Enterprise Linux 7.0”并回车。

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_38.png)

安装过程需要注意以下几个关键点：

*   **虚拟化支持**：如果启动时报错提示“虚拟化技术未开启”，则需要进入电脑BIOS设置中开启VT（虚拟化技术）。具体方法因电脑品牌和主板型号而异，请根据型号百度搜索“BIOS开启VT”教程。
*   **安装选项**：在语言选择界面，建议选择“English”。在“SOFTWARE SELECTION”界面，**务必选择“Server with GUI”**（带图形界面的服务器安装），否则会安装成纯命令行界面。
*   **安装目的地**：直接选择提供的硬盘，点击“Done”即可。
*   **网络与主机名**：网络可以暂时不开启，后续章节会详细讲解配置方法。
*   **设置密码**：为root管理员用户设置一个密码。同时可以创建一个普通用户（建议用户名与教程保持一致，便于后续实验）。

安装过程需要一定时间，请耐心等待。安装完成后，点击“Reboot”重启系统。

## Linux软件管理基础

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_60.png)

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_61.png)

在系统安装的等待时间里，我们补充一些重要的理论知识，帮助您理解Linux软件的安装机制。

Linux软件安装方式主要有以下三种，它们的发展是为了不断降低安装难度：

1.  **源代码安装**
    *   **早期方式**：直接编译程序的源代码进行安装。
    *   **弊端**：安装复杂，且需要手动解决软件间的依赖关系（即一个软件运行需要先安装其他软件）。

2.  **RPM（红帽软件包管理器）**
    *   **改进**：将程序的源代码和一套安装规则打包在一起，形成 `.rpm` 文件。
    *   **目的**：**降低软件的安装难度**。用户无需手动编译和配置。
    *   **公式**：`源代码 + 安装规则 = RPM软件包`

3.  **YUM仓库**
    *   **进一步改进**：将大量常用的RPM软件包集中存放在一个服务器（仓库）中。
    *   **目的**：**进一步降低软件的安装难度**。自动解决软件依赖关系，并根据用户需求自动安装所有相关软件包。
    *   **原理**：`用户命令` -> `YUM` -> `从仓库查找软件包并解决依赖` -> `自动下载并安装`

在后续一个半月的课程中，我们将频繁使用YUM仓库来安装和管理软件。

## 系统服务管理命令

系统安装好后，我们需要学习如何管理它。在RHEL 7中，系统服务由 `systemd` 进程管理，使用 `systemctl` 命令进行操作。

以下是必须掌握的5条核心服务管理命令，请务必熟记：

| 命令 | 作用 |
| :--- | :--- |
| `systemctl start 服务名` | 启动某个服务 |
| `systemctl stop 服务名` | 停止某个服务 |
| `systemctl restart 服务名` | 重启某个服务 |
| `systemctl enable 服务名` | 将服务设为开机自启 |
| `systemctl status 服务名` | 查看服务的运行状态 |

> **命令示例**：`systemctl start firewalld` （启动防火墙服务）

## Shell：人与硬件的翻译官

要执行命令，首先需要理解命令的“运行环境”——Shell。

我们可以将计算机抽象为以下层次：
`人` -> `Shell` -> `内核` -> `硬件`

*   **内核**：直接管理硬件的核心程序。
*   **Shell**：充当“翻译官”的角色。它接收用户输入的命令，将其“翻译”成内核能理解的指令来操作硬件，同时将硬件的执行结果“翻译”成人能看懂的信息反馈回来。
*   **Bash Shell**：它是当前大多数Linux系统默认的Shell解释器。我们学习它的原因有两点：1) **通用性强**；2) **非常好用**。

## Linux命令格式解析

一条完整的Linux命令通常由三部分组成：

**命令 参数 对象**

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_72.png)

*   **命令**：要执行的操作，即“做什么”。
*   **参数**：对命令的修饰，指定如何执行，即“怎么做”。通常以 `-`（短格式）或 `--`（长格式）开头。
*   **对象**：命令操作的目标，即“对谁做”。

![](img/c913bddf0d3af5f0c3173d50ef5ec72c_74.png)

**格式规则：**
1.  命令、参数、对象之间**必须用空格分隔**（至少一个空格）。
2.  参数可有可无，对象也依命令而定。
3.  多个**短格式参数可以合并**，只保留一个 `-`。
    *   **正确示例**：`ls -l -a` 可以合并为 `ls -la`
    *   **错误示例**：长格式之间、长格式与短格式之间不能合并。

**类比举例：**
*   **命令**：`亲` （动作）
*   **参数**：`温柔的` （修饰动作的方式）
*   **对象**：`小红` （动作的承受者）
*   **完整命令**：`温柔的 亲 小红`
