# Linux运维进阶：P15：用户管理、用户信息文件详解 👤

![](img/57d30da4735109c2906dca92138060c9_1.png)

在本节课中，我们将要学习Linux系统中的用户管理，包括用户账号的作用、分类，以及如何创建和管理用户。我们将重点解析存储用户核心信息的 `/etc/passwd` 文件，并学习 `useradd` 命令的常用选项。

## 用户账号概述

![](img/57d30da4735109c2906dca92138060c9_3.png)

![](img/57d30da4735109c2906dca92138060c9_5.png)

用户账号是登录系统所必需的凭证。在Linux系统中，主要存在两类用户：
*   **超级管理员 (root)**：拥有系统最高权限的用户。
*   **普通用户**：由管理员创建，权限受到限制的用户。

![](img/57d30da4735109c2906dca92138060c9_7.png)

![](img/57d30da4735109c2906dca92138060c9_9.png)

![](img/57d30da4735109c2906dca92138060c9_11.png)

在企业环境中，`root` 账号权限过大，通常只分配给核心运维人员或部门领导。普通员工具备的通常是普通用户账号，以降低误操作风险。

## 用户模板目录

在深入了解用户创建之前，我们先了解一个概念：用户模板目录 `/etc/skel/`。

这个目录下的隐藏文件（如 `.bashrc`, `.bash_profile`）会在创建新用户时，被自动复制到该用户的家目录中。这些文件为用户提供了初始的命令别名、环境变量等配置。了解此目录的存在即可，无需深入探究。

## 创建用户：`useradd` 命令

![](img/57d30da4735109c2906dca92138060c9_13.png)

![](img/57d30da4735109c2906dca92138060c9_15.png)

创建用户的命令是 `useradd`，此命令仅限 `root` 用户使用。

![](img/57d30da4735109c2906dca92138060c9_17.png)

其基本语法非常简单：`useradd [选项] 用户名`。

![](img/57d30da4735109c2906dca92138060c9_19.png)

例如，创建一个名为 `user1` 的用户：
```bash
useradd user1
```
用户创建后，需要为其设置密码才能用于登录系统。

![](img/57d30da4735109c2906dca92138060c9_21.png)

## 用户信息文件：`/etc/passwd`

![](img/57d30da4735109c2906dca92138060c9_23.png)

用户创建后，其基本信息存储在 `/etc/passwd` 文件中。这个文件至关重要，每一行代表一个用户，每行内容以冒号 `:` 分隔为7个字段。

以下是查看该文件内容的命令：
```bash
cat /etc/passwd
```

接下来，我们详细解释这7个字段的含义：

**1. 用户名**
第一列是用户的登录名。

![](img/57d30da4735109c2906dca92138060c9_25.png)

**2. 密码占位符**
第二列历史上用于存放加密密码，现在密码已移至 `/etc/shadow` 文件。此处统一用 `x` 表示。

**3. 用户ID (UID)**
第三列是用户的唯一身份标识号。
*   **UID = 0**：超级管理员 (`root`)。
*   **UID = 1-999**：系统伪用户，通常用于运行系统服务，不能登录系统。
*   **UID >= 1000**：普通用户。

![](img/57d30da4735109c2906dca92138060c9_27.png)

**4. 组ID (GID)**
第四列是用户**基本组（初始组）**的ID号。创建用户时，系统会默认创建一个与用户名同名的组作为其基本组。

**5. 描述信息**
第五列是用户的备注信息，如部门、联系电话等，可为空。

**6. 家目录**
第六列是用户登录后所在的初始目录，即家目录。普通用户通常在 `/home/用户名` 下。

![](img/57d30da4735109c2906dca92138060c9_29.png)

**7. 登录Shell**
第七列指定用户登录后使用的命令解释器。
*   `/bin/bash`：标准的、可交互登录的Shell。
*   `/sbin/nologin`：禁止用户登录系统的Shell，常用于系统伪用户。

![](img/57d30da4735109c2906dca92138060c9_31.png)

![](img/57d30da4735109c2906dca92138060c9_33.png)

## `useradd` 命令常用选项

上一节我们介绍了用户信息文件的构成，本节中我们来看看如何在使用 `useradd` 命令时定制这些属性。

![](img/57d30da4735109c2906dca92138060c9_35.png)

![](img/57d30da4735109c2906dca92138060c9_37.png)

`useradd` 命令支持多个选项来指定用户的属性：

以下是常用选项列表：
*   `-u UID`：手动指定用户的UID。
    ```bash
    useradd -u 6666 user2
    ```
*   `-d 目录`：指定用户的家目录路径。
    ```bash
    useradd -d /opt/user3 user3
    ```
*   `-c “备注”`：添加用户的描述信息。
    ```bash
    useradd -c “运维部，电话：123456” user4
    ```
*   `-g 组名`：指定用户的基本组（初始组）。
    ```bash
    useradd -g developers user5
    ```
*   `-G 组名1,组名2,...`：指定用户的附加组。
    ```bash
    useradd -G dev,test user6
    ```
*   `-s Shell路径`：指定用户的登录Shell。
    ```bash
    useradd -s /sbin/nologin user7 # 创建不能登录的用户
    ```

这些选项可以组合使用，例如：
```bash
useradd -u 2001 -c “开发工程师” -G dev,ops -s /bin/bash user8
```

## 查看用户和组信息：`id` 命令

创建用户后，可以使用 `id` 命令查看用户的UID、GID以及所属的所有组。

命令格式为：`id [用户名]`。

![](img/57d30da4735109c2906dca92138060c9_39.png)

例如：
```bash
id user1
```
输出会显示用户的 `uid`、`gid` 以及 `groups`（所属的组列表）。
