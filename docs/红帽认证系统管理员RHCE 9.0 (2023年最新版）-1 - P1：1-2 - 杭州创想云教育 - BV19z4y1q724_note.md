## 使用GNOME桌面

我们当前使用的是基于Wayland驱动的GNOME 40桌面。登录成功后，界面主要区域如下：

1.  **活动概览 (Activities)**：点击左上角或按 `Windows` 键，可以查看所有窗口的缩略图。
2.  **功能区域**：屏幕右上角，可以查看网络连接、电源模式、进行系统设置等。
3.  **仪表盘 (Dash)**：屏幕底部，显示收藏和最近使用的应用程序图标。
4.  **显示应用程序 (Show Applications)**：点击仪表盘右侧的网格图标，显示所有已安装的应用程序。
5.  **面板 (Top Bar)**：屏幕顶部，显示时间、日期和搜索栏。
6.  **工作空间 (Workspaces)**：可以创建多个虚拟桌面来组织工作。

### 常用桌面操作

以下是几个基本的桌面操作步骤：

*   **更换壁纸**：在桌面空白处点击鼠标右键，选择 **Change Background**，然后选择喜欢的壁纸。
*   **系统设置**：点击右上角功能区域，选择 **Settings**。在这里可以管理网络、蓝牙、显示器分辨率、电源选项等。
    *   **注意**：课堂环境中请勿更改网络设置。
*   **打开终端**：点击 **Activities**，在搜索栏或仪表盘中找到并点击 **Terminal** 图标。

## 命令语法结构

在终端中，命令的基本格式如下：
```
$ command [options] [arguments]
```
*   **`$`**：代表命令提示符。
*   **`command`**：命令本身，本质上是一个可执行的二进制文件。
*   **`[options]`**：选项，用于修改命令的行为，可选。
*   **`[arguments]`**：参数，命令执行的目标，可选。

可以将这个结构类比为：
*   **命令**：汽车（执行操作的工具）。
*   **选项**：导航仪（改变行驶行为，如路线）。
*   **参数**：目的地（操作的目标）。

### 选项的格式

常见的选项有两种风格：
1.  **UNIX风格**：短横线后跟单个字母，例如 `-l`。
2.  **GNU风格**：两个短横线后跟完整单词，例如 `--help`。

**示例**：使用 `ls` 命令列出文件。
*   `ls`：仅列出当前目录下的文件名。
*   `ls -l`：以长列表格式显示详细信息（UNIX风格）。
*   `ls --help`：显示该命令的帮助信息（GNU风格）。

![](img/07b351b483f3f5847fabc8e46ce9d93f_39.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_41.png)

**短格式优点**：可以合并，例如 `ls -l -a` 可以写成 `ls -la`。
**长格式优点**：含义更清晰，易于记忆。

## 常用基础命令

以下是一些初学者必须掌握的基础命令：

![](img/07b351b483f3f5847fabc8e46ce9d93f_43.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_45.png)

*   **`date`**：显示或设置系统日期和时间。
    *   `date`：显示当前日期和时间。
    *   `date +%s`：显示自1970年1月1日（Unix纪元）以来的秒数。
*   **`wc`**：统计文件的行数、单词数和字符数。
    *   `wc -l /etc/passwd`：统计 `/etc/passwd` 文件的行数。
*   **`cat`**：连接文件并打印到标准输出，常用于查看较短的文本文件内容。
    *   `cat /etc/passwd`：查看 `/etc/passwd` 文件内容。
*   **`file`**：判断文件类型。
    *   `file /dev/tty1`：判断 `/dev/tty1` 文件的类型。
*   **`head` / `tail`**：查看文件的开头或结尾部分。
    *   `head -n 5 /etc/passwd`：查看文件前5行。
    *   `tail -n 5 /etc/passwd`：查看文件末尾5行。
*   **`more` / `less`**：分页查看长文本文件。
    *   `less /etc/passwd`：使用 `less` 查看文件，支持上下翻页和搜索。

## 终端实用技巧

熟练使用以下技巧可以极大提高命令行操作效率：

*   **命令自动补全**：输入命令或路径的前几个字母后，按 `Tab` 键可以自动补全。如果有多项匹配，按两次 `Tab` 会列出所有可能性。
*   **命令历史**：
    *   `history`：查看之前执行过的命令历史列表。
    *   `!36`：执行历史记录中第36行的命令。
    *   `!ls`：执行最近一次以 `ls` 开头的命令。
*   **快捷键**：
    *   `Ctrl + A`：光标移动到行首。
    *   `Ctrl + E`：光标移动到行尾。
    *   `Ctrl + U`：删除光标之前的所有内容。
    *   `Ctrl + K`：删除光标之后的所有内容。
    *   `Ctrl + R`：反向搜索历史命令。

## 文件系统基础

![](img/07b351b483f3f5847fabc8e46ce9d93f_47.png)

Linux文件系统采用倒置的树状结构，根目录 (`/`) 位于最顶端。以下是几个重要的顶级目录：

![](img/07b351b483f3f5847fabc8e46ce9d93f_49.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_51.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_53.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_55.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_57.png)

*   `/bin`：存放基本的用户命令二进制文件。
*   `/boot`：存放系统启动所需的文件。
*   `/dev`：存放设备文件。
*   `/etc`：存放系统配置文件。
*   `/home`：普通用户的家目录。
*   `/root`：root用户的家目录。
*   `/usr`：存放用户应用程序和文件。
*   `/var`：存放经常变化的文件，如日志。

### 路径

![](img/07b351b483f3f5847fabc8e46ce9d93f_59.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_61.png)

路径分为两种：
*   **绝对路径**：从根目录 (`/`) 开始的完整路径，如 `/home/student/Documents`。
*   **相对路径**：相对于当前工作目录的路径，如 `./Documents` 或 `../otherdir`。
    *   `.` 代表当前目录。
    *   `..` 代表上一级目录。

**查看当前路径**：`pwd`
**切换目录**：`cd [目录路径]`
**列出目录内容**：`ls`

## 文件与目录管理

以下是管理文件和目录的核心命令：

*   **创建目录**：`mkdir`
    *   `mkdir newdir`：创建名为 `newdir` 的目录。
    *   `mkdir -p parent/child`：创建多级目录，如果父目录不存在则一并创建。
*   **复制**：`cp`
    *   `cp file1 file2`：复制 `file1` 到 `file2`。
    *   `cp -r dir1 dir2`：递归复制目录 `dir1` 到 `dir2`。
*   **移动/重命名**：`mv`
    *   `mv oldname newname`：重命名文件或目录。
    *   `mv file /tmp/`：将文件移动到 `/tmp` 目录。
*   **删除**：
    *   `rm file`：删除文件。
    *   `rm -r dir`：递归删除目录及其内容。
    *   `rmdir dir`：删除空目录。

**重要警告**：`rm` 命令删除的文件不会进入回收站，很难恢复。建议使用 `-i` 选项进行交互式删除（`rm -i file`），或养成备份习惯。

## 链接文件

链接分为硬链接和软链接（符号链接），类似于Windows的快捷方式，但有本质区别。

![](img/07b351b483f3f5847fabc8e46ce9d93f_66.png)

*   **创建硬链接**：`ln source_file hard_link`
    *   硬链接与源文件共享相同的inode（数据块），删除源文件不影响硬链接访问数据。
    *   不能跨文件系统创建，不能为目录创建。
*   **创建软链接**：`ln -s source_file soft_link`
    *   软链接是一个独立的文件，内容是指向源文件的路径。
    *   可以跨文件系统，可以为目录创建。
    *   如果源文件被删除，软链接将失效（称为“断链”）。

![](img/07b351b483f3f5847fabc8e46ce9d93f_68.png)

![](img/07b351b483f3f5847fabc8e46ce9d93f_70.png)

**示例**：
```
$ touch source.txt
$ ln source.txt hardlink.txt    # 创建硬链接
$ ln -s source.txt softlink.txt # 创建软链接
$ ls -li # 查看inode号，硬链接与源文件相同，软链接不同
```

## Shell通配符

通配符用于匹配文件名，非常高效：

*   `*`：匹配任意多个字符（包括零个）。
    *   `ls *.txt`：列出所有 `.txt` 文件。
*   `?`：匹配任意单个字符。
    *   `ls file?.txt`：列出 `file1.txt`, `fileA.txt` 等。
*   `[ ]`：匹配括号内的任意一个字符。
    *   `ls file[123].txt`：列出 `file1.txt`, `file2.txt`, `file3.txt`。
*   `[! ]` 或 `[^ ]`：匹配不在括号内的任意一个字符。
    *   `ls file[!1].txt`：列出不以 `file1.txt` 命名的其他 `file?.txt`。

**花括号扩展**：`{ }`
*   `touch file_{a,b,c}.txt`：创建 `file_a.txt`, `file_b.txt`, `file_c.txt` 三个文件。

## 获取帮助：man手册

当你不熟悉某个命令时，最好的老师是 `man`（manual）手册。

**手册章节**：`man` 手册分为9个章节，常用的是：
*   `1`：用户命令。
*   `5`：文件格式。
*   `8`：系统管理命令。

**使用 `man`**：
*   `man ls`：查看 `ls` 命令的手册（默认进入第1章）。
*   `man 5 passwd`：查看 `/etc/passwd` 文件格式的说明（第5章）。

**在 `man` 页面中导航**：
*   `空格键` / `Page Down`：向下翻页。
*   `b` / `Page Up`：向上翻页。
*   `/keyword`：搜索关键词，按 `n` 查找下一个，`N` 查找上一个。
*   `q`：退出手册。

**搜索手册**：
*   `man -k keyword`：在所有手册页中搜索包含 `keyword` 的描述。
*   如果搜索无结果，可能需要以root权限更新手册数据库：`sudo mandb`。
