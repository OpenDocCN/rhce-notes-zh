## 概述

在本节课中，我们将要学习Linux系统中的磁盘管理。我们将从磁盘的物理结构和工作原理讲起，了解不同的磁盘接口类型，然后重点学习两种主流的磁盘分区格式：MBR和GPT。最后，我们会掌握在Linux系统中查看和管理磁盘的常用命令，为后续的实际分区操作打下基础。

## 磁盘物理结构与工作原理

上一节我们介绍了课程的整体内容，本节中我们来看看磁盘的物理结构。

在传统的物理服务器中，硬盘的外观类似于一块“板砖”。服务器机箱内部有专门的硬盘插槽，硬盘可以直接插入。现代的服务器硬盘大多支持**热插拔**，这意味着可以在服务器开机运行的状态下直接插入或拔出硬盘，系统能够立即识别，就像使用U盘一样方便。

硬盘的内部结构主要由以下几部分组成：
*   **主轴马达**：带动盘片旋转。
*   **盘片**：数据实际存储的介质，类似于光盘。
*   **机械臂与读写磁头**：机械臂带动读写磁头在高速旋转的盘片上移动，进行数据的读取和写入。

需要注意的是，硬盘内部是**真空**的，以防止灰尘进入导致盘片划伤和数据丢失。因此，普通用户切勿自行拆解硬盘。

## 磁盘的核心概念：扇区

了解了物理结构后，我们需要理解一个核心逻辑概念：扇区。

扇区是磁盘存储数据的**最小单元**。操作系统在划分和管理磁盘空间时，是以扇区为单位进行计算的。
*   默认情况下，一个扇区的大小是 **512字节**。
*   当我们创建一个10GB的分区时，系统实际上是在计算需要多少个512字节的扇区才能组成10GB的空间。

对于用户和管理员来说，我们通常直接指定分区大小（如10GB、20GB），而无需关心底层的扇区数量换算。

## 磁盘接口类型

随着技术发展，出现了多种磁盘接口类型，它们在速度、价格和应用场景上有所不同。

以下是几种常见的磁盘接口类型：
*   **IDE**：一种古老的接口，数据传输速度慢，价格便宜。大约在2009年后已在服务器领域淘汰，曾主要用于个人计算机。
*   **SCSI**：在2013年至2017年间在服务器领域非常流行。数据传输速度快，并支持热插拔。
*   **SATA**：目前个人计算机和部分服务器常用的接口，速度较快。
*   **SAS**：目前在服务器领域主流的接口类型，由SCSI发展而来，性能更优。
*   **SSD (固态硬盘)**：使用闪存颗粒存储数据，没有机械部件。其数据传输速度远快于传统的机械硬盘，因为利用了类似内存的快速存储机制。
*   **NVMe**：一种用于固态硬盘的高速通信协议，能够充分发挥SSD的性能。

选择哪种接口取决于实际需求（如速度、可靠性）和预算。对于企业级应用，通常选择SAS或支持NVMe协议的高性能SSD。

## 磁盘分区格式：MBR vs GPT

有了硬盘之后，必须对其进行分区才能使用。分区格式决定了如何组织和管理磁盘上的分区。主要有两种格式：MBR和GPT。

### MBR (主引导记录)

MBR是一种较老的分区格式。
*   **分区数量限制**：一块磁盘最多只能创建**4个主分区**。
*   **扩展分区与逻辑分区**：为了突破4个分区的限制，可以将第四个分区创建为**扩展分区**。扩展分区本身不能直接存储数据，但它是一个“容器”，可以在其内部创建多个**逻辑分区**来存储数据。
    *   IDE磁盘最多可划分59个逻辑分区。
    *   SCSI/SAS等磁盘最多可划分11个逻辑分区。
*   **磁盘容量限制**：MBR最大只支持 **2TB** 以内的磁盘容量。对于大于2TB的磁盘，无法使用MBR格式进行全部分区。

### GPT (GUID 分区表)

![](img/db3503b58daa94374d15395f108d8123_10.png)

![](img/db3503b58daa94374d15395f108d8123_12.png)

![](img/db3503b58daa94374d15395f108d8123_13.png)

![](img/db3503b58daa94374d15395f108d8123_14.png)

![](img/db3503b58daa94374d15395f108d8123_16.png)

![](img/db3503b58daa94374d15395f108d8123_18.png)

GPT是一种现代的分区格式，旨在克服MBR的限制。
*   **分区数量**：几乎无限制，理论上可创建**128个主分区**，无需使用扩展分区和逻辑分区这种复杂概念。
*   **磁盘容量限制**：最大支持 **18EB** 的磁盘容量。容量换算关系如下：
    `1024 Byte = 1 KB; 1024 KB = 1 MB; 1024 MB = 1 GB; 1024 GB = 1 TB; 1024 TB = 1 PB; 1024 PB = 1 EB`
    18EB是一个极其庞大的存储空间。

![](img/db3503b58daa94374d15395f108d8123_20.png)

![](img/db3503b58daa94374d15395f108d8123_22.png)

![](img/db3503b58daa94374d15395f108d8123_24.png)

**总结与选择**：
*   对于小于2TB的磁盘，MBR和GPT都适用。
*   对于大于2TB的磁盘，必须使用GPT格式。
*   对于需要大量分区（超过4个）的场景，GPT更为简单直接。

以下是管理磁盘时最常用的几条命令：

![](img/db3503b58daa94374d15395f108d8123_67.png)

**1. `lsblk` 命令**
此命令用于列出系统中所有块设备（如硬盘、分区）的简明信息，是查看磁盘布局最直观的工具。

![](img/db3503b58daa94374d15395f108d8123_73.png)

![](img/db3503b58daa94374d15395f108d8123_75.png)

常用示例：
```bash
lsblk
```
该命令会以树状结构显示所有磁盘（如 `sda`, `sdb`）及其下的分区（如 `sda1`, `sda2`）。

![](img/db3503b58daa94374d15395f108d8123_77.png)

![](img/db3503b58daa94374d15395f108d8123_78.png)

**2. `df` 命令**
此命令用于报告文件系统（即已挂载分区）的磁盘空间使用情况。

![](img/db3503b58daa94374d15395f108d8123_80.png)

![](img/db3503b58daa94374d15395f108d8123_82.png)

![](img/db3503b58daa94374d15395f108d8123_84.png)

![](img/db3503b58daa94374d15395f108d8123_86.png)

常用示例：
```bash
df -h
```
选项 `-h` 代表“人类可读”，会以 K, M, G 为单位显示空间使用量，而不是以字节为单位。该命令可以查看各个挂载分区的总容量、已用量、可用量及使用百分比。

![](img/db3503b58daa94374d15395f108d8123_88.png)

![](img/db3503b58daa94374d15395f108d8123_90.png)

![](img/db3503b58daa94374d15395f108d8123_92.png)

![](img/db3503b58daa94374d15395f108d8123_94.png)

**3. `du` 命令**
此命令用于估算文件或目录的磁盘使用空间。与 `ls` 命令查看文件“大小”不同，`du` 统计的是文件实际占用的“磁盘空间”。

![](img/db3503b58daa94374d15395f108d8123_96.png)

![](img/db3503b58daa94374d15395f108d8123_98.png)

常用示例：
```bash
du -sh /etc
```
*   `-s`：仅显示总计，不显示每个子目录的详情。
*   `-h`：以人类可读格式显示。
*   `/etc`：要统计的目录路径。

![](img/db3503b58daa94374d15395f108d8123_100.png)

![](img/db3503b58daa94374d15395f108d8123_102.png)

![](img/db3503b58daa94374d15395f108d8123_104.png)

**4. `blkid` 命令**
此命令用于显示块设备（如分区）的属性，特别是文件系统类型和UUID（全局唯一标识符）。

![](img/db3503b58daa94374d15395f108d8123_105.png)

![](img/db3503b58daa94374d15395f108d8123_107.png)

![](img/db3503b58daa94374d15395f108d8123_109.png)

常用示例：
```bash
blkid /dev/sda1
```

![](img/db3503b58daa94374d15395f108d8123_111.png)

**命令功能小结**：
*   想快速了解系统中有几块硬盘、每个硬盘有几个分区，用 **`lsblk`**。
*   想查看各个正在使用的分区还剩多少空间，用 **`df -h`**。
*   想统计某个目录或文件实际占了多大磁盘空间，用 **`du -sh`**。
*   想查看某个分区具体的文件系统类型和UUID，用 **`blkid`** 或 **`lsblk -f`**。
