在本节课中，我们将学习搭建RHCE8学习环境所需的准备工作。主要内容包括VMware Workstation Pro软件的安装与基本操作，以及Xmanager远程连接工具的安装与配置。掌握这些工具是后续进行Linux系统实验的基础。

## 软件安装要求与环境准备

为了不破坏物理机的原有环境，我们将在虚拟机中演示这两个软件的安装过程。以下是学习RHCE8课程对硬件环境的基本要求。

以下是硬件配置建议：
*   **物理机内存**：建议至少8GB，推荐12GB至16GB，以保证虚拟机流畅运行。
*   **CPU**：普通i5处理器即可满足大部分需求。
*   **硬盘**：建议使用固态硬盘（SSD），128GB容量可满足基本环境需求。若考虑未来扩展，建议选择256GB或512GB。
*   **网卡**：普通千兆网卡即可。
*   **操作系统**：Windows 7或Windows 10系统均可。

### 安装VMware Workstation Pro

![](img/72f4f99ccece242c02714b477487a4b0_15.png)

![](img/72f4f99ccece242c02714b477487a4b0_17.png)

以下是安装步骤：
1.  以管理员身份运行安装程序。
2.  在安装向导中，选择“下一步”。
3.  勾选“我接受许可协议中的条款”，然后选择“下一步”。
4.  使用默认安装路径，建议勾选“增强型键盘驱动程序”，然后选择“下一步”。
5.  取消勾选“启动时检查产品更新”和“加入VMware客户体验提升计划”，然后选择“下一步”。
6.  保持默认的快捷方式创建选项，选择“下一步”并开始安装。
7.  安装完成后，输入有效的许可证密钥以激活软件，否则仅有30天试用期。
8.  若提示重启，可选择“否”，待下次重启时生效。

### VMware Workstation Pro基本操作

![](img/72f4f99ccece242c02714b477487a4b0_28.png)

![](img/72f4f99ccece242c02714b477487a4b0_30.png)

![](img/72f4f99ccece242c02714b477487a4b0_32.png)

软件安装完成后，我们来熟悉其基本界面和功能。

![](img/72f4f99ccece242c02714b477487a4b0_34.png)

以下是主要功能区域介绍：
*   **查看菜单**：
    *   **全屏**：使虚拟机界面全屏显示。将鼠标移至屏幕顶部可呼出控制栏退出全屏。
    *   **控制台视图**：左侧显示虚拟机硬件配置，右侧显示虚拟机操作系统界面。
    *   **自定义**：可控制是否显示库侧边栏（快捷键`F9`）、缩略图栏（快捷键`F8`）等。
    *   **拉伸模式**：建议选择“自由拉伸”，使虚拟机显示自适应窗口大小。
*   **虚拟机电源管理**：
    *   **启动**：为虚拟机上电开机。
    *   **关机**：关闭虚拟机电源。
    *   **挂起**：使虚拟机进入睡眠状态，可快速唤醒。
    *   **发送Ctrl-Alt-Delete**：此功能将组合键信号发送给虚拟机，而非物理机。
*   **快照管理**：
    *   **拍摄快照**：保存虚拟机当前状态。在系统升级或进行有风险的操作前，建议创建快照。
    *   **恢复快照**：将虚拟机恢复到之前保存的快照状态。
    *   **管理快照**：可查看、删除或克隆已有的快照。
*   **虚拟机设置**：可修改虚拟机的硬件配置，如内存、CPU、硬盘、网卡等。**建议在虚拟机关机状态下进行修改**。
*   **打开现有虚拟机**：通过“文件”->“打开”，选择以`.vmx`结尾的虚拟机配置文件，即可导入并使用已有的虚拟机。

## Xmanager的安装与配置

![](img/72f4f99ccece242c02714b477487a4b0_40.png)

掌握了虚拟机的基本操作后，我们通常不会直接在虚拟机窗口内操作Linux系统，而是使用更便捷的远程工具。本节中我们来看看Xmanager的安装与网络配置。

![](img/72f4f99ccece242c02714b477487a4b0_42.png)

### 安装Xmanager

![](img/72f4f99ccece242c02714b477487a4b0_44.png)

以下是安装步骤：
1.  以管理员身份运行安装程序。
2.  在安装向导中，选择“下一步”。
3.  接受许可证条款，输入用户名、公司名和产品密钥。
4.  选择“定制”安装类型，勾选全部组件（尤其是Xshell和XFTP），然后选择“下一步”。
5.  选择“中文简体”作为安装语言，完成安装。

### 配置网络连接

![](img/72f4f99ccece242c02714b477487a4b0_46.png)

![](img/72f4f99ccece242c02714b477487a4b0_48.png)

要使用Xmanager远程连接虚拟机内的Linux系统，需要确保网络连通。我们提供的虚拟机默认使用“仅主机模式（Host-Only）”。

**网络模式原理**：
在仅主机模式下，VMware会在物理机上创建一个虚拟交换机（如`VMnet1`）和一块虚拟网卡。虚拟机和物理机的这块虚拟网卡都连接到这个虚拟交换机上，从而构成一个独立的私有网络，两者可以相互通信。

![](img/72f4f99ccece242c02714b477487a4b0_50.png)

![](img/72f4f99ccece242c02714b477487a4b0_52.png)

以下是配置步骤：
1.  在VMware中，确保虚拟机的网络适配器设置为“仅主机模式”。
2.  打开VMware的“编辑”->“虚拟网络编辑器”。
3.  选择`VMnet1`（仅主机模式），确保“将主机虚拟适配器连接到此网络”已勾选。
4.  设置子网IP（例如`172.25.0.0`）和子网掩码（`255.255.255.0`）。
5.  **关闭**此界面中的DHCP服务，避免与虚拟机内的DHCP服务冲突。
6.  物理机虚拟网卡（`VMnet1`）的IP地址应设置为与虚拟机同一网段（例如`172.25.0.1`）。

配置完成后，可以在物理机的命令提示符中使用 `ping` 命令测试连通性：
```bash
ping 172.25.0.250
```

### 使用Xshell连接Linux

![](img/72f4f99ccece242c02714b477487a4b0_54.png)

![](img/72f4f99ccece242c02714b477487a4b0_56.png)

![](img/72f4f99ccece242c02714b477487a4b0_57.png)

网络连通后，即可使用Xshell进行远程连接。

![](img/72f4f99ccece242c02714b477487a4b0_59.png)

![](img/72f4f99ccece242c02714b477487a4b0_61.png)

![](img/72f4f99ccece242c02714b477487a4b0_63.png)

以下是连接与配置步骤：
1.  打开Xshell，点击“新建会话”。
2.  输入虚拟机Linux系统的IP地址（如`172.25.0.250`）和端口（默认22）。
3.  输入用户名（如`kiosk`）进行连接。
4.  首次连接会提示接受主机密钥，输入密码（如`redhat`）即可登录。
5.  为方便日后使用，可将当前会话保存：在登录后的会话窗口，点击“文件”->“另存为”，或直接拖拽到链接栏。可以在会话属性中自定义名称、颜色方案和字体大小。

### 使用XFTP传输文件

Xmanager套件中的XFTP工具可以方便地在物理机和虚拟机之间传输文件。

![](img/72f4f99ccece242c02714b477487a4b0_69.png)

![](img/72f4f99ccece242c02714b477487a4b0_71.png)

以下是使用方法：
1.  在Xshell中，点击工具栏上的“新建文件传输”按钮，会自动启动XFTP并连接到当前会话的服务器。
2.  XFTP界面左侧为本地计算机目录，右侧为远程服务器目录。
3.  通过拖拽或右键菜单，即可上传或下载文件。
