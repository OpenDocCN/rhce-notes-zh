## 移动与重命名文件：mv命令 🚚

上一节我们介绍了`cp`拷贝命令，本节中我们来看看`mv`命令。`mv`命令用于移动文件或目录到其他位置，其功能类似于Windows中的“剪切”。同时，它也可以用来修改文件或目录的名称。

![](img/6d41361962c594c243aea3574c7c86a3_9.png)

![](img/6d41361962c594c243aea3574c7c86a3_11.png)

![](img/6d41361962c594c243aea3574c7c86a3_13.png)

命令格式与`cp`命令类似：
```bash
mv [选项] 源文件或目录 目标文件或目录
```

以下是`mv`命令的基本操作：

![](img/6d41361962c594c243aea3574c7c86a3_15.png)

*   **移动文件**：将当前目录下的`hello`文件移动到`/opt/t1/`目录。
    ```bash
    mv hello /opt/t1/
    ```
*   **移动多个项目**：将当前目录下的`t6`、`t7`、`t8`目录移动到`/opt/test/`目录。
    ```bash
    mv t6 t7 t8 /opt/test/
    ```
*   **移动并重命名**：将`test`目录移动到根目录`/`下，并重命名为`xxxxx`。
    ```bash
    mv test /xxxxx
    ```
*   **仅重命名**：如果源和目标在同一目录下，则实现重命名效果。例如，将当前目录下的`test1`重命名为`newname`。
    ```bash
    mv test1 newname
    ```

## 查看文件内容：cat命令 👀

![](img/6d41361962c594c243aea3574c7c86a3_21.png)

在Linux中，我们使用命令来查看文件内容。`cat`命令是最基本的查看工具，它会一次性将文件的全部内容输出到终端屏幕上。

![](img/6d41361962c594c243aea3574c7c86a3_23.png)

命令格式非常简单：
```bash
cat [选项] 文件名
```

![](img/6d41361962c594c243aea3574c7c86a3_25.png)

最常用的选项是`-n`，它可以显示行号。

![](img/6d41361962c594c243aea3574c7c86a3_27.png)

![](img/6d41361962c594c243aea3574c7c86a3_29.png)

![](img/6d41361962c594c243aea3574c7c86a3_30.png)

![](img/6d41361962c594c243aea3574c7c86a3_32.png)

以下是使用`cat`查看一些常见系统文件的例子：
```bash
# 查看系统用户信息文件，并显示行号
cat -n /etc/passwd

![](img/6d41361962c594c243aea3574c7c86a3_34.png)

![](img/6d41361962c594c243aea3574c7c86a3_36.png)

![](img/6d41361962c594c243aea3574c7c86a3_38.png)

# 查看文件系统挂载配置文件
cat /etc/fstab

# 查看当前系统的主机名
cat /etc/hostname

# 查看系统版本信息
cat /etc/centos-release  # 或 /etc/redhat-release

# 查看系统用户组信息
cat -n /etc/group

![](img/6d41361962c594c243aea3574c7c86a3_40.png)

![](img/6d41361962c594c243aea3574c7c86a3_42.png)

# 查看网卡配置文件（请根据实际网卡名调整，如ens33）
cat /etc/sysconfig/network-scripts/ifcfg-ens33
```

![](img/6d41361962c594c243aea3574c7c86a3_44.png)

![](img/6d41361962c594c243aea3574c7c86a3_46.png)

`cat`命令的缺点是，当文件内容很长时（例如`/etc/services`文件），所有内容会瞬间刷屏，导致你无法方便地查看开头部分的内容。因此，`cat`更适合查看内容较少的文件。

## 分页查看大文件：less命令 📖

![](img/6d41361962c594c243aea3574c7c86a3_48.png)

![](img/6d41361962c594c243aea3574c7c86a3_50.png)

![](img/6d41361962c594c243aea3574c7c86a3_52.png)

为了解决查看大文件的问题，我们可以使用`less`命令。`less`是一个分页显示工具，允许你像看书一样一页一页地浏览文件内容，并支持上下翻页、跳转等操作。

![](img/6d41361962c594c243aea3574c7c86a3_54.png)

命令格式如下：
```bash
less [选项] 文件名
```

常用选项`-N`可以显示行号。

进入`less`视图后，可以使用以下快捷键进行操作：

*   **上下箭头键**：向上或向下滚动一行。
*   **Page Down / Page Up** 或 **空格键 / b键**：向下或向上翻一页。
*   **`g`** 和 **`G`**：跳转到文件首行或末行。
*   **`/关键词`**：向下搜索指定关键词。
*   **`?关键词`**：向上搜索指定关键词。
*   **`:行号`**：跳转到指定行号（例如 `:1000` 跳转到第1000行）。
*   **`q`**：退出`less`视图。

![](img/6d41361962c594c243aea3574c7c86a3_56.png)

![](img/6d41361962c594c243aea3574c7c86a3_58.png)

例如，分页查看`/etc/services`文件：
```bash
less -N /etc/services
```

![](img/6d41361962c594c243aea3574c7c86a3_60.png)

![](img/6d41361962c594c243aea3574c7c86a3_62.png)

## 查看文件首尾：head与tail命令 🔍

有时我们只需要查看文件的开头或结尾部分，这时`head`和`tail`命令就非常有用。

### head命令

![](img/6d41361962c594c243aea3574c7c86a3_64.png)

![](img/6d41361962c594c243aea3574c7c86a3_66.png)

`head`命令默认显示文件的前10行。

![](img/6d41361962c594c243aea3574c7c86a3_68.png)

命令格式：
```bash
head [选项] 文件名
```

常用选项`-n`用于指定显示的行数。

以下是`head`命令的用法示例：
```bash
# 查看文件前10行（默认）
head /etc/passwd

# 查看文件前20行
head -n 20 /etc/passwd
# 或简写为
head -20 /etc/passwd
```

![](img/6d41361962c594c243aea3574c7c86a3_70.png)

### tail命令

![](img/6d41361962c594c243aea3574c7c86a3_72.png)

![](img/6d41361962c594c243aea3574c7c86a3_74.png)

![](img/6d41361962c594c243aea3574c7c86a3_76.png)

`tail`命令与`head`相反，默认显示文件的末尾10行。

![](img/6d41361962c594c243aea3574c7c86a3_78.png)

命令格式：
```bash
tail [选项] 文件名
```

![](img/6d41361962c594c243aea3574c7c86a3_80.png)

![](img/6d41361962c594c243aea3574c7c86a3_82.png)

常用选项：
*   `-n`：指定显示末尾的行数。
*   `-f`：动态追踪文件末尾内容。当文件有新增内容时（如日志文件被写入），`tail -f`会实时显示出来。按 `Ctrl+C` 终止追踪。

以下是`tail`命令的用法示例：
```bash
# 查看文件末尾10行（默认）
tail /etc/passwd

![](img/6d41361962c594c243aea3574c7c86a3_84.png)

![](img/6d41361962c594c243aea3574c7c86a3_85.png)

# 查看文件末尾5行
tail -n 5 /etc/sysconfig/network-scripts/ifcfg-ens33

# 动态追踪日志文件（例如系统消息日志）
tail -f /var/log/messages
```

## 删除文件与目录：rm命令 🗑️ (请谨慎使用！)

`rm`命令用于删除文件或目录。**请注意，Linux系统默认没有回收站，删除操作需格外谨慎。**

命令格式：
```bash
rm [选项] 文件或目录...
```

常用选项：
*   `-r` 或 `-R`：递归删除，用于删除目录及其内部所有内容。
*   `-f`：强制删除，不进行任何确认提示。
*   `-i`：交互式删除，删除前逐一确认（默认行为在某些系统上可能被别名覆盖）。

以下是`rm`命令的操作示例：

*   **删除文件**：删除普通文件不需要`-r`选项。
    ```bash
    rm file.txt        # 删除前会询问
    rm -f file.txt     # 强制删除，不询问
    ```
*   **删除空目录**：可以使用`rmdir`，但`rm -r`更通用。
    ```bash
    rmdir empty_dir
    ```
*   **删除非空目录**：必须使用`-r`选项。
    ```bash
    rm -r mydir        # 删除目录，会逐一确认
    rm -rf mydir       # 强制递归删除目录，不询问
    ```
*   **使用通配符`*`**：`*`代表当前目录下的所有项目。
    ```bash
    rm *.log           # 删除所有.log结尾的文件
    rm -rf /opt/*      # 强制删除/opt目录下的所有内容（慎用！）
    ```

**⚠️ 极度危险的命令示例（切勿在真实环境尝试！）**：
```bash
rm -rf /*             # 强制递归删除根目录下的所有文件，会导致系统崩溃！
```

**安全建议**：在执行`rm -rf`命令前，务必**再三确认路径**。可以先使用`ls`命令列出要删除的内容，确认无误后再执行删除。

## 路径与斜线（/）的注意事项

在操作目录和文件时，需要注意路径中斜线`/`的含义：
*   **路径分隔符**：在路径中，`/`用于分隔目录层级，例如`/home/user/file`。
*   **目录标识**：在命令中指定一个目录时，末尾可以加`/`，也可以不加，效果相同。例如`ls /opt` 和 `ls /opt/` 是等价的。
*   **文件与目录的区别**：**文件末尾绝对不能加`/`**。系统会认为以`/`结尾的是一个目录。例如，`rm file/` 会报错，因为`file`不是一个目录。
