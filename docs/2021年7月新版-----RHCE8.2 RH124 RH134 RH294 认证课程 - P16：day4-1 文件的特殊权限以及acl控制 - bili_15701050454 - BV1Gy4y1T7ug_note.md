![](img/837fe77f74627a69c89c34e6e24e947f_3.png)

## 回顾：用户、用户组与基本权限

![](img/837fe77f74627a69c89c34e6e24e947f_5.png)

昨天我们主要讲解了用户和用户组的管理，以及文件的基本权限。

![](img/837fe77f74627a69c89c34e6e24e947f_7.png)

![](img/837fe77f74627a69c89c34e6e24e947f_9.png)

以下是相关的四个核心配置文件：
*   `/etc/passwd`：用户账户信息配置文件。
*   `/etc/shadow`：用户密码相关配置文件。
*   `/etc/group`：用户组信息配置文件。
*   `/etc/gshadow`：用户组密码相关配置文件。

![](img/837fe77f74627a69c89c34e6e24e947f_11.png)

![](img/837fe77f74627a69c89c34e6e24e947f_13.png)

以下是用户管理常用命令：
*   创建用户：`useradd [选项] 用户名`。常用选项包括 `-r`（创建系统用户）、`-u`（指定UID）、`-g`（指定主组GID）、`-G`（指定附加组）、`-s`（指定登录shell）。
*   删除用户：`userdel [选项] 用户名`。常用选项 `-r` 会同时删除用户主目录。
*   修改用户属性：`usermod [选项] 用户名`。常用选项与 `useradd` 类似，如 `-u`、`-g`、`-G`、`-s`，还可通过 `-l` 修改登录名。
*   修改用户密码：`passwd [选项] [用户名]`。仅root用户可以修改其他用户的密码。

以下是用户组管理常用命令：
*   创建组：`groupadd [选项] 组名`。常用选项 `-g` 用于指定GID。
*   删除组：`groupdel 组名`。如果组内有用户以其为主组，则无法删除，可配合 `-f` 选项强制删除。
*   修改组信息：`groupmod [选项] 组名`。常用选项 `-n` 用于修改组名。

![](img/837fe77f74627a69c89c34e6e24e947f_15.png)

以下是文件权限管理核心概念：
*   文件归属：包括所有者（owner）和所属组（group）。使用 `chown` 命令修改。
*   普通权限：分为读（r）、写（w）、执行（x）三类，分别对应所有者、所属组和其他人（others）。使用 `chmod` 命令修改。
*   权限修改的三种方式：
    1.  符号法：例如 `chmod u=rwx,g=rx,o=r file`。
    2.  加减法：例如 `chmod u+x,o-w file`。
    3.  数字法：r=4，w=2，x=1。例如 `chmod 755 file`。
*   默认权限：由 `umask` 值决定。新建文件默认权限为 `666 - umask`，新建目录默认权限为 `777 - umask`。可通过 `umask` 命令查看或临时修改，永久修改需写入 `/etc/profile` 或 `/etc/bashrc` 等配置文件。

## 特殊权限位 🛡️

![](img/837fe77f74627a69c89c34e6e24e947f_23.png)

![](img/837fe77f74627a69c89c34e6e24e947f_25.png)

![](img/837fe77f74627a69c89c34e6e24e947f_27.png)

除了基本的rwx权限，Linux还有三种特殊权限位：SUID、SGID和Sticky Bit。

![](img/837fe77f74627a69c89c34e6e24e947f_29.png)

### SUID（Set User ID）

SUID作用于**可执行的二进制程序文件**。其作用是：当其他用户执行这个程序时，会临时拥有该程序**所有者**的权限，而不是执行者本身的权限。

![](img/837fe77f74627a69c89c34e6e24e947f_31.png)

![](img/837fe77f74627a69c89c34e6e24e947f_33.png)

![](img/837fe77f74627a69c89c34e6e24e947f_35.png)

一个典型例子是 `/usr/bin/passwd` 命令。查看其权限：
```bash
ls -l /usr/bin/passwd
```
输出中，所有者的执行位是 `s` 而非 `x`，这表示SUID权限。

**作用原理**：例如，普通用户student执行 `passwd` 命令来修改自己的密码。这个操作需要写入 `/etc/shadow` 文件，而该文件通常只有root可写。由于 `passwd` 命令设置了SUID且所有者是root，student执行它时，产生的进程实际以root身份运行，从而获得了修改 `/etc/shadow` 的权限。

**设置与取消**：
*   设置：`chmod u+s file` 或使用数字法 `chmod 4755 file`（4代表SUID）。
*   取消：`chmod u-s file`。
*   **注意**：如果文件原来没有可执行（x）权限，设置SUID后会显示为大写 `S`，表示SUID有效但文件本身不可执行。

![](img/837fe77f74627a69c89c34e6e24e947f_37.png)

### SGID（Set Group ID）

![](img/837fe77f74627a69c89c34e6e24e947f_39.png)

![](img/837fe77f74627a69c89c34e6e24e947f_41.png)

SGID可作用于**文件**和**目录**，作用不同。

![](img/837fe77f74627a69c89c34e6e24e947f_43.png)

![](img/837fe77f74627a69c89c34e6e24e947f_45.png)

![](img/837fe77f74627a69c89c34e6e24e947f_46.png)

**作用于文件（二进制程序）时**：与SUID类似，但进程将获得文件**所属组**的权限。

![](img/837fe77f74627a69c89c34e6e24e947f_48.png)

![](img/837fe77f74627a69c89c34e6e24e947f_50.png)

![](img/837fe77f74627a69c89c34e6e24e947f_51.png)

**作用于目录时（更常用）**：在该目录下创建的任何新文件或子目录，其所属组都会**自动继承该目录的所属组**，而不是创建者的主组。

![](img/837fe77f74627a69c89c34e6e24e947f_53.png)

![](img/837fe77f74627a69c89c34e6e24e947f_55.png)

**设置与取消**：
*   设置：`chmod g+s directory` 或使用数字法 `chmod 2755 directory`（2代表SGID）。
*   取消：`chmod g-s directory`。
*   **注意**：如果目录原来没有组可执行（x）权限，设置SGID后会显示为大写 `S`。

![](img/837fe77f74627a69c89c34e6e24e947f_57.png)

![](img/837fe77f74627a69c89c34e6e24e947f_59.png)

![](img/837fe77f74627a69c89c34e6e24e947f_61.png)

### Sticky Bit（粘滞位）

![](img/837fe77f74627a69c89c34e6e24e947f_63.png)

Sticky Bit只作用于**目录**。其作用是：在具有写权限的目录中，用户只能删除或重命名**自己创建的文件和目录**，而不能删除或重命名其他用户的文件。root用户除外。

![](img/837fe77f74627a69c89c34e6e24e947f_65.png)

一个典型例子是系统的 `/tmp` 临时目录。

![](img/837fe77f74627a69c89c34e6e24e947f_67.png)

![](img/837fe77f74627a69c89c34e6e24e947f_69.png)

**设置与取消**：
*   设置：`chmod o+t directory` 或使用数字法 `chmod 1755 directory`（1代表Sticky Bit）。
*   取消：`chmod o-t directory`。
*   **注意**：如果目录原来没有其他人可执行（x）权限，设置Sticky Bit后会显示为大写 `T`。

### 特殊权限的数字表示法

![](img/837fe77f74627a69c89c34e6e24e947f_71.png)

在数字表示法中，特殊权限位作为第四位数字放在普通权限之前：
*   SUID = 4
*   SGID = 2
*   Sticky Bit = 1

可以组合使用，例如：
*   `chmod 6755 file`：设置SUID(4)和SGID(2)，即6。
*   `chmod 3755 directory`：设置SGID(2)和Sticky Bit(1)，即3。

![](img/837fe77f74627a69c89c34e6e24e947f_73.png)

## 文件隐藏属性 🔒

除了特殊权限，文件还有隐藏属性，使用 `lsattr` 命令查看，使用 `chattr` 命令修改。最常用的两个属性是 `a` 和 `i`。

![](img/837fe77f74627a69c89c34e6e24e947f_75.png)

![](img/837fe77f74627a69c89c34e6e24e947f_77.png)

以下是 `a` 和 `i` 属性的区别：
*   **`a`属性 (append only)**：文件只能被追加内容，不能被修改、覆盖或删除。设置命令：`chattr +a filename`。
*   **`i`属性 (immutable)**：文件不能被修改、删除、重命名、追加内容或创建硬链接，是最高级别的保护。设置命令：`chattr +i filename`。

**注意**：即使是root用户，在文件被设置 `i` 属性后也无法直接修改或删除。必须先使用 `chattr -i filename` 移除该属性。

![](img/837fe77f74627a69c89c34e6e24e947f_79.png)

![](img/837fe77f74627a69c89c34e6e24e947f_81.png)

## 访问控制列表（ACL） 📋

基本权限模型（所有者、所属组、其他人）有时不够灵活。ACL（Access Control List）提供了更精细的权限控制，允许为任意用户或用户组设置独立的文件权限。

### 启用与查看ACL

现代Linux文件系统（如XFS, ext4）通常默认支持ACL。主要使用两个命令：
*   `setfacl`：设置ACL规则。
*   `getfacl`：查看ACL规则。

查看一个文件的ACL信息：
```bash
getfacl filename
```

### 设置ACL规则

![](img/837fe77f74627a69c89c34e6e24e947f_83.png)

![](img/837fe77f74627a69c89c34e6e24e947f_85.png)

使用 `setfacl` 命令。

![](img/837fe77f74627a69c89c34e6e24e947f_87.png)

以下是常用选项示例：
*   为用户设置权限：`setfacl -m u:username:rwx filename`
*   为用户组设置权限：`setfacl -m g:groupname:rx filename`
*   删除特定ACL条目：`setfacl -x u:username filename`
*   删除所有ACL条目（恢复基本权限）：`setfacl -b filename`

![](img/837fe77f74627a69c89c34e6e24e947f_89.png)

**示例**：文件 `f1` 其他人权限为只读（r--）。现在需要允许特定用户 `user3` 可写。
```bash
# 查看初始权限
ls -l f1
# 为user3添加rw权限
setfacl -m u:user3:rw f1
# 再次查看，权限位后会出现一个‘+’号，表示有ACL
ls -l f1
# 使用getfacl查看详细ACL
getfacl f1
```
此时，`user3` 可以对 `f1` 进行读写，而其他不属于所有者或所属组的用户仍然只有读权限。

### 有效权限掩码（mask）

`getfacl` 命令的输出中包含一条 `mask` 条目。它定义了除所有者和其他人外，所有**通过ACL指定的用户和组**所能拥有的**最大有效权限**。

**mask的动态性**：当您设置ACL时，如果新条目的权限超出了当前的 `mask`，`mask` 值会自动提升以匹配最高权限。
**mask的手动限制**：您可以手动降低 `mask` 值（如 `setfacl -m m::r file`），这将像一个“天花板”，限制所有ACL条目用户的权限不超过此 `mask`，即使他们被赋予了更高的权限。

**理解mask**：可以将ACL权限看作“水”，`mask` 是“水桶”的高度。自动提升时，桶变高以容纳所有水；手动降低时，桶变矮，多余的水（超出mask的权限）就会溢出失效。

![](img/837fe77f74627a69c89c34e6e24e947f_91.png)

![](img/837fe77f74627a69c89c34e6e24e947f_93.png)

### 默认ACL规则

可以为目录设置默认ACL规则（使用 `-d` 选项），这样在该目录下**新建**的文件或子目录会自动继承这些ACL规则。
```bash
setfacl -m d:u:user3:rwx mydir/
```

## 总结

本节课中我们一起学习了Linux文件系统的高级权限管理功能。

我们首先回顾了用户、用户组和基本文件权限的管理命令与概念。接着，深入探讨了三种特殊权限位：SUID让程序运行时临时切换所有者身份；SGID在目录上能强制继承所属组；Sticky Bit保护目录内容不被非所有者删除。

然后，我们了解了文件的隐藏属性 `a` 和 `i`，它们能对文件实施更严格的保护，防止误修改或删除。

最后，我们学习了强大的访问控制列表（ACL）。它突破了基本权限模型的限制，允许我们为任意用户或组设置独立的文件访问权限，并通过有效权限掩码（mask）进行灵活控制，实现了真正精细化的权限管理。
