## 环境准备与网络规划 🌐

![](img/a9b18736649b17868e2027f5ee301783_12.png)

![](img/a9b18736649b17868e2027f5ee301783_13.png)

上一节我们介绍了防火墙的基本概念，本节中我们来看看如何为实验搭建网络环境。一个典型的防火墙实验环境需要模拟内部网络、外部网络以及防火墙本身。

以下是实验所需的网络拓扑与IP规划：
*   **防火墙主机 (iptables)**:
    *   内网网卡 (ens33): `192.168.0.80`
    *   外网网卡 (ens34): `192.168.1.100`
*   **内部网站服务器 (web26)**:
    *   IP地址: `192.168.0.26`
    *   网关: `192.168.0.80` (指向防火墙内网IP)
*   **外部客户端 (client)**:
    *   IP地址: `192.168.1.24`
    *   网关: `192.168.1.100` (指向防火墙外网IP)

### 配置防火墙外网网卡

![](img/a9b18736649b17868e2027f5ee301783_29.png)

在防火墙主机上，第二块网卡(ens34)默认没有配置文件，我们需要为其配置一个永久IP地址。可以使用`nmtui`命令进行交互式配置。

```bash
nmtui
```
执行上述命令后，在图形界面中选择“编辑连接” -> “有线连接 1” -> 将连接名称改为`ens34` -> 在“IPv4配置”中选择“手动” -> 添加地址`192.168.1.100`，无需配置网关 -> 保存并退出。

![](img/a9b18736649b17868e2027f5ee301783_31.png)

![](img/a9b18736649b17868e2027f5ee301783_33.png)

配置完成后，重启网络服务并检查。
```bash
systemctl restart network
ip addr show ens34
```
此时，查看`/etc/sysconfig/network-scripts/`目录，会发现生成了`ifcfg-ens34`配置文件，IP地址已永久生效。

## 配置iptables转发与基础规则 🛡️

上一节我们准备好了所有服务，本节中我们来看看如何配置iptables的核心功能。首先，需要在防火墙主机上开启Linux内核的IP转发功能。

![](img/a9b18736649b17868e2027f5ee301783_98.png)

```bash
# 临时开启IP转发
echo 1 > /proc/sys/net/ipv4/ip_forward

![](img/a9b18736649b17868e2027f5ee301783_99.png)

![](img/a9b18736649b17868e2027f5ee301783_101.png)

![](img/a9b18736649b17868e2027f5ee301783_103.png)

# 永久开启IP转发
echo “net.ipv4.ip_forward = 1” >> /etc/sysctl.conf
sysctl -p
```

### 配置FORWARD链规则

![](img/a9b18736649b17868e2027f5ee301783_105.png)

我们的目标是让外部客户端能够通过防火墙访问内部Web服务器。默认情况下，`FORWARD`链（负责转发数据包）的策略可能是`DROP`。我们需要添加允许转发的规则。

但在添加允许规则前，我们先清空所有现有规则，避免干扰。
```bash
iptables -F
iptables -t nat -F
iptables -t mangle -F
```

现在，配置允许转发的规则。假设我们只允许访问内部服务器(192.168.0.26)的80端口。
```bash
iptables -A FORWARD -d 192.168.0.26 -p tcp --dport 80 -j ACCEPT
iptables -A FORWARD -s 192.168.0.26 -p tcp --sport 80 -j ACCEPT
```
第一条规则允许目标为`192.168.0.26:80`的请求通过；第二条规则允许来自该服务的返回数据通过。

此时，从客户端(`192.168.1.24`)访问`192.168.0.26`，应该可以成功看到“web26”页面。这证明了防火墙的转发功能正常工作。

### 实施访问控制

![](img/a9b18736649b17868e2027f5ee301783_107.png)

![](img/a9b18736649b17868e2027f5ee301783_108.png)

现在，我们来实现核心安全功能：拒绝特定客户端的访问。例如，我们要拒绝IP为`192.168.1.24`的客户端访问Web服务。

![](img/a9b18736649b17868e2027f5ee301783_110.png)

在`FORWARD`链中插入一条拒绝规则（使用`-I`插入到链的顶部，使其优先匹配）。
```bash
iptables -I FORWARD -s 192.168.1.24 -p tcp --dport 80 -j REJECT
```
配置后，客户端再次访问将被拒绝。这就是网络型防火墙保护内部服务器的基本原理：在转发链上对数据包进行过滤。

![](img/a9b18736649b17868e2027f5ee301783_112.png)

![](img/a9b18736649b17868e2027f5ee301783_114.png)

## iptables扩展功能应用 🔧

![](img/a9b18736649b17868e2027f5ee301783_115.png)

基础的IP过滤有时不够灵活，iptables提供了丰富的扩展模块。

![](img/a9b18736649b17868e2027f5ee301783_117.png)

![](img/a9b18736649b17868e2027f5ee301783_119.png)

### 基于MAC地址过滤

![](img/a9b18736649b17868e2027f5ee301783_121.png)

如果客户端更改了IP地址，基于IP的规则就会失效。此时可以基于网卡的MAC地址进行过滤，因为MAC地址通常更难更改。

首先，在防火墙主机上安装扫描工具并获取客户端的MAC地址。
```bash
yum -y install nmap
nmap -sn 192.168.1.24
```
在输出信息中找到`MAC Address`。

![](img/a9b18736649b17868e2027f5ee301783_123.png)

![](img/a9b18736649b17868e2027f5ee301783_124.png)

![](img/a9b18736649b17868e2027f5ee301783_126.png)

然后，使用`mac`模块添加规则。
```bash
iptables -I FORWARD -m mac --mac-source <客户端MAC地址> -p tcp --dport 80 -j REJECT
```
将`<客户端MAC地址>`替换为实际扫描到的地址。这样，无论客户端如何更换IP，只要网卡不变，都会被拒绝。

### 基于多端口过滤

![](img/a9b18736649b17868e2027f5ee301783_128.png)

![](img/a9b18736649b17868e2027f5ee301783_130.png)

当需要同时控制对多个端口的访问时（例如FTP的21端口、HTTPS的443端口、MySQL的3306端口），可以使用`multiport`模块。

![](img/a9b18736649b17868e2027f5ee301783_132.png)

以下是配置示例：
```bash
iptables -A FORWARD -p tcp -m multiport --dports 21,80,443,3306 -j DROP
```
这条规则会丢弃所有目标端口为21, 80, 443, 3306的TCP数据包。

### 基于IP地址范围过滤

如果需要禁止一个IP段内的所有主机，可以使用`iprange`模块。

![](img/a9b18736649b17868e2027f5ee301783_134.png)

![](img/a9b18736649b17868e2027f5ee301783_136.png)

以下是配置示例：
```bash
iptables -A FORWARD -m iprange --src-range 192.168.1.20-192.168.1.30 -j REJECT
```
这条规则会拒绝源IP在`192.168.1.20`到`192.168.1.30`这个范围内的所有访问请求。

## 配置SNAT实现共享上网 🌍

![](img/a9b18736649b17868e2027f5ee301783_138.png)

iptables的另一个重要功能是源地址转换(SNAT)，常用于让内网使用私有IP的机器通过防火墙的公网IP访问互联网。

![](img/a9b18736649b17868e2027f5ee301783_140.png)

![](img/a9b18736649b17868e2027f5ee301783_142.png)

我们调整一下实验角色：
*   **内部客户端**: `192.168.1.25` (原实验中的client)
*   **防火墙内网IP**: `192.168.1.100` (ens34)
*   **防火墙外网IP (模拟公网)**: `192.168.0.80` (ens33)
*   **外部服务器**: `192.168.0.26` (web26，模拟互联网服务器)

![](img/a9b18736649b17868e2027f5ee301783_144.png)

![](img/a9b18736649b17868e2027f5ee301783_145.png)

![](img/a9b18736649b17868e2027f5ee301783_147.png)

目标：让内部客户端(`192.168.1.25`)能够访问外部服务器(`192.168.0.26`)，并且在外部服务器上看到的访问来源是防火墙的外网IP(`192.168.0.80`)。

![](img/a9b18736649b17868e2027f5ee301783_149.png)

![](img/a9b18736649b17868e2027f5ee301783_151.png)

在防火墙上配置SNAT规则：
```bash
iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -p tcp -j SNAT --to-source 192.168.0.80
```
*   `-t nat`: 指定操作`nat`表。
*   `-A POSTROUTING`: 在`POSTROUTING`链（路由后处理，数据包即将发出时）添加规则。
*   `-s 192.168.1.0/24`: 匹配源IP为`192.168.1.0/24`网段的数据包。
*   `-j SNAT`: 执行SNAT动作。
*   `--to-source 192.168.0.80`: 将源IP修改为`192.168.0.80`。

![](img/a9b18736649b17868e2027f5ee301783_153.png)

配置完成后，在内部客户端上访问外部Web服务器。然后在外网服务器上查看Apache访问日志(`/var/log/httpd/access_log`)，会发现访问者的IP是`192.168.0.80`，而不是内部客户端的真实IP。这证明了SNAT转换成功。

![](img/a9b18736649b17868e2027f5ee301783_155.png)

## 规则的保存与清除 💾

![](img/a9b18736649b17868e2027f5ee301783_157.png)

iptables的规则默认是临时的，重启后会丢失。如果需要永久保存，可以使用以下命令：

![](img/a9b18736649b17868e2027f5ee301783_159.png)

```bash
# 保存当前规则到默认配置文件
service iptables save
# 或
iptables-save > /etc/sysconfig/iptables
```
保存后，规则会被写入`/etc/sysconfig/iptables`文件。系统重启时，`iptables`服务会自动加载这个文件中的规则。

![](img/a9b18736649b17868e2027f5ee301783_161.png)

若要清除所有规则并恢复默认状态，可以执行：
```bash
iptables -F
iptables -t nat -F
iptables -t mangle -F
# 删除保存的规则文件
rm -f /etc/sysconfig/iptables
# 重启iptables服务（如果规则文件不存在，会加载空规则）
systemctl restart iptables
```
