在本节课中，我们将学习一个非常实际的Linux操作例子：如何修改系统的`PATH`环境变量，以添加一个自定义的目录（例如`/opt`）。这对于安装和运行第三方程序至关重要。

## 概述：为什么需要修改PATH？

在Linux系统中，当我们输入一个命令（如`ls`或`echo`）时，系统会在一系列预设的目录中查找这个命令的可执行文件。这个目录列表就是`PATH`环境变量。有时，我们安装的第三方软件默认安装到非标准目录（如`/opt`），为了让系统能直接找到并运行这些程序，我们需要将它们的安装路径添加到`PATH`变量中。

上一节我们介绍了环境变量的基本概念，本节中我们来看看如何通过一个具体操作来修改`PATH`变量。

## 查看当前的PATH

首先，我们需要查看当前系统的`PATH`变量是什么。这可以通过`echo`命令实现。

```bash
echo $PATH
```

执行上述命令后，终端会显示一串由冒号分隔的目录路径。这代表了系统当前会搜索命令的所有位置。

## 修改PATH变量

假设我们需要将`/opt`目录添加到现有的`PATH`变量末尾。我们不能直接覆盖`PATH`，而是需要在原有值的基础上追加新路径。

以下是修改`PATH`变量的核心步骤：

1.  **创建新的PATH变量**：我们将创建一个名为`PATH`的新变量（注意Linux中变量名通常大写），其值等于旧的`PATH`值加上`:/opt`。
2.  **使用引用符号**：在引用旧`PATH`的值时，必须使用美元符号`$`。如果不加`$`，就只是将字符串“PATH”赋值给新变量，而不是其内容。

具体的操作命令如下：

```bash
PATH=$PATH:/opt
```

**公式解释**：
`新PATH = 旧PATH + “:” + “/opt”`
其中，`$PATH`代表获取旧PATH变量的值，冒号`:`是Linux中分隔多个路径的标准符号。

执行这条命令后，系统似乎没有给出任何明显反馈。这属于正常现象，因为赋值操作通常是静默的。

## 验证修改结果

为了确认我们的修改是否成功，需要再次查看`PATH`变量的内容。

再次执行查看命令：

```bash
echo $PATH
```

现在，观察输出结果。与第一次查看时相比，路径列表的末尾应该已经增加了`:/opt`。这证明我们已经成功地将`/opt`目录添加到了系统的搜索路径中。

## 总结

本节课中我们一起学习了如何通过命令行修改Linux的`PATH`环境变量。我们首先使用`echo $PATH`查看了变量的原始值，然后通过`PATH=$PATH:/opt`命令在原有路径后追加了`/opt`目录，最后再次验证确认了修改成功。掌握这个方法，你就能让系统正确找到安装在不同目录下的应用程序了。

![](img/72793759f656e973723909259dd759e3_2.png)

需要注意的是，这种直接在终端中修改`PATH`的方法只在当前会话中有效。一旦关闭终端或重新登录，修改就会失效。若想永久修改`PATH`，需要将上述命令写入用户的shell配置文件（如`~/.bashrc`或`~/.bash_profile`）中，这将是后续课程可能涉及的内容。
