## 概述：什么是FTP？

文件共享服务FTP与NFS是比较古老的文件传输方式。它的功能类似于我们日常生活中使用的百度网盘，用于共享文件、录屏等。但在企业环境中，出于数据保密性的考虑，通常不会使用公网上的网盘服务，而是选择在企业内部搭建自己的文件共享服务器。这样，数据就存储在企业自己的服务器上，而非第三方平台。

FTP的全称是 **File Transfer Protocol**，即文件传输协议。它是一种专门用于在网络上进行文件传输的协议。无论是音乐文件（MP3）、视频文件（MP4）、脚本、图片（JPG、PNG、GIF）还是文本文件（.txt）、代码文件（.java、.html、.py、.json）等，FTP都能传输。因此，你可以将企业内部的FTP服务器理解为一个“私有的百度网盘”。

与用于访问网站的HTTP/HTTPS协议不同，FTP协议的应用场景专注于**企业内部的数据传输**。FTP是一种基于TCP协议的端到端数据传输协议。这意味着客户端与服务器建立连接时，底层需要通过TCP协议进行“三次握手”来确保连接的可靠性，而上层的文件传输则通过FTP协议来完成。

## FTP的架构与工作模式

FTP采用经典的**C/S（Client/Server，客户端/服务器）架构**。服务器端负责共享数据，客户端则负责连接服务器以下载或上传数据。

FTP协议默认使用**20和21**两个端口：
*   **21端口**：命令端口，用于接收客户端发送的指令（如下载、上传命令）。
*   **20端口**：数据端口，在主动模式下用于数据传输。

FTP有两种主要的工作模式，区别在于**数据传输连接由谁主动发起**：

### 主动模式
1.  客户端使用一个随机端口（例如1024）作为命令端口，连接服务器的21端口，并发送指令（如请求下载文件）。在指令中，客户端会告知服务器：“我的数据端口是1025”。
2.  服务器收到指令后，用20端口**主动连接**客户端告知的1025端口。
3.  连接建立后，开始数据传输。

![](img/59a67e61577812aab5a9d162010693b0_5.png)

**潜在问题**：如果客户端开启了防火墙，且没有放行对随机高端口（如1025）的入站连接，那么服务器的主动连接可能会被防火墙拦截，导致数据传输失败。

![](img/59a67e61577812aab5a9d162010693b0_7.png)

### 被动模式
1.  客户端连接服务器的21端口，并发送`PASV`指令，进入被动模式。
2.  服务器随机开启一个端口（例如5000）作为数据端口，并将这个端口号告知客户端。
3.  客户端使用另一个随机端口**主动连接**服务器的5000端口。
4.  连接建立后，开始数据传输。

![](img/59a67e61577812aab5a9d162010693b0_9.png)

![](img/59a67e61577812aab5a9d162010693b0_11.png)

![](img/59a67e61577812aab5a9d162010693b0_13.png)

![](img/59a67e61577812aab5a9d162010693b0_15.png)

**特点**：在被动模式下，数据传输连接是由客户端主动发起的。这通常能更好地适应客户端防火墙的环境，因为出站连接（客户端访问服务器）通常是被允许的。**vsftpd默认使用被动模式**。

![](img/59a67e61577812aab5a9d162010693b0_17.png)

![](img/59a67e61577812aab5a9d162010693b0_19.png)

## 搭建FTP服务器：vsftpd

![](img/59a67e61577812aab5a9d162010693b0_21.png)

![](img/59a67e61577812aab5a9d162010693b0_23.png)

上一节我们介绍了FTP的基本原理和工作模式，本节中我们来看看如何在Linux系统上实际搭建一个FTP服务器。我们将使用**vsftpd**这款软件，它的名字是“Very Secure FTP Daemon”的缩写，意为“非常安全的FTP守护进程”，在企业中应用广泛。

![](img/59a67e61577812aab5a9d162010693b0_25.png)

### 安装vsftpd

![](img/59a67e61577812aab5a9d162010693b0_27.png)

![](img/59a67e61577812aab5a9d162010693b0_29.png)

![](img/59a67e61577812aab5a9d162010693b0_31.png)

首先，我们需要在作为服务器的机器上进行安装。确保系统仓库可用，并关闭或正确配置防火墙（如firewalld）和SELinux（学习环境中可临时关闭以简化操作）。

![](img/59a67e61577812aab5a9d162010693b0_33.png)

以下是安装步骤：
```bash
# 1. 安装vsftpd软件包
yum install -y vsftpd

![](img/59a67e61577812aab5a9d162010693b0_35.png)

![](img/59a67e61577812aab5a9d162010693b0_37.png)

# 2. 启动vsftpd服务
systemctl start vsftpd

# 3. 设置开机自启（生产环境建议，学习环境可选）
systemctl enable vsftpd

# 4. 检查服务状态
systemctl status vsftpd
```

### vsftpd的用户访问模式

![](img/59a67e61577812aab5a9d162010693b0_39.png)

vsftpd支持三种用户访问模式，决定了客户端如何认证并访问服务器：

![](img/59a67e61577812aab5a9d162010693b0_41.png)

1.  **本地用户模式**：客户端必须使用服务器系统上存在的真实用户账号和密码进行登录。例如，在服务器上创建用户`ftp_user`，客户端则使用`ftp_user`及其密码连接。
2.  **匿名用户模式**：客户端无需输入用户名和密码，或使用通用账号（如`anonymous`或`ftp`）即可直接访问。这是vsftpd**默认启用**的模式。
3.  **虚拟用户模式**：用户信息存储在独立的数据库（如MySQL）中，而非系统本地。安全性更高，配置也相对复杂。

![](img/59a67e61577812aab5a9d162010693b0_43.png)

![](img/59a67e61577812aab5a9d162010693b0_45.png)

对于大多数内部共享场景，**匿名用户模式**因其简便性而使用较多。本节课我们将重点配置匿名模式。

![](img/59a67e61577812aab5a9d162010693b0_47.png)

![](img/59a67e61577812aab5a9d162010693b0_49.png)

### 配置匿名用户访问

![](img/59a67e61577812aab5a9d162010693b0_51.png)

![](img/59a67e61577812aab5a9d162010693b0_53.png)

默认情况下，匿名用户登录后，会进入服务器上的 `/var/ftp/` 目录。该目录下有一个 `pub/` 子目录，意为“public”，是设计用来存放共享文件的。

![](img/59a67e61577812aab5a9d162010693b0_55.png)

![](img/59a67e61577812aab5a9d162010693b0_57.png)

然而，默认配置下匿名用户可能只有**下载**权限，没有**上传、创建目录**等权限。我们需要修改配置文件来开放这些权限。

![](img/59a67e61577812aab5a9d162010693b0_59.png)

以下是关键的配置步骤：
```bash
# 1. 编辑vsftpd的主配置文件
vi /etc/vsftpd/vsftpd.conf

![](img/59a67e61577812aab5a9d162010693b0_61.png)

![](img/59a67e61577812aab5a9d162010693b0_63.png)

# 2. 找到并确保以下行是启用的（默认已启用，无需修改）
anonymous_enable=YES           # 允许匿名登录

# 3. 找到以下行，去掉行首的 `#` 以取消注释，启用相应功能
anon_upload_enable=YES         # 允许匿名用户上传文件
anon_mkdir_write_enable=YES    # 允许匿名用户创建目录
# 有时还需要启用写入权限
write_enable=YES               # 允许写入（全局设置，通常已存在）

![](img/59a67e61577812aab5a9d162010693b0_65.png)

![](img/59a67e61577812aab5a9d162010693b0_67.png)

# 4. 保存并退出编辑器，然后重启vsftpd服务使配置生效
systemctl restart vsftpd
```

### 设置共享目录权限

仅修改配置文件可能还不够，还需要确保匿名用户访问的目录具有正确的文件系统权限。建议使用 `/var/ftp/pub/` 目录作为共享点，并将其所有者改为vsftpd服务运行时使用的`ftp`用户。

```bash
# 1. 修改/var/ftp/pub目录的所有者为ftp用户和组
chown ftp:ftp /var/ftp/pub/

![](img/59a67e61577812aab5a9d162010693b0_69.png)

# 2. 为该目录设置足够的权限（例如755，所有者可读写执行，其他人只读执行）
chmod 755 /var/ftp/pub/
# 如果需要匿名用户也能删除文件，可以赋予777权限（测试环境，生产环境慎用）
# chmod 777 /var/ftp/pub/
```

![](img/59a67e61577812aab5a9d162010693b0_71.png)

![](img/59a67e61577812aab5a9d162010693b0_73.png)

![](img/59a67e61577812aab5a9d162010693b0_75.png)

完成以上配置后，匿名用户应该能够在 `/pub/` 目录中进行上传、下载、创建文件夹等操作了。

上一节我们完成了服务器的搭建与配置，本节中我们来看看客户端如何连接并使用FTP服务。在Linux客户端上，我们可以使用命令行工具进行访问。

### 安装客户端工具

![](img/59a67e61577812aab5a9d162010693b0_88.png)

![](img/59a67e61577812aab5a9d162010693b0_90.png)

常用的FTP命令行客户端工具有 `ftp` 和其增强版 `lftp`。`lftp` 功能更强大，支持标签补全、颜色显示等。

```bash
# 安装客户端工具
yum install -y ftp lftp
```

### 使用lftp连接服务器

`lftp` 默认会尝试以匿名方式连接。

```bash
# 连接FTP服务器（假设服务器IP是192.168.0.40）
lftp 192.168.0.40
```
连接成功后，命令行提示符会发生变化，表示你已进入FTP会话。

![](img/59a67e61577812aab5a9d162010693b0_92.png)

![](img/59a67e61577812aab5a9d162010693b0_94.png)

### 常用FTP命令

进入`lftp`会话后，你可以使用很多命令。输入 `help` 可以查看支持的命令列表。以下是一些最常用的命令：

*   `ls`：列出服务器当前目录的文件和子目录。
*   `cd [目录名]`：切换服务器上的目录。例如 `cd pub` 进入pub目录。
*   `pwd`：显示在服务器上的当前目录。
*   `get [文件名]`：从服务器下载单个文件到客户端当前目录。
    ```bash
    get hello.txt
    ```
*   `put [文件名]`：将客户端当前目录的单个文件上传到服务器。
    ```bash
    put local_file.txt
    ```
*   `mget [通配符]`：批量下载文件。
    ```bash
    mget *.txt
    ```
*   `mput [通配符]`：批量上传文件。
*   `mkdir [目录名]`：在服务器上创建目录（需有权限）。
*   `rm [文件名]`：删除服务器上的文件（需有权限）。
*   `exit` 或 `bye`：退出lftp会话。

**操作示例**：
1.  连接后，使用 `ls` 查看初始目录。
2.  使用 `cd pub` 进入共享目录。
3.  使用 `put myfile.txt` 尝试上传一个文件。
4.  使用 `get hello.txt` 下载一个文件到本地。
5.  操作完成后，输入 `exit` 退出。

## 防火墙注意事项

在实践过程中，如果无法连接或传输失败，很可能是防火墙阻止了连接。企业Linux系统通常使用 `firewalld`。

*   **对于服务器端**：需要放行FTP服务。
    ```bash
    # 方法一：将FTP服务添加到public区域的允许列表
    firewall-cmd --permanent --zone=public --add-service=ftp
    firewall-cmd --reload

    # 方法二（学习环境）：直接停止防火墙（不推荐生产环境）
    systemctl stop firewalld
    systemctl disable firewalld
    ```
*   **对于客户端**：在主动模式下，如果客户端开启了严格防火墙，可能需要放行来自服务器20端口的高位随机端口入站连接。在被动模式下，此问题通常不存在。最简便的方法是确保客户端防火墙不会干扰测试。
