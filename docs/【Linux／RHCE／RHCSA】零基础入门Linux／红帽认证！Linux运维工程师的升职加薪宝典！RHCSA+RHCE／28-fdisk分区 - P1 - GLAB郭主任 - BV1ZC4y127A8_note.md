# Linux存储管理：第28章：fdisk分区

在本节课中，我们将要学习Linux系统中关于存储管理的基础知识，特别是如何使用`fdisk`工具对硬盘进行分区。我们将从存储的基本概念讲起，逐步深入到分区的原理、类型以及实际操作。

## 概述：Linux存储基础

上一节我们介绍了Linux系统的基本概念，本节中我们来看看存储管理。在Linux操作系统中，存储管理与Windows有显著不同。Linux没有“C盘”、“D盘”这样的盘符概念，所有的存储空间都必须**挂载**到某个目录上才能使用。这意味着数据是基于目录结构存放的，而不是基于独立的磁盘分区。

为了实现挂载，一块硬盘需要经历几个关键步骤：
1.  **分区**：将物理硬盘划分为逻辑上的独立区域。
2.  **格式化**：在分区上建立文件系统，确定数据的组织方式。
3.  **挂载**：将格式化好的分区关联到系统目录树中的某个目录。

接下来，我们将逐一探讨这些步骤。

## 硬盘分区详解

### 分区方案：MBR vs GPT

对硬盘进行分区，本质上是修改硬盘起始位置的一块特殊区域——**分区表**。分区表记录了每个分区的起始和结束位置等信息。目前主流的两种分区表格式是MBR和GPT。

**MBR**
*   **全称**：Master Boot Record（主引导记录）。
*   **特点**：
    *   分区表大小固定为**64字节**。
    *   每个分区记录占用16字节，因此最多只能定义**4个主分区**。
    *   单个分区最大支持**2TB**容量。
*   **扩展分区与逻辑分区**：为了突破4个分区的限制，MBR引入了扩展分区。一个扩展分区可以包含多个**逻辑分区**。系统必须至少有一个**主分区**（通常用于安装操作系统），逻辑分区则用于普通数据存储。

**GPT**
*   **全称**：GUID Partition Table（全局唯一标识分区表）。
*   **特点**：
    *   是UEFI标准的一部分，用于取代MBR。
    *   支持几乎无限数量的分区（通常实现为128个）。
    *   支持远超2TB的极大磁盘容量（如8ZB）。
    *   在磁盘的**头部和尾部**存储分区表副本，提供冗余和校验机制，可靠性更高。

**重要提示**：对硬盘进行分区操作，本质上只是修改分区表，**理论上并不会删除分区内原有的数据**。数据丢失通常发生在后续的格式化或写入操作中。

### 使用fdisk进行分区

`fdisk`是一个交互式的命令行磁盘分区工具。下面我们通过实际操作来学习其用法。

首先，查看当前系统的磁盘情况：
```bash
lsblk
```
此命令会列出所有块设备（如硬盘）及其分区信息。

假设我们要对第二块硬盘 `/dev/vdb` 进行MBR格式的分区，创建2个主分区和包含3个逻辑分区的扩展分区。
```bash
fdisk /dev/vdb
```
进入`fdisk`交互界面后，可以输入 `m` 查看所有可用命令。

以下是实现我们目标的分步操作：

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_1.png)

1.  确保分区表为MBR（DOS）格式。输入 `o` 创建新的MBR分区表。
2.  输入 `p` 打印当前分区表，确认初始状态。
3.  创建第一个主分区：
    *   输入 `n` 创建新分区。
    *   选择 `p` 为主分区。
    *   分区号接受默认值 `1`。
    *   起始扇区接受默认值。
    *   设置大小为 `+500M`。
4.  创建第二个主分区：
    *   再次输入 `n`。
    *   选择 `p`。
    *   分区号为 `2`。
    *   起始扇区默认。
    *   设置大小为 `+500M`。
5.  创建扩展分区以容纳逻辑分区：
    *   输入 `n`。
    *   选择 `e` 为扩展分区。
    *   分区号为 `3`。
    *   将剩余所有空间分配给扩展分区（直接按回车使用默认最大值）。
6.  在扩展分区内创建逻辑分区：
    *   输入 `n`。此时会自动开始创建逻辑分区。
    *   分区号从 `5` 开始（1-4为主/扩展分区预留）。
    *   设置大小为 `+500M`。重复此步骤创建第6、7个逻辑分区。
7.  输入 `p` 查看最终分区布局，确认无误。
8.  输入 `w` 将更改写入磁盘并退出。

在`fdisk`交互界面中，以下命令非常常用：
*   `n`：创建新分区。
*   `d`：删除分区。
*   `p`：打印（显示）分区表。
*   `t`：更改分区的系统ID（例如，改为Linux Swap类型）。
*   `l`：列出所有已知的分区类型ID。
*   `w`：保存更改并退出。
*   `q`：不保存更改并退出。
*   `m`：显示帮助菜单。

## 格式化与挂载分区

分区完成后，分区还不能直接存储文件，需要先**格式化**以创建文件系统（如ext4， XFS）。

例如，将 `/dev/vdb1` 格式化为ext4文件系统：
```bash
mkfs.ext4 /dev/vdb1
```
或者格式化为XFS文件系统：
```bash
mkfs.xfs /dev/vdb1
```

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_30.png)

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_32.png)

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_34.png)

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_36.png)

格式化后，需要**挂载**到目录才能使用。例如，临时挂载到 `/mnt` 目录：
```bash
mount /dev/vdb1 /mnt
```
现在，所有存入 `/mnt` 目录的文件实际都保存在 `/dev/vdb1` 分区上。

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_38.png)

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_40.png)

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_42.png)

查看挂载信息：
```bash
df -Th
```

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_44.png)

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_46.png)

卸载分区：
```bash
umount /mnt
# 或
umount /dev/vdb1
```
**注意**：在对分区进行删除等操作前，务必先卸载该分区。

## 使用parted进行分区（简介）

![](img/48d75b23b3b02f72b77b8ee2f1d528c0_54.png)

除了`fdisk`，`parted`也是一个功能强大的分区工具，并且是当前官方更推荐的工具。它支持更丰富的功能和非交互式操作。其基本用法如下：
```bash
parted /dev/vdc
```
在`parted`交互界面中，可以使用 `help` 查看命令，`print` 查看分区，`mklabel` 设置分区表类型（如 `gpt` 或 `msdos`），`mkpart` 创建分区等。
