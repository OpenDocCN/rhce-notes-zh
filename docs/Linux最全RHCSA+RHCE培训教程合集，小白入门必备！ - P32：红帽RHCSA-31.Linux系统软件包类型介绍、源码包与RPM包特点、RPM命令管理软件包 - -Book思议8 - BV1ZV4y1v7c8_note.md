Linux系统中的软件主要分为**开源软件**和**闭源软件**。

![](img/10da61195e68401a031f4a801f8a50c2_25.png)

![](img/10da61195e68401a031f4a801f8a50c2_27.png)

*   **开源软件**：软件的源代码公开发布，用户可以获取并允许进行二次开发。例如，淘宝基于Nginx二次开发了Tengine。开源软件的优势包括用户使用放心（代码公开可审查）、即使原开发者停止维护，社区也可能继续维护。
*   **闭源软件**：软件的源代码不公开，用户无法查看和修改。例如，Windows操作系统。闭源软件的缺点是，一旦开发者停止维护，用户遇到问题将难以自行解决。

我们主要学习和使用的是开源软件。

## 源码包与RPM包特点

了解了软件包的基本分类后，我们来深入比较源码包和RPM包各自的优缺点。

### 源码包特点

![](img/10da61195e68401a031f4a801f8a50c2_41.png)

![](img/10da61195e68401a031f4a801f8a50c2_43.png)

源码包是直接包含软件源代码的压缩包。例如，从Nginx官网下载的 `nginx-1.18.0.tar.gz` 文件。

![](img/10da61195e68401a031f4a801f8a50c2_45.png)

![](img/10da61195e68401a031f4a801f8a50c2_47.png)

![](img/10da61195e68401a031f4a801f8a50c2_49.png)

*   **优点**：
    1.  **开放源代码**：可以直接查看和修改程序的源代码。
    2.  **安装灵活**：在编译过程中，可以**自定义安装路径**和**选择需要的功能模块**进行安装，有利于节约系统资源。
    3.  **卸载方便**：由于可以自定义安装目录，卸载时直接删除该目录即可，干净彻底。
*   **缺点**：
    1.  **安装过程复杂**：需要用户手动执行 **`./configure`**, **`make`**, **`make install`** 等步骤进行编译。
    2.  **依赖关系需手动解决**：安装前需要自行安装该软件所依赖的其他库或软件。

![](img/10da61195e68401a031f4a801f8a50c2_51.png)

![](img/10da61195e68401a031f4a801f8a50c2_53.png)

### RPM包特点

![](img/10da61195e68401a031f4a801f8a50c2_55.png)

![](img/10da61195e68401a031f4a801f8a50c2_57.png)

![](img/10da61195e68401a031f4a801f8a50c2_59.png)

RPM包是Red Hat Package Manager的缩写，是一种已经编译好的二进制包，文件后缀名为 **`.rpm`**。例如，`vsftpd-3.0.2-25.el7.x86_64.rpm`。

![](img/10da61195e68401a031f4a801f8a50c2_61.png)

*   **优点**：
    1.  **安装简单快捷**：因为已经提前编译好，通常使用一条命令（如 `yum install` 或 `rpm -ivh`）即可快速安装。
*   **缺点**：
    1.  **无法自定义功能**：安装的是官方预编译的版本，包含了大部分功能，无法像源码包那样按需选择。
    2.  **无法自定义安装路径**：软件会安装到默认的系统路径中。
    3.  **看不到源代码**：因为是二进制文件，用户无法查看其源代码。
    4.  **依赖关系复杂**：在安装时可能会遇到复杂的依赖关系问题（如环形依赖），需要手动或借助工具解决。

![](img/10da61195e68401a031f4a801f8a50c2_63.png)

![](img/10da61195e68401a031f4a801f8a50c2_65.png)

**如何选择**：对于简单的软件或快速部署，使用RPM包更为便捷；对于有定制化需求（如特定功能、安装路径）或安全性要求极高的生产环境，则可能需要使用源码包进行编译安装。

![](img/10da61195e68401a031f4a801f8a50c2_67.png)

![](img/10da61195e68401a031f4a801f8a50c2_69.png)

## RPM命令管理软件包

![](img/10da61195e68401a031f4a801f8a50c2_71.png)

前面我们了解了两种包的特点，从本节开始，我们将重点学习如何使用`rpm`命令来管理RPM软件包。首先，我们需要知道软件包从哪里来。

### 软件包来源

Linux系统中的软件包主要有两个来源：
1.  **网络源**：从互联网上的软件仓库下载。
2.  **本地源（系统安装镜像）**：系统安装光盘或ISO镜像文件中自带了大量的软件包。

我们的CentOS系统安装镜像（约4GB）中就包含了超过4000个软件包。要使用这些包，需要先将镜像文件挂载到系统目录。

以下是挂载镜像并访问其中软件包的步骤：
```bash
# 1. 在 /mnt 目录下创建一个挂载点
mkdir /mnt/centos

# 2. 将光驱设备（通常是 /dev/cdrom）挂载到该目录
mount /dev/cdrom /mnt/centos

# 3. 进入软件包目录
cd /mnt/centos/Packages

# 4. 查看包的数量
ls | wc -l
```
挂载后，你就可以在 `/mnt/centos/Packages` 目录下看到所有的 `.rpm` 软件包文件。

### RPM命令格式与常用选项

![](img/10da61195e68401a031f4a801f8a50c2_75.png)

![](img/10da61195e68401a031f4a801f8a50c2_77.png)

`rpm` 命令功能强大，选项繁多。对于初学者，我们首先掌握最常用的安装、查询和卸载选项。

![](img/10da61195e68401a031f4a801f8a50c2_79.png)

![](img/10da61195e68401a031f4a801f8a50c2_81.png)

![](img/10da61195e68401a031f4a801f8a50c2_83.png)

![](img/10da61195e68401a031f4a801f8a50c2_85.png)

基本命令格式为：
```
rpm [选项] [软件包全名/程序名]
```
*   **安装软件**时，需要指定软件包的**全名**（包括路径和完整的文件名）。
*   **查询、升级、卸载**软件时，只需要指定**程序名**（即包名）。

以下是`rpm`命令的黄金组合和常用查询选项：

**安装软件包**
*   `-ivh`：安装软件包并显示详细信息和进度条。
    *   `-i`：安装（install）。
    *   `-v`：显示详细信息（verbose）。
    *   `-h`：显示安装进度（hash）。

![](img/10da61195e68401a031f4a801f8a50c2_87.png)

![](img/10da61195e68401a031f4a801f8a50c2_89.png)

![](img/10da61195e68401a031f4a801f8a50c2_91.png)

![](img/10da61195e68401a031f4a801f8a50c2_93.png)

**查询软件包**
*   `-q`：查询指定软件包是否安装。
*   `-qa`：查询系统中所有已安装的软件包列表。常与 `grep` 命令结合进行过滤。
*   `-qi`：查询已安装软件包的详细信息（版本、开发商、安装时间等）。
*   `-ql`：查询指定软件包在系统中安装了哪些文件以及安装位置。
*   `-qf`：查询某个文件是由哪个软件包安装产生的。

![](img/10da61195e68401a031f4a801f8a50c2_98.png)

**卸载软件包**
*   `-e`：卸载指定的软件包。

### 命令实践示例

![](img/10da61195e68401a031f4a801f8a50c2_108.png)

![](img/10da61195e68401a031f4a801f8a50c2_110.png)

现在，让我们在挂载了镜像的 `Packages` 目录中进行实际操作。

![](img/10da61195e68401a031f4a801f8a50c2_112.png)

![](img/10da61195e68401a031f4a801f8a50c2_114.png)

**1. 安装软件包**
我们以安装 `vsftpd` 软件包为例。
```bash
# 进入软件包目录
cd /mnt/centos/Packages

![](img/10da61195e68401a031f4a801f8a50c2_116.png)

# 使用tab键自动补全包名，然后安装
rpm -ivh vsftpd-3.0.2-25.el7.x86_64.rpm
```
执行后，你会看到安装进度和完成提示。

![](img/10da61195e68401a031f4a801f8a50c2_118.png)

![](img/10da61195e68401a031f4a801f8a50c2_120.png)

![](img/10da61195e68401a031f4a801f8a50c2_122.png)

**2. 查询软件包**
安装完成后，我们可以使用各种查询选项来验证和查看软件信息。
```bash
# 查询vsftpd是否安装
rpm -q vsftpd

# 从所有已安装的包中过滤出包含‘vsft’的包
rpm -qa | grep vsft

# 查询vsftpd软件的详细信息
rpm -qi vsftpd

# 查询vsftpd软件在系统中安装了哪些文件
rpm -ql vsftpd
```

**3. 查询文件归属**
如果我们想知道系统中的一个特定文件或命令是由哪个软件包提供的，可以使用 `-qf` 选项。
```bash
# 查询 /etc/passwd 文件由哪个包安装
rpm -qf /etc/passwd

# 查询 ls 命令由哪个包提供
# 首先用 which 找到 ls 命令的路径
which ls
# 输出可能是 /usr/bin/ls
# 然后查询该文件所属的包
rpm -qf /usr/bin/ls
```

**4. 卸载软件包**
如果需要卸载软件，使用 `-e` 选项。
```bash
# 卸载vsftpd软件包
rpm -e vsftpd
```
**注意**：卸载时使用程序名 `vsftpd`，而不是安装时的全名 `vsftpd-3.0.2-25.el7.x86_64.rpm`。
