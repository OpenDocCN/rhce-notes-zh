在本节课中，我们将要学习Linux系统中最基础也是最重要的概念之一：文件系统的目录结构，以及如何通过绝对路径和相对路径来定位文件。理解这些内容是后续所有文件操作和管理的基础。

## 目录结构概述

首先，我们来了解一下Linux的目录结构。与Windows系统使用C盘、D盘等盘符不同，Linux系统采用单一的树形结构，所有文件和目录都从根目录（`/`）开始。

我们可以使用 `tree` 命令来直观地查看目录的树形结构。如果你的系统尚未安装此命令，可以通过包管理器进行安装。

![](img/968d1eb939092ab41f13e2640f7f3274_9.png)

```bash

# 或 CentOS 8 中使用
dnf install tree
```

```bash

```

## 核心目录详解

![](img/968d1eb939092ab41f13e2640f7f3274_25.png)

![](img/968d1eb939092ab41f13e2640f7f3274_27.png)

根目录下包含许多重要的子目录，每个目录都有其特定的用途。以下是主要目录及其功能的介绍：

![](img/968d1eb939092ab41f13e2640f7f3274_29.png)

![](img/968d1eb939092ab41f13e2640f7f3274_31.png)

![](img/968d1eb939092ab41f13e2640f7f3274_32.png)

![](img/968d1eb939092ab41f13e2640f7f3274_33.png)

*   **`/`（根目录）**：Linux文件系统树形结构的顶端。
*   **`/bin`**：存放系统最基本的二进制可执行命令（如 `ls`, `cat`），所有用户都可以使用。
*   **`/boot`**：存放系统启动时所需的文件，如内核（`vmlinuz`）、引导加载程序（GRUB）等。
*   **`/dev`**：存放设备文件。在Linux中，一切硬件设备（如磁盘、光驱、终端）都被抽象为文件。例如，第一块磁盘通常是 `/dev/sda`。
*   **`/etc`**：存放系统和应用程序的配置文件。例如，用户账户文件 `/etc/passwd`，挂载配置文件 `/etc/fstab`。
*   **`/home`**：普通用户的家目录。每个用户登录后默认进入自己的家目录，如用户 `mk` 的家目录是 `/home/mk`。
*   **`/lib` 和 `/lib64`**：存放系统最基本的共享库文件（类似于Windows的DLL文件），为 `/bin` 和 `/sbin` 中的程序提供支持。
*   **`/media` 和 `/mnt`**：用于临时挂载存储设备（如光盘、U盘、移动硬盘）的目录。
*   **`/opt`**：通常用于安装第三方可选应用程序。
*   **`/proc`**：一个虚拟文件系统，存放系统运行时进程和内核信息（如CPU信息 `/proc/cpuinfo`）。系统重启后该目录内容会消失。
*   **`/root`**：系统管理员（root用户）的家目录。
*   **`/run`**：存放系统运行时的数据，如进程ID文件、服务锁文件等。
*   **`/sbin`**：存放系统管理相关的二进制命令（如 `fdisk`, `mkfs`），通常需要root权限才能执行。
*   **`/tmp`**：存放临时文件，所有用户都可读写。系统会定期清理此目录。注意其权限末尾的 `t`（粘滞位），表示只有文件所有者和root才能删除其中的文件。
*   **`/usr`**：存放用户级的应用程序和文件，可视为“UNIX Software Resource”的缩写。其下也有 `/usr/bin`, `/usr/sbin`, `/usr/lib` 等子目录，通常存放后来安装的软件。
*   **`/var`**：存放经常变化的文件，如日志（`/var/log`）、邮件、缓存数据等。

## 绝对路径与相对路径

![](img/968d1eb939092ab41f13e2640f7f3274_44.png)

![](img/968d1eb939092ab41f13e2640f7f3274_46.png)

路径用于唯一确定文件或目录在文件系统中的位置。主要有两种表示方法：绝对路径和相对路径。

![](img/968d1eb939092ab41f13e2640f7f3274_48.png)

![](img/968d1eb939092ab41f13e2640f7f3274_49.png)

### 绝对路径

![](img/968d1eb939092ab41f13e2640f7f3274_50.png)

![](img/968d1eb939092ab41f13e2640f7f3274_51.png)

![](img/968d1eb939092ab41f13e2640f7f3274_52.png)

![](img/968d1eb939092ab41f13e2640f7f3274_54.png)

![](img/968d1eb939092ab41f13e2640f7f3274_56.png)

绝对路径是从根目录（`/`）开始的完整路径。无论当前位于哪个目录，使用绝对路径都能准确定位到目标。

![](img/968d1eb939092ab41f13e2640f7f3274_58.png)

![](img/968d1eb939092ab41f13e2640f7f3274_60.png)

**示例：**
*   `/etc/passwd`
*   `/home/mk/Documents/report.txt`

![](img/968d1eb939092ab41f13e2640f7f3274_62.png)

![](img/968d1eb939092ab41f13e2640f7f3274_64.png)

使用 `pwd` 命令可以打印出当前工作目录的绝对路径。

![](img/968d1eb939092ab41f13e2640f7f3274_66.png)

### 相对路径

![](img/968d1eb939092ab41f13e2640f7f3274_68.png)

相对路径是相对于当前工作目录的路径。它使用以下符号：
*   `.`（一个点）：代表当前目录。
*   `..`（两个点）：代表上一级目录（父目录）。
*   直接使用文件名或目录名：代表当前目录下的文件或子目录。

![](img/968d1eb939092ab41f13e2640f7f3274_70.png)

![](img/968d1eb939092ab41f13e2640f7f3274_71.png)

![](img/968d1eb939092ab41f13e2640f7f3274_73.png)

**示例：**
假设当前目录是 `/home/mk`。
*   访问当前目录下的 `file1`：`./file1` 或直接 `file1`。
*   访问上一级目录 `/home`：`..`。
*   访问兄弟目录 `mk2` 下的文件：`../mk2/file2`。

![](img/968d1eb939092ab41f13e2640f7f3274_75.png)

![](img/968d1eb939092ab41f13e2640f7f3274_76.png)

![](img/968d1eb939092ab41f13e2640f7f3274_77.png)

![](img/968d1eb939092ab41f13e2640f7f3274_79.png)

以下是使用路径的常见操作：

![](img/968d1eb939092ab41f13e2640f7f3274_81.png)

*   **切换目录**：使用 `cd` 命令。
    ```bash
    cd /etc          # 使用绝对路径切换到 /etc
    cd ..            # 切换到上级目录
    cd ./sysconfig   # 切换到当前目录下的 sysconfig 子目录
    ```

![](img/968d1eb939092ab41f13e2640f7f3274_83.png)

![](img/968d1eb939092ab41f13e2640f7f3274_85.png)

![](img/968d1eb939092ab41f13e2640f7f3274_87.png)

*   **访问文件**：在命令中指定文件路径。
    ```bash
    cat /etc/passwd          # 使用绝对路径查看文件
    cat ../../etc/group      # 使用相对路径查看文件
    ```

![](img/968d1eb939092ab41f13e2640f7f3274_90.png)

本节课中我们一起学习了Linux系统的核心基础知识。我们首先认识了Linux独特的树形目录结构，了解了 `/`, `/bin`, `/etc`, `/home` 等关键目录的作用。接着，我们掌握了定位文件的两种方法：从根目录开始的**绝对路径**和相对于当前位置的**相对路径**（使用 `.` 和 `..`）。理解目录结构和路径是高效管理Linux系统文件的第一步，请务必通过实践加深印象。
