# Linux 从入门到精通：P23：重定向技术2（案例分析篇） 🧩

在本节课中，我们将通过分析几个具体的案例，来综合运用和巩固上一节学习的重定向技术。我们将学习如何将命令的输出和错误信息分别处理，包括丢弃、保存到文件以及同时显示在屏幕上等操作。

## 案例一：显示输出并忽略所有错误

上一节我们介绍了标准输出和标准错误。本案例要求将命令的正常输出显示在终端，同时忽略所有错误信息。

忽略错误信息通常的做法是将其重定向到系统“垃圾桶” `/dev/null`。这是一个特殊的设备文件，任何写入它的数据都会被系统丢弃。

以下是实现此需求的命令格式：
```bash
command 2> /dev/null
```
在这个命令中，`2>` 表示将标准错误（文件描述符2）重定向，而 `/dev/null` 就是丢弃这些信息的目的地。

## 案例二：输出与错误分别发送到不同文件

本案例要求将命令的标准输出发送到一个文件，同时将标准错误发送到另一个文件。

这需要分别对标准输出（`1`）和标准错误（`2`）进行重定向操作。

以下是实现此需求的命令格式：
```bash
command 1> file_output.txt 2> file_error.txt
```
这个命令会将 `command` 的正确输出覆盖写入 `file_output.txt`，同时将其错误信息覆盖写入 `file_error.txt`。

## 案例三：输出与错误发送到同一个新文件

有时我们需要将命令的所有输出（包括正确和错误信息）都收集到同一个文件中。这可以通过 `&>` 操作符来实现。

`&>` 操作符是 `1>` 和 `2>` 的组合简写，表示将标准输出和标准错误都重定向到同一个目标。

以下是实现此需求的命令格式：
```bash
command &> file_all.txt
```
执行此命令后，`file_all.txt` 文件将被创建（或覆盖），并包含命令的全部输出内容。

## 案例四：输出与错误追加到同一现有文件

上一个案例会覆盖目标文件。本案例要求将输出和错误都发送到同一个文件，但必须保留该文件原有的内容。

这需要用到追加重定向 `>>`，并分两步处理：先处理标准输出，再让标准错误指向已重定向的标准输出。

以下是实现此需求的命令格式：
```bash
command >> file_exist.txt 2>&1
```
这个命令的含义是：
1.  `>> file_exist.txt`：将标准输出追加到 `file_exist.txt` 文件末尾。
2.  `2>&1`：将标准错误（文件描述符2）重定向到标准输出（文件描述符1）当前的位置，即同一个文件。

## 案例五：丢弃所有终端显示

本案例要求运行一个命令，并丢弃所有可能的输出（包括正确和错误信息），不让任何内容显示在终端。

这实际上是案例一和案例三的结合，我们需要将全部输出流重定向到 `/dev/null`。

以下是实现此需求的命令格式：
```bash
command &> /dev/null
```
或者使用更明确的写法：
```bash
command > /dev/null 2>&1
```
这两个命令的效果相同，都会将命令产生的任何输出都丢弃。

## 案例六：将输出同时发送到屏幕和文件

有时我们既想在终端看到命令的执行结果，又想将其保存到文件中。这需要用到 `tee` 命令。

`tee` 命令的作用是从标准输入读取数据，并同时写入标准输出（屏幕）和一个或多个文件。它通常与管道 `|` 结合使用。

以下是实现此需求的命令格式：
```bash
command | tee file_save.txt
```
例如，我们有一个文件 `aa.txt`，内容为 `count`。执行以下命令：
```bash
cat aa.txt | tee file_save.txt
```
执行后，`aa.txt` 的内容会显示在屏幕上，同时也会被写入（或覆盖）到 `file_save.txt` 文件中。

## 案例七：保存输出到文件并丢弃错误

本案例结合了两个需求：将命令的标准输出保存到文件，同时丢弃其错误信息。

这需要分别处理两个流：用 `>` 保存标准输出，用 `2>` 将标准错误丢弃到 `/dev/null`。

以下是实现此需求的命令格式：
```bash
command 1> output.txt 2> /dev/null
```
或者简写为：
```bash
command > output.txt 2> /dev/null
```
这个命令会将正确结果保存到 `output.txt`，而错误信息则被静默丢弃。
