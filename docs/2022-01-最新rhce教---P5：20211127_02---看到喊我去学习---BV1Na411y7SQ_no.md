![](img/fff4dece6bbbdcb14e021fc817c91630_0.png)

![](img/fff4dece6bbbdcb14e021fc817c91630_2.png)

在本节课中，我们将要学习 Ansible 中两个核心概念：**变量** 和 **事实**。变量允许我们定义动态值，而事实则是 Ansible 从被管理主机自动收集的系统信息。掌握它们对于编写灵活、可复用的 Playbook 至关重要。

![](img/fff4dece6bbbdcb14e021fc817c91630_3.png)

## 概述 📋

![](img/fff4dece6bbbdcb14e021fc817c91630_4.png)

Ansible 变量用于存储和引用在 Playbook 或角色中可能发生变化的值。Ansible 事实则是 Ansible 在执行任务前，自动从目标主机收集的、反映其当前状态的信息（如 IP 地址、操作系统版本等）。理解如何定义、使用变量以及如何利用事实，是构建高效自动化任务的基础。

![](img/fff4dece6bbbdcb14e021fc817c91630_5.png)

上一节我们介绍了 Ansible 的基本结构和 Playbook 的编写，本节中我们来看看如何让 Playbook 变得更加动态和智能。

![](img/fff4dece6bbbdcb14e021fc817c91630_7.png)

## 1. 定义与使用变量

![](img/fff4dece6bbbdcb14e021fc817c91630_8.png)

变量可以在多个地方定义，例如在 Playbook 中、在独立的变量文件中、或在命令行传递。定义后，可以使用 Jinja2 模板语法 `{{ variable_name }}` 来引用它们。

![](img/fff4dece6bbbdcb14e021fc817c91630_10.png)

以下是定义变量的几种常见方式：

![](img/fff4dece6bbbdcb14e021fc817c91630_11.png)

*   **在 Playbook 中直接定义：**
    ```yaml
    - hosts: webservers
      vars:
        http_port: 80
        max_clients: 200
      tasks:
        - name: Ensure Apache is at the latest version
          yum:
            name: httpd
            state: latest
        - name: Write the configuration file
          template:
            src: /srv/httpd.j2
            dest: /etc/httpd.conf
          notify:
            - restart apache
      handlers:
        - name: restart apache
          service:
            name: httpd
            state: restarted
    ```

![](img/fff4dece6bbbdcb14e021fc817c91630_13.png)

![](img/fff4dece6bbbdcb14e021fc817c91630_14.png)

*   **在独立的变量文件中定义（`vars_files`）：**
    ```yaml
    - hosts: all
      vars_files:
        - vars.yml
      tasks:
        - debug:
            msg: "Server {{ inventory_hostname }} uses {{ preferred_db }}"
    ```
    其中 `vars.yml` 文件内容可能为：
    ```yaml
    preferred_db: mysql
    package_name: nginx
    ```

![](img/fff4dece6bbbdcb14e021fc817c91630_15.png)

*   **通过命令行传递变量（`-e` 或 `--extra-vars`）：**
    ```bash
    ansible-playbook release.yml --extra-vars "version=1.23.45 other_variable=foo"
    ```

![](img/fff4dece6bbbdcb14e021fc817c91630_17.png)

## 2. 理解 Ansible 事实

![](img/fff4dece6bbbdcb14e021fc817c91630_19.png)

事实是 Ansible 在 Playbook 执行开始时，通过一个特殊的“收集事实”阶段自动获取的信息。你可以使用 `setup` 模块来查看所有事实。

![](img/fff4dece6bbbdcb14e021fc817c91630_21.png)

以下是查看事实的方法：

![](img/fff4dece6bbbdcb14e021fc817c91630_22.png)

*   使用 `ansible` 命令查看指定主机的所有事实：
    ```bash
    ansible hostname -m setup
    ```
*   在 Playbook 中，事实会被自动注入，并可以直接引用。例如，`{{ ansible_facts[‘default_ipv4’][‘address’] }}` 获取主机的 IPv4 地址。

![](img/fff4dece6bbbdcb14e021fc817c91630_23.png)

上一节我们介绍了如何手动定义变量，本节中我们来看看系统自动提供的事实变量如何增强我们的自动化脚本。

![](img/fff4dece6bbbdcb14e021fc817c91630_24.png)

## 3. 在任务中使用变量和事实

![](img/fff4dece6bbbdcb14e021fc817c91630_26.png)

变量和事实可以用于任务的任何部分，使任务具备条件性和适应性。

![](img/fff4dece6bbbdcb14e021fc817c91630_27.png)

以下是在任务中使用的几个例子：

![](img/fff4dece6bbbdcb14e021fc817c91630_28.png)

![](img/fff4dece6bbbdcb14e021fc817c91630_29.png)

*   **在模块参数中使用：**
    ```yaml
    - name: Install a package
      yum:
        name: "{{ package_name }}"
        state: present
    ```

![](img/fff4dece6bbbdcb14e021fc817c91630_31.png)

*   **在模板文件中使用（Jinja2）：**
    模板文件 `index.html.j2` 可能包含：
    ```html
    <html>
    <body>
        <p>Welcome to {{ ansible_facts['fqdn'] }}!</p>
        <p>This server has {{ ansible_facts['memtotal_mb'] }} MB of memory.</p>
    </body>
    </html>
    ```

![](img/fff4dece6bbbdcb14e021fc817c91630_33.png)

*   **结合条件判断（`when`）：**
    ```yaml
    - name: Shut down Debian flavored systems
      command: /sbin/shutdown -t now
      when: ansible_facts['os_family'] == "Debian"
    ```

![](img/fff4dece6bbbdcb14e021fc817c91630_34.png)

## 4. 变量优先级

![](img/fff4dece6bbbdcb14e021fc817c91630_35.png)

Ansible 变量可以从多个来源定义，它们遵循一个明确的优先级顺序。高优先级的变量会覆盖低优先级的变量。

![](img/fff4dece6bbbdcb14e021fc817c91630_36.png)

以下是变量优先级从低到高的简化列表（越靠后优先级越高）：

![](img/fff4dece6bbbdcb14e021fc817c91630_38.png)

![](img/fff4dece6bbbdcb14e021fc817c91630_39.png)

1.  命令行值（`-e`）
2.  角色默认值（`roles/x/defaults/main.yml`）
3.  清单变量（`inventory` 文件或脚本中定义）
4.   Playbook 变量（`vars:` 块）
5.  角色变量（`roles/x/vars/main.yml`）
6.  块变量（`block:` 中的 `vars:`）
7.  任务变量（在任务中直接定义的 `vars:`）
8.  `set_fact` 模块设置的变量
9.  通过 `-e` 传递的额外变量
10. 主机事实 / 缓存的事实

![](img/fff4dece6bbbdcb14e021fc817c91630_41.png)

了解优先级有助于在变量冲突时进行调试。

## 总结 🎯

![](img/fff4dece6bbbdcb14e021fc817c91630_43.png)

本节课中我们一起学习了 Ansible 中变量与事实的管理。我们了解了如何在不同位置定义和使用变量，如何利用 Ansible 自动收集的事实信息来编写自适应的 Playbook，以及变量覆盖的优先级规则。变量和事实是 Ansible 动态性和强大功能的核心，熟练掌握它们将使你的自动化脚本更加灵活和健壮。
