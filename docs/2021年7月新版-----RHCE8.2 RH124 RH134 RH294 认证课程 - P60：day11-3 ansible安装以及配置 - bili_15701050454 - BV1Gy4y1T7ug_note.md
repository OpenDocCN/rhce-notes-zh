## 概述

Ansible是一个基于Python开发的自动化运维工具，它通过SSH协议管理节点，无需在目标主机上安装代理。其核心特性包括模块化、幂等性和使用YAML编排任务，能够高效地完成批量、重复性的系统配置与管理任务。

上一节我们介绍了课程安排，本节中我们来看看Ansible的安装与基础配置。

## Ansible简介与核心概念

Ansible是一个配置管理系统。它通过SSH访问服务器或设备，采用推送方式而非代理方式执行任务，可以将代码部署到任意数量的服务器上。

其核心特性如下：
*   **模块化**：Ansible的功能通过模块实现，用户主要任务是使用模块。
*   **基于Python**：工具本身由Python实现。
*   **部署简单**：主要依赖Python和SSH。
*   **安全**：基于SSH协议。
*   **支持Playbook**：使用YAML格式编排任务。
*   **幂等性**：无论执行多少次，最终效果一致，不会因重复执行导致意外结果。例如，一个确保软件包安装的模块，多次运行也只会安装一次。

Ansible的架构包含几个核心组件：
*   **模块**：用于执行具体任务的代码单元。
*   **SSH**：用于与托管节点通信的协议。
*   **资产清单**：定义了被管理的主机列表。

![](img/e89e0f9a3960aa72ad62d57d084aed67_28.png)

以下是Ansible执行过程的简化描述：
```
用户 -> Ansible命令/Playbook -> 核心引擎 -> (通过模块 & 资产清单 & SSH) -> 托管主机
```

## 安装Ansible

![](img/e89e0f9a3960aa72ad62d57d084aed67_32.png)

在RHEL 8或CentOS 8系统上，可以通过启用EPEL仓库来安装Ansible。

![](img/e89e0f9a3960aa72ad62d57d084aed67_34.png)

以下是安装步骤：
1.  确保系统已挂载安装介质（如光盘），以便解决依赖。
2.  添加EPEL仓库并安装Ansible。

![](img/e89e0f9a3960aa72ad62d57d084aed67_36.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_38.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_40.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_42.png)

安装命令示例：
```bash
sudo yum install epel-release -y
sudo yum install ansible -y
```

![](img/e89e0f9a3960aa72ad62d57d084aed67_44.png)

安装完成后，可以验证版本：
```bash
ansible --version
```

## 配置Ansible

![](img/e89e0f9a3960aa72ad62d57d084aed67_46.png)

Ansible的配置文件优先级从高到低为：
1.  环境变量 `ANSIBLE_CONFIG` 指定的文件。
2.  当前工作目录下的 `ansible.cfg`。
3.  用户家目录下的 `~/.ansible.cfg`。
4.  系统默认的 `/etc/ansible/ansible.cfg`。

![](img/e89e0f9a3960aa72ad62d57d084aed67_48.png)

实践中，通常在项目目录中创建 `ansible.cfg` 文件以便管理。

![](img/e89e0f9a3960aa72ad62d57d084aed67_50.png)

一个基础的配置文件示例如下：
```ini
[defaults]
inventory = ./inventory
remote_user = student

![](img/e89e0f9a3960aa72ad62d57d084aed67_52.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_54.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_55.png)

[privilege_escalation]
become = True
become_method = sudo
become_user = root
become_ask_pass = False
```

![](img/e89e0f9a3960aa72ad62d57d084aed67_57.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_59.png)

配置项说明：
*   `inventory`: 指定资产清单文件的路径。
*   `remote_user`: 指定在托管主机上执行任务的用户。
*   `become`, `become_method`, `become_user`: 配置权限提升（提权）为root用户。
*   `become_ask_pass`: 提权时是否询问密码。

![](img/e89e0f9a3960aa72ad62d57d084aed67_61.png)

**重要**：若配置了提权，必须在所有托管节点上为 `remote_user`（如student）配置相应的sudo权限，无需密码。例如，在 `/etc/sudoers.d/` 下添加文件：
```
student ALL=(root) NOPASSWD: ALL
```

### 定义主机与分组

![](img/e89e0f9a3960aa72ad62d57d084aed67_75.png)

清单文件的基本格式是INI或YAML。主机可以单独定义，也可以分组。

![](img/e89e0f9a3960aa72ad62d57d084aed67_77.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_78.png)

定义单个主机：
```
servera.example.com
172.25.250.10
```

定义主机组（组名用方括号`[]`括起）：
```
[dev]
servera.example.com
serverb.example.com

![](img/e89e0f9a3960aa72ad62d57d084aed67_80.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_82.png)

[test]
serverc.example.com
serverd.example.com
```

### 使用清单

![](img/e89e0f9a3960aa72ad62d57d084aed67_90.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_92.png)

使用 `ansible` 命令的 `--list-hosts` 选项可以列出匹配的主机，而不执行任何操作：
```bash
ansible dev --list-hosts
ansible all --list-hosts
```

![](img/e89e0f9a3960aa72ad62d57d084aed67_96.png)

使用临时命令测试与主机的连接（使用 `ping` 模块）：
```bash

```

### 高级清单语法

![](img/e89e0f9a3960aa72ad62d57d084aed67_106.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_108.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_110.png)

1.  **嵌套组**：一个组可以包含其他组作为成员。
    ```
    [web_servers]
    servera.example.com
    serverb.example.com

    [app_servers]
    serverc.example.com

    [datacenter:children] # 定义父组，使用`:children`后缀
    web_servers
    app_servers
    ```

![](img/e89e0f9a3960aa72ad62d57d084aed67_112.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_114.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_116.png)

2.  **主机范围**：可以使用数字或字母范围简化连续主机的定义。
    ```
    [range_group]
    192.168.1.[10:20] # 表示192.168.1.10到192.168.1.20
    server[01:10].example.com
    ```

![](img/e89e0f9a3960aa72ad62d57d084aed67_118.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_120.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_122.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_124.png)

![](img/e89e0f9a3960aa72ad62d57d084aed67_126.png)

3.  **模式匹配**：在指定目标主机时支持通配符`*`、逻辑或`,`、逻辑与`:`、逻辑非`!`等。
    ```bash
    ansible ‘server*’ --list-hosts # 所有以server开头的主机
    ansible ‘dev:test’ --list-hosts # 在dev组且同时在test组的主机（与）
    ansible ‘dev,!servera’ --list-hosts # 在dev组但排除servera主机（非）
    ```
