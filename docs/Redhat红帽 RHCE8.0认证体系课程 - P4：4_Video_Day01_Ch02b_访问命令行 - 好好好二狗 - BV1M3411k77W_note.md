## 系统登录方式

![](img/4573f474d93e43f49bfd07fe6765fb44_8.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_10.png)

对于服务器而言，我们通常不会直接连接键盘和显示器。除了可以像个人电脑（PC）一样直接连接外，服务器更常见的方式是通过远程控制口（如iDRAC、iLO）进行管理。无论是直接连接还是远程连接，本质上都是通过一个控制台来操作系统。

![](img/4573f474d93e43f49bfd07fe6765fb44_12.png)

本地登录相当于直接为服务器接上键盘、鼠标和显示器。但在Linux系统中，图形界面下的鼠标使用较少，很多时候仅凭键盘即可完成操作。

![](img/4573f474d93e43f49bfd07fe6765fb44_14.png)

*   **图形界面**：提供可视化的桌面环境，用户可以通过点击图标和菜单进行操作。
*   **文本界面**：也称为终端或控制台，用户通过输入文本命令来操作系统。

![](img/4573f474d93e43f49bfd07fe6765fb44_16.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_18.png)

在文本界面中，存在两种终端类型：
*   **tty**：物理终端或虚拟控制台，通常指直接连接到服务器显示器或通过快捷键切换出来的界面。
*   **pts**：伪终端，通常指通过SSH等远程连接工具打开的终端窗口。

![](img/4573f474d93e43f49bfd07fe6765fb44_20.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_22.png)

例如，在图形界面中打开的终端窗口属于`pts`，而从图形界面切换到纯文本的黑色屏幕则属于`tty`。

![](img/4573f474d93e43f49bfd07fe6765fb44_24.png)

## 界面切换

![](img/4573f474d93e43f49bfd07fe6765fb44_26.png)

上一节我们介绍了两种主要的操作界面。本节中我们来看看如何在它们之间进行切换。

### 临时切换

![](img/4573f474d93e43f49bfd07fe6765fb44_28.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_30.png)

在已经启动的系统中，我们可以使用快捷键在不同界面间临时切换。

![](img/4573f474d93e43f49bfd07fe6765fb44_32.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_34.png)

以下是切换方法：
*   从**图形界面**切换到**文本界面**：按下 `Ctrl + Alt + F2` 到 `F6` 中的任意一个（例如 `Ctrl + Alt + F3`）。
*   从**文本界面**切换回**图形界面**：按下 `Ctrl + Alt + F1`。
> **注意**：对于某些笔记本电脑，可能需要配合 `Fn` 键使用（例如 `Ctrl + Alt + Fn + F2`）。

![](img/4573f474d93e43f49bfd07fe6765fb44_36.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_38.png)

### 永久切换

![](img/4573f474d93e43f49bfd07fe6765fb44_40.png)

除了临时切换，我们还可以设置系统默认的启动目标，即开机后直接进入图形界面还是文本界面。

![](img/4573f474d93e43f49bfd07fe6765fb44_42.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_44.png)

1.  **查看当前默认启动目标**：
    ```bash
    systemctl get-default
    ```
    此命令会显示当前默认目标，例如 `graphical.target`（图形界面）或 `multi-user.target`（多用户文本界面）。

![](img/4573f474d93e43f49bfd07fe6765fb44_46.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_48.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_50.png)

2.  **设置永久启动目标**：
    *   设置为文本界面：
        ```bash
        sudo systemctl set-default multi-user.target
        ```
    *   设置为图形界面：
        ```bash
        sudo systemctl set-default graphical.target
        ```
    设置完成后，需要重启系统才能生效。

![](img/4573f474d93e43f49bfd07fe6765fb44_52.png)

3.  **临时切换启动目标（无需重启）**：
    *   从图形界面切换到文本界面：
        ```bash
        sudo systemctl isolate multi-user.target
        ```
    *   从文本界面切换到图形界面：
        ```bash
        sudo systemctl isolate graphical.target
        ```
    `isolate` 命令会立即切换到指定目标，但下次启动时仍会恢复为默认设置。

> **重要**：不能将默认启动目标设置为 `reboot.target`（重启）、`poweroff.target`（关机）或 `rescue.target`（救援模式）。

![](img/4573f474d93e43f49bfd07fe6765fb44_54.png)

## 远程登录与注销

![](img/4573f474d93e43f49bfd07fe6765fb44_56.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_58.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_60.png)

在实际工作场景中，管理员通常从一台Windows或Mac电脑远程连接到Linux服务器进行操作。

![](img/4573f474d93e43f49bfd07fe6765fb44_62.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_64.png)

*   **从Windows连接Linux**：Windows系统通常不自带SSH客户端，需要使用第三方工具，例如：
    *   Xshell
    *   PuTTY
    *   SecureCRT
*   **从Mac或Linux连接Linux**：系统终端直接支持SSH命令。

![](img/4573f474d93e43f49bfd07fe6765fb44_66.png)

![](img/4573f474d93e43f49bfd07fe6765fb44_68.png)

远程登录的命令格式为：
```bash
ssh 用户名@服务器IP地址
```
例如：`ssh root@192.168.1.100`。执行后会提示输入对应用户的密码。

![](img/4573f474d93e43f49bfd07fe6765fb44_70.png)

登录成功后，如果需要结束当前会话并退出，有两种方法：
1.  输入命令 `exit`
2.  按下快捷键 `Ctrl + D`

![](img/4573f474d93e43f49bfd07fe6765fb44_72.png)

## 命令执行基础

成功连接到命令行后，核心工作就是执行命令。本节我们来学习命令执行的一些基本逻辑和技巧。

### 命令组合执行

有时我们需要依次执行多个命令，Linux Shell提供了几种逻辑运算符来实现。

以下是三种常用的命令连接方式：
*   **分号 `;`**：顺序执行。无论前一条命令是否成功，都会执行后面的命令。
    ```bash
    cd /tmp; ls  # 先切换到/tmp目录，然后列出其内容
    ```
*   **逻辑与 `&&`**：只有前一条命令**执行成功**（返回状态码为0），才会执行后一条命令。
    ```bash
    mkdir test && cd test  # 创建test目录，只有创建成功才会进入该目录
    ```
*   **逻辑或 `||`**：只有前一条命令**执行失败**（返回状态码非0），才会执行后一条命令。
    ```bash
    cd /nonexistent || echo "目录不存在"  # 尝试进入一个不存在的目录，失败后输出提示
    ```

### 长命令换行

当命令非常长时，为了提高可读性，可以使用反斜杠 `\` 进行换行。Shell会将反斜杠后的换行符忽略，视为同一行命令继续输入。

![](img/4573f474d93e43f49bfd07fe6765fb44_74.png)

```bash
tail -n 1 \
/etc/passwd \
/etc/group
```
这条命令等价于 `tail -n 1 /etc/passwd /etc/group`，用于查看两个文件的最后一行。

## 命令行实用技巧

![](img/4573f474d93e43f49bfd07fe6765fb44_80.png)

熟练使用命令行快捷键和历史记录功能，可以极大提升操作效率。

### 命令补全 (Tab键)

![](img/4573f474d93e43f49bfd07fe6765fb44_82.png)

Tab键是命令行中最常用的效率工具。

*   **按一次Tab**：如果输入的开头能唯一确定一个命令、文件或目录名，系统会自动补全。
    ```bash
    # 输入 `sys`， 然后按Tab，可能会补全为 `systemctl`
    sys<Tab>
    ```
*   **按两次Tab**：如果输入的开头有多个匹配项，系统会列出所有可能的选择。
    ```bash
    # 输入 `net`， 然后按两次Tab，会列出所有以net开头的命令，如 netstat, networkctl等
    net<Tab><Tab>
    ```
此功能依赖于 `bash-completion` 软件包，标准安装的RHEL系统通常已包含。

![](img/4573f474d93e43f49bfd07fe6765fb44_84.png)

### 编辑快捷键

![](img/4573f474d93e43f49bfd07fe6765fb44_86.png)

在输入命令时，以下快捷键能帮助你快速移动光标和编辑文本。

以下是常用的命令行编辑快捷键：
*   `Ctrl + A`：移动光标到**行首**。
*   `Ctrl + E`：移动光标到**行尾**。
*   `Ctrl + U`：删除从光标处到**行首**的所有内容。
*   `Ctrl + K`：删除从光标处到**行尾**的所有内容。
*   `Ctrl + 左箭头`：将光标向左移动一个**单词**。
*   `Ctrl + 右箭头`：将光标向右移动一个**单词**。
*   `Ctrl + L`：清屏，等同于 `clear` 命令。

![](img/4573f474d93e43f49bfd07fe6765fb44_88.png)

### 历史命令搜索与调用

![](img/4573f474d93e43f49bfd07fe6765fb44_90.png)

Shell会记录你执行过的命令历史。

![](img/4573f474d93e43f49bfd07fe6765fb44_92.png)

*   **查看历史记录**：使用 `history` 命令可以列出最近执行过的命令列表（默认保存1000条）。
*   **执行历史命令**：
    *   使用上下方向键可以翻阅之前执行过的命令。
    *   输入 `!序号` 可以直接执行历史记录中对应编号的命令。例如 `!101`。
    *   输入 `!!` 可以执行上一条命令。
*   **搜索历史命令**：按下 `Ctrl + R`，然后输入关键词，Shell会反向搜索历史命令。找到后按回车即可执行，按左右方向键可以编辑该命令。

![](img/4573f474d93e43f49bfd07fe6765fb44_94.png)
