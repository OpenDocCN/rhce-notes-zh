## 第五章节收尾：用户身份切换与权限委派

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_4.png)

上一节我们介绍了Linux系统中的三类用户身份。本节中，我们来看看如何在不同的用户身份之间进行切换，以及如何安全地授予普通用户执行特定管理任务的权限。

### 用户身份切换：`su` 命令

`su` 命令用于切换用户身份。其基本语法非常简单。

```bash
su [用户名]
```

例如，要从当前用户切换到名为 `lin` 的用户，只需执行：

```bash
su lin
```

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_6.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_8.png)

执行此命令后，终端会话将切换到 `lin` 用户的环境。

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_10.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_11.png)

#### 完全变更与保留环境

`su` 命令有一个重要的参数 `-`（减号）。这个参数决定了切换用户时是否完全变更环境。

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_13.png)

*   **不加 `-` 参数**：仅切换用户身份，但会保留原始用户的环境变量信息。
*   **加 `-` 参数**：完全变更到新用户身份，包括环境变量、家目录等都会切换到新用户的配置。

**建议在切换用户时使用 `su - [用户名]`，以获得一个干净、完整的新用户会话。**

**权限差异**：
*   管理员用户（如 `root`）切换到任何普通用户都**不需要输入密码**。
*   普通用户切换到管理员或其他用户身份时，**需要输入目标用户的密码**。

### 权限精准委派：`sudo` 服务

有时，管理员希望普通用户能执行某些特定的管理任务（如重启服务），但又不想告知其管理员密码。这时就需要 `sudo` 服务。

`sudo` 允许系统管理员授权给特定的普通用户，使其能够以管理员或其他用户的身份运行特定的命令，而不是获得完整的身份变更。

#### 配置 `sudo` 权限

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_15.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_16.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_17.png)

`sudo` 的配置文件是 `/etc/sudoers`。**切勿直接使用普通文本编辑器修改此文件**，应使用专用的 `visudo` 命令进行编辑，该命令会进行语法检查，防止配置错误导致系统无法使用 `sudo`。

```bash
visudo
```

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_19.png)

在 `/etc/sudoers` 文件中，我们可以为用户或用户组添加授权。配置行的基本格式如下：

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_20.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_22.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_24.png)

```
[用户] [来自的主机]=(可切换的身份) [可执行的命令列表]
```

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_26.png)

**配置示例**：
假设我们想授权用户 `zhangsan` 可以重启服务器和查看系统日志。

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_28.png)

1.  首先，找到命令的完整路径：
    ```bash
    which reboot
    which cat
    ```
    假设输出为 `/usr/sbin/reboot` 和 `/usr/bin/cat`。

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_30.png)

2.  在 `/etc/sudoers` 文件中（建议在约100行后添加），写入：
    ```
    zhangsan ALL=(ALL) /usr/sbin/reboot, /usr/bin/cat
    ```
    *   `zhangsan`：被授权的用户名。
    *   `ALL`：允许从任何主机登录。
    *   `(ALL)`：可以以任何用户身份运行命令（通常指 `root`）。
    *   最后是允许执行的命令**完整路径**，多个命令用逗号分隔。

**重要安全提醒**：**切勿授予用户编辑系统配置文件的权限（如 `vim`, `vi`, `nano`）**。因为Linux中一切皆文件，配置服务就是修改文件。一旦用户获得编辑器权限，他就可以修改 `/etc/sudoers` 文件来扩大自己的权限，造成安全风险。

#### 使用 `sudo` 执行命令

配置完成后，用户 `zhangsan` 在执行被授权的命令前加上 `sudo` 即可。

```bash
# 普通用户无法直接重启
reboot
# 使用 sudo 提权执行
sudo reboot
```

系统会提示输入**当前用户（zhangsan）自己的密码**，而不是管理员密码。验证通过后，命令将以管理员权限执行。

## 第六章节：磁盘结构与管理入门

本节我们将开始学习Linux的磁盘管理。首先从理解文件系统的目录结构开始。

### 文件系统层次结构标准（FHS）

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_32.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_34.png)

FHS（Filesystem Hierarchy Standard）定义了Linux系统中各个目录的用途，确保不同发行版之间结构的一致性，方便用户和管理员查找文件。

以下是一些核心目录及其作用：

*   **`/`**：根目录，是所有目录的起点。
*   **`/boot`**：存放系统启动所需的文件，如内核、引导程序。
*   **`/dev`**：存放**设备文件**。如硬盘（`/dev/sda`）、光盘（`/dev/cdrom`）、终端等。**这是需要重点记住的目录之一**。
*   **`/etc`**：存放**系统及服务的配置文件**。**这是需要重点记住的目录之一**。
*   **`/home`**：普通用户的**家目录**。每个用户在此拥有一个以自己用户名命名的子目录。**这是需要重点记住的目录之一**。
*   **`/root`**：系统管理员（root）的家目录。
*   **`/bin`, `/sbin`**：存放二进制可执行命令。`/sbin` 下的命令通常需要管理员权限。
*   **`/usr`**：存放用户安装的应用程序和文件。
*   **`/var`**：存放经常发生变动的数据，如日志、数据库文件等。
*   **`/tmp`**：临时文件目录，所有用户都可读写。
*   **`/proc`**：虚拟文件系统，映射系统运行状态和内核信息（如内存、CPU使用情况），不是真实文件。
*   **`/mnt`, `/media`**：用于临时挂载文件系统（如U盘、光盘）的目录。

### 物理设备的命名规则

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_36.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_37.png)

在Linux中，一切皆文件，硬件设备也被映射为 `/dev` 目录下的文件。命名遵循一定的规则（由 `udev` 服务管理）。

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_39.png)

#### 硬盘设备

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_41.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_43.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_44.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_46.png)

*   **主流的硬盘（包括SATA、SCSI、USB闪存盘）**：名称格式为 `/dev/sd[a-z]`。
    *   `sd` 表示设备类型。
    *   `[a-z]` 表示系统的识别顺序，第一个被识别的硬盘是 `sda`，第二个是 `sdb`，以此类推。

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_48.png)

#### 硬盘分区

在硬盘名称后会附加分区编号，例如 `/dev/sda1`。

*   **主分区或扩展分区**：编号为 **1-4**。
*   **逻辑分区**：编号从 **5** 开始。

**示例分析**：
*   `/dev/sdb3`：系统第二个硬盘上的第三个主分区或扩展分区。
*   `/dev/sda5`：系统第一个硬盘上的第一个逻辑分区（编号5及以上一定是逻辑分区）。

#### 光盘设备

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_50.png)

光盘设备通常被命名为 `/dev/cdrom`，这是一个指向实际设备（如 `/dev/sr0`）的链接。如果有多个光驱，则依次为 `/dev/sr0`, `/dev/sr1`...

### 文件系统类型

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_60.png)

就像Windows有NTFS、FAT32一样，Linux也有多种文件系统格式。

*   **EXT4**：红帽RHEL 6及许多当前发行版的默认文件系统。在EXT3基础上提升了性能和单文件大小支持。
*   **XFS**：红帽RHEL 7/8的默认文件系统。特别适合处理大容量存储，支持高达18EB的单文件系统，适用于大数据和云计算场景。
*   **虚拟文件系统（VFS）**：Linux内核中的一个抽象层，它为用户程序提供了统一的文件操作接口，屏蔽了底层不同文件系统（如EXT4、XFS、Btrfs）的差异。用户无需关心文件保存在哪种格式的分区上。

### 磁盘分区、格式化与挂载

给系统添加一块新硬盘后，需要经过三个步骤才能使用：

1.  **分区**：使用 `fdisk` 或 `parted` 等工具，将一块物理硬盘划分成多个逻辑区域。
    *   **类比**：买回一大张纸，根据用途裁剪成不同大小的小张。

2.  **格式化**：使用 `mkfs` 命令，在分区上创建特定的文件系统（如EXT4、XFS）。
    *   **类比**：给裁剪好的纸画上格子（作文本、数学本），规定书写格式。
    *   **命令示例**：`mkfs.xfs /dev/sdb1`

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_62.png)

![](img/bc2f1d2b8400fc7b7c3472df1f395a6b_64.png)

3.  **挂载**：使用 `mount` 命令，将格式化好的分区关联到系统目录树中的一个空目录上。
    *   **本质**：将设备文件与一个目录进行关联的动作。
    *   **目的**：使得用户可以通过访问该目录来使用硬盘分区中的资源。
    *   **命令示例**：`mount /dev/sdb1 /mnt/newdisk`
    *   **卸载命令**：`umount /dev/sdb1` 或 `umount /mnt/newdisk`

**注意**：使用 `mount` 命令进行的挂载是临时的，重启后失效。如需永久挂载，需要编辑 `/etc/fstab` 配置文件。

#### 操作流程概览

假设为系统添加了一块新硬盘，被识别为 `/dev/sdb`。

```bash
# 1. 分区
fdisk /dev/sdb
# 在交互界面中，使用 n 创建新分区，w 保存退出
# 通知内核分区表已更新
partprobe

# 2. 格式化
mkfs.xfs /dev/sdb1

# 3. 创建挂载点并挂载
mkdir /mnt/newdisk
mount /dev/sdb1 /mnt/newdisk

# 4. 查看挂载结果
df -h
```
