# Linux文本处理工具：1：文本提取工具详解

在本节课中，我们将要学习Linux中用于查看和提取文本内容的基础工具。这些工具是处理日志、编写脚本和进行系统管理的基础，掌握它们对于打好Linux基础至关重要。

## 概述：文本处理的重要性

![](img/2efbf0aec4f82f3f952c3af74359a409_1.png)

文本处理工具的核心功能是查看、过滤、修改、查找、替换、统计和排序文本内容。无论是处理日志文件还是编写脚本，这些工具都扮演着关键角色。本章内容分为三个部分：文本提取、文本分析统计以及文本操作。本节我们将重点学习文本提取部分。

## 查看文件内容：cat, more, less

上一节我们介绍了文本处理的重要性，本节中我们来看看最基础的文本查看工具。

![](img/2efbf0aec4f82f3f952c3af74359a409_3.png)

### cat命令：连接并显示文件

![](img/2efbf0aec4f82f3f952c3af74359a409_5.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_7.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_9.png)

`cat`命令是最简单的文本查看工具，用于将文件内容打印到终端。

**基本语法**：
```bash
cat [选项] 文件名
```

**常用选项**：
*   `-n`：显示行号。

**示例**：
1.  查看单个文件：
    ```bash
    cat /etc/passwd
    ```
2.  查看文件并显示行号：
    ```bash
    cat -n /etc/passwd
    ```
3.  合并查看多个文件：
    ```bash
    cat /etc/passwd /etc/profile
    ```
4.  将多个文件内容合并输出到一个新文件：
    ```bash
    cat file1 file2 > newfile
    ```

![](img/2efbf0aec4f82f3f952c3af74359a409_11.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_13.png)

`cat`会一次性输出全部内容，对于大文件，浏览不便。

### more命令：分页显示（仅向前）

`more`命令用于分页显示文件内容，适合查看较长的文件。

![](img/2efbf0aec4f82f3f952c3af74359a409_15.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_17.png)

**基本语法**：
```bash
more 文件名
```

![](img/2efbf0aec4f82f3f952c3af74359a409_19.png)

**交互操作**：
*   **空格键**：向下翻一页。
*   **回车键**：向下滚动一行。
*   浏览到文件末尾后，命令自动退出。

![](img/2efbf0aec4f82f3f952c3af74359a409_21.png)

`more`只能向前浏览，无法回看。

### less命令：分页显示（可前后翻看）

`less`命令是`more`的增强版，可以前后自由翻看文件内容，功能更强大。

![](img/2efbf0aec4f82f3f952c3af74359a409_23.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_25.png)

**基本语法**：
```bash
less 文件名
```

![](img/2efbf0aec4f82f3f952c3af74359a409_27.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_29.png)

**交互操作**：
以下是`less`命令的常用操作键：
*   **空格键 / PageDown**：向下翻一页。
*   **PageUp**：向上翻一页。
*   **上下箭头键**：向上/下滚动一行。
*   `/关键词**：**搜索指定关键词，按`n`键向下查找下一个，按`N`键向上查找上一个。
*   **v键**：进入`vi`编辑器模式，可直接编辑当前文件。编辑后保存退出（`:wq`）会返回`less`视图。
*   **q键**：退出`less`。

**提示**：Linux的`man`帮助手册使用的就是`less`作为分页器，因此操作方式完全相同。

![](img/2efbf0aec4f82f3f952c3af74359a409_31.png)

### 命令组合与效率

![](img/2efbf0aec4f82f3f952c3af74359a409_33.png)

可以使用管道将`cat`和分页命令组合，例如：
```bash
cat /etc/passwd | less
```
这种方式灵活，便于后续添加其他处理命令（如`grep`）。但对于大文件，直接使用`less 文件名`效率更高，因为它避免了一次额外的`cat`进程和管道传输。

## 提取文件局部内容：head 与 tail

刚刚我们学习了如何查看文件的全部内容，但有时我们只关心文件的开头、结尾或特定部分。

### head命令：查看文件开头

`head`命令默认显示文件的前10行。

**基本语法**：
```bash
head [选项] 文件名
```

**常用选项**：
*   `-n NUM`：显示文件的前`NUM`行。例如`-n 5`显示前5行。
*   `-NUM`：显示文件除了最后`NUM`行以外的所有行。例如`-6`表示不显示最后6行。

**示例**：
1.  查看`/etc/passwd`的前5行：
    ```bash
    head -n 5 /etc/passwd
    ```
2.  查看`/etc/passwd`除了最后6行以外的所有内容：
    ```bash
    head -6 /etc/passwd
    ```
3.  结合`cat -n`查看带行号的前10行：
    ```bash
    cat -n /etc/passwd | head
    ```

### tail命令：查看文件末尾

![](img/2efbf0aec4f82f3f952c3af74359a409_35.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_37.png)

`tail`命令默认显示文件的最后10行。它最重要的功能是实时监控日志文件。

![](img/2efbf0aec4f82f3f952c3af74359a409_39.png)

**基本语法**：
```bash
tail [选项] 文件名
```

**常用选项**：
*   `-n NUM`：显示文件的最后`NUM`行。例如`-n 4`显示最后4行。
*   `+NUM`：从文件的第`NUM`行开始显示，直到文件末尾。例如`+4`表示从第4行开始显示。
*   `-f`：实时追踪显示文件新增的内容。这是监控日志的利器。

![](img/2efbf0aec4f82f3f952c3af74359a409_41.png)

**示例**：
1.  查看`/etc/passwd`的最后4行：
    ```bash
    tail -n 4 /etc/passwd
    ```
2.  从`/etc/passwd`的第4行开始显示：
    ```bash
    tail +4 /etc/passwd
    ```
3.  **实时监控系统主日志文件**：
    ```bash
    tail -f /var/log/messages
    ```
    执行此命令后，终端会持续显示`/var/log/messages`文件中新写入的日志，便于观察系统动态。按 `Ctrl+C` 终止监控。
4.  监控`/var/log`目录下所有日志文件的末尾（每个文件最后10行）：
    ```bash
    tail -f /var/log/*
    ```

### 组合使用head和tail

通过管道，可以组合`head`和`tail`来提取文件的中间部分。

![](img/2efbf0aec4f82f3f952c3af74359a409_43.png)

![](img/2efbf0aec4f82f3f952c3af74359a409_45.png)

**示例**：查看`/etc/passwd`文件的第10行到第20行。
思路：先取前20行，再从结果中取最后11行。
```bash
head -n 20 /etc/passwd | tail -n 11
```
**公式推导**：`tail`所需的行数 = `head`截取的行数(`20`) - 起始行(`10`) + 1 = `11`。

![](img/2efbf0aec4f82f3f952c3af74359a409_47.png)

## 总结

![](img/2efbf0aec4f82f3f952c3af74359a409_49.png)

本节课中我们一起学习了Linux中基础的文本提取工具。
*   **cat** 用于快速查看或合并文件。
*   **more** 和 **less** 用于分页浏览文件，其中`less`功能更全面，支持前后翻页、搜索和编辑。
*   **head** 和 **tail** 用于查看文件的开头和结尾部分，`tail -f`在监控实时日志时非常有用。

![](img/2efbf0aec4f82f3f952c3af74359a409_51.png)

这些工具是文本处理的基础，通过管道符`|`将它们灵活组合，可以应对各种查看和提取需求。请务必熟练掌握每个命令的常用选项和基本用法。
