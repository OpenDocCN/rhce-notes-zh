# Linux运维基础：P20：常用特殊符号与grep文件内容过滤 📚

在本节课中，我们将学习Linux系统中几个常用的特殊符号（通配符）以及`grep`命令。特殊符号能帮助我们高效地匹配文件名，而`grep`命令则是从文件内容中筛选信息的利器。掌握它们是提升Linux操作效率的关键一步。

![](img/b19457104e10f05e0e6a5df99e5dff3c_1.png)

## 常用特殊符号 🔤

在Linux系统中，键盘上的许多符号，如 `!`、`@`、`#`、`$`、`%`、`^`、`&`、`*`、`{}`、`()`、`<>`、`?`、`/`、`\`、反撇号 `` ` `` 以及点号 `.` 和管道符 `|` 等，都属于特殊符号，各自有特定的功能。本节我们重点学习其中四个：`*`、`{}`、`?` 和 `[]`。

### 星号 `*`

星号 `*` 是一个通配符，用于在文件名中代表**任意多个任意字符**。这里的“任意”指所有字符类型（数字、字母、符号等），“多个”指长度不限。

**核心功能**：匹配任意长度的任意字符串。

**常用场景**：
*   **搜索文件**：当你只记得文件名的一部分时，可以用 `*` 补全。
*   **批量操作**：对符合特定模式的一批文件进行操作。

以下是几个应用示例：

*   **搜索以特定字符串开头的文件**：
    ```bash
    ls /etc/pass*
    ```
    此命令会列出 `/etc` 目录下所有以 `pass` 开头的文件。

![](img/b19457104e10f05e0e6a5df99e5dff3c_9.png)

![](img/b19457104e10f05e0e6a5df99e5dff3c_11.png)

*   **搜索以特定扩展名结尾的文件**：
    ```bash
    ls /var/log/*.log
    ```
    此命令会列出 `/var/log` 目录下所有以 `.log` 结尾的日志文件。

*   **批量复制文件**：
    ```bash
    cp /var/log/*.log /opt/
    ```
    此命令会将 `/var/log` 目录下所有 `.log` 文件复制到 `/opt/` 目录。

![](img/b19457104e10f05e0e6a5df99e5dff3c_13.png)

`*` 非常灵活，可以放在文件名开头、中间或结尾进行匹配。

### 问号 `?`

![](img/b19457104e10f05e0e6a5df99e5dff3c_15.png)

问号 `?` 也是一个通配符，用于在文件名中代表**任意单个字符**。

![](img/b19457104e10f05e0e6a5df99e5dff3c_17.png)

**核心功能**：匹配恰好一个任意字符。

**常用场景**：当你明确知道文件名中某个位置有一个字符，但不记得具体是什么时使用。

![](img/b19457104e10f05e0e6a5df99e5dff3c_19.png)

![](img/b19457104e10f05e0e6a5df99e5dff3c_21.png)

![](img/b19457104e10f05e0e6a5df99e5dff3c_23.png)

以下是应用示例：

*   **搜索特定格式的文件名**：
    ```bash
    ls /dev/tty?
    ```
    此命令会列出 `/dev` 目录下所有形如 `tty` 后接**一个**字符的文件（如 `tty0`, `tty1`, `ttyA`）。

*   **匹配两个字符**：
    ```bash
    ls /dev/tty??
    ```
    此命令会匹配 `tty` 后接**两个**字符的文件名。

### 中括号 `[]`

![](img/b19457104e10f05e0e6a5df99e5dff3c_25.png)

![](img/b19457104e10f05e0e6a5df99e5dff3c_27.png)

中括号 `[]` 用于在文件名中匹配**括号内指定的一个字符**或**一个连续范围内的一个字符**。

**核心功能**：匹配一个来自指定集合或范围的字符。

**常用场景**：匹配数字或字母范围。

以下是应用示例：

*   **匹配单个数字**：
    ```bash
    ls /dev/tty[0-9]
    ```
    此命令会匹配 `/dev` 目录下 `tty0` 到 `tty9` 的文件。

*   **匹配单个字母**：
    ```bash
    ls /var/log/*[a-z]*
    ```
    此命令会列出 `/var/log` 目录下文件名中包含任意小写字母的文件（由于结合了 `*`，匹配可能不精确）。

**注意**：`[]` 在匹配不连续的数字范围（如 `[15-25]`）时可能无法按预期工作，此时更适合使用大括号 `{}`。

![](img/b19457104e10f05e0e6a5df99e5dff3c_29.png)

### 大括号 `{}`

![](img/b19457104e10f05e0e6a5df99e5dff3c_31.png)

大括号 `{}` 功能强大，常用于生成序列或匹配多组字符串，尤其在处理数字序列时非常高效。

![](img/b19457104e10f05e0e6a5df99e5dff3c_33.png)

**核心功能**：生成一个序列或枚举列表。

![](img/b19457104e10f05e0e6a5df99e5dff3c_35.png)

**常用场景**：批量创建、删除或匹配有规律的数字序列文件。

![](img/b19457104e10f05e0e6a5df99e5dff3c_37.png)

以下是应用示例：

![](img/b19457104e10f05e0e6a5df99e5dff3c_39.png)

![](img/b19457104e10f05e0e6a5df99e5dff3c_41.png)

*   **批量创建文件**：
    ```bash
    touch test{1..100}.txt
    ```
    此命令会快速创建 `test1.txt` 到 `test100.txt` 共100个文件。

*   **批量删除文件**：
    ```bash
    rm -f test{1..50}.txt
    ```
    此命令会删除 `test1.txt` 到 `test50.txt` 这50个文件。

*   **匹配枚举项**：
    ```bash
    ls /etc/*.{conf,cfg}
    ```
    此命令会列出 `/etc` 目录下以 `.conf` 或 `.cfg` 结尾的文件。

**小结**：在初级阶段，最需要熟练掌握的是 `*` 和 `{}`，它们的使用频率最高。`?` 和 `[]` 了解其功能即可。

## grep文件内容过滤 🔍

`grep` 命令用于在文件中搜索并显示包含指定字符串的行。当文件内容很多时，手动查找效率低下，`grep` 可以快速定位目标信息。

**核心语法**：
```bash
grep [选项] “搜索字符串” 文件名
```

### 基本用法

*   **在文件中搜索特定字符串**：
    ```bash
    grep “tom” /etc/passwd
    ```
    此命令会显示 `/etc/passwd` 文件中所有包含 “tom” 的行。

*   **显示行号**（`-n` 选项）：
    ```bash
    grep -n “root” /etc/passwd
    ```
    此命令会显示包含 “root” 的行及其在文件中的行号。

### 常用选项

*   **忽略大小写**（`-i` 选项）：
    ```bash
    grep -i “user” /etc/passwd
    ```
    此命令会匹配 “user”、”User”、”USER” 等，不区分大小写。

*   **反向选择**（`-v` 选项）：显示**不包含**匹配字符串的行。
    ```bash
    grep -v “nologin” /etc/passwd
    ```
    此命令会显示 `/etc/passwd` 中所有不使用 `/sbin/nologin` 解释器的用户行（即可登录系统的用户）。

### 结合正则表达式

`grep` 可以结合更强大的正则表达式符号进行精确过滤。

*   **匹配行首**（`^` 符号）：
    ```bash
    grep “^root” /etc/passwd
    ```
    此命令只匹配以 “root” **开头**的行。

![](img/b19457104e10f05e0e6a5df99e5dff3c_43.png)

*   **匹配行尾**（`$` 符号）：
    ```bash
    grep “bash$” /etc/passwd
    ```
    此命令只匹配以 “bash” **结尾**的行。

![](img/b19457104e10f05e0e6a5df99e5dff3c_45.png)

*   **匹配空行**（`^$`）：
    ```bash
    grep “^$” /etc/login.defs
    ```
    此命令会匹配文件中的所有空行。

### 高级应用：组合过滤

`grep` 常与管道符 `|` 和其他命令结合，实现复杂过滤。

![](img/b19457104e10f05e0e6a5df99e5dff3c_47.png)

*   **过滤命令输出**：
    ```bash
    ifconfig | grep “inet”
    ```
    此命令只显示 `ifconfig` 输出中包含 “inet” 的行（通常是IP地址信息）。

![](img/b19457104e10f05e0e6a5df99e5dff3c_49.png)

*   **组合排除**：先排除注释行（以`#`开头），再排除空行。
    ```bash
    grep -v “^#” /etc/login.defs | grep -v “^$”
    ```
    此命令能清晰显示配置文件中的有效配置项。

![](img/b19457104e10f05e0e6a5df99e5dff3c_51.png)

*   **统计匹配行数**：
    ```bash
    grep -c “nologin” /etc/passwd
    ```
    或结合 `wc`：
    ```bash
    grep “nologin” /etc/passwd | wc -l
    ```
    这两个命令都可以统计使用 `/sbin/nologin` 解释器的用户数量。

![](img/b19457104e10f05e0e6a5df99e5dff3c_53.png)

本节课中我们一起学习了Linux中四个常用的特殊符号（`*`, `?`, `[]`, `{}`）以及强大的 `grep` 命令。
*   **特殊符号**主要用于**文件名**的匹配和批量操作，能极大提升文件管理效率。
*   **`grep` 命令**则专注于**文件内容**的搜索与过滤，是查看日志、分析配置文件的必备工具。
