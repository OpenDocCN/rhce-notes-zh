## 2.3 系统状态监控：`top`命令详解 📊

![](img/11d9492249f8d4d42978433a2fda118f_41.png)

![](img/11d9492249f8d4d42978433a2fda118f_43.png)

![](img/11d9492249f8d4d42978433a2fda118f_45.png)

上一节我们介绍了`ps`命令，它可以查看系统进程信息。本节中，我们来看看一个更加强大的动态进程查看工具——`top`命令。

![](img/11d9492249f8d4d42978433a2fda118f_51.png)

![](img/11d9492249f8d4d42978433a2fda118f_53.png)

![](img/11d9492249f8d4d42978433a2fda118f_55.png)

`top`命令可以看作是一个加强版的Windows任务管理器。它不仅能够实时显示系统中最重要的进程信息，还能动态更新，让我们直观地了解CPU、内存等资源的使用情况。

![](img/11d9492249f8d4d42978433a2fda118f_57.png)

![](img/11d9492249f8d4d42978433a2fda118f_59.png)

![](img/11d9492249f8d4d42978433a2fda118f_61.png)

![](img/11d9492249f8d4d42978433a2fda118f_63.png)

![](img/11d9492249f8d4d42978433a2fda118f_65.png)

以下是`top`命令输出信息的详细解读：

![](img/11d9492249f8d4d42978433a2fda118f_67.png)

![](img/11d9492249f8d4d42978433a2fda118f_69.png)

![](img/11d9492249f8d4d42978433a2fda118f_71.png)

![](img/11d9492249f8d4d42978433a2fda118f_73.png)

**第一行：系统概况**
*   `top`：命令本身。
*   `up 34 min`：系统已运行时间。
*   `1 user`：当前登录用户数（注意：在RHEL 5/6中，此数字代表终端数量）。
*   `load average`：系统负载，分别对应最近1分钟、5分钟、15分钟的平均负载。数值越低越好，长期超过5可能意味着系统不稳定。

![](img/11d9492249f8d4d42978433a2fda118f_77.png)

![](img/11d9492249f8d4d42978433a2fda118f_85.png)

![](img/11d9492249f8d4d42978433a2fda118f_87.png)

![](img/11d9492249f8d4d42978433a2fda118f_89.png)

![](img/11d9492249f8d4d42978433a2fda118f_91.png)

**第二行：进程概况**
*   `Tasks`：进程总数及其状态分布。常见的进程状态有：
    *   `R`：运行状态，正在消耗资源提供服务。
    *   `S`：休眠状态，等待被调用。
    *   `D`：不可中断的休眠状态。
    *   `Z`：僵尸进程，消耗资源但不提供服务。
    *   `T`：停止状态。

![](img/11d9492249f8d4d42978433a2fda118f_93.png)

![](img/11d9492249f8d4d42978433a2fda118f_95.png)

![](img/11d9492249f8d4d42978433a2fda118f_97.png)

![](img/11d9492249f8d4d42978433a2fda118f_99.png)

![](img/11d9492249f8d4d42978433a2fda118f_101.png)

![](img/11d9492249f8d4d42978433a2fda118f_103.png)

**第三行：CPU使用情况**
*   我们主要关注`us`（用户进程占用百分比）和`id`（空闲百分比）。`us`值高表示CPU繁忙。

![](img/11d9492249f8d4d42978433a2fda118f_105.png)

![](img/11d9492249f8d4d42978433a2fda118f_107.png)

![](img/11d9492249f8d4d42978433a2fda118f_109.png)

![](img/11d9492249f8d4d42978433a2fda118f_111.png)

**第四、五行：内存使用情况**
*   `Mem`：物理内存使用情况。
*   `Swap`：交换分区使用情况。交换分区是硬盘上划出的一块空间，用于在物理内存不足时临时存放数据。如果交换分区被长期、大量使用，通常意味着物理内存不足。

![](img/11d9492249f8d4d42978433a2fda118f_113.png)

![](img/11d9492249f8d4d42978433a2fda118f_115.png)

![](img/11d9492249f8d4d42978433a2fda118f_117.png)

**进程列表字段**
*   `PID`：进程的唯一标识号，类似身份证。
*   `USER`：进程发起者。
*   `PR` / `NI`：进程优先级。**数字越低，优先级越高**。范围是-20（最高）到19（最低）。
*   `%CPU` / `%MEM`：进程瞬时的CPU和内存占用百分比。
*   `TIME+`：进程累计占用CPU时间。
*   `COMMAND`：启动该进程的命令名称。

![](img/11d9492249f8d4d42978433a2fda118f_119.png)

![](img/11d9492249f8d4d42978433a2fda118f_121.png)

![](img/11d9492249f8d4d42978433a2fda118f_123.png)

**`top`命令的交互操作**
在`top`运行界面，可以执行一些快捷操作：
*   按 `k` 后输入PID，可以结束指定进程。
*   按 `P` 按CPU使用率排序。
*   按 `M` 按内存使用率排序。
*   按 `q` 退出`top`。

![](img/11d9492249f8d4d42978433a2fda118f_127.png)

**`ifconfig`：查看网卡与网络状态**
此命令用于查看网络接口配置和状态。
```bash
ifconfig
```
输出信息包括：
*   网卡名称（如 `ens160`，不同系统版本命名规则不同）。
*   `inet`：网卡的IP地址。
*   `netmask`：子网掩码。
*   `ether`：网卡的MAC地址（物理地址）。
*   `RX` / `TX`：自上次重启后接收/发送的数据包统计。

此命令用于查看系统内核版本、主机名和硬件架构。
```bash
uname -a
```
输出示例：`Linux linuxprobe 4.18.0-xxx.x86_64 ...`，其中`4.18.0`即为内核版本。

此命令可以快速查看系统运行时间、登录用户数和平均负载，是`top`命令第一行信息的精简版。
```bash

    ping -c 4 www.linuxprobe.com
    ```
    参数 `-c 4` 表示发送4个数据包后停止。
*   `traceroute`：追踪数据包到达目标主机所经过的路由路径，用于诊断网络故障点。
    ```bash
    traceroute www.linuxprobe.com
    ```

**`netstat`：查看网络连接状态**
此命令用于显示网络连接、路由表、接口统计等信息。
```bash
netstat -an
```
常用参数组合 `-an` 可以显示所有连接和监听端口，并以数字形式显示地址。

**`history`：查看历史命令**
此命令显示当前用户执行过的命令历史记录。
```bash
history
```
*   使用 `!编号` 可以快速重复执行历史记录中的某条命令，例如 `!53`。
*   使用 `history -c` 可以清空当前会话的历史记录。
*   注意：命令历史记录保存在用户家目录的 `.bash_history` 文件中，通常在下一次登录或注销时才会同步写入。

**`sosreport`：收集系统诊断信息**
当系统出现故障需要他人协助排查时，可以使用此命令收集系统配置和日志信息，生成一个诊断报告。
```bash
sosreport
```

```bash
cd /etc # 切换到 /etc 目录
cd ~    # 切换到当前用户的家目录
cd ..   # 切换到上一级目录
cd -    # 切换到上一次所在的目录
```

```bash
ls -l   # 以长格式显示，包含文件权限、所有者、大小、修改时间等详细信息
```
在默认配色中，白色通常代表普通文件（如文本、配置文件），蓝色代表目录。

速度非常快，因为它搜索的是一个预建的文件名数据库。使用前可能需要先更新数据库：`updatedb`。
```bash
locate passwd
```

**`whereis`：查找命令及其相关文件**
专门用于查找二进制程序（命令本身）、源代码和帮助手册页的位置。
```bash
whereis ls
```

![](img/11d9492249f8d4d42978433a2fda118f_331.png)

**`which`：查找命令的路径**
在环境变量 `$PATH` 设置的目录中查找某个命令的绝对路径。
```bash
which ls
```

## 总结 🎯

![](img/11d9492249f8d4d42978433a2fda118f_337.png)

本节课中我们一起学习了：
1.  使用 `top` 命令动态监控系统进程和资源（CPU、内存、负载）的使用情况。
2.  使用 `ifconfig`、`uname`、`uptime`、`free`、`who`、`last`、`ping`、`traceroute`、`netstat` 等一系列命令来查看系统的基本状态和网络信息。
3.  使用 `history` 管理命令历史，使用 `sosreport` 收集系统诊断报告。
4.  掌握了在文件系统中导航的基本命令：`pwd`（查看当前位置）、`cd`（切换目录）、`ls`（列出内容）。
5.  学习了四种查找文件的命令：`find`（全盘精确查找）、`locate`（快速数据库查找）、`whereis`（查找命令及相关文件）、`which`（查找命令路径）。
