### 扩容前提：检查卷组空间

![](img/4be8896352b607db1f11574f4b1819a4_15.png)

![](img/4be8896352b607db1f11574f4b1819a4_17.png)

在对逻辑卷扩容前，必须确保其所属的卷组有足够的剩余空间。

以下是检查卷组和逻辑卷状态的命令：
```bash
vgs
lvs
```
`vgs`命令查看卷组剩余空间，`lvs`命令查看逻辑卷当前大小。

### 扩展逻辑卷空间

确认卷组有空间后，即可使用`lvextend`命令扩展逻辑卷。

扩展逻辑卷的命令格式如下：
```bash
lvextend -L +[大小] /dev/[卷组名]/[逻辑卷名]
```
例如，为逻辑卷`myLV`增加19GB空间：
```bash
lvextend -L +19G /dev/system_vg/myLV
```
命令执行后，逻辑卷的**物理空间**会立即增大。

![](img/4be8896352b607db1f11574f4b1819a4_19.png)

### 扩展文件系统

逻辑卷空间扩展后，其内部的**文件系统**并不会自动增大。这会导致使用`df -h`命令查看时，已用空间显示未变。

![](img/4be8896352b607db1f11574f4b1819a4_21.png)

这是因为新增的空间没有文件系统。需要扩展文件系统以管理新增空间。

![](img/4be8896352b607db1f11574f4b1819a4_23.png)

以下是扩展不同文件系统的命令：
*   对于XFS文件系统：`xfs_growfs [挂载点]`
*   对于EXT2/3/4文件系统：`resize2fs [设备路径]`

![](img/4be8896352b607db1f11574f4b1819a4_25.png)

例如，我们的逻辑卷挂载在`/web_back`，使用XFS文件系统，则执行：
```bash
xfs_growfs /web_back
```
执行此命令后，文件系统容量才会与逻辑卷物理容量同步，使用`df -h`即可看到空间已真正变大。

![](img/4be8896352b607db1f11574f4b1819a4_27.png)

### 扩容卷组

![](img/4be8896352b607db1f11574f4b1819a4_29.png)

如果卷组空间不足，需要先对卷组进行扩容。可以将新的物理磁盘或分区加入卷组。

扩展卷组的命令格式如下：
```bash
vgextend [卷组名] [设备路径1] [设备路径2] ...
```
例如，将新硬盘`/dev/sdd`加入到`system_vg`卷组：
```bash
vgextend system_vg /dev/sdd
```
卷组扩容后，便可继续按上述步骤扩展逻辑卷及其文件系统。

### 逻辑卷缩容与删除 ⚠️

![](img/4be8896352b607db1f11574f4b1819a4_39.png)

逻辑卷支持缩容，但操作风险极高，可能导致数据丢失，企业环境中极少使用。

若需删除逻辑卷，必须遵循以下顺序：
1.  卸载逻辑卷：`umount [挂载点]`
2.  删除逻辑卷：`lvremove /dev/[卷组名]/[逻辑卷名]`
3.  删除卷组（可选）：`vgremove [卷组名]`
4.  删除物理卷属性（可选）：`pvremove [设备路径]`

**注意**：删除操作不可逆，会清除该逻辑卷上的所有数据。

## RAID磁盘阵列简介 🛡️

![](img/4be8896352b607db1f11574f4b1819a4_43.png)

![](img/4be8896352b607db1f11574f4b1819a4_45.png)

接下来，我们离开逻辑卷，了解一下RAID磁盘阵列技术。RAID的核心目标是提升数据可靠性和/或存储性能。

![](img/4be8896352b607db1f11574f4b1819a4_47.png)

### 什么是RAID？

RAID（Redundant Array of Independent Disks），即独立磁盘冗余阵列。其原理是通过硬件（RAID卡）或软件，将多块独立的物理磁盘组合成一个逻辑上的大容量磁盘来使用。

主要优点包括：
*   **提高数据可靠性**：通过数据冗余，允许部分磁盘损坏时不丢失数据。
*   **提升读写性能**：通过数据条带化分布，实现并行读写。

### 常见RAID级别

![](img/4be8896352b607db1f11574f4b1819a4_49.png)

![](img/4be8896352b607db1f11574f4b1819a4_51.png)

以下是几种常见的RAID级别简介：

![](img/4be8896352b607db1f11574f4b1819a4_53.png)

![](img/4be8896352b607db1f11574f4b1819a4_55.png)

![](img/4be8896352b607db1f11574f4b1819a4_57.png)

*   **RAID 0（条带化）**：
    *   **原理**：将数据分块后，交替写入多个磁盘。
    *   **优点**：读写速度最快。
    *   **缺点**：无冗余，任何一块磁盘损坏将导致所有数据丢失。
    *   **最少磁盘数**：2块。

![](img/4be8896352b607db1f11574f4b1819a4_58.png)

![](img/4be8896352b607db1f11574f4b1819a4_60.png)

*   **RAID 1（镜像）**：
    *   **原理**：将相同的数据同时写入两块磁盘，实现完全备份。
    *   **优点**：数据安全性高，读取速度较快。
    *   **缺点**：磁盘利用率只有50%，写入速度无提升。
    *   **最少磁盘数**：2块。

![](img/4be8896352b607db1f11574f4b1819a4_62.png)

![](img/4be8896352b607db1f11574f4b1819a4_64.png)

![](img/4be8896352b607db1f11574f4b1819a4_66.png)

*   **RAID 5（带奇偶校验的条带化）**：
    *   **原理**：数据与校验信息交替存储在多个磁盘上。
    *   **优点**：兼顾性能、容量和安全性。允许损坏一块磁盘而不丢失数据。
    *   **缺点**：写入速度较慢（需计算校验位）。
    *   **最少磁盘数**：3块。

*   **RAID 10（先镜像再条带化）**：
    *   **原理**：先两两组成RAID 1，再将多个RAID 1组组成RAID 0。
    *   **优点**：兼具RAID 1的安全性和RAID 0的高性能。
    *   **缺点**：成本高，磁盘利用率只有50%。
    *   **最少磁盘数**：4块。

### RAID与LVM的结合应用

在实际生产环境中，RAID常与LVM结合使用，以构建既安全又灵活的存储架构：
1.  使用多块硬盘创建RAID阵列（如RAID 5），得到一个可靠的大容量“物理盘”。
2.  将此RAID阵列作为物理卷（PV）加入卷组（VG）。
3.  在卷组上创建可动态扩展的逻辑卷（LV）供系统使用。

这种架构既通过RAID保证了底层数据的冗余安全，又通过LVM实现了上层存储空间的灵活管理。
