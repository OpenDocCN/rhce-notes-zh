# Linux用户管理：027：用户管理进阶（echo、SHELL、usermod）

在本节课中，我们将继续深入学习Linux用户管理。我们将探讨如何查看和修改用户的默认Shell，如何使用`usermod`命令修改用户属性，以及如何通过命令行高效地设置用户密码。这些技能对于系统管理员管理用户账户至关重要。

## 查看与修改默认Shell

上一节我们介绍了用户的基本创建，本节中我们来看看如何管理用户的登录环境。每个用户都有一个默认的Shell，它决定了用户登录后使用的命令行解释器。

使用`echo $SHELL`命令可以查看当前用户的默认Shell。系统默认通常使用`/bin/bash`。

在创建用户时，如果需要为其指定一个特定的Shell，可以使用`useradd`命令的`-s`选项。例如，指定`/sbin/nologin`作为Shell是一种特殊做法。

**核心概念**：`/sbin/nologin`与`/bin/bash`等标准Shell有本质区别。一旦为用户指定了`/sbin/nologin`，该用户将被锁定，无法登录系统。

例如，为用户`user1`指定`/sbin/nologin`后，尝试切换（`su`）到该用户会提示“该账户不可用”。

## 使用usermod修改用户属性

创建用户后，如果需要修改其信息，可以使用`usermod`命令。

### 修改用户附加组

`usermod`命令的`-G`选项可以更改用户的附加组（扩展组）。

例如，用户`ddd`原本的附加组是`pro3`。执行以下命令可以将其附加组改为`pro2`：
```bash
usermod -G pro2 ddd
```
执行`id ddd`命令后，可以看到其附加组已更新。

### 锁定与解锁用户

系统管理员可能需要临时禁止某个用户登录。以下是相关操作步骤：

*   **锁定用户**：使用`usermod -L`命令可以锁定用户。例如，锁定用户`aa`：
    ```bash
    usermod -L aa
    ```
    锁定后，该用户将无法登录系统。Root用户仍可切换到该用户环境，但普通用户`aa`本身无法登录。

*   **解锁用户**：使用`usermod -U`命令可以解锁用户。例如，解锁用户`aa`：
    ```bash
    usermod -U aa
    ```

### 修改用户Shell与UID

`usermod`命令还可以修改用户的Shell和用户ID（UID）。

*   **修改Shell**：使用`-s`选项。例如，将之前被锁定的用户`user1`的Shell改回`/bin/bash`，使其可以正常登录：
    ```bash
    usermod -s /bin/bash user1
    ```

*   **修改UID**：使用`-u`选项。例如，将用户`eee`的UID从113改为114：
    ```bash
    usermod -u 114 eee
    ```
    执行`id eee`命令可以验证UID已更改。

## 设置与修改用户密码

之前创建的用户都没有设置密码。`passwd`命令用于修改用户密码。

### 交互式修改密码

直接运行`passwd [用户名]`会进入交互模式，提示输入新密码。例如，为`user1`设置密码：
```bash
passwd user1
```
系统可能会提示密码过于简单。

### 通过标准输入设置密码

为了自动化或批量设置复杂密码，可以通过管道将密码传递给`passwd`命令。

**核心概念**：使用`echo`命令生成密码字符串，然后通过管道`|`传递给`passwd --stdin`命令。

例如，为用户`user1`设置密码为`ZhangYang123456`：
```bash
echo “ZhangYang123456” | passwd --stdin user1
```
此方法会提示密码更新成功，允许通过标准输入流非交互式地修改密码。
