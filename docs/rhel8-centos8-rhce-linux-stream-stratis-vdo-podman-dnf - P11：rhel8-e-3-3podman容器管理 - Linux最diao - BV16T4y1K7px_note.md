# Podman容器管理：P11：Podman容器管理基础 🐳

在本节课中，我们将学习如何使用Podman运行和管理容器。我们将从运行一个简单的Web服务器容器开始，逐步介绍如何查看、进入、停止和删除容器，以及相关的常用命令。

## 概述

上一节我们介绍了Podman的基本概念和运行交互式容器。本节中我们来看看如何运行一个后台服务容器，并学习管理容器生命周期的完整流程。

![](img/a01d51df499345482a8d3dc0aae239c4_1.png)

## 运行后台容器

我们可以使用`podman run`命令运行一个后台容器。如果本地没有指定的镜像，Podman会自动从注册表下载。

以下是运行一个Apache HTTPD后台容器的命令：

```bash
podman run -d -p 8080:80 --name myweb01 httpd
```

*   **`-d`**：代表后台运行。
*   **`-p 8080:80`**：进行端口转发。将宿主机的8080端口映射到容器的80端口。
*   **`--name myweb01`**：为容器指定一个唯一的名称。
*   **`httpd`**：要运行的镜像名称。

执行后，Podman会下载`httpd`镜像并运行容器。我们可以通过宿主机的8080端口访问容器内的Web服务。

![](img/a01d51df499345482a8d3dc0aae239c4_3.png)

## 查看容器状态

容器运行后，我们需要查看其状态。

![](img/a01d51df499345482a8d3dc0aae239c4_5.png)

以下是查看运行中容器的命令：

![](img/a01d51df499345482a8d3dc0aae239c4_7.png)

```bash
podman ps
```

![](img/a01d51df499345482a8d3dc0aae239c4_9.png)

该命令会显示容器的ID、名称、使用的镜像、创建时间、状态和端口映射等信息。

若要查看所有容器（包括已停止的），可以使用：

```bash
podman ps -a
```

## 进入运行中的容器

![](img/a01d51df499345482a8d3dc0aae239c4_11.png)

![](img/a01d51df499345482a8d3dc0aae239c4_13.png)

对于后台运行的容器，我们可以进入其内部执行命令。

以下是进入一个运行中容器的命令：

```bash
podman exec -it myweb01 /bin/bash
```

*   **`exec`**：在运行中的容器内执行命令。
*   **`-it`**：分配一个交互式终端。
*   **`myweb01`**：目标容器的名称。
*   **`/bin/bash`**：要执行的命令，这里是启动一个Bash shell。

![](img/a01d51df499345482a8d3dc0aae239c4_15.png)

进入容器后，可以像在普通系统中一样执行命令。使用`exit`命令可以退出容器。

## 管理容器生命周期

![](img/a01d51df499345482a8d3dc0aae239c4_17.png)

我们需要掌握容器的启动、停止和重启操作。

以下是管理容器状态的常用命令：

*   **停止容器**：`podman stop myweb01`
*   **启动已停止的容器**：`podman start myweb01`
*   **重启容器**：`podman restart myweb01`
*   **强制停止（杀死）容器**：`podman kill myweb01`

![](img/a01d51df499345482a8d3dc0aae239c4_19.png)

## 删除容器

![](img/a01d51df499345482a8d3dc0aae239c4_21.png)

当容器不再需要时，可以将其删除。删除容器不会影响其使用的镜像。

![](img/a01d51df499345482a8d3dc0aae239c4_23.png)

![](img/a01d51df499345482a8d3dc0aae239c4_25.png)

以下是删除容器的命令：

```bash
podman rm myweb01
```

此命令只能删除已停止的容器。

若要强制删除一个正在运行的容器，可以添加`-f`参数：

```bash
podman rm -f myweb01
```

若要删除所有已停止的容器，可以使用`-a`参数：

![](img/a01d51df499345482a8d3dc0aae239c4_27.png)

```bash
podman rm -a
```

![](img/a01d51df499345482a8d3dc0aae239c4_29.png)

## 容器与镜像的关系

![](img/a01d51df499345482a8d3dc0aae239c4_31.png)

需要明确的是，容器是基于镜像运行的实例。删除容器并不会删除镜像。

若要删除镜像，需要使用`podman rmi`命令：

![](img/a01d51df499345482a8d3dc0aae239c4_33.png)

```bash
podman rmi httpd
```

![](img/a01d51df499345482a8d3dc0aae239c4_35.png)

**注意**：不能删除正在被容器使用的镜像。如果强制删除，需要先使用`-f`参数删除相关容器。

## 总结

![](img/a01d51df499345482a8d3dc0aae239c4_37.png)

本节课中我们一起学习了Podman容器管理的基础操作。我们实践了如何运行一个后台Web服务容器，学会了使用`podman ps`查看容器状态，使用`podman exec`进入容器，以及使用`stop`、`start`、`restart`、`rm`等命令管理容器的整个生命周期。理解容器与镜像的独立关系是进行有效管理的关键。
