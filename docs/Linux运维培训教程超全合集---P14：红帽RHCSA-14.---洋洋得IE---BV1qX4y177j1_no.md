## 📊 wc统计命令

![](img/463e0225f30aca6ce0d76f91da580270_9.png)

![](img/463e0225f30aca6ce0d76f91da580270_11.png)

`wc`命令用于统计文件中的行数、单词数和字节数。它最常用的功能是统计文件的行数。

![](img/463e0225f30aca6ce0d76f91da580270_13.png)

命令的基本格式是：
```bash
wc [选项] 文件名
```

以下是`wc`命令常用的选项：
*   `-l`：仅统计行数。
*   `-w`：仅统计单词数（以空格分隔的字符串）。
*   `-c`：仅统计字节数。

例如，要统计`/etc/passwd`文件的总行数，可以执行：
```bash
wc -l /etc/passwd
```
这条命令会输出文件的行数，这是我们最常使用的功能。

## 🔗 管道符

![](img/463e0225f30aca6ce0d76f91da580270_15.png)

管道符`|`是一个非常强大的工具，它可以将前一个命令的输出结果，作为后一个命令的输入参数进行处理。这使得我们可以将多个简单的命令组合起来，实现复杂的功能。

![](img/463e0225f30aca6ce0d76f91da580270_17.png)

管道符的格式是：
```bash
命令A | 命令B
```

例如，我们想查看`/etc/passwd`文件的前10行中的最后5行。这个需求很难用单个命令完成，但使用管道符就很简单：
```bash
head -10 /etc/passwd | tail -5
```
这里，`head -10 /etc/passwd`先输出文件的前10行，然后通过管道`|`将这个结果传递给`tail -5`命令，`tail -5`则输出接收到的内容的最后5行。

管道符可以无限连接。例如，我们还可以继续统计这5行的数量：
```bash
head -10 /etc/passwd | tail -5 | wc -l
```
这个组合命令最终会输出数字`5`。

## 📥 重定向操作

![](img/463e0225f30aca6ce0d76f91da580270_23.png)

重定向操作可以将命令的输出结果保存到文件中，而不是显示在屏幕上。它主要通过大于号`>`和`>>`来实现。

![](img/463e0225f30aca6ce0d76f91da580270_25.png)

### 输出重定向

![](img/463e0225f30aca6ce0d76f91da580270_27.png)

![](img/463e0225f30aca6ce0d76f91da580270_29.png)

![](img/463e0225f30aca6ce0d76f91da580270_31.png)

一个大于号`>`代表覆盖重定向。它会将命令的输出结果写入指定文件，如果文件已存在，则会覆盖原有内容。
```bash
命令 > 文件名
```
例如，将`ifconfig`命令查看网卡信息的前两行保存到文件`net_ens32`中：
```bash
ifconfig ens32 | head -2 > net_ens32
```
执行后，屏幕没有输出，但`net_ens32`文件中已经保存了结果。如果文件不存在，重定向操作会自动创建它。

![](img/463e0225f30aca6ce0d76f91da580270_33.png)

两个大于号`>>`代表追加重定向。它也会将命令的输出结果写入文件，但如果文件已存在，新内容会追加在原有内容的末尾，而不是覆盖。
```bash
命令 >> 文件名
```

![](img/463e0225f30aca6ce0d76f91da580270_35.png)

![](img/463e0225f30aca6ce0d76f91da580270_37.png)

### 错误输出重定向

![](img/463e0225f30aca6ce0d76f91da580270_39.png)

默认情况下，`>`和`>>`只收集命令执行成功的输出（标准输出）。命令执行失败的错误信息（标准错误输出）不会被收集。

![](img/463e0225f30aca6ce0d76f91da580270_41.png)

`2>`用于覆盖重定向错误输出。
`2>>`用于追加重定向错误输出。

![](img/463e0225f30aca6ce0d76f91da580270_43.png)

例如，查看一个不存在的文件，并将错误信息保存：
```bash
ls /etc/abc.txt 2> error.log
```

![](img/463e0225f30aca6ce0d76f91da580270_45.png)

![](img/463e0225f30aca6ce0d76f91da580270_47.png)

### 合并正确与错误输出

![](img/463e0225f30aca6ce0d76f91da580270_49.png)

`&>`可以将命令的正确输出和错误输出都进行覆盖重定向。
`&>>`可以将命令的正确输出和错误输出都进行追加重定向。

![](img/463e0225f30aca6ce0d76f91da580270_51.png)

例如，无论命令成功与否，都将所有输出保存到文件：
```bash
ls /etc/passwd /etc/abc.txt &>> all_output.log
```

![](img/463e0225f30aca6ce0d76f91da580270_53.png)

![](img/463e0225f30aca6ce0d76f91da580270_55.png)

## 💬 echo命令

`echo`命令用于在终端输出指定的字符串或变量值。

命令格式很简单：
```bash
echo 字符串
```
例如，`echo Hello`会在屏幕上输出`Hello`。

![](img/463e0225f30aca6ce0d76f91da580270_57.png)

虽然单独使用`echo`看起来作用不大，但结合重定向，它可以方便地向文件写入内容或清空文件。例如，快速修改主机名文件：
```bash
echo “new_hostname” > /etc/hostname
```
这条命令会将“new_hostname”写入`/etc/hostname`文件，覆盖原有内容。如果`echo`后面不跟任何内容，则会产生一个空输出，常用于清空文件：`echo > file.txt`。

![](img/463e0225f30aca6ce0d76f91da580270_59.png)

## ⏳ sleep命令

![](img/463e0225f30aca6ce0d76f91da580270_61.png)

![](img/463e0225f30aca6ce0d76f91da580270_63.png)

`sleep`命令用于让当前进程（或脚本）暂停（休眠）一段时间。

![](img/463e0225f30aca6ce0d76f91da580270_65.png)

![](img/463e0225f30aca6ce0d76f91da580270_67.png)

命令格式为：
```bash
sleep 时间
```
时间单位可以是：
*   `s`：秒（默认）
*   `m`：分钟
*   `h`：小时
*   `d`：天

![](img/463e0225f30aca6ce0d76f91da580270_69.png)

![](img/463e0225f30aca6ce0d76f91da580270_71.png)

![](img/463e0225f30aca6ce0d76f91da580270_73.png)

例如，`sleep 5`会让终端暂停5秒后才显示下一个提示符。

`sleep`命令在编写Shell脚本时非常有用，可以控制命令的执行间隔，避免CPU过载。例如，在一个循环中快速输出信息会导致CPU使用率飙升，加入`sleep 0.1`让每次循环暂停0.1秒，可以显著降低CPU负载。
```bash
while true; do echo “Hello”; sleep 0.1; done
```
