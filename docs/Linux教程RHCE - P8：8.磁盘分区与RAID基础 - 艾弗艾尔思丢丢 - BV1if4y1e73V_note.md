# Linux教程RHCE：P8：磁盘分区与RAID基础

## 概述

在本节课中，我们将学习Linux系统中关于磁盘管理的核心知识。主要内容包括文件系统的基本概念、磁盘分区的创建与管理、格式化操作、挂载与卸载设备，以及磁盘配额和软硬链接技术。最后，我们还将初步了解RAID技术的基础知识。这些内容是管理Linux服务器存储资源的基础。

## 文件系统基础

上一节我们介绍了课程概述，本节中我们来看看文件系统的基础知识。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_1.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_2.png)

文件系统是操作系统用于明确存储设备（如硬盘）上文件的方法和数据结构。在Windows系统中，常见的文件系统有FAT32、NTFS等。在Linux系统中，文件系统格式经历了多次演变。

*   在Red Hat Enterprise Linux 5及更早版本中，主要使用EXT3文件系统。
*   到了RHEL 6，默认文件系统升级为EXT4。
*   在当前的RHEL 7中，默认文件系统是XFS。

从EXT3到EXT4的主要改进是提升了读写速度（I/O性能）。而RHEL 7选择XFS的主要原因在于其支持**超大容量**的单个分区，最大可达**18EB**（Exabytes）。这个决定是为了适应2014年前后兴起的“大数据”和“云计算”趋势，确保系统在未来几年内能满足海量数据存储的需求。

每个文件在存储时，实际上由两部分组成：
1.  **Inode**：存储文件的**元数据**，如文件名、大小、所有者、权限、时间戳等。
2.  **Data Block**：存储文件的**真实内容**。

Inode会通过指针指向存储其内容的Data Block。需要注意的是，即使一个文件的实际大小很小（例如1.5KB），它也会占用一个完整的Data Block（例如4KB），这会造成一定的空间浪费。

## 虚拟文件系统（VFS）

上一节我们了解了文件系统的构成，本节中我们来看看系统是如何统一管理不同文件系统的。

Linux系统支持多种文件系统（如EXT4， XFS）。为了让用户和应用程序无需关心底层文件系统的具体差异，Linux内核提供了一个抽象层——**虚拟文件系统**。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_4.png)

VFS的作用是**屏蔽底层不同文件系统的差异**，为用户提供统一的文件操作接口。无论文件存储在EXT4分区还是XFS分区，用户都可以使用相同的命令（如`cat`， `cp`）进行访问和操作。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_6.png)

## 使用硬盘的三个步骤

上一节我们介绍了VFS的作用，本节中我们来看看将一块新硬盘投入使用需要经历哪些标准步骤。

通常，使用一块新硬盘需要经过三个步骤：**分区**、**格式化**和**挂载**。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_8.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_9.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_10.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_12.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_14.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_15.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_16.png)

1.  **分区**：将一块大容量的物理硬盘逻辑上划分为多个独立的区域。这类似于将一大张纸裁剪成合适尺寸的小张。在Linux中，分区类型包括主分区、扩展分区和逻辑分区。
2.  **格式化**：在分区上创建特定的文件系统结构（如EXT4或XFS）。这类似于在纸上画好格子，以便规整地写入数据。格式化操作会建立Inode表、Data Block等结构。
3.  **挂载**：将格式化好的分区关联到Linux目录树中的一个空目录上。此后，用户通过访问这个目录（称为**挂载点**）即可使用该分区中的存储空间。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_18.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_19.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_20.png)

**挂载的本质**是将设备文件与某个目录相关联的动作。关联后，该目录就成为访问设备内容的入口。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_22.png)

## 分区、格式化与挂载实践

上一节我们学习了理论步骤，本节中我们通过实际操作来掌握这些命令。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_24.png)

以下是操作一块新硬盘（例如 `/dev/sdb`）的完整流程演示。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_26.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_28.png)

### 1. 分区

使用 `fdisk` 命令对硬盘进行分区。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_30.png)

```bash
fdisk /dev/sdb
```

![](img/03d9bd8f6248b5fc022f955fe7a7be25_32.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_33.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_34.png)

进入交互界面后，常用命令如下：
*   `n`：新建分区。
*   `p`：打印当前分区表。
*   `d`：删除分区。
*   `w`：保存并退出。
*   `q`：不保存退出。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_36.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_38.png)

创建分区时，可以指定分区大小，例如 `+5G` 表示创建一个5GB的分区。分区完成后，有时系统可能无法立即识别新分区，可以运行 `partprobe` 命令强制内核重新读取分区表。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_40.png)

### 2. 格式化

使用 `mkfs` 命令对分区进行格式化。例如，格式化为EXT4文件系统：

```bash
mkfs.ext4 /dev/sdb1
```

### 3. 挂载

![](img/03d9bd8f6248b5fc022f955fe7a7be25_42.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_44.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_46.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_47.png)

使用 `mount` 命令进行挂载。首先创建一个目录作为挂载点，然后将分区挂载上去。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_49.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_50.png)

```bash
mkdir /xiaodou
mount /dev/sdb1 /xiaodou
```

![](img/03d9bd8f6248b5fc022f955fe7a7be25_52.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_53.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_55.png)

使用 `df -h` 命令可以查看当前的挂载信息。

### 4. 卸载

![](img/03d9bd8f6248b5fc022f955fe7a7be25_57.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_59.png)

使用 `umount` 命令可以卸载设备。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_61.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_62.png)

```bash

# 或

```

![](img/03d9bd8f6248b5fc022f955fe7a7be25_67.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_69.png)

### 5. 实现永久挂载

![](img/03d9bd8f6248b5fc022f955fe7a7be25_72.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_74.png)

通过 `mount` 命令进行的挂载在重启后会失效。要实现永久挂载，需要编辑 `/etc/fstab` 文件。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_76.png)

`/etc/fstab` 文件的每一行代表一个自动挂载项，格式如下：

```
设备文件路径 挂载点目录 文件系统类型 挂载参数 是否备份 是否自检
```

例如，添加以下行可使 `/dev/sdb1` 在开机时自动挂载到 `/xiaodou`：

![](img/03d9bd8f6248b5fc022f955fe7a7be25_78.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_79.png)

```
/dev/sdb1 /xiaodou ext4 defaults 0 0
```

## 交换分区（SWAP）

![](img/03d9bd8f6248b5fc022f955fe7a7be25_83.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_85.png)

上一节我们完成了普通数据分区的管理，本节中我们来看看一种特殊的分区——交换分区。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_87.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_88.png)

交换分区（SWAP）的作用是**当物理内存不足时，将内存中不常用的“冷数据”临时移动到硬盘空间**，以降低内存负载压力。这可以防止因内存耗尽导致系统崩溃。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_90.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_91.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_92.png)

### 创建与启用交换分区

![](img/03d9bd8f6248b5fc022f955fe7a7be25_94.png)

1.  **分区**：使用 `fdisk` 创建一个新分区（例如 `/dev/sdb5`）。
2.  **格式化**：使用 `mkswap` 命令专门格式化交换分区。

    ```bash
    mkswap /dev/sdb5
    ```
3.  **启用**：使用 `swapon` 命令启用交换分区。

    ```bash
    swapon /dev/sdb5
    ```
4.  **查看**：使用 `free -h` 命令查看交换分区大小。
5.  **永久生效**：同样需要将交换分区信息写入 `/etc/fstab`。

    ```
    /dev/sdb5 swap swap defaults 0 0
    ```

![](img/03d9bd8f6248b5fc022f955fe7a7be25_98.png)

## 磁盘配额（Quota）

![](img/03d9bd8f6248b5fc022f955fe7a7be25_100.png)

上一节我们配置了系统资源，本节中我们学习如何限制用户对磁盘资源的使用。

磁盘配额（Quota）技术用于**限制用户或组在特定文件系统上可使用的磁盘空间大小或文件数量**。这可以防止个别用户过度占用磁盘空间，影响其他用户或系统服务。

![](img/03d9bd8f6248b5fc022f955fe7a7be25_102.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_103.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_104.png)

在RHEL 7的XFS文件系统上，启用配额需要为挂载点添加 `uquota` 参数（在 `/etc/fstab` 中）。

### 配置磁盘配额

![](img/03d9bd8f6248b5fc022f955fe7a7be25_106.png)

![](img/03d9bd8f6248b5fc022f955fe7a7be25_107.png)

1.  **重新挂载文件系统**（或重启）使配额参数生效。
2.  **为用户设置配额规则**：使用 `xfs_quota` 命令。

    ```bash
    xfs_quota -x -c ‘limit bsoft=3m bhard=6m isoft=3 ihard=6 username‘ /挂载点目录
    ```
    *   `bsoft`/`bhard`：磁盘容量的软/硬限制。
    *   `isoft`/`ihard`：文件数量的软/硬限制。
    *   软限制：超过后会警告，但仍允许使用。
    *   硬限制：绝对上限，超过后禁止操作。
3.  **验证**：切换到相应用户，尝试创建超出限制的文件，会收到错误提示。
4.  **修改配额**：管理员可以使用 `edquota -u username` 命令交互式地修改用户的配额限制。

## 软链接与硬链接

上一节我们学习了磁盘配额，本节中我们来看看Linux中文件的两种链接方式。

链接（Link）类似于Windows系统中的“快捷方式”，它允许通过多个路径访问同一个文件。Linux中有两种链接：**软链接**和**硬链接**。

*   **软链接（Symbolic Link）**：
    *   创建命令：`ln -s 源文件 链接文件`
    *   本质是一个独立的文件，内容中存储了**源文件的路径**。
    *   如果删除源文件，软链接将失效（成为“断链”）。
    *   可以跨文件系统创建，也可以为目录创建。

*   **硬链接（Hard Link）**：
    *   创建命令：`ln 源文件 链接文件`
    *   本质是**为同一个Inode创建了另一个文件名引用**。
    *   删除任何一个硬链接（包括源文件），只要Inode还有引用，数据就不会丢失。
    *   不能跨文件系统创建，也不能为目录创建。

使用 `ls -li` 命令查看文件时，第二列的数字表示该Inode的**硬链接计数**。每创建一个硬链接，这个计数就会加1。

## RAID技术简介

上一节我们探讨了文件链接，本节中我们初步了解提升磁盘性能和可靠性的RAID技术。

RAID（Redundant Array of Independent Disks，独立磁盘冗余阵列）技术通过将多块硬盘组合起来，实现**提升数据读写速度**和**增强数据安全性**的目标。

RAID技术最初旨在将多个廉价的小容量硬盘组合成一个大容量逻辑盘使用。随着发展，其价值更多体现在性能与冗余上。

RAID有多个级别（Level），例如RAID 0， RAID 1， RAID 5， RAID 10等，每种级别在性能、容量和冗余能力上各有侧重。我们将在后续课程中详细探讨。
