# RHCE8.0 红帽认证入门教程：Day05：文件管理与Web控制台

在本节课中，我们将要学习Linux系统中的文件定位、查找方法，并了解红帽系统提供的图形化Web管理工具Cockpit。这些技能对于系统管理和日常运维至关重要。

## 文件定位

上一节我们介绍了Linux的基本操作，本节中我们来看看如何定位系统中的文件。

### 使用 `which` 命令定位可执行文件

`which` 命令用于在环境变量 `PATH` 定义的路径中查找可执行文件。其基本语法如下：

![](img/e934b67dac107f443c2e801be0646374_1.png)

```bash
which <命令名>
```

![](img/e934b67dac107f443c2e801be0646374_3.png)

![](img/e934b67dac107f443c2e801be0646374_5.png)

例如，查找 `dnf` 命令的位置：

![](img/e934b67dac107f443c2e801be0646374_7.png)

```bash
which dnf
```

如果目标命令不在 `PATH` 变量包含的路径中，`which` 命令将无法找到。

### 使用 `locate` 命令定位普通文件

![](img/e934b67dac107f443c2e801be0646374_9.png)

`locate` 命令通过预建的索引数据库来快速查找文件。首次使用前需要更新数据库。

以下是使用 `locate` 的基本步骤：

![](img/e934b67dac107f443c2e801be0646374_11.png)

1.  更新索引数据库：
    ```bash
    updatedb
    ```
2.  使用 `locate` 查找文件：
    ```bash
    locate <文件名或关键字>
    ```
    例如，查找所有包含 “student” 的文件：
    ```bash
    locate student
    ```

![](img/e934b67dac107f443c2e801be0646374_13.png)

`locate` 的索引数据库通常位于 `/var/lib/mlocate/mlocate.db`。若要强制重建索引，可删除此数据库文件后再次执行 `updatedb`。

![](img/e934b67dac107f443c2e801be0646374_14.png)

### 挂载点目录

Linux中有几个常用的挂载点目录，用于挂载文件系统：

![](img/e934b67dac107f443c2e801be0646374_16.png)

*   **`/mnt`**： 临时挂载文件系统的传统目录。
*   **`/media`**： 通常用于自动挂载可移动介质（如U盘、光盘）。
*   **`/tmp`**： 存放临时文件。

挂载命令的基本用法如下：

![](img/e934b67dac107f443c2e801be0646374_18.png)

```bash
mount <设备> <挂载点>
```

例如，将设备 `/dev/sdb1` 挂载到 `/mnt/data`：
```bash
mount /dev/sdb1 /mnt/data
```

![](img/e934b67dac107f443c2e801be0646374_20.png)

卸载命令使用 `umount` 后跟挂载点：
```bash
umount /mnt/data
```

![](img/e934b67dac107f443c2e801be0646374_22.png)

若要实现开机自动挂载，需要编辑 `/etc/fstab` 文件进行配置，这部分内容将在后续磁盘管理的课程中详细讲解。

## 使用 `find` 命令进行条件查找

![](img/e934b67dac107f443c2e801be0646374_24.png)

上一节我们介绍了简单的文件定位方法，本节中我们来看看功能更强大的条件查找工具 `find`。

![](img/e934b67dac107f443c2e801be0646374_26.png)

`find` 命令用于在指定目录及其子目录中，根据用户设定的条件搜索文件和目录。其基本语法结构为：

![](img/e934b67dac107f443c2e801be0646374_28.png)

```bash
find <起始路径> <查找条件> [后续操作]
```

![](img/e934b67dac107f443c2e801be0646374_29.png)

![](img/e934b67dac107f443c2e801be0646374_31.png)

*   **起始路径**： 指定开始搜索的目录，例如 `/`（根目录）或 `.`（当前目录）。
*   **查找条件**： 定义搜索的过滤标准。
*   **后续操作**： 对搜索结果进行处理，如 `-exec` 或通过管道传递给 `xargs`。

以下是 `find` 命令的常用条件示例：

### 按文件拥有者查找

查找 `/home` 目录下所有属于用户 `test2` 的文件：

```bash
find /home -user test2
```

### 按文件大小查找

![](img/e934b67dac107f443c2e801be0646374_33.png)

查找 `/boot` 目录下大小等于 2MB 的文件：

```bash
find /boot -size 2M
```

查找大小大于 2MB 的文件（使用 `+`）：

```bash
find /boot -size +2M
```

![](img/e934b67dac107f443c2e801be0646374_35.png)

![](img/e934b67dac107f443c2e801be0646374_36.png)

查找大小小于 2MB 的文件（使用 `-`）：

```bash
find /boot -size -2M
```

### 按文件名查找

![](img/e934b67dac107f443c2e801be0646374_38.png)

在 `/etc` 目录下查找名为 `fstab` 的文件：

```bash
find /etc -name fstab
```

### 按文件权限查找

查找 `/home` 目录下权限恰好为 `644`（即所有者可读写，其他人只读）的文件：

```bash
find /home -perm 644
```

也可以使用符号模式查找，例如查找所有者可读可写 (`u=rw`) 的文件：

![](img/e934b67dac107f443c2e801be0646374_40.png)

```bash
find /home -perm u=rw
```

### 按文件类型查找

查找 `/var/log` 目录下的所有普通文件 (`-type f`)：

```bash
find /var/log -type f
```

查找所有目录 (`-type d`)：

```bash
find /var/log -type d
```

![](img/e934b67dac107f443c2e801be0646374_42.png)

### 按时间查找

![](img/e934b67dac107f443c2e801be0646374_44.png)

Linux文件有三个主要时间戳：
*   **访问时间 (atime)**： 文件最后一次被读取的时间。
*   **修改时间 (mtime)**： 文件内容最后一次被修改的时间。
*   **变更时间 (ctime)**： 文件元数据（如权限、所有者）最后一次被改变的时间。修改内容通常也会改变 `ctime`。

查找 `/tmp` 目录下内容在 **2天前** 被修改过的文件：

![](img/e934b67dac107f443c2e801be0646374_46.png)

```bash
find /tmp -mtime +2
```

查找内容在 **2天以内** 被修改过的文件：

```bash
find /tmp -mtime -2
```

查找内容修改时间 **恰好是2天前** 的文件：

```bash
find /tmp -mtime 2
```

**注意**：`+2` 表示大于2天（即至少3天前），`-2` 表示小于2天（即最近2天内），`2` 表示在2天到3天之间。

### 对查找结果执行操作

![](img/e934b67dac107f443c2e801be0646374_48.png)

`find` 命令常与 `-exec` 参数结合，对找到的文件执行后续命令。

例如，在 `/usr/share/doc` 目录下，查找深度不超过3级的普通文件，并将其复制到 `/tmp/find_results` 目录中：

![](img/e934b67dac107f443c2e801be0646374_50.png)

```bash
mkdir /tmp/find_results
find /usr/share/doc -maxdepth 3 -type f -exec cp {} /tmp/find_results \;
```

*   `-maxdepth 3`： 指定最大搜索深度为3级子目录。
*   `-exec cp {} /tmp/find_results \;`： 对每个找到的文件执行 `cp` 命令，`{}` 会被替换为当前找到的文件路径，`\;` 表示命令结束。

也可以使用 `xargs` 命令达到类似效果，但 `-exec` 更为常用和直接。

## 使用 Cockpit 进行 Web 管理

![](img/e934b67dac107f443c2e801be0646374_52.png)

![](img/e934b67dac107f443c2e801be0646374_53.png)

前面我们学习了命令行下的文件管理，本节中我们来看看红帽系统提供的图形化Web管理工具Cockpit。

![](img/e934b67dac107f443c2e801be0646374_55.png)

Cockpit 是一个基于网页的服务器管理界面，适用于红帽企业版 Linux 7 及更高版本。它提供了系统状态监控、服务管理、存储配置、网络设置等功能。

![](img/e934b67dac107f443c2e801be0646374_57.png)

### 启用与访问 Cockpit

1.  **启用服务**： Cockpit 默认可能未启动。使用以下命令启用并立即启动它：
    ```bash
    systemctl enable --now cockpit.socket
    ```
    `--now` 参数同时实现了 `enable`（开机自启）和 `start`（立即启动）的效果。

![](img/e934b67dac107f443c2e801be0646374_59.png)

2.  **配置防火墙**： 如果系统防火墙开启，需要放行 Cockpit 服务（默认使用9090端口）：
    ```bash
    firewall-cmd --permanent --add-service=cockpit
    firewall-cmd --reload
    ```

![](img/e934b67dac107f443c2e801be0646374_61.png)

3.  **访问界面**： 打开浏览器，访问 `https://<服务器IP地址>:9090`。使用系统管理员（如 `root`）的用户名和密码登录。

### Cockpit 主要功能

登录后，您可以看到一个包含以下功能模块的仪表盘：

*   **系统状态**： 实时监控 CPU、内存、磁盘 I/O 和网络流量。
*   **电源管理**： 执行重启、关机等操作。
*   **日志查看器**： 查看系统日志，可按级别筛选。
*   **存储管理**： 查看磁盘、分区，甚至可以创建新的分区或逻辑卷。
*   **网络配置**： 配置网络接口、创建网桥或绑定（bonding）。
*   **账户管理**： 创建和管理系统用户账户。
*   **服务管理**： 查看、启动、停止系统服务，类似于图形化的 `systemctl`。
*   **终端访问**： 提供内置的 Web 终端，可直接执行命令。
*   **诊断报告**： 生成系统诊断报告。

![](img/e934b67dac107f443c2e801be0646374_63.png)

**请注意**：软件更新和红帽订阅管理等功能需要有效的红帽订阅账户。虽然 Cockpit 功能强大且对新手友好，但 RHCE 认证考试主要考察命令行技能，因此熟练掌握命令仍是学习重点。
