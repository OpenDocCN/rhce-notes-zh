# Linux软件包管理：第6章：RPM包管理器详解 🛠️

在本节课中，我们将要学习Linux系统中的软件包管理，特别是RPM包管理器的核心概念和基本操作。RPM是Red Hat系列发行版的基础包管理工具，理解它是掌握系统软件管理的第一步。

## 软件包管理概述

最初，开源软件以源代码形式发布，用户需要下载源码压缩包，在本地编译后才能运行。这种方式较为繁琐。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_1.png)

后来，Debian系统率先推出了`dpkg`包管理工具，将编译好的软件打包成`.deb`文件，首次引入了“包”的概念。受此启发，Red Hat开发了RPM包管理系统。

软件包管理器的主要功能是将编译好的应用程序及其相关文件打包成一个或几个包文件，从而方便地实现程序的安装、卸载、查询、升级和校验等操作。

在RHEL/CentOS等系统中，主要使用RPM包管理器，其包文件通常以`.rpm`结尾。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_3.png)

## RPM包命名规则

一个RPM包的名称遵循特定格式，其核心组成部分可以用以下公式表示：

**`软件名-版本号-发布号.系统架构.rpm`**

常见的系统架构（arch）包括：
*   `x86`：指32位系统，如`i386`，`i486`，`i586`，`i686`。
*   `x86_64`：指64位系统，同`amd64`。
*   `noarch`：表示与平台无关，32位和64位系统均可使用。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_5.png)

## 访问RPM包文件

RPM包通常存储在系统光盘或软件仓库中。要使用光盘中的包，需要先将其挂载到目录。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_7.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_9.png)

以下是挂载光盘并查看其中RPM包的步骤：
1.  创建挂载点目录：`mkdir /mnt/cdrom`
2.  挂载光盘设备：`mount /dev/sr0 /mnt/cdrom`
3.  进入包目录：`cd /mnt/cdrom/AppStream/Packages/` (RHEL 8将包分为`BaseOS`和`AppStream`两部分)

在目录中，你可以看到许多类似 `wget-1.19.5-10.el8.x86_64.rpm` 的包文件。

## 包的分类与依赖关系

RPM包可以分为主包和子包。主包名称如 `application-version-arch.rpm`。子包则可能包括：
*   `-utils`：工具子包
*   `-devel`：开发子包
*   `-libs`：库子包

![](img/d36591e00e3a07c8bc32b93bf46ffde2_11.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_13.png)

安装主包时，通常会自动安装其必需的子包。

软件包之间存在依赖关系。例如，安装`ansible`依赖于`python`。如果系统中没有`python`，则`ansible`安装会失败。更复杂的情况是循环依赖（A依赖B，B依赖C，C又依赖A），这会导致安装无法进行。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_15.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_17.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_19.png)

为了解决依赖问题，Red Hat引入了更高级的工具：`yum` 或它的新一代版本 `dnf`。在RHEL 8中，`yum`命令实际上是`dnf-3`的一个软链接，底层使用的是`dnf`。

## RPM包管理器基本操作

RPM命令是管理`.rpm`程序包的核心工具，主要功能包括安装、卸载、升级、查询和校验。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_21.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_23.png)

### 安装软件包

![](img/d36591e00e3a07c8bc32b93bf46ffde2_25.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_27.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_28.png)

安装RPM包的基本命令格式是 `rpm -i [选项] package_file.rpm`。

常用选项组合是 `-ivh`：
*   `-i`：安装
*   `-v`：显示详细信息
*   `-h`：显示安装进度条

例如，安装`wget`包：
```bash
rpm -ivh wget-1.19.5-10.el8.x86_64.rpm
```
在安装来自官方源的包时，可能需要导入GPG密钥来验证包完整性：
```bash
rpm --import /mnt/cdrom/RPM-GPG-KEY-redhat-release
```

### 查询软件包

查询是RPM最常用的功能之一，使用 `-q` 选项。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_30.png)

以下是常用的查询操作：

![](img/d36591e00e3a07c8bc32b93bf46ffde2_32.png)

**查询系统所有已安装的包**
```bash
rpm -qa
```

![](img/d36591e00e3a07c8bc32b93bf46ffde2_34.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_36.png)

**查询某个文件是由哪个包安装的**
此功能在误删文件后找回时非常有用。
```bash
rpm -qf /usr/bin/tar
```

![](img/d36591e00e3a07c8bc32b93bf46ffde2_38.png)

**查询某个已安装包的详细信息**
```bash
rpm -qi package_name
```

![](img/d36591e00e3a07c8bc32b93bf46ffde2_40.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_42.png)

**列出某个包安装的所有文件**
此功能常用于查找软件的配置文件、二进制程序位置。
```bash
rpm -ql package_name
```

![](img/d36591e00e3a07c8bc32b93bf46ffde2_44.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_46.png)

### 卸载与升级软件包

![](img/d36591e00e3a07c8bc32b93bf46ffde2_48.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_50.png)

**卸载包**使用 `-e` 选项：
```bash
rpm -e package_name
```
注意：卸载包通常不会删除用户创建的配置文件。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_52.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_54.png)

**升级包**使用 `-U` 选项。如果包未安装，则执行安装；如果已安装旧版本，则升级到新版本。
```bash
rpm -Uvh new_package.rpm
```

![](img/d36591e00e3a07c8bc32b93bf46ffde2_56.png)

### 校验与文件提取

![](img/d36591e00e3a07c8bc32b93bf46ffde2_58.png)

**校验包文件完整性**使用 `-K` 选项（需已导入密钥）：
```bash
rpm -K package_file.rpm
```

![](img/d36591e00e3a07c8bc32b93bf46ffde2_60.png)

**从RPM包中提取特定文件**，这在恢复误删的系统命令时非常关键：
1.  解压RPM包内容（不安装）：
    ```bash
    rpm2cpio package_file.rpm | cpio -idv
    ```
2.  从解压出的文件中找到需要的二进制文件或库，复制到正确位置（如`/usr/bin/`）。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_62.png)

![](img/d36591e00e3a07c8bc32b93bf46ffde2_64.png)

## RPM数据库

RPM维护一个数据库（通常位于`/var/lib/rpm/`），记录所有已安装软件包的信息、文件列表和依赖关系。**切勿随意破坏此数据库**，否则`rpm`查询命令可能无法正常工作，系统软件管理将陷入混乱。

![](img/d36591e00e3a07c8bc32b93bf46ffde2_66.png)
