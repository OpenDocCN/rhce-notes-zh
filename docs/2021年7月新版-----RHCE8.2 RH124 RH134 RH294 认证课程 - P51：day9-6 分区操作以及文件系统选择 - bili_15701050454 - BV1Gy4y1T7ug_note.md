## 为什么要进行磁盘分区 💡

![](img/21a7d88b2cd1361958a4651ee739d004_1.png)

![](img/21a7d88b2cd1361958a4651ee739d004_3.png)

对磁盘进行分区主要有以下几个目的：
*   **优化I/O性能**：将数据分散到不同分区，可以提高读写效率。
*   **实现磁盘空间配额限制**：可以为不同用户或组设置磁盘使用上限。
*   **提高修复速度**：当文件系统损坏时，修复一个较小的分区比修复整个大磁盘更快。
*   **隔离系统和程序**：将操作系统、应用程序和用户数据放在不同分区，可以提高系统安全性和稳定性。
*   **安装多个操作系统**：每个操作系统通常需要安装在独立的主分区上。
*   **采用不同的文件系统**：可以根据数据特点，在不同分区上使用最合适的文件系统。

## 磁盘分区格式 🔧

![](img/21a7d88b2cd1361958a4651ee739d004_22.png)

目前主要有两种磁盘分区格式：**MBR** 和 **GPT**。

![](img/21a7d88b2cd1361958a4651ee739d004_24.png)

### MBR分区格式

![](img/21a7d88b2cd1361958a4651ee739d004_26.png)

![](img/21a7d88b2cd1361958a4651ee739d004_27.png)

MBR是一种传统且较为老旧的分区格式。它的结构特点如下：
*   一个MBR磁盘最多只能有**4个主分区**。
*   可以通过**扩展分区**来突破4个分区的限制。扩展分区本身不存储数据，它只是一个容器，可以在其内部创建多个**逻辑分区**。
*   分区组合方式可以是：4个主分区；或3个主分区 + 1个扩展分区；或2个主分区 + 1个扩展分区；或1个主分区 + 1个扩展分区。
*   MBR使用32位表示扇区数，因此**单个分区大小不能超过2TB**。
*   MBR信息存储在磁盘的第一个扇区（512字节），其结构为：
    *   前**446字节**：存放引导程序（boot loader）。
    *   中间**64字节**：存放分区表信息（每个分区占16字节）。
    *   最后**2字节**：固定为`0x55AA`，是校验位。

我们可以使用`hexdump`命令查看MBR的原始数据：
```bash
hexdump -C /dev/sda -n 512
```

![](img/21a7d88b2cd1361958a4651ee739d004_29.png)

### GPT分区格式

GPT是一种更现代的分区格式，逐步取代MBR。它的特点如下：
*   支持**128个主分区**，没有扩展分区和逻辑分区的概念。
*   使用64位，**支持超过2TB的磁盘和分区**。
*   在磁盘首尾部分别保存了分区表信息，提供了更好的冗余性。

## 管理磁盘与分区的工具 🛠️

![](img/21a7d88b2cd1361958a4651ee739d004_31.png)

![](img/21a7d88b2cd1361958a4651ee739d004_33.png)

在Linux中，我们可以使用`lsblk`命令列出所有块设备（如硬盘、分区）。
```bash
lsblk
```
常见的设备命名规则：
*   `/dev/sd[a-z]`：SCSI、SATA、USB等接口的磁盘。
*   `/dev/vd[a-z]`：KVM虚拟化环境中的虚拟磁盘。
*   `/dev/nvme0n[1-9]`：NVMe协议的固态硬盘（SSD）。

接下来，我们介绍三种常用的分区工具。

![](img/21a7d88b2cd1361958a4651ee739d004_35.png)

### 1. `fdisk` 工具 (适用于MBR分区)

![](img/21a7d88b2cd1361958a4651ee739d004_37.png)

![](img/21a7d88b2cd1361958a4651ee739d004_39.png)

![](img/21a7d88b2cd1361958a4651ee739d004_41.png)

`fdisk`是一个交互式命令行工具，常用于管理小于2TB的MBR分区磁盘。

以下是`fdisk`的基本操作步骤：
1.  运行`fdisk /dev/sdX`进入交互界面（`sdX`为你的磁盘，如`sdb`）。
2.  输入`m`可以查看所有命令帮助。
3.  常用命令：
    *   `n`：创建新分区。
    *   `d`：删除分区。
    *   `p`：打印当前分区表。
    *   `t`：更改分区类型。
    *   `w`：保存更改并退出。
    *   `q`：不保存更改并退出。

**重要提示**：对正在使用的系统磁盘操作需极其谨慎，误操作可能导致数据丢失。

### 2. `gdisk` 工具 (适用于GPT分区)

`gdisk`是用于管理GPT分区磁盘的工具，其用法与`fdisk`非常相似。
```bash
gdisk /dev/sdX
```
**警告**：切勿在MBR格式的磁盘上使用`gdisk`，或在GPT格式的磁盘上使用`fdisk`，这会破坏原有分区表导致数据丢失。使用前可用`fdisk -l`或`gdisk -l`查看磁盘的当前分区格式。

### 3. `parted` 工具 (支持MBR和GPT，适用于大容量磁盘)

![](img/21a7d88b2cd1361958a4651ee739d004_43.png)

![](img/21a7d88b2cd1361958a4651ee739d004_45.png)

![](img/21a7d88b2cd1361958a4651ee739d004_47.png)

`parted`是一个更强大的工具，既支持交互模式也支持非交互（命令行）模式，并且是管理**超过2TB磁盘**的必备工具。

进入交互模式：
```bash
parted /dev/sdX
```
常用`parted`命令：
*   `print`：打印分区信息。
*   `mklabel`：创建新的磁盘标签（即选择`msdos`(MBR)或`gpt`格式）。
*   `mkpart`：创建新分区。
*   `rm`：删除分区。
*   `quit`：退出。

使用`parted`时，必须先用`mklabel`明确指定分区格式，然后再进行分区操作。

## 文件系统概念与格式化 📂

创建分区后，需要为其创建文件系统才能存储数据，这个过程称为“格式化”。

### 常见的Linux文件系统

*   **ext2/ext3/ext4**：Linux上历史最悠久、最常用的文件系统系列。ext4是目前默认的推荐选择，支持最大1EB的文件系统。
*   **XFS**：高性能日志文件系统，特别擅长处理大文件，支持最大8EB的文件系统。
*   **swap**：交换分区专用的文件系统，用于虚拟内存。
*   **iso9660**：光盘标准文件系统。

![](img/21a7d88b2cd1361958a4651ee739d004_53.png)

![](img/21a7d88b2cd1361958a4651ee739d004_55.png)

可以使用以下命令查看当前内核支持的文件系统类型：
```bash
cat /proc/filesystems
```

![](img/21a7d88b2cd1361958a4651ee739d004_57.png)

### 格式化分区

![](img/21a7d88b2cd1361958a4651ee739d004_59.png)

![](img/21a7d88b2cd1361958a4651ee739d004_61.png)

使用`mkfs`（make filesystem）命令家族来格式化分区。

**方法一：使用`mkfs`通用命令**
指定文件系统类型进行格式化：
```bash
mkfs -t ext4 /dev/sdX1
```
或者使用更简洁的格式：
```bash
mkfs.ext4 /dev/sdX1
```
对于XFS文件系统：
```bash
mkfs.xfs /dev/sdX1
```

![](img/21a7d88b2cd1361958a4651ee739d004_63.png)

**方法二：使用文件系统专用工具**
例如，对于ext系列，可以使用`mke2fs`命令：
```bash
mke2fs -t ext4 /dev/sdX1
```

![](img/21a7d88b2cd1361958a4651ee739d004_65.png)

![](img/21a7d88b2cd1361958a4651ee739d004_67.png)

格式化完成后，可以使用`blkid`命令查看分区的UUID和文件系统类型：
```bash
blkid /dev/sdX1
```
