## 实验环境介绍与使用 🖥️

首先，我们来了解一下本次课程所使用的实验环境。这个环境是为了完成教材课后习题而设计的，它模拟了真实的考试和实验场景。

![](img/e789b578e76f2c9079d43081ccbf7941_18.png)

![](img/e789b578e76f2c9079d43081ccbf7941_20.png)

环境的核心是一台名为 **foundation0** 的物理机（在本课程中以虚拟机形式提供），其内部运行着多台虚拟机。以下是环境中的关键虚拟机及其角色：

![](img/e789b578e76f2c9079d43081ccbf7941_22.png)

*   **classroom**: 这是整个环境中最关键的服务器，充当DHCP等服务端角色。**必须首先启动此虚拟机**，其他虚拟机才能正常获取IP并运行。
*   **bastion**: 堡垒机/跳板机，用于连接两个不同网段（172.25.252.0/24 和 172.25.250.0/24）。
*   **workstation**: 带有图形界面的客户端虚拟机，是许多实验的操作起点。
*   **servera & serverb**: 两台没有图形界面的服务器虚拟机，是课程中主要的操作对象。

### 环境管理命令

![](img/e789b578e76f2c9079d43081ccbf7941_28.png)

![](img/e789b578e76f2c9079d43081ccbf7941_30.png)

在 `foundation0` 机器上，可以使用以下专用命令来管理这些虚拟机（这些是环境自定义命令，非通用Linux命令）：

![](img/e789b578e76f2c9079d43081ccbf7941_32.png)

*   **查看虚拟机状态**:
    ```bash
    rht-vmctl status
    ```
*   **启动所有虚拟机（不包括classroom）**:
    ```bash
    rht-vmctl start all
    ```
*   **启动指定虚拟机（如servera）**:
    ```bash
    rht-vmctl start servera
    ```
*   **停止指定虚拟机**:
    ```bash
    rht-vmctl stop servera
    ```
*   **彻底重置虚拟机（推荐）**:
    ```bash
    rht-vmctl fullreset servera
    ```

![](img/e789b578e76f2c9079d43081ccbf7941_34.png)

![](img/e789b578e76f2c9079d43081ccbf7941_36.png)

**重要提示**:
*   所有虚拟机的默认 root 用户密码均为 `redhat`。
*   考试环境与本实验环境类似，但操作会通过图形化按钮完成，而非命令行。
*   该环境资源消耗较大，建议主机内存不低于8GB，并使用SSD硬盘以获得流畅体验。

## RHEL8 操作系统安装指南 💿

上一节我们了解了实验环境，本节中我们来看看如何从零开始安装一个RHEL8操作系统。我们将使用虚拟机进行演示。

![](img/e789b578e76f2c9079d43081ccbf7941_40.png)

![](img/e789b578e76f2c9079d43081ccbf7941_42.png)

![](img/e789b578e76f2c9079d43081ccbf7941_44.png)

### 安装前准备

1.  在VMware Workstation中创建一台新的虚拟机。
2.  选择“稍后安装操作系统”，类型选择“Linux”，版本选择“Red Hat Enterprise Linux 8 64位”。
3.  为虚拟机分配存储空间（例如20GB）并完成创建。
4.  将RHEL8的ISO镜像文件加载到虚拟机的光驱中。

![](img/e789b578e76f2c9079d43081ccbf7941_46.png)

![](img/e789b578e76f2c9079d43081ccbf7941_48.png)

### 安装步骤详解

启动虚拟机后，将进入安装界面。

1.  **选择安装语言**：在安装向导中，**建议选择英文（English）**。这里选择的语言仅影响安装过程界面，并非系统安装后的语言。在生产环境、考试及日常使用中，均推荐使用英文系统，以避免可能的乱码问题，且命令行操作与界面语言无关。
2.  **软件选择**：在“SOFTWARE SELECTION”中，可以选择安装模式。对于初学者，建议选择带有图形界面的 **“Server with GUI”**。企业环境中常选择 **“Minimal Install”**（最小化安装）以节省资源和提高安全性。
3.  **安装目的地（分区）**：这是安装的核心步骤。选择“Custom”进行手动分区。
    *   点击“+”创建挂载点。
    *   **挂载点** 可以暂时理解为Windows中的盘符，但Linux中所有存储设备都通过挂载点接入唯一的文件系统树。
    *   常见的必须挂载点包括：
        *   **`/` (根)**: 文件系统的起点，必须存在且唯一。
        *   **`/boot`**: 存放系统启动所需的内核和引导文件。建议分配1GB左右，无需过大。
        *   **`/home`**: 普通用户的家目录所在地。可根据需要分配较大空间。
        *   **`swap`**: 交换分区，相当于虚拟内存。当物理内存不足时，将不活跃的内存页交换到此分区。**建议分配**，大小通常为物理内存的1-1.5倍。
4.  **分区策略与主/逻辑分区**：Linux硬盘设备名取决于接口类型（如SATA接口为`sda`，NVMe接口为`nvme0n1`）。MBR分区表格式的硬盘最多只能有**4个主分区**。安装程序会自动优化，当主分区可能用尽时，会将后续分区创建为**扩展分区内的逻辑分区**。逻辑分区的编号从5开始（例如`sda5`）。在考试或复杂分区需求中，需要注意此限制，合理规划主分区和逻辑分区。
5.  **完成配置**：设置root密码，并创建一个普通用户（安装程序强制要求，旨在提倡不使用root进行日常登录）。点击“Begin Installation”开始安装。
6.  **安装后设置**：安装完成后重启，首次启动需要接受许可证协议，即可进入系统。

### 命令提示符与语法

![](img/e789b578e76f2c9079d43081ccbf7941_88.png)

![](img/e789b578e76f2c9079d43081ccbf7941_90.png)

![](img/e789b578e76f2c9079d43081ccbf7941_92.png)

打开终端后，你会看到命令提示符，例如：`[kiosk@foundation0 ~]$`
*   `kiosk`: 当前登录的用户名。
*   `@`: 分隔符。
*   `foundation0`: 主机名。
*   `~`: 当前所在目录（`~`代表用户的家目录）。
*   `$`: 提示符，`$`表示普通用户，`#`表示root管理员。

![](img/e789b578e76f2c9079d43081ccbf7941_94.png)

![](img/e789b578e76f2c9079d43081ccbf7941_96.png)

![](img/e789b578e76f2c9079d43081ccbf7941_98.png)

![](img/e789b578e76f2c9079d43081ccbf7941_100.png)

Linux命令的基本语法格式为：
```
命令 [选项] [参数]
```
*   **命令**：必须存在，表示要执行的操作。
*   **选项**：用于修改命令的行为，通常以 `-`（短选项）或 `--`（长选项）开头。多个短选项可以合并，如 `-la` 等同于 `-l -a`。
*   **参数**：命令作用的对象，如文件名、目录名等。
*   各部分之间必须用**空格**分隔。

![](img/e789b578e76f2c9079d43081ccbf7941_102.png)

**示例**:
```bash
ls -l /home
```
*   `ls` 是命令（列出目录内容）。
*   `-l` 是选项（以长格式显示详细信息）。
*   `/home` 是参数（要查看的目录）。

### 常用基础命令

![](img/e789b578e76f2c9079d43081ccbf7941_116.png)

![](img/e789b578e76f2c9079d43081ccbf7941_118.png)

以下是初学者必须掌握的几个基础命令：

![](img/e789b578e76f2c9079d43081ccbf7941_120.png)

*   **`pwd`**: 打印当前工作目录。
*   **`ls`**: 列出目录内容。
    *   `ls -l`: 长格式列表，显示权限、所有者、大小、时间等。
    *   `ls -a`: 显示所有文件，包括隐藏文件（以 `.` 开头的文件）。
*   **`cd`**: 切换目录。
    *   `cd /home`: 切换到绝对路径 `/home`。
    *   `cd ..`: 切换到上级目录。
    *   `cd` 或 `cd ~`: 切换到当前用户的家目录。
*   **`passwd`**: 修改用户密码。
    *   **root用户**：可以为任何用户修改密码，且密码可以不满足复杂性要求。`passwd username`
    *   **普通用户**：只能修改自己的密码，且必须满足复杂性要求。直接输入 `passwd`，后面不跟用户名。
*   **`file`**: 判断文件类型。`file filename`
*   **查看文件内容**:
    *   **`cat`**: 连接并显示整个文件内容，适合小文件。`cat -n filename`（显示行号）。
    *   **`less` / `more`**: 分页显示文件内容，适合大文件。支持上下翻页（`less` 更强大）。
    *   **`head`**: 显示文件开头部分，默认10行。`head -n 5 filename`（显示前5行）。
    *   **`tail`**: 显示文件末尾部分，默认10行。`tail -n 5 filename`（显示后5行）。`tail -f filename`（实时追踪文件新增内容，常用于查看日志）。
*   **`wc`**: 统计文件的行数、单词数和字符数。
    *   `wc -l filename`: 统计行数。
    *   `wc -w filename`: 统计单词数。
    *   `wc -c filename`: 统计字符数。

### 提高效率的技巧

![](img/e789b578e76f2c9079d43081ccbf7941_128.png)

![](img/e789b578e76f2c9079d43081ccbf7941_130.png)

![](img/e789b578e76f2c9079d43081ccbf7941_132.png)

*   **命令补全**：按 `Tab` 键可以自动补全命令、文件名或目录名。按一次补全唯一项，按两次显示所有可能项。
*   **历史命令**：
    *   `history`: 查看之前执行过的命令历史列表。
    *   `!n`: 执行历史记录中第 n 条命令。
    *   `!!`: 执行上一条命令。
    *   上下箭头键：在历史命令中浏览。
    *   `Ctrl + R`: 进入反向搜索模式，输入关键词搜索历史命令。
    *   历史记录保存在用户家目录的隐藏文件 `~/.bash_history` 中。使用 `history -c` 可以清空当前会话的历史记录（不影响文件）。

## Linux文件系统层次结构 📁

![](img/e789b578e76f2c9079d43081ccbf7941_154.png)

![](img/e789b578e76f2c9079d43081ccbf7941_156.png)

要熟练管理文件，必须理解Linux文件系统的组织方式。这与Windows有根本区别。

### 核心概念：唯一的树形结构

![](img/e789b578e76f2c9079d43081ccbf7941_158.png)

![](img/e789b578e76f2c9079d43081ccbf7941_160.png)

![](img/e789b578e76f2c9079d43081ccbf7941_162.png)

![](img/e789b578e76f2c9079d43081ccbf7941_164.png)

Linux文件系统采用一个**倒置的、唯一的树形结构**。整个系统只有一个起点，称为**根目录**，用 `/` 表示。

![](img/e789b578e76f2c9079d43081ccbf7941_166.png)

![](img/e789b578e76f2c9079d43081ccbf7941_168.png)

*   **与Windows的区别**：Windows每个驱动器（C:， D:）都是一个独立的树根。而Linux中，所有的硬盘分区、光盘、U盘等存储设备，都必须“挂载”到这棵唯一的树上的某个目录（挂载点）才能访问。
*   **路径表示**：所有文件和目录的路径都从根 `/` 开始。例如，`/home/student/file.txt`。
*   **挂载点**：在安装系统分区时，我们为 `/`, `/home`, `/boot` 等分配了独立空间。它们虽然是独立分区，但在访问时，依然通过像 `/home` 这样的路径进行，这体现了逻辑上的统一性。如果 `/` 分区损坏，未独立分区的目录（如 `/etc`）将丢失，但独立分区的 `/home` 数据可能得以保全，这强调了关键目录单独分区的重要性。

![](img/e789b578e76f2c9079d43081ccbf7941_170.png)

![](img/e789b578e76f2c9079d43081ccbf7941_172.png)

### 重要目录简介

![](img/e789b578e76f2c9079d43081ccbf7941_174.png)

以下是一些标准目录及其常见用途：
*   **`/`**: 根目录，所有目录的起点。
*   **`/boot`**: 存放系统引导文件。
*   **`/home`**: 普通用户的个人目录。
*   **`/root`**: 系统管理员（root）的家目录。
*   **`/etc`**: 存放系统配置文件。
*   **`/var`**: 存放经常变化的文件，如日志、邮件。
*   **`/tmp`**: 存放临时文件，所有用户都可读写。
*   **`/bin`, `/sbin`**: 存放系统基本命令。`/sbin` 下的命令通常需要管理员权限。
*   **`/usr`**: 存放用户安装的应用程序和文件。
