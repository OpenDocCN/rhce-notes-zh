## 过滤空行与行首/行尾匹配

上一节我们介绍了`grep`的基本用法，本节中我们来看看如何使用正则表达式进行更精确的匹配。

`grep`命令可以配合正则表达式来过滤文本。例如，要过滤文件中的空行，可以使用`^$`这个正则表达式。`^`代表行首，`$`代表行尾，`^$`组合在一起就代表一个空行。

![](img/59bd55cc81ac868764993c157b32687c_4.png)

**命令示例：**
```bash
grep '^$' filename
```

如果想在过滤掉包含特定字符串（如`root`）的行的同时，也排除空行，可以组合使用`-v`选项（反向选择）和`^$`。

**命令示例：**
```bash
grep -v root filename | grep -v '^$'
```
这个命令管道的第一步会过滤掉所有包含`root`的行，第二步则会过滤掉第一步结果中所有的空行。

## 单词边界与任意字符匹配

![](img/59bd55cc81ac868764993c157b32687c_10.png)

除了匹配整行，我们还可以匹配行中的特定单词。`grep`提供了单词边界的匹配符。

`\<`和`\>`分别用于匹配单词的开头和结尾。例如，`\<root`会匹配以`root`开头的单词（如`root`、`rooter`），而`root\>`则会匹配以`root`结尾的单词（如`root`、`broot`）。

**命令示例：**
```bash
grep '\<root' /etc/passwd   # 匹配以root开头的单词
grep 'root\>' /etc/passwd   # 匹配以root结尾的单词
```

![](img/59bd55cc81ac868764993c157b32687c_12.png)

![](img/59bd55cc81ac868764993c157b32687c_14.png)

`.`（点）在正则表达式中代表匹配任意一个字符。

**命令示例：**
```bash
grep 'r.t' filename
```
这个命令会匹配像`rat`、`rot`、`r@t`这样的字符串，因为`.`可以代表任何单个字符。

## 字符集与范围匹配

![](img/59bd55cc81ac868764993c157b32687c_21.png)

中括号`[]`用于定义一个字符集合，匹配集合中的任意一个字符。

![](img/59bd55cc81ac868764993c157b32687c_23.png)

![](img/59bd55cc81ac868764993c157b32687c_25.png)

`^`在中括号内和外含义不同。在中括号外，`^`表示行首；在中括号内，`^`表示“非”，即排除后面的字符。

![](img/59bd55cc81ac868764993c157b32687c_26.png)

![](img/59bd55cc81ac868764993c157b32687c_28.png)

**命令示例：**
```bash
grep '^[ABC]' filename   # 匹配以A或B或C开头的行
grep '^[^ABC]' filename  # 匹配不以A、B、C开头的行
```

可以在中括号内使用`-`指定一个连续的范围。

![](img/59bd55cc81ac868764993c157b32687c_30.png)

![](img/59bd55cc81ac868764993c157b32687c_32.png)

**命令示例：**
```bash
grep '[a-z]' filename   # 匹配包含任意小写字母的行
grep '[A-Z]' filename   # 匹配包含任意大写字母的行
grep '[0-9]' filename   # 匹配包含任意数字的行
```

## 重复匹配与转义字符

`*`（星号）表示对其前面的字符进行0次或多次的重复匹配。

![](img/59bd55cc81ac868764993c157b32687c_40.png)

**命令示例：**
```bash
grep 'ro*t' filename
```
这个命令会匹配`rt`（0个o）、`rot`（1个o）、`root`（2个o）等。

![](img/59bd55cc81ac868764993c157b32687c_42.png)

反斜杠`\`是转义字符，用于取消其后字符的特殊含义，使其被当作普通字符处理。这在匹配空格、点等特殊字符时非常有用。

![](img/59bd55cc81ac868764993c157b32687c_44.png)

![](img/59bd55cc81ac868764993c157b32687c_45.png)

**命令示例：**
```bash
grep 'root\ user' filename  # 匹配“root user”这个字符串（中间有空格）
```

## 扩展正则表达式

![](img/59bd55cc81ac868764993c157b32687c_51.png)

![](img/59bd55cc81ac868764993c157b32687c_53.png)

扩展正则表达式（Extended Regular Expressions）提供了更多元字符。要使用它们，需要给`grep`加上`-E`选项，或者直接使用`egrep`命令。

`|`（竖线）表示“或”的关系，匹配其左边或右边的模式。

![](img/59bd55cc81ac868764993c157b32687c_55.png)

**命令示例：**
```bash
egrep 'root|user' /etc/passwd
# 或
grep -E 'root|user' /etc/passwd
```
这个命令会匹配包含`root`或`user`的行。

`+`表示对其前面的字符进行1次或多次匹配。
`?`表示对其前面的字符进行0次或1次匹配。
`{m,n}`表示对其前面的字符进行m到n次匹配。

**命令示例：**
```bash
egrep 'ro+t' filename   # 匹配rot, root, rooot...（至少1个o）
egrep 'ro?t' filename   # 匹配rt, rot（0个或1个o）
egrep 'ro{2,3}t' filename # 匹配root（2个o）或rooot（3个o）
```

## cut命令：按列提取文本

![](img/59bd55cc81ac868764993c157b32687c_59.png)

![](img/59bd55cc81ac868764993c157b32687c_61.png)

与`grep`按行处理不同，`cut`命令用于从文件的每一行中提取指定的列（字段）。

使用`cut`时，通常需要指定两个关键选项：
*   `-d`：指定字段分隔符（默认为制表符）。
*   `-f`：指定要提取的字段编号（从1开始）。

![](img/59bd55cc81ac868764993c157b32687c_63.png)

**命令示例：**
```bash
cut -d ':' -f 1 /etc/passwd
```
这个命令会以冒号`:`作为分隔符，提取`/etc/passwd`文件每一行的第一列（即用户名）。

![](img/59bd55cc81ac868764993c157b32687c_65.png)

## 实践：组合使用grep与cut

在实际工作中，我们经常需要组合多个命令来完成复杂任务。

例如，想要从`df -h`命令的输出中提取特定磁盘的使用率百分比（如26%），可能会遇到字段间空格数量不固定的问题。

一个解决方法是先用`tr -s`命令压缩连续的空格为单个空格，然后再用`cut`处理。

**命令示例：**
```bash
df -h | grep '/dev/sda1' | tr -s ' ' | cut -d ' ' -f 5
```
这个命令管道：
1.  `df -h`：查看磁盘使用情况。
2.  `grep '/dev/sda1'`：过滤出包含`/dev/sda1`的行。
3.  `tr -s ' '`：将行中连续的空格压缩成一个空格。
4.  `cut -d ' ' -f 5`：以空格为分隔符，提取第5列（使用率百分比）。

另一种更强大的工具是`awk`，它能更优雅地处理默认以空白字符（空格或制表符）分隔的字段。

**命令示例：**
```bash
df -h | grep '/dev/sda1' | awk '{print $5}'
```
`awk`默认以空白字符分割字段，`{print $5}`表示打印第5个字段。
