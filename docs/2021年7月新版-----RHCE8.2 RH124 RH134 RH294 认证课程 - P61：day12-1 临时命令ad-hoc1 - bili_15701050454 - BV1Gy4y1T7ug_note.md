# Ansible课程：Day12-1：临时命令 (ad-hoc) 入门

在本节课中，我们将要学习Ansible临时命令（ad-hoc）的基本概念和使用方法。临时命令是一种无需编写完整剧本（playbook），直接在命令行中执行简单任务的方式，非常适合快速测试和一次性操作。

## 回顾：Systemd管理容器服务

上一节我们介绍了如何使用Systemd管理容器服务。本节中我们来看看Ansible的临时命令。

![](img/348837acf91c4dd28f66bb076b721ebe_1.png)

![](img/348837acf91c4dd28f66bb076b721ebe_3.png)

管理容器服务有两种方式：基于`systemctl`和基于`podman`。

以下是基于`systemctl`管理服务的步骤：
1.  首先，创建一个容器。
2.  然后，以该容器为模板，生成一个Service文件。
3.  生成Service文件的命令是：`podman generate systemd --name <容器名> --files`。
4.  生成的文件默认在当前目录，名为`<容器名>.service`。
5.  执行命令前，需要切换到`/etc/systemd/system`目录。
6.  生成文件后，需要停止并删除原来的容器。
7.  最后，使用`systemctl enable --now <服务名>`命令设置服务开机自启并立即启动。

以下是基于普通用户管理服务的步骤：
1.  步骤与上述类似，首先创建容器并以之为模板生成Service文件。
2.  生成文件时，需要切换到用户家目录下的`.config/systemd/user/`路径。
3.  生成文件后，需要重新加载用户级Systemd配置：`systemctl --user daemon-reload`。
4.  最后，使用`systemctl --user enable --now <服务名>`命令启用并启动服务。

![](img/348837acf91c4dd28f66bb076b721ebe_5.png)

![](img/348837acf91c4dd28f66bb076b721ebe_7.png)

![](img/348837acf91c4dd28f66bb076b721ebe_9.png)

需要注意，普通用户创建的容器服务，不能以`root`或`sudo`身份启动，必须以该用户身份直接登录系统来管理。

## Ansible基础配置

![](img/348837acf91c4dd28f66bb076b721ebe_13.png)

接下来，我们回顾Ansible的基础配置。

![](img/348837acf91c4dd28f66bb076b721ebe_15.png)

Ansible的配置文件可以是默认的`/etc/ansible/ansible.cfg`。配置文件的读取优先级如下：
*   最高优先级：环境变量定义。
*   次高优先级：当前目录创建的`ansible.cfg`。
*   较低优先级：用户家目录下创建的`.ansible.cfg`。
*   最低优先级：默认的`/etc/ansible/ansible.cfg`。

配置文件中最常修改的两项是：
*   `inventory`：指定资产清单文件的路径。
*   `remote_user`：指定在工作节点上使用的用户账号。

关于权限提升（提权）的配置：
*   `become = yes`：是否启用提权。
*   `become_method = sudo`：提权方式，例如`sudo`。
*   `become_user = root`：提权到哪个用户，例如`root`。
*   `become_ask_pass = yes`：提权时是否需要输入密码。

![](img/348837acf91c4dd28f66bb076b721ebe_18.png)

![](img/348837acf91c4dd28f66bb076b721ebe_20.png)

如果使用`sudo`提权，且工作节点配置了需要密码，则必须配置`become_ask_pass`。

## 资产清单 (Inventory) 编写

![](img/348837acf91c4dd28f66bb076b721ebe_26.png)

![](img/348837acf91c4dd28f66bb076b721ebe_28.png)

![](img/348837acf91c4dd28f66bb076b721ebe_30.png)

资产清单文件定义了Ansible要管理的主机。

以下是资产清单的几种写法：
*   单主机写法：可以是IP地址或FQDN（完全限定域名）。
*   主机组写法：先写组名，下方缩进写组内主机的IP或FQDN。
*   嵌套组写法：先写父组名，后跟英文冒号`:`，然后写子组成员。

列出资产清单中的主机，可以使用命令：`ansible all --list-hosts`。
该命令后面可以指定具体的主机组、主机名或使用模式匹配。

## 临时命令 (ad-hoc) 入门

现在，我们开始学习Ansible临时命令。

临时命令允许我们通过命令行直接向目标主机推送并执行简单的任务，而无需编写完整的剧本。

![](img/348837acf91c4dd28f66bb076b721ebe_32.png)

![](img/348837acf91c4dd28f66bb076b721ebe_34.png)

临时命令的基本格式是：
```bash
ansible <主机模式> -m <模块名> -a “<模块参数>”
```
其中，`-i`选项可以指定资产清单文件。

![](img/348837acf91c4dd28f66bb076b721ebe_36.png)

例如，我们使用`ping`模块检查与所有节点的连通性：
```bash
ansible all -m ping
```
如果返回结果为绿色且显示`pong`，则表示连接正常。如果返回其他颜色或有非零的返回码（RC），则需要检查SSH免密登录、网络连接等问题。

![](img/348837acf91c4dd28f66bb076b721ebe_38.png)

## Ansible命令行工具详解

`ansible`命令行工具提供了一些常用选项：
*   `ansible --version`：查看Ansible版本。
*   `-a`：指定模块参数。
*   `-m`：指定模块，默认为`command`模块。
*   `-v`, `-vv`, `-vvv`：显示更详细的执行过程。
*   `--list-hosts`：列出匹配的主机列表。
*   `-C`：检查模式（Dry Run），只检查不实际执行。
*   `-u`：指定远程执行用户。
*   `--become-user`：指定提权到的用户。
*   `-b`或`--become`：启用提权。
*   `-K`：提权时询问密码。
*   `--ask-pass`：SSH连接时询问密码。

## 常用模块实践

![](img/348837acf91c4dd28f66bb076b721ebe_64.png)

### command 和 shell 模块
`command`模块是默认模块，用于在远程节点执行命令。但它不支持变量、操作符（如`|`, `>`, `<`）等。对于需要这些功能的场景，应使用`shell`模块。

使用`command`模块执行命令：
```bash
ansible servera -m command -a “echo $PS1”
```
使用`shell`模块执行命令：
```bash
ansible servera -m shell -a “echo $PS1”
```
注意命令中引号的使用，单引号和双引号可能会影响变量解析。

### script 模块
当命令过于复杂时，`shell`模块也可能失败。此时可以使用`script`模块。
`script`模块将控制节点本地的脚本复制到远程节点执行，执行完毕后会自动删除临时脚本文件。

例如，执行一个本地的复杂脚本：
```bash
ansible servera -m script -a “/path/to/local/script.sh”
```

![](img/348837acf91c4dd28f66bb076b721ebe_66.png)

### copy 模块
`copy`模块用于将文件从控制节点复制到受管节点，或直接在受管节点上生成文件。

![](img/348837acf91c4dd28f66bb076b721ebe_68.png)

![](img/348837acf91c4dd28f66bb076b721ebe_70.png)

以下是`copy`模块的关键参数：
*   `backup`：在覆盖前备份原文件。
*   `content`：直接用指定的内容生成文件，替代`src`参数。
*   `dest`：目标路径（必须为绝对路径）。
*   `force`：是否强制覆盖。`yes`为强制覆盖，`no`则只在目标不存在时复制。
*   `mode`：设置文件权限。
*   `src`：源文件路径。

![](img/348837acf91c4dd28f66bb076b721ebe_72.png)

![](img/348837acf91c4dd28f66bb076b721ebe_74.png)

使用`content`参数直接生成文件：
```bash
ansible servera -m copy -a “content=‘hello world’ dest=/tmp/test.txt”
```
使用`src`和`dest`参数复制文件：
```bash
ansible servera -m copy -a “src=/local/file.txt dest=/remote/path/file.txt”
```
如果目标文件已存在且`force=no`，则不会复制。`force`默认为`yes`。
