在本节课中，我们将继续学习如何解读 `ntpq -p` 命令的输出。我们将重点关注输出结果最左侧的符号，这些符号代表了每个远程NTP服务器的状态。理解这些符号对于诊断NTP同步问题和了解系统时间源的状态至关重要。

上一节我们介绍了 `ntpq -p` 命令输出的基本结构，本节中我们来看看这些状态符号的具体含义。

以下是 `ntpq -p` 命令输出中可能出现的状态符号及其解释：

![](img/71518267e162651f7ee6cedb89e15440_11.png)

*   **星号 (`*`)**：这是最重要的符号。它表示**当前系统正在使用此远程服务器作为时间源来同步时钟**。例如，如果你将某个服务器配置为首选服务器，它旁边通常会显示星号。在我们的例子中，系统正使用之前配置的中央NTP服务器进行同步。

![](img/71518267e162651f7ee6cedb89e15440_13.png)

![](img/71518267e162651f7ee6cedb89e15440_14.png)

![](img/71518267e162651f7ee6cedb89e15440_15.png)

![](img/71518267e162651f7ee6cedb89e15440_16.png)

*   **加号 (`+`)**：这个符号表示该服务器被包含在组合算法中，是一个**良好的候选服务器**。你可以将其视为**备用服务器**。如果当前正在使用的星号标记的服务器因故失效，系统将自动切换到加号标记的服务器。

![](img/71518267e162651f7ee6cedb89e15440_18.png)

![](img/71518267e162651f7ee6cedb89e15440_20.png)

![](img/71518267e162651f7ee6cedb89e15440_22.png)

*   **减号 (`-`)**：这个符号表示该服务器被**集群算法丢弃**。这意味着该服务器目前未被计入有效的时间源候选列表。

![](img/71518267e162651f7ee6cedb89e15440_24.png)

*   **井号 (`#`)**：这个符号表示该服务器是一个良好的远程服务器，可作为**替代的备用服务器**。通常，只有在配置了超过10个远程服务器时，这个符号才会显示。

![](img/71518267e162651f7ee6cedb89e15440_25.png)

![](img/71518267e162651f7ee6cedb89e15440_26.png)

*   **字母 `o`**：这个符号表示一个**脉冲每秒（PPS）对等体**。它通常与GPS时间源一起使用，任何提供PPS信号的时间源都会显示此符号。需要注意的是，`o` 符号和星号 (`*`) 符号**不会同时显示**。这就是为什么在我们的输出中看到了星号，但没有看到 `o`。

![](img/71518267e162651f7ee6cedb89e15440_28.png)

![](img/71518267e162651f7ee6cedb89e15440_29.png)

![](img/71518267e162651f7ee6cedb89e15440_30.png)

*   **字母 `x`**：这个符号表示该服务器被**交集算法丢弃**。

![](img/71518267e162651f7ee6cedb89e15440_31.png)

![](img/71518267e162651f7ee6cedb89e15440_32.png)

![](img/71518267e162651f7ee6cedb89e15440_34.png)

本节课中我们一起学习了 `ntpq -p` 命令输出中各种状态符号的含义，包括星号 (`*`)、加号 (`+`)、减号 (`-`)、井号 (`#`)、字母 `o` 和字母 `x`。理解这些符号能帮助你快速判断系统正在使用哪个时间源，以及哪些服务器是备用的或已被排除，这对于维护精确的系统时间非常有帮助。
