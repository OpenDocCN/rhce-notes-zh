# Unix&Linux快速入门超详细教程：P12：03-1-2 虚拟机环境下的磁盘模式设置 💾

在本节课中，我们将学习在虚拟机环境下设置磁盘模式。我们将了解几种不同的虚拟磁盘配置方式，包括它们的原理、区别以及适用场景，帮助你为实验环境选择合适的磁盘模式。

上一节我们介绍了物理磁盘的分区格式，本节中我们来看看在虚拟化环境中如何配置虚拟磁盘。

## 虚拟磁盘文件（VMDK）

![](img/b22842a0df3dfe736ed91315507827a4_1.png)

![](img/b22842a0df3dfe736ed91315507827a4_3.png)

在建立虚拟硬盘时，需要指定磁盘大小（例如100GB）。创建完成后，会生成一个扩展名为 `.vmdk` 的文件，它代表了虚拟机的磁盘。这个文件的大小通常与你指定的虚拟硬盘大小相关，但实际占用物理硬盘空间的方式取决于你选择的磁盘模式。

![](img/b22842a0df3dfe736ed91315507827a4_5.png)

## 虚拟磁盘的三种模式

![](img/b22842a0df3dfe736ed91315507827a4_7.png)

以下是虚拟机环境中常见的三种磁盘配置模式。

### 1. 厚置备延迟置零（Zeroed Thick）

![](img/b22842a0df3dfe736ed91315507827a4_9.png)

在这种模式下，创建虚拟硬盘时会立即分配全部指定大小的空间（例如400GB），并生成对应大小的VMDK文件。但是，这400GB空间并不会立即全部写入零（初始化），而是根据实际写入的数据量逐步进行初始化。

**核心概念**：`分配空间 = 指定大小`，但`初始化空间 = 已写入数据量`。

![](img/b22842a0df3dfe736ed91315507827a4_11.png)

举例来说，如果你有一个500GB的物理硬盘，创建了一个400GB的“厚置备延迟置零”虚拟硬盘。一开始，VMDK文件会显示为400GB，但物理硬盘可能只被占用了100GB（假设你写入了100GB数据）。剩余的300GB空间会在将来需要写入数据时再进行初始化和占用。

![](img/b22842a0df3dfe736ed91315507827a4_12.png)

### 2. 厚置备置零（Eager Zeroed Thick）

![](img/b22842a0df3dfe736ed91315507827a4_14.png)

![](img/b22842a0df3dfe736ed91315507827a4_16.png)

这种模式与上一种类似，也会立即分配全部指定大小的空间。不同之处在于，它在创建时就会将整个分配的空间全部写入零（完全初始化）。

![](img/b22842a0df3dfe736ed91315507827a4_18.png)

**核心概念**：`分配空间 = 初始化空间 = 指定大小`。

![](img/b22842a0df3dfe736ed91315507827a4_19.png)

继续上面的例子，创建400GB的“厚置备置零”虚拟硬盘后，物理硬盘会立即被占用400GB，并且这400GB空间已经完成了初始化。下次使用时可以直接写入数据，无需等待初始化过程。

**厚置备延迟置零与厚置备置零的区别**：两者都会预先占用物理硬盘的全部指定空间。区别在于初始化的时机：前者是“用时再初始化”，后者是“创建即初始化”。

![](img/b22842a0df3dfe736ed91315507827a4_21.png)

### 3. 精简置备（Thin Provision）

![](img/b22842a0df3dfe736ed91315507827a4_23.png)

![](img/b22842a0df3dfe736ed91315507827a4_25.png)

这是最节省物理空间的一种模式。创建虚拟硬盘时，系统并不会立即分配全部指定大小的空间，而是生成一个很小的VMDK文件。随着虚拟机内数据的不断增加，VMDK文件才会动态增长，直到达到指定的最大容量。

**核心概念**：`分配空间 = 已使用空间`，`最大可分配空间 = 指定大小`。

例如，在500GB物理硬盘上创建一个400GB的“精简置备”虚拟硬盘。如果你只使用了10GB数据，那么VMDK文件实际只占用了10GB的物理空间。随着使用量增加到20GB、50GB，占用的物理空间也会相应增加，但永远不会超过400GB。

![](img/b22842a0df3dfe736ed91315507827a4_27.png)

![](img/b22842a0df3dfe736ed91315507827a4_28.png)

## 模式对比与选择建议

![](img/b22842a0df3dfe736ed91315507827a4_30.png)

![](img/b22842a0df3dfe736ed91315507827a4_32.png)

在VMware等虚拟化软件创建磁盘时，通常会有一个选项，例如“立即分配所有磁盘空间”。勾选此选项相当于选择“厚置备”模式（置零或延迟置零），不勾选则通常对应“精简置备”模式。

![](img/b22842a0df3dfe736ed91315507827a4_34.png)

为了节省物理硬盘空间，在一般的应用测试环境中，**建议选择“精简置备”模式**。

![](img/b22842a0df3dfe736ed91315507827a4_35.png)

![](img/b22842a0df3dfe736ed91315507827a4_37.png)

然而，在某些对磁盘I/O性能要求极高的场景下，例如大数据处理、数据库服务器等，由于“精简置备”模式在动态增长时可能带来性能开销，**建议选择“厚置备置零”模式**，以换取更稳定、更直接的磁盘性能。

## 知识回顾与延伸

结合之前所学，在安装Linux系统时，根分区（`/`）、启动分区（`/boot`）和交换分区（`swap`）是必须的。在Linux中，频繁与磁盘进行数据交互的分区通常包括存放日志的目录（如`/var/log`）、安装软件的目录（如`/usr`， `/opt`）以及交换分区（`swap`）。操作系统的缓存（cache）和缓冲区（buffer）机制也会与磁盘频繁交互，其管理会涉及到交换空间。

![](img/b22842a0df3dfe736ed91315507827a4_39.png)

![](img/b22842a0df3dfe736ed91315507827a4_41.png)

本节课中我们一起学习了虚拟机环境下的三种磁盘模式：厚置备延迟置零、厚置备置零和精简置备。理解了它们的工作原理和区别后，你就可以根据实际需求（是优先考虑节省空间，还是追求极致性能）来为你的虚拟实验环境做出合适的选择。
