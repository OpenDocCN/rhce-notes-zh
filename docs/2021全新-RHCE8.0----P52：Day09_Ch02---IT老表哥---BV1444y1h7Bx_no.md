### 临时命令语法

![](img/6afce8a452fbf1d7dba9b86adb1e9832_8.png)

临时命令的基本语法结构如下：

![](img/6afce8a452fbf1d7dba9b86adb1e9832_11.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_12.png)

```
ansible <主机模式> -m <模块名> -a "<模块参数>"
```

![](img/6afce8a452fbf1d7dba9b86adb1e9832_14.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_16.png)

*   **`<主机模式>`**：指资产清单中定义的主机组、具体主机名或IP地址。
*   **`-m <模块名>`**：指定要使用的功能模块。
*   **`-a "<模块参数>"`**：传递给模块的参数。

例如，要在 `node1` 主机上创建一个用户，可以使用 `user` 模块，它相当于结合了 `useradd`、`usermod`、`userdel` 三条命令的功能。

![](img/6afce8a452fbf1d7dba9b86adb1e9832_18.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_20.png)

```
ansible node1 -m user -a "name=ts"
```

![](img/6afce8a452fbf1d7dba9b86adb1e9832_22.png)

### 幂等性 (Idempotency)

幂等性是Ansible的一个核心设计理念。它指的是：**无论一个操作执行多少次，最终系统的状态都与只执行一次的结果相同。**

![](img/6afce8a452fbf1d7dba9b86adb1e9832_30.png)

当我们使用Ansible模块执行任务时，它会先检查受管主机的当前状态。
*   如果当前状态已经符合目标状态（即“已经完成”），则Ansible不会进行任何更改，并返回 `"changed": false`。
*   如果当前状态不符合目标状态，Ansible则会执行必要的操作以达到目标状态，并返回 `"changed": true`。

例如，第一次创建用户 `ts` 时，状态会发生改变 (`changed: true`)。再次执行相同的创建命令时，因为用户已存在，状态符合目标，所以不会改变 (`changed: false`)。这避免了重复操作，确保了操作的安全性和一致性。

![](img/6afce8a452fbf1d7dba9b86adb1e9832_32.png)

## 文件管理模块详解

![](img/6afce8a452fbf1d7dba9b86adb1e9832_34.png)

Ansible内置了丰富的管理模块。接下来，我们将聚焦于文件操作相关的几个常用模块。

![](img/6afce8a452fbf1d7dba9b86adb1e9832_36.png)

### 1. copy 模块 📋

![](img/6afce8a452fbf1d7dba9b86adb1e9832_38.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_39.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_40.png)

`copy` 模块用于将文件从控制节点复制到受管主机。

以下是 `copy` 模块的常用参数：
*   **`src`**：控制节点上的源文件路径（绝对或相对路径）。如果是目录，需在路径末尾加 `/` 才会复制目录内容，否则会复制目录本身。此参数与 `content` 二选一。
*   **`content`**：直接在受管主机上生成文件内容，而不是复制文件。
*   **`dest`**：**（必需）** 受管主机上的目标路径。
*   **`owner`**：设置文件所有者。
*   **`group`**：设置文件所属组。
*   **`mode`**：设置文件权限（如 `0755`）。
*   **`seuser`/`selevel`/`setype`**：设置SELinux上下文。
*   **`backup`**：在覆盖前备份原文件，值为 `yes` 或 `no`。

**应用示例：**

![](img/6afce8a452fbf1d7dba9b86adb1e9832_42.png)

```bash
# 示例1：复制本地文件到远程主机的/tmp目录
ansible node1 -m copy -a "src=copy_file dest=/tmp/copy_file"

# 示例2：直接在远程主机上创建文件并写入内容
ansible node1 -m copy -a "content='ts copy\n' dest=/tmp/f1"

# 示例3：创建文件并指定所有者、权限和SELinux上下文
ansible node1 -m copy -a "dest=/tmp/f3 owner=student mode=0755 setype=httpd_sys_content_t"

![](img/6afce8a452fbf1d7dba9b86adb1e9832_44.png)

# 示例4：修改文件内容前进行备份
ansible node1 -m copy -a "src=copy_file dest=/tmp/f1 backup=yes"
```

![](img/6afce8a452fbf1d7dba9b86adb1e9832_46.png)

### 2. file 模块 📁

![](img/6afce8a452fbf1d7dba9b86adb1e9832_48.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_50.png)

`file` 模块用于在受管主机上设置文件、目录、链接的属性，或创建它们。

以下是 `file` 模块的常用参数：
*   **`path`**：**（必需）** 受管主机上的文件/目录路径。
*   **`state`**：目标状态。
    *   `directory`：创建目录。
    *   `touch`：创建空文件。
    *   `link`：创建软链接。
    *   `hard`：创建硬链接。
    *   `absent`：删除。
*   **`owner`**/**`group`**/**`mode`**：同 `copy` 模块。

**应用示例：**

```bash
# 示例1：创建目录
ansible node1 -m file -a "path=/tmp/dir1 state=directory owner=student group=student mode=0755 setype=httpd_sys_content_t"

# 示例2：创建空文件（使用 touch，而非 file）
ansible node1 -m file -a "path=/tmp/f1 state=touch owner=student group=student mode=0644"

![](img/6afce8a452fbf1d7dba9b86adb1e9832_52.png)

# 示例3：创建软链接和硬链接
ansible node1 -m file -a "src=/tmp/f1 dest=/tmp/f2 state=link"  # 软链接
ansible node1 -m file -a "src=/tmp/f1 dest=/tmp/f3 state=hard"  # 硬链接
```

### 3. fetch 模块 ⬇️

`fetch` 模块与 `copy` 相反，用于从受管主机拉取文件到控制节点。

以下是 `fetch` 模块的常用参数：
*   **`src`**：**（必需）** 受管主机上的源文件路径。
*   **`dest`**：**（必需）** 控制节点上保存文件的目录。
*   **`flat`**：如果设置为 `yes`，则文件直接保存在 `dest` 指定的路径下；否则会按 `主机名/文件路径` 的目录结构保存。

**应用示例：**

![](img/6afce8a452fbf1d7dba9b86adb1e9832_54.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_56.png)

```bash
# 示例1：拉取文件，默认按主机名创建目录结构
ansible node1 -m fetch -a "src=/tmp/copy_file dest=/home/student/ansible"

![](img/6afce8a452fbf1d7dba9b86adb1e9832_58.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_59.png)

# 示例2：拉取文件并直接保存到指定路径（不创建主机名目录）
ansible node1 -m fetch -a "src=/tmp/f1 dest=/home/student/ansible/f1 flat=yes"
```

![](img/6afce8a452fbf1d7dba9b86adb1e9832_61.png)

![](img/6afce8a452fbf1d7dba9b86adb1e9832_63.png)

### 4. lineinfile 模块 ✏️

`lineinfile` 模块用于确保某一行文本存在于（或不存在于）文件中，非常适合修改配置文件。它以“行”为单位进行处理。

以下是 `lineinfile` 模块的常用参数：
*   **`path`**：**（必需）** 要修改的文件路径。
*   **`regexp`**：使用正则表达式定位要修改的行。
*   **`line`**：替换或插入的最终行内容。
*   **`state`**：`present`（确保行存在，默认）或 `absent`（确保行不存在，即删除）。
*   **`insertafter`**/**`insertbefore`**：在匹配行的后面或前面插入新行。
*   **`backup`**：修改前备份。

**应用示例：**

```bash
# 首先，复制一个配置文件到/tmp目录进行操作，避免破坏系统文件
ansible node1 -m copy -a "src=/etc/selinux/config dest=/tmp/selinux"

# 示例1：替换以 SELINUX= 开头的行的内容
ansible node1 -m lineinfile -a "path=/tmp/selinux regexp='^SELINUX=' line='SELINUX=disabled'"

# 示例2：在以 SELINUX= 开头的行后面插入一行注释
ansible node1 -m lineinfile -a "path=/tmp/selinux regexp='^SELINUX=' insertafter=yes line='# disabled SELINUX'"

# 示例3：删除以 # disabled 开头的行
ansible node1 -m lineinfile -a "path=/tmp/selinux regexp='^# disabled' state=absent"

# 示例4：修改文件并创建备份
ansible node1 -m lineinfile -a "path=/tmp/selinux regexp='^SELINUX=' line='SELINUX=enforcing' backup=yes"
```

### 5. synchronize 模块 🔄

`synchronize` 模块是对 `rsync` 命令的封装，用于在控制节点和受管主机之间同步目录或文件，功能强大。

以下是 `synchronize` 模块的常用参数：
*   **`src`**：**（必需）** 控制节点上的源路径。
*   **`dest`**：**（必需）** 受管主机上的目标路径。
*   **`delete`**：同步后，删除目标端存在而源端不存在的文件（谨慎使用）。需结合 `recursive=yes`。
*   **`recursive`**：递归同步目录。

**应用示例：**

```bash
# 示例1：将本地目录同步到远程主机
ansible node1 -m synchronize -a "src=test_dir/ dest=/home/student/test_dir"

# 示例2：同步并删除目标端多余文件（危险操作，注意备份）
# ansible node1 -m synchronize -a "src=test_dir/ dest=/home/student/test_dir delete=yes recursive=yes"
```

请使用Ansible临时命令在 `node1` 主机上完成以下任务：

1.  创建一个名为 `/opt/dir_ansible` 的目录。

3.  设置该目录的权限为 `2770`（即设置了SGID位）。
4.  在该目录下创建一个指向 `/etc/hosts` 的符号链接，名为 `hosts_link`。
5.  在该目录下创建一个名为 `readme.txt` 的文件，内容为“This is a managed directory.”。

**提示**：你可能需要先创建 `devops` 用户和组。

## 总结 📝

本节课中我们一起学习了Ansible临时命令的基础语法和核心概念——幂等性。我们深入探讨了五个常用的文件管理模块：
*   **`copy`**：用于从控制节点向受管主机复制文件或直接生成内容。
*   **`file`**：用于管理文件、目录、链接的属性及创建它们。
*   **`fetch`**：用于从受管主机拉取文件到控制节点。
*   **`lineinfile`**：用于精确地修改文件中的特定行，是配置管理的利器。
*   **`synchronize`**：基于 `rsync` 的强大的文件同步工具。

掌握这些模块是进行Ansible自动化操作的基础。请务必通过随堂练习和查阅 `ansible-doc` 来巩固所学知识。下一节，我们将继续学习软件包管理、系统服务管理等其他常用模块。
