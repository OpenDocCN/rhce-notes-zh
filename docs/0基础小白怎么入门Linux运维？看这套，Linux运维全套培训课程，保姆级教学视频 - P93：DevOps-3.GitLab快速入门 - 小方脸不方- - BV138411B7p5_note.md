## 语言设置与界面熟悉

![](img/db45649d7c037c57bd33856454545151_9.png)

![](img/db45649d7c037c57bd33856454545151_11.png)

上一节我们完成了GitLab的安装与访问。本节中，我们首先来调整界面语言，使其更易于操作。

GitLab Web界面的默认语言是英文。我们需要将其调整为简体中文。

操作步骤如下：
1.  登录GitLab后，点击右上角用户头像，选择 **“Preferences”**（偏好设置）。
2.  在左侧菜单中找到并点击 **“Localization”**（本地化）。
3.  在“Language”（语言）下拉菜单中，选择 **“简体中文”**。系统提示当前翻译覆盖率为67%。
4.  点击页面底部的 **“Save changes”**（保存更改）按钮。
5.  保存后，刷新浏览器页面。

完成以上步骤后，界面将切换为简体中文，包括用户资料、账号、应用等菜单项都将以中文显示，方便后续操作。

## 修改管理员密码

在首次登录后，必须修改默认的root管理员密码。因为初始密码存放在一个临时文件中，该文件将在24小时或系统首次重新配置后自动删除。

以下是修改密码的步骤：
1.  点击右上角用户头像，选择 **“编辑个人资料”**。
2.  在左侧菜单中点击 **“密码”**。
3.  在“当前密码”输入框中，填入安装完成后显示的初始密码（可在安装服务器的 `/etc/gitlab/initial_root_password` 文件中查看）。
4.  在“新密码”和“确认新密码”输入框中，设置符合要求的新密码（GitLab要求密码长度至少为8个字符）。
5.  点击 **“保存密码”**。

例如，将密码修改为 `12345678`。修改成功后，下次登录请使用新密码。

## 创建群组与项目

在将本地代码推送到远程仓库前，我们需要在GitLab上创建用于存放代码的“项目”（即仓库）。为了更好地管理权限，通常先创建“群组”。

### 创建群组

群组（Group）用于集中管理项目和用户权限。可以将项目归属到某个群组，并将用户加入该群组，从而实现组内成员共享项目访问权限。

![](img/db45649d7c037c57bd33856454545151_13.png)

![](img/db45649d7c037c57bd33856454545151_15.png)

创建群组的步骤如下：
1.  点击左侧导航栏的 **“菜单”**，选择 **“管理”**。
2.  在管理区域，点击 **“新建群组”**。
3.  填写群组信息：
    *   **群组名称**：例如 `test`。
    *   **群组URL**：系统会自动生成，例如 `http://your-gitlab-address/test`。
    *   **群组描述**：（可选）例如“这是一个测试项目组”。
    *   **可见性级别**：选择 **“私有”**。这意味着只有被明确添加到该群组的成员才能查看组内的项目。
4.  其他选项如“允许创建项目”等保持默认，点击 **“创建群组”**。

### 创建项目

![](img/db45649d7c037c57bd33856454545151_17.png)

项目（Project）即代码仓库。我们将在这个仓库中存储代码。

![](img/db45649d7c037c57bd33856454545151_18.png)

以下是创建项目的步骤：
1.  在“管理”区域，点击 **“新建项目”**。
2.  选择 **“创建空白项目”**。
3.  填写项目信息：
    *   **项目名称**：例如 `my_lottery`。
    *   **项目URL**：选择将其归属到刚才创建的 `test` 群组下，URL会自动变为 `http://your-gitlab-address/test/my_lottery`。
    *   **项目描述**：（可选）例如“这是一个幸运转盘抽奖项目，目前处于测试阶段”。
    *   **可见性级别**：同样选择 **“私有”**。
4.  点击 **“新建项目”**。

![](img/db45649d7c037c57bd33856454545151_20.png)

项目创建成功后，页面会显示一个空的仓库，并给出命令行操作指引。

## 推送本地代码到GitLab仓库

![](img/db45649d7c037c57bd33856454545151_28.png)

现在，我们有了一个空的远程仓库。接下来，需要将本地已有的Git仓库中的代码推送上去。这模拟了开发人员提交代码到版本库的过程。

![](img/db45649d7c037c57bd33856454545151_30.png)

假设本地仓库位于 `/data` 目录，且已包含项目代码。

![](img/db45649d7c037c57bd33856454545151_31.png)

![](img/db45649d7c037c57bd33856454545151_33.png)

以下是推送代码的核心步骤与命令：

![](img/db45649d7c037c57bd33856454545151_35.png)

![](img/db45649d7c037c57bd33856454545151_37.png)

**1. 添加远程仓库地址到本地配置**

首先，需要将GitLab上的远程仓库地址关联到本地仓库。使用 `git remote add` 命令。

![](img/db45649d7c037c57bd33856454545151_39.png)

![](img/db45649d7c037c57bd33856454545151_41.png)

```bash
cd /data
git remote add origin http://your-gitlab-address/test/my_lottery.git
```
*   `origin` 是为这个远程仓库起的别名，可以自定义。
*   使用 `git remote -v` 命令可以查看已配置的远程仓库。

**2. 推送代码到远程仓库**

将本地仓库的所有内容推送到远程仓库的 `master` 分支。

```bash
git push -u origin master
```
执行此命令时，会弹出认证窗口，要求输入有权限访问该仓库的GitLab用户名和密码。这里我们使用 `root` 用户和之前修改的密码（例如 `12345678`）进行认证。

推送成功后，刷新GitLab项目页面，即可看到本地代码文件已全部出现在远程仓库中。

## Jenkins简介与安装

![](img/db45649d7c037c57bd33856454545151_48.png)

![](img/db45649d7c037c57bd33856454545151_50.png)

代码成功入库后，下一步是部署持续集成（CI）服务器来自动化拉取代码、构建和部署。这里我们介绍Jenkins。

![](img/db45649d7c037c57bd33856454545151_52.png)

### Jenkins概述

Jenkins是一款用Java开发的开源CI/CD工具，用于自动化各种任务，包括拉取项目、构建、测试、部署等。其核心功能通过丰富的插件体系实现，有超过1000个插件来支持各种需求。

### 安装Jenkins

![](img/db45649d7c037c57bd33856454545151_54.png)

![](img/db45649d7c037c57bd33856454545151_56.png)

以下是在CentOS/RHEL系统上安装Jenkins的简要步骤：

![](img/db45649d7c037c57bd33856454545151_58.png)

![](img/db45649d7c037c57bd33856454545151_60.png)

![](img/db45649d7c037c57bd33856454545151_61.png)

![](img/db45649d7c037c57bd33856454545151_63.png)

**1. 安装Java环境与字体包**
Jenkins需要Java运行环境。我们安装OpenJDK和一个字体配置包。
```bash
yum install -y fontconfig java-11-openjdk
```

![](img/db45649d7c037c57bd33856454545151_65.png)

**2. 安装Jenkins RPM包**
使用从清华大学镜像站下载的稳定版RPM包进行安装（例如 `jenkins-2.332.3-1.1.noarch.rpm`）。
```bash
rpm -ivh jenkins-2.332.3-1.1.noarch.rpm
```

**3. 启动并设置开机自启**
```bash
systemctl start jenkins
systemctl enable jenkins
```
Jenkins服务默认监听 `8080` 端口，启动后可通过 `http://<服务器IP>:8080` 访问。

**4. 初始化Jenkins**
首次访问Web界面时，需要输入初始管理员密码。该密码存储在Jenkins服务器的日志文件中：
```bash
cat /var/lib/jenkins/secrets/initialAdminPassword
```
将输出的密码填入Web页面，然后选择“安装推荐的插件”。系统会自动下载并安装常用插件，此过程可能需要一些时间。插件安装完成后，创建管理员用户即可完成初始化。
