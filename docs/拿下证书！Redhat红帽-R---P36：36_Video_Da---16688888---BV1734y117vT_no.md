# Red Hat RHCE 8.0 认证体系课程：RH134：Ch02a：计划任务 🕐

在本节课中，我们将要学习 Linux 系统中的计划任务管理。计划任务允许系统在指定的时间自动执行预定义的操作，是系统自动化运维的重要工具。我们将分别介绍一次性计划任务和周期性计划任务的配置与管理方法。

## 概述

![](img/93631d97467316fff361fb8be288e670_7.png)

计划任务分为两种主要类型：一次性计划任务和周期性计划任务。一次性计划任务在设定的时间点执行一次后即结束。周期性计划任务则会根据设定的时间条件循环往复地执行。

![](img/93631d97467316fff361fb8be288e670_9.png)

## 一次性计划任务

一次性计划任务通过 `atd` 后台守护进程来控制。它允许用户安排一个命令或脚本在未来某个特定时间点执行一次。

![](img/93631d97467316fff361fb8be288e670_11.png)

![](img/93631d97467316fff361fb8be288e670_12.png)

### 定义一次性计划任务

![](img/93631d97467316fff361fb8be288e670_14.png)

![](img/93631d97467316fff361fb8be288e670_16.png)

![](img/93631d97467316fff361fb8be288e670_18.png)

![](img/93631d97467316fff361fb8be288e670_19.png)

使用 `at` 命令可以定义一次性计划任务。其基本语法是 `at [时间]`。定义后，在 `at>` 提示符下输入要执行的命令，最后按 `Ctrl+D` 结束输入。

![](img/93631d97467316fff361fb8be288e670_20.png)

![](img/93631d97467316fff361fb8be288e670_22.png)

![](img/93631d97467316fff361fb8be288e670_24.png)

![](img/93631d97467316fff361fb8be288e670_26.png)

以下是定义一次性计划任务的步骤示例：

![](img/93631d97467316fff361fb8be288e670_28.png)

![](img/93631d97467316fff361fb8be288e670_30.png)

1.  查看当前时间，确定任务执行时间点。
    ```bash
    date
    ```
2.  使用 `at` 命令指定时间并输入要执行的命令。
    ```bash
    at 17:25 2020-07-12
    at> /usr/bin/touch /tmp/test_at_file
    at> <EOT> # 此处按 Ctrl+D 结束输入
    job 3 at Mon Jul 12 17:25:00 2020
    ```
3.  任务执行后，其定义文件会被自动删除。

### `at` 命令的其他时间格式

![](img/93631d97467316fff361fb8be288e670_32.png)

除了指定具体时间点，`at` 命令还支持相对时间格式。

*   `at now + 1 day`：一天后执行。
*   `at now + 3 minutes`：三分钟后执行。
*   `at now + 2 hours`：两小时后执行。

![](img/93631d97467316fff361fb8be288e670_34.png)

![](img/93631d97467316fff361fb8be288e670_36.png)

### 管理 `at` 任务

定义任务后，可以使用以下命令进行管理：

![](img/93631d97467316fff361fb8be288e670_38.png)

![](img/93631d97467316fff361fb8be288e670_39.png)

*   `atq` 或 `at -l`：列出当前用户所有等待执行的 `at` 任务。
*   `atrm [任务编号]` 或 `at -r [任务编号]`：删除指定的 `at` 任务。
*   `at -c [任务编号]`：查看指定 `at` 任务的具体内容。

![](img/93631d97467316fff361fb8be288e670_41.png)

![](img/93631d97467316fff361fb8be288e670_43.png)

上一节我们介绍了如何创建和管理一次性计划任务，本节中我们来看看功能更强大的周期性计划任务。

## 周期性计划任务

![](img/93631d97467316fff361fb8be288e670_45.png)

![](img/93631d97467316fff361fb8be288e670_47.png)

![](img/93631d97467316fff361fb8be288e670_49.png)

周期性计划任务由 `crond` 服务监管，允许任务按设定的周期（如每分钟、每小时、每天）重复执行。

![](img/93631d97467316fff361fb8be288e670_51.png)

### 用户级周期性任务

![](img/93631d97467316fff361fb8be288e670_53.png)

![](img/93631d97467316fff361fb8be288e670_55.png)

普通用户可以使用 `crontab` 命令管理自己的周期性任务。

![](img/93631d97467316fff361fb8be288e670_57.png)

![](img/93631d97467316fff361fb8be288e670_58.png)

![](img/93631d97467316fff361fb8be288e670_59.png)

*   `crontab -l`：列出当前用户的周期性任务。
*   `crontab -e`：编辑当前用户的周期性任务。
*   `crontab -u [用户名] -e`：**仅 `root` 用户可用**，编辑指定用户的周期性任务。

![](img/93631d97467316fff361fb8be288e670_61.png)

![](img/93631d97467316fff361fb8be288e670_63.png)

![](img/93631d97467316fff361fb8be288e670_65.png)

![](img/93631d97467316fff361fb8be288e670_67.png)

使用 `crontab -e` 命令会调用默认文本编辑器（如 `vim`）。任务条目的格式如下：

![](img/93631d97467316fff361fb8be288e670_68.png)

![](img/93631d97467316fff361fb8be288e670_70.png)

```
分 时 日 月 周 命令
```

![](img/93631d97467316fff361fb8be288e670_72.png)

![](img/93631d97467316fff361fb8be288e670_74.png)

![](img/93631d97467316fff361fb8be288e670_75.png)

![](img/93631d97467316fff361fb8be288e670_77.png)

以下是 `crontab` 条目的格式说明和示例：

![](img/93631d97467316fff361fb8be288e670_79.png)

![](img/93631d97467316fff361fb8be288e670_81.png)

*   `*`：代表任意值。例如，在“分”字段使用 `*` 表示每分钟。
*   `*/N`：代表每隔 N 个单位时间。例如，在“时”字段使用 `*/2` 表示每两小时。
*   `A,B,C`：代表在具体的 A、B、C 时间点。例如，在“日”字段使用 `10,20,30` 表示每月 10号、20号、30号。
*   `0-6`：代表一个范围。在“周”字段，`0` 和 `7` 都代表星期日，`1` 代表星期一，以此类推。

示例条目：
*   `0 1 * * * /usr/bin/systemctl restart httpd`：每天凌晨 1 点重启 httpd 服务。
*   `0 */2 * * * /usr/bin/systemctl restart httpd`：每两小时的 0 分重启 httpd 服务。
*   `0 8 10,20,30 * * /usr/bin/wall “Monthly reminder”`：每月 10号、20号、30号的早上 8 点向所有用户发送广播信息。

用户使用 `crontab -e` 编辑的任务，实际保存在 `/var/spool/cron/` 目录下以用户名命名的文件中。

### 系统级周期性任务

![](img/93631d97467316fff361fb8be288e670_89.png)

![](img/93631d97467316fff361fb8be288e670_91.png)

![](img/93631d97467316fff361fb8be288e670_93.png)

系统级的周期性任务通常不直接编辑 `/etc/crontab` 文件，而是通过以下两种方式定义：

![](img/93631d97467316fff361fb8be288e670_95.png)

**方法一：在预定义目录中放置脚本**
系统提供了按周期执行的目录，将可执行脚本放入相应目录即可。
*   `/etc/cron.hourly/`：每小时执行。
*   `/etc/cron.daily/`：每天执行。
*   `/etc/cron.weekly/`：每周执行。
*   `/etc/cron.monthly/`：每月执行。
例如，系统的日志轮转任务 `logrotate` 就是通过放置在 `/etc/cron.daily/` 目录下的脚本来实现的。

**方法二：在 `/etc/cron.d/` 目录中创建配置文件**
在此目录下创建的配置文件格式与 `/etc/crontab` 类似，但可以自定义文件名。例如，创建一个名为 `myweeklyjob` 的文件：
```
# 分 时 日 月 周 用户 命令
0 3 * * 0 root /usr/local/bin/backup.sh
```
这表示每周日（`0`）凌晨 3 点，以 `root` 用户身份执行 `/usr/local/bin/backup.sh` 脚本。

### 任务执行机制与延迟

![](img/93631d97467316fff361fb8be288e670_97.png)

系统周期性任务的执行参数（如延迟时间）在 `/etc/anacrontab` 文件中定义。例如，`daily` 任务最多可能延迟 5 分钟执行，`weekly` 任务可能延迟 25 分钟。这种随机延迟机制有助于错开高峰，避免所有系统任务同时启动，并优先保证用户任务的执行。

![](img/93631d97467316fff361fb8be288e670_99.png)

## 总结

本节课中我们一起学习了 Linux 计划任务的核心管理方法。我们掌握了如何使用 `at` 和 `atd` 服务来安排**一次性计划任务**，包括任务的创建、查看和删除。接着，我们深入探讨了使用 `crontab` 命令和 `crond` 服务来配置**周期性计划任务**，涵盖了用户级任务的编辑格式（`分 时 日 月 周 命令`）以及系统级任务的两种定义方式（通过 `cron.[hourly|daily|weekly|monthly]` 目录或 `/etc/cron.d/` 配置文件）。理解并熟练运用这些工具，是实现系统自动化运维的基础。
