## 用户密码设置

![](img/d547e7218808e938a2edfb7c75cdffaa_8.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_10.png)

为用户设置密码是账户管理的基础操作。在Linux系统中，我们使用 `passwd` 命令来完成这项工作。

### `passwd` 命令基本用法

`passwd` 命令的格式非常简单。如果不指定用户名，则默认修改当前登录用户的密码。

![](img/d547e7218808e938a2edfb7c75cdffaa_12.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_14.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_16.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_18.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_19.png)

**命令格式：**
```bash
passwd [用户名]
```

*   **修改自己的密码**：直接输入 `passwd` 命令，系统会提示输入当前密码（root用户除外）和新密码。
*   **修改其他用户的密码**：只有root用户有此权限，命令后需跟上目标用户名，例如 `passwd user1`。

### 密码规范与注意事项

系统对密码有严格的安全规范，普通用户修改密码时必须遵守，而root用户则不受此限制。

以下是主要的密码规范：
*   **密码长度**：不能少于8个字符。
*   **密码复杂度**：需包含数字、字母（区分大小写）以及特殊字符。
*   **密码历史**：新密码不能与旧密码过于相似。

如果普通用户设置的密码不符合规范，系统会拒绝修改。root用户则可以直接强制设置任何密码，包括简单密码。

### 文件结构与权限

![](img/d547e7218808e938a2edfb7c75cdffaa_31.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_32.png)

`/etc/shadow` 文件权限特殊，只有root用户能够读取和修改，这极大地增强了密码的安全性。

![](img/d547e7218808e938a2edfb7c75cdffaa_34.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_36.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_38.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_40.png)

**查看文件权限：**
```bash
ls -l /etc/shadow
```
输出结果类似 `---------- 1 root root`，表示所有用户（包括root）都无直接读写权限，但root可以通过特权工具访问。

### 字段含义解读

与 `/etc/passwd` 类似，`/etc/shadow` 文件的每一行代表一个用户的密码信息，以冒号 `:` 分隔为9个字段。

我们以用户 `user1` 的一行信息为例进行解读：
```
user1:$6$...加密字符串...:19077:0:99999:7:::
```

以下是每个字段的详细说明：

1.  **用户名**：用户的登录名。
2.  **加密密码**：用户密码经过加密算法（如SHA512）处理后的字符串。如果该字段为 `*` 或 `!!`，则表示账户被锁定或未设置密码。
3.  **上次修改密码的日期**：表示从1970年1月1日（Unix纪元）到上次修改密码之日所经过的天数。例如 `19077`。
4.  **密码最短有效天数**：密码修改后，必须经过多少天才可以再次修改。`0` 表示随时可以修改。
5.  **密码最长有效天数**：密码的有效期，超过此天数后密码即过期。默认 `99999` 天意味着几乎永不过期。
6.  **密码过期前警告天数**：在密码即将过期前多少天开始向用户发出警告。例如 `7` 表示提前7天警告。
7.  **密码过期后宽限天数**：密码过期后，用户仍然可以在此宽限期内登录，但登录后会强制要求修改密码。宽限期过后账户将被锁定。
8.  **账户失效日期**：指定一个绝对日期（同样以1970年以来的天数表示），在此日期之后，账户将无法使用。
9.  **保留字段**：目前未使用，留作未来扩展。

### 密码策略管理实践

了解字段含义后，我们可以通过修改 `/etc/shadow` 文件或使用专用命令来实施密码策略。

**常用管理命令：**

![](img/d547e7218808e938a2edfb7c75cdffaa_42.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_44.png)

*   **强制用户首次登录时修改密码**：此操作通过将“上次修改密码日期”（第3字段）设置为 `0` 来实现。
    ```bash
    chage -d 0 username
    ```
    用户首次使用初始密码登录时，系统会强制其立即设置新密码。

![](img/d547e7218808e938a2edfb7c75cdffaa_46.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_48.png)

*   **设置密码有效期**：例如，要求用户每60天必须更改一次密码。
    ```bash
    chage -M 60 username
    ```
    这修改的是第5个字段。

![](img/d547e7218808e938a2edfb7c75cdffaa_50.png)

*   **查看用户密码信息**：
    ```bash
    passwd -S username
    或
    chage -l username
    ```
    这些命令会清晰地显示上述各个字段的信息。

![](img/d547e7218808e938a2edfb7c75cdffaa_52.png)

*   **锁定与解锁用户密码**：
    ```bash
    passwd -l username  # 锁定用户，禁止其登录
    passwd -u username  # 解锁用户
    ```

*   **非交互式设置密码（用于脚本）**：
    ```bash
    echo "NewPassword" | passwd --stdin username
    ```
    **注意**：直接在命令行中明文传递密码存在安全风险，仅建议在自动化脚本中使用。

## 用户身份切换 `su`

![](img/d547e7218808e938a2edfb7c75cdffaa_60.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_62.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_63.png)

在管理多用户系统时，有时需要临时切换至其他用户身份执行操作，这时可以使用 `su` 命令。

**命令格式与用法：**
```bash
su [选项] [用户名]
```

*   **切换用户**：`su username` 切换到指定用户，但环境变量和工作目录可能保持不变。
*   **完全切换（推荐）**：`su - username` 或 `su -l username`。这会模拟一次完整的登录，加载目标用户的环境配置并切换到其家目录。
*   **退出切换**：执行 `exit` 命令或按 `Ctrl+D` 即可返回原用户身份。

![](img/d547e7218808e938a2edfb7c75cdffaa_65.png)

![](img/d547e7218808e938a2edfb7c75cdffaa_67.png)

**重要提示**：root用户切换到任何其他用户都**不需要输入密码**。普通用户使用 `su` 命令切换身份时，则需要输入目标用户的密码。

![](img/d547e7218808e938a2edfb7c75cdffaa_69.png)
