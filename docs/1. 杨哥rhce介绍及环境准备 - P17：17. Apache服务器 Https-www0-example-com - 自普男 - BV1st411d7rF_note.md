在本节课中，我们将学习如何将一个已存在的Apache网站配置为支持HTTPS加密访问。我们将使用私有证书颁发机构签发的证书，实现通过443端口的安全访问。

![](img/4f49c0e539ce2c7e9327a5be5a622213_6.png)

上一节我们介绍了如何配置一个基本的HTTP网站，本节中我们来看看如何为其增加HTTPS加密支持。

## 核心概念与准备工作

![](img/4f49c0e539ce2c7e9327a5be5a622213_8.png)

配置HTTPS需要几个关键组件：
*   **SSL/TLS模块**：Apache服务器必须安装并启用`mod_ssl`模块以支持HTTPS。
*   **服务器证书**：由证书颁发机构签发的文件，用于向客户端证明服务器身份。其路径由 `SSLCertificateFile` 指令指定。
*   **私钥文件**：与证书配对的密钥文件，用于建立加密连接。其路径由 `SSLCertificateKeyFile` 指令指定。
*   **CA证书**：签发服务器证书的机构的根证书。客户端需要信任此证书，才能验证服务器证书的有效性。

## 配置步骤

![](img/4f49c0e539ce2c7e9327a5be5a622213_10.png)

以下是配置Apache服务器支持HTTPS的具体步骤。

![](img/4f49c0e539ce2c7e9327a5be5a622213_12.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_14.png)

### 1. 下载证书与私钥

![](img/4f49c0e539ce2c7e9327a5be5a622213_16.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_18.png)

首先，需要在Apache服务器上获取由CA签发的网站证书和对应的私钥文件。通常可以通过`wget`命令从指定地址下载。

![](img/4f49c0e539ce2c7e9327a5be5a622213_20.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_22.png)

```bash
# 进入Apache配置目录（或其他自定义目录）
cd /etc/httpd
# 下载服务器证书
wget http://提供的URL地址/www0.crt
# 下载服务器私钥
wget http://提供的URL地址/www0.key
```

![](img/4f49c0e539ce2c7e9327a5be5a622213_24.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_25.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_26.png)

### 2. 编辑虚拟主机配置文件

![](img/4f49c0e539ce2c7e9327a5be5a622213_28.png)

接下来，需要为HTTPS访问创建一个新的虚拟主机配置，或修改现有配置。配置文件通常位于 `/etc/httpd/conf.d/` 目录下。

![](img/4f49c0e539ce2c7e9327a5be5a622213_30.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_31.png)

以下是配置示例，核心是启用SSL引擎并指定证书和私钥的路径：

![](img/4f49c0e539ce2c7e9327a5be5a622213_32.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_34.png)

```apache
<VirtualHost *:443>
    # 服务器名称（与HTTP版本一致）
    ServerName www0.example.com
    # 网站文档根目录（与HTTP版本一致）
    DocumentRoot /var/www/html/www0

    # 启用SSL引擎
    SSLEngine on
    # 指定SSL证书文件路径
    SSLCertificateFile /etc/httpd/www0.crt
    # 指定SSL私钥文件路径
    SSLCertificateKeyFile /etc/httpd/www0.key

    # 其他可选SSL配置（如协议版本）
    # SSLProtocol all -SSLv2 -SSLv3
</VirtualHost>
```

![](img/4f49c0e539ce2c7e9327a5be5a622213_36.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_38.png)

**关键指令说明**：
*   `<VirtualHost *:443>`：定义监听443端口（HTTPS默认端口）的虚拟主机。
*   `SSLEngine on`：这是启用SSL/TLS支持的必要指令。
*   `SSLCertificateFile` 和 `SSLCertificateKeyFile`：必须正确指向第一步下载的文件。

![](img/4f49c0e539ce2c7e9327a5be5a622213_39.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_41.png)

### 3. 重启Apache服务

配置完成后，需要重启Apache服务以使更改生效。务必检查配置语法是否正确。

```bash
# 测试配置文件语法
apachectl configtest
# 重启Apache服务
systemctl restart httpd
# 确认443端口已监听
ss -tlpn | grep -E ‘:(443|80)’
```

## 客户端访问测试

![](img/4f49c0e539ce2c7e9327a5be5a622213_43.png)

由于使用的是私有CA签发的证书，客户端浏览器默认不信任它，直接访问会显示安全警告。需要进行额外配置。

![](img/4f49c0e539ce2c7e9327a5be5a622213_45.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_47.png)

以下是让客户端浏览器信任该网站证书的步骤。

![](img/4f49c0e539ce2c7e9327a5be5a622213_49.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_51.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_53.png)

### 1. 下载并导入CA根证书

![](img/4f49c0e539ce2c7e9327a5be5a622213_55.png)

在客户端机器上，下载签发服务器证书的CA根证书。

![](img/4f49c0e539ce2c7e9327a5be5a622213_56.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_57.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_58.png)

1.  从指定地址下载CA根证书文件（例如 `ca.crt`）。
2.  打开客户端浏览器（以Firefox为例），进入设置。
3.  搜索“证书”或“隐私与安全”，找到“查看证书”选项。
4.  在“证书管理器”中，选择“证书机构”选项卡。
5.  点击“导入”按钮，选择下载的 `ca.crt` 文件。
6.  在导入对话框中，勾选“信任此CA以标识网站”等选项，完成导入。

![](img/4f49c0e539ce2c7e9327a5be5a622213_60.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_61.png)

### 2. 访问HTTPS网站

![](img/4f49c0e539ce2c7e9327a5be5a622213_63.png)

完成CA证书导入后，在客户端浏览器地址栏输入 `https://www0.example.com` 进行访问。此时浏览器地址栏应显示安全锁标志，点击可以查看证书详情，确认是由已导入的CA签发，表示HTTPS连接已成功建立且受信任。

![](img/4f49c0e539ce2c7e9327a5be5a622213_65.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_66.png)

![](img/4f49c0e539ce2c7e9327a5be5a622213_68.png)

## 要点总结

![](img/4f49c0e539ce2c7e9327a5be5a622213_70.png)

本节课中我们一起学习了为Apache网站配置HTTPS的全过程。主要步骤和要点如下：

![](img/4f49c0e539ce2c7e9327a5be5a622213_72.png)

1.  **服务端配置**：确保`mod_ssl`模块已安装。下载服务器证书和私钥，并在虚拟主机配置中通过 `SSLEngine on`、`SSLCertificateFile` 和 `SSLCertificateKeyFile` 指令正确引用它们。
2.  **客户端配置**：对于私有CA环境，必须在客户端浏览器中导入CA的根证书，否则会遇到证书不受信任的警告。
3.  **配置分离**：HTTPS配置通常独立于HTTP配置，使用不同的端口和SSL相关指令，但可以共享 `ServerName` 和 `DocumentRoot` 等基本设置。

![](img/4f49c0e539ce2c7e9327a5be5a622213_74.png)

通过以上步骤，你可以成功地将一个HTTP网站升级为同时支持HTTP和HTTPS访问的安全网站。
