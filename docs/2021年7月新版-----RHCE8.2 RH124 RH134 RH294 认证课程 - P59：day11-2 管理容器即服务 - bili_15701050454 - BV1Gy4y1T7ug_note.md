*   `podman run`：运行容器。
    *   `-d`：后台运行。
    *   `-t`：提供终端。
    *   `-p`：端口映射。
    *   `-e`：定义环境变量。
    *   `-v`：将物理机目录挂载到容器。
*   `podman search`：搜索镜像。
*   `podman pull`：拉取镜像。
*   `podman push`：推送镜像。
*   `podman rm`：删除容器。
    *   `-a`：删除所有容器。
*   `podman stop`：停止运行容器。
    *   `-a`：停止所有容器。
*   `podman start`：继续运行容器。
*   `podman images`：查看当前拥有的镜像。
*   `podman rmi`：删除镜像。
*   `podman exec`：进入容器。
*   `podman ps`：查看容器状态。
    *   `-a`：查看所有容器状态。

## 下午内容：管理容器即服务

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_52.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_54.png)

上午我们学习了容器的基本操作，如端口映射、环境变量和后台运行。本节中，我们来看看如何将容器作为系统服务来管理。

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_56.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_58.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_60.png)

与 Docker 不同，Podman 没有守护进程。但我们可以通过生成 systemd 单元文件，将容器配置为服务，从而实现开机自启和便捷的服务管理。

**重要前提：**
此功能要求 Podman 版本在 1.9 以上。RHEL 8.2 默认提供的版本是 1.6.4，因此需要在实验环境中操作。实验环境会通过 `lab containers-registry` 命令配置私有仓库并安装 Podman 1.9.3。

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_62.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_63.png)

### Root 用户配置容器服务

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_65.png)

以下是配置步骤：

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_67.png)

1.  **运行一个示例容器：**
    我们首先运行一个带有端口映射和卷挂载的 Web 服务器容器。
    ```bash
    podman run -d --name myweb -p 8080:80 -v /app/data:/var/www/html:Z docker.io/library/httpd
    ```
    此命令会运行一个名为 `myweb` 的 Apache 容器，将宿主机的 8080 端口映射到容器的 80 端口，并将 `/app/data` 目录挂载到容器的网页根目录。

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_69.png)

2.  **生成 systemd 单元文件：**
    使用 `podman generate systemd` 命令，以正在运行的容器为模板生成服务文件。
    ```bash
    cd /etc/systemd/system
    podman generate systemd --name myweb --files --new
    ```
    *   `--name`：指定现有容器的名称。
    *   `--files`：生成单元文件。
    *   `--new`：此选项至关重要。它表示停止服务时会删除容器，启动服务时会创建新容器。如果不加此选项，服务控制可能无法正常工作。

    命令执行后，会在当前目录生成一个名为 `container-myweb.service` 的文件。

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_71.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_73.png)

3.  **启用并启动服务：**
    重新加载 systemd 配置，启用服务并立即启动它。
    ```bash
    systemctl daemon-reload
    systemctl enable --now container-myweb.service
    ```
    现在，容器将作为系统服务运行。使用 `systemctl stop container-myweb.service` 停止服务时，容器会被删除；再次启动服务时，会创建一个新的容器实例。

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_77.png)

### 普通用户配置容器服务

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_83.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_85.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_87.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_89.png)

对于普通用户，操作类似，但配置文件的存放位置和部分命令不同。**关键点：必须使用该用户身份直接登录系统或 SSH，不能使用 `su` 或 `sudo` 切换。**

1.  **创建必要的目录结构：**
    在用户家目录下创建 systemd 用户单元文件的存放路径。
    ```bash
    mkdir -p ~/.config/systemd/user
    ```

2.  **运行容器并生成单元文件：**
    以普通用户身份运行容器并生成服务文件。
    ```bash
    podman run -d --name myweb -p 8081:80 -v ~/html:/var/www/html:Z docker.io/library/httpd
    cd ~/.config/systemd/user
    podman generate systemd --name myweb --files --new
    ```

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_91.png)

3.  **管理用户服务：**
    普通用户的服务管理命令需要添加 `--user` 参数。
    ```bash
    systemctl --user daemon-reload
    systemctl --user enable --now container-myweb.service
    ```

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_93.png)

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_95.png)

4.  **启用用户级服务开机自启：**
    普通用户的服务需要额外授权才能实现开机自启。
    ```bash
    loginctl enable-linger $USER
    ```
    执行此命令后，用户的服务便可在系统启动时自动运行。

## 总结

![](img/16ff3b35aa4ef63620c4c3d77032cb7e_115.png)

本节课中我们一起学习了如何将 Podman 容器管理为系统服务。
*   我们回顾了 Podman 的核心命令。
*   学习了为 **root 用户**配置容器服务的方法，包括生成 systemd 单元文件、启用和启动服务。
*   学习了为 **普通用户**配置容器服务的步骤，特别注意了目录路径、命令差异和启用开机自启所需的 `loginctl enable-linger` 授权。
