## 界面语言设置 🌐

![](img/0e34fa49c10f720fba111c731d40d9de_9.png)

![](img/0e34fa49c10f720fba111c731d40d9de_11.png)

上一节我们完成了GitLab的安装，本节我们首先来调整界面语言。GitLab的默认界面语言是英文。我们需要将其调整为简体中文。

操作步骤如下：
1.  登录GitLab后，点击右上角用户头像，进入 **“偏好设置”**。
2.  在设置页面中找到 **“语言”** 选项。
3.  将语言从“English”更改为 **“简体中文 (zh_CN)”**。
4.  点击 **“保存更改”** 按钮。
5.  刷新浏览器页面，界面将切换为简体中文。

> **注意**：当前的简体中文翻译覆盖度约为67%，并非100%完整翻译，但核心功能已汉化，便于理解。

完成设置后，界面中的“用户资料”、“账号”、“应用”等内容将变为中文，更加直观易懂。

## 修改管理员密码 🔐

在首次登录后，必须修改默认的root管理员密码。因为初始密码存储在文件中，并且该文件会在24小时后或系统首次重新配置时被自动删除。

以下是修改密码的步骤：
1.  点击右上角用户头像，进入 **“偏好设置”**。
2.  点击左侧菜单中的 **“密码”** 选项。
3.  在“当前密码”栏中，输入安装时生成的初始密码（可在安装提示或特定文件中找到）。
4.  在“新密码”和“确认密码”栏中，输入符合要求的新密码。
    *   **要求**：密码长度至少为8个字符。
    *   **示例**：`12345678`
5.  点击 **“保存”** 按钮。
6.  使用新密码 `root` / `12345678` 重新登录系统。

## 创建群组与项目 📁

现在，我们可以开始创建用于管理代码的仓库。在GitLab中，通常先创建群组，再在群组下创建项目（即代码仓库）。

### 理解核心概念

![](img/0e34fa49c10f720fba111c731d40d9de_13.png)

在开始操作前，需要理解三个核心概念：
*   **项目**：即代码仓库，用于存储代码。
*   **用户**：可以登录GitLab平台的账号。
*   **群组**：用于将项目和用户进行统一管理的容器。一个项目可以属于某个群组，一个用户也可以加入某个群组，从而获得访问该群组内所有项目的权限。

![](img/0e34fa49c10f720fba111c731d40d9de_15.png)

### 创建群组

![](img/0e34fa49c10f720fba111c731d40d9de_17.png)

![](img/0e34fa49c10f720fba111c731d40d9de_18.png)

首先，我们来创建一个测试群组。
1.  点击左侧边栏的 **“菜单”** -> **“管理员”**。
2.  在管理区域，点击 **“新建群组”**。
3.  填写群组信息：
    *   **群组名称**：例如 `test`
    *   **群组URL**：系统会自动生成，例如 `http://your-gitlab-address/test`
    *   **群组描述**：（可选）例如“这是一个测试项目的组”
    *   **可见性级别**：选择 **“私有”**
        *   **私有**：仅群组成员可见。
        *   **内部**：所有登录用户可见。
        *   **公开**：互联网上任何人无需登录即可访问（适用于开源项目）。
4.  其他选项保持默认，点击 **“创建群组”**。

![](img/0e34fa49c10f720fba111c731d40d9de_20.png)

### 创建项目

群组创建完成后，接下来在该群组下创建一个项目。
1.  在管理区域，点击 **“新建项目”**。
2.  选择 **“创建空白项目”**。
3.  填写项目信息：
    *   **项目名称**：例如 `my_lottery`
    *   **项目URL**：选择将其放置在刚才创建的 `test` 群组下，URL会变为 `http://your-gitlab-address/test/my_lottery`
    *   **项目描述**：（可选）例如“这是一个幸运转盘抽奖项目，目前处于测试阶段”
    *   **可见性级别**：同样选择 **“私有”**。
4.  点击 **“新建项目”**。

![](img/0e34fa49c10f720fba111c731d40d9de_24.png)

![](img/0e34fa49c10f720fba111c731d40d9de_26.png)

项目创建成功后，你会看到一个空的代码仓库，并附有如何将本地代码推送至此仓库的命令指引。

## 推送本地代码到GitLab 🚚

![](img/0e34fa49c10f720fba111c731d40d9de_30.png)

![](img/0e34fa49c10f720fba111c731d40d9de_31.png)

![](img/0e34fa49c10f720fba111c731d40d9de_33.png)

我们已经有了本地的Git仓库（例如在 `/data` 目录下），现在需要将其推送到刚刚创建的GitLab远程仓库。

![](img/0e34fa49c10f720fba111c731d40d9de_35.png)

![](img/0e34fa49c10f720fba111c731d40d9de_37.png)

以下是操作步骤：

![](img/0e34fa49c10f720fba111c731d40d9de_39.png)

### 1. 添加远程仓库地址到本地

![](img/0e34fa49c10f720fba111c731d40d9de_41.png)

在本地仓库的根目录（例如 `/data`）下执行命令，将GitLab上的远程仓库添加为本地仓库的一个远程源。

```bash
git remote add origin http://your-gitlab-address/test/my_lottery.git
```
*   `origin` 是为这个远程仓库起的别名，可以自定义。

使用以下命令验证是否添加成功：
```bash
git remote -v
```
该命令会列出所有已配置的远程仓库及其地址。

### 2. 推送代码到远程仓库

![](img/0e34fa49c10f720fba111c731d40d9de_43.png)

![](img/0e34fa49c10f720fba111c731d40d9de_45.png)

![](img/0e34fa49c10f720fba111c731d40d9de_47.png)

![](img/0e34fa49c10f720fba111c731d40d9de_48.png)

执行推送命令，将本地 `master` 分支的所有内容推送到远程 `origin` 仓库。

![](img/0e34fa49c10f720fba111c731d40d9de_50.png)

![](img/0e34fa49c10f720fba111c731d40d9de_52.png)

```bash
git push -u origin master
```
*   `-u`：建立本地 `master` 分支与远程 `origin/master` 分支的追踪关系，后续推送可简化为 `git push`。
*   `origin`：远程仓库别名。
*   `master`：要推送的本地分支名。

**首次推送会触发身份验证**：
*   命令行会提示输入用户名 (`root`) 和密码 (`12345678`)。
*   验证成功后，代码即开始上传。

![](img/0e34fa49c10f720fba111c731d40d9de_54.png)

### 3. 在GitLab中验证

![](img/0e34fa49c10f720fba111c731d40d9de_56.png)

![](img/0e34fa49c10f720fba111c731d40d9de_58.png)

![](img/0e34fa49c10f720fba111c731d40d9de_60.png)

![](img/0e34fa49c10f720fba111c731d40d9de_61.png)

推送完成后，刷新GitLab中 `my_lottery` 项目的页面。你将看到本地 `/data` 目录下的所有文件都已成功出现在项目的文件列表中。

![](img/0e34fa49c10f720fba111c731d40d9de_63.png)

![](img/0e34fa49c10f720fba111c731d40d9de_65.png)

至此，我们模拟了开发人员将本地代码提交到团队远程代码仓库的完整流程。
