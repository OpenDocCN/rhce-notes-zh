## 系统信息与状态查看命令

上一节我们学习了系统安装和基础命令结构，本节中我们来看看如何查看和管理系统的基本信息。

### echo命令：输出文本或变量
`echo`命令用于将指定的字符串或变量值输出到终端屏幕。

**基本语法：**
```bash
echo [字符串]
```
例如，输入 `echo Hello World` 会将“Hello World”输出到屏幕。

它也可以用于输出变量的值。变量是存储数据的名称，使用美元符号`$`进行引用。
```bash
echo $PATH
```
此命令会输出`PATH`变量中存储的值（通常是系统的可执行文件路径）。

**注意：** 命令、参数和对象之间必须用空格分隔。Linux系统严格区分大小写。

### date命令：显示与设置系统时间
`date`命令用于显示或设置系统的日期和时间。

**基本语法：**
```bash
date [选项]... [+格式]
```
不加任何参数时，`date`会显示完整的系统时间。
```bash
date
```
输出示例：`Sun Mar 10 19:11:54 CST 2019`

![](img/22c220998a1a15db2d391239d6e8d382_14.png)

可以使用`+`号加格式参数来定制输出内容。这是`date`命令参数格式的一个特例。
```bash
date +%Y
```
此命令仅输出当前年份，如`2019`。

![](img/22c220998a1a15db2d391239d6e8d382_16.png)

![](img/22c220998a1a15db2d391239d6e8d382_18.png)

**常用格式参数：**
*   `%Y`：四位年份
*   `%m`：月份
*   `%d`：日期
*   `%H`：小时
*   `%M`：分钟
*   `%S`：秒

组合示例：
```bash
date +%Y-%m-%d
date +%H:%M:%S
```

**重要提示：**
1.  大多数命令使用`-`作为参数前缀，但少数命令（如`date`）有其特定格式。
2.  命令的参数和对象可根据需要省略。

![](img/22c220998a1a15db2d391239d6e8d382_20.png)

### reboot与poweroff命令：重启与关机
这两个命令用于重启和关闭系统，无需额外参数。

![](img/22c220998a1a15db2d391239d6e8d382_22.png)

**语法：**
```bash
reboot
poweroff
```
执行后系统将立即重启或关机。

### wget命令：下载网络文件
`wget`命令用于从网络上下载文件。

![](img/22c220998a1a15db2d391239d6e8d382_24.png)

**基本语法：**
```bash
wget [URL]
```
例如：`wget http://example.com/file.tar.gz`
此命令会将文件下载到当前所在的目录。

**常用参数：**
*   `-c`：支持断点续传。如果下载中断，重新执行此命令可以继续下载。

**注意：** 执行下载前请确保系统已连接网络。

## 进程管理命令

了解了系统基本信息后，我们需要学习如何查看和管理系统中运行的程序，即进程。

![](img/22c220998a1a15db2d391239d6e8d382_30.png)

### ps命令：查看进程状态
`ps`命令用于查看当前系统的进程状态。进程是正在执行的程序实例。

![](img/22c220998a1a15db2d391239d6e8d382_32.png)

Linux系统中，进程主要有5种状态：
*   **R**：正在运行或等待运行。
*   **S**：休眠中，可被唤醒。
*   **D**：不可中断的休眠（通常在进行I/O操作）。
*   **Z**：僵尸进程。进程已终止，但其资源未被完全释放。
*   **T`：已停止。

仅输入`ps`命令信息有限。通常使用固定参数组合来查看详细信息。

![](img/22c220998a1a15db2d391239d6e8d382_34.png)

**常用命令：**
```bash
ps aux
```
*   `a`：显示所有用户的进程。
*   `u`：显示进程的详细信息（如用户、CPU/内存占用率）。
*   `x`：显示没有控制终端的进程（如系统后台服务）。

![](img/22c220998a1a15db2d391239d6e8d382_36.png)

此命令会列出系统所有进程的详细信息。

### top命令：动态监视系统状态
`top`命令提供了一个动态、实时更新的系统状态和进程信息视图，类似于增强版的Windows任务管理器。

执行`top`命令后，屏幕上半部分显示系统概览，下半部分显示进程列表。

**系统概览区关键信息解读：**
1.  **系统时间与运行时间**：当前时间及系统已运行时长。
2.  **用户与负载**：登录用户数及系统平均负载。负载值显示最近1、5、15分钟的平均负载情况。对于单核CPU，建议负载值长期不要超过5。
3.  **进程统计**：总进程数，以及处于运行、休眠、停止、僵尸状态的进程数。
4.  **CPU使用率**：重点关注`us`（用户空间占用）和`id`（空闲率）。
5.  **内存使用率**：显示物理内存总量、已用量、空闲量。
6.  **交换分区使用率**：显示虚拟内存的使用情况。

![](img/22c220998a1a15db2d391239d6e8d382_38.png)

![](img/22c220998a1a15db2d391239d6e8d382_39.png)

![](img/22c220998a1a15db2d391239d6e8d382_41.png)

![](img/22c220998a1a15db2d391239d6e8d382_42.png)

**进程列表区关键列：**
*   `PID`：进程的唯一标识符。
*   `USER`：启动该进程的用户。
*   `%CPU` / `%MEM`：进程占用CPU和内存的百分比。
*   `COMMAND`：启动进程的命令名称。

![](img/22c220998a1a15db2d391239d6e8d382_43.png)

在`top`界面中，可以按`q`键退出。

![](img/22c220998a1a15db2d391239d6e8d382_45.png)

### 管理进程：pidof与kill命令
我们不仅需要查看进程，有时还需要管理它们。

**pidof命令：查找服务对应的进程号**
```bash
pidof [服务名称]
```
例如：`pidof sshd` 会返回`sshd`服务对应的进程ID（PID）。

![](img/22c220998a1a15db2d391239d6e8d382_47.png)

![](img/22c220998a1a15db2d391239d6e8d382_48.png)

**kill命令：终止进程**
```bash
kill [PID]
```
例如，如果`pidof sshd`返回`1829`，则 `kill 1829` 会终止SSH服务进程。

**killall命令：终止指定名称的所有进程**
```bash
killall [进程名称]
```
例如：`killall sshd` 会终止所有名为`sshd`的进程。这比逐个`kill` PID更方便。

## 系统状态检查命令

当拿到一台新服务器时，我们需要快速了解其硬件和系统状态。以下命令将帮助我们完成初步检查。

### ifconfig命令：查看网络配置
`ifconfig`用于查看和配置网络接口信息。

**执行命令：**
```bash
ifconfig
```
输出信息较多，我们主要关注四点：
1.  **网卡名称**：如 `ens33`。
2.  **IP地址与子网掩码**：`inet`后面的地址。
3.  **MAC地址**：`ether`后面的物理地址。
4.  **网络流量**：`RX`（接收）和`TX`（发送）的数据包大小。

### uname命令：查看系统内核信息
`uname`命令用于打印系统信息。

**常用命令：**
```bash
uname -a
```
`-a`参数表示显示所有信息。输出包括：
*   内核名称（如Linux）
*   主机名
*   内核发行版本
*   内核版本
*   硬件架构（如x86_64）
*   操作系统名称

### uptime命令：查看系统负载
`uptime`命令可以快速查看系统已经运行了多长时间以及平均负载。
```bash
uptime
```
输出信息与`top`命令第一行类似，显示了当前时间、运行时间、用户数和平均负载。

### free命令：查看内存使用量
`free`命令显示系统内存的使用情况。

**建议使用`-h`参数，以人类易读的格式显示：**
```bash
free -h
```
输出显示物理内存和交换分区的总量、已用量、空闲量等信息。

### who与last命令：查看登录信息
**who命令：** 显示当前登录系统的用户信息。
```bash
who
```
**last命令：** 显示系统近期的登录记录。
```bash
last
```
**注意：** 这些记录可以被修改，因此不能作为系统是否被入侵的绝对依据。

### history命令：查看命令历史
`history`命令可以查看当前用户在终端中执行过的命令历史记录。
```bash
history
```
每条命令前有一个编号。可以使用 `!编号` 的形式快速重新执行某条历史命令。
例如：`!45` 会重新执行历史记录中编号为45的命令。

### sosreport命令：收集系统信息
当系统出现故障需要向技术支持求助时，可以使用`sosreport`命令收集详细的系统配置和日志信息。
```bash
sosreport
```
命令执行后，会生成一个压缩包文件。可以将此文件发送给技术支持人员，帮助他们分析问题。

## 文件与目录管理命令

系统状态检查完毕后，我们进入文件系统操作的学习。在Linux中，一切皆文件，因此文件操作至关重要。

### pwd与cd命令：定位与切换目录
**pwd命令：** 显示当前所在的工作目录。
```bash
pwd
```
**cd命令：** 用于切换工作目录。
```bash
cd [目录路径]
```
**常用快捷操作：**
*   `cd -`：返回上一次所在的目录。
*   `cd ..`：返回上一级目录。
*   `cd ~`：返回当前用户的家目录。

![](img/22c220998a1a15db2d391239d6e8d382_50.png)

### ls命令：列出目录内容
`ls`命令用于列出目录中的文件和子目录。

**常用参数：**
*   `-a`：显示所有文件，包括以`.`开头的隐藏文件。
*   `-l`：以长格式显示详细信息，包括文件权限、所有者、大小、修改时间等。
*   参数可以合并使用，例如 `ls -al` 表示以详细信息列表显示所有文件。

![](img/22c220998a1a15db2d391239d6e8d382_52.png)

### 文件内容查看命令
根据文件大小和查看目的，有不同的命令可供选择。

**cat命令：查看小型文件**
`cat`命令适合查看内容较少的文件。
```bash
cat [文件名]
```
它会一次性将文件所有内容输出到屏幕。

**more命令：分页查看文件**
`more`命令适合查看内容较多的文件。它会一页一页地显示，按空格键翻页，按回车键向下滚动一行。
```bash
more [文件名]
```

**head与tail命令：查看文件首尾**
*   `head`：查看文件开头部分。`head -n 5 [文件名]` 查看前5行。
*   `tail`：查看文件末尾部分。`tail -n 5 [文件名]` 查看后5行。
*   `tail -f [文件名]`：持续刷新显示文件末尾内容，常用于监控日志文件。

### 文件内容处理命令
以下命令用于对文件内容进行统计、转换和比较。

**wc命令：统计文件信息**
`wc`命令用于统计文件的行数、单词数、字节数。
```bash
wc -l [文件名]  # 统计行数
wc -c [文件名]  # 统计字节数
```

**tr命令：替换或删除字符**
`tr`命令用于转换或删除文件中的字符。它通常需要与管道符`|`配合使用，从标准输入读取数据。
```bash
cat [文件名] | tr [a-z] [A-Z]
```
此命令会将文件内容中的所有小写字母转换为大写字母并输出到屏幕。

**stat命令：查看文件详细信息**
`stat`命令可以查看文件的详细属性，包括三个重要的时间戳：
*   **Access Time**：最后一次访问文件的时间。
*   **Modify Time**：最后一次修改文件**内容**的时间。
*   **Change Time**：最后一次更改文件**属性**（如权限、所有者）的时间。
```bash
stat [文件名]
```

**cut命令：按列提取文本**
`cut`命令用于从文本文件的每一行中提取特定字段。
```bash
cut -d: -f1 [文件名]
```
*   `-d:`：指定冒号`:`作为字段分隔符。
*   `-f1`：提取第一个字段。

![](img/22c220998a1a15db2d391239d6e8d382_54.png)

此命令常用于处理`/etc/passwd`这类以冒号分隔的配置文件。

**diff命令：比较文件差异**
`diff`命令用于比较两个文件的差异。
```bash
diff [文件A] [文件B]
```
如果只关心文件是否不同，可以使用`--brief`参数。
```bash
diff --brief [文件A] [文件B]
```
如果需要查看具体的差异内容，可以使用`-c`参数，它会以上下文格式显示差异。

## 总结
本节课我们一起学习了大量实用的Linux命令，涵盖了系统信息查看、进程管理、网络配置、文件与目录操作等多个核心领域。我们学习了如何用`ps`和`top`监控进程，用`ifconfig`和`free`检查系统资源，用`cd`、`ls`、`cat`、`more`等命令熟练操作文件系统，并用`diff`、`cut`等命令处理文件内容。掌握这些命令是成为合格Linux系统管理员的第一步。请务必通过课后练习巩固所学知识，为后续更深入的学习打下坚实基础。
