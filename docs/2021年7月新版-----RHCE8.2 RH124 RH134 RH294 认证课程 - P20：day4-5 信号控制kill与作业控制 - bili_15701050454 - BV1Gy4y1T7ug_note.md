## 概述
在本节课中，我们将要学习Linux系统中如何通过信号来管理进程，以及如何进行作业控制。信号是系统与进程之间通信的一种方式，而作业控制则允许我们在前台和后台之间灵活地运行和管理任务。

## 信号管理

![](img/cee9e57117c3355be2e83865e64a9158_14.png)

上一节我们介绍了进程的基本概念，本节中我们来看看如何通过信号来控制进程。

![](img/cee9e57117c3355be2e83865e64a9158_16.png)

![](img/cee9e57117c3355be2e83865e64a9158_18.png)

通过`kill`命令可以向进程发送信号。信号并非总是“杀死”进程，而是向系统或进程发送指令，使其执行特定操作。信号有很多种，对于进程管理，我们主要关注其中几个。

![](img/cee9e57117c3355be2e83865e64a9158_20.png)

以下是系统支持的部分信号列表，可以使用 `kill -L` 命令查看全部64个信号：
```bash
kill -L
```

常用信号及其含义如下：
*   **SIGHUP (1)**：重新加载配置文件，无需关闭进程。
*   **SIGINT (2)**：中断进程，相当于在终端按 `Ctrl+C`。
*   **SIGKILL (9)**：强制终止进程，进程无法忽略此信号。
*   **SIGTERM (15)**：终止进程，这是`kill`命令发送的默认信号，进程可以捕获并处理它。
*   **SIGSTOP (19)**：暂停（休眠）进程。
*   **SIGCONT (18)**：继续运行被暂停的进程。

![](img/cee9e57117c3355be2e83865e64a9158_22.png)

![](img/cee9e57117c3355be2e83865e64a9158_24.png)

信号`SIGTERM(15)`和`SIGKILL(9)`都可以终止进程，但两者有区别。`SIGTERM`允许进程进行清理工作，而`SIGKILL`会强制立即终止进程，不给进程任何处理机会。

![](img/cee9e57117c3355be2e83865e64a9158_26.png)

信号名称前的`SIG`可以省略。我们可以使用信号编号或名称来发送信号。例如，终止进程号为1234的进程：
```bash
kill 1234          # 发送默认的SIGTERM(15)信号
kill -15 1234      # 发送SIGTERM信号
kill -TERM 1234    # 发送SIGTERM信号
kill -9 1234       # 发送SIGKILL信号强制终止
kill -KILL 1234    # 发送SIGKILL信号强制终止
```

![](img/cee9e57117c3355be2e83865e64a9158_28.png)

有一个特殊的信号`0`，它不发送任何信号，仅用于测试目标进程是否存在。但需要注意，它无法检测进程是否健康（例如僵尸进程也会被判定为存在）。
```bash
kill -0 <PID>
```

## 信号应用示例

![](img/cee9e57117c3355be2e83865e64a9158_39.png)

接下来，我们通过具体例子来看看信号的实际应用。

![](img/cee9e57117c3355be2e83865e64a9158_41.png)

![](img/cee9e57117c3355be2e83865e64a9158_43.png)

**示例1：重新加载服务配置（SIGHUP）**
`SIGHUP(1)`信号常用于让服务重新加载其配置文件，而无需重启服务，从而避免中断现有连接。例如，修改SSH服务的端口后，可以向其主进程发送`SIGHUP`信号。
```bash
# 1. 查找SSH服务主进程ID
ps aux | grep sshd
# 2. 向该进程发送SIGHUP信号，使其重新加载配置文件
kill -1 <主进程PID>
```

![](img/cee9e57117c3355be2e83865e64a9158_45.png)

![](img/cee9e57117c3355be2e83865e64a9158_47.png)

![](img/cee9e57117c3355be2e83865e64a9158_49.png)

**示例2：使用进程名终止进程（killall/pkill）**
有时我们不知道进程的具体PID，但知道进程名。这时可以使用`killall`或`pkill`命令，它们的功能类似，能根据进程名发送信号。
```bash
# 终止所有名为“httpd”的进程
killall httpd
# 或使用pkill
pkill httpd
```
这适用于需要关闭一个服务，但该服务可能包含多个进程的情况。

## 作业控制

![](img/cee9e57117c3355be2e83865e64a9158_67.png)

上一节我们介绍了如何用信号控制单个进程，本节中我们来看看如何管理一系列任务，即作业控制。

![](img/cee9e57117c3355be2e83865e64a9158_69.png)

![](img/cee9e57117c3355be2e83865e64a9158_71.png)

![](img/cee9e57117c3355be2e83865e64a9158_73.png)

![](img/cee9e57117c3355be2e83865e64a9158_75.png)

作业分为前台作业和后台作业。前台作业启动后会占用当前终端，用户无法在同一个终端执行其他命令。后台作业则在后台运行，不会占用终端，用户可以在前台继续执行其他任务。

![](img/cee9e57117c3355be2e83865e64a9158_77.png)

![](img/cee9e57117c3355be2e83865e64a9158_79.png)

![](img/cee9e57117c3355be2e83865e64a9158_81.png)

![](img/cee9e57117c3355be2e83865e64a9158_83.png)

**管理作业的常用命令：**
*   `jobs`：列出当前会话中的后台作业及其状态和编号。
*   `fg %n`：将编号为`n`的后台作业切换到前台运行。
*   `bg %n`：将编号为`n`的已停止的后台作业继续在后台运行。
*   `Ctrl+Z`：快捷键，将当前前台作业暂停，并放入后台（状态为Stopped）。

![](img/cee9e57117c3355be2e83865e64a9158_85.png)

以下是作业状态转换的常见操作流程：
1.  启动一个前台作业（例如 `ping baidu.com`）。
2.  按下 `Ctrl+Z`，该作业被暂停并放入后台，状态为`Stopped`。
3.  使用 `jobs` 命令查看作业列表和编号。
4.  使用 `bg %1` 让1号作业在后台继续运行。
5.  使用 `fg %1` 将1号作业调回前台。
6.  也可以直接向作业发送`SIGCONT(18)`信号使其继续运行：`kill -18 %1`

![](img/cee9e57117c3355be2e83865e64a9158_87.png)

![](img/cee9e57117c3355be2e83865e64a9158_89.png)

![](img/cee9e57117c3355be2e83865e64a9158_91.png)

**启动后台作业的方法：**
除了先将命令在前台运行再转到后台，也可以在命令末尾添加 `&` 符号，使其直接在后台启动运行。
```bash
# 命令将在后台立即运行
dd if=/dev/zero of=/dev/null &
```

![](img/cee9e57117c3355be2e83865e64a9158_93.png)

![](img/cee9e57117c3355be2e83865e64a9158_95.png)

![](img/cee9e57117c3355be2e83865e64a9158_97.png)

![](img/cee9e57117c3355be2e83865e64a9158_99.png)

**使作业与终端无关：**
默认情况下，后台作业的输出仍会显示在启动它的终端上。如果希望作业在终端关闭后仍能继续运行，且输出不干扰当前终端，可以使用 `nohup` 命令。
```bash
# 启动一个与终端无关的后台作业，输出默认重定向到 nohup.out 文件
nohup ping baidu.com &
```
这样，即使关闭当前终端，这个`ping`命令也会继续在系统后台运行。

## 总结
本节课中我们一起学习了Linux的进程信号管理和作业控制。我们了解了常用信号（如TERM、KILL、HUP）的作用与用法，掌握了通过`kill`、`killall`、`pkill`命令向进程发送信号。同时，我们也学习了作业的前后台切换（`fg`/`bg`）、列表查看（`jobs`）以及如何启动与终端无关的后台作业（`nohup`）。这些技能对于在Linux环境下高效地管理和控制运行中的任务至关重要。
