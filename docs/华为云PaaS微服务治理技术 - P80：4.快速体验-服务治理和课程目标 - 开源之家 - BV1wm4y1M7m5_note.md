# 华为云PaaS微服务治理技术：P80：4.快速体验-服务治理和课程目标 🎯

![](img/8dd914e34d32b37d3872c835772f44ca_1.png)

在本节课中，我们将快速体验如何在华为云平台上对微服务进行治理，并明确本课程的学习目标。我们将通过一个名为“WeatherMap”的微服务应用，演示服务注册、监控和降级等核心治理功能。

## 服务注册与发现

上一节我们完成了服务的启动。现在，服务信息已经注册到云平台。

![](img/8dd914e34d32b37d3872c835772f44ca_3.png)

在云平台微服务引擎的界面中，“服务目录”里显示的就是已经注册到云平台的服务实例。

![](img/8dd914e34d32b37d3872c835772f44ca_4.png)

点击“服务治理”菜单，进入治理界面。我们主要关注前端的“WeatherMap”服务，它当前处于高亮状态，表示服务正在运行。

![](img/8dd914e34d32b37d3872c835772f44ca_6.png)

服务治理界面展示了一个调用关系图。图中，`fusion-weather` 是网关服务，前端请求首先到达这里。网关会将请求转发到具体的微服务。

具体的微服务有两个：`weather`（当前天气查询服务）和 `forecast`（未来天气查询服务）。其中 `weather` 服务启动了两个实例（对应两个版本）。图中的箭头清晰地表示了网关与微服务之间的调用关系。

这个图形界面和“服务目录”中的信息，共同说明了服务已成功注册到注册中心，并且正在运行。

## 服务监控 📊

![](img/8dd914e34d32b37d3872c835772f44ca_8.png)

![](img/8dd914e34d32b37d3872c835772f44ca_9.png)

![](img/8dd914e34d32b37d3872c835772f44ca_10.png)

服务治理包含多种策略，我们将快速体验其中两项。首先，我们来看服务的监控。服务治理就是通过云平台的图形化界面，快速地对服务进行监控和管理。

以下是服务监控的一个示例图，其中包含了许多性能指标。

我们可以点击服务来查看其具体的监控指标。例如，点击 `fusion-weather` 服务，可以看到其吞吐量、熔断状态、实例数等各种指标。

这些指标包括每秒请求数、请求失败数量、请求超时数量等。因为是快速体验，我们先简单了解。

![](img/8dd914e34d32b37d3872c835772f44ca_12.png)

![](img/8dd914e34d32b37d3872c835772f44ca_13.png)

![](img/8dd914e34d32b37d3872c835772f44ca_14.png)

目前图形显示的数据可能为零，因为它展示的是近10秒的信息。为了让数据出现，我们需要让服务接收外部请求。

![](img/8dd914e34d32b37d3872c835772f44ca_15.png)

在浏览器中访问应用，例如搜索“郑州”的天气。然后返回监控界面，可以看到图表中已经出现了数据。

当服务开始处理外部请求时，监控数据就会产生。通过图形界面，我们可以直观地看到服务的运行情况，这就是服务监控。

## 服务降级 ⬇️

![](img/8dd914e34d32b37d3872c835772f44ca_17.png)

![](img/8dd914e34d32b37d3872c835772f44ca_18.png)

接下来，我们快速体验另一个功能：服务降级。服务降级是容错的一种形式。容错是指微服务在遇到错误时的处理能力。

![](img/8dd914e34d32b37d3872c835772f44ca_20.png)

降级的具体含义是：当服务压力过大、资源紧张时，可以暂时关闭一些非核心服务，以保障核心服务的资源。

例如，在“双十一”期间，订单服务压力巨大。此时，可以保留订单服务，而将修改密码、维护个人信息等次要服务暂时降级（关闭），从而为订单服务节省出系统资源。

服务降级并不是简单地杀掉进程，因为进程杀掉后还需要手动重启。云平台提供了更智能的图形化方式来实现降级。

例如，现在我们只想查看当前天气，不希望查询未来天气。我们可以通过降级功能来实现。

![](img/8dd914e34d32b37d3872c835772f44ca_22.png)

操作步骤如下：
1.  在服务治理界面，点击网关服务 `fusion-weather`。
2.  点击“降级”选项，然后点击“新增”。
3.  在“所有微服务”中选择 `forecast` 服务。
4.  因为我们希望关闭该服务的所有方法，所以选择“所有方法”。（也可以选择性地关闭特定方法）
5.  点击“确定”。

![](img/8dd914e34d32b37d3872c835772f44ca_24.png)

此时，降级策略就启动成功了。这意味着网关将不再把请求转发给 `forecast` 服务。

为了测试效果，我们刷新天气预报页面。可以看到，页面中未来天气的部分变为空白，但当前天气信息仍然正常显示。这是因为我们只对 `forecast` 服务进行了降级，而 `weather` 服务仍在正常运行。

![](img/8dd914e34d32b37d3872c835772f44ca_26.png)

![](img/8dd914e34d32b37d3872c835772f44ca_27.png)

通过这种方式，我们可以非常快速、智能地对服务进行降级操作。

![](img/8dd914e34d32b37d3872c835772f44ca_29.png)

![](img/8dd914e34d32b37d3872c835772f44ca_30.png)

当系统压力过去，需要恢复服务时，操作也很简单：只需删除刚才创建的降级规则即可。

![](img/8dd914e34d32b37d3872c835772f44ca_32.png)

删除降级规则后，服务间的调用关系立即恢复。再次刷新天气预报页面，未来天气的信息就会重新显示出来。

![](img/8dd914e34d32b37d3872c835772f44ca_34.png)

通过这个快速体验，我们可以直观地感受到，通过云平台治理微服务是非常智能和高效的，能显著降低系统的维护成本。

当然，服务治理还包括很多其他策略，如负载均衡、限流、熔断等，这些内容我们将在后续课程中详细讲解。

## 课程总结与目标 🎯

![](img/8dd914e34d32b37d3872c835772f44ca_36.png)

![](img/8dd914e34d32b37d3872c835772f44ca_37.png)

至此，我们快速体验了如何使用云平台治理微服务。现在我们来总结一下，并明确本课程的目标。

![](img/8dd914e34d32b37d3872c835772f44ca_39.png)

首先，要想使用云平台治理微服务，必须先将服务信息注册到云平台。我们可以在“服务目录”中查看注册成功的服务。

服务注册成功后，就可以通过“服务治理”的图形化界面，智能地进行服务治理。

本次快速体验让我们对课程目标有了直观了解。但需要指出，完整的“上云”流程还包括将项目部署到云平台。这不仅仅是注册服务，而是要将服务应用本身部署到云平台的服务器上，通常采用 Docker 容器化技术来实现快速、智能的部署。

![](img/8dd914e34d32b37d3872c835772f44ca_41.png)

![](img/8dd914e34d32b37d3872c835772f44ca_42.png)

![](img/8dd914e34d32b37d3872c835772f44ca_43.png)

在后续课程中，我们会讲解“云容器引擎”，基于容器化技术将微服务部署到云端。部署上云后，再结合本节课体验的治理功能，就能实现完整的云平台微服务治理。

本次课程的目标包含以下三个部分：
1.  **采用微服务引擎CSE开发微服务**。
2.  **将微服务在华为云上部署**。
3.  **采用云平台来治理微服务**。

整个课程将以一个大型实战综合项目——“学成在线”（一个在线教育平台）作为案例进行讲解。最终目标是实现将“学成在线”项目部署到华为云平台，并使用云平台对其进行全面的治理。

![](img/8dd914e34d32b37d3872c835772f44ca_45.png)

本节课中，我们一起学习了如何在华为云平台快速体验服务注册、服务监控和服务降级等微服务治理功能，并明确了本课程将围绕开发、部署、治理三大目标展开。
