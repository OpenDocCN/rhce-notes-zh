## 字符串截取 ✂️

上一节我们介绍了Shell脚本的基本结构，本节中我们来看看如何对字符串进行截取。字符串截取是指从一个字符串变量中提取出指定位置和长度的子串。

其基本语法格式如下：

![](img/5fc03571253b7d3bfb0db590c906e161_5.png)

```bash
${变量名:起始位置:长度}
```

以下是关于字符串截取的具体说明和示例：

![](img/5fc03571253b7d3bfb0db590c906e161_7.png)

![](img/5fc03571253b7d3bfb0db590c906e161_9.png)

![](img/5fc03571253b7d3bfb0db590c906e161_11.png)

*   **起始位置**：从0开始计数。例如，字符串“12345678”中，数字“1”的位置是0，“2”的位置是1，以此类推。
*   **长度**：指定要截取的字符个数。如果省略长度，则会从起始位置截取到字符串末尾。
*   **示例**：假设我们有一个变量 `phone="13812345678"`，想要截取后四位“5678”。
    *   首先确定起始位置：从0开始数，“5”是第7位（0-1, 1-3, 2-8, 3-1, 4-2, 5-3, 6-4, 7-5, 8-6, 9-7, 10-8）。
    *   执行命令：`echo ${phone:7:4}`，输出结果为“5678”。

![](img/5fc03571253b7d3bfb0db590c906e161_13.png)

这个功能不仅限于数字，也可以用于截取文件内容或其他文本。

## 字符串替换 🔄

![](img/5fc03571253b7d3bfb0db590c906e161_15.png)

![](img/5fc03571253b7d3bfb0db590c906e161_17.png)

字符串替换是指将字符串中指定的部分内容更改为新的内容。

其基本语法格式如下：

```bash
${变量名/旧内容/新内容}
```

![](img/5fc03571253b7d3bfb0db590c906e161_19.png)

![](img/5fc03571253b7d3bfb0db590c906e161_21.png)

以下是关于字符串替换的具体说明：

![](img/5fc03571253b7d3bfb0db590c906e161_23.png)

*   **单次替换**：使用一个斜杠`/`，只替换第一个匹配到的旧内容。
    *   示例：`str="1234234"; echo ${str/3/*}` 会将第一个“3”替换为“*”，输出“12*4234”。
*   **全局替换**：使用两个斜杠`//`，替换所有匹配到的旧内容。
    *   示例：`str="1234234"; echo ${str//3/*}` 会将所有“3”替换为“*”，输出“12*42*4”。

![](img/5fc03571253b7d3bfb0db590c906e161_25.png)

这个功能类似于文本编辑器（如Vim）中的替换功能，但它是直接对变量值进行操作。

## 字符串掐头去尾 ✂️

“掐头去尾”是指删除字符串开头或结尾的指定部分。

以下是关于字符串掐头去尾的具体说明：

*   **掐头（删除开头）**：使用 `#` 符号。
    *   `#关键词`：从开头删除**最短匹配**到“关键词”的部分。
    *   `##关键词`：从开头删除**最长匹配**到“关键词”的部分。
*   **去尾（删除结尾）**：使用 `%` 符号。
    *   `%关键词`：从结尾删除**最短匹配**到“关键词”的部分。
    *   `%%关键词`：从结尾删除**最长匹配**到“关键词”的部分。

**示例**：
假设 `file="backup.tar.gz"`
*   `echo ${file#*.}` 输出 `tar.gz` （掐头，最短匹配，删除第一个点号及之前的部分）。
*   `echo ${file##*.}` 输出 `gz` （掐头，最长匹配，删除最后一个点号及之前的部分）。
*   `echo ${file%.*}` 输出 `backup.tar` （去尾，最短匹配，删除最后一个点号及之后的部分）。
*   `echo ${file%%.*}` 输出 `backup` （去尾，最长匹配，删除第一个点号及之后的部分）。

这个功能在批量修改文件扩展名时非常实用。

## 实践应用：批量修改文件扩展名 📁

结合循环和字符串处理，我们可以轻松实现批量操作。例如，将当前目录下所有 `.txt` 文件的扩展名改为 `.doc`。

以下是两种实现方法的示例脚本：

**方法一：使用字符串去尾和拼接**
```bash
#!/bin/bash
for file in *.txt
do
    mv "$file" "${file%.txt}.doc"
done
```

**方法二：使用字符串替换**
```bash
#!/bin/bash
for file in *.txt
do
    mv "$file" "${file/.txt/.doc}"
done
```

![](img/5fc03571253b7d3bfb0db590c906e161_27.png)

这两个脚本都能达到相同的效果。在实际工作中，可以根据具体情况和个人习惯选择合适的方法。

## 学习策略：借鉴与修改 💡

![](img/5fc03571253b7d3bfb0db590c906e161_29.png)

![](img/5fc03571253b7d3bfb0db590c906e161_31.png)

在运维工作中，完全从零开始编写复杂脚本的情况并不多见。更高效的学习和工作方式是学会借鉴和修改。

![](img/5fc03571253b7d3bfb0db590c906e161_33.png)

![](img/5fc03571253b7d3bfb0db590c906e161_35.png)

*   **利用现有资源**：当需要实现某个功能时（如生成随机密码、测试网络连通性），首先在网上搜索相关的脚本。很多常见需求都有成熟的解决方案。
*   **理解与调试**：将找到的脚本拿到自己的环境中运行测试。如果报错，需要根据错误信息并结合已学的Shell知识进行调试和修改（例如，修正IP地址、路径或命令选项）。
*   **掌握核心即可**：学习Shell脚本，首要目标是能够读懂别人的脚本，并知道如何修改以适应自己的需求，而不是死记硬背所有语法细节。遇到不熟悉的命令（如 `cut`, `sort`），可以随时通过 `man` 命令或搜索引擎查询其用法。

![](img/5fc03571253b7d3bfb0db590c906e161_39.png)

![](img/5fc03571253b7d3bfb0db590c906e161_41.png)

这是一种高效且实用的学习路径，即“前人种树，后人乘凉”。具备看懂和修改脚本的能力，远比从零创造更重要。
