# RHCE8.0 红帽认证入门教程：Day08：Stratis 与 VDO 存储管理 🚀

在本节课中，我们将要学习红帽企业版 Linux 8.0 中的两个新特性存储管理工具：Stratis 和 VDO。Stratis 是一个精简配置的共享存储池管理工具，而 VDO 则是一个虚拟数据优化器，用于数据去重和压缩。我们将通过实际操作来掌握它们的配置和使用方法。

## Stratis：精简配置存储池管理 💾

上一节我们介绍了课程概述，本节中我们来看看 Stratis。Stratis 是一种本地存储管理工具，它将文件系统构建在一个精简配置的共享存储池中，旨在实现精简配置、快照和基于池的管理监控等高级存储功能。

### 安装与启动服务

以下是安装和启动 Stratis 服务的步骤。

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_9.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_11.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_12.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_13.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_14.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_16.png)

1.  安装必要的软件包。
    ```bash
    dnf install -y stratisd stratis-cli
    ```
2.  启动并启用 Stratis 守护进程服务。
    ```bash
    systemctl enable --now stratisd
    systemctl status stratisd
    ```

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_18.png)

### 创建存储池与文件系统

安装并启动服务后，我们可以创建一个存储池，并在其上建立文件系统。

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_26.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_28.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_29.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_30.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_32.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_34.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_36.png)

1.  使用一个空白磁盘（例如 `/dev/nvme0n3`）创建名为 `tpool` 的存储池。
    ```bash
    stratis pool create tpool /dev/nvme0n3
    stratis pool list
    ```
2.  在存储池 `tpool` 上创建一个名为 `tfs` 的文件系统。
    ```bash
    stratis filesystem create tpool tfs
    stratis filesystem list
    ```

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_38.png)

### 挂载文件系统

创建文件系统后，需要将其挂载到目录才能使用。这里介绍永久挂载的方法。

1.  首先，获取文件系统的 UUID。
    ```bash
    stratis filesystem list
    ```
2.  创建挂载点目录。
    ```bash
    mkdir /mnt/tfs
    ```
3.  编辑 `/etc/fstab` 文件，添加挂载配置。**关键** 是需要在 `defaults` 选项后，添加 `x-systemd.requires=stratisd.service` 以确保挂载依赖于 Stratis 服务。
    ```bash
    # 将以下行添加到 /etc/fstab 中，请替换为你的实际 UUID
    UUID=<你的_UUID> /mnt/tfs xfs defaults,x-systemd.requires=stratisd.service 0 0
    ```
4.  执行挂载并检查。
    ```bash
    mount -a
    df -Th
    ```
    你将看到文件系统已被挂载，并且其显示大小（如 1T）是精简配置的逻辑大小，而非实际磁盘大小。

### 管理存储池

Stratis 存储池支持动态扩展。以下是向现有存储池添加新磁盘的方法。

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_40.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_42.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_43.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_45.png)

1.  假设我们有两块新磁盘 `/dev/nvme0n4` 和 `/dev/nvme0n5`，将它们添加到 `tpool` 池中。
    ```bash
    stratis pool add-data tpool /dev/nvme0n4 /dev/nvme0n5
    ```
2.  再次列出存储池，可以看到总容量已增加。
    ```bash
    stratis pool list
    ```
    **重要提示**：虽然逻辑空间可以超额配置，但实际写入的数据量**绝对不能超过**存储池的实际物理总容量，否则会导致严重问题。

## VDO：虚拟数据优化器 🔄

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_57.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_59.png)

上一节我们介绍了 Stratis，本节中我们来看看 VDO。VDO 是一个位于块设备顶层的存储优化层，它通过零块消除、重复数据删除和压缩技术，有效减少存储设备上的空间占用。

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_63.png)

### 安装与创建 VDO 卷

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_69.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_71.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_72.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_74.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_76.png)

以下是安装软件包和创建 VDO 卷的步骤。

1.  安装必要的软件包。
    ```bash
    dnf install -y vdo kmod-kvdo
    ```
2.  使用一个物理磁盘（例如 `/dev/nvme0n6`）创建 VDO 卷。这里创建一个逻辑大小为 50G（大于物理磁盘的 20G）的卷，名为 `tvdovol`。
    ```bash
    vdo create --name=tvdovol --device=/dev/nvme0n6 --vdoLogicalSize=50G
    ```
3.  查看创建的 VDO 卷。
    ```bash
    vdo list
    ```

创建 VDO 卷后，可以像使用普通块设备一样对其进行格式化和挂载。

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_94.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_95.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_97.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_98.png)

1.  检查 VDO 卷的压缩和去重功能状态。
    ```bash
    vdo status --name=tvdovol | grep -i -E "compression|deduplication"
    ```
2.  在 VDO 卷上创建文件系统并挂载。
    ```bash
    # 格式化
    mkfs.xfs -K /dev/mapper/tvdovol
    # 创建挂载点
    mkdir /mnt/tvdo
    # 获取 UUID 并写入 /etc/fstab 实现永久挂载
    blkid /dev/mapper/tvdovol
    # 编辑 /etc/fstab，添加如下行（替换为你的 UUID）
    # UUID=<你的_UUID> /mnt/tvdo xfs defaults 0 0
    mount -a
    df -Th /mnt/tvdo
    ```
    此时 `df` 命令会显示逻辑大小 50G。

我们可以通过写入数据来验证 VDO 的零块消除和去重效果。

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_119.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_121.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_123.png)

![](img/88c8ade2bf2b8aa1c8a63d83698fb02f_125.png)

1.  **零块消除测试**：向卷中写入一个 10G 的全零文件。
    ```bash
    dd if=/dev/zero of=/mnt/tvdo/zero.file bs=1G count=10
    ```
    写入后，使用 `vdo status --name=tvdovol --human-readable` 命令查看实际物理空间占用，你会发现占用远小于 10G。
2.  **重复数据删除测试**：复制一个已有的文件（例如一个 4G 的 ISO 镜像）到挂载点的不同目录。
    ```bash
    # 第一次复制
    cp /path/to/large.iso /mnt/tvdo/dir1/
    # 观察 vdo status 中的实际数据占用增长
    vdo status --name=tvdovol --human-readable
    # 第二次复制相同文件到另一个目录
    cp /path/to/large.iso /mnt/tvdo/dir2/
    # 再次观察，实际数据占用几乎不会增加，但 `df` 命令显示的逻辑使用量会增加
    vdo status --name=tvdovol --human-readable
    df -h /mnt/tvdo
    ```
    这个实验清晰地展示了 VDO 的去重功能：存储相同内容的多个副本，实际只占用一份空间。
