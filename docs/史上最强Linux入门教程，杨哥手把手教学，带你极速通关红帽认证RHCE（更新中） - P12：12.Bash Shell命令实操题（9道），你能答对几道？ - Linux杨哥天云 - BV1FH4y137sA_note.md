### **题目一：创建特定结构的目录与文件**

首先，我们来看一道关于创建目录和文件的题目。这需要综合运用`mkdir`和`touch`命令。

请在`/home`目录下创建如下结构：
*   目录 `rhce`
*   在`rhce`目录下创建子目录 `dir1`, `dir2`, `dir3`
*   在`dir1`目录下创建文件 `file1.txt`, `file2.txt`
*   在`dir2`目录下创建文件 `file3.txt`

**解题命令：**
```bash
mkdir -p /home/rhce/{dir1,dir2,dir3}
touch /home/rhce/dir1/file1.txt /home/rhce/dir1/file2.txt /home/rhce/dir2/file3.txt
```
**命令解析：**
*   `mkdir -p`：`-p`选项确保可以创建多级目录，如果父目录不存在则会一并创建。
*   `{dir1,dir2,dir3}`：这是Bash的大括号扩展功能，能一次性生成多个参数，等效于分别输入`dir1` `dir2` `dir3`。
*   `touch`：用于创建空文件或更新文件的时间戳。

### **题目二：批量重命名文件**

上一节我们创建了文件，本节中我们来看看如何批量修改它们。这道题考察的是使用通配符和`mv`命令进行批量操作。

将`dir1`目录下的所有`.txt`文件重命名为`.bak`文件。

**解题命令：**
```bash
cd /home/rhce/dir1
mv *.txt *.bak
```
**命令解析：**
*   `cd`：切换当前工作目录到`/home/rhce/dir1`。
*   `*.txt`：通配符`*`匹配所有以`.txt`结尾的文件名。
*   需要注意的是，`mv *.txt *.bak`在Bash中的实际效果是将第一个匹配的`.txt`文件重命名为`*.bak`，然后报错。更安全的批量重命名应使用循环或`rename`命令。但根据题意及简单场景理解，此命令意在演示通配符匹配。

### **题目三：查找并归档文件**

接下来是文件查找与打包操作。这需要组合使用`find`和`tar`命令。

查找`/home/rhce`目录下所有`.bak`文件，并将它们打包成`backup.tar.gz`，存放在`/tmp`目录下。

**解题命令：**
```bash
find /home/rhce -name "*.bak" -exec tar -czf /tmp/backup.tar.gz {} +
```
**命令解析：**
*   `find /home/rhce -name "*.bak"`：在`/home/rhce`目录下查找所有文件名匹配`*.bak`的文件。
*   `-exec ... {} +`：`-exec`选项对查找到的每个文件执行后续命令。`{}`代表找到的文件路径，`+`表示将找到的所有文件一次性传递给后面的命令。
*   `tar -czf /tmp/backup.tar.gz`：`-c`创建归档，`-z`使用gzip压缩，`-f`指定归档文件名。这里将`find`找到的所有文件打包压缩。

### **题目四：文件内容过滤与统计**

现在转向文本处理。这道题涉及使用`grep`过滤内容，并用`wc`进行统计。

统计`/etc/passwd`文件中包含“bash”的行数。

**解题命令：**
```bash
grep "bash" /etc/passwd | wc -l
```
**命令解析：**
*   `grep "bash" /etc/passwd`：`grep`命令在`/etc/passwd`文件中搜索包含字符串“bash”的行并输出。
*   `|`：管道符，将前一个命令的标准输出作为后一个命令的标准输入。
*   `wc -l`：`wc`命令统计行数、单词数和字节数。`-l`选项使其只输出行数。

### **题目五：系统进程管理**

以下是进程管理相关的操作，我们需要使用`ps`、`grep`和`kill`命令。

![](img/0fef9f5b740a3b76f8363290c1369f19_1.png)

查找名为“nginx”的进程，并终止它。

**解题命令：**
```bash
ps aux | grep nginx | grep -v grep | awk '{print $2}' | xargs kill -9
```
**命令解析：**
*   `ps aux`：列出所有用户的详细进程信息。
*   `grep nginx`：从上一步的输出中筛选出包含“nginx”的行。
*   `grep -v grep`：排除掉`grep`进程自身（因为`grep nginx`这个命令也会出现在进程列表中）。
*   `awk '{print $2}'`：`awk`是一个文本处理工具，这里打印出每一行的第二列（即进程ID，PID）。
*   `xargs kill -9`：`xargs`将前面得到的PID作为参数传递给`kill -9`命令。`-9`是SIGKILL信号，强制终止进程。

### **题目六：磁盘空间检查**

系统管理离不开监控。这道题要求我们检查磁盘使用情况。

检查根分区（`/`）的磁盘使用情况，并以人类可读的格式显示。

**解题命令：**
```bash
df -h /
```
**命令解析：**
*   `df`：报告文件系统磁盘空间使用情况。
*   `-h`：`--human-readable`的缩写，以易读的格式（如K， M， G）显示大小。
*   `/`：指定要检查的文件系统为根分区。

### **题目七：计划任务设置**

自动化是Linux运维的核心。以下是配置计划任务的题目。

设置一个计划任务，每天凌晨2点整执行脚本 `/opt/backup.sh`。

**解题命令：**
```bash
(crontab -l 2>/dev/null; echo "0 2 * * * /opt/backup.sh") | crontab -
```
**命令解析：**
*   `crontab -l`：列出当前用户的cron任务。
*   `2>/dev/null`：将错误输出（例如，当没有cron任务时）重定向到空设备，避免报错信息干扰。
*   `echo "0 2 * * * /opt/backup.sh"`：输出要添加的cron行。时间格式为`分 时 日 月 周 命令`，`0 2 * * *`表示每天2点0分。
*   `(...) | crontab -`：将括号内组合命令的输出（即旧任务列表+新任务行）通过管道传递给`crontab -`，从而设置新的cron任务。

### **题目八：文件权限与所有权管理**

权限管理是系统安全的基础。这道题综合了`chmod`和`chown`命令。

将`/home/rhce/dir3`目录的所属组改为“users”，并为该目录设置`setgid`权限，使得在此目录下创建的新文件自动继承目录的所属组。

**解题命令：**
```bash
chown :users /home/rhce/dir3
chmod g+s /home/rhce/dir3
```
**命令解析：**
*   `chown :users /home/rhce/dir3`：`chown`命令修改文件所有者或所属组。`:`前为空表示只修改组，这里将目录的所属组改为`users`。
*   `chmod g+s /home/rhce/dir3`：`chmod`命令修改文件权限。`g+s`表示设置`setgid`位。对目录设置`setgid`后，任何用户在此目录下创建的文件或子目录，其所属组都将自动设置为该目录的所属组（即`users`）。

### **题目九：网络配置与测试**

最后，我们处理一个网络相关的题目。这需要配置IP并测试连通性。

将eth0网卡的IP地址临时设置为192.168.1.100/24，并测试到网关192.168.1.1的连通性。

**解题命令：**
```bash
ip addr add 192.168.1.100/24 dev eth0
ping -c 4 192.168.1.1
```
**命令解析：**
*   `ip addr add 192.168.1.100/24 dev eth0`：`ip`命令是强大的网络配置工具。此子命令为`eth0`网卡添加一个IP地址`192.168.1.100`，子网掩码为24位（即255.255.255.0）。此配置重启网络服务或系统后会失效。
*   `ping -c 4 192.168.1.1`：`ping`命令用于测试网络连通性。`-c 4`表示发送4个ICMP回显请求包后停止。
