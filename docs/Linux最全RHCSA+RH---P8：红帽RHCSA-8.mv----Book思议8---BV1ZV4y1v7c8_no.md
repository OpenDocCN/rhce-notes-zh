在本节课中，我们将要学习Linux系统中几个非常核心的文件操作命令。我们将从移动和重命名文件开始，逐步深入到如何查看文件内容，最后学习如何安全地删除文件和目录。这些命令是日常系统管理的基础，掌握它们对于任何Linux初学者都至关重要。

## 移动与重命名：mv命令 🚚

![](img/f6e565553c16fbe5688c4868f7732987_9.png)

![](img/f6e565553c16fbe5688c4868f7732987_11.png)

![](img/f6e565553c16fbe5688c4868f7732987_13.png)

上一节我们介绍了`cp`命令用于复制文件，本节中我们来看看`mv`命令。`mv`命令用于**移动**文件或目录到其他位置，其功能类似于Windows系统中的“剪切”。同时，它也可以用来修改文件或目录的名称。

![](img/f6e565553c16fbe5688c4868f7732987_15.png)

命令的基本格式与`cp`命令类似：
```bash
mv [选项] 源文件或目录 目标文件或目录
```

![](img/f6e565553c16fbe5688c4868f7732987_17.png)

![](img/f6e565553c16fbe5688c4868f7732987_19.png)

![](img/f6e565553c16fbe5688c4868f7732987_21.png)

### 移动文件
例如，将当前目录下的`hello`文件移动到`/opt/t1`目录中：
```bash
mv hello /opt/t1
```
执行后，`hello`文件将从原位置消失，出现在`/opt/t1`目录下。

![](img/f6e565553c16fbe5688c4868f7732987_23.png)

### 移动并重命名
`mv`命令在移动的同时可以重命名目标。例如，将`test`目录移动到根目录`/`下，并重命名为`xxxxx`：
```bash
mv test /xxxxx
```
执行后，根目录下会出现一个名为`xxxxx`的目录，其内容与原`test`目录相同。

![](img/f6e565553c16fbe5688c4868f7732987_25.png)

![](img/f6e565553c16fbe5688c4868f7732987_27.png)

### 移动多个项目
可以一次性移动多个文件或目录。例如，将当前目录下的`t6`、`t7`、`t8`目录移动到`/opt/test`目录中：
```bash
mv t6 t7 t8 /opt/test
```

![](img/f6e565553c16fbe5688c4868f7732987_29.png)

![](img/f6e565553c16fbe5688c4868f7732987_30.png)

![](img/f6e565553c16fbe5688c4868f7732987_32.png)

![](img/f6e565553c16fbe5688c4868f7732987_34.png)

![](img/f6e565553c16fbe5688c4868f7732987_36.png)

![](img/f6e565553c16fbe5688c4868f7732987_38.png)

**核心概念**：`mv`命令的本质是修改文件在文件系统中的路径入口。当源和目标在同一文件系统时，它仅修改元数据（如inode中的目录项），速度极快；当跨文件系统时，它实际执行“复制+删除”操作。

## 查看文件内容：cat命令 👀

在图形界面中，我们双击文件即可查看内容。在Linux命令行中，我们需要使用命令来查看文件内容，`cat`命令就是最基础的一个。

![](img/f6e565553c16fbe5688c4868f7732987_40.png)

![](img/f6e565553c16fbe5688c4868f7732987_42.png)

`cat`命令用于连接文件并打印到标准输出设备上，常用来显示文件内容。
```bash
cat [选项] 文件名
```

![](img/f6e565553c16fbe5688c4868f7732987_44.png)

![](img/f6e565553c16fbe5688c4868f7732987_46.png)

### 基本查看
查看`/etc/passwd`文件的内容：
```bash
cat /etc/passwd
```

![](img/f6e565553c16fbe5688c4868f7732987_48.png)

![](img/f6e565553c16fbe5688c4868f7732987_50.png)

### 显示行号
使用`-n`选项可以显示行号，这对于查看或调试文件非常有用：
```bash
cat -n /etc/passwd
```

![](img/f6e565553c16fbe5688c4868f7732987_52.png)

![](img/f6e565553c16fbe5688c4868f7732987_54.png)

以下是使用`cat`命令查看系统常见文件的示例：
*   **查看主机名**：`cat /etc/hostname`
*   **查看系统版本**：`cat /etc/centos-release` 或 `cat /etc/redhat-release`
*   **查看用户组信息**：`cat /etc/group`
*   **查看网卡配置**：`cat /etc/sysconfig/network-scripts/ifcfg-ens33`

**注意**：`cat`命令会一次性将整个文件内容输出到屏幕。对于小文件这很方便，但对于内容很多的大文件（如超过1万行的`/etc/services`），所有内容会瞬间刷屏，导致你无法查看之前输出的内容，且难以浏览文件开头部分。

## 分页查看大文件：less命令 📄

![](img/f6e565553c16fbe5688c4868f7732987_56.png)

![](img/f6e565553c16fbe5688c4868f7732987_58.png)

![](img/f6e565553c16fbe5688c4868f7732987_60.png)

![](img/f6e565553c16fbe5688c4868f7732987_62.png)

为了解决`cat`查看大文件的不便，我们可以使用`less`命令。`less`是一个分页显示工具，允许你像看书一样一页一页地浏览文件内容，非常适合查看大型日志或配置文件。

```bash
less [选项] 文件名
```
常用选项 `-N` 可以显示行号。

进入`less`视图后，可以使用以下快捷键进行操作：
*   **上下箭头键**：向上或向下滚动一行。
*   **Page Down / Page Up 或 Fn+下/上键**：向下或向上翻一页。
*   **g**：跳转到文件**第一行**。
*   **G**：跳转到文件**最后一行**。
*   **:数字**：输入英文冒号后加行号（如 `:1000`），然后按回车，可跳转到指定行。
*   **q**：退出`less`视图。

![](img/f6e565553c16fbe5688c4868f7732987_64.png)

![](img/f6e565553c16fbe5688c4868f7732987_66.png)

![](img/f6e565553c16fbe5688c4868f7732987_68.png)

例如，分页查看`/etc/services`文件：
```bash
less -N /etc/services
```

## 查看文件首尾：head 和 tail 命令 🔍

有时我们并不需要查看整个文件，而只关心文件的开头或结尾部分。这时`head`和`tail`命令就派上用场了。

![](img/f6e565553c16fbe5688c4868f7732987_70.png)

### head 命令
`head`命令用于显示文件的开头部分内容，**默认显示前10行**。
```bash
head [选项] 文件名
```
使用`-n`选项可以指定显示的行数。以下两种写法等效，都是查看文件前20行：
```bash
head -n 20 /etc/services
head -20 /etc/services # “-”和数字间不能有空格
```

![](img/f6e565553c16fbe5688c4868f7732987_72.png)

![](img/f6e565553c16fbe5688c4868f7732987_74.png)

![](img/f6e565553c16fbe5688c4868f7732987_76.png)

![](img/f6e565553c16fbe5688c4868f7732987_78.png)

### tail 命令
`tail`命令与`head`相反，用于显示文件的末尾部分内容，**默认显示最后10行**。
```bash
tail [选项] 文件名
```
同样使用`-n`指定行数。查看网卡配置文件`/etc/sysconfig/network-scripts/ifcfg-ens33`的最后5行，通常包含关键的IP、网关等信息：
```bash
tail -5 /etc/sysconfig/network-scripts/ifcfg-ens33
```

![](img/f6e565553c16fbe5688c4868f7732987_80.png)

### tail 的实时监控功能
`tail`命令有一个非常强大的 `-f` 选项，可以**动态显示**文件新增的内容。这对于实时监控日志文件的变化至关重要。
```bash
tail -f 文件名
```
例如，在一个终端执行 `tail -f /opt/test1.log`，然后在另一个终端向该文件写入内容（`echo “new log” >> /opt/test1.log`），第一个终端会立即显示出新增的“new log”行。按 `Ctrl+C` 可以终止监控。

![](img/f6e565553c16fbe5688c4868f7732987_84.png)

![](img/f6e565553c16fbe5688c4868f7732987_86.png)

## 删除文件与目录：rm命令 🗑️ (谨慎使用！)

![](img/f6e565553c16fbe5688c4868f7732987_88.png)

![](img/f6e565553c16fbe5688c4868f7732987_89.png)

`rm`命令用于删除文件或目录。**请务必谨慎使用此命令，因为Linux系统默认没有回收站功能，删除操作通常是不可逆的。**

```bash
rm [选项] 文件或目录
```

### 删除文件
删除普通文件不需要特殊选项，但系统会要求确认：
```bash
rm test.txt
# 输出：rm: remove regular file ‘test.txt’? y
```
输入 `y` 确认删除，输入 `n` 取消。

### 强制删除文件
使用 `-f` (force) 选项可以强制删除，**跳过确认提示**：
```bash
rm -f test.txt
```

### 删除目录
删除目录必须使用 `-r` (recursive，递归) 选项，因为需要删除目录及其内部所有内容：
```bash
rm -r test_dir/
# 系统会对目录内每个项目进行确认
```

### 强制递归删除（最常用也最危险）
组合使用 `-rf` 选项，可以**强制递归删除**目录及其下所有内容，且不进行任何确认：
```bash
rm -rf test_dir/
```

### 使用通配符 `*`
`*`（星号）是一个通配符，代表“任意所有”字符。它可以用来匹配多个文件。
*   删除`/opt`目录下所有内容：`rm -rf /opt/*`
    *   **注意**：这条命令删除的是`/opt`目录下的所有子项，而不是`/opt`目录本身。
*   **警告：著名的“删库”命令**
    ```bash
    rm -rf /*
    ```
    这条命令会强制递归删除根目录`/`下的所有文件。**绝对不要在生产环境或重要系统中尝试此命令**，它会摧毁整个操作系统。

### 路径分隔符 `/` 的注意事项
*   在路径中，`/` 是目录分隔符（如 `/home/user/file`）。
*   在查看或操作**目录**时，路径末尾的 `/` 是可选的（`ls /opt` 和 `ls /opt/` 效果相同）。
*   在查看或操作**文件**时，路径末尾**不能**有 `/`，否则系统会误认为它是一个目录而报错。
