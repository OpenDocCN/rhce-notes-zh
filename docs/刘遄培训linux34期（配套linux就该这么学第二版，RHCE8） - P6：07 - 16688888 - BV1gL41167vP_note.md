## 循环语句：`while`循环

![](img/f2c079cf902301385f38c2904e630b03_23.png)

![](img/f2c079cf902301385f38c2904e630b03_25.png)

![](img/f2c079cf902301385f38c2904e630b03_27.png)

![](img/f2c079cf902301385f38c2904e630b03_29.png)

上一节我们介绍了`for`循环语句，本节中我们来看看`while`循环。

![](img/f2c079cf902301385f38c2904e630b03_31.png)

![](img/f2c079cf902301385f38c2904e630b03_33.png)

`for`循环语句是根据一个文件中的用户列表，或者一个IP地址范围来进行循环的。也就是说，`for`循环是根据范围来进行循环的，只要列表或范围内包含信息，就会根据这个列表进行批量循环。

而我们现在要讲的`while`循环语句，并不是根据范围来进行循环，它是根据一个条件来进行循环的。也就是说，只要条件满足，就会一直循环下去。例如，大家可能听说过一种叫做蠕虫病毒的病毒，它会一直消耗系统的计算和存储资源，直到系统崩溃为止。这就是一种循环条件，它会无限制地执行，只要条件为真，就会一直循环下去。只有当条件为假，即条件不成立时，循环才会停止。

![](img/f2c079cf902301385f38c2904e630b03_35.png)

![](img/f2c079cf902301385f38c2904e630b03_37.png)

所以，`while`循环语句的理论基础是：只要条件为真，则一直循环；只有条件为假，循环才会停止。

### 脚本声明与注释

![](img/f2c079cf902301385f38c2904e630b03_59.png)

![](img/f2c079cf902301385f38c2904e630b03_61.png)

首先，回忆一下Shell脚本的编写格式。一个Shell脚本必须有脚本声明，指定由哪个解释器来执行。脚本声明是`#!/bin/bash`，这表示可以通过`bash`解释器来读取、解析和执行这个Shell脚本。

其次，是脚本的注释。注释可以有一行或多行，主要用于解释脚本的功能或某行参数的作用。注释是给人看的，程序不会读取，因此没有语法规则。例如，我们可以写`# This is a test script`。

![](img/f2c079cf902301385f38c2904e630b03_63.png)

### 定义变量

![](img/f2c079cf902301385f38c2904e630b03_65.png)

![](img/f2c079cf902301385f38c2904e630b03_67.png)

![](img/f2c079cf902301385f38c2904e630b03_69.png)

我们将使用一个名为`RANDOM`的变量来生成一个随机数字。这个变量会产生一个随机数，范围可能很大。为了将范围限制在0到1000之间，我们需要使用取余运算符`%`。取余运算会返回除法运算后的余数。例如，`$RANDOM % 1000`会得到一个0到999之间的随机数。

![](img/f2c079cf902301385f38c2904e630b03_77.png)

为了进行计算，我们需要使用`$((...))`语法来执行算术运算，并将结果赋值给一个变量。这与使用反引号`` `...` ``执行命令并返回结果的效果是相同的。

以下是定义变量的步骤：
1.  生成一个0到999的随机数并赋值给变量`price`。
2.  定义一个变量`times`，初始值为0，用于记录猜测次数。

### 构建`while`循环

![](img/f2c079cf902301385f38c2904e630b03_79.png)

我们将设置一个条件永远为真的`while`循环，这意味着循环会一直执行下去，直到遇到退出指令才会结束。

![](img/f2c079cf902301385f38c2904e630b03_81.png)

![](img/f2c079cf902301385f38c2904e630b03_83.png)

![](img/f2c079cf902301385f38c2904e630b03_85.png)

![](img/f2c079cf902301385f38c2904e630b03_89.png)

![](img/f2c079cf902301385f38c2904e630b03_91.png)

![](img/f2c079cf902301385f38c2904e630b03_93.png)

在循环体内，我们将：
1.  使用`read`命令提示用户输入一个数字，并将输入的值赋给一个变量（例如`guess`）。
2.  每次循环时，让`times`变量自增1，以记录猜测次数。
3.  使用条件测试语句`if`来比较用户输入的值`$guess`和随机数`$price`。
    *   如果相等，则输出恭喜信息并显示猜测次数，然后使用`break`退出循环。
    *   如果用户输入的值大于随机数，则提示“太高了”。
    *   如果用户输入的值小于随机数，则提示“太低了”。

### 一次性计划任务：`at`

![](img/f2c079cf902301385f38c2904e630b03_139.png)

![](img/f2c079cf902301385f38c2904e630b03_141.png)

![](img/f2c079cf902301385f38c2904e630b03_143.png)

![](img/f2c079cf902301385f38c2904e630b03_145.png)

![](img/f2c079cf902301385f38c2904e630b03_147.png)

`at`命令用于安排一次性计划任务。例如，我们可以在指定时间执行一个命令。

![](img/f2c079cf902301385f38c2904e630b03_149.png)

![](img/f2c079cf902301385f38c2904e630b03_151.png)

**使用方法：**
1.  使用`at [时间]`来安排任务。
2.  输入要执行的命令。
3.  按`Ctrl + D`保存并退出。

![](img/f2c079cf902301385f38c2904e630b03_153.png)

![](img/f2c079cf902301385f38c2904e630b03_155.png)

![](img/f2c079cf902301385f38c2904e630b03_157.png)

**管理`at`任务：**
*   `at -l`：列出当前的计划任务。
*   `at -c [任务编号]`：查看特定任务的详细信息。
*   `atrm [任务编号]`：删除指定的计划任务。

![](img/f2c079cf902301385f38c2904e630b03_159.png)

![](img/f2c079cf902301385f38c2904e630b03_161.png)

![](img/f2c079cf902301385f38c2904e630b03_163.png)

`at`任务执行一次后就会被删除。

### 周期性计划任务：`cron`

`cron`服务用于安排周期性计划任务。我们通过编辑`crontab`来配置任务。

![](img/f2c079cf902301385f38c2904e630b03_165.png)

**`crontab`格式：**
任务配置遵循“分 时 日 月 星期 命令”的格式。
*   **字段说明：** 分钟 (0-59), 小时 (0-23), 日 (1-31), 月 (1-12), 星期 (0-7，0和7都代表周日)。
*   **特殊字符：**
    *   `*`：代表所有可能的值，用作占位符。
    *   `,`：指定多个值，例如`1,3,5`。
    *   `-`：指定一个范围，例如`1-5`。
    *   `/`：指定间隔频率，例如在分钟字段写`*/2`表示每两分钟。

![](img/f2c079cf902301385f38c2904e630b03_167.png)

![](img/f2c079cf902301385f38c2904e630b03_169.png)

![](img/f2c079cf902301385f38c2904e630b03_171.png)

**配置方法：**
使用`crontab -e`命令来编辑当前用户的计划任务。这个命令会调用默认的文本编辑器（如`vi`），并且具有语法检查功能，比直接编辑`/etc/crontab`文件更安全。

![](img/f2c079cf902301385f38c2904e630b03_173.png)

**示例：**
*   `20 3 * * * /sbin/reboot`：每天凌晨3:20重启系统。
*   `0 */2 * * * /sbin/reboot`：每两小时重启一次系统。

**管理`cron`服务：**
如果计划任务没有生效，请检查`cron`服务是否正在运行：
```bash
systemctl status crond
```
如果服务未运行，使用以下命令启动并设置开机自启：
```bash
systemctl restart crond
systemctl enable crond
```

## 用户身份与权限（第五章引言）

![](img/f2c079cf902301385f38c2904e630b03_181.png)

![](img/f2c079cf902301385f38c2904e630b03_183.png)

![](img/f2c079cf902301385f38c2904e630b03_185.png)

![](img/f2c079cf902301385f38c2904e630b03_187.png)

在Linux系统中，用户身份决定了其权限和能力。用户主要分为三类：
1.  **管理员 (root)**：UID为0，拥有系统的最高管理权限。
2.  **系统用户**：UID通常为1-999（RHEL 7/8），不能登录系统，用于运行系统服务，以限制服务权限，增强安全性。
3.  **普通用户**：UID从1000开始（RHEL 7/8），用于日常工作和系统登录。

![](img/f2c079cf902301385f38c2904e630b03_189.png)

![](img/f2c079cf902301385f38c2904e630b03_191.png)

![](img/f2c079cf902301385f38c2904e630b03_193.png)

![](img/f2c079cf902301385f38c2904e630b03_195.png)

![](img/f2c079cf902301385f38c2904e630b03_197.png)

**注意：** UID只是一个标识符。虽然UID=0一定是管理员，但系统用户和普通用户的UID范围只是惯例，并非强制。一个UID为666的用户也可以是普通用户。

**用户组：**
每个用户都有一个**基本组**（Primary Group），在用户创建时自动生成。用户还可以加入多个**扩展组**（Supplementary Group），以获得额外的文件访问权限。

![](img/f2c079cf902301385f38c2904e630b03_199.png)

![](img/f2c079cf902301385f38c2904e630b03_201.png)

**相关命令：**
*   `id [用户名]`：查看用户信息（UID、GID、所属组）。
*   `useradd [选项] 用户名`：创建新用户。
*   `usermod [选项] 用户名`：修改用户属性。
*   `passwd [用户名]`：设置或修改用户密码。
*   `userdel [-r] 用户名`：删除用户。`-r`选项会同时删除用户的家目录。

![](img/f2c079cf902301385f38c2904e630b03_203.png)

![](img/f2c079cf902301385f38c2904e630b03_205.png)

![](img/f2c079cf902301385f38c2904e630b03_207.png)

**底层原理：**
Linux中“一切皆文件”，用户信息也不例外。主要涉及以下几个文件：
*   `/etc/passwd`：存储用户账户信息。
*   `/etc/shadow`：存储加密后的用户密码。
*   `/etc/group`：存储用户组信息。
通过直接编辑这些文件（需谨慎操作），也可以手动创建用户，这印证了Linux的底层文件管理机制。
