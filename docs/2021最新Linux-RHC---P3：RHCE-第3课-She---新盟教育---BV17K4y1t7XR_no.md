## 脚本编写规范与目的 📝

上一节我们介绍了SSH和Rsync的基础使用。本节中，我们来看看编写Shell脚本的标准化规范。

脚本编写的核心目的是实现自动化运维的第一步，旨在提升工作效率、执行效率和系统容错率。脚本是将重复性、繁琐的工作组织成文档，并让其自动执行。

以下是脚本编写的基本原则：

*   **命名规则**：脚本名称应体现其功能，并以 `.sh` 结尾。
*   **内容结构**：
    1.  第一行指定解释器，例如 `#!/bin/bash`。
    2.  第二至四行注释脚本的创建时间、作者邮箱和主要用途。
    3.  脚本中定义的变量通常使用大写字母。
    4.  函数可以使用小写字母，通常用 `function` 关键字定义。

## 案例一：自动化系统备份脚本 💾

在生产环境中，备份是必不可少的。我们可能需要备份数据库、网站数据、日志等。高效的备份方案通常结合完整备份和增量备份。

### 备份策略与工具

常见的备份周期有日备、周备和月备。以周备为例，通常在访问量最低的时段（如周日凌晨）进行一次完整备份，其他时间则进行增量备份。备份工具可以选择 `tar`、`rsync` 或 `scp`，对于大量数据传输，压缩和异地备份是常见做法。

`tar` 命令的 `-g` 参数可以指定快照文件，用于实现增量备份。

*   **完整备份命令示例**：
    ```bash
    tar -g /path/to/snapshot -czvf backup_full.tar.gz /data/to/backup
    ```
*   **增量备份命令示例**：
    ```bash
    tar -g /path/to/snapshot -czvf backup_incremental_$(date +%Y%m%d).tar.gz /data/to/backup
    ```

### 脚本编写思路

以下是编写自动化备份脚本的核心思路：

1.  **确定备份周期**：按天、周或月进行备份，并使用时间戳命名备份文件。
2.  **定义备份函数**：分别编写完整备份 (`total_backup`) 和增量备份 (`add_backup`) 的函数块。
3.  **逻辑判断**：根据当前是星期几，决定调用哪个备份函数。
4.  **自动化执行**：将脚本加入 `crontab`，实现定时自动执行。

### 关键代码逻辑

![](img/9b3b30465b3de7584957f4df66e70952_5.png)

![](img/9b3b30465b3de7584957f4df66e70952_7.png)

脚本首先获取当前是星期几，然后进行判断。

![](img/9b3b30465b3de7584957f4df66e70952_9.png)

![](img/9b3b30465b3de7584957f4df66e70952_11.png)

```bash
WEEK=$(date +%u) # %u 表示星期几（1-7，1代表星期一）
if [ "$WEEK" -eq 7 ]; then # 假设周日进行完整备份
    total_backup
else
    add_backup
fi
```


## 案例三：拒绝恶意IP登录脚本 🛡️

![](img/9b3b30465b3de7584957f4df66e70952_87.png)

![](img/9b3b30465b3de7584957f4df66e70952_89.png)

此脚本用于增强SSH服务安全，自动分析登录日志，将多次尝试失败（恶意登录）的IP地址加入防火墙黑名单。

![](img/9b3b30465b3de7584957f4df66e70952_91.png)

![](img/9b3b30465b3de7584957f4df66e70952_93.png)

### 实现原理

![](img/9b3b30465b3de7584957f4df66e70952_95.png)

![](img/9b3b30465b3de7584957f4df66e70952_96.png)

SSH的失败登录记录保存在 `/var/log/secure`（CentOS/RHEL）或 `/var/log/auth.log`（Ubuntu/Debian）中。脚本的核心是分析这些日志。

![](img/9b3b30465b3de7584957f4df66e70952_98.png)

1.  **提取失败IP**：使用 `grep` 过滤出 “Failed” 登录行，并用 `awk` 提取IP地址。
    ```bash
    grep "Failed password" /var/log/secure | awk '{print $(NF-3)}' | sort | uniq -c
    ```
    这条命令会统计每个IP的失败次数。
2.  **判断与拦截**：循环遍历上一步的结果，如果某个IP的失败次数超过阈值（例如10次），则使用 `iptables` 命令将其丢弃。
    ```bash
    iptables -A INPUT -s $MALICIOUS_IP -j DROP
    ```
3.  **持久化规则**：根据需要，可以将 `iptables` 规则保存，防止重启后失效。

### 输入验证与逻辑

![](img/9b3b30465b3de7584957f4df66e70952_128.png)

![](img/9b3b30465b3de7584957f4df66e70952_129.png)

![](img/9b3b30465b3de7584957f4df66e70952_131.png)

![](img/9b3b30465b3de7584957f4df66e70952_133.png)

这是一个交互式脚本，需要很强的输入验证逻辑。例如，在修改IP时：

![](img/9b3b30465b3de7584957f4df66e70952_135.png)

![](img/9b3b30465b3de7584957f4df66e70952_136.png)

*   需要验证用户输入的字符串是否符合IP地址格式（如 `192.168.1.100`）。
*   可以使用正则表达式进行匹配验证。
*   脚本应包含一个主菜单 (`case` 语句)，让用户选择要执行的功能。

```bash
function validate_ip() {
    local ip=$1
    local stat=1
    if [[ $ip =~ ^[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}$ ]]; then
        # 进一步验证每个数字是否小于等于255
        OIFS=$IFS
        IFS='.'
        ip=($ip)
        IFS=$OIFS
        [[ ${ip[0]} -le 255 && ${ip[1]} -le 255 && ${ip[2]} -le 255 && ${ip[3]} -le 255 ]]
        stat=$?
    fi
    return $stat
}
```

## 总结 🎯

本节课中我们一起学习了四个企业级Shell脚本的实战案例。从自动化备份到信息收集，从安全防护到系统配置，这些脚本涵盖了运维工作的多个关键方面。

编写脚本的关键在于：
1.  **明确需求**：清楚要自动化什么。
2.  **熟悉系统**：知道所需的信息在系统的哪个文件中，用什么命令获取。
3.  **分步实现**：将大任务拆解为多个小步骤，逐个用命令实现。
4.  **加入逻辑**：使用条件判断、循环和函数，让脚本更智能、健壮。
5.  **注重规范**：良好的注释和结构能让脚本更易维护。
