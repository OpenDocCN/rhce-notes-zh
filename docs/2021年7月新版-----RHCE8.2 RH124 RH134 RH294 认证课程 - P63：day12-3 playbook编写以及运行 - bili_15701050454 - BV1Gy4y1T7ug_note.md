## Playbook 核心概念

![](img/becbc7578fc2feee982d68cc207275dd_3.png)

上一节我们介绍了 Ansible 的临时命令。本节中，我们来看看如何将多个任务组织起来。

Playbook 是 Ansible 的配置、部署和编排语言。它可以将多个任务组织在一起，实现复杂的自动化流程。

*   **Task（任务）**：一个任务对应一个 Ansible 模块的调用，相当于一条临时命令。
*   **Play（剧本段）**：多个任务（Task）的集合。
*   **Playbook（剧本）**：一个 YAML 文件，包含一个或多个 Play。

一个 Playbook 的结构可以概括为：
```
一个 Playbook 包含多个 Play。
一个 Play 包含多个 Task。
```

## YAML 语法基础

Playbook 使用 YAML 格式编写。YAML 是一种对人类友好的数据序列化语言，其语法严谨且易于阅读。

以下是编写 Playbook 时需要了解的 YAML 核心语法规则：

![](img/becbc7578fc2feee982d68cc207275dd_9.png)

*   **文件开头**：通常以三个连字符 `---` 开始。
*   **键值对**：使用冒号 `:` 表示，冒号后面必须有一个空格。例如：`name: Install Apache`
*   **列表**：使用短横线 `-` 表示列表项，每个 `-` 后面要有空格。例如：
    ```yaml
    packages:
      - httpd
      - mariadb
    ```
*   **缩进**：使用空格进行缩进来表示层级关系，同一层级的元素必须左对齐。**强烈建议使用 2 个空格作为缩进**，不要使用 Tab 键。
*   **注释**：使用井号 `#` 开头。
*   **字符串**：通常不需要引号，但如果字符串中包含特殊字符（如冒号），则需要使用单引号或双引号括起来。

## 编写第一个 Playbook

现在，让我们动手编写一个简单的 Playbook。这个 Playbook 的目标是在目标主机上安装、启动 Apache HTTP 服务器，并配置防火墙。

### Playbook 结构解析

![](img/becbc7578fc2feee982d68cc207275dd_13.png)

一个最基本的 Play 包含以下核心元素：

*   `hosts`：指定在哪些主机或主机组上执行任务。
*   `tasks`：定义要执行的任务列表。
*   `become`：是否进行权限提升（例如使用 `sudo`），通常设置为 `true`。

以下是我们的 Playbook 文件 `install_apache.yml` 的内容：

```yaml
---
- name: Configure Apache Web Server
  hosts: web_servers
  become: true
  tasks:
    - name: Install httpd package
      yum:
        name: httpd
        state: latest

    - name: Start and enable httpd service
      service:
        name: httpd
        state: started
        enabled: true

    - name: Open firewall port for http
      firewalld:
        service: http
        permanent: true
        state: enabled
        immediate: yes

    - name: Create a simple index.html
      copy:
        content: "Hello from Ansible!\n"
        dest: /var/www/html/index.html
        owner: apache
        group: apache
        mode: '0644'
```

**代码解释**：
1.  `name: Configure Apache Web Server`：整个 Play 的描述。
2.  `hosts: web_servers`：指定在名为 `web_servers` 的主机组上运行。你需要在 Ansible 的 `inventory` 文件中定义这个组。
3.  `become: true`：以 root 权限运行任务。
4.  在 `tasks` 下，我们定义了四个任务，分别使用 `yum`、`service`、`firewalld` 和 `copy` 模块来完成软件安装、服务管理、防火墙配置和文件创建。

使用 `--syntax-check` 参数可以验证 Playbook 的 YAML 语法是否正确。

![](img/becbc7578fc2feee982d68cc207275dd_19.png)

![](img/becbc7578fc2feee982d68cc207275dd_25.png)

### 模拟运行（试运行）

使用 `-C` 或 `--check` 参数可以进行试运行。Ansible 会模拟执行任务，但不会对目标主机做出任何实际更改。这对于验证 Playbook 逻辑非常有用。

```bash
ansible-playbook install_apache.yml --check
```

## 编写卸载 Playbook

![](img/becbc7578fc2feee982d68cc207275dd_61.png)

![](img/becbc7578fc2feee982d68cc207275dd_63.png)

根据“安装”的逻辑，我们可以轻松编写一个“卸载”的 Playbook `remove_apache.yml`。

![](img/becbc7578fc2feee982d68cc207275dd_65.png)

```yaml
---
- name: Remove Apache Web Server
  hosts: web_servers
  become: true
  tasks:
    - name: Stop httpd service
      service:
        name: httpd
        state: stopped

    - name: Disable httpd service
      service:
        name: httpd
        enabled: false

    - name: Remove httpd package
      yum:
        name: httpd
        state: absent

    - name: Close firewall port for http
      firewalld:
        service: http
        permanent: true
        state: disabled
        immediate: yes

    - name: Remove index.html
      file:
        path: /var/www/html/index.html
        state: absent
```

![](img/becbc7578fc2feee982d68cc207275dd_67.png)

**注意**：卸载任务通常按照停止服务 -> 禁用服务 -> 删除软件 -> 清理配置/文件的顺序进行。

## 权限提升（Privilege Escalation）

![](img/becbc7578fc2feee982d68cc207275dd_77.png)

![](img/becbc7578fc2feee982d68cc207275dd_79.png)

![](img/becbc7578fc2feee982d68cc207275dd_81.png)

![](img/becbc7578fc2feee982d68cc207275dd_83.png)

在 Playbook 中，我们使用 `become: true` 来提权。`become` 关键字提供了更精细的权限控制：

![](img/becbc7578fc2feee982d68cc207275dd_85.png)

![](img/becbc7578fc2feee982d68cc207275dd_86.png)

*   `become: true`：启用权限提升，默认使用 `sudo` 方式并切换到 `root` 用户。
*   `become_user: <username>`：指定要切换到的用户，例如 `become_user: apache`。
*   `become_method: <method>`：指定提权方式，如 `sudo`、`su`、`doas` 等。

![](img/becbc7578fc2feee982d68cc207275dd_88.png)

![](img/becbc7578fc2feee982d68cc207275dd_90.png)

如果已经在 Ansible 的配置文件 (`ansible.cfg`) 中全局设置了提权，那么在 Playbook 中可以省略 `become` 设置。但如果 Playbook 需要不同的权限，则必须在 Playbook 中明确声明。

![](img/becbc7578fc2feee982d68cc207275dd_92.png)

![](img/becbc7578fc2feee982d68cc207275dd_94.png)

## 实验与总结

![](img/becbc7578fc2feee982d68cc207275dd_100.png)

![](img/becbc7578fc2feee982d68cc207275dd_102.png)

本节课中我们一起学习了 Ansible Playbook 的核心知识。我们了解了 Playbook 与临时命令的关系，学习了 YAML 的基本语法，并亲手编写了用于安装和卸载 Apache 的完整 Playbook。

![](img/becbc7578fc2feee982d68cc207275dd_104.png)

![](img/becbc7578fc2feee982d68cc207275dd_106.png)

![](img/becbc7578fc2feee982d68cc207275dd_108.png)

**关键点总结**：
1.  Playbook 是 Ansible 自动化的核心，用于编排复杂任务。
2.  YAML 语法强调缩进和键值对结构，编写时需格外注意格式。
3.  一个 Play 至少包含 `hosts` 和 `tasks`。
4.  使用 `--syntax-check` 和 `--check` 参数可以在运行前检查和测试 Playbook。
5.  通过 `become` 关键字管理任务执行权限。
