## 条件判断：`when`语句 ⚖️

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_27.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_29.png)

判断任务的核心是`when`语句。它写在任务模块之后，用于指定任务执行的条件。只有当条件为真（`True`）时，该任务才会被执行。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_31.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_33.png)

以下是`when`语句的几种常见用法。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_35.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_37.png)

### 1. 基于主机名判断

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_38.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_39.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_41.png)

我们可以根据Ansible的魔法变量`inventory_hostname`来判断任务在哪些主机上执行。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_43.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_45.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_46.png)

**示例剧本**：
```yaml
---
- name: 条件判断示例一
  hosts: all
  tasks:
    - name: 收集事实
      setup:
    - name: 为特定主机创建文件
      copy:
        content: "此文件仅创建于node1\n"
        dest: /tmp/test.txt
      when: inventory_hostname == "node1.lab.example.com"
```
在这个例子中，`copy`任务只会在主机名为`node1.lab.example.com`的主机上执行。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_48.png)

### 2. 判断变量是否在列表中

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_50.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_52.png)

我们可以使用`in`关键字来判断一个变量的值是否存在于某个列表中。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_54.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_55.png)

**示例剧本**：
```yaml
---
- name: 条件判断示例二
  hosts: all
  vars:
    test: "ansible_test"
    one_list:
      - "redhat"
      - "centos"
      - "ansible_test"
  tasks:
    - name: 收集事实
      setup:
    - name: 如果变量在列表中则创建文件
      copy:
        content: "条件成立，创建文件。\n"
        dest: /tmp/test2.txt
      when: test in one_list
```
这里定义了一个变量`test`和一个列表`one_list`。`when: test in one_list`会检查`test`的值是否在`one_list`中，只有条件成立，才会创建文件。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_57.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_59.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_60.png)

**重要提示**：`when`语句应该与任务模块的**参数**保持**同级缩进**，而不是写在模块参数内部。

### 列表类型变量

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_84.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_85.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_87.png)

列表变量包含一系列值，在循环中使用`item`关键字引用当前项。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_89.png)

**示例剧本**：
```yaml
---
- name: 列表变量循环示例
  hosts: all
  vars:
    package_list:
      - httpd
      - mariadb-server
      - php
  tasks:
    - name: 安装软件包列表
      yum:
        name: "{{ item }}"
        state: present
      loop: "{{ package_list }}"
```
在这个任务中，`loop`会遍历`package_list`，每次循环的当前值通过`{{ item }}`引用。

### 字典类型变量

字典变量包含键值对。在循环字典时，需要使用`dict2items`过滤器将其转换为列表，然后通过`item.key`和`item.value`来引用。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_91.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_93.png)

**示例剧本**：
```yaml
---
- name: 字典变量循环示例
  hosts: all
  vars:
    user_info:
      alice:
        name: "Alice"
        shell: "/bin/bash"
      bob:
        name: "Bob"
        shell: "/sbin/nologin"
  tasks:
    - name: 创建用户
      user:
        name: "{{ item.value.name }}"
        shell: "{{ item.value.shell }}"
      loop: "{{ user_info | dict2items }}"
```
这里，`dict2items`将`user_info`字典转换成了可以循环的列表，每个`item`包含`key`（如`alice`）和`value`（如`{"name": "Alice", "shell": "/bin/bash"}`）。

## 注册结果与循环的结合 📥

我们可以使用`register`关键字将一个任务（尤其是循环任务）的执行结果保存到变量中，然后在后续任务中调用和分析这些结果。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_105.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_107.png)

以下是注册结果与循环结合的一个复杂示例。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_109.png)

**示例剧本**：
```yaml
---
- name: 注册结果与循环结合示例
  hosts: all
  tasks:
    - name: 执行多条命令并注册结果
      shell: "{{ item }}"
      ignore_errors: yes
      loop:
        - "ls -ld /tmp"
        - "ls -ld /tmp/nonexistent"
      register: test_shell

    - name: 仅输出成功执行命令的结果
      debug:
        var: item
      loop: "{{ test_shell.results }}"
      when: item.rc == 0
```

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_111.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_113.png)

**代码解析**：
1.  **第一个任务**：使用`shell`模块循环执行两条命令。`ignore_errors: yes`确保即使某条命令失败（如`ls`一个不存在的目录），剧本也会继续执行。所有命令的执行结果被注册到变量`test_shell`中。
2.  **第二个任务**：使用`debug`模块输出信息。它循环遍历`test_shell.results`（这是注册结果中存放每个命令详细结果的列表）。`when: item.rc == 0`是一个判断条件，它检查每个结果的返回码（`rc`），只输出返回码为0（即成功执行）的命令的详细信息。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_115.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_117.png)

**核心概念**：
*   `item`：在`loop`循环中，它代表当前被遍历到的元素。其具体内容取决于循环的对象。
    *   在第一个任务的循环中，`item`是具体的命令字符串（如`"ls -ld /tmp"`）。
    *   在第二个任务的循环中，`item`是第一个任务注册结果`test_shell.results`列表中的每一项，它是一个包含了`cmd`、`stdout`、`stderr`、`rc`等键的字典。
*   `rc`：代表“Return Code”，是命令执行后的状态返回码。`0`通常表示成功，非`0`表示失败。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_118.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_120.png)

通过这种方式，我们可以精细地筛选和处理循环中每个步骤的执行结果。

## 综合练习：安装并测试HTTPD服务 🌐

现在，让我们运用所学的判断、循环和变量知识，来完成一个综合练习。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_139.png)

**目标**：编写一个剧本，在节点上安装`httpd`（Apache）服务，启动它，并测试服务是否能够正常访问。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_141.png)

**思路与步骤**：
1.  使用`yum`模块安装`httpd`软件包。
2.  使用`systemd`模块启动`httpd`服务，并设置为开机自启。
3.  使用`uri`模块（或`shell`模块调用`curl`）来测试本地HTTP服务是否返回成功状态码（如200）。
4.  可以使用`register`保存测试结果，并用`debug`模块输出测试是否成功的信息。

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_143.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_144.png)

![](img/bdb8ccf43be22ad7268f3fc602c54b7a_146.png)

**提示**：
*   安装和启动服务是“改变系统状态”的任务，通常需要`become: yes`来提权。
*   测试任务可以在安装启动之后进行。
*   使用`uri`模块时，需要检查返回的`status`值。
