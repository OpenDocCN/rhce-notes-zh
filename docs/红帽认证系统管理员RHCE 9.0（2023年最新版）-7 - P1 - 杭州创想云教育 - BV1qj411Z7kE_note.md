# RHCE 9.0 课程：第7章：软件仓库管理与系统工具

## 概述
在本节课中，我们将学习如何启用和管理 DNF 软件仓库，识别系统设备与文件系统，以及使用查找文件等系统管理工具。这些是系统管理员日常工作中的核心技能。

## 启用 DNF 软件仓库

上一节我们介绍了软件包的基本管理，本节中我们来看看如何配置软件源。

![](img/70b1b51fae218632feb039c766538789_1.png)

![](img/70b1b51fae218632feb039c766538789_3.png)

软件包可以来源于本地文件、互联网或特定的文件存储服务器。直接使用 RPM 安装可能会遇到依赖问题。更高效的方法是使用配置好的软件仓库。

![](img/70b1b51fae218632feb039c766538789_5.png)

![](img/70b1b51fae218632feb039c766538789_7.png)

红帽通过 CDN（内容分发网络）提供官方的软件仓库。客户端系统（如 RHEL 9）可以订阅并访问这些仓库来安装软件。此外，也存在第三方公共镜像仓库，如阿里云、中科大的 EPEL 仓库。

在生产环境中，常见的做法是：
1.  使用红帽的卫星服务器同步 CDN 的软件包，再分发给内部服务器。
2.  自行搭建 HTTP 或 FTP 服务器，将所需软件包同步到本地，并配置为仓库供内部客户端使用。

在我们的实验环境中，仓库服务器已由 `classroom` 主机搭建好。

### 配置仓库的方法

![](img/70b1b51fae218632feb039c766538789_13.png)

![](img/70b1b51fae218632feb039c766538789_14.png)

以下是配置软件仓库的两种主要方法。

#### 方法一：使用 `dnf config-manager` 命令
此方法通过命令直接添加仓库，但生成的文件名较长。

**关键步骤：确定仓库地址**
正确的仓库地址是能看到 `repodata` 目录的路径。例如，对于路径 `.../errata/`，如果其下包含 `Packages` 和 `repodata` 目录，则仓库地址应写到 `errata` 这一级。

添加仓库的命令格式为：
```bash
dnf config-manager --add-repo <仓库地址URL>
```
添加后，仓库配置文件会保存在 `/etc/yum.repos.d/` 目录下，文件以 `.repo` 结尾。可以将其重命名为更易识别的名称。

![](img/70b1b51fae218632feb039c766538789_16.png)

![](img/70b1b51fae218632feb039c766538789_17.png)

使用 `dnf repolist` 命令可以列出已启用的仓库。使用 `dnf search` 或 `dnf info` 可以验证仓库中的软件包。

#### 方法二：手动编辑 `.repo` 文件
建议掌握此方法，以便更灵活地管理仓库配置。

![](img/70b1b51fae218632feb039c766538789_19.png)

![](img/70b1b51fae218632feb039c766538789_21.png)

仓库配置文件通常包含以下核心字段：
*   `[repository-id]`: 仓库的唯一标识符。
*   `name`: 仓库的描述信息。
*   `baseurl`: 仓库的地址。
*   `gpgcheck`: 是否进行 GPG 签名检查（`1` 为检查，`0` 为不检查）。
*   `gpgkey`: GPG 公钥的地址（当 `gpgcheck=1` 时需要）。

一个简单的手动配置示例如下：
```bash
[RHCSA-practice-errata]
name = RHCSA Practice Errata
baseurl = http://classroom.example.com/content/rhel9.0/x86_64/rhcsa-practice/errata
gpgcheck = 0
```
配置文件创建后，运行 `dnf repolist` 和 `dnf makecache` 来更新仓库列表和缓存。

![](img/70b1b51fae218632feb039c766538789_23.png)

![](img/70b1b51fae218632feb039c766538789_25.png)

![](img/70b1b51fae218632feb039c766538789_26.png)

![](img/70b1b51fae218632feb039c766538789_27.png)

**关于 GPG 检查**
对于官方或公共仓库，建议启用 GPG 检查 (`gpgcheck=1`) 并指定 `gpgkey`，以确保软件包完整性和来源可信。对于自建的内网仓库，可以关闭此项检查 (`gpgcheck=0`)。

### 文件系统与挂载
文件系统用于组织和存储数据，常见的 Linux 文件系统有 ext4, XFS, Btrfs 等。RHEL 7 之后默认使用 XFS。

![](img/70b1b51fae218632feb039c766538789_37.png)

![](img/70b1b51fae218632feb039c766538789_38.png)

![](img/70b1b51fae218632feb039c766538789_39.png)

在 Linux 中，用户不能像在 Windows 中那样直接访问磁盘设备。需要将文件系统与一个**空目录**关联起来，这个过程称为**挂载**。对该目录的读写操作即是对所关联文件系统的操作。

### 设备识别
文件系统创建于磁盘设备或其分区之上。Linux 设备文件位于 `/dev` 目录下。

![](img/70b1b51fae218632feb039c766538789_41.png)

![](img/70b1b51fae218632feb039c766538789_42.png)

可以使用 `lsblk` 命令清晰地列出所有块设备（磁盘和分区）：
```bash
lsblk
```
也可以使用 `ls -l /dev/disk/by-*` 通过不同属性（如路径、UUID）查看设备。

![](img/70b1b51fae218632feb039c766538789_44.png)

![](img/70b1b51fae218632feb039c766538789_45.png)

设备命名规则取决于其接口类型：
*   **/dev/sd[a-z]**: SATA, SAS, USB 接口的磁盘。
*   **/dev/vd[a-z]**: 虚拟化环境（如 KVM）中的 VirtIO 块设备。
*   **/dev/nvme[0-9]n[1-9]**: NVMe 固态硬盘。
设备名后的数字（如 `sda1`, `sda2`) 表示分区编号。

### 查看挂载与磁盘使用情况
`mount` 命令（不加参数）可以列出当前所有挂载项。

`df` 命令用于报告文件系统的磁盘空间使用情况。
常用选项：
*   `-h`: 以易读的单位（如 G, M）显示（1024进制）。
*   `-H`: 以易读的单位显示（1000进制）。
*   `-T`: 同时显示文件系统类型。

`du` 命令用于估算文件和目录的磁盘使用量。
常用选项：
*   `-s`: 仅显示总计。
*   `-h`: 以易读格式显示。

## 挂载和卸载文件系统

![](img/70b1b51fae218632feb039c766538789_48.png)

### 挂载命令
挂载的基本语法是：
```bash
mount [选项] <设备名或UUID> <挂载点目录>
```
例如：
```bash
mount /dev/vdb1 /mnt
```
更推荐使用 UUID 进行挂载，因为设备名可能变动，而 UUID 是唯一的。使用 `blkid <设备名>` 可以查看设备的 UUID。
```bash
mount UUID="xxxx-xxxx-..." /mnt
```

![](img/70b1b51fae218632feb039c766538789_53.png)

![](img/70b1b51fae218632feb039c766538789_55.png)

![](img/70b1b51fae218632feb039c766538789_57.png)

![](img/70b1b51fae218632feb039c766538789_59.png)

### 卸载命令
卸载使用 `umount` 命令，后跟设备名或挂载点：
```bash
umount /mnt
```
如果卸载时提示“设备忙”，可以使用 `lsof +D /mnt` 或 `lsof | grep /mnt` 查看哪个进程正在使用该挂载点，结束进程后再尝试卸载。

![](img/70b1b51fae218632feb039c766538789_61.png)

图形化界面下，可移动介质通常会自动挂载到 `/run/media/<用户名>/<卷标>` 目录下。

## 查找文件

上一节我们处理了存储设备，本节中我们来看看如何在系统中高效地查找文件。

![](img/70b1b51fae218632feb039c766538789_69.png)

![](img/70b1b51fae218632feb039c766538789_71.png)

### locate 命令
`locate` 命令基于预建的数据库进行快速查找，效率很高。
```bash
locate filename
```
如果文件是新建的，可能需要先更新数据库：`sudo updatedb`。数据库默认由计划任务每日更新。

### find 命令
`find` 命令进行实时搜索，功能强大但语法相对复杂。
基本语法：
```bash
find [路径] [选项] [操作]
```
*   **路径**：指定搜索的起始目录，默认为当前目录。
*   **选项**：定义搜索条件。
*   **操作**：对找到的结果执行的动作。

**常用搜索条件示例：**
*   按名称查找：`find / -name "ssh_config"`
*   按类型查找：`find /home -type f -user student` (查找 student 用户的普通文件)
*   按大小查找：`find / -type f -size +200M` (查找大于 200MB 的文件)
*   按权限查找：`find / -type f -perm 4755` (查找 SUID 位被设置且权限为 755 的文件)

![](img/70b1b51fae218632feb039c766538789_73.png)

![](img/70b1b51fae218632feb039c766538789_74.png)

**对查找结果执行操作**
使用 `-exec` 选项可以对找到的每个文件执行一个命令。`{}` 代表找到的文件名，`\;` 表示命令结束。
例如，将找到的所有属于 `devops` 用户的文件复制到 `/tmp`：
```bash
find / -user devops -type f -exec cp {} /tmp \;
```
例如，验证找到的文件的权限：
```bash
find / -type f -perm 4755 -exec ls -l {} \;
```

### Cockpit Web 控制台
从 RHEL 8 开始，系统提供了基于 Web 的管理工具 Cockpit（端口 9090）。它可以用于监控系统状态、查看日志、管理服务、用户和软件更新等，适合初学者。

![](img/70b1b51fae218632feb039c766538789_95.png)

![](img/70b1b51fae218632feb039c766538789_97.png)

安装与启用：
```bash
sudo dnf install cockpit
sudo systemctl enable --now cockpit.socket
sudo firewall-cmd --add-service=cockpit --permanent
sudo firewall-cmd --reload
```
然后通过浏览器访问 `https://<服务器IP>:9090` 即可。

![](img/70b1b51fae218632feb039c766538789_99.png)

![](img/70b1b51fae218632feb039c766538789_101.png)

## 红帽认证考试与账户注册流程

### 学习与考试流程概述
1.  **课程学习**：完成 RHCSA 和 RHCE 课程内容。
2.  **课后练习**：课程期间及结束后，按要求练习 EX200 和 EX294 的题目。
3.  **考前辅导**：在充分练习后，参加考前辅导进行查漏补缺。
4.  **模拟考试**：每人有一次模拟考试机会，用于评估准备情况。
5.  **正式考试**：根据准备情况决定是否参加正式考试。

### 账户注册关键步骤
1.  **注册红帽账户 (RHN)**：
    *   访问 `redhat.com`，点击登录/注册。
    *   选择“个人”账户类型。
    *   **重要**：`登录名`不要使用邮箱，使用自定义英文名。`姓名`按“名 First Name + 姓 Last Name”格式填写英文。`国家`选择 China，`电话`加国家代码 86。
    *   使用**个人常用邮箱**接收激活邮件并完成激活。
2.  **注册 Credly 账户**：
    *   访问 `credly.com` 注册账户。
    *   **重要**：注册使用的`邮箱`应与红帽账户邮箱**一致**。`姓名`使用英文。
    *   激活账户。
3.  **关联与下载证书**：
    *   考试通过约3个工作日后，红帽会发邮件通知。
    *   登录红帽网站，在个人资料中“关联证书”。
    *   在隐私设置中，允许“红帽认证专业人才搜索”。
    *   Credly 会收到徽章推送邮件，接受后即可在 Credly 账户中看到徽章。
    *   在 Credly 的徽章页面，可选择“共享”->“下载证书”来获取 PDF 版证书。

![](img/70b1b51fae218632feb039c766538789_107.png)

### 常见问题与注意事项
*   **姓名**：全程使用**英文**，否则证书可能出现乱码。
*   **多邮箱**：若有其他认证（如 CKA）使用不同邮箱，可在 Credly 账户设置中添加该邮箱。
*   **账户被禁用**：若 Credly 账户无故被禁用，可能是触发了制裁名单关键词匹配。可通过网站底部 `Support` 提交申诉请求，说明情况并附上身份证明（如护照）。
*   **未收到徽章**：检查红帽账户邮箱是否正确，或在红帽网站提交申诉请求重新发送徽章链接。
*   **考试证件**：参加线下考试务必携带有效身份证件。
