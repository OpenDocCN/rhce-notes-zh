## 进程状态监控：top命令详解

上一节我们介绍了基础的进程概念，本节中我们来看看如何使用`top`命令动态监控系统进程状态。`top`命令提供了一个实时更新的系统概览。

运行`top`命令后，界面顶部会显示多行系统摘要信息。

第一行显示系统的基本信息：
*   **当前系统时间**
*   **系统已运行时间**
*   **当前登录的用户会话数**
*   **系统在过去1、5、15分钟内的平均负载**

*   **总进程数**

*   **休眠（`sleeping`）的进程数**
*   **停止（`stopped`）的进程数**

![](img/874abd1773ec862ce8a08ac88ee809ac_22.png)

僵尸进程是指已经终止但其父进程尚未对其进行善后处理的进程。如果发现僵尸进程，通常需要找到并结束其父进程。

![](img/874abd1773ec862ce8a08ac88ee809ac_26.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_27.png)

第三行显示CPU的使用情况，其中需要重点关注的是 **`id`（空闲）百分比**。如果这个值持续过低，表明CPU非常繁忙，需要进一步排查是哪个进程消耗了大量资源。

第四行和第五行分别显示物理内存（`Mem`）和交换分区（`Swap`）的使用情况。不过，查看内存更推荐使用 `free -h` 命令，因为它以更易读的单位（如GB、MB）显示信息。

在`top`的交互界面中，可以使用以下快捷键对进程列表进行排序，以便快速定位资源消耗大户：
*   按 **`P`** （大写）：根据CPU使用率降序排列。
*   按 **`M`** （大写）：根据内存使用率降序排列。
*   按 **`q`** ：退出`top`命令。

## 进程检索：pgrep命令

![](img/874abd1773ec862ce8a08ac88ee809ac_33.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_35.png)

除了动态监控，我们有时需要精确查找特定进程。`pgrep`命令可以根据进程名、用户等信息快速检索进程ID（PID）。

以下是`pgrep`命令的常用选项示例：
*   `pgrep sshd`：查找所有名为`sshd`的进程。
*   `pgrep -l sshd`：查找名为`sshd`的进程并显示进程名和PID。
*   `pgrep -u test`：查找属于用户`test`的所有进程。
*   `pgrep -t pts/1`：查找在终端`pts/1`上运行的所有进程。
*   `pgrep -x mysqld`：精确匹配进程名为`mysqld`的进程（而非包含该字符串的进程）。

![](img/874abd1773ec862ce8a08ac88ee809ac_37.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_39.png)

`pgrep`命令返回的信息较为简洁，适用于快速获取PID。如果需要更详细的信息，可以结合`ps aux | grep`命令使用。

## 进程的前后台调度

在Linux中，进程可以在前台或后台运行。前台进程会占用当前终端，直到其结束；而后台进程则在“背后”运行，不占用终端，允许用户继续输入其他命令。

![](img/874abd1773ec862ce8a08ac88ee809ac_49.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_50.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_52.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_54.png)

将一个命令放入后台运行，只需在命令末尾加上 **`&`** 符号。
```bash
sleep 300 &
```
这条命令会让系统休眠300秒，但由于加了`&`，它会在后台运行，不会阻塞当前终端。

要查看当前用户放入后台的作业（进程），可以使用`jobs -l`命令。

![](img/874abd1773ec862ce8a08ac88ee809ac_56.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_58.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_60.png)

我们还可以对后台进程进行调度：
*   `fg %1`：将后台编号为1的作业调回**前台**运行。
*   按下 **`Ctrl + Z`**：将当前**前台**运行的进程**挂起**（暂停）。
*   `bg %1`：将后台编号为1的**挂起**的作业在**后台恢复**运行。

如果想结束在前台运行的进程，可以按下 **`Ctrl + C`**。

## 杀死进程：kill、pkill、killall

有时我们需要手动结束进程。根据不同的需求，有多个命令可供选择。

![](img/874abd1773ec862ce8a08ac88ee809ac_62.png)

**1. kill命令：根据PID结束进程**
`kill`命令通过向进程发送信号来工作。默认信号是`15`（`SIGTERM`），即请求进程正常终止。
```bash
kill 1234        # 优雅地结束PID为1234的进程
kill -9 1234     # 强制结束PID为1234的进程（发送SIGKILL信号）
```

![](img/874abd1773ec862ce8a08ac88ee809ac_64.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_66.png)

**2. pkill命令：根据进程名结束进程**
`pkill`命令可以根据进程名称来结束进程，适用于结束多个同名进程。
```bash
pkill sleep      # 结束所有名为sleep的进程
pkill -9 -u test # 强制结束用户test的所有进程
```

![](img/874abd1773ec862ce8a08ac88ee809ac_68.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_69.png)

**3. killall命令：根据进程名结束进程（更彻底）**
`killall`与`pkill`类似，但通常结束得更彻底。它同样支持信号。
```bash
killall sleep    # 结束所有名为sleep的进程
killall -9 sleep # 强制结束所有名为sleep的进程
```

**4. pkill按终端结束进程**
如果想结束特定终端上的所有进程，可以使用`pkill -t`。
```bash
pkill -9 -t pts/3 # 强制结束在终端pts/3上运行的所有进程
```

![](img/874abd1773ec862ce8a08ac88ee809ac_71.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_73.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_75.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_77.png)

![](img/874abd1773ec862ce8a08ac88ee809ac_79.png)

**关于信号：**
*   **`-15` （`SIGTERM`）**：默认信号，通知进程自行清理并退出，是“优雅”的结束方式。
*   **`-9` （`SIGKILL`）**：强制信号，立即终止进程，不给进程任何清理的机会。通常在进程无响应时使用。

## 用户登录分析

了解谁正在或曾经登录系统，是系统安全管理的重要部分。

**查看当前已登录用户：**
*   `users`：仅显示当前登录的用户名列表，信息最简洁。
*   `who`：显示更详细的信息，包括用户名、终端、登录时间及来源IP地址。
*   `w`：显示最详细的信息，包括`who`命令的所有信息，以及用户当前正在执行的命令和系统负载（类似`top`的第一行）。

![](img/874abd1773ec862ce8a08ac88ee809ac_87.png)

**查看历史登录记录：**
*   `last`：查看**历史登录成功**的记录，包括用户名、终端、登录时间、时长和来源IP。
*   `lastb`：查看**历史登录失败**的记录。这对于发现恶意登录尝试非常有用。通常需要root权限查看。
