# Linux权限管理：2.08：附加权限 📚

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_1.png)

在本节课中，我们将要学习Linux系统中的附加权限。上一节我们介绍了基本的文件权限和归属，以及用于解决归属分类过于笼统的访问控制列表（ACL）。本节中我们来看看，除了基本的读写执行（RWX）权限外，Linux还提供了一些特殊的附加权限，它们能实现更精细的访问控制。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_3.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_4.png)

## 概述
Linux的基本权限模型（用户、组、其他人）有时无法满足复杂的需求。除了使用ACL进行补充，系统还提供了三种特殊的附加权限：Set UID、Set GID和粘滞位。这些权限通过修改文件或目录执行位（X）的特殊标记来实现，赋予程序或目录额外的能力。本节课我们将重点学习Set GID权限，并完成一个创建共用目录的实践题目。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_6.png)

## 特殊权限的引入与Set UID 🔐
Linux文档的权限标记只有9个位置（用户、组、其他人的RWX），当需要赋予超出基本RWX的特殊能力时，就需要用到附加权限。它们通过改变执行位（X）的显示来实现。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_8.png)

第一种特殊权限是**Set UID**。当它被设置在一个**可执行程序文件**上时，任何用户在执行该程序期间，会临时获得该程序**所有者**的权限。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_10.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_12.png)

**公式/代码表示：**
*   基本权限标记变化：用户的 `x` -> `s`
*   设置命令：`chmod u+s 文件名`

一个典型的例子是 `/usr/bin/passwd` 命令。普通用户没有权限直接修改存储密码的 `/etc/shadow` 文件，但`passwd`命令被设置了Set UID权限（属于root）。因此，当普通用户执行`passwd`时，便能临时获得root权限，从而合法地修改自己的密码。

如果移除此权限（`chmod u-s /usr/bin/passwd`），普通用户将无法成功修改密码。此权限需谨慎使用，考试中不常考，但有助于理解附加权限的概念。

## 核心考点：Set GID 权限 🗂️
第二种特殊权限是**Set GID**。它有两种应用场景：

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_14.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_16.png)

1.  **设置在程序文件上**：类似于Set UID，但执行者会临时获得该文件**所属组**的权限。
2.  **设置在目录上（重点考点）**：当一个目录被设置Set GID权限后，任何用户在此目录下创建的新文件或子目录，其**所属组会自动继承该目录的所属组**，而非创建者默认的主组。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_18.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_20.png)

**公式/代码表示：**
*   基本权限标记变化：组的 `x` -> `s`
*   设置命令：`chmod g+s 目录名`

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_22.png)

这个功能对于创建团队协作的共用目录至关重要。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_24.png)

## 实践：创建共用目录 🛠️
以下是创建共用目录的典型需求与操作步骤，其中就用到了Set GID权限。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_26.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_28.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_30.png)

**需求描述：**
1.  在 `/home` 下创建目录 `tools`。
2.  目录的所属组应为 `admins` 组。
3.  `admins` 组的成员对该目录拥有读、写、执行访问权限。
4.  其他用户（除root外）没有任何权限。
5.  在此目录下创建的任何文件，其所属组自动设置为 `admins` 组。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_32.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_34.png)

**操作步骤：**

1.  **创建目录**
    ```bash
    mkdir /home/tools
    ```

2.  **更改目录归属**
    将目录的所属组改为 `admins`。
    ```bash
    chown :admins /home/tools
    ```

3.  **设置基本权限**
    为所属组（`admins`）添加读写执行（rwx）权限，并移除其他人（other）的所有权限。
    ```bash
    chmod g+rwx,o-rwx /home/tools
    ```
    *（此时权限应为 `drwxrwx---`，所属组为 `admins`）*

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_36.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_38.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_40.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_42.png)

4.  **设置Set GID附加权限**
    这是实现“自动继承组”功能的关键。为目录的组权限位添加 `s` 标记。
    ```bash
    chmod g+s /home/tools
    ```
    *（此时权限变为 `drwxrws---`）*

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_44.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_46.png)

**验证：**
以root身份在 `/home/tools` 下创建文件，该文件的所属组将自动变为 `admins`，而不是root的默认组。
```bash
touch /home/tools/testfile
ls -l /home/tools/testfile
```

## 权限的数字表示法 🔢
权限可以用八进制数字简写，方便批量设置。
*   `r` = 4， `w` = 2， `x` = 1
*   用户权限 = 三者之和，组权限 = 三者之和，其他人权限 = 三者之和
*   例如：`rwxr-xr--` 表示为 `751` (4+2+1=7, 4+0+1=5, 4+0+0=4)

附加权限也有对应的数字，放在最前面：
*   Set UID = 4
*   Set GID = 2
*   粘滞位 = 1

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_48.png)

因此，要设置一个权限为 `rwxrws---` 的目录，其数字表示为 `2770`。
```bash
chmod 2770 /home/tools
```

## 补充知识：粘滞位 (Sticky Bit) 🍬
第三种特殊权限是**粘滞位**，通常设置在**目录**的其他人执行位（X）上。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_50.png)

**公式/代码表示：**
*   基本权限标记变化：其他人的 `x` -> `t`
*   设置命令：`chmod o+t 目录名` 或 数字表示时加前缀 `1`（如 `1777`）

它的作用是：在具有粘滞位的目录中，用户只能删除或重命名**自己创建**的文件和目录，即使他拥有该目录的写权限。这常用于临时目录（如 `/tmp`），以维持公共目录的秩序，防止用户随意删除他人的文件。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_52.png)

## 总结
本节课中我们一起学习了Linux的三种附加权限：
1.  **Set UID**：使程序执行者临时获得文件所有者的权限。
2.  **Set GID（重点）**：使目录下新建文件的所属组自动继承目录的所属组，这是创建团队共用目录的关键。
3.  **粘滞位**：保护公共目录中的文件，防止被非所有者删除。

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_54.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_56.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_58.png)

![](img/0898b7d0251f83a0dd32d4eb0f3898e2_60.png)

我们还掌握了使用 `chmod` 命令的符号法（`u+s`， `g+s`， `o+t`）和数字法（如 `2770`）来设置这些权限。理解并熟练运用Set GID权限，是解决Linux系统下协作目录管理问题的核心技能。
