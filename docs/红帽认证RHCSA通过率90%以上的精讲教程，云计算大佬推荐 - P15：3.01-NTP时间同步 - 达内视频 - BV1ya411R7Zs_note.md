## 理解NTP时间同步

![](img/c7e2df03655071cc2b75ed3781699b79_11.png)

上一节我们介绍了系统基础命令和配置，本节中我们来看看如何配置时间同步。

![](img/c7e2df03655071cc2b75ed3781699b79_13.png)

NTP（Network Time Protocol）是一种网络时间协议。通俗地说，通过这个协议，我们可以将一台标准时间服务器的时钟推送给网络中的其他计算机。提供标准时间的机器称为NTP服务器，使用该标准时间的机器称为NTP客户端。时间同步的目标是让所有客户端与服务器的时间保持一致。

在红帽系统（RHEL 8及以后版本）中，用于时间同步的服务已经从传统的`ntpd`更换为`chrony`。`chrony`软件包对应的系统服务是`chronyd`，它既可以作为NTP服务器，也可以作为NTP客户端。本课程主要关注如何配置客户端。

## 配置NTP客户端

![](img/c7e2df03655071cc2b75ed3781699b79_15.png)

![](img/c7e2df03655071cc2b75ed3781699b79_17.png)

配置NTP客户端通常包含三个核心步骤：安装软件包、修改配置文件、启动并启用服务。以下是详细的操作流程。

![](img/c7e2df03655071cc2b75ed3781699b79_19.png)

### 1. 安装chrony软件包

首先，需要确认系统是否已安装`chrony`软件包。在大多数情况下，系统默认已安装。如果未安装，可以使用YUM包管理器进行安装。

```bash
yum -y install chrony
```

### 2. 修改配置文件

![](img/c7e2df03655071cc2b75ed3781699b79_21.png)

![](img/c7e2df03655071cc2b75ed3781699b79_23.png)

`chrony`的主要配置文件位于`/etc/chrony.conf`。我们需要修改此文件，指定要同步的NTP服务器地址。

![](img/c7e2df03655071cc2b75ed3781699b79_25.png)

使用文本编辑器（如`vim`）打开配置文件：
```bash
vim /etc/chrony.conf
```

在配置文件中，找到以`pool`或`server`开头的行。默认配置可能指向公共NTP服务器池（如`pool.ntp.org`）。根据考试或实际需求，需要将其修改为指定的NTP服务器域名。

![](img/c7e2df03655071cc2b75ed3781699b79_27.png)

**配置语法说明：**
*   **`pool`**：后跟一个域名，该域名在DNS上通常解析为多个IP地址（服务器池）。适用于有多个冗余服务器的情况。
*   **`server`**：后跟一个域名或IP地址，通常解析为单个服务器。适用于指向特定服务器的情况。

![](img/c7e2df03655071cc2b75ed3781699b79_29.png)

对于指向单个特定服务器的场景（如考试要求），更规范的做法是使用`server`指令。建议注释掉或删除原有的`pool`行，并添加新的`server`行。

![](img/c7e2df03655071cc2b75ed3781699b79_31.png)

例如，将服务器设置为`server1.lab.example.com`：
```bash
#pool 2.rhel.pool.ntp.org iburst
server server1.lab.example.com iburst
```
参数`iburst`的作用是：在服务启动初期，会快速发送多个数据包以尽快完成首次时间同步，这对于快速验证配置很有帮助。

![](img/c7e2df03655071cc2b75ed3781699b79_33.png)

修改完成后，保存并退出编辑器。

### 3. 启动并启用服务

![](img/c7e2df03655071cc2b75ed3781699b79_35.png)

配置文件修改完成后，需要重启`chronyd`服务以使更改生效，并设置其开机自动启动。

```bash
systemctl restart chronyd
systemctl enable chronyd
```

## 验证配置

配置完成后，需要验证时间同步是否正常工作。以下是几种验证方法。

### 方法一：使用`timedatectl`命令

`timedatectl`命令可以查看系统时间和日期相关的状态。

![](img/c7e2df03655071cc2b75ed3781699b79_37.png)

```bash
timedatectl
```
在输出中，关注以下两行：
*   `NTP service: active` – 表示NTP服务正在运行。
*   `System clock synchronized: yes` – 表示系统时钟已成功同步。如果服务刚启动或时间差异不大，此处可能显示`no`，需要等待同步周期或手动触发。

### 方法二：使用`chronyc`命令检查源

![](img/c7e2df03655071cc2b75ed3781699b79_39.png)

`chronyc`是`chrony`的命令行客户端工具，用于监控和诊断。检查时间源状态是最直接的验证方法。

执行以下命令查看当前使用的时间源：
```bash
chronyc sources -v
```
在输出列表中，您应该能看到配置的NTP服务器地址（如`server1.lab.example.com`）。

![](img/c7e2df03655071cc2b75ed3781699b79_41.png)

![](img/c7e2df03655071cc2b75ed3781699b79_42.png)

![](img/c7e2df03655071cc2b75ed3781699b79_44.png)

**关键标记说明（输出列表最左侧两列）：**
*   **`M`（模式）**：`^`表示服务器，`=`表示对等点，`#`表示本地连接的参考时钟。
*   **`S`（状态）**：`*`表示当前同步使用的源，`+`表示可接受的备用源，`?`表示源丢失或不可达。

![](img/c7e2df03655071cc2b75ed3781699b79_46.png)

一个健康的配置通常能看到`^`和`*`标记，表示已配置了指定的服务器并且当前正在使用它进行同步。如果看到`?`，则说明无法连接到该服务器，需要检查网络或配置。

![](img/c7e2df03655071cc2b75ed3781699b79_48.png)

### 方法三：手动修改时间测试（可选）

![](img/c7e2df03655071cc2b75ed3781699b79_50.png)

为了直观验证同步效果，可以手动将系统时间设错，然后观察是否会被自动纠正。

![](img/c7e2df03655071cc2b75ed3781699b79_52.png)

![](img/c7e2df03655071cc2b75ed3781699b79_54.png)

1.  将系统时间设置为一个过去的错误时间：
    ```bash
    date -s “2002-02-02 10:00:00”
    ```
2.  重启`chronyd`服务或等待下一个同步周期（通常几分钟）：
    ```bash
    systemctl restart chronyd
    ```
3.  再次查看当前时间：
    ```bash
    date
    ```
    如果时间恢复为正确的当前时间，则证明同步成功。

![](img/c7e2df03655071cc2b75ed3781699b79_55.png)

![](img/c7e2df03655071cc2b75ed3781699b79_56.png)

## 补充知识：硬件时钟同步

![](img/c7e2df03655071cc2b75ed3781699b79_58.png)

系统存在两个时钟：系统时钟（软件维护）和硬件时钟（主板CMOS维护）。`chrony`同步的是系统时钟。如果需要将正确的系统时间写入硬件时钟，可以使用`hwclock`命令。

![](img/c7e2df03655071cc2b75ed3781699b79_60.png)

*   将系统时间同步到硬件时钟：
    ```bash
    hwclock -w
    ```
*   根据硬件时钟设置系统时间：
    ```bash
    hwclock -s
    ```
在RHCSA考试中，通常不需要操作硬件时钟，了解即可。

## 总结

本节课中我们一起学习了如何配置NTP时间客户端。核心步骤可以概括为“装包、配配、起服务”：
1.  确保`chrony`软件包已安装。
2.  编辑`/etc/chrony.conf`配置文件，使用`server`指令指定正确的NTP服务器地址。
3.  重启并启用`chronyd`服务。

![](img/c7e2df03655071cc2b75ed3781699b79_62.png)

![](img/c7e2df03655071cc2b75ed3781699b79_64.png)

验证时，推荐使用`chronyc sources -v`命令，查看配置的源是否可达且状态为`^*`。理解并掌握这些操作，就能轻松完成RHCSA考试中关于时间同步的题目，并应对实际工作中的类似需求。
