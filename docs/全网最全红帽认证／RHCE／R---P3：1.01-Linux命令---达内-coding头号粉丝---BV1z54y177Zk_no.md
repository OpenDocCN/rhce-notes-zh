# Linux命令基础：1.01：命令行概述与基础概念

在本节课中，我们将学习Linux命令行的核心概念，包括命令行的定义、解释器的作用、Linux目录结构以及命令的基本格式和常用快捷键。掌握这些基础知识是后续高效使用Linux系统的关键。

## 命令行与解释器

上一节我们提到了通过命令行控制Linux系统。本节中，我们来详细看看什么是命令行以及系统如何理解我们的命令。

**命令行**是管理员输入的一串字符，用于完成特定的系统任务。例如，输入 `ip address` 并按回车键提交，这就是一个命令行。

**解释器**（通常称为Shell）是Linux系统中一个特殊的程序，它的作用是将用户输入的命令行翻译成操作系统内核能够理解和执行的指令。如果没有解释器，用户将无法与系统内核进行有效沟通。在大多数Linux系统中，默认的解释器程序是 `/bin/bash`。

**内核**是操作系统的核心部分，负责直接管理计算机的硬件资源，如CPU、内存和磁盘。用户通过解释器与内核交互，从而控制整个系统。

在命令行下，最顶层的目录称为**根目录**，用单个斜杠 `/` 表示。斜杠 `/` 也用作不同目录层级之间的分隔符。

以下是Linux系统中一些常见的一级目录及其用途：

*   **`/bin` 与 `/sbin`**：存放可执行程序文件。`/bin` 下的命令普通用户和管理员都可使用，`/sbin` 下的命令通常仅供管理员使用。
*   **`/dev`**：存放各种硬件设备的设备文件。
*   **`/home`**：存放所有普通用户的主目录（家目录）。
*   **`/root`**：系统管理员（root用户）的主目录。
*   **`/boot`**：存放系统启动所需的文件，包括内核。切勿随意删除。
*   **`/etc`**：存放系统的各种配置文件。
*   **`/var`**：存放经常变化的数据，如系统日志、邮件等。
*   **`/tmp`**：存放临时文件。
*   **`/mnt`** 与 **`/media`**：用于挂载外部设备（如U盘、光盘）的目录。`/mnt` 通常用于管理员手动挂载，`/media` 用于系统自动识别并挂载。

## 命令行的基本格式

![](img/f5e039f6f38975195cba779349241a2e_28.png)

![](img/f5e039f6f38975195cba779349241a2e_30.png)

在Linux中，命令通常遵循一个基本格式。理解这个格式有助于我们正确使用各种命令。

一个标准的命令行格式如下：
`命令字 [选项] [参数]`

*   **命令字**：即命令的名称，是必须的部分。它决定了要执行的操作，例如 `ls` 代表“列出”。
*   **选项**：用于调节命令的执行方式或效果，以短横线 `-` 开头。例如，`ls -l` 表示以长格式（详细信息）列出文件。
*   **参数**：指定命令操作的对象，通常是文件或目录的路径。例如，`ls /boot` 表示列出 `/boot` 目录下的内容。

选项和参数都可以有多个。多个单字母选项可以合并，例如 `ls -l -h` 可以简写为 `ls -lh`，表示以人类可读的格式（如KB、MB）显示文件的详细信息。

## 常用快捷键

为了提高在命令行下的操作效率，Linux提供了一些实用的快捷键。

以下是几个最常用的快捷键：

*   **`Tab` 键**：命令/路径自动补全。输入命令或路径的前几个字符后按 `Tab` 键，系统会自动补全或给出提示。
*   **`Ctrl + L`**：清空当前终端屏幕，效果等同于执行 `clear` 命令。
*   **`Ctrl + C`**：终止当前正在运行的前台任务或命令。
*   **`Esc + .`**：快速粘贴上一条命令的最后一个参数到当前光标位置。
