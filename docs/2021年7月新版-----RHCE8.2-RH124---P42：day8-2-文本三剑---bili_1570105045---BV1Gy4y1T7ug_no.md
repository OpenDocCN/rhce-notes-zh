# Linux文本处理：P42：文本三剑客之awk

在本节课中，我们将要学习文本处理三剑客中的最后一个强大工具——`awk`。`awk`不仅仅是一个文本搜索工具，它更是一个完整的文本分析和报告生成语言。我们将从它的基本工作原理开始，逐步学习其语法、变量和常用技巧，最终能够运用`awk`高效地处理和分析文本数据。

## 工作原理与版本

上一节我们介绍了`sed`，本节中我们来看看`awk`。首先，了解`awk`的工作原理和版本。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_1.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_3.png)

`awk`有多种实现版本，例如`nawk`和`mawk`。但在Linux系统中，我们通常使用的是`gawk`（GNU awk），它是其他版本的超集，功能最全且兼容性好。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_5.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_7.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_9.png)

`awk`的主要功能包括：
*   文本处理与格式化输出。
*   执行算术与字符串运算。
*   生成格式化的文本报表。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_11.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_13.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_15.png)

它的基本命令格式如下：
```bash
awk [选项] ‘程序’ 文件
```
或者，将程序写入脚本文件并用`-f`选项指定：
```bash
awk -f 脚本文件 输入文件
```

## 程序结构与选项

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_23.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_25.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_27.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_29.png)

了解了`awk`的概貌后，我们深入其程序结构。一个`awk`程序通常由三部分组成，被包含在单引号中。

以下是`awk`程序的核心结构：
1.  **BEGIN块**：在处理任何输入行之前执行一次，常用于初始化。
2.  **模式匹配块**：对于每一行输入，如果匹配指定的模式，则执行对应的动作。模式可以是正则表达式、条件判断等。
3.  **END块**：在处理完所有输入行之后执行一次，常用于汇总输出。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_31.png)

`awk`的常用选项有：
*   `-F`：指定输入字段的分隔符。例如 `-F “:“` 表示用冒号分隔。
*   `-v`：定义一个变量，可以在`awk`程序中使用。例如 `-v var=value`。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_33.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_35.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_37.png)

程序的基本格式为：`模式 { 动作 }`。如果省略模式，则对每一行执行动作；如果省略动作，则默认打印匹配的行。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_39.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_41.png)

## 工作模式详解

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_43.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_45.png)

现在，我们通过实例来理解`awk`的逐行工作模式。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_47.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_49.png)

`awk`的工作流程是：
1.  首先执行`BEGIN`块（如果有）。
2.  然后逐行读取输入文件。
3.  对于每一行，根据`-F`指定的分隔符将行分割成多个字段。字段依次用 `$1`, `$2`, `$3`… 表示，整行内容用 `$0` 表示。
4.  将当前行与程序中的每个`模式`进行匹配。
5.  如果匹配成功，则执行该模式对应的`{ 动作 }`。
6.  处理完所有行后，执行`END`块（如果有）。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_51.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_53.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_55.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_57.png)

我们通过一个简单命令来观察：
```bash
echo “a” | awk ‘{print “hello”}’
```
输入 “a”，`awk`读取该行，执行动作 `print “hello”`，于是输出 “hello”。它并不处理原文本”a”，而是执行我们指定的动作。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_59.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_61.png)

再看一个序列生成的例子：
```bash
seq 3 | awk ‘{print “hello”}’
```
`seq 3` 生成三行（1, 2, 3）。`awk`逐行处理，对每一行都执行 `print “hello”`，所以最终输出三个 “hello”。

## 字段分割与提取

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_63.png)

理解了工作模式，我们就可以利用字段分割来提取特定信息了。`awk`默认使用连续的空格或制表符作为分隔符。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_70.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_72.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_74.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_76.png)

以下命令演示了字段提取：
```bash
# 查看/etc/passwd文件，使用冒号分隔
awk -F “:” ‘{print $1}’ /etc/passwd # 打印所有用户名（第一列）
awk -F “:” ‘{print $1, $7}’ /etc/passwd # 打印用户名和对应的shell（第一和第七列）
```
在`/etc/passwd`文件中，每行由冒号分隔成多个字段。`$1`代表用户名，`$7`代表用户的登录shell。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_78.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_80.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_82.png)

我们也可以用`OFS`（输出字段分隔符）变量控制打印时的分隔符：
```bash
awk -F “:” -v OFS=“\t” ‘{print $1, $7}’ /etc/passwd
```
这样，用户名和shell在输出时会用制表符隔开，更清晰。

## 内置变量应用

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_90.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_92.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_94.png)

`awk`提供了强大的内置变量，让文本处理更加灵活。两个最常用的是`NF`和`NR`。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_96.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_98.png)

以下是`awk`的关键内置变量：
*   **NF**：当前行被分割后的字段总数。
*   **`$NF`**：代表当前行的最后一个字段。
*   **NR**：当前正在处理的行号（累计行数）。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_100.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_102.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_104.png)

应用示例：
```bash
# 打印每行的字段数
awk ‘{print NF}’ filename
# 打印每行的最后一个字段
awk ‘{print $NF}’ filename
# 打印文件内容并加上行号
awk ‘{print NR, $0}’ filename
# 仅打印第5行
awk ‘NR==5 {print}’ filename
# 打印第2到第4行
awk ‘NR>=2 && NR<=4 {print}’ filename
```

## 实践案例解析

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_110.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_112.png)

掌握了基本概念和变量后，我们来看几个综合性的实践案例。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_114.png)

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_116.png)

**案例1：提取磁盘使用率百分比（去除`%`符号）**
```bash
df -h | awk -F “[[:space:]]+|%” ‘/^\/dev/ {print $(NF-1)}’
```
*   `-F “[[:space:]]+|%”`：使用一个或多个连续空格**或**百分号`%`作为分隔符。
*   `/^\/dev/`：模式匹配，只处理以`/dev`开头的行（即物理磁盘）。
*   `{print $(NF-1)}`：打印倒数第二个字段。因为用`%`分割后，使用率数字和`%`分开了，数字变成了倒数第二列。

![](img/3276329ad9d84c0c70db0b5bfcd0cf80_118.png)

**案例2：从日志中提取主机名**
假设有日志行：`www.baidu.com [28/Feb/2024:13:17:10]`，想提取`www.baidu.com`。
```bash
echo “www.baidu.com [28/Feb/2024:13:17:10]” | awk -F “[ .]+” ‘{print $1}’
```
*   `-F “[ .]+”`：使用空格或点号`.`作为分隔符（`[]`表示字符集，`+`表示一个或多个）。
*   `{print $1}`：打印分割后的第一个字段，即`www`。如果要得到完整域名，可能需要调整分隔符或字段索引。

**案例3：获取本机IP地址**
```bash
ip addr show eth0 | awk ‘NR==3 {print $2}’ | awk -F “/” ‘{print $1}’
```
*   `NR==3`：取`ip addr`命令输出的第三行（通常包含IPv4地址）。
*   `{print $2}`：打印该行的第二个字段，格式如 `192.168.1.10/24`。
*   `awk -F “/” ‘{print $1}’`：再次用`/`分割，取第一个字段，得到纯IP地址 `192.168.1.10`。
