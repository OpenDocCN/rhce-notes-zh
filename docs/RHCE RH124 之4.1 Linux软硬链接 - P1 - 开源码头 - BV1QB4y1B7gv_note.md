# Linux基础：4.1：Linux软硬链接详解 🔗

![](img/1b1404c725bcb4172b85fb0e46ec6b36_1.png)

在本节课中，我们将要学习Linux系统中一个重要的概念——链接。链接分为软链接和硬链接两种，它们允许我们以不同的方式引用文件。理解这两种链接的区别和用途，是掌握Linux文件系统管理的基础。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_3.png)

## 打开终端并创建文件

我们首先打开一个终端窗口。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_5.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_1.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_7.png)

使用 `ls` 命令查看当前桌面目录的内容。可以看到桌面上有两个目录：`classroom_doc` 和 `HUSV8`。回收站是系统图标，不在用户家目录的桌面显示范围内。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_8.png)

现在，我们想在桌面上新建一个文件。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_3.png)

创建新文件的方法是使用 `touch` 命令。

```bash
touch newfile1.txt
```

执行命令后，桌面上会出现一个新文件 `newfile1.txt`。

接下来，我们双击打开这个文件，并向其中写入一些内容。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_5.png)

在文件中输入：“您好，这是我的第一个文件。” 然后关闭并保存文件。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_7.png)
![](img/1b1404c725bcb4172b85fb0e46ec6b36_8.png)

我们也可以使用 `cat` 命令在终端中查看文件内容。按下 `Windows键 + 空格键` 切换到英文输入法，然后输入：

```bash
cat newfile1.txt
```

文件内容会显示在命令行窗口中。

## 创建软链接（符号链接）

现在，我们想在桌面上为 `newfile1.txt` 创建一个软链接，类似于Windows系统中的“快捷方式”。

创建软链接的命令是 `ln -s`。

```bash
ln -s newfile1.txt newfile_symbol.txt
```

![](img/1b1404c725bcb4172b85fb0e46ec6b36_10.png)

执行后，桌面上会出现一个新文件 `newfile_symbol.txt`。它的图标通常带有一个箭头，表示它是一个指向其他文件的链接。

双击这个符号链接文件，打开的内容仍然是原始的 `newfile1.txt` 文件。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_12.png)

我们可以使用 `ll`（`ls -l` 的别名）命令查看详细信息。

```bash
ll
```

输出结果会显示两个文件：
*   `newfile1.txt` 是原始文件，大小为41字节（包含实际内容）。
*   `newfile_symbol.txt` 是符号链接文件，大小为12字节。这12个字节仅仅存储了它所指向的文件名 `newfile1.txt`。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_14.png)

**核心概念**：软链接文件本身不存储实际数据，只存储目标文件的路径名。访问软链接等同于访问其指向的目标文件。

我们可以通过符号链接修改文件。双击打开 `newfile_symbol.txt`，添加一些新内容（例如：“这是打开符号文件添加的。”），然后保存关闭。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_10.png)
![](img/1b1404c725bcb4172b85fb0e46ec6b36_12.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_16.png)

再次使用 `ll` 命令查看，会发现 `newfile1.txt` 的文件大小增加了（因为内容变多），而 `newfile_symbol.txt` 的大小依然是12字节，因为它存储的路径名没有改变。

## 跨目录创建软链接

软链接可以跨目录创建。例如，我们在 `/tmp` 目录下创建一个新文件。

```bash
touch /tmp/file2.txt
```

![](img/1b1404c725bcb4172b85fb0e46ec6b36_20.png)

然后使用图形界面或编辑器（如 `gedit`）打开 `/tmp/file2.txt`，写入内容：“这是我的第二个文件。” 并保存。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_14.png)
![](img/1b1404c725bcb4172b85fb0e46ec6b36_16.png)
![](img/1b1404c725bcb4172b85fb0e46ec6b36_18.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_22.png)

现在，我们在桌面上为这个位于 `/tmp` 目录的文件创建一个软链接。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_24.png)

```bash
ln -s /tmp/file2.txt ~/Desktop/file2_symbol.txt
```

![](img/1b1404c725bcb4172b85fb0e46ec6b36_25.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_20.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_27.png)

执行后，桌面上会出现 `file2_symbol.txt`。双击它，打开的是 `/tmp/file2.txt` 的内容。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_22.png)
![](img/1b1404c725bcb4172b85fb0e46ec6b36_24.png)
![](img/1b1404c725bcb4172b85fb0e46ec6b36_25.png)

使用 `ll` 命令查看，可以看到 `file2_symbol.txt` 的大小是14字节，对应它存储的路径字符串 `/tmp/file2.txt` 的长度。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_27.png)

**重要提示**：软链接不仅可以链接文件，也可以链接目录。在Linux根目录中，许多目录（如 `/bin` -> `/usr/bin`）本身就是符号链接。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_29.png)

## 创建硬链接

![](img/1b1404c725bcb4172b85fb0e46ec6b36_31.png)

上一节我们介绍了软链接，它类似于快捷方式。本节中我们来看看Linux特有的硬链接。硬链接允许**多个不同的文件名指向磁盘上同一份数据**。

首先，创建一个新文件。

```bash
touch double.txt
```

![](img/1b1404c725bcb4172b85fb0e46ec6b36_29.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_33.png)

双击打开 `double.txt`，写入内容：“这个文件内容，我们计划对应到两个文件名。” 然后保存关闭。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_31.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_35.png)

现在，我们为 `double.txt` 创建一个硬链接。**创建硬链接时不使用 `-s` 选项**。

```bash
ln double.txt repeat.txt
```

![](img/1b1404c725bcb4172b85fb0e46ec6b36_33.png)

执行后，桌面上会出现 `repeat.txt`。使用 `ll` 命令查看：

![](img/1b1404c725bcb4172b85fb0e46ec6b36_37.png)

![](img/1b1404c725bcb4172b85fb0e46ec6b36_35.png)

观察输出：
1.  两个文件（`double.txt` 和 `repeat.txt`）的大小相同（55字节），因为它们内容相同。
2.  关键区别在于第二列的“链接数”。创建硬链接前，`double.txt` 的链接数是 `1`。创建后，`double.txt` 和 `repeat.txt` 的链接数都变成了 `2`。

**核心概念**：硬链接使多个文件名直接关联到同一个数据块（inode）。`ll` 命令第二列的数字表示有多少个文件名链接到这份数据。

双击打开 `repeat.txt`，可以看到和 `double.txt` 完全相同的内容。通过 `repeat.txt` 修改文件内容并保存。

![](img/1b1404c725bcb4172b85fb0e46ec6b36_37.png)

再次打开 `double.txt`，会发现内容也同步改变了。这说明两个文件名操作的是同一份数据。

如果删除其中一个文件名，例如删除 `double.txt`：

```bash
rm double.txt
```

使用 `ll` 查看，`double.txt` 已消失，但 `repeat.txt` 依然存在，且其链接数从 `2` 变回 `1`。只要链接数不为零，磁盘上的数据就不会被清除。

硬链接也可以跨目录创建：

```bash
ln repeat.txt /tmp/repeat2.txt
```

此时，`repeat.txt` 的链接数又会变为 `2`。使用 `cat /tmp/repeat2.txt` 查看，内容一致。

## 硬链接的特点与限制

硬链接有一些重要的特点和限制：

1.  **共享元数据**：硬链接不仅共享文件内容，也共享文件的元数据（如权限、所有者、时间戳）。修改一个硬链接文件的权限，其他所有指向同一数据的硬链接权限都会同步改变。
    *   尝试修改权限：`chmod u=r,g=,o= repeat.txt`
    *   查看 `/tmp/repeat2.txt`，其权限也已同步更改。

2.  **主要限制**：**硬链接不能用于目录**（为了防止在目录树中创建循环，导致文件系统混乱）。而软链接则可以对目录进行链接。

3.  **文件系统限制**：硬链接通常要求所有链接必须在同一个文件系统（分区）内，因为inode编号在跨文件系统时不唯一。软链接没有此限制，它可以跨文件系统指向任意路径。

## 总结

![](img/1b1404c725bcb4172b85fb0e46ec6b36_39.png)

本节课中我们一起学习了Linux中的两种链接机制：

![](img/1b1404c725bcb4172b85fb0e46ec6b36_41.png)

*   **软链接（符号链接）**：使用 `ln -s <源文件> <链接名>` 创建。它是一个独立的文件，内容仅为目标文件的路径，类似于Windows的快捷方式。可以链接文件和目录，可以跨文件系统。
*   **硬链接**：使用 `ln <源文件> <链接名>` 创建。它不是一个独立的文件，而是给现有数据块增加一个新的文件名引用。`ll` 命令第二列的数值表示硬链接的数量。硬链接不能用于目录，且通常不能跨文件系统。

理解软硬链接的区别，对于管理Linux文件系统、理解文件存储方式以及进行一些高级系统操作都至关重要。
