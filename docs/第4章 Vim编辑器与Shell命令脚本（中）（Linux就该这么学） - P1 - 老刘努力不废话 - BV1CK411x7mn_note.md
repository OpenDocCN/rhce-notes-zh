## Shell脚本简介

![](img/0cc562c5fc084fd0a368cbdff949926b_5.png)

![](img/0cc562c5fc084fd0a368cbdff949926b_7.png)

通过前几章的学习，我们已经掌握了在Linux中执行命令和编写文件的基本技术。接下来，我们可以将命令写入文件，这就形成了一个Shell脚本。

Shell脚本主要分为两种类型：**交互式**和**批处理式**。交互式脚本在执行过程中需要用户输入信息来补全参数。批处理式脚本则根据预设的命令和逻辑自动执行，常用于自动化运维。我们大多数情况下使用的是批处理式脚本。

## Shell脚本的基本结构

无论Shell脚本是长是短，其基本结构都是不变的。它由三个部分组成：脚本声明、脚本注释和脚本命令。

以下是脚本的三个组成部分：

![](img/0cc562c5fc084fd0a368cbdff949926b_11.png)

1.  **脚本声明**：指定脚本由哪个解释器执行。通常使用 `#!/bin/bash` 来声明使用Bash解释器。
2.  **脚本注释**：对脚本功能或某行代码进行说明，以 `#` 开头。注释可以没有，也可以有多行。
3.  **脚本命令**：脚本要执行的具体命令，是脚本的核心内容。最简单的形式就是命令从上到下依次执行。

一个最简单的Shell脚本就是命令的堆积。例如，我们可以编写一个脚本，依次执行显示当前目录、列出文件、等待15秒然后重启。

![](img/0cc562c5fc084fd0a368cbdff949926b_13.png)

![](img/0cc562c5fc084fd0a368cbdff949926b_15.png)

```bash
#!/bin/bash
# Welcome to linuxprobe.com
pwd
ls
sleep 15
reboot
```

要执行这个脚本，可以使用 `bash` 命令后跟脚本文件名（例如 `bash haha.sh`）。在系统授予脚本执行权限之前，这是推荐的方法。

## 接收用户参数

![](img/0cc562c5fc084fd0a368cbdff949926b_29.png)

Shell脚本需要有能力接收用户输入的参数，并根据不同的参数执行不同的操作。例如，`ls -a` 和 `ls -l` 会显示不同的信息。

![](img/0cc562c5fc084fd0a368cbdff949926b_31.png)

Shell脚本通过一些内置的变量来接收用户参数：

*   **`$0`**：代表脚本本身的名称。
*   **`$1`、`$2`、`$3`...**：代表脚本接收到的第1、2、3...个参数。
*   **`$#`**：代表脚本总共接收到的参数个数。
*   **`$*`**：代表脚本接收到的所有参数。
*   **`$?`**：代表上一条命令的返回值（0通常表示成功，非0表示失败）。

![](img/0cc562c5fc084fd0a368cbdff949926b_33.png)

例如，执行 `bash haha.sh A B C D E F G`，在脚本中使用 `echo "$0, $1, $3, $5"` 会输出 `haha.sh, A, C, E`。

![](img/0cc562c5fc084fd0a368cbdff949926b_35.png)

## 使用判断语句

![](img/0cc562c5fc084fd0a368cbdff949926b_41.png)

脚本接收参数后，还需要有能力根据不同的参数进行判断，从而执行不同的命令。这就需要用到判断语句。

![](img/0cc562c5fc084fd0a368cbdff949926b_43.png)

在Linux中，判断语句使用 `[ 条件表达式 ]` 的格式（注意方括号内两端必须有空格）。它可以对文件、整数、字符串和逻辑进行判断。

![](img/0cc562c5fc084fd0a368cbdff949926b_45.png)

![](img/0cc562c5fc084fd0a368cbdff949926b_47.png)

### 1. 文件测试

文件测试用于判断文件的属性，例如是否存在、是否是目录、是否可读等。

![](img/0cc562c5fc084fd0a368cbdff949926b_49.png)

常用的文件测试运算符包括：
*   `-d`：测试是否为目录。
*   `-f`：测试是否为一般文件。
*   -e`：测试文件或目录是否存在。
*   `-r`：测试当前用户是否有读取权限。
*   `-w`：测试当前用户是否有写入权限。

例如，`[ -d /root ] && echo "存在"` 会判断 `/root` 是否为目录，如果是则输出“存在”。`$?` 变量可以查看上一条判断语句的返回值。

![](img/0cc562c5fc084fd0a368cbdff949926b_51.png)

### 2. 逻辑测试

逻辑测试允许我们将多个条件表达式或命令组合使用。主要有三个逻辑操作符：

*   **与（`&&`）**：只有前面的命令执行成功（返回值为0），才会执行后面的命令。
    *   **格式**：`命令1 && 命令2`
    *   **含义**：若 `命令1` 成功，则执行 `命令2`。
*   **或（`||`）**：只有前面的命令执行失败（返回值为非0），才会执行后面的命令。
    *   **格式**：`命令1 || 命令2`
    *   **含义**：若 `命令1` 失败，则执行 `命令2`。
*   **非（`!`）**：对条件测试的结果取反值。
    *   **格式**：`[ ! 条件表达式 ]`
    *   **含义**：若条件表达式成立，则结果为不成立；反之亦然。

例如，我们可以根据当前登录用户显示不同信息：
```bash
[ $USER = root ] && echo “Manager” || echo “Staff”
```

**注意**：逻辑“非”是 `!`，而不是 `!=`。`!=` 是字符串比较运算符。

### 3. 整数测试

整数测试用于比较两个数值的大小关系。

**重要提示**：在 `[ ]` 中比较整数时，**不能**使用 `>`、`<`、`=` 这些符号，因为它们会被Shell解释为重定向或赋值操作。必须使用专用的整数比较运算符：

*   `-eq`：等于（Equal）
*   `-ne`：不等于（Not Equal）
*   `-gt`：大于（Greater Than）
*   `-lt`：小于（Less Than）
*   `-ge`：大于或等于（Greater or Equal）
*   `-le`：小于或等于（Less or Equal）

例如，判断5是否大于3，应使用 `[ 5 -gt 3 ]`。

### 4. 字符串测试

字符串测试用于比较两个字符串是否相同，或判断字符串是否为空。

常用运算符：
*   `=`：比较两个字符串是否相同。
*   `!=`：比较两个字符串是否不同。
*   `-z`：判断字符串是否为空值。

一个常见的应用场景是检查变量是否已被定义，避免覆盖：
```bash
[ -z $PRICE ] && PRICE=10
```
这行代码会先判断变量 `$PRICE` 是否为空，如果为空（即未定义），则将其赋值为10。
