# RHCE认证课程：P58：Podman运行容器 🐳

## 概述
在本节课中，我们将学习如何使用Podman工具来运行和管理容器。Podman是一个无守护进程的容器引擎，用于在Linux系统上开发、管理和运行OCI容器。我们将从基础概念开始，逐步学习如何搜索、拉取、运行、停止、进入和删除容器，并了解端口映射、环境变量设置以及数据持久化等核心操作。

## 容器基础概念回顾
上一节我们介绍了容器与虚拟机的区别。本节中，我们来看看容器引擎的核心组成部分。

![](img/6c2de2868e2ef156a41c847b996f86e7_5.png)

一个完整的容器环境通常包含以下几个部分：
*   **主机（Host）**：运行容器的物理机或虚拟机，也称为宿主机。
*   **仓库（Registry）**：用于保存容器镜像的仓库。常见的官方仓库有 `docker.io` 和 `quay.io`。Podman默认使用的是 `quay.io`。
*   **镜像（Image）**：可以理解为创建容器实例使用的模板，类似于RPM或DEB软件包。
*   **容器（Container）**：从镜像运行起来的、对外提供服务的实例，类似于RPM包中的一个运行起来的程序。

![](img/6c2de2868e2ef156a41c847b996f86e7_7.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_9.png)

镜像的完整名称格式通常为：`{仓库FQDN}/{用户名}/{镜像名}:{标签}`。如果不指定标签，则默认为 `latest`。

### 搜索与拉取镜像
在运行容器之前，我们通常需要从仓库中拉取镜像。以下是相关操作。

![](img/6c2de2868e2ef156a41c847b996f86e7_24.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_25.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_27.png)

**搜索镜像**
```bash
podman search ubi8
```
此命令会在配置的仓库中搜索包含“ubi8”关键词的镜像。


此命令会从 `quay.io` 仓库拉取 `ubi8/ubi` 镜像的 `latest` 标签到本地。

**前台运行**
```bash

```
这种方式会占用当前终端。如果镜像没有设计前台守护进程，容器会执行完命令后立即退出。

**后台运行**
```bash
podman run -d quay.io/ubi8/ubi:latest
```
`-d` 选项让容器在后台运行，不占用当前终端。

![](img/6c2de2868e2ef156a41c847b996f86e7_79.png)

**交互式运行并进入容器**
```bash
podman run -it quay.io/ubi8/ubi:latest /bin/bash
```
`-it` 选项组合为容器分配一个伪终端并保持标准输入打开，允许用户与容器交互。`/bin/bash` 是指定容器启动后运行的命令。

### 端口映射（-P）
默认情况下，容器内的网络服务无法直接从宿主机外部访问。需要使用 `-p` 选项进行端口映射。

![](img/6c2de2868e2ef156a41c847b996f86e7_214.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_216.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_218.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_220.png)

**命令格式**
```bash
podman run -d -p <宿主机端口>:<容器端口> <镜像名>
```

![](img/6c2de2868e2ef156a41c847b996f86e7_222.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_224.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_226.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_230.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_232.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_234.png)

**应用示例**
运行一个Apache容器，并将宿主机的8080端口映射到容器的80端口：
```bash
podman run -d --name web1 -p 8080:80 quay.io/centos/httpd-24-centos7
```
执行后，可以通过访问 `http://localhost:8080` 来访问容器内的Web服务。

**应用示例**
运行一个MySQL容器，并设置root密码和初始数据库：
```bash

  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=mysecretpassword \
  -e MYSQL_DATABASE=mydb \
  -e MYSQL_USER=myuser \
  -e MYSQL_PASSWORD=mypassword \
  docker.io/mysql:5.7
```

### 数据持久化与卷挂载（-v）
容器内的数据默认是临时的，容器删除后数据会丢失。为了实现数据持久化，可以将宿主机的目录挂载到容器内。

**命令格式**
```bash
podman run -v <宿主机路径>:<容器路径>:Z <镜像名>
```
`:Z` 选项用于自动配置SELinux上下文，在启用SELinux的系统上很重要。

![](img/6c2de2868e2ef156a41c847b996f86e7_297.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_299.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_301.png)

![](img/6c2de2868e2ef156a41c847b996f86e7_303.png)

**应用示例**
运行一个Web容器，并将宿主机的 `~/webdata` 目录挂载到容器的Web根目录：
```bash
mkdir ~/webdata
echo “Hello from Persistent Storage” > ~/webdata/index.html
podman run -d --name web2 -p 8081:80 -v ~/webdata:/var/www/html:Z quay.io/centos/httpd-24-centos7
```
现在，访问 `http://localhost:8081` 会显示 `~/webdata/index.html` 的内容。即使删除并重新创建容器，只要挂载同一个目录，数据依然存在。

## 总结
本节课我们一起学习了Podman的核心操作。我们了解了容器的基础概念，并实践了如何使用Podman进行镜像的搜索、拉取和管理。我们重点学习了容器的生命周期管理，包括运行、停止、启动、进入和删除。此外，我们还掌握了三种关键的容器高级配置技巧：通过 `-p` 进行端口映射使服务可被外部访问；通过 `-e` 设置环境变量来配置应用；通过 `-v` 进行卷挂载实现容器数据的持久化，这是生产环境中保证数据安全的关键步骤。这些技能是使用容器技术的基础，也为后续学习更复杂的容器编排工具打下了坚实的基础。
