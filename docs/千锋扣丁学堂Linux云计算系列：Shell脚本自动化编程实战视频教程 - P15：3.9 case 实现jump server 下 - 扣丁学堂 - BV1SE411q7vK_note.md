## 实现免密登录 🔑

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_8.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_9.png)

目前我们的脚本通过密码方式从跳板机连接到后端服务器。为了提高安全性和便利性，我们可以采用SSH密钥对进行认证。

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_11.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_13.png)

以下是配置SSH免密登录的步骤：

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_15.png)

1.  **生成密钥对**：在跳板机上，以最终登录用户（例如 `alice`）的身份生成SSH密钥。
    ```bash
    ssh-keygen
    ```
    此命令会在 `~/.ssh/` 目录下生成私钥 `id_rsa` 和公钥 `id_rsa.pub`。

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_17.png)

2.  **分发公钥**：将公钥拷贝到所有需要连接的后端服务器上。
    ```bash
    ssh-copy-id alice@192.168.241.241
    ssh-copy-id alice@192.168.241.242
    ssh-copy-id alice@192.168.241.243
    ```
    执行命令时需要输入目标服务器上 `alice` 用户的密码。完成后，从跳板机连接这些服务器将不再需要密码。

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_19.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_21.png)

配置完成后，再次运行跳板机脚本，选择对应服务器编号即可实现免密直接登录。

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_23.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_25.png)

## 放置与自启动脚本 📂

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_27.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_29.png)

为了让用户一登录跳板机就自动进入跳板机菜单，我们需要将脚本放置在合适的位置并设置自启动。

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_31.png)

脚本应该放在跳板机上，并且由客户端登录时使用的账号（例如 `alice`）来执行。一个常见的做法是将脚本的调用命令添加到该用户的Shell配置文件中。

以下是具体操作：

1.  将编写好的 `jump_server.sh` 脚本保存在 `alice` 用户的家目录下，例如 `/home/alice/jump_server.sh`，并确保其有执行权限。
2.  编辑 `alice` 用户的Shell配置文件（如 `~/.bash_profile` 或 `~/.bashrc`），在文件末尾添加执行脚本的命令。
    ```bash
    /home/alice/jump_server.sh
    ```
3.  当用户 `alice` 通过SSH登录跳板机时，系统会读取此配置文件并自动执行跳板机脚本，用户将直接看到服务器选择菜单。

## 防止脚本被意外中断 🛡️

目前用户可以通过按 `Ctrl+C` 中断脚本，退回到跳板机的Shell提示符，这不符合跳板机“只允许连接后端服务器”的设计初衷。我们需要捕捉中断信号，并忽略它。

在Shell脚本中，可以使用 `trap` 命令来捕捉信号。

以下是实现方法：
在脚本的开头部分添加如下代码：
```bash
trap '' HUP INT
```
*   `trap` 用于捕捉信号。
*   `''`（空命令）表示捕捉到信号后不执行任何操作。
*   `HUP INT` 是要捕捉的信号类型，这里包括挂起信号和键盘中断信号（`Ctrl+C`）。

添加后，用户将无法通过 `Ctrl+C` 退出脚本，只能通过选择菜单中的退出选项（如果提供）或直接关闭终端来结束会话。

## 美化脚本输出 🎨

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_33.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_35.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_37.png)

我们可以优化菜单的显示，例如添加颜色和更好的格式，提升用户体验。可以使用 `echo -e` 命令来输出带颜色的文本。

以下是定义颜色变量和美化输出的示例：
```bash
# 定义颜色变量
RED='\033[31m'
GREEN='\033[32m'
RESET='\033[0m'

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_39.png)

# 美化菜单输出
echo -e "${GREEN}=================================${RESET}"
echo -e "${GREEN}        欢迎使用跳板机系统        ${RESET}"
echo -e "${GREEN}=================================${RESET}"
echo -e "1. 连接 Web 服务器 1"
echo -e "2. 连接 Web 服务器 2"
echo -e "3. 连接 DB 服务器"
echo -e "q. 退出"

# 美化输入提示，-n 参数使输入不换行
echo -e -n "${RED}请输入您的选择 [1-3, q]: ${RESET}"
read choice
```
*   `\033[31m` 设置红色，`\033[32m` 设置绿色，`\033[0m` 重置颜色。
*   `echo -e` 启用反斜杠转义解释。
*   `echo -n` 使输出后不自动换行，让输入提示紧跟在后面。

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_41.png)

你可以根据喜好调整边框、颜色和布局，使界面更友好。

## 生产环境限制与设计回顾 📝

让我们回顾一下跳板机所要解决的生产环境常见限制，以及我们的设计方案：

1.  **限制一**：业务服务器不允许直接连接，必须通过跳板机中转。
    *   **实现**：通过网络防火墙策略控制，只允许跳板机的IP访问业务服务器的SSH端口。

2.  **限制二**：业务服务器禁止root用户直接登录。
    *   **实现**：在所有业务服务器上创建统一的普通管理账号（如 `alice`），并修改SSH配置 `PermitRootLogin no` 来禁用root登录。

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_45.png)

我们的跳板机设计核心在于：
*   **统一入口**：所有运维人员通过同一个跳板机账号访问后台。
*   **权限最小化**：使用普通权限账号 `alice` 进行运维操作。
*   **操作可溯**：脚本本身可以扩展，用于记录用户选择了连接哪台服务器（更复杂的操作审计需要更专业的工具如Python跳板机）。
*   **强制访问**：通过修改用户Shell配置和捕捉信号，确保用户登录后只能使用跳板机功能，无法进入跳板机本身的Shell环境。

## 脚本编写模式总结 🧩

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_47.png)

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_49.png)

通过跳板机、系统工具箱等案例，我们可以总结出一个典型的交互式Shell脚本的通用模式：

1.  **显示菜单**：使用 `echo` 命令向用户展示可用的选项。
2.  **获取输入**：使用 `read` 命令读取用户的选择。
3.  **处理逻辑**：使用 `case...esac` 语句对用户输入进行模式匹配，并执行相应的代码块。

其基本结构如下：
```bash
#!/bin/bash
# 1. 显示菜单
echo "请选择操作："
echo "1) 选项A"
echo "2) 选项B"
echo "3) 退出"

![](img/91a44a5b4c0a82e4eadc1255ad5c4bc7_51.png)

# 2. 获取输入
read -p "请输入数字: " option

# 3. 处理逻辑
case $option in
    1)
        # 执行选项A的操作
        ;;
    2)
        # 执行选项B的操作
        ;;
    3|q)
        echo "退出。"
        exit 0
        ;;
    *)
        echo "无效输入！"
        ;;
esac
```
掌握这个模式，你就可以快速创建出各种用于系统管理、工具集成的交互式脚本。脚本的难点往往不在于语法，而在于对实际需求和环境（如用户登录流程、网络结构）的理解。
