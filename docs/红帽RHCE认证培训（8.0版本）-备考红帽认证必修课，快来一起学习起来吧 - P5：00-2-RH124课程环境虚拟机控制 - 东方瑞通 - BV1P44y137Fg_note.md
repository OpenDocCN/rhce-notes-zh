## 核心控制工具：`rht-vmctl`

控制这些嵌套虚拟机的核心工具是 `rht-vmctl` 命令。这个命令的名称含义如下：
*   `rht`: Red Hat Training（红帽培训）
*   `vmctl`: Virtual Machine Control（虚拟机控制）

![](img/79508d253976f337467e41353cb4f208_24.png)

因此，这是一个专门用于管理红帽培训环境虚拟机的命令。

![](img/79508d253976f337467e41353cb4f208_26.png)

### 命令基本语法

`rht-vmctl` 命令的基本使用格式如下：

```bash
rht-vmctl <command> <vm_name>
```

以下是参数说明：
*   **`<command>`**: 要执行的操作命令，例如 `status`（查看状态）、`start`（启动）、`stop`（关机）等。
*   **`<vm_name>`**: 目标虚拟机的名称，例如 `servera`、`all`（代表除`classroom`外的所有机器）或 `classroom`。

**注意**：`all` 参数**不包含** `classroom` 虚拟机，`classroom` 需要单独管理。

## 虚拟机基本操作

![](img/79508d253976f337467e41353cb4f208_28.png)

![](img/79508d253976f337467e41353cb4f208_30.png)

以下是使用 `rht-vmctl` 进行虚拟机管理的核心操作步骤。

### 1. 查看虚拟机状态

![](img/79508d253976f337467e41353cb4f208_32.png)

在操作前，通常需要先查看虚拟机的当前状态。

```bash
rht-vmctl status servera
rht-vmctl status all
```

命令执行后，可能会看到以下几种状态：
*   **`running`**: 虚拟机正在运行。
*   **`shut off`**: 虚拟机关闭。
*   **`missing`**: 虚拟机未定义（首次使用实验环境时常见）。遇到此状态无需担心，执行启动操作即可。

### 2. 启动虚拟机

启动虚拟机有严格的顺序要求，这是确保实验环境正常工作的关键。

**启动顺序必须遵循以下两步：**
1.  首先启动 `classroom` 虚拟机。它充当资源服务器，其他虚拟机首次启动时会从它下载必要的镜像文件。
2.  然后启动其他所有虚拟机。

对应的命令如下：

```bash
rht-vmctl start classroom
rht-vmctl start all
```

**注意**：首次启动 `servera`、`serverb` 等虚拟机时，会有下载镜像文件的过程，这属于正常现象。下载完成后，虚拟机文件会变大，但后续启动将不再需要下载。

### 3. 访问虚拟机控制台

当虚拟机启动后，如果需要查看其图形界面或文本控制台，可以使用 `view` 命令。

![](img/79508d253976f337467e41353cb4f208_34.png)

```bash
rht-vmctl view servera
```

![](img/79508d253976f337467e41353cb4f208_36.png)

![](img/79508d253976f337467e41353cb4f208_38.png)

执行此命令会打开一个窗口，显示 `servera` 虚拟机的控制台界面。关闭此窗口**不会**关闭虚拟机，只是断开显示连接。

### 4. 关闭虚拟机

![](img/79508d253976f337467e41353cb4f208_40.png)

![](img/79508d253976f337467e41353cb4f208_42.png)

以下是关闭虚拟机的两种方式：

![](img/79508d253976f337467e41353cb4f208_44.png)

![](img/79508d253976f337467e41353cb4f208_46.png)

*   **正常关机**：相当于在系统内执行关机命令。
    ```bash
    rht-vmctl stop servera
    ```
*   **强制断电**：模拟直接切断电源，较为暴力。
    ```bash
    rht-vmctl poweroff servera
    ```

使用 `poweroff` 命令时，默认会进行交互式确认。如果希望跳过确认直接执行，可以添加 `-y` 或 `-q` 选项。

```bash
rht-vmctl poweroff servera -y
```

![](img/79508d253976f337467e41353cb4f208_48.png)

![](img/79508d253976f337467e41353cb4f208_50.png)

### 5. 交互式操作

![](img/79508d253976f337467e41353cb4f208_52.png)

你可以使用 `-i` 选项进行交互式操作，系统会对每个受影响的虚拟机逐一询问确认。

```bash
rht-vmctl poweroff -i all
```

### 6. 重置虚拟机

在实验过程中，如果虚拟机被修改，需要恢复至初始纯净状态，可以使用 `reset` 命令。这类似于将虚拟机还原到某个快照点。

![](img/79508d253976f337467e41353cb4f208_54.png)

```bash
rht-vmctl reset servera
```

**重要提示**：如果 `reset` 命令无效（例如因之前下载中断导致虚拟机镜像损坏），可以使用 `fullreset` 命令。该命令会从 `classroom` 服务器重新下载完整的虚拟机镜像，但仅在万不得已时使用。

```bash
rht-vmctl fullreset servera -y
```

## 使用技巧与图形化工具

### 命令补全技巧

在输入 `rht-vmctl` 命令时，可以多使用 `Tab` 键进行自动补全，提高效率并减少拼写错误。
*   输入 `rht-vm` 后按两次 `Tab`，会显示可用的命令补全（如 `rht-vmctl`）。
*   输入 `rht-vmctl start ` 后按两次 `Tab`，会列出可启动的虚拟机名称（如 `servera`, `all`）。

### 图形化工具（了解即可）

实验环境中也存在一个图形化的虚拟机管理工具 `virt-manager`。它提供了类似 VMware 的界面，可以查看、开关机和管理虚拟机。然而，**强烈建议管理员使用命令行工具 `rht-vmctl`**，因为这是更专业和高效的运维方式。
