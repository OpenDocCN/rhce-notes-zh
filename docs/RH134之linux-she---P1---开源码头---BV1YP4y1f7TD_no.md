## 脚本的基本概念

上一节我们介绍了脚本编程的目的。本节中，我们来看看什么是脚本以及它的核心工作原理。

脚本的本质是将一系列命令按顺序写入一个文本文件。计算机的核心优势在于高速执行**循环**和**逻辑判断**。所有编程语言，包括Shell脚本，都是围绕这两个核心来组织命令，让计算机自动、高效地工作。

对于Shell脚本，需要一个解释器来逐行读取并执行文件中的命令。在Linux中，最常用的解释器就是Bash。

### 创建与执行脚本

以下是创建一个基本脚本的步骤：

1.  **指定解释器（魔术行）**：脚本的第一行通常称为“魔术行”，用于指定使用哪个解释器来执行后续命令。其格式为：
    ```bash
    #!/bin/bash
    ```
    这行代码告诉系统，使用`/bin/bash`这个程序来解释本文件。

2.  **编写命令序列**：在魔术行之后，可以按顺序写入需要执行的命令，就像在终端中手动输入一样。

3.  **赋予执行权限**：创建好的文本文件默认没有执行权限。需要使用`chmod`命令为其添加执行权。
    ```bash
    chmod +x script_name.sh
    ```

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_5.png)

4.  **执行脚本**：可以通过以下方式执行脚本：
    *   使用绝对或相对路径：`./script_name.sh`
    *   如果脚本位于`PATH`环境变量包含的目录中，可以直接输入脚本名执行。

## 输入与输出操作

理解了脚本的基础后，我们来看看如何让脚本与用户交互，即处理输入和输出。

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_10.png)

### 输出信息

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_12.png)

在脚本中向屏幕输出信息最常用的命令是`echo`。

**示例：输出文本**
```bash
#!/bin/bash
echo “Hello, World!”
echo “当前时间是：$(date)”
```
`echo`命令会将后面的字符串打印到标准输出（通常是终端屏幕）。`$(date)`是命令替换的语法，会先执行`date`命令，然后用其输出结果替换该部分。

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_14.png)

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_16.png)

### 接收输入

要让脚本接收用户的输入，可以使用`read`命令。

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_18.png)

**示例：读取用户输入**
```bash
#!/bin/bash
echo “请输入您的名字：”
read username
echo “您好，$username！”
```
`read`命令会等待用户输入，并将输入的内容赋值给指定的变量（本例中是`username`）。之后，可以通过`$变量名`的格式来引用这个变量的值。

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_20.png)

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_21.png)

**示例：读取多个输入**
```bash
#!/bin/bash
read -p “请输入用户名：” user
read -sp “请输入密码：” pass
echo
echo “用户 $user 的密码已接收。”
```
*   `-p`选项允许在等待输入的同时显示提示符。
*   `-s`选项使输入静默，不显示在屏幕上，常用于输入密码。

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_23.png)

### 位置参数

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_25.png)

除了交互式输入，脚本还可以在启动时通过命令行接收参数，这些参数称为位置参数。

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_27.png)

![](img/d1195b430d0f21fc3eb4ca6d140f7b05_29.png)

**示例：使用位置参数**
```bash
#!/bin/bash
echo “脚本名称：$0”
echo “第一个参数：$1”
echo “第二个参数：$2”
echo “所有参数：$@”
echo “参数总数：$#”
```
执行该脚本：`./myscript.sh arg1 arg2`
*   `$0`：脚本自身的名称。
*   `$1`, `$2`, ...：第1个、第2个...参数。
*   `$@`：所有参数的列表。
*   `$#`：传递给脚本的参数个数。
