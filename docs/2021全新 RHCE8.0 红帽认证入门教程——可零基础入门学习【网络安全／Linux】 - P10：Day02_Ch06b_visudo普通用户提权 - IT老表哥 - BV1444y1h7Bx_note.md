## 为什么需要 `visudo`？

![](img/f3d64a8424d785fbdee1106fbb9dd54c_22.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_23.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_24.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_25.png)

在 Linux 系统中，普通用户默认无法执行像 `fdisk -l`（查看磁盘分区）这样的管理员命令。直接赋予普通用户完整的 root 权限是危险的。`visudo` 工具允许我们安全地编辑 `/etc/sudoers` 配置文件，为特定用户或组授予执行特定命令的临时权限。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_27.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_28.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_29.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_30.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_31.png)

`/etc/sudoers` 文件的权限设置非常严格，普通用户甚至 root 用户都不应直接使用 `vi` 编辑它。以下是查看其权限的命令：
```bash
ll /etc/sudoers
```
你会看到该文件对其他用户只有读权限。因此，我们必须使用 `visudo` 命令来编辑，因为它会在保存前进行语法检查，防止配置错误导致系统无法使用 `sudo`。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_33.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_34.png)

## 如何编辑 `/etc/sudoers` 文件

![](img/f3d64a8424d785fbdee1106fbb9dd54c_39.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_40.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_41.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_42.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_44.png)

在 root 用户下，执行以下命令开始编辑：
```bash
visudo
```
该命令会调用一个临时副本进行编辑。配置文件的基本结构包含注释（以 `#` 开头）和实际的授权规则。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_46.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_48.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_49.png)

我们重点关注授权规则的格式。基本格式如下：
```
用户名 主机名=(可切换到的用户) 可执行的命令
```

![](img/f3d64a8424d785fbdee1106fbb9dd54c_52.png)

### 1. 为单个用户授权

![](img/f3d64a8424d785fbdee1106fbb9dd54c_62.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_63.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_64.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_65.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_66.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_67.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_68.png)

假设我们需要让用户 `test2` 能够执行 `fdisk -l` 命令。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_70.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_71.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_73.png)

首先，使用 `which` 命令查找 `fdisk` 的绝对路径：
```bash
which fdisk
```
通常路径是 `/usr/sbin/fdisk`。

然后，在 `visudo` 编辑器中添加一行（例如，在文件末尾）：
```
test2    ALL=(ALL)       /usr/sbin/fdisk -l
```
这表示：用户 `test2` 可以从任何主机登录，并可以切换到任何用户身份来执行 `/usr/sbin/fdisk -l` 命令。

保存并退出后，语法无误则配置生效。

### 2. 用户如何使用授权

切换到 `test2` 用户：
```bash
su - test2
```
现在，使用 `sudo` 来执行被授权的命令：
```bash
sudo fdisk -l
```
系统会提示输入 `test2` 用户自己的密码（**不是 root 密码**）。首次成功验证后，在接下来的 **5 分钟** 内再次执行 `sudo` 命令可能不再需要密码（取决于配置）。

### 3. 配置无需密码执行

如果希望授权用户执行特定命令时无需输入密码，可以在配置行中添加 `NOPASSWD:` 选项。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_75.png)

例如，修改 `test2` 的配置为：
```
test2    ALL=(ALL)       NOPASSWD: /usr/sbin/fdisk -l
```
保存后，`test2` 用户执行 `sudo fdisk -l` 时将不再需要输入密码。

### 4. 为用户组授权

![](img/f3d64a8424d785fbdee1106fbb9dd54c_77.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_78.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_79.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_81.png)

除了单个用户，也可以为整个用户组授权。在用户名前加上 `%` 符号表示组名。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_83.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_84.png)

例如，授权 `testgroup` 组的所有成员可以执行 `fdisk`：
```
%testgroup    ALL=(ALL)       /usr/sbin/fdisk
```

![](img/f3d64a8424d785fbdee1106fbb9dd54c_86.png)

### 5. 使用别名简化管理

![](img/f3d64a8424d785fbdee1106fbb9dd54c_88.png)

`/etc/sudoers` 文件支持使用别名来管理用户、主机和命令的集合，使配置更清晰。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_90.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_92.png)

以下是定义和使用别名的示例：

![](img/f3d64a8424d785fbdee1106fbb9dd54c_93.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_95.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_97.png)

*   **定义用户别名**：将多个用户归为一个组。
    ```
    User_Alias TESTADMINS = test1, test2
    ```

![](img/f3d64a8424d785fbdee1106fbb9dd54c_99.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_100.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_102.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_104.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_106.png)

*   **定义命令别名**：将多个命令归为一个集合。
    ```
    Cmnd_Alias STORAGECMDS = /usr/sbin/fdisk, /usr/bin/passwd
    ```
    **注意**：别名必须使用大写字母。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_108.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_109.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_111.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_113.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_114.png)

*   **使用别名进行授权**：
    ```
    TESTADMINS    ALL=(ALL)       NOPASSWD: STORAGECMDS
    ```
    这表示 `TESTADMINS` 别名下的所有用户，可以在所有主机上无需密码地执行 `STORAGECMDS` 别名下的所有命令。

## 总结

![](img/f3d64a8424d785fbdee1106fbb9dd54c_120.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_122.png)

本节课中我们一起学习了 `visudo` 的核心用法。我们了解到：

![](img/f3d64a8424d785fbdee1106fbb9dd54c_123.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_125.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_127.png)

1.  **目的**：`visudo` 用于安全地配置 `sudo` 权限，让普通用户能够临时、有限地执行管理员命令，提升系统安全性。
2.  **核心配置**：授权规则的基本格式是 `用户 主机=(身份) 命令`。
3.  **关键选项**：使用 `NOPASSWD:` 可以让授权用户执行命令时免密码验证。
4.  **扩展管理**：可以通过为用户组授权以及使用**用户别名**、**命令别名**来更高效地管理大量用户和复杂的权限需求。

![](img/f3d64a8424d785fbdee1106fbb9dd54c_129.png)

![](img/f3d64a8424d785fbdee1106fbb9dd54c_131.png)

通过合理配置 `sudo`，我们可以在日常运维中实现权限分离和安全管控，例如允许普通用户查看系统信息或重启特定服务，但禁止其执行关机或删除关键文件等危险操作。
