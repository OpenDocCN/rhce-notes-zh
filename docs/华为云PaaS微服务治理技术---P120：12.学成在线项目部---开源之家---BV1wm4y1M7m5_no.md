# 华为云PaaS微服务治理技术 - P120：12.学成在线项目部署-edgeservice-创建工作负载 🚀

在本节课程中，我们将学习如何为“学成在线”项目的网关服务（edgeservice）在华为云平台上创建工作负载，完成其部署，并验证网关的路由功能。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_1.png)

上一节我们介绍了如何为其他微服务创建工作负载，本节中我们来看看如何部署网关服务。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_2.png)

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_4.png)

## 创建工作负载

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_5.png)

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_7.png)

首先，我们需要在华为云控制台为网关服务创建一个无状态工作负载。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_9.png)

1.  在控制台导航中，点击“工作负载”，然后选择“无状态负载”。
2.  点击“创建”按钮，开始创建工作负载。
3.  在基本信息页面，输入工作负载名称，例如 `xc-govern-edgeservice`。
4.  设置“实例数量”为1。
5.  点击“下一步”，进入容器配置页面。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_11.png)

以下是添加容器镜像的步骤：
*   选择之前已上传的 `edgeservice` 镜像。
*   设置容器规格，例如CPU为0.5核，内存限制为1024Mi，内存申请为2048Mi。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_13.png)

配置完成后，点击“下一步”。

## 配置服务访问

接下来，我们需要配置环境变量和数据卷。根据当前服务配置，无需额外设置环境变量，因此直接点击“下一步”。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_15.png)

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_17.png)

现在，我们需要为网关服务添加一个外部访问地址，以便能够从公网访问。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_19.png)

以下是添加服务访问的步骤：
*   服务名称保持为 `edgeservice`。
*   访问类型选择“公网访问”。
*   服务亲和性保持默认。
*   端口配置中，容器端口和服务端口均设置为 `50201`。
*   选择“弹性IP”作为访问方式。

配置完成后，点击“确定”，然后点击“创建”。工作负载创建成功后，状态将显示为“运行中”。

## 检查日志与验证部署

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_21.png)

工作负载启动后，我们需要观察其日志以确保服务正常启动。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_23.png)

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_25.png)

1.  在工作负载列表页，点击新创建的 `edgeservice` 工作负载名称。
2.  进入详情页后，切换到“日志”页签。
3.  查看日志输出，确认服务启动过程中没有报错信息，并成功注册到服务注册中心。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_27.png)

日志检查无误后，我们可以开始测试网关的路由功能。网关支持两种路由方式：默认路由和基于URL路径的路由。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_29.png)

### 测试默认路由

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_31.png)

默认路由的访问路径格式为：`http://{网关外部IP}:{端口}/api/{微服务名称}/{具体API路径}`。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_33.png)

例如，要访问学习中心服务（`xc-service-learning`）的课程查询接口：
*   首先，在“工作负载”详情页的“访问方式”中，找到 `edgeservice` 的外部访问地址（IP和端口）。
*   构造访问URL：`http://{网关IP}:50201/api/xc-service-learning/learning/course/list?pageNo=1&pageSize=2`
*   在浏览器或API测试工具中访问该URL，若能成功返回课程数据，则证明默认路由配置生效。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_34.png)

### 测试URL路径路由

URL路径路由的访问格式为：`http://{网关外部IP}:{端口}/{自定义路由前缀}/{具体API路径}`。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_36.png)

根据网关配置，我们设置了 `openapi` 作为搜索服务的路由前缀。
*   构造访问URL：`http://{网关IP}:50201/openapi/search/course/list?pageNo=1&pageSize=2`
*   在浏览器或API测试工具中访问该URL，若能成功返回搜索数据，则证明URL路径路由配置生效。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_38.png)

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_40.png)

## 查看服务拓扑图

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_41.png)

最后，我们可以通过华为云的服务治理功能查看当前部署的微服务拓扑关系图。

1.  在控制台进入“应用服务网格”或“服务目录”相关页面。
2.  找到服务拓扑图功能，可以清晰地看到 `edgeservice` 网关以及其下游的各个微服务实例。
3.  此视图直观地展示了所有服务均已成功部署在云平台，并且通过网关进行统一路由和管理。

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_43.png)

![](img/a53bd89f3f2b8e96becc81ae0d5395ac_45.png)

本节课中我们一起学习了如何在华为云平台为网关服务创建工作负载，完成了从部署到验证的全过程。我们掌握了查看服务日志的方法，并成功测试了网关的两种路由模式。至此，“学成在线”项目微服务层的云端部署已全部完成。
