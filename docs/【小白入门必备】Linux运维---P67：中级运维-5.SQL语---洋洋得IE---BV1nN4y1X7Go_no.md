## 删除数据：DELETE命令详解 🔍

上一节我们介绍了插入数据，本节中我们来看看如何删除数据。DELETE命令用于从数据库表中移除记录。

**`DELETE FROM`** 语句可以删除表中的所有数据，其基本语法如下：
```sql
DELETE FROM 表名;
```
例如，`DELETE FROM xh;` 会删除 `xh` 表中的所有行。这个命令需要谨慎使用，因为它会清空整个表。

![](img/42f9cd3d26757f825d6113640708e835_7.png)

通常，我们更希望删除特定的行，而不是全部数据。这时就需要用到 **`WHERE`** 子句来指定删除条件。

![](img/42f9cd3d26757f825d6113640708e835_9.png)

**`WHERE`** 子句的作用是限制操作的范围，它不仅用于DELETE，也用于UPDATE和SELECT命令。其语法是：
```sql
DELETE FROM 表名 WHERE 条件;
```
条件可以基于表中的任何列（字段）。例如，要删除 `id` 为5的行：
```sql
DELETE FROM xh WHERE id=5;
```
为了确保只删除唯一的一行，最好使用具有唯一性的列作为条件，例如主键（ID）、身份证号等。

以下是使用WHERE子句进行条件删除的几种常见方法：
*   **基于唯一ID删除**：`DELETE FROM xh WHERE id=5;`
*   **基于其他唯一字段删除**：`DELETE FROM xh WHERE db=‘星海’;`
*   **组合多个条件删除**：`DELETE FROM xh WHERE id=5 AND db=‘星海’;`

![](img/42f9cd3d26757f825d6113640708e835_11.png)

![](img/42f9cd3d26757f825d6113640708e835_13.png)

如果条件匹配了多行，那么所有这些行都会被删除。DELETE命令只能以行为单位进行删除。

## 更新数据：UPDATE命令详解 ✏️

![](img/42f9cd3d26757f825d6113640708e835_15.png)

学会了删除，接下来我们学习如何修改已有的数据。UPDATE命令用于修改表中现有的记录。

**`UPDATE`** 语句的基本结构如下：
```sql
UPDATE 表名 SET 列名1=新值1, 列名2=新值2 WHERE 条件;
```
与DELETE类似，**如果不使用WHERE子句，UPDATE会修改表中所有行的数据**。例如，`UPDATE xh SET system=‘否’;` 会把所有行的 `system` 列都改为“否”。

![](img/42f9cd3d26757f825d6113640708e835_17.png)

![](img/42f9cd3d26757f825d6113640708e835_19.png)

![](img/42f9cd3d26757f825d6113640708e835_21.png)

因此，在大多数情况下，我们必须使用WHERE子句来限定要更新的行。例如，将id为5的行的system字段改为“是”：
```sql
UPDATE xh SET system=‘是’ WHERE id=5;
```
WHERE子句在这里同样是确保操作精确性的关键。

## 数据控制：权限管理（GRANT & REVOKE） 🛡️

![](img/42f9cd3d26757f825d6113640708e835_23.png)

![](img/42f9cd3d26757f825d6113640708e835_25.png)

在管理数据库时，控制不同用户的访问权限至关重要。这属于数据控制语言（DCL）的范畴。

![](img/42f9cd3d26757f825d6113640708e835_27.png)

MySQL中使用 **`GRANT`** 命令授予权限，使用 **`REVOKE`** 命令撤销权限。

**`GRANT`** 命令的基本语法如下：
```sql
GRANT 权限 ON 数据库名.表名 TO ‘用户名’@‘主机’ IDENTIFIED BY ‘密码’;
```
*   **权限**：可以是 `ALL PRIVILEGES`（所有权限），或具体的如 `SELECT`, `INSERT`, `UPDATE` 等。
*   **数据库名.表名**：指定权限的作用范围。`*.*` 代表所有数据库的所有表。
*   **‘用户名’@‘主机’**：指定用户和允许其连接的主机。`‘用户名’@‘localhost’` 代表仅允许本地连接，`‘用户名’@‘%’` 代表允许所有主机连接。
*   **IDENTIFIED BY ‘密码’**：为用户设置密码。如果用户不存在，此命令会创建该用户。

例如，授予用户 `mysql` 本地连接的所有权限：
```sql
GRANT ALL PRIVILEGES ON *.* TO ‘mysql’@‘localhost’ IDENTIFIED BY ‘password’;
```

![](img/42f9cd3d26757f825d6113640708e835_29.png)

**`REVOKE`** 命令用于撤销权限，语法与GRANT相似：
```sql
REVOKE 权限 ON 数据库名.表名 FROM ‘用户名’@‘主机’;
```
例如，撤销刚才授予 `mysql` 用户的所有权限：
```sql
REVOKE ALL PRIVILEGES ON *.* FROM ‘mysql’@‘localhost’;
```
需要注意的是，普通用户即使拥有 `ALL PRIVILEGES`，通常也不具备 `GRANT OPTION`（授权权限），该权限一般仅限root用户拥有。

## 数据类型：数值型（整数与小数） 🔢

![](img/42f9cd3d26757f825d6113640708e835_31.png)

![](img/42f9cd3d26757f825d6113640708e835_33.png)

为了更有效地创建和修改表结构，我们需要了解MySQL的数据类型。首先来看数值类型，它主要分为整数型和小数型（浮点型）。

![](img/42f9cd3d26757f825d6113640708e835_35.png)

![](img/42f9cd3d26757f825d6113640708e835_37.png)

### 整数型（INTEGER）

![](img/42f9cd3d26757f825d6113640708e835_39.png)

整数类型用于存储没有小数部分的数字。MySQL提供了多种整数类型，区别主要在于存储范围。

![](img/42f9cd3d26757f825d6113640708e835_41.png)

![](img/42f9cd3d26757f825d6113640708e835_43.png)

![](img/42f9cd3d26757f825d6113640708e835_45.png)

以下是主要的整数类型及其范围：
*   **TINYINT**：非常小的整数，范围约 -128 到 127。
*   **SMALLINT**：小整数，范围约 -32768 到 32767。
*   **MEDIUMINT**：中等大小整数。
*   **INT**：标准整数，范围约 -21亿 到 21亿。**最常用**。
*   **BIGINT**：大整数，范围极大。

![](img/42f9cd3d26757f825d6113640708e835_47.png)

在定义整数类型时，可以指定显示宽度，例如 `INT(11)`。但需要注意的是，**这个宽度只是显示宽度（一个提示），并不严格限制能存储的数值范围**。真正的范围由数据类型本身（如INT）决定。例如，`TINYINT(2)` 仍然只能存储 -128 到 127 之间的数，超过两位的数字也能存入。

### 小数型/浮点型（FLOAT & DOUBLE）

![](img/42f9cd3d26757f825d6113640708e835_49.png)

![](img/42f9cd3d26757f825d6113640708e835_51.png)

小数类型用于存储带有小数点的数字。

定义浮点类型时，可以指定精度：
```sql
FLOAT(M, D)
DOUBLE(M, D)
```
*   **M**：表示总位数（整数位+小数位）。
*   **D**：表示小数位数。

例如，`FLOAT(5,2)` 可以存储像 `123.45` 这样的数（总位数为5，小数位为2）。如果不指定 `(M, D)`，则会使用默认精度。

![](img/42f9cd3d26757f825d6113640708e835_53.png)

![](img/42f9cd3d26757f825d6113640708e835_55.png)

**`FLOAT`** 是单精度浮点数。
**`DOUBLE`** 是双精度浮点数，精度比FLOAT更高。

![](img/42f9cd3d26757f825d6113640708e835_57.png)

![](img/42f9cd3d26757f825d6113640708e835_59.png)

插入浮点数时，直接写入数字即可，无需引号：
```sql
INSERT INTO test4 (a) VALUES (123.45);
```
