## 概述：系统启动与 `/etc/fstab`

![](img/e9ca6520bd524879514e918fac3a0387_11.png)

![](img/e9ca6520bd524879514e918fac3a0387_12.png)

上一节我们介绍了文件系统挂载的基本概念。本节中我们来看看，如果负责自动挂载的配置文件 `/etc/fstab` 出现错误，会导致什么后果，以及如何修复。

`/etc/fstab` 文件是系统启动时自动挂载文件系统的配置文件。如果其中的条目存在错误（例如，指定了不存在的设备或错误的挂载点），系统在启动过程中就会失败，无法进入正常的登录界面。

## 模拟故障场景

![](img/e9ca6520bd524879514e918fac3a0387_17.png)

首先，我们通过修改 `/etc/fstab` 文件来人为制造一个启动故障。

![](img/e9ca6520bd524879514e918fac3a0387_19.png)

![](img/e9ca6520bd524879514e918fac3a0387_21.png)

1.  编辑 `/etc/fstab` 文件，在末尾添加一个错误的挂载条目。
2.  例如，添加一行无效的配置，如将一个不存在的设备挂载到 `/test` 目录。
3.  保存文件并重启系统，系统将因挂载错误而无法正常启动。

![](img/e9ca6520bd524879514e918fac3a0387_23.png)

![](img/e9ca6520bd524879514e918fac3a0387_25.png)

![](img/e9ca6520bd524879514e918fac3a0387_26.png)

![](img/e9ca6520bd524879514e918fac3a0387_28.png)

**核心操作命令：**
```bash
# 编辑 fstab 文件
vi /etc/fstab
# 在文件末尾添加一行错误配置，例如：
# /dev/sdx1 /test xfs defaults 0 0
```

## 进入救援模式

![](img/e9ca6520bd524879514e918fac3a0387_33.png)

![](img/e9ca6520bd524879514e918fac3a0387_35.png)

当系统因 `/etc/fstab` 错误无法启动时，我们需要进入救援模式进行修复。

![](img/e9ca6520bd524879514e918fac3a0387_36.png)

![](img/e9ca6520bd524879514e918fac3a0387_38.png)

![](img/e9ca6520bd524879514e918fac3a0387_40.png)

以下是进入救援模式并获取root shell的步骤：

![](img/e9ca6520bd524879514e918fac3a0387_42.png)

![](img/e9ca6520bd524879514e918fac3a0387_44.png)

1.  在系统启动的GRUB菜单界面，选择要启动的内核，然后按下 `e` 键进入编辑模式。
2.  在编辑界面中，找到以 `linux` 或 `linux16` 开头的行。
3.  在该行的末尾，添加 `rd.break` 参数。
4.  按下 `Ctrl + X` 或 `F10` 使用修改后的参数启动系统。
5.  系统将进入一个临时的救援环境，并最终提供一个 `switch_root` 提示符。

## 修复 `/etc/fstab` 文件

![](img/e9ca6520bd524879514e918fac3a0387_51.png)

![](img/e9ca6520bd524879514e918fac3a0387_53.png)

在救援模式的shell中，我们拥有root权限，可以重新挂载根文件系统并编辑有问题的 `/etc/fstab` 文件。

![](img/e9ca6520bd524879514e918fac3a0387_55.png)

以下是修复步骤：

1.  重新以读写模式挂载根文件系统：
    ```bash
    mount -o remount,rw /sysroot
    ```
2.  切换根目录到 `/sysroot`：
    ```bash
    chroot /sysroot
    ```
3.  使用文本编辑器（如 `vi`）打开 `/etc/fstab` 文件：
    ```bash
    vi /etc/fstab
    ```
4.  找到导致启动失败的错误行。通常有两种处理方式：
    *   **注释掉错误行**：在该行行首添加 `#` 号。
    *   **直接删除错误行**。
5.  保存并退出编辑器（在 `vi` 中按 `ESC`，然后输入 `:wq`）。
6.  退出 `chroot` 环境并重启系统：
    ```bash
    exit
    reboot
    ```

**关键点：** 在RHCE考试中，如果因为新增磁盘或修改挂载导致系统无法启动，有很高的概率是 `/etc/fstab` 配置问题。掌握此修复方法是避免得零分的关键之一。
