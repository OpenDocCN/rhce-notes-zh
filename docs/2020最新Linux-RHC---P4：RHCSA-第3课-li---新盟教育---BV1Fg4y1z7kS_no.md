## Linux目录结构 🌳

![](img/1c0f37417efd1109d3597f97dc34e0e9_7.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_9.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_11.png)

上一节我们介绍了Linux系统启动过程，本节中我们来看看Linux的文件系统是如何组织的。与Windows使用C盘、D盘等盘符不同，Linux采用一种倒置的树状结构，一切皆从根目录（`/`）开始。

![](img/1c0f37417efd1109d3597f97dc34e0e9_13.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_15.png)

Linux遵循“一切皆文件”的原则，无论是硬件设备、软件服务还是存储分区，最终都以文件的形式呈现。同时，Linux系统严格区分大小写。

![](img/1c0f37417efd1109d3597f97dc34e0e9_17.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_19.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_21.png)

以下是Linux系统中一些重要目录及其功能的介绍：

![](img/1c0f37417efd1109d3597f97dc34e0e9_23.png)

*   **`/` (根分区)**： 整个文件系统的起点，存放着`/etc`、`/bin`、`/dev`等核心目录。
*   **`/boot`**： 存放系统启动所需的引导文件，例如内核文件（`vmlinuz-*`）和初始化内存盘文件（`initramfs-*.img`）。
*   **`/dev`**： 存放设备文件。例如，光盘设备通常有一个软链接指向这里（如`/dev/cdrom`）。
*   **`/etc`**： 存放系统及应用程序的配置文件。例如，使用`yum`安装Apache后，其配置文件通常在`/etc/httpd/`目录下。
*   **`/home`**： 普通用户的家目录所在地。系统会为每个普通用户在此创建同名目录（如`/home/zhangsan`），用户在自己的家目录内拥有最高权限。
*   **`/root`**： 超级管理员`root`用户的家目录。
*   **`/bin`**： 存放二进制可执行文件，这些命令在单用户模式下仍可使用。
*   **`/sbin`**： 存放系统管理命令，通常只有`root`用户有权执行，普通用户需要提权。
*   **`/lib`**： 存放系统库文件，为`/bin`和`/sbin`中的程序提供函数支持。
*   **`/media` 与 `/mnt`**： 用于挂载外部存储设备，如U盘、光盘或额外的硬盘分区。
*   **`/opt`**： 通常用于存放第三方软件或源码编译安装的应用程序。
*   **`/tmp`**： 临时文件目录，所有用户均可读写。该目录通常设置了特殊的“粘滞位”（Sticky Bit）权限，确保用户只能删除自己的文件。
*   **`/proc`**： 虚拟文件系统，存放进程和内核信息（如CPU信息`/proc/cpuinfo`，内存信息`/proc/meminfo`），用于系统监控和调优。
*   **`/usr/local`**： 用户通过源码编译安装软件时的默认目录。
*   **`/var/log`**： 存放系统及应用程序的日志文件，如系统日志`/var/log/messages`，服务日志`/var/log/httpd/`。

理解了目录结构后，我们需要明确如何定位文件。这引出了两个概念：绝对路径和相对路径。

![](img/1c0f37417efd1109d3597f97dc34e0e9_40.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_42.png)

*   **绝对路径**： 从根目录（`/`）开始，完整地描述文件或目录的位置。例如：`/etc/sysconfig/network-scripts/ifcfg-ens33`。
*   **相对路径**： 从当前工作目录开始，描述目标位置的路径。例如，当前在`/etc`目录下，进入`sysconfig`可以写作 `cd sysconfig` 或 `cd ./sysconfig`。
    *   `.` 代表当前目录。
    *   `..` 代表上一级目录。

## 链接文件 🔗

![](img/1c0f37417efd1109d3597f97dc34e0e9_48.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_50.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_52.png)

在`/dev`目录中我们提到了“软链接”，Linux中有两种链接文件：

*   **硬链接 (Hard Link)**： 相当于给原文件创建了一个完整的副本（相同的inode号），但不允许对目录创建。删除原文件，硬链接文件依然有效。创建命令：`ln 源文件 目标文件`
*   **软链接 (Soft Link / Symbolic Link)**： 类似于Windows的快捷方式（显示为`l`），可以指向文件或目录。删除原文件，软链接将失效。创建命令：`ln -s 源文件 目标文件`

![](img/1c0f37417efd1109d3597f97dc34e0e9_54.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_56.png)

可以使用 `ls -i` 或 `ll -i` 查看文件的inode号来区分。

## 用户与用户组管理 👥

Linux是一个多用户操作系统，为了安全和便于管理，引入了用户和用户组的概念。

![](img/1c0f37417efd1109d3597f97dc34e0e9_70.png)

### 用户分类
Linux用户主要分为三类：

![](img/1c0f37417efd1109d3597f97dc34e0e9_72.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_74.png)

1.  **超级用户 (root)**： 用户ID（UID）和组ID（GID）均为0的用户，拥有系统最高权限。
2.  **系统用户**： UID范围（RHEL/CentOS 7+为1-999），用于运行系统服务，不可登录系统，无家目录。
3.  **普通用户**： UID范围（RHEL/CentOS 7+为1000+），由`root`用户创建，权限由`root`分配，拥有自己的家目录。

用户信息存储在 `/etc/passwd` 文件中，密码的加密信息存储在 `/etc/shadow` 文件中。组信息存储在 `/etc/group` 和 `/etc/gshadow` 中。

![](img/1c0f37417efd1109d3597f97dc34e0e9_76.png)

### 用户基本操作
以下是用户管理的基本命令：

![](img/1c0f37417efd1109d3597f97dc34e0e9_78.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_80.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_81.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_83.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_85.png)

*   **创建用户**： `useradd [选项] 用户名`
    *   常用选项：
        *   `-M`： 不创建家目录（常用于创建系统用户）。
        *   `-s`： 指定用户的登录Shell（如`/sbin/nologin`）。
        *   `-u`： 指定UID。
        *   `-g`： 指定主组（GID或组名）。
        *   `-G`： 指定附加组。
*   **删除用户**： `userdel -r 用户名` （`-r`选项会同时删除用户家目录）
*   **修改用户属性**： `usermod [选项] 用户名`
    *   常用选项：`-g`（改主组）、`-G`（改附加组）、`-s`（改Shell）、`-L`（锁定用户）、`-U`（解锁用户）。
*   **设置/修改密码**： `passwd [用户名]`
    *   `root`用户可以为任何用户设置密码：`passwd 用户名`
    *   普通用户只能修改自己的密码：`passwd`
*   **查看用户信息**： `id 用户名` 可以查看用户的UID、GID及所属组。

![](img/1c0f37417efd1109d3597f97dc34e0e9_87.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_89.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_90.png)

![](img/1c0f37417efd1109d3597f97dc34e0e9_92.png)

### 用户组概念
每个用户必须属于一个主组（通常是与用户名同名的组），也可以属于多个附加组。组的存在便于对具有相同权限需求的用户进行批量管理。文件或目录的权限会针对“属主”、“属组”和“其他用户”分别设置。
