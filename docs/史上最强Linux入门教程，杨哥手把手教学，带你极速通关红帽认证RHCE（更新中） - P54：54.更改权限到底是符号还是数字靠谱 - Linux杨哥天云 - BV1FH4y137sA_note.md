在本节课中，我们将要学习Linux中设置文件权限的两种主要方法——符号法和数字法，并分析它们各自的适用场景与优缺点。

上一节我们介绍了如何通过符号（如 `u+x`）和数字（如 `755`）来更改文件权限。本节中我们来看看在实际操作中，哪种方法更常用或更“靠谱”。

## 两种方法概述 📋

![](img/18b1d3065184e4466bef2f74920db5ef_15.png)

两种方法都有其存在的价值。数字法在某些情况下显得非常简洁高效，而符号法在处理局部权限调整时则更加直观和严谨。

![](img/18b1d3065184e4466bef2f74920db5ef_17.png)

![](img/18b1d3065184e4466bef2f74920db5ef_19.png)

## 数字法的优势与应用场景

![](img/18b1d3065184e4466bef2f74920db5ef_21.png)

![](img/18b1d3065184e4466bef2f74920db5ef_23.png)

数字法通过三位八进制数（如 `644`、`777`）一次性设置所有者、所属组和其他人的所有权限，非常简洁。

![](img/18b1d3065184e4466bef2f74920db5ef_25.png)

![](img/18b1d3065184e4466bef2f74920db5ef_27.png)

以下是数字法适用的典型场景：
*   **权限推倒重来**：当你不关心文件原有权限，希望直接赋予全新且完整的权限时。例如，设置文件为所有者可读写执行（`700`），或所有人可读写执行（`777`）。
*   **记忆与批量设置**：对于常见的权限组合，数字形式更容易记忆和书写，也便于在脚本中进行批量操作。

![](img/18b1d3065184e4466bef2f74920db5ef_29.png)

![](img/18b1d3065184e4466bef2f74920db5ef_31.png)

**命令示例**：
```bash
# 设置文件权限为所有者读写，组和其他人只读
chmod 644 /home/download/password.txt
# 设置文件为所有用户拥有全部权限
chmod 777 filename
# 设置文件为仅所有者拥有全部权限
chmod 700 filename
```

![](img/18b1d3065184e4466bef2f74920db5ef_33.png)

![](img/18b1d3065184e4466bef2f74920db5ef_35.png)

## 符号法的优势与应用场景

![](img/18b1d3065184e4466bef2f74920db5ef_37.png)

![](img/18b1d3065184e4466bef2f74920db5ef_39.png)

符号法通过操作符（`+`， `-`， `=`）和身份标识（`u`， `g`， `o`， `a`）来增加、移除或设置特定权限。

![](img/18b1d3065184e4466bef2f74920db5ef_41.png)

![](img/18b1d3065184e4466bef2f74920db5ef_43.png)

符号法在需要对现有权限进行微调时优势明显。你无需计算文件当前的数字权限，可以直接针对特定对象进行操作。

![](img/18b1d3065184e4466bef2f74920db5ef_44.png)

![](img/18b1d3065184e4466bef2f74920db5ef_46.png)

以下是符号法适用的典型场景：
*   **局部权限调整**：当你只想为某个特定身份增加或移除一项权限，而保持其他权限不变时。
*   **操作明确直观**：命令直接表达了“给谁”进行“何种操作”，意图清晰，不易因计算错误而导致意外结果。

![](img/18b1d3065184e4466bef2f74920db5ef_48.png)

![](img/18b1d3065184e4466bef2f74920db5ef_50.png)

**命令示例**：
假设有一个脚本文件 `a.sh`，当前没有执行权限。
```bash
# 为所有用户添加执行权限，无需考虑原有权限数字
chmod +x a.sh
# 或仅为文件所有者添加执行权限
chmod u+x a.sh
# 为所属组成员添加写权限
chmod g+w a.sh
# 移除其他人的所有权限
chmod o= a.sh
```

![](img/18b1d3065184e4466bef2f74920db5ef_52.png)

![](img/18b1d3065184e4466bef2f74920db5ef_54.png)

## 核心对比与选择建议 ⚖️

![](img/18b1d3065184e4466bef2f74920db5ef_56.png)

![](img/18b1d3065184e4466bef2f74920db5ef_58.png)

上一节我们介绍了两种方法的具体操作，通过本节的场景分析，我们可以得出以下结论：

![](img/18b1d3065184e4466bef2f74920db5ef_60.png)

![](img/18b1d3065184e4466bef2f74920db5ef_62.png)

*   **数字法** 适用于**全局、覆盖式**的权限设置，追求效率和简洁。
*   **符号法** 适用于**局部、增量式**的权限调整，强调精确和直观。

![](img/18b1d3065184e4466bef2f74920db5ef_64.png)

![](img/18b1d3065184e4466bef2f74920db5ef_66.png)

没有绝对意义上的“更靠谱”，关键在于根据实际需求选择合适的方法。如果需要对权限进行彻底重置，数字法很快捷；如果只是进行细微修改，符号法则更严谨方便。

![](img/18b1d3065184e4466bef2f74920db5ef_68.png)

![](img/18b1d3065184e4466bef2f74920db5ef_70.png)

本节课中我们一起学习了Linux权限管理中符号法与数字法的对比。总结来说，数字法适合全局权限设置，符号法适合局部权限调整，在实际工作中应根据具体场景灵活选用这两种方法。
