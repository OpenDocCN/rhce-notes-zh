在本节课中，我们将学习如何配置NTP时间同步客户端。NTP（Network Time Protocol，网络时间协议）用于在网络中同步计算机的系统时间。我们将了解其基本概念，并掌握在红帽系统上配置NTP客户端的完整步骤。

![](img/285e42451315adf6064b1fca83090abd_7.png)

## 概述与背景

![](img/285e42451315adf6064b1fca83090abd_9.png)

![](img/285e42451315adf6064b1fca83090abd_11.png)

上一节我们介绍了系统权限管理，本节中我们来看看如何配置NTP时间同步。在RHCSA/RHCE考试中，通常会有一道题目要求你将虚拟机配置为指定NTP服务器的时间客户端。理解并掌握这一配置过程，对于通过考试和实际工作都至关重要。

![](img/285e42451315adf6064b1fca83090abd_13.png)

NTP服务分为服务器端和客户端。提供标准时间的机器称为NTP服务器，使用该标准时间的机器称为NTP客户端。从红帽7系统开始，默认的时间同步服务由`ntpd`更换为`chrony`。`chrony`服务既可以作为NTP服务器，也可以作为NTP客户端。

## 配置NTP客户端的基本步骤

配置NTP客户端主要包含三个步骤：安装软件包、修改配置文件、启动并启用服务。

![](img/285e42451315adf6064b1fca83090abd_15.png)

![](img/285e42451315adf6064b1fca83090abd_17.png)

以下是详细操作流程：

### 1. 安装软件包

![](img/285e42451315adf6064b1fca83090abd_19.png)

首先，需要确认并安装`chrony`软件包。在大多数考试环境中，该软件包可能已经预装。

```bash
yum -y install chrony
```

如果系统提示已安装，则可跳过此步骤。

![](img/285e42451315adf6064b1fca83090abd_21.png)

![](img/285e42451315adf6064b1fca83090abd_23.png)

### 2. 修改配置文件

![](img/285e42451315adf6064b1fca83090abd_25.png)

`chrony`服务的主配置文件位于`/etc/chrony.conf`。我们需要修改此文件，指定要同步的NTP服务器。

使用文本编辑器打开配置文件：

![](img/285e42451315adf6064b1fca83090abd_27.png)

```bash
vim /etc/chrony.conf
```

![](img/285e42451315adf6064b1fca83090abd_29.png)

在配置文件中，找到以`pool`或`server`开头的行（通常在文件开头部分）。考试中，需要将默认的时间服务器地址更改为题目指定的域名。

![](img/285e42451315adf6064b1fca83090abd_31.png)

**配置项说明：**
*   **`pool`**：后跟一个域名，该域名在DNS上可能解析为多个IP地址（地址池）。适用于拥有多个备用服务器的情况。
*   **`server`**：后跟一个域名或IP地址，通常解析为单个服务器。适用于指定单一NTP服务器的情况，是考试中的规范做法。

![](img/285e42451315adf6064b1fca83090abd_33.png)

**操作建议：**
将原有的`pool`行注释掉或修改为`server`行，并填入题目要求的服务器域名。同时，保留`iburst`参数，它可以让客户端在初始同步时更快地联系服务器。

例如，将默认的：
```
pool 2.rhel.pool.ntp.org iburst
```
修改为（假设题目指定服务器为`server1.example.com`）：
```
server server1.example.com iburst
```

![](img/285e42451315adf6064b1fca83090abd_35.png)

修改完成后，保存并退出编辑器。

### 3. 启动并启用服务

配置文件修改完成后，需要重启`chronyd`服务以使更改生效，并设置其开机自动启动。

```bash
systemctl restart chronyd
systemctl enable chronyd
```

## 验证配置结果

配置完成后，可以通过以下几种方法验证NTP客户端是否工作正常。

![](img/285e42451315adf6064b1fca83090abd_37.png)

以下是常用的验证命令：

### 方法一：使用 `timedatectl` 命令

此命令可以查看系统时间状态和NTP服务是否活跃。

![](img/285e42451315adf6064b1fca83090abd_39.png)

```bash
timedatectl
```
关注输出中的以下信息：
*   `NTP service: active` 表示NTP服务正在运行。
*   `NTP synchronized: yes` 表示时间已成功同步（可能需要等待片刻或重启服务后才会显示为`yes`）。

![](img/285e42451315adf6064b1fca83090abd_41.png)

![](img/285e42451315adf6064b1fca83090abd_42.png)

### 方法二：使用 `chronyc` 命令检查源

![](img/285e42451315adf6064b1fca83090abd_44.png)

这是最直接有效的验证方法，可以查看当前配置的时间源及其状态。

![](img/285e42451315adf6064b1fca83090abd_46.png)

```bash
chronyc sources -v
```
在输出列表中，找到你配置的服务器地址。关注前两列的标记：
*   **`^*`**（星号）：表示当前正在使用的、已成功同步的时间源。
*   **`^+`**（加号）：表示可用的备用时间源。
*   **`^?`**（问号）：表示连接失败的时间源。

![](img/285e42451315adf6064b1fca83090abd_48.png)

如果能看到配置的服务器地址，并且标记为`^*`，则说明配置正确且同步成功。

![](img/285e42451315adf6064b1fca83090abd_50.png)

![](img/285e42451315adf6064b1fca83090abd_52.png)

### 方法三：手动修改时间测试

![](img/285e42451315adf6064b1fca83090abd_54.png)

可以临时将系统时间改为一个错误值，然后观察其是否会被自动纠正。

![](img/285e42451315adf6064b1fca83090abd_55.png)

![](img/285e42451315adf6064b1fca83090abd_56.png)

![](img/285e42451315adf6064b1fca83090abd_58.png)

1.  设置一个过去的时间：
    ```bash
    date -s “2002-02-02 10:00:00”
    ```
2.  重启`chronyd`服务或等待一段时间（通常几分钟）后，再次运行`date`命令，查看时间是否已恢复为正确的网络时间。

## 补充知识：硬件时钟同步

![](img/285e42451315adf6064b1fca83090abd_60.png)

系统时间同步后，有时需要将正确的系统时间写入硬件时钟（BIOS时间）。可以使用`hwclock`命令。

*   将系统时间同步到硬件时钟：
    ```bash
    hwclock -w
    ```
*   根据硬件时钟设置系统时间：
    ```bash
    hwclock -s
    ```
> **注意**：此操作在考试中通常不要求，但在实际工作中可能用到。
