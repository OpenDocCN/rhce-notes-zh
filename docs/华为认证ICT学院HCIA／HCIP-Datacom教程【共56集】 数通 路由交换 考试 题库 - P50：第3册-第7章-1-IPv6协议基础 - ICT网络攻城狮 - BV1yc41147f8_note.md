## IPv4的缺陷与IPv6的提出 🚀

![](img/5c2021d764e80dbbabd8d3815c62991a_3.png)

上一节我们介绍了课程概述，本节中我们来看看IPv4协议目前存在的主要缺陷，以及IPv6协议是如何被提出的。

![](img/5c2021d764e80dbbabd8d3815c62991a_5.png)

IPv4协议主要存在以下几个缺陷：

![](img/5c2021d764e80dbbabd8d3815c62991a_7.png)

1.  **IP地址空间耗尽**：IPv4地址长度为32比特，理论上可提供约43亿个地址。由于网络爆炸式增长和早期地址分配不合理（如预留私网地址段、测试地址段等），全球IPv4地址已于2011年2月由互联网地址管理机构IANA分配完毕，根源已枯竭。
2.  **缺乏自动配置技术**：IPv4协议本身不具备自动配置能力。主机接入网络时，必须由管理员手动配置或依赖额外的DHCP服务，增加了部署和维护的复杂性。
3.  **头部字段处理负担重**：IPv4头部定义了较多字段，给网络中间设备带来较大处理负担。例如，与分片/重组相关的标识、标志、片偏移字段，以及可选项和头部校验和字段，都增加了处理的复杂度。
4.  **缺乏原生安全机制**：IPv4在设计之初未充分考虑安全性，数据传输缺乏认证和加密机制，必须依赖IPSec等额外技术来保障安全。

![](img/5c2021d764e80dbbabd8d3815c62991a_9.png)

为了解决上述问题，互联网工程任务组IETF在上世纪90年代就提出了下一代IP协议，即IPv6。

![](img/5c2021d764e80dbbabd8d3815c62991a_11.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_13.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_15.png)

IPv6的核心优势在于：
*   **近乎无限的地址空间**：IPv6地址长度为128比特，其地址数量级极其庞大，足以满足未来所有设备的联网需求。
*   **支持即插即用**：IPv6协议原生支持无状态地址自动配置，主机接入网络后可自动获取地址，无需管理员干预。
*   **简化的报文头部**：IPv6采用固定40字节的基本报头，去除了IPv4中的冗余字段，使处理更高效。
*   **内置安全特性**：IPv6通过扩展报头（如AH和ESP）原生支持认证和加密，提供了“贴身”的安全保障。

![](img/5c2021d764e80dbbabd8d3815c62991a_17.png)

## IPv6数据包封装格式 📦

![](img/5c2021d764e80dbbabd8d3815c62991a_19.png)

上一节我们介绍了IPv4的缺陷和IPv6的优势，本节中我们来看看IPv6数据包的具体封装格式，并与IPv4进行对比。

![](img/5c2021d764e80dbbabd8d3815c62991a_21.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_23.png)

以下是IPv6与IPv4头部字段的对比分析：

![](img/5c2021d764e80dbbabd8d3815c62991a_25.png)

为了更清晰地对比，我们将对IPv4头部中存在但在IPv6基本报头中已移除的字段进行说明，并解释其功能在IPv6中如何实现。

![](img/5c2021d764e80dbbabd8d3815c62991a_27.png)

*   **标识、标志、片偏移**：这些字段在IPv4中用于数据包的分片与重组。在IPv6中，为了减轻中间转发设备的处理负担，路由器不再执行分片操作。如果数据包超出链路MTU，路由器会丢弃该包并向源设备发送“数据包过大”ICMPv6消息，提示其进行**路径MTU发现**。分片工作改由源设备在发送前完成。
*   **头部校验和**：为了提升网络性能，IPv6删除了头部校验和字段。数据完整性校验交由上层传输协议（如TCP/UDP）和底层链路层协议负责。
*   **可选项**：IPv4的可选字段功能被移至IPv6的**扩展报头**中。一般情况下，中间设备只需处理基本报头，仅在特定场景下才需要处理扩展报头。

![](img/5c2021d764e80dbbabd8d3815c62991a_29.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_31.png)

IPv6基本报头中保留或改良自IPv4的字段包括：

![](img/5c2021d764e80dbbabd8d3815c62991a_33.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_35.png)

*   **版本**：取值为`6`。
*   **流量类别**：类似于IPv4中的服务类型字段，用于QoS。
*   **流标签**：新增字段，用于标识特定流，以提高QoS处理效率。一条IPv6流可以由**三元组**（源地址、目的地址、流标签）来标识。
*   **有效载荷长度**：指示IPv6基本报头之后的数据长度（包含扩展报头）。
*   **下一个报头**：类似于IPv4的协议字段，指示紧接在基本报头之后的是哪种上层协议或扩展报头。
*   **跳数限制**：类似于IPv4的TTL，每经过一台路由器减1，减至0时数据包被丢弃。
*   **源地址与目的地址**：各占128比特。

![](img/5c2021d764e80dbbabd8d3815c62991a_37.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_39.png)

## IPv6地址的表示方式 🔢

![](img/5c2021d764e80dbbabd8d3815c62991a_41.png)

上一节我们分析了IPv6的报文格式，本节中我们来看看IPv6地址独特的表示方法。

![](img/5c2021d764e80dbbabd8d3815c62991a_43.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_45.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_47.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_49.png)

IPv6地址长度为128比特，采用**冒号十六进制表示法**。其规则是将128位地址分为8组，每组16位，用4个十六进制数表示，组之间用冒号`:`分隔。

例如：
`2001:0db8:0000:0000:0008:0800:200c:417a`

![](img/5c2021d764e80dbbabd8d3815c62991a_51.png)

为了书写简洁，IPv6定义了两种压缩规则：

![](img/5c2021d764e80dbbabd8d3815c62991a_53.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_55.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_57.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_59.png)

以下是IPv6地址的压缩规则示例：

![](img/5c2021d764e80dbbabd8d3815c62991a_61.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_63.png)

1.  **省略前导零**：每组中的前导零可以省略。
    *   例如：`0db8` 可写为 `db8`；`0008` 可写为 `8`。
    *   压缩后示例变为：`2001:db8:0:0:8:800:200c:417a`
2.  **压缩连续全零组**：如果地址中包含连续两个或多个全零组（`0:0`或`0:0:0...`），可以将其压缩为双冒号`::`。**注意：一个IPv6地址中只能使用一次双冒号**。
    *   例如，上面的地址可进一步压缩为：`2001:db8::8:800:200c:417a`
    *   双冒号`::`代表被省略的连续全零组。还原时，根据总组数为8的规则，将`::`展开为适当数量的`0:`即可。

![](img/5c2021d764e80dbbabd8d3815c62991a_65.png)

![](img/5c2021d764e80dbbabd8d3815c62991a_67.png)
