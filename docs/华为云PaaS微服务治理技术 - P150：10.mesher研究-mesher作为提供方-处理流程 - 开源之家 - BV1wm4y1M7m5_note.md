在本节课中，我们将要学习Mesher作为服务提供方时，是如何被其他微服务调用的。我们将详细解析其处理流程，并通过简单的步骤说明如何配置Mesher来代理一个不具备微服务能力的本地服务。

![](img/cad1b63bddea977299b38983bb744173_3.png)

## 概述

![](img/cad1b63bddea977299b38983bb744173_5.png)

上一节我们介绍了Mesher作为消费方的角色。本节中，我们来看看Mesher作为服务提供方时的工作流程。当一个服务本身不具备微服务能力时，可以通过在其侧部署Mesher来获得微服务治理能力，从而允许其他微服务通过Mesher来调用它。

## 处理流程解析

![](img/cad1b63bddea977299b38983bb744173_10.png)

以下是Mesher作为提供方时的调用流程示意图：

![](img/cad1b63bddea977299b38983bb744173_12.png)

![](img/cad1b63bddea977299b38983bb744173_14.png)

![](img/cad1b63bddea977299b38983bb744173_1.png)

流程的核心步骤如下：

![](img/cad1b63bddea977299b38983bb744173_16.png)

![](img/cad1b63bddea977299b38983bb744173_17.png)

![](img/cad1b63bddea977299b38983bb744173_18.png)

![](img/cad1b63bddea977299b38983bb744173_19.png)

1.  **服务消费者发起调用**：一个基于CSE框架开发的微服务（消费者）需要调用目标服务。它使用标准的微服务调用地址格式：`cse://{微服务名称}`。
2.  **从注册中心发现Mesher**：消费者向服务注册中心查询目标微服务名称对应的地址。由于目标服务本身不具备微服务能力，其地址并未注册。取而代之的是，代理它的Mesher实例的地址被注册到了服务中心。因此，消费者实际获取到的是Mesher的地址。
3.  **请求转发至Mesher**：消费者根据获取到的地址，将请求发送给Mesher。
4.  **Mesher代理调用本地服务**：Mesher接收到请求后，根据其配置，将请求转发给它所代理的本地服务提供方（例如，一个运行在`127.0.0.1:40000`的普通HTTP服务）。
5.  **响应返回**：本地服务处理请求并返回结果给Mesher，Mesher再将结果返回给最初的服务消费者。

![](img/cad1b63bddea977299b38983bb744173_20.png)

![](img/cad1b63bddea977299b38983bb744173_21.png)

通过这个过程，一个原本非微服务的应用，在Mesher的代理下，具备了服务注册、发现和治理等微服务能力。

![](img/cad1b63bddea977299b38983bb744173_23.png)

![](img/cad1b63bddea977299b38983bb744173_25.png)

## 配置步骤详解

![](img/cad1b63bddea977299b38983bb744173_27.png)

理解了流程之后，我们需要完成以下三步配置，才能使Mesher成功作为提供方工作。

![](img/cad1b63bddea977299b38983bb744173_29.png)

![](img/cad1b63bddea977299b38983bb744173_31.png)

以下是具体的配置步骤：

![](img/cad1b63bddea977299b38983bb744173_32.png)

1.  **消费者配置调用地址**：服务消费者在代码中配置调用地址为 `cse://{目标微服务名称}`。这是标准做法，消费者无需关心背后是Mesher还是原生微服务。
    ```yaml
    # 示例：在消费者的配置中指定服务提供者的名称
    servicecomb:
      references:
        provider:
          name: target-service-name # Mesher代理的服务名
    ```

![](img/cad1b63bddea977299b38983bb744173_34.png)

![](img/cad1b63bddea977299b38983bb744173_35.png)

2.  **配置Mesher监听的本地服务地址**：这是关键步骤。我们需要通过环境变量告诉Mesher，它应该将接收到的请求转发到哪个本地服务地址。IP地址固定为`127.0.0.1`，端口则是本地服务实际监听的端口。
    ```bash
    # 设置环境变量，例如本地服务运行在40000端口
    export LOCAL_SERVICE_ADDR=127.0.0.1:40000
    ```

3.  **编写并注册接口契约**：为了让其他服务能够正确调用，Mesher需要将本地服务的API接口以OpenAPI契约的形式注册到服务中心。这通常通过提供一个`swagger.yaml`或`openapi.yaml`文件来实现。
    ```yaml
    # 示例：一个简单的OpenAPI契约文件片段
    openapi: 3.0.1
    info:
      title: 本地服务API
      version: 1.0.0
    paths:
      /api/resource:
        get:
          summary: 获取资源
          responses:
            '200':
              description: 成功
    ```

## 总结

![](img/cad1b63bddea977299b38983bb744173_41.png)

![](img/cad1b63bddea977299b38983bb744173_42.png)

本节课中我们一起学习了Mesher作为服务提供方的处理流程与配置方法。核心在于理解Mesher充当了一个“适配器”或“代理”的角色，它接收来自微服务框架的标准调用，并将其转发给不具备微服务能力的本地应用。通过配置调用地址、本地服务地址和接口契约这三个关键步骤，即可将传统服务快速接入微服务治理体系。
