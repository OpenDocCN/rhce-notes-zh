## 日志系统回顾 📝

上一节我们介绍了Linux的日志系统。系统有两个主要的日志服务：`systemd-journald`和`rsyslog`。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_13.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_15.png)

`systemd-journald`服务负责收集系统启动后的所有日志，这些日志以二进制格式存储，不进行分类。

`rsyslog`服务则将`systemd-journald`收集的日志转换为文本格式，进行分类处理，并存放至指定的文件中。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_17.png)

`rsyslog`的配置文件是`/etc/rsyslog.conf`。该配置文件主要包含三个模块：
*   **`module`模块**：配置`rsyslog`加载的模块，例如开启网络日志收集功能。
*   **`global`模块**：配置全局设置。
*   **`rules`模块**：配置日志收集规则，这是我们关注的重点。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_19.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_20.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_22.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_24.png)

日志规则主要关注**设施（facility）**和**级别（priority）**。设施代表日志的来源（如`auth`, `cron`, `kern`等），级别代表日志的严重程度（从`debug`到`emerg`共8个级别）。通常，我们需要处理级别在`warning`及以上的日志。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_26.png)

规则的基本格式为：`设施.级别 存放位置`。例如，`authpriv.* /var/log/secure`表示将所有授权相关的日志（任何级别）记录到`/var/log/secure`文件中。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_28.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_30.png)

此外，可以通过`imudp`和`imtcp`模块开启网络日志收集，将日志发送到指定的日志服务器。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_32.png)

`rsyslog`的日志文件默认保存在`/var/log`目录下。常用的日志查看工具是`journalctl`，它有一些实用选项：
*   `-b`：列出本次启动后的日志。
*   `-p`：按优先级过滤日志。
*   `--since` 和 `--until`：按时间范围筛选日志。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_34.png)

## 使用 `find` 命令查找文件 🔍

上一节我们回顾了日志系统，本节中我们来看看如何在文件系统中高效地查找文件。`find`命令功能强大，但条件丰富，查找速度相对较慢，因为它进行实时查找。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_36.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_38.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_40.png)

以下是`find`命令的一些常见查找条件：

![](img/b64bf86c92e81d27ed1ed16d2f35700e_42.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_44.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_46.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_48.png)

*   **按名称查找**：使用`-name`进行精确查找，使用`-iname`进行不区分大小写的查找。
*   **按类型查找**：使用`-type`指定文件类型（如`f`代表普通文件，`d`代表目录）。
*   **按大小查找**：使用`-size`指定文件大小。例如，`-size +10M`查找大于10MB的文件，`-size -1k`查找小于1KB的文件。
*   **按时间查找**：使用`-atime`（访问时间）、`-mtime`（修改时间）、`-ctime`（状态改变时间）。`-7`表示7天以内，`+7`表示7天以前。
*   **按权限查找**：使用`-perm`指定权限。`-perm 644`精确匹配权限为644的文件。`-perm /222`表示任意用户（u,g,o）有写权限即匹配。`-perm -222`表示必须所有用户（u,g,o）都有写权限才匹配。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_50.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_52.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_53.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_54.png)

`find`命令的强大之处在于可以对找到的文件执行操作。以下是常用的处理动作：

![](img/b64bf86c92e81d27ed1ed16d2f35700e_56.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_58.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_60.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_62.png)

*   `-print`：默认动作，将结果打印到标准输出。
*   `-ls`：以`ls -dils`的格式列出当前文件。
*   `-delete`：删除查找到的文件。
*   `-ok command {} \;`：对每个找到的文件执行指定的`command`命令，但在执行前会进行交互式确认。
*   `-exec command {} \;`：对每个找到的文件执行指定的`command`命令，**不会**进行确认。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_64.png)

其中，`-exec`的用法需要特别注意。其语法结构是固定的：`find [路径] [条件] -exec 命令 {} \;`。这里的`{}`代表`find`命令找到的**每个文件本身**（而非文件名文本），`\;`用于结束命令。这相当于一个高效的“管道”，将找到的文件对象传递给后续命令处理。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_66.png)

例如，以下命令查找`/home/student`目录下所有属于`student`用户的文件，并将它们复制到`/tmp/backup`目录：
```bash
find /home/student -user student -exec cp {} /tmp/backup \;
```

![](img/b64bf86c92e81d27ed1ed16d2f35700e_68.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_70.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_72.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_74.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_76.png)

## 文件压缩工具 📦

![](img/b64bf86c92e81d27ed1ed16d2f35700e_78.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_80.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_81.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_83.png)

学会了查找文件，我们常常需要对找到的文件进行归档或压缩以节省空间或便于传输。Linux下有多种压缩工具，我们先来看三种只压缩单个文件的工具：`gzip`、`bzip2`和`xz`。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_85.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_87.png)

它们的通用选项相似：
*   `-k`：保留原始文件（不删除）。
*   `-d`：解压缩。
*   `-c`：将压缩结果输出到标准输出，保留原文件不变。
*   `-数字`：指定压缩级别（1-9），数字越大，压缩率越高，耗时越长。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_89.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_91.png)

**压缩率对比**：通常，`xz`的压缩率最高，生成的压缩文件最小；其次是`bzip2`；`gzip`的压缩率相对较低。但压缩率高并不意味着总是最好，因为高压缩率会消耗更多的CPU时间和内存，解压时也可能更慢。需要根据实际情况在压缩率和性能之间权衡。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_93.png)

## 打包与归档工具 🗃️

![](img/b64bf86c92e81d27ed1ed16d2f35700e_95.png)

然而，`gzip`、`bzip2`、`xz`这些工具通常只能压缩单个文件，无法直接处理目录。为了打包多个文件或整个目录，我们需要用到`tar`命令。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_97.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_99.png)

`tar`命令本身是一个**打包**工具，并不压缩内容，但它可以方便地配合压缩工具使用。它的主要选项有：
*   `-c`：创建新的归档文件。
*   `-x`：从归档文件中提取文件。
*   `-t`：列出归档文件的内容。
*   `-v`：显示详细的处理过程。
*   `-f`：指定归档文件名（**这个选项后面必须紧跟文件名**）。
*   `-C`：指定解压到的目标目录。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_101.png)

`tar`命令最常用的功能是**打包并压缩**。它通过以下选项调用对应的压缩算法：
*   `-z` 或 `--gzip`：使用`gzip`进行压缩/解压，生成`.tar.gz`或`.tgz`文件。
*   `-j` 或 `--bzip2`：使用`bzip2`进行压缩/解压，生成`.tar.bz2`文件。
*   `-J` 或 `--xz`：使用`xz`进行压缩/解压，生成`.tar.xz`文件。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_103.png)

例如，将`/etc`目录打包并用`gzip`压缩：
```bash
tar -czvf etc_backup.tar.gz /etc
```
解压这个包到`/tmp`目录：
```bash
tar -xzvf etc_backup.tar.gz -C /tmp
```

![](img/b64bf86c92e81d27ed1ed16d2f35700e_105.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_106.png)

**注意**：使用`tar`打包时，如果使用绝对路径（如`/etc`），解压时可能会覆盖系统中的原有路径。通常建议在打包时先进入目标目录的父目录，或使用`-P`选项保留绝对路径，但需格外小心。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_108.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_110.png)

`tar`的另一个优点是支持向已存在的**未压缩**的tar包中追加文件（使用`-r`选项），但一旦tar包被压缩（如`.tar.gz`），就无法再直接追加内容。

![](img/b64bf86c92e81d27ed1ed16d2f35700e_112.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_114.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_116.png)

![](img/b64bf86c92e81d27ed1ed16d2f35700e_118.png)
