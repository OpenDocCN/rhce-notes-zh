## 管道符号 `|` 的使用

上一节我们介绍了基本的文件操作命令，本节中我们来看看如何将多个命令组合起来完成更复杂的任务，这就需要用到管道符号。

![](img/3e99b316070e121c89e20d965c5dfa7e_16.png)

管道符号用竖线 `|` 表示。它的作用是将一个命令的输出，作为另一个命令的输入，从而将多个命令串联起来，完成单个命令无法实现的功能。

![](img/3e99b316070e121c89e20d965c5dfa7e_18.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_20.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_21.png)

**公式：**
```
命令1 | 命令2 | 命令3 ...
```

![](img/3e99b316070e121c89e20d965c5dfa7e_23.png)

例如，`ls -l /etc` 命令会列出 `/etc` 目录下的所有内容，但内容太多，一屏显示不全。我们可以使用管道将输出传递给 `more` 命令进行分页显示：

![](img/3e99b316070e121c89e20d965c5dfa7e_25.png)

```
ls -l /etc | more
```

![](img/3e99b316070e121c89e20d965c5dfa7e_27.png)

这样，`ls -l /etc` 的输出就成为了 `more` 命令的输入，实现了翻屏查看的效果。

管道符号的强大之处在于可以连接多个命令。例如，我们想从 `ifconfig` 命令的输出中，仅提取出 `ens33` 网卡的IP地址 `172.25.254.250`，可以通过一系列命令组合实现：

![](img/3e99b316070e121c89e20d965c5dfa7e_29.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_31.png)

```
ifconfig | grep ens33 | grep -v inet6 | awk '{print $2}'
```

以下是这条命令的分解步骤：
1.  `ifconfig`： 显示所有网络接口信息。
2.  `grep ens33`： 从上一步的输出中，筛选出包含 `ens33` 的行。
3.  `grep -v inet6`： 从上一步的输出中，排除包含 `inet6` 的行（即IPv6地址）。
4.  `awk '{print $2}'`： 提取出上一步结果中的第二个字段，即我们需要的IP地址。

通过三个管道符号，我们结合了四个命令，最终精确地得到了所需信息。这体现了Linux命令行灵活而强大的特性。

## `tee` 命令：分流输出

![](img/3e99b316070e121c89e20d965c5dfa7e_39.png)

管道将数据从左流向右。`tee` 命令的作用像一个“三通”，它既能把数据继续传递给下一个命令，也能同时将数据保存到一个文件中。

![](img/3e99b316070e121c89e20d965c5dfa7e_41.png)

**图示逻辑：**
```
命令1 | tee 文件名 | 命令2
```

![](img/3e99b316070e121c89e20d965c5dfa7e_43.png)

例如，我们想查看 `/etc/passwd` 文件的内容，同时将其保存到文件 `abc.txt` 中，然后再筛选出包含 `root` 的行：

![](img/3e99b316070e121c89e20d965c5dfa7e_45.png)

```
cat /etc/passwd | tee abc.txt | grep root
```

![](img/3e99b316070e121c89e20d965c5dfa7e_47.png)

执行后，屏幕会显示包含 `root` 的行，同时文件 `abc.txt` 中会保存 `/etc/passwd` 的全部内容。

![](img/3e99b316070e121c89e20d965c5dfa7e_49.png)

`tee` 命令有一个常用选项 `-a`，代表追加（append）。如果不加 `-a`，`tee` 会覆盖目标文件；加上 `-a`，则会将新内容追加到文件末尾。

![](img/3e99b316070e121c89e20d965c5dfa7e_51.png)

例如，先将 `/etc/passwd` 内容保存到 `abc.txt`，再追加 `/etc/group` 的内容：
```
cat /etc/passwd | tee abc.txt | grep root
cat /etc/group | tee -a abc.txt | grep root
```
执行后，`abc.txt` 文件中将包含两个文件的内容。

![](img/3e99b316070e121c89e20d965c5dfa7e_53.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_55.png)

## 用户管理基础

![](img/3e99b316070e121c89e20d965c5dfa7e_63.png)

### 用户的概念与分类

在Linux系统中，每个使用者都必须通过一个账户（用户）登录系统进行操作。用户通过唯一的用户ID（UID）来区分身份，而非用户名。

![](img/3e99b316070e121c89e20d965c5dfa7e_65.png)

Linux用户主要分为三类：
1.  **管理员（root）**： UID 永远为 **0**。拥有系统的最高权限，可以执行任何操作。生产环境中应谨慎使用。
2.  **系统用户**： UID 范围通常为 **1-999**（RHEL 7/8）。这类用户由系统或应用程序创建，用于运行特定的系统服务（如 `mysql`、`apache`），**默认不允许登录系统**。
3.  **普通用户**： UID 从 **1000** 开始（RHEL 7/8）。由管理员创建，用于日常操作，权限受到限制。

![](img/3e99b316070e121c89e20d965c5dfa7e_67.png)

### 用户相关文件

1.  **`/etc/passwd`**： 保存用户的基本信息（**不包含密码**）。每一行代表一个用户，字段用冒号 `:` 分隔。

    我们需要记住这7个字段的含义：
    ```
    用户名:密码占位符:UID:GID:描述信息:家目录:登录Shell
    ```
    *   例如：`root:x:0:0:root:/root:/bin/bash`
    *   第二个字段 `x` 表示密码已移至 `/etc/shadow` 文件。
    *   最后一个字段如果是 `/sbin/nologin`，则表示该用户不允许登录系统。

2.  **`/etc/shadow`**： 保存用户的加密密码及密码策略。同样以冒号分隔字段，包含密码过期时间、失效时间等信息，权限严格，只有root可读。

### 用户管理命令

![](img/3e99b316070e121c89e20d965c5dfa7e_69.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_71.png)

以下是管理用户的核心命令：

*   **创建用户**： `useradd [选项] 用户名`
    *   常用选项：
        *   `-u UID`： 指定用户的UID。
        *   `-d 目录`： 指定用户的家目录。
        *   `-s Shell`： 指定用户的登录Shell，如 `/sbin/nologin` 可禁止登录。
        *   `-c “备注”`： 添加用户的描述信息。
        *   `-g 组名`： 指定用户的主要组（初始组）。
        *   `-G 组名1,组名2,...`： 指定用户的附加组。

*   **修改用户属性**： `usermod [选项] 用户名`
    *   选项与 `useradd` 类似，用于修改已存在用户的属性。
    *   特别选项：
        *   `-L`： 锁定用户，禁止其登录。
        *   `-U`： 解锁用户。
        *   `-aG 组名`： **将用户追加到某个附加组**（与 `-G` 单独使用时会覆盖原有附加组列表不同）。

![](img/3e99b316070e121c89e20d965c5dfa7e_73.png)

*   **删除用户**： `userdel -r 用户名`
    *   `-r` 选项会在删除用户的同时，**删除其家目录和邮件池**。如果不加 `-r`，则只删除用户，家目录会保留。

*   **查看用户ID信息**： `id [用户名]`
    *   不加用户名则显示当前用户的信息。输出包括UID、主要组GID和所属的附加组。

*   **设置/修改密码**： `passwd [用户名]`
    *   root用户可以为任何用户设置密码。普通用户只能修改自己的密码（不跟用户名）。

![](img/3e99b316070e121c89e20d965c5dfa7e_75.png)

**示例：**
1.  创建UID为666，备注为“Test User”的用户 `user100`：
    ```
    useradd -u 666 -c "Test User" user100
    ```
2.  将用户 `user100` 附加到组 `admins` 和 `developers` 中：
    ```
    usermod -aG admins,developers user100
    ```
3.  彻底删除用户 `user100`：
    ```
    userdel -r user100
    ```

## 群组管理基础

### 群组的概念与分类

![](img/3e99b316070e121c89e20d965c5dfa7e_83.png)

群组是用户的集合，用于方便地对一组用户分配相同的权限。每个组也有一个唯一的组ID（GID）。

组分为两类：
1.  **主要组（初始组/私有组）**： 每个用户有且只有一个主要组。创建用户时，系统会默认创建一个与用户同名的组作为其主要组。用户的GID（`/etc/passwd` 中第四个字段）即其主要组的GID。
2.  **附加组（补充组）**： 一个用户可以属于零个或多个附加组。用于给用户分配额外的权限。

![](img/3e99b316070e121c89e20d965c5dfa7e_85.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_87.png)

### 群组相关文件

*   **`/etc/group`**： 保存组的基本信息。每一行代表一个组。
    ```
    组名:组密码占位符:GID:组成员列表
    ```
    *   例如：`wheel:x:10:student`
    *   最后一个字段列出以该组为**附加组**的成员用户名，用逗号分隔。

*   **`/etc/gshadow`**： 保存组的加密密码（较少使用）。

![](img/3e99b316070e121c89e20d965c5dfa7e_89.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_91.png)

### 群组管理命令

![](img/3e99b316070e121c89e20d965c5dfa7e_93.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_95.png)

*   **创建组**： `groupadd [选项] 组名`
    *   常用选项 `-g GID` 用于指定GID。
*   **修改组属性**： `groupmod [选项] 组名`
    *   选项与 `groupadd` 类似。
*   **删除组**： `groupdel 组名`
    *   注意：不能删除某个用户的主要组（必须先修改该用户的主要组或删除该用户）。

**示例：**
1.  创建GID为2000的组 `admins`：
    ```
    groupadd -g 2000 admins
    ```
2.  删除组 `developers`：
    ```
    groupdel developers
    ```

## 密码策略管理 `chage`

![](img/3e99b316070e121c89e20d965c5dfa7e_105.png)

`passwd` 命令用于设置密码，而 `chage` 命令用于管理密码的时效策略。

![](img/3e99b316070e121c89e20d965c5dfa7e_107.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_109.png)

**常用选项：**
*   `-l`： 列出用户的密码策略信息。
*   `-m 天数`： 密码可更改的最小天数（0表示随时可改）。
*   `-M 天数`： 密码保持有效的最大天数（99999通常表示永不过期）。
*   `-W 天数`： 密码到期前，提前多少天开始警告用户。
*   `-I 天数`： 密码过期后，账户被锁定前的宽限天数。
*   `-E 日期`： 账户到期的绝对日期，格式为 YYYY-MM-DD。

![](img/3e99b316070e121c89e20d965c5dfa7e_111.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_113.png)

**示例：**
1.  设置用户 `user100` 的密码90天后过期，并在过期前7天开始警告：
    ```
    chage -M 90 -W 7 user100
    ```
2.  查询 `root` 用户的密码策略：
    ```
    chage -l root
    ```
3.  设置用户下次登录时必须更改密码：
    ```
    chage -d 0 user100
    ```

![](img/3e99b316070e121c89e20d965c5dfa7e_115.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_117.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_119.png)

**计算日期小技巧：**
使用 `date` 命令计算未来日期，用于 `-E` 选项：
```
date -d "+45 days" +%Y-%m-%d
```
这条命令会输出45天后的日期。

## 权限提升 `sudo`

![](img/3e99b316070e121c89e20d965c5dfa7e_129.png)

在Linux中，许多系统管理命令（如 `useradd`）只有root用户才能执行。`sudo` 命令允许被授权的普通用户，以root（或其他用户）的身份来运行命令，而无需知道root密码，提高了安全性和可审计性。

![](img/3e99b316070e121c89e20d965c5dfa7e_131.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_133.png)

### 配置 `sudo` 权限

管理员通过编辑 `/etc/sudoers` 文件来授权。**强烈建议使用 `visudo` 命令来编辑此文件**，因为它会在保存前检查语法，防止配置错误导致所有sudo权限失效。

![](img/3e99b316070e121c89e20d965c5dfa7e_135.png)

**配置语法示例：**
在 `/etc/sudoers` 文件中，找到类似以下的行进行修改或添加：
```
## Allows people in group wheel to run all commands
%wheel  ALL=(ALL)       ALL
```
*   `%wheel`： 授权给 `wheel` 组的所有成员。如果授权给单个用户，则直接写用户名，如 `user100`。
*   第一个 `ALL`： 允许在**所有主机**上执行。
*   `(ALL)`： 允许以**所有用户**的身份执行（通常指root）。
*   最后一个 `ALL`： 允许运行**所有命令**。

![](img/3e99b316070e121c89e20d965c5dfa7e_137.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_139.png)

**精细授权示例：**
只允许用户 `user100` 执行 `useradd` 和 `groupadd` 命令：
```
user100 ALL=(root) /usr/sbin/useradd, /usr/sbin/groupadd
```
这里必须使用命令的**绝对路径**（可通过 `which useradd` 查询）。

![](img/3e99b316070e121c89e20d965c5dfa7e_141.png)

### 使用 `sudo`

![](img/3e99b316070e121c89e20d965c5dfa7e_143.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_145.png)

被授权的用户在执行命令前加上 `sudo` 即可：
```
sudo useradd newuser
```
系统会提示输入**该用户自己的密码**进行验证（并非root密码）。验证成功后，命令将以root权限执行。

![](img/3e99b316070e121c89e20d965c5dfa7e_147.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_149.png)

第一次使用 `sudo` 需要输入密码，短时间内再次使用可能不需要。这是一种安全与便利的平衡。

**`sudo` 的重要性**： 在后续学习自动化工具（如Ansible）时，普通用户通过剧本管理多台服务器，当剧本中包含需要root权限的命令时，就必须配置和使用 `sudo` 来完成权限提升。

![](img/3e99b316070e121c89e20d965c5dfa7e_151.png)

![](img/3e99b316070e121c89e20d965c5dfa7e_153.png)

## 总结

本节课中我们一起学习了Linux用户和群组管理的核心内容：

1.  **管道 `|`**： 掌握了连接多个命令以实现复杂功能的方法。
2.  **`tee` 命令**： 学会了在管道中同时输出到屏幕和文件。
3.  **用户与群组概念**： 理解了UID/GID、用户/组的分类（管理员、系统用户、普通用户；主要组、附加组）及其相关配置文件（`/etc/passwd`, `/etc/group`, `/etc/shadow`）。
4.  **管理命令**： 熟练运用 `useradd`/`usermod`/`userdel`、`groupadd`/`groupmod`/`groupdel`、`id`、`passwd` 来创建、修改、删除和查询用户与组。
5.  **密码策略**： 使用 `chage` 命令管理用户密码的过期时间等策略。
6.  **权限提升**： 理解了 `sudo` 机制的重要性，并学会了如何使用 `visudo` 配置授权，以及如何使用 `sudo` 命令临时获得管理员权限执行特定命令。
