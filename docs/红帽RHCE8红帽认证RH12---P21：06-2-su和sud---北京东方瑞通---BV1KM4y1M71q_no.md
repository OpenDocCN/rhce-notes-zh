## 理解su命令 🔄

![](img/78b834a100b10cdc70769cd2fecf521f_11.png)

![](img/78b834a100b10cdc70769cd2fecf521f_13.png)

![](img/78b834a100b10cdc70769cd2fecf521f_15.png)

上一节我们提到了管理任务需要超级用户权限。本节中，我们来看看如何使用 `su` 命令切换用户身份。

![](img/78b834a100b10cdc70769cd2fecf521f_17.png)

`su` 命令用于切换当前用户身份。其基本语法可以通过 `su --help` 查看。我们将通过几个常见例子来理解其用法。

![](img/78b834a100b10cdc70769cd2fecf521f_19.png)

![](img/78b834a100b10cdc70769cd2fecf521f_20.png)

![](img/78b834a100b10cdc70769cd2fecf521f_22.png)

以下是 `su` 命令的两种主要切换方式及其区别：

*   **`su [username]`**：以非登录式Shell切换用户。此方式会**继承**当前Shell的大部分环境变量。
*   **`su - [username]`** 或 **`su -l [username]`**：以登录式Shell切换用户。此方式会**重新加载**目标用户的Shell环境配置，包括其专属的环境变量。

![](img/78b834a100b10cdc70769cd2fecf521f_24.png)

![](img/78b834a100b10cdc70769cd2fecf521f_26.png)

环境变量是否传递给子Shell，取决于它是否被导出（`export`）。登录式Shell会读取并执行一系列环境配置文件，而非登录式Shell则不会。

![](img/78b834a100b10cdc70769cd2fecf521f_28.png)

![](img/78b834a100b10cdc70769cd2fecf521f_30.png)

当 `su` 命令后不指定用户名时，默认切换到 `root` 用户。例如，`su` 和 `su -` 都表示切换到 `root`，但前者继承环境，后者使用 `root` 用户的全新环境。

## su命令的进阶用法 ⚙️

![](img/78b834a100b10cdc70769cd2fecf521f_32.png)

理解了基本的用户切换后，我们来看看 `su` 命令的一些高级选项，它们能提供更灵活的控制。

`su` 命令的 `-c` 选项允许我们以指定用户的身份执行单个命令，而无需完全切换到该用户的Shell。命令执行完毕后，控制权会立刻返回给原用户。

![](img/78b834a100b10cdc70769cd2fecf521f_34.png)

![](img/78b834a100b10cdc70769cd2fecf521f_36.png)

以下是使用 `-c` 选项的示例：
```bash
# 以root身份执行 `id` 命令
su - root -c ‘id‘
```
**注意**：如果执行的命令中包含空格等特殊字符，务必使用单引号 `‘ ‘` 将整个命令括起来，否则Shell会错误解析。

![](img/78b834a100b10cdc70769cd2fecf521f_38.png)

此外，`su` 命令还支持 `-s` 选项来指定执行命令的Shell程序，以及 `-g` 选项来指定主要组或补充组，从而以不同的权限上下文执行命令。

![](img/78b834a100b10cdc70769cd2fecf521f_40.png)

## 引入sudo命令 🛡️

![](img/78b834a100b10cdc70769cd2fecf521f_42.png)

![](img/78b834a100b10cdc70769cd2fecf521f_44.png)

虽然 `su` 可以切换用户，但它要求我们知道目标用户的密码。这在多管理员场景下会带来审计和职责不清的问题。为了解决这些问题，我们引入了功能更强大的 `sudo` 命令。

![](img/78b834a100b10cdc70769cd2fecf521f_46.png)

`sudo` 允许授权用户以其他用户（通常是 `root`）的身份执行命令，但它具有以下关键优势：

![](img/78b834a100b10cdc70769cd2fecf521f_48.png)

1.  **身份验证**：用户需要输入**自己的**密码，而非目标用户的密码，这增强了安全性并明确了操作责任人。
2.  **操作审计**：所有通过 `sudo` 执行的命令都会被详细记录在系统日志（如 `/var/log/secure`）中，便于追踪和审计。
3.  **精细授权**：管理员可以精确配置每个用户能够以何种身份执行哪些特定命令，实现最小权限原则。

![](img/78b834a100b10cdc70769cd2fecf521f_50.png)

## 配置sudo权限 📝

`sudo` 的强大功能依赖于其配置文件。现在，我们来看看如何通过配置文件来定义授权规则。

`sudo` 的主配置文件是 `/etc/sudoers`。**强烈建议使用 `visudo` 命令编辑此文件**，因为它能进行语法检查，防止配置错误导致 `sudo` 不可用。

![](img/78b834a100b10cdc70769cd2fecf521f_52.png)

该文件的基本规则语法如下：
```
用户    主机名=(可切换的身份)    可执行的命令列表
```
以下是配置示例及其含义：

![](img/78b834a100b10cdc70769cd2fecf521f_54.png)

*   **`root    ALL=(ALL)       ALL`**
    *   `root` 用户可以在**所有**主机上，切换为**任何**用户，执行**所有**命令。
*   **`%wheel  ALL=(ALL)       ALL`**
    *   `wheel` 组内的**所有成员**，可以在所有主机上，切换为任何用户，执行所有命令。
*   **`%wheel  ALL=(ALL)       NOPASSWD: ALL`**
    *   `wheel` 组成员执行 `sudo` 时**无需输入密码**，即可执行所有命令。
*   **`user1   ALL=(ALL)       /usr/bin/mount, /usr/bin/umount`**
    *   `user1` 用户可以在所有主机上，以任何用户身份，但**仅能执行** `mount` 和 `umount` 命令。

除了主配置文件，`/etc/sudoers.d/` 目录下的文件也会被读取。这便于进行模块化的权限管理，例如为不同应用程序创建独立的授权文件。

![](img/78b834a100b10cdc70769cd2fecf521f_56.png)

![](img/78b834a100b10cdc70769cd2fecf521f_58.png)

![](img/78b834a100b10cdc70769cd2fecf521f_60.png)

本节课中我们一起学习了 `su` 和 `sudo` 命令。`su` 用于直接切换用户身份，而 `sudo` 提供了更安全、可审计、可精细控制的特权执行机制。理解并正确配置 `sudo` 是Linux系统管理中的重要技能。在接下来的课程中，我们将深入探讨 `sudo` 配置的更复杂写法和实践案例。
