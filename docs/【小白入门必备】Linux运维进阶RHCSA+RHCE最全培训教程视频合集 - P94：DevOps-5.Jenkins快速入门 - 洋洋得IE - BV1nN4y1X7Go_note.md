## 配置SSH免密登录与脚本发布

![](img/6c5311a3353ff39457d417d487835a7a_15.png)

![](img/6c5311a3353ff39457d417d487835a7a_17.png)

![](img/6c5311a3353ff39457d417d487835a7a_19.png)

![](img/6c5311a3353ff39457d417d487835a7a_20.png)

上一节我们介绍了Jenkins的基本概念，本节中我们来看看如何通过脚本实现自动化项目发布。首先，需要在Jenkins服务器与目标Web服务器之间配置SSH免密登录，以确保脚本可以无交互地执行。

![](img/6c5311a3353ff39457d417d487835a7a_22.png)

![](img/6c5311a3353ff39457d417d487835a7a_24.png)

![](img/6c5311a3353ff39457d417d487835a7a_26.png)

![](img/6c5311a3353ff39457d417d487835a7a_27.png)

![](img/6c5311a3353ff39457d417d487835a7a_28.png)

以下是配置免密登录的核心步骤：
1.  在Jenkins服务器生成SSH密钥对：`ssh-keygen`
2.  将公钥复制到目标Web服务器：`ssh-copy-id root@web_server_ip`
3.  在首次连接时输入`yes`并验证目标服务器的root密码。

![](img/6c5311a3353ff39457d417d487835a7a_30.png)

![](img/6c5311a3353ff39457d417d487835a7a_32.png)

![](img/6c5311a3353ff39457d417d487835a7a_34.png)

配置成功后，执行部署脚本将不再需要输入密码。我们的发布脚本主要功能是将项目文件从Jenkins服务器同步到多台Web服务器。

![](img/6c5311a3353ff39457d417d487835a7a_36.png)

![](img/6c5311a3353ff39457d417d487835a7a_38.png)

![](img/6c5311a3353ff39457d417d487835a7a_40.png)

```bash
#!/bin/bash
# 这是一个简化的项目发布脚本示例
TARGET_SERVERS=("192.168.0.33" "192.168.0.34")
PROJECT_DIR="/var/www/project_latest"

![](img/6c5311a3353ff39457d417d487835a7a_42.png)

for server in ${TARGET_SERVERS[@]}; do
    scp -r $PROJECT_DIR root@$server:/usr/share/nginx/html/
done
```

![](img/6c5311a3353ff39457d417d487835a7a_44.png)

![](img/6c5311a3353ff39457d417d487835a7a_46.png)

脚本执行后，项目文件会被分发到所有Web服务器的指定目录。在Web服务器上，`/usr/share/nginx/html/`目录通常是一个软链接，指向实际的项目目录（如`/usr/share/nginx/html/web/`）。只要该目录下有内容，Web服务即可访问。

![](img/6c5311a3353ff39457d417d487835a7a_48.png)

## 在Jenkins中集成自动化构建

![](img/6c5311a3353ff39457d417d487835a7a_75.png)

虽然手动执行脚本可以完成发布，但这并不智能。更优的做法是将此脚本集成到Jenkins的构建流程中，实现一键发布。

![](img/6c5311a3353ff39457d417d487835a7a_77.png)

![](img/6c5311a3353ff39457d417d487835a7a_79.png)

![](img/6c5311a3353ff39457d417d487835a7a_81.png)

以下是集成步骤：
1.  在Jenkins任务配置页面，找到“构建”环节。
2.  增加一个构建步骤，选择“Execute shell”。
3.  在命令框中，填写执行脚本的绝对路径，例如：`bash /opt/scripts/deploy.sh`

![](img/6c5311a3353ff39457d417d487835a7a_83.png)

![](img/6c5311a3353ff39457d417d487835a7a_85.png)

![](img/6c5311a3353ff39457d417d487835a7a_87.png)

保存配置后，每次需要发布时，只需在Jenkins任务界面点击“Build Now”。Jenkins会按照配置的顺序执行：首先从代码仓库拉取最新代码，然后执行我们集成的Shell脚本，自动将项目发布到所有Web服务器。

![](img/6c5311a3353ff39457d417d487835a7a_89.png)

![](img/6c5311a3353ff39457d417d487835a7a_91.png)

## 项目回滚操作

![](img/6c5311a3353ff39457d417d487835a7a_93.png)

![](img/6c5311a3353ff39457d417d487835a7a_95.png)

![](img/6c5311a3353ff39457d417d487835a7a_97.png)

发布新版本后，如果出现问题，需要快速回滚到上一个可用版本。由于我们的发布脚本不会删除历史版本目录，因此回滚操作相对简单。

![](img/6c5311a3353ff39457d417d487835a7a_99.png)

![](img/6c5311a3353ff39457d417d487835a7a_101.png)

回滚的本质是将Web服务器的软链接指向旧版本的项目目录。假设旧版本目录为`/usr/share/nginx/html/project_old/`。

![](img/6c5311a3353ff39457d417d487835a7a_103.png)

以下是手动回滚命令：
```bash
# 进入Web服务器操作
cd /usr/share/nginx/html
# 删除当前软链接
rm -f html
# 创建指向旧版本的软链接
ln -s /usr/share/nginx/html/project_old/ html
```
执行后，刷新浏览器，页面就会恢复到旧版本。同理，也可以再次修改软链接以切回新版本。对于需要编译的Java等项目，回滚流程会更复杂，但基本思路一致：替换运行的文件或目录。

## GitLab配置与管理

之前我们介绍了使用HTTP方式连接GitLab，实际上更安全便捷的方式是配置SSH密钥认证，实现免密推送代码。

配置方法如下：
1.  在客户端生成SSH密钥：`ssh-keygen`
2.  将公钥（`~/.ssh/id_rsa.pub`的内容）添加到GitLab账户的“SSH Keys”设置中。
3.  此后，该客户端在向GitLab推送或拉取代码时，就不再需要输入用户名和密码。

此外，作为管理员，可能需要对GitLab实例进行一些定制化管理：
*   **定制外观**：在“Admin Area -> Settings -> Appearance”中，可以上传自定义Logo、设置页眉页脚文本和系统标题。
*   **注册限制**：在“Admin Area -> Settings -> General -> Sign-up restrictions”中，可以禁用公开注册功能。禁用后，只能由管理员手动创建用户账号，这更符合企业内部管理的需求。
*   **偏好设置**：可以调整界面主题、代码高亮样式等，提升用户体验。
