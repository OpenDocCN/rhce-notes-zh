## 📁 `ls`命令详解

![](img/9b61e36a7e8c4dda1be3310885830b49_7.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_9.png)

`ls`命令用于列出目录内容，即查看目录和文件。通过添加不同的选项，我们可以获取更详细或特定格式的信息。

![](img/9b61e36a7e8c4dda1be3310885830b49_11.png)

### 使用 `ls -l` 查看详细信息

![](img/9b61e36a7e8c4dda1be3310885830b49_13.png)

`ls -l` 命令可以显示文件和目录的详细信息。以下是其输出中各字段的含义：

![](img/9b61e36a7e8c4dda1be3310885830b49_15.png)

以下是`ls -l`命令输出中每个字段的详细解释：

*   **文件类型与权限（第1字段）**：第一个字符表示文件类型，后续九个字符表示权限。
    *   `-`：普通文件
    *   `d`：目录
    *   `l`：符号链接（快捷方式）
    *   `b`：块设备文件（如磁盘）
    *   `c`：字符设备文件（如终端、键盘）
*   **链接数（第2字段）**：
    *   对于**文件**，此数字表示该文件的硬链接数量。
    *   对于**目录**，此数字至少为2，表示该目录下子目录的数量（包含隐藏的`.`和`..`目录）。
*   **拥有者（第3字段）**：文件或目录的所有者用户。
*   **所属组（第4字段）**：文件或目录所属的用户组。
*   **大小（第5字段）**：文件的大小，单位通常是字节。
*   **最后修改时间（第6字段）**：文件或目录内容最后一次被修改的时间。
*   **名称（第7字段）**：文件或目录的名称。

### 其他常用 `ls` 选项

![](img/9b61e36a7e8c4dda1be3310885830b49_23.png)

上一节我们介绍了`ls -l`的详细信息，本节中我们来看看其他几个常用选项。

![](img/9b61e36a7e8c4dda1be3310885830b49_25.png)

*   `ls -a`：显示所有文件，包括以`.`开头的隐藏文件。
*   `ls -R`：递归显示目录及其所有子目录下的内容。
*   `ls -d`：仅显示目录本身的信息，而不是其内容。这在查看目录权限时非常有用。

例如，`ls -ld /etc` 将只显示`/etc`目录本身的详细信息，而`ls -l /etc`则会列出`/etc`目录下的所有文件和目录。

## 📋 `cp`命令：复制文件与目录

![](img/9b61e36a7e8c4dda1be3310885830b49_33.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_35.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_37.png)

`cp`命令用于复制文件或目录。其基本语法是 `cp [选项] 源文件... 目标路径`。

### 复制文件时，目标的三种情况

复制文件时，根据“目标”的不同，命令行为有所区别。以下是三种主要情况：

![](img/9b61e36a7e8c4dda1be3310885830b49_39.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_41.png)

1.  **目标是一个目录**：文件将被复制到该目录下，并保留原文件名。
    *   命令示例：`cp file.txt /tmp/`
2.  **目标是一个已存在的文件**：系统会询问是否覆盖（若使用`-i`选项）。确认后，源文件内容将覆盖目标文件。
    *   命令示例：`cp file.txt existing_file.txt`
3.  **目标是一个不存在的文件**：文件被复制并重命名为指定的新文件名。
    *   命令示例：`cp file.txt newfile.txt`

![](img/9b61e36a7e8c4dda1be3310885830b49_43.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_45.png)

### 复制目录

与复制文件不同，复制目录必须使用 `-r`（或`-R`，递归）选项。

命令示例：`cp -r /etc /tmp/`

复制目录时，目标的三种情况与复制文件类似，但有一个关键区别：**无法用一个目录去覆盖一个已存在的普通文件**。

![](img/9b61e36a7e8c4dda1be3310885830b49_47.png)

## 🚚 `mv`命令：移动与重命名

`mv`命令用于移动或重命名文件/目录。其语法与`cp`类似：`mv [选项] 源文件... 目标路径`。

![](img/9b61e36a7e8c4dda1be3310885830b49_51.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_53.png)

### 移动文件/目录

![](img/9b61e36a7e8c4dda1be3310885830b49_55.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_56.png)

移动操作会将源文件/目录从原位置转移到目标位置，原位置的文件会消失。

![](img/9b61e36a7e8c4dda1be3310885830b49_58.png)

命令示例：`mv file.txt /home/user/`

![](img/9b61e36a7e8c4dda1be3310885830b49_60.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_61.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_62.png)

### 重命名文件/目录

在同一个目录下使用`mv`命令，即为重命名操作。

命令示例：`mv oldname.txt newname.txt`

**注意**：重命名时务必指定正确的路径，否则可能变成移动操作。例如，在`/root`目录下，应使用 `mv a a.bak` 来重命名，而不是 `mv a.bak`（这会将文件移动到当前目录并改名）。

### 移动时目标的三种情况

`mv`命令在处理目标时，逻辑与`cp`命令的三种情况完全一致。

## 🔗 复制链接文件的注意事项

![](img/9b61e36a7e8c4dda1be3310885830b49_68.png)

![](img/9b61e36a7e8c4dda1be3310885830b49_70.png)

在复制符号链接文件时，默认行为是复制链接所指向的**原始文件**，而不是链接本身。

如果想复制链接文件本身（保持其链接属性），需要使用 `-a` 或 `-d` 选项。
*   `cp -a`：归档模式，保留所有文件属性，并递归复制目录。
*   `cp -d`：保持链接关系，复制链接文件本身。

命令示例：`cp -a /etc/systemd/system/default.target /tmp/`

---

## 📝 总结

本节课中我们一起学习了Linux文件操作的三个核心命令：
1.  **`ls`**：用于查看目录内容，常用选项`-l`（详情）、`-a`（显示隐藏文件）、`-R`（递归）、`-d`（查看目录本身）。
2.  **`cp`**：用于复制。复制目录需加`-r`选项。理解目标为目录、已存在文件、不存在文件时的三种行为是关键。
3.  **`mv`**：用于移动和重命名。其行为逻辑与`cp`类似，但会删除源文件。

![](img/9b61e36a7e8c4dda1be3310885830b49_72.png)
