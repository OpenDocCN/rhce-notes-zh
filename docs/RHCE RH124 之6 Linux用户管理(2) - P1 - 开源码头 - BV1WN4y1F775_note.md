## 用户与组信息存储位置

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_3.png)

上一节我们介绍了用户和组的基本概念，本节中我们来看看这些信息在系统中是如何存储的。

用户账户信息存储在 `/etc/passwd` 文件中。这个文件包含了所有用户的基本信息，例如用户名、用户ID（UID）、主组ID（GID）和家目录路径等。

用户密码信息则存储在 `/etc/shadow` 文件中。出于安全考虑，密码以不可逆的散列加密方式存储。该文件还记录了密码的有效期等信息。

组信息存储在 `/etc/group` 文件中，其中包含了组名、组ID（GID）以及组成员列表。

虽然理论上可以直接编辑这些文件来管理用户和组，但这样做既繁琐又容易出错。因此，系统提供了专门的命令来安全、高效地完成这些任务。

## 使用命令管理用户和组

接下来，我们将学习如何使用命令行工具来管理用户和组。

### 创建用户

创建用户最直接的方法是使用 `useradd` 命令。该命令会自动在 `/etc/passwd`、`/etc/shadow` 和 `/etc/group` 文件中创建相应条目，并为用户创建家目录。

**命令格式**：
```bash
useradd [选项] 用户名
```

例如，创建一个名为“张三”的用户：
```bash
useradd 张三
```

创建后，可以使用 `id` 命令查看用户的详细信息：
```bash
id 张三
```
输出会显示用户的UID、主组GID以及所属的所有组。

### 创建组

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_5.png)

使用 `groupadd` 命令可以创建一个新的组。

**命令格式**：
```bash
groupadd 组名
```

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_7.png)

例如，创建一个名为“销售组”的组：
```bash
groupadd 销售组
```

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_9.png)

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_10.png)

### 修改用户属性

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_11.png)

要将用户添加到附属组（即除了主组之外的其他组），可以使用 `usermod` 命令。

**命令格式**：
```bash
usermod -aG 组名 用户名
```
其中，`-aG` 选项表示“追加到附属组”（Append to supplementary Groups）。

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_13.png)

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_14.png)

例如，将用户“张三”添加到“销售组”：
```bash
usermod -aG 销售组 张三
```
再次执行 `id 张三`，可以看到“销售组”已出现在其所属组列表中。

## `useradd` 命令常用选项详解

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_18.png)

`useradd` 命令提供了丰富的选项来定制新用户的属性。以下是部分常用选项：

以下是 `useradd` 命令的部分常用选项列表：

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_20.png)

*   **`-c “注释”`**：为用户账户添加一段描述性注释。
*   **`-d /path/to/home`**：指定用户的家目录路径，而非使用默认的 `/home/用户名`。
*   **`-e YYYY-MM-DD`**：设置账户的过期日期。
*   **`-g 组名或GID`**：指定用户的主组。
*   **`-G 组列表`**：指定用户的附属组列表，多个组用逗号分隔。
*   **`-s /path/to/shell`**：指定用户的登录Shell。例如，设置为 `/sbin/nologin` 可禁止该用户交互式登录。
*   **`-u UID`**：指定用户的UID。必须确保不与现有UID冲突。

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_22.png)

![](img/09d163b2b78c70c1b4c2d40ae96e85b0_24.png)

通常情况下，使用默认值创建用户即可满足大部分需求。如需查看所有选项的详细说明，可以运行 `man useradd` 命令。
