# Linux基础教程：23：IO输入输出与重定向详解 🖥️

![](img/8f06462f190db6f8705ceb22759f93fc_1.png)

在本节课中，我们将要学习Linux系统中的输入（Input）和输出（Output），以及如何通过重定向来改变输入输出的默认方向。这是进行高级操作和自动化脚本编写的基础。

![](img/8f06462f190db6f8705ceb22759f93fc_3.png)

## 什么是IO？ 🤔

![](img/8f06462f190db6f8705ceb22759f93fc_5.png)

IO是指输入和输出。在计算机中，输入是指将数据送入系统，输出是指系统将处理结果送出。例如，在终端中，键盘是输入设备，屏幕是输出设备。

![](img/8f06462f190db6f8705ceb22759f93fc_7.png)

## 标准输入、输出与错误输出 📊

![](img/8f06462f190db6f8705ceb22759f93fc_9.png)

Linux系统为每个进程定义了三种标准的数据流：

![](img/8f06462f190db6f8705ceb22759f93fc_11.png)

![](img/8f06462f190db6f8705ceb22759f93fc_13.png)

*   **标准输入 (stdin)**：文件描述符为 `0`，通常来自键盘。
*   **标准输出 (stdout)**：文件描述符为 `1`，通常输出到屏幕。
*   **标准错误输出 (stderr)**：文件描述符为 `2`，通常也输出到屏幕，用于显示错误信息。

上一节我们介绍了IO的基本概念，本节中我们来看看如何改变这些标准流的去向，即“重定向”。

## 输出重定向 📤

默认情况下，命令的执行结果会输出到终端屏幕。输出重定向可以将这些结果保存到文件中。

### 覆盖重定向

使用单个大于号 `>` 可以将命令的标准输出重定向到文件。如果文件不存在，则创建它；如果文件已存在，则会**覆盖**原有内容。

![](img/8f06462f190db6f8705ceb22759f93fc_15.png)

![](img/8f06462f190db6f8705ceb22759f93fc_17.png)

```bash
ls > ls_output.txt
```
执行后，`ls` 命令的结果不会显示在屏幕上，而是被写入 `ls_output.txt` 文件。

**警告**：此操作会直接覆盖目标文件，使用时需格外小心。例如，清空历史记录文件：
```bash
echo "" > ~/.bash_history
```

### 追加重定向

使用两个大于号 `>>` 可以将命令的输出**追加**到文件末尾，而不会覆盖原有内容。

```bash
ls >> ls_output.txt
```

![](img/8f06462f190db6f8705ceb22759f93fc_21.png)

## 错误输出重定向 ❌

有时命令会产生错误信息（标准错误输出）。我们可以单独重定向错误输出。

使用 `2>` 可以将错误输出重定向到文件。同样，`2>>` 用于追加。

![](img/8f06462f190db6f8705ceb22759f93fc_23.png)

```bash
# 以普通用户查找/etc/passwd，权限不足的部分会报错
find /etc -name passwd 2> error.log
```
上述命令将正确的查找结果输出到屏幕，而将权限错误信息写入 `error.log` 文件。

![](img/8f06462f190db6f8705ceb22759f93fc_25.png)

## 合并与丢弃输出 🔄

![](img/8f06462f190db6f8705ceb22759f93fc_27.png)

以下是处理多种输出的常用方法：

*   **将正确和错误输出重定向到不同文件**：
    ```bash
    find /etc -name passwd > output.log 2> error.log
    ```

*   **将正确和错误输出合并重定向到同一文件**：
    ```bash
    find /etc -name passwd &> all_output.log
    # 或等价的写法
    find /etc -name passwd > all_output.log 2>&1
    ```
    `2>&1` 表示“将标准错误输出重定向到标准输出所指向的地方”。

![](img/8f06462f190db6f8705ceb22759f93fc_29.png)

*   **丢弃所有输出（输出到“黑洞”）**：
    Linux有一个特殊的设备文件 `/dev/null`，它可以接收任何数据并立即丢弃，不占空间。常用于屏蔽脚本中不必要的输出。
    ```bash
    find /etc -name passwd &> /dev/null
    ```
    命令执行后，屏幕将没有任何输出。

## 输入重定向 📥

![](img/8f06462f190db6f8705ceb22759f93fc_37.png)

输入重定向使用小于号 `<`，它可以将文件的内容作为命令的输入。

例如，`wc -l` 命令可以统计行数，通常需要从键盘输入内容。使用输入重定向可以直接统计文件行数：
```bash
wc -l < /etc/passwd
```

## 管道符 | 🔗

![](img/8f06462f190db6f8705ceb22759f93fc_39.png)

![](img/8f06462f190db6f8705ceb22759f93fc_41.png)

管道符 `|` 用于连接两个命令，将前一个命令的**标准输出**作为后一个命令的**标准输入**。这是Linux中组合简单命令完成复杂任务的强大工具。

```bash
# 列出/etc目录下内容，并通过grep过滤出包含“pass”的行
ls -l /etc | grep pass
```

管道和重定向常常结合使用，构建出高效的数据处理流程。
