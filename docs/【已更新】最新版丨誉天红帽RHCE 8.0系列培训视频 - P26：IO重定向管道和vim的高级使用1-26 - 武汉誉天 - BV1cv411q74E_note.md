## 文件查找与输出重定向

上一节我们介绍了基本的文件操作，本节中我们来看看如何使用`find`命令查找文件，并理解其输出。

![](img/60b24f7b6385510c2455f502964c1bc9_5.png)

`find`命令用于在指定目录下查找文件。例如，查找`/etc`目录下名为`password`的文件：
```bash
find /etc -name password
```
这条命令会以文件名作为查找条件。`-name`是一个选项，虽然它是一个单词，但这里使用了短选项格式`-`，这是因为`find`是一个古老的命令，没有严格遵循后来的长选项（`--`）规范。

![](img/60b24f7b6385510c2455f502964c1bc9_7.png)

![](img/60b24f7b6385510c2455f502964c1bc9_9.png)

当以root用户执行时，会找到两个文件。但以普通用户执行时，会因为权限不足而报错。

## 标准输出与错误输出重定向

![](img/60b24f7b6385510c2455f502964c1bc9_11.png)

现在，我们尝试将查找结果重定向到一个文件中。
```bash
find /etc -name password > find.out
```
执行后，终端仍然有输出，而`find.out`文件中只写入了两条正确的查找结果，错误信息并未写入。

![](img/60b24f7b6385510c2455f502964c1bc9_13.png)

![](img/60b24f7b6385510c2455f502964c1bc9_15.png)

![](img/60b24f7b6385510c2455f502964c1bc9_17.png)

这是因为`>`符号默认只重定向**标准输出**。在Linux中，标准输出用文件描述符`1`表示，标准错误输出用`2`表示。上面的命令等价于：
```bash
find /etc -name password 1> find.out
```
所以，错误输出依旧显示在终端上。

![](img/60b24f7b6385510c2455f502964c1bc9_19.png)

如果只想看到正确结果，而将错误信息重定向到文件，可以使用`2>`：
```bash
find /etc -name password 2> fderror
```
此时，正确结果输出到终端，错误信息则写入`fderror`文件。查看该文件，可以看到因目录无`r-x`权限而导致的“权限拒绝”错误。

关于重定向的覆盖与追加：
*   `>` 和 `2>`：覆盖目标文件原有内容。
*   `>>` 和 `2>>`：将输出追加到目标文件末尾。

![](img/60b24f7b6385510c2455f502964c1bc9_21.png)

![](img/60b24f7b6385510c2455f502964c1bc9_23.png)

## 合并与丢弃输出

有时我们需要将正确和错误的输出分别重定向到不同文件：
```bash
find /etc -name password > find.out 2> find.error
```
这样，当前终端将没有任何输出。

如果希望将所有输出（正确和错误）都重定向到同一个文件，可以使用`&>`：
```bash
find /etc -name password &> find.all
```
这等同于命令 `find /etc -name password > find.all 2>&1`。这里的 `2>&1` 表示将标准错误输出重定向到标准输出所指向的地方。

![](img/60b24f7b6385510c2455f502964c1bc9_25.png)

![](img/60b24f7b6385510c2455f502964c1bc9_27.png)

一个特殊的用法是将输出丢弃，既不显示也不保存。这可以通过重定向到空设备文件 `/dev/null` 实现：
```bash
find /etc -name password 2> /dev/null
```
`/dev/null` 是一个特殊的字符设备文件，它像一个“黑洞”，接收任何数据但不保存也不显示。这在脚本中非常有用，可以屏蔽不必要的输出，只通过命令的退出状态码（`$?`，0表示成功，非0表示失败）来判断命令执行结果。

## 管道

管道符 `|` 是连接多个命令的强大工具，它允许将一个命令的**标准输出**作为另一个命令的**标准输入**。

基本语法如下：
```bash
command1 | command2 | command3 ...
```
其含义是：`command1` 的输出作为 `command2` 的输入，`command2` 的输出又作为 `command3` 的输入，依此类推。

例如，统计查找到的文件数量：
```bash
find /etc -name password | wc -l
```
这里，`find` 命令的输出（两个文件名）通过管道传递给 `wc -l` 命令，后者统计行数，最终输出 `2`。

![](img/60b24f7b6385510c2455f502964c1bc9_33.png)

![](img/60b24f7b6385510c2455f502964c1bc9_35.png)

需要注意的是，**默认情况下，标准错误输出不能通过管道**。观察以下命令：
```bash
find /etc -name password 2>&1 | wc -l
```
此时，错误信息被重定向到标准输出，从而也能通过管道被 `wc -l` 统计到行数。

![](img/60b24f7b6385510c2455f502964c1bc9_37.png)

管道的强大之处在于能将多个单一功能的命令组合起来，完成复杂的数据处理。例如，可以组合多个文本处理命令来提取特定信息。

*   将内容发送给打印机（如有）：`echo “Test Print” | lpr -P printer_name`

关于命令组合的常见误区：管道传递的是文本流，而非命令本身。例如，`mkdir /data/test | chmod 777 /data/test` 是错误的，因为 `mkdir` 的输出是空，`chmod` 无法从管道获得有效的路径参数。正确的做法是分步执行或使用逻辑操作符（如 `&&`）。

本节课中我们一起学习了IO重定向和管道。我们了解了如何区分并控制标准输出与错误输出，如何将它们重定向到文件或丢弃。更重要的是，我们掌握了管道符 `|` 的用法，它能够将多个命令像流水线一样连接起来，极大地增强了命令行的处理能力。这些是Linux高效运维和脚本编写的基础，请务必熟练掌握。
