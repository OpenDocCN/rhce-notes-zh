## wc统计命令 📊

![](img/6f7086927b26760ee1230e406b732910_5.png)

![](img/6f7086927b26760ee1230e406b732910_7.png)

`wc`命令用于统计文件中的行数、单词数和字节数。它非常常用，尤其是在需要快速了解文件基本信息时。

![](img/6f7086927b26760ee1230e406b732910_9.png)

### 基本用法

![](img/6f7086927b26760ee1230e406b732910_11.png)

![](img/6f7086927b26760ee1230e406b732910_13.png)

`wc`命令的基本语法是：
```bash
wc [选项] 文件名
```
如果不加任何选项，`wc`会同时显示文件的行数、单词数和字节数。

例如，统计`/etc/passwd`文件的信息：
```bash
wc /etc/passwd
```
执行后，你会看到类似`43 43 2381 /etc/passwd`的输出，分别代表行数、单词数和字节数。

### 常用选项

以下是`wc`命令最常用的几个选项：

*   **`-l`**：仅统计行数。这是最常用的选项。
    ```bash
    wc -l /etc/passwd
    ```
*   **`-w`**：仅统计单词数。单词是由空格、制表符或换行符分隔的字符串。
*   **`-c`**：仅统计字节数。

![](img/6f7086927b26760ee1230e406b732910_15.png)

对于初学者，记住`wc -l`用于统计行数就足够了，这在日常工作中最为实用。

## 管道符 `|` 🔗

![](img/6f7086927b26760ee1230e406b732910_17.png)

上一节我们介绍了如何用`wc`统计一个文件的信息。本节中我们来看看如何将多个命令串联起来，实现更复杂的功能，这就需要用到管道符。

管道符`|`的功能非常强大，它可以将前一个命令的标准输出，作为后一个命令的标准输入。简单说，就是把一个命令的结果交给另一个命令去处理。

### 管道符的应用

以下是管道符的一些典型应用场景：

![](img/6f7086927b26760ee1230e406b732910_19.png)

![](img/6f7086927b26760ee1230e406b732910_21.png)

*   **组合命令**：例如，你想查看`/etc/passwd`文件的前10行中的最后5行。单独使用`head`或`tail`都无法直接实现。
    ```bash
    head -10 /etc/passwd | tail -5
    ```
    这条命令先取出文件的前10行，然后将这10行结果通过管道交给`tail -5`命令，最终显示出这10行中的最后5行。

![](img/6f7086927b26760ee1230e406b732910_23.png)

*   **无限链接**：管道可以连接多个命令。例如，统计上述操作最终输出的行数：
    ```bash
    head -10 /etc/passwd | tail -5 | wc -l
    ```
    这条命令链会输出数字`5`。

![](img/6f7086927b26760ee1230e406b732910_25.png)

*   **过滤输出**：查看网络配置，但只关心前两行（通常包含网卡名和IP地址）：
    ```bash
    ifconfig ens32 | head -2
    ```

![](img/6f7086927b26760ee1230e406b732910_27.png)

管道符是Linux命令行中提高效率的利器，通过组合简单的命令，可以完成复杂的文本处理任务。

![](img/6f7086927b26760ee1230e406b732910_29.png)

![](img/6f7086927b26760ee1230e406b732910_31.png)

## 重定向操作 📥📤

![](img/6f7086927b26760ee1230e406b732910_33.png)

我们学会了用管道将命令的结果传递给另一个命令。那么，如果想把命令的结果保存到一个文件里，或者从一个文件读取数据作为命令的输入，该怎么做呢？这就需要用到重定向操作。

![](img/6f7086927b26760ee1230e406b732910_35.png)

![](img/6f7086927b26760ee1230e406b732910_37.png)

重定向操作通过符号`>`、`>>`、`<`等来控制数据的流向。

![](img/6f7086927b26760ee1230e406b732910_39.png)

### 输出重定向

![](img/6f7086927b26760ee1230e406b732910_41.png)

输出重定向是最常用的，它把命令的输出结果写入文件，而不是显示在屏幕上。

*   **`>` 覆盖重定向**：将命令的输出写入指定文件。**如果文件已存在，则会清空原有内容**。
    ```bash
    # 将 ifconfig ens32 的前两行结果保存到 net_ens32.txt 文件
    ifconfig ens32 | head -2 > net_ens32.txt
    # 查看文件内容
    cat net_ens32.txt
    ```
    **注意**：如果`>`后面的文件不存在，系统会自动创建它。

![](img/6f7086927b26760ee1230e406b732910_43.png)

*   **`>>` 追加重定向**：将命令的输出**追加**到指定文件的末尾，不会清空原有内容。
    ```bash
    # 将当前日期时间追加到 log.txt 文件
    date >> log.txt
    ```

![](img/6f7086927b26760ee1230e406b732910_45.png)

### 区分标准输出和错误输出

![](img/6f7086927b26760ee1230e406b732910_47.png)

命令执行后通常有两种输出：“标准输出”（正确结果）和“标准错误输出”（错误信息）。默认情况下，两者都显示在屏幕。重定向时可以分别处理它们。

![](img/6f7086927b26760ee1230e406b732910_49.png)

*   **`1>` 或 `>`**：重定向**标准输出**（正确结果）。
*   **`2>`**：重定向**标准错误输出**（错误信息）。
*   **`&>`**：重定向**标准输出和标准错误输出**（全部结果）。

![](img/6f7086927b26760ee1230e406b732910_51.png)

以下是具体用法示例：

![](img/6f7086927b26760ee1230e406b732910_53.png)

*   保存命令的正确结果：
    ```bash
    ls /etc/passwd > output.txt  # output.txt 中会保存文件列表
    ```
*   保存命令的错误信息（例如，列出一个不存在的文件）：
    ```bash
    ls /nonexistent_file 2> error.log # error.log 中会保存“未找到文件”的错误信息
    ```
*   同时保存正确和错误结果（覆盖方式）：
    ```bash
    ls /etc/passwd /nonexistent_file &> all_output.txt
    ```
*   同时保存正确和错误结果（追加方式）：
    ```bash
    ls /etc/passwd /nonexistent_file &>> all_output.log
    ```

![](img/6f7086927b26760ee1230e406b732910_55.png)

**应用场景**：在服务器维护中，经常需要批量检查多台主机是否在线（使用`ping`命令）。我们可以将`ping`不通（错误输出）的主机IP记录到一个文件里，方便后续排查故障。
```bash
ping -c 2 192.168.1.100 2>> offline_hosts.txt
```

## echo与sleep命令 ⏳

最后，我们来了解两个看似简单但很有用的命令：`echo`和`sleep`。

![](img/6f7086927b26760ee1230e406b732910_57.png)

### echo命令

![](img/6f7086927b26760ee1230e406b732910_59.png)

`echo`命令用于在终端输出一行文本或变量的值。

![](img/6f7086927b26760ee1230e406b732910_61.png)

它的基本语法非常简单：
```bash
echo [字符串]
```
例如：
```bash
echo Hello World
echo $HOME # 输出环境变量HOME的值，即当前用户的家目录路径
```

![](img/6f7086927b26760ee1230e406b732910_63.png)

**结合重定向的妙用**：`echo`常与重定向联用，快速创建或修改文件内容，而无需使用文本编辑器。
```bash
# 快速创建一个包含内容的文件
echo “This is a config line” > myconfig.conf
# 快速清空一个文件（写入空字符串）
echo “” > myconfig.conf
# 快速修改主机名（需要root权限）
echo “new-hostname” > /etc/hostname
```

![](img/6f7086927b26760ee1230e406b732910_65.png)

![](img/6f7086927b26760ee1230e406b732910_67.png)

### sleep命令

![](img/6f7086927b26760ee1230e406b732910_69.png)

![](img/6f7086927b26760ee1230e406b732910_71.png)

`sleep`命令用于让当前 shell 进程（或脚本）暂停（睡眠）一段时间。

![](img/6f7086927b26760ee1230e406b732910_73.png)

![](img/6f7086927b26760ee1230e406b732910_75.png)

命令格式为：
```bash
sleep 数字[后缀]
```
后缀可以是：
*   `s` （秒，默认）
*   `m` （分钟）
*   `h` （小时）
*   `d` （天）

![](img/6f7086927b26760ee1230e406b732910_77.png)

例如：
```bash
sleep 5    # 暂停5秒
sleep 2m   # 暂停2分钟
sleep 0.1  # 暂停0.1秒
```

![](img/6f7086927b26760ee1230e406b732910_79.png)

![](img/6f7086927b26760ee1230e406b732910_80.png)

![](img/6f7086927b26760ee1230e406b732910_82.png)

**应用场景**：`sleep`在编写Shell脚本时非常有用。
1.  **控制执行节奏**：在循环或连续操作中，插入短暂的`sleep`可以避免CPU使用率瞬间达到100%，让系统有喘息之机，防止过载。
    ```bash
    # 一个简单的演示：每秒输出一次Hello，但CPU占用很低
    while true; do
      echo “Hello”
      sleep 1
    done
    ```
2.  **定时任务**：在某些需要等待后台操作完成后再执行下一步的场景中，可以用`sleep`做简单的延时。
