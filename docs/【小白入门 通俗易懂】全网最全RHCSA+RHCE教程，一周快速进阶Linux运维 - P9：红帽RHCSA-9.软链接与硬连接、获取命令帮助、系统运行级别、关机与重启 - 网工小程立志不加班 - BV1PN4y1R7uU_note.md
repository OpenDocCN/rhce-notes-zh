## 软链接与硬链接 🔗

![](img/581022123e9eef570082f8d0a0926c06_31.png)

![](img/581022123e9eef570082f8d0a0926c06_33.png)

![](img/581022123e9eef570082f8d0a0926c06_35.png)

![](img/581022123e9eef570082f8d0a0926c06_37.png)

![](img/581022123e9eef570082f8d0a0926c06_39.png)

上一节我们介绍了文件和目录的基本操作，本节中我们来看看Linux中特殊的“链接”文件。链接文件类似于Windows系统中的“快捷方式”，但Linux中分为软链接和硬链接两种，它们有显著的区别。

### 软链接（符号链接）

![](img/581022123e9eef570082f8d0a0926c06_41.png)

![](img/581022123e9eef570082f8d0a0926c06_43.png)

软链接可以理解为指向另一个文件或目录的快捷方式。它的特点如下：
*   **可以跨分区**：源文件和链接文件可以位于不同的磁盘分区。
*   **可以对目录创建链接**。
*   **依赖于源文件**：如果删除源文件，软链接将失效（变成“坏链”）。

创建软链接的命令格式是：
```bash
ln -s <源文件路径> <链接文件路径>
```
**注意**：创建链接时，建议使用源文件的**绝对路径**，以避免链接文件失效。

例如，将`/opt/hello_soft`文件软链接到`/media`目录：
```bash
ln -s /opt/hello_soft /media/hello_soft_link
```
创建后，对源文件或链接文件的修改会同步到另一方。删除源文件后，链接文件将不可用。

![](img/581022123e9eef570082f8d0a0926c06_47.png)

### 硬链接

![](img/581022123e9eef570082f8d0a0926c06_49.png)

![](img/581022123e9eef570082f8d0a0926c06_51.png)

![](img/581022123e9eef570082f8d0a0926c06_52.png)

硬链接更像是给同一个文件数据块起了另一个“名字”。它的特点与软链接几乎相反：
*   **不能跨分区**：源文件和硬链接必须在同一分区。
*   **不能对目录创建硬链接**。
*   **独立于源文件**：删除源文件后，硬链接文件仍然可用，因为文件数据块并未被删除。

创建硬链接的命令格式是（省略`-s`选项）：
```bash
ln <源文件路径> <链接文件路径>
```
例如，在同一个分区内创建硬链接：
```bash
ln /opt/hello_hard /media/hello_hard_link
```
硬链接文件看起来和普通文件没有颜色区别。它和`cp -p`命令的复制不同之处在于，硬链接与源文件始终保持数据同步，而复制出的文件是独立的副本。

简单来说，软链接是“快捷方式”，硬链接是“文件的别名”。在实际使用中，软链接更为常见。

## 获取命令帮助 ❓

在Linux中，当你忘记某个命令的用法或想了解其详细选项时，可以使用系统提供的帮助功能。

![](img/581022123e9eef570082f8d0a0926c06_56.png)

以下是几种获取命令帮助的方法：
*   **`<命令> --help`**：这是最常用、最快捷的方式，可以查看命令的基本用法和选项说明。例如：`ls --help`。
*   **`man <命令>`**：查看命令的详细手册页（manual），内容比`--help`更详尽，包含描述、选项、示例等。例如：`man ls`。查看时可以使用与`less`命令相同的快捷键（如空格翻页，`q`退出）。
*   **`help <命令>`**：主要用于查看Shell**内部命令**的帮助（如`cd`, `echo`），对于大多数外部命令（如`ls`, `cp`）不适用。
*   **`info <命令>`**：提供另一种格式的帮助文档，但如今已较少使用。

对于初学者，如果英文帮助文档阅读困难，完全可以通过互联网搜索（如“ls命令常用选项”）来获取更易理解的中文资料。

## 系统运行级别 🖥️

Linux系统有不同的运行级别（Runlevel），它定义了系统启动后运行哪些服务。主要有7个级别：

![](img/581022123e9eef570082f8d0a0926c06_72.png)

![](img/581022123e9eef570082f8d0a0926c06_74.png)

![](img/581022123e9eef570082f8d0a0926c06_76.png)

以下是各运行级别的说明：
*   **0**：关机。
*   **1**：单用户模式。仅允许root用户登录，用于系统维护（如忘记密码）。所有网络服务通常关闭。
*   **2**：多用户模式（无NFS网络文件共享）。
*   **3**：完全的多用户模式（命令行界面），这是服务器最常用的标准运行级别。
*   **4**：未使用。
*   **5**：图形化界面模式。带有GUI的系统的标准运行级别。
*   **6**：重启。

![](img/581022123e9eef570082f8d0a0926c06_77.png)

![](img/581022123e9eef570082f8d0a0926c06_79.png)

![](img/581022123e9eef570082f8d0a0926c06_81.png)

**查看当前运行级别**：
```bash
runlevel
```
**切换运行级别**（例如切换到单用户模式）：
```bash
init 1
```
**查看和设置默认运行级别**：
```bash
systemctl get-default # 查看默认级别
systemctl set-default multi-user.target # 设置默认级别为3（命令行）
systemctl set-default graphical.target # 设置默认级别为5（图形界面）
```
通常，服务器默认运行级别为3（无图形界面），个人桌面版默认级别为5。我们一般不需要修改运行级别。

## 关机与重启 ⏻

![](img/581022123e9eef570082f8d0a0926c06_95.png)

![](img/581022123e9eef570082f8d0a0926c06_97.png)

![](img/581022123e9eef570082f8d0a0926c06_99.png)

在命令行下管理系统的开关机是运维人员的基本功。

![](img/581022123e9eef570082f8d0a0926c06_101.png)

![](img/581022123e9eef570082f8d0a0926c06_103.png)

![](img/581022123e9eef570082f8d0a0926c06_105.png)

常用的关机与重启命令如下：
*   **`poweroff`**：立即关机。
*   **`reboot`**：立即重启。
*   **`shutdown`**：功能更丰富的关机命令。
    *   `shutdown -h now`：立即关机。
    *   `shutdown -h +10`：10分钟后关机。
    *   `shutdown -r now`：立即重启。
*   **`init 0`**：切换到运行级别0（关机）。
*   **`init 6`**：切换到运行级别6（重启）。

![](img/581022123e9eef570082f8d0a0926c06_107.png)

![](img/581022123e9eef570082f8d0a0926c06_109.png)

![](img/581022123e9eef570082f8d0a0926c06_111.png)

![](img/581022123e9eef570082f8d0a0926c06_113.png)

![](img/581022123e9eef570082f8d0a0926c06_115.png)

对于日常使用，记住`poweroff`和`reboot`这两个最直接的命令即可。
