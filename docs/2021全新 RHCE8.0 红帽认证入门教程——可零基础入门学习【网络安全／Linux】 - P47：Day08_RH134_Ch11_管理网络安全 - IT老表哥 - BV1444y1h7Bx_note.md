# RHCE8.0 红帽认证入门教程：Day08：管理网络安全 🔒

在本节课中，我们将要学习 Linux 系统中的防火墙管理。防火墙是网络安全的核心组件，它通过定义规则来控制网络流量的进出。我们将重点学习红帽系统 8.0 中使用的 `firewalld` 服务，了解其区域（zone）概念、基本规则配置以及高级的富规则（rich rule）应用。

## 防火墙基础概念

![](img/4d4b859996c186d0625436d4f9113fcc_5.png)

![](img/4d4b859996c186d0625436d4f9113fcc_7.png)

![](img/4d4b859996c186d0625436d4f9113fcc_9.png)

上一节我们介绍了网络安全的重要性，本节中我们来看看防火墙的具体实现。在 Linux 系统中，`firewalld` 服务负责执行包过滤操作。要配置防火墙，我们需要掌握两点：如何编写规则以及如何匹配规则。

![](img/4d4b859996c186d0625436d4f9113fcc_11.png)

![](img/4d4b859996c186d0625436d4f9113fcc_13.png)

![](img/4d4b859996c186d0625436d4f9113fcc_14.png)

![](img/4d4b859996c186d0625436d4f9113fcc_16.png)

防火墙通过“区域”来管理规则。每个区域包含一组预定义的规则。数据包会根据其源地址或进入的网卡被关联到某个区域，然后在该区域内匹配规则。这类似于实体防火墙的配置逻辑。

![](img/4d4b859996c186d0625436d4f9113fcc_18.png)

![](img/4d4b859996c186d0625436d4f9113fcc_19.png)

![](img/4d4b859996c186d0625436d4f9113fcc_21.png)

![](img/4d4b859996c186d0625436d4f9113fcc_22.png)

在配置防火墙规则前，需要先配置数据源（源地址）和网卡所属的区域。默认情况下，所有网卡都属于 `public` 区域。

![](img/4d4b859996c186d0625436d4f9113fcc_24.png)

红帽 7 系统有 9 个区域，而 8 系统增加了一个 `libvirt` 区域（供虚拟机专用）。以下是常见区域的默认行为：

*   **trusted**：允许所有传输流量，表示完全信任。
*   **block**：拒绝所有传入流量，除非与传出流量相关。
*   **DMZ**：非军事区，用于隔离外部网络。
*   **home** / **internal**：用于家庭或内部网络，规则较为宽松。
*   **work**：用于工作场所网络。
*   **public**：默认区域，通常只放行 SSH、DHCPv6 等少数服务，拒绝其他所有传入连接。

![](img/4d4b859996c186d0625436d4f9113fcc_26.png)

![](img/4d4b859996c186d0625436d4f9113fcc_27.png)

### 管理数据源（源地址）

![](img/4d4b859996c186d0625436d4f9113fcc_126.png)

![](img/4d4b859996c186d0625436d4f9113fcc_128.png)

![](img/4d4b859996c186d0625436d4f9113fcc_130.png)

![](img/4d4b859996c186d0625436d4f9113fcc_132.png)

以下是管理源地址与区域绑定的操作：

![](img/4d4b859996c186d0625436d4f9113fcc_134.png)

![](img/4d4b859996c186d0625436d4f9113fcc_136.png)

![](img/4d4b859996c186d0625436d4f9113fcc_138.png)

![](img/4d4b859996c186d0625436d4f9113fcc_139.png)

![](img/4d4b859996c186d0625436d4f9113fcc_141.png)

![](img/4d4b859996c186d0625436d4f9113fcc_142.png)

*   **添加数据源到区域**：将 `192.168.145.0/24` 网段添加到 `home` 区域。
    ```bash
    firewall-cmd --permanent --add-source=192.168.145.0/24 --zone=home
    firewall-cmd --reload
    ```
    此后，来自该网段的数据包将进入 `home` 区域匹配规则。
*   **列出区域内的数据源**：
    ```bash
    firewall-cmd --list-sources --zone=home
    ```
*   **查询数据源是否在区域内**：
    ```bash
    firewall-cmd --query-source=192.168.145.0/24 --zone=home
    ```
    命令返回 `yes` 或 `no`。
*   **更改数据源所属区域**：将上述网段从 `home` 改到 `public` 区域。
    ```bash
    firewall-cmd --permanent --change-source=192.168.145.0/24 --zone=public
    firewall-cmd --reload
    ```
*   **从区域中移除数据源**：移除后，该数据源将回归默认区域的规则管理。
    ```bash
    firewall-cmd --permanent --remove-source=192.168.145.0/24 --zone=public
    firewall-cmd --reload
    ```

网卡也必须属于某个区域。通常使用 `change-interface` 来修改其所属区域。

![](img/4d4b859996c186d0625436d4f9113fcc_158.png)

![](img/4d4b859996c186d0625436d4f9113fcc_159.png)

![](img/4d4b859996c186d0625436d4f9113fcc_161.png)

![](img/4d4b859996c186d0625436d4f9113fcc_163.png)

![](img/4d4b859996c186d0625436d4f9113fcc_164.png)

*   **更改网卡区域**：将网卡 `ens160` 关联到 `home` 区域。
    ```bash
    firewall-cmd --permanent --change-interface=ens160 --zone=home
    firewall-cmd --reload
    ```
*   **将网卡恢复到默认区域**：
    ```bash
    firewall-cmd --permanent --remove-interface=ens160 --zone=home
    firewall-cmd --reload
    ```
*   **添加网卡到指定区域**（如果网卡尚未被任何区域管理）：
    ```bash
    firewall-cmd --permanent --add-interface=ens160 --zone=work
    firewall-cmd --reload
    ```

## 配置防火墙规则

![](img/4d4b859996c186d0625436d4f9113fcc_175.png)

现在我们已经知道如何划分区域，接下来学习如何在区域内配置具体的允许或拒绝规则。我们主要使用 `firewall-cmd` 命令进行操作。

### 基本规则

![](img/4d4b859996c186d0625436d4f9113fcc_177.png)

![](img/4d4b859996c186d0625436d4f9113fcc_179.png)

![](img/4d4b859996c186d0625436d4f9113fcc_180.png)

![](img/4d4b859996c186d0625436d4f9113fcc_182.png)

![](img/4d4b859996c186d0625436d4f9113fcc_184.png)

基本规则允许我们基于服务、端口或数据源来放行流量，这本质上是一种白名单模式。

![](img/4d4b859996c186d0625436d4f9113fcc_186.png)

![](img/4d4b859996c186d0625436d4f9113fcc_187.png)

![](img/4d4b859996c186d0625436d4f9113fcc_189.png)

![](img/4d4b859996c186d0625436d4f9113fcc_191.png)

![](img/4d4b859996c186d0625436d4f9113fcc_193.png)

*   **基于数据源放行**：添加一个网段到区域即意味着放行来自该网段的所有流量（受区域其他规则限制）。
    ```bash
    firewall-cmd --permanent --add-source=192.168.146.0/24 --zone=trusted
    firewall-cmd --reload
    ```
*   **基于服务放行**：在 `public` 区域放行 SMTP 服务（端口 25）。
    ```bash
    firewall-cmd --permanent --add-service=smtp --zone=public
    firewall-cmd --reload
    ```
*   **基于端口放行**：在 `public` 区域放行 TCP 协议的 80 端口。
    ```bash
    firewall-cmd --permanent --add-port=80/tcp --zone=public
    firewall-cmd --reload
    ```

### 富规则 (Rich Rules)

![](img/4d4b859996c186d0625436d4f9113fcc_204.png)

![](img/4d4b859996c186d0625436d4f9113fcc_205.png)

当基本规则无法满足复杂需求时，需要使用富规则。富规则使用一种表达性语言，可以实现更精细的控制，例如针对单个 IP 地址、记录日志、端口转发等。

![](img/4d4b859996c186d0625436d4f9113fcc_207.png)

富规则的基本语法结构如下：
```
rule [family="ipv4|ipv6"]
     [source|destination address="address[/mask]"]
     [service name="service名"]
     [port port="端口号" protocol="tcp|udp"]
     [log|audit]
     [accept|reject|drop]
```

![](img/4d4b859996c186d0625436d4f9113fcc_209.png)

规则在区域内的匹配优先级顺序为：
1.  端口转发和伪装规则。
2.  日志记录规则。
3.  允许（accept）规则。
4.  拒绝（reject/drop）规则。

以下是管理富规则的命令示例：

![](img/4d4b859996c186d0625436d4f9113fcc_211.png)

*   **添加富规则**：拒绝 IP 为 `192.168.145.129` 的主机访问本机的 SSH 服务。
    ```bash
    firewall-cmd --permanent --add-rich-rule='rule family=ipv4 source address=192.168.145.129 service name=ssh reject'
    firewall-cmd --reload
    ```
    `reject` 会返回拒绝信息，而 `drop` 会直接丢弃数据包不留痕迹。通常对友好网络使用 `reject`，对恶意网络使用 `drop`。
*   **移除富规则**：
    ```bash
    firewall-cmd --permanent --remove-rich-rule='rule family=ipv4 source address=192.168.145.129 service name=ssh reject'
    firewall-cmd --reload
    ```
*   **允许特定网段访问特定端口**：允许 `192.168.145.0/24` 网段访问本机的 8080/TCP 端口。
    ```bash
    firewall-cmd --permanent --add-rich-rule='rule family=ipv4 source address=192.168.145.0/24 port port=8080 protocol=tcp accept'
    firewall-cmd --reload
    ```

![](img/4d4b859996c186d0625436d4f9113fcc_213.png)

### 端口伪装与转发

![](img/4d4b859996c186d0625436d4f9113fcc_215.png)

![](img/4d4b859996c186d0625436d4f9113fcc_217.png)

![](img/4d4b859996c186d0625436d4f9113fcc_219.png)

*   **端口伪装**：用于网络地址转换（NAT），隐藏内部网络的实际 IP。
    ```bash
    firewall-cmd --permanent --add-rich-rule='rule family=ipv4 source address=192.168.145.0/24 masquerade'
    firewall-cmd --reload
    ```
*   **端口转发**：将到达本机某个端口（如 5423）的流量转发到另一个端口（如 80）或另一台机器。这可以用于隐藏真实服务端口。
    ```bash
    # 将本机5423端口的TCP流量转发到本机的80端口
    firewall-cmd --permanent --add-rich-rule='rule family=ipv4 source address=192.168.145.0/24 forward-port port=5423 protocol=tcp to-port=80'
    firewall-cmd --reload
    ```

## 管理 SELinux 端口上下文

![](img/4d4b859996c186d0625436d4f9113fcc_223.png)

![](img/4d4b859996c186d0625436d4f9113fcc_225.png)

在启用 SELinux 的情况下，即使防火墙放行了某个端口，如果该端口没有正确的 SELinux 安全上下文，服务也可能无法被访问。这对于自定义端口（如 8909）尤其重要。

![](img/4d4b859996c186d0625436d4f9113fcc_227.png)

![](img/4d4b859996c186d0625436d4f9113fcc_229.png)

![](img/4d4b859996c186d0625436d4f9113fcc_230.png)

*   **列出所有端口上下文**：
    ```bash
    semanage port -l
    ```
*   **添加端口上下文**：将 TCP 协议的 8909 端口标记为 `http_port_t` 类型，以便 HTTP 服务可以使用它。
    ```bash
    semanage port -a -t http_port_t -p tcp 8909
    ```
*   **修改或删除端口上下文**：使用 `-m` 选项修改，使用 `-d` 选项删除。

本节课中我们一起学习了红帽 Linux 8.0 中防火墙的全面管理。我们从 `firewalld` 的区域模型开始，学习了如何查看和修改默认区域，如何将数据源和网络接口分配到不同的区域。接着，我们深入探讨了如何配置基本规则来放行服务或端口，以及如何使用更强大的富规则来实现精细化的流量控制（如拒绝特定IP、记录日志）。此外，我们还介绍了端口伪装（NAT）和端口转发的配置方法。最后，我们补充了在 SELinux 环境下管理自定义端口上下文的知识，确保安全策略不会阻止合法的网络访问。掌握这些技能，是构建和管理安全 Linux 系统环境的基础。
