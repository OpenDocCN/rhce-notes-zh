## 文件与目录操作

![](img/799fdd1a55621f03da3465262e546234_4.png)

![](img/799fdd1a55621f03da3465262e546234_6.png)

上一节我们介绍了基础命令行，本节中我们来看看如何创建、删除文件和目录，以及路径的概念。

![](img/799fdd1a55621f03da3465262e546234_8.png)

### 文件创建与删除
使用 `touch` 命令可以创建空文件，相当于图形界面中的“新建文本文档”。

```bash
touch filename.txt
```

删除文件使用 `rm` 命令。**超级管理员（root）** 默认会为 `rm` 命令设置别名 `rm -i`，删除前会询问确认，这是为了防止误操作。而**普通用户**默认没有此别名，删除时不会询问。

```bash
rm filename.txt
```

若要强制删除文件且不询问，可以使用 `-f` 选项。

```bash
rm -f filename.txt
```

### 目录创建与删除
创建目录使用 `mkdir` 命令。

```bash
mkdir directory_name
```

删除**空目录**可以使用 `rmdir` 命令或 `rm -d` 命令。

```bash
rmdir directory_name
# 或
rm -d directory_name
```

若要删除**非空目录及其内部所有内容**，需要使用 `rm -r`（递归删除）命令。结合 `-f` 选项可以强制删除，不进行任何确认。

```bash
rm -rf directory_name
```

**警告**：`rm -rf /` 命令会删除根目录下的所有文件，导致系统崩溃，使用时务必谨慎。

### 路径：绝对路径与相对路径
路径描述文件或目录的位置，有两种形式：
*   **绝对路径**：从根目录（`/`）开始，完整地描述目标位置。例如：`/home/redhat/file.txt`
*   **相对路径**：相对于当前工作目录来描述目标位置。例如，当前在 `/home` 目录下，`redhat/file.txt` 就是相对路径。

![](img/799fdd1a55621f03da3465262e546234_22.png)

### 批量操作与通配符
可以批量创建文件或目录。

```bash
# 枚举创建
touch file{1,2,3,4,5}.txt
# 范围创建
touch file{6..15}.txt
```

使用通配符 `*` 可以匹配多个文件，常用于批量删除。

```bash
# 删除当前目录下所有文件
rm -rf *
```

### 修改文件时间
`touch` 命令还可以用来更新文件的访问和修改时间戳。更新所有文件时间为当前时间：

```bash
touch *
```

### 递归创建目录
使用 `-p` 选项可以递归创建目录结构，如果父目录不存在，则会一并创建。

```bash
mkdir -p /tmp/a/b/c
```

### 目录列表显示选项
`ls` 命令有一些有用的选项：
*   `ls -r`：将结果反向排序输出。
*   `ls -R`：递归列出子目录内容。

## Vim 文本编辑器

![](img/799fdd1a55621f03da3465262e546234_42.png)

文件操作是基础，而编辑文件内容则需要一个强大的工具。本节我们将学习功能强大的 Vim 编辑器。

![](img/799fdd1a55621f03da3465262e546234_44.png)

Vim 是一个模式化编辑器，主要包含三种模式：**命令模式**、**编辑模式**和**末行模式**。

![](img/799fdd1a55621f03da3465262e546234_46.png)

![](img/799fdd1a55621f03da3465262e546234_48.png)

### 模式切换与基本操作
1.  **启动与命令模式**：输入 `vim filename` 进入 Vim，初始状态为**命令模式**，此模式下按键被解释为命令。
2.  **进入编辑模式**：在命令模式下，按以下键可进入**编辑模式**进行文本输入：
    *   `i`：在光标前插入。
    *   `a`：在光标后插入。
    *   `I`：在行首插入。
    *   `A`：在行尾插入。
    *   `o`：在下一行插入。
    *   `O`：在上一行插入。
3.  **返回命令模式**：在编辑模式下，按 `Esc` 键返回命令模式。
4.  **进入末行模式**：在命令模式下，输入 `:` 进入**末行模式**，可以执行保存、退出等操作。
5.  **保存与退出（末行模式）**：
    *   `:w`：保存文件。
    *   `:q`：退出 Vim（如果文件有未保存的修改，则不允许退出）。
    *   `:q!`：强制退出，不保存任何修改。
    *   `:wq` 或 `:x`：保存并退出。

### 常用命令模式操作
*   **显示/隐藏行号**：
    ```vim
    :set number
    :set nonumber
    ```
*   **光标跳转**：
    *   `gg`：跳到文件第一行。
    *   `G`：跳到文件最后一行。
    *   `20gg`：跳到第20行。
*   **查找**：
    *   `/keyword`：在文件中向下查找 `keyword`。
    *   `n`：查找下一个匹配项。
    *   `N`：查找上一个匹配项。
    *   `/keyword\c`：查找时忽略大小写。
*   **替换**：
    *   `:s/old/new`：替换当前行第一个 `old` 为 `new`。
    *   `:s/old/new/g`：替换当前行所有 `old` 为 `new`。
    *   `:%s/old/new/g`：替换全文所有 `old` 为 `new`。
*   **复制、剪切、粘贴**：
    *   `yy`：复制当前行。
    *   `5yy`：复制从当前行开始的5行。
    *   `dd`：剪切（删除）当前行。
    *   `p`：在光标后粘贴。
    *   `P`：在光标前粘贴。
*   **撤销**：`u` 撤销上一次操作。

### 处理 Vim 异常
如果 Vim 异常退出（如终端意外关闭），再次编辑同一文件时可能会提示交换文件（`.swp`）已存在。通常选择 `(D)elete` 删除交换文件，或 `(R)ecover` 恢复内容。

## 用户管理基础

管理多用户系统是 Linux 的核心功能之一。本节我们初步了解用户和组的管理。

### 用户与组的概念
*   **用户**：系统资源的访问主体。每个进程都以特定用户身份运行。
*   **组**：用户的集合，便于批量分配权限。每个用户必须属于一个**主组**，同时可以属于零个或多个**附加组**。

![](img/799fdd1a55621f03da3465262e546234_50.png)

### 用户信息文件
用户账户信息存储在 `/etc/passwd` 文件中，每行代表一个用户，由冒号 `:` 分隔为7个字段：
1.  用户名
2.  密码占位符（`x` 表示密码存储在 `/etc/shadow`）
3.  用户ID (UID)
4.  主组ID (GID)
5.  用户全名或描述
6.  家目录
7.  登录 Shell（如 `/bin/bash`）

![](img/799fdd1a55621f03da3465262e546234_52.png)

### 用户密码文件
加密后的密码及相关策略存储在 `/etc/shadow` 文件中，同样由冒号分隔，包含密码哈希、最后修改日期、密码有效期、过期警告时间等信息。

### UID 范围
*   `0`：超级管理员 (root)。
*   `1-200`：系统静态分配给系统进程。
*   `201-999`：系统动态分配给文件系统对象。
*   `1000+`：普通用户。

### 用户管理命令
*   **创建用户**：`useradd` 或 `adduser`。默认会创建同名组和家目录。
    ```bash
    useradd username
    ```
*   **修改用户属性**：`usermod`。例如修改家目录：
    ```bash
    usermod -d /new/home/dir -m username
    ```
*   **删除用户**：
    *   `userdel username`：仅删除用户，保留家目录。
    *   `userdel -r username`：删除用户及其家目录、邮件等所有相关文件。
*   **查看用户ID**：`id username`
*   **切换用户**：
    *   `su - username`：切换用户并加载其环境变量和家目录。
    *   `su username`：切换用户，但环境不变。
    *   从 root 切换到其他用户无需密码，普通用户切换任何用户都需要密码。
    *   `exit`：退出当前用户会话，返回上一层。

### 为用户创建家目录
如果用户家目录丢失或未创建，可以手动创建并复制默认配置文件。

```bash
# 1. 创建家目录
mkdir /home/username
# 2. 复制骨架文件
cp -r /etc/skel/. /home/username
# 3. 修改所有权
chown -R username:username /home/username
```
