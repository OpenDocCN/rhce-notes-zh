## 创建文件与touch命令

上一节我们介绍了文件的复制和移动操作，本节中我们来看看如何创建和删除文件。

创建文件使用`touch`命令。`touch`命令的主要功能是创建一个空文件。

![](img/d0660e41793368cd9722346be5bbb13c_11.png)

**命令格式**：
```bash
touch 文件名
```

![](img/d0660e41793368cd9722346be5bbb13c_13.png)

![](img/d0660e41793368cd9722346be5bbb13c_15.png)

![](img/d0660e41793368cd9722346be5bbb13c_17.png)

例如，执行`touch file01`会创建一个名为`file01`的空文件。使用`ls -l`命令查看，可以看到该文件的大小为0。

除了创建文件，`touch`命令还有一个重要作用，那就是更新文件的时间戳。

![](img/d0660e41793368cd9722346be5bbb13c_19.png)

## 理解文件时间戳

要查看文件的详细时间信息，我们可以使用`stat`命令。

**命令格式**：
```bash
stat 文件名
```

执行该命令后，会显示文件的三个重要时间戳，通常简称为`atime`、`mtime`和`ctime`。

### 1. Access Time (atime)
`atime`指文件最后一次被访问的时间。例如，当你使用`cat`、`less`或`vi`（仅查看不修改）命令读取文件内容时，`atime`就会被更新到当前时间。

![](img/d0660e41793368cd9722346be5bbb13c_21.png)

它的一个作用是记录文件是否被他人查看过。对于需要频繁被访问的文件（如网页文件），频繁更新`atime`会增加磁盘I/O压力，因此在某些场景下可以关闭`atime`的更新以提升性能。

![](img/d0660e41793368cd9722346be5bbb13c_23.png)

![](img/d0660e41793368cd9722346be5bbb13c_24.png)

![](img/d0660e41793368cd9722346be5bbb13c_26.png)

### 2. Modify Time (mtime)
`mtime`指文件内容最后一次被修改的时间。当你编辑并保存文件后，`mtime`就会更新。

![](img/d0660e41793368cd9722346be5bbb13c_28.png)

`ls -l`命令默认显示的时间就是`mtime`。这个时间戳在**增量备份**中非常有用，备份软件可以通过比较`mtime`来判断哪些文件内容发生了变动，从而只备份修改过的文件，提高备份效率。

![](img/d0660e41793368cd9722346be5bbb13c_30.png)

### 3. Change Time (ctime)
`ctime`指文件状态（或元数据）最后一次发生变化的时间。这里的“状态”包括文件的权限、所有者、所属组、链接数等属性。

![](img/d0660e41793368cd9722346be5bbb13c_32.png)

![](img/d0660e41793368cd9722346be5bbb13c_34.png)

需要注意的是：
*   当`mtime`改变（文件内容被修改）时，`ctime`也一定会随之改变。
*   当`ctime`改变（如修改了文件权限）时，`mtime`不一定改变。

![](img/d0660e41793368cd9722346be5bbb13c_36.png)

## 使用touch更新时间戳

我们可以使用`touch`命令手动更新文件的时间戳。默认情况下，`touch 文件名`会将文件的`atime`和`mtime`都更新为当前时间。

![](img/d0660e41793368cd9722346be5bbb13c_38.png)

![](img/d0660e41793368cd9722346be5bbb13c_40.png)

![](img/d0660e41793368cd9722346be5bbb13c_42.png)

这在某些场景下很有用，例如，如果你有一个非常重要的文件只备份了一次，可以通过更新其时间戳，让备份系统在下一次增量备份时再次备份该文件，实现“多次备份”的效果。

![](img/d0660e41793368cd9722346be5bbb13c_44.png)

`touch`命令还支持更精细的控制：
*   `touch -a 文件名`：只更新`atime`。
*   `touch -m 文件名`：只更新`mtime`。

关于`touch`命令的更多选项，可以通过`man touch`命令查看手册。

![](img/d0660e41793368cd9722346be5bbb13c_46.png)

![](img/d0660e41793368cd9722346be5bbb13c_48.png)

## 删除文件与rm命令

下面我们来看如何删除文件和目录，这需要使用`rm`命令。

### 删除文件
直接使用`rm 文件名`会删除指定文件。在红帽企业版Linux（RHEL）中，系统为`rm`命令设置了别名（alias），默认添加了`-i`选项，因此在删除前会进行交互式确认。

### 删除目录
删除目录需要使用`-r`（递归）选项，因为删除目录的本质是进入目录并删除其中的所有文件和子目录。

例如，删除一个名为`test`的目录：
```bash
rm -r test
```
同样，系统会提示确认是否删除目录中的每一个文件，这对于包含大量文件的目录来说非常繁琐。

![](img/d0660e41793368cd9722346be5bbb13c_50.png)

### 强制删除
为了跳过确认提示，可以加上`-f`（强制）选项。组合使用`-r`和`-f`选项可以强制删除目录及其内容，且无需确认。

![](img/d0660e41793368cd9722346be5bbb13c_52.png)

**命令格式**：
```bash
rm -rf 目录名
```

**重要警告**：`rm -rf`命令极其危险，一旦执行，数据将无法通过常规手段恢复。请务必谨慎使用，尤其是在使用通配符（如`*`）时。**绝对不要执行`rm -rf /*`这样的命令**，它会尝试删除根目录下的所有文件，导致系统崩溃和数据永久丢失。

### 关于命令别名
在RHEL等系统中，`rm`、`cp`、`mv`等危险命令通常被设置了`-i`别名以增加安全性。你可以通过`alias`命令查看现有别名。但请注意，并非所有Linux发行版都默认设置这些别名。因此，不能依赖系统的提示，养成良好的操作习惯和备份意识才是根本。
