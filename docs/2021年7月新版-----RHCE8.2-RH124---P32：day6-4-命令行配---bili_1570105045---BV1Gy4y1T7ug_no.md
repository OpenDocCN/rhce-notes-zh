## 上午内容回顾 📝

上一节我们介绍了文件查找、远程传输以及压缩打包工具。本节中，我们来看看网络配置的基础知识。

上午的三个重点内容是：
1.  **`find` 命令的搜索与处理**
2.  **文件传输工具**
3.  **压缩与打包工具**

以下是 `find` 命令结合 `exec` 处理文件的标准格式：
```bash
find [路径] [条件] -exec command {} \;
```
*   `command` 是要执行的命令。
*   `{}` 指代 `find` 找到的每个文件或目录。
*   `\;` 是固定的结束格式。

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_3.png)

以下是三种常用的远程文件传输工具：

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_5.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_7.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_9.png)

*   **SCP**：用于安全复制文件。
    ```bash
    scp [本地文件] [user@host:远程路径]
    scp [user@host:远程文件] [本地路径]
    ```
*   **Rsync**：支持增量传输，效率高。使用前需确保双方机器都已安装 `rsync`。它会比较双方目录的文件差异，只传输不同的文件。使用 `--delete` 选项可以删除目标路径中存在而源路径中没有的文件。
*   **SFTP**：交互式的安全文件传输程序。

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_11.png)

以下是常用的压缩与打包命令：

*   **压缩工具**：
    *   `gzip` / `gunzip`：处理 `.gz` 格式。
    *   `bzip2` / `bunzip2`：处理 `.bz2` 格式。
    *   `xz` / `unxz`：处理 `.xz` 格式。
*   **打包工具**：`tar` 是最常用的打包工具，它也可以结合压缩选项。
    *   `-c`：创建打包文件。
    *   `-x`：解开打包文件。
    *   `-t`：查看打包文件的详细信息。
    *   `-f`：指定档案文件名。
    *   `-r`：向打包文件末尾追加文件。
    *   `-z`：调用 `gzip` 进行压缩或解压。
    *   `-j`：调用 `bzip2` 进行压缩或解压。
    *   `-J`：调用 `xz` 进行压缩或解压。
    *   `-C`：指定解压到的目录。

### 使用 `ip` 命令

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_22.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_24.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_26.png)

`ip` 命令是功能强大的网络配置工具。

*   **查看IP地址**：
    ```bash
    ip addr show
    ```
*   **添加IP别名**：
    ```bash
    ip addr add [IP地址/前缀] dev [设备名] label [设备名:别名]
    ```
*   **删除IP地址**：
    ```bash
    ip addr del [IP地址/前缀] dev [设备名]
    ```
*   **启用/禁用网络连接**：
    ```bash
    ip link set dev [设备名] up
    ip link set dev [设备名] down
    ```

### 网卡配置文件

通过编辑网卡配置文件可以永久修改网络设置。配置文件路径通常为 `/etc/sysconfig/network-scripts/ifcfg-[连接名]`。

以下是配置文件中的主要参数：

*   `TYPE`：网络类型（如 Ethernet）。
*   `NAME`：连接名称。
*   `DEVICE`：设备名称。
*   `BOOTPROTO`：IP获取方式（`none`/`static` 表示静态，`dhcp` 表示动态）。
*   `IPADDR`：IP地址。
*   `NETMASK`：子网掩码（也可用 `PREFIX` 指定前缀长度）。
*   `GATEWAY`：网关地址。
*   `DNS1`、`DNS2`：DNS服务器地址。
*   `ONBOOT`：是否在系统启动时激活此连接。

### 本地主机名解析

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_28.png)

类似于 Windows 的 hosts 文件，Linux 可以通过 `/etc/hosts` 文件进行本地解析。格式为 `IP地址 主机名`。

## 使用 `nmcli` 配置网络 🛠️

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_30.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_32.png)

上一节我们介绍了基础的网络配置方法，本节中我们来看看更先进的网络管理工具 `nmcli`。

`nmcli` 是 NetworkManager 的命令行工具，用于管理网络设置。相关术语：
*   **设备 (Device)**：物理或虚拟网络接口。
*   **连接 (Connection)**：针对设备的配置文件。一个设备可以有多个连接配置，但同一时间只能有一个连接处于激活状态。

其基本命令格式为：`nmcli [选项] 对象 {命令 | help}`。

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_34.png)

### 查看网络信息

常用的查看命令：

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_36.png)

*   查看所有设备状态：`nmcli device status`
*   查看设备详细信息：`nmcli device show`
*   查看所有连接配置：`nmcli connection show`

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_38.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_40.png)

### 通过命令行配置网络连接

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_50.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_52.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_54.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_56.png)

我们可以完全使用 `nmcli` 命令来配置网络，无需手动编辑文件。

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_58.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_60.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_62.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_64.png)

**1. 创建新的连接配置**
```bash
nmcli connection add type ethernet ifname [设备名] con-name [连接名]
```

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_66.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_68.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_70.png)

**2. 修改连接配置（如设置静态IP）**
```bash
nmcli connection modify [连接名] ipv4.addresses [IP地址/前缀] ipv4.method manual ipv4.gateway [网关] ipv4.dns [DNS]
```
*   `ipv4.addresses`：设置 IPv4 地址和子网前缀。
*   `ipv4.method manual`：将获取IP方式改为静态。
*   `ipv4.gateway`：设置网关。
*   `ipv4.dns`：设置 DNS 服务器。

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_72.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_74.png)

**3. 激活连接**
```bash
nmcli connection up [连接名]
```

### 创建 Team 接口

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_103.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_105.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_107.png)

**1. 创建 Team 连接**
```bash
nmcli connection add type team con-name [Team连接名] ifname [Team接口名] config '{"runner": {"name":"loadbalance"}}'
```
*   `config` 参数指定了 Team 的运行模式，例如 `loadbalance`（负载均衡）。

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_109.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_111.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_113.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_115.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_116.png)

**2. 将物理网卡作为端口（port）加入 Team**
```bash
nmcli connection add type team-slave con-name [端口连接名] ifname [物理设备名] master [Team连接名]
```
需要为每个要加入的物理网卡执行此命令。

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_118.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_120.png)

**3. 为 Team 接口配置 IP 地址**
```bash
nmcli connection modify [Team连接名] ipv4.addresses [IP地址/前缀] ipv4.method manual
```

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_122.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_124.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_126.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_128.png)

**4. 启动连接**
```bash
nmcli connection up [Team连接名]
nmcli connection up [端口连接名1]
nmcli connection up [端口连接名2]
```

### Team 的运行模式

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_137.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_139.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_141.png)

![](img/8e98d02ddd1b76fc0455d8b8173ded1d_143.png)

通过在创建时指定不同的 `runner`，可以改变 Team 的工作模式：
*   `loadbalance`：负载均衡。
*   `activebackup`：主备模式，一个接口活跃，另一个备用。
*   `broadcast`：广播所有数据。
*   `roundrobin`：轮询传输数据包。
*   `lacp`：实现 IEEE 802.3ad 链路聚合控制协议。
