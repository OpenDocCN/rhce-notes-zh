## 软链接与硬链接 🔗

![](img/92bf33848bb628d649fd49c0858d977b_19.png)

![](img/92bf33848bb628d649fd49c0858d977b_21.png)

上一节我们介绍了文件和目录的基本操作，本节中我们来看看Linux系统中的链接文件。链接文件类似于Windows系统中的快捷方式，但分为软链接和硬链接两种类型，它们有各自的特点。

### 软链接

![](img/92bf33848bb628d649fd49c0858d977b_31.png)

![](img/92bf33848bb628d649fd49c0858d977b_33.png)

![](img/92bf33848bb628d649fd49c0858d977b_35.png)

软链接（Symbolic Link）类似于Windows的快捷方式。以下是其主要特点：
*   **可以跨分区**：源文件和链接文件可以位于不同的磁盘分区。
*   **可以对目录创建链接**：可以为目录创建软链接。
*   **依赖源文件**：如果删除源文件，软链接将失效（变为“坏链接”）。

![](img/92bf33848bb628d649fd49c0858d977b_37.png)

![](img/92bf33848bb628d649fd49c0858d977b_39.png)

创建软链接的命令格式是 `ln -s`。**关键点在于，必须使用源文件的绝对路径**。

```bash
# 创建软链接的正确方式（使用绝对路径）
ln -s /opt/hello_soft /media/
```

如果创建时未使用绝对路径，链接文件可能会显示为闪烁状态，表示链接不可用。

![](img/92bf33848bb628d649fd49c0858d977b_41.png)

![](img/92bf33848bb628d649fd49c0858d977b_43.png)

软链接文件的权限显示为所有用户都有读写执行权限（`rwxrwxrwx`），但其实际有效的权限取决于源文件的权限。对软链接或源文件的任何修改都会同步到另一方。

### 硬链接

硬链接（Hard Link）与软链接有显著区别，可以理解为给文件创建了一个“同步的副本”。以下是其主要特点：
*   **不能跨分区**：源文件和硬链接必须在同一个文件系统（分区）内。
*   **不能对目录创建**：只能对文件创建硬链接。
*   **独立于源文件**：删除源文件后，硬链接文件仍然可用，内容保持不变。

创建硬链接的命令是 `ln`（不加 `-s` 选项）。

![](img/92bf33848bb628d649fd49c0858d977b_45.png)

```bash
# 创建硬链接
ln /opt/hello_hard /media/
```

![](img/92bf33848bb628d649fd49c0858d977b_47.png)

硬链接文件在外观和属性上与普通文件无异。它与 `cp -p` 命令的复制操作类似，但不同之处在于，硬链接与源文件会保持数据同步，而复制出的文件是独立的。

简单来说，软链接是指向文件路径的“快捷方式”，而硬链接是共享同一存储数据的多个文件入口。

## 获取命令帮助 📖

在Linux中，当我们不熟悉某个命令的用法时，可以查阅其自带的帮助文档。以下是几种常见的方法。

### 使用 `--help` 选项

大多数命令都支持 `--help` 选项，它能列出该命令的基本用法和选项说明。这是最常用、最快捷的获取帮助的方式。

![](img/92bf33848bb628d649fd49c0858d977b_54.png)

```bash
# 查看 ls 命令的帮助
ls --help
```

### 使用 `man` 手册

`man`（manual）命令提供更详细、更官方的命令说明文档，通常包括描述、选项、示例等。

```bash
# 查看 ls 命令的详细手册
man ls
```

![](img/92bf33848bb628d649fd49c0858d977b_56.png)

在 `man` 手册页面中，可以使用类似文本查看器 `less` 的快捷键进行浏览（如空格翻页，`q` 退出）。

### 其他方式

![](img/92bf33848bb628d649fd49c0858d977b_58.png)

![](img/92bf33848bb628d649fd49c0858d977b_60.png)

*   `help` 命令：主要用于查看Shell内置命令的帮助（如 `help cd`）。
*   `info` 命令：提供另一种格式的帮助信息，但如今已较少使用。

对于初学者，如果英文帮助文档阅读困难，更推荐通过互联网搜索（如百度）来获取中文解释和常用用例，这通常更易于理解。

## 系统运行级别 🏃

![](img/92bf33848bb628d649fd49c0858d977b_72.png)

![](img/92bf33848bb628d649fd49c0858d977b_74.png)

Linux系统有不同的运行级别（Runlevel），它定义了系统启动后运行哪些服务。共有7个运行级别（0-6）。

![](img/92bf33848bb628d649fd49c0858d977b_76.png)

以下是各个运行级别的含义：
*   **0**：关机。
*   **1**：单用户模式。仅允许root用户登录，用于系统维护（如忘记root密码时）。所有网络服务通常关闭。
*   **2**：多用户模式（无NFS网络文件共享）。
*   **3**：完全的多用户模式（命令行界面），这是服务器最常用的标准运行级别。
*   **4**：未使用。
*   **5**：图形化界面模式。带GUI的系统默认级别。
*   **6**：重启。

![](img/92bf33848bb628d649fd49c0858d977b_78.png)

![](img/92bf33848bb628d649fd49c0858d977b_80.png)

### 查看与切换运行级别

![](img/92bf33848bb628d649fd49c0858d977b_82.png)

![](img/92bf33848bb628d649fd49c0858d977b_84.png)

![](img/92bf33848bb628d649fd49c0858d977b_86.png)

*   查看当前运行级别：
    ```bash
    runlevel
    ```
    输出如 `N 3`，表示前一个级别是N，当前级别是3。
*   临时切换运行级别（例如切换到单用户模式）：
    ```bash
    init 1
    ```
    **注意**：切换到级别1或6（重启）可能导致当前会话断开。

![](img/92bf33848bb628d649fd49c0858d977b_88.png)

![](img/92bf33848bb628d649fd49c0858d977b_90.png)

![](img/92bf33848bb628d649fd49c0858d977b_92.png)

![](img/92bf33848bb628d649fd49c0858d977b_94.png)

### 设置默认运行级别

系统默认运行级别在文件 `/etc/inittab` 中定义（新版本系统可能不同）。更常用的查看和设置命令是：
*   查看默认级别：
    ```bash
    systemctl get-default
    ```
*   设置默认级别（例如设置为命令行模式）：
    ```bash
    systemctl set-default multi-user.target  # 对应级别3
    ```
*   设置默认级别（例如设置为图形界面模式）：
    ```bash
    systemctl set-default graphical.target   # 对应级别5
    ```
    请注意，如果系统未安装图形界面，设置为级别5无效。

## 关机与重启 ⏻

![](img/92bf33848bb628d649fd49c0858d977b_106.png)

![](img/92bf33848bb628d649fd49c0858d977b_108.png)

![](img/92bf33848bb628d649fd49c0858d977b_110.png)

![](img/92bf33848bb628d649fd49c0858d977b_112.png)

在命令行下管理服务器，需要掌握关机与重启的命令。

以下是常用的命令：
*   **`poweroff`**：立即关闭系统。（**推荐记忆**）
*   **`reboot`**：立即重启系统。（**推荐记忆**）
*   **`shutdown`**：功能强大的关机命令。
    ```bash
    shutdown -h now      # 立即关机
    shutdown -h +10      # 10分钟后关机
    shutdown -r 22:30    # 在22:30重启
    ```
*   **`init 0`**：切换到运行级别0（关机）。
*   **`halt`**：停止系统运行，通常也需要接着手动关机。

![](img/92bf33848bb628d649fd49c0858d977b_114.png)

对于日常使用，记住 `poweroff` 和 `reboot` 这两个简单直接的命令即可。
