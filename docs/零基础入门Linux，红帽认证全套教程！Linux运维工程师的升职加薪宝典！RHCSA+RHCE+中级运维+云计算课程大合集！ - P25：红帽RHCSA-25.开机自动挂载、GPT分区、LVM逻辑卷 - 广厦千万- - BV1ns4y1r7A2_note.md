## 回顾与过渡：磁盘管理基础步骤

![](img/87303f98e1993086fff1f45b1681bf7d_13.png)

![](img/87303f98e1993086fff1f45b1681bf7d_15.png)

上一节我们介绍了使用`mount`命令进行临时挂载。本节中，我们来看看如何让挂载永久生效。

对硬盘进行分区并存储数据，通常需要经历以下几个步骤：
1.  **添加硬盘**：将物理硬盘连接到服务器。
2.  **识别硬盘**：使用 `lsblk` 命令查看系统是否识别到新硬盘。
3.  **分区**：使用分区工具（如 `fdisk` 或 `gdisk`）对硬盘进行分区。
4.  **格式化**：使用 `mkfs` 命令为分区创建文件系统（如 `xfs`）。
5.  **挂载**：创建挂载点目录，并使用 `mount` 命令将分区挂载到目录。
6.  **查看**：使用 `df -hT` 命令查看分区使用情况和文件系统类型。

![](img/87303f98e1993086fff1f45b1681bf7d_17.png)

![](img/87303f98e1993086fff1f45b1681bf7d_18.png)

![](img/87303f98e1993086fff1f45b1681bf7d_20.png)

`df -hT` 命令可以列出所有已挂载分区的详细信息，包括文件系统类型。例如，普通数据分区通常使用 `xfs` 类型，而 `/dev` 目录用于存放硬件设备，有其特殊的文件系统类型。

## 开机自动挂载

我们之前使用的 `mount` 命令属于临时挂载。其优点是立即生效，但缺点是服务器重启后，挂载就会失效。为了实现重启后自动挂载，我们需要配置开机自动挂载。

开机自动挂载通过修改 `/etc/fstab` 配置文件实现。系统启动时会自动读取此文件，并将其中指定的文件系统挂载到指定目录。

![](img/87303f98e1993086fff1f45b1681bf7d_24.png)

![](img/87303f98e1993086fff1f45b1681bf7d_26.png)

![](img/87303f98e1993086fff1f45b1681bf7d_28.png)

以下是配置 `/etc/fstab` 文件的步骤和格式详解。

![](img/87303f98e1993086fff1f45b1681bf7d_30.png)

![](img/87303f98e1993086fff1f45b1681bf7d_32.png)

![](img/87303f98e1993086fff1f45b1681bf7d_34.png)

打开配置文件：
```bash
vi /etc/fstab
```

![](img/87303f98e1993086fff1f45b1681bf7d_36.png)

配置文件的每一行代表一个需要自动挂载的文件系统，每行共有6列，以空格分隔。

![](img/87303f98e1993086fff1f45b1681bf7d_38.png)

![](img/87303f98e1993086fff1f45b1681bf7d_40.png)

**配置文件各列含义：**
1.  **设备路径**：要挂载的分区设备文件，例如 `/dev/sdb1`。
2.  **挂载点**：分区要挂载到的目录路径，例如 `/db_backup`。
3.  **文件系统类型**：分区的文件系统类型，例如 `xfs`。
4.  **挂载参数**：挂载时使用的选项。`defaults` 代表使用默认参数（包含读写权限、异步写入等），通常直接填写 `defaults` 即可。
5.  **备份标记**：`dump` 备份工具是否对此分区进行备份。`0` 表示不备份。
6.  **开机检查顺序**：系统启动时 `fsck` 检查磁盘的顺序。`0` 表示不检查。根分区 `/` 应设置为 `1`，其他分区通常设为 `0` 或 `2`。

![](img/87303f98e1993086fff1f45b1681bf7d_42.png)

![](img/87303f98e1993086fff1f45b1681bf7d_44.png)

例如，添加以下行以实现 `/dev/sdb1` 的开机自动挂载：
```
/dev/sdb1 /db_backup xfs defaults 0 0
```

![](img/87303f98e1993086fff1f45b1681bf7d_46.png)

![](img/87303f98e1993086fff1f45b1681bf7d_48.png)

配置完成后，可以使用 `mount -a` 命令让系统立即读取 `/etc/fstab` 文件，并挂载所有未挂载的设备。此命令常用于测试配置是否正确。

![](img/87303f98e1993086fff1f45b1681bf7d_50.png)

**`mount` 命令常用选项：**
*   `mount -a`：根据 `/etc/fstab` 配置，挂载所有未挂载的设备。
*   `mount -o`：指定额外的挂载参数（如 `ro` 只读）。此选项在特殊维护场景下使用。

![](img/87303f98e1993086fff1f45b1681bf7d_52.png)

![](img/87303f98e1993086fff1f45b1681bf7d_53.png)

## GPT分区格式

![](img/87303f98e1993086fff1f45b1681bf7d_58.png)

之前我们学习了MBR分区格式，它最多支持4个主分区或3个主分区加1个扩展分区，且最大支持2TB磁盘。对于超过2TB的硬盘或需要更多分区的场景，需要使用GPT分区格式。

![](img/87303f98e1993086fff1f45b1681bf7d_60.png)

![](img/87303f98e1993086fff1f45b1681bf7d_61.png)

![](img/87303f98e1993086fff1f45b1681bf7d_63.png)

GPT分区格式支持最多128个主分区，且支持远超2TB的磁盘容量。

![](img/87303f98e1993086fff1f45b1681bf7d_65.png)

以下是使用 `gdisk` 命令对硬盘进行GPT分区的步骤。

![](img/87303f98e1993086fff1f45b1681bf7d_67.png)

![](img/87303f98e1993086fff1f45b1681bf7d_69.png)

首先，确保系统已安装 `gdisk` 工具包：
```bash
yum -y install gdisk
```

然后对目标磁盘进行分区：
```bash
gdisk /dev/sdc
```

![](img/87303f98e1993086fff1f45b1681bf7d_71.png)

![](img/87303f98e1993086fff1f45b1681bf7d_72.png)

进入交互界面后，常用命令如下：
*   `n`：创建新分区。
*   `p`：显示当前分区表。
*   `d`：删除分区。
*   `w`：保存分区表并退出。
*   `q`：不保存更改并退出。
*   `?`：显示帮助信息。

分区过程与 `fdisk` 类似：输入 `n` 创建分区，指定分区编号、起始扇区（通常默认）、分区大小（如 `+10G`），最后选择分区类型（默认 `Linux filesystem` 即可）。使用 `p` 查看分区信息，确认无误后输入 `w` 保存并退出。

分区完成后，后续的格式化、挂载和配置自动挂载步骤与MBR分区完全相同，例如：
```bash
mkfs.xfs /dev/sdc1
mkdir /web_backup
# 将 /dev/sdc1 /web_backup xfs defaults 0 0 添加到 /etc/fstab
mount -a
```

## LVM逻辑卷管理

![](img/87303f98e1993086fff1f45b1681bf7d_86.png)

### 逻辑卷概述

物理分区一旦创建，其大小固定，无法灵活扩展。当分区空间用尽时，只能创建新分区并迁移数据，导致数据存储分散，管理不便。

LVM（Logical Volume Manager，逻辑卷管理）通过抽象层解决了这个问题。它的核心功能是**动态扩容**。

### LVM核心概念与优势

![](img/87303f98e1993086fff1f45b1681bf7d_88.png)

![](img/87303f98e1993086fff1f45b1681bf7d_90.png)

LVM将底层的多个**物理卷**整合成一个大的**卷组**，然后从卷组中划分出**逻辑卷**供系统使用。

**工作流程简述：**
1.  将物理分区创建为**物理卷**。
2.  将一个或多个物理卷加入**卷组**，形成一个统一的存储池。
3.  从卷组中划分出**逻辑卷**。
4.  将逻辑卷格式化并挂载使用。

![](img/87303f98e1993086fff1f45b1681bf7d_92.png)

**优势：**
*   **空间动态扩展**：当逻辑卷空间不足时，可以从卷组的剩余空间中进行扩展，而无需迁移数据或卸载文件系统。
*   **存储池管理**：管理员可以专注于管理卷组的总空间，而非单个物理硬盘。
*   **数据集中存储**：避免了因分区空间用尽而导致的数据分散存储问题。

![](img/87303f98e1993086fff1f45b1681bf7d_94.png)

### 逻辑卷与物理分区对比

假设有四个100GB的物理分区。
*   **传统方式**：每个分区独立挂载。一个分区存满后，需使用另一个分区，数据分散。
*   **LVM方式**：将四个分区加入一个卷组，获得400GB的总空间。从中创建一个200GB的逻辑卷并挂载使用。当此逻辑卷空间用尽时，可以直接从卷组剩余的200GB空间中扩容，所有数据仍集中存储在同一挂载点。

![](img/87303f98e1993086fff1f45b1681bf7d_96.png)

![](img/87303f98e1993086fff1f45b1681bf7d_97.png)

**请注意**：LVM主要提供灵活性，并不直接提供数据冗余保护。若底层物理硬盘损坏，数据可能会丢失。数据安全需通过RAID磁盘阵列或定期备份来保障。

## 总结

本节课我们一起学习了Linux磁盘管理的三个重要进阶主题。
1.  **开机自动挂载**：通过编辑 `/etc/fstab` 配置文件，并配合 `mount -a` 命令，可以实现文件系统在系统启动时自动挂载。
2.  **GPT分区**：使用 `gdisk` 工具管理GPT分区格式，它突破了MBR在分区数量和磁盘容量上的限制。
3.  **LVM逻辑卷**：理解了LVM的基本概念和优势。LVM通过物理卷、卷组、逻辑卷的层级管理，实现了存储空间的动态扩展，为灵活高效的存储管理奠定了基础。

![](img/87303f98e1993086fff1f45b1681bf7d_99.png)

掌握这些技能，将使你能够更从容地规划和管理服务器存储资源。
