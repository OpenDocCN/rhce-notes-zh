在本节课中，我们将要学习DNS服务器的基本概念和工作原理。DNS是互联网的“电话簿”，负责将我们熟悉的域名（如 `www.baidu.com`）转换为计算机能够识别的IP地址（如 `180.149.131.98`）。理解DNS是构建和维护网络服务的基础。

## 什么是DNS？

![](img/b4db8c342ba3e183faa1f22f46767df4_10.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_11.png)

DNS是 **Domain Name System（域名系统）** 的缩写。它的核心功能是在域名和IP地址之间进行双向解析。

![](img/b4db8c342ba3e183faa1f22f46767df4_13.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_14.png)

*   **正向解析**：将域名转换为IP地址。例如，输入 `www.baidu.com`，DNS会返回对应的IP地址。
*   **反向解析**：将IP地址转换为域名。

![](img/b4db8c342ba3e183faa1f22f46767df4_16.png)

计算机在网络中通信时，只识别IP地址。但IP地址（如 `202.99.100.68`）难以记忆，因此我们使用更直观的域名。DNS服务器的作用就是完成这个转换工作。

![](img/b4db8c342ba3e183faa1f22f46767df4_18.png)

此外，许多网站出于安全考虑，不允许用户直接通过IP地址访问，必须使用域名。这使得DNS服务在生产环境中至关重要。

![](img/b4db8c342ba3e183faa1f22f46767df4_20.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_22.png)

## 域名的结构

![](img/b4db8c342ba3e183faa1f22f46767df4_24.png)

域名采用分层级的树状结构，类似于文件系统的目录。一个完整的域名（FQDN）从右向左，层级逐渐降低。

![](img/b4db8c342ba3e183faa1f22f46767df4_25.png)

例如，`www.baidu.com.` 的完整结构是：
*   `.` （根域）
*   `com.` （顶级域/一级域）
*   `baidu.com.` （二级域）
*   `www.baidu.com.` （主机名）

![](img/b4db8c342ba3e183faa1f22f46767df4_27.png)

我们通常省略末尾的根域符号 `.`。这种层级结构意味着DNS信息是分布式存储的，没有任何一台服务器能存储全互联网的所有域名信息。

![](img/b4db8c342ba3e183faa1f22f46767df4_29.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_30.png)

## 根域服务器

![](img/b4db8c342ba3e183faa1f22f46767df4_32.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_34.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_35.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_37.png)

根域服务器位于这个层级结构的顶端，存储着所有顶级域（如 `.com`， `.cn`， `.net`）的DNS服务器信息。

![](img/b4db8c342ba3e183faa1f22f46767df4_39.png)

*   全球共有13组逻辑根域服务器（编号A-M），实际通过任播技术在全球部署了数百个物理镜像。
*   这些服务器主要分布在美国，其他国家（包括中国）部署的是其镜像服务器，用于分担查询压力和提高访问速度。
*   根域服务器是互联网DNS解析的基石。

![](img/b4db8c342ba3e183faa1f22f46767df4_41.png)

## DNS查询过程

![](img/b4db8c342ba3e183faa1f22f46767df4_43.png)

DNS查询是一个分层、协作的过程。为了便于理解，我们将其分为内网查询和外网迭代查询两部分。

![](img/b4db8c342ba3e183faa1f22f46767df4_45.png)

上一节我们介绍了域名的层级结构，本节中我们来看看一次完整的DNS查询是如何在这些层级中穿梭完成的。

![](img/b4db8c342ba3e183faa1f22f46767df4_46.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_48.png)

### 客户端本地查询

![](img/b4db8c342ba3e183faa1f22f46767df4_50.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_51.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_53.png)

当一台计算机（客户端）需要访问一个域名时，它会按以下顺序进行查询：

![](img/b4db8c342ba3e183faa1f22f46767df4_54.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_55.png)

1.  **查询本地Hosts文件**：检查操作系统本地的 `hosts` 文件（如 `/etc/hosts`）。如果文件中存在该域名的记录，则直接使用，查询结束。
2.  **查询本地DNS缓存**：检查操作系统是否有该域名的缓存记录。如果有，则直接使用。
3.  **向配置的DNS服务器发起查询**：如果上述两步都没有结果，客户端会向网络配置中指定的DNS服务器（通常是企业内网的缓存DNS服务器）发起查询请求。

### 企业内网DNS架构

![](img/b4db8c342ba3e183faa1f22f46767df4_57.png)

在企业生产环境中，DNS服务通常采用分层架构以提升性能和可靠性：

![](img/b4db8c342ba3e183faa1f22f46767df4_59.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_60.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_61.png)

*   **主DNS服务器**：存储企业内网所有主机域名与IP地址的映射记录（如 `server1.tech.yzm.com -> 192.168.1.10`）。
*   **缓存DNS服务器**：不存储原始记录，其数据从主DNS同步或通过缓存外网查询结果获得。客户端通常被指向这些缓存服务器，以分散主服务器的压力。

### 完整的递归与迭代查询

以下是客户端首次访问一个外网域名（如 `www.baidu.com`）时，查询请求的完整旅程：

```
客户端 -> 缓存DNS -> 主DNS -> 运营商DNS -> 根域服务器 -> 顶级域服务器 -> 二级域服务器 -> ... -> 返回结果
```

1.  **递归查询（客户端与本地DNS之间）**：客户端向本地DNS服务器（缓存DNS）提问：“`www.baidu.com` 的IP是多少？” 并等待最终答案。本地DNS服务器有责任去找到这个答案并返回给客户端。
2.  **迭代查询（本地DNS与外部DNS之间）**：本地DNS服务器为了找到答案，会从根开始一层层向上级DNS服务器询问。
    *   它先问根域服务器：“`www.baidu.com` 的IP是多少？”
    *   根域服务器回答：“我不知道，但我知道管理 `.com` 的服务器地址，你去问它们。”
    *   本地DNS服务器再去问 `.com` 顶级域服务器。
    *   `.com` 服务器回答：“我不知道，但我知道管理 `baidu.com` 的服务器地址，你去问它们。”
    *   本地DNS服务器最后去问 `baidu.com` 的权威服务器。
    *   `baidu.com` 的权威服务器给出最终答案：“`www.baidu.com` 的IP是 `180.149.131.98`。”
3.  **结果返回与缓存**：本地DNS服务器将最终IP地址返回给客户端。同时，查询路径上的每一台DNS服务器（缓存DNS、运营商DNS等）都会将这条记录缓存一段时间。当同一内网的其他客户端再次查询相同域名时，缓存DNS服务器可以直接回复，极大地提高了效率。

**核心概念简化**：
*   **递归查询**：“你帮我问到底，然后把最终结果给我。”
*   **迭代查询**：“我只告诉你下一个该问谁，你自己去问。”

![](img/b4db8c342ba3e183faa1f22f46767df4_63.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_64.png)

![](img/b4db8c342ba3e183faa1f22f46767df4_65.png)

对于网络管理员而言，重点在于管理和维护**企业内网的DNS架构**（主DNS和缓存DNS），而外网的迭代查询过程由公共互联网基础设施自动完成。

## 总结

本节课我们一起学习了DNS的基础知识：
1.  **DNS的作用**：实现域名与IP地址的双向解析，是互联网的关键服务。
2.  **域名的结构**：采用树状的层级结构（根域. -> 顶级域 -> 二级域 -> 主机名），信息分布式存储。
3.  **根域服务器**：全球13组，是DNS查询的起点。
4.  **DNS查询原理**：结合了客户端的**递归查询**和DNS服务器间的**迭代查询**。查询结果会在路径上的服务器中缓存，以提升后续查询速度。
5.  **企业DNS架构**：通常采用主DNS服务器加多台缓存DNS服务器的模式，以保证内网域名解析的高效和安全。

![](img/b4db8c342ba3e183faa1f22f46767df4_67.png)

理解这些概念是后续实际搭建和配置DNS服务器的基础。
