## wc统计命令
`wc`命令用于统计文件中的行数、单词数和字节数。它在分析日志文件或检查配置文件大小时非常有用。

![](img/d749696050d7b6770e4874d0ba87f287_9.png)

### 基本用法
命令的基本格式是 `wc [选项] 文件名`。如果不加任何选项，`wc`会同时显示行数、单词数和字节数。

![](img/d749696050d7b6770e4874d0ba87f287_11.png)

![](img/d749696050d7b6770e4874d0ba87f287_13.png)

**示例代码：**
```bash
wc /etc/passwd
```
执行上述命令会输出类似 `43 89 2657 /etc/passwd` 的结果，分别代表行数、单词数和字节数。

### 常用选项
以下是`wc`命令最常用的几个选项：

*   **`-l`**：仅统计文件的行数。这是最常用的选项。
*   **`-w`**：仅统计文件的单词数。单词是由空格或换行符分隔的字符串。
*   **`-c`**：仅统计文件的字节数。

**示例代码：**
```bash
wc -l /etc/passwd
```
这个命令只会输出文件`/etc/passwd`的总行数。

上一节我们介绍了`wc`命令，它擅长对单个文件进行统计。本节中我们来看看如何将多个命令串联起来，实现更复杂的功能。

## 管道符 `|`
管道符 `|` 是一个强大的工具，它可以将前一个命令的标准输出，作为后一个命令的标准输入。这意味着你可以将多个简单的命令组合起来，完成复杂的文本处理任务。

### 管道符的功能
它的核心功能是连接命令，让数据在命令之间“流动”。例如，你想查看一个文件的前10行，但只关心最后5行。

**示例代码：**
```bash
head -10 /etc/passwd | tail -5
```
这个命令组合首先用`head -10`获取文件的前10行，然后通过管道`|`将这10行结果传递给`tail -5`命令，`tail -5`则显示接收到的输入的最后5行。

### 管道的链式调用
管道可以无限连接，只要你有需求。你可以将多个处理步骤串联起来。

![](img/d749696050d7b6770e4874d0ba87f287_19.png)

**示例代码：**
```bash
cat -n /etc/passwd | head -10 | tail -5 | wc -l
```
这个命令链的执行过程是：
1.  `cat -n` 显示文件内容并加上行号。
2.  管道将结果传给 `head -10`，取前10行。
3.  再管道传给 `tail -5`，取这10行中的最后5行。
4.  最后管道传给 `wc -l`，统计这5行的行数（结果自然是5）。

![](img/d749696050d7b6770e4874d0ba87f287_21.png)

管道符让我们能够灵活地处理数据流。接下来，我们将学习如何将命令的输出结果保存到文件中，这就是重定向操作。

## 重定向操作
重定向操作符用于改变命令输入和输出的默认方向（默认是屏幕）。最常用的是输出重定向，它可以将命令的执行结果保存到文件中。

![](img/d749696050d7b6770e4874d0ba87f287_27.png)

![](img/d749696050d7b6770e4874d0ba87f287_29.png)

### 输出重定向：`>` 和 `>>`
以下是两种主要的输出重定向方式：

![](img/d749696050d7b6770e4874d0ba87f287_31.png)

*   **`>` （覆盖重定向）**：将命令的输出写入指定文件。如果文件已存在，则会**清空原有内容**再写入；如果文件不存在，则会创建该文件。
*   **`>>` （追加重定向）**：将命令的输出**追加**到指定文件的末尾。不会清空原有内容。

![](img/d749696050d7b6770e4874d0ba87f287_33.png)

![](img/d749696050d7b6770e4874d0ba87f287_35.png)

**示例代码（覆盖）：**
```bash
ifconfig ens33 | head -2 > net_ens33.txt
```
这条命令将`ifconfig ens33`结果的前两行，保存（覆盖）到`net_ens33.txt`文件中。

![](img/d749696050d7b6770e4874d0ba87f287_37.png)

![](img/d749696050d7b6770e4874d0ba87f287_39.png)

**示例代码（追加）：**
```bash
echo “New line of text” >> net_ens33.txt
```
这条命令会将字符串“New line of text”追加到`net_ens33.txt`文件的末尾。

![](img/d749696050d7b6770e4874d0ba87f287_41.png)

### 错误输出重定向：`2>` 和 `2>>`
命令执行可能产生正确输出（标准输出）和错误输出（标准错误）。默认情况下，两者都显示在屏幕上。`2>`和`2>>`专门用于收集错误信息。

*   **`2>`**：将命令的**错误输出**覆盖写入文件。
*   **`2>>`**：将命令的**错误输出**追加写入文件。

![](img/d749696050d7b6770e4874d0ba87f287_43.png)

**示例代码：**
```bash
cat /etc/nonexistent_file 2> error.log
```
由于`/etc/nonexistent_file`文件不存在，`cat`命令会报错。这个错误信息不会被打印到屏幕，而是被保存到`error.log`文件中。

### 合并正确与错误输出：`&>` 和 `&>>`
有时我们需要同时收集正确和错误的输出。

![](img/d749696050d7b6770e4874d0ba87f287_45.png)

![](img/d749696050d7b6770e4874d0ba87f287_47.png)

*   **`&>`**：将命令的**正确输出和错误输出都**覆盖写入同一个文件。
*   **`&>>`**：将命令的**正确输出和错误输出都**追加写入同一个文件。

**示例代码：**
```bash
ls /etc/passwd /etc/nonexistent_file &> all_output.log
```
这条命令会列出存在的`/etc/passwd`和不存在的文件信息，无论对错，所有输出都会保存到`all_output.log`。

![](img/d749696050d7b6770e4874d0ba87f287_49.png)

![](img/d749696050d7b6770e4874d0ba87f287_51.png)

## echo 与 sleep 命令

### echo 命令
`echo`命令用于在终端输出指定的字符串或变量值。它在脚本中常用于显示信息、生成文件内容或调试。

**基本用法：**
```bash
echo “Hello, World!”
```
这行命令会在屏幕上输出 `Hello, World!`。

**结合重定向创建或修改文件：**
`echo`配合重定向可以快速向文件写入内容，这是一个非常实用的技巧。

![](img/d749696050d7b6770e4874d0ba87f287_57.png)

![](img/d749696050d7b6770e4874d0ba87f287_59.png)

**示例代码：**
```bash
echo “new_hostname” > /etc/hostname
```
这条命令会将字符串“new_hostname”写入`/etc/hostname`文件，从而快速修改系统主机名（注意需要权限）。如果只使用 `> file` 而不跟任何字符串，则会创建一个空文件或清空已有文件。

![](img/d749696050d7b6770e4874d0ba87f287_61.png)

### sleep 命令
`sleep`命令用于使当前进程暂停（睡眠）一段时间。它在脚本中用于等待操作完成、控制任务节奏或降低CPU使用率。

![](img/d749696050d7b6770e4874d0ba87f287_63.png)

**基本用法：**
`sleep`后跟数字和时间单位。
*   `s`：秒（默认）
*   `m`：分钟
*   `h`：小时
*   `d`：天

![](img/d749696050d7b6770e4874d0ba87f287_65.png)

**示例代码：**
```bash
sleep 5
```
这条命令会让终端等待5秒后才显示下一个提示符。

![](img/d749696050d7b6770e4874d0ba87f287_67.png)

![](img/d749696050d7b6770e4874d0ba87f287_69.png)

**应用场景：**
在循环脚本中，如果不使用`sleep`，命令会以CPU最高速度运行，可能导致CPU占用率瞬间达到100%。加入`sleep`可以让CPU在每次循环间有短暂的休息。

![](img/d749696050d7b6770e4874d0ba87f287_71.png)

**示例代码：**
```bash
while true; do echo “CPU is working”; sleep 0.1; done
```
这个循环会每秒输出约10次信息，通过`sleep 0.1`（休眠0.1秒）有效控制了循环速度，避免了CPU持续满载。

![](img/d749696050d7b6770e4874d0ba87f287_73.png)

![](img/d749696050d7b6770e4874d0ba87f287_75.png)

## 总结
本节课中我们一起学习了Linux中几个核心的文本处理和流程控制工具。
1.  **`wc`命令**：用于统计文件的行数、单词数和字节数，是分析文本文件的基础工具。
2.  **管道符 `|`**：功能强大，能够将多个命令连接起来，使前一个命令的输出作为后一个命令的输入，实现复杂的数据流处理。
3.  **重定向操作**：包括`>`、`>>`、`2>`、`&>`等，它们能灵活地将命令输出导向文件，方便日志记录和结果保存。
4.  **`echo`命令**：主要用于输出文本，常与重定向结合来快速创建或修改文件。
5.  **`sleep`命令**：用于在脚本或命令中引入延迟，有助于控制程序执行节奏和降低系统负载。
