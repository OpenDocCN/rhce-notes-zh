## 官方描述与基本概念

`grep`命令用于在文件中搜索匹配指定模式的文本行。根据官方手册的描述，其基本功能是：**搜索每个文件中匹配给定模式的行**。

![](img/463af17a557015f4a6a83738656a53af_9.png)

如果给定的文件名是 `-`，则表示从**标准输入**读取数据。标准输入通常指键盘输入，而标准输出通常指显示器。

![](img/463af17a557015f4a6a83738656a53af_11.png)

![](img/463af17a557015f4a6a83738656a53af_13.png)

`grep`命令还有两个变种：`egrep` 和 `fgrep`。它们分别等价于 `grep -E` 和 `grep -F`，主要用于处理扩展正则表达式和固定字符串，但官方不推荐直接使用这两个变种命令。

## 基础用法：在文件中搜索

上一节我们介绍了`grep`的基本概念，本节中我们来看看它的基础用法。其基本命令格式为：
```bash
grep [选项] “搜索模式” 文件名
```

![](img/463af17a557015f4a6a83738656a53af_19.png)

例如，要在文件 `source.txt` 中搜索字符串 “src”，可以执行：
```bash
grep “src” source.txt
```
命令执行后，会输出所有包含“src”字符串的行，并且匹配的文本会默认高亮显示。

## 从标准输入读取数据

`grep`命令可以处理来自标准输入的数据，这在处理管道数据流时非常有用。

![](img/463af17a557015f4a6a83738656a53af_26.png)

要使用标准输入，只需将文件名参数替换为 `-`。例如：
```bash
grep “src” -
```
执行此命令后，终端会等待你输入文本。输入完成后按回车，`grep`会从你输入的内容中搜索“src”并输出匹配的行。如果没有匹配项，则不会有任何输出。

## 递归搜索目录

在日常工作中，我们经常需要在多层目录中搜索文件内容。`grep`命令配合特定参数可以实现递归搜索。

![](img/463af17a557015f4a6a83738656a53af_36.png)

以下是实现递归搜索时常用的参数组合介绍：
*   **-r** 或 **-R**：递归搜索指定目录下的所有文件。`-R` 会同时搜索符号链接指向的文件，而 `-r` 则不会。
*   **-l**：仅列出包含匹配模式的文件路径，而不显示具体的匹配行。这在需要定位文件时非常有用。

一个常用的组合命令是：
```bash
grep -rl “搜索模式” 目录路径
```
这个命令会递归搜索指定目录，并列出所有包含“搜索模式”的文件路径。

## 实时过滤数据流

`grep`命令一个非常强大的功能是实时过滤数据流，这在监控日志或网络流量时极其有用。

例如，使用 `tail -f` 命令实时查看日志文件时，信息滚动非常快。我们可以通过管道将输出传递给 `grep` 进行过滤，只显示我们关心的内容。
```bash
tail -f /var/log/nginx/access.log | grep “ERROR”
```
这样，屏幕上就只会显示包含“ERROR”关键词的日志行，极大地简化了问题排查过程。

同样，对于快速滚动的网络连接信息，也可以使用类似的方法：
```bash
tcpdump -i eth0 | grep “192.168.1.100”
```

## 高亮显示搜索结果

默认情况下，`grep`会高亮显示匹配的文本。我们也可以自定义高亮的颜色和样式，使结果更醒目。

`grep`通过 `--color` 参数控制高亮。其值由三部分组成，格式通常为 `--color=‘A;B;C’`：
*   **A (属性)**：控制显示效果，如 `01`（高亮）、`04`（下划线）、`05`（闪烁）等。
*   **B (前景色/字体颜色)**：控制匹配文本的颜色，范围是 `30-37`。
*   **C (背景色)**：控制匹配文本的背景色，范围是 `40-47`。

例如，要设置紫色高亮显示，可以执行：
```bash
grep --color=‘01;35’ “pattern” file.txt
```
你也可以通过设置环境变量 `GREP_COLOR` 来永久改变默认高亮颜色。

## 结合正则表达式进行高级搜索

![](img/463af17a557015f4a6a83738656a53af_53.png)

![](img/463af17a557015f4a6a83738656a53af_55.png)

`grep`命令的强大之处在于它能与正则表达式完美结合，进行复杂模式的匹配。这需要使用 `-E` 选项来启用扩展正则表达式。

![](img/463af17a557015f4a6a83738656a53af_57.png)

![](img/463af17a557015f4a6a83738656a53af_59.png)

一个典型的应用场景是从大量文本中提取符合特定格式的字符串，例如IP地址。

![](img/463af17a557015f4a6a83738656a53af_61.png)

以下是一个用于匹配IP地址的正则表达式示例，结合 `-o` 选项可以只输出匹配到的部分：
```bash
grep -E -o “([0-9]{1,3}\.){3}[0-9]{1,3}” file.txt
```
这个命令会从 `file.txt` 中找出所有格式为 `xxx.xxx.xxx.xxx` 的IP地址并单独输出。

![](img/463af17a557015f4a6a83738656a53af_63.png)

在实际运维中，你可以将此命令与其他命令结合，例如从系统网络配置或日志中快速提取所有IP：
```bash
ip a | grep -E -o “([0-9]{1,3}\.){3}[0-9]{1,3}”
```
