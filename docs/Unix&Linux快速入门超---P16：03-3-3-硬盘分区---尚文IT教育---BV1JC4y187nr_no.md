# Unix&Linux快速入门超详细教程-7天通关RHCE：P16：03-3-3 硬盘分区演示 💾

在本节课中，我们将学习如何在安装Linux系统时进行硬盘分区。这是系统安装过程中至关重要的一步，我们将详细演示如何手动配置分区，包括引导分区、根分区、交换分区以及自定义分区。

## 安装目标选择

上一节我们介绍了安装前的准备工作，本节中我们来看看如何选择安装目标并进行分区。

在安装界面中，找到“Installation Destination”选项。点击进入后，会看到可用的磁盘列表。

*   **本地标准磁盘**：通常标识为 `sda`、`sdb` 等。例如，`sda` 就是我们为虚拟机创建的20GB虚拟磁盘，类型为“Local Standard Disks”。
*   **专用与网络磁盘**：这部分包含来自外部存储设备的磁盘，需要额外配置才能识别和使用。

以下是几种常见的外部磁盘类型：

![](img/8878706ffc89a43af893ddfda7b59411_3.png)

*   **Multipath Devices**：多路径设备。这是一种高可用性技术，允许服务器通过多条物理路径访问同一个存储设备。即使一条路径出现故障，数据仍可通过其他路径传输。实现此功能需要在操作系统中安装 `mpio` 软件。
*   **SAN Devices**：存储区域网络设备。这是一种通过网络（通常是光纤通道）提供块级存储访问的技术。
*   **NVMe-oF Devices**：基于NVMe协议的网络存储设备，能提供极高的性能。

简单来说，Linux可识别的磁盘分为两大类：**本地磁盘**和**外部磁盘**。外部磁盘可以通过多种协议连接，例如光纤通道、基于以太网的FC协议或iSCSI协议。

![](img/8878706ffc89a43af893ddfda7b59411_5.png)

## 分区配置方式

![](img/8878706ffc89a43af893ddfda7b59411_7.png)

选择好目标磁盘后，接下来需要配置分区方式。

系统提供了两种主要的分区配置方式：

1.  **自动配置分区**：系统会根据磁盘大小自动创建分区方案。
2.  **手动配置分区**：允许用户完全自定义分区方案，这是我们需要重点掌握的部分。

默认选项是“自动配置分区”。如果选择此选项并点击完成，系统会生成一个默认的分区布局（例如，可能分配1GB给 `/boot`，其余空间给 `/`）。但这通常不符合我们的实际需求。

因此，我们选择“手动配置分区”。点击“我要配置分区”后，会进入分区布局界面。

## 手动分区演示

![](img/8878706ffc89a43af893ddfda7b59411_11.png)

现在，我们开始手动创建分区。点击“+”号来添加新的挂载点。

![](img/8878706ffc89a43af893ddfda7b59411_13.png)

以下是创建各个分区的步骤：

![](img/8878706ffc89a43af893ddfda7b59411_15.png)

**1. 创建 `/boot` 引导分区**

*   **挂载点**：选择 `/boot`。
*   **期望容量**：输入 `200M`。
*   **设备类型**：系统默认会将其设为“标准分区”。
*   **文件系统**：默认可能是 `xfs`。对于RHEL 7，我们通常选择 `ext4`。因此，将文件系统类型改为 `ext4`。

**2. 创建 `swap` 交换分区**

![](img/8878706ffc89a43af893ddfda7b59411_17.png)

![](img/8878706ffc89a43af893ddfda7b59411_19.png)

*   **挂载点**：选择 `swap`。
*   **期望容量**：通常设置为物理内存的1.5到2倍。假设物理内存为4GB，我们可以设置为 `6G`。
*   **设备类型**：系统会自动将其配置为 **LVM** 逻辑卷。你可以看到它属于一个名为 `rhel` 的卷组，逻辑卷名称为 `swap`。卷组和逻辑卷的名称都可以自定义修改。

**3. 创建 `/` 根分区**

![](img/8878706ffc89a43af893ddfda7b59411_21.png)

![](img/8878706ffc89a43af893ddfda7b59411_23.png)

*   **挂载点**：选择 `/`。
*   **期望容量**：例如，输入 `12G`。
*   **设备类型**：同样选择 **LVM**。你可以选择将其放入已有的卷组（如 `rhel`），或创建新的卷组。逻辑卷名称可以自定义，例如 `root` 或直接使用 `lv_root`。
*   **文件系统**：选择 `ext4` 或 `xfs`。

![](img/8878706ffc89a43af893ddfda7b59411_25.png)

![](img/8878706ffc89a43af893ddfda7b59411_27.png)

**4. 创建自定义分区（例如 `/app`）**

![](img/8878706ffc89a43af893ddfda7b59411_29.png)

*   **挂载点**：输入自定义的挂载点，例如 `/app`。这可以用于存放应用程序数据。
*   **期望容量**：如果不指定，将使用所有剩余空间。这里我们直接添加。
*   **设备类型**：选择 **LVM**。系统可能会自动将其标记为“数据”用途。我们可以将其放入指定的卷组，并为其逻辑卷命名，例如 `app_lv`。
*   **文件系统**：选择 `ext4`。

![](img/8878706ffc89a43af893ddfda7b59411_31.png)

完成所有分区配置后，点击“完成”并接受更改。此时，分区方式就从“自动”变为了“自定义”。

![](img/8878706ffc89a43af893ddfda7b59411_33.png)

## 其他安装设置

分区完成后，我们还需要关注几个重要的安装设置。

**Kdump**
这是一个内核崩溃转储机制。当系统内核发生严重错误时，Kdump可以捕获内存状态，用于事后分析。建议保持 **启用** 状态，并采用自动分配的内存大小。

![](img/8878706ffc89a43af893ddfda7b59411_35.png)

**网络与主机名**
在这里可以配置网络和系统主机名。

![](img/8878706ffc89a43af893ddfda7b59411_37.png)

*   **主机名**：相当于计算机在网络中的名称，应保持唯一。例如，可以设置为 `server01`。
*   **网络配置**：点击“配置”可以设置网卡。默认是DHCP自动获取IP。我们可以改为手动配置：
    *   **IPv4设置**：选择“手动”，添加IP地址（如 `10.60.100.2`）、子网掩码（如 `255.255.255.0`）和网关（如 `10.60.100.1`）。
    *   配置完成后，系统安装好就会使用这个静态IP。

![](img/8878706ffc89a43af893ddfda7b59411_39.png)

**安全策略**
系统会默认启用安全策略（如SELinux）。在初学阶段，可以先保持默认开启，但需要知道它可能会影响某些服务的正常运行。

## 开始安装

所有配置确认无误后，就可以点击“开始安装”了。

在安装过程中，你需要做两件事：
1.  为 **root（管理员）用户** 设置一个强密码。
2.  （可选）创建一个 **普通用户** 账户。

安装进度条会显示正在安装的软件包数量。Linux系统的安装过程，本质上就是根据你选择的软件包组合（最小化安装或带图形界面），将一个个 **RPM** 格式的软件包安装到硬盘上。等待所有包安装完成，系统即可使用。
