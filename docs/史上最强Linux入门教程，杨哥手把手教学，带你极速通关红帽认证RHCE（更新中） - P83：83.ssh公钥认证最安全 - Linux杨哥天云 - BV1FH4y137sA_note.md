Linux入门与红帽认证RHCE通关教程：P83：SSH公钥认证配置指南 🔑

在本节课中，我们将学习如何配置SSH的公钥认证。这是一种比传统密码认证更安全、更便捷的远程登录方式。我们将从生成密钥对开始，逐步完成公钥传输与验证配置。

上一节我们介绍了SSH的基本连接，本节中我们来看看如何通过公钥认证实现免密登录。

### 实验环境准备
为了演示效果，我们使用两台机器：一台作为服务器（IP: 192.168.10.20，SSH端口：20），另一台作为客户端。我们将以用户“杨哥”的身份进行操作。

首先，在服务器端切换到“杨哥”用户，并查看其家目录下的初始文件。

### 生成SSH密钥对
配置的第一步是在**客户端**生成密钥对。以下是操作步骤：

![](img/0cbf4fd16134391ecb3bf9538c56b811_9.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_10.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_11.png)

1.  在客户端，使用 `ssh-keygen` 命令生成密钥。
    ```bash
    ssh-keygen
    ```
    生成过程中，所有提示均可直接按回车键接受默认值。

2.  密钥对默认生成在用户家目录的 `.ssh/` 文件夹中。其中：
    *   `id_rsa` 是**私钥**，必须妥善保管在客户端。
    *   `id_rsa.pub` 是**公钥**，需要传输到服务器。

### 传输公钥至服务器
生成密钥对后，需要将客户端的公钥传输到目标服务器。我们使用 `ssh-copy-id` 命令来完成。

![](img/0cbf4fd16134391ecb3bf9538c56b811_13.png)

1.  在客户端执行以下命令，将公钥传给服务器上的“杨哥”用户。
    ```bash
    ssh-copy-id -p 20 yangge@192.168.10.20
    ```
    此命令会提示输入服务器上“杨哥”用户的密码。这是配置过程中**唯一一次**需要输入密码。

![](img/0cbf4fd16134391ecb3bf9538c56b811_14.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_15.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_16.png)

2.  传输成功后，可以在服务器上“杨哥”用户家目录的 `.ssh/authorized_keys` 文件中找到上传的公钥。

![](img/0cbf4fd16134391ecb3bf9538c56b811_17.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_18.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_19.png)

### 测试公钥认证
公钥传输完成后，即可进行测试。

![](img/0cbf4fd16134391ecb3bf9538c56b811_20.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_21.png)

1.  从客户端尝试登录服务器。
    ```bash
    ssh -p 20 yangge@192.168.10.20
    ```
    此时，系统将优先使用公钥进行认证，无需再次输入密码即可直接登录。

### 增强安全性：禁用密码认证
在公钥认证配置无误后，为了进一步提升安全性，可以关闭服务器的密码认证功能。

![](img/0cbf4fd16134391ecb3bf9538c56b811_23.png)

1.  编辑服务器上的SSH服务配置文件 `/etc/ssh/sshd_config`。
2.  找到 `PasswordAuthentication` 配置项，将其值改为 `no`。
    ```bash
    PasswordAuthentication no
    ```
3.  保存文件并重启SSH服务使配置生效。
    ```bash
    systemctl restart sshd
    ```
    **请注意**：此操作必须在公钥认证测试成功**之后**进行，否则可能导致无法登录。

![](img/0cbf4fd16134391ecb3bf9538c56b811_24.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_25.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_26.png)

### 重要注意事项
公钥认证是基于**用户**和**密钥对**的。上述配置意味着：客户端的“杨哥”用户可以通过密钥免密登录服务器的“杨哥”用户。

![](img/0cbf4fd16134391ecb3bf9538c56b811_27.png)

如果换用客户端的其他用户（例如“天云”用户）尝试登录，由于该用户没有对应的私钥，系统将回退到密码认证或直接拒绝连接。

![](img/0cbf4fd16134391ecb3bf9538c56b811_29.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_30.png)

因此，务必确保进行密钥认证的客户端用户与服务器端目标用户相匹配。

![](img/0cbf4fd16134391ecb3bf9538c56b811_31.png)

![](img/0cbf4fd16134391ecb3bf9538c56b811_32.png)

本节课中我们一起学习了SSH公钥认证的完整配置流程。我们掌握了生成密钥对、传输公钥、测试登录以及通过禁用密码认证来强化安全性的方法。理解并熟练运用公钥认证，是保障Linux系统远程管理安全的重要技能。
