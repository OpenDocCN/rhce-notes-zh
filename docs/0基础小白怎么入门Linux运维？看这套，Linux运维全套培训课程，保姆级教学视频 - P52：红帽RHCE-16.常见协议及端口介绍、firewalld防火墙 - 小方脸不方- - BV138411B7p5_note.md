## 网络协议与端口号

![](img/2e4a5fa0891f9a32c710484758376e35_3.png)

上一节我们介绍了Linux系统的基本操作，本节中我们来看看网络通信的基石——协议和端口。

![](img/2e4a5fa0891f9a32c710484758376e35_5.png)

![](img/2e4a5fa0891f9a32c710484758376e35_7.png)

在互联网中，计算机之间的数据传输需要遵循特定的规则，这些规则被称为**协议**。协议规定了数据如何打包、传输和接收。例如，当你访问淘宝网站时，你的浏览器和淘宝服务器之间的数据传输就基于**HTTP**或**HTTPS**协议。

*   **HTTP**：超文本传输协议，用于传输网页内容。它是明文协议，数据在传输过程中可能被截获和分析。
    *   默认端口：**80**
*   **HTTPS**：安全的超文本传输协议，在HTTP基础上增加了加密层（SSL/TLS），确保数据传输的安全性。
    *   默认端口：**443**

**端口**是网络通信的入口。一台服务器上可能运行着多个应用程序（服务），如网站、文件共享服务等。端口号用于区分这些不同的服务。客户端通过服务器的IP地址和特定的端口号，才能访问到对应的服务。

可以把服务器想象成一个大型商场，每个店铺（服务）都有一个唯一的门牌号（端口）。顾客（客户端）需要根据门牌号才能找到想去的店铺。

以下是互联网中一些常见协议及其默认端口：

*   **FTP**：文件传输协议，用于在网络上传输文件。类似于公司内部的网盘系统。
    *   默认端口：**20** (数据), **21** (控制)
*   **SSH**：安全外壳协议，用于远程加密登录和管理Linux服务器。
    *   默认端口：**22**
*   **DNS**：域名系统协议，用于将域名（如 `www.taobao.com`）解析为IP地址。
    *   默认端口：**53**
*   **Telnet**：远程登录协议，由于是明文传输，不安全，现已基本被SSH取代。
    *   默认端口：**23**

## 防火墙基础概念

![](img/2e4a5fa0891f9a32c710484758376e35_19.png)

![](img/2e4a5fa0891f9a32c710484758376e35_21.png)

了解了协议和端口后，我们来看看如何控制对这些端口的访问，这就是防火墙的作用。

![](img/2e4a5fa0891f9a32c710484758376e35_23.png)

![](img/2e4a5fa0891f9a32c710484758376e35_25.png)

![](img/2e4a5fa0891f9a32c710484758376e35_27.png)

防火墙工作在主机或网络的边缘，根据预先设定的规则，对进出的网络数据包进行检查和过滤。对于匹配规则的数据包，会执行相应的处理（如允许通过或拒绝）。

![](img/2e4a5fa0891f9a32c710484758376e35_29.png)

![](img/2e4a5fa0891f9a32c710484758376e35_31.png)

防火墙主要分为两类：
1.  **软件防火墙**：通过安装在操作系统上的软件来实现包过滤功能。成本低，配置灵活。
2.  **硬件防火墙**：独立的物理设备，专门用于网络安全防护。性能强，但成本高。

![](img/2e4a5fa0891f9a32c710484758376e35_33.png)

![](img/2e4a5fa0891f9a32c710484758376e35_35.png)

![](img/2e4a5fa0891f9a32c710484758376e35_37.png)

根据保护范围，又可分为：
*   **主机防火墙**：保护单台主机，规则仅对该主机生效。
*   **网络防火墙**：保护一个网络区域，通常部署在网络入口处。

在CentOS/RHEL 7及之后版本中，默认的防火墙管理工具是 **firewalld**，而更早的版本（如CentOS 6）使用的是 **iptables**。虽然firewalld是官方推荐的新工具，但很多企业环境由于历史习惯，仍然广泛使用iptables。内核中实际负责数据包过滤的模块叫做 **netfilter**，firewalld和iptables都是用于配置netfilter规则的用户空间工具。

## firewalld防火墙初探

接下来，我们通过一个简单的实验来理解firewalld的基本操作。

首先，我们安装并启动一个Web服务器（httpd）来作为测试对象。

```bash
yum -y install httpd
systemctl start httpd
systemctl status httpd
```

使用 `ss` 命令可以查看httpd服务监听的端口。

![](img/2e4a5fa0891f9a32c710484758376e35_39.png)

![](img/2e4a5fa0891f9a32c710484758376e35_41.png)

```bash
ss -ntlp | grep httpd
```
输出可能显示 `LISTEN 0 128 :::80 :::*`，这表明httpd正在监听80端口。

![](img/2e4a5fa0891f9a32c710484758376e35_43.png)

![](img/2e4a5fa0891f9a32c710484758376e35_45.png)

此时，如果从浏览器访问服务器的IP地址（如 `http://192.168.0.40`），会发现无法打开页面。这是因为默认开启的防火墙阻止了访问。

![](img/2e4a5fa0891f9a32c710484758376e35_47.png)

### firewalld的区域（Zone）

![](img/2e4a5fa0891f9a32c710484758376e35_49.png)

![](img/2e4a5fa0891f9a32c710484758376e35_51.png)

![](img/2e4a5fa0891f9a32c710484758376e35_53.png)

firewalld引入了“区域”的概念，每个区域定义了一组预设的信任级别和规则。常用的区域有：

![](img/2e4a5fa0891f9a32c710484758376e35_55.png)

![](img/2e4a5fa0891f9a32c710484758376e35_57.png)

*   **public（默认）**：仅允许访问本机SSH、DHCP等少量服务。
*   **trusted**：允许所有访问。
*   **block**：拒绝所有访问，但会向请求方返回拒绝信息。
*   **drop**：丢弃所有访问请求，不做任何回应（更隐蔽）。

![](img/2e4a5fa0891f9a32c710484758376e35_59.png)

![](img/2e4a5fa0891f9a32c710484758376e35_61.png)

![](img/2e4a5fa0891f9a32c710484758376e35_63.png)

![](img/2e4a5fa0891f9a32c710484758376e35_65.png)

我们可以使用 `firewall-cmd` 命令来管理防火墙。

![](img/2e4a5fa0891f9a32c710484758376e35_67.png)

![](img/2e4a5fa0891f9a32c710484758376e35_69.png)

![](img/2e4a5fa0891f9a32c710484758376e35_71.png)

查看当前默认区域：
```bash
firewall-cmd --get-default-zone
```

![](img/2e4a5fa0891f9a32c710484758376e35_73.png)

修改默认区域为 `trusted` 以允许所有访问：
```bash
firewall-cmd --set-default-zone=trusted
```
修改后，刷新浏览器，应该就能看到Apache的测试页面了。

![](img/2e4a5fa0891f9a32c710484758376e35_75.png)

![](img/2e4a5fa0891f9a32c710484758376e35_77.png)

查看某个区域（如public）的详细规则：
```bash
firewall-cmd --zone=public --list-all
```

![](img/2e4a5fa0891f9a32c710484758376e35_79.png)

![](img/2e4a5fa0891f9a32c710484758376e35_81.png)

![](img/2e4a5fa0891f9a32c710484758376e35_83.png)

### 管理防火墙规则

![](img/2e4a5fa0891f9a32c710484758376e35_85.png)

![](img/2e4a5fa0891f9a32c710484758376e35_87.png)

在默认的public区域，默认不允许HTTP访问。我们可以为public区域添加一条规则，允许HTTP协议的数据通过。

![](img/2e4a5fa0891f9a32c710484758376e35_89.png)

![](img/2e4a5fa0891f9a32c710484758376e35_91.png)

![](img/2e4a5fa0891f9a32c710484758376e35_93.png)

![](img/2e4a5fa0891f9a32c710484758376e35_95.png)

添加允许HTTP服务的规则：
```bash
firewall-cmd --zone=public --add-service=http --permanent
firewall-cmd --reload
```
*   `--add-service=http`：添加http服务（对应80端口）。
*   `--permanent`：使规则永久生效（否则重启后失效）。
*   `--reload`：重新加载防火墙配置，使永久规则立即生效。

添加后，即使默认区域是public，也能访问Web页面了。

删除规则：
```bash
firewall-cmd --zone=public --remove-service=http --permanent
firewall-cmd --reload
```

## 总结

![](img/2e4a5fa0891f9a32c710484758376e35_97.png)

![](img/2e4a5fa0891f9a32c710484758376e35_99.png)

本节课中我们一起学习了网络基础与Linux防火墙入门知识。

我们首先了解了**协议**是网络通信的规则，**端口**是服务器上不同服务的访问入口，并记住了一些关键协议（HTTP/HTTPS/SSH）及其端口（80/443/22）。

接着，我们认识了防火墙的作用和分类，并重点操作了CentOS 7的默认防火墙工具**firewalld**。我们学习了如何查看和切换不同的安全**区域**，以及如何通过添加或删除**服务规则**来控制对特定端口（如Web服务的80端口）的访问。
