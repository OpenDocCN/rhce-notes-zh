## 进程、用户与文件权限的关系

上一节我们介绍了用户和组的基本概念，本节中我们来看看进程如何与文件权限进行交互。

一个进程在运行时，会携带其所属用户的身份标识（UID）和组标识（GID）。当这个进程尝试访问一个文件时，系统会根据进程的UID/GID与文件的拥有人/拥有组进行匹配，从而决定进程以何种身份（拥有人、拥有组或其他用户）来访问该文件。

例如，当用户 `user4` 执行 `vim` 命令时，会生成一个进程。该进程的UID和GID通常就是 `user4` 的UID和GID。如果 `vim` 进程要访问 `/etc/passwd` 文件（拥有人和拥有组均为 `root`），系统会进行如下匹配：

![](img/50399c343991cb14cf540f7396348e3c_11.png)

1.  进程的UID (`user4`) 是否等于文件的拥有人 (`root`)？ **否**。
2.  进程的GID (`user4` 的主组) 是否等于文件的拥有组 (`root`)？ **否**。
3.  以上都不匹配，因此进程将以 **其他用户** 的身份访问该文件。

![](img/50399c343991cb14cf540f7396348e3c_13.png)

## 权限匹配的三种身份与优先级

![](img/50399c343991cb14cf540f7396348e3c_15.png)

在访问文件时，进程的身份被划分为三类，系统会按照固定优先级进行匹配。

![](img/50399c343991cb14cf540f7396348e3c_17.png)

![](img/50399c343991cb14cf540f7396348e3c_19.png)

以下是身份匹配的规则：
*   **拥有人 (Owner)**：如果进程的UID与文件的UID匹配，则进程以文件拥有人的身份访问。这是最高优先级。
*   **拥有组 (Group)**：如果拥有人不匹配，但进程的GID（或其附加组GID）与文件的GID匹配，则进程以文件拥有组的身份访问。
*   **其他用户 (Others)**：如果以上两者均不匹配，则进程以其他用户的身份访问。

![](img/50399c343991cb14cf540f7396348e3c_21.png)

**总结**：权限匹配遵循 **拥有人 > 拥有组 > 其他用户** 的优先级顺序。一旦在某一级匹配成功，就使用该身份对应的权限，不再继续向下匹配。

![](img/50399c343991cb14cf540f7396348e3c_23.png)

![](img/50399c343991cb14cf540f7396348e3c_25.png)

## 解读文件权限：rwx

![](img/50399c343991cb14cf540f7396348e3c_27.png)

确定了访问身份后，接下来就需要查看该身份对文件拥有哪些具体权限。文件的权限通过9个字符表示，每3个字符为一组，分别对应拥有人、拥有组和其他用户的权限。

![](img/50399c343991cb14cf540f7396348e3c_29.png)

每组中的三个字符依次代表：
*   **r (Read)**：读权限。
*   **w (Write)**：写权限。
*   **x (Execute)**：执行权限。

![](img/50399c343991cb14cf540f7396348e3c_31.png)

![](img/50399c343991cb14cf540f7396348e3c_33.png)

如果某个位置是 `-`，则表示没有对应的权限。例如，权限字符串 `-rwxr-xr--` 表示：
*   拥有人权限：`rwx` (可读、可写、可执行)
*   拥有组权限：`r-x` (可读、不可写、可执行)
*   其他用户权限：`r--` (只读)

## 文件与目录的rwx权限详解

`rwx` 权限对于普通文件和目录的含义有显著区别。

![](img/50399c343991cb14cf540f7396348e3c_35.png)

### 文件的rwx权限
*   **r (读)**：允许查看文件内容。例如，使用 `cat`、`less` 命令。
*   **w (写)**：允许修改文件内容。例如，使用 `vim` 编辑并保存文件。
*   **x (执行)**：允许将文件作为程序或脚本运行。例如，执行 `./script.sh` 或 `/usr/bin/ls`。

![](img/50399c343991cb14cf540f7396348e3c_37.png)

### 目录的rwx权限
*   **r (读)**：允许列出目录内的文件和子目录名称（使用 `ls` 命令），但**不能**查看详细信息（如 `ls -l`）或进入目录。
*   **w (写)**：允许在目录内**创建、删除、重命名**文件和子目录。**注意**：对目录拥有 `w` 权限，就可以删除该目录下的任何文件，**即使你不是那个文件的拥有人**。
*   **x (执行)**：允许进入该目录（使用 `cd` 命令），并访问目录内已知的文件/子目录的元数据（这是使用 `ls -l` 查看详细信息的前提）。

![](img/50399c343991cb14cf540f7396348e3c_39.png)

![](img/50399c343991cb14cf540f7396348e3c_41.png)

**常见组合**：
*   对目录拥有 `rx` 权限：可以进入目录并列出内容。
*   对目录拥有 `wx` 权限：可以进入目录并在其中创建、删除文件，但无法列出已有文件列表（因为无 `r` 权限）。
*   对目录拥有 `rwx` 权限：拥有完整控制权。

![](img/50399c343991cb14cf540f7396348e3c_43.png)

## 修改文件拥有人和拥有组 (chown, chgrp)

![](img/50399c343991cb14cf540f7396348e3c_45.png)

了解如何查看权限后，我们学习如何修改文件的归属。这需要使用 `chown` (change owner) 和 `chgrp` (change group) 命令。

以下是常用命令格式示例：
```bash
# 修改文件拥有人
chown user4 /path/to/file

# 修改文件拥有组
chgrp group1 /path/to/file

![](img/50399c343991cb14cf540f7396348e3c_47.png)

# 同时修改拥有人和拥有组
chown user4:group1 /path/to/file

![](img/50399c343991cb14cf540f7396348e3c_49.png)

# 仅修改拥有组（使用冒号前缀）
chown :group1 /path/to/file

![](img/50399c343991cb14cf540f7396348e3c_51.png)

# 递归修改目录及其内部所有内容的拥有人
chown -R user4 /path/to/directory/
```

![](img/50399c343991cb14cf540f7396348e3c_53.png)

![](img/50399c343991cb14cf540f7396348e3c_55.png)

![](img/50399c343991cb14cf540f7396348e3c_57.png)

**重要规则**：
1.  只有 **root 用户** 可以任意修改文件的拥有人。
2.  文件的 **拥有人** 可以修改文件的拥有组，但**前提是**该拥有人自身必须是目标组的成员。
3.  **root 用户** 可以修改文件的拥有组为任何存在的组。
