在本节课中，我们将要学习Linux系统中一个极其强大的工具——`find`命令。与之前学习的`locate`命令不同，`find`命令直接在文件系统中实时搜索，功能更为灵活和强大。我们将从基础用法开始，逐步探索如何控制搜索深度、按类型和名称过滤、按文件大小搜索，并最终学习其最强大的功能：对搜索结果执行其他命令。

## `find`命令基础

![](img/e199445e297039582c60eb60aa6a9992_5.png)

![](img/e199445e297039582c60eb60aa6a9992_7.png)

上一节我们介绍了`locate`命令，它依赖于数据库进行快速搜索。本节中我们来看看`find`命令，它直接在文件系统上工作，因此搜索结果总是最新的。

`find`命令的默认行为是列出指定起点目录下的**每一个文件和文件夹**，并递归地进入所有子目录，无限深度地展开所有内容。

![](img/e199445e297039582c60eb60aa6a9992_9.png)

![](img/e199445e297039582c60eb60aa6a9992_11.png)

如果我们直接在终端中运行`find`命令，它会从**当前工作目录**开始搜索。

![](img/e199445e297039582c60eb60aa6a9992_13.png)

![](img/e199445e297039582c60eb60aa6a9992_15.png)

![](img/e199445e297039582c60eb60aa6a9992_17.png)

```bash
find
```

![](img/e199445e297039582c60eb60aa6a9992_19.png)

执行上述命令，会列出当前目录及其所有子目录中的每个文件和文件夹。例如，如果当前目录是家目录(`~`)，它会显示`Pictures`、`Desktop`等文件夹，并继续进入这些文件夹，列出其中的所有内容（如`Pictures/Holiday/`和`Pictures/findme6.txt`）。

![](img/e199445e297039582c60eb60aa6a9992_21.png)

![](img/e199445e297039582c60eb60aa6a9992_23.png)

`find`命令与`locate`命令的核心区别在于：
*   `find`：实时搜索文件系统，无需数据库，结果最新但速度相对较慢。
*   `locate`：搜索预建的数据库，速度极快但结果可能不是最新的。

![](img/e199445e297039582c60eb60aa6a9992_25.png)

![](img/e199445e297039582c60eb60aa6a9992_27.png)

![](img/e199445e297039582c60eb60aa6a9992_29.png)

![](img/e199445e297039582c60eb60aa6a9992_30.png)

你可以为`find`命令指定任何搜索起点。例如，从根目录开始搜索将遍历整个文件系统（这可能需要很长时间）：
```bash
find /
```
从特定目录开始搜索：
```bash
find /home/ziad/Documents
```
即使你不在目标目录中，也可以指定其路径进行搜索。`find`命令会从指定点开始，深入文件系统，列出该点以下的所有内容，无论文件或文件夹是否隐藏。

![](img/e199445e297039582c60eb60aa6a9992_32.png)

![](img/e199445e297039582c60eb60aa6a9992_34.png)

## 控制搜索深度与类型

![](img/e199445e297039582c60eb60aa6a9992_36.png)

![](img/e199445e297039582c60eb60aa6a9992_38.png)

默认情况下，`find`命令会进行无限深度搜索。但我们可以使用`-maxdepth`选项来控制它向下搜索的层数。

![](img/e199445e297039582c60eb60aa6a9992_40.png)

![](img/e199445e297039582c60eb60aa6a9992_42.png)

首先，我们创建一个多级目录结构用于演示：
```
Documents/
└── level2/
    ├── two.txt
    └── level3/
        ├── three.txt
        └── level4/
            ├── four.txt
            └── level5/
                └── five.txt
```

![](img/e199445e297039582c60eb60aa6a9992_44.png)

![](img/e199445e297039582c60eb60aa6a9992_46.png)

`-maxdepth`选项后面跟一个数字，表示搜索的最大深度。**注意**：`find`命令的许多选项使用单个短横线(`-`)，而不是常见的双横线(`--`)。

![](img/e199445e297039582c60eb60aa6a9992_48.png)

![](img/e199445e297039582c60eb60aa6a9992_50.png)

以下是使用`-maxdepth`的示例：
*   `-maxdepth 1`：仅列出当前目录的内容。
    ```bash
    find . -maxdepth 1
    ```
*   `-maxdepth 2`：列出当前目录及下一级子目录的内容。
    ```bash
    find . -maxdepth 2
    ```
*   不指定`-maxdepth`：默认无限深度搜索。

另一个常用选项是`-type`，用于按类型过滤结果。默认`find`会列出文件和目录，而`-type`允许我们只查找其中一种。

以下是`-type`选项的用法：
*   `-type f`：仅查找**文件**。
    ```bash
    find . -type f
    ```
*   `-type d`：仅查找**目录/文件夹**。
    ```bash
    find . -type d
    ```

你可以组合使用`-maxdepth`和`-type`选项来精确控制搜索。**重要**：当组合多个选项时，应将`-maxdepth`选项放在前面。
```bash
find . -maxdepth 2 -type d
```
这个命令将搜索当前目录下最大深度为2的所有目录。

## 按名称和大小搜索

除了控制深度和类型，`find`命令还能根据文件名和文件大小进行精准搜索。

**按名称搜索**使用`-name`选项。搜索时，可以使用通配符（如`*`代表任意字符，`?`代表单个字符）。为了确保通配符被正确解析，建议将搜索模式用双引号括起来。

以下是按名称搜索的示例：
*   查找名为`five.txt`的文件：
    ```bash
    find . -name “five.txt”
    ```
*   查找所有以`.txt`结尾的文件：
    ```bash
    find . -name “*.txt”
    ```
*   进行不区分大小写的搜索，使用`-iname`选项：
    ```bash
    find . -iname “*.TXT”
    ```

**按文件大小搜索**使用`-size`选项。大小单位可以是`k`（千字节）、`M`（兆字节）、`G`（千兆字节）。使用`+`表示“大于”，`-`表示“小于”。

![](img/e199445e297039582c60eb60aa6a9992_52.png)

以下是按大小搜索的示例，通常结合`-type f`只搜索文件：
*   查找大于100KB的文件：
    ```bash
    find / -type f -size +100k
    ```
*   查找小于5MB的文件：
    ```bash
    find / -type f -size -5M
    ```
*   查找大小在100KB到5MB之间的文件：
    ```bash
    find / -type f -size +100k -size -5M
    ```
    这个命令使用了逻辑**与**（AND），要求文件同时满足两个条件。

**注意**：从根目录(`/`)开始搜索系统文件可能需要管理员权限，否则会遇到“权限被拒绝”的错误。可以在命令前添加`sudo`以超级用户身份运行。
```bash
sudo find / -type f -size +100k
```
要计算找到的文件数量，可以将结果通过管道(`|`)传递给`wc -l`命令（统计行数）：
```bash
sudo find / -type f -size +100k | wc -l
```

## 对搜索结果执行命令：`-exec` 与 `-ok`

![](img/e199445e297039582c60eb60aa6a9992_54.png)

`find`命令最强大的功能之一是能够对搜索到的每一个结果执行另一个命令。这是通过`-exec`或`-ok`选项实现的。

假设我们想将系统中所有大小在100KB到5MB之间的文件复制到一个特定文件夹（例如`~/Desktop/copy_here`）。

![](img/e199445e297039582c60eb60aa6a9992_56.png)

![](img/e199445e297039582c60eb60aa6a9992_58.png)

基本命令结构如下：
```bash
find <搜索起点> <条件> -exec <要执行的命令> {} \;
```
*   `{}`：这是一个占位符，在命令执行时会被替换为`find`找到的**每一个文件路径**。
*   `\;`：这标志着`-exec`选项的结束。反斜杠(`\`)用于转义分号，使其被正确识别。

具体命令示例：
```bash
sudo find / -type f -size +100k -size -5M -exec cp {} ~/Desktop/copy_here \;
```
这个命令会：
1.  从根目录开始。
2.  查找所有类型为文件(`-type f`)、大小在100KB到5MB之间(`-size +100k -size -5M`)的对象。
3.  对每一个找到的文件，执行`cp`（复制）命令，将其复制到`~/Desktop/copy_here`目录。

`-ok`选项的功能与`-exec`完全相同，但它在执行每个命令前会**交互式地询问用户是否确认**。这对于执行删除(`rm`)等危险操作时非常有用。
```bash
find . -name “*.tmp” -ok rm {} \;
```
运行此命令时，对于每个找到的`.tmp`文件，终端都会询问“`rm ./file.tmp?`”，输入`y`确认删除，输入`n`跳过。

## 实战：大海捞针

![](img/e199445e297039582c60eb60aa6a9992_60.png)

让我们通过一个综合练习来体验`find`命令的强大。目标：在成千上万个文件中找到一个特定的文件。

1.  **创建“干草堆”**：在桌面上创建一个包含大量文件和文件夹的复杂结构。
    ```bash
    # 创建主文件夹
    mkdir ~/Desktop/haystack
    # 使用大括号扩展创建500个子文件夹
    mkdir ~/Desktop/haystack/folder{1..500}
    # 在每个子文件夹中创建100个文件
    touch ~/Desktop/haystack/folder{1..500}/file{1..100}
    ```
    现在，我们有了500个文件夹，每个文件夹里有100个文件，总计50，000个文件。

![](img/e199445e297039582c60eb60aa6a9992_62.png)

![](img/e199445e297039582c60eb60aa6a9992_64.png)

2.  **藏起“针”**：在其中一个随机文件夹里创建一个名为`needle.txt`的特殊文件（具体位置可随机选择，例如`folder449`）。
    ```bash
    touch ~/Desktop/haystack/folder449/needle.txt
    ```

![](img/e199445e297039582c60eb60aa6a9992_66.png)

3.  **寻找“针”**：使用`find`命令在“干草堆”中找出这个唯一的文件。
    ```bash
    find ~/Desktop/haystack -type f -name “needle.txt”
    ```
    命令会返回该文件的完整路径，例如`/home/ziad/Desktop/haystack/folder449/needle.txt`。

![](img/e199445e297039582c60eb60aa6a9992_68.png)

![](img/e199445e297039582c60eb60aa6a9992_70.png)

![](img/e199445e297039582c60eb60aa6a9992_72.png)

4.  **移动“针”**：找到后，我们不仅想看到它，还想直接把它移动到桌面。这可以一步完成：
    ```bash
    find ~/Desktop/haystack -type f -name “needle.txt” -exec mv {} ~/Desktop \;
    ```
    这个命令组合了查找、定位和移动操作，从5万个文件中精准地找到了目标并完成了任务。

![](img/e199445e297039582c60eb60aa6a9992_74.png)

![](img/e199445e297039582c60eb60aa6a9992_76.png)

本节课中我们一起学习了`find`命令的核心用法。我们从其基础递归搜索开始，学习了如何用`-maxdepth`和`-type`控制搜索行为，接着探索了按`-name`和`-size`进行过滤，最后掌握了其杀手锏功能——使用`-exec`或`-ok`对搜索结果执行操作。`find`命令是Linux命令行中不可或缺的工具，结合通配符、大括号扩展等其他技巧，可以让你高效地管理复杂的文件系统任务。
