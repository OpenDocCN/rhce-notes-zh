# 华为云PaaS微服务治理技术 - P114：06.学成在线项目部署-portalview-创建工作负载 🚀

![](img/8ead4a12d2bdfcec89df7dadbb60b997_1.png)

在本节课中，我们将要学习如何在华为云CCE上为“学成在线”项目的`portalview`微服务创建工作负载。我们将通过创建无状态工作负载、配置容器镜像、设置环境变量以及添加服务访问方式，完成该微服务的部署。

上一节我们完成了`portalview`微服务镜像的上传，本节中我们来看看如何基于该镜像创建工作负载，实现服务的部署。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_3.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_4.png)

## 创建工作负载类型选择

![](img/8ead4a12d2bdfcec89df7dadbb60b997_6.png)

首先需要确定创建的工作负载类型。如果应用在运行过程中需要对数据进行持久化并记录状态变化，则需要创建有状态工作负载。`portalview`微服务运行的是Java程序，可以方便地复制多份，因此我们将其创建为无状态工作负载。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_8.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_10.png)

## 创建工作负载基础配置

![](img/8ead4a12d2bdfcec89df7dadbb60b997_12.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_13.png)

以下是创建工作负载的基础步骤。

1.  点击“创建”按钮。
2.  输入工作负载的名称，例如 `portalview`。
3.  选择所属集群，例如 `EDU02`。
4.  实例数保持默认的1个容器。
5.  点击“下一步”。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_15.png)

## 配置容器

现在需要为工作负载添加容器。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_17.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_19.png)

1.  点击“添加容器”。
2.  在镜像选择页面，找到并选择之前上传的 `portalview` 镜像。
3.  点击“确定”。
4.  为容器设置一个名称。
5.  配置容器资源。对于Spring Boot开发的Java程序，建议适当提高CPU和内存配置。例如：
    *   CPU申请：`0.5`核
    *   内存申请：`1024` MiB
    *   CPU限制：`1`核
    *   内存限制：`2048` MiB
6.  点击“下一步”。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_20.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_22.png)

## 配置环境变量与存储

![](img/8ead4a12d2bdfcec89df7dadbb60b997_24.png)

此步骤需要关注微服务是否有外部依赖配置。打开`portalview`程序的配置文件，我们发现它需要连接MongoDB数据库。因此，必须通过环境变量将MongoDB的连接地址传入容器。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_26.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_28.png)

1.  在“环境变量”配置区域，点击“添加环境变量”。
2.  变量名填写程序配置中对应的键，例如 `spring.data.mongodb.uri`。
3.  变量值需要填写MongoDB在**集群内部**的访问地址，而不是外网地址。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_30.png)

**如何获取MongoDB内网地址？**

![](img/8ead4a12d2bdfcec89df7dadbb60b997_32.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_34.png)

因为`portalview`容器与MongoDB容器同在一个集群内，它们之间应通过内网通信。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_36.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_38.png)

1.  在CCE控制台，找到之前部署的MongoDB有状态工作负载。
2.  进入其详情页，查看“访问方式”。
3.  如果只有外网访问方式，需要点击“添加”来创建一个**集群内访问**的服务。
4.  设置服务名称，访问类型选择“集群内访问”，容器端口填写MongoDB的服务端口 `27017`。
5.  创建后，系统会生成一个内网域名或IP地址，将此地址作为MongoDB连接串的一部分，填入`portalview`的环境变量值中。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_40.png)

例如，环境变量配置可能如下：
```
变量名：spring.data.mongodb.uri
变量值：mongodb://<内网服务地址>:27017/xc_portal_db
```

关于数据存储，由于`portalview`是无状态服务，通常无需挂载持久化数据卷。因此，“数据存储”部分可以保持默认，无需额外配置。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_42.png)

## 添加服务（Service）

工作负载创建后，需要为其配置访问方式，以便外部或其他服务能够调用。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_44.png)

以下是添加服务的步骤。

![](img/8ead4a12d2bdfcec89df7dadbb60b997_46.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_47.png)

1.  在“服务配置”步骤，点击“添加服务”。
2.  服务名称可设置为 `portalview-service`。
3.  访问类型选择“公网访问”。
4.  容器端口填写`portalview`微服务自身的端口，根据其配置文件可知为 `40200`。
5.  服务端口（即公网访问端口）可以由系统自动生成，也可以手动指定一个范围（如30000-32767）。
6.  点击“下一步”，然后确认并“创建”工作负载。

## 部署完成与验证

![](img/8ead4a12d2bdfcec89df7dadbb60b997_49.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_50.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_52.png)

完成上述所有步骤后，`portalview`微服务的工作负载就创建成功了。在CCE控制台的工作负载列表中，可以看到其状态变为“运行中”。

稍后，我们可以通过生成的服务公网地址和端口，对`portalview`服务进行访问测试，以验证部署是否成功。

## 总结

![](img/8ead4a12d2bdfcec89df7dadbb60b997_55.png)

![](img/8ead4a12d2bdfcec89df7dadbb60b997_56.png)

本节课中我们一起学习了在华为云CCE上部署`portalview`微服务的完整流程。关键步骤包括：创建无状态工作负载、选择并配置容器镜像、通过环境变量注入MongoDB内网连接地址、以及添加公网访问服务。整个过程的核心在于理解容器间如何通过内网服务进行通信，并将外部依赖正确配置到应用环境中。
