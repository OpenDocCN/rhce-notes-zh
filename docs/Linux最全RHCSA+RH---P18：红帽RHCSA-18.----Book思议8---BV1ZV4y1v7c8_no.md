## 归属关系与基本权限

任何文件或目录都有三类归属关系，分别用首字母表示：
*   **U (User)**: 文件的所有者，通常是创建者。
*   **G (Group)**: 文件的所有组。
*   **O (Other)**: 既不是所有者，也不在所属组内的其他所有用户。

使用 `ls -l` 命令可以查看文件的详细信息，其中包含了权限信息。权限位由9个字符组成，每3个字符为一组，顺序固定为：**所有者权限 | 所属组权限 | 其他人权限**。

每个权限位由三个字符 `r`、`w`、`x` 或 `-` 表示：
*   **r (Read)**: 读取权限。
*   **w (Write)**: 写入（增、删、改）权限。
*   **x (Execute)**: 执行权限。
*   **-**: 表示没有对应权限。

例如，一个文件的权限显示为 `-rwxr-xr--`，我们可以这样解读：
*   所有者(`u`): `rwx` -> 可读、可写、可执行。
*   所属组(`g`): `r-x` -> 可读、不可写、可执行。
*   其他人(`o`): `r--` -> 仅可读。

## 修改文件归属关系：chown命令

在设置权限前，有时需要先修改文件的归属关系。`chown` 命令用于更改文件的所有者和所属组。

![](img/58a1d08eae07a5c15441ebb839fca52f_18.png)

![](img/58a1d08eae07a5c15441ebb839fca52f_20.png)

![](img/58a1d08eae07a5c15441ebb839fca52f_22.png)

以下是 `chown` 命令的几种常用格式：

```bash
# 同时修改所有者和所属组
chown 所有者:所属组 文件名
# 示例：将文件hello的所有者改为tom，所属组改为tom的初始组
chown tom:tom hello

# 仅修改所有者
chown 所有者 文件名
# 示例：将文件hello的所有者改回root
chown root hello

# 仅修改所属组
chown :所属组 文件名
# 示例：将文件hello的所属组改为test组
chown :test hello
```

**选项 `-R`**：递归处理，将指定目录下的所有文件及子目录一并修改归属关系。

## 修改文件权限：chmod命令

`chmod` 命令是修改文件权限的核心工具。它可以通过 **字母法** 或 **数字法** 来操作。

### 字母表示法

字母表示法直观易懂，格式为：`chmod [ugoa][+-=][rwx] 文件名`。

*   **身份指定符**:
    *   `u`: 所有者 (User)
    *   `g`: 所属组 (Group)
    *   `o`: 其他人 (Other)
    *   `a`: 所有人 (All)，是 `ugo` 的简写
*   **操作符**:
    *   `+`: 添加权限
    *   `-`: 移除权限
    *   `=`: 重新设定权限（覆盖原有权限）
*   **权限指定符**:
    *   `r`: 读
    *   `w`: 写
    *   `x`: 执行

以下是具体操作示例：

```bash
# 给所有者添加执行权限
chmod u+x hello

# 移除所属组的写权限
chmod g-w hello

![](img/58a1d08eae07a5c15441ebb839fca52f_24.png)

# 设置其他人只有读权限
chmod o=r hello

![](img/58a1d08eae07a5c15441ebb839fca52f_26.png)

# 同时为不同身份设置权限（用逗号分隔）
chmod u=rwx,g=rx,o=r hello

# 为所有人添加写权限
chmod a+w hello
```

### 数字表示法

![](img/58a1d08eae07a5c15441ebb839fca52f_28.png)

![](img/58a1d08eae07a5c15441ebb839fca52f_30.png)

数字表示法更为简洁，它将权限转换为一个三位八进制数。每个权限位对应一个数字：
*   `r` (读) = 4
*   `w` (写) = 2
*   `x` (执行) = 1
*   `-` (无) = 0

计算每组权限时，将拥有的权限对应的数字相加即可。

例如：
*   `rwx` = 4+2+1 = **7**
*   `r-x` = 4+0+1 = **5**
*   `r--` = 4+0+0 = **4**
*   `---` = 0+0+0 = **0**

![](img/58a1d08eae07a5c15441ebb839fca52f_32.png)

因此，权限 `rwxr-xr--` 对应的数字就是 **754**。

![](img/58a1d08eae07a5c15441ebb839fca52f_34.png)

![](img/58a1d08eae07a5c15441ebb839fca52f_36.png)

使用数字法修改权限：
```bash
# 将文件hello的权限设置为 rwxr-xr-- (754)
chmod 754 hello
```
**重要提示**：使用数字法时，必须完整指定三个数字（所有者、所属组、其他人），即使你只想修改其中一部分。例如，`chmod 6 hello` 会被系统理解为 `chmod 006 hello`，导致所有者和所属组权限被清零。

## 文件与目录权限的差异

理解权限的关键在于区分它对 **文件** 和 **目录** 的不同含义：

| 权限 | 对文件的影响 | 对目录的影响 |
| :--- | :--- | :--- |
| **r (读)** | 可以查看文件内容 | 可以列出目录中的文件列表（如 `ls`） |
| **w (写)** | 可以修改文件内容（增、删、改） | 可以在目录内 **创建、删除、重命名** 文件或子目录 |
| **x (执行)** | 可以运行该文件（如果是程序或脚本） | 可以进入该目录（如 `cd`） |

**核心区别**：对文件的 `w` 权限**不能**删除文件本身，只能修改其内容。能否删除一个文件，取决于用户对其**父目录**是否拥有 `w` 权限。这就是为什么普通用户通常无法删除 `/etc`、`/boot` 等系统目录下的文件——因为他们对这些目录没有 `w` 权限。

## 默认权限与umask

当我们新建文件或目录时，系统会赋予一个默认的权限。这个默认权限由 `umask`（权限掩码）决定。

`umask` 值表示要从默认权限中“减去”的权限。查看当前 `umask`：
```bash
umask          # 数字形式显示，如 0022
umask -S       # 符号形式显示，如 u=rwx,g=rx,o=rx
```
常见的默认 `umask` 是 **0022**，其含义是：
*   新建文件时，默认权限为 `666 - 022 = 644` (即 `rw-r--r--`)。
*   新建目录时，默认权限为 `777 - 022 = 755` (即 `rwxr-xr-x`)。

![](img/58a1d08eae07a5c15441ebb839fca52f_44.png)

![](img/58a1d08eae07a5c15441ebb839fca52f_46.png)

你可以临时修改 `umask` 值来改变新建文件的默认权限，但通常不需要这样做。

## 特殊权限简介（扩展）

除了基本的 `rwx`，Linux还有三个特殊权限位，它们出现在权限位的第一个字符组（所有者执行位）：
*   **SUID (Set User ID)**: 当具有SUID权限的**文件**被执行时，执行者将临时获得文件所有者的身份。
*   **SGID (Set Group ID)**: 对于**文件**，效果类似SUID，但获得的是文件所属组的身份。对于**目录**，在该目录下新建的文件，其所属组会自动继承该目录的所属组。
*   **Sticky Bit (粘滞位)**: 通常用于公共目录（如 `/tmp`）。在此目录下，用户只能删除自己创建的文件，而不能删除其他用户的文件。

![](img/58a1d08eae07a5c15441ebb839fca52f_52.png)

![](img/58a1d08eae07a5c15441ebb839fca52f_54.png)

设置特殊权限同样可以使用 `chmod` 命令（数字法在首位加一个数字，如 `4755`；字母法使用 `s` 和 `t`）。
