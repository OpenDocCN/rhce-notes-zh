在开始创建虚拟机之前，需要了解一些前置条件和注意事项。

![](img/6af4433e9459bf849712e3f27b418863_24.png)

![](img/6af4433e9459bf849712e3f27b418863_26.png)

*   **CPU兼容性**：目前主流的Intel或AMD CPU均可正常使用VMware Workstation。
*   **软件版本**：安装RHEL 8.2系统，**必须使用VMware Workstation 15或16版本**。12及以下版本存在内核不兼容问题，无法安装。
*   **虚拟机配置**：虚拟机流畅运行的最低推荐配置为**2核CPU**和**2GB内存**。仅用于启动的话，1核1GB也可运行，但性能较慢。

## 创建RHEL 8虚拟机 🛠️

![](img/6af4433e9459bf849712e3f27b418863_49.png)

![](img/6af4433e9459bf849712e3f27b418863_51.png)

![](img/6af4433e9459bf849712e3f27b418863_55.png)

上一节我们介绍了环境准备，本节中我们来看看创建虚拟机的具体步骤。

1.  打开VMware Workstation 16，点击“创建新的虚拟机”。
2.  选择“自定义（高级）”配置，点击下一步。
3.  硬件兼容性选择默认的“Workstation 16.x”，点击下一步。
4.  选择“稍后安装操作系统”，点击下一步。
5.  客户机操作系统选择“Linux”，版本选择“Red Hat Enterprise Linux 8 64位”，点击下一步。
6.  为虚拟机命名（例如`RHEL8-ServerA`），并选择虚拟机文件的存储位置，点击下一步。
7.  处理器配置：数量选择1，每个处理器的核心数选择2（即2核CPU），点击下一步。
8.  内存配置：为此虚拟机分配2048 MB（即2GB）内存，点击下一步。
9.  网络类型：选择“使用网络地址转换（NAT）”，这便于后续从外部主机连接，点击下一步。
10. I/O控制器类型和磁盘类型均使用推荐选项，点击下一步。
11. 选择“创建新虚拟磁盘”，点击下一步。
12. 指定磁盘容量：例如20 GB，选择“将虚拟磁盘存储为单个文件”，点击下一步。
13. 指定磁盘文件名称，使用默认即可，点击下一步。
14. 点击“完成”，虚拟机硬件创建完毕。

## 安装RHEL 8操作系统 📀

虚拟机创建完成后，我们需要挂载系统镜像并开始安装。

![](img/6af4433e9459bf849712e3f27b418863_63.png)

1.  在虚拟机设置中，点击“CD/DVD (SATA)”，选择“使用ISO映像文件”，浏览并加载RHEL 8.2的ISO镜像文件。
2.  移除不需要的硬件（如打印机），点击“确定”保存设置。
3.  启动虚拟机。在启动界面，使用键盘上下键选择第一项“Install Red Hat Enterprise Linux 8.2”并按回车，开始安装。
4.  系统加载完成后，进入安装向导。语言和键盘均选择“English”，点击“Continue”。
5.  在“INSTALLATION SUMMARY”界面，需要配置以下三项：
    *   **TIME & DATE**：时区选择“Asia/Shanghai”，并手动设置正确时间。
    *   **SOFTWARE SELECTION**：选择安装模式。对于学习环境，选择带图形界面的“Server with GUI”。生产环境通常选择更精简的“Server”或“Minimal Install”。
    *   **INSTALLATION DESTINATION**：点击进入磁盘分区界面。对于实验环境，直接使用自动配置，点击“Done”即可。
6.  完成以上配置后，右下角的“Begin Installation”按钮变为可用，点击开始安装。
7.  安装过程中，设置**root用户密码**。输入密码（例如`redhat`）并确认。若密码强度太弱，系统会提示“Weak password”，需要点击“Done”两次以确认。
8.  同时，可以创建一个普通用户（例如用户名为`student`，密码也为`student`）。同样，若密码弱需点击两次“Done”。
9.  等待安装进度条完成，点击“Reboot System”重启。

![](img/6af4433e9459bf849712e3f27b418863_65.png)

重启后，系统首次启动会要求接受许可协议，点击同意即可完成初始化设置。

![](img/6af4433e9459bf849712e3f27b418863_80.png)

### 配置网络

![](img/6af4433e9459bf849712e3f27b418863_84.png)

1.  启动虚拟机，以root用户登录图形界面。
2.  打开终端（Terminal）。
3.  检查网卡名称，通常为`ens160`或类似。使用命令 `ip addr` 查看。
4.  编辑网卡配置文件，启用开机自动连接并获取IP地址。
    ```bash
    vi /etc/sysconfig/network-scripts/ifcfg-ens160
    ```
5.  找到 `ONBOOT=no` 这一行，将其改为 `ONBOOT=yes`，保存退出。
6.  重启网络服务使配置生效。
    ```bash
    nmcli connection reload
    nmcli connection up ens160
    ```
7.  使用 `ip addr` 命令再次查看，确认网卡已获取到IP地址（例如 `192.168.131.128`）。

### 使用CRT远程连接

![](img/6af4433e9459bf849712e3f27b418863_89.png)

![](img/6af4433e9459bf849712e3f27b418863_91.png)

![](img/6af4433e9459bf849712e3f27b418863_93.png)

![](img/6af4433e9459bf849712e3f27b418863_95.png)

![](img/6af4433e9459bf849712e3f27b418863_97.png)

![](img/6af4433e9459bf849712e3f27b418863_99.png)

![](img/6af4433e9459bf849712e3f27b418863_101.png)

1.  在物理机（Windows系统）上打开SecureCRT软件。
2.  点击“File” -> “Quick Connect”。
3.  在对话框中输入虚拟机的IP地址、端口（22）、用户名（如`root`）。
4.  点击“Connect”，输入密码，即可成功连接到虚拟机。
5.  CRT的优势在于可以多标签管理会话、方便地复制粘贴命令、通过SFTP上传下载文件，并能记录会话日志，非常适合服务器管理。

## 克隆虚拟机与修改配置 🔄

![](img/6af4433e9459bf849712e3f27b418863_110.png)

![](img/6af4433e9459bf849712e3f27b418863_111.png)

![](img/6af4433e9459bf849712e3f27b418863_113.png)

![](img/6af4433e9459bf849712e3f27b418863_115.png)

为了后续实验，我们可能需要多台相同配置的虚拟机。可以通过克隆快速创建。

![](img/6af4433e9459bf849712e3f27b418863_117.png)

![](img/6af4433e9459bf849712e3f27b418863_119.png)

1.  将源虚拟机（如ServerA）正常关机。
2.  在VMware中，右键点击该虚拟机，选择“管理” -> “克隆”。
3.  在克隆向导中，选择“虚拟机中的当前状态”，点击下一步。
4.  克隆类型选择“创建完整克隆”，点击下一步。
5.  为新虚拟机命名（如`ServerB`）并选择存储位置，点击“完成”。
6.  克隆完成后，启动ServerB。由于克隆机的唯一标识（如UUID）与源机相同，需要修改以避免冲突。
7.  修改主机名：
    ```bash
    hostnamectl set-hostname serverb.example.com
    ```
8.  修改网卡UUID（可选但推荐）：编辑网卡配置文件，删除或注释掉 `UUID` 这一行。
    ```bash
    vi /etc/sysconfig/network-scripts/ifcfg-ens160
    # 在UUID行前加#号注释，或直接删除该行
    ```
9.  重启网络服务。由于是DHCP获取，克隆机会自动获得一个新的IP地址（如`192.168.131.129`）。
10. 现在可以使用CRT分别连接ServerA和ServerB，它们处于同一网络，可以互相通信。

## 重置Root用户密码 🔑

![](img/6af4433e9459bf849712e3f27b418863_134.png)

![](img/6af4433e9459bf849712e3f27b418863_136.png)

如果忘记了root用户的密码，可以通过以下步骤在单用户模式下重置。

![](img/6af4433e9459bf849712e3f27b418863_138.png)

![](img/6af4433e9459bf849712e3f27b418863_139.png)

![](img/6af4433e9459bf849712e3f27b418863_140.png)

![](img/6af4433e9459bf849712e3f27b418863_142.png)

![](img/6af4433e9459bf849712e3f27b418863_144.png)

以下是操作步骤：
1.  重启目标虚拟机。在VMware启动界面出现时，迅速按下键盘的 `e` 键，进入内核编辑模式。
2.  找到以 `linux` 开头的一行，将光标移动至该行末尾。
3.  在行末添加 `rd.break` 参数（注意前面有个空格），然后按 `Ctrl+X` 启动。
4.  系统会进入一个临时的`switch_root` shell环境。
5.  以读写方式重新挂载根文件系统：
    ```bash
    mount -o remount,rw /sysroot
    ```
6.  切换根目录到`/sysroot`：
    ```bash
    chroot /sysroot
    ```
7.  修改root密码：
    ```bash
    passwd root
    ```
    输入新密码并确认。
8.  如果系统启用了SELinux，必须更新文件安全上下文标签：
    ```bash
    touch /.autorelabel
    ```
9.  退出`chroot`环境并重启系统：
    ```bash
    exit
    reboot
    ```
10. 系统重启后，即可使用新设置的root密码登录。
