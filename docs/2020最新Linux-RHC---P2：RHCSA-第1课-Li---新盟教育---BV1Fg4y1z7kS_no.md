## 操作系统启动概念与流程 🔄

上一节我们完成了Linux系统的安装，本节中我们来看看计算机是如何启动并加载操作系统的。这个过程涉及硬件、固件和软件的协同工作。

### 核心概念

1.  **BIOS (基本输入输出系统)**
    *   全称：Basic Input/Output System。
    *   性质：它是一个存储在主板ROM芯片中的固件程序。
    *   功能：进行开机自检（POST），初始化硬件，并定位可启动设备（如硬盘、U盘）。
    *   关键设置：例如，开启CPU虚拟化支持（`Virtualization Technology`）通常需要在BIOS中配置。

2.  **MBR (主引导记录)**
    *   位置：硬盘的第一个扇区（512字节）。
    *   功能：存储磁盘分区信息和一小段引导程序（Boot Loader）。它是系统加载时，BIOS读取硬盘后执行的第一个程序。
    *   限制：MBR分区表只能支持最大2TB的硬盘，且最多4个主分区。
    *   引导地址：MBR的内容会被加载到物理内存地址 `0x7C00` 处执行，这是为了兼容早期的8088处理器。

3.  **GPT (GUID 分区表)**
    *   作用：正在逐步取代MBR的新分区方案。
    *   优势：支持超过2TB的大容量硬盘，分区数量几乎没有限制。
    *   搭配：通常与UEFI（统一可扩展固件接口）启动方式配合使用。

![](img/b3d5381a456d0974fe21e00860a02566_9.png)

![](img/b3d5381a456d0974fe21e00860a02566_11.png)

4.  **GRUB (GRand Unified Bootloader)**
    *   作用：多操作系统引导程序。当电脑安装多个系统（如Windows和Linux）时，GRUB会提供一个菜单供用户选择要启动的系统。
    *   版本：CentOS 6使用GRUB，CentOS 7使用GRUB 2。

![](img/b3d5381a456d0974fe21e00860a02566_13.png)

![](img/b3d5381a456d0974fe21e00860a02566_15.png)

### 启动流程详解
以下是Linux系统启动的十个关键步骤：

![](img/b3d5381a456d0974fe21e00860a02566_17.png)

![](img/b3d5381a456d0974fe21e00860a02566_19.png)

1.  **通电**：为计算机硬件供电。
2.  **BIOS自检**：BIOS检测并初始化关键硬件（CPU、内存、硬盘等）。
3.  **读取MBR/GPT**：BIOS根据设置，从指定启动设备读取MBR或GPT中的引导信息。
4.  **加载GRUB**：执行MBR中的引导程序，加载GRUB。
5.  **加载内核**：根据GRUB的配置，加载Linux内核镜像（`vmlinuz-*`）到内存。
6.  **初始化进程**：内核启动第一个用户空间进程 `init`（CentOS 6）或 `systemd`（CentOS 7）。
7.  **确定运行级别**：根据配置，确定系统的运行模式。
8.  **加载内核模块**：读取 `/etc/modprobe.d/` 等目录的配置，加载必要的硬件驱动模块。
9.  **执行启动脚本**：根据运行级别，按顺序执行 `/etc/rc.d/rc[0-6].d/` 目录下的启动（`S`开头）或停止（`K`开头）脚本。
10. **启动登录管理器**：最后，启动登录界面（如 `getty`），等待用户输入用户名和密码。

### 运行级别 (Runlevel)
Linux系统定义了不同的运行级别，代表不同的操作模式。

以下是各级别的含义：
*   **0**: 关机。不可设为默认级别。
*   **1**: 单用户模式。类似Windows安全模式，仅root可登录，用于系统修复（如重置root密码）。
*   **2**: 多用户模式（无网络支持）。不带NFS服务。
*   **3**: 完整的多用户模式（命令行界面）。服务器最常用的级别。
*   **4**: 保留未使用。
*   **5**: 图形界面模式。安装了桌面环境后默认进入的级别。
*   **6**: 重启。不可设为默认级别。

**查看与切换运行级别命令**：
```bash
# 查看当前和上一次运行级别
runlevel
# 输出示例：N 3 （N表示未切换过，当前是3）

![](img/b3d5381a456d0974fe21e00860a02566_25.png)

![](img/b3d5381a456d0974fe21e00860a02566_27.png)

![](img/b3d5381a456d0974fe21e00860a02566_31.png)

![](img/b3d5381a456d0974fe21e00860a02566_33.png)

# 临时切换运行级别（例如切换到图形界面）

# 或
systemctl isolate graphical.target (CentOS 7)
```

![](img/b3d5381a456d0974fe21e00860a02566_35.png)

**运行级别对应的脚本目录**：
在 `/etc/rc.d/` 目录下，有 `rc0.d` 到 `rc6.d` 的文件夹，分别对应各个运行级别。里面的脚本文件以 `S`（Start）或 `K`（Kill）开头，后面跟数字表示启动/停止顺序。
```bash
ls -l /etc/rc.d/rc3.d/
```

上一节我们了解了系统启动的通用流程，本节中我们来看看CentOS的两个主要版本在管理上的重要差异。理解这些区别对于运维工作至关重要。

以下是CentOS 6和7的主要区别列表：

![](img/b3d5381a456d0974fe21e00860a02566_63.png)

![](img/b3d5381a456d0974fe21e00860a02566_64.png)

![](img/b3d5381a456d0974fe21e00860a02566_66.png)

| 特性 | CentOS 6 | CentOS 7 | 说明 |
| :--- | :--- | :--- | :--- |
| **初始化系统** | `init` (SysVinit) | `systemd` | 7用systemd统一管理进程和服务，速度更快。 |
| **默认文件系统** | `ext4` | `xfs` | xfs支持更大容量和文件，但ext4更成熟稳定。 |
| **内核版本** | 2.6.x | 3.10.x | 内核更新带来更好的硬件支持和性能。 |
| **引导程序** | GRUB | GRUB 2 | GRUB 2配置更灵活。 |
| **防火墙** | `iptables` | `firewalld` (底层仍为iptables) | firewalld动态管理规则，但生产环境仍多用iptables。 |
| **默认数据库** | MySQL | MariaDB | MariaDB是MySQL的分支，完全兼容。 |
| **主机名修改** | 修改 `/etc/sysconfig/network` | 使用 `hostnamectl` 命令或修改 `/etc/hostname` | 7的修改方式更直接。 |
| **时间同步** | `ntpd` 服务 | `chronyd` 服务 | chrony更快更精准，同时兼容ntpd。 |
| **服务管理命令** | `service 服务名 start/stop` | `systemctl start/stop 服务名` | systemctl是7的核心管理命令。 |
| **开机自启命令** | `chkconfig 服务名 on/off` | `systemctl enable/disable 服务名` | |

### 1. 网络配置
网卡配置文件位于 `/etc/sysconfig/network-scripts/` 目录下，通常名为 `ifcfg-ens33` 或类似。

![](img/b3d5381a456d0974fe21e00860a02566_97.png)

![](img/b3d5381a456d0974fe21e00860a02566_99.png)

![](img/b3d5381a456d0974fe21e00860a02566_100.png)

**关键配置项**：
```bash
TYPE=Ethernet
BOOTPROTO=none # 静态IP设为‘none‘或‘static‘；DHCP设为‘dhcp‘
NAME=ens33
DEVICE=ens33
ONBOOT=yes # 开机自启
IPADDR=192.168.1.100 # IP地址
PREFIX=24 # 或使用 NETMASK=255.255.255.0
GATEWAY=192.168.1.1 # 网关
DNS1=8.8.8.8 # 主DNS
DNS2=114.114.114.114 # 备DNS
```
**修改DNS的注意事项**：修改DNS最有效的方法是直接在网卡配置文件中设置。`/etc/resolv.conf` 文件是由网络服务动态生成的，直接修改可能无效。

### 2. 配置YUM软件仓库
YUM是RPM包的高级管理工具，能自动解决依赖关系。我们需要为其配置软件源（仓库）。

![](img/b3d5381a456d0974fe21e00860a02566_109.png)

![](img/b3d5381a456d0974fe21e00860a02566_111.png)

![](img/b3d5381a456d0974fe21e00860a02566_113.png)

![](img/b3d5381a456d0974fe21e00860a02566_114.png)

**配置本地YUM源步骤**：
1.  挂载系统安装镜像到目录（如 `/mnt`）。
    ```bash
    mount /dev/cdrom /mnt
    ```
2.  在 `/etc/yum.repos.d/` 目录下创建新的 `.repo` 文件（例如 `local.repo`）。
    ```bash
    vi /etc/yum.repos.d/local.repo
    ```
3.  写入以下内容：
    ```ini
    [local-base] # 仓库唯一ID
    name=Local Repository # 仓库描述
    baseurl=file:///mnt # 仓库路径，file://表示本地
    enabled=1 # 启用此仓库
    gpgcheck=0 # 不进行GPG签名校验（本地源可关闭）
    ```
4.  清除缓存并测试。
    ```bash
    yum clean all
    yum makecache
    yum install -y tree # 测试安装一个小软件
    ```

### 实战：重置Root密码（单用户模式）
忘记root密码时，可以通过单用户模式重置。

![](img/b3d5381a456d0974fe21e00860a02566_172.png)

![](img/b3d5381a456d0974fe21e00860a02566_173.png)

![](img/b3d5381a456d0974fe21e00860a02566_175.png)

![](img/b3d5381a456d0974fe21e00860a02566_177.png)

**操作步骤**：
1.  重启系统，在GRUB菜单界面，按 `e` 键编辑启动参数。
2.  找到以 `linux16` 开头的行，将光标移到最后，添加 `rd.break` 参数。
3.  按 `Ctrl + X` 启动，系统会进入紧急救援模式。
4.  重新挂载根目录为可读写：
    ```bash
    mount -o remount,rw /sysroot
    ```
5.  切换根目录环境：
    ```bash
    chroot /sysroot
    ```
6.  修改root密码：
    ```bash
    echo "你的新密码" | passwd --stdin root
    ```
    *（如果密码中包含难以区分的字符如‘0‘和‘O‘，此方法可确保准确性）*
7.  创建自动重新标记SELinux上下文文件（重要）：
    ```bash
    touch /.autorelabel
    ```
8.  退出并重启：
    ```bash
    exit
    reboot
    ```
