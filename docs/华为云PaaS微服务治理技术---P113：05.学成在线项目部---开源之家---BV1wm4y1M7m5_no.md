上一节我们已经完成了Dockerfile和Maven插件的配置。本节中，我们来看看如何执行构建脚本，将代码转化为云平台上的可用镜像。

## 脚本配置回顾与建议 📝

刚才我们已经完成了构建镜像所需脚本的编写，包括Dockerfile文件和Maven插件的配置。

对于Maven插件的配置，其核心参数均引用了我们在项目配置中定义的变量名称，并未写死。因此，该配置可以被复用于其他微服务模块的镜像构建。

以下是配置复用的建议：
*   建议为Docker镜像构建单独创建一个Maven Profile，例如命名为 `docker`。这样，当我们需要构建Docker镜像时，只需激活此Profile即可，实现关注点分离。

## 执行构建：代码提交与克隆 🔄

![](img/6d5d0fe1d62b32bf454564e3ae80580e_7.png)

现在，我们开始执行构建流程。首先，需要将本地代码推送到GitLab代码仓库。

![](img/6d5d0fe1d62b32bf454564e3ae80580e_9.png)

![](img/6d5d0fe1d62b32bf454564e3ae80580e_10.png)

> 注：GitLab的安装与搭建过程不在本课程范围内，可参考学成在线项目相关课程或自行查阅资料。

![](img/6d5d0fe1d62b32bf454564e3ae80580e_12.png)

假设GitLab环境已就绪，我们需要将修改后的代码文件提交并推送至远程仓库。

![](img/6d5d0fe1d62b32bf454564e3ae80580e_14.png)

1.  将代码提交至本地Git仓库。
2.  将本地提交推送（Push）至远程GitLab仓库。

![](img/6d5d0fe1d62b32bf454564e3ae80580e_16.png)

推送成功后，可以在GitLab仓库中查看到新增的构建脚本文件，例如 `image_build.sh`。

接下来，我们需要登录到用于构建的服务器（本地或CI/CD服务器），从GitLab克隆代码。

1.  登录服务器，进入合适目录，例如创建并进入 `/root/xcedu-pod-02`。
2.  执行Git克隆命令，将远程仓库代码拉取到服务器本地。
    ```bash
    git clone http://your-gitlab-server:port/your-project.git
    ```
    克隆过程中需要输入GitLab的账号密码进行认证。

![](img/6d5d0fe1d62b32bf454564e3ae80580e_18.png)

## 执行构建：认证与脚本运行 ⚙️

![](img/6d5d0fe1d62b32bf454564e3ae80580e_20.png)

代码克隆成功后，即可执行构建脚本。但在执行前，有一个关键步骤：登录华为云容器镜像仓库。

若未登录，则无法将本地构建的镜像推送（Push）至云平台。因此，需要先在服务器上执行Docker登录命令。

1.  从华为云SWR容器镜像服务获取临时的登录指令。
2.  在服务器上执行该指令，完成一次性认证。
    ```bash
    docker login -u [区域项目名]@[AK] -p [临时密码] [镜像仓库地址]
    ```

![](img/6d5d0fe1d62b32bf454564e3ae80580e_22.png)

认证通过后，即可运行构建脚本。我们以 `image_build.sh` 脚本为例。

![](img/6d5d0fe1d62b32bf454564e3ae80580e_24.png)

以下是脚本执行的核心步骤解析：
*   **清理**：删除服务器上已有的旧版本镜像。
*   **安装依赖**：将父工程及公共模块打包并安装到本地Maven仓库，为后续编译提供依赖。
*   **编译打包**：进入微服务工程目录，执行Maven命令进行清理和打包。
    ```bash
    mvn clean package -P docker
    ```
*   **构建镜像**：使用Docker命令，根据Dockerfile构建新的应用镜像。
    ```bash
    docker build -t [镜像名称:版本] .
    ```
*   **重命名与推送**：将构建好的本地镜像重命名为符合云平台规范的名称，并推送至华为云SWR仓库。
    ```bash
    docker tag [旧镜像名] [新镜像名]
    docker push [新镜像名]
    ```

**重要提示**：执行脚本时，通常需要传入版本号等参数。例如，若脚本中通过 `$1` 引用第一个参数，则执行命令应为：
```bash
./image_build.sh v1.0
```

![](img/6d5d0fe1d62b32bf454564e3ae80580e_26.png)

![](img/6d5d0fe1d62b32bf454564e3ae80580e_27.png)

脚本执行过程是全自动的。成功执行后，你将看到镜像构建成功以及向云平台推送进度的日志。
