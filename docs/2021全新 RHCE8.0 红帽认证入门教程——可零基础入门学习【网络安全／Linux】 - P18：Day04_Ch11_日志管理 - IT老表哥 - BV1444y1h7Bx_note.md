# 红帽认证入门教程：Day04：日志管理 📝

在本节课中，我们将要学习Linux系统中的日志管理。日志是系统运行的“摄像头”，它详细记录了用户的操作、服务的状态以及系统的各种事件。理解日志的生成、分类、查看和管理，是系统运维和网络安全的基础技能。

---

## 日志系统概述

一个完备的操作系统都拥有日志系统。在RHEL 8系统中，日志主要通过 **rsyslog** 协议来记录原始信息。为了使这些原始信息变得可读，需要其他服务进行整理。

上一节我们介绍了日志的基本概念，本节中我们来看看两个核心服务的关系。

*   **rsyslog**：这是一个日志记录协议和标准。
*   **systemd-journald**：这是一个记录与系统服务、开机引导相关日志的服务。它只在服务运行时记录日志。

`systemd-journald` 服务默认已在运行。它以二进制形式将日志保存在内存中。系统重启后，这些内存中的日志会被清空。

那么，我们平常如何查看这些日志呢？这就要依靠 `rsyslog.service` 服务。它的作用是将 `systemd-journald.service` 保存的二进制日志流，实时或定期地转换成文本文档，并保存在 `/var/log/` 目录下。

## 日志分类与格式

理解了日志的流转过程后，我们来看看系统中有哪些常见的日志文件。它们都位于 `/var/log/` 目录下。

以下是几个最重要的日志文件及其作用：

![](img/7ff9fcd98e489d5ba100d37142290828_5.png)

*   **`/var/log/messages`**：记录大多数系统日志消息，内容最全也最杂。但身份验证、邮件处理等特定日志不在此处。
*   **`/var/log/secure`**：记录所有与安全和身份验证相关的日志，例如用户登录成功或失败。这类似于Windows系统中的“安全审核”日志。
*   **`/var/log/maillog`**：记录与邮件服务器（如Postfix）相关的所有日志。
*   **`/var/log/cron`**：记录所有计划任务（cron job）的执行结果。
*   **`/var/log/boot.log`**：记录系统启动过程中的消息。

系统日志通常有固定的格式，便于阅读和工具分析。一条典型的日志格式如下：

```
时间戳 主机名 服务名[进程号]: 具体事件描述
```

![](img/7ff9fcd98e489d5ba100d37142290828_7.png)

![](img/7ff9fcd98e489d5ba100d37142290828_9.png)

![](img/7ff9fcd98e489d5ba100d37142290828_11.png)

例如一条SSH登录日志：
```
Jul  3 21:13:24 localhost sshd[36698]: Accepted password for root from 192.168.1.100 port 50000 ssh2
```

![](img/7ff9fcd98e489d5ba100d37142290828_13.png)

![](img/7ff9fcd98e489d5ba100d37142290828_15.png)

*   `Jul 3 21:13:24`：事件发生的时间。
*   `localhost`：产生该日志的主机名。
*   `sshd[36698]`：产生日志的服务名称 (`sshd`) 及其进程号 (`36698`)。
*   `Accepted password...`：具体的日志事件描述。

![](img/7ff9fcd98e489d5ba100d37142290828_17.png)

此外，许多应用程序（如Apache HTTPD, Nginx）会定义自己的日志格式和存储路径，通常在它们的配置文件中设置。

## 查看日志的方法

![](img/7ff9fcd98e489d5ba100d37142290828_29.png)

在日常运维中，我们需要知道如何查看日志。根据日志类型，主要有以下几种方法：

![](img/7ff9fcd98e489d5ba100d37142290828_31.png)

1.  **查看系统通用日志**：直接查看 `/var/log/` 目录下的文件，如 `messages`, `secure` 等。
2.  **查看服务状态及日志**：使用 `systemctl status` 命令，它可以显示服务的状态和最近10条日志。
    ```bash
    systemctl status sshd.service
    ```
3.  **查看特定应用日志**：根据应用自身的配置，查看其指定的日志文件，例如Apache的 `/var/log/httpd/access_log`。
4.  **查看内存日志（journal）**：使用 `journalctl` 命令查看由 `systemd-journald` 管理的日志。
    ```bash
    journalctl -u sshd.service # 查看sshd服务的日志
    journalctl --since "2021-07-03 09:00:00" --until "2021-07-03 10:00:00" # 查看特定时间段的日志
    ```

## 配置 rsyslog 日志规则

![](img/7ff9fcd98e489d5ba100d37142290828_47.png)

![](img/7ff9fcd98e489d5ba100d37142290828_49.png)

`rsyslog` 的强大之处在于可以自定义日志规则。其配置文件为 `/etc/rsyslog.conf`。

![](img/7ff9fcd98e489d5ba100d37142290828_51.png)

![](img/7ff9fcd98e489d5ba100d37142290828_52.png)

![](img/7ff9fcd98e489d5ba100d37142290828_54.png)

![](img/7ff9fcd98e489d5ba100d37142290828_56.png)

在该文件中，`RULES` 部分定义了日志的筛选和存储规则。规则的基本格式是：
```
设施.优先级 动作
```
*   **设施**：指定日志来源，如 `auth`（认证）、`mail`（邮件）、`kern`（内核）等。
*   **优先级**：指定日志级别，从高到低为：`emerg` > `alert` > `crit` > `err` > `warning` > `notice` > `info` > `debug`。
*   **动作**：指定日志的去向，如保存到文件 (`/var/log/xxx`)、发送到远程主机或显示到控制台。

![](img/7ff9fcd98e489d5ba100d37142290828_58.png)

![](img/7ff9fcd98e489d5ba100d37142290828_60.png)

例如，以下规则将所有认证相关的、级别在 `info` 及以上的日志保存到 `/var/log/auth.log`：
```
auth.info /var/log/auth.log
```

## 日志轮转（Log Rotation）

![](img/7ff9fcd98e489d5ba100d37142290828_73.png)

![](img/7ff9fcd98e489d5ba100d37142290828_75.png)

如果日志只写入一个文件，该文件会无限增大。为了解决这个问题，系统使用 **logrotate** 工具进行日志轮转。

![](img/7ff9fcd98e489d5ba100d37142290828_77.png)

![](img/7ff9fcd98e489d5ba100d37142290828_78.png)

![](img/7ff9fcd98e489d5ba100d37142290828_80.png)

![](img/7ff9fcd98e489d5ba100d37142290828_82.png)

![](img/7ff9fcd98e489d5ba100d37142290828_84.png)

![](img/7ff9fcd98e489d5ba100d37142290828_85.png)

日志轮转规则主要有两种：
1.  **按时间**：例如每天、每周轮转一次。
2.  **按大小**：当日志文件达到指定大小时（如10MB）进行轮转。

![](img/7ff9fcd98e489d5ba100d37142290828_86.png)

![](img/7ff9fcd98e489d5ba100d37142290828_88.png)

![](img/7ff9fcd98e489d5ba100d37142290828_90.png)

![](img/7ff9fcd98e489d5ba100d37142290828_92.png)

通常两者可以结合使用。轮转时，旧日志文件会被重命名（通常加上日期后缀），然后新建一个空日志文件继续写入，并可以配置压缩或删除过旧的日志备份。

系统默认的轮转配置文件是 `/etc/logrotate.conf`。针对特定服务的轮转配置则放在 `/etc/logrotate.d/` 目录下，例如 `/etc/logrotate.d/httpd`。

一个简单的轮转配置示例：
```
/var/log/myapp.log {
    daily          # 每天轮转一次
    rotate 7       # 保留最近7个备份
    compress       # 压缩旧日志
    missingok      # 如果日志文件不存在，不报错
    notifempty     # 如果日志文件为空，不轮转
}
```

## 管理 systemd-journal 日志

之前提到，`systemd-journald` 的日志默认存储在内存中。我们也可以将其持久化保存到磁盘。

**启用 journal 日志持久化：**
1.  创建存储目录：`sudo mkdir -p /var/log/journal`
2.  编辑配置文件：`sudo vim /etc/systemd/journald.conf`
3.  找到 `#Storage=auto` 这一行，取消注释并改为 `Storage=persistent`
4.  重启服务：`sudo systemctl restart systemd-journald`

之后，日志将持久化保存在 `/var/log/journal/` 目录下，但仍需使用 `journalctl` 命令查看。

## 系统时间与时钟同步

系统时间的准确性对日志分析、计划任务等至关重要。RHEL 8 使用 **chronyd** 服务进行网络时间协议（NTP）同步。

**查看和设置时区：**
```bash
timedatectl status # 查看当前时间和时区
sudo timedatectl set-timezone Asia/Shanghai # 设置时区为上海（东八区）
```

![](img/7ff9fcd98e489d5ba100d37142290828_98.png)

**配置 NTP 时间同步：**
1.  编辑配置文件：`sudo vim /etc/chrony.conf`
2.  在 `pool` 部分添加或修改NTP服务器地址。例如，指向内部时间服务器：
    ```
    server classroom.example.com iburst
    ```
    `iburst` 参数可以让客户端在启动后快速进行时间同步。
3.  重启服务：`sudo systemctl restart chronyd`
4.  检查同步状态：`chronyc sources -v`

![](img/7ff9fcd98e489d5ba100d37142290828_100.png)

![](img/7ff9fcd98e489d5ba100d37142290828_101.png)

![](img/7ff9fcd98e489d5ba100d37142290828_103.png)

如果需要手动设置时间，需要先关闭NTP同步：
```bash
sudo timedatectl set-ntp false
sudo timedatectl set-time “2021-07-04 12:00:00”
sudo timedatectl set-ntp true # 设置完成后重新开启
```
