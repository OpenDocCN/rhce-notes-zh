## Ansible简介

Ansible是一个开源的自动化平台，也是一种简单的自动化语言。它能够通过编写剧本（Playbook）来完美地描述和管理IT基础架构。与人工部署相比，Ansible可以运行剧本来自动化复杂的任务，并能适应不同的工作流程，从而极大地提升工作效率。

## Ansible的核心特点

以下是Ansible的几个主要特点：

*   **简单易用**：Ansible提供人类可读的自动化脚本（剧本）。这些剧本易于阅读、理解和修改，不需要掌握复杂的编程技能。其设计使得初学者也能快速上手并看到成效。
*   **功能强大**：Ansible主要用于配置管理、工作流程自动化和网络自动化。它可以编排从应用部署、上线到回收的整个生命周期。
*   **无需代理**：Ansible不需要在受管主机上安装任何代理程序。它通过SSH协议（通常使用基于密钥的免密认证）来管理主机并运行任务。任务执行后，相关模块会被清理，这使得它更加高效和安全。

## Ansible的基本架构

理解Ansible的架构是掌握其用法的关键。它的工作模式类似于一个指挥部指挥士兵：

*   **控制节点 (Controller Node)**：也称为管理节点，是发出指令的“司令部”。它负责定义任务（剧本）和指定执行任务的主机（清单）。
*   **受管主机 (Managed Nodes)**：是接受并执行命令的“士兵”。它们只需要与控制节点建立SSH信任关系，无需安装额外软件。
*   **清单 (Inventory)**：定义了所有受管主机的列表，可以按IP、主机名分组。控制节点通过清单知道要对哪些主机发号施令。
*   **剧本 (Playbook)**：定义了要执行的具体任务列表，相当于指挥部的“作战计划”或演员的“台本”。它按顺序描述了一系列操作。
*   **模块 (Module)**：是Ansible执行特定任务（如创建用户、安装软件包）的小程序。剧本中的每个任务都会调用一个模块。

**简单流程**：控制节点通过SSH连接到清单中的受管主机，并将需要执行的模块推送到主机上运行，以达到预期的系统状态。

## RHCE8.0 (EX294) 考试要点

![](img/2f1758469b33a35a9c2f56a9ea613b40_35.png)

![](img/2f1758469b33a35a9c2f56a9ea613b40_37.png)

上一节我们介绍了Ansible的基本架构，本节中我们来看看它在RHCE8.0考试中的具体应用。EX294考试主要考察使用Ansible进行批量服务器管理的能力。

考试满分300分，及格线210分，时长为4小时。主要考察范围包括以下10个部分：

1.  **理解Ansible核心组件**：了解控制节点、受管主机、模块、清单等核心概念。
2.  **安装和配置Ansible控制节点**：在控制节点上正确安装和设置Ansible。
3.  **配置受管节点和运行临时命令**：建立SSH免密认证，并使用`ansible`命令行工具执行临时任务（例如配置软件仓库）。
4.  **使用Ansible模块执行系统管理任务**：使用模块完成如软件包管理、服务管理、文件操作、磁盘管理等任务。
5.  **创建Ansible剧本**：编写Playbook来定义和执行一系列自动化任务。
6.  **使用Jinja2模板管理配置文件**：利用模板动态生成自定义的配置文件。
7.  **使用变量和事实**：在Playbook中定义和使用变量，并利用从受管主机收集的“事实”（系统信息）。
8.  **创建和使用角色**：将复杂的Playbook组织成可重用的“角色”。
9.  **从Ansible Galaxy下载和使用角色**：使用`ansible-galaxy`工具获取社区共享的角色。
10. **使用Ansible Vault保护敏感数据**：对剧本中的密码等敏感信息进行加密。
