## 联合查询的注意事项

上一节我们介绍了连接查询和联合查询，本节中我们来看看联合查询使用时需要特别注意的关键点。

![](img/632e816642889baff36577c27562ba89_7.png)

联合查询使用 `UNION` 连接两个 `SELECT` 语句，其查询结果必须保证列数相同，否则执行会报错。但列数相同只是最基本的要求。

例如，以下查询虽然列数相同，但数据含义不一致，导致查询结果混乱：
```sql
SELECT id, name FROM students
UNION
SELECT id, class_name FROM class;
```
查询结果的前半部分是学生的ID和姓名，后半部分则是班级的ID和名称。这样的联合查询在实际应用中意义不大。

因此，联合查询连接的两个 `SELECT` 语句，其查询结果**从列数到每列的含义都应保持一致**，这样查询出的数据才是清晰、有意义的。通常，联合查询用于合并结构完全相同的两个数据集（例如，两个分公司的销售数据表）。

## 多表查询内容总结

![](img/632e816642889baff36577c27562ba89_11.png)

以下是关于SELECT查询，包括单表和多表查询的核心用法总结：

![](img/632e816642889baff36577c27562ba89_13.png)

![](img/632e816642889baff36577c27562ba89_15.png)

![](img/632e816642889baff36577c27562ba89_17.png)

**SELECT子句**：用于指定要查询的字段。
*   `*`：查询所有字段。
*   字段名：查询指定字段。
*   聚合函数：如 `COUNT()`, `SUM()`, `AVG()` 等。

![](img/632e816642889baff36577c27562ba89_19.png)

**FROM子句**：用于指定数据来源。
*   单表查询：直接写表名。
*   多表查询：写多个表名，并使用 `JOIN ... ON ...` 指定连接条件。
*   嵌套查询：可以将一个完整的 `SELECT` 查询语句（其结果是一个虚拟表）放在 `FROM` 后，但**必须为其起一个别名**。

**WHERE子句**：用于对数据进行首次过滤，条件非常灵活，支持各种运算符和逻辑判断。

**GROUP BY子句**：用于对结果进行分组，常与聚合函数配合使用。

**HAVING子句**：用于对分组后的结果进行二次过滤。`WHERE` 在分组前过滤，`HAVING` 在分组后过滤。

**ORDER BY子句**：用于对结果进行排序。

**LIMIT子句**：用于限制返回的记录数（分页），可以加在任何查询语句的末尾。

**多表连接查询**：MySQL主要支持三种连接方式。
*   内连接 (`INNER JOIN`)：返回两个表匹配的行。
*   左外连接 (`LEFT OUTER JOIN`)：返回左表所有行，以及右表匹配的行。
*   右外连接 (`RIGHT OUTER JOIN`)：返回右表所有行，以及左表匹配的行。
通过左外连接和右外连接的组合，可以实现其他四种连接效果（如全外连接），因为MySQL不直接支持 `FULL OUTER JOIN` 命令。

**嵌套查询**：将一个查询语句嵌套在另一个查询语句中。常见位置：
*   嵌套在 `WHERE` 子句中，作为一个条件值。
*   嵌套在 `FROM` 子句中，作为一个数据源（必须起别名）。
嵌套层数不宜过多，通常一至两层即可，否则语句会过于冗长复杂。

## 复制表的操作

查询结束后，我们接下来学习一个实用的小命令：复制表。复制表主要有两种用途：复制表结构和复制表数据。

![](img/632e816642889baff36577c27562ba89_21.png)

### 1. 复制表结构

使用 `CREATE TABLE ... LIKE ...` 命令可以完整复制源表的结构（包括字段、约束、索引等），但不会复制数据。
```sql
CREATE TABLE new_student LIKE students;
```
这条命令创建了一个名为 `new_student` 的新表，其结构与 `students` 表完全相同。这在你需要创建一个结构类似的表时非常有用，无需重新设计。

![](img/632e816642889baff36577c27562ba89_23.png)

### 2. 复制表结构和数据

![](img/632e816642889baff36577c27562ba89_25.png)

通常，完整的复制分为两步：先复制结构，再复制数据。
*   **复制结构**：同上，使用 `CREATE TABLE ... LIKE ...`。
*   **复制数据**：使用 `INSERT INTO ... SELECT ...` 命令。
```sql
INSERT INTO new_student SELECT * FROM students;
```
这条命令将 `students` 表中的所有数据插入到 `new_student` 表中。前提是两个表的结构必须一致。

![](img/632e816642889baff36577c27562ba89_27.png)

![](img/632e816642889baff36577c27562ba89_29.png)

### 3. 快速创建表并导入数据（不推荐用于完整复制）

![](img/632e816642889baff36577c27562ba89_31.png)

![](img/632e816642889baff36577c27562ba89_33.png)

还有一种方式是使用 `CREATE TABLE ... SELECT ...` 直接创建表并导入数据。
```sql
CREATE TABLE student_backup SELECT * FROM students;
```
但这种方法**主要复制数据**，表结构中的一些关键信息（如主键约束、自增属性、索引等）可能会丢失。因此，除非你只需要数据快照，否则建议使用前两种方法进行完整的表复制。

## 破解MySQL root密码

![](img/632e816642889baff36577c27562ba89_35.png)

如果忘记了MySQL的root用户密码，可以通过以下步骤在服务器本地进行破解重置。**请注意，此操作需要直接接触服务器，远程无法完成。**

![](img/632e816642889baff36577c27562ba89_37.png)

1.  **修改配置文件，跳过密码验证**：
    编辑MySQL的配置文件 `my.cnf`（位置可能为 `/etc/my.cnf` 或 `/usr/local/mysql/etc/my.cnf`，取决于安装方式）。
    在 `[mysqld]` 配置段中添加一行：
    ```
    skip-grant-tables
    ```
    保存并退出。这一行的作用是让MySQL服务启动时跳过权限验证。

![](img/632e816642889baff36577c27562ba89_39.png)

![](img/632e816642889baff36577c27562ba89_41.png)

![](img/632e816642889baff36577c27562ba89_43.png)

2.  **重启MySQL服务**：
    ```bash
    systemctl restart mysqld
    ```

3.  **免密登录并修改密码**：
    重启后，可以直接用root用户免密登录。
    ```bash
    mysql -u root
    ```
    登录后，使用 `UPDATE` 语句直接修改 `mysql.user` 表中的密码字段。
    ```sql
    UPDATE mysql.user SET authentication_string=PASSWORD('你的新密码') WHERE User='root';
    ```
    **注意**：MySQL 5.7和8.0的密码字段名和函数可能不同，上述示例适用于5.7。8.0版本可能需要使用 `ALTER USER` 命令。

4.  **恢复配置文件并重启服务**：
    密码修改成功后，再次编辑 `my.cnf` 文件，**注释掉或删除**刚才添加的 `skip-grant-tables` 行。
    保存后，再次重启MySQL服务。
    ```bash
    systemctl restart mysqld
    ```
    之后，就可以使用新密码正常登录了。

## 授权远程访问数据库

![](img/632e816642889baff36577c27562ba89_47.png)

默认情况下，MySQL只允许本地连接。如果需要从其他机器远程访问数据库，需要进行授权。

![](img/632e816642889baff36577c27562ba89_49.png)

![](img/632e816642889baff36577c27562ba89_51.png)

![](img/632e816642889baff36577c27562ba89_52.png)

![](img/632e816642889baff36577c27562ba89_54.png)

![](img/632e816642889baff36577c27562ba89_56.png)

![](img/632e816642889baff36577c27562ba89_58.png)

授权命令的基本格式如下：
```sql
GRANT 权限列表 ON 数据库名.表名 TO '用户名'@'客户端IP' IDENTIFIED BY '密码';
```

![](img/632e816642889baff36577c27562ba89_59.png)

*   **权限列表**：如 `ALL PRIVILEGES`（所有权限）、`SELECT`、`INSERT`、`UPDATE`等。
*   **数据库名.表名**：`*.*` 代表所有数据库的所有表。
*   **客户端IP**：指定允许连接的客户端IP地址。使用 `%` 代表允许任何IP连接（生产环境慎用）。
*   **密码**：该用户从指定IP连接时使用的密码。

**示例**：
1.  允许IP为 `192.168.1.100` 的机器，用户 `report` 用密码 `123456` 访问所有数据库的只读权限：
    ```sql
    GRANT SELECT ON *.* TO 'report'@'192.168.1.100' IDENTIFIED BY '123456';
    ```
2.  允许任何IP的用户 `admin` 用密码 `admin_pwd` 访问 `app_db` 数据库的所有权限：
    ```sql
    GRANT ALL PRIVILEGES ON app_db.* TO 'admin'@'%' IDENTIFIED BY 'admin_pwd';
    ```

![](img/632e816642889baff36577c27562ba89_61.png)

![](img/632e816642889baff36577c27562ba89_63.png)

**重要提示**：授权后，还需要确保服务器防火墙放行了MySQL的默认端口（3306）。否则，远程连接会被防火墙拦截。
*   **放行端口**：
    ```bash
    firewall-cmd --zone=public --add-port=3306/tcp --permanent
    firewall-cmd --reload
    ```
*   **或临时关闭防火墙（仅用于测试）**：
    ```bash
    systemctl stop firewalld
    ```
