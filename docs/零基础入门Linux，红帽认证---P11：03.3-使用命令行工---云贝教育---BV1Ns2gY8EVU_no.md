## 创建目录 📁

![](img/f98b1998d41f65e43affbb955d293ca3_9.png)

![](img/f98b1998d41f65e43affbb955d293ca3_11.png)

创建目录的命令是 `mkdir`。该命令在不同Linux发行版中选项略有差异，但核心功能一致。

![](img/f98b1998d41f65e43affbb955d293ca3_13.png)

以下是 `mkdir` 命令的常用选项：

![](img/f98b1998d41f65e43affbb955d293ca3_15.png)

*   **`-p`**：创建父目录。如果目标目录的上级目录不存在，此选项会一并创建。
*   **`-m`**：设置目录权限（mode）。类似于后续会学到的 `chmod` 命令。
*   **`-v`**：显示详细的操作信息。

![](img/f98b1998d41f65e43affbb955d293ca3_17.png)

如果忘记这些选项，可以使用 `mkdir --help` 命令查看帮助。

![](img/f98b1998d41f65e43affbb955d293ca3_19.png)

![](img/f98b1998d41f65e43affbb955d293ca3_21.png)

现在我们来演示 `mkdir` 命令的用法。首先，在当前目录创建一个名为 `download` 的目录：
```bash
mkdir download
```
要创建多个并列的目录，可以在命令后跟上多个目录名：
```bash
mkdir dir1 dir2 dir3
```
如果要创建嵌套的目录结构（例如 `files/archives`），而父目录 `files` 不存在，直接使用 `mkdir files/archives` 会报错。此时需要加上 `-p` 选项，它会自动创建所有不存在的父目录：
```bash
mkdir -p files/archives
```
选项 `-v` 可以在任何 `mkdir` 命令后使用，以显示操作详情。`-m` 选项涉及权限管理，我们将在后续课程中详细介绍。

![](img/f98b1998d41f65e43affbb955d293ca3_23.png)

![](img/f98b1998d41f65e43affbb955d293ca3_25.png)

## 复制文件和目录 📋

复制文件和目录的命令是 `cp`。

![](img/f98b1998d41f65e43affbb955d293ca3_27.png)

![](img/f98b1998d41f65e43affbb955d293ca3_28.png)

其基本语法是：
```bash
cp [源文件/目录] [目标位置]
```
例如，将 `/etc/fstab` 文件复制到当前目录：
```bash
cp /etc/fstab ./
```
这代表将**文件复制到某个目录下**。

如果在当前目录下复制文件并重命名，可以理解为**备份**或**在当前目录复制文件**：
```bash
cp fstab newfstab
```
接下来是**复制目录**。如果直接复制一个包含内容的目录，系统会提示忽略目录。需要使用 `-r`（递归）选项来复制目录及其内部所有内容：
```bash
cp -r files documents/
```
`cp` 命令还有其他重要选项，使用 `cp --help` 可以查看。以下是几个关键选项：

![](img/f98b1998d41f65e43affbb955d293ca3_30.png)

![](img/f98b1998d41f65e43affbb955d293ca3_32.png)

![](img/f98b1998d41f65e43affbb955d293ca3_34.png)

*   **`-a`**：归档复制，保留文件所有属性（权限、所有者、时间戳等），相当于 `-dR --preserve=all`。可以理解为100%复制。
*   **`-p`**：保留权限、所有者和时间戳。这是默认行为。
*   **`-r` 或 `-R`**：递归复制，用于复制目录。
*   **`-i`**：交互模式。如果目标位置存在同名文件，会询问用户是否覆盖。这是一个重要的安全选项，因为默认情况下 `cp` 会直接覆盖目标文件。

![](img/f98b1998d41f65e43affbb955d293ca3_36.png)

![](img/f98b1998d41f65e43affbb955d293ca3_38.png)

## 移动文件和目录 🚚

移动文件和目录的命令是 `mv`（move）。其用法简单，基本语法为：
```bash
mv [源文件/目录] [目标位置]
```
如果源和目标在同一目录下，此操作相当于**重命名**。`mv` 命令在移动时会保留文件的属性信息。

![](img/f98b1998d41f65e43affbb955d293ca3_40.png)

![](img/f98b1998d41f65e43affbb955d293ca3_42.png)

例如，将当前目录下的 `newfstab` 文件移动到 `files` 目录中：
```bash
mv newfstab files/
```
与 `cp` 命令类似，`mv` 也有 `-i` 选项，用于在覆盖同名文件前进行交互确认，以避免误操作。需要注意的是，Linux 系统严格区分文件名的大小写。

## 删除文件和目录 🗑️

删除文件和目录的命令是 `rm`。

![](img/f98b1998d41f65e43affbb955d293ca3_44.png)

`rm` 命令有几个关键选项：

![](img/f98b1998d41f65e43affbb955d293ca3_46.png)

![](img/f98b1998d41f65e43affbb955d293ca3_47.png)

![](img/f98b1998d41f65e43affbb955d293ca3_49.png)

![](img/f98b1998d41f65e43affbb955d293ca3_51.png)

*   **`-i`**：交互模式，删除前询问用户确认。
*   **`-r`**：递归删除，用于删除目录及其内容。
*   **`-f`**：强制删除，不进行任何询问，会覆盖 `-i` 选项的效果。

![](img/f98b1998d41f65e43affbb955d293ca3_53.png)

需要特别注意：**`rm` 命令删除文件并非将其移至回收站，一旦删除，通常需要专业数据恢复软件才能找回。** 因此使用时必须格外小心。

![](img/f98b1998d41f65e43affbb955d293ca3_54.png)

在红帽系统中，`root` 用户默认执行 `rm` 相当于 `rm -i`（会询问），而普通用户默认则不会询问。但 `rm` 命令本身设计就是不询问的，这是一个需要注意的事项。

![](img/f98b1998d41f65e43affbb955d293ca3_56.png)

示例：删除当前目录下的 `fstab` 文件（不加 `-i` 则直接删除）：
```bash
rm fstab
```
进入 `files` 目录，使用 `-i` 选项删除文件，系统会询问是否确认：
```bash
rm -i fstab
```
要删除一个目录，必须使用 `-r` 选项：
```bash
rm -r files
```
**警告：** 类似 `rm -rf /` 这样的命令极其危险，它会尝试删除根目录下的所有内容，可能导致系统崩溃和数据永久丢失，务必避免使用。通配符 `*` 与 `rm` 结合使用时也需非常谨慎。

为了更安全地管理文件，这里有一些建议：

1.  **对于文件**：可以考虑用 `mv` 命令替代 `rm`。例如，先建立一个“垃圾箱”目录（如 `~/trash`），将想删除的文件移动进去，并加上时间戳，待确认无用后再彻底删除。
    ```bash
    mv important_file ~/trash/important_file_20231218_1619
    ```
2.  **对于目录**：可以使用 `rmdir` 命令替代 `rm -r`。因为 `rmdir` 仅能删除**空目录**。如果目录非空，它会报错，这给了我们一个检查的机会。你可以先手动清空目录内容，再使用 `rmdir` 删除空目录。`rmdir -p` 选项可以删除目录及其空的父目录。
    ```bash
    rmdir empty_directory
    ```

![](img/f98b1998d41f65e43affbb955d293ca3_58.png)

养成谨慎操作和定期备份的习惯至关重要。

本节课中我们一起学习了Linux命令行下管理文件和目录的核心命令：`mkdir`（创建）、`cp`（复制）、`mv`（移动/重命名）和 `rm`（删除）。虽然这些命令基础，但正确、安全地使用它们是成为合格系统管理员的基石。请务必理解每个命令的选项，特别是涉及覆盖和删除操作时，善用交互模式（`-i`）来增加安全屏障。
