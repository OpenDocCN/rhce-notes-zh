## 练习一：使用任务块处理异常与状态

![](img/6d54e878fa964677a2b1fe71d5b9b688_1.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_2.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_3.png)

上一节我们回顾了 Ansible 的基础知识，本节中我们来看看如何构建一个具备错误处理和状态追踪功能的 Playbook。

![](img/6d54e878fa964677a2b1fe71d5b9b688_5.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_6.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_7.png)

以下是第一题的详细步骤与代码实现：

![](img/6d54e878fa964677a2b1fe71d5b9b688_9.png)

1.  **列出目录内容**：使用 `command` 模块执行 `ls` 命令。
    ```yaml
    - name: List directory contents
      command: ls /tmp
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_11.png)

2.  **任务状态改变时创建文件**：使用 `file` 模块，并添加 `when: changed` 条件。
    ```yaml
    - name: Create file if task changed
      file:
        path: /tmp/testfile
        state: touch
      when: changed
    ```

3.  **任务异常时创建目录**：在 `rescue` 块中使用 `file` 模块创建目录。
    ```yaml
    rescue:
      - name: Create directory if task fails
        file:
          path: /tmp/testDir
          state: directory
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_13.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_14.png)

4.  **无论成功与否都输出信息**：在 `always` 块中使用 `lineinfile` 模块写入文件。
    ```yaml
    always:
      - name: Always output a message
        lineinfile:
          path: /tmp/tfile
          line: "Task execution completed."
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_16.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_18.png)

完整的 Playbook 结构如下：
```yaml
- name: 练习01
  hosts: all
  ignore_errors: yes
  tasks:
    - block:
        - name: List directory contents
          command: ls /tmp
        - name: Create file if task changed
          file:
            path: /tmp/testfile
            state: touch
          when: changed
      rescue:
        - name: Create directory if task fails
          file:
            path: /tmp/testDir
            state: directory
      always:
        - name: Always output a message
          lineinfile:
            path: /tmp/tfile
            line: "Task execution completed."
```

## 练习二：使用加密变量创建用户

![](img/6d54e878fa964677a2b1fe71d5b9b688_22.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_23.png)

在掌握了基础任务控制后，本节我们将学习如何使用加密的变量文件来安全地管理敏感信息（如密码），并基于主机组动态创建用户。

![](img/6d54e878fa964677a2b1fe71d5b9b688_24.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_26.png)

以下是完成此综合练习的关键步骤：

![](img/6d54e878fa964677a2b1fe71d5b9b688_28.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_30.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_32.png)

1.  **准备加密的变量文件**：
    *   创建一个包含明文密码的文件（如 `secret.txt`）。
    *   使用 `ansible-vault create` 命令和该密码文件来加密生成 `locker.yml`。
    ```bash
    ansible-vault create --vault-password-file=secret.txt locker.yml
    ```
    *   在 `locker.yml` 中定义加密后的变量，例如：
        ```yaml
        pw_develop: $6$加密后的密码哈希
        pw_manager: $6$加密后的密码哈希
        ```

2.  **准备用户列表变量文件**：创建一个普通的 YAML 文件（如 `user_list.yml`）定义用户信息。
    ```yaml
    users:
      dev:
        - name: alice
          job: developer
        - name: bob
          job: developer
      mgr:
        - name: charlie
          job: manager
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_34.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_36.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_37.png)

3.  **编写创建用户的 Playbook**：Playbook 需要引用上述变量文件，并根据主机组循环创建用户。
    ```yaml
    - name: Create users from vault
      hosts: dev, mgr
      vars_files:
        - user_list.yml
        - locker.yml
      tasks:
        - name: Create users for dev group
          user:
            name: "{{ item.name }}"
            password: "{{ pw_develop }}"
            comment: "{{ item.job }}"
          loop: "{{ users.dev }}"
          when: inventory_hostname in groups['dev']

        - name: Create users for mgr group
          user:
            name: "{{ item.name }}"
            password: "{{ pw_manager }}"
            comment: "{{ item.job }}"
          loop: "{{ users.mgr }}"
          when: inventory_hostname in groups['mgr']
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_39.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_40.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_41.png)

4.  **运行 Playbook**：执行时需要提供解密密码文件。
    ```bash
    ansible-playbook create_users.yml --vault-password-file=secret.txt
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_45.png)

## 练习三：基于主机组替换文件内容

![](img/6d54e878fa964677a2b1fe71d5b9b688_53.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_55.png)

最后，我们来看一个文件操作题目，学习如何根据目标主机所属的不同组，来动态替换其文件内容。

![](img/6d54e878fa964677a2b1fe71d5b9b688_57.png)

以下是实现此需求的几种方法示例：

![](img/6d54e878fa964677a2b1fe71d5b9b688_59.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_60.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_61.png)

*   **方法一：使用 `copy` 模块覆盖文件**
    为每个组准备不同的源文件，然后根据条件复制。
    ```yaml
    - name: Replace file content by group
      hosts: all
      tasks:
        - name: Copy dev content
          copy:
            dest: /etc/somefile
            content: "This is development content."
          when: inventory_hostname in groups['dev']

        - name: Copy mgr content
          copy:
            dest: /etc/somefile
            content: "This is manager content."
          when: inventory_hostname in groups['mgr']
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_62.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_64.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_66.png)

*   **方法二：使用 `lineinfile` 或 `replace` 模块修改文件**
    如果只需修改文件中特定行或字符串，可以使用这些模块。
    ```yaml
    - name: Replace specific line
      lineinfile:
        path: /etc/somefile
        regexp: '^OldText.*$'
        line: 'NewText for {{ group }}'
      when: inventory_hostname in groups['dev']
    ```

![](img/6d54e878fa964677a2b1fe71d5b9b688_67.png)

![](img/6d54e878fa964677a2b1fe71d5b9b688_69.png)

**核心思路**：关键在于利用 `inventory_hostname in groups[‘group_name’]` 这个条件判断，来区分不同主机组并执行相应的任务。

## 总结

本节课中我们一起学习了三个 Ansible 综合练习题：
1.  使用 **`block`、`rescue`、`always`** 构建健壮的任务流程，处理异常和状态变化。
2.  使用 **`ansible-vault`** 加密敏感变量，并在 Playbook 中安全引用，实现基于主机组的动态用户创建。
3.  利用 **`when` 条件判断** 结合主机组信息，使用 `copy`、`lineinfile` 等模块对目标主机文件进行差异化配置。
