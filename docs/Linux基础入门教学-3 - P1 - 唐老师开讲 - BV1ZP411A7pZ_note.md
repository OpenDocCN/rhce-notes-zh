## 回顾与衔接：Linux核心特性
上一节我们介绍了Linux的基本安装和网络配置。本节中，我们来看看Linux系统的两个核心特性：万物皆文件/文件夹的概念，以及如何通过颜色区分文件类型。

在Linux系统中，一切都被抽象为文件或文件夹。最大的文件夹是根目录，用 `/` 表示。

系统通过颜色来区分不同类型的文件：
*   **普通文件**：默认显示为**白色**。
*   **文件夹（目录）**：通常显示为**深蓝色**。
*   **可执行文件或命令**：通常显示为**绿色**。

这与Windows通过文件后缀（如 `.exe`）来标识可执行文件的方式不同。

## 第一部分：Linux一级目录结构详解 🗂️
理解Linux的目录结构是掌握系统的第一步。根目录 `/` 下的每个文件夹都有其特定的用途，类似于一个公司或国家的各个职能部门。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_9.png)

![](img/d7cf6d2f2f747f72ffa61d10148bf243_11.png)

以下是根目录下一级核心目录的详细说明：

![](img/d7cf6d2f2f747f72ffa61d10148bf243_13.png)

![](img/d7cf6d2f2f747f72ffa61d10148bf243_14.png)

*   **`/bin`**
    *   **作用**：存放系统**最基本的命令**（二进制可执行文件），这些命令在单用户维护模式下也能使用。
    *   **类比**：相当于Windows系统中存放基础系统工具的位置。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_16.png)

*   **`/boot`**
    *   **作用**：存放系统的**启动文件**、**内核**以及**内核引导程序**。
    *   **重要性**：如果此目录损坏，系统可能无法启动。在安装系统时，我们通常为其分配独立分区（约500MB-1GB）。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_18.png)

*   **`/dev`**
    *   **作用**：**设备文件目录**。该目录下的文件对应着系统的硬件设备（如硬盘、光驱、USB设备）。
    *   **类比**：相当于Windows中的“设备管理器”。例如，光驱设备可能对应 `/dev/sr0` 文件。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_20.png)

![](img/d7cf6d2f2f747f72ffa61d10148bf243_22.png)

*   **`/etc`**
    *   **作用**：存放系统及应用程序的**主配置文件**。这是Linux系统中**最重要**的目录之一。
    *   **重要性**：修改这里的配置文件可以改变系统或软件的行为。误操作可能导致服务无法启动或系统异常。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_24.png)

*   **`/home`**
    *   **作用**：**普通用户的家目录**。每个普通用户登录后，都会进入其在此目录下的个人文件夹（如用户 `zhangsan` 的家目录是 `/home/zhangsan`）。
    *   **注意**：超级管理员 `root` 的家目录是 `/root`，不在此目录下。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_26.png)

*   **`/lib` 与 `/lib64`**
    *   **作用**：存放系统**最基本的共享库文件**和**内核模块文件**。这些文件类似于Windows中的 `.dll` 文件，是许多命令和程序运行所依赖的。
    *   **重要性**：库文件缺失会导致依赖它的命令或程序无法执行。

*   **`/media`**
    *   **作用**：系统**自动挂载目录**。当插入U盘、光盘等可移动介质时，系统通常会在此目录下自动创建一个子目录并挂载设备。

*   **`/mnt`**
    *   **作用**：**手动挂载目录**。系统管理员临时挂载文件系统（如网络驱动器、额外的硬盘分区）时，通常习惯挂载到此目录下。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_28.png)

![](img/d7cf6d2f2f747f72ffa61d10148bf243_29.png)

*   **`/opt`**
    *   **作用**：**第三方应用程序安装目录**。一些非系统自带的软件包可以选择安装在此处。
    *   **类比**：类似于在Windows中专门为软件创建的 `D:\Program Files` 目录。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_31.png)

*   **`/proc`**
    *   **作用**：**虚拟文件系统**，存放当前**系统内核与进程的运行状态信息**。这里的文件并非真实磁盘文件，而是内存的映射。
    *   **特点**：系统重启后，此目录内容会被清空重置。
    *   **类比**：类似于Windows的“任务管理器”所显示的信息。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_33.png)

![](img/d7cf6d2f2f747f72ffa61d10148bf243_34.png)

*   **`/root`**
    *   **作用**：**系统超级管理员（root）的家目录**。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_36.png)

*   **`/sbin`**
    *   **作用**：存放**系统管理命令**，通常只有超级管理员（root）才有权限使用。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_38.png)

![](img/d7cf6d2f2f747f72ffa61d10148bf243_40.png)

*   **`/tmp`**
    *   **作用**：**临时文件目录**。所有用户都可在此存放临时文件。系统可能会定期清理此目录的内容。

*   **`/usr`**
    *   **作用**：存放**用户应用程序和文件**，类似于Windows的 `Program Files` 目录。包含大量的子目录，如 `/usr/bin`（用户命令）、`/usr/lib`（用户库文件）、`/usr/share`（共享数据）等。

*   **`/var`**
    *   **作用**：存放**经常变化的文件**，如日志文件 (`/var/log`)、邮件队列、缓存数据等。

**核心概念强调**：
*   **`/etc`** 是**配置中心**，存放核心设置。
*   **`/home`** 和 **`/root`** 是用户的**工作空间**。
*   **`/bin`**, **`/sbin`**, **`/usr`** 是**命令和程序的仓库**。
*   **`/dev`**, **`/proc`** 是**系统状态的窗口**。

**重要警告**：由于Linux中一切皆文件，目录也可以被“挂载”覆盖。**切勿将其他设备或分区挂载到 `/`、`/etc`、`/bin` 等关键系统目录上**，否则会覆盖原有内容，导致系统崩溃。

## 第二部分：硬盘分区基础概念 💽
上一节我们理清了Linux的“软件”结构，本节中我们来看看“硬件”存储的基础——分区。理解分区是进行磁盘管理和多系统安装的前提。

### 主分区、扩展分区与逻辑分区
一块全新的硬盘需要分区后才能使用。分区主要涉及以下三种类型：

1.  **主分区**
    *   **定义**：能够直接安装操作系统并引导启动的分区。
    *   **限制**：一块**硬盘上最多只能创建4个主分区**。

2.  **扩展分区**
    *   **定义**：一种特殊的主分区，它本身不能直接存储数据，其作用是在4个主分区的限制下，**划出额外的空间来创建更多的逻辑分区**。
    *   **限制**：一块硬盘上**最多只能有1个扩展分区**。扩展分区会占用1个主分区的名额。

![](img/d7cf6d2f2f747f72ffa61d10148bf243_42.png)

3.  **逻辑分区**
    *   **定义**：在**扩展分区内部**创建的分区，用于实际存储数据。
    *   **标识**：逻辑分区的编号从 **5** 开始（因为1-4预留给主分区和扩展分区）。

### 分区方案示例
假设有一块500GB的硬盘，我们来看两种典型的分区方案：

**方案一：仅使用主分区（不推荐）**
*   `/dev/sda1` (主分区，100GB， 安装Windows)
*   `/dev/sda2` (主分区，100GB， 安装Linux)
*   `/dev/sda3` (主分区，100GB， 存放数据)
*   `/dev/sda4` (主分区，200GB， 存放数据)
*   **结果**：4个主分区名额用尽，无法再创建更多分区。如果每个分区空间规划不当，可能导致空间浪费或不足。

**方案二：使用主分区+扩展分区（推荐）**
*   `/dev/sda1` (主分区，100GB， 安装Windows)
*   `/dev/sda2` (主分区，100GB， 安装Linux)
*   `/dev/sda3` (扩展分区，300GB)
    *   `/dev/sda5` (逻辑分区，100GB， 在扩展分区内)
    *   `/dev/sda6` (逻辑分区，100GB， 在扩展分区内)
    *   `/dev/sda7` (逻辑分区，100GB， 在扩展分区内)
*   **结果**：只用了2个主分区名额，通过扩展分区创建了3个逻辑分区，更灵活地利用了磁盘空间，并保留了未来创建更多逻辑分区或主分区的可能性。

**核心公式与编号规则**：
```
主分区 + 扩展分区 <= 4
其中，扩展分区 <= 1
逻辑分区编号 >= 5
```
在Linux中，磁盘设备通常命名为 `/dev/sdX`（如 `/dev/sda`），分区则在后面加数字，如 `/dev/sda1`、`/dev/sda2`...
