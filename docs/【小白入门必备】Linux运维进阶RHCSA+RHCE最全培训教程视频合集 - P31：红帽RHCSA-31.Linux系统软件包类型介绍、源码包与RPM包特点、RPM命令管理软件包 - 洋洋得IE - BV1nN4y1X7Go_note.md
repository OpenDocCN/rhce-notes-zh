在Linux系统中，软件包主要分为两大类：**源码包**和**二进制包**（通常指RPM包）。理解它们的区别是选择合适安装方式的关键。

### 开源软件与闭源软件

在深入了解软件包类型前，需要理解开源的概念。开源意味着软件的源代码被公开发布，允许用户查看、修改甚至进行二次开发。

![](img/b1d014006c61e1a6375801a1da67fe57_29.png)

![](img/b1d014006c61e1a6375801a1da67fe57_31.png)

![](img/b1d014006c61e1a6375801a1da67fe57_33.png)

![](img/b1d014006c61e1a6375801a1da67fe57_35.png)

*   **开源软件示例**：Nginx是一款开源的Web服务器软件。淘宝网基于Nginx进行了二次开发，创建了满足自身大访问量需求的Tengine。
*   **开源优势**：
    1.  **允许二次开发**：企业可以根据自身需求修改源代码。
    2.  **使用放心**：公开的代码经社区审查，降低了内置恶意代码的风险。
    3.  **持续维护**：即使原开发者停止维护，开源社区也可能会接手。

![](img/b1d014006c61e1a6375801a1da67fe57_37.png)

![](img/b1d014006c61e1a6375801a1da67fe57_39.png)

与开源相对的是闭源软件，其源代码不公开。例如Windows操作系统。闭源软件的缺点是用户无法进行二次开发，且一旦官方停止维护，用户将难以自行修复问题。

**注意**：开源不等于完全免费，有些开源软件会针对高级功能、技术支持或特定服务收费。

### 源码包的特点

源码包是包含软件原始代码的压缩包，例如 `nginx-1.18.0.tar.gz`。用户需要手动将其“编译”成计算机可执行的二进制形式。

![](img/b1d014006c61e1a6375801a1da67fe57_47.png)

![](img/b1d014006c61e1a6375801a1da67fe57_49.png)

以下是源码包的主要特点：

![](img/b1d014006c61e1a6375801a1da67fe57_51.png)

![](img/b1d014006c61e1a6375801a1da67fe57_53.png)

*   **优点**：
    1.  **可查看源代码**：便于学习和二次开发。
    2.  **安装灵活**：在编译过程中，可以自定义安装路径（如 `/opt/nginx`）和选择需要的功能模块。
    3.  **卸载干净**：直接删除自定义的安装目录即可，无残留文件。
*   **缺点**：
    1.  **安装过程复杂**：需要用户手动执行配置、编译、安装等步骤。
    2.  **需解决依赖关系**：需要自行安装该软件所依赖的其他库或软件。

**适用场景**：对软件功能有定制化需求、追求系统轻量化或学习研究源代码时使用。

### RPM包的特点

![](img/b1d014006c61e1a6375801a1da67fe57_63.png)

![](img/b1d014006c61e1a6375801a1da67fe57_65.png)

![](img/b1d014006c61e1a6375801a1da67fe57_67.png)

RPM包（Red Hat Package Manager）是已经预先编译好的二进制包，文件后缀名为 `.rpm`，例如 `vsftpd-3.0.2-25.el7.x86_64.rpm`。我们之前用 `yum` 命令安装的软件都是RPM包。

![](img/b1d014006c61e1a6375801a1da67fe57_69.png)

以下是RPM包的主要特点：

![](img/b1d014006c61e1a6375801a1da67fe57_71.png)

*   **优点**：
    1.  **安装简单快捷**：无需编译，直接安装即可使用。
    2.  **功能全面**：官方编译时通常包含了大多数常用功能。
*   **缺点**：
    1.  **无法自定义**：不能选择安装路径和功能模块。
    2.  **看不到源代码**：因为是二进制格式。
    3.  **依赖关系复杂**：安装时可能需要手动解决复杂的依赖问题。

**适用场景**：需要快速安装软件、对功能无特殊定制要求时使用。日常管理中，RPM包的使用更为频繁。

**命名规则解析**：
一个典型的RPM包名 `vsftpd-3.0.2-25.el7.x86_64.rpm` 包含以下信息：
*   `vsftpd`：软件包名称。
*   `3.0.2`：软件版本号（主版本.次版本.修订版本）。
*   `25`：软件包发布次数（补丁次数）。
*   `el7`：适用的操作系统（Red Hat Enterprise Linux 7 / CentOS 7）。
*   `x86_64`：适用的CPU架构（64位）。
*   `rpm`：文件扩展名。

## RPM命令管理软件包

上一节我们了解了两种软件包的特点，本节中我们重点学习如何使用 `rpm` 命令来管理RPM软件包。首先，我们需要知道软件包从哪里来。

### 软件包来源

Linux系统中的软件包主要有两个来源：

1.  **本地镜像文件**：系统安装光盘或ISO镜像文件中包含了大量常用的RPM包。这些包通常挂载在 `/mnt/` 目录下，例如 `/mnt/Packages/` 目录中可能包含数千个软件包。
2.  **网络仓库**：互联网上的软件仓库（如CentOS官方仓库、EPEL仓库），可以通过 `yum` 命令（下节课讲解）自动从网络下载并安装。

#### 安装软件包

安装RPM包需要使用 `-ivh` 选项组合，并指定软件包的**完整名称**（包括版本、架构等信息）。

![](img/b1d014006c61e1a6375801a1da67fe57_87.png)

**命令格式**：
```bash
rpm -ivh 软件包全名.rpm
```
*   `-i`：安装（install）。
*   `-v`：显示详细信息（verbose）。
*   `-h`：显示安装进度条（hash）。

![](img/b1d014006c61e1a6375801a1da67fe57_89.png)

![](img/b1d014006c61e1a6375801a1da67fe57_91.png)

![](img/b1d014006c61e1a6375801a1da67fe57_93.png)

**操作示例**：
假设我们位于软件包所在目录 `/mnt/Packages/`，安装 `vsftpd` 软件包。
```bash
# 使用Tab键自动补全包名
rpm -ivh vsftpd-3.0.2-25.el7.x86_64.rpm
```

#### 查询软件包

![](img/b1d014006c61e1a6375801a1da67fe57_102.png)

![](img/b1d014006c61e1a6375801a1da67fe57_104.png)

查询是 `rpm` 命令最强大的功能之一，以下是常用查询选项：

![](img/b1d014006c61e1a6375801a1da67fe57_106.png)

![](img/b1d014006c61e1a6375801a1da67fe57_108.png)

![](img/b1d014006c61e1a6375801a1da67fe57_110.png)

以下是常用的查询命令列表：

![](img/b1d014006c61e1a6375801a1da67fe57_112.png)

*   `rpm -q 软件名`：查询指定软件是否已安装。
    ```bash
    rpm -q vsftpd
    ```
*   `rpm -qa`：查询系统中所有已安装的RPM包。
    ```bash
    rpm -qa | wc -l        # 统计已安装包的数量
    rpm -qa | grep vsftpd  # 在所有包中模糊搜索包含‘vsftpd’的包
    ```
*   `rpm -qi 软件名`：查询已安装软件的详细信息（版本、描述、安装时间等）。
    ```bash
    rpm -qi vsftpd
    ```
*   `rpm -ql 软件名`：列出软件安装的所有文件及其位置。
    ```bash
    rpm -ql coreutils      # 查看提供基础命令（如ls, cat）的coreutils包安装了哪些文件
    ```
*   `rpm -qf 文件名`：查询某个文件是由哪个软件包安装的。
    ```bash
    rpm -qf /etc/passwd
    rpm -qf $(which cat)   # 查询cat命令来自哪个包
    ```

![](img/b1d014006c61e1a6375801a1da67fe57_114.png)

![](img/b1d014006c61e1a6375801a1da67fe57_116.png)

**重要提示**：安装后，管理软件（查询、卸载）时使用的是**软件名**（如 `vsftpd`），而不是安装时的**包全名**（如 `vsftpd-3.0.2-25.el7.x86_64.rpm`）。绝大多数情况下，软件名与包名的主体部分相同。

![](img/b1d014006c61e1a6375801a1da67fe57_118.png)

#### 卸载软件包

卸载软件包使用 `-e` 选项。

**命令格式**：
```bash
rpm -e 软件名
```
**操作示例**：
```bash
rpm -e vsftpd
```
**注意**：卸载时同样使用软件名，并且如果其他软件依赖此软件，卸载可能会失败。

### RPM包的依赖问题

使用 `rpm` 命令手动安装时，经常需要处理依赖关系，即一个软件包正常运行需要其他软件包或库文件的支持。依赖关系可能非常复杂，主要类型有：

1.  **树形依赖**：A包依赖B、C、D包。需要先安装所有依赖包，再安装A包。
2.  **环形依赖**：A依赖B，B依赖C，C又依赖A，形成循环。手动解决非常困难。
3.  **模块依赖**：依赖的不是具体的软件包，而是一个模块文件（如 `.so` 文件），且不清楚该文件由哪个包提供。

手动解决依赖极其繁琐。因此，在实际工作中，我们更常使用能自动解决依赖关系的 `yum` 命令来安装RPM包。`rpm` 命令的价值主要体现在强大的查询功能上。
