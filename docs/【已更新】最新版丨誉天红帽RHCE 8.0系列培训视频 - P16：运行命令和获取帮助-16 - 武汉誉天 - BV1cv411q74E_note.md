## man手册页的章节结构 📚

![](img/5a17a2488ff13a8ee09a12303692428a_1.png)

![](img/5a17a2488ff13a8ee09a12303692428a_3.png)

上一节我们介绍了`man`命令的基本用法，本节中我们来看看`man`手册页的章节结构。`man`手册页被划分为9个章节，每个章节包含特定类型的信息。

以下是`man`手册页的9个章节及其主要用途：

![](img/5a17a2488ff13a8ee09a12303692428a_5.png)

1.  **第1章节：用户命令** - 包含普通用户可以在shell中执行的命令。例如：`ls`, `cp`。
2.  **第2章节：系统调用** - 包含由内核提供的函数接口。例如：`open`, `read`。
3.  **第3章节：库函数** - 包含程序库中的函数。例如：`printf`, `fopen`。
4.  **第4章节：特殊文件** - 包含位于`/dev`目录下的设备文件信息。例如：磁盘、键盘。
5.  **第5章节：文件格式** - 包含系统配置文件的格式说明。例如：`/etc/passwd`文件的格式。
6.  **第6章节：游戏** - 包含游戏相关的说明。
7.  **第7章节：杂项** - 包含其他不便归类的主题，如宏命令包、约定等。
8.  **第8章节：系统管理命令** - 包含通常只有`root`管理员才能执行的命令。例如：`fdisk`, `useradd`。
9.  **第9章节：内核例程** - 包含内核相关的文档。

![](img/5a17a2488ff13a8ee09a12303692428a_7.png)

对于初学者，不需要记住所有章节。只需重点关注以下三个最常用的章节：

![](img/5a17a2488ff13a8ee09a12303692428a_9.png)

![](img/5a17a2488ff13a8ee09a12303692428a_11.png)

*   **第1章节**：查询普通命令的用法。
*   **第5章节**：查询配置文件的格式和参数含义。
*   **第8章节**：查询系统管理命令。

## 指定章节查询man手册 🔍

了解了章节结构后，我们来看看如何查询指定章节的内容。`man`命令默认会显示命令的第一个可用章节。

以下是使用`man`命令查询不同章节的示例：

![](img/5a17a2488ff13a8ee09a12303692428a_13.png)

*   查询`passwd`命令（第1章节）的用法：
    ```bash
    man passwd
    # 或明确指定章节
    man 1 passwd
    ```

![](img/5a17a2488ff13a8ee09a12303692428a_15.png)

*   查询`/etc/passwd`文件（第5章节）的格式说明：
    ```bash
    man 5 passwd
    ```

![](img/5a17a2488ff13a8ee09a12303692428a_17.png)

*   查询系统管理命令`fdisk`（通常在第8章节）：
    ```bash
    man fdisk
    # 如果只有第8章节，则默认显示第8章
    ```

![](img/5a17a2488ff13a8ee09a12303692428a_19.png)

**核心概念**：如果一个命令或主题在多个章节中存在（如`passwd`同时是命令和文件名），使用`man [章节号] [主题]`来指定查询的章节。若不指定章节号，`man`会显示编号最小的那个可用章节。

## 使用man -k进行关键字搜索 🔎

有时我们不确定完整的命令名，或者想查找与某个主题相关的所有手册页，这时可以使用`man -k`命令进行关键字搜索。

`man -k`命令基于`whatis`数据库进行模糊查询。它会列出描述信息中包含指定关键词的所有手册页条目。

![](img/5a17a2488ff13a8ee09a12303692428a_21.png)

例如，搜索所有与“password”相关的手册页：
```bash
man -k password
```
搜索结果可能包含`passwd`命令、`chpasswd`命令以及一些配置文件的描述。

## 其他帮助工具：info与pinfo 📖

![](img/5a17a2488ff13a8ee09a12303692428a_28.png)

除了`man`，Linux系统还提供了`info`和`pinfo`作为帮助工具。它们通常提供比`man`更详细、组织更严密（像一本书）的文档。

![](img/5a17a2488ff13a8ee09a12303692428a_30.png)

*   **`info`**：文档结构更层次化，但导航相对复杂。
    ```bash
    info ls
    ```
*   **`pinfo`**：`info`的增强版，支持彩色高亮和更好的鼠标交互，用户体验更友好。
    ```bash
    pinfo ls
    ```

![](img/5a17a2488ff13a8ee09a12303692428a_32.png)

![](img/5a17a2488ff13a8ee09a12303692428a_34.png)

![](img/5a17a2488ff13a8ee09a12303692428a_36.png)

对于日常使用，`man`命令通常已足够。`info`和`pinfo`可作为深入了解命令的补充资源。

## 访问官方在线文档 🌐

![](img/5a17a2488ff13a8ee09a12303692428a_38.png)

![](img/5a17a2488ff13a8ee09a12303692428a_39.png)

对于更复杂或最新的主题，红帽（Red Hat）等发行版提供了详细的在线产品文档。

红帽官方文档网站是：`https://access.redhat.com/documentation`

在该网站上，你可以找到：
*   系统安装指南
*   管理手册（如逻辑卷管理、网络配置）
*   安全指南
*   版本发行说明和新特性介绍

![](img/5a17a2488ff13a8ee09a12303692428a_42.png)

**学习建议**：初学者遇到问题，可先使用`man`命令和搜索引擎。当问题涉及特定产品的深入配置或新特性时，再查阅官方在线文档。养成查阅一手官方资料的习惯，是成为高级用户的重要技能。
