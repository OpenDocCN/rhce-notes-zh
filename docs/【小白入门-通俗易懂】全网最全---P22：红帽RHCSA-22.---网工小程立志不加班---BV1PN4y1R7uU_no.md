## 压缩的概念与好处
压缩的主要目的是减小文件的体积。这可以带来两个直接的好处：一是节约磁盘存储空间，二是提高文件在网络中的传输速度。互联网上常见的图片（如JPG、PNG）、音频（MP3）、视频（MP4）等格式都是经过压缩的，这使得我们能够快速加载和访问这些内容。

## Linux下的常见压缩格式
在Linux系统中，有三种主流的压缩命令，它们对应不同的压缩格式，在压缩速度和压缩比例上各有侧重。

### gzip压缩格式
`gzip`命令的压缩速度较快，但压缩比例相对一般。压缩后的文件以`.gz`结尾。

**命令格式如下：**
```bash
gzip [文件名]
```

![](img/049c247a2390e66268bff2e50178a2d1_15.png)

![](img/049c247a2390e66268bff2e50178a2d1_17.png)

**操作示例：**
1.  首先，我们复制一个文件并查看其原始大小。
    ```bash
    cp /etc/services .
    du -h services
    ```
    输出显示文件大小约为656K。

![](img/049c247a2390e66268bff2e50178a2d1_19.png)

![](img/049c247a2390e66268bff2e50178a2d1_21.png)

2.  使用`gzip`进行压缩。
    ```bash
    gzip services
    ```
    压缩后，原文件`services`会变成`services.gz`。

![](img/049c247a2390e66268bff2e50178a2d1_23.png)

![](img/049c247a2390e66268bff2e50178a2d1_25.png)

3.  再次查看文件大小，会发现它显著减小（例如变为136K）。
    ```bash
    du -h services.gz
    ```

**查看压缩文件内容：**
不能使用`cat`命令直接查看，否则会显示乱码。需要使用`zcat`命令。
```bash
zcat services.gz
```

![](img/049c247a2390e66268bff2e50178a2d1_27.png)

![](img/049c247a2390e66268bff2e50178a2d1_29.png)

![](img/049c247a2390e66268bff2e50178a2d1_31.png)

**解压缩文件：**
使用`-d`选项进行解压。
```bash
gzip -d services.gz
```
解压后，文件恢复为原始格式和大小。

![](img/049c247a2390e66268bff2e50178a2d1_33.png)

## 压缩格式的选择
上一节我们介绍了三种压缩工具，本节中我们来看看如何根据场景选择。
*   **追求速度**：选择`gzip`。适用于需要快速压缩和解压的场景，例如流媒体传输。
*   **追求压缩率**：选择`xz`。适用于对存储空间敏感、不频繁访问的长期备份。
*   **平衡选择**：选择`bzip2`。在速度和压缩率之间取得较好的平衡。

需要注意的是，以上三种命令都**只能对单个文件进行压缩，无法直接压缩目录**。

## tar打包工具
当我们需要对目录或大量文件进行压缩时，就需要先使用打包工具。`tar`命令可以将多个文件或目录打包成一个单独的文件（称为tar包，通常以`.tar`结尾），然后再对这个打包后的文件进行压缩。

![](img/049c247a2390e66268bff2e50178a2d1_98.png)

### 打包与压缩
`tar`命令选项较多，以下是创建打包文件时的核心选项组合：
*   `-c`：创建新的打包文件。
*   `-f`：指定打包后的文件名。**此选项必须放在所有选项的最后**。
*   `-z`：调用`gzip`进行压缩/解压。
*   `-j`：调用`bzip2`进行压缩/解压。
*   `-J`：调用`xz`进行压缩/解压。
*   `-v`：显示详细的处理过程（可选）。

**命令基本格式如下：**
```bash
tar -czf [压缩包名.tar.gz] [要打包的目录或文件]
```

**操作示例：**
1.  将`/var/log`目录打包并用`gzip`压缩。
    ```bash
    tar -czf log_backup.tar.gz /var/log
    ```
    这条命令会创建一个名为`log_backup.tar.gz`的压缩包。

2.  如果想在过程中看到正在打包的文件列表，可以加上`-v`选项。
    ```bash
    tar -czvf log_backup.tar.gz /var/log
    ```

### 查看打包文件内容
使用`-t`选项可以列出打包文件内的内容，同样需要`-f`指定文件名。
```bash
tar -tzf log_backup.tar.gz
```

### 解包与解压缩
使用`-x`选项进行解包/解压。
*   `-x`：解包/解压文件。
*   `-C`：指定解压到的目标目录（可选）。

**操作示例：**
1.  解压到当前目录。
    ```bash
    tar -xzf log_backup.tar.gz
    ```

2.  解压到指定的`/opt`目录。
    ```bash
    tar -xzf log_backup.tar.gz -C /opt
    ```

**重要提示**：在打包系统目录（如`/var/log`）时，`tar`命令默认会移除路径开头的“/”（根目录）。这是为了防止解压时文件被绝对路径覆盖到系统原始位置，从而允许你安全地将文件解压到任何指定目录。
