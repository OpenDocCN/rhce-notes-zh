# Ansible入门教程：Day10_Ch04a：管理变量 📚

![](img/8878277a9f3968c4e4183362a3a5ee48_1.png)

在本节课中，我们将要学习Ansible中变量的不同类型及其使用方法。变量是Ansible剧本中存储和引用数据的关键，理解它们对于编写高效、灵活的自动化脚本至关重要。

## 概述 📖

我们将介绍四种主要的变量类型：常规变量、数组变量、字典变量和注册变量。每种类型都有其特定的使用场景和语法规则。

## 常规变量 🔧

![](img/8878277a9f3968c4e4183362a3a5ee48_10.png)

上一节我们介绍了变量的基本概念，本节中我们来看看最基础的变量类型——常规变量。

![](img/8878277a9f3968c4e4183362a3a5ee48_12.png)

常规变量用于存储单个值，其定义和取值方式如下：

*   **定义方式**：`key=value` 或 `key: value`
*   **注意事项**：在资产清单（Inventory）文件中，必须使用等号 `=` 进行定义，使用冒号 `:` 会导致错误。

---

## 数组变量 📦

当我们需要定义一系列同类型的变量时，例如一批用户名或软件包列表，可以使用数组变量。

![](img/8878277a9f3968c4e4183362a3a5ee48_14.png)

以下是数组变量的定义和使用示例：

![](img/8878277a9f3968c4e4183362a3a5ee48_16.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_18.png)

```yaml
- hosts: servers
  vars:
    user_name: ['test_user0', 'test_user1', 'test_user2', 'test_user3']
  tasks:
    - name: Create a user from array
      user:
        name: "{{ user_name[0] }}"
        state: present
```

![](img/8878277a9f3968c4e4183362a3a5ee48_19.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_20.png)

**关键点**：
*   数组索引从 `0` 开始。
*   调用数组中的特定值使用 `数组名[索引]` 的语法，例如 `{{ user_name[0] }}`。
*   不能直接将整个数组传递给某些模块参数（如 `user` 模块的 `name` 参数），这会导致错误。

![](img/8878277a9f3968c4e4183362a3a5ee48_22.png)

## 字典变量 📚

![](img/8878277a9f3968c4e4183362a3a5ee48_28.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_30.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_31.png)

当变量的信息包含多种不同类型的元素时，例如一个用户的详细信息（名称、Shell类型、UID、注释），适合使用字典变量。

![](img/8878277a9f3968c4e4183362a3a5ee48_33.png)

字典变量允许我们以键值对的形式组织复杂数据。以下是字典的定义和引用方法：

![](img/8878277a9f3968c4e4183362a3a5ee48_35.png)

```yaml
- hosts: servers
  vars:
    user_info:
      test_user10:
        name: 'test_user10'
        shell: '/sbin/nologin'
        comment: 'testuser10'
      test_user11:
        name: 'test_user11'
        shell: '/bin/bash'
        comment: 'testuser11'
  tasks:
    - name: Create users via dictionary
      user:
        name: "{{ user_info['test_user10'].name }}"
        shell: "{{ user_info['test_user10'].shell }}"
        state: present
```

![](img/8878277a9f3968c4e4183362a3a5ee48_37.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_38.png)

字典变量有两种引用方式：

![](img/8878277a9f3968c4e4183362a3a5ee48_40.png)

1.  **使用方括号和引号**：`"{{ user_info['test_user10']['name'] }}"`
2.  **使用点号（`.`）**：`"{{ user_info.test_user10.name }}"`。注意，如果键名包含点号或特殊字符，则必须使用第一种方式。

![](img/8878277a9f3968c4e4183362a3a5ee48_42.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_44.png)

## 注册变量 💾

![](img/8878277a9f3968c4e4183362a3a5ee48_49.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_51.png)

注册变量（Register Variable）用于将一个任务的执行结果保存到变量中，以便后续任务引用。这在需要根据前一个任务的结果来决定后续操作时非常有用。

![](img/8878277a9f3968c4e4183362a3a5ee48_53.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_55.png)

以下是如何使用注册变量的示例：

![](img/8878277a9f3968c4e4183362a3a5ee48_57.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_58.png)

```yaml
- hosts: servers
  tasks:
    - name: Create a user and register the result
      user:
        name: test_user12
        state: present
      register: user_result  # 将创建用户任务的结果注册到变量 user_result

    - name: Display the registered result
      debug:
        msg: "The UID of the created user is {{ user_result.uid }}"
```

![](img/8878277a9f3968c4e4183362a3a5ee48_60.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_62.png)

**关键点**：
*   `register` 关键字与任务项处于同一层级。
*   注册的结果是一个包含丰富信息的字典，例如 `changed`（状态是否改变）、`msg`（消息）、`uid`（用户ID）等。
*   可以通过点号 `.` 或方括号 `[]` 来引用结果中的特定字段。
*   注册的变量可以传递给后续任务，实现信息的迭代使用。

![](img/8878277a9f3968c4e4183362a3a5ee48_64.png)

## 总结 🎯

![](img/8878277a9f3968c4e4183362a3a5ee48_66.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_68.png)

本节课中我们一起学习了Ansible中四种核心的变量类型：

![](img/8878277a9f3968c4e4183362a3a5ee48_70.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_72.png)

1.  **常规变量**：用于存储简单的键值对，定义方式为 `key=value`。
2.  **数组变量**：用于存储同类型数据的列表，通过索引 `数组名[索引]` 引用单个元素。
3.  **字典变量**：用于存储具有多种属性的复杂数据，通过键名引用，如 `字典名.键名` 或 `字典名['键名']`。
4.  **注册变量**：用于捕获任务的执行结果，通过 `register` 关键字定义，便于在后续任务中引用和分析。

![](img/8878277a9f3968c4e4183362a3a5ee48_74.png)

![](img/8878277a9f3968c4e4183362a3a5ee48_76.png)

掌握这些变量类型将帮助你更好地组织数据，编写出更强大、更灵活的Ansible自动化剧本。
