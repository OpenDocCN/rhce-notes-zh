# Linux Shell编程：P43：了解bash shell-43

在本节课中，我们将要学习bash shell中的一些核心概念，包括命令替换、大括号扩展以及变量的定义和使用。这些知识是编写高效shell脚本和熟练使用命令行环境的基础。

## 命令替换：`$(command)`

![](img/2348d3f989cdc7053b26085198415a26_1.png)

![](img/2348d3f989cdc7053b26085198415a26_3.png)

![](img/2348d3f989cdc7053b26085198415a26_5.png)

上一节我们介绍了基本的命令执行，本节中我们来看看如何将命令的输出结果作为值来使用。这通过命令替换实现。

![](img/2348d3f989cdc7053b26085198415a26_7.png)

命令替换的语法是 `$(command)`。它的作用是执行括号内的命令，并将该命令的标准输出结果作为值返回。这个值可以赋值给变量，或者直接嵌入到其他命令中。

![](img/2348d3f989cdc7053b26085198415a26_9.png)

![](img/2348d3f989cdc7053b26085198415a26_11.png)

![](img/2348d3f989cdc7053b26085198415a26_13.png)

以下是命令替换的一个典型应用场景：
*   假设我们经常需要获取当前日期，可以执行 `date` 命令。
*   通过命令替换 `$(date)`，我们可以将日期字符串赋值给一个变量：`CURRENT_DATE=$(date)`。
*   之后，通过引用变量 `$CURRENT_DATE` 或 `${CURRENT_DATE}`，就可以在任何地方使用这个日期值，而无需重复执行 `date` 命令。

![](img/2348d3f989cdc7053b26085198415a26_15.png)

**核心概念公式**：`variable_name=$(shell_command)`

## 大括号扩展：`{}`

接下来，我们学习一种用于生成序列或组合的快捷方式——大括号扩展。

大括号 `{}` 用于生成一系列字符串。它有两种主要形式：枚举列表和序列。

以下是使用大括号扩展的几种情况：
*   **枚举列表**：大括号内用逗号分隔的项会被依次展开，并与外部字符串（如果有）进行组合。
    *   例如，`file{1,2,3}.txt` 会展开为 `file1.txt file2.txt file3.txt`。
    *   例如，创建目录 `mkdir -p dir{A,B,C}/{1,2,3}`，会生成9个目录：`dirA/1`, `dirA/2`, `dirA/3`, `dirB/1`...
*   **序列生成**：使用 `..` 可以生成一个连续的序列（数字或字母）。
    *   例如，`file{4..9}.txt` 会展开为 `file4.txt file5.txt ... file9.txt`。
    *   例如，`{a..d}` 会展开为 `a b c d`。

![](img/2348d3f989cdc7053b26085198415a26_17.png)

![](img/2348d3f989cdc7053b26085198415a26_19.png)

**核心概念代码**：`echo prefix{A,B,C}suffix` 或 `echo item{1..10}`

## 变量基础

现在，我们进入一个编程中的核心概念——变量。变量是用于存储数据的内存空间，其值可以改变。

![](img/2348d3f989cdc7053b26085198415a26_21.png)

### 变量的定义与赋值

![](img/2348d3f989cdc7053b26085198415a26_23.png)

在shell中定义变量非常简单：`变量名=值`。注意，等号两边不能有空格。

![](img/2348d3f989cdc7053b26085198415a26_25.png)

![](img/2348d3f989cdc7053b26085198415a26_27.png)

以下是定义变量时需要遵循的规则和操作：
*   变量名通常使用大写字母，且应以字母或下划线开头，不能以数字开头。
*   为变量赋值：`MY_VAR=100`
*   引用变量的值：在变量名前加美元符号 `$`，如 `echo $MY_VAR`。
*   查看所有已定义的变量：使用 `set` 命令。
*   取消一个变量：使用 `unset` 命令，如 `unset MY_VAR`。

**核心概念代码**：
```bash
MY_VARIABLE="Hello World"  # 定义变量
echo $MY_VARIABLE          # 引用变量
unset MY_VARIABLE          # 取消变量
```

### 变量的引用与`${}`

![](img/2348d3f989cdc7053b26085198415a26_29.png)

![](img/2348d3f989cdc7053b26085198415a26_30.png)

当变量名与其他字符紧邻，可能导致混淆时，需要使用 `${}` 将变量名括起来，以明确其边界。

![](img/2348d3f989cdc7053b26085198415a26_32.png)

例如，假设有一个变量 `A=100`，我们想输出 “100B”。如果写成 `echo $AB`，shell会尝试寻找名为 `AB` 的变量。正确的写法是 `echo ${A}B`，这样shell就能正确识别出变量 `A` 和字符串 `B`。

**核心概念代码**：`echo "The value is ${VAR}units"`

![](img/2348d3f989cdc7053b26085198415a26_34.png)

![](img/2348d3f989cdc7053b26085198415a26_36.png)

![](img/2348d3f989cdc7053b26085198415a26_38.png)

## 变量的作用域：本地变量与环境变量

变量根据其生效的范围，可分为本地变量和环境变量。

### 本地变量

仅在定义它的当前shell进程中有效的变量称为本地变量。一旦退出该shell或开启新的子shell，本地变量将不可见。

![](img/2348d3f989cdc7053b26085198415a26_40.png)

![](img/2348d3f989cdc7053b26085198415a26_42.png)

![](img/2348d3f989cdc7053b26085198415a26_44.png)

**核心概念**：本地变量的作用域是**当前shell**。

![](img/2348d3f989cdc7053b26085198415a26_46.png)

### 环境变量

环境变量不仅在当前shell中有效，还会被传递到从当前shell启动的**任何子shell**中。这使得环境变量非常适合用于配置工作环境（如PATH、HOME）。

![](img/2348d3f989cdc7053b26085198415a26_48.png)

如何定义环境变量呢？有以下两种方法：
*   先定义本地变量，再用 `export` 命令导出：`MY_VAR=value; export MY_VAR`
*   直接定义并导出：`export MY_VAR=value`

![](img/2348d3f989cdc7053b26085198415a26_50.png)

查看所有环境变量可以使用 `env` 或 `printenv` 命令。

**核心概念代码**：`export PATH=$PATH:/my/new/path`

![](img/2348d3f989cdc7053b26085198415a26_52.png)

![](img/2348d3f989cdc7053b26085198415a26_54.png)

### 重要系统环境变量示例

![](img/2348d3f989cdc7053b26085198415a26_56.png)

![](img/2348d3f989cdc7053b26085198415a26_58.png)

系统预定义了许多环境变量，这里介绍两个关键的：

![](img/2348d3f989cdc7053b26085198415a26_60.png)

*   **PS1**：定义了shell的主提示符格式。例如，`\u@\h:\w\$` 表示“用户名@主机名:当前工作目录$”。
*   **PATH**：这是极其重要的变量，它定义了当用户输入一个命令时，shell去哪些目录中查找该命令的可执行文件。目录之间用冒号 `:` 分隔。

![](img/2348d3f989cdc7053b26085198415a26_62.png)

**核心概念**：`PATH` 变量决定了系统如何查找命令。
