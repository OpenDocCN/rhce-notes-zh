当我们在系统中添加一块新硬盘后，它就像一块未经划分的“空地”。在存储文件之前，我们通常需要对其进行“分区”和“格式化”两个步骤。

*   **分区**：类似于将一个大房间划分成多个独立的小房间（如系统区、学习区、娱乐区）。每个分区在逻辑上是独立的存储空间。
*   **格式化**：类似于对每个小房间进行“装修”，建立文件存储的规则和结构（即文件系统），之后才能存放文件。

首先，我们需要确认新硬盘是否被系统识别。可以通过以下命令查看：

```bash
cat /proc/partitions
```
或者
```bash
lsblk
```

如果新添加的硬盘没有立即显示，可能需要重启系统或重新扫描磁盘。

## 分区表格式：MBR vs GPT
在开始分区之前，我们需要了解两种主流的分区表格式：MBR和GPT。它们决定了如何管理磁盘上的分区信息。

### MBR（主引导记录）
MBR是一种较老的分区格式。它将磁盘开头的512字节用于存储引导程序和分区表。
*   **结构**：
    *   前446字节：引导程序。
    *   中间64字节：分区表，每条分区记录占用16字节，因此最多只能记录**4条**主分区信息。
    *   最后2字节：结束标志。
*   **分区类型**：
    1.  **主分区**：可以直接使用，用于安装系统或存储数据。最多只能创建**4个**。
    2.  **扩展分区**：一种特殊的主分区，不能直接存储数据。它的作用是**容纳逻辑分区**。主分区 + 扩展分区 ≤ 4。
    3.  **逻辑分区**：在扩展分区内部创建，数量可以很多，用于实际存储数据。**逻辑分区必须创建在扩展分区之内**。

### GPT（GUID分区表）
GPT是一种现代的分区格式，克服了MBR的限制。
*   支持超过2TB的大容量磁盘。
*   理论上可以创建**128个**主分区，无需使用扩展分区和逻辑分区的概念。
*   具有更强的数据完整性和备份机制。

## 使用 `fdisk` 管理MBR分区
`fdisk` 是用于管理MBR格式磁盘的经典交互式工具。其基本操作模式是进入一个命令行界面，通过输入单字母命令进行操作。

![](img/db460d943495b55234570285edb59b1e_37.png)

![](img/db460d943495b55234570285edb59b1e_39.png)

![](img/db460d943495b55234570285edb59b1e_41.png)

以下是 `fdisk` 的常用命令列表：
*   `m`：显示帮助菜单。
*   `n`：创建一个新分区。
*   `d`：删除一个分区。
*   `p`：打印当前分区表。
*   `t`：更改分区的类型ID（标签）。
*   `w`：将分区表写入磁盘并退出。
*   `q`：不保存更改并退出。

**重要提示**：在 `fdisk` 中执行的所有操作，在输入 `w` 命令之前都仅保存在内存中。如果直接输入 `q` 退出，所有更改都将丢失。

### 操作示例
假设对新硬盘 `/dev/nvme0n2` 进行操作：

1.  **进入交互模式**：
    ```bash
    fdisk /dev/nvme0n2
    ```

2.  **创建主分区**：
    输入 `n` 创建分区，选择 `p` 创建主分区，指定分区编号（如1），设置起始扇区（通常默认），然后设置分区大小（如 `+1G`）。

3.  **查看分区表**：
    输入 `p` 查看当前分区信息。

4.  **创建扩展分区和逻辑分区**：
    *   再次输入 `n` 创建分区。
    *   选择 `e` 创建扩展分区，并分配较大空间（如 `+10G`）。
    *   输入 `n` 创建分区，此时会自动选择 `l` 创建逻辑分区，并设置其大小。

5.  **更改分区类型**：
    输入 `t`，选择分区编号，然后输入类型ID。常用ID有：
    *   `83`：Linux 文件系统（默认）
    *   `82`：Linux 交换分区
    *   `8e`：Linux LVM（逻辑卷）

6.  **保存并退出**：
    确认无误后，输入 `w` 保存所有更改。

有时，分区信息更改后系统未能立即识别，可以使用 `partprobe` 命令手动通知内核重新读取分区表：
```bash
partprobe /dev/nvme0n2
```

## 使用 `gdisk` 管理GPT分区
`gdisk` 是用于管理GPT格式磁盘的工具，其用法与 `fdisk` 非常相似。

*   如果对一个原本是MBR格式的磁盘使用 `gdisk`，工具会尝试在内存中将其转换为GPT格式进行操作。
*   同样，如果对GPT磁盘使用 `fdisk`，也会尝试转换。
*   `gdisk` 支持创建大量分区（最多128个），操作流程与 `fdisk` 的交互模式基本一致。

![](img/db460d943495b55234570285edb59b1e_43.png)

![](img/db460d943495b55234570285edb59b1e_45.png)

基本命令：
```bash
gdisk /dev/nvme0n2
```
进入后，可以使用 `n`, `d`, `p`, `w`, `q` 等命令进行分区操作。

## 使用 `parted` 进行高级分区操作
`parted` 是一个更强大、支持脚本化的分区工具，它可以处理MBR和GPT两种格式。

### 非交互式修改分区表格式
`parted` 的一个常见用途是直接修改磁盘的标签（即分区表格式），而无需进入复杂的交互模式。

*   **将磁盘转换为GPT格式**：
    ```bash
    parted -s /dev/nvme0n2 mklabel gpt
    ```
*   **将磁盘转换为MBR（MSDOS）格式**：
    ```bash
    parted -s /dev/nvme0n2 mklabel msdos
    ```
*   **清除磁盘标签**：
    ```bash
    parted -s /dev/nvme0n2 mklabel loop
    ```

**注意**：`mklabel` 操作会**销毁磁盘上所有的分区信息**，请谨慎使用。

我们首先理解了**分区**和**格式化**是两个必要的步骤。然后，我们深入探讨了**MBR**和**GPT**两种分区表格式的原理与区别，特别是MBR中主分区、扩展分区和逻辑分区的关系。

在实践部分，我们重点掌握了三个分区工具：
1.  **`fdisk`**：用于管理MBR格式磁盘的经典交互式工具。
2.  **`gdisk`**：用于管理GPT格式磁盘，用法与 `fdisk` 类似。
3.  **`parted`**：功能强大的工具，支持两种格式，并能进行非交互式的磁盘标签修改。
