### 什么是程序？
程序是用计算机语言编写的命令序列的集合，用来实现特定的目标或解决特定的问题。

![](img/152ccd2f1b8f80774e97a68364d60758_24.png)

![](img/152ccd2f1b8f80774e97a68364d60758_26.png)

用通俗的话来说，程序就是我们从软件商店下载到电脑里的软件。例如，下载一个聊天工具到电脑后，它就成为了一个应用程序。程序在未运行时，占用的是磁盘空间，它是**静态**且**永久**的。

![](img/152ccd2f1b8f80774e97a68364d60758_28.png)

![](img/152ccd2f1b8f80774e97a68364d60758_30.png)

![](img/152ccd2f1b8f80774e97a68364d60758_32.png)

**核心概念公式**：
```
程序 = 存储在磁盘上的软件/应用
程序状态 = 静态
占用资源 = 磁盘空间
```

![](img/152ccd2f1b8f80774e97a68364d60758_34.png)

![](img/152ccd2f1b8f80774e97a68364d60758_36.png)

### 什么是进程？
进程是正在运行中的程序。当程序被加载到内存中并开始执行时，它就变成了一个进程。

![](img/152ccd2f1b8f80774e97a68364d60758_38.png)

![](img/152ccd2f1b8f80774e97a68364d60758_40.png)

进程占用的是内存空间，它是**动态**的，并且有**生命周期**。生命周期是指进程从启动到结束的整个过程。有些进程（如一条简单的命令`ls`）生命周期极短，瞬间结束；而有些进程（如一个持续运行的后台服务）则需要手动结束。

每个进程启动时，系统都会为它分配独立的内存空间和一个唯一的身份标识——**PID**。此外，每个进程通常都有一个**父进程**，父进程可以创建多个**子进程**，形成父子关系。

**核心概念公式**：
```
进程 = 运行中的程序
进程状态 = 动态
占用资源 = 内存空间
标识 = PID (进程ID)
关系 = 父进程 (PPID) -> 子进程
```

![](img/152ccd2f1b8f80774e97a68364d60758_42.png)

### 什么是线程？
线程，常被称为**轻量级进程**，是进程中实际运作的单元。一个进程可以并发多个线程。

我们可以用一个比喻来理解：将CPU比作一个工厂，CPU的**核心**就是工厂里的生产车间，而**线程**就是每个车间里真正干活的员工。例如，一个“四核八线程”的CPU，就像一个有4个车间，每个车间有2个员工的工厂。线程是真正执行计算、处理任务的最小单位。

对于运维工作而言，我们只需要了解线程是进程中实际干活的单元即可。

**核心概念公式**：
```
线程 ⊆ 进程
线程 = 进程中实际执行任务的单元
比喻：CPU核心 = 车间， 线程 = 员工
```

### 使用 `pstree` 命令查看进程树
`pstree` 命令可以以树状结构显示进程信息，清晰地展示进程之间的父子关系。

![](img/152ccd2f1b8f80774e97a68364d60758_50.png)

以下是 `pstree` 命令的一些常用选项：
*   `-p`：显示每个进程的PID。
*   `-u`：显示进程所属的用户名。
*   `-a`：显示完整的命令行。

![](img/152ccd2f1b8f80774e97a68364d60758_52.png)

![](img/152ccd2f1b8f80774e97a68364d60758_54.png)

**操作示例**：
```bash
# 安装 pstree 命令（如果系统未预装）
yum install -y psmisc

![](img/152ccd2f1b8f80774e97a68364d60758_56.png)

![](img/152ccd2f1b8f80774e97a68364d60758_58.png)

![](img/152ccd2f1b8f80774e97a68364d60758_60.png)

![](img/152ccd2f1b8f80774e97a68364d60758_61.png)

# 以树状结构显示进程
pstree

# 显示进程树及其PID
pstree -p

![](img/152ccd2f1b8f80774e97a68364d60758_63.png)

![](img/152ccd2f1b8f80774e97a68364d60758_65.png)

![](img/152ccd2f1b8f80774e97a68364d60758_67.png)

![](img/152ccd2f1b8f80774e97a68364d60758_69.png)

![](img/152ccd2f1b8f80774e97a68364d60758_71.png)

# 显示指定用户（如test）的进程树
pstree test

![](img/152ccd2f1b8f80774e97a68364d60758_73.png)

![](img/152ccd2f1b8f80774e97a68364d60758_75.png)

# 显示进程树、PID及所属用户
pstree -up
```
在树状结构中，根进程通常是 `systemd`（PID为1），它是所有进程的“祖先”（父进程）。如果进程名后带有数字和星号（如`6*`），星号代表线程，数字代表该进程下的线程数量。

### 使用 `ps` 命令查看进程快照
`ps` 命令用于查看当前时刻的进程状态快照。它有两种常见的选项风格：

![](img/152ccd2f1b8f80774e97a68364d60758_77.png)

![](img/152ccd2f1b8f80774e97a68364d60758_79.png)

**1. BSD风格（无`-`）**
命令 `ps aux` 最为常用。
*   `a`：显示当前终端所有用户的进程。
*   `u`：以用户为主的格式输出。
*   `x`：显示所有进程，包括未控制终端的进程。

**2. Linux标准风格（带`-`）**
命令 `ps -ef` 或 `ps -elf` 也经常使用。
*   `-e`：显示所有进程。
*   `-f`：显示完整格式信息。
*   `-l`：长格式输出。

**`ps aux` 输出列含义解析**：
以下是各列信息的简要说明：
*   **USER**：进程所有者。
*   **PID**：进程ID。
*   **%CPU**：CPU使用百分比。
*   **%MEM**：内存使用百分比。
*   **VSZ**：进程使用的虚拟内存大小。
*   **RSS**：进程使用的物理内存大小。
*   **TTY**：进程运行的终端。
    *   `tty1-tty63`：本地终端。
    *   `pts/0-pts/255`：远程终端。
    *   `?`：内核或后台进程，无控制终端。
*   **STAT**：进程状态。这是需要重点关注的列。
    *   **R**：正在运行。
    *   **S**：休眠中。
    *   **T**：已停止。
    *   **Z**：僵尸进程。
    *   **s**：该进程是会话首进程（包含子进程）。
    *   **+**：位于后台进程组。
*   **START**：进程启动时间。
*   **TIME**：进程占用CPU的总时间。
*   **COMMAND**：启动进程的命令。

![](img/152ccd2f1b8f80774e97a68364d60758_81.png)

**操作示例**：
```bash
# 查看系统所有进程
ps aux

![](img/152ccd2f1b8f80774e97a68364d60758_83.png)

![](img/152ccd2f1b8f80774e97a68364d60758_85.png)

# 结合 grep 过滤特定进程（如 sshd）
ps aux | grep sshd

![](img/152ccd2f1b8f80774e97a68364d60758_87.png)

![](img/152ccd2f1b8f80774e97a68364d60758_89.png)

# 查看进程的完整格式，包含父进程PID（PPID）
ps -ef
# 或
ps -elf

![](img/152ccd2f1b8f80774e97a68364d60758_91.png)

# 查找僵尸进程及其父进程（关键：STAT状态为‘Z’）
ps -eo pid,ppid,stat,cmd | grep -w ‘Z’
```

![](img/152ccd2f1b8f80774e97a68364d60758_93.png)

![](img/152ccd2f1b8f80774e97a68364d60758_95.png)

![](img/152ccd2f1b8f80774e97a68364d60758_97.png)

**关于僵尸进程**：
僵尸进程（状态为`Z`）是指子进程已经终止，但其父进程未能成功回收其资源。僵尸进程本身无法被直接“杀死”，通常需要结束其父进程，由系统父进程`systemd`来回收资源。如果父进程也无法被回收，可能意味着系统出现了更深层次的问题。

## 总结
本节课我们一起学习了操作系统中的核心概念。我们明确了**程序**是静态的磁盘实体，**进程**是动态运行的程序实例，而**线程**是进程中实际执行任务的单元。我们还掌握了在Linux中使用 `pstree` 命令查看进程树状关系，以及使用 `ps aux` 或 `ps -ef` 查看进程详细状态的方法，特别是理解了输出中 `STAT` 状态列的含义。这些知识是后续进行系统监控、性能分析和故障排查的基础。
