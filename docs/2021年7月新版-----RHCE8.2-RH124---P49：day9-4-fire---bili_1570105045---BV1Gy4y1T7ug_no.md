## 流程控制回顾

上一节我们介绍了脚本编写的基础，本节中我们来看看流程控制结构。流程控制主要包括条件控制和循环控制。

### 条件控制

![](img/6b5052a3f37622b180b428bc70a9433e_1.png)

![](img/6b5052a3f37622b180b428bc70a9433e_3.png)

条件控制用于根据不同的测试结果执行不同的代码块。以下是主要的条件控制结构：

![](img/6b5052a3f37622b180b428bc70a9433e_5.png)

![](img/6b5052a3f37622b180b428bc70a9433e_7.png)

*   **if 语句**：用于基本的条件判断。
    ```bash
    if [ 条件测试 ]; then
        # 执行命令
    fi
    ```
*   **if-else 语句**：当条件不满足时，执行另一段代码。
    ```bash
    if [ 条件测试 ]; then
        # 条件为真时执行
    else
        # 条件为假时执行
    fi
    ```
*   **if-elif-else 语句**：用于多分支条件判断。
    ```bash
    if [ 条件测试1 ]; then
        # 条件1为真时执行
    elif [ 条件测试2 ]; then
        # 条件2为真时执行
    else
        # 所有条件都为假时执行
    fi
    ```
*   **case 选择语句**：基于一个变量的值进行多路匹配选择。
    ```bash
    case $变量 in
        模式1)
            # 匹配模式1时执行
            ;;
        模式2)
            # 匹配模式2时执行
            ;;
        *)
            # 默认情况，无匹配时执行
            echo "输入错误"
            ;;
    esac
    ```

### 循环控制

![](img/6b5052a3f37622b180b428bc70a9433e_9.png)

循环控制用于重复执行一段代码。以下是主要的循环结构：

*   **for 循环**：遍历一个列表或执行指定次数的循环。
    *   遍历列表：
        ```bash
        for i in 列表; do
            # 执行命令
        done
        ```
    *   C语言风格：
        ```bash
        for ((i=0; i<10; i++)); do
            # 执行命令
        done
        ```
*   **while 循环**：当条件为真时，持续执行循环。
    ```bash
    while [ 条件测试 ]; do
        # 执行命令
    done
    ```

### 循环控制关键字

在循环中，可以使用特定关键字来控制流程：

![](img/6b5052a3f37622b180b428bc70a9433e_11.png)

![](img/6b5052a3f37622b180b428bc70a9433e_13.png)

*   **continue**：跳过当前循环的剩余代码，直接进入下一轮循环。
*   **break**：立即终止整个循环。

## SELinux 安全上下文管理

在深入防火墙之前，我们先简要回顾SELinux的管理。SELinux为系统和文件提供了额外的安全标签。

### SELinux 状态管理

![](img/6b5052a3f37622b180b428bc70a9433e_15.png)

![](img/6b5052a3f37622b180b428bc70a9433e_17.png)

以下是管理SELinux状态的基本命令：

![](img/6b5052a3f37622b180b428bc70a9433e_19.png)

*   **查看状态**：`getenforce`
*   **临时设置**：
    *   启用：`setenforce 1`
    *   禁用：`setenforce 0`
*   **永久设置**：编辑配置文件 `/etc/selinux/config`，将 `SELINUX=` 的值改为 `disabled`，然后重启系统。

![](img/6b5052a3f37622b180b428bc70a9433e_21.png)

### SELinux 标签管理

![](img/6b5052a3f37622b180b428bc70a9433e_23.png)

SELinux的标签关联着系统的安全策略。

![](img/6b5052a3f37622b180b428bc70a9433e_25.png)

![](img/6b5052a3f37622b180b428bc70a9433e_27.png)

*   **查看文件标签**：`ls -lZ` 或 `ls -ldZ`（针对目录）
*   **查看进程标签**：`ps -efZ`
*   **修改文件标签**：使用 `semanage` 和 `restorecon` 命令。
    *   查询文件默认上下文：`semanage fcontext -l | grep 文件路径`
    *   添加新的上下文规则：`semanage fcontext -a -t 上下文类型 “文件路径正则表达式”`
    *   恢复文件默认上下文：`restorecon -v 文件或目录路径`

## Firewalld 防火墙基本规则 🔥

现在，我们进入本节课的核心内容——防火墙。Firewalld通过“区域（zone）”来管理网络流量规则，每个区域可以定义不同的规则集。

![](img/6b5052a3f37622b180b428bc70a9433e_33.png)

![](img/6b5052a3f37622b180b428bc70a9433e_35.png)

### 工作原理与规则匹配

![](img/6b5052a3f37622b180b428bc70a9433e_37.png)

![](img/6b5052a3f37622b180b428bc70a9433e_39.png)

防火墙的工作基于区域。规则定义于每一个区域中，不同的区域可以定义不同的规则。

防火墙如何选择使用哪个区域的规则呢？匹配顺序如下：
1.  首先，通过**源IP地址（source）**匹配。
2.  其次，通过数据包进入的**网络接口（interface）**匹配。
3.  如果以上都未匹配，则使用该接口所属的**默认区域（zone）**的规则。

### 规则配置实践

![](img/6b5052a3f37622b180b428bc70a9433e_41.png)

![](img/6b5052a3f37622b180b428bc70a9433e_43.png)

防火墙规则可以通过多种条件来定义。

以下是添加规则的几种主要方式：

![](img/6b5052a3f37622b180b428bc70a9433e_45.png)

![](img/6b5052a3f37622b180b428bc70a9433e_47.png)

*   **通过服务（service）添加**：允许特定的预定义服务（如http、ssh）。
    ```bash
    firewall-cmd --zone=work --add-service=http --permanent
    ```
*   **通过端口（port）添加**：允许特定的端口和协议。
    ```bash
    firewall-cmd --zone=public --add-port=80/tcp --permanent
    ```
*   **指定源IP（source）**：仅允许来自特定IP或网段的流量访问本区域规则。
    ```bash
    firewall-cmd --zone=trusted --add-source=192.168.1.0/24 --permanent
    ```
*   **指定接口（interface）**：将某个网络接口（如ens160）绑定到特定区域，从此接口进入的流量应用该区域规则。
    ```bash
    firewall-cmd --zone=work --change-interface=ens160 --permanent
    ```

![](img/6b5052a3f37622b180b428bc70a9433e_49.png)

![](img/6b5052a3f37622b180b428bc70a9433e_51.png)

**注意**：使用 `--permanent` 选项表示永久生效，配置后需执行 `firewall-cmd --reload` 重载配置。不加此选项则为运行时规则，重启后失效。

### 默认区域与规则目标（Target）

每个区域都有一个 `target` 设置。它的作用是：当进入该区域的流量**没有匹配到任何一条明确允许的规则**时，将按照 `target` 执行默认操作。

`target` 的常见值有：
*   **ACCEPT**：接受流量。
*   **DROP**：丢弃流量，不回应任何信息。
*   **REJECT**：拒绝流量，并向发送方返回拒绝信息。
*   **default**：跳转到默认区域（通常为 `public`）的规则继续进行匹配。

你可以使用以下命令修改区域的默认 `target`：
```bash
firewall-cmd --zone=work --set-target=ACCEPT --permanent
firewall-cmd --reload
```

**`DROP` 与 `REJECT` 的区别**：
*   `DROP` 像“已读不回”，对方会一直等待直到超时，体验更“隐蔽”。
*   `REJECT` 像“直接拒绝”，对方会立刻收到拒绝响应，知道访问被阻。
