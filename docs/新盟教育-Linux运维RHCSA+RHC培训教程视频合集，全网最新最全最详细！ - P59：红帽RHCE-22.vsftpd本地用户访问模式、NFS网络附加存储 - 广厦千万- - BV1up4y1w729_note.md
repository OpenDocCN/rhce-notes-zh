## 禁用匿名用户访问模式 🔒

![](img/8585e1271d952e9cea9ea257891d6929_7.png)

![](img/8585e1271d952e9cea9ea257891d6929_8.png)

上一节我们介绍了vsftpd的匿名用户访问模式。本节中，我们来看看如何禁用匿名访问，转而使用本地用户模式。

![](img/8585e1271d952e9cea9ea257891d6929_10.png)

![](img/8585e1271d952e9cea9ea257891d6929_12.png)

企业环境中，出于安全考虑，可能希望只允许特定的本地用户访问FTP服务器，而不是任何人都能匿名访问。这需要禁用匿名用户模式。

以下是禁用匿名用户访问的步骤：

1.  编辑vsftpd的主配置文件 `/etc/vsftpd/vsftpd.conf`。
2.  找到控制匿名访问的参数 `anonymous_enable`。
3.  将其值修改为 `NO`。**注意**：注释掉该行（前面加`#`）默认是允许匿名访问的，因此必须明确设置为 `NO`。
    ```bash
    anonymous_enable=NO
    ```
4.  保存配置文件并重启vsftpd服务使更改生效。
    ```bash
    systemctl restart vsftpd
    ```
5.  配置完成后，客户端尝试使用匿名方式连接将失败。

## 配置与使用本地用户 👤

![](img/8585e1271d952e9cea9ea257891d6929_14.png)

![](img/8585e1271d952e9cea9ea257891d6929_16.png)

![](img/8585e1271d952e9cea9ea257891d6929_18.png)

![](img/8585e1271d952e9cea9ea257891d6929_20.png)

禁用匿名访问后，我们就可以配置并使用本地用户来访问FTP服务器了。

首先，需要在服务器上创建一个用于FTP访问的本地用户，例如 `ftpuser`。
```bash
useradd ftpuser
```
接下来，客户端可以使用此用户名和密码进行连接。连接命令如下：
```bash
lftp ftpuser@192.168.0.40
# 或
ftp 192.168.0.40
# 然后在提示下输入用户名 ftpuser 及其密码
```
成功连接后，用户会进入其在服务器上的家目录（例如 `/home/ftpuser`）。用户只能看到和操作自己家目录下的文件。管理员可以将需要共享的文件放置于此目录下。

![](img/8585e1271d952e9cea9ea257891d6929_22.png)

![](img/8585e1271d952e9cea9ea257891d6929_24.png)

![](img/8585e1271d952e9cea9ea257891d6929_26.png)

本地用户默认具备对其家目录的完整读写权限（如创建、删除、重命名文件）。这是因为系统权限和vsftpd配置共同作用的结果。

![](img/8585e1271d952e9cea9ea257891d6929_28.png)

## 限制本地用户的活动范围 🚧

默认情况下，本地FTP用户登录后可以切换到服务器文件系统的其他目录，这存在安全风险。通常我们希望将用户的活动范围限制在其家目录内。

vsftpd通过 `chroot_local_user` 参数来实现这一功能，这被称为“切根”（chroot），即将用户的根目录切换到其家目录。

![](img/8585e1271d952e9cea9ea257891d6929_30.png)

以下是限制本地用户活动范围的步骤：

![](img/8585e1271d952e9cea9ea257891d6929_32.png)

1.  在配置文件 `/etc/vsftpd/vsftpd.conf` 中，找到并取消 `chroot_local_user=YES` 这一行的注释。
    ```bash
    chroot_local_user=YES
    ```
2.  保存并重启vsftpd服务。
3.  此时，如果用户家目录对用户本身具有写权限（`w`），连接可能会失败并提示错误。这是vsftpd出于安全考虑的设计。
4.  需要移除用户对其家目录的写权限。
    ```bash
    chmod a-w /home/ftpuser
    ```
5.  为了允许用户上传文件，可以在其家目录下创建一个具有写权限的子目录（例如 `pub`），并将该目录的所有者改为相应用户。
    ```bash
    mkdir /home/ftpuser/pub
    chown ftpuser:ftpuser /home/ftpuser/pub
    ```
6.  完成上述设置后，用户登录后将只能在其家目录内活动，并且可以在 `pub` 目录内进行上传等写操作。

![](img/8585e1271d952e9cea9ea257891d6929_34.png)

## 控制本地用户的文件操作权限 ⚙️

![](img/8585e1271d952e9cea9ea257891d6929_36.png)

我们可以通过vsftpd的配置文件进一步控制本地用户的操作权限。

![](img/8585e1271d952e9cea9ea257891d6929_38.png)

![](img/8585e1271d952e9cea9ea257891d6929_40.png)

![](img/8585e1271d952e9cea9ea257891d6929_42.png)

![](img/8585e1271d952e9cea9ea257891d6929_44.png)

![](img/8585e1271d952e9cea9ea257891d6929_46.png)

以下是一些关键参数及其作用：

![](img/8585e1271d952e9cea9ea257891d6929_48.png)

![](img/8585e1271d952e9cea9ea257891d6929_50.png)

*   `write_enable=YES`：此参数控制是否允许本地用户进行上传、删除、重命名等写操作。如果设置为 `NO`，则用户只能下载和查看文件。
*   `local_umask=022`：此参数设置本地用户上传文件的默认权限掩码。`022` 表示创建的文件权限为 `644`（所有者可读写，其他人只读），目录权限为 `755`。

![](img/8585e1271d952e9cea9ea257891d6929_52.png)

![](img/8585e1271d952e9cea9ea257891d6929_54.png)

修改这些参数后，需要重启vsftpd服务才能生效。

## 用户访问控制名单 📋

vsftpd提供了用户名单机制来控制允许或拒绝访问的用户。

![](img/8585e1271d952e9cea9ea257891d6929_56.png)

![](img/8585e1271d952e9cea9ea257891d6929_58.png)

![](img/8585e1271d952e9cea9ea257891d6929_59.png)

![](img/8585e1271d952e9cea9ea257891d6929_61.png)

以下是相关的文件和配置：

![](img/8585e1271d952e9cea9ea257891d6929_63.png)

*   `/etc/vsftpd/user_list`：此文件用途灵活，可配置为黑名单或白名单。
*   `/etc/vsftpd/vsftpd.conf` 中的相关参数：
    *   `userlist_enable=YES`：启用 `user_list` 文件。
    *   `userlist_deny=YES`：当设置为 `YES` 时，`user_list` 文件作为**黑名单**，列出的用户被拒绝访问。
    *   `userlist_deny=NO`：当设置为 `NO` 时，`user_list` 文件作为**白名单**，只允许列出的用户访问，其他用户均被拒绝。
*   `/etc/vsftpd/ftpusers`：这是一个固定的**黑名单**文件，其中列出的用户（默认包括root等系统用户）禁止通过FTP登录。

![](img/8585e1271d952e9cea9ea257891d6929_65.png)

![](img/8585e1271d952e9cea9ea257891d6929_66.png)

在实际应用中，通常使用 `ftpusers` 作为黑名单，而 `user_list` 的复杂配置较少使用。

## 配置NFS网络附加存储 💾

![](img/8585e1271d952e9cea9ea257891d6929_72.png)

![](img/8585e1271d952e9cea9ea257891d6929_73.png)

NFS（Network File System）是另一种常见的网络文件共享服务，在数据备份等场景中应用广泛。其配置比FTP更为简洁。

![](img/8585e1271d952e9cea9ea257891d6929_75.png)

![](img/8585e1271d952e9cea9ea257891d6929_77.png)

下面我们来看如何快速搭建一个NFS服务器。

首先，在服务器端安装必要的软件包并启动服务：
```bash
yum install -y nfs-utils
systemctl start nfs
systemctl enable nfs
```
然后，创建一个准备共享的目录：
```bash
mkdir /nfs_upload
```
接着，编辑NFS的配置文件 `/etc/exports`，指定共享目录、允许访问的客户端及权限：
```bash
/nfs_upload 192.168.0.10(rw)
# 格式：本地目录 客户端IP或网段(权限参数)
```
保存配置文件后，重新导出共享列表使配置生效：
```bash
exportfs -arv
```
现在，客户端可以查看NFS服务器提供的共享目录：
```bash
showmount -e 192.168.0.40
```
最后，客户端将远程共享目录挂载到本地的一个空目录下使用：
```bash
mkdir /local_nfs
mount -t nfs 192.168.0.40:/nfs_upload /local_nfs
```
挂载成功后，客户端向 `/local_nfs` 目录写入数据，实际上会存储到NFS服务器的 `/nfs_upload` 目录中，实现了网络存储。

为了使挂载在客户端重启后依然有效，需要将挂载信息写入 `/etc/fstab` 文件：
```bash
192.168.0.40:/nfs_upload /local_nfs nfs defaults,_netdev 0 0
```
参数 `_netdev` 指明这是一个网络设备，防止因网络或服务器未就绪导致系统启动失败。

## NFS的权限问题与解决 🔑

NFS服务在处理权限时，默认会将客户端的root用户映射为服务器端的nfsnobody用户，这可能导致权限不足。

以下是常见的解决方法：

![](img/8585e1271d952e9cea9ea257891d6929_82.png)

![](img/8585e1271d952e9cea9ea257891d6929_84.png)

1.  **允许root用户保持权限**：在服务器端的 `/etc/exports` 文件中，为共享目录添加 `no_root_squash` 参数。
    ```bash
    /nfs_upload 192.168.0.10(rw,no_root_squash)
    ```
2.  **放宽目录权限**：如果客户端有多个普通用户需要访问，可以简单地将服务器端的共享目录权限设置为 `777`。这在备份场景中较为常见。
    ```bash
    chmod 777 /nfs_upload
    ```
3.  **使用ACL进行精细控制**：对于更复杂的权限需求，可以使用ACL（访问控制列表），但这超出了基础使用的范围。

本节课中我们一起学习了vsftpd和NFS两种重要的网络文件共享服务。

*   对于**vsftpd**，我们掌握了如何从匿名模式切换到更安全的本地用户模式，包括创建用户、禁用匿名访问、限制用户活动范围（chroot）以及控制用户的读写权限。我们还了解了用户访问控制名单（黑名单/白名单）的基本概念。
