![](img/3337d0897255e075a092a686e6588977_1.png)

在本节课中，我们将要学习Linux系统中的两个核心技能：**重定向**和**vim编辑器**。重定向允许我们控制命令的输入和输出流向，而vim则是一个功能强大的文本编辑器，是系统管理员和开发者的必备工具。掌握它们对于高效地进行系统管理和文件操作至关重要。

## 重定向

上一节我们回顾了基础命令，本节中我们来看看如何控制命令的输入和输出。默认情况下，命令的执行结果会输出到终端屏幕上。但我们可以通过**重定向**的方式，将这些结果导入到文件中，或者从文件中读取输入。

这种情况主要应用于日志收集和排错。因为输出到屏幕上的信息稍纵即逝，而重定向可以帮助我们保存重要的信息。如果信息没有价值，我们也可以将其丢弃到系统“垃圾桶”——`/dev/null`设备。

### 输出类型与重定向符号

![](img/3337d0897255e075a092a686e6588977_7.png)

![](img/3337d0897255e075a092a686e6588977_8.png)

![](img/3337d0897255e075a092a686e6588977_10.png)

![](img/3337d0897255e075a092a686e6588977_11.png)

在Linux系统中，命令的输出分为两种类型：
*   **标准输出 (stdout)**：命令的正常输出结果，其文件描述符为 **1**。
*   **标准错误 (stderr)**：命令的异常或错误输出结果，其文件描述符为 **2**。

![](img/3337d0897255e075a092a686e6588977_13.png)

![](img/3337d0897255e075a092a686e6588977_14.png)

此外，命令的输入（文件描述符为 **0**）也可以被重定向。

![](img/3337d0897255e075a092a686e6588977_16.png)

![](img/3337d0897255e075a092a686e6588977_17.png)

![](img/3337d0897255e075a092a686e6588977_19.png)

![](img/3337d0897255e075a092a686e6588977_20.png)

以下是重定向的核心操作符：

![](img/3337d0897255e075a092a686e6588977_22.png)

![](img/3337d0897255e075a092a686e6588977_23.png)

*   `>` 或 `1>`：将**标准输出**覆盖到指定文件。
*   `>>` 或 `1>>`：将**标准输出**追加到指定文件末尾。
*   `2>`：将**标准错误**覆盖到指定文件。
*   `2>>`：将**标准错误**追加到指定文件末尾。
*   `&>` 或 `2>&1`：将**标准输出和标准错误**同时重定向到同一个文件。
*   `<`：将文件内容作为命令的输入。

### 重定向示例

![](img/3337d0897255e075a092a686e6588977_25.png)

让我们通过一些例子来理解这些操作。

![](img/3337d0897255e075a092a686e6588977_27.png)

![](img/3337d0897255e075a092a686e6588977_29.png)

**示例1：将命令输出保存到文件**
```bash
# 将 ls -l /tmp 的结果覆盖保存到文件 /tmp/f1 中
ls -l /tmp > /tmp/f1
# 查看文件内容
cat /tmp/f1
```

**示例2：追加输出到文件**
```bash
# 将当前工作目录信息追加到文件 /tmp/f1 的末尾
pwd >> /tmp/f1
```

![](img/3337d0897255e075a092a686e6588977_31.png)

![](img/3337d0897255e075a092a686e6588977_32.png)

![](img/3337d0897255e075a092a686e6588977_34.png)

![](img/3337d0897255e075a092a686e6588977_36.png)

**示例3：重定向错误输出**
```bash
# 尝试切换到无权限的目录，并将错误信息保存到文件 error.log
cd /root 2> error.log
# 查看错误信息
cat error.log
```

![](img/3337d0897255e075a092a686e6588977_38.png)

![](img/3337d0897255e075a092a686e6588977_40.png)

**示例4：丢弃输出**
```bash
# 执行命令，并将所有输出（包括错误）丢弃到 /dev/null
some_command &> /dev/null
```

![](img/3337d0897255e075a092a686e6588977_42.png)

![](img/3337d0897255e075a092a686e6588977_44.png)

![](img/3337d0897255e075a092a686e6588977_46.png)

**示例5：同时重定向标准输出和错误**
```bash
# 查找文件，并将所有输出（正确和错误）都保存到 result.txt
find /home -user student &> result.txt
```

管道符 `|` 用于将一个命令的**标准输出**作为另一个命令的**标准输入**。这是组合简单命令完成复杂任务的强大工具。

![](img/3337d0897255e075a092a686e6588977_73.png)

![](img/3337d0897255e075a092a686e6588977_75.png)

![](img/3337d0897255e075a092a686e6588977_77.png)

**管道示例**：
```bash
# 查看 /etc/passwd 文件，并通过 more 命令分页显示
cat /etc/passwd | more
# 统计 /etc/passwd 文件的行数
cat /etc/passwd | wc -l
# 或者更简洁的写法
wc -l < /etc/passwd
```

![](img/3337d0897255e075a092a686e6588977_79.png)

![](img/3337d0897255e075a092a686e6588977_81.png)

**tee命令**：`tee` 命令用于同时将数据输出到屏幕（标准输出）和文件。
```bash
# 查看文件内容，并在屏幕上显示的同时保存到另一个文件
cat /tmp/f1 | tee /tmp/f2
```

### Vim的三种模式

![](img/3337d0897255e075a092a686e6588977_97.png)

Vim的操作主要围绕三种模式进行，理解它们是熟练使用Vim的关键：

![](img/3337d0897255e075a092a686e6588977_103.png)

![](img/3337d0897255e075a092a686e6588977_105.png)

![](img/3337d0897255e075a092a686e6588977_107.png)

![](img/3337d0897255e075a092a686e6588977_108.png)

![](img/3337d0897255e075a092a686e6588977_110.png)

1.  **命令模式 (Command Mode)**：启动Vim后默认进入的模式。在此模式下可以执行复制、粘贴、删除、查找等快捷操作，但不能直接输入文本。按 `ESC` 键可以从其他模式返回此模式。
2.  **插入模式 (Insert Mode)**：在此模式下可以像普通编辑器一样输入和编辑文本。从命令模式按下以下键进入：
    *   `i`：在光标当前位置前开始插入。
    *   `I`：在光标所在行的行首开始插入。
    *   `a`：在光标当前位置后开始插入。
    *   `A`：在光标所在行的行末开始插入。
    *   `o`：在光标所在行的下方新建一行并开始插入。
    *   `O`：在光标所在行的上方新建一行并开始插入。
3.  **末行模式 (Last Line Mode)**：在命令模式下按 `:` 进入。用于执行保存、退出、查找替换、显示行号等需要输入命令的操作。

以下是三种模式下最常用的操作列表。

![](img/3337d0897255e075a092a686e6588977_135.png)

![](img/3337d0897255e075a092a686e6588977_137.png)

**命令模式下的操作**：
*   **光标移动**：
    *   `gg`：跳到文件第一行。
    *   `G`：跳到文件最后一行。
    *   `:n`：在末行模式下，跳转到第n行（如 `:10` 跳转到第10行）。
*   **复制、粘贴、删除**：
    *   `yy`：复制当前行。
    *   `nyy`：复制从当前行开始的n行（如 `3yy`）。
    *   `p`：在光标下一行粘贴。
    *   `P`：在光标上一行粘贴。
    *   `dd`：剪切（删除）当前行。
    *   `ndd`：剪切（删除）从当前行开始的n行。
    *   `x`：删除光标所在位置的字符。
    *   `X`：删除光标前一个字符。
*   **撤销与替换**：
    *   `u`：撤销上一次操作。
    *   `r`：替换光标所在处的一个字符（按`r`后再按新字符）。
*   **查找**：
    *   `/keyword`：从光标处向后查找关键字 `keyword`，按 `n` 查找下一个，按 `N` 查找上一个。

![](img/3337d0897255e075a092a686e6588977_139.png)

![](img/3337d0897255e075a092a686e6588977_141.png)

![](img/3337d0897255e075a092a686e6588977_143.png)

![](img/3337d0897255e075a092a686e6588977_145.png)

**末行模式下的操作**：
*   **保存与退出**：
    *   `:w`：保存文件。
    *   `:q`：退出Vim（如果文件有修改未保存，会提示）。
    *   `:wq` 或 `:x`：保存并退出。
    *   `:q!`：强制退出，不保存修改。
    *   `:wq!`：强制保存并退出（常用于只读文件的保存，需要权限）。
*   **显示与设置**：
    *   `:set nu` 或 `:set number`：显示行号。
    *   `:set nonu` 或 `:set nonumber`：取消显示行号。
*   **替换文本**：
    *   `:%s/old/new/g`：将文件中所有的 `old` 替换为 `new`。
    *   `:n,ms/old/new/g`：将第n行到第m行之间的所有 `old` 替换为 `new`。
    *   **注意**：如果 `old` 或 `new` 中包含 `/`，需要用 `\` 进行转义，例如 `:%s/\/bin\/nologin/\/bin\/bash/g`。
*   **执行外部命令与读入内容**：
    *   `:!command`：暂时离开Vim执行外部命令 `command`（如 `:!ls -l`），按回车返回。
    *   `:r filename`：将文件 `filename` 的内容读入到光标下方。
    *   `:r !command`：将外部命令 `command` 的执行结果读入到光标下方。

### 高级功能

**分屏编辑**：
*   `vim -o file1 file2`：水平分屏同时编辑多个文件。
*   `vim -O file1 file2`：垂直分屏同时编辑多个文件。
*   在分屏间切换：`Ctrl+w` 然后按方向键（`h`, `j`, `k`, `l`）。

![](img/3337d0897255e075a092a686e6588977_149.png)

![](img/3337d0897255e075a092a686e6588977_151.png)

**可视化模式**：
*   `v`：进入字符可视化模式，可选中字符。
*   `V`：进入行可视化模式，可整行选中。
*   `Ctrl+v`：进入块可视化模式，可进行矩形区域选择。
    在可视化模式下选中内容后，可以进行 `y`（复制）、`d`（删除）等操作。

![](img/3337d0897255e075a092a686e6588977_153.png)

![](img/3337d0897255e075a092a686e6588977_155.png)

**恢复文件**：
Vim在编辑文件时会生成一个交换文件（`.filename.swp`），用于在异常退出时恢复。如果打开文件时看到相关警告，可以选择：
*   `:recover`：恢复文件。
*   `vim -r filename`：恢复文件。
*   直接删除交换文件（`rm .filename.swp`），如果确认不需要恢复。

![](img/3337d0897255e075a092a686e6588977_159.png)

**多行注释/取消注释（实用技巧）**：
*   **添加注释**：在命令模式下，`Ctrl+v` 进入块选择模式，选中需要注释的行首，按 `I` 进入插入模式，输入 `#`，然后按 `ESC`，所选行首将全部添加 `#`。
*   **取消注释**：`Ctrl+v` 进入块选择模式，选中行首的 `#` 字符，按 `d` 删除。
