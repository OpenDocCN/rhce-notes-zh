## Linux核心概念解析 🐧

了解了课程全景后，我们进入正题，解析Linux的核心概念。

**Linux是什么？**
Linux严格来说指的是**内核**（Kernel），它是介于硬件和操作系统之间的一层核心程序。我们常说的“Linux系统”是指**内核**加上各种软件组成的**发行版**（Distribution）。

**Linux为何流行？（对比Windows）**
以下是Linux的主要优势：
*   **开源**：开放源代码，允许根据业务进行深度定制和优化。**注意：开源不等于免费**。
*   **免费**：大多数发行版和软件可免费使用。
*   **稳定高效**：无需图形界面等冗余组件，系统精简，占用资源少。
*   **安全性高**：代码公开接受监督，权限控制严格（遵循最小权限原则）。
*   **多用户多任务**：支持多个用户同时登录并执行各自任务。
*   **应用广泛**：是云计算、大数据、人工智能、区块链等领域的基石。

**内核版本规则**
内核版本号格式为 `主版本.次版本.修订版本`。例如 `3.10.0`。
*   **次版本为奇数**：开发版，不稳定。
*   **次版本为偶数**：稳定版。**生产环境务必选择稳定版**。

## 主流Linux发行版 🏷️

![](img/24b7df5d9f73bb2840f7ca5be6615860_13.png)

理解了Linux内核，我们来看看基于它的各种发行版。以下是主流发行版及其特点：

*   **Red Hat Enterprise Linux (RHEL)**：红帽企业版，面向企业收费，提供技术支持。常见于银行、证券等传统行业。
*   **CentOS**：社区企业操作系统，被红帽收购。**完全免费**，且与同版本RHEL**二进制兼容**，是互联网公司首选。
*   **Fedora**：红帽的桌面版及新技术“试验田”。
*   **Ubuntu**：基于Debian，流行的个人桌面及开发环境发行版。
*   **Debian**：以稳定著称，在欧洲流行。
*   **Kali Linux**：专为渗透测试和安全审计设计。
*   **openSUSE**：德国开发的发行版（SUSE企业版收费）。
*   **麒麟**：国产优秀发行版（已停止维护）。
*   **Raspberry Pi OS**：树莓派官方系统，常用于物联网、教育。

**学习建议**：为了就业，运维学习应**以CentOS/RHEL 7为主**。开发方向可关注Ubuntu。

![](img/24b7df5d9f73bb2840f7ca5be6615860_27.png)

### 1. 创建虚拟机
使用VMware Workstation创建新虚拟机。
*   选择“典型”安装，“稍后安装操作系统”。
*   客户机操作系统选择“Linux”，版本选择“CentOS 7 64位”。
*   虚拟机名称和位置可自定义。
*   **磁盘选择“将虚拟磁盘拆分成多个文件”**。
*   在“自定义硬件”中，移除不必要的设备（如USB控制器、声卡、打印机）。
*   **网络适配器选择“桥接模式”**，这最接近生产环境，可使虚拟机与物理机处于同一局域网。

![](img/24b7df5d9f73bb2840f7ca5be6615860_33.png)

![](img/24b7df5d9f73bb2840f7ca5be6615860_35.png)

**桥接模式原理**：虚拟网卡将物理网卡当作交换机，虚拟机与物理机IP在同一网段，可直接通信。

### 2. 安装CentOS 7
*   加载CentOS 7镜像，启动虚拟机。
*   选择第一项“Install CentOS 7”开始安装。
*   语言建议选择“English”。
*   **安装目的地**：选择虚拟磁盘，分区方案选择默认的“自动配置”即可（生产环境或学习时会详解手动分区）。
*   **软件选择**：**务必选择“Minimal Install”（最小化安装）**，不安装图形界面。
*   **设置root密码**：为root管理员账户设置密码（如密码太简单需点击两次完成）。
*   开始安装，等待完成并重启。

![](img/24b7df5d9f73bb2840f7ca5be6615860_45.png)

![](img/24b7df5d9f73bb2840f7ca5be6615860_47.png)

![](img/24b7df5d9f73bb2840f7ca5be6615860_49.png)

### 3. 基础网络配置
系统启动后，使用root账户登录。首先配置静态IP地址以便远程连接。
1.  编辑网卡配置文件（网卡名可能是ens33、eth0等）：
    ```bash
    vi /etc/sysconfig/network-scripts/ifcfg-ens33
    ```
2.  按 `i` 键进入编辑模式（INSERT），修改以下关键参数：
    *   `BOOTPROTO=static` # 将DHCP改为静态
    *   `ONBOOT=yes` # 开机自启
    *   添加以下配置（IP地址根据你的实际网络环境填写）：
        ```
        IPADDR=192.168.1.100   # 静态IP
        NETMASK=255.255.255.0  # 子网掩码
        GATEWAY=192.168.1.1    # 网关
        DNS1=114.114.114.114   # DNS服务器
        ```
3.  按 `ESC` 键退出编辑模式，输入 `:wq` 保存并退出。
4.  重启网络服务使配置生效：
    ```bash
    systemctl restart network
    ```

### 4. 使用SSH远程连接
在物理机上使用SSH客户端（如Xshell、SecureCRT、PuTTY）连接虚拟机。
*   主机地址：填你刚才设置的静态IP（如`192.168.1.100`）。
*   端口：22。
*   用户名：root。
*   密码：你设置的root密码。
连接成功后，即可在更友好的界面中进行后续操作。
