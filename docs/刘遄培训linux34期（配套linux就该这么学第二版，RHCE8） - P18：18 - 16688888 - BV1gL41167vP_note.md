**核心概念**：Apache的访问控制可以通过`SetEnvIf`指令和`Order`指令实现，基于用户代理（User-Agent）等条件进行限制。

![](img/15de64435522c49018a342e00938485b_32.png)

![](img/15de64435522c49018a342e00938485b_33.png)

**操作步骤**：
1.  编辑Apache主配置文件 `/etc/httpd/conf/httpd.conf`。
2.  在对应的`<Directory>`区域（例如`/var/www/html/server`）内添加以下配置：
    ```apache
    SetEnvIf User-Agent “Firefox” ff=1
    Order allow,deny
    Allow from env=ff
    ```
    **公式/代码解释**：
    *   `SetEnvIf User-Agent “Firefox” ff=1`：如果用户浏览器的标识（User-Agent）包含“Firefox”，则设置环境变量`ff`的值为1。
    *   `Order allow,deny`：先处理`Allow`指令，再处理`Deny`指令。
    *   `Allow from env=ff`：仅允许环境变量`ff`为1的请求访问。
3.  重启Apache服务使配置生效：`systemctl restart httpd`。

![](img/15de64435522c49018a342e00938485b_35.png)

![](img/15de64435522c49018a342e00938485b_37.png)

**排错思路**：如果配置后访问失败，请按顺序检查：
1.  Apache服务是否正常运行。
2.  网络连通性。
3.  防火墙是否放行了HTTP/HTTPS端口（80/443）。

### 三种验证模式详解

![](img/15de64435522c49018a342e00938485b_81.png)

VSFTPD支持三种主要的用户验证模式，每种模式登录后的默认目录不同。

![](img/15de64435522c49018a342e00938485b_88.png)

![](img/15de64435522c49018a342e00938485b_90.png)

以下是三种模式的对比与登录路径：

![](img/15de64435522c49018a342e00938485b_92.png)

![](img/15de64435522c49018a342e00938485b_94.png)

| 模式 | 描述 | 登录后默认目录 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **匿名公开模式** | 无需密码，使用`anonymous`或`ftp`用户登录。 | `/var/ftp/` | 公共文件下载 |
| **本地用户模式** | 使用系统本地用户账号和密码登录。 | 用户自己的家目录（如`/home/username`） | 内部用户管理 |
| **虚拟用户模式** | 使用独立于系统的虚拟用户账号登录，映射到某个本地用户。 | **映射用户**的家目录 | 安全隔离的FTP服务 |

#### 1. 匿名公开模式配置

![](img/15de64435522c49018a342e00938485b_103.png)

![](img/15de64435522c49018a342e00938485b_104.png)

![](img/15de64435522c49018a342e00938485b_106.png)

编辑`/etc/vsftpd/vsftpd.conf`，确保以下参数：
```bash
anonymous_enable=YES # 允许匿名登录
anon_upload_enable=YES # 允许匿名用户上传
anon_mkdir_write_enable=YES # 允许匿名用户创建目录
anon_other_write_enable=YES # 允许匿名用户其他写操作（重命名、删除）
```
此外，需要确保匿名用户目录`/var/ftp/`有相应写入权限，并通过SELinux命令开放权限：
```bash
setsebool -P ftpd_anon_write on
setsebool -P ftpd_full_access on
```

#### 3. 虚拟用户模式配置（进阶）

![](img/15de64435522c49018a342e00938485b_122.png)

![](img/15de64435522c49018a342e00938485b_124.png)

虚拟用户模式更安全，用户独立于系统。

![](img/15de64435522c49018a342e00938485b_126.png)

![](img/15de64435522c49018a342e00938485b_128.png)

![](img/15de64435522c49018a342e00938485b_129.png)

**配置步骤**：
1.  **创建用户密码文件**：如`/etc/vsftpd/vuser.list`，奇数行为用户名，偶数行为密码。
2.  **生成数据库文件**：使用`db_load`命令加密明文文件。
    ```bash
    db_load -T -t hash -f /etc/vsftpd/vuser.list /etc/vsftpd/vuser.db
    ```
3.  **创建本地映射用户**：`useradd -d /var/ftproot -s /sbin/nologin vuser`
4.  **配置PAM认证**：创建`/etc/pam.d/vsftpd.vu`文件，指向生成的数据库。
5.  **修改主配置文件**：在`/etc/vsftpd/vsftpd.conf`中添加：
    ```bash
    pam_service_name=vsftpd.vu # 指定PAM文件
    guest_enable=YES # 启用虚拟用户
    guest_username=vuser # 映射到本地用户vuser
    user_config_dir=/etc/vsftpd/vusers_dir # 虚拟用户独立配置目录
    ```
6.  **为虚拟用户设置独立权限**：在`/etc/vsftpd/vusers_dir/`目录下，创建以用户名命名的文件，可单独配置权限参数。

### 补充：简单文件传输协议（TFTP）

如果只需要极简的、无需验证的文件传输，可以使用TFTP（Trivial File Transfer Protocol）。

![](img/15de64435522c49018a342e00938485b_146.png)

![](img/15de64435522c49018a342e00938485b_147.png)

![](img/15de64435522c49018a342e00938485b_148.png)

![](img/15de64435522c49018a342e00938485b_150.png)

![](img/15de64435522c49018a342e00938485b_151.png)

![](img/15de64435522c49018a342e00938485b_153.png)

**特点**：UDP 69端口，无需验证，功能单一（仅上传/下载）。
**配置简述**：
1.  安装：`dnf install tftp-server tftp`
2.  主服务由`xinetd`管理，启用`/etc/xinetd.d/tftp`中的服务。
3.  默认根目录为`/var/lib/tftpboot/`。
4.  客户端使用`tftp`命令连接：`tftp 192.168.10.10`，然后使用`get`/`put`命令传输文件。

### 基础共享配置

![](img/15de64435522c49018a342e00938485b_187.png)

![](img/15de64435522c49018a342e00938485b_189.png)

![](img/15de64435522c49018a342e00938485b_190.png)

1.  **备份并编辑主配置文件**：`/etc/samba/smb.conf`。可以删除大量注释和打印机示例，保留核心。
2.  **配置全局参数**：
    ```ini
    [global]
        workgroup = SAMBA # 工作组名称
        security = user # 安全模式为用户密码验证
    ```
3.  **定义一个共享**：
    ```ini
    [linux] # 共享名，客户端通过此名访问
        comment = Linux Share # 描述信息
        path = /home/share # 服务器上实际共享的目录路径
        browseable = yes
        writable = yes
        public = no # 需要用户验证
    ```
4.  **创建Samba用户**：必须将系统用户添加到Samba数据库。
    ```bash
    smbpasswd -a linux # ‘linux’是已存在的系统用户
    ```
5.  **设置目录权限与SELinux**：
    ```bash
    chmod -R 777 /home/share
    setsebool -P samba_export_all_rw on
    ```
6.  **启动服务**：`systemctl enable --now smb`

### 客户端访问

![](img/15de64435522c49018a342e00938485b_197.png)

![](img/15de64435522c49018a342e00938485b_198.png)

![](img/15de64435522c49018a342e00938485b_200.png)

![](img/15de64435522c49018a342e00938485b_202.png)

*   **Windows**：文件资源管理器地址栏输入 `\\服务器IP\共享名`（如`\\192.168.10.10\linux`），输入Samba用户名密码。
*   **Linux**：
    *   查看共享：`smbclient -L 192.168.10.10 -U linux`
    *   连接共享：`smbclient //192.168.10.10/linux -U linux`
    *   挂载共享（更常用）：
        ```bash
        mount -t cifs -o username=linux,password=redhat //192.168.10.10/linux /mnt
        ```
        若要永久挂载，可将配置写入`/etc/fstab`。

## 课程总结

本节课中我们一起学习了三个重要的网络服务：
1.  **Apache访问控制**：学会了使用`SetEnvIf`和`Order`指令基于条件限制网站访问。
2.  **VSFTPD文件传输服务**：深入理解了匿名、本地用户、虚拟用户三种模式的原理、配置方法与区别，并了解了简单的TFTP服务。
3.  **Samba文件共享服务**：掌握了配置Samba实现跨平台文件共享的完整流程，包括服务端配置、用户添加和客户端访问/挂载。
