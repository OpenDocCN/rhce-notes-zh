## 理论讲解：硬件、内核与Shell 🐚

电脑的硬件由各种元素组成，例如铁、硅等。硬件本身无法直接处理用户的请求，例如直接向硬盘写入信息。我们需要一个操作系统来为我们进行数据传输和控制硬件吞吐。

硬件无法直接被人控制，但它的价值很高，具有实用性。为了控制硬件，我们需要一个中间层。

首先，电脑硬件是最底层，它不能被直接控制。例如，一个单独的硬盘，即使通电，你也无法直接向其写入信息。硬件之所以能工作，是因为其外部还有一层叫做**内核**的程序。

内核包裹在硬件外部，可以完成一些非常基本的工作。例如，我们可以调用内核中的功能来调试硬件。在Linux中，如果错误地调整内核参数，可能导致硬件损坏。内核非常复杂且危险。

为了更安全、高效地使用内核，就有了**API**。如果你学习过C语言，会知道通过`#include`加载函数库来调用接口，这些就是API。通过调用程序接口，可以完成基本功能，例如让主机蜂鸣器响一声。API由内核提供，我们通过调用API来让内核控制硬件，这样更安全。

然而，如果任务复杂，例如通过网络发送信息，可能需要调用几十个API，这对用户来说不现实。因此，在最外层还有**应用程序**，例如微信、QQ、抖音。这些应用程序通过调用大量API，结合自身功能，形成一套完整的程序。用户只需通过应用程序就能实现对硬件的控制和完成复杂任务。

总结来说，硬件是最底层，人不能直接与硬件交互。我们需要软件，软件分为操作系统（如Windows、Linux）和应用程序（如微信）。在Linux中，这个与用户交互的应用程序名词叫做**Shell**。

Shell这个词来自英文单词“壳”。它的功能是让用户通过Shell终端与操作系统交互。我们执行命令就是在Shell里输入。

为什么叫Shell呢？这个过程就像一层层的壳。例如，一只蜗牛，我们对硬件的控制过程就像穿过蜗牛的一层层壳。在Linux中，Shell就是用来控制系统硬件的应用程序。

我们可以将Shell理解为人与硬件之间的**翻译官**。它将人的需求翻译给硬件，硬件处理后再将结果翻译成人能理解的语言返回。

既然Shell是一个“职业”的统称，那么在红帽RHEL 9中，我们使用的是哪个“翻译官”呢？这就像看病要找最好的医生一样，我们会选择一个最好的Shell终端来工作。在红帽RHEL 9中，默认使用的是**Bash解释器**。

![](img/bcf3b8b0b44060f33f80e3308268a791_1.png)

Bash解释器有四个主要优势：
1.  **方向键调取历史命令**：可以通过上下方向键快速找到并重复执行之前输入过的命令，节省输入时间。
2.  **Tab键自动补齐**：输入命令、参数或文件名的前几个字母后，按`Tab`键可以自动补全。这不同于网络设备中的命令缩写，Linux中不允许缩写，必须补全。
3.  **强大的批处理脚本支持**：Bash脚本功能强大，可以实现自动化任务，无需Python或C语言。我们会在第四章详细学习循环、判断等语句。
4.  **强大的环境变量支持**：支持为每个用户配置独立的工作环境，这是多用户系统的重要特性。

![](img/bcf3b8b0b44060f33f80e3308268a791_2.png)

在红帽RHEL 9中，我们默认使用Bash解释器。了解它的优势有助于我们更好地工作。

## 命令、参数与对象的关系 🔧

上一节我们介绍了Shell作为用户与系统交互的桥梁。本节中，我们来看看在这个桥梁上具体如何下达指令，即理解命令、参数和对象。

命令、参数和对象三者之间通过**空格**进行间隔。格式通常为：`命令 [参数] [对象]`。参数与参数之间也需要空格。

*   **命令**：相当于英语中的动词，表示要执行的动作。它位于最前面。例如，“创建用户”中的“创建”就是命令。
*   **对象**：是命令的承受方，相当于宾语。例如，“创建用户”中的“用户”就是对象；“打开文件”中的“文件”也是对象。
*   **参数**：用于修饰命令，使其更贴合具体的工作需求。它相当于副词，对动作进行补充说明。例如，“狠狠地打”中的“狠狠地”就是参数。在Linux中，参数可以指定命令执行的具体细节，例如创建用户时指定其UID。

为了更好地理解，我们举一个例子：制作寿司时切饭团。
*   **命令**：是“切”这个动作，或者代表这个动作的“刀”。
*   **对象**：是被切的“饭团”。
*   **参数**：为了让切得更完美（米饭不粘刀），我们可能会在刀上打孔或洒水。这个“打孔”和“洒水”就是参数，它们让“切”这个动作更好地完成了工作。

## 参数的格式：长格式与短格式 ⚙️

![](img/bcf3b8b0b44060f33f80e3308268a791_6.png)

理解了基本概念后，我们来看看参数的具体形式。参数分为两种格式：**长格式**和**短格式**。

![](img/bcf3b8b0b44060f33f80e3308268a791_8.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_9.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_10.png)

1.  **长格式**：通常由两个减号`--`后跟一个完整的单词组成。例如：`--all`， `--list`。
2.  **短格式**：通常由一个减号`-`后跟一个字母组成。例如：`-a`， `-l`。

![](img/bcf3b8b0b44060f33f80e3308268a791_11.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_12.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_13.png)

关于参数格式，有以下几个要点：
*   **功能相同**：长格式和短格式通常功能相同，可以二选一使用。
*   **非绝对**：并非所有命令都同时支持长格式和短格式，具体以命令为准。
*   **推荐短格式**：短格式效率更高，因为多个短格式参数可以合并。
*   **短格式合并**：多个短格式参数可以合并，只保留第一个减号，后面连续跟上所有字母。例如，`-a -l -p` 可以合并为 `-alp`。字母顺序一般无关。
*   **禁止合并**：短格式与长格式之间不能合并；长格式与长格式之间也不能合并。
*   **空格间隔**：无论是否合并，命令、参数、对象之间都必须用至少一个空格间隔。

![](img/bcf3b8b0b44060f33f80e3308268a791_15.png)

让我们通过`ls`命令的例子来巩固一下。`ls`命令用于列出目录内容。
*   `-a` 参数：显示所有文件，包括隐藏文件。
*   `-l` 参数：以长格式显示文件详细信息。

看看以下几种写法是否正确：
1.  `ls -a -l`：**正确**。两个短格式参数用空格分开。
2.  `ls -al`：**正确**。两个短格式参数合并。
3.  `ls -a --list`：**正确**。短格式和长格式混合使用（`--list` 是 `-l` 的长格式）。
4.  `ls -al`：**错误**。短格式合并时，字母应直接跟在减号后，应为 `-al`。
5.  `ls file.txt -l`：**通常可以**。命令、参数、对象的顺序有时可以调整，命令必须在最前。但有些命令（如`tar`）对参数顺序有严格要求。

### 常用快捷键
在开始输入命令前，先学习几个常用快捷键，让操作更高效：
*   `Ctrl + Shift + +`：放大终端字体。
*   `Ctrl + -`：缩小终端字体。
*   `Ctrl + C`：强制终止当前正在运行的命令。
*   `Ctrl + Z`：将当前命令暂停并放入后台。
*   `Ctrl + L`：清屏（实际上是翻页，历史输出仍可滚动查看）。
*   `方向键上/下`：调取历史命令。

![](img/bcf3b8b0b44060f33f80e3308268a791_26.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_28.png)

### 1. `man` 命令：查看帮助手册
当我们不知道一个命令怎么用时，可以求助`man`命令（manual的缩写）。

**命令格式**：`man [命令名]`
例如，查看`ls`命令的帮助：`man ls`
查看`man`命令本身的帮助：`man man`

帮助信息是英文的，但最为准确和全面。按`空格键`翻页，按`q`键退出。

Linux系统有数千个命令，每个命令又有数十个参数，组合方式极其繁多。我们不需要全部记忆，只需掌握最常用的即可。

### 2. `echo` 命令：输出信息
`echo`命令的作用是“回声”，即将指定的字符串或变量值输出到屏幕。

**命令格式**：`echo [字符串]`
例如：`echo LinuxProbe.Com`
屏幕上就会显示：`LinuxProbe.Com`

![](img/bcf3b8b0b44060f33f80e3308268a791_30.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_31.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_33.png)

它还可以输出变量的值。变量就像一个容器，里面存储着数据。使用`$`符号可以提取变量的值。
例如，有一个变量叫`hostname`，存储着主机名。
*   `echo hostname`：输出字符串“hostname”。
*   `echo $hostname`：提取变量`hostname`的值并输出。

![](img/bcf3b8b0b44060f33f80e3308268a791_35.png)

`echo`命令在后面学习脚本和管道时会非常有用。

![](img/bcf3b8b0b44060f33f80e3308268a791_37.png)

### 3. `date` 命令：查看与设置时间
`date`命令用于查看或设置系统日期和时间。

![](img/bcf3b8b0b44060f33f80e3308268a791_39.png)

**查看时间**：直接输入`date`。
**按格式输出时间**：`date “+%Y-%m-%d %H:%M:%S”`
*   `%Y`：四位年份
*   `%m`：月份
*   `%d`：日期
*   `%H`：小时（24小时制）
*   `%M`：分钟
*   `%S`：秒

![](img/bcf3b8b0b44060f33f80e3308268a791_40.png)

例如，生成一个带时间戳的文件名：`date “+backup-%Y%m%d.tar.gz”`

**设置时间**：需要使用`timedatectl`命令。
1.  设置时区为上海：`timedatectl set-timezone Asia/Shanghai`
2.  关闭网络时间同步：`timedatectl set-ntp false`
3.  设置日期：`timedatectl set-time “2025-05-18”`
4.  设置时间：`timedatectl set-time “08:30:00”`

### 4. `reboot` 与 `poweroff` 命令：重启与关机
*   **重启系统**：`reboot`
*   **关闭系统**：`poweroff`

![](img/bcf3b8b0b44060f33f80e3308268a791_42.png)

这两个命令需要管理员权限。

### 5. `wget` 命令：下载网络文件
`wget`命令用于从网络上下载文件。

**命令格式**：`wget [文件URL]`
例如：`wget https://www.linuxprobe.com/docs/LinuxProbe.pdf`
（此示例需要系统联网，实验环境中若无网络可暂不操作）

### 6. `ps` 命令：查看进程状态
`ps`命令用于查看系统当前进程的信息。Linux进程主要有5种状态：
1.  **R (Running)**：正在运行或准备运行，正在消耗CPU资源。
2.  **S (Sleeping)**：休眠中，等待被唤醒，不消耗CPU资源。
3.  **D (Uninterruptible Sleep)**：不可中断的休眠，通常在进行I/O操作，会消耗资源。
4.  **Z (Zombie)**：僵尸进程。进程已终止，但其父进程未正确回收资源，会占用系统资源。
5.  **T (Stopped)**：进程已被停止（如通过`Ctrl+Z`）。

![](img/bcf3b8b0b44060f33f80e3308268a791_44.png)

![](img/bcf3b8b0b44060f33f80e3308268a791_45.png)

**常用参数组合**：`ps aux`
*   `a`：显示所有用户的进程。
*   `u`：显示进程的详细信息（如用户、CPU、内存占用）。
*   `x`：显示没有控制终端的进程。

### 7. `pstree` 命令：以树状图显示进程
`ps aux`的输出比较扁平。`pstree`命令可以以树状结构显示进程，清晰地展示进程间的父子关系。

![](img/bcf3b8b0b44060f33f80e3308268a791_47.png)

### 8. `top` 命令：动态实时查看进程
`top`命令类似于Windows的任务管理器，可以动态、实时地显示系统状态和进程信息，并自动刷新。

`top`界面顶部几行显示了关键的系统状态信息：
*   **系统负载**：`load average: 0.01, 0.05, 0.18`。这3个值分别代表过去1分钟、5分钟、15分钟的系统平均负载。对于单核CPU，负载持续高于1可能表示繁忙，高于5则需要关注。数值从右向左看，趋势由旧到新。
*   **进程统计**：总进程数、运行中、休眠、停止、僵尸进程的数量。
*   **CPU使用率**：`%Cpu(s)`行，关注`us`（用户进程占用）和`id`（空闲率）。
*   **内存使用**：`MiB Mem`行，关注`total`（总量）、`free`（空闲）、`used`（已用）。
*   **交换分区**：`MiB Swap`行，如果`used`值持续很高，说明物理内存可能不足。

界面下部是进程列表，包含PID、用户、优先级、状态、CPU和内存占用率等信息。

在`top`界面中，按`q`键退出。

### 9. `nice` 命令：调整进程优先级
`nice`命令可以调整进程运行的优先级。优先级数值范围是**-20到19**，数值越小，优先级越高，获得的CPU资源越多。

![](img/bcf3b8b0b44060f33f80e3308268a791_49.png)

**命令格式**：`nice -n [优先级数字] [命令]`
例如，以最高优先级运行`top`命令：`nice -n -20 top`

### 10. `pidof` 命令：查询进程PID
`pidof`命令用于查询某个指定服务进程的PID号码。

![](img/bcf3b8b0b44060f33f80e3308268a791_51.png)

**命令格式**：`pidof [服务名]`
例如：`pidof sshd`

![](img/bcf3b8b0b44060f33f80e3308268a791_53.png)

### 11. `kill` 与 `killall` 命令：终止进程
用于终止指定的进程。
*   `kill [PID]`：通过进程PID来终止单个进程。例如：`kill 1234`
*   `killall [进程名]`：通过进程名称来终止所有同名进程。例如：`killall sshd`

更规范的服务管理方式是使用`systemctl`命令（上节课已学）：
*   `systemctl stop sshd`：停止sshd服务。
*   `systemctl start sshd`：启动sshd服务。
