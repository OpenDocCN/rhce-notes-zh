# RHCE 8.2 认证课程：P53：LVM 逻辑卷管理教程 🗂️

在本节课中，我们将要学习逻辑卷管理器（LVM）的核心概念和基本操作。LVM 是一种对磁盘存储进行灵活管理的强大技术，允许我们动态调整存储空间，是系统运维和认证考试的重点内容。

## 逻辑卷管理器（LVM）概述

逻辑卷管理器（Logical Volume Manager，简称 LVM）是对磁盘分区进行管理操作的一个抽象层。它允许我们弹性地更改文件系统的大小，并能在多个物理设备间重新组织文件系统。

## LVM 的组成与工作原理

上一节我们介绍了 LVM 的基本概念，本节中我们来看看它的核心组成和工作原理。LVM 通过分层结构实现灵活的存储管理。

![](img/12d11924ab150c7aec4d288cc140981b_1.png)

![](img/12d11924ab150c7aec4d288cc140981b_3.png)

LVM 的实现过程主要分为三个层次：
1.  **物理卷（Physical Volume， PV）**：这是 LVM 的基础，可以是整个硬盘或硬盘分区。
2.  **卷组（Volume Group， VG）**：由一个或多个物理卷组成，形成一个大的存储池。
3.  **逻辑卷（Logical Volume， LV）**：从卷组中划分出来的逻辑存储单元，我们可以在其上创建文件系统并挂载使用。

![](img/12d11924ab150c7aec4d288cc140981b_5.png)

卷组（VG）将物理卷的空间汇集起来，并以称为 **PE（Physical Extent）** 的最小单位进行管理。创建逻辑卷（LV）时，实际上是从卷组中分配一定数量的 PE。例如，如果设定每个 PE 为 4MB，那么分配 100 个 PE 就会创建一个 400MB 的逻辑卷。

**核心关系公式**：
`LV 空间大小 = PE 大小 × 分配的 PE 数量`

## 管理工具安装与 PV 管理

![](img/12d11924ab150c7aec4d288cc140981b_7.png)

```bash
rpm -qa | grep lvm2
```

如果未安装，请使用 `yum install lvm2` 进行安装。

![](img/12d11924ab150c7aec4d288cc140981b_13.png)

### 物理卷（PV）管理

![](img/12d11924ab150c7aec4d288cc140981b_16.png)

以下是 PV 管理的三个核心命令：
*   **`pvcreate`**：将块设备（如分区）初始化为物理卷。
*   **`pvremove`**：移除物理卷属性。
*   **`pvdisplay` / `pvs` / `pvscan`**：查看物理卷的详细信息。

![](img/12d11924ab150c7aec4d288cc140981b_18.png)

**创建 PV 示例**：
假设我们有两个空闲分区 `/dev/sdb1` 和 `/dev/sdb2`，可以将它们初始化为物理卷。
```bash
pvcreate /dev/sdb1 /dev/sdb2
```
创建后，可以使用 `pvs` 或 `pvdisplay` 命令查看结果。

## 卷组（VG）管理

![](img/12d11924ab150c7aec4d288cc140981b_20.png)

成功创建物理卷后，下一步是将它们组合成卷组。

![](img/12d11924ab150c7aec4d288cc140981b_22.png)

![](img/12d11924ab150c7aec4d288cc140981b_24.png)

### 创建与查看卷组

创建卷组的命令是 `vgcreate`。可以使用 `-s` 选项指定 PE 的大小（默认为 4MB）。
```bash
vgcreate -s 8M myvg /dev/sdb1 /dev/sdb2
```
以上命令创建了一个名为 `myvg` 的卷组，PE 大小为 8MB，并包含了 `/dev/sdb1` 和 `/dev/sdb2` 两个物理卷。

![](img/12d11924ab150c7aec4d288cc140981b_26.png)

查看卷组信息的命令同样有三个：
*   **`vgs`**：简要信息。
*   **`vgdisplay`**：详细信息。
*   **`vgscan`**：扫描系统中的卷组。

使用 `vgdisplay myvg` 可以查看该卷组的总大小、PE 数量等信息。

## 逻辑卷（LV）管理

卷组创建好后，我们就可以像从池子里取水一样，从中划分逻辑卷了。

![](img/12d11924ab150c7aec4d288cc140981b_28.png)

### 创建逻辑卷

![](img/12d11924ab150c7aec4d288cc140981b_30.png)

创建逻辑卷使用 `lvcreate` 命令。有两个常用选项来指定大小：
*   **`-L`**：直接指定容量大小（如 `-L 2G`）。
*   **`-l`**：指定分配的 PE 数量（如 `-l 100`）或百分比（如 `-l 50%VG` 表示卷组总空间的50%，`-l 100%FREE` 表示所有剩余空间）。

![](img/12d11924ab150c7aec4d288cc140981b_32.png)

![](img/12d11924ab150c7aec4d288cc140981b_34.png)

**创建 LV 示例**：
```bash
# 创建一个大小为 2G，名为 mydata 的逻辑卷
lvcreate -L 2G -n mydata myvg

# 使用剩余空间的 30% 创建一个名为 mydb 的逻辑卷
lvcreate -l 30%FREE -n mydb myvg
```
创建后，使用 `lvs` 或 `lvdisplay` 命令查看逻辑卷。

![](img/12d11924ab150c7aec4d288cc140981b_36.png)

![](img/12d11924ab150c7aec4d288cc140981b_37.png)

![](img/12d11924ab150c7aec4d288cc140981b_39.png)

![](img/12d11924ab150c7aec4d288cc140981b_41.png)

### 逻辑卷的设备文件

逻辑卷创建后，系统会在 `/dev` 目录下生成对应的设备文件，主要有两种路径：
1.  `/dev/mapper/` 目录下，如 `/dev/mapper/myvg-mydata`。
2.  `/dev/卷组名/` 目录下，如 `/dev/myvg/mydata`。

![](img/12d11924ab150c7aec4d288cc140981b_43.png)

![](img/12d11924ab150c7aec4d288cc140981b_44.png)

这两个路径都是指向实际设备（如 `/dev/dm-0`）的符号链接，建议使用前者进行挂载等操作，因为其名称更具可读性。

![](img/12d11924ab150c7aec4d288cc140981b_46.png)

## 使用逻辑卷：格式化与挂载

![](img/12d11924ab150c7aec4d288cc140981b_48.png)

![](img/12d11924ab150c7aec4d288cc140981b_50.png)

![](img/12d11924ab150c7aec4d288cc140981b_52.png)

创建逻辑卷后，其使用方式与普通分区完全一致。

![](img/12d11924ab150c7aec4d288cc140981b_54.png)

![](img/12d11924ab150c7aec4d288cc140981b_56.png)

**格式化与挂载示例**：
```bash
# 格式化逻辑卷为 ext4 文件系统
mkfs.ext4 /dev/myvg/mydata

![](img/12d11924ab150c7aec4d288cc140981b_58.png)

![](img/12d11924ab150c7aec4d288cc140981b_60.png)

# 格式化逻辑卷为 xfs 文件系统
mkfs.xfs /dev/myvg/mydb

![](img/12d11924ab150c7aec4d288cc140981b_62.png)

# 创建挂载点并挂载
mkdir /data /database
mount /dev/myvg/mydata /data
mount /dev/myvg/mydb /database
```
为了让挂载永久生效，需要将配置写入 `/etc/fstab` 文件。

![](img/12d11924ab150c7aec4d288cc140981b_64.png)

![](img/12d11924ab150c7aec4d288cc140981b_66.png)

## LVM 的核心优势：在线扩容

![](img/12d11924ab150c7aec4d288cc140981b_70.png)

![](img/12d11924ab150c7aec4d288cc140981b_72.png)

LVM 最强大的功能之一是支持在线扩容，即无需卸载文件系统即可扩展空间。

### 扩展卷组（VG）

当卷组空间不足时，可以添加新的物理卷来扩展。
```bash
# 将新的物理卷 /dev/sdc1 添加到现有卷组 myvg
vgextend myvg /dev/sdc1
```

### 扩展逻辑卷（LV）及文件系统

![](img/12d11924ab150c7aec4d288cc140981b_74.png)

![](img/12d11924ab150c7aec4d288cc140981b_76.png)

扩展逻辑卷分为两步：先扩展 LV 本身，再扩展其上的文件系统。

**分步操作示例（针对 ext4）**：
```bash
# 1. 将逻辑卷 mydata 扩展 3G
lvextend -L +3G /dev/myvg/mydata

# 2. 调整 ext4 文件系统大小，使其充满整个逻辑卷
resize2fs /dev/myvg/mydata
```

![](img/12d11924ab150c7aec4d288cc140981b_78.png)

**分步操作示例（针对 xfs）**：
```bash
# 1. 将逻辑卷 mydb 扩展 3G
lvextend -L +3G /dev/myvg/mydb

![](img/12d11924ab150c7aec4d288cc140981b_80.png)

# 2. 调整 xfs 文件系统大小
xfs_growfs /database
```

![](img/12d11924ab150c7aec4d288cc140981b_82.png)

**一步到位操作**：
使用 `lvresize` 命令的 `-r` 选项，可以同时扩展逻辑卷和文件系统，适用于 ext4 和 xfs。
```bash
lvresize -r -L +3G /dev/myvg/mydata
```

![](img/12d11924ab150c7aec4d288cc140981b_84.png)

## LVM 的缩容操作 ⚠️

![](img/12d11924ab150c7aec4d288cc140981b_86.png)

![](img/12d11924ab150c7aec4d288cc140981b_88.png)

与扩容相比，缩容风险较高，且 **xfs 文件系统不支持缩容**。缩容必须离线进行（即先卸载文件系统）。

**缩容 ext4 逻辑卷步骤**：
1.  **卸载文件系统**：`umount /data`
2.  **强制检查文件系统**：`e2fsck -f /dev/myvg/mydata`
3.  **缩小文件系统**：`resize2fs /dev/myvg/mydata 10G` （缩小至 10G）
4.  **缩小逻辑卷**：`lvreduce -L 10G /dev/myvg/mydata`
5.  **重新挂载**：`mount /dev/myvg/mydata /data`

![](img/12d11924ab150c7aec4d288cc140981b_90.png)

> **注意**：缩容前务必做好数据备份，并确保缩容后的尺寸大于已存数据总量。

![](img/12d11924ab150c7aec4d288cc140981b_92.png)

![](img/12d11924ab150c7aec4d288cc140981b_94.png)

## 删除 LVM 组件

![](img/12d11924ab150c7aec4d288cc140981b_96.png)

![](img/12d11924ab150c7aec4d288cc140981b_98.png)

![](img/12d11924ab150c7aec4d288cc140981b_100.png)

删除顺序与创建顺序相反：先删 LV，再删 VG，最后删 PV。
```bash
# 1. 删除逻辑卷
lvremove /dev/myvg/mydata

![](img/12d11924ab150c7aec4d288cc140981b_102.png)

![](img/12d11924ab150c7aec4d288cc140981b_104.png)

# 2. 删除卷组（确保其中已无逻辑卷）
vgremove myvg

![](img/12d11924ab150c7aec4d288cc140981b_106.png)

![](img/12d11924ab150c7aec4d288cc140981b_108.png)

![](img/12d11924ab150c7aec4d288cc140981b_110.png)

# 3. 移除物理卷的 LVM 属性
pvremove /dev/sdb1 /dev/sdb2
```
