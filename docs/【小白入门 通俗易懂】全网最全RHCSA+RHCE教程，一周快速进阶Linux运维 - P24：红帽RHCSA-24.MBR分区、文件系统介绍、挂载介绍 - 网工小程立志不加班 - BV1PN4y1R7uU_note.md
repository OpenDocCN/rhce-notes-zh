## 认识设备文件路径 🔍

![](img/289932d662a4014bfe35189a5f5e2c27_11.png)

在开始分区之前，我们需要了解Linux中硬件设备的表示方式。所有硬件设备都以文件的形式存在于 `/dev` 目录下。

以下是 `/dev` 目录下一些常见设备文件的介绍：

![](img/289932d662a4014bfe35189a5f5e2c27_19.png)

![](img/289932d662a4014bfe35189a5f5e2c27_21.png)

![](img/289932d662a4014bfe35189a5f5e2c27_25.png)

*   **硬盘设备**：通常以 `sd` 开头，例如 `sda`、`sdb`、`sdc`。
*   **旧式IDE硬盘**：以 `hd` 开头（现在较少见）。
*   **软盘驱动器**：以 `fd` 开头。
*   **虚拟终端**：以 `tty` 开头，例如 `tty1`、`tty2`，代表可以登录的终端界面。
*   **光驱设备**：通常表示为 `sr0`，其快捷方式为 `cdrom`。
*   **内存设备**：以 `mem` 开头，代表物理内存。
*   **空设备（黑洞设备）**：`/dev/null`。任何写入此设备的数据都会被丢弃。常用于清空文件内容，例如：`cat /dev/null > filename`。
*   **满设备**：`/dev/full`。任何写入此设备的数据操作都会失败。
*   **随机数设备**：`/dev/random` 和 `/dev/urandom`，用于生成随机数。

## 使用fdisk进行MBR分区 ✂️

![](img/289932d662a4014bfe35189a5f5e2c27_39.png)

![](img/289932d662a4014bfe35189a5f5e2c27_41.png)

上一节我们认识了设备文件，本节中我们来看看如何使用 `fdisk` 命令对磁盘进行MBR格式的分区。`fdisk` 是一个交互式的磁盘分区工具。

**命令基本格式**：
```bash
fdisk /dev/设备名
```
例如，对第二块硬盘操作：`fdisk /dev/sdb`

进入 `fdisk` 交互界面后，所有操作都暂存于内存，只有在执行保存命令后才会写入磁盘。输入 `m` 可以获取帮助菜单。

以下是 `fdisk` 交互模式下的常用命令列表：

*   **`p`**：打印当前磁盘的分区表。可以查看分区信息、磁盘总大小以及 **磁盘标签类型**（如 `DOS` 代表MBR格式，`GPT` 代表GPT格式）。**一块磁盘只能使用一种分区格式（MBR或GPT），混用会导致数据丢失。**
*   **`n`**：创建新分区。
*   **`d`**：删除分区。
*   **`w`**：保存所有更改并退出。
*   **`q`**：不保存更改并退出。

**MBR分区实战步骤**：
1.  输入 `n` 创建新分区。
2.  选择分区类型：`p` 为主分区，`e` 为扩展分区。MBR最多允许 **4个主分区**，或 **3个主分区 + 1个扩展分区**。扩展分区是一个容器，可以在其中创建多个**逻辑分区**。
3.  设置分区编号（1-4）。
4.  设置起始扇区（通常直接回车使用默认值）。
5.  设置分区大小：可以使用 `+size` 的格式，如 `+10G` 表示分配10GB空间。
6.  重复以上步骤创建所需分区。**当主分区数量达到上限后，若想创建更多分区，必须将其中一个主分区替换为扩展分区（先删除，再创建为扩展分区），然后在扩展分区内创建逻辑分区。逻辑分区的编号从5开始**。
7.  所有操作确认无误后，输入 `w` 保存并退出。

## 文件系统：分区的“管理员” 📂

![](img/289932d662a4014bfe35189a5f5e2c27_47.png)

![](img/289932d662a4014bfe35189a5f5e2c27_49.png)

分区创建完成后，并不能直接存储数据。我们需要为分区建立一个“管理系统”，即**文件系统**，它负责管理分区内的文件存储、属性、日志等。

目前Linux系统主流的文件系统有两种：

**1. ext4文件系统**：
*   特点：具有日志功能，可避免意外宕机时的数据丢失。
*   容量限制：单个文件系统（分区）最大支持 **1EB**；单个文件最大支持 **16TB**。
*   常见于：较旧的系统（如RHEL 6）。

![](img/289932d662a4014bfe35189a5f5e2c27_51.png)

**2. XFS文件系统**：
*   特点：具有日志功能，**数据恢复速度比ext4更快**，性能更优。
*   容量限制：单个文件系统（分区）最大支持 **8EB**；单个文件最大支持 **500TB**。
*   常见于：现代主流系统（如RHEL 7/8/9）的默认文件系统。

![](img/289932d662a4014bfe35189a5f5e2c27_53.png)

**格式化命令**：
为分区创建文件系统的过程称为“格式化”。
```bash
# 格式化为XFS文件系统
mkfs.xfs /dev/sdb1

# 格式化为ext4文件系统
mkfs.ext4 /dev/sdb1
```
格式化后，可以使用 `lsblk -f` 命令查看分区的文件系统类型。

## 挂载：使用分区的必经之路 🔗

![](img/289932d662a4014bfe35189a5f5e2c27_65.png)

![](img/289932d662a4014bfe35189a5f5e2c27_67.png)

在Linux中，用户不能直接访问硬件设备分区。必须通过**挂载（Mount）**操作，将分区关联到一个**目录**（挂载点），用户通过访问这个目录来间接使用分区的存储空间。

**挂载命令**：
```bash
mount /dev/设备名 /挂载点目录
```
例如：`mount /dev/sdb1 /mnt/data`

**挂载注意事项**：
1.  **挂载点通常是一个空目录**。如果挂载点非空，挂载后原目录内的文件会被暂时隐藏（卸载后恢复）。
2.  **一个分区不能同时挂载到多个目录**。
3.  **一个目录不能同时挂载多个分区**。
4.  使用 `df -h` 命令可以查看已挂载的文件系统及其使用情况。

![](img/289932d662a4014bfe35189a5f5e2c27_69.png)

![](img/289932d662a4014bfe35189a5f5e2c27_70.png)

![](img/289932d662a4014bfe35189a5f5e2c27_72.png)

![](img/289932d662a4014bfe35189a5f5e2c27_74.png)

**卸载命令**：
当需要停止使用某个分区，或更换挂载点时，需要先卸载。
```bash
umount /挂载点目录
# 或
umount /dev/设备名
```
例如：`umount /mnt/data`

![](img/289932d662a4014bfe35189a5f5e2c27_76.png)

![](img/289932d662a4014bfe35189a5f5e2c27_78.png)

**工作流程示例**：
1.  创建目录作为挂载点：`mkdir /backup`
2.  格式化分区：`mkfs.xfs /dev/sdb1`
3.  挂载分区：`mount /dev/sdb1 /backup`
4.  使用分区：此时所有存入 `/backup` 目录的文件，实际都存储在 `/dev/sdb1` 分区中。
5.  卸载分区：`umount /backup`。卸载后，`/backup` 目录恢复为空，但分区中的数据完好无损。
