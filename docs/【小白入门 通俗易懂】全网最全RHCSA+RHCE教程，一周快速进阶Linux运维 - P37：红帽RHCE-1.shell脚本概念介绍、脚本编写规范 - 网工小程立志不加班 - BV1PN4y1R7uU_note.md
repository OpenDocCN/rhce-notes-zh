## 什么是Shell？它能做什么？

上一节我们介绍了课程的整体安排，本节中我们来看看Shell脚本的基础概念。

![](img/7e55455409b7ad398e6524c66e4e41d2_9.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_11.png)

Shell本身是一门编程语言。编程语言是人与计算机交流的工具。计算机只能理解二进制语言（如0101），而我们使用的英文字母命令（如`ls`, `cat`）之所以能被计算机识别，是因为有“翻译官”的存在，这个“翻译官”就是Shell解释器。

![](img/7e55455409b7ad398e6524c66e4e41d2_13.png)

编程语言主要分为两类：
*   **编译型语言**：如C、C++、Go。程序运行前需要专门的编译器将其整体转换为计算机能直接执行的二进制文件。
    *   **公式/代码示例**：`gcc hello.c -o hello` (编译C程序)
*   **解释型语言**：如Shell、Python、PHP。程序运行时，由解释器一边读取代码一边逐行翻译并执行。
    *   **公式/代码示例**：`python hello.py` (由Python解释器执行)

我们日常在终端输入的所有命令（`ls`, `pwd`等）都属于Shell语言。因此，你已经在不知不觉中学习并使用一门编程语言了。

![](img/7e55455409b7ad398e6524c66e4e41d2_15.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_17.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_19.png)

Shell语言功能强大，支持变量、数组、判断（if）、循环（for）、函数等大多数编程语言都具备的功能。我们可以用Shell脚本实现：
*   服务器的批量维护与初始化
*   软件的自动化安装与部署
*   系统监控与故障自动报警
*   定时任务（如凌晨自动备份）

Shell属于**高级语言**，离用户更近，使用起来相对方便简单。很多常用功能（如`cat`, `ls`）已经由别人实现好了，我们直接组合使用即可，这极大地提高了工作效率。

![](img/7e55455409b7ad398e6524c66e4e41d2_21.png)

## 为什么需要Shell脚本？

![](img/7e55455409b7ad398e6524c66e4e41d2_25.png)

脚本的本质是**命令的堆积**。

想象一个场景：你需要为50台新服务器部署一套完整的LNMP（Linux, Nginx, MySQL, PHP）环境。手动在一台台服务器上重复输入几十条命令，不仅效率低下，而且容易出错。

以下是解决此问题的步骤：
1.  将所有部署所需的命令，按照正确顺序写入一个文件（例如 `deploy_lnmp.sh`）。
2.  为这个文件添加执行权限。
3.  在任何新服务器上，只需要执行这个脚本文件，它就会自动按序运行所有命令。
4.  在脚本运行期间，你可以去处理其他事务。

另一个常见场景是**凌晨备份**。你可以将备份命令写成脚本，然后结合计划任务（如cron），设定在每天凌晨3点自动执行。这样，你无需熬夜，工作也能自动完成。

![](img/7e55455409b7ad398e6524c66e4e41d2_27.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_29.png)

Shell脚本能将运维人员从重复、繁琐的手工操作中解放出来，是提升幸福感和专业度的关键工具。

## 如何编写一个规范的Shell脚本？

![](img/7e55455409b7ad398e6524c66e4e41d2_31.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_33.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_35.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_36.png)

现在，我们开始学习如何动手编写一个Shell脚本。

首先，创建一个用于练习的目录，并进入该目录：
```bash
mkdir /script
cd /script
```

然后，我们创建一个脚本文件。脚本文件名通常以 `.sh` 结尾，这是一种约定，便于识别文件类型。
```bash
vim hello.sh
```

![](img/7e55455409b7ad398e6524c66e4e41d2_40.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_42.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_43.png)

一个规范的Shell脚本通常包含以下几个部分：

![](img/7e55455409b7ad398e6524c66e4e41d2_45.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_47.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_49.png)

### 1. 脚本声明（环境声明）
脚本的第一行需要指定用哪个解释器来执行脚本中的命令。这通过 `#!`（称为shebang）实现。
```bash
#!/bin/bash
```
*   `#!` 是一个特殊标记，告诉系统该文件需要被指定的解释器执行。
*   `/bin/bash` 是Bash解释器的路径，它是Linux系统最常用的Shell解释器。
*   你可以通过 `cat /etc/shells` 命令查看系统可用的所有解释器。

![](img/7e55455409b7ad398e6524c66e4e41d2_51.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_53.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_55.png)

### 2. 注释信息
注释以 `#` 开头，其后的内容不会被解释器执行。注释用于说明脚本的作者、功能、版本等信息，对于代码维护和他人阅读至关重要。
```bash
# Author: NetEngineer_XiaoCheng
# Description: This script prints “Hello World” to the screen.
# Version: 1.0
```

![](img/7e55455409b7ad398e6524c66e4e41d2_57.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_59.png)

### 3. 可执行语句（命令）
这是脚本的主体部分，包含了需要依次执行的各种命令。
```bash
echo “Hello World”
```
`echo` 命令用于向屏幕输出文本。

![](img/7e55455409b7ad398e6524c66e4e41d2_61.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_63.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_64.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_66.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_68.png)

将以上三部分组合起来，一个完整的 `hello.sh` 脚本内容如下：
```bash
#!/bin/bash
# Author: NetEngineer_XiaoCheng
# Description: This script prints “Hello World” to the screen.
# Version: 1.0

![](img/7e55455409b7ad398e6524c66e4e41d2_70.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_72.png)

echo “Hello World”
```

保存并退出编辑器后，该文件只是一个普通文本文件，无法直接运行。需要为其添加**执行权限**：
```bash
chmod +x hello.sh
```
执行脚本：
```bash
./hello.sh
```
*   `./` 表示当前目录。直接输入 `hello.sh` 系统会将其当作命令去查找，而不会执行当前目录下的文件，因此必须指明路径。

![](img/7e55455409b7ad398e6524c66e4e41d2_74.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_76.png)

![](img/7e55455409b7ad398e6524c66e4e41d2_78.png)

执行后，终端将输出：`Hello World`。

![](img/7e55455409b7ad398e6524c66e4e41d2_80.png)

本节课中我们一起学习了Shell脚本的核心概念和编写规范。我们了解到Shell是一门强大的解释型编程语言，是运维自动化的基石。一个规范的脚本应包含脚本声明、注释信息和可执行命令三部分。记住，脚本的本质是命令的堆积，它通过将手动操作流程固化到文件中，实现了工作的自动化与高效化。从下一个简单的“Hello World”脚本开始，你已经踏入了Shell编程的大门。
