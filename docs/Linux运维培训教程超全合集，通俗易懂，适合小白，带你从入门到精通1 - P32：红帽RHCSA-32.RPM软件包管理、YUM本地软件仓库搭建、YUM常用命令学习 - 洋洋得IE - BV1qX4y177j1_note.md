## RPM软件包升级与卸载

![](img/74b4cf750d48ccd4680684d14c25467b_3.png)

### 软件包升级注意事项
升级软件包时，必须做好相关配置文件的备份。在企业环境中，软件升级可能因修复漏洞或功能更新而进行。升级操作会将旧版本的所有文件（包括配置文件）替换为新版本的文件。如果你希望保留旧版本的特定配置，就需要提前备份这些文件。

**核心操作**：升级后，用备份的旧配置文件覆盖新版本的同名文件，即可在享受新版本功能的同时保留原有配置。

以下是升级命令的格式：
```bash
rpm -Uvh 软件包全名.rpm
```
其中，`-U` 代表升级，`-v` 显示详细信息，`-h` 显示安装进度条。

![](img/74b4cf750d48ccd4680684d14c25467b_5.png)

### 软件包卸载
卸载软件包使用 `-e` 选项。通常，服务器上已安装的软件不会轻易卸载，只需停止相关服务即可。

![](img/74b4cf750d48ccd4680684d14c25467b_7.png)

以下是卸载命令：
```bash
rpm -e 软件包名
```
卸载后，可以使用 `rpm -q 软件包名` 查询确认是否已卸载。

### 处理卸载时的依赖关系
在卸载软件包时，可能会遇到依赖关系问题。一个软件包（如A）可能被其他软件包（如X）所依赖。如果卸载A时连带卸载了其依赖包B，可能导致X无法运行。

可以使用 `--nodeps` 选项忽略依赖关系，强制卸载指定软件包，但需谨慎使用。
```bash
rpm -e --nodeps 软件包名
```

## RPM软件包签名验证

![](img/74b4cf750d48ccd4680684d14c25467b_9.png)

![](img/74b4cf750d48ccd4680684d14c25467b_11.png)

在安装软件包时，系统可能会提示缺少GPG密钥签名。这类似于商品上的安全认证，用于验证软件包的来源是否可信。警告不影响安装，但若需消除警告，可以导入红帽的签名密钥文件。

![](img/74b4cf750d48ccd4680684d14c25467b_13.png)

![](img/74b4cf750d48ccd4680684d14c25467b_15.png)

以下是导入签名文件的命令：
```bash
rpm --import /mnt/cdrom/RPM-GPG-KEY-redhat-release
```
导入后，系统便能识别软件包的合法签名，安装时不再出现警告。签名文件通常位于 `/etc/pki/rpm-gpg/` 目录下。

![](img/74b4cf750d48ccd4680684d14c25467b_17.png)

![](img/74b4cf750d48ccd4680684d14c25467b_19.png)

## YUM软件包管理器简介

![](img/74b4cf750d48ccd4680684d14c25467b_21.png)

![](img/74b4cf750d48ccd4680684d14c25467b_23.png)

### 为何需要YUM
使用RPM手动安装软件时，经常需要处理复杂的依赖关系，效率低下。YUM（Yellowdog Updater Modified）是一个基于RPM的软件包管理器，能够自动从软件仓库下载软件包并处理依赖关系。

![](img/74b4cf750d48ccd4680684d14c25467b_25.png)

**核心概念**：软件仓库是存放大量RPM软件包及其元数据的服务器或本地目录。YUM从中获取并安装软件。

![](img/74b4cf750d48ccd4680684d14c25467b_27.png)

![](img/74b4cf750d48ccd4680684d14c25467b_29.png)

### 软件仓库类型
软件仓库主要分为两类：
1.  **本地仓库**：软件包存放在本地文件系统（如系统安装光盘镜像）。无需网络即可使用，但需要手动配置。
2.  **网络仓库**：由互联网服务商（如阿里云、清华大学）维护的仓库。需要网络连接，但通常无需复杂配置即可直接使用。

![](img/74b4cf750d48ccd4680684d14c25467b_31.png)

![](img/74b4cf750d48ccd4680684d14c25467b_33.png)

对于内网或无网络环境，搭建本地YUM仓库是必备技能。

![](img/74b4cf750d48ccd4680684d14c25467b_35.png)

## 搭建本地YUM仓库 🛠️

![](img/74b4cf750d48ccd4680684d14c25467b_37.png)

![](img/74b4cf750d48ccd4680684d14c25467b_39.png)

![](img/74b4cf750d48ccd4680684d14c25467b_41.png)

![](img/74b4cf750d48ccd4680684d14c25467b_43.png)

上一节我们了解了YUM的优势，本节中我们来看看如何手动搭建一个本地YUM仓库。我们将使用系统安装光盘作为软件包来源。

![](img/74b4cf750d48ccd4680684d14c25467b_45.png)

以下是搭建本地YUM仓库的步骤：

1.  **挂载系统镜像**：首先，将系统安装光盘镜像挂载到指定目录。我们将其永久挂载到 `/mnt/cdrom`。
    ```bash
    # 编辑 /etc/fstab 文件，添加以下行
    /dev/cdrom /mnt/cdrom iso9660 defaults 0 0
    # 执行挂载
    mount -a
    ```

2.  **配置YUM仓库文件**：在 `/etc/yum.repos.d/` 目录下创建新的仓库配置文件，例如 `local.repo`。
    ```bash
    vim /etc/yum.repos.d/local.repo
    ```

![](img/74b4cf750d48ccd4680684d14c25467b_47.png)

![](img/74b4cf750d48ccd4680684d14c25467b_49.png)

3.  **编写仓库配置**：在配置文件中输入以下内容。`baseurl` 指定软件包所在路径，`enabled=1` 启用此仓库，`gpgcheck=0` 禁用GPG检查（本地仓库可关闭）。
    ```bash
    [LocalRepo]
    name=Local Repository
    baseurl=file:///mnt/cdrom
    enabled=1
    gpgcheck=0
    ```

![](img/74b4cf750d48ccd4680684d14c25467b_51.png)

![](img/74b4cf750d48ccd4680684d14c25467b_53.png)

![](img/74b4cf750d48ccd4680684d14c25467b_55.png)

4.  **清理并测试**：清除YUM缓存，然后测试新仓库是否可用。
    ```bash
    yum clean all
    yum makecache
    yum list
    ```
    如果能看到软件包列表，说明本地仓库搭建成功。

## YUM常用命令 📝

![](img/74b4cf750d48ccd4680684d14c25467b_57.png)

![](img/74b4cf750d48ccd4680684d14c25467b_59.png)

本地仓库搭建完成后，我们就可以使用YUM命令来方便地管理软件了。以下是YUM最常用的命令列表：

![](img/74b4cf750d48ccd4680684d14c25467b_61.png)

*   **安装软件包**：`yum install 软件包名`
*   **重新安装软件包**：`yum reinstall 软件包名`
*   **升级软件包**：`yum update 软件包名` (不指定包名则升级所有可升级软件包)
*   **卸载软件包**：`yum remove 软件包名`
*   **搜索软件包**：`yum search 关键词`
*   **查看软件包信息**：`yum info 软件包名`
*   **列出所有可用软件包**：`yum list`
*   **列出已安装软件包**：`yum list installed`
*   **检查可更新的软件包**：`yum check-update`
*   **清除缓存**：`yum clean all`

![](img/74b4cf750d48ccd4680684d14c25467b_63.png)

![](img/74b4cf750d48ccd4680684d14c25467b_65.png)

![](img/74b4cf750d48ccd4680684d14c25467b_67.png)

**使用示例**：安装HTTPD服务，YUM会自动解决所有依赖。
```bash
yum -y install httpd
```
`-y` 选项表示对所有询问自动回答“是”。
