在本节课中，我们将学习Linux系统中的四种特殊权限：SUID、SGID、SBIT和ACL。这些权限扩展了基本权限（rwx）的功能，用于满足更精细和特殊的安全控制需求。我们将逐一了解它们的作用、应用场景及设置方法。

上一节我们介绍了Linux文件的基本权限和归属关系。本节中，我们来看看几种特殊的权限机制。特殊权限主要用于可执行文件或目录，以实现普通用户临时提权、限制文件删除或为用户/组定制独立权限等功能。理解这些权限对于系统安全管理至关重要。

## 1. SUID (Set User ID) 👑

SUID权限主要针对**可执行文件**。当一个可执行文件被设置了SUID权限后，任何普通用户在执行该文件时，会**临时拥有文件所有者（通常是root）的身份**。该权限仅在程序执行过程中有效，程序执行完毕后用户恢复原有身份。

### 功能与应用场景
其核心功能是让普通用户能够执行一些需要更高权限才能完成的操作。最典型的例子是`/usr/bin/passwd`命令。

*   **问题**：用户密码存储在`/etc/shadow`文件中，该文件普通用户没有任何写权限。那么普通用户如何能使用`passwd`命令成功修改自己的密码呢？
*   **原理**：查看`/usr/bin/passwd`文件的权限：
    ```bash
    ls -l /usr/bin/passwd
    ```
    你会发现所有者的执行位是 `s` 而非 `x`。这表示该文件拥有SUID权限。当普通用户执行`passwd`时，进程会临时获得文件所有者`root`的身份，从而拥有修改`/etc/shadow`文件的权限。

### 设置与取消SUID
设置SUID的命令是 `chmod u+s <文件名>`，取消则是 `chmod u-s <文件名>`。

**注意**：SUID权限非常强大，如果错误地授予给如`vim`这样的编辑器，普通用户将能以root身份修改任何系统文件，造成严重安全问题，因此**切勿随意设置**。

## 2. SGID (Set Group ID) 👥

SGID权限同样主要针对**可执行文件**，其作用对象是**组**。设置了SGID的文件，任何用户在执行时，会临时拥有文件**所属组的身份**。

![](img/1f596c6b545eb08d8907569a9f29a05b_19.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_20.png)

### 功能与应用场景
其作用是让执行者临时继承文件所属组的权限。系统命令`/usr/bin/locate`是一个例子。

*   **问题**：`locate`命令通过读取`/var/lib/mlocate/mlocate.db`数据库文件来快速查找文件。普通用户对这个数据库文件没有读权限，为何能使用`locate`命令？
*   **原理**：查看`/usr/bin/locate`文件，其所属组的执行位是 `s`。同时，数据库文件`mlocate.db`的所属组是`slocate`，且组权限为`r--`（可读）。当普通用户执行`locate`时，进程临时获得`slocate`组的身份，从而继承了读取数据库文件的权限。

![](img/1f596c6b545eb08d8907569a9f29a05b_22.png)

### 设置与取消SGID
设置SGID的命令是 `chmod g+s <文件名>`，取消则是 `chmod g-s <文件名>`。

![](img/1f596c6b545eb08d8907569a9f29a05b_24.png)

**注意**：SUID和SGID权限会通过文件颜色进行提示（如红色/黄色背景）。对于系统管理员而言，了解其原理即可，通常不需要主动设置。

## 3. SBIT (Sticky Bit) 🚫

![](img/1f596c6b545eb08d8907569a9f29a05b_36.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_38.png)

SBIT权限**只针对目录有效**。它对目录的写权限（w）进行了额外的限制。

![](img/1f596c6b545eb08d8907569a9f29a05b_40.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_41.png)

### 功能与应用场景
当一个目录被设置了SBIT权限后，**除了root用户和文件所有者本人**，其他用户即使对该目录有完整的rwx权限，也无法删除或重命名该目录下**不属于自己的文件或目录**。

*   **典型场景**：系统的`/tmp`和`/var/tmp`目录。这些目录权限为`1777`，其中`1`就代表SBIT权限。所有用户都可以在其中创建文件，但只能管理自己的文件，无法删除他人的文件。
*   **示例**：目录`/test`权限为777，用户A和B都可以在其中创建和删除文件，包括对方的文件。这存在风险。为`/test`设置SBIT权限后，用户A将无法删除用户B创建的文件。

![](img/1f596c6b545eb08d8907569a9f29a05b_43.png)

### 设置与取消SBIT
设置SBIT的命令是 `chmod o+t <目录名>`，取消则是 `chmod o-t <目录名>`。设置了SBIT的目录，在其他人的执行位会显示为 `t`。

![](img/1f596c6b545eb08d8907569a9f29a05b_45.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_47.png)

## 4. ACL (Access Control List) 访问控制列表 🎯

![](img/1f596c6b545eb08d8907569a9f29a05b_51.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_53.png)

ACL提供了比传统“所有者-所属组-其他人”更精细的权限控制。它可以为**任意指定的用户或用户组**，单独设置对某个文件或目录的访问权限。

![](img/1f596c6b545eb08d8907569a9f29a05b_55.png)

### 功能与应用场景
当需要为某个特定用户（非文件所有者，也不在文件所属组内）分配特殊权限，而又不想改变文件的基本权限或用户组关系时，ACL就非常有用。

![](img/1f596c6b545eb08d8907569a9f29a05b_57.png)

**场景示例**：有一个`/devops`目录，属于`devops`组，组权限为`rwx`。组员`tom`和`jerry`拥有完整权限。现在实习生`xiaowei`需要访问该目录，但只希望给她读和执行权限（`r-x`），防止误操作。将她加入`devops`组会赋予过大权限，这时就可以使用ACL为她单独授权。

![](img/1f596c6b545eb08d8907569a9f29a05b_59.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_61.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_62.png)

### ACL常用命令
以下是ACL权限管理的核心命令：

![](img/1f596c6b545eb08d8907569a9f29a05b_64.png)

*   **设置ACL权限**：`setfacl -m u:<用户名>:<权限> <文件/目录>`
    *   例如，为用户`xiaowei`对`/devops`目录添加读和执行权限：`setfacl -m u:xiaowei:rx /devops`
    *   为用户组设置：`setfacl -m g:<组名>:<权限> <文件/目录>`

*   **查看ACL权限**：`getfacl <文件/目录>`
    *   输出结果中，`user:`和`group:`后面的行就是基本权限，而`user:xiaowei:`这样的行就是设置的ACL条目。

*   **删除指定ACL条目**：`setfacl -x u:<用户名> <文件/目录>`
    *   例如，删除用户`xiaowei`的ACL权限：`setfacl -x u:xiaowei /devops`

*   **删除所有ACL条目**：`setfacl -b <文件/目录>`

![](img/1f596c6b545eb08d8907569a9f29a05b_66.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_68.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_70.png)

**提示**：设置了ACL权限的文件或目录，使用`ls -l`查看时，权限位末尾会显示一个加号`+`，例如 `drwxrwx---+`。

## 总结 📝

本节课我们一起学习了Linux中的四种特殊权限：
1.  **SUID**：作用于可执行文件，让执行者临时拥有文件所有者的身份。
2.  **SGID**：作用于可执行文件，让执行者临时拥有文件所属组的身份；也可作用于目录，使在该目录下新建的文件自动继承目录的所属组。
3.  **SBIT**：作用于目录，保护目录内的文件，防止用户删除他人的文件（root除外）。
4.  **ACL**：提供更精细的权限控制，可以为特定的用户或组单独设置文件/目录的访问权限。

![](img/1f596c6b545eb08d8907569a9f29a05b_72.png)

![](img/1f596c6b545eb08d8907569a9f29a05b_74.png)

其中，**SBIT**和**ACL**在实际运维工作中应用更为广泛。理解并合理运用这些权限，能够帮助你构建更安全、更灵活的Linux系统访问控制体系。
