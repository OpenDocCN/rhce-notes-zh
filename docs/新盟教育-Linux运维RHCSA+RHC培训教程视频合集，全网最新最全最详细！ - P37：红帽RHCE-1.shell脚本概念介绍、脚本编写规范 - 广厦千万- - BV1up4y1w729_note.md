## 什么是Shell脚本？

![](img/b60b437f5b194511dd3c8c6efed6101a_7.png)

上一节我们介绍了课程的整体安排，本节中我们来看看Shell脚本到底是什么。

Shell脚本本质上是一门编程语言。我们与计算机系统交互时输入的命令（如 `ls`、`pwd`）都属于Shell语言的范畴。Shell语言是一种解释型语言，这意味着它通过一个称为“解释器”的程序，将我们编写的命令逐行翻译成计算机能理解的二进制指令来执行。

![](img/b60b437f5b194511dd3c8c6efed6101a_9.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_11.png)

与C/C++等需要提前编译的“编译型语言”不同，Shell脚本是“边执行边解释”的。这使得它用起来更加方便，属于“高级语言”，即更贴近人类自然习惯的语言。

**核心公式：**
```
用户命令 -> Shell解释器（如/bin/bash） -> 二进制指令 -> 计算机执行
```

![](img/b60b437f5b194511dd3c8c6efed6101a_13.png)

Shell脚本的功能非常强大，它支持变量、数组、条件判断（if）、循环（for/while）、函数等大多数编程语言都具备的功能。因此，我们可以利用Shell脚本实现许多自动化任务。

![](img/b60b437f5b194511dd3c8c6efed6101a_15.png)

以下是Shell脚本常见的应用场景：
*   **批量操作**：对多台服务器进行相同的配置或软件安装。
*   **自动化部署**：一键搭建复杂的应用环境（如LNMP架构）。
*   **系统初始化**：新服务器上线时，自动完成基础环境配置。
*   **监控与告警**：定期检查系统状态，发现异常时自动触发通知。
*   **定时任务**：结合计划任务，在指定时间（如凌晨）自动执行备份等操作。

## 为什么学习Shell脚本？

理解了Shell脚本是什么之后，本节我们来看看为什么它对运维工程师至关重要。

![](img/b60b437f5b194511dd3c8c6efed6101a_21.png)

手动重复执行相同的命令不仅效率低下，而且容易出错。Shell脚本可以将一系列命令固化到一个文件中，实现“一次编写，多次运行”。这极大地提升了工作效率和准确性，是运维工程师实现自动化、提升“幸福指数”的核心技能。

![](img/b60b437f5b194511dd3c8c6efed6101a_22.png)

例如，需要为50台新服务器部署相同的环境。手动操作可能需要数天，且难免出错。而编写一个部署脚本后，这项工作可能在几分钟内就能完成。

![](img/b60b437f5b194511dd3c8c6efed6101a_24.png)

## 如何编写一个规范的Shell脚本？

知道了Shell脚本的威力，本节我们来学习如何动手编写一个规范的脚本。

一个规范的Shell脚本通常包含以下几个部分，我们将通过一个输出“Hello World”的简单脚本来演示。

### 1. 创建脚本文件

![](img/b60b437f5b194511dd3c8c6efed6101a_26.png)

首先，我们创建一个专门存放脚本的目录，并新建脚本文件。脚本文件的后缀通常为 `.sh`，这只是为了方便识别，并非强制要求。

![](img/b60b437f5b194511dd3c8c6efed6101a_27.png)

```bash
mkdir -p /scripts
cd /scripts
vim hello.sh
```

### 2. 脚本的组成部分

![](img/b60b437f5b194511dd3c8c6efed6101a_29.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_31.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_33.png)

打开 `hello.sh` 文件后，我们按照规范顺序编写内容。

![](img/b60b437f5b194511dd3c8c6efed6101a_35.png)

**第一部分：环境声明（Shebang）**
脚本的第一行必须指明用哪个解释器来执行本脚本。格式为 `#!` 后接解释器的绝对路径。

```bash
#!/bin/bash
```
这行代码告诉系统，使用 `/bin/bash` 这个程序来解释本脚本中的命令。

**第二部分：注释信息**
注释以 `#` 开头，用于解释脚本的功能、作者、更新时间等信息。注释不会被当作命令执行，但对脚本的维护和他人阅读至关重要。

```bash
# 作者：YourName
# 功能：在屏幕上输出 Hello World
# 时间：2023-10-27
```

![](img/b60b437f5b194511dd3c8c6efed6101a_37.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_38.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_39.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_41.png)

**第三部分：可执行命令**
从这一部分开始，编写需要依次执行的命令。我们的目标是输出“Hello World”，所以使用 `echo` 命令。

![](img/b60b437f5b194511dd3c8c6efed6101a_43.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_45.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_47.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_49.png)

```bash
echo “Hello World”
```

![](img/b60b437f5b194511dd3c8c6efed6101a_51.png)

### 3. 完整的脚本示例

![](img/b60b437f5b194511dd3c8c6efed6101a_53.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_54.png)

将以上三部分组合起来，`hello.sh` 文件的完整内容如下：

![](img/b60b437f5b194511dd3c8c6efed6101a_56.png)

```bash
#!/bin/bash
# 作者：YourName
# 功能：在屏幕上输出 Hello World
# 时间：2023-10-27

![](img/b60b437f5b194511dd3c8c6efed6101a_58.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_59.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_61.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_63.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_65.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_67.png)

echo “Hello World”
```

### 4. 执行脚本

脚本编写完成后，它只是一个文本文件，需要赋予其“执行权限”才能运行。

![](img/b60b437f5b194511dd3c8c6efed6101a_69.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_71.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_73.png)

以下是执行脚本的步骤：
1.  为脚本文件添加执行权限。
    ```bash
    chmod +x hello.sh
    ```
2.  执行脚本。注意，执行当前目录下的脚本时，需要在脚本名前加上 `./` 来指明路径。
    ```bash
    ./hello.sh
    ```
3.  执行后，终端将会显示输出结果：`Hello World`。

![](img/b60b437f5b194511dd3c8c6efed6101a_75.png)

## 总结

本节课中我们一起学习了Shell脚本的核心概念。我们了解到Shell脚本是一门强大的解释型编程语言，是运维自动化工作的基石。我们学习了编写一个规范脚本的完整流程：从创建文件、编写环境声明（`#!/bin/bash`）和注释，到填入具体的执行命令，最后通过修改权限并执行脚本。

![](img/b60b437f5b194511dd3c8c6efed6101a_77.png)

![](img/b60b437f5b194511dd3c8c6efed6101a_79.png)

记住，脚本的本质是**命令的堆积**。将你平时在命令行中手动输入的命令，按照逻辑顺序保存到一个文件中，就构成了一个脚本。从下一个简单的“Hello World”开始，你将逐步学会用脚本解决更复杂的实际问题。
