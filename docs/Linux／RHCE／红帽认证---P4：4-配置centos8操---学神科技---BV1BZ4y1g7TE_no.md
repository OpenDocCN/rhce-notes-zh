# CentOS 8 系统安装教程：P4：配置操作系统参数与自定义分区 🖥️

在本节课中，我们将学习如何配置 CentOS 8 操作系统的安装参数，并进行自定义磁盘分区。这是安装过程中至关重要的一步，直接关系到系统未来的稳定性和可扩展性。

## 语言与时区设置 🌍

![](img/0340e0314a104a8e7846868539ce4b67_5.png)

安装程序启动后，首先进入语言选择界面。默认语言为 English，我们需要选择简体中文（Chinese (Simplified)）以方便后续操作。选择后点击“继续”。

![](img/0340e0314a104a8e7846868539ce4b67_7.png)

接下来需要设置时区。时区设置会影响系统日志记录和时间同步。请选择“亚洲/上海”作为系统时区。

## 软件选择与安装源 📦

在“软件选择”部分，初学者建议选择“带 GUI 的服务器”，以便拥有图形界面进行操作。如果已经熟悉 Linux 命令行，可以选择“最小安装”以加快安装速度并减少潜在的安全风险。

无论选择哪种安装模式，**必须勾选“开发工具”**。这个选项会安装 GCC 等编译工具，为后续可能需要的源码编译做好准备。

![](img/0340e0314a104a8e7846868539ce4b67_9.png)

安装源保持默认的“本地介质”即可。

### 分区方案详解

![](img/0340e0314a104a8e7846868539ce4b67_25.png)

![](img/0340e0314a104a8e7846868539ce4b67_27.png)

![](img/0340e0314a104a8e7846868539ce4b67_29.png)

![](img/0340e0314a104a8e7846868539ce4b67_31.png)

在分区方案界面，选择“自定义”并点击“完成”。分区类型建议选择“标准分区”而非 LVM，以简化初期的学习过程。

![](img/0340e0314a104a8e7846868539ce4b67_33.png)

以下是创建分区的步骤和推荐大小：

![](img/0340e0314a104a8e7846868539ce4b67_35.png)

1.  **引导分区 (/boot)**：
    *   **作用**：存放系统内核和引导程序（GRUB）。
    *   **文件系统**：`xfs`
    *   **期望容量**：`1024 MiB` (1 GiB)
    *   **说明**：在 CentOS 8 中，建议分配 1GB 空间，为系统内核更新预留足够空间。传统教程中分配的 200MB 在多次系统升级后可能不足。

2.  **交换分区 (swap)**：
    *   **作用**：当物理内存不足时，系统会将部分内存数据临时存储到此分区。
    *   **文件系统类型**：`swap`
    *   **期望容量**：`2048 MiB` (2 GiB)
    *   **说明**：对于拥有较大物理内存（如 16GB 以上）的系统，无需遵循“物理内存 1.5-2 倍”的旧规则。分配 2GB 足以应对一般情况。如果系统频繁使用交换分区，通常意味着需要增加物理内存。

3.  **根分区 (/) **：
    *   **作用**：系统的主要分区，存放操作系统核心文件、应用程序和日志等。
    *   **文件系统**：`xfs`
    *   **期望容量**：`51200 MiB` (50 GiB)
    *   **说明**：分配足够的空间以避免因系统日志增长导致根分区被填满，从而引发系统问题。

![](img/0340e0314a104a8e7846868539ce4b67_37.png)

![](img/0340e0314a104a8e7846868539ce4b67_39.png)

![](img/0340e0314a104a8e7846868539ce4b67_41.png)

4.  **数据分区 (可选，如 /data)**：
    *   **作用**：用于存放用户数据，与系统文件分离，便于管理和数据安全。
    *   **文件系统**：`xfs`
    *   **期望容量**：使用剩余的所有磁盘空间。
    *   **说明**：这种分离类似于 Windows 中将系统安装在 C 盘，数据存放在 D 盘。当系统需要重装时，数据分区的内容得以保留。

![](img/0340e0314a104a8e7846868539ce4b67_42.png)

![](img/0340e0314a104a8e7846868539ce4b67_44.png)

## 内核转储与网络配置 🌐

*   **Kdump**：这是内核崩溃转储机制，用于系统崩溃时收集调试信息。对于初学者和学习环境，可以**取消勾选“启用 Kdump”**，以节省内存资源。
*   **网络和主机名**：这是确保系统安装后能够联网的关键步骤。
    1.  找到你的网卡（常见名称如 `ens33`, `ens160`），将开关从“关闭”切换到“打开”。如果网络支持 DHCP，系统会自动获取 IP 地址。
    2.  为了后续远程连接的便利，建议配置静态 IP。点击“配置”按钮。
    3.  在 IPv4 设置中，将方法改为“手动”。
    4.  点击“添加”按钮，依次填写：
        *   **地址**：例如 `192.168.1.63`
        *   **子网掩码**：例如 `24` (代表 255.255.255.0)
        *   **网关**：例如 `192.168.1.1`
        *   **DNS 服务器**：例如 `192.168.1.1` 或 `8.8.8.8`
    5.  点击“保存”。
    6.  在主机名输入框中，设置一个易于识别的主机名，例如 `xuegod63`，然后点击“应用”。

## 开始安装与密码设置 🔐

检查所有配置无误后，点击“开始安装”。

在安装过程中，需要为 root 管理员用户设置密码。点击“ROOT 密码”进行设置。为了学习方便，本教程使用 `123456` 作为示例密码。**请注意，在生产环境中必须使用高强度复杂密码。**

可以跳过创建普通用户的步骤，安装完成后根据需要再创建。

系统将开始安装软件包，这个过程需要一些时间。请耐心等待直至安装完成。
