## 软件版本升级与降级

![](img/0051b122324fb7906641032526536b21_7.png)

![](img/0051b122324fb7906641032526536b21_9.png)

上一节我们介绍了YUM的基本安装与查询操作，本节中我们来看看如何管理软件的具体版本。

有时，我们需要安装或升级到特定的软件版本，而不是默认的最新版。YUM提供了相应的选项来实现这一点。

![](img/0051b122324fb7906641032526536b21_11.png)

![](img/0051b122324fb7906641032526536b21_13.png)

以下是列出仓库中某个软件包所有可用版本的命令：
```bash
yum list --showduplicates <软件包名>
```
例如，要列出`nginx`的所有版本，可以执行：
```bash
yum list --showduplicates nginx
```
执行后，你会看到仓库中所有可用的nginx版本列表。

![](img/0051b122324fb7906641032526536b21_15.png)

![](img/0051b122324fb7906641032526536b21_17.png)

要安装指定版本的软件包，需要使用以下命令格式：
```bash
yum install <软件包名>-<版本号>
```
例如，要安装`nginx`的`1.18.0`版本，可以执行：
```bash
yum install nginx-1.18.0
```
YUM会自动处理该版本所需的依赖关系。

### 升级软件与配置文件保留

![](img/0051b122324fb7906641032526536b21_19.png)

![](img/0051b122324fb7906641032526536b21_21.png)

![](img/0051b122324fb7906641032526536b21_23.png)

![](img/0051b122324fb7906641032526536b21_25.png)

升级软件时，一个常见的担忧是自定义的配置文件会被覆盖。现代软件包管理器（如YUM）通常具备智能的升级机制。

![](img/0051b122324fb7906641032526536b21_27.png)

1.  首先，修改当前已安装软件的配置文件（例如`/etc/nginx/nginx.conf`），添加一些自定义配置。
2.  然后，使用升级命令安装新版本：
    ```bash
    yum update nginx
    ```
3.  升级完成后，检查配置文件。你会发现，之前添加的自定义配置通常会被保留下来，而不会被新版本的默认配置文件覆盖。

![](img/0051b122324fb7906641032526536b21_29.png)

**注意**：虽然现代包管理机制已经比较完善，但为了保险起见，在升级重要软件前，手动备份关键配置文件（如`/etc/`目录下的相关文件）仍然是一个好习惯。

### 软件卸载

![](img/0051b122324fb7906641032526536b21_37.png)

卸载软件使用`remove`命令：
```bash
yum remove <软件包名>
```
例如，卸载`nginx`：
```bash
yum remove nginx
```
现在的YUM在卸载时通常只移除指定的主软件包，而不会自动删除其依赖包，这避免了误删其他程序所需依赖的风险。

![](img/0051b122324fb7906641032526536b21_39.png)

## 其他常用YUM命令

![](img/0051b122324fb7906641032526536b21_41.png)

![](img/0051b122324fb7906641032526536b21_43.png)

以下是几个非常实用的YUM命令。

`yum provides`命令用于查找某个特定命令或文件是由哪个软件包提供的。当系统提示“command not found”时，这个命令尤其有用。
```bash
yum provides <命令或文件路径>
```
例如，如果你需要`ab`命令（Apache HTTP服务器性能测试工具），但系统未安装，可以运行：
```bash
yum provides ab
```
命令会返回提供`ab`命令的软件包名称（例如`httpd-tools`），然后你就可以安装这个包来获得该命令。

`yum makecache`命令用于将仓库的元数据（如软件包列表、依赖信息）下载到本地缓存。这可以加速后续的软件搜索和安装操作，尤其是在使用网络仓库时。
`yum clean all`命令用于清除所有本地YUM缓存。当仓库源发生变化（例如本地光盘镜像被移除），但YUM仍使用旧的缓存信息导致报错时，就需要使用此命令清除缓存，让YUM重新获取最新的仓库信息。

## 配置网络YUM仓库

![](img/0051b122324fb7906641032526536b21_49.png)

![](img/0051b122324fb7906641032526536b21_51.png)

本地YUM仓库的软件包可能有限。为了获取更丰富、更新的软件，我们可以配置网络YUM仓库，如阿里云、清华大学等开源镜像站。

![](img/0051b122324fb7906641032526536b21_53.png)

配置网络仓库非常简单，通常只需下载镜像站提供的仓库配置文件即可。

![](img/0051b122324fb7906641032526536b21_55.png)

![](img/0051b122324fb7906641032526536b21_57.png)

以下是配置阿里云CentOS 7 YUM源的步骤：
1.  使用`wget`命令下载仓库文件（如果系统没有`wget`，可先执行`yum install -y wget`安装）：
    ```bash
    wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo
    ```
2.  下载完成后，仓库即配置完毕。你可以列出所有仓库来验证：
    ```bash
    yum repolist
    ```
    输出中会显示`base`、`updates`、`extras`等多个来自阿里云的仓库及其包含的软件包数量。

![](img/0051b122324fb7906641032526536b21_59.png)

![](img/0051b122324fb7906641032526536b21_61.png)

![](img/0051b122324fb7906641032526536b21_63.png)

网络仓库的配置文件里可能定义了多个软件源（如基础包、更新包、额外包），并可能已经设置了**仓库优先级**。优先级会影响YUM选择软件包的顺序。

![](img/0051b122324fb7906641032526536b21_65.png)

## 设置YUM仓库优先级

![](img/0051b122324fb7906641032526536b21_67.png)

当系统配置了多个YUM仓库（如本地仓库和多个网络仓库）时，YUM需要决定优先从哪个仓库下载软件。这时，仓库优先级就非常重要。

![](img/0051b122324fb7906641032526536b21_69.png)

默认情况下，如果仓库未明确设置优先级，YUM的行为可能不符合预期。我们希望的是：优先从本地仓库下载（速度最快），如果本地没有，再去网络仓库查找。

![](img/0051b122324fb7906641032526536b21_71.png)

以下是设置本地仓库优先级最高的步骤：

1.  安装优先级插件：
    ```bash
    yum install -y yum-plugin-priorities
    ```
2.  编辑本地仓库的配置文件（例如`/etc/yum.repos.d/local.repo`），在`[local]`段落中添加优先级设置：
    ```
    [local]
    name=Local Repository
    baseurl=file:///mnt/cdrom
    enabled=1
    gpgcheck=0
    priority=1
    ```
    其中，**`priority=1`** 是关键。优先级数字范围为1-99，**数字越小，优先级越高**。设置为1意味着此仓库拥有最高优先级。

3.  保存文件后，再进行软件安装时，YUM就会优先从本地仓库查找软件包。只有当本地仓库不存在该软件时，才会去查询优先级较低的网络仓库。

**请注意**：只有在配置了多个仓库（尤其是网络仓库）时，设置优先级才有意义。如果只有一个本地仓库，则无需设置。
