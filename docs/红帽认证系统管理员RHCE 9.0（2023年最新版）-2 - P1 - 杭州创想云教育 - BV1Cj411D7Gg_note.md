# RHCE 9.0 课程：第五章：创建和查看文本文件

在本节课中，我们将要学习Linux中关于文本文件创建、查看以及Shell环境的一些核心概念，包括重定向、管道、文本编辑器以及环境变量。

## 第一节：理解并掌握Shell重定向

上一节我们回顾了Linux的基础命令和文件系统结构。本节中，我们来看看如何控制命令的输入和输出，即Shell重定向。

计算机执行命令的标准流程是：用户通过键盘（标准输入，STDIN，文件描述符0）输入指令，系统处理后将正确结果输出到控制台（标准输出，STDOUT，文件描述符1），或将错误信息输出到控制台（标准错误，STDERR，文件描述符2）。

重定向就是改变这个默认的输入/输出方向。例如，将命令的输出结果保存到文件中，而不是显示在屏幕上。

![](img/49ffbb291276797a45e0e34280613c42_1.png)

![](img/49ffbb291276797a45e0e34280613c42_2.png)

![](img/49ffbb291276797a45e0e34280613c42_3.png)

![](img/49ffbb291276797a45e0e34280613c42_4.png)

以下是重定向的核心操作符：

*   **`>`**：将标准输出重定向到文件，会覆盖文件原有内容。
    *   示例：`ls -l example1.txt > output.log`
*   **`>>`**：将标准输出追加到文件末尾，不会覆盖原有内容。
    *   示例：`ls -l example1.txt >> output.log`
*   **`2>`**：将标准错误重定向到文件。
    *   示例：`ls -l nonexistent.txt 2> error.log`
*   **`&>`** 或 **`2>&1`**：将标准输出和标准错误都重定向到同一个文件。
    *   示例：`ls -l *.txt &> all_output.log`
*   **`<`**：将文件内容作为命令的标准输入。
    *   示例：`sort < input.txt`
*   **`/dev/null`**：一个特殊的设备文件，任何写入它的数据都会被丢弃。常用于丢弃不需要的输出。
    *   示例：`ls -l 2> /dev/null` （忽略所有错误信息）

## 第二节：使用管道和Tee命令处理数据流

上一节我们介绍了如何将命令输出重定向到文件。本节中我们来看看如何将一个命令的输出作为另一个命令的输入，即管道操作。

管道符 `|` 可以将一个命令的标准输出连接到另一个命令的标准输入，实现数据的流水线处理。

**管道的基本语法：**
`command1 | command2`

![](img/49ffbb291276797a45e0e34280613c42_6.png)

**示例：**
`ls -l /usr/bin | less` （将 `ls` 的长列表结果通过 `less` 分页查看）

![](img/49ffbb291276797a45e0e34280613c42_8.png)

![](img/49ffbb291276797a45e0e34280613c42_10.png)

![](img/49ffbb291276797a45e0e34280613c42_12.png)

Tee命令则像一个“三通”，它在管道中分流数据：既将数据传递给下一个命令，又同时将数据保存到文件或输出到屏幕。

**Tee命令的基本语法：**
`command1 | tee [选项] 文件名 | command2`

**常用选项：**
*   `-a`：追加到文件，而不是覆盖。

**示例：**
`ls -l | tee filelist.txt | wc -l` （将列表既保存到 `filelist.txt`，又传递给 `wc -l` 统计行数）

## 第三节：使用Vim文本编辑器

![](img/49ffbb291276797a45e0e34280613c42_39.png)

![](img/49ffbb291276797a45e0e34280613c42_40.png)

在Linux服务器管理中，经常需要编辑配置文件。Vim是一个功能强大、基于文本界面的编辑器，是系统管理员必备技能。

Vim有三种基本模式：
1.  **命令模式**：启动Vim后的默认模式，可以执行移动光标、删除、复制等命令。
2.  **插入模式**：在此模式下可以输入和编辑文本。按 `i`, `a`, `o` 等键进入。
3.  **末行模式（扩展命令模式）**：在命令模式下按 `:` 进入，可以执行保存、退出、搜索替换等操作。

**基本工作流程：**
1.  `vim filename` 打开文件（进入命令模式）。
2.  按 `i` 进入插入模式，编辑文本。
3.  按 `Esc` 键返回命令模式。
4.  输入 `:wq` 保存并退出。

**常用命令模式操作：**
*   `h` `j` `k` `l` 或方向键：移动光标。
*   `x`：删除光标处的字符。
*   `dd`：删除当前行。
*   `yy`：复制当前行。
*   `p`：粘贴。
*   `u`：撤销。
*   `/关键词`：搜索。

**常用末行模式命令：**
*   `:w`：保存。
*   `:q`：退出。
*   `:q!`：强制退出，不保存。
*   `:wq` 或 `:x`：保存并退出。
*   `:set nu`：显示行号。

## 第四节：管理Shell环境变量

Shell环境变量是存储系统信息和用户配置的键值对，它们影响着Shell的行为和外观。

变量可分为临时变量和永久变量：
*   **临时变量**：仅在当前Shell会话中有效。定义方法：`变量名=值`
*   **永久变量**：需要将定义写入Shell的配置文件中，如 `~/.bashrc`（对当前用户）或 `/etc/bashrc`（对所有用户）。

**定义和引用变量：**
*   **定义**：`MY_VAR="Hello World"`
*   **引用**：`echo $MY_VAR` 或 `echo ${MY_VAR}`（推荐后者，避免歧义）。

**重要环境变量示例：**
*   `PATH`：系统查找可执行文件的路径列表。
    *   示例：添加自定义脚本目录到PATH：`export PATH=$PATH:/home/user/scripts`
*   `HOME`：当前用户的家目录。
*   `USER`：当前用户名。
*   `LANG`：系统语言设置。

**查看环境变量命令：**
*   `env` 或 `printenv`：显示所有环境变量。
*   `set`：显示所有Shell变量（包括环境变量和局部变量）。
*   `echo $变量名`：查看特定变量。

**使变量永久生效：**
将 `export 变量名=值` 这行命令添加到 `~/.bashrc` 文件末尾，然后执行 `source ~/.bashrc` 或重新登录即可。
