## 什么是容器？

![](img/367a0910486a31ad88f841454e6c2d83_7.png)

![](img/367a0910486a31ad88f841454e6c2d83_8.png)

容器，顾名思义，是用于容纳物品的载体。在计算机领域，容器是一种轻量级的虚拟化技术，它可以将应用程序及其运行所需的所有环境（如代码、运行时、系统工具、库和设置）打包在一起。

![](img/367a0910486a31ad88f841454e6c2d83_10.png)

![](img/367a0910486a31ad88f841454e6c2d83_12.png)

## 容器解决了什么问题？

上一节我们介绍了容器的基本定义，本节中我们来看看容器技术具体解决了企业软件部署中的哪些痛点。在企业中，软件的部署方式主要经历了三个阶段。

![](img/367a0910486a31ad88f841454e6c2d83_14.png)

![](img/367a0910486a31ad88f841454e6c2d83_16.png)

### 传统部署方式

传统部署方式是将软件直接安装在物理服务器上。

![](img/367a0910486a31ad88f841454e6c2d83_18.png)

![](img/367a0910486a31ad88f841454e6c2d83_20.png)

*   **优点**：部署简单直接。
*   **缺点**：无法限制单个程序对系统资源（如CPU、内存）的使用量。一个程序出现故障（如内存泄漏）可能耗尽所有资源，导致同一服务器上的其他所有程序崩溃。

![](img/367a0910486a31ad88f841454e6c2d83_22.png)

![](img/367a0910486a31ad88f841454e6c2d83_24.png)

![](img/367a0910486a31ad88f841454e6c2d83_26.png)

### 虚拟化部署方式

虚拟化技术（如VMware）可以在单台物理服务器上创建多个相互隔离的虚拟机。

![](img/367a0910486a31ad88f841454e6c2d83_28.png)

![](img/367a0910486a31ad88f841454e6c2d83_30.png)

*   **优点**：实现了程序间的隔离，一个虚拟机内的程序故障不会影响其他虚拟机。提高了硬件资源的利用率。
*   **缺点**：每个虚拟机都需要虚拟出一套完整的操作系统和硬件，占用资源较多，启动较慢。

![](img/367a0910486a31ad88f841454e6c2d83_32.png)

![](img/367a0910486a31ad88f841454e6c2d83_33.png)

![](img/367a0910486a31ad88f841454e6c2d83_34.png)

### 容器化部署方式

![](img/367a0910486a31ad88f841454e6c2d83_36.png)

![](img/367a0910486a31ad88f841454e6c2d83_38.png)

容器化是一种更轻量级的虚拟化方案。

*   **优点**：
    1.  **隔离性**：每个容器相互隔离，互不影响。
    2.  **高效性**：容器共享主机操作系统内核，无需虚拟完整操作系统和硬件，因此更加轻量，启动速度极快（秒级），资源利用率更高。
    3.  **一致性**：容器将应用及其依赖环境打包，确保了应用在开发、测试、生产等任何环境中的运行一致性，解决了“水土不服”的问题。

## Docker是什么？

理解了容器带来的好处后，我们来看看如何管理容器。Docker就是这样一个管理容器的工具。

Docker是一个用Go语言编写的开源项目，它是一套完整的容器管理系统。其核心理念是 **“一次封装，随处运行”**。

*   **Docker与内核的关系**：容器的底层支持来源于Linux内核的功能，但用户无法直接操作。Docker作为工具，提供了简单的命令，让我们能够方便地使用内核的容器功能。
*   **Docker的Logo寓意**：Docker的Logo是一只鲸鱼驮着许多集装箱。鲸鱼代表宿主机操作系统，每个集装箱代表一个容器。这形象地体现了容器的两大特点：**隔离性**（集装箱间互不影响）和**高效性**（集装箱可堆叠，节约空间）。

## Docker的核心组成

![](img/367a0910486a31ad88f841454e6c2d83_40.png)

![](img/367a0910486a31ad88f841454e6c2d83_42.png)

Docker系统主要由三个核心部分组成：镜像、容器和仓库。

以下是三者的关系与工作流程：
1.  用户从**仓库**拉取**镜像**。
2.  使用Docker命令将**镜像**运行起来，成为**容器**。
3.  应用程序在**容器**中运行。

### 镜像 (Image)

镜像是创建容器的模板。你可以将其理解为一个特殊的文件系统，里面包含了运行某个软件所需的所有代码、运行时环境、库、环境变量和配置文件。

![](img/367a0910486a31ad88f841454e6c2d83_44.png)

![](img/367a0910486a31ad88f841454e6c2d83_46.png)

**公式表示**：`镜像 = 应用程序代码 + 依赖环境 + 基础系统层`

### 容器 (Container)

容器是镜像的运行实例。你可以把镜像看作“类”，容器就是根据这个“类”创建出来的“对象”。容器是应用程序的真正载体和运行环境。

**代码表示启动容器**：`docker run [镜像名称]`

### 仓库 (Repository)

仓库是集中存放镜像的地方，类似于代码仓库或应用商店。

![](img/367a0910486a31ad88f841454e6c2d83_48.png)

![](img/367a0910486a31ad88f841454e6c2d83_50.png)

![](img/367a0910486a31ad88f841454e6c2d83_52.png)

*   **公有仓库**：如 Docker Hub (`hub.docker.com`)，所有人都可以访问和下载镜像。
*   **私有仓库**：企业或个人搭建的私人仓库，用于存放内部镜像，如 Harbor。

![](img/367a0910486a31ad88f841454e6c2d83_54.png)
