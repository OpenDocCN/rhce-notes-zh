## 开源与闭源软件

![](img/c2dce9fc87b198e5020bf7a608d469b0_31.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_33.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_35.png)

在深入了解软件包类型之前，我们需要先理解开源和闭源的概念。

![](img/c2dce9fc87b198e5020bf7a608d469b0_37.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_39.png)

*   **开源软件**：指软件的源代码被公开发布，允许用户查看、修改和进行二次开发。例如，淘宝基于开源的Nginx二次开发出了Tengine服务器。开源的优势在于用户使用放心（代码公开透明，不易藏有恶意代码），并且即使原开发者停止维护，社区也可能会继续维护和更新。
*   **闭源软件**：指软件的源代码不公开，用户无法查看和修改。例如，微软的Windows操作系统。闭源软件的缺点是用户无法进行二次开发，且一旦开发者停止维护（如Windows 7），用户将很难自行修复问题。

需要注意的是，开源并不完全等同于免费，有些开源软件会针对高级功能或技术支持进行收费。

### 源码包

![](img/c2dce9fc87b198e5020bf7a608d469b0_57.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_59.png)

源码包是直接包含软件原始代码的压缩包。例如，从Nginx官网下载的 `nginx-1.18.0.tar.gz` 文件。

![](img/c2dce9fc87b198e5020bf7a608d469b0_61.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_63.png)

以下是源码包的主要特点：

![](img/c2dce9fc87b198e5020bf7a608d469b0_65.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_67.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_69.png)

*   **优点**：
    1.  **开放源代码**：用户可以查看和修改源代码。
    2.  **安装灵活**：在编译（将源代码转换为机器可执行的二进制文件）过程中，可以**自定义安装路径**和**选择需要的功能模块**，实现按需安装，有利于节约系统资源。
    3.  **卸载方便**：由于文件集中安装在自定义目录下，直接删除该目录即可干净卸载。
*   **缺点**：
    1.  **安装复杂**：安装过程需要用户手动执行配置、编译、安装等步骤。
    2.  **需解决依赖**：需要用户自行解决软件依赖的其他库或软件包。

![](img/c2dce9fc87b198e5020bf7a608d469b0_71.png)

### RPM包（二进制包）

RPM包是已经由软件发行商预先编译好的二进制文件包，其后缀名通常为 `.rpm`，例如 `vsftpd-3.0.2-25.el7.x86_64.rpm`。我们之前使用 `yum install` 命令安装的软件都是RPM包。

以下是RPM包的主要特点：

*   **优点**：
    1.  **安装简单快捷**：因为已经编译好，无需手动编译，安装速度非常快。安装命令示例：`rpm -ivh 软件包全名.rpm`
    2.  **功能齐全**：官方编译的包通常会包含软件的绝大多数功能。
*   **缺点**：
    1.  **无法自定义**：不能选择安装路径，也无法自定义安装哪些功能。
    2.  **看不到源代码**：因为是二进制格式，用户无法查看其源代码。
    3.  **依赖关系复杂**：在安装时可能需要手动处理复杂的依赖关系。

**如何选择？** 对于简单的工具软件（如文件传输工具lrzsz），追求效率，使用RPM包安装即可。对于企业核心应用，如果对安全性、定制化有严格要求，则可能选择源码包安装。

## RPM包命名规则与软件源

一个典型的RPM包名称包含丰富的信息，例如：`vsftpd-3.0.2-25.el7.x86_64.rpm`

![](img/c2dce9fc87b198e5020bf7a608d469b0_73.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_75.png)

*   `vsftpd`：软件包名称。
*   `3.0.2`：软件版本号（主版本.次版本.修订号）。
*   `25`：发行号，代表该版本被打过25次补丁。
*   `el7`：适合在Red Hat Enterprise Linux 7或其衍生系统（如CentOS 7）上运行。
*   `x86_64`：适合的CPU架构（64位）。
*   `.rpm`：文件扩展名，表明这是一个RPM包。

![](img/c2dce9fc87b198e5020bf7a608d469b0_77.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_79.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_81.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_83.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_85.png)

Linux系统中的软件包主要有两个来源：
1.  **本地镜像**：系统安装光盘或ISO镜像文件中自带的软件包仓库。这些包通常挂载在 `/mnt/` 目录下，例如 `/mnt/Packages/` 目录中可能包含数千个RPM包。
2.  **网络仓库**：互联网上的软件仓库，如官方源、EPEL源等，可以通过 `yum` 命令访问（下节课详述）。

### 安装软件包

![](img/c2dce9fc87b198e5020bf7a608d469b0_95.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_96.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_104.png)

安装RPM包的基本命令格式是 `rpm -ivh 软件包全名`。
*   `-i`：安装（install）。
*   `-v`：显示详细信息（verbose）。
*   `-h`：显示安装进度条（hash）。

![](img/c2dce9fc87b198e5020bf7a608d469b0_106.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_108.png)

**操作示例**：
假设我们已挂载光盘，并进入软件包目录 `/mnt/Packages/`，要安装vsftpd软件包。
```bash
cd /mnt/Packages/
rpm -ivh vsftpd-3.0.2-25.el7.x86_64.rpm
```
注意：安装时必须指定**软件包的全名**（包括版本、架构等信息），可以使用 `Tab` 键自动补全。

### 查询软件包

![](img/c2dce9fc87b198e5020bf7a608d469b0_116.png)

查询是 `rpm` 命令非常强大且常用的功能。

![](img/c2dce9fc87b198e5020bf7a608d469b0_118.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_120.png)

![](img/c2dce9fc87b198e5020bf7a608d469b0_122.png)

以下是常用的查询选项：

*   **查询软件是否安装**：`rpm -q 软件名`
    ```bash
    rpm -q vsftpd
    ```

*   **查询系统中所有已安装的软件包**：`rpm -qa`
    ```bash
    rpm -qa | wc -l        # 统计已安装包的数量
    rpm -qa | grep vsftpd  # 在所有包中过滤出包含‘vsftpd’的包
    ```

*   **查询软件包的详细信息**：`rpm -qi 软件名`
    ```bash
    rpm -qi vsftpd
    ```
    此命令会显示软件名称、版本、描述、安装时间、大小、官方网址等详细信息。

*   **查询文件由哪个软件包安装**：`rpm -qf 文件绝对路径`
    ```bash
    rpm -qf /etc/passwd
    which cat               # 先查找cat命令的路径
    rpm -qf /usr/bin/cat
    ```

*   **查询软件包安装了哪些文件**：`rpm -ql 软件名`
    ```bash
    rpm -ql coreutils       # 查看coreutils包安装了哪些文件（包含很多基础命令）
    ```

**重要提示**：安装时需要提供“软件包全名”，而安装后的查询、卸载等操作，通常使用“软件名”（即程序名）即可。有些软件的程序名和包名可能不同，可以通过 `rpm -qa | grep 关键字` 来查找确认。
