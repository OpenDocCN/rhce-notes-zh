# Linux基础：021：软硬链接详解 🔗

在本节课中，我们将要学习Linux系统中的两种重要链接方式：软链接和硬链接。理解它们的区别和工作原理，对于管理文件和目录至关重要。

## 概述

在Windows系统中，我们常用“快捷方式”来从不同位置访问同一个文件。Linux系统提供了更强大的链接功能，分为**软链接**和**硬链接**两种。它们看似相似，但底层机制和特性有显著不同。本节我们将深入探讨这两种链接的创建、工作原理及区别。

## 软链接（符号链接）

上一节我们介绍了链接的基本概念，本节中我们来看看第一种链接方式——软链接。

软链接，也称为符号链接。它仅仅包含所链接文件的**路径名**，就像一个记录了地址的标签。当原始文件被删除或移动后，软链接会失效，因为它指向的路径不存在了。软链接可以链接到文件或目录，并且可以跨不同的文件系统创建。

其工作原理可以理解为：
*   **操作者** -> **软链接**（存储路径） -> **原始文件** -> **硬盘数据**

如果原始文件被删除，软链接因找不到目标路径而失效。

以下是创建和验证软链接的步骤：

1.  首先，创建一个测试文件并写入内容。
    ```bash
    echo "这是一些测试内容" > aa.txt
    cat aa.txt
    ```

2.  使用 `ln -s` 命令创建软链接。`-s` 选项代表 `soft`（软链接）。
    ```bash
    ln -s aa.txt abc.link
    ```

3.  通过软链接查看内容，它与原始文件内容一致。
    ```bash
    cat abc.link
    ```

4.  删除原始文件，再次通过软链接访问，系统会提示“没有那个文件或目录”，链接已失效。
    ```bash
    rm aa.txt
    cat abc.link
    ```

## 硬链接

了解了像“快捷方式”一样的软链接后，我们再来看看Linux中另一种独特的链接方式——硬链接。

硬链接可以理解为指向原始文件数据块的**直接指针**。系统会创建一个与原始文件完全相同的 **inode** 信息块，这个inode直接指向硬盘上的数据块。因此，删除原始文件并不会影响硬链接对数据的访问。可以将inode的引用计数视为指针数量，只要计数不为零，数据就依然可以通过任何链接访问。

其工作原理与软链接不同：
*   **原始文件**、**硬链接1**、**硬链接2** 都拥有相同的 **inode**，并**直接指向硬盘数据**。
*   **操作者**可以通过其中任何一个入口访问数据。

以下是创建和验证硬链接的步骤：

1.  创建一个新文件。
    ```bash
    echo "这是硬链接测试内容" > hard.txt
    cat hard.txt
    ```

2.  使用 `ln` 命令（不加 `-s` 选项）创建硬链接。默认创建的就是硬链接。
    ```bash
    ln hard.txt hh.link
    ln hard.txt hh2.link
    ```

3.  分别查看原始文件和硬链接，它们内容相同。
    ```bash
    cat hh.link
    cat hh2.link
    ```

4.  使用 `ls -l` 可以查看文件的链接数（即inode引用计数）。初始的 `hard.txt` 链接数为3。
    ```bash
    ls -l hard.txt
    ```

5.  删除原始文件 `hard.txt`，通过硬链接 `hh.link` 和 `hh2.link` 依然可以访问原始数据。
    ```bash
    rm hard.txt
    cat hh.link
    cat hh2.link
    ```

6.  继续删除 `hh2.link`，此时查看 `hh.link` 的链接数已变为1。只要链接数不为0，数据就依然存在。
    ```bash
    rm hh2.link
    ls -l hh.link
    cat hh.link
    ```

7.  最后删除 `hh.link`，当链接数降为0时，系统才会真正释放该数据块占用的空间。
    ```bash
    rm hh.link
    ```

## 核心区别总结

本节课中我们一起学习了Linux的软链接和硬链接。为了更清晰地对比，以下是它们的主要区别：

![](img/bdfc6a7c53ae354ef94edca4f61d33ed_1.png)

*   **本质不同**：软链接是**存储路径**的独立文件；硬链接是共享**相同inode和数据块**的文件别名。
*   **inode**：软链接有自己的inode；硬链接与原始文件inode相同。
*   **跨文件系统**：软链接支持；硬链接不支持。
*   **链接目标**：软链接可链接目录；硬链接通常只能链接文件。
*   **原始文件删除后**：软链接失效（悬空链接）；硬链接不受影响。
*   **命令区别**：创建软链接用 `ln -s`；创建硬链接用 `ln`。

![](img/bdfc6a7c53ae354ef94edca4f61d33ed_2.png)

理解并熟练运用这两种链接，能够让你更高效、灵活地管理Linux文件系统。
