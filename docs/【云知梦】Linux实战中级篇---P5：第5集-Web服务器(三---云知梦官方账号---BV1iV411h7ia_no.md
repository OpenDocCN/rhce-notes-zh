## 静态网页与动态网页

上一节我们介绍了Web服务器的基本配置，本节中我们来看看更主流的动态网站技术。

![](img/4e24657a2ca6951725eb363a489759ba_5.png)

![](img/4e24657a2ca6951725eb363a489759ba_6.png)

![](img/4e24657a2ca6951725eb363a489759ba_7.png)

### 静态网页
静态网页是指网页内容固定不变的页面。它通常使用HTML格式编写，页面中直接包含文字、图片、音频、视频等具体内容。

**特点：**
*   **优点**：制作简单，访问时对服务器资源消耗小。
*   **缺点**：后期维护困难。例如，一个包含1万页的公司网站，若公司电话变更，则需要手动修改每一页的内容。

### 动态网页
动态网页的内容不是固定的，而是根据数据库中的数据动态生成的。网页中需要变化的部分（如电话号码、新闻内容、用户信息）会被替换成一个变量，这个变量与数据库中的某个位置关联。

**工作原理：**
当用户访问网页时，服务器会执行程序（如PHP），从数据库（如MySQL）中读取相应的数据，填充到网页的变量位置，再将完整的页面发送给用户的浏览器。

![](img/4e24657a2ca6951725eb363a489759ba_9.png)

**核心优势：**
后期维护极其方便。同样以修改公司电话为例，管理员只需在数据库中修改一次数据，所有关联该数据的网页在访问时都会自动显示更新后的内容。

![](img/4e24657a2ca6951725eb363a489759ba_11.png)

![](img/4e24657a2ca6951725eb363a489759ba_12.png)

**主流技术组合：**
*   **小型网站**：ASP + Access
*   **中大型网站（主流）**：PHP + MySQL
*   **其他技术**：JSP (Java)、.NET等

## 什么是LAMP

![](img/4e24657a2ca6951725eb363a489759ba_20.png)

![](img/4e24657a2ca6951725eb363a489759ba_22.png)

![](img/4e24657a2ca6951725eb363a489759ba_23.png)

LAMP是一套用于搭建动态网站的开源软件组合，其名称来源于四个组件的首字母。

![](img/4e24657a2ca6951725eb363a489759ba_25.png)

![](img/4e24657a2ca6951725eb363a489759ba_26.png)

**LAMP 结构解析：**
*   **L**：代表 **Linux** 操作系统，是整个环境的基础平台。
*   **A**：代表 **Apache** HTTP服务器，负责处理网页请求和响应。
*   **M**：代表 **MySQL** 或 **MariaDB** 数据库，用于存储网站的动态数据。
*   **P**：代表 **PHP** 编程语言，用于编写处理业务逻辑和连接数据库的网页程序。

![](img/4e24657a2ca6951725eb363a489759ba_28.png)

![](img/4e24657a2ca6951725eb363a489759ba_29.png)

这套组合因其性能、稳定性和开源免费的特性，在生产环境中被广泛使用。与之对应的还有 **LNMP**，即用 Nginx 替换了 Apache。

## 实验：搭建LAMP环境并部署Discuz论坛

![](img/4e24657a2ca6951725eb363a489759ba_44.png)

![](img/4e24657a2ca6951725eb363a489759ba_45.png)

接下来，我们将动手搭建一个完整的LAMP环境，并部署一个开源的Discuz论坛程序。

### 第一步：安装必要的软件包
我们需要在Linux系统上安装Apache、MariaDB（MySQL的替代品）和PHP。

执行以下命令进行安装：
```bash
yum install -y httpd mariadb-server mariadb php php-mysql
```

### 第二步：准备网站程序
动态网站需要对应的程序源码。我们可以从互联网上获取大量开源的PHP程序。

![](img/4e24657a2ca6951725eb363a489759ba_47.png)

以下是获取源码的步骤：
1.  从Discuz官网或其他源码站点下载论坛程序压缩包（如 `Discuz_X3.2_SC_UTF8.zip`）。
2.  将压缩包上传到Linux服务器。
3.  解压程序包。我们创建一个专用目录并解压到该目录下。
```bash
mkdir /discuz
unzip /root/Desktop/Discuz_X3.2_SC_UTF8.zip -d /discuz
```
4.  程序包中 `upload` 文件夹内的文件是主站程序。将其复制到Web服务器的默认目录。
```bash
cp -rf /discuz/upload/* /var/www/html/
```

### 第三步：配置目录权限
Apache服务进程默认以 `apache` 用户身份运行。为了让它能够读写网站目录下的文件（例如安装程序需要创建配置文件），我们必须修改目录的所有者。

执行以下命令更改权限：
```bash
chown -R apache:apache /var/www/html/*
```
**注意**：如果跳过此步骤，在后续安装向导中会提示目录“不可写”。

### 第四步：启动服务并配置数据库
1.  重启Apache服务，使其加载PHP模块的配置。
    ```bash
    systemctl restart httpd
    ```
2.  启动MariaDB数据库服务，并为数据库的root用户设置密码。
    ```bash
    systemctl start mariadb
    mysqladmin -u root password "123456" # 设置数据库root密码为123456
    systemctl restart mariadb # 重启数据库服务使密码生效
    ```

### 第五步：通过Web界面安装网站
1.  打开浏览器，访问服务器的IP地址（如 `http://192.168.100.1`）。
2.  浏览器会自动跳转到Discuz的安装向导页面。
3.  按照向导提示进行操作：
    *   同意许可协议。
    *   检查环境，确认目录权限正常（若第三步已做，此处应全为绿色“可写”）。
    *   **填写数据库信息**：
        *   数据库服务器：`localhost` (如果数据库在本机)
        *   数据库名：保持默认或新建一个
        *   数据库用户名：`root`
        *   数据库密码：`123456` (即第四步设置的密码)
        *   数据表前缀：可以修改（如改为 `pre_`），用于在同一数据库中区分不同网站的数据表。
    *   **设置管理员账号**：
        *   输入论坛后台的管理员用户名和密码（如 `admin` / `admin`）。
4.  点击下一步，系统会自动创建数据库表并完成安装。
5.  安装完成后，即可访问论坛首页，并使用管理员账号登录后台进行版块设置、用户管理等操作。

### 第六步：验证网站可访问性
从网络内的其他计算机访问服务器的IP地址，应该同样能够看到部署好的论坛网站，这证明LAMP环境搭建成功且网站服务已正常发布。

## 总结
本节课我们一起学习了动态网站与静态网站的核心区别，并详细实践了LAMP环境的搭建流程。我们掌握了从安装Apache、MariaDB、PHP软件包，到配置目录权限、初始化数据库，最后通过Web界面部署一个完整PHP论坛的全过程。这套流程是部署绝大多数PHP开源程序（如WordPress、DedeCMS等）的基础，理解并掌握它对于Linux运维和Web服务管理至关重要。
