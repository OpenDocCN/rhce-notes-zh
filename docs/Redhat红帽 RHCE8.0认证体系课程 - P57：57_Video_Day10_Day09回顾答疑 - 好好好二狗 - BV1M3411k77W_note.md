### Ansible配置文件优先级 📊
上一节我们介绍了Ansible的基础架构，本节中我们来看看配置文件的优先级问题，这是许多同学容易出错的地方。

Ansible配置文件的优先级从高到低依次为：
1.  当前工作目录下的 `ansible.cfg` 文件。
2.  用户家目录下的 `.ansible.cfg` 文件。
3.  系统默认的 `/etc/ansible/ansible.cfg` 文件。

![](img/6db4cc8249d8f2f53715015bd1859458_13.png)

![](img/6db4cc8249d8f2f53715015bd1859458_15.png)

**核心概念**：优先级最高的配置文件生效范围最小，影响也最小。在生产或考试环境中，建议在工作目录下使用自定义的 `ansible.cfg`，以实现对特定项目的最小化控制。

![](img/6db4cc8249d8f2f53715015bd1859458_17.png)

**常见错误**：未切换到正确的工作目录，导致使用了错误的配置文件，使得后续所有操作基于错误的资产清单或配置执行，最终任务失败。

![](img/6db4cc8249d8f2f53715015bd1859458_19.png)

**解决方案**：在执行任何Ansible命令前，务必使用 `pwd` 命令确认当前工作目录，并使用 `ansible --version` 命令检查当前生效的配置文件路径。

### 资产清单与连接配置 🔗
以下是关于资产清单和免密认证的要点总结。

*   **资产清单定义**：清单文件定义了受管主机。可以按单条记录、分组或嵌套分组（使用 `:children`）的方式组织。
*   **主机名解析**：在实验环境中，可以通过 `/etc/hosts` 文件进行主机名解析。
*   **免密认证**：这是控制节点与受管主机建立SSH信任关系的关键步骤。在考试或特定练习环境中，可能已预先配置好，但自行部署时必须完成此步骤。
*   **配置示例**：一个基础的 `ansible.cfg` 配置文件应包含资产清单路径和权限提升设置。

**权限提升配置示例**：
```ini
[privilege_escalation]
become=True
become_method=sudo
become_user=root
become_ask_pass=False
```
建议使用 `sudo` 方式，而非 `su`。

### Ansible临时命令格式与模块回顾 ⚙️
我们回顾了Ansible临时命令的基本格式和两个核心模块。

**临时命令通用格式**：
```bash
ansible <主机或主机组> [-i 清单文件] -m <模块名> -a “<模块参数>”
```
如果使用 `shell` 或 `command` 模块，`-m` 参数可以省略。

**核心特性**：幂等性。即同一命令多次执行的结果与执行一次一致。

![](img/6db4cc8249d8f2f53715015bd1859458_39.png)

**模块使用建议**：不熟悉模块参数时，善用 `ansible-doc <模块名>` 命令查看官方文档和示例。

![](img/6db4cc8249d8f2f53715015bd1859458_41.png)

#### 文件模块关键点
以下是文件操作相关模块的总结。

![](img/6db4cc8249d8f2f53715015bd1859458_45.png)

*   **copy模块**：不仅可以从控制节点复制文件到受管主机，还可以通过 `content` 参数直接在受管主机上创建文件并写入内容。
*   **file模块**：用于管理文件和目录的属性，如创建（`state=directory` 或 `state=touch`）、设置权限、创建软链接等。**创建软链接时，注意 `src`（源）和 `dest`（目标）参数不要写反**。
*   **fetch模块**：用于将受管主机上的文件拉取到控制节点。
*   **lineinfile模块**：确保文件中某一行内容存在（或不存在），常用于修改配置文件。可以通过正则表达式（`regexp`）定位要操作的行。

![](img/6db4cc8249d8f2f53715015bd1859458_47.png)

![](img/6db4cc8249d8f2f53715015bd1859458_49.png)

#### 软件包模块关键点
以下是软件包管理相关模块的总结。

*   **yum_repository模块**：管理Yum仓库。可以在一个任务中配置多个仓库（在YAML中按列表格式书写）。
*   **yum模块**：管理软件包。核心参数包括 `name`（包名）、`state`（状态，如 `present` 安装、`latest` 更新、`absent` 删除）。

**学习技巧**：直接参考 `ansible-doc` 给出的 **EXAMPLES** 部分，可以快速获得正确的任务写法，避免因手动输入错误导致失败。
