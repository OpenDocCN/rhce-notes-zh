# Linux运维教程：P27：逻辑卷扩容与RAID磁盘阵列

![](img/b0b9854daa070f985ce52bbd3befde38_1.png)

在本节课中，我们将学习逻辑卷的扩容操作，并初步了解RAID磁盘阵列的基本概念。逻辑卷管理（LVM）的核心优势在于其灵活的扩容能力，而RAID技术则用于提升数据的安全性和磁盘性能。我们将从实践出发，掌握如何为已满的逻辑卷增加空间，并理解RAID在企业环境中的作用。

## 逻辑卷扩容操作

![](img/b0b9854daa070f985ce52bbd3befde38_3.png)

上一节我们介绍了逻辑卷的创建。本节中我们来看看当逻辑卷空间不足时，如何进行扩容。其核心思想在于，逻辑卷允许我们根据数据增长的需求，动态地增加存储空间，而无需在初始时就分配过大的容量。

![](img/b0b9854daa070f985ce52bbd3befde38_5.png)

![](img/b0b9854daa070f985ce52bbd3befde38_7.png)

### 检查现有空间

![](img/b0b9854daa070f985ce52bbd3befde38_9.png)

![](img/b0b9854daa070f985ce52bbd3befde38_11.png)

![](img/b0b9854daa070f985ce52bbd3befde38_13.png)

在对逻辑卷扩容前，必须首先确认其所属的卷组（VG）是否有足够的剩余空间。以下是检查命令：

![](img/b0b9854daa070f985ce52bbd3befde38_15.png)

![](img/b0b9854daa070f985ce52bbd3befde38_17.png)

```bash
# 查看卷组剩余空间
vgs
# 查看逻辑卷当前大小
lvs
```

### 扩展逻辑卷容量

确认卷组有空间后，即可使用 `lvextend` 命令扩展逻辑卷。此操作支持在线进行，无需重启服务器。

命令格式为：`lvextend -L +[要增加的大小] [逻辑卷设备路径]`

例如，为逻辑卷 `/dev/system_vg/my_lv` 增加19GB空间：

```bash
lvextend -L +19G /dev/system_vg/my_lv
```
命令执行后，逻辑卷的底层存储空间即被扩大。

![](img/b0b9854daa070f985ce52bbd3befde38_19.png)

### 扩展文件系统

扩容逻辑卷后，**文件系统**并不会自动增大以使用新增的空间。这是因为最初格式化时创建的文件系统只管理原始大小的区域。新增的空间没有文件系统，因此操作系统无法识别和使用。

![](img/b0b9854daa070f985ce52bbd3befde38_21.png)

对于XFS文件系统，使用 `xfs_growfs` 命令扩展其容量；对于ext2/3/4文件系统，则使用 `resize2fs` 命令。

![](img/b0b9854daa070f985ce52bbd3befde38_23.png)

由于我们的逻辑卷格式化为XFS，因此执行：

![](img/b0b9854daa070f985ce52bbd3befde38_25.png)

```bash
# 指定挂载点扩展文件系统
xfs_growfs /web_backup
# 或直接指定逻辑卷设备
# xfs_growfs /dev/system_vg/my_lv
```
执行此命令后，使用 `df -h` 查看，逻辑卷的可用空间才会显示为扩容后的总大小。

### 再次扩容流程

![](img/b0b9854daa070f985ce52bbd3befde38_27.png)

![](img/b0b9854daa070f985ce52bbd3befde38_29.png)

如果空间再次不足，可以重复上述流程：
1.  检查卷组剩余空间 (`vgs`)。
2.  若卷组空间不足，则需要先**扩展卷组**。
3.  扩展卷组后，再扩展逻辑卷和文件系统。

扩展卷组的命令是 `vgextend`，可以将新的物理卷（PV，如硬盘或分区）加入现有卷组。

![](img/b0b9854daa070f985ce52bbd3befde38_31.png)

命令格式：`vgextend [卷组名] [新设备路径]`

![](img/b0b9854daa070f985ce52bbd3befde38_33.png)

![](img/b0b9854daa070f985ce52bbd3befde38_35.png)

![](img/b0b9854daa070f985ce52bbd3befde38_37.png)

例如，将新硬盘 `/dev/sdd` 加入 `system_vg` 卷组：

![](img/b0b9854daa070f985ce52bbd3befde38_39.png)

```bash
vgextend system_vg /dev/sdd
```
卷组扩容后，即可继续对逻辑卷进行扩容操作。

## 逻辑卷缩容与删除

虽然逻辑卷支持缩容，但此操作风险极高，可能导致数据丢失，且需要卸载文件系统后才能进行，因此**企业环境中极少使用**。通常我们只进行扩容操作。

如果需要删除不再使用的逻辑卷，必须遵循正确的顺序，且操作前务必**卸载文件系统并备份数据**。

![](img/b0b9854daa070f985ce52bbd3befde38_41.png)

以下是操作步骤：

![](img/b0b9854daa070f985ce52bbd3befde38_43.png)

![](img/b0b9854daa070f985ce52bbd3befde38_45.png)

1.  **卸载文件系统**：`umount /web_backup`
2.  **删除逻辑卷**：`lvremove /dev/system_vg/my_lv`
3.  **删除卷组**：`vgremove system_vg`
4.  **删除物理卷（可选）**：`pvremove /dev/sdb1` （如需将磁盘恢复为普通分区）

![](img/b0b9854daa070f985ce52bbd3befde38_47.png)

删除后，原有的磁盘或分区可重新格式化为普通文件系统使用。

## RAID磁盘阵列简介

在了解了逻辑卷的灵活管理后，我们来看另一个提升存储可靠性和性能的技术——RAID。RAID（独立磁盘冗余阵列）通过将多块物理磁盘组合起来，形成一个逻辑上的大容量、高可靠或高性能的虚拟磁盘。

### RAID的核心作用

![](img/b0b9854daa070f985ce52bbd3befde38_49.png)

RAID的主要目的是：
*   **提高数据可靠性（冗余）**：即使某一块硬盘损坏，数据也不会丢失。
*   **提升读写性能**：通过并行读写多块磁盘，加快数据访问速度。
*   **增加存储容量**：将多块小容量磁盘合并为一个大容量逻辑盘。

![](img/b0b9854daa070f985ce52bbd3befde38_51.png)

![](img/b0b9854daa070f985ce52bbd3befde38_53.png)

![](img/b0b9854daa070f985ce52bbd3befde38_55.png)

### 常见RAID级别

![](img/b0b9854daa070f985ce52bbd3befde38_57.png)

![](img/b0b9854daa070f985ce52bbd3befde38_58.png)

以下是几种常见的RAID级别及其特点：

![](img/b0b9854daa070f985ce52bbd3befde38_60.png)

*   **RAID 0（条带化）**：
    *   **原理**：将数据拆分后，并行写入多块磁盘。
    *   **优点**：读写速度最快，容量利用率100%（总容量为所有磁盘之和）。
    *   **缺点**：无冗余，任何一块磁盘损坏将导致所有数据丢失。
    *   **适用场景**：对性能要求极高，但对数据安全要求不高的场景，如视频缓存。

![](img/b0b9854daa070f985ce52bbd3befde38_62.png)

![](img/b0b9854daa070f985ce52bbd3befde38_64.png)

![](img/b0b9854daa070f985ce52bbd3befde38_66.png)

*   **RAID 1（镜像）**：
    *   **原理**：将相同的数据同时写入两块磁盘，实现完全备份。
    *   **优点**：数据安全性高，读取速度有所提升。
    *   **缺点**：容量利用率只有50%（总容量为单块磁盘容量），写入速度无提升。
    *   **适用场景**：对数据安全要求极高的场景，如操作系统盘、数据库日志。

*   **RAID 5（带奇偶校验的条带化）**：
    *   **原理**：数据与校验信息（奇偶校验）交叉存储在至少三块磁盘上。
    *   **优点**：兼顾性能、容量和安全性。允许损坏一块磁盘而不丢失数据。
    *   **缺点**：写入速度受校验计算影响。重建数据时系统压力大。
    *   **适用场景**：通用文件服务器、应用服务器。

*   **RAID 10（先镜像再条带化）**：
    *   **原理**：先做RAID 1镜像对，再将多个镜像对组成RAID 0。
    *   **优点**：兼具RAID 1的安全性和RAID 0的高性能。
    *   **缺点**：成本高，容量利用率只有50%。
    *   **适用场景**：对性能和安全性都有极高要求的场景，如数据库服务器。

### RAID与LVM的结合

在实际生产环境中，RAID和LVM常结合使用以发挥最大优势：
1.  使用多块物理硬盘组建RAID（如RAID 5或RAID 10），得到一个可靠且高性能的“大硬盘”（RAID阵列）。
2.  将这个RAID阵列作为物理卷（PV）加入LVM的卷组（VG）。
3.  在卷组上创建灵活的逻辑卷（LV），并享受LVM带来的**在线扩容、快照**等便利。

这种架构既通过RAID保证了底层硬件的可靠性，又通过LVM实现了上层存储资源的灵活管理。
