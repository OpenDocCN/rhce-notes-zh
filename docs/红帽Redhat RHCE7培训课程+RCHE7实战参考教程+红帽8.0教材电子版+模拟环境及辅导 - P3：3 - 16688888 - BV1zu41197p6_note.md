### 帮助系统概览
在Linux中，有多种方式可以获取命令的帮助信息，最常用的是`--help`选项和`man`命令。

### 使用`--help`选项
大多数命令都支持`--help`选项，它可以快速显示命令的基本用法和选项。

**示例：**
```bash
ls --help
```

### 使用`man`手册
`man`命令是查看命令详细手册页的标准工具。手册页分为多个章节，每个章节代表不同类型的文档。

**man手册章节：**
*   **1**：用户命令（可执行程序或shell命令）
*   **5**：文件格式（配置文件的结构说明）

**常用`man`命令操作：**
*   `man passwd`：查看`passwd`命令的帮助（默认从章节1开始查找）。
*   `man 5 passwd`：查看`/etc/passwd`文件格式的说明。
*   `man -k keyword`：在所有手册页中搜索包含`keyword`的内容。
*   `man man`：查看`man`命令自身的帮助。

**在`man`手册中导航：**
*   向下翻页：`空格键` 或 `Page Down`
*   向上翻页：`Page Up`
*   查找：`/`后跟搜索词，按`n`查找下一个，`N`查找上一个。
*   退出：`q`

### 其他帮助资源
*   `whereis`：定位命令的二进制文件、源代码和手册页位置。
*   `/usr/share/doc`：存放已安装软件包的详细文档。
*   红帽知识库：访问 `access.redhat.com` 获取官方文档和支持。
*   `sosreport`：收集系统配置和诊断信息，用于向红帽支持提交问题报告。

**示例：生成诊断报告**
```bash
su - root
sosreport
```

## 第四章：输入输出重定向与管道

![](img/cdb487a3e96a2862241b0af5ed2b41ae_14.png)

![](img/cdb487a3e96a2862241b0af5ed2b41ae_15.png)

上一节我们介绍了如何获取系统帮助，本节中我们来看看如何控制命令的输入和输出。

### 标准输入、输出和错误
在Linux中，每个命令运行时都会打开三个标准的“文件”：
*   **标准输入 (stdin)**：默认来自键盘，文件描述符为 **0**。
*   **标准输出 (stdout)**：默认输出到显示器，文件描述符为 **1**。
*   **标准错误 (stderr)**：默认也输出到显示器，文件描述符为 **2**。

![](img/cdb487a3e96a2862241b0af5ed2b41ae_17.png)

### 输出重定向
我们可以改变命令输出的目的地，将其从屏幕重定向到文件或其他设备。

![](img/cdb487a3e96a2862241b0af5ed2b41ae_19.png)

![](img/cdb487a3e96a2862241b0af5ed2b41ae_21.png)

**以下是输出重定向的常用符号：**

*   `>`：将**标准输出**重定向到文件，**覆盖**文件原有内容。
    ```bash
    echo "Hello" > file.txt
    ```
*   `>>`：将**标准输出**重定向到文件，**追加**到文件末尾。
    ```bash
    echo "World" >> file.txt
    ```
*   `2>`：将**标准错误**重定向到文件，覆盖。
    ```bash
    ls /nonexistent 2> error.log
    ```
*   `&>`：将**标准输出和标准错误**都重定向到同一个文件。
    ```bash
    command &> output.log
    ```
*   `> /dev/null`：将输出丢弃到空设备（不保存）。`/dev/null`是一个特殊的设备文件，写入它的任何数据都会被丢弃。
    ```bash
    find / -name *.iso 2> /dev/null
    ```

### 输入重定向
我们可以改变命令输入的来源，使其从文件读取而不是键盘。

**以下是输入重定向的符号：**

*   `<`：将文件内容作为命令的**标准输入**。
    ```bash
    mail -s "Subject" student < message.txt
    ```
*   `<<`：称为“Here Document”，用于在命令行中直接输入多行文本，直到遇到指定的终止符。
    ```bash
    cat > newfile.txt << EOF
    This is line 1.
    This is line 2.
    EOF
    ```

### 管道
管道符 `|` 用于将一个命令的**标准输出**作为另一个命令的**标准输入**。这允许我们组合多个简单命令来完成复杂任务。

![](img/cdb487a3e96a2862241b0af5ed2b41ae_23.png)

![](img/cdb487a3e96a2862241b0af5ed2b41ae_25.png)

**管道应用示例：**

![](img/cdb487a3e96a2862241b0af5ed2b41ae_27.png)

![](img/cdb487a3e96a2862241b0af5ed2b41ae_28.png)

1.  **过滤输出**：使用`grep`搜索包含特定文本的行。
    ```bash
    man nmcli | grep team
    ```
2.  **组合命令**：连续使用多个管道进行复杂处理。
    ```bash
    man nmcli | grep -A2 team
    ```
3.  **设置密码**：通过管道将密码传递给`passwd`命令。
    ```bash
    echo "MyPassword" | passwd --stdin username
    ```
4.  **分页查看**：使用`less`或`more`分页查看长输出。
    ```bash
    ls -l /etc | less
    ```
5.  **计数**：使用`wc`统计行数、单词数等。
    ```bash
    ls -l | wc -l
    ```
6.  **查看文件首尾**：使用`head`和`tail`查看文件的开头或结尾部分。
    ```bash
    head -5 /etc/passwd  # 查看前5行
    tail -3 /var/log/messages # 查看最后3行
    ```
7.  **三通命令`tee`**：在管道中，`tee`命令将数据同时输出到屏幕（或下一个管道）和文件，常用于调试和保存中间结果。
    ```bash
    ls -l | tee filelist.txt | wc -l
    ```

## 第五章：文本编辑器VI/VIM

上一节我们掌握了数据流的控制方法，本节中我们来看看Linux中最强大也最常用的文本编辑器——VI/VIM。

### VI编辑器简介
VI是Linux系统自带的文本编辑器，功能强大，完全通过键盘操作。VIM是VI的增强版本。

![](img/cdb487a3e96a2862241b0af5ed2b41ae_35.png)

![](img/cdb487a3e96a2862241b0af5ed2b41ae_37.png)

### VI的三种模式
VI编辑器有三种主要工作模式，理解它们是熟练使用VI的关键：

1.  **命令模式**：启动VI后的默认模式。在此模式下，键盘输入被视为命令（如移动光标、删除、复制等），而不是文本。按`ESC`键可确保进入此模式。
2.  **插入模式**：在此模式下，可以像普通编辑器一样输入和编辑文本。从命令模式按 `i`, `a`, `o` 等键进入。
3.  **末行模式**：在命令模式下按 `:` 进入。可以执行保存、退出、查找替换等操作。

**VI基本使用流程（必须掌握的三步）：**
1.  `vi filename`：打开/创建文件，进入**命令模式**。
2.  按 `i`：进入**插入模式**，开始编辑文本。
3.  按 `ESC`：返回**命令模式**。
4.  按 `:wq` 或 `:x`：进入**末行模式**，执行保存并退出。

![](img/cdb487a3e96a2862241b0af5ed2b41ae_39.png)

### 常用命令

**模式切换：**
*   `i`：在光标前插入。
*   `a`：在光标后插入。
*   `o`：在当前行下方新建一行并插入。
*   `I`：在行首插入。
*   `A`：在行尾插入。
*   `O`：在当前行上方新建一行并插入。
*   `ESC`：从插入模式返回命令模式。
*   `:`：从命令模式进入末行模式。

**光标移动（命令模式下）：**
*   `h`：左
*   `j`：下
*   `k`：上
*   `l`：右
*   `0` 或 `^`：移动到行首
*   `$`：移动到行尾
*   `gg`：移动到文件第一行
*   `G`：移动到文件最后一行
*   `:n`：移动到第n行（例如 `:10` 跳到第10行）

**编辑操作（命令模式下）：**
*   `x`：删除光标处的字符。
*   `dd`：删除（剪切）当前行。
*   `yy`：复制当前行。
*   `p`：在光标后粘贴。
*   `P`：在光标前粘贴。
*   `u`：撤销上一次操作。
*   `Ctrl + r`：重做被撤销的操作。

**末行模式命令：**
*   `:w`：保存文件。
*   `:q`：退出VI（如果文件未修改）。
*   `:q!`：强制退出，不保存修改。
*   `:wq` 或 `:x`：保存并退出。
*   `:set number` 或 `:set nu`：显示行号。
*   `:set nonumber` 或 `:set nonu`：隐藏行号。
*   `/pattern`：向前搜索`pattern`。
*   `?pattern`：向后搜索`pattern`。

### 图形界面编辑器
对于习惯图形界面的用户，可以使用`gedit`，它的操作方式与Windows记事本类似。

**启动gedit：**
```bash
gedit filename &
```
