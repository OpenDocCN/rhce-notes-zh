Linux入门教程：63：灵魂拷问三：对文件有所有权限就能删除吗？

在本节课中，我们将探讨一个关于Linux文件权限的常见误解：即使对一个文件拥有全部权限（读、写、执行），是否就一定能删除它？我们将通过实际操作来揭示背后的核心规则。

![](img/cd8b7f79d243fb88325e3af42363e013_1.png)

![](img/cd8b7f79d243fb88325e3af42363e013_2.png)

上一节我们介绍了文件的基本权限，本节中我们来看看删除文件这一特殊操作所依赖的条件。

---

### 场景准备

首先，我们需要创建一个实验环境。在 `/home` 目录下创建一个名为 `public` 的目录，并在其中放置一个文件。

![](img/cd8b7f79d243fb88325e3af42363e013_4.png)

![](img/cd8b7f79d243fb88325e3af42363e013_5.png)

以下是操作步骤：

![](img/cd8b7f79d243fb88325e3af42363e013_6.png)

![](img/cd8b7f79d243fb88325e3af42363e013_7.png)

1.  创建目录：
    ```bash
    mkdir /home/public
    ```
2.  复制一个文件到该目录（例如 `/etc/hosts`）：
    ```bash
    cp /etc/hosts /home/public/
    ```
3.  赋予这个文件高权限，确保所有用户都能访问：
    ```bash
    chmod 777 /home/public/hosts
    ```

![](img/cd8b7f79d243fb88325e3af42363e013_8.png)

现在，任何普通用户（例如 `tianyun` 或 `alice`）都应该能对这个文件进行读、写、执行操作。

![](img/cd8b7f79d243fb88325e3af42363e013_9.png)

![](img/cd8b7f79d243fb88325e3af42363e013_11.png)

---

### 权限测试：尝试删除文件

![](img/cd8b7f79d243fb88325e3af42363e013_13.png)

我们以普通用户 `tianyun` 的身份进行操作。

![](img/cd8b7f79d243fb88325e3af42363e013_14.png)

![](img/cd8b7f79d243fb88325e3af42363e013_15.png)

1.  进入目录：
    ```bash
    cd /home/public
    ```
2.  验证对文件的权限：
    *   **读取**：使用 `cat hosts` 查看文件内容，可以成功。
    *   **修改**：使用 `vim` 或 `echo` 命令修改文件内容，可以成功。
    *   **执行**：虽然此文件不是脚本，但理论上拥有执行权限。

然而，当我们尝试删除这个文件时：
```bash
rm hosts
```
操作会失败，系统提示“权限不足”。

![](img/cd8b7f79d243fb88325e3af42363e013_17.png)

**结论**：即使对文件本身拥有全部权限，也不一定能删除它。

![](img/cd8b7f79d243fb88325e3af42363e013_18.png)

---

### 核心原理：删除权限的归属

![](img/cd8b7f79d243fb88325e3af42363e013_20.png)

![](img/cd8b7f79d243fb88325e3af42363e013_21.png)

为什么不能删除？关键在于：**在Linux中，删除一个文件的权限并不取决于文件本身的权限，而是取决于其所在目录的权限。**

![](img/cd8b7f79d243fb88325e3af42363e013_22.png)

具体来说：
*   要对一个目录内的文件进行**创建**或**删除**操作，用户必须对该目录拥有**写（w）权限**。
*   文件自身的权限只影响对文件内容的操作（读、写、执行），不影响其“存在与否”。

在当前例子中，用户 `tianyun` 对 `/home/public/hosts` 文件有全部权限，但对它的父目录 `/home/public` 没有写权限。因此，他无法删除目录内的任何文件。

我们可以用以下命令验证目录权限：
```bash
ls -ld /home/public
```
如果输出中用户权限部分没有 `w`，则证实了这一点。

![](img/cd8b7f79d243fb88325e3af42363e013_24.png)

### 反向验证：目录权限决定一切

![](img/cd8b7f79d243fb88325e3af42363e013_27.png)

![](img/cd8b7f79d243fb88325e3af42363e013_28.png)

为了更深刻地理解这个规则，我们进行一个反向实验。

![](img/cd8b7f79d243fb88325e3af42363e013_29.png)

1.  首先，确保用户对目录有写权限：
    ```bash
    sudo chmod 777 /home/public
    ```
    现在，`tianyun` 用户对 `/home/public` 目录拥有了读、写、执行权限。
2.  此时，用户可以在目录内自由创建和删除文件，无论目标文件的权限如何。
3.  我们甚至可以做极端测试：将一个文件的权限设置为 `000`（即无任何权限）。
    ```bash
    chmod 000 /home/public/hosts
    ```
4.  此时，用户 `tianyun` 无法读取或修改此文件内容。但是，由于他对目录有写权限，他**仍然可以删除这个文件**。
    ```bash
    rm hosts  # 操作成功
    ```

![](img/cd8b7f79d243fb88325e3af42363e013_30.png)

这个实验强有力地证明：**文件的“生杀大权”掌握在其父目录手中。**

### 总结

![](img/cd8b7f79d243fb88325e3af42363e013_32.png)

![](img/cd8b7f79d243fb88325e3af42363e013_33.png)

本节课我们一起学习了Linux中关于文件删除的重要权限规则：

![](img/cd8b7f79d243fb88325e3af42363e013_34.png)

![](img/cd8b7f79d243fb88325e3af42363e013_35.png)

1.  **核心概念**：删除文件的权限由**其所在目录的写权限**决定，与**文件自身的权限无关**。
2.  **关键区别**：
    *   文件权限（`rwx`）：控制对文件**内容**的操作。
    *   目录的写权限（`w`）：控制对目录**条目**的修改，即创建或删除其中的文件。
3.  **实用技巧**：当遇到无法删除文件的情况时，不应只检查文件权限，更应检查其父目录的权限。

![](img/cd8b7f79d243fb88325e3af42363e013_36.png)

理解这一区别，对于管理Linux文件系统和排查权限问题至关重要。
