# Linux运维100问：P11：K8S是运维必会的技能了吗？ 🚢

在本节课中，我们将探讨Kubernetes（K8S）是否已成为现代运维工程师的必备技能。我们将了解K8S的核心概念、主要应用领域，并分析其在不同业务场景下的重要性。

## 概述

Kubernetes（常简称为K8S）是一个开源的容器编排平台。如果大家希望向高级运维或自动化运维方向发展，掌握K8S技术是必要的。

我们可以将K8S理解为一个**自动化的运维框架**。例如，K8S可以帮助我们实现从代码提交、代码发布、版本更新、负载均衡、流量监控到最终监控告警等一系列功能的一体化实现。

## K8S的应用领域

上一节我们介绍了K8S作为自动化框架的概念，本节中我们来看看它的主要应用场景。K8S并非所有公司都会使用。目前，K8S主要的应用领域集中在微服务和虚拟化方面。

### 微服务领域

在微服务领域，K8S是常用的技术。微服务架构是指将一个庞大、整体的系统架构拆分成多个独立、小型服务的方法。这样拆分易于管理，避免了管理一个庞大单体架构的复杂性。

庞大架构的一个缺点是“牵一发而动全身”。而将其拆分为多个小服务后，就可以独立地管理这些小服务。通常在微服务领域，我们肯定会用到K8S。如果想详细了解微服务，可以参考相关的往期视频。

### 虚拟化领域

虚拟化领域也会用到K8S。例如，中国的阿里云云平台目前就是基于K8S实现的。

## 总结

本节课中，我们一起学习了Kubernetes（K8S）在现代运维中的角色。我们了解到K8S是一个强大的自动化运维框架，能整合从开发到监控的完整流程。它的核心应用在于支持微服务架构和云平台虚拟化，是迈向高级运维和自动化运维的关键技能之一。
