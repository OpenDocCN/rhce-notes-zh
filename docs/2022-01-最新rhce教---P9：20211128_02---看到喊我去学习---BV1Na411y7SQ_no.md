上一节我们介绍了管道的基本用法，本节中我们来看看`tee`命令。`tee`命令能够读取标准输入，并将其内容同时输出到标准输出（屏幕）和一个或多个文件中。它常与管道符 `|` 结合使用。

其基本命令格式如下：
```
命令1 | tee [选项] 文件名 | 命令2
```
执行时，`命令1`的输出结果不仅会传递给`命令2`继续处理，还会保存到指定的`文件名`中。

以下是`tee`命令的核心选项：
*   **-a**：以追加（Append）模式写入文件。如果不使用此选项，新内容会覆盖文件的原有内容。

### tee命令的应用实例

以下是`tee`命令的几个典型应用场景。

**1. 保存命令输出结果**
例如，我们可以计算一个表达式并将结果保存下来：
```
echo "2+3" | bc | tee bc_result.txt
```
这条命令会计算2+3，将结果`5`同时显示在屏幕上并保存到`bc_result.txt`文件中。

![](img/fcc30b9342976bdf9eda981e7c696f72_24.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_26.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_28.png)

**2. 调试复杂管道**
在复杂的管道命令中，`tee`可以帮助我们查看中间步骤的输出，便于排查问题。例如，查找特定进程并保存查找过程：
```
ps aux | tee process.log | grep sshd
```
这条命令将`ps aux`的完整进程列表保存到`process.log`文件，同时将结果传递给`grep`来筛选包含`sshd`的行。

![](img/fcc30b9342976bdf9eda981e7c696f72_30.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_31.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_32.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_33.png)

**3. 同时记录正确和错误输出**
使用`tee`可以捕获命令执行的所有输出（包括标准输出和标准错误），这在生产环境日志记录和故障排查中非常有用。
```
某个命令 2>&1 | tee all_output.log
```
`2>&1` 表示将标准错误（文件描述符2）重定向到标准输出（文件描述符1），这样`tee`就能将正确和错误信息都写入`all_output.log`。

## vi编辑器：入门与三种模式

![](img/fcc30b9342976bdf9eda981e7c696f72_40.png)

接下来，我们将开始学习`vi`编辑器。`vi`是Linux系统中最经典的文本编辑器，功能强大。在最小化安装的系统中可能默认不安装，但图形界面或服务器版通常会包含其增强版本`vim`。

![](img/fcc30b9342976bdf9eda981e7c696f72_42.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_44.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_46.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_47.png)

我们可以使用`rpm -q vim`命令来检查是否已安装。启动`vi`的基本命令是：
```
vi [选项] 文件名
```

![](img/fcc30b9342976bdf9eda981e7c696f72_49.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_50.png)

以下是`vi`的一些常用启动选项：
*   `-v`： 显示版本信息。
*   `+n`： 打开文件后，光标直接定位到第n行。例如 `vi +10 file.txt`。
*   `+/pattern`： 打开文件后，光标直接定位到第一个匹配`pattern`字符串的行首。
*   `-b`： 以二进制模式打开文件。
*   `-d file1 file2`： 比较多个文件，高亮显示差异。
*   `-R`： 以只读模式打开文件，无法直接保存修改。
*   `-y`： 简易模式，启动后直接进入插入模式。

如果打开的文件不存在，`vi`会在首次保存时创建它。

### vi编辑器的三种主要模式

![](img/fcc30b9342976bdf9eda981e7c696f72_57.png)

`vi`编辑器的强大源于其不同的操作模式，初学者需要理解它们之间的切换关系。

![](img/fcc30b9342976bdf9eda981e7c696f72_63.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_65.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_67.png)

**1. 普通模式**
这是启动`vi`后的默认模式。在此模式下，可以移动光标、复制、粘贴、删除文本，但不能直接输入字符。屏幕底部会显示文件名、总行数和当前光标位置等信息。

![](img/fcc30b9342976bdf9eda981e7c696f72_69.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_70.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_71.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_72.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_73.png)

![](img/fcc30b9342976bdf9eda981e7c696f72_74.png)

**2. 插入模式**
在此模式下，可以像普通编辑器一样输入和修改文本。从普通模式进入插入模式的常用键有：
*   `i`： 在光标当前位置前插入。
*   `I`（Shift+i）： 在当前行的行首插入。
*   `a`： 在光标当前位置后插入。
*   `A`（Shift+a）： 在当前行的行尾插入。
*   `o`： 在当前行的下方新建一行并进入插入模式。
*   `O`（Shift+o）： 在当前行的上方新建一行并进入插入模式。

![](img/fcc30b9342976bdf9eda981e7c696f72_75.png)

**3. 命令行模式（扩展模式）**
在普通模式下按冒号 `:` 进入。在此模式下可以执行保存、退出、搜索替换等高级命令。输入命令后按回车执行。
*   `:w`： 保存文件。
*   `:q`： 退出`vi`。
*   `:wq` 或 `:x`： 保存并退出。
*   `:q!`： 不保存强制退出。
*   `:set nu`： 显示行号。
*   `:%s/旧文本/新文本/g`： 将文件中所有的“旧文本”替换为“新文本”。

**模式切换总结：**
*   启动`vi` -> **普通模式**。
*   普通模式下按 `i， I, a, A, o, O` 等键 -> **插入模式**。
*   插入模式下按 `Esc` 键 -> **普通模式**。
*   普通模式下按 `:` 键 -> **命令行模式**。
*   命令行模式下执行命令后或按 `Esc` 键 -> **普通模式**。

### 实用技巧：可视化块模式

除了上述三种基本模式，`vi/vim`还有一个非常实用的**可视化块模式**。
*   在普通模式下按 `Ctrl + v` 进入。
*   此时可以使用方向键（或 `h， j, k, l`）高亮选择一个矩形文本区域。
*   选择后，按 `Shift + i` 进入插入模式，输入内容，然后按 `Esc`，之前选中的所有行都会在块前插入相同的内容。这个功能非常适合批量注释或修改多行代码。
