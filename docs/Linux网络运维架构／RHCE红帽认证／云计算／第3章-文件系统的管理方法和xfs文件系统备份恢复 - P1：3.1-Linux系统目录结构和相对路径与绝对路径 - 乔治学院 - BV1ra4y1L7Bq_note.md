## 目录结构概述

Linux系统采用一种树形的目录结构，与Windows系统使用盘符（如C盘、D盘）不同，Linux只有一个根目录（`/`），所有文件和目录都从根目录开始延伸。首先，我们可以使用`tree`命令来直观地查看目录结构。如果系统没有安装该命令，可以通过包管理器进行安装。

![](img/c729070ce7aa720641f917bf4a9e3d05_11.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_13.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_15.png)

**安装tree命令示例：**
```bash
yum install tree
# 或 CentOS 8 中使用
dnf install tree
```

![](img/c729070ce7aa720641f917bf4a9e3d05_17.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_19.png)

安装完成后，可以执行`tree /`来查看根目录下的完整结构。不过，我们更关注的是其中一些核心目录的作用。

## 核心目录详解

以下是Linux根目录下一些最重要目录的说明，了解它们的功能对于系统管理和日常操作至关重要。

![](img/c729070ce7aa720641f917bf4a9e3d05_35.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_37.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_39.png)

*   **`/`（根目录）**：这是整个文件系统层次结构的起点，所有其他目录和文件都挂载在其下。
*   **`/bin`（二进制文件目录）**：存放系统最基本的、供所有用户使用的可执行命令（二进制文件），例如`ls`、`cat`、`cp`等。
*   **`/boot`（启动目录）**：存放系统启动时所需的文件，如Linux内核（`vmlinuz-*`）、引导加载程序（GRUB）的配置文件等。
*   **`/dev`（设备文件目录）**：存放设备文件。在Linux中，一切皆文件，硬件设备（如硬盘、光驱、终端）也以文件形式存在于此。例如，第一块硬盘通常是`/dev/sda`，其分区是`/dev/sda1`、`/dev/sda2`等。
*   **`/etc`（配置文件目录）**：存放系统和应用程序的配置文件。例如，用户账户信息文件`/etc/passwd`、系统挂载表`/etc/fstab`等。
*   **`/home`（用户家目录）**：普通用户的个人目录所在地。每个用户通常在此目录下有一个以自己用户名命名的子目录（如`/home/mk`），用于存放个人文件和配置。
*   **`/lib` 和 `/lib64`（库文件目录）**：存放系统最基本的共享库文件（以`.so`结尾）和内核模块，这些库是`/bin`和`/sbin`目录下程序运行所必需的。`/lib64`专门存放64位系统的库文件。
*   **`/media` 和 `/mnt`（挂载点目录）**：通常用于临时挂载外部存储设备，如光盘、USB驱动器等。`/media`常用于自动挂载的可移动媒体，`/mnt`常用于管理员手动挂载的文件系统。
*   **`/opt`（可选应用程序目录）**：用于安装第三方或可选的应用程序软件包。
*   **`/proc`（进程信息虚拟文件系统）**：一个虚拟文件系统，提供内核和进程信息的接口。它不是真实的磁盘文件，系统关闭后即消失。例如，`/proc/cpuinfo`文件包含CPU的详细信息。
*   **`/root`（root用户家目录）**：系统管理员（root用户）的个人目录。
*   **`/run`（运行时数据目录）**：存放自系统启动以来描述系统信息的文件，如当前登录用户、运行中的进程ID等。这是一个临时文件系统。
*   **`/sbin`（系统二进制文件目录）**：存放供系统管理员（root用户）使用的系统管理命令，如`fdisk`（磁盘分区）、`mkfs`（格式化）等。
*   **`/tmp`（临时文件目录）**：供所有用户存放临时文件。系统会定期清理此目录下的内容。注意，此目录有一个特殊的“粘滞位（t）”权限，意味着只有文件的所有者或root用户才能删除其中的文件。
*   **`/usr`（用户程序目录）**：存放用户安装的应用程序和文件，是另一个重要的层级。其子目录结构类似于根目录，例如：
    *   `/usr/bin`：用户后期安装的大多数命令。
    *   `/usr/lib`：应用程序的库文件。
    *   `/usr/local`：本地编译安装的软件通常放置于此。
*   **`/var`（可变数据目录）**：存放经常变化的文件，如日志文件（`/var/log`）、邮件队列、数据库文件等。

## 绝对路径与相对路径

![](img/c729070ce7aa720641f917bf4a9e3d05_58.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_60.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_62.png)

路径是用来定位文件或目录在文件系统中位置的字符串。在Linux中，主要有两种表示路径的方式：绝对路径和相对路径。

![](img/c729070ce7aa720641f917bf4a9e3d05_64.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_66.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_68.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_70.png)

*   **绝对路径**：从根目录（`/`）开始写起的完整路径。无论当前位于哪个目录，绝对路径都能唯一、准确地指向目标。
    *   **示例**：`/etc/passwd`、`/home/mk/Documents/report.txt`
    *   **特点**：总是以斜杠（`/`）开头。

![](img/c729070ce7aa720641f917bf4a9e3d05_72.png)

*   **相对路径**：相对于当前工作目录（Current Working Directory）的路径。它不从根目录开始。
    *   **特殊符号**：
        *   `.`（一个点）：代表当前目录本身。
        *   `..`（两个点）：代表当前目录的上一级目录（父目录）。
    *   **示例**：假设当前目录是`/home/mk`。
        *   `./Documents` 或 `Documents` 指向 `/home/mk/Documents`。
        *   `../alice` 指向 `/home/alice`。
        *   如果当前在`/etc/sysconfig/network-scripts`，使用`cd ..`会切换到`/etc/sysconfig`目录。

![](img/c729070ce7aa720641f917bf4a9e3d05_74.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_75.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_77.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_79.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_80.png)

**路径使用对比示例：**
假设我们要查看`/etc/passwd`文件。
*   使用绝对路径：`cat /etc/passwd`
*   使用相对路径（假设当前已在`/etc`目录）：`cat passwd` 或 `cat ./passwd`

![](img/c729070ce7aa720641f917bf4a9e3d05_81.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_83.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_85.png)

绝对路径的优点是明确无误，缺点是较长。相对路径在操作临近文件时更加简洁。理解并熟练运用这两种路径是高效使用命令行的基础。

## 总结

![](img/c729070ce7aa720641f917bf4a9e3d05_96.png)

![](img/c729070ce7aa720641f917bf4a9e3d05_98.png)

本节课中我们一起学习了Linux文件系统的两大基石。首先，我们认识了从根目录（`/`）开始的树形目录结构，了解了`/bin`、`/etc`、`/home`、`/var`等核心目录的用途。接着，我们掌握了定位文件的两种方法：**绝对路径**（从`/`开始）和**相对路径**（使用`.`、`..`或相对于当前目录）。这些知识是后续进行文件创建、删除、复制、移动以及更高级系统管理操作的必备前提。请务必通过实践命令（如`cd`、`pwd`、`ls`）来加深对这些概念的理解。
