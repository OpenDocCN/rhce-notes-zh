在本节课中，我们将要学习Linux命令行中三个非常强大且常用的概念：管道符、重定向以及通配符。它们能极大地提升我们处理数据和执行命令的效率。今天我们先重点学习**重定向**和**管道符**，明天再继续学习转义符和环境变量。

![](img/c9311cbac9658aab45e8f54e822ab43d_5.png)

## 概述 📋

重定向和管道符是Linux命令行的核心技巧。简单来说，**重定向**用于改变命令输入和输出的默认方向（通常是屏幕），而**管道符**则用于将一个命令的输出作为另一个命令的输入，实现命令间的协作。

## 一、重定向：改变数据的流向 🔄

重定向分为两种：**输入重定向**和**输出重定向**。它们改变了命令与数据源（通常是文件）之间的交互方式。

### 1. 输出重定向

![](img/c9311cbac9658aab45e8f54e822ab43d_9.png)

输出重定向是指，将命令原本要显示在屏幕上的结果，“重定向”到写入一个文件中。

![](img/c9311cbac9658aab45e8f54e822ab43d_11.png)

![](img/c9311cbac9658aab45e8f54e822ab43d_13.png)

其基本符号是一个大于号 `>`。

![](img/c9311cbac9658aab45e8f54e822ab43d_15.png)

#### 标准输出与错误输出

![](img/c9311cbac9658aab45e8f54e822ab43d_17.png)

命令的执行结果分为两类：
*   **标准输出**：命令正常执行后产生的信息。
*   **错误输出**：命令执行出错时产生的报错信息。

![](img/c9311cbac9658aab45e8f54e822ab43d_19.png)

在重定向时，我们需要对它们加以区分。

*   **标准输出重定向**：使用单个 `>` 符号。
    ```bash
    ls -l /etc/passwd > result.txt
    ```
    这条命令将 `ls -l` 的正常输出结果写入 `result.txt` 文件，屏幕上不再显示。

*   **错误输出重定向**：使用 `2>` 符号。
    ```bash
    ls -l /nonexistent_file 2> error.log
    ```
    这条命令将查看一个不存在文件时产生的错误信息，写入 `error.log` 文件。

*   **合并输出重定向**：使用 `&>` 符号。它将不区分标准输出和错误输出，全部写入指定文件。
    ```bash
    ls -l /etc/passwd /nonexistent_file &> all_output.txt
    ```

#### 覆盖写入与追加写入

默认情况下，使用 `>` 或 `2>` 会**清空**目标文件原有内容，再写入新内容，这称为**覆盖（清空）写入**。

![](img/c9311cbac9658aab45e8f54e822ab43d_21.png)

如果希望保留文件原有内容，在末尾添加新内容，则需要使用**追加写入**，符号是 `>>`（对于错误输出是 `2>>`）。

![](img/c9311cbac9658aab45e8f54e822ab43d_23.png)

```bash
# 第一次写入
echo "First line" > myfile.txt
# 第二次追加写入
echo "Second line" >> myfile.txt
```
执行后，`myfile.txt` 中将有两行内容。

### 2. 输入重定向

![](img/c9311cbac9658aab45e8f54e822ab43d_25.png)

输入重定向是指，将文件的内容作为命令的输入，而不是从键盘等标准输入设备读取。

![](img/c9311cbac9658aab45e8f54e822ab43d_27.png)

其基本符号是一个小于号 `<`。

一个典型的例子是 `wc` 命令。通常我们这样统计文件行数：
```bash
wc -l /etc/passwd
```
输出会包含行数和文件名。

![](img/c9311cbac9658aab45e8f54e822ab43d_29.png)

如果使用输入重定向：
```bash
wc -l < /etc/passwd
```
输出则只显示行数，不显示文件名。这是因为命令的“处理对象”变成了从文件“流入”的数据流本身，而非文件名。

另一个例子是 `tr` 命令，它不能直接读取文件，但可以通过输入重定向或管道（稍后介绍）来工作：
```bash
# 使用输入重定向将文件内容转换为大写
tr [a-z] [A-Z] < myfile.txt
```

## 二、管道符：连接命令的桥梁 🛠️

![](img/c9311cbac9658aab45e8f54e822ab43d_35.png)

![](img/c9311cbac9658aab45e8f54e822ab43d_37.png)

管道符的作用是将**命令A**的输出结果，直接作为**命令B**的输入，进行二次处理。其符号是一个竖线 `|`（在键盘上通常是 Shift + \）。

![](img/c9311cbac9658aab45e8f54e822ab43d_39.png)

你可以将其想象成一个传送带或流水线：上一个工序的产出，直接传给下一个工序继续加工。

### 管道符的基本用法

![](img/c9311cbac9658aab45e8f54e822ab43d_41.png)

![](img/c9311cbac9658aab45e8f54e822ab43d_43.png)

```bash
command_A | command_B
```

例如，我们想统计 `/etc` 目录下有多少个文件：
```bash
ls /etc | wc -l
```
这里，`ls /etc` 列出所有文件（每行一个），这个列表通过管道 `|` 传给 `wc -l` 命令，由后者统计行数，最终得出文件数量。

### 管道符的嵌套使用

管道符可以连续使用，实现多级处理。
```bash
command_A | command_B | command_C | command_D
```
就像工厂的流水线，每个命令（工序）只负责一项专门任务，组合起来就能完成复杂工作。

### 实用案例

1.  **分页查看长列表**：`ls -l` 的结果可能很长，用 `more` 或 `less` 分页查看。
    ```bash
    ls -l /usr/bin | more
    ```

2.  **过滤进程信息**：从 `ps aux` 的众多进程中，快速找到与 `ssh` 相关的。
    ```bash
    ps aux | grep ssh
    ```

3.  **自动化输入**：在脚本中自动为 `root` 用户设置密码（`--stdin` 参数表示从标准输入读取密码）。
    ```bash
    echo "new_password" | passwd --stdin root
    ```

## 三、通配符：模糊匹配的利器 🎯

通配符主要用于匹配文件名或路径名，特别适合处理一批具有共同特征的文件。

### 常用的通配符

![](img/c9311cbac9658aab45e8f54e822ab43d_45.png)

1.  **星号 `*`**：匹配**零个或多个**任意字符。
    *   `ls /dev/sda*`：匹配 `/dev` 目录下所有以 `sda` 开头的文件（如 `sda`, `sda1`, `sda2`）。

2.  **问号 `?`**：匹配**一个**任意字符。
    *   `ls /dev/sda?`：匹配 `/dev` 目录下以 `sda` 开头且后面**恰好有一个字符**的文件（如 `sda1`，但不匹配 `sda` 或 `sda12`）。

![](img/c9311cbac9658aab45e8f54e822ab43d_47.png)

![](img/c9311cbac9658aab45e8f54e822ab43d_49.png)

3.  **中括号 `[]`**：匹配括号内**任意一个**字符。
    *   `ls /dev/sda[0-9]`：匹配 `sda0` 到 `sda9`。
    *   `ls /dev/sda[135]`：只匹配 `sda1`, `sda3`, `sda5`。
    *   `ls /dev/sda[a-z]`：匹配 `sda` 后跟一个小写字母的文件。

### 通配符的实用技巧：批量创建文件

![](img/c9311cbac9658aab45e8f54e822ab43d_51.png)

使用大括号 `{}` 扩展，可以批量创建或操作文件。
```bash
touch file_{A,B,C}.txt
```
这条命令会一次性创建三个文件：`file_A.txt`, `file_B.txt`, `file_C.txt`。

![](img/c9311cbac9658aab45e8f54e822ab43d_53.png)

大括号 `{}` 内的内容用逗号分隔，会被逐一展开，与前后缀组合。

![](img/c9311cbac9658aab45e8f54e822ab43d_55.png)

![](img/c9311cbac9658aab45e8f54e822ab43d_57.png)

**注意**：`[]` 用于“匹配”已有文件，`{}` 用于“生成”文件列表，两者用途不同。

![](img/c9311cbac9658aab45e8f54e822ab43d_59.png)

## 总结 📝

本节课中我们一起学习了Linux命令行的三个高效工具：

1.  **重定向**：我们学会了用 `>` 和 `>>` 将命令输出保存到文件，用 `<` 将文件内容作为命令输入，并区分了标准输出和错误输出。
2.  **管道符**：我们掌握了用 `|` 连接多个命令，让前一个命令的输出成为后一个命令的输入，从而构建强大的命令流水线。
3.  **通配符**：我们了解了 `*`、`?`、`[]` 和 `{}` 的用法，能够使用它们来快速匹配和批量操作文件。

![](img/c9311cbac9658aab45e8f54e822ab43d_61.png)

这些技能是成为Linux熟练用户的基础，它们能让你摆脱重复劳动，用更优雅的方式解决问题。下节课，我们将继续学习本章的剩余内容：**转义符**和**环境变量**。
