## 系统安装回顾与分区概念

上一节我们完成了RHEL 8系统的安装。在安装过程中，分区是一个关键步骤。本节中，我们来看看分区背后的逻辑和一些核心概念。

系统分区的主要目的是隔离不同类型的数据，以提高系统的稳定性和管理效率。例如，将用户数据（`/home`）与系统文件（`/`）分开，可以防止用户数据占满整个系统盘导致系统无法运行。

以下是分区时需要考虑的几个要点：
*   **根分区 (`/`)**：这是系统核心文件所在的分区。对于初学者实验环境，分配20GB左右通常足够。如果只分一个根分区，所有数据都将存放在这里。
*   **启动分区 (`/boot`)**：存放系统启动所需的文件。分配200MB到1GB空间即可，分配过多会造成浪费。
*   **交换分区 (`swap`)**：相当于虚拟内存，当物理内存不足时使用。其大小通常设置为物理内存的1到2倍。
*   **用户家目录分区 (`/home`)**：存放普通用户的个人文件。将其独立分区，可以避免用户数据占满根分区空间。
*   **其他目录**：像 `/etc`、`/var` 等系统关键目录通常不能独立于根分区，它们必须与根目录在同一个分区，这与系统启动流程有关。

**Kdump内核转储机制**
Kdump是一种内核崩溃转储机制。当系统内核发生严重错误（类似Windows蓝屏）时，它会将内存中的数据保存到一个文件中，供后续分析崩溃原因。在实验环境中，如果内存紧张，可以禁用此功能以节省资源（约160MB）。启用命令在图形界面中对应一个复选框。

## 用户账户设置与系统初始化

系统安装包的过程中，需要设置管理员密码并创建普通用户。

**Root用户**
Root是Linux系统的超级管理员账户，拥有最高权限。安装时必须为其设置密码，例如：
```
密码：redhat
```
**重要提示**：请务必牢记设置的root密码。如果忘记，在实验环境中通常只能通过重装系统来解决。

**普通用户**
系统建议创建一个普通用户（如 `admin`）用于日常登录，以降低直接使用root账户带来的风险。在图形界面安装中，创建普通用户是必要步骤；如果是“最小化安装”，则可以跳过。

安装完成后，系统会重启。首次启动时会显示许可协议，点击接受即可。随后会进入“订阅管理器”界面，如果您购买了红帽的商业支持服务，可以在此注册；对于学习和实验，可以跳过此步骤，系统可以正常使用，但不会获得官方技术支持。

首次以root用户登录图形界面后，系统会进行一些初始化设置（如选择键盘布局、隐私设置等），按照提示操作即可完成。

## 开源软件简介

Linux本身是一个开源操作系统。理解“开源”概念是理解Linux世界的基础。

开源软件是指其源代码向所有人开放，允许用户自由使用、学习、修改和分发的软件。这与“闭源”软件（如大多数Windows软件）形成对比，闭源软件的源代码是保密的。

自由软件基金会定义了使用开源软件的四大自由：
1.  出于任何目的运行软件的自由。
2.  学习和修改软件源代码的自由。
3.  重新分发软件的自由。
4.  发布软件修改版（衍生版本）的自由。

使用开源软件时，通常需要遵循相关许可协议，例如注明源代码出处，以尊重原作者的贡献。

开源软件的优势包括：
*   **低风险**：由社区维护，避免因单一公司倒闭导致软件无人维护。
*   **低成本**：可免费使用，进行二次开发的成本也较低。
*   **高品质**：由全球开发者共同维护，bug修复迅速，功能迭代快。
*   **高安全**：代码公开透明，便于审查，难以植入隐蔽的后门。

## Linux发展简史

要理解Linux，需要从其历史渊源说起。

**Unix的诞生**
Linux深受Unix操作系统的影响。Unix诞生于1969年的AT&T贝尔实验室，最初是开源的。1970年1月1日成为了Unix系统的纪元时间，也是计算机世界的常用时间起点。后来Unix逐渐走向商业化和闭源，并且与特定硬件紧密绑定。

**GNU计划**
1983年，理查德·斯托曼发起了GNU计划，旨在创建一个完全自由的操作系统。该计划开发了许多重要的开源工具（如GCC编译器），但当时缺少一个核心部件：操作系统内核。

**Linux内核的出现**
1991年，芬兰大学生林纳斯·托瓦兹出于个人兴趣，参考Unix的设计理念，独立编写了一个操作系统内核，并将其开源，这就是Linux内核。请注意，Linux内核没有使用任何Unix的源代码。
内核是操作系统的核心，负责管理硬件资源。但仅有内核无法直接使用，还需要外围的应用程序。

**完整的操作系统**
GNU计划提供了丰富的应用程序，而Linux内核则提供了核心。两者结合，构成了一个完整的、可用的操作系统，即我们今天所说的“GNU/Linux”系统。这种组合体现了开源协作的精神。

## Linux主要发行版介绍

Linux内核是开源的，任何个人或组织都可以将其与不同的软件包组合，形成自己的“发行版”。红帽（Red Hat）是其中最具影响力的公司之一。

**红帽系列发行版**
红帽旗下主要有三个发行版，关系如下图所示：
```
Fedora -> (测试与创新) -> RHEL -> (代码重组) -> CentOS
```
*   **Red Hat Enterprise Linux (RHEL) 🎩**：红帽企业版。特点是**极其稳定**，每3-4年发布一个大版本，提供**付费的商业技术支持**。是企业服务器的首选之一。
*   **Fedora**：红帽社区版。特点是**软件版本新**，是新技术和新特性的“试验田”。通常每6个月发布一个新版本，**适合桌面用户和开发者体验前沿技术**。
*   **CentOS**：社区企业操作系统。它是对RHEL源代码进行重新编译后产生的免费发行版，与RHEL在功能上**几乎完全相同**，但**不提供商业支持**。由于红帽已收购CentOS项目，两者版本保持高度同步。CentOS是许多预算有限或无需商业支持的企业和个人的热门选择。

**其他流行发行版**
*   **Debian/Ubuntu**：以易用性著称，拥有庞大的社区和软件库，在桌面和服务器领域都很流行。
*   **SUSE/openSUSE**：在欧洲尤其流行的企业级发行版。
*   **国内发行版**：如麒麟、统信UOS、OpenEuler等，多基于开源技术进行二次开发，以满足本土化需求。

选择红帽（RHEL/CentOS）系列学习，是因为其拥有完善的认证体系（如RHCE），在企业市场占有率极高，且学会了它，再学习其他发行版也会触类旁通。

## Linux的基本原则

最后，我们了解几条Linux系统的基本设计哲学，这对后续学习至关重要。

1.  **一切皆文件**：这是Linux最核心的理念。包括硬件设备、进程、配置等，在Linux中都以文件的形式存在和访问。
2.  **小型、单一目的的程序**：Linux鼓励开发功能单一且精悍的小程序。通过管道和脚本将多个小程序组合起来，以完成复杂任务。这比一个庞大、多功能的程序更灵活、高效。
    ```bash
    # 例如：组合使用 ls, grep, wc 来统计文件数量
    ls -l | grep "^-" | wc -l
    ```
3.  **文本文件存储配置**：系统的配置信息通常存储在纯文本文件中。这使得配置易于查看、修改和备份，也便于通过脚本进行自动化管理。
4.  **避免与用户交互**：许多命令行工具设计为“非交互式”，通过参数和管道接收输入，以便于自动化。
