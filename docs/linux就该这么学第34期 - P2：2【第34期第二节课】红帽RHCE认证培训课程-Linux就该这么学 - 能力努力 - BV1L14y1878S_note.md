## 创建虚拟机硬件平台

![](img/d2cb9791081c75bc9b750053043f324d_23.png)

![](img/d2cb9791081c75bc9b750053043f324d_25.png)

安装好虚拟机软件后，第一步并非直接安装系统，而是先虚拟出一个硬件平台。

1.  点击“创建新的虚拟机”。
2.  选择“高级（自定义）”安装模式，以便进行更详细的配置。
3.  硬件兼容性选择最高版本。
4.  **关键步骤**：选择“稍后安装操作系统”。如果此处选择了镜像文件，系统会以最小化模式安装，与我们后续的实验环境不符。
5.  选择客户机操作系统为“Linux”，版本为“Red Hat Enterprise Linux 8 64位”。
6.  为虚拟机命名（例如“跟老刘学习Linux第一课”），并选择一个空间充足的磁盘路径。
7.  处理器配置：核心数可根据物理机CPU情况设置（可通过任务管理器查看），设置为1也无妨。
8.  内存配置：如果物理机内存大于4GB，建议为虚拟机分配2048MB（2GB）。
9.  网络类型：暂时选择“仅主机模式”，具体原理将在第8章详解。
10. 磁盘控制器类型：选择默认的SCSI或SATA均可，避免选择NVMe。
11. 创建新虚拟磁盘，大小设为20GB，选择“将虚拟磁盘拆分成多个文件”。
12. 完成硬件平台的创建。

## 安装红帽RHEL 8系统

硬件平台准备就绪后，现在开始安装操作系统。

1.  编辑虚拟机设置，在“CD/DVD”选项中加载下载好的RHEL 8系统镜像文件。
2.  开启此虚拟机。
3.  在启动界面，使用方向键选择第一项“Install Red Hat Enterprise Linux 8”，并按下回车键。
4.  选择系统语言为“English”。虽然可以选择中文，但为了适应工作环境和考试环境（均为英文），建议从开始就使用英文。
5.  配置时间和日期，选择“Asia/Shanghai”代表中国时区。
6.  软件选择：选择“Server with GUI”（带有图形界面的服务器）。这并非鼓励依赖图形界面，而是此模式包含了后续学习所需的基本命令和工具。
7.  安装目标：即磁盘分区。关于分区的详细知识（如 `/`, `/boot`, `/swap` 等）将在第6、7章深入讲解，此处使用自动配置即可。
8.  关闭“Kdump”内核崩溃转储功能以节省内存。
9.  网络和主机名：可以暂时启用网卡，主机名可设置为 `linuxprobe.com` 以便后续实验保持一致。
10. 安全策略和系统目的暂时无需设置。
11. 点击“Begin Installation”开始安装。
12. 在安装过程中，设置root管理员密码（可简单设置，如 `redhat`）并创建一个普通用户（如 `linuxprobe`）。
13. 安装完成后，点击“Reboot”重启系统。
14. 重启后，接受许可协议，并跳过红帽订阅注册。
15. 完成初始设置，选择使用root用户登录，以便拥有完整权限进行学习。

## Linux系统初始化进程与软件管理

系统安装好后，我们了解一下红帽系统的发展和一些核心概念。

在RHEL 7/8中，系统的初始化进程由传统的 `init` 升级为 `systemd`。`systemd` 不仅仅是初始化进程，还接管了系统服务的管理。因此，一些旧的服务管理命令（如 `service`, `chkconfig`）已被新的 `systemctl` 命令取代。

![](img/d2cb9791081c75bc9b750053043f324d_50.png)

![](img/d2cb9791081c75bc9b750053043f324d_51.png)

以下是常用的 `systemctl` 命令：
*   `systemctl start 服务名`：启动服务。
*   `systemctl stop 服务名`：停止服务。
*   `systemctl restart 服务名`：重启服务（会中断服务）。
*   `systemctl reload 服务名`：重新加载服务配置（不中断服务）。
*   `systemctl enable 服务名`：设置服务开机自启。
*   `systemctl disable 服务名`：禁止服务开机自启。
*   `systemctl status 服务名`：查看服务状态。

![](img/d2cb9791081c75bc9b750053043f324d_53.png)

![](img/d2cb9791081c75bc9b750053043f324d_54.png)

![](img/d2cb9791081c75bc9b750053043f324d_55.png)

![](img/d2cb9791081c75bc9b750053043f324d_56.png)

![](img/d2cb9791081c75bc9b750053043f324d_58.png)

**设置服务开机自启非常重要**，这类似于考试或生产环境中，服务需要在重启后依然生效。

Linux中软件的安装也经历了演变：
1.  **源代码编译安装**：早期方式，需要本地搭建编译环境并解决依赖，非常复杂。
2.  **RPM软件包**：红帽软件包管理器。它将软件的**编译规则**与**源代码**打包，简化了安装，但依赖关系仍需手动解决。
    *   **公式**：`源代码 + 安装规则 => RPM包`
3.  **YUM/DNF软件仓库**：进一步将大量常用的RPM包及其依赖信息集中管理。用户只需说出需求（软件名），仓库会自动解决依赖并安装。
    *   **目的**：进一步降低软件安装难度。
    *   **现状**：在RHEL 8中，`yum` 命令实际上是 `dnf` 的软链接，两者使用方式几乎相同，`dnf` 解决了 `yum` 的一些历史问题（如只能单线程安装、元数据臃肿）。

## Shell命令行基础

![](img/d2cb9791081c75bc9b750053043f324d_63.png)

现在，我们进入Linux的核心部分——命令行。图形界面中，点击“Activities” -> “Terminal”即可打开终端。

### Shell是什么？

Shell是人与计算机硬件之间的“翻译官”。用户通过Shell输入命令，Shell将其“翻译”成机器能理解的指令，操作硬件，再将结果“翻译”成人能看懂的信息反馈回来。

常见的Shell有bash、zsh、csh等。在大多数Linux发行版中，默认使用的是 **Bash**（Bourne-Again Shell）。它因其**好用**（支持命令历史、Tab补全、批处理脚本、环境变量）和**通用**而被广泛采用。

### 命令的格式

![](img/d2cb9791081c75bc9b750053043f324d_65.png)

一个完整的命令行通常包含三部分：
*   **命令**：要执行的操作（动词），例如 `新建`、`删除`。
*   **参数**：对命令的修饰，使其更贴合工作需要（副词），例如 `温柔地`、`详细地`。参数通常以 `-`（短格式，如 `-l`）或 `--`（长格式，如 `--list`）开头。
*   **对象**：命令的作用目标（名词），例如 `文件`、`用户`。

![](img/d2cb9791081c75bc9b750053043f324d_67.png)

![](img/d2cb9791081c75bc9b750053043f324d_68.png)

**格式规则**：
*   命令、参数、对象之间必须用**空格**分隔。
*   命令和参数严格**区分大小写**。
*   短格式参数可以合并，只保留一个 `-`，如 `-l -a` 可以写成 `-la`。长格式参数不能合并。

![](img/d2cb9791081c75bc9b750053043f324d_69.png)

![](img/d2cb9791081c75bc9b750053043f324d_70.png)

![](img/d2cb9791081c75bc9b750053043f324d_72.png)

**注意**：以上是通用规范，但某些命令的作者可能有特殊写法（如使用 `+` 而非 `-`），无需死记硬背，常用自然熟。

## 常用命令初体验

![](img/d2cb9791081c75bc9b750053043f324d_81.png)

![](img/d2cb9791081c75bc9b750053043f324d_83.png)

以下是几个简单的命令，让大家感受一下命令行的操作。

1.  **`echo`**：将文本或变量值输出到屏幕。
    *   **代码**：`echo RHCE.net`
    *   **作用**：在屏幕上显示“RHCE.net”。

2.  **`date`**：查看或设置系统时间与日期。
    *   **代码**：`date`
    *   **作用**：显示当前系统时间。
    *   **带参数示例**：`date "+%Y-%m-%d"` 可以只显示“年-月-日”格式的日期。

![](img/d2cb9791081c75bc9b750053043f324d_85.png)

![](img/d2cb9791081c75bc9b750053043f324d_87.png)

![](img/d2cb9791081c75bc9b750053043f324d_88.png)

![](img/d2cb9791081c75bc9b750053043f324d_89.png)

3.  **`timedatectl`**：设置系统时间（需要先关闭NTP时间同步）。
    *   **代码**：`timedatectl set-time "2022-06-01 08:00:00"`

![](img/d2cb9791081c75bc9b750053043f324d_100.png)

4.  **`reboot`**：重启系统。
    *   **代码**：`reboot`

![](img/d2cb9791081c75bc9b750053043f324d_102.png)

![](img/d2cb9791081c75bc9b750053043f324d_104.png)

5.  **`poweroff`**：关闭系统。
    *   **代码**：`poweroff`

6.  **`wget`**：从网络下载文件。
    *   **代码**：`wget 网址`

7.  **`ps`**：查看系统进程状态。
    *   **代码**：`ps aux`
    *   **作用**：查看系统中所有进程的详细信息。
    *   **进程状态**：
        *   **R**：运行中。
        *   **S**：休眠中。
        *   **D**：不可中断的休眠（类似“未响应”）。
        *   **Z**：僵尸进程（已终止但未释放资源）。
        *   **T**：已停止。

8.  **`pstree`**：以树状图形式显示进程及其子进程的依赖关系。

**提示**：`ps aux` 的输出信息量大且是静态的。下一节我们将学习 `top` 命令，它可以动态地、交互式地监控进程状态，类似于Windows的任务管理器。
