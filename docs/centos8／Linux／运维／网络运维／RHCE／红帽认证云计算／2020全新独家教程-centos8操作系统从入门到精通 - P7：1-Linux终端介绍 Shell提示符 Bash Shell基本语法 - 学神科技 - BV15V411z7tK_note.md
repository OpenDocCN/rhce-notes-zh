## 什么是Linux终端

上一节我们介绍了课程概述，本节中我们来看看什么是Linux终端。Linux终端是用户与系统进行交互的命令行界面。例如，登录Linux系统后，可以通过点击图标打开一个终端窗口。

![](img/6e2d15546b376398fc8ea42f0cfff9d7_4.png)

![](img/6e2d15546b376398fc8ea42f0cfff9d7_1.png)

![](img/6e2d15546b376398fc8ea42f0cfff9d7_6.png)

![](img/6e2d15546b376398fc8ea42f0cfff9d7_2.png)

我们当前使用的图形界面终端称为Gnome Terminal。Linux有不同的桌面环境风格，例如Gnome和KDE，就像Windows、macOS有不同的界面风格一样。此外，还有如Xshell、SecureCRT等远程连接工具。

输入 `tty` 命令可以查看当前终端的类型。

![](img/6e2d15546b376398fc8ea42f0cfff9d7_8.png)

## 修改终端样式

![](img/6e2d15546b376398fc8ea42f0cfff9d7_14.png)

默认终端通常是黑底白字。您可以根据个人喜好修改样式。

![](img/6e2d15546b376398fc8ea42f0cfff9d7_16.png)

## 终端间通信

在早期多用户共享服务器的场景下，Linux提供了终端间通信的功能。虽然现在不常用，但了解其原理有助于理解Linux的多用户特性。

![](img/6e2d15546b376398fc8ea42f0cfff9d7_18.png)

`echo` 命令可以向屏幕输出信息。通过重定向符号 `>`，可以将信息发送到指定终端设备文件（如 `/dev/pts/1`）。

![](img/6e2d15546b376398fc8ea42f0cfff9d7_19.png)

```bash
echo "Hello" > /dev/pts/1
```

系统广播消息（如关机通知）也会发送到所有登录的终端。例如，执行关机命令：

```bash
shutdown -h +10  # 10分钟后关机
```

![](img/6e2d15546b376398fc8ea42f0cfff9d7_21.png)

使用 `wall` 命令可以向所有用户广播消息：

![](img/6e2d15546b376398fc8ea42f0cfff9d7_23.png)

```bash
wall "系统将于10分钟后维护，请保存工作。"
```

## Shell 简介

终端是界面，而真正解释和执行命令的是Shell。Shell是用户与系统内核（Kernel）之间的接口，它接收用户输入的命令，并将其交给内核执行。

Shell本身也是一种编程语言，允许编写包含循环、分支等逻辑的脚本。

其工作流程可概括为：
1.  用户输入命令。
2.  Shell判断该命令是**内部命令**还是**外部命令**。
3.  **内部命令**：Shell自身包含的命令，系统启动时已调入内存，执行效率高。
4.  **外部命令**：通常是独立的应用程序，需要从硬盘读取到内存再执行。
5.  最终由内核调用系统资源完成命令指定的操作。

使用 `type` 命令可以判断一个命令是内部命令还是外部命令。

```bash
type cd      # cd 是 shell 内建命令
type ls      # ls 是外部命令 (通常位于 /usr/bin/ls)
```

通过Shell，我们可以实现用户管理、权限控制、磁盘管理、软件安装、网络配置等几乎所有系统操作。

## Shell 提示符

Shell等待用户输入命令时显示的字符串称为**Shell提示符**。它包含了重要的上下文信息。

一个典型的提示符如下所示：
`[root@localhost ~]#`

以下是各部分的含义：
*   **`root`**：当前登录的用户名。
*   **`@`**：分隔符。
*   **`localhost`**：主机名。
*   **`~`**：当前所在目录。`~` 代表当前用户的**家目录**。
*   **`#`** 或 **`$`**：提示符结尾。`#` 表示当前是**root超级管理员**，`$` 表示当前是**普通用户**。

当切换目录时，提示符中的路径部分会随之改变。

```bash
cd /etc
# 提示符可能变为 [root@localhost /etc]#
```

输入 `exit` 命令可以退出当前Shell或终端会话。

## Shell 的类型

![](img/6e2d15546b376398fc8ea42f0cfff9d7_38.png)

Linux系统支持多种Shell，就像有多种Linux发行版一样。常见的Shell有Bash、Zsh、Ksh等。

查看系统支持的Shell类型：

```bash
cat /etc/shells
```

查看当前用户默认使用的Shell：

```bash
head -1 /etc/passwd
```
在输出结果的最后一列，可以看到用户默认的Shell程序路径，例如 `/bin/bash`。

**学习初期注意事项**：在执行命令出错时，通常有两个主要原因：
1.  **命令拼写错误**。
2.  **当前所在路径不正确**。

熟练使用 `Tab` 键进行命令和路径补全，可以有效避免拼写错误并提高效率。
