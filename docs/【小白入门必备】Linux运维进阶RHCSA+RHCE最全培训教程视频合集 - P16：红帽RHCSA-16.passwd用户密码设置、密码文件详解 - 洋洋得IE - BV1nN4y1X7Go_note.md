`passwd` 命令用于设置或修改用户的登录密码。

### 基本用法

*   **修改当前用户密码**：直接执行 `passwd` 命令，系统会提示输入新密码。
    ```bash
    passwd
    ```
*   **修改指定用户密码**：需要管理员权限，使用 `passwd [用户名]`。
    ```bash
    passwd user1
    ```

### 密码规范

![](img/4fed760d832a2f490db9a6e15a8b31ef_24.png)

系统对密码有严格的复杂度要求，普通用户必须遵守，而 `root` 用户则不受此限制。以下是主要规范：
*   密码长度不少于8个字符。
*   密码应包含数字、字母（区分大小写）和特殊字符。
*   新密码不能与旧密码过于相似。

### 常用选项

![](img/4fed760d832a2f490db9a6e15a8b31ef_36.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_38.png)

以下是 `passwd` 命令的一些常用选项：
*   `-S`：查看用户的密码状态信息。
    ```bash
    passwd -S user1
    ```
*   `-l`：锁定用户密码，使其无法登录。
    ```bash
    passwd -l user1
    ```
*   `-u`：解锁已被锁定的用户密码。
    ```bash
    passwd -u user1
    ```
*   `-d`：删除用户密码（清空密码字段）。
    ```bash
    passwd -d user1
    ```
*   `--stdin`：非交互式设置密码，通常用于脚本中。
    ```bash
    echo "NewPassword123!" | passwd --stdin user1
    ```

## 密码文件 `/etc/shadow` 详解

用户密码的加密信息存储在 `/etc/shadow` 文件中，该文件权限严格，只有 `root` 用户可以读取和修改。文件中的每一行对应一个用户，由冒号 `:` 分隔为9个字段。

以下是每个字段的含义解析（以 `user1:$6$...:19077:0:99999:7:::` 为例）：

1.  **用户名**：用户的登录名，例如 `user1`。
2.  **加密密码**：经过加密算法（如SHA-512）处理后的密码字符串。如果该字段为 `!!` 或 `*`，则表示账户未设置密码或密码已锁定。
3.  **上次修改密码日期**：从1970年1月1日（Unix纪元）到上次修改密码的天数。例如 `19077`。
4.  **密码最短有效期**：密码修改后，必须经过多少天才可以再次修改。`0` 表示随时可以修改。
5.  **密码最长有效期**：密码保持有效的最大天数。例如 `99999` 表示几乎永久有效。管理员常将此值设为如 `60`，要求用户每两个月更改一次密码。
6.  **密码过期警告期**：在密码到期前多少天开始向用户发出警告。例如 `7` 表示到期前7天发出警告。
7.  **密码过期宽限期**：密码过期后，账户仍可登录的宽限天数。在此期间登录会强制要求修改密码。超过此期限，账户将被锁定。
8.  **账户失效日期**：指定账户自动失效的日期（同样以1970年以来的天数计算）。到达此日期后，账户将无法登录。
9.  **保留字段**：目前未使用，留作未来扩展。

## 强制用户首次登录修改密码

![](img/4fed760d832a2f490db9a6e15a8b31ef_48.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_50.png)

在企业环境中，管理员分发初始账户时，常希望用户首次登录就必须更改默认密码。这可以通过修改 `/etc/shadow` 文件中用户的“上次修改密码日期”（第3个字段）为 `0` 来实现。

![](img/4fed760d832a2f490db9a6e15a8b31ef_52.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_54.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_56.png)

使用 `chage` 命令可以方便地完成此操作：
```bash
chage -d 0 user1
```
执行后，用户 `user1` 下次登录时，系统将强制要求其更改密码。

## 用户身份切换

![](img/4fed760d832a2f490db9a6e15a8b31ef_60.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_62.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_64.png)

`su` 命令用于切换当前用户身份到其他用户。

![](img/4fed760d832a2f490db9a6e15a8b31ef_66.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_68.png)

![](img/4fed760d832a2f490db9a6e15a8b31ef_69.png)

*   **切换用户（不切换环境变量）**：`su [用户名]`
    ```bash
    su user5
    ```
*   **完全切换用户（包括环境变量和家目录）**：`su - [用户名]`
    ```bash
    su - user5
    ```
*   **退出切换的身份**：执行 `exit` 命令。
    ```bash
    exit
    ```
> **注意**：`root` 用户切换到任何其他用户都无需输入密码。
