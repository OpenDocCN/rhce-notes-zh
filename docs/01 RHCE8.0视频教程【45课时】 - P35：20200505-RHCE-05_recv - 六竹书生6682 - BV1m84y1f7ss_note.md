# Ansible教程：01：Ansible简介与基础安装

在本节课中，我们将要学习自动化运维工具Ansible的基础知识，包括其核心概念、特点、架构以及如何安装和进行最基本的连接测试。

## Ansible简介

Ansible是一款用于自动化配置管理、应用部署和任务执行的工具。它主要用于管理Web服务器、数据库等设备。如果只有一台设备，可以直接登录进行更新。但当需要管理10台或100台设备时，逐台操作就变得不合理，此时就需要借助Ansible这类运维工具。

Ansible的第一个版本发布于2012年3月9日，后于2015年10月17日被红帽公司收购。它最初由红帽内部员工开发，后出售给红帽公司。Ansible是基于Python语言开发的工具，在国内应用广泛，因为Python语言普及度高，便于对Ansible进行二次开发，以更好地适应公司业务需求。

## Ansible核心特性

上一节我们介绍了Ansible的基本概念，本节中我们来看看它有哪些核心特性。

### 无代理架构

![](img/590233ce53597947b43057267c0ed7ed_1.png)

![](img/590233ce53597947b43057267c0ed7ed_2.png)

Ansible的一个显著特点是**无代理**。这意味着被管理的节点上无需预先安装任何Ansible客户端软件。传统的网管工具通常需要在被控端安装代理软件。虽然无代理架构的功能可能比有代理架构少一些，但它更加轻量级，非常适合中小型企业环境。

因此，在Ansible的架构中，我们主要讨论**控制节点**和**被控节点**，而不涉及客户端/服务器模式。其架构是一台主控服务器控制多台设备。

### 管理方式

关于如何管理被控端，主要有两种方式：
*   **有代理**：需要在被控端预先安装代理软件，功能更强大，但前期部署工作量较大。
*   **无代理**：Ansible采用这种方式，基于SSH协议进行访问。这又分为基于密码的访问和基于密钥的访问两种形式。

## Ansible的执行方式

![](img/590233ce53597947b43057267c0ed7ed_14.png)

Ansible提供了多种执行任务的方式，以适应不同复杂度的需求。

![](img/590233ce53597947b43057267c0ed7ed_16.png)

![](img/590233ce53597947b43057267c0ed7ed_17.png)

以下是Ansible的三种主要执行方式：
1.  **Ad-Hoc命令**：执行单条的Linux命令来解决简单问题。
2.  **Playbook**：将一系列命令整合成剧本的形式执行，用于完成复杂或重复的任务。
3.  **Role**：当单一Playbook无法满足复杂需求时，可以使用Role在逻辑上整合多个Playbook，使其结构更清晰、更易复用。

### 幂等性

![](img/590233ce53597947b43057267c0ed7ed_65.png)

![](img/590233ce53597947b43057267c0ed7ed_67.png)

![](img/590233ce53597947b43057267c0ed7ed_69.png)

**幂等性**是Ansible一个非常重要的特性。它意味着一个任务无论执行一次还是多次，最终的效果都是一样的。

![](img/590233ce53597947b43057267c0ed7ed_71.png)

考虑一个普通Linux命令：`mkdir /tmp/aa`。第一次执行时，如果目录不存在，则会创建。第二次执行时，因为目录已存在，系统会报错。

![](img/590233ce53597947b43057267c0ed7ed_73.png)

而在Ansible中，使用相应模块执行创建目录操作时：
*   第一次执行：目录不存在，则创建。
*   第二次执行：发现目录已存在，则不会再次执行，也不会报错。

这避免了因重复执行命令而导致的错误。

![](img/590233ce53597947b43057267c0ed7ed_75.png)

### 体系架构

![](img/590233ce53597947b43057267c0ed7ed_88.png)

Ansible的体系架构主要包含以下组件：
*   **控制节点**：安装并运行Ansible的主机。
*   **被控节点**：被Ansible管理的主机，可以有N台。
*   **用户**：通过Ansible控制被控节点的操作者。
*   **主机清单**：一个文件（默认位于`/etc/ansible/hosts`），其中记录了所有需要被管理的主机的IP地址或主机名。只有记录在此清单中的主机才能被管理。
*   **模块**：Ansible执行任务的基本单位，可以通过Ad-Hoc命令或Playbook调用。
*   **插件**：用于扩展Ansible核心功能。
*   **Playbook**：将多个模块任务组织起来的剧本。

![](img/590233ce53597947b43057267c0ed7ed_90.png)

![](img/590233ce53597947b43057267c0ed7ed_92.png)

![](img/590233ce53597947b43057267c0ed7ed_93.png)

用户通过Ansible的核心命令，结合主机清单，调用模块或Playbook，通过SSH协议网络发送指令到被控主机执行任务。Ansible不仅可以管理Linux/Unix主机和Windows主机（仅作为被控端），还能管理网络设备（如路由器、交换机）。它还可以与CMDB（配置管理数据库）集成，调用其中的配置信息来管理主机。

![](img/590233ce53597947b43057267c0ed7ed_95.png)

![](img/590233ce53597947b43057267c0ed7ed_97.png)

### 工作流程

![](img/590233ce53597947b43057267c0ed7ed_99.png)

![](img/590233ce53597947b43057267c0ed7ed_101.png)

当控制节点向被控节点发送命令时，其内部执行流程如下：
1.  **加载配置文件**：读取Ansible的配置文件（如`/etc/ansible/ansible.cfg`）。
2.  **加载模块文件**：加载任务对应的模块代码。
3.  **生成临时脚本**：在控制节点本地生成临时的Python脚本文件。
4.  **传输脚本文件**：将生成的临时脚本文件传输到被控节点上。
5.  **修改脚本权限**：在被控节点上为临时脚本添加可执行权限。
6.  **执行脚本并返回结果**：在被控节点上执行脚本，并将结果返回给控制节点。
7.  **删除临时脚本**：任务执行完毕后，删除被控节点上的临时脚本文件。

### 安装Ansible

![](img/590233ce53597947b43057267c0ed7ed_126.png)

![](img/590233ce53597947b43057267c0ed7ed_127.png)

在控制节点（`control.xtt.com`）上执行以下步骤：

![](img/590233ce53597947b43057267c0ed7ed_129.png)

1.  配置EPEL源和本地YUM源，确保可以安装软件包。
    ```bash
    # 上传并安装EPEL源包（具体包名可能不同）
    rpm -ivh epel-release-latest-8.noarch.rpm

    # 配置本地BaseOS和AppStream源
    cat > /etc/yum.repos.d/local.repo << EOF
    [baseos]
    name=Local BaseOS
    baseurl=file:///mnt/BaseOS
    gpgcheck=0

    [appstream]
    name=Local AppStream
    baseurl=file:///mnt/AppStream
    gpgcheck=0
    EOF
    ```
2.  安装Ansible。
    ```bash
    yum install ansible -y
    ```
3.  验证安装。
    ```bash
    ansible --version
    ```
    此命令会显示Ansible版本、配置文件路径、模块路径等信息。

#### 配置文件

![](img/590233ce53597947b43057267c0ed7ed_146.png)

![](img/590233ce53597947b43057267c0ed7ed_147.png)

![](img/590233ce53597947b43057267c0ed7ed_149.png)

主配置文件为`/etc/ansible/ansible.cfg`。初始时，许多配置都被注释，使用默认值。有几个关键配置项：
*   `inventory`: 指定主机清单文件路径，默认为`/etc/ansible/hosts`。
*   `remote_tmp`: 远程主机上临时脚本的存放目录。
*   `local_tmp`: 控制节点上临时脚本的存放目录。
*   `forks`: 并发连接数，默认为5。
*   `host_key_checking`: 是否检查SSH主机密钥。建议设置为`False`以避免首次连接时的确认提示。
*   `log_path`: 指定日志文件路径，如`/var/log/ansible.log`。取消注释以启用日志记录。

例如，禁用主机密钥检查并启用日志：
```ini
host_key_checking = False
log_path = /var/log/ansible.log
```

#### 管理主机清单

![](img/590233ce53597947b43057267c0ed7ed_151.png)

主机清单文件`/etc/ansible/hosts`支持多种格式：

![](img/590233ce53597947b43057267c0ed7ed_153.png)

1.  **管理单台主机**：直接写入IP或主机名。
    ```
    192.168.127.201
    ```
2.  **管理主机组**：使用`[组名]`格式。
    ```
    [webservers]
    192.168.127.201
    192.168.127.202
    ```
3.  **管理嵌套组**：使用`:children`后缀。
    ```
    [webservers]
    192.168.127.201
    192.168.127.202

    [dbservers]
    192.168.127.211

    [allservers:children]
    webservers
    dbservers
    ```
    也支持使用通配符和范围：
    ```
    [web]
    192.168.127.[201:202] # 表示201和202

    [all]
    192.168.127.*
    ```

### 首次连接测试

![](img/590233ce53597947b43057267c0ed7ed_155.png)

![](img/590233ce53597947b43057267c0ed7ed_157.png)

![](img/590233ce53597947b43057267c0ed7ed_159.png)

使用`ping`模块测试与被控节点的连通性。`ping`模块是基于SSH的连通性测试，并非ICMP ping。

![](img/590233ce53597947b43057267c0ed7ed_162.png)

![](img/590233ce53597947b43057267c0ed7ed_164.png)

1.  **测试单台主机**（需要输入被控节点密码）：
    ```bash
    ansible 192.168.127.201 -m ping -k
    ```
    系统会提示输入SSH密码（如`root`用户的密码）。成功后返回`pong`。
2.  **测试主机组**：
    ```bash
    ansible webservers -m ping -k
    ```
3.  **查看清单中的主机**：
    ```bash
    ansible webservers --list-hosts
    ansible allservers --list-hosts
    ```

### Ansible命令常用选项

![](img/590233ce53597947b43057267c0ed7ed_170.png)

以下是一些常用的`ansible`命令选项：
*   `--version` 或 `-v`: 查看版本信息 (`-vvv`, `-vvvv` 输出更详细)。
*   `--module-path` 或 `-M`: 指定模块路径。
*   `--list-hosts`: 列出匹配的主机。
*   `--ask-pass` 或 `-k`: 连接时询问SSH密码。
*   `--ask-become-pass` 或 `-K`: 提权时询问密码（如sudo）。
*   `--user` 或 `-u`: 指定远程执行用户，如 `-u redhat`。
*   `--become` 或 `-b`: 提权执行（通常配合`-K`使用）。
*   `--timeout` 或 `-T`: 设置连接超时时间。

![](img/590233ce53597947b43057267c0ed7ed_176.png)

**示例**：以`redhat`用户连接并测试：

### 主机模式匹配

![](img/590233ce53597947b43057267c0ed7ed_180.png)

![](img/590233ce53597947b43057267c0ed7ed_182.png)

![](img/590233ce53597947b43057267c0ed7ed_184.png)

在命令行中可以直接使用模式来匹配主机清单中的主机：
*   `all` 或 `*`: 匹配所有主机。
*   `192.168.127.*`: 匹配该网段所有主机。
*   `webservers:dbservers`: **并集**，匹配两个组中的所有主机（去重）。
*   `webservers:&dbservers`: **交集**，匹配同时属于两个组的主机。
*   `webservers:!dbservers`: **差集**，匹配在`webservers`组但不在`dbservers`组的主机。

![](img/590233ce53597947b43057267c0ed7ed_186.png)

![](img/590233ce53597947b43057267c0ed7ed_188.png)

**示例**：
```bash
# 测试所有主机
ansible all -m ping -k

![](img/590233ce53597947b43057267c0ed7ed_190.png)

![](img/590233ce53597947b43057267c0ed7ed_192.png)

# 测试webservers和dbservers的并集
ansible 'webservers:dbservers' -m ping -k

![](img/590233ce53597947b43057267c0ed7ed_194.png)

# 测试webservers和dbservers的交集（假设只有201同时属于两个组）
ansible 'webservers:&dbservers' -m ping -k

# 测试在webservers但不在dbservers的主机（如202）
ansible 'webservers:!dbservers' -m ping -k
```
