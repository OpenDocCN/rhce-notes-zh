## 软件仓库模块 (`yum_repository`) 📦

上一节我们介绍了Ansible临时命令的基础，本节中我们来看看如何管理软件仓库。`yum_repository` 模块的主要作用是创建或删除YUM/DNF软件仓库配置文件。其前提是软件仓库必须存在。

![](img/539e2c5f9e4aa07cd5e0163596527be5_3.png)

该模块的常用参数与手动创建repo文件时的配置项类似。

![](img/539e2c5f9e4aa07cd5e0163596527be5_4.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_6.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_7.png)

以下是 `yum_repository` 模块的关键参数：
*   **`name`**: 软件仓库的名称（对应repo文件的 `[repository_id]`）。
*   **`baseurl`**: 软件仓库的地址。
*   **`description`**: 软件仓库的描述信息。
*   **`enabled`**: 是否启用该仓库（1为启用，0为禁用）。
*   **`file`**: 保存配置的文件名，通常为 `[name].repo`。
*   **`gpgcheck`**: 是否进行GPG完整性验证（1为检查，0为不检查）。
*   **`gpgkey`**: GPG密钥文件的URL。

![](img/539e2c5f9e4aa07cd5e0163596527be5_8.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_10.png)

### 配置本地软件仓库示例

我们以在 `node1.lab.example.com` 上配置红帽8系统的 `BaseOS` 和 `AppStream` 本地仓库为例。

![](img/539e2c5f9e4aa07cd5e0163596527be5_12.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_14.png)

首先，创建挂载点并挂载ISO镜像。这里会用到 `file` 和 `mount` 模块。

![](img/539e2c5f9e4aa07cd5e0163596527be5_16.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_18.png)

```bash
# 1. 创建挂载目录
ansible node1.lab.example.com -m file -a "path=/mnt/iso state=directory"

![](img/539e2c5f9e4aa07cd5e0163596527be5_20.png)

# 2. 挂载光盘镜像
ansible node1.lab.example.com -m mount -a "src=/dev/sr0 path=/mnt/iso fstype=iso9660 state=mounted"
```

![](img/539e2c5f9e4aa07cd5e0163596527be5_22.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_24.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_26.png)

接着，使用 `yum_repository` 模块创建两个软件仓库配置文件。

![](img/539e2c5f9e4aa07cd5e0163596527be5_28.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_30.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_32.png)

```bash
# 3. 创建 BaseOS 仓库
ansible node1.lab.example.com -m yum_repository -a "name=BaseOS description='BaseOS Repository' baseurl=file:///mnt/iso/BaseOS enabled=1 gpgcheck=0 state=present"

![](img/539e2c5f9e4aa07cd5e0163596527be5_33.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_35.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_36.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_38.png)

# 4. 创建 AppStream 仓库 (注意目录名大小写)
ansible node1.lab.example.com -m yum_repository -a "name=AppStream description='AppStream Repository' baseurl=file:///mnt/iso/AppStream enabled=1 gpgcheck=0 state=present"
```

![](img/539e2c5f9e4aa07cd5e0163596527be5_40.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_42.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_43.png)

最后，我们可以验证仓库是否配置成功。

![](img/539e2c5f9e4aa07cd5e0163596527be5_45.png)

```bash
# 方法一：直接查看生成的文件
ansible node1.lab.example.com -m shell -a "cat /etc/yum.repos.d/BaseOS.repo"
ansible node1.lab.example.com -m shell -a "cat /etc/yum.repos.d/AppStream.repo"

![](img/539e2c5f9e4aa07cd5e0163596527be5_47.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_48.png)

```

## 软件包管理模块 (`yum`) 🔧

![](img/539e2c5f9e4aa07cd5e0163596527be5_55.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_57.png)

配置好软件仓库后，我们就可以使用 `yum` 模块来管理软件包了。该模块用于安装、卸载、更新软件包或软件包组。

![](img/539e2c5f9e4aa07cd5e0163596527be5_59.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_61.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_63.png)

`yum` 模块的核心参数如下：
*   **`name`**: 指定软件包或软件包组的名称。
*   **`state`**: 指定期望的状态。
    *   `present` 或 `installed`: 确保软件包已安装。
    *   `absent` 或 `removed`: 确保软件包已卸载。
    *   `latest`: 确保软件包更新到最新版本。

![](img/539e2c5f9e4aa07cd5e0163596527be5_65.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_66.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_68.png)

### 软件包管理操作示例

![](img/539e2c5f9e4aa07cd5e0163596527be5_70.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_72.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_73.png)

以下是在 `node1` 上进行的几种常见操作。

![](img/539e2c5f9e4aa07cd5e0163596527be5_75.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_77.png)

**1. 安装单个软件包 (如 httpd)**
```bash
ansible node1.lab.example.com -m yum -a "name=httpd state=present"
```

![](img/539e2c5f9e4aa07cd5e0163596527be5_79.png)

**2. 卸载软件包**
```bash
ansible node1.lab.example.com -m yum -a "name=httpd state=absent"
```

![](img/539e2c5f9e4aa07cd5e0163596527be5_81.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_83.png)

**3. 安装软件包组 (如 MariaDB)**
安装组时，需要在组名前加 `@` 并用引号括起来。
```bash
ansible node1.lab.example.com -m yum -a "name='@mariadb' state=present"
```

![](img/539e2c5f9e4aa07cd5e0163596527be5_85.png)

![](img/539e2c5f9e4aa07cd5e0163596527be5_87.png)

**4. 更新所有软件包到最新版本**
使用通配符 `*` 可以匹配所有包。
```bash
ansible node1.lab.example.com -m yum -a "name=* state=latest"
```

## 总结 📝

![](img/539e2c5f9e4aa07cd5e0163596527be5_91.png)

本节课中我们一起学习了Ansible中管理软件生态的两个重要模块：
1.  **`yum_repository` 模块**：用于自动化配置YUM/DNF软件仓库，其参数与repo文件内容对应，是考试和实践中配置软件源的关键。
2.  **`yum` 模块**：用于执行软件包的安装 (`present`)、卸载 (`absent`)、组安装 (`@group`) 和更新 (`latest`) 操作，是实现系统软件批量管理的核心。

通过这两个模块，我们可以轻松地将软件仓库配置和软件包管理任务自动化，无需在每台主机上手动执行重复命令。
