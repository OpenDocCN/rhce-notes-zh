# Linux文本处理工具：第八章：文本处理工具

## 概述
在本节课中，我们将要学习Linux系统中一系列强大的文本处理工具。这些工具是系统管理和脚本编写的基础，能够帮助我们高效地查看、过滤、分析和修改文本内容。掌握它们对于处理日志文件、编写自动化脚本至关重要。

## 第一节：查看文件内容

![](img/6fb25a3454c36adb1f141adf6d45d269_1.png)

上一节我们介绍了本章的重要性，本节中我们来看看如何查看文件的全部内容。

### cat命令
`cat`命令是最简单的文本查看工具，用于将文件内容一次性打印到终端。

**基本语法：**
```bash
cat [选项] 文件名
```

![](img/6fb25a3454c36adb1f141adf6d45d269_3.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_5.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_7.png)

**常用选项：**
*   `-n`：在输出内容前显示行号。

![](img/6fb25a3454c36adb1f141adf6d45d269_9.png)

**示例：**
1.  查看文件内容并显示行号：
    ```bash
    cat -n /etc/passwd
    ```
2.  合并查看多个文件：
    ```bash
    cat /etc/passwd /etc/profile
    ```
3.  将多个文件内容合并输出到一个新文件：
    ```bash
    cat file1.txt file2.txt > combined.txt
    ```

![](img/6fb25a3454c36adb1f141adf6d45d269_11.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_12.png)

### more和less命令
当文件内容较长时，`cat`命令会一次性输出所有内容，不便阅读。`more`和`less`命令可以进行分页显示。

**more命令：**
`more`命令允许用户逐页浏览文件内容。
*   按**空格键**向下翻一页。
*   按**回车键**向下滚动一行。
*   浏览到文件末尾后，命令自动退出。

![](img/6fb25a3454c36adb1f141adf6d45d269_14.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_16.png)

**示例：**
```bash
more /etc/passwd
```

**less命令：**
`less`命令比`more`更强大，支持前后翻页和搜索，是查看大文件的理想工具。
*   按**上下箭头键**或**Page Up/Page Down**键滚动。
*   按 **`/`** 后输入关键词进行搜索，按 **`n`** 跳转到下一个匹配项，按 **`N`** 跳转到上一个匹配项。
*   按 **`v`** 键会调用`vi`编辑器打开当前文件，方便直接编辑。
*   按 **`q`** 键退出。

**示例：**
```bash
less /etc/passwd
```

![](img/6fb25a3454c36adb1f141adf6d45d269_18.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_20.png)

> **提示：** Linux系统的`man`帮助手册默认使用的就是`less`作为分页器，因此其操作方式与`less`命令一致。

## 第二节：提取文件部分内容

上一节我们学习了如何查看整个文件，本节中我们来看看如何提取文件的特定部分，例如开头或结尾的若干行。

![](img/6fb25a3454c36adb1f141adf6d45d269_26.png)

### head命令
`head`命令用于显示文件的开头部分，默认显示前10行。

![](img/6fb25a3454c36adb1f141adf6d45d269_28.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_30.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_32.png)

**基本语法：**
```bash
head [选项] 文件名
```

**常用选项：**
*   `-n <行数>`：指定显示文件开头多少行。

**示例：**
以下是`head`命令的几种用法：
1.  查看文件前5行：
    ```bash
    head -n 5 /etc/passwd
    ```
2.  查看除了最后6行以外的所有行（即显示总行数减6行）：
    ```bash
    head -n -6 /etc/passwd
    ```

![](img/6fb25a3454c36adb1f141adf6d45d269_34.png)

### tail命令
`tail`命令用于显示文件的末尾部分，默认显示最后10行。它特别常用于实时监控日志文件。

**基本语法：**
```bash
tail [选项] 文件名
```

**常用选项：**
*   `-n <行数>`：指定显示文件末尾多少行。
*   `-f`：实时追踪文件末尾的新增内容，常用于监控日志。按 `Ctrl+C` 终止。
*   `+<行号>`：从指定行号开始显示到文件末尾。

**示例：**
以下是`tail`命令的几种用法：
1.  查看文件末尾5行：
    ```bash
    tail -n 5 /etc/passwd
    ```
2.  从第20行开始显示到文件末尾：
    ```bash
    tail -n +20 /etc/passwd
    ```
3.  **实时监控系统日志文件**：
    ```bash
    tail -f /var/log/messages
    ```
4.  同时监控日志目录下所有文件的更新（显示每个文件末尾10行）：
    ```bash
    tail -f /var/log/*
    ```

### 组合使用head和tail
通过管道符`|`组合`head`和`tail`，可以提取文件中任意连续的行。

**示例：**
提取文件第10行到第20行的内容：
```bash
cat /etc/passwd | head -n 20 | tail -n 11
```
**命令解析：**
1.  `cat /etc/passwd`：输出文件全部内容。
2.  `head -n 20`：取前20行。
3.  `tail -n 11`：从`head`的结果中取最后11行（即第10行到第20行）。

## 第三节：按行过滤内容（grep）

![](img/6fb25a3454c36adb1f141adf6d45d269_36.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_38.png)

上一节我们学习了按位置提取行，本节中我们来看看如何根据内容来过滤行，这是文本处理中最核心的操作之一。

![](img/6fb25a3454c36adb1f141adf6d45d269_40.png)

### grep命令简介
`grep`命令用于在文件中搜索包含指定模式（字符串或正则表达式）的行，并将匹配到的行打印出来。

**基本语法：**
```bash
grep [选项] ‘搜索模式’ 文件名
```

![](img/6fb25a3454c36adb1f141adf6d45d269_42.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_44.png)

**常用选项：**
*   `-i`：忽略大小写进行匹配。
*   `-v`：反向选择，即显示**不包含**匹配模式的所有行。
*   `-n`：显示匹配行及其行号。
*   `-c`：仅显示匹配到的行数。
*   `-r` 或 `-R`：递归搜索指定目录下的所有文件。
*   `-E`：使用扩展正则表达式（等同于`egrep`命令）。

![](img/6fb25a3454c36adb1f141adf6d45d269_46.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_47.png)

**示例：**
以下是`grep`命令的基础用法：
1.  在`/etc/passwd`中搜索包含“root”的行：
    ```bash
    grep ‘root’ /etc/passwd
    ```
2.  搜索时忽略大小写：
    ```bash
    grep -i ‘hello’ file.txt
    ```
3.  显示不包含“nologin”的行及其行号：
    ```bash
    grep -v -n ‘nologin’ /etc/passwd
    ```
4.  统计文件中包含“error”关键词的行数：
    ```bash
    grep -c ‘error’ /var/log/messages
    ```
5.  在当前目录及其子目录的所有文件中递归搜索“192.168.1.1”：
    ```bash
    grep -r ‘192.168.1.1’ .
    ```

> **核心概念：** `grep`的强大之处在于它支持**正则表达式**，可以进行非常复杂和灵活的文本匹配。例如，`grep ‘^root’`匹配以“root”开头的行，`grep ‘bash$’`匹配以“bash”结尾的行。

## 总结
本节课中我们一起学习了Linux下基础的文本处理工具。
*   我们使用 `cat`、`more`、`less` 来查看文件内容。
*   我们使用 `head` 和 `tail` 来提取文件的头部或尾部特定行，其中 `tail -f` 是监控日志的利器。
*   我们初步了解了 `grep` 命令，它能够根据内容过滤行，是文本搜索的核心工具。

![](img/6fb25a3454c36adb1f141adf6d45d269_57.png)

![](img/6fb25a3454c36adb1f141adf6d45d269_59.png)
