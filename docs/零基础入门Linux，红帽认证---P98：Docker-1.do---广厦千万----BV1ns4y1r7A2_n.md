## 什么是容器？

![](img/f8d8408766318f02c51a4f43c369f417_10.png)

![](img/f8d8408766318f02c51a4f43c369f417_12.png)

容器，顾名思义，是用于容纳物品的载体。在计算机领域，容器技术是一种虚拟化技术，它允许我们将应用程序及其运行环境打包在一起，形成一个独立、可移植的单元。

![](img/f8d8408766318f02c51a4f43c369f417_14.png)

## 容器解决了什么问题？

![](img/f8d8408766318f02c51a4f43c369f417_16.png)

上一节我们介绍了容器的基本定义，本节中我们来看看容器技术具体解决了哪些实际问题。这需要从企业软件部署方式的演变说起。

![](img/f8d8408766318f02c51a4f43c369f417_20.png)

在企业中，软件的部署方式主要经历了三个阶段：

![](img/f8d8408766318f02c51a4f43c369f417_24.png)

![](img/f8d8408766318f02c51a4f43c369f417_26.png)

1.  **传统部署方式**：直接将软件部署在物理服务器上。
    *   **优点**：部署简单。
    *   **缺点**：无法限制程序对系统资源（如CPU、内存）的使用量。一个程序出现故障（如内存泄漏）可能耗尽所有资源，导致同一服务器上的其他所有程序崩溃。

![](img/f8d8408766318f02c51a4f43c369f417_28.png)

![](img/f8d8408766318f02c51a4f43c369f417_29.png)

![](img/f8d8408766318f02c51a4f43c369f417_31.png)

![](img/f8d8408766318f02c51a4f43c369f417_33.png)

![](img/f8d8408766318f02c51a4f43c369f417_34.png)

![](img/f8d8408766318f02c51a4f43c369f417_35.png)

2.  **虚拟化部署方式**：通过虚拟机（VM）软件（如VMware）在一台物理服务器上虚拟出多台完整的“子计算机”。
    *   **优点**：实现了程序间的**隔离**。每个虚拟机拥有独立的操作系统和虚拟硬件，一个虚拟机内的程序故障不会影响其他虚拟机。
    *   **缺点**：**资源消耗较大**。每个虚拟机都需要虚拟一整套完整的操作系统和硬件，其启动速度慢，占用的系统资源远大于单个程序本身的需求。

![](img/f8d8408766318f02c51a4f43c369f417_37.png)

3.  **容器化部署方式**：容器也是一种虚拟化技术，但它比虚拟机更加轻量。
    *   **优点**：
        *   **隔离性**：每个容器相互隔离，互不影响。
        *   **高效性**：容器共享宿主机的操作系统内核，无需虚拟完整的操作系统和硬件，因此**启动速度极快**（像启动一个进程），且**资源利用率极高**。
        *   **一致性**：容器封装了应用程序及其所有依赖环境（如系统库、运行时环境），确保了应用在开发、测试、生产等任何环境中的运行一致性，解决了“水土不服”的问题。

![](img/f8d8408766318f02c51a4f43c369f417_39.png)

简单来说，容器技术完美地平衡了资源利用率和环境一致性，是现代软件开发和运维的核心技术之一。

## Docker是什么？

了解了容器解决的问题，我们来看看如何管理容器。Docker就是这样一个管理容器的工具。

Docker是一个用Go语言编写的开源项目，它是一套完整的容器管理系统。Linux内核本身具备容器功能，但用户无法直接操作。Docker提供了友好的命令行工具和API，让我们能够轻松地创建、管理和运行容器。

![](img/f8d8408766318f02c51a4f43c369f417_41.png)

![](img/f8d8408766318f02c51a4f43c369f417_43.png)

Docker的理念是 **“一次封装，随处运行”** 。开发者将应用及其环境打包成一个Docker镜像，这个镜像可以在任何安装了Docker的Linux系统上运行，不受底层系统具体发行版（如Ubuntu、CentOS）的影响。

Docker的Logo是一条鲸鱼驮着许多集装箱，其设计思想正来源于集装箱：
*   **隔离**：每个集装箱（容器）内的货物（应用）互不影响。
*   **高效**：集装箱可以标准化堆叠，极大提高了运输效率（资源利用率）。

在这里，鲸鱼代表宿主机操作系统，集装箱则代表运行在其上的各个Docker容器。

## Docker的核心组成

Docker生态系统主要由三个核心组件构成：镜像、容器和仓库。

以下是三者的关系与工作流程：

1.  **镜像（Image）**：镜像是创建容器的**模板**。它是一个只读的静态文件，包含了运行应用程序所需的代码、运行时环境、系统工具、库和设置。你可以将镜像理解为一个软件的“安装包”。
2.  **容器（Container）**：容器是镜像的**运行实例**。当镜像被运行时，它就变成了一个容器。容器是应用程序的**载体**，是独立、安全的运行环境。
3.  **仓库（Repository）**：仓库是集中存放镜像的**地方**，类似于代码仓库或软件应用商店。
    *   **公有仓库**：如 Docker Hub (`hub.docker.com`)，所有人都可以从中拉取镜像。
    *   **私有仓库**：企业或个人搭建的私人仓库，用于存放内部镜像。

它们的工作流程如下：
**用户操作 -> Docker命令 -> 从仓库拉取镜像 -> 在本地创建并运行容器**
