# Linux基础教程：P5：红帽发行版本和其他Linux版本介绍 🐧

在本节课中，我们将要学习红帽公司旗下的主要Linux发行版本，了解它们各自的特点和定位，并简要介绍其他常见的Linux发行版家族。通过学习，你将明白为何红帽企业版Linux（RHEL）在企业和认证领域占据重要地位。

## 红帽旗下的三个发行版本

上一节我们介绍了Linux的起源，本节中我们来看看具体的发行版本。首先，红帽公司旗下有三个主要的Linux发行版本。

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_1.png)

以下是红帽旗下的三个版本：

1.  **红帽企业版Linux**
    这是我们今天安装的系统。它的特点是应用程序和功能非常稳定，更新周期较慢，大约每4年发布一个大的新版本。它提供专业的技术支持服务，但这项服务是收费的。

2.  **Fedora**
    这个版本主要面向个人用户和桌面环境，其图形界面非常炫丽。Fedora的功能更新很快，但可能不够稳定。红帽公司会先在Fedora上测试新功能，收集用户反馈并修复问题，待功能稳定后才会引入到红帽企业版Linux中。Fedora由社区支持，不提供红帽官方的付费技术支持。

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_3.png)

3.  **CentOS**
    它的全称是 **Community Enterprise Operating System**，即社区企业操作系统。它最初并非红帽旗下，而是由一个社区维护。CentOS本质上是红帽企业版Linux的复制品，社区将RHEL的源代码重新编译，发布一个功能完全相同的免费版本。红帽后来收购了CentOS项目，但保持了其原有的免费模式。CentOS与RHEL版本几乎同步，唯一的区别是它不提供官方的商业技术支持。

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_5.png)

面对如此多的发行版，我们课程选择以红帽企业版Linux为重点进行学习。

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_49.png)

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_51.png)

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_53.png)

选择红帽的主要原因包括：
1.  红帽在开源界具有领导者地位。
2.  红帽拥有非常完整和权威的认证体系（如RHCSA、RHCE）。
3.  许多其他企业级发行版（如Oracle Linux、华为的EulerOS）都基于红帽体系衍生，学会RHEL有助于触类旁通。
4.  红帽被IBM收购后，获得了更广阔的市场和发展平台。

## Linux系统的基本原则

在结束本章之前，我们需要了解Linux系统设计的几个基本原则，这些理念贯穿整个系统的使用。

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_55.png)

以下是Linux的几个核心原则：

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_57.png)

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_59.png)

1.  **一切皆文件**
    在Linux中，一切资源都以文件的形式存在，包括硬件设备、进程信息等。这个概念可以抽象为：**系统中的所有操作对象都可以通过文件接口进行访问**。

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_61.png)

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_62.png)

2.  **小型、单一用途的程序**
    Linux鼓励使用功能单一的小程序。每个程序只做好一件事。例如，一个查看工具就只负责查看。当需要完成复杂任务时，通过将多个小程序组合起来实现。

3.  **通过文本文件存储配置**
    系统的配置信息通常保存在纯文本文件中。这意味着你可以通过编辑文本文件来修改系统设置，这种方式透明且灵活。例如，网络配置可能保存在 `/etc/sysconfig/network-scripts/` 目录下的一个文本文件中。

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_64.png)

![](img/6cb4c8850e084c3ab8a9a2cafdec8446_66.png)
