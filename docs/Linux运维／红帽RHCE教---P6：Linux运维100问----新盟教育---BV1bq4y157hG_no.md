# Linux运维100问：P6：Linux运维如何避免被当成网管

在本节课中，我们将探讨一个在运维工程师群体中常见的话题：如何避免在工作中被误解为“网管”。我们将分析两者职责的区别，并理解这种误解产生的原因。

## 网管与运维工程师的区别

上一节我们介绍了课程的主题，本节中我们来看看“网管”与“运维工程师”的核心区别。

网管在公司内部通常被称为“桌面运维”。其工作范围主要涵盖办公区域内可见的硬件与基础网络维护。而运维工程师的职责则覆盖了从产品设计、发布到后期维护的整个生命周期。运维工程师通常具备系统运维、应用运维等专业技能。

## 误解产生的原因

理解了基本区别后，我们进一步探讨这种误解为何会产生。

如果一个运维工程师被当成网管，通常说明所在公司可能同时需要运维工程师和桌面运维人员。许多小型公司在招聘运维岗位时，期望一名工程师能够承担所有技术相关工作，他们可能并不清晰区分运维与网管的职责边界。因此，将运维误解为网管的情况便自然出现了。

以下是导致误解的几个常见场景：
*   公司规模较小，岗位职责划分模糊。
*   招聘方对技术岗位的具体技能要求不明确。
*   运维人员需要兼顾部分桌面支持工作。

## 总结

本节课中我们一起学习了“网管”与“运维工程师”的职责区别，并分析了运维被当成网管这一现象背后的常见原因。核心在于公司对岗位定位的模糊以及对专业技能范围认知的不同。
