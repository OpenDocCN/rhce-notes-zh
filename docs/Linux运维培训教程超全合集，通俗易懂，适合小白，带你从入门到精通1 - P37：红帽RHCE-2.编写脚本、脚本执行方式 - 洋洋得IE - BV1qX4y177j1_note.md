上一节我们介绍了脚本的基本概念。本节中，我们来看看编写一个脚本的通用思维逻辑。这个逻辑与解决生活中的问题相似，主要分为以下几步：

![](img/006735f42e032ba8e652bc975877e4e9_13.png)

1.  **明确任务需求**：确定脚本最终要实现什么目标。
2.  **整理实现步骤**：将达成目标所需的具体操作分解为清晰的步骤。
3.  **编写与测试**：将步骤转化为命令写入脚本，并运行测试。
4.  **优化与完善**：根据测试结果修改错误，并优化脚本逻辑和输出。

## 第一个脚本：Hello World

![](img/006735f42e032ba8e652bc975877e4e9_23.png)

学习任何编程语言，通常都从输出“Hello World”开始。这是一种传统，也适用于Shell脚本。

![](img/006735f42e032ba8e652bc975877e4e9_25.png)

以下是创建一个简单“Hello World”脚本的步骤：

1.  使用文本编辑器（如`vim`）创建一个新文件，例如 `hello.sh`。
2.  在文件的第一行，通过 `#!` 指定脚本的解释器。对于Bash脚本，通常是：
    ```bash
    #!/bin/bash
    ```
3.  在下一行，使用 `echo` 命令输出文本。
    ```bash
    echo “Hello World”
    ```
4.  保存并退出编辑器。
5.  为脚本文件添加执行权限。
    ```bash
    chmod +x hello.sh
    ```
6.  执行脚本。如果脚本在当前目录，使用 `./` 前缀。
    ```bash
    ./hello.sh
    ```

执行后，终端将显示 `Hello World`。这个脚本的本质，就是将能在命令行直接运行的命令 `echo “Hello World”` 写入一个文件，并让这个文件可以像命令一样执行。

## 脚本编写的核心注意事项

从简单的“Hello World”脚本，我们过渡到更实用的脚本。在编写过程中，有一个至关重要的原则需要牢记：**脚本应尽量避免交互式命令**。

![](img/006735f42e032ba8e652bc975877e4e9_29.png)

### 什么是交互式命令？

![](img/006735f42e032ba8e652bc975877e4e9_31.png)

交互式命令在执行过程中会等待用户输入信息，然后才能继续。例如：
*   `passwd <用户名>`：设置密码时，会提示你输入并确认密码。
*   `vim <文件名>`：编辑文件时，会进入编辑器界面等待你的操作。

### 为什么脚本要避免交互？

脚本的目标是自动化执行。如果脚本中包含交互式命令，执行到该命令时就会“卡住”，等待人工输入。这对于需要定时运行（如凌晨备份）或无人值守的脚本来说是灾难性的。

### 如何解决？使用非交互方式。

对于必须的交互操作，我们需要找到其非交互的替代方法。例如，为用户设置密码：

*   **交互式方式（不适用于脚本）**：
    ```bash
    passwd username
    # 然后手动输入密码
    ```

![](img/006735f42e032ba8e652bc975877e4e9_33.png)

![](img/006735f42e032ba8e652bc975877e4e9_35.png)

*   **非交互式方式（适用于脚本）**：
    ```bash
    echo “密码” | passwd --stdin username
    ```
    这条命令通过管道 `|` 将密码字符串直接传递给 `passwd` 命令，无需人工干预。

![](img/006735f42e032ba8e652bc975877e4e9_37.png)

因此，在编写脚本时，如果遇到需要交互的命令，应优先查询其非交互的执行方式。

![](img/006735f42e032ba8e652bc975877e4e9_39.png)

## 实践：编写一个系统信息查看脚本

现在，让我们应用所学知识，编写一个查看基本系统信息的脚本。这个脚本将执行多条命令，并将结果输出到屏幕。

以下是脚本内容示例，我们将其保存为 `sys_info.sh`：

![](img/006735f42e032ba8e652bc975877e4e9_41.png)

![](img/006735f42e032ba8e652bc975877e4e9_43.png)

![](img/006735f42e032ba8e652bc975877e4e9_45.png)

```bash
#!/bin/bash

![](img/006735f42e032ba8e652bc975877e4e9_47.png)

![](img/006735f42e032ba8e652bc975877e4e9_49.png)

![](img/006735f42e032ba8e652bc975877e4e9_51.png)

# 查看系统版本信息
echo “第一步：查看系统版本”
cat /etc/centos-release
sleep 2  # 暂停2秒，增强可读性

# 查看内核版本
echo “第二步：查看内核版本”
uname -rs
sleep 2

# 查看根分区磁盘使用情况
echo “第三步：查看根分区使用情况”
df -h /
sleep 2

# 查看指定网卡信息（请根据你的系统替换ens33）
echo “第四步：查看网卡信息”
ifconfig ens33
sleep 2

# 查看主机名
echo “第五步：查看主机名”
hostname
```

**脚本解析**：
*   `#!/bin/bash`：指定Bash作为解释器。
*   `echo “描述信息”`：在每条实际命令前输出提示，使脚本执行过程更清晰。
*   `sleep 2`：让脚本暂停2秒，避免输出刷屏，提升观看体验。
*   其余行都是标准的Linux命令。

![](img/006735f42e032ba8e652bc975877e4e9_53.png)

![](img/006735f42e032ba8e652bc975877e4e9_55.png)

**执行脚本**：
1.  添加执行权限：`chmod +x sys_info.sh`
2.  运行脚本：`./sys_info.sh`

![](img/006735f42e032ba8e652bc975877e4e9_57.png)

![](img/006735f42e032ba8e652bc975877e4e9_59.png)

这个脚本展示了如何将多个命令有序地组织在一起，并通过 `echo` 和 `sleep` 命令改善输出效果，形成一个有用的小工具。

## 脚本的存放与共享

关于脚本文件放在哪里，有以下建议：
*   **个人使用**：可以放在任何你有权限的目录，例如家目录下的某个文件夹。
*   **与他人共享**：如果需要让其他用户也能使用你的脚本，应将其放在一个公共目录（如 `/usr/local/bin` 或你自定义的 `/scripts`），并确保其他用户对该目录和脚本文件有相应的读取和执行权限。
