## `cut`命令：截取文本

`cut`命令用于从文件的每一行中截取指定部分。它主要有两种用法：按列截取和按字符截取。

![](img/1191fc95585fe05d1125fd37ef143b91_14.png)

![](img/1191fc95585fe05d1125fd37ef143b91_16.png)

上一节我们介绍了文本查看工具，本节中我们来看看如何精确地截取文本内容。

![](img/1191fc95585fe05d1125fd37ef143b91_18.png)

![](img/1191fc95585fe05d1125fd37ef143b91_19.png)

**按列截取**的语法是：`cut -d‘分隔符’ -f列号`。例如，`cut -d‘:’ -f3 /etc/passwd`会以冒号为分隔符，输出`/etc/passwd`文件的第三列（即用户ID）。

![](img/1191fc95585fe05d1125fd37ef143b91_21.png)

![](img/1191fc95585fe05d1125fd37ef143b91_23.png)

**按字符截取**的语法是：`cut -c起始位置-结束位置`。例如，`cut -c1-3 /etc/passwd`会输出文件每一行的前三个字符。这个功能在某些特定场景下有用，比如截取随机生成的字符串中的某一段。

### `sort`命令：排序文本

`sort`命令用于对文本行进行排序。默认按每行的首个字符的ASCII码值进行升序排列。

![](img/1191fc95585fe05d1125fd37ef143b91_51.png)

以下是`sort`命令的核心选项和用法：
*   `sort -r`：进行降序排序。
*   `sort -t‘分隔符’ -k列号`：指定分隔符和排序列。例如，`sort -t‘:’ -k3 /etc/passwd`会以冒号为分隔符，按第三列排序。
*   `sort -n`：按数值大小而非字符串顺序进行排序。结合`-k`选项，`sort -t‘:’ -k3 -n /etc/passwd`能正确按用户ID的数值大小排序。
*   `sort -u`：在排序的同时，去除重复行。
*   `sort 文件1 文件2`：可以对多个文件一起进行排序。

![](img/1191fc95585fe05d1125fd37ef143b91_53.png)

![](img/1191fc95585fe05d1125fd37ef143b91_54.png)

![](img/1191fc95585fe05d1125fd37ef143b91_56.png)

`sort`命令用途广泛，例如，我们可以用`ps aux | sort -k3 -rn | head -5`来找出CPU占用率最高的前5个进程（`-rn`表示按数值降序）。

![](img/1191fc95585fe05d1125fd37ef143b91_58.png)

![](img/1191fc95585fe05d1125fd37ef143b91_60.png)

## `uniq`命令：报告或忽略重复行

`uniq`命令用于处理相邻的重复行。它通常与`sort`命令结合使用。

![](img/1191fc95585fe05d1125fd37ef143b91_62.png)

`uniq`命令默认会**删除相邻的重复行**。如果文件中的重复行不相邻，则需要先使用`sort`排序，再用`uniq`处理。`sort -u`的效果等同于`sort file | uniq`。

![](img/1191fc95585fe05d1125fd37ef143b91_64.png)

`uniq`命令的一个有用选项是`-c`，它可以在行首显示该重复行出现的次数。例如，`sort file | uniq -c`可以用于统计文件中每个不同行出现的频率，这在分析日志、统计词频等场景中非常实用。

![](img/1191fc95585fe05d1125fd37ef143b91_66.png)

![](img/1191fc95585fe05d1125fd37ef143b91_68.png)

## `diff`命令：比较文件差异

`diff`命令用于比较两个文件的差异，并输出哪些行有所不同。这对于对比配置文件修改前后版本非常有用。

基本语法是：`diff 文件1 文件2`。输出结果会指示第一个文件（`文件1`）需要如何修改才能变成第二个文件（`文件2`）。

## `tr`命令：转换或删除字符

![](img/1191fc95585fe05d1125fd37ef143b91_74.png)

![](img/1191fc95585fe05d1125fd37ef143b91_75.png)

![](img/1191fc95585fe05d1125fd37ef143b91_76.png)

`tr`命令用于转换或删除字符。它从标准输入读取数据，经过转换后输出到标准输出。

![](img/1191fc95585fe05d1125fd37ef143b91_78.png)

![](img/1191fc95585fe05d1125fd37ef143b91_80.png)

基本语法是：`tr ‘原字符集’ ‘目标字符集’`。例如，`cat /etc/passwd | tr ‘:’ ‘\n’`会将文件中的所有冒号替换为换行符。

## `sed`命令：流编辑器

![](img/1191fc95585fe05d1125fd37ef143b91_84.png)

`sed`是一个功能强大的流编辑器，它可以对文本进行过滤、替换、删除、打印等操作，而**默认不修改原文件**（除非使用`-i`选项）。

以下是`sed`命令的一些基本用法：
*   **打印指定行**：`sed -n ‘5,10p’ /etc/passwd`会静默模式（`-n`）下打印第5到第10行。
*   **查找与替换**：`sed ‘s/查找模式/替换内容/’ file`是常见的替换语法。例如，`sed ‘s/root/admin/’ /etc/passwd`会将每行第一个`root`替换为`admin`。
*   **修改原文件**：使用`-i`选项会直接修改原文件。为了安全，常配合备份使用，如`sed -i.bak ‘s/old/new/’ file`，这会在修改前将原文件备份为`file.bak`。

`sed`的工作原理是将文件内容读入“模式空间”（一块内存缓冲区），在缓冲区中进行编辑操作，然后默认输出缓冲区内容。加`-n`选项是为了禁止默认输出，只输出我们明确指定的部分（如`p`命令）。`-i`选项则是将缓冲区的内容写回原文件。

![](img/1191fc95585fe05d1125fd37ef143b91_86.png)

本节课中我们一起学习了`cut`、`wc`、`sort`、`uniq`、`diff`、`tr`和`sed`这七个核心的文本处理工具。它们各有专长，通过管道（`|`）组合使用，可以解决Linux环境下绝大多数文本处理需求。掌握这些工具是成为高效Linux用户或管理员的关键一步。
