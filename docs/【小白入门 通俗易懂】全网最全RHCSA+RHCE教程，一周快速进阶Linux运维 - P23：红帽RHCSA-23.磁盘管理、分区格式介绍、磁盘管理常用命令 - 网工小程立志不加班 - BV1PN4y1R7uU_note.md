## 磁盘基础概念 💽

上一节我们介绍了Linux系统的基本操作，本节中我们来看看计算机中存储数据的核心硬件——磁盘。

磁盘是用于存储数据的物理设备。在企业中，如果使用云主机，你可能看不到物理硬盘的样子。在传统的物理服务器上，硬盘的外观类似一块板砖，可以插入服务器机箱的专用插槽中。现代服务器硬盘通常支持热插拔，这意味着可以在服务器开机运行时直接插入或拔出硬盘，就像使用U盘一样。

硬盘内部主要由以下几个部件组成：
*   **主轴马达**：带动盘片旋转。
*   **盘片**：用于存储数据的圆形碟片，数据就保存在盘片的磁道上。
*   **机械手臂与读写磁头**：机械手臂带动读写磁头在高速旋转的盘片上移动，进行数据的读取和写入。

硬盘内部是真空的，以防止灰尘进入导致盘片损坏和数据丢失。

在逻辑层面，硬盘空间被划分为许多小的存储单元，称为**扇区**。它是磁盘存储数据的最小单元，默认一个扇区的大小是 **512字节**。操作系统在划分分区时，实际上是在计算需要多少个这样的扇区。

## 磁盘接口与类型 🔌

购买硬盘时，需要注意接口类型，必须与服务器或主板的接口匹配。常见的接口类型有：
*   **IDE**：古老接口，数据传输慢，价格便宜，现已基本淘汰。
*   **SCSI**：早期服务器领域常用接口，数据传输速度快，支持热插拔。
*   **SATA**：目前个人计算机领域主流接口，服务器也有使用。
*   **SAS**：当前服务器领域流行接口，性能优于SATA。
*   **SSD (固态硬盘)**：使用内存机制存储数据，速度远快于传统机械硬盘，价格较高。
*   **NVMe**：一种用于固态硬盘的高速传输协议，能极大提升数据传输速度。

简单来说，硬件性能与投入成本直接相关，更好的硬件通常能提供更快的速度和更可靠的性能。

## 分区格式介绍 🗂️

有了硬盘之后，需要对其进行分区才能使用。分区格式决定了如何组织和识别硬盘上的分区。主要有两种格式：

### MBR (主引导记录) 格式
这是一种较古老的分区格式。
*   **分区数量限制**：一块硬盘最多只能创建**4个主分区**。为了突破限制，可以将第四个主分区改为**扩展分区**（一个特殊的容器），然后在扩展分区内创建多个**逻辑分区**。
*   **容量限制**：最大只支持 **2.2TB** 以内的硬盘容量。
*   **适用场景**：适用于小容量硬盘或旧系统。

以下是MBR分区结构的示意图：
```
硬盘
├── 主分区1 (可存储数据)
├── 主分区2 (可存储数据)
├── 主分区3 (可存储数据)
└── 扩展分区 (容器，不可直接存数据)
    ├── 逻辑分区5 (可存储数据)
    ├── 逻辑分区6 (可存储数据)
    └── ... (可继续划分更多逻辑分区)
```

### GPT (GUID 分区表) 格式
这是一种现代的分区格式。
*   **分区数量**：理论上可创建**128个主分区**，无需扩展分区和逻辑分区的复杂概念。
*   **容量支持**：支持极大容量的硬盘，最高可达 **18EB** (1EB = 1024PB， 1PB = 1024TB)。
*   **适用场景**：适用于大容量硬盘（超过2.2TB）和新式计算机。

### 1. `lsblk` 命令
这条命令用于列出系统中所有**块设备**（如硬盘、分区）的信息。
```bash
lsblk
```
执行后会显示所有硬盘（如 `sda`, `sdb`）及其下的分区（如 `sda1`, `sda2`）。

![](img/d8b8db1c156c1b87444a1092d31ea3b7_46.png)

![](img/d8b8db1c156c1b87444a1092d31ea3b7_48.png)

![](img/d8b8db1c156c1b87444a1092d31ea3b7_50.png)

**常用选项**：
*   `lsblk -d`：仅显示磁盘本身信息，不显示分区详情。
*   `lsblk -f`：显示文件系统类型、UUID等详细信息。

**通常用法**：查看系统中有多少块硬盘，以及每块硬盘上有哪些分区。

### 2. `df` 命令
这条命令用于报告**文件系统**的磁盘空间使用情况，即查看已挂载分区的使用率。
```bash
df -h
```
*   `-h` 选项表示以人类易读的格式（如K, M, G）显示大小。
*   `-T` 选项可以同时显示文件系统类型。

![](img/d8b8db1c156c1b87444a1092d31ea3b7_72.png)

**通常用法**：查看各个已使用分区的空间还剩多少，使用率是否过高。

![](img/d8b8db1c156c1b87444a1092d31ea3b7_74.png)

### 3. `du` 命令
这条命令用于估算**文件或目录**的磁盘使用空间。
```bash
du -sh /path/to/directory
```
*   `-s` 选项显示指定目录的总大小，而不列出其子目录。
*   `-h` 选项以人类易读的格式显示。

![](img/d8b8db1c156c1b87444a1092d31ea3b7_80.png)

![](img/d8b8db1c156c1b87444a1092d31ea3b7_82.png)

**与 `ls` 的区别**：`ls -lh` 命令在查看目录时，只显示目录条目本身的大小（通常很小），而 `du` 命令会递归计算目录下所有文件和子目录的实际数据总大小。

### 4. `blkid` 命令
这条命令用于显示**块设备**的属性，如UUID和文件系统类型。
```bash
blkid /dev/sda1
```
其功能与 `lsblk -f /dev/sda1` 类似，但 `blkid` 不显示挂载点信息。

**命令小结**：
*   想查看系统有几块硬盘和分区：用 **`lsblk`**。
*   想查看各个分区用了多少空间：用 **`df -h`**。
*   想查看某个目录或文件占多大空间：用 **`du -sh`**。
*   想查看分区的UUID或文件系统类型：用 **`blkid`** 或 **`lsblk -f`**。
