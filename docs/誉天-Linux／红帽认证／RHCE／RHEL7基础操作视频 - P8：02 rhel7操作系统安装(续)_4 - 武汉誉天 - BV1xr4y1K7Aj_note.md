### `/boot`分区
`/boot`分区的作用是存放系统启动过程中所需的文件。换句话说，如果这个分区的文件丢失，系统将无法正常启动。它主要起引导作用。

![](img/a72a042c6f145384270b831d1e1c0516_13.png)

该分区一般分配**500MB**空间即可，实际上200MB也足够，这里我们多分配一些。

![](img/a72a042c6f145384270b831d1e1c0516_15.png)

![](img/a72a042c6f145384270b831d1e1c0516_5.png)

![](img/a72a042c6f145384270b831d1e1c0516_17.png)

500MB空间足够。

![](img/a72a042c6f145384270b831d1e1c0516_19.png)

![](img/a72a042c6f145384270b831d1e1c0516_7.png)

![](img/a72a042c6f145384270b831d1e1c0516_21.png)

添加分区后，可以看到系统自动将其标识为第二个分区（sda2）。理论上它应该是第一个分区，但这里名称发生了变化。

![](img/a72a042c6f145384270b831d1e1c0516_13.png)

### `swap`交换分区
除了上述分区，还有一个非常重要的分区叫做`swap`分区，即交换分区。

![](img/a72a042c6f145384270b831d1e1c0516_21.png)

其他分区我们暂时不创建，后续课程会讲解。

![](img/a72a042c6f145384270b831d1e1c0516_23.png)

`swap`分区的作用类似于Windows系统中的虚拟内存。为了理解它，我们需要先了解CPU、内存和磁盘之间的关系。

计算机组件的基本关系可以简化为：
```
CPU <-> 内存 <-> 磁盘
```

通常，文件存储在磁盘上。当CPU需要打开一个文件时，它不会直接读取速度较慢的磁盘，而是先将文件数据读取到速度更快的内存中，然后CPU再从内存中读取数据。

早期计算机内存容量较小。当同时打开多个文件导致内存不足时，系统可能会卡住或无法打开新文件。`swap`分区就是硬盘上划出的一部分空间，用作内存的扩展。

![](img/a72a042c6f145384270b831d1e1c0516_25.png)

**`swap`分区不能直接当作内存使用**。它的作用是**暂存内存中暂时不被访问的数据**。

举个例子：假设你打开了Word文档等学习资料，占用了部分内存。然后你暂时切换到游戏程序，游戏也需要大量内存。此时，系统可能会将暂时不用的学习资料数据从内存移动到`swap`分区，从而为游戏腾出内存空间。当你切回学习界面时，系统再将这些数据从`swap`分区移回内存。

![](img/a72a042c6f145384270b831d1e1c0516_27.png)

所以，交换分区是内存的临时“后备仓库”。

![](img/a72a042c6f145384270b831d1e1c0516_28.png)

![](img/a72a042c6f145384270b831d1e1c0516_25.png)

![](img/a72a042c6f145384270b831d1e1c0516_30.png)

![](img/a72a042c6f145384270b831d1e1c0516_32.png)

![](img/a72a042c6f145384270b831d1e1c0516_27.png)
![](img/a72a042c6f145384270b831d1e1c0516_28.png)

![](img/a72a042c6f145384270b831d1e1c0516_38.png)

## 用户配置
点击右下角的“开始安装”按钮。

![](img/a72a042c6f145384270b831d1e1c0516_48.png)

![](img/a72a042c6f145384270b831d1e1c0516_44.png)

在安装过程中，需要创建两个用户。

![](img/a72a042c6f145384270b831d1e1c0516_50.png)

首先需要为**root用户**设置密码。root用户是Linux系统的超级管理员，权限极大，类似于Windows中的`Administrator`。

![](img/a72a042c6f145384270b831d1e1c0516_46.png)

Linux系统默认不推荐直接使用root用户登录，因此**必须创建一个普通用户**。系统倾向于使用普通用户进行日常操作。

![](img/a72a042c6f145384270b831d1e1c0516_52.png)

![](img/a72a042c6f145384270b831d1e1c0516_48.png)

设置root密码。如果密码过于简单，系统会提示，需要点击两次确认。

接着，创建一个普通用户，例如用户名为“张三”，并为其设置密码。

![](img/a72a042c6f145384270b831d1e1c0516_50.png)

此时，安装界面会显示进度条，表示正在安装软件包。数字（如1189）代表需要安装的软件包总数。如果选择“最小化安装”，则只有300多个包，安装速度更快，但不会安装图形界面。

![](img/a72a042c6f145384270b831d1e1c0516_52.png)

**补充说明**：如果在分区时选择“自动分区”，系统会使用**逻辑卷（LVM）** 方式进行分区，你将看不到详细的手动分区过程。对于服务器管理等高级场景，手动分区是重要的技能。

![](img/a72a042c6f145384270b831d1e1c0516_54.png)

## 总结
本节课我们一起学习了RHEL7安装过程中手动分区的关键步骤。我们明确了：
*   **`/boot`分区**：用于系统引导，建议分配500MB。
*   **`/home`分区**：用户家目录，用于存放私人文件。
*   **`swap`交换分区**：作为内存的扩展，用于暂存不活跃的数据，类似于Windows的虚拟内存。
*   **用户创建**：必须为root用户设置密码，并创建一个普通用户以供日常使用。

![](img/a72a042c6f145384270b831d1e1c0516_54.png)

通过手动分区，你可以更精细地控制磁盘空间的布局，这对于系统管理和性能优化非常重要。安装过程的剩余步骤将在后续课程中继续。
