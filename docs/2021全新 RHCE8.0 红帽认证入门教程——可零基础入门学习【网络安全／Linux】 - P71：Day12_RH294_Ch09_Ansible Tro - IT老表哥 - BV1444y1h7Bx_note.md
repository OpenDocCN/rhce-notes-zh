# Ansible 自动化运维：第九章：故障排除 🐛

![](img/98a2e6c514aa85875b943d366d20d80c_0.png)

在本章中，我们将学习 Ansible 剧本执行过程中遇到问题时，如何进行有效的故障排除。掌握这些技巧能帮助你快速定位和解决问题。

![](img/98a2e6c514aa85875b943d366d20d80c_2.png)

上一节我们介绍了 Ansible 的各种模块和高级用法，本节中我们来看看当剧本运行不顺利时，我们可以采取哪些排查手段。

## 日志路径查看

![](img/98a2e6c514aa85875b943d366d20d80c_4.png)

第一个方法是查看日志。Ansible 允许我们定义日志路径来记录执行过程。

![](img/98a2e6c514aa85875b943d366d20d80c_5.png)

例如，可以在配置中定义一个日志路径 `s5.logg`，之后便可以通过该路径查看对应的日志来排查问题。

## 使用 Debug 模块

Debug 模块我们已经讲过了，它有两种主要用途：
*   使用 `msg` 参数直接在屏幕上显示指定内容。
*   将输出保存到变量中，然后再导出显示。

![](img/98a2e6c514aa85875b943d366d20d80c_7.png)

## 语法检测

![](img/98a2e6c514aa85875b943d366d20d80c_9.png)

使用 `ansible-playbook --syntax-check` 命令可以检测剧本的语法是否正确，这是一个很实用的预检查步骤。

![](img/98a2e6c514aa85875b943d366d20d80c_11.png)

## 分布调试

使用 `ansible-playbook --step` 命令可以进入分布调试模式。

![](img/98a2e6c514aa85875b943d366d20d80c_13.png)

以下是该模式下的操作选项：
*   按 `y`：执行当前任务。
*   按 `n`：跳过当前任务。
*   按 `c`：继续执行剩余所有任务，不再询问。

![](img/98a2e6c514aa85875b943d366d20d80c_15.png)

## 从指定任务开始运行

![](img/98a2e6c514aa85875b943d366d20d80c_17.png)

使用 `--start-at-task` 参数可以从剧本中指定的某个任务开始执行。

![](img/98a2e6c514aa85875b943d366d20d80c_18.png)

命令格式为：`ansible-playbook --start-at-task="任务名称"`。这样只会运行该任务及其之后的内容。

![](img/98a2e6c514aa85875b943d366d20d80c_20.png)

![](img/98a2e6c514aa85875b943d366d20d80c_22.png)

## 显示详细过程

使用 `-v`、`-vv`、`-vvv` 参数可以显示越来越详细的执行过程信息，有助于观察剧本执行的每一个细节。

![](img/98a2e6c514aa85875b943d366d20d80c_24.png)

## 冒烟测试（空运行）

![](img/98a2e6c514aa85875b943d366d20d80c_25.png)

![](img/98a2e6c514aa85875b943d366d20d80c_26.png)

使用 `--check` 参数可以进行“冒烟测试”或“空运行”。此模式不会在实际主机上做出任何更改，仅模拟执行过程，用于预测剧本运行结果。

![](img/98a2e6c514aa85875b943d366d20d80c_28.png)

## 使用 URI 模块检测服务

![](img/98a2e6c514aa85875b943d366d20d80c_30.png)

![](img/98a2e6c514aa85875b943d366d20d80c_32.png)

URI 模块可用于检测 URL 的访问状态，常用来验证 Web 服务是否正常。

![](img/98a2e6c514aa85875b943d366d20d80c_34.png)

以下是一个示例剧本，用于检测 Nginx 服务能否正常访问：

![](img/98a2e6c514aa85875b943d366d20d80c_36.png)

```yaml
- name: Test web service
  hosts: web_servers
  tasks:
    - name: Check Nginx status
      uri:
        url: "http://{{ inventory_hostname }}/"
      register: nginx_result

    - name: Output result
      debug:
        msg: "OK"
      when: nginx_result.status == 200
```

![](img/98a2e6c514aa85875b943d366d20d80c_38.png)

这个剧本的作用是：
1.  向受管主机的根 URL 发起请求。
2.  将请求结果注册到变量 `nginx_result` 中。
3.  判断结果中的状态码 (`status`) 是否为 200，如果是则输出 “OK” 信息。

## 使用 Script 模块

![](img/98a2e6c514aa85875b943d366d20d80c_40.png)

![](img/98a2e6c514aa85875b943d366d20d80c_41.png)

Script 模块用于在受管主机上运行控制主机上的脚本。

![](img/98a2e6c514aa85875b943d366d20d80c_43.png)

如果脚本的退出状态码非零，则任务会被标记为失败。例如，一个脚本 `test.sh` 的内容是 `echo “Hello Ansible”`，如果执行失败（返回非零码），则任务失败。

![](img/98a2e6c514aa85875b943d366d20d80c_45.png)

## 使用 Fail 模块

![](img/98a2e6c514aa85875b943d366d20d80c_47.png)

Fail 模块是 `when` 条件的另一种替代选择，用于主动让任务失败。它通常与条件判断 `when` 一起使用，用于指定任务失败的条件。

![](img/98a2e6c514aa85875b943d366d20d80c_48.png)

## 连接故障排查

![](img/98a2e6c514aa85875b943d366d20d80c_50.png)

![](img/98a2e6c514aa85875b943d366d20d80c_52.png)

当 Ansible 执行任务失败或无法连接受管主机时，首先应检查连接问题。

![](img/98a2e6c514aa85875b943d366d20d80c_54.png)

以下是连接故障的主要检查要点：
1.  **远程用户**：检查是否指定了正确的远程连接用户。
2.  **权限提升**：如果使用特权用户（如 root），检查是否正确配置了提权方式（如 `become`）。
3.  **受管主机配置**：检查受管主机上的 SSH 服务配置（如端口、密钥认证等），这一点常被忽略。如果预先的配置没有做好，连接肯定会出问题。

![](img/98a2e6c514aa85875b943d366d20d80c_56.png)

## 使用 Ping 和 Command 模块

![](img/98a2e6c514aa85875b943d366d20d80c_58.png)

`ping` 模块用于测试主机连通性，`command` 或 `shell` 模块用于执行命令，这些基础模块在故障排查时也经常用到，我们之前已经详细讲解过。

**本章总结**

本节课中我们一起学习了 Ansible 故障排除的多种方法。故障排除并不算难，核心在于知道出问题时该如何着手调查。主要思路包括：查看日志、使用 debug 输出、进行语法检查和空运行测试，以及利用 `-v` 参数查看详细过程。掌握这些技巧，你就能更加从容地应对剧本执行中的各种问题。
