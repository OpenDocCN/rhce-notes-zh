# Linux运维培训教程：P21：find文件查找命令详解 🔍

在本节课中，我们将要学习Linux系统中一个非常强大的文件查找命令——`find`。`find`命令能够根据多种条件（如文件名、类型、大小、时间等）在指定目录及其子目录中递归地搜索文件，并支持对搜索结果进行进一步处理。掌握`find`命令对于系统管理和日常运维工作至关重要。

## 命令格式与基本查找

`find`命令的基本格式是：`find [路径] [查找条件]`。首先，你需要指定一个搜索的起始路径。

上一节我们介绍了命令的基本概念，本节中我们来看看如何按照不同类型进行查找。

### 按文件类型查找

`find`命令可以通过`-type`选项指定要查找的文件类型。常见的类型包括：
*   `f`：普通文件
*   `d`：目录
*   `l`：符号链接文件

以下是按类型查找的示例：
```bash
# 在 /var/log 目录下查找所有普通文件
find /var/log -type f

![](img/c1693e9994f6e86a0e206562f488fbef_1.png)

![](img/c1693e9994f6e86a0e206562f488fbef_3.png)

![](img/c1693e9994f6e86a0e206562f488fbef_5.png)

# 在 /var/log 目录下查找所有目录
find /var/log -type d

# 在 /etc 目录下查找所有符号链接文件
find /etc -type l
```
`find`命令会进行**递归查找**，即它不仅搜索指定路径下的文件，还会深入该路径下的所有子目录进行搜索。

### 按文件名查找

更常见的需求是按文件名进行查找，这需要使用`-name`选项。你可以使用通配符（如`*`）进行模糊匹配。

以下是按文件名查找的示例：
```bash
# 在 /var/log 目录下递归查找所有以 .log 结尾的文件
find /var/log -name "*.log"

# 在 /etc 目录下递归查找所有以 .conf 结尾的文件
find /etc -name "*.conf"
```
**注意**：`find -name` 是根据**文件名**在文件系统中查找文件；而 `grep` 命令是根据**文件内容**进行文本过滤。两者功能不同，请勿混淆。

![](img/c1693e9994f6e86a0e206562f488fbef_7.png)

与`ls`命令不同，`ls`只列出指定目录下的内容（非递归），而`find`默认进行递归查找，因此`find`的搜索结果通常更全面。

![](img/c1693e9994f6e86a0e206562f488fbef_9.png)

你还可以使用`-iname`选项进行不区分大小写的文件名查找。

## 组合条件与按属性查找

除了单一条件，`find`命令允许你组合多个条件进行更精确的查找。

### 按文件大小查找

使用`-size`选项可以根据文件大小进行查找。单位可以是`k`（KB），`M`（MB），`G`（GB）。`+`号表示“大于”，`-`号表示“小于”。

以下是按文件大小查找的示例：
```bash
# 查找 /var/log 目录下大于10KB的文件
find /var/log -size +10k -type f

# 查找 /var/log 目录下小于10KB的文件
find /var/log -size -10k -type f
```
你可以使用`du -h 文件名`命令来验证文件的实际大小。

### 组合条件：与（-a）和或（-o）

你可以使用`-a`（and，逻辑与）和`-o`（or，逻辑或）来连接多个查找条件。

![](img/c1693e9994f6e86a0e206562f488fbef_19.png)

![](img/c1693e9994f6e86a0e206562f488fbef_21.png)

![](img/c1693e9994f6e86a0e206562f488fbef_23.png)

![](img/c1693e9994f6e86a0e206562f488fbef_25.png)

以下是组合条件查找的示例：
```bash
# 查找大小在10KB到20KB之间的文件（必须同时满足）
find /var/log -size +10k -a -size -20k -type f

# 查找以 .log 结尾 或 小于10KB 的文件（满足其一即可）
find /var/log -name "*.log" -o -size -10k -type f
```

使用`-user`选项可以查找属于特定用户的文件。

以下是按用户查找的示例：
```bash
# 在根目录 / 下查找属于用户“laowang”的所有文件
find / -user laowang
```
搜索结果会显示该用户在系统中所有位置的文件。

### 按文件修改时间查找

使用`-mtime`选项可以根据文件的最后修改时间进行查找。`+n`表示`n`天之前，`-n`表示`n`天之内。

以下是按时间查找的示例：
```bash
# 查找 /var/log 目录下10天前被修改过的文件
find /var/log -mtime +10 -type f

# 查找 /var/log 目录下10天内被修改过的文件
find /var/log -mtime -10 -type f

![](img/c1693e9994f6e86a0e206562f488fbef_41.png)

# 查找 /var/log 目录下当天（24小时内）被修改过的文件
find /var/log -mtime 0 -type f
```
对于更精细的时间控制（如几分钟内），可以使用`-cmin`选项（按分钟查找修改时间）。

![](img/c1693e9994f6e86a0e206562f488fbef_43.png)

## 对查找结果执行操作

`find`命令的强大之处在于，它不仅能找到文件，还能直接对找到的结果执行操作。这是通过`-exec`选项实现的，其格式为：`-exec 命令 {} \;`。其中，`{}`代表`find`找到的每一个文件，`\;`表示命令结束。

**注意**：`find`命令的输出不能直接通过管道（`|`）传递给某些命令（如`cp`, `mv`, `rm`）进行处理，必须使用`-exec`。

以下是对查找结果执行操作的示例：
```bash
# 1. 将找到的文件备份到 /opt 目录
find /var/log -name "*.log" -type f -exec cp {} /opt \;

# 2. 删除找到的文件（请谨慎使用！）
# find /var/log -name "*.tmp" -type f -exec rm {} \;

![](img/c1693e9994f6e86a0e206562f488fbef_45.png)

# 3. 清空找到的文件内容（利用 /dev/null 黑洞设备）
find /var/log -name "*.log" -size +10k -type f -exec cp /dev/null {} \;
```
**清空文件内容技巧**：Linux系统有一个特殊的设备文件`/dev/null`，被称为“黑洞”。任何写入它的数据都会消失。通过将`/dev/null`的内容“拷贝”到目标文件，即可实现清空文件内容的目的。
