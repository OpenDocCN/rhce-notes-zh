## wc统计命令

`wc`命令用于统计文件中的行数、单词数和字节数，是日常文本处理中非常常用的工具。

命令的基本格式为：`wc [选项] 文件名`。

![](img/9290079bdc17ff9efc8a458c31658b23_15.png)

以下是`wc`命令常用的选项：
*   `-l`：仅统计行数。
*   `-w`：仅统计单词数（以空格分隔的字符串）。
*   `-c`：仅统计字节数。

![](img/9290079bdc17ff9efc8a458c31658b23_17.png)

例如，统计`/etc/passwd`文件的信息：
```bash
wc /etc/passwd
```
输出会显示该文件的行数、单词数和字节数。

![](img/9290079bdc17ff9efc8a458c31658b23_19.png)

如果我们只关心文件有多少行，可以这样使用：
```bash
wc -l /etc/passwd
```
这条命令会直接输出文件的总行数，这是`wc`命令最常用的功能。

## 管道符 `|`

![](img/9290079bdc17ff9efc8a458c31658b23_25.png)

![](img/9290079bdc17ff9efc8a458c31658b23_27.png)

![](img/9290079bdc17ff9efc8a458c31658b23_29.png)

上一节我们介绍了如何统计文件内容，本节中我们来看看如何将多个命令串联起来，实现更复杂的功能，这就需要用到管道符。

![](img/9290079bdc17ff9efc8a458c31658b23_31.png)

![](img/9290079bdc17ff9efc8a458c31658b23_33.png)

管道符`|`的功能是将前一个命令的标准输出，作为后一个命令的标准输入。它允许我们将多个简单的命令组合成一个强大的处理流程。

![](img/9290079bdc17ff9efc8a458c31658b23_35.png)

![](img/9290079bdc17ff9efc8a458c31658b23_37.png)

![](img/9290079bdc17ff9efc8a458c31658b23_39.png)

以下是管道符的一些应用示例：
*   **查看文件末尾内容**：`head -10 /etc/passwd | tail -5`。这条命令先取出文件的前10行，再从这个结果中取出末尾5行，最终效果是显示原文件的第6到第10行。
*   **统计过滤后的行数**：`cat -n /etc/passwd | head -10 | tail -5 | wc -l`。这条命令组合先给文件加上行号并全部列出，然后取前10行，再从这10行中取末尾5行，最后统计这5行的行数（结果当然是5）。
*   **查看网络配置的特定信息**：`ifconfig ens32 | head -2`。这条命令查看`ens32`网卡的信息，但只显示前两行（通常是网卡名和IP地址）。

![](img/9290079bdc17ff9efc8a458c31658b23_41.png)

![](img/9290079bdc17ff9efc8a458c31658b23_43.png)

![](img/9290079bdc17ff9efc8a458c31658b23_45.png)

管道符可以无限连接，只要你有需求，就能构建出复杂的命令流水线，它是Linux命令行高效性的核心体现之一。

![](img/9290079bdc17ff9efc8a458c31658b23_47.png)

## 重定向操作

![](img/9290079bdc17ff9efc8a458c31658b23_49.png)

管道符实现了命令之间的数据传递，而重定向操作则用于控制命令的输入输出与文件之间的交互。简单说，重定向可以将命令的输出保存到文件，或者从文件读取数据作为命令的输入。

![](img/9290079bdc17ff9efc8a458c31658b23_51.png)

![](img/9290079bdc17ff9efc8a458c31658b23_53.png)

重定向操作主要通过以下几个符号实现：
*   `>`：覆盖重定向输出。将命令的正确输出结果覆盖写入到指定文件。如果文件不存在，则创建它。
    *   示例：`ifconfig ens32 | head -2 > net_ens32.txt` 将网卡信息的前两行保存到`net_ens32.txt`文件。
*   `>>`：追加重定向输出。将命令的正确输出结果追加到指定文件的末尾。
    *   示例：`free -h >> info.txt` 将内存使用信息追加到`info.txt`文件末尾，不会覆盖原有内容。
*   `2>`：覆盖重定向错误输出。专门收集命令执行失败（错误）的输出信息。
    *   示例：`cat /etc/nonexistentfile 2> error.log` 将查看一个不存在文件时产生的错误信息保存到`error.log`。
*   `2>>`：追加重定向错误输出。以追加方式收集命令的错误输出。
*   `&>`：覆盖重定向所有输出。无论命令执行正确还是错误，其输出都会被收集。
    *   示例：`cat /etc/hostname /etc/nonexistent &> all_output.txt` 将正确和错误的输出都保存到`all_output.txt`。
*   `&>>`：追加重定向所有输出。以追加方式收集命令的所有输出。

![](img/9290079bdc17ff9efc8a458c31658b23_55.png)

![](img/9290079bdc17ff9efc8a458c31658b23_57.png)

**应用场景提示**：在企业中，当你需要批量检测上百台服务器是否在线时，可以使用`ping`命令配合`2>>`。将无法ping通的服务器（产生错误输出）的IP地址追加记录到一个特定文件中，便于后续集中排查故障机。

## echo与sleep命令

最后，我们来学习两个看似简单但非常有用的命令：`echo`和`sleep`。

![](img/9290079bdc17ff9efc8a458c31658b23_65.png)

`echo`命令用于在终端输出指定的字符串或变量值。
命令格式非常简单：`echo [字符串]`。
例如，`echo Hello World`会在屏幕输出“Hello World”。

![](img/9290079bdc17ff9efc8a458c31658b23_67.png)

![](img/9290079bdc17ff9efc8a458c31658b23_69.png)

![](img/9290079bdc17ff9efc8a458c31658b23_71.png)

它的一个常见用法是与重定向结合，快速向文件写入内容或清空文件：
*   `echo “new_hostname” > /etc/hostname`：将主机名直接改为“new_hostname”。
*   `echo “” > file.txt` 或 `echo > file.txt`：快速清空`file.txt`文件的内容。

![](img/9290079bdc17ff9efc8a458c31658b23_73.png)

![](img/9290079bdc17ff9efc8a458c31658b23_75.png)

`sleep`命令用于使当前进程（或脚本）暂停（睡眠）一段时间。
命令格式为：`sleep 时间`。时间单位可以是秒（默认）、分钟（`m`）、小时（`h`）、天（`d`）。
例如：
*   `sleep 5`：暂停5秒。
*   `sleep 0.1`：暂停0.1秒。

![](img/9290079bdc17ff9efc8a458c31658b23_77.png)

![](img/9290079bdc17ff9efc8a458c31658b23_79.png)

![](img/9290079bdc17ff9efc8a458c31658b23_81.png)

![](img/9290079bdc17ff9efc8a458c31658b23_83.png)

`sleep`在编写Shell脚本时特别有用，可以控制命令的执行节奏。一个直观的例子是，在循环中快速执行`echo`命令会瞬间占用大量CPU资源。如果在每次循环中加入`sleep 0.1`，让CPU每次循环后休息0.1秒，就能显著降低CPU使用率，避免系统因单个脚本而过载。

![](img/9290079bdc17ff9efc8a458c31658b23_87.png)
