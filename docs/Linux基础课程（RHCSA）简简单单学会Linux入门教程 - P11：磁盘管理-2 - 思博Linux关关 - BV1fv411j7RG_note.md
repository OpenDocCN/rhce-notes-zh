## MBR与GPT磁盘格式 💽

MBR磁盘格式的存储容量限制在2TB以下，这与它的表结构设计有关。在生产环境中，如果硬盘容量大于2TB，使用MBR格式会遇到问题。通常的解决方案是将大硬盘分成多个小于2TB的分区，或者直接使用GPT格式。

GPT格式没有MBR的2TB容量限制，并且支持更多的主分区。MBR的限制主要源于其分区表仅占用512字节，其中只能记录4个主分区信息。

**核心概念**：MBR使用512字节的引导扇区，其分区表结构限制了寻址能力。

## 文件系统格式化与挂载 🔧

分区创建后，需要格式化为特定的文件系统才能使用。

### 格式化分区

可以使用 `mkfs` 系列命令来格式化分区。例如，将 `/dev/sdb1` 格式化为 ext4 文件系统：

```bash
mkfs.ext4 /dev/sdb1
```

格式化过程会创建文件系统的元数据结构，包括inode表和块位图，因此会占用少量磁盘空间。

### 挂载分区

![](img/e4a01833dd6415ed583f01c155f80e35_5.png)

格式化后，需要将分区挂载到一个目录才能访问：

![](img/e4a01833dd6415ed583f01c155f80e35_7.png)

```bash
mount /dev/sdb1 /mnt
```

![](img/e4a01833dd6415ed583f01c155f80e35_9.png)

挂载后，对 `/mnt` 目录的操作实际上就是对 `/dev/sdb1` 分区的操作。可以使用 `df -h` 命令查看挂载情况和磁盘使用量。

### 验证挂载与写入数据

![](img/e4a01833dd6415ed583f01c155f80e35_11.png)

挂载后，可以向目录写入数据来测试。例如，使用 `dd` 命令创建一个3MB的文件：

![](img/e4a01833dd6415ed583f01c155f80e35_13.png)

```bash
dd if=/dev/zero of=/mnt/test.file bs=1M count=3
```

此命令从 `/dev/zero` 设备读取数据（全是零），写入到 `/mnt/test.file` 文件中，块大小（`bs`）为1MB，共写入3个块（`count=3`）。

## 理解Inode与块 🧩

文件系统的两个核心概念是inode（索引节点）和块（block）。inode存储文件的元数据（如权限、所有者、时间戳、数据块指针），但不包含文件名。文件名存储在目录的条目中，并指向对应的inode。

### 实验：Inode数量限制

我们可以通过一个实验来直观理解inode的限制。首先，格式化一个分区并指定较少的inode数量：

![](img/e4a01833dd6415ed583f01c155f80e35_15.png)

```bash
mkfs.ext3 -N 20 /dev/sdb1
```

![](img/e4a01833dd6415ed583f01c155f80e35_17.png)

然后挂载分区，并使用 `dumpe2fs` 命令查看inode信息：

```bash
dumpe2fs /dev/sdb1 | grep -i “inode”
```

![](img/e4a01833dd6415ed583f01c155f80e35_19.png)

![](img/e4a01833dd6415ed583f01c155f80e35_21.png)

输出会显示该分区总的inode数量（例如32个）和已使用的数量。接着，在挂载点尝试创建超过inode数量的空文件：

```bash
for i in {1..25}; do touch /mnt/file$i; done
```

![](img/e4a01833dd6415ed583f01c155f80e35_23.png)

当创建的文件数超过可用inode时（例如第22个文件），即使磁盘空间还有剩余，系统也会报告“设备上没有空间”。通过 `df -i` 命令可以查看inode的使用率已达到100%。

![](img/e4a01833dd6415ed583f01c155f80e35_24.png)

![](img/e4a01833dd6415ed583f01c155f80e35_26.png)

这个实验表明，磁盘空间耗尽可能有两种情况：一是块（存储空间）用尽，二是inode（文件数量上限）用尽。在实际业务中，inode用尽的情况较少见，但可能发生在产生大量小文件的场景（如邮件队列）。

## 自动挂载配置 ⚙️

![](img/e4a01833dd6415ed583f01c155f80e35_28.png)

手动挂载的分区在重启后会失效。要实现开机自动挂载，需要编辑 `/etc/fstab` 文件。

![](img/e4a01833dd6415ed583f01c155f80e35_30.png)

以下是 `/etc/fstab` 文件条目的格式：

![](img/e4a01833dd6415ed583f01c155f80e35_32.png)

![](img/e4a01833dd6415ed583f01c155f80e35_34.png)

```
<设备或UUID> <挂载点> <文件系统类型> <挂载选项> <dump备份标记> <fsck检查顺序>
```

例如，将 `/dev/sdb1` 自动挂载到 `/mnt/data`：

![](img/e4a01833dd6415ed583f01c155f80e35_36.png)

```
/dev/sdb1 /mnt/data ext4 defaults 0 0
```

![](img/e4a01833dd6415ed583f01c155f80e35_38.png)

![](img/e4a01833dd6415ed583f01c155f80e35_40.png)

其中 `defaults` 挂载选项包含了一系列常用参数：`rw`（读写）、`suid`（允许SUID）、`dev`（允许设备文件）、`exec`（允许执行）、`auto`（开机自动挂载）、`nouser`（禁止普通用户挂载）、`async`（异步I/O）。

配置完成后，可以使用 `mount -a` 命令测试并立即挂载所有在 `/etc/fstab` 中定义但未挂载的设备。

![](img/e4a01833dd6415ed583f01c155f80e35_42.png)

![](img/e4a01833dd6415ed583f01c155f80e35_44.png)

![](img/e4a01833dd6415ed583f01c155f80e35_46.png)

**警告**：编辑 `/etc/fstab` 文件时要非常小心，错误的配置可能导致系统无法正常启动。建议修改前先备份原文件。

![](img/e4a01833dd6415ed583f01c155f80e35_48.png)

![](img/e4a01833dd6415ed583f01c155f80e35_50.png)

## 交换分区管理 🔄

![](img/e4a01833dd6415ed583f01c155f80e35_52.png)

交换分区（swap）在系统物理内存不足时，用作内存的扩展。我们可以使用磁盘分区来创建或扩展交换空间。

### 创建交换分区

1.  使用 `fdisk` 或 `gdisk` 创建一个新分区。
2.  将分区的类型代码更改为 `82`（Linux swap）。
3.  格式化该分区为交换空间：

    ```bash
    mkswap /dev/sdb2
    ```
4.  激活交换分区：

    ```bash
    swapon /dev/sdb2
    ```

![](img/e4a01833dd6415ed583f01c155f80e35_54.png)

![](img/e4a01833dd6415ed583f01c155f80e35_55.png)

使用 `free -m` 或 `swapon -s` 命令可以查看当前生效的交换空间。

![](img/e4a01833dd6415ed583f01c155f80e35_57.png)

### 设置交换分区自动激活

同样，可以将交换分区加入 `/etc/fstab` 以实现开机自动激活：

![](img/e4a01833dd6415ed583f01c155f80e35_59.png)

```
/dev/sdb2 swap swap defaults 0 0
```

![](img/e4a01833dd6415ed583f01c155f80e35_61.png)

## 高级挂载技巧 🛠️

`mount` 命令功能强大，除了挂载磁盘分区，还有多种高级用法。

![](img/e4a01833dd6415ed583f01c155f80e35_63.png)

### 指定挂载选项

挂载时可以使用 `-o` 选项指定参数。例如，以只读（`ro`）方式挂载：

```bash
mount -o ro /dev/sdb1 /mnt
```

### 挂载不同文件系统

*   **光盘/U盘**：通常系统会自动识别。手动挂载光盘设备：`mount /dev/sr0 /mnt`。
*   **NTFS文件系统（来自Windows）**：需要 `ntfs-3g` 支持。`mount -t ntfs-3g /dev/sdc1 /mnt/usb`。
*   **ISO镜像文件**：使用回环（loop）设备挂载。`mount -o loop /path/to/image.iso /mnt/iso`。
*   **网络文件系统（如NFS、Samba）**：`mount -t nfs server:/share /mnt/nfs`。

### 绑定挂载

`mount` 命令的 `--bind`（或 `-B`）选项可以将一个目录或文件“绑定”到另一个位置，实现镜像访问。

**绑定文件**：
```bash
mount --bind /path/to/file1 /path/to/file2
```
执行后，访问 `file2` 看到的内容就是 `file1` 的内容。这并非文件同步，而是一种“覆盖”或“重定向”。卸载绑定后，`file2` 恢复原有内容。

**绑定目录**：
目录也可以进行绑定挂载，原理相同。

**应用场景**：这在某些运维场景中很有用，例如需要临时替换服务的配置文件进行测试。可以将新配置文件绑定到原配置文件的位置，服务重启后即使用新配置。如果出现问题，只需卸载绑定，即可瞬间回滚到旧配置。

**如何识别绑定挂载**：使用 `stat` 命令查看文件的inode号。绑定挂载后，目标文件会显示与源文件相同的inode号（在同一个设备上）。而硬链接（`ln` 命令创建）虽然inode也相同，但文件的链接数会增加。
