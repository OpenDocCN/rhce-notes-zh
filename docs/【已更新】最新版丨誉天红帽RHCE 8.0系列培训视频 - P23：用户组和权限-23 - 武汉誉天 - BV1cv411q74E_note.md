# Linux用户与权限管理：P23：用户组和权限-23

在本节课中，我们将学习Linux系统中用户和组的管理，以及文件权限的基本概念。我们将通过具体的命令操作来理解如何创建、修改、删除用户和组，并探讨权限如何影响进程对文件的访问。

## 用户组管理

上一节我们介绍了用户的基本操作，本节中我们来看看用户组的管理以及如何修改用户属性。

### 修改用户属性

![](img/21189702d3cff42dc02f2a7a46dbd024_1.png)

使用 `usermod` 命令可以修改现有用户的属性。该命令支持多种选项来调整用户信息。

以下是 `usermod` 命令的一些常用选项：
*   `-G`：为用户设置新的附加组列表。注意，这会覆盖用户原有的所有附加组。
*   `-O`：允许将用户的UID修改为一个非唯一的值（即可能已被其他用户占用的UID）。
*   `-U`：修改用户的UID。
*   `-s`：修改用户的登录Shell。

例如，将用户 `user1` 的UID修改为2022：
```bash
usermod -U 2022 user1
```

### 管理用户附加组

![](img/21189702d3cff42dc02f2a7a46dbd024_3.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_5.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_7.png)

为用户添加附加组时，需要注意 `-G` 选项的行为。直接使用 `-G` 会设置新的附加组列表并覆盖旧的。如果需要追加附加组而不覆盖，需要结合 `-a`（append）选项。

![](img/21189702d3cff42dc02f2a7a46dbd024_8.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_10.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_11.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_12.png)

以下是操作示例：
1.  将用户 `user1` 的附加组设置为 `it1`（这会覆盖原有附加组）：
    ```bash
    usermod -G it1 user1
    ```
2.  为用户 `user1` 追加附加组 `it1`（保留原有附加组）：
    ```bash
    usermod -aG it1 user1
    ```
    选项的顺序很重要，`-aG` 是正确的，而 `-Ga` 会导致错误。

### 理解私有组与附加组

![](img/21189702d3cff42dc02f2a7a46dbd024_18.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_20.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_22.png)

*   **私有组（主组）**：创建用户时默认生成的同名组，或通过 `-g` 选项指定的组。用户的 `GID` 指向其私有组。
*   **附加组**：用户所属的其他组。用户可以通过附加组获得额外的权限。

一个用户只能有一个私有组，但可以属于多个附加组。私有组关系如同“亲生父子”，而附加组关系如同用户作为“成员”加入其他组。

## 用户与组的删除操作

理解了如何创建和修改后，我们来看看如何安全地删除用户和组。

### 删除用户

使用 `userdel` 命令删除用户。默认删除操作不会移除用户的家目录和邮件文件，这可能在新创建同名用户时导致权限问题。

以下是删除用户的注意事项：
*   仅删除用户账户：
    ```bash
    userdel username
    ```
*   删除用户的同时，移除其家目录和邮件文件（`/var/mail/username`）：
    ```bash
    userdel -r username
    ```
*   如果删除用户时未使用 `-r` 选项，之后又创建了同名用户，可能会因为遗留文件的所有权问题导致新用户无法正常登录。此时需要手动删除旧的家目录和邮件文件。

![](img/21189702d3cff42dc02f2a7a46dbd024_24.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_26.png)

### 删除组

使用 `groupdel` 命令删除组。需要注意的是，如果某个组是系统中任一用户的私有组（主组），则该组无法被删除。

例如，尝试删除用户 `user1` 的私有组 `user1` 会失败：
```bash
groupdel user1 # 这将报错，因为user1组是user1用户的私有组
```
而删除一个空组或仅作为某些用户附加组的群组则可以成功。

### 管理组成员

![](img/21189702d3cff42dc02f2a7a46dbd024_28.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_30.png)

`gpasswd` 命令可用于管理组，包括为组设置密码。`newgrp` 命令允许用户临时切换其生效的主组到另一个已设置密码的组。

![](img/21189702d3cff42dc02f2a7a46dbd024_32.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_34.png)

1.  为组 `group1` 设置密码：
    ```bash
    gpasswd group1
    ```
2.  用户临时切换到 `group1` 组（需要输入组密码）：
    ```bash
    newgrp group1
    ```
    执行后，用户的当前 `GID` 会临时变为 `group1` 的GID。

## Linux文件安全与权限基础

管理好用户和组之后，我们进入权限部分，了解Linux如何通过权限控制来保障系统安全。

![](img/21189702d3cff42dc02f2a7a46dbd024_36.png)

### 文件所有权

![](img/21189702d3cff42dc02f2a7a46dbd024_38.png)

每个Linux文件都关联着一个所有者和一个所属组，这体现在文件的 `UID` 和 `GID` 上。使用 `ls -l` 命令可以查看这些信息。

![](img/21189702d3cff42dc02f2a7a46dbd024_40.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_42.png)

### 进程权限

![](img/21189702d3cff42dc02f2a7a46dbd024_44.png)

![](img/21189702d3cff42dc02f2a7a46dbd024_46.png)

进程在运行时，会携带其启动用户的身份信息，这决定了进程能访问哪些资源。

核心概念如下：
*   任何进程运行时都带有一个 `UID`（用户身份）和一个或多个 `GID`（组身份）。
*   进程的 `UID` 和 `GID` 通常继承自启动该进程的用户。
*   由于一个用户可以属于多个组（附加组），因此进程也可能携带多个 `GID` 信息。

当进程尝试访问一个文件时，系统会检查进程的 `UID/GID` 与文件的 `UID/GID` 及权限设置，以此决定是否允许访问。例如，由 `root` 用户启动的进程拥有 `root` 的权限，而由普通用户 `user4` 启动的进程则只有 `user4` 的权限。
