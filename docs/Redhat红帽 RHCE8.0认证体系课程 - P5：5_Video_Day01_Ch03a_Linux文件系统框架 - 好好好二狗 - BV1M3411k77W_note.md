Linux文件系统采用树形结构，所有文件和目录都从根目录 `/` 开始。文件并非随意创建，而是遵循文件系统层次结构标准，特定的功能目录应按照其功能存放。本节将详细介绍这些核心目录的用途。

*   **`/etc`**
    *   **用途**：存放系统及应用程序的配置文件。
    *   **示例**：网络配置文件、YUM软件仓库配置文件 (`/etc/yum.repos.d/`) 等都位于此目录。

*   **`/boot`**
    *   **用途**：存放与系统启动相关的文件，包括内核文件、引导加载程序（如GRUB）以及初始化内存盘镜像（initramfs）。
    *   **说明**：系统启动时首先加载此处的文件，然后切换到真正的根目录。

*   **`/dev`**
    *   **用途**：存放设备文件，代表系统中的硬件设备。
    *   **示例**：磁盘设备文件（如 `/dev/nvme0n1p1`）、终端设备等。所有硬件设备相关的文件都存放在这里。

*   **`/root`**
    *   **用途**：系统超级管理员 `root` 用户的**家目录**。
    *   **说明**：存放 `root` 用户的私有配置文件（如环境变量 `.bashrc`）。其访问权限通常设置为仅 `root` 用户及其所属组可读写，其他用户无法访问。

*   **`/home`**
    *   **用途**：存放所有普通用户的**家目录**。
    *   **说明**：每个用户在此目录下有一个以自己用户名命名的子目录（如 `/home/oracle`），用于存放个人文件和私有环境配置。`~` 符号代表当前用户的家目录。

*   **`/var`**
    *   **用途**：存放经常变化的（Variable）数据，例如日志文件、邮件、计划任务等。
    *   **示例**：系统日志通常位于 `/var/log`；邮件数据在 `/var/mail`；周期性计划任务在 `/var/spool/cron`。

*   **`/tmp`**
    *   **用途**：存放临时文件。
    *   **说明**：许多程序在运行或安装时会在此创建临时文件。类似于Windows的 `C:\Windows\Temp`。需要确保此目录有足够的磁盘空间。

*   **`/proc` 与 `/sys`**
    *   **用途**：虚拟文件系统，映射系统内存中的数据，不占用实际磁盘空间。
    *   **说明**：
        *   `/proc`：提供进程和内核信息的接口（如 `/proc/cpuinfo` 显示CPU信息）。
        *   `/sys`：与 `/proc` 类似，但提供了更结构化的内核数据结构访问方式。

    *   **用途**：存放用户安装的应用程序及其相关文件（如命令、库文件、文档等）。
    *   **说明**：可以理解为 “User Software Resources” 的缩写，是系统软件资源的主要存放地。

    *   **说明**：通常用于安装那些非发行版软件仓库提供的、需要独立目录的大型软件。

## 总结

本节课中我们一起学习了Linux文件系统的核心框架。我们了解到Linux采用树形结构，从根目录 `/` 开始，每个标准目录都有其特定的用途，例如 `/etc` 存放配置，`/home` 存放用户数据，`/var` 存放可变数据等。同时，我们明确了 `/etc`、`/boot` 等关键目录必须与根分区共存的原则。掌握这些知识是后续进行文件管理、系统配置和故障排查的重要基础。
