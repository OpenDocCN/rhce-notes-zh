## 命令的基本语法

![](img/135c602c12f95a9014acca210f68200c_21.png)

![](img/135c602c12f95a9014acca210f68200c_23.png)

上一节我们介绍了如何登录系统和执行简单命令，本节中我们来看看命令的通用语法结构。

Linux命令的通用语法格式如下：
```
command [option(s)] [argument(s)]
```
*   **command**：命令本身，表示要执行的操作，必须放在最前面。
*   **option(s)**：选项，用于修饰命令的行为，通常以 `-` 或 `--` 开头，是可选的。
*   **argument(s)**：参数，是命令操作的具体对象（如文件名、用户名等），也是可选的。

### 命令与选项

![](img/135c602c12f95a9014acca210f68200c_43.png)

命令表示要执行的操作，例如 `ls` 表示列出目录内容，`date` 表示显示日期。

![](img/135c602c12f95a9014acca210f68200c_49.png)

![](img/135c602c12f95a9014acca210f68200c_50.png)

![](img/135c602c12f95a9014acca210f68200c_52.png)

选项用于改变命令的默认行为。选项分为短选项和长选项：
*   **短选项**：通常是一个字母，前面加一个短横线 `-`。例如 `ls -l` 表示以长格式（详细信息）列出文件。
*   **长选项**：通常是一个完整的单词，前面加两个短横线 `--`。例如 `ls --all` 表示列出所有文件（包括隐藏文件）。

![](img/135c602c12f95a9014acca210f68200c_54.png)

![](img/135c602c12f95a9014acca210f68200c_56.png)

![](img/135c602c12f95a9014acca210f68200c_58.png)

![](img/135c602c12f95a9014acca210f68200c_59.png)

多个短选项可以合并使用，只需一个 `-`。例如 `ls -l -a` 可以合并为 `ls -la`。长选项不能合并。

### Tab键自动补全

![](img/135c602c12f95a9014acca210f68200c_96.png)

![](img/135c602c12f95a9014acca210f68200c_98.png)

![](img/135c602c12f95a9014acca210f68200c_100.png)

Tab键可以自动补全命令、选项和文件路径。
*   输入命令或路径的前几个字符后按一次Tab，如果唯一，则会自动补全。
*   如果按一次Tab没有反应，可能是输入有误，或者有多个匹配项。此时按两次Tab，会列出所有可能的匹配项。

### 使用命令历史记录

![](img/135c602c12f95a9014acca210f68200c_110.png)

系统会记录之前执行过的命令，方便我们再次调用。

![](img/135c602c12f95a9014acca210f68200c_112.png)

![](img/135c602c12f95a9014acca210f68200c_114.png)

![](img/135c602c12f95a9014acca210f68200c_116.png)

![](img/135c602c12f95a9014acca210f68200c_118.png)

以下是调用历史记录的几种方法：
*   `history`：查看所有历史命令列表，每条命令前有编号。
*   `!n`：执行历史记录中编号为 `n` 的命令。例如 `!58`。
*   `!string`：执行最近一条以 `string` 开头的命令。例如 `!sys`。
*   上下方向键：在历史命令中逐条翻阅。
*   `Ctrl + R`：反向搜索历史命令。输入关键词后，会动态匹配并显示最近一条包含该关键词的命令。
*   `Alt + .` 或 `Esc + .`：快速调用上一条命令的最后一个参数。

## 如何获取命令帮助 ❓

Linux系统拥有丰富的内置帮助资源，我们不需要记住所有命令的细节，但必须学会如何查找帮助。

### 1. `which` - 查找命令位置
`which` 命令可以显示某个命令对应的可执行文件的具体路径。
```
which ls
```
如果命令不存在，`which` 将不会有输出，这解释了为何有时会报“command not found”错误。

### 2. `whatis` - 显示简短描述
`whatis` 命令可以显示命令的简短功能描述。
```
whatis ls
```
注意：新安装的系统可能需要先运行 `mandb` 命令更新数据库后才能使用 `whatis`。

![](img/135c602c12f95a9014acca210f68200c_126.png)

### 3. `--help` - 获取命令用法摘要
大多数命令都支持 `--help` 选项（或 `-h`），它会输出该命令的用法摘要、选项和参数说明。
```
ls --help
```
`--help` 的输出通常比较简洁，适合快速查阅常用选项。

![](img/135c602c12f95a9014acca210f68200c_128.png)

![](img/135c602c12f95a9014acca210f68200c_130.png)

![](img/135c602c12f95a9014acca210f68200c_132.png)

### 4. `man` - 联机帮助手册（最重要）
`man`（manual的缩写）是Linux系统最权威、最详细的帮助工具。它提供了完整的命令手册。
```
man ls
```
进入man页面后，可以使用以下键进行操作：
*   `空格键` / `Page Down`：向下翻页。
*   `Page Up`：向上翻页。
*   `/关键词`：搜索指定关键词，按 `n` 查找下一个，按 `N` 查找上一个。
*   `q`：退出man页面。

![](img/135c602c12f95a9014acca210f68200c_134.png)

![](img/135c602c12f95a9014acca210f68200c_136.png)

man手册分为多个章节，常用章节有：
*   **1**：用户命令（可执行程序）。
*   **5**：文件格式（配置文件的语法）。
*   **8**：系统管理命令（通常需要root权限）。

### 5. 其他帮助资源
*   `info`：另一种格式的帮助文档，通常比man更详细，结构像一本书。
*   `pinfo`：`info` 的增强版，支持颜色和高亮，浏览更友好。
*   **红帽官方文档**：访问 [Red Hat Customer Portal](https://access.redhat.com)，在“产品文档”中可以找到最权威、最详细的官方指南和参考手册。这是解决复杂问题的重要资源。
