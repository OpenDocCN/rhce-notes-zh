## 跨系统SSH密钥认证

上一节我们介绍了Linux系统之间的密钥认证。本节中，我们来看看如何实现Windows与Linux系统之间的SSH密钥认证。这通常需要借助第三方工具，例如Xshell。

![](img/6822e23845a16ed30ad72830a04d68c1_11.png)

![](img/6822e23845a16ed30ad72830a04d68c1_13.png)

![](img/6822e23845a16ed30ad72830a04d68c1_15.png)

![](img/6822e23845a16ed30ad72830a04d68c1_17.png)

![](img/6822e23845a16ed30ad72830a04d68c1_19.png)

![](img/6822e23845a16ed30ad72830a04d68c1_21.png)

以下是使用Xshell生成密钥对并配置免密登录的步骤：

![](img/6822e23845a16ed30ad72830a04d68c1_23.png)

![](img/6822e23845a16ed30ad72830a04d68c1_25.png)

1.  **生成密钥对**：在Xshell中，点击“工具” -> “新建用户密钥生成向导”。密钥类型通常选择RSA，长度默认为2048位。生成后，会得到公钥和私钥。
2.  **保存私钥**：私钥文件（如 `id_rsa`）需要妥善保存在本地Windows电脑上，用于后续的身份验证。
3.  **部署公钥**：将生成的公钥内容，复制到目标Linux服务器的对应用户家目录下的 `~/.ssh/authorized_keys` 文件中。如果该文件不存在，可以创建它。
    ```bash
    # 在Linux服务器上操作示例
    echo “你的公钥内容” >> ~/.ssh/authorized_keys
    ```
4.  **使用密钥连接**：在Xshell新建会话时，在“用户身份验证”方法中选择“Public Key”，并指定之前保存的私钥文件。连接时，将使用密钥而非密码进行认证。

![](img/6822e23845a16ed30ad72830a04d68c1_27.png)

![](img/6822e23845a16ed30ad72830a04d68c1_29.png)

通过以上步骤，即可实现从Windows到Linux服务器的免密SSH登录。

## 远程文件传输：SCP命令

![](img/6822e23845a16ed30ad72830a04d68c1_37.png)

![](img/6822e23845a16ed30ad72830a04d68c1_39.png)

除了远程登录，OpenSSH还提供了安全的文件传输功能，即SCP命令。它的功能类似于本地的 `cp` 命令，但用于在网络主机间拷贝文件。

![](img/6822e23845a16ed30ad72830a04d68c1_43.png)

![](img/6822e23845a16ed30ad72830a04d68c1_45.png)

![](img/6822e23845a16ed30ad72830a04d68c1_47.png)

![](img/6822e23845a16ed30ad72830a04d68c1_49.png)

![](img/6822e23845a16ed30ad72830a04d68c1_51.png)

![](img/6822e23845a16ed30ad72830a04d68c1_53.png)

以下是SCP命令的基本用法：

![](img/6822e23845a16ed30ad72830a04d68c1_54.png)

![](img/6822e23845a16ed30ad72830a04d68c1_55.png)

![](img/6822e23845a16ed30ad72830a04d68c1_57.png)

![](img/6822e23845a16ed30ad72830a04d68c1_59.png)

![](img/6822e23845a16ed30ad72830a04d68c1_61.png)

![](img/6822e23845a16ed30ad72830a04d68c1_63.png)

![](img/6822e23845a16ed30ad72830a04d68c1_65.png)

![](img/6822e23845a16ed30ad72830a04d68c1_67.png)

*   **从本地拷贝文件到远程主机**：
    ```bash
    scp /本地/文件/路径 root@远程主机IP:/远程/目标/路径
    ```
    *示例*：`scp /root/hello.txt root@192.168.0.112:/root/` 将本地的 `hello.txt` 文件拷贝到远程主机的 `/root/` 目录下。

![](img/6822e23845a16ed30ad72830a04d68c1_69.png)

![](img/6822e23845a16ed30ad72830a04d68c1_71.png)

![](img/6822e23845a16ed30ad72830a04d68c1_73.png)

*   **从远程主机拉取文件到本地**：
    ```bash
    scp root@远程主机IP:/远程/文件/路径 /本地/目标/路径
    ```
    *示例*：`scp root@192.168.0.112:/etc/fstab /opt/` 将远程主机上的 `/etc/fstab` 文件拉取到本地的 `/opt/` 目录下。

**注意**：如果已配置密钥认证，SCP操作将无需输入密码；否则，需要输入远程用户的密码。

## 提高SSH服务安全性

![](img/6822e23845a16ed30ad72830a04d68c1_85.png)

![](img/6822e23845a16ed30ad72830a04d68c1_87.png)

![](img/6822e23845a16ed30ad72830a04d68c1_89.png)

![](img/6822e23845a16ed30ad72830a04d68c1_91.png)

OpenSSH服务的主配置文件是 `/etc/ssh/sshd_config`。通过修改此文件，可以显著提升系统的访问安全。

以下是几个常见的安全配置项：

![](img/6822e23845a16ed30ad72830a04d68c1_93.png)

![](img/6822e23845a16ed30ad72830a04d68c1_95.png)

1.  **修改默认端口**：将 `Port 22` 改为一个非标准端口（如 `Port 2222`），可以减少自动化攻击脚本的扫描。修改后需重启SSH服务 (`systemctl restart sshd`) 并在防火墙开放新端口。
2.  **禁止Root用户直接登录**：找到 `PermitRootLogin` 配置项，将其值改为 `no`。这样可以防止攻击者直接暴力破解root密码。管理员应使用普通用户登录，再通过 `su` 或 `sudo` 提权。
3.  **禁用密码认证，强制使用密钥登录**：找到 `PasswordAuthentication` 配置项，将其值改为 `no`。这要求所有用户都必须使用密钥对才能登录，彻底杜绝密码被暴力破解或窃听的风险。
4.  **设置用户访问控制**：
    *   **白名单**：使用 `AllowUsers` 指令，例如 `AllowUsers alice@192.168.1.0/24 bob`，表示只允许用户alice从指定网段登录，用户bob可以从任何地方登录。
    *   **黑名单**：使用 `DenyUsers` 指令，例如 `DenyUsers charlie`，表示禁止用户charlie登录。

![](img/6822e23845a16ed30ad72830a04d68c1_97.png)

![](img/6822e23845a16ed30ad72830a04d68c1_99.png)

**重要提示**：修改配置文件前建议备份。每次修改后，都需要执行 `systemctl restart sshd` 重启服务以使配置生效。在应用“禁用密码认证”或“禁止root登录”等严格策略前，请务必确保已配置并测试好替代的登录方式（如密钥），以免将自己锁在服务器外。

## 本地主机名解析：/etc/hosts 文件

在进行网络管理时，我们经常需要配置主机名解析。`/etc/hosts` 文件提供了本地的主机名到IP地址的映射，优先级高于DNS查询。

![](img/6822e23845a16ed30ad72830a04d68c1_109.png)

![](img/6822e23845a16ed30ad72830a04d68c1_111.png)

![](img/6822e23845a16ed30ad72830a04d68c1_113.png)

![](img/6822e23845a16ed30ad72830a04d68c1_115.png)

文件格式非常简单：
```
IP地址    主机名或域名
```
例如，添加一行 `192.168.0.112 web-srv`，保存后，在本机上就可以使用 `ping web-srv` 或 `ssh web-srv` 来访问该IP地址对应的服务器。

![](img/6822e23845a16ed30ad72830a04d68c1_117.png)

**Windows系统的对应文件**位于 `C:\Windows\System32\drivers\etc\hosts`，格式相同。修改此文件可以让Windows电脑也使用自定义的主机名进行解析。
