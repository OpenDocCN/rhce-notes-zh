## 反斜杠的转义作用

上一节我们介绍了 Shell 中的特殊字符，本节中我们来看看如何使用反斜杠来“转义”这些字符的特殊含义。

反斜杠 `\` 可以去除其**后面一个字符**的特殊含义，使其被当作普通字符处理。例如，空格在 Shell 中通常是命令或参数的分隔符，但如果我们想将其作为参数的一部分，就需要使用反斜杠。

**代码示例：**
```bash
# 尝试将带空格的文件名作为参数
touch my\ file.txt
```
在这个例子中，反斜杠去掉了后面空格的分隔符功能，Shell 会将 `my file.txt` 作为一个整体参数传递给 `touch` 命令。

![](img/bfb49d535f336a162c787f5e40370e80_9.png)

需要注意的是，反斜杠**只对其紧邻的下一个字符生效**。如果后面有多个特殊字符需要转义，就需要为每个字符都加上反斜杠。

![](img/bfb49d535f336a162c787f5e40370e80_11.png)

![](img/bfb49d535f336a162c787f5e40370e80_13.png)

**代码示例：**
```bash
# 输出一个美元符号
echo \$5.00
```
这里，反斜杠去掉了 `$` 的变量引用功能，使其原样输出为 `$5.00`。

![](img/bfb49d535f336a162c787f5e40370e80_15.png)

## 单引号与双引号的对比

![](img/bfb49d535f336a162c787f5e40370e80_17.png)

理解了反斜杠的用法后，我们来看看引号。引号可以批量处理字符的特殊含义，但单引号和双引号的行为有显著区别。

![](img/bfb49d535f336a162c787f5e40370e80_19.png)

单引号 `‘ ’` 会**去除引号内所有字符的特殊含义**，所有内容都将被原样输出。

![](img/bfb49d535f336a162c787f5e40370e80_21.png)

**代码示例：**
```bash
# 单引号内的所有内容原样输出
echo ‘You cost $5.00’
# 输出：You cost $5.00
```

![](img/bfb49d535f336a162c787f5e40370e80_23.png)

![](img/bfb49d535f336a162c787f5e40370e80_25.png)

双引号 `“ ”` 则会**保留大部分特殊字符的含义**，但会将其内容作为一个整体，防止被空格等分隔符拆开。然而，有四种情况双引号无法去除其特殊含义：

![](img/bfb49d535f336a162c787f5e40370e80_27.png)

![](img/bfb49d535f336a162c787f5e40370e80_29.png)

以下是双引号无法去除特殊含义的四种情况：
1.  **变量引用符 `$`**：双引号内的 `$variable` 仍会被解析为变量值。
2.  **命令替换符 `$()` 或反引号 ``**：双引号内的命令替换仍会执行。
3.  **转义字符 `\` 本身**：双引号内的反斜杠仍具有转义功能。
4.  **历史命令调用符 `!`**：双引号内的 `!` 仍会尝试调用历史命令。

**代码示例对比：**
```bash
# 双引号：$ 的特殊含义被保留，变量被解析
A=100
echo “The value is $A”  # 输出：The value is 100

# 单引号：$ 被当作普通字符
echo ‘The value is $A’  # 输出：The value is $A
```

因此，当需要原封不动地输出字符串时，使用**单引号**。当需要字符串中的变量或命令替换生效，同时又想保持字符串整体性时，使用**双引号**。

## 定义永久生效的环境变量

之前我们学习的本地变量和环境变量在 Shell 会话结束后就会消失。本节中我们来看看如何定义永久生效的变量。

变量定义在磁盘文件中，通过 `source` 命令（或 `.` 命令）加载到当前 Shell 的内存中才能生效。

**代码示例：**
```bash
# 1. 编辑配置文件，添加一行：export MY_VAR=”Hello”
vim /etc/profile

# 2. 使配置在当前 Shell 生效
source /etc/profile

# 3. 检查变量
echo $MY_VAR
```

根据配置文件的位置，变量分为两种作用域：

![](img/bfb49d535f336a162c787f5e40370e80_31.png)

![](img/bfb49d535f336a162c787f5e40370e80_33.png)

![](img/bfb49d535f336a162c787f5e40370e80_34.png)

### 全局环境变量

![](img/bfb49d535f336a162c787f5e40370e80_36.png)

定义在 `/etc/profile` 文件中的环境变量，对所有用户生效。

![](img/bfb49d535f336a162c787f5e40370e80_38.png)

**特点**：任何用户登录系统时，都会读取此文件，因此可以继承其中用 `export` 导出的变量。

![](img/bfb49d535f336a162c787f5e40370e80_40.png)

### 用户环境变量

定义在用户家目录下的 `~/.bash_profile` 或 `~/.bashrc` 文件中的环境变量，仅对该用户生效。

![](img/bfb49d535f336a162c787f5e40370e80_42.png)

**特点**：当该用户登录时，会读取自己的配置文件。如果用户变量与全局变量同名，则**用户变量的值会覆盖全局变量**。

**优先级总结**：`用户环境变量` > `全局环境变量`。

![](img/bfb49d535f336a162c787f5e40370e80_44.png)

## 总结

本节课中我们一起学习了 Bash Shell 的几个核心概念：
1.  **反斜杠 `\`** 用于转义单个特殊字符。
2.  **单引号 `‘ ’`** 强制所有内容原样输出。
3.  **双引号 `“ ”`** 在保持字符串整体性的同时，允许变量替换和命令替换，但无法转义 `$`、`$()`、`\` 和 `!`。
4.  **永久环境变量**通过配置文件定义，使用 `source` 命令加载。`/etc/profile` 用于定义全局变量，`~/.bash_profile` 用于定义用户个人变量。

![](img/bfb49d535f336a162c787f5e40370e80_48.png)

掌握这些知识，你将能更精确地控制 Shell 的行为，并有效地管理系统的运行环境。
