# Linux权限管理：P13：2.08-附加权限 🔐

在本节课中，我们将要学习Linux系统中的附加权限。之前我们介绍了基本的文件权限、归属以及访问控制列表（ACL），它们解决了权限分类不够精细的问题。本节中，我们来看看除了读写执行（RWX）之外，系统还提供的一些特殊权限，它们能实现更高级的访问控制功能。

![](img/9f07926a5822222fcf610bf4945b48dc_1.png)

## 概述 📋

![](img/9f07926a5822222fcf610bf4945b48dc_3.png)

Linux的基本权限机制（用户、组、其他人）有时过于笼统。为了解决更复杂的权限需求，系统引入了访问控制列表（ACL）。同样，基本的RWX权限也可能不够用，因此Linux提供了几种特殊的附加权限，它们通过修改权限位中的执行位（X）来实现特殊功能。

![](img/9f07926a5822222fcf610bf4945b48dc_4.png)

![](img/9f07926a5822222fcf610bf4945b48dc_6.png)

## 特殊权限的类型

Linux系统中有三种主要的特殊权限，它们分别附加在用户、组和其他人的执行位上。

### Set UID (SUID) 🗝️

![](img/9f07926a5822222fcf610bf4945b48dc_8.png)

Set UID权限附加在**文件所有者**的执行位上。当一个可执行程序被设置了SUID权限后，任何用户在执行该程序时，都将临时获得该程序**所有者**的权限。

![](img/9f07926a5822222fcf610bf4945b48dc_10.png)

![](img/9f07926a5822222fcf610bf4945b48dc_12.png)

**作用**：这类似于“尚方宝剑”，让普通用户能借助程序所有者的身份执行特定操作。

**示例**：系统命令 `/usr/bin/passwd` 用于修改用户密码。普通用户需要将新密码写入 `/etc/shadow` 文件，但该文件通常只有root可写。`passwd` 命令通过SUID权限，让执行它的用户临时获得root权限，从而完成密码修改。

**查看与设置**：
使用 `ls -l` 命令查看，所有者的执行位会显示为 `s` 而非 `x`。
```bash
ls -l /usr/bin/passwd
```
输出类似：`-rwsr-xr-x. 1 root root ...`

设置SUID权限的命令是：
```bash
chmod u+s 文件名
```
移除SUID权限：
```bash
chmod u-s 文件名
```

**重要提示**：SUID权限需谨慎使用，错误地赋予普通程序（如文本编辑器）SUID权限可能导致严重的安全风险。RHCSA考试不直接考查设置SUID。

![](img/9f07926a5822222fcf610bf4945b48dc_14.png)

### Set GID (SGID) 📁

![](img/9f07926a5822222fcf610bf4945b48dc_16.png)

Set GID权限可以附加在**文件**或**目录**的**所属组**执行位上。

![](img/9f07926a5822222fcf610bf4945b48dc_18.png)

![](img/9f07926a5822222fcf610bf4945b48dc_20.png)

*   **对文件的作用**：与SUID类似，用户执行该文件时，会临时获得文件**所属组**的权限。此用法较少见。
*   **对目录的作用（重点）**：这是RHCSA的考点。当一个目录被设置了SGID权限后，任何用户在此目录下创建的新文件或子目录，其**所属组会自动继承该目录的所属组**，而不是创建者本人的主要组。

**作用**：便于在团队协作的共享目录中，保证所有新建文件都属于同一个工作组。

**查看与设置**：
使用 `ls -l` 命令查看，所属组的执行位会显示为 `s`。
设置目录的SGID权限：
```bash
chmod g+s 目录名
```

![](img/9f07926a5822222fcf610bf4945b48dc_22.png)

![](img/9f07926a5822222fcf610bf4945b48dc_24.png)

### 粘滞位 (Sticky Bit) 🍬

![](img/9f07926a5822222fcf610bf4945b48dc_26.png)

粘滞位权限附加在**其他人**的执行位上，通常只用于目录。

![](img/9f07926a5822222fcf610bf4945b48dc_28.png)

![](img/9f07926a5822222fcf610bf4945b48dc_30.png)

**作用**：在具有写权限的公共目录（如 `/tmp`）中，粘滞位可以防止用户删除或重命名其他用户的文件（即使他们对该目录有写权限）。用户只能删除自己创建的文件。

**查看与设置**：
使用 `ls -l` 命令查看，其他人的执行位会显示为 `t`。
设置目录的粘滞位：
```bash
chmod o+t 目录名
```

![](img/9f07926a5822222fcf610bf4945b48dc_32.png)

**典型示例**：系统的 `/tmp` 临时目录默认就设置了粘滞位，确保了多用户环境下的文件安全。

## 实战：创建共用目录 🛠️

以下是RHCSA考试中的一个典型题目，综合运用了权限、归属和SGID知识。

**题目要求**：
1.  创建目录 `/home/tools`。
2.  该目录的所属组应为 `admins`。
3.  `admins` 组的成员对该目录应具有读、写、执行访问权限。
4.  其他用户（除root外）没有任何权限。
5.  在此目录下创建的任何文件，其所属组自动设置为 `admins` 组。

**解题步骤与命令**：

![](img/9f07926a5822222fcf610bf4945b48dc_34.png)

![](img/9f07926a5822222fcf610bf4945b48dc_36.png)

首先，我们创建所需的目录。
```bash
mkdir /home/tools
```

![](img/9f07926a5822222fcf610bf4945b48dc_38.png)

![](img/9f07926a5822222fcf610bf4945b48dc_40.png)

接着，修改目录的所属组为 `admins`。
```bash
chown :admins /home/tools
```

![](img/9f07926a5822222fcf610bf4945b48dc_42.png)

![](img/9f07926a5822222fcf610bf4945b48dc_44.png)

然后，设置目录的基本权限：所属组（admins）有全部权限（rwx），其他人无任何权限（---）。所有者root默认拥有全部权限，无需更改。
```bash
chmod g=rwx,o= /home/tools
# 或使用数字表示法：chmod 770 /home/tools
```

最后，为目录设置SGID权限，以实现“新建文件自动继承组”的功能。
```bash
chmod g+s /home/tools
```
设置完成后，使用 `ls -ld /home/tools` 查看，应能看到所属组的执行位是 `s`（如 `drwxrws---`）。

**验证**：
以root身份在 `/home/tools` 下创建文件，检查其所属组是否自动变为 `admins`。
```bash
touch /home/tools/testfile
ls -l /home/tools/testfile
```

## 权限的数字表示法（八进制）🔢

![](img/9f07926a5822222fcf610bf4945b48dc_46.png)

除了 `rwx` 这种符号表示法，权限也可以用三位或四位的八进制数字表示，这通常更为简洁。

**基本规则**：
*   `r` (读) = 4
*   `w` (写) = 2
*   `x` (执行) = 1
*   无权限 = 0

将用户（u）、组（g）、其他人（o）各自的权限值相加，得到三个数字。
例如：权限 `rwxr-xr--` 的计算如下：
*   u: rwx = 4+2+1 = **7**
*   g: r-x = 4+0+1 = **5**
*   o: r-- = 4+0+0 = **4**
因此，对应的数字权限是 **754**。

**设置权限**：
```bash
chmod 754 文件名
```

![](img/9f07926a5822222fcf610bf4945b48dc_48.png)

**包含特殊权限的数字表示法**：
特殊权限作为第四个数字，加在三位基本权限数字之前。
*   SUID = 4
*   SGID = 2
*   粘滞位 = 1

例如，要设置权限为 `rwxrws---`（即基本权限770，并加上SGID）：
*   基本权限 770
*   SGID 对应数字 2
*   完整命令：`chmod 2770 目录名`
