## 特殊权限：SUID、SGID与SBIT

![](img/7015529701a8ea1875b95fdb8df93780_5.png)

上一节我们介绍了用户身份和一般文件权限，本节中我们来看看三种特殊的权限位：SUID、SGID和SBIT。它们为一般权限提供了重要的补充功能。

![](img/7015529701a8ea1875b95fdb8df93780_7.png)

### SUID权限
SUID权限的核心是让程序的执行者临时获取文件所有者的身份。这类似于“钦差大臣”持有“尚方宝剑”，可以临时行使更高权限来完成特定任务。

**设置SUID权限的命令格式如下：**
```bash
chmod u+s 文件名
```
或者使用数字法，在一般权限数字前加上4（例如`4755`）。

![](img/7015529701a8ea1875b95fdb8df93780_9.png)

### SGID权限
SGID权限与SUID类似，但它让程序的执行者临时获取文件所有组的身份。其主要功能是让目录内新建的文件自动继承目录所属组的属性。

![](img/7015529701a8ea1875b95fdb8df93780_10.png)

**设置SGID权限的命令格式如下：**
```bash
chmod g+s 目录名
```
通常会对目录使用`-R`参数进行递归设置。使用数字法时，在一般权限数字前加上2（例如`2755`）。

### SBIT权限（保护位）
SBIT权限，常被称为保护位或粘滞位，用于保护目录内的文件。它确保目录内的文件只能被其所有者或root用户删除，其他用户无法删除。这在公共目录（如`/tmp`）中非常有用。

**设置SBIT权限的命令格式如下：**
```bash
chmod o+t 目录名
```
使用数字法时，在一般权限数字前加上1（例如`1755`）。

## 权限的数字表示法

特殊权限也可以使用数字法表示，并置于一般权限数字之前，形成一个四位数的完整权限表示。

![](img/7015529701a8ea1875b95fdb8df93780_23.png)

以下是特殊权限的数字对应关系：
*   **SUID**：4
*   **SGID**：2
*   **SBIT**：1

![](img/7015529701a8ea1875b95fdb8df93780_25.png)

**转换示例：**
将权限`rws rw- rwt`转换为数字法。
1.  一般权限部分：`rwx`=7，`rw-`=6，`rwt`中的`rw-`=6（`t`是特殊权限位）。
2.  特殊权限部分：所有者位有小写`s`（表示原有`x`权限），值为4；其他位有大写`T`（表示原无`x`权限），值为1。
3.  组合：特殊权限`4+1=5`，一般权限`766`。
4.  最终数字表示为：`5766`。

## 隐藏权限：chattr与lsattr

![](img/7015529701a8ea1875b95fdb8df93780_37.png)

![](img/7015529701a8ea1875b95fdb8df93780_39.png)

除了上述权限，Linux还提供了隐藏权限，需要使用特定命令`chattr`设置和`lsattr`查看。

![](img/7015529701a8ea1875b95fdb8df93780_41.png)

**以下是两个核心命令：**
*   `chattr`：用于设置文件的隐藏属性。
*   `lsattr`：用于查看文件的隐藏属性。

![](img/7015529701a8ea1875b95fdb8df93780_43.png)

**常用参数：**
*   `+a`：设置该属性后，文件内容只能追加，不能删除或修改，常用于保护日志文件。
*   `+i`：设置该属性后，文件不能被删除、修改、重命名或创建链接，提供最高级别的保护。

**设置与取消隐藏权限的示例：**
```bash
# 为文件添加只能追加的属性
chattr +a 文件名
# 为文件添加不可更改的属性
chattr +i 文件名
# 取消文件的不可更改属性
chattr -i 文件名
# 查看文件的隐藏属性
lsattr 文件名
```

## 文件访问控制列表（FACL）

之前介绍的权限都是针对一类人（所有者、组、其他人）生效的。FACL（文件访问控制列表）允许我们对单个用户或用户组进行更精细的权限指派。

**以下是两个核心命令：**
*   `setfacl`：用于设置FACL规则。
*   `getfacl`：用于查看FACL规则。

![](img/7015529701a8ea1875b95fdb8df93780_52.png)

![](img/7015529701a8ea1875b95fdb8df93780_54.png)

**设置FACL的示例：**
假设有一个目录`/public`，我们希望用户`zhangsan`对其拥有完全访问权限，而其他用户无权限。
```bash
# 首先，设置目录的一般权限为000
chmod 000 /public
# 然后，为zhangsan用户设置rwx权限
setfacl -Rm u:zhangsan:rwx /public
```
设置后，使用`ls -ld /public`查看，权限位末尾会有一个`+`号，表示该文件或目录设置了FACL。使用`getfacl /public`可以查看具体的ACL条目。

![](img/7015529701a8ea1875b95fdb8df93780_56.png)

## Linux文件系统目录结构（FHS）

Linux系统遵循文件系统层次结构标准（FHS），将文件和目录组织在一个以根目录（`/`）为起点的树形结构中。理解主要目录的作用至关重要。

以下是核心目录及其作用的简要说明：

*   **`/boot`**：存放系统启动所需的文件，如内核、引导程序。
*   **`/dev`**：存放设备文件。在Linux中，一切皆文件，硬件设备也以文件形式存在于此目录。
*   **`/etc`**：存放系统及各种服务的配置文件。
*   **`/home`**：普通用户的家目录所在地，每个用户拥有一个以用户名命名的子目录。
*   **`/root`**：系统管理员（root）的家目录。
*   **`/bin`, `/sbin`, `/usr/bin`, `/usr/sbin`**：存放可执行命令（二进制程序）的目录。`/sbin`和`/usr/sbin`下的命令通常需要管理员权限。
*   **`/lib`, `/usr/lib`**：存放系统库文件。
*   **`/tmp`**：临时文件目录，所有用户可读写。通常具有SBIT权限，防止用户删除他人文件。
*   **`/var`**：存放经常变动的数据，如日志、数据库文件。
*   **`/mnt`, `/media`**：用于临时挂载存储设备（如光盘、U盘）的目录。
*   **`/opt`**：用于安装第三方应用程序的目录。
*   **`/usr`**：存放用户安装的应用程序、库、文档等。
*   **`/proc`**：一个虚拟文件系统，存放系统运行时的进程和内核信息（如内存、CPU数据），是内存的映射，并非真实磁盘文件。

## 设备文件命名规则

在`/dev`目录下，存储设备有特定的命名规则：
*   **硬盘**：通常以`sd`开头，如`sda`、`sdb`。后面的字母`a`、`b`...表示系统识别到的顺序，**并非绝对对应物理插槽顺序**。
*   **分区**：在硬盘名称后加上数字，如`sda1`、`sdb5`。数字是分区编号。
    *   主分区和扩展分区的编号为`1-4`。
    *   逻辑分区的编号从`5`开始。
*   **光驱**：通常命名为`cdrom`或`sr0`。

**示例分析：**
`/dev/sdb3`
*   `sd`：表示SCSI或SATA类磁盘。
*   `b`：表示这是系统识别到的第二块此类磁盘。
*   `3`：表示该设备上的第三个分区。由于编号在1-4之间，它可能是一个主分区或扩展分区，但**一定不是逻辑分区**。

## 路径：绝对路径与相对路径

描述文件位置有两种方式：
*   **绝对路径**：从根目录（`/`）开始写起的完整路径，如`/home/zhangsan/file.txt`。在任何当前目录下使用绝对路径都能准确定位到文件。
*   **相对路径**：相对于当前工作目录的路径。例如，当前在`/home`目录，要访问`zhangsan`下的文件，相对路径就是`zhangsan/file.txt`。使用`..`表示上一级目录，`.`表示当前目录。
