## 系统重启与许可协议

![](img/c420e689ae4548f2df3d6926b1c88b4b_5.png)

![](img/c420e689ae4548f2df3d6926b1c88b4b_6.png)

操作系统安装完成后，系统会提示重启。重启后，我们将进入操作系统的引导界面。

![](img/c420e689ae4548f2df3d6926b1c88b4b_7.png)

引导进入系统后，首先会遇到 **许可协议** 的提示。系统会显示 `License not accepted`，表示需要接受许可协议才能继续使用产品。

以下是接受许可协议的操作步骤：
1.  点击 **I accept the license agreement** 按钮。
2.  点击 **Done** 按钮确认。

此时，界面可能还会提供创建用户等选项，我们可以暂时跳过，直接点击 **Finish configuration** 来完成初始配置。

![](img/c420e689ae4548f2df3d6926b1c88b4b_9.png)

## 初始系统设置

完成许可协议确认后，系统会进入一系列初始设置向导。

上一节我们完成了许可协议的确认，本节中我们来看看后续的系统个性化设置步骤。

![](img/c420e689ae4548f2df3d6926b1c88b4b_13.png)

以下是初始设置的主要步骤：
1.  **欢迎与隐私设置**：系统会显示欢迎界面，并询问是否启用 **位置服务**。此设置可根据个人偏好选择，例如点击 **Next** 继续。
2.  **连接在线账户**：系统会提示连接在线账户（如 Google、Microsoft 等）。我们可以选择跳过此步骤，点击 **Skip**。
3.  **创建本地用户**：这是关键步骤，需要为系统创建一个用于日常登录的非 root 用户。
    *   如果企业环境需要，可以选择 **Enterprise Login** 进行域账户登录。
    *   对于个人学习或测试，我们创建本地用户。例如，输入用户名 `shangwen`。
    *   点击 **Next** 后，为该用户设置一个符合复杂度要求的密码。
    *   再次点击 **Next**。
4.  **准备就绪**：完成上述步骤后，系统会显示 `Ready to Go` 之类的提示，点击确认即可。

![](img/c420e689ae4548f2df3d6926b1c88b4b_14.png)

![](img/c420e689ae4548f2df3d6926b1c88b4b_15.png)

![](img/c420e689ae4548f2df3d6926b1c88b4b_16.png)

现在，我们就可以使用新创建的用户（如 `shangwen`）登录系统了。此外，在安装系统时我们也设置了 `root` 用户的密码，因此也可以直接使用 `root` 用户登录。

## 图形界面简介

![](img/c420e689ae4548f2df3d6926b1c88b4b_21.png)

成功登录系统后，我们将进入 RHEL 8 的图形桌面环境，默认为 **GNOME**。

初始登录后，系统可能会有一些入门向导，但对于有经验的用户，这些可以忽略。图形界面的基本操作与 Windows 类似，直观易懂。

以下是图形界面中几个核心组件的介绍：
*   **文件管理器**：在 GNOME 中称为 **Files**（或 `nautilus`）。它的功能类似于 Windows 的“我的电脑”，用于浏览和管理家目录（如 `Documents`, `Downloads` 等）及其他位置的文件。
*   **软件中心**：名为 **Software** 的应用，可以查看系统已安装的软件包，并浏览、安装新的应用程序。
*   **终端**：名为 **Terminal** 的应用。这是我们后续学习和运维工作中**最常用**的工具。通过命令行终端，可以执行绝大部分系统管理、配置和故障排查任务。

![](img/c420e689ae4548f2df3d6926b1c88b4b_23.png)

![](img/c420e689ae4548f2df3d6926b1c88b4b_24.png)

![](img/c420e689ae4548f2df3d6926b1c88b4b_25.png)

![](img/c420e689ae4548f2df3d6926b1c88b4b_26.png)

对于准备 RHCE 考试或其他运维工作的学员，需要重点掌握以下两个操作：
1.  打开 **文件管理器**，用于查看考题文档（如 HTML 格式的考试说明）。
2.  打开 **终端**，用于执行考试要求的所有命令行操作。

![](img/c420e689ae4548f2df3d6926b1c88b4b_27.png)

在实际操作或考试中，经常需要在 **终端** 和 **文件管理器/浏览器** 之间进行切换。虽然初始运行时可能因资源分配感觉略慢，但这不影响基本功能的使用。
