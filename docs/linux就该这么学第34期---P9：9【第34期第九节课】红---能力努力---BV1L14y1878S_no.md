## 6.1：Linux存储结构 📂

上一节我们介绍了Linux的基本操作，本节中我们来看看Linux是如何组织和管理文件的。

在Windows系统中，我们使用盘符（如C盘、D盘）来保存数据。但在Linux系统中，没有盘符的概念。Linux的一切都从根目录（`/`）开始，不同的设备会被挂载到不同的目录上以保存数据。

因此，Windows使用盘符的形式，而Linux则从根目录开始。我们可以将根目录理解为万物的起始点，所有文件和设备都从这个目录开始向下延伸。Linux遵循一个名为FHS的协议，该协议划分了每个子目录的作用，并根据这些作用来存放文件。这样做的好处是，当我们需要找到某个文件或某类文件时，可以根据这个协议，大概率在对应的目录名称下找到。

所以，我们现在需要先了解这个协议是如何规范每个子目录含义的。请看下图，我们来详细聊聊。

![](img/46e60561a3406eabb4c12a18f2565145_1.png)

这张图展示了Linux的存储结构拓扑图，或者说是一张示意图。我们可以看到，一切都从根目录开始，这就像一棵倒过来的树，根在上面，下面有很多枝繁叶茂的叶子。这张倒树图展示了我们常见的一些子目录名称。

今天我们将带大家过一遍这些目录，让大家有个印象。当见到这些名称时，能大概知道它们的作用即可。我们会在后续的一个月时间里不断使用和巩固这些知识，加深印象。

### 常见目录详解

以下是Linux根目录下常见子目录及其作用的介绍：

*   **`/root`**：此目录用于保存管理员（`root`用户）的家目录数据。家目录（或主目录）是存储用户私人文件的地方。
*   **`/bin`**：此目录存放二进制可执行文件，即命令文件。`/bin`目录下的命令是所有用户都可以执行的，例如查看目录、查看文件、编辑文件等简单命令。
*   **`/sbin`**：此目录也存放命令文件，但这里的命令只有管理员（`root`）才可以执行，例如 `reboot`（重启）、`poweroff`（关机）等命令。`/bin` 和 `/sbin` 的核心区别在于命令的执行权限。
    > **历史背景**：早期有些书籍解释`/bin`存放与开机无关的命令，`/sbin`存放与开机相关的命令。这种说法源于早期使用软盘存储系统的历史遗留问题，但现代更认可基于权限的划分方式。
*   **`/boot`**：此目录存放与开机引导相关的文件，例如内核文件、GRUB引导菜单等。一般情况下，我们无需手动修改此目录。
*   **`/dev`**：此目录存放设备文件。在Linux“一切皆文件”的哲学下，硬件设备（如硬盘、鼠标、键盘）也会被映射成文件形式存放在这里，方便管理。
*   **`/etc`**：此目录**非常重要**，用于存放系统的配置文件，以及大多数用户自行安装的软件的配置文件。后续课程中，我们部署服务基本上就是修改此目录下的配置文件。
    > **注意**：绝大多数（99%）的配置文件在此目录，但也有例外，例如DNS服务的配置文件可能在`/var/named`目录下。
*   **`/home`**：此目录用于存放**普通用户**的家目录数据。系统会为每个普通用户在此目录下创建一个与其用户名同名的文件夹，用于存放该用户的私人文件。可以将其理解为“公寓楼”，而`/root`目录则是管理员的“独栋别墅”。
*   **`/lib`**：此目录存放系统开机时所需的函数库文件，以及软件运行所需的共享库。一般情况下，用户无需手动管理此目录。
*   **`/media`**：此目录用作设备的挂载点。当我们需要使用光盘、U盘等移动设备时，通常会将其挂载到此目录下的子目录中。
*   **`/mnt`**：此目录与`/media`类似，也常用于临时挂载文件系统。
*   **`/opt`**：此目录用于存放第三方应用程序。用户可以将自己编译或安装的软件放在此目录下。
*   **`/proc`**：此目录是一个**虚拟文件系统**，它存放的是系统运行时进程和内核信息（如CPU、内存使用情况）的映射文件，而非真实的磁盘文件。其中的数据是动态变化的。
*   **`/srv`**：此目录存放一些网络服务的数据文件，但通常此目录为空。
*   **`/sys`**：此目录也是一个虚拟文件系统，用于映射系统硬件和设备驱动信息。
*   **`/tmp`**：此目录存放临时文件。所有用户都可以在此目录中创建文件，其默认权限为`777`（满权限），并带有`sticky bit`保护位，确保用户只能删除自己的文件。适合用于用户间临时共享文件。
*   **`/usr`**：此目录存放用户安装的应用程序和文件，类似于Windows的“Program Files”目录。其下的`/usr/local`常用于存放用户手动编译安装的软件。
*   **`/var`**：此目录存放经常发生变动的数据，例如日志文件（`/var/log`）、数据库文件、网站数据等。

## 6.2：绝对路径与相对路径 🧭

上一节我们了解了目录结构，本节中我们来看看如何在目录树中定位文件。

![](img/46e60561a3406eabb4c12a18f2565145_31.png)

![](img/46e60561a3406eabb4c12a18f2565145_33.png)

在Linux中，定位一个文件有两种方式：**绝对路径**和**相对路径**。

*   **绝对路径**：从根目录（`/`）开始的完整路径。
    *   **特点**：准确性高，无论当前位于哪个目录，使用绝对路径都能唯一确定目标文件。
    *   **缺点**：路径通常较长，书写和输入较繁琐。
    *   **示例**：`/home/linuxprobe/documents/report.txt`
    *   **判断依据**：只要路径以斜杠（`/`）开头，就是绝对路径。

*   **相对路径**：相对于当前工作目录的路径。
    *   **特点**：便捷性高，路径通常较短。
    *   **缺点**：其有效性依赖于当前所在目录。如果当前目录改变，相同的相对路径可能指向不同的文件或无效。
    *   **示例**：假设当前目录是`/home/linuxprobe`，那么 `documents/report.txt` 就是一个相对路径。
    *   **特殊符号**：
        *   `.` 代表当前目录。
        *   `..` 代表上一级目录。

![](img/46e60561a3406eabb4c12a18f2565145_35.png)

**操作示例**：
假设我们要查看 `/home/linuxprobe/test.txt` 文件的内容。

![](img/46e60561a3406eabb4c12a18f2565145_37.png)

![](img/46e60561a3406eabb4c12a18f2565145_39.png)

```bash
# 使用绝对路径 (无论当前在哪都能执行)
cat /home/linuxprobe/test.txt

![](img/46e60561a3406eabb4c12a18f2565145_41.png)

# 使用相对路径 (必须先进入/home/linuxprobe目录)
cd /home/linuxprobe
cat test.txt
```

**路径末尾的斜杠**：在大多数情况下，目录路径末尾是否加斜杠（如 `/home/` 与 `/home`）没有区别，系统会自动处理。但在某些特定场景（如早期 SELinux 管理工具）下可能有要求，现代使用中可视为等同。

## 6.3：设备文件命名规则（udev服务）🔧

上一节我们学会了如何定位文件，本节中我们来看看Linux如何命名硬件设备对应的文件。

我们已知：
1.  Linux中“一切皆文件”。
2.  硬件设备也以文件形式存在于 `/dev` 目录。
3.  如果这些设备文件随意命名（如 `a`, `b`, `c`），用户将无法通过文件名判断其作用。

为了解决这个问题，Linux引入了 **udev 服务**。udev服务负责在系统运行时管理 `/dev` 目录下的设备节点，其核心功能之一就是**根据一套规则命名设备文件，让用户能通过文件名直观了解设备类型**。

### 常见设备文件命名规则

以下是udev服务定义的一些常见设备文件命名前缀：

| 设备类型 | 文件前缀 | 示例 | 说明 |
| :--- | :--- | :--- | :--- |
| **SCSI/SATA/U盘/固态硬盘** | `sd` | `/dev/sda`, `/dev/sdb1` | **最常见**。`sd` 表示SCSI磁盘。 |
| **虚拟化环境中的硬盘** | `vd` | `/dev/vda` | 常用于KVM、Xen等虚拟化平台。 |
| **老式IDE硬盘** | `hd` | `/dev/hda` | 现已较少见。 |
| **软盘驱动器** | `fd` | `/dev/fd0` | 历史设备。 |
| **打印机** | `lp` | `/dev/lp0` | |
| **光盘驱动器** | `cdrom` 或 `sr` | `/dev/cdrom`, `/dev/sr0` | `cdrom` 通常是链接文件。 |
| **磁带机** | `st` | `/dev/st0` | |

### 硬盘设备文件名解析

以最常见的 `/dev/sdb5` 为例，我们来解析其含义：

1.  **`/dev/`**：根据FHS，这是一个**设备文件**。
2.  **`sd`**：根据udev规则，这是一个**SCSI/SATA/U盘或固态硬盘**。
3.  **`b`**：这是**识别编号**。它表示这是系统中**第二个被识别到的**此类存储设备（a=第一，b=第二，依此类推）。**注意**：这个顺序由内核识别顺序决定，可能与物理插槽顺序一致，但并非直接由插槽决定。
4.  **`5`**：这是**分区编号**。编号 **1-4** 保留给**主分区**或**扩展分区**。编号从 **5** 开始，则一定属于**逻辑分区**。

**重要结论**：
*   看到 `/dev/sdX5`（X为字母，5为分区号）这类文件，可以**肯定**它是一个逻辑分区。
*   看到 `/dev/sdX3`（编号为1-4），无法直接判断它是主分区还是扩展分区。
*   分区编号可以由管理员手动指定，因此可能出现“跳跃”的情况（例如，只有分区1和分区5，没有2、3、4）。

## 6.4：文件系统与虚拟文件系统（VFS）💾

上一节我们了解了设备如何命名，本节中我们来看看数据是如何在硬盘上被组织和访问的。

### 文件系统发展简史

Linux支持多种文件系统，不同版本的红帽系统默认文件系统有所不同：

*   **RHEL 5**：`ext3` （在 `ext2` 基础上增加了**日志功能**，防止意外断电导致数据丢失）。
*   **RHEL 6**：`ext4` （性能较 `ext3` 有显著提升）。
*   **RHEL 7/8**：`xfs` （支持超大容量，最大可达18EB，适合大数据和云计算场景）。

> **容量概念**：1 EB = 1024 PB = 1024 * 1024 TB。`xfs` 文件系统的容量上限在可预见的未来都足够使用。

### 文件存储原理

在Linux文件系统（如 `ext4`, `xfs`）中，一个文件在磁盘上由两部分组成：

1.  **inode（索引节点）**：保存文件的**元数据**，包括：
    *   文件权限、所有者、所属组。
    *   文件大小、时间戳（访问、修改、变化时间）。
    *   指向数据块的指针。
2.  **block（数据块）**：实际保存文件**内容**的数据块。默认大小通常为1KB、2KB或4KB。

**工作原理**：
*   如果文件很小（小于1个block），该block剩余空间会被浪费（类似于Windows中“文件大小”和“占用空间”的差异）。
*   如果文件很大，系统会分配多个block，并通过inode中的指针将它们串联起来，形成文件的完整内容。

### 虚拟文件系统（VFS）

用户可能发现，尽管底层有 `ext4`、`xfs` 等不同文件系统，但我们在操作文件时（如 `touch`, `cp`, `rm`）使用的是统一的命令，无需关心底层细节。这要归功于 **Virtual File System，虚拟文件系统**。

**VFS的作用**：
*   它在用户应用程序和具体的文件系统之间提供了一个抽象层。
*   当用户执行文件操作命令时，VFS会将其“翻译”成底层具体文件系统能理解的指令。
*   **核心目的**：**让用户无需关心底层文件系统是什么**，使用统一的接口进行操作。

正因为VFS非常稳定和高效，我们平时几乎感觉不到它的存在，但它却是Linux文件操作兼容性和便利性的基石。

## 6.5：硬盘管理实战：分区、格式化与挂载 🛠️

上一节我们学习了文件系统的理论，本节中我们通过一个完整的流程来管理一块新硬盘。

假设我们新购买并连接了一块硬盘到系统。要使用它，需要经过三个步骤：
1.  **分区**：将大硬盘划分成逻辑上的独立区域。
2.  **格式化**：在分区上创建文件系统，以便操作系统能识别和存储数据。
3.  **挂载**：将分区关联到一个目录，以便用户访问。

### 第一步：分区

分区工具我们使用 `fdisk` 命令。它是一个交互式的分区工具。

**操作流程**（以 `/dev/sdb` 为例）：
```bash
# 1. 使用fdisk操作指定硬盘
fdisk /dev/sdb

![](img/46e60561a3406eabb4c12a18f2565145_51.png)

![](img/46e60561a3406eabb4c12a18f2565145_53.png)

![](img/46e60561a3406eabb4c12a18f2565145_55.png)

![](img/46e60561a3406eabb4c12a18f2565145_57.png)

# 2. 进入交互界面后，输入 `n` 新建分区。
# 3. 选择分区类型：`p` 为主分区，`e` 为扩展分区。这里选 `p`。
# 4. 设置分区编号（1-4），例如 `1`。
# 5. 设置起始扇区，通常直接回车使用默认值。
# 6. 设置结束扇区，可以指定大小，如 `+300M` 表示创建300MB的分区。
# 7. 输入 `p` 查看分区表，确认信息。
# 8. 输入 `w` 保存并退出。
```

![](img/46e60561a3406eabb4c12a18f2565145_59.png)

![](img/46e60561a3406eabb4c12a18f2565145_61.png)

![](img/46e60561a3406eabb4c12a18f2565145_63.png)

![](img/46e60561a3406eabb4c12a18f2565145_65.png)

![](img/46e60561a3406eabb4c12a18f2565145_67.png)

**分区概念回顾**：
*   **主分区**：最多4个，可直接存储数据。
*   **扩展分区**：最多1个，不能直接存数据，它像一个容器，内部可划分多个**逻辑分区**。
*   **逻辑分区**：编号从5开始，在扩展分区内创建，用于存储数据。

![](img/46e60561a3406eabb4c12a18f2565145_69.png)

![](img/46e60561a3406eabb4c12a18f2565145_71.png)

### 第二步：格式化

![](img/46e60561a3406eabb4c12a18f2565145_73.png)

![](img/46e60561a3406eabb4c12a18f2565145_75.png)

格式化是在分区上创建文件系统的过程，使用 `mkfs` 命令。

![](img/46e60561a3406eabb4c12a18f2565145_77.png)

**操作流程**：
```bash
# 将 /dev/sdb1 分区格式化为 ext4 文件系统
mkfs.ext4 /dev/sdb1
```
也可以使用 `mkfs -t ext4 /dev/sdb1` 命令。

![](img/46e60561a3406eabb4c12a18f2565145_79.png)

### 第三步：挂载

![](img/46e60561a3406eabb4c12a18f2565145_81.png)

挂载是将设备文件与一个目录进行关联的操作，使用 `mount` 命令。关联后，用户通过访问该目录即可读写设备中的数据。

![](img/46e60561a3406eabb4c12a18f2565145_83.png)

**操作流程**：
```bash
# 1. 创建一个目录作为挂载点
mkdir /mnt/mydisk

![](img/46e60561a3406eabb4c12a18f2565145_85.png)

![](img/46e60561a3406eabb4c12a18f2565145_87.png)

![](img/46e60561a3406eabb4c12a18f2565145_89.png)

# 2. 执行挂载
mount /dev/sdb1 /mnt/mydisk

![](img/46e60561a3406eabb4c12a18f2565145_91.png)

# 3. 使用 df -h 命令查看挂载结果
df -h
```
此时，访问 `/mnt/mydisk` 目录就相当于访问 `/dev/sdb1` 分区。

![](img/46e60561a3406eabb4c12a18f2565145_93.png)

![](img/46e60561a3406eabb4c12a18f2565145_95.png)

![](img/46e60561a3406eabb4c12a18f2565145_97.png)

**重要提示**：使用 `mount` 命令进行的挂载是**临时生效**的，重启系统后会失效。

![](img/46e60561a3406eabb4c12a18f2565145_99.png)

![](img/46e60561a3406eabb4c12a18f2565145_101.png)

### 配置永久挂载

为了让挂载在系统重启后依然有效，需要编辑 `/etc/fstab` 配置文件。

**文件格式**：
```
<设备文件> <挂载目录> <文件系统类型> <挂载参数> <备份标记> <自检顺序>
```

**编辑示例**：
在 `/etc/fstab` 文件末尾添加一行：
```
/dev/sdb1 /mnt/mydisk ext4 defaults 0 0
```

![](img/46e60561a3406eabb4c12a18f2565145_103.png)

![](img/46e60561a3406eabb4c12a18f2565145_105.png)

**参数解释**：
*   `defaults`：默认挂载参数（包含读写、执行等基本权限）。
*   第一个 `0`：表示不使用 `dump` 工具进行备份。
*   第二个 `0`：表示开机时不进行磁盘检查（`fsck`）。

**使配置生效**：
```bash
# 重新加载 /etc/fstab 文件，或直接重启系统
mount -a
```
