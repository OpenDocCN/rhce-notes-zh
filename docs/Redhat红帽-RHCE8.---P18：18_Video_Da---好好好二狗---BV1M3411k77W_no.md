# Redhat红帽 RHCE8.0认证体系课程：P18：配置和管理SSH服务 🔐

在本节课中，我们将要学习如何配置和管理SSH服务。SSH是远程管理Linux系统最常用且最安全的工具之一。我们将重点讲解SSH的加密原理、如何通过密钥对实现免密认证，以及一些关键的服务器端配置，例如修改默认端口和禁用root登录。

## 概述：SSH与加密基础

![](img/1bbd34f4f1f1073656bd88209010c1a9_1.png)

上一节我们介绍了进程和系统服务的管理。本节中，我们来看看如何安全地进行远程连接。

![](img/1bbd34f4f1f1073656bd88209010c1a9_3.png)

SSH（Secure Shell）是一种加密的网络传输协议，用于在不安全的网络中提供安全的远程登录和其他安全网络服务。它比传统的Telnet协议安全得多，因为Telnet使用明文传输数据，而SSH对通信过程进行了加密。

SSH主要使用两种加密方式：
*   **对称加密**：加密和解密使用同一个密钥。特点是速度快，但安全性相对较低，因为密钥需要在通信双方之间安全地共享。
    *   常见算法：DES, AES
*   **非对称加密**：使用一对密钥，即**公钥**和**私钥**。公钥可以公开给任何人，用于加密数据或验证签名；私钥必须严格保密，用于解密数据或创建签名。这种方式更安全，但速度较慢。
    *   常见算法：RSA, ECDSA

![](img/1bbd34f4f1f1073656bd88209010c1a9_5.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_7.png)

在SSH连接中，非对称加密用于初始的密钥交换和身份验证阶段，之后会协商出一个临时的对称密钥用于加密后续的会话数据，兼顾了安全性和效率。

![](img/1bbd34f4f1f1073656bd88209010c1a9_10.png)

## SSH连接建立过程

理解了加密基础后，我们来看看一次典型的SSH密码认证是如何建立的。

当客户端A尝试连接服务器B时，会发生以下过程：
1.  A向B发起连接请求。
2.  B将自己的**公钥**发送给A。
3.  A收到B的公钥后，会将其保存在本地的 `~/.ssh/known_hosts` 文件中（首次连接时）。这个文件记录了所有已连接过的主机及其公钥。
4.  A使用B的公钥加密一个随机生成的“会话密钥”，然后发送给B。
5.  B使用自己的**私钥**解密，获得“会话密钥”。
6.  此后，双方使用这个“会话密钥”进行对称加密通信。A会提示用户输入在服务器B上的账户密码以完成身份验证。
7.  验证通过后，连接建立。

![](img/1bbd34f4f1f1073656bd88209010c1a9_14.png)

`known_hosts` 文件的作用是：当再次连接同一台主机时，客户端会比对接收到的公钥和本地保存的是否一致。如果一致，则跳过警告直接进入密码验证；如果不一致（可能遭遇中间人攻击或服务器密钥变更），则会发出严重警告。

![](img/1bbd34f4f1f1073656bd88209010c1a9_16.png)

## 实现基于密钥的免密认证 🔑

使用密码认证每次都需要手动输入，且存在被暴力破解的风险。更安全、更便捷的方式是使用**密钥对认证**。

其核心思想是：将客户端的公钥上传到服务器上。当客户端连接时，服务器会用这个公钥发起一个挑战，客户端用对应的私钥应答。验证通过则无需输入密码。

以下是建立从服务器A到服务器B的免密认证步骤：

![](img/1bbd34f4f1f1073656bd88209010c1a9_18.png)

**1. 在客户端（服务器A）生成密钥对**
使用 `ssh-keygen` 命令生成密钥对。通常使用更安全的ECDSA或RSA算法。
```bash
ssh-keygen -t ecdsa
```
命令执行后，会询问密钥保存路径（默认 `~/.ssh/id_ecdsa`）和私钥密码（可直接回车留空，实现完全免密）。生成的公钥文件通常以 `.pub` 结尾（如 `id_ecdsa.pub`）。

![](img/1bbd34f4f1f1073656bd88209010c1a9_20.png)

**2. 将公钥上传到服务器（服务器B）**
使用 `ssh-copy-id` 命令可以自动完成公钥上传。
```bash
ssh-copy-id root@192.168.145.129
```
系统会提示输入服务器B上root用户的密码。输入后，客户端的公钥就会被自动添加到服务器B的 `~/.ssh/authorized_keys` 文件中。

![](img/1bbd34f4f1f1073656bd88209010c1a9_22.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_24.png)

**3. 测试免密登录**
完成上述步骤后，再次从服务器A连接服务器B，就应该不再需要输入密码了。
```bash
ssh root@192.168.145.129
```

**重要说明**：
*   免密认证是**基于用户**的。上面的例子建立了服务器A的`root`用户到服务器B的`root`用户的信任。如果需要为其他用户建立信任，需要在该用户目录下重复此过程。
*   上述过程是**单向**的。如果要实现服务器B也能免密登录服务器A，需要在服务器B上生成密钥对并上传到服务器A。
*   这种机制是自动化运维工具（如Ansible）和跳板机（堡垒机）的基础。

![](img/1bbd34f4f1f1073656bd88209010c1a9_26.png)

## 使用SSH执行远程命令

SSH不仅可以用于登录交互式Shell，还可以直接在命令行中执行单条远程命令，这对于自动化脚本非常有用。

![](img/1bbd34f4f1f1073656bd88209010c1a9_28.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_30.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_32.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_33.png)

其基本语法是：`ssh [user@]hostname [command]`

![](img/1bbd34f4f1f1073656bd88209010c1a9_35.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_37.png)

例如，要在服务器B上执行 `hostname` 命令并立即返回结果：
```bash
ssh root@192.168.145.129 hostname
```
执行这条命令后，会要求输入密码（如果未配置免密认证），然后显示服务器B的主机名，随后连接关闭。如果不指定 `command`，则会进入服务器的交互式Shell。

![](img/1bbd34f4f1f1073656bd88209010c1a9_39.png)

## 关键SSH服务器配置 ⚙️

![](img/1bbd34f4f1f1073656bd88209010c1a9_41.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_43.png)

作为系统管理员，我们经常需要修改SSH服务器的默认配置以增强安全性。配置文件位于 `/etc/ssh/sshd_config`。

![](img/1bbd34f4f1f1073656bd88209010c1a9_45.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_47.png)

**修改配置后，必须重启SSHD服务才能生效：**
```bash
systemctl restart sshd
```

![](img/1bbd34f4f1f1073656bd88209010c1a9_48.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_49.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_51.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_52.png)

以下是两个最常用的安全配置：

![](img/1bbd34f4f1f1073656bd88209010c1a9_54.png)

**1. 禁止root用户直接登录**
直接使用root远程登录存在风险。更佳实践是使用普通用户登录，然后通过 `su` 或 `sudo` 提权。
*   编辑 `/etc/ssh/sshd_config` 文件。
*   找到 `#PermitRootLogin yes` 这一行。
*   去掉行首的 `#` 注释，并将 `yes` 改为 `no`。
    ```
    PermitRootLogin no
    ```

![](img/1bbd34f4f1f1073656bd88209010c1a9_55.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_57.png)

![](img/1bbd34f4f1f1073656bd88209010c1a9_59.png)

**2. 修改默认SSH端口（22）**
22端口是标准SSH端口，也是攻击者扫描和尝试暴力破解的首要目标。修改为非常用端口可以显著减少此类攻击。
*   编辑 `/etc/ssh/sshd_config` 文件。
*   找到 `#Port 22` 这一行。
*   去掉行首的 `#` 注释，并将 `22` 改为一个未被系统其他服务占用的端口号（如 `2222`）。
    ```
    Port 2222
    ```
*   **重要**：修改端口后，客户端连接时必须使用 `-p` 参数指定新端口：
    ```bash
    ssh -p 2222 root@192.168.145.129
    ```
*   同时，需要确保新端口在防火墙（如firewalld或iptables）中是放行的。
